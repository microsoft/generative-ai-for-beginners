# ಶುರುಮಾಡುವವರಿಗೆ ಜನರೇಟಿವ್ AI ಗಾಗಿ ಸಣ್ಣ ಭಾಷಾ ಮಾದರಿಗಳಿಗೆ ಪರಿಚಯ
ಜನರೇಟಿವ್ AI ಎಂಬುದು ಹೊಸ ವಿಷಯಗಳನ್ನು ಉತ್ಪಾದಿಸಲು ಕೌಶಲ್ಯ ಹೊಂದಿದ ವ್ಯವಸ್ಥೆಗಳ ಸೃಷ್ಟಿಯ ಮೇಲೆ ಗುರಿಯಾಗಿರುವ ಕೃತಕ ಬುದ್ಧಿಮತ್ತೆಯ ಒಂದು ಆಕರ್ಷಕ ಕ್ಷೇತ್ರವಾಗಿದೆ. ಈ ವಿಷಯವು ಪಠ್ಯ ಮತ್ತು ಚಿತ್ರಗಳಿಂದ ಸಂಗೀತ ಮತ್ತು ಸಂಪೂರ್ಣ ವರ್ಚುವಲ್ ಪರಿಸರಗಳಿಗೆ ಬದಲಾಗಬಹುದು. ಜನರೇಟಿವ್ AI ಯ ಅತ್ಯಂತ ರೋಚಕ ಅನ್ವಯಿಕತೆಗಳಲ್ಲಿ ಒಂದಾಗಿದೆ ಭಾಷಾ ಮಾದರಿಗಳ ಕ್ಷೇತ್ರ.

## ಸಣ್ಣ ಭಾಷಾ ಮಾದರಿಗಳು ಎಂದರೆ ಏನು?

ಸಣ್ಣ ಭಾಷಾ ಮಾದರಿ (SLM) ಅತಿ ದೊಡ್ಡ ಭಾಷಾ ಮಾದರಿಗಳ (LLMs) ಕೆಲವೊಂದು ಮೂಲತತ್ವಗಳು ಮತ್ತು ತಂತ್ರಗಳನ್ನು ಬಳಸಿಕೊಂಡು, ಗಣನೀಯವಾಗಿ ಕಡಿಮೆಯಾದ ಗಣನೆ ಪರಿಮಾಣವನ್ನು ಹೊಂದಿರುವ ಒಂದು ಸ್ಕೇಲ್-ಡೌನ್ ರೂಪವಾಗಿದೆ.

SLMಗಳು ಮಾನವರಂಗೆ ಪಠ್ಯವನ್ನು ಹುಟ್ಟಿಸುವಂತೆ ವಿನ್ಯಾಸಗೊಳ್ಳಲಾದ ಭಾಷಾ ಮಾದರಿಗಳ ಉಪಸಮೂಹವಾಗಿದೆ. GPT-4 ಹೀಗೆ ದೊಡ್ಡ ಮಾದರಿಗಳಿಗಿಂತ ಭಿನ್ನವಾಗಿ, SLMಗಳು ಹೆಚ್ಚಿನದಾಗಿ ಚಿಕ್ಕದಾಗಿ ಮತ್ತು ಪರಿಣಾಮಕಾರಿಯಾಗಿ, ಗಣನಾಸಂಬಂಧಿ ಸಂಪನ್ಮೂಲಗಳು ಮಿತವಾಗಿರುವ ಬಳಕೆಗಳಿಗೆ ಸೂಕ್ತವಾಗಿರುತ್ತವೆ. ಚಿಕ್ಕದಾಗಿರುವ باوجودವೂ, ಇವು ವಿವಿಧ ಕೆಲಸಗಳನ್ನು ನಿರ್ವಹಿಸಬಹುದು. ಸಾಮಾನ್ಯವಾಗಿ, SLMಗಳನ್ನು LLMಗಳನ್ನು ಸಂಕುಚಿತಗೊಳಿಸುವುದರ ಮೂಲಕ ನಿರ್ಮಿಸಲಾಗುತ್ತದೆ, ಮೂಲ ಮಾದರಿಯ ಕಾರ್ಯಕ್ಷಮತೆ ಮತ್ತು ಭಾಷಾ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಸಾಕಷ್ಟು ಮಟ್ಟದಲ್ಲಿ ಉಳಿಸಿಕೊಳ್ಳಲು ಪ್ರಯತ್ನಿಸುತ್ತವೆ. ಈ ಮಾದರಿ ಗಾತ್ರದ ಕಡಿತವು ಸಂಪೂರ್ಣ ಜೀವಂತತೆಯನ್ನು ಕಡಿಮೆ ಮಾಡುತ್ತದೆ ಮತ್ತು ಸ್ಮೃತಿ ಬಳಕೆ ಮತ್ತು ಗಣಕೀಯ ಅವಶ್ಯಕತೆಗಳಲ್ಲಿಯೂ ಪರಿಣಾಮಕಾರಿತೆಯನ್ನು ಹೆಚ್ಚಿಸುತ್ತವೆ. ಈ ಸದುಪಚಾರಿತೆಯ ನಂತರವೂ, SLMಗಳು ವಿವಿಧ ನೈಸರ್ಗಿಕ ಭಾಷಾ ಸಂಸ್ಕರಣೆ (NLP) ಕೆಲಸಗಳನ್ನು ಮಾಡಬಹುದು:

- ಪಠ್ಯ ರಚನೆ: ಅವ್ಯವಸ್ಥಿತ ಮತ್ತು ಪ್ರಾಸಂಗಿಕವಾಗಿ ಸೂಕ್ತವಾದ ವಾಕ್ಯಗಳು ಅಥವಾ ಪ್ಯಾರಾಗ್ರಾಫ್‌ಗಳನ್ನು ಸೃಷ್ಟಿಸುವುದು.
- ಪಠ್ಯದ ಪೂರ್ಣಗೊಳಿಸುವಿಕೆ: ನೀಡಲಾದ ಪ್ರಾಂಪ್ಟ್ ಆಧಾರದಲ್ಲಿ ವಾಕ್ಯಗಳನ್ನು ಊಹಿಸುವುದು ಮತ್ತು ಪೂರ್ಣಗೊಳಿಸುವುದು.
- ಭಾಷಾಂತರ: ಒಂದು ಭಾಷೆಯಿಂದ ಮತ್ತೊಂದು ಭಾಷೆಗೆ ಪಠ್ಯವನ್ನು ಪರಿವರ್ತಿಸುವುದು.
- ಸಾರಾಂಶ: ದೀರ್ಘ ಪಠ್ಯವನ್ನು ಚಿಕ್ಕದು ಮತ್ತು ಸುಲಭವಾಗಿ ಅರಿಯಬಹುದಾದ ಸಾರಾಂಶಗಳಿಗೆ ಸಂಕ್ಷೇಪಿಸುವುದು.

ತಾವು ದೊಡ್ಡ ಮಾದರಿಗಳಿಗಿಂತ ಕಾರ್ಯಕ್ಷಮತೆ ಅಥವಾ ಆಳವಾದ ತಿಳಿವಳಿಕೆಯ ಕೆಲವು ವ್ಯತ್ಯಾಸಗಳೊಂದಿಗೆ ಈ ಸಲುವಾಗಿವೆ.

## ಸಣ್ಣ ಭಾಷಾ ಮಾದರಿಗಳು ಹೇಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ?
SLMಗಳನ್ನು ಬೃಹತ್ ಪಠ್ಯ ಡೇಟಾ ಮೇಲೆ ತರಬೇತಿ ನೀಡಲಾಗುತ್ತದೆ. ತರಬೇತಿಯ ಸಮಯದಲ್ಲಿ, ಅವು ಭಾಷೆಯ ರೂಪರೇಖೆಗಳ ಮತ್ತು ರೂಪಾಂತರಣಗಳನ್ನು ಕಲಿಯುತ್ತವೆ, ಇದರಿಂದ ವ್ಯಾಕರಣಾತ್ಮಕವಾಗಿ ಸರಿಯಾಗಿರುವ ಮತ್ತು ಪ್ರಾಸಂಗಿಕವಾಗಿ ಹೊಂದಾಣಿಕೆಯ ಪಠ್ಯವನ್ನು ಸೃಷ್ಟಿಸುವ ಸಾಮರ್ಥ್ಯವನ್ನು ಹೊಂದುತ್ತವೆ. ತರಬೇತಿ ಪ್ರಕ್ರಿಯೆಯಲ್ಲಿ:

- ಡೇಟಾ ಸಂಗ್ರಹಣೆ: ವಿವಿಧ ಮೂಲಗಳಿಂದ ದೊಡ್ಡ ಪಠ್ಯ ಡೇಟಾಸೆಟ್‌ಗಳನ್ನು ಸಂಗ್ರಹಿಸುವುದು.
- ಪೂರ್ವಪ್ರಕ್ರಿಯೆ: ತರಬೇತಿಗೆ ತಕ್ಕಂತೆ ಡೇಟಾವನ್ನು ಶುದ್ಧಮಾಡಿ, ಸಂಘಟಿಸುವುದು.
- ತರಬೇತಿ: ಪಠ್ಯವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಮತ್ತು ರಚಿಸಲು ಮಾದರಿಯನ್ನು ಕಲಿಸಲು ಯಂತ್ರ ಕಲಿಕೆಯ ಆಲ್ಗಾರಿದಮ್‌ಗಳನ್ನು ಬಳಸುವುದು.
- ಸೂಕ್ಷ್ಮ-ಸೇವೆ: ನಿರ್ದಿಷ್ಟ ಕೆಲಸಗಳಲ್ಲಿ ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಸುಧಾರಿಸಲು ಮಾದರಿಯನ್ನು ಹೊಂದಿಸಿಕೊಳ್ಳುವುದು.

SLMಗಳ ಅಭಿವೃದ್ಧಿ ಸಂಪನ್ಮೂಲ-ಮಿತ ಪರಿಸರಗಳಲ್ಲಿ, ಉದಾಹರಣೆಗೆ ಮೊಬೈಲ್ ಸಾಧನಗಳು ಅಥವಾ ಎಡ್ಜ್ ಕಂಪ್ಯೂಟಿಂಗ್ ವೇದಿಕೆಗಳು, ಶಕ್ತಿಶಾಲಿ ಸಂಪನ್ಮೂಲ ಅಗತ್ಯಗಳ ಕಾರಣ ಪೂರ್ಣ ಪ್ರಮಾಣದ LLMಗಳನ್ನು ಅನ್ವಯಿಸಬೇಕಾಗದ ಅಗತ್ಯವನ್ನು ಸೊಪ್ಪುಗೊಳಿಸುತ್ತದೆ. ಪರಿಣಾಮಕಾರಿತೆಯನ್ನು ಗಮನವಿಟ್ಟು, SLMಗಳು ಕಾರ್ಯಕ್ಷಮತೆಯೊಂದಿಗೆ ಸೌಲಭ್ಯತೆಯನ್ನು ಸಮತೋಲಗೊಳಿಸುತ್ತವೆ ಮತ್ತು ವಿಭಿನ್ನ ಕ್ಷೇತ್ರಗಳಲ್ಲಿ ಹೆಚ್ಚಿನ ಅನ್ವಯಿಕತೆಯನ್ನು ಸರುವುದು.

## ಕಲಿಕೆಯ ಉದ್ದೇಶಗಳು

ಈ ಪಾಠದಲ್ಲಿ, ನಾವು SLM ಬಗ್ಗೆ ಅರಿವು ನೀಡಲು ಮತ್ತು ಅದನ್ನು ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೈ-3 ನೊಂದಿಗೆ ಸಂಯೋಜಿಸಿ ಪಠ್ಯ ವಿಷಯ, ದೃಶ್ಯ ಮತ್ತು MoE ನಲ್ಲಿ ವಿಭಿನ್ನ ದೃಶ್ಯಾವಳಿಗಳನ್ನು ಕಲಿಯಲು ಆಶಿಸುತ್ತೇವೆ.

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ, ನೀವು ಕೆಳಗಿನ ಪ್ರಶ್ನೆಗಳಿಗೆ ಉತ್ತರ ನೀಡಬಹುದು:

- SLM ಎಂದರೆ ಏನು?
- SLM ಮತ್ತು LLM ನಡುವಿನ ವ್ಯತ್ಯಾಸವೇನು?
- ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೈ-3/3.5 ಕುಟುಂಬವೇನು?
- ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೈ-3/3.5 ಕುಟುಂಬದೊಂದಿಗೆ ಊಹಿಸಲು ಹೇಗೆ ಚಲಿಸುವುದು?

ಸಜ್ಜಿದಿರಾ? ಪ್ರಾರಂಭಿಸೋಣ.

## ದೊಡ್ಡ ಭಾಷಾ ಮಾದರಿಗಳು (LLMs) ಮತ್ತು ಸಣ್ಣ ಭಾಷಾ ಮಾದರಿಗಳ (SLMs) ಹೊಸ ಹೋಲಿಕೆಗಳು

LLMಗಳು ಮತ್ತು SLMಗಳು probabilistic machine learning ಸಹಿತ ಆಧುನಿಕ ತಂತ್ರ, ತರಬೇತಿ ವಿಧಾನಗಳು, ಡೇಟಾ ಉತ್ಪಾದನಾ ಪ್ರಕ್ರಿಯೆಗಳು ಮತ್ತು ಮಾದರಿ ಮೌಲ್ಯಮಾಪನ ತಂತ್ರಗಳು ಸೂಕ್ತವಾಗಿರುತ್ತವೆ. ಆದರೆ, ಈ ಎರಡು ಮಾದರಿಗಳಲ್ಲಿ ಕೆಲವು ಮುಖ್ಯ ವ್ಯತ್ಯಾಸಗಳಿವೆ.

## ಸಣ್ಣ ಭಾಷಾ ಮಾದರಿಗಳ ಅನ್ವಯಿಕತೆಗಳು

SLMಗಳ ಅನೇಕ ಅನ್ವಯಿಕತೆಗಳಿವೆ, ಅವುಗಳಲ್ಲಿ ಕೆಲವು:

- ಚಾಟ್‌ಬಾಟ್‌ಗಳು: ಗ್ರಾಹಕ ಬೆಂಬಲ ಒದಗಿಸುವುದು ಮತ್ತು ಬಳಕೆದಾರರೊಂದಿಗೆ ಸಂವಹನ ನಡೆಸುವುದು.
- ವಿಷಯ ರಚನೆ: ಲೇಖಕರಿಗೆ ಒಂದಿಷ್ಟು ಸೃಜನಶೀಲತೆ ನೀಡುವುದು ಅಥವಾ ಸಂಪೂರ್ಣ ಲೇಖನಗಳನ್ನು ಮೇಲ್ವಿಚಾರಣೆ ಮಾಡುವುದು.
- ಶಿಕ್ಷಣ: ವಿದ್ಯಾರ್ಥಿಗಳಿಗೆ ಬರವಣಿಗೆಗೆ ಸಹಾಯಮಾಡುವುದು ಅಥವಾ ಹೊಸ ಭಾಷೆಗಳನ್ನು ಕಲಿಯುವುದು.
- ಪ್ರಾಪ್ಯಾಂಗತೆ: ಅಂಗವಿಕಲ ವ್ಯಕ್ತಿಗಳಿಗಾಗಿ ಸಾಧನಗಳನ್ನು ನಿರ್ಮಿಸುವುದು, ಉದಾಹರಣೆಗೆ ಪಠ್ಯ-ದಿಂದ-ಮಾತಿಗೆ ವ್ಯವಸ್ಥೆಗಳು.

**ಗಾತ್ರ**
  
LLMಗಳ ಮತ್ತು SLMಗಳ ಮಧ್ಯೆ ಮೂಲಭೂತ ವ್ಯತ್ಯಾಸ ಮಾದರಿಯ ಗಾತ್ರದಲ್ಲಿದೆ. ChatGPT (GPT-4) ಹೀಗೆ LLMಗಳು ಶೇಕಡಾ 1.76 ಟ್ರಿಲಿಯನ್ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳನ್ನು ಹೊಂದಿರಬಹುದು, ಆದರೆ ಮುಕ್ತ ಮೂಲ SLMಗಳಾದ Mistral 7B ಅಂದಾಜಿಗೆ ಸುಮಾರು 7 ಬಿಲಿಯನ್ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳನ್ನೇ ಹೊಂದಿವೆ. ಈ ಭೇದವು ಮುಖ್ಯವಾಗಿ ಮಾದರಿ ಆರ್ಕಿಟೆಕ್ಟಚರ್ ಮತ್ತು ತರಬೇತಿ ಪ್ರಕ್ರಿಯೆಗಳ ವ್ಯತ್ಯಾಸಗಳಿಗೆ ಕಾರಣ. ಉದಾಹರಣೆಗೆ, ChatGPT ಸ್ವಯಂ-ಧ್ಯಾನ ಯಂತ್ರವನ್ನು ಬಳಸಿ ಎನ್ಕೋಡರ್-ಡೀಕೊಡರ್ ರಚನೆಯಲ್ಲಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ, ಆದರೆ Mistral 7B ಸ್ಲೈಡಿಂಗ್ ವಿಂಡೋ ಅಟೆನ್ಷನ್ ನ್ನು ಬಳಸಿ ಡೀಕೊಡರ್ ಮಾತ್ರ ಮಾದರಿಯಲ್ಲಿ ಹೆಚ್ಚು ಪರಿಣಾಮಕಾರಿ ತರಬೇತಿಗೆ ಸಹಾಯಕವಾಗಿರುತ್ತದೆ. ಈ ವಿನ್ಯಾಸ ವ್ಯತ್ಯಾಸವು ಮಾದರಿಗಳ ಸಂಕೀರ್ಣತೆ ಮತ್ತು ಕಾರ್ಯಕ್ಷಮತೆಯಲ್ಲಿ ಮಹತ್ವಪೂರ್ಣ ಪರಿಣಾಮಗಳನ್ನು ಉಂಟುಮಾಡುತ್ತದೆ.

**ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವಿಕೆ**

SLMಗಳು ವಿಶೇಷ ಕ್ಷೇತ್ರಗಳಲ್ಲಿ ಕಾರ್ಯಕ್ಷಮತೆಗೆ ಸೂಕ್ತವಾಗಿರಲು ಅತ್ಯುತ್ತಮಗೊಳಿಸಲ್ಪಟ್ಟಿರುವುದರಿಂದ, ಅವು ಹೆಚ್ಚು ವಿಶಿಷ್ಟವಾಗಿರುತ್ತವೆ ಆದರೆ ಬೃಹತ್ ಕ್ಷೇತ್ರಗಳಲ್ಲಿ ವ್ಯಾಪಕ ಸಾಂದರ್ಭಿಕ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವಿಕೆಗೆ ಸಮರ್ಥವಲ್ಲದಿರಬಹುದು. ಇದರ ವಿರುದ್ಧವಾಗಿ, LLMಗಳು ಮಾನವರಂಗೆ ಬುದ್ಧಿಮತ್ತೆಯನ್ನು ಸಮಗ್ರ ಮಟ್ಟದಲ್ಲಿ ಅನುಕರಿಸಲು ಉದ್ದೇಶಿಸಲಾಗಿದೆ. ಬೃಹತ್, ವೈವಿಧ್ಯಮಯ ಡೇಟಾಸೆಟ್‌ಗಳಲ್ಲಿ ತರಬೇತಿ ಪಡೆದ LLMಗಳು ವಿವಿಧ ಕ್ಷೇತ್ರಗಳಲ್ಲಿ ಉತ್ತಮ ಸಾಧನೆ ಮಾಡುವಂತೆ ರೂಪಿಸಲ್ಪಟ್ಟಿದ್ದು, ಹೆಚ್ಚುವರಿ ಅನುವಾದಕೀಯ ಮತ್ತು ಹೊಂದಾಣಿಕೆಯ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ನೀಡುತ್ತವೆ. ಆದ್ದರಿಂದ, LLMಗಳು ನೈಸರ್ಗಿಕ ಭಾಷಾ ಸಂಸ್ಕರಣೆ ಮತ್ತು ಕಾರ್ಯನಿರ್ವಹಣೆ ಸೇರಿದಂತೆ ಹೆಚ್ಚಿನ ಕಾರ್ಯಗಳಿಗೆ ಸೂಕ್ತವಾಗಿರುತ್ತವೆ.

**ಗಣನೆಯ ಸಾಮರ್ಥ್ಯ**

LLMಗಳ ತರಬೇತಿ ಮತ್ತು ನಿಯೋಜನೆ ಸಂಪನ್ಮೂಲ-ತೀವ್ರ ಪ್ರಕ್ರಿಯೆಗಳು ಆಗಿದ್ದು, ಹೆಚ್ಚಿನಪಡಿನ GPU ಕ್ಲಸ್ಟರ್‌ಗಳನ್ನು ಒಳಗೊಂಡಿರಬಹುದು. ಉದಾಹರಣೆಗೆ, ChatGPT ಮಾದರಿಯನ್ನು ಆರಂಭದಿಂದ ತೆಗೆಯಲು ಸಾವಿರಾರು GPUಗಳ ಅವಶ್ಯಕತೆ ಪ್ರಬಲವಾಗಿದೆ. ಇದರ ವಿರುದ್ಧವಾಗಿ, SLMಗಳು ತನ್ನ ಕಡಿಮೆ ಪ್ಯಾರಾಮೀಟರ್ ಸಂಖ್ಯೆಗಳಿಂದ ಗಣಕೀಯ ಸಂಪನ್ಮೂಲಗಳಿಗೆ ಸುಲಭವಾಗಿ ಲಭ್ಯವಿರುತ್ತವೆ. Mistral 7B ಮಾದರಿಗಳು ಮಧ್ಯಮ GPU ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಹೊಂದಿರುವ ಸ್ಥಳೀಯ ಯಂತ್ರಗಳಲ್ಲಿ ತರಬೇತಿ ಮತ್ತು ಕಾರ್ಯನಿರ್ವಹಣೆಗೆ ತಕ್ಕವೆಯಾದರೂ, ತರಬೇತಿಗೆ ಹಲವಾರು ಗಂಟೆಗಳ ಅವಧಿ ಅಗತ್ಯವಿದೆ.

**ಪಕ್ಷಪಾತ**

LLMಗಳಲ್ಲಿ ಪಕ್ಷಪಾತವು ಟ್ರೇನಿಂಗ್ ಡೇಟಾದ ಸ್ವರೂಪದ ಕಾರಣಾಗಿ ಪ್ರಸಿದ್ಧ ಸಮಸ್ಯೆಯಾಗಿದೆ. ಈ ಮಾದರಿಗಳು ಅಂತರ್ಜಾಲದಿಂದ ಲಭ್ಯವಾಗುವ الخام ಡೇಟಾಧಾರಿತವಾಗಿ ಕೆಲಸಮಾಡುತ್ತವೆ, ಇದು ಕೆಲ ಸಮುದಾಯಗಳನ್ನು ಅಲ್ಪಪ್ರತಿನಿಧಿಸುವುದಾಗಿ ತೋರುತ್ತದೆ ಅಥವಾ ತಪ್ಪಾಗಿದೆ ಎಂದು ಲೇಬಲ್ ಮಾಡುವ, ಭಾಷಾ ಪರಿಕಲ್ಪನೆಗಳನ್ನು ಪ್ರಭಾವಿಸುವ ಸಾಕಲ್ಯ, ಭೌಗೋಳಿಕ ಬದಲಾಗುವುದರ ಮೂಲಕ ಅಥವಾ ವ್ಯಾಕರಣ ನಿಯಮಗಳಿಂದಾಗಿ ಉಂಟಾಗುವ ಸಾಧ್ಯತೆ ಇದೆ. ಜೊತೆಗೇ, LLMಗಳ ಸಂಕೀರ್ಣ ರಚನೆ ಪಕ್ಷಪಾತವನ್ನು ಹೆಚ್ಚಿಸುವ ಸಾಧ್ಯತೆಯಿದೆ, ಫೈನ್ಟ್ಯೂನಿಂಗ್ ಇಲ್ಲದಿದ್ದರೆ ಈ ಅವಗಾಹನೆಯು ಕಾಣೆಯಾಗಬಹುದು. ಈಗಿನಂತೆ, SLMಗಳು ನಿರ್ದಿಷ್ಟ ಕ್ಷೇತ್ರಗಳ ಡೇಟಾ ಮೇಲೆ ತರಬೇತಿಗೊಳ್ಳುವುದರಿಂದ, ಇವು ಪಕ್ಷಪಾತಕ್ಕೆ ಕಡಿಮೆ ಒಳಗಾಗುತ್ತವೆ, ಆದರೆ ಸಂಪೂರ್ಣವಾಗಿ ಮುಕ್ತಳಾಗಿಲ್ಲ.

**ಊಹಿಸುವಿಕೆ**

SLMಗಳ ಕಡಿಮೆ ಗಾತ್ರವು ಅವುಗಳಿಗೆ ಸ್ಥಳೀಯ ಸಾಧನಗಳಲ್ಲಿ ಬಹುಸಂಖ್ಯಾಕ ಬಹುಪ್ರಕ್ರಿಯೆ ಅಗತ್ಯವಿಲ್ಲದೆ ತ್ವರಿತವಾಗಿ ಔಟ್ಪುಟ್ ಉತ್ಪಾದಿಸುವುದರಲ್ಲಿ ಮಹತ್ವದ ಪ್ರಯೋಜನ ನೀಡುತ್ತದೆ. ಇದಕ್ಕೆ ಎದುರಾಗಿ, LLMಗಳು ಅವುಗಳ ಗಾತ್ರ ಮತ್ತು ಸಂಕೀರ್ಣತೆಗೆ ಕಾರಣವಾಗಿ ಅನುಕೂಲಕರ ಛಂದಸ್ಸು ಸಮಯಗಳನ್ನು ಪಡೆಯಲು ದೊಡ್ಡ ಪರಮಾಣು ಗಣನೀಯ ಸಂಪನ್ಮೂಲಗಳನ್ನು ಬೇಕಾಗಿಸುತ್ತವೆ. ಬಹುತೇಕ ಬಳಕೆದಾರರಿದ್ದಾಗ LLMಗಳ ಪ್ರತಿಕ್ರಿಯೆ ಸಮಯ ಇನ್ನಷ್ಟು ನಿಧಾನವಾಗುತ್ತದೆ, ವಿಶೇಷವಾಗಿ ಹೆಚ್ಚಿನ ಪ್ರಮಾಣದಲ್ಲಿ ನಿಯೋಜಿತಾಗಿದ್ದಾಗ.

ಸಾರಾಂಶವಾಗಿ, LLMಗಳ ಮತ್ತು SLMಗಳಿಬ್ಬರಲ್ಲಿಯೂ ಯಂತ್ರ ಕಲಿಕೆಯ ಮೂಲಭೂತ ತತ್ವಗಳು ಇದ್ದರೂ, ಅವುಗಳ ಗಾತ್ರ, ಸಂಪನ್ಮೂಲ ಅವಶ್ಯಕತೆ, ಸಾಂದರ್ಭಿಕ ತಿಳಿವಳಿಕೆ, ಪಕ್ಷಪಾತ ಹೊಂದಿರುವ ಸಾಧ್ಯತೆ ಮತ್ತು ಊಹಿಸುವಿಕೆಯ ವೇಗದಲ್ಲಿ ಪ್ರಮುಖ ವ್ಯತ್ಯಾಸಗಳಿವೆ. ಈ ವ್ಯತ್ಯಾಸಗಳು ಸಾದ್ಯವಿರುವ ಬಳಕೆಗಾಗಿ ಅನುಕೂಲತೆಯನ್ನು ಸೂಚಿಸುತ್ತವೆ, LLMಗಳು ಹೆಚ್ಚು ಸೌಕರ್ಯದಲ್ಲಿದ್ದರೂ ಸಂಪನ್ಮೂಲ-ಗಾಳಾ, SLMಗಳು ಕಡಿಮೆ ಗಣಕೀಯ ಅಗತ್ಯಗಳೊಂದಿಗೆ ಕ್ಷೇತ್ರ ವಿಶಿಷ್ಟತೆ ನೀಡುತ್ತವೆ.

***ಗಮನಿಸಿ: ಈ ಪಾಠದಲ್ಲಿ, ನಾವು Microsoft Phi-3 / 3.5 ಉಪಯೋಗಿಸಿ SLM ಅನ್ನು ಪರಿಚಯಿಸುವೆವು.***

## ಫೈ-3 / ಫೈ-3.5 ಕುಟುಂಬವನ್ನು ಪರಿಚಯಿಸಿ

ಫೈ-3 / 3.5 ಕುಟುಂಬವು ಮುಖ್ಯವಾಗಿ ಪಠ್ಯ, ದೃಶ್ಯ, ಮತ್ತು ಏಜೆಂಟ್ (MoE) ಅನ್ವಯಿಕ ದೃಶ್ಯಗಳಿಗೆ ಗುರಿಯಾಗಿರುತ್ತದೆ:

### ಫೈ-3 / 3.5 ಸೂಚನೆ

ಮುಖ್ಯವಾಗಿ ಪಠ್ಯ ರಚನೆ, ಚಾಟ್ ಪೂರ್ಣಗೊಳಿಸುವಿಕೆ, ವಿಷಯ ಮಾಹಿತಿ ತೆಗೆಯುವಿಕೆ ಇತ್ಯಾದಿಗಳಿಗೆ.

**ಫೈ-3-ಮಿನಿ**

3.8B ಭಾಷಾ ಮಾದರಿ Microsoft Foundry, Hugging Face, ಮತ್ತು Ollama ನಲ್ಲಿ ಲಭ್ಯವಿದೆ. ಫೈ-3 ಮಾದರಿಗಳು ಸಮಾನ ಗಾತ್ರ ಅಥವಾ ದೊಡ್ಡ ಗಾತ್ರದ ಭಾಷಾ ಮಾದರಿಗಳ ಮೇಲ್ನೋಟದಲ್ಲಿ ಪ್ರಮುಖ ನಿರೀಕ್ಷಿತ ಕಾರ್ಯಗಳನ್ನು ಬಹುತೇಕ ಉತ್ತಮ ಸಾಧನೆ ಮಾಡುತ್ತವೆ (ಕೆಳಗಿನ ಬೆಂಚ್ಮಾರ್ಕ್ ಸಂಖ್ಯೆಯನ್ನು ನೋಡಿ, ಹೆಚ್ಚು ಸಂಖ್ಯೆ ಉತ್ತಮ). ಫೈ-3-ಮಿನಿ ತನ್ನ ಗಾತ್ರದ ಎರಡುಮಟ್ಟಿಗೆಳ್ಳಿರುವ ಮಾದರಿಗಳನ್ನು ಮೀರಿಸುತ್ತದೆ, ಮತ್ತು ಫೈ-3-ಸಣ್ಣ ಮತ್ತು ಫೈ-3-ಮಧ್ಯಮವು ದೊಡ್ಡ ಮಾದರಿಗಳನ್ನು, જેમાં GPT-3.5 ಕೂಡ ಇದ್ದಾನೆ, ಹೆಚ್ಚು ಸಾಧನೆ ಮಾಡುತ್ತವೆ.

**ಫೈ-3-ಸಣ್ಣ ಮತ್ತು ಮಧ್ಯಮ**

ಮಾತ್ರ 7B ಪ್ಯಾರಾಮೀಟರ್‌ಗಳೊಂದಿಗೆ, ಫೈ-3-ಸಣ್ಣವು GPT-3.5T ಅನ್ನುಭಾಷೆ, ತರ್ಕ, ಕೋಡಿಂಗ್ ಮತ್ತು ಗಣಿತೀಯ ಬೆಂಚ್ಮಾರ್ಕ್‌ಗಳಲ್ಲಿ ಮೆಚ್ಚುಗೆಯವರೆಗೆ ಮೀರುತ್ತದೆ.

14B ಪ್ಯಾರಾಮೀಟರ್‌ಗಳೊಂದಿಗೆ ಫೈ-3-ಮಧ್ಯಮ ಈ ಪರಂಪರೆಯನ್ನು ಮುಂದುವರೆಸಿದ್ದು, ಜೆಮಿನಿ 1.0 ಪ್ರೊವನ್ನು ಮೀರಿಸುತ್ತದೆ.

**ಫೈ-3.5-ಮಿನಿ**

ಇದನ್ನು ಫೈ-3-ಮಿನಿಯ ನವೀಕರಣವೆಂದು ಪರಿಗಣಿಸಬಹುದು. ಪ್ಯಾರಾಮೀಟರ್‌ಗಳು ಬದಲಾಗದಿದ್ದರೂ, ಇದು ವಿವಿಧ ಭಾಷೆಗಳನ್ನು ಬೆಂಬಲಿಸುವ ಸಾಮರ್ಥ್ಯವನ್ನು ಸುಧಾರಿಸುತ್ತದೆ (20+ ಭಾಷೆಗಳ ಬೆಂಬಲ: ಅರೇಬಿಕ್, ಚೈನೀಸ್, ಚೆಕ್, ಡ್ಯಾನಿಷ್, ಡಚ್, ಇಂಗ್ಲೀಷ್, ಫಿನ್ನಿಶ್, ಫ್ರೆಂಚ್, ಜರ್ಮನ್, ಹೀಬ್ರೂ, ಹಂಗೇರಿಯನ್, ಇಟಾಲಿಯನ್, ಜಪಾನೀಸ್, ಕೊರಿಯನ್, ನಾರ್ವೆಜಿಯನ್, ಪೋಲಿಶ್, ಪೋರ್ಟುಗೀಸ್, ರಶಿಯನ್, ಸ್ಪ್ಯಾನಿಷ್, ಸ್ವೀಡಿಶ್, ಥಾಯ್, ಟರ್ಕಿಶ್, ಉಕ್ರೇನಿಯನ್) ಮತ್ತು ದೀರ್ಘ ಸಾಂದರ್ಭಿಕತೆಗೆಷ್ಟು ಹೆಚ್ಚಿನ ಬೆಂಬಲವನ್ನು ವಿವರಿಸುತ್ತದೆ.

3.8B ಪ್ಯಾರಾಮೀಟರ್‌ಗಳೊಂದಿಗೆ ಫೈ-3.5-ಮಿನಿ ತನ್ನ ಗಾತ್ರದ ಭಾಷಾ ಮಾದರಿಗಳನ್ನು ಮೀರಿಸುತ್ತಿದ್ದು, ಎರಡುಮಟ್ಟಿಗೆಳ್ಳಿರುವ ಮಾದರಿಗಳ ಸಮತೋಲನವಾಗಿದೆ.

### ಫೈ-3 / 3.5 ದೃಶ್ಯ

ನಾವು ಫೈ-3/3.5 ಸೂಚನಾ ಮಾದರಿಯನ್ನು ಫೈನ ಸಾಮರ್ಥ್ಯವಾಗಿ ಪರಿಗಣಿಸಬಹುದು, ಮತ್ತು ದೃಶ್ಯವು ಫೈಗೆ ಲೋಕವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಕಣ್ಣುಗಳನ್ನು ನೀಡುತ್ತದೆ.


**ಫೈ-3-ದೃಶ್ಯ**

ಕೇವಲ 4.2B ಪ್ಯಾರಾಮೀಟರ್‌ಗಳೊಂದಿಗೆ ಫೈ-3-ದೃಶ್ಯ ಈ ಪ್ರವೃತ್ತಿಯನ್ನು ಮುಂದುವರೆಸುತ್ತಿದ್ದು, Claude-3 Haiku ಮತ್ತು Gemini 1.0 Pro V ಗಿಂತ ದೊಡ್ಡ ಮಾದರಿಗಳ ಮೇಲೆ ಸಾಮಾನ್ಯ ದೃಶ್ಯ ತರ್ಕ ಕಾರ್ಯಗಳು, OCR ಮತ್ತು ಟೇಬಲ್ ಹಾಗೂ ಚಾರ್ಟ್ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವ ನಿಯೋಜನೆಗಳಲ್ಲಿ ಮೇಲುಕಾಣಿ ಸಾಧಿಸುತ್ತದೆ.


**ಫೈ-3.5-ದೃಶ್ಯ**

ಫೈ-3.5-ದೃಶ್ಯವು ಫೈ-3-ದೃಶ್ಯದ ನವೀಕರಣವಾಗಿದ್ದು, ಹಲವಾರು ಚಿತ್ರಗಳ ಬೆಂಬಲವನ್ನು ನೂತನಗೊಳಿಸುತ್ತದೆ. ನೀವು ಇದನ್ನು ದೃಶ್ಯದಲ್ಲಿ ಸುಧಾರಣೆ ಎಂದು ಭಾವಿಸಬಹುದು, ಬೇಸರ ಇಲ್ಲ, ನೀವು ಚಿತ್ರಗಳನ್ನು ಮಾತ್ರವಲ್ಲದೆ ವೀಡಿಯೋಗಳನ್ನು ಸಹ ನೋಡಬಹುದು.

ಫೈ-3.5-ದೃಶ್ಯ Claude-3.5 Sonnet ಮತ್ತು Gemini 1.5 Flash ಮುಂತಾದ ದೊಡ್ಡ ಮಾದರಿಗಳಲ್ಲಿ OCR, ಟೇಬಲ್ ಮತ್ತು ಚಾರ್ಟ್ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವ ಕಾರ್ಯಗಳಲ್ಲಿ ಮೇಲುಕಾಣಿ ಸಾಧಿಸುವುದಲ್ಲದೆ, ಸಾಮಾನ್ಯ ದೃಶ್ಯ ಜ್ಞಾನ ತರ್ಕ ಕಾರ್ಯಗಳಲ್ಲಿ ಸಮತೋಲನವಾಗಿದೆ. ಬಹು-ಫ್ರೇಮ್ ಇನ್ಪುಟ್ ಬೆಂಬಲ, ಅಂದರೆ ಬಹುಚಿತ್ರ ಇನ್ಪುಟ್ಗಳ ಮೇಲೆ ತರ್ಕ ನಿರ್ವಹಣೆ.


### ಫೈ-3.5-MoE

*** especialistaಗಳ ಮಿಶ್ರಣ (Mixture of Experts, MoE)*** ಮಾದರಿಗಳನ್ನು ಕಡಿಮೆ ಗಣಕೀಯದಿಂದ ಪ್ರೀಟ್ರೇನಿಂಗ್ ಮಾಡಲು ನೆರವಾಗುತ್ತದೆ, ಅರ್ಥಾತ್ ನೀವು ಸDense ಮಾದರಿಯ ಸಮಾನ ಕಂಗಳಬಜೆಟ್ ಹೊಂದಿದ್ದರೂ ಮಾದರಿ ಅಥವಾ ಡೇಟಾಸೆಟ್ ಗಾತ್ರವನ್ನು ಬಹಳ ಮಟ್ಟಿಗೆ ವಿಸ್ತರಿಸಬಹುದು. ವಿಶೇಷವಾಗಿ, MoE ಮಾದರಿ ತನ್ನ ದಪ್ಪ ಸಮಾನ ಪ್ರಕಾರದಂತೆ ತ್ವರಿತವಾಗಿ ಪ್ರೀಟ್ರೇನಿಂಗ್ ಸಮಯದಲ್ಲಿ ಸಮಾನ ಗುಣಮಟ್ಟ ಸಾಧಿಸಬೇಕು.

ಫೈ-3.5-MoE 16x3.8B ತಜ್ಞ ಘಟಕಗಳನ್ನು ಹೊಂದಿದೆ. ಕೇವಲ 6.6B ಸಕ್ರಿಯ ಪ್ಯಾರಾಮೀಟರ್‌ಗಳೊಂದಿಗೆ ಫೈ-3.5-MoE ಹೆಚ್ಚಿದ ಮಾದರಿಗಳಂತೆ ತರ್ಕ, ಭಾಷಾ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವಿಕೆ ಮತ್ತು ಗಣಿತ ಸಾಧಿಸಲು ಸಾಧ್ಯವಿದೆ.

ನಾವು ವಿಭಿನ್ನ ದೃಶ್ಯಗಳ ಆಧಾರದಲ್ಲಿ ಫೈ-3/3.5 ಕುಟುಂಬದ ಮಾದರಿಗಳನ್ನು ಬಳಸಬಹುದು. LLMಗಿಂತ ಭಿನ್ನವಾಗಿ, ನೀವು Phi-3/3.5-ಮಿನಿ ಅಥವಾ Phi-3/3.5-ದೃಶ್ಯವನ್ನು ಎಡ್ಜ್ ಸಾಧನಗಳಲ್ಲಿ ನಿಯೋಜಿಸಬಹುದು.

ಫೈ-3/3.5 ಅನ್ನು ವಿಭಿನ್ನ ದೃಶ್ಯಗಳಲ್ಲಿ ಬಳಸಲು ನಾವು ಆಶಿಸುತ್ತೇವೆ. ಮುಂದಿನದಲ್ಲಿಯೇ ನಾವು ವಿವಿಧ ದೃಶ್ಯಗಳ ಆಧಾರದ ಮೇಲೆ Phi-3/3.5 ಉಪಯೋಗಿಸುವೆವು.

**NVIDIA NIM**

ಕ್ಲೌಡ್ ಆಧಾರಿತ Microsoft Foundry Models ಕ್ಯಾಟಾಲಾಗ್ ಜೊತೆಗೆ, ನೀವು [NVIDIA NIM](https://developer.nvidia.com/nim?WT.mc_id=academic-105485-koreyst) ಬಳಸಿ ಸಂಬಂಧಿತ ಕರೆಯಗಳನ್ನು ಪೂರ್ಣಗೊಳಿಸಬಹುದು. ನೀವು NVIDIA NIM ಗೆ ಭೇಟಿ ನೀಡಿ Phi-3/3.5 ಕುಟುಂಬದ API ಕರೆಯಗಳನ್ನು ಪೂರ್ಣಗೊಳಿಸಬಹುದು. NVIDIA NIM (NVIDIA Inference Microservices) ವಿವಿಧ ಪರಿಸರಗಳಲ್ಲಿ, ಕ್ಲೌಡ್, ಡೇಟಾ ಸೆಂಟರ್, ಮತ್ತು ವರ್ಕ್‌ಸ್ಟೇಷನ್‌ಗಳಿಗೆ AI ಮಾದರಿಗಳನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ನಿಯೋಜಿಸಲು ವಿನ್ಯಾಸಗೊಳಿಸಿದ ವೇಗಗತಿಯಾದ ಊಹಿಸುವಿಕೆ ಮೈಕ್ರೋಸರ್ವೀಸಿಗಳ ಸಂಗ್ರಹವಾಗಿದೆ.

ಇದಲ್ಲದೆ NVIDIA NIMನ ಕೆಲ ಪ್ರಮುಖ বৈಶಿಷ್ಟ್ಯಗಳಿವೆ:

- **ನಿಯೋಜನೆ ಸರಳತೆ:** NIM ಒಬ್ಬ ಆಜ್ಞೆಯೊಂದಿಗೆ AI ಮಾದರಿಗಳನ್ನು ನಿಯೋಜಿಸಲು ಸಾಧ್ಯವಾಗಿಸುವುದರಿಂದ, ಇದನ್ನು ಈಗಾಗಲೇ ಇದ್ದ ಕಾರ್ಯಪ್ರವಾಹಗಳಲ್ಲಿ ಸುಲಭವಾಗಿ ಅಳವಡಿಸಬಹುದು.

- **ಅತ್ಯುತ್ತಮ ಕಾರ್ಯಕ್ಷಮತೆ:** ಇದು ಕಡಿಮೆ ವಿಳಂಬ ಮತ್ತು ಹೆಚ್ಚಿದ ಸಾಧನಶೀಲತೆಗಾಗಿ, TensorRT ಮತ್ತು TensorRT-LLM ಎಂಬ NVIDIA ನ ಪೂರ್ವ-ಆಪ್ಟಿಮೈಜ್ಡ್ ಇನ್ಫರೆನ್ಸ್ ಎಂಜಿನ್‌ಗಳನ್ನು ಬಳಸುತ್ತದೆ.
- **ಸ್ಕೇಲಬಿಲಿಟಿ:** NIM ಕುಬರ್ನೇಟಿಸ್‌ನಲ್ಲಿ ಸ್ವಯಂಚಾಲಿತ ವಿಸ್ತರಣೆಯನ್ನು (autoscaling) ಬೆಂಬಲಿಸುತ್ತದೆ, ಇದರಿಂದ ಇದು ವಿಭಿನ್ನ ಕೆಲಸ ಭಾರಗಳನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ನಿಭಾಯಿಸಬಹುದು.
- **ಭದ್ರತೆ ಮತ್ತು ನಿಯಂತ್ರಣ:** ಸಂಸ್ಥೆಗಳು ತಮ್ಮ ದತ್ತಾಂಶ ಮತ್ತು ಅನ್ವಯಿಕೆಗಳ ಮೇಲಿನ ನಿಯಂತ್ರಣವನ್ನು ತಮ್ಮದೇ ನಿರ್ವಹಿಸುವ ಮೂಲಸೌಕರ್ಯದಲ್ಲಿ NIM ಮೈಕ್ರೋಸರ್ವಿಸ್‌ಗಳನ್ನು ಸ್ವಾಯತ್ತವಾಗಿ ಹೋಸ್ಟ್ ಮೂಲಕ ಸಾಕಷ್ಟು ಇಡಬಹುದು.
- **ಪ್ರಮಾಣಿತ APIs:** NIM ದರ್ಶನ ಚಾಟ್‌ಬಾಟ್‌ಗಳು, AI ಸಹಾಯಕರು, ಮತ್ತು ಇನ್ನಿತರ AI ಅನ್ವಯಿಕೆಗಳನ್ನು ಸುಲಭವಾಗಿ ನಿರ್ಮಿಸಲು ಮತ್ತು ಇಂಟಿಗ್ರೇಟ್ ಮಾಡಲು ಕೈಗಾರಿಕಾ ಪ್ರಮಾಣಿತ API ಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ.

NIM NVIDIA AI ಎಂಟರ್‌ಪ್ರೈಸ್ ಭಾಗವಾಗಿದ್ದು, AI ಮಾದರಿಗಳ ನಿಯೋಜನೆ ಮತ್ತು ಕಾರ್ಯಾಚರಣೆಯನ್ನು ಸರಳಗೊಳಿಸುವ ಉದ್ದೇಶ ಹೊಂದಿದೆ, ಮತ್ತು ಅವುಗಳನ್ನು NVIDIA GPU ಗಳಲ್ಲಿ ಪರಿಣಾಮಕಾರಿಯಾಗಿ ಚಲಿಸುವಂತೆ ಮಾಡುತ್ತದೆ.

### Phi-3/3.5 ಅನ್ನು ಸ್ಥಳೀಯವಾಗಿ ನಡೆಸುವುದು
Phi-3 ಅಥವಾ ಯಾವುದೇ ಭಾಷಾ ಮಾದರಿ, ಉದಾ. GPT-3, ನ ಇನ್ಫರೆನ್ಸ್ ಎಂಬುದು ಪೂರೈಕೆ ಮಾಡಿದ ಇನ್ಪುಟ್ ಆಧಾರದಲ್ಲಿ ಪ್ರತಿಕ್ರಿಯೆಗಳು ಅಥವಾ ಭವಿಷ್ಯವಾಣಿಗಳನ್ನು ನಿರ್ಮಿಸುವ ಪ್ರಕ್ರಿಯೆಯಾಗಿದ್ದು, Phi-3 ಇದಕ್ಕೆ ಹೊಂದಿಕೊಂಡ ನಿಯಂತ್ರಿತ ನ್ಯೂರಲ್ ನೆಟ್‌ور್ಕ್ ಬಳಸಿ ತರಬೇತಿಗೆ ಒಳಗಾದ ಡೇಟಾದ ಮಾದರಿಗಳನ್ನು ವಿಶ್ಲೇಷಿಸಿ ಅತ್ಯಂತ ಸಂಬಂಧಿತ ಉತ್ತರವನ್ನು ಉತ್ಪಾದಿಸುತ್ತದೆ.

**ಹಗ್ಗಿಂಗ್ ಫೇಸ್ ಟ್ರಾನ್ಸ್‌ಫಾರ್ಮರ್**
ಹಗ್ಗಿಂಗ್ ಫೇಸ್ ಟ್ರಾನ್ಸ್‌ಫಾರ್ಮರ್‌ಗಳು ನೈಸರ್ಗಿಕ ಭಾಷಾ ಪ್ರಕ್ರಿಯೆ (NLP) ಮತ್ತು ಇತರ ಯಂತ್ರಕಲಿಕಾ ಕಾರ್ಯಗಳಿಗೆ ವಿನ್ಯಾಸಗೊಳಿಸಿರುವ ಪ್ರಬಲ ಗ್ರಂಥಾಲಯವಾಗಿದೆ. ಇದರ ಕೆಲವು ಪ್ರಮುಖ ಅಂಶಗಳು:

1. **ಪೂರ್ವ ತರಬೇತಿಗೊಳಿಸಿದ ಮಾದರಿಗಳು**: ಪಠ್ಯ ವರ್ಗೀಕರಣ, ಹೆಸರು ಗುರುತಿಸುವಿಕೆ, ಪ್ರಶ್ನೋತ್ತರ, ಸಾರಾಂಶ, ಭಾಷಾಂತರ ಮತ್ತು ಪಠ್ಯ ಉತ್ಪಾದನೆ ಸೇರಿದಂತೆ ವಿವಿಧ ಕಾರ್ಯಗಳಿಗೆ ಸಾವಿರಾರು ಪೂರ್ವ ತರಬೇತಿಗೊಳಿಸಿದ ಮಾದರಿಗಳನ್ನು ನೀಡುತ್ತದೆ.

2. **ಫ್ರೆ임್ವರ್ಕ್ ಅನುಗುಣತೆ**: ಪೈಟಾರ್ಚ್, ಟೆನ್ಸರ್‌ಫ್ಲೋ, ಜ್ಯಾಕ್ಸ್ ಮುಂತಾದ ಹಲವು ಗುಣೆಮಟ್ಟದ ಯಂತ್ರಕಲಿಕಾ ಫ್ರೆ임್ವರ್ಕ್‌ಗಳನ್ನೂ ಬೆಂಬಲಿಸಿ, ಒಂದು ಫ್ರೆ임್ವರ್ಕ್‌ನಲ್ಲಿ ತರಬೇತಿಯನ್ನು ಮಾಡಿಸಿ ಮತ್ತೊಂದರಲ್ಲಿ ಬಳಸಲು ಅವಕಾಶ ನೀಡುತ್ತದೆ.

3. **ಬಹುಮಾಧ್ಯಮ ಸಾಮರ್ಥ್ಯಗಳು**: NLP ಯಲ್ಲದೆ ಚಿತ್ರ ವರ್ಗೀಕರಣ, ವಸ್ತು ಪತ್ತೆ (ಕಂಪ್ಯೂಟರ್ ವೀಕ್ಷಣೆ) ಮತ್ತು ಧ್ವನಿ ಮಾನ್ಯತೆ, ಧ್ವನಿ ವರ್ಗೀಕರಣ (ಆಡಿಯೋ ಪ್ರಕ್ರಿಯೆ) ಕಾರ್ಯಗಳಿಗೂ ಬೆಂಬಲ ನೀಡುತ್ತದೆ.

4. **ಬಳಕೆ ಸರಳತೆ**: ಸುಲಭವಾಗಿ ಡೌನ್‌ಲೋಡ್ ಮಾಡಿಕೊಳ್ಳಲು ಮತ್ತು ಫೈನ್-ಟ್ಯೂನ್ ಮಾಡಲು API ಗಳು ಮತ್ತು ಉಪಕರಣಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ, ಇದರಿಂದ ಪ್ರಾರಂಭಿಕರಿಂದ ಪರಿಣತರಿಗೂ ಅದು ಸುಲಭವಾಗುತ್ತದೆ.

5. **ಸಮುದಾಯ ಮತ್ತು ಸಂಪನ್ಮೂಲಗಳು**: ಹಗ್ಗಿಂಗ್ ಫೇಸ್ ಸಮುದಾಯದಲ್ಲಿ ನಿಖರ ಡಾಕ್ಯುಮೆಂಟೇಶನ್, ಪಾಠಗಳು ಮತ್ತು ಮಾರ್ಗದರ್ಶಿ ಲಭ್ಯವಿದ್ದು ಬಳಕೆದಾರರಿಗೆ ಸಹಾಯ ಮಾಡುತ್ತದೆ.

**GenAI ಗೆ ONNX ರನ್‌ಟೈಮ್**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) ಬಹುವಿಧ ವೇದಿಕೆಗಳಲ್ಲಿ ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಓಪನ್-ಸೋರ್ಸ್ ಯಂತ್ರಕಲಿಕಾ ತಂತ್ರಾಂಶ ಮತ್ತು ತರಬೇತಿ ವೇಗವರ್ಧಕವಾಗಿದೆ. ONNX Runtime for Generative AI (GENAI) ಉದ್ದಿಮೆಗಳಿಗೆ ಟೇರಾ ವೇಗದಲ್ಲಿ ಜನರೇಟಿವ್ AI ಮಾದರಿಗಳನ್ನು ಸಕ್ರೀಯವಾಗಿ ನಡೆಸಲು ನೆರವಾಗುತ್ತದೆ.

## ONNX ರನ್‌ಟೈಮ್ ಎಂದರೆ ಏನು?
ONNX ರನ್‌ಟೈಮ್ ಒಂದು ಮುಕ್ತ ಮೂಲ ಯೋಜನೆಯಾಗಿದ್ದು, ಯಂತ್ರಕಲಿಕಾ ಮಾದರಿಗಳ ಹೆಚ್ಚಿನ ಕಾರ್ಯಕ್ಷಮತೆಯ ಇನ್ಫರೆನ್ಸ್‌ಗಳನ್ನು ಸಾಥ್ ಮಾಡುತ್ತದೆ. ಅದು Open Neural Network Exchange (ONNX) ಮಾದರಿಯನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ, ಇದು ಯಂತ್ರಕಲಿಕಾ ಮಾದರಿಗಳನ್ನು ಪ್ರತಿನಿಧಿಸುವ ಪ್ರಮಾಣವಾಗಿದೆ. ONNX ರನ್‌ಟೈಮ್ ಇನ್ಫರೆನ್ಸ್ ಗ್ರಾಹಕರ ಅನುಭವವನ್ನು ವೇಗವಾಗಿಸುತ್ತದೆ ಮತ್ತು ವೆಚ್ಚಗಳನ್ನು ಕಡಿಮೆಮಾಡುತ್ತದೆ. ಅದು PyTorch ಮತ್ತು TensorFlow/Keras ಗಿಂತಲೂ ಪಾರಂಪರಿಕ ಯಂತ್ರಕಲಿಕಾ ಪುಸ್ತಕಗಳಾದ scikit-learn, LightGBM, XGBoost ಮತ್ತು ಇತರೆಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ. ONNX ರನ್‌ಟೈಮ್ ವಿವಿಧ ಹಾರ್ಡ್‌ವೇರ್‌ಗಳು, ಚಾಲಕರು ಮತ್ತು ಆಪರೇಟಿಂಗ್ ಸಿಸ್ಟಮ್‌ಗಳೊಂದಿಗೆ ಹೊಂದಾಣಿಕೆಯಾಗಿದ್ದು, ಹಾರ್ಡ್‌ವೇರ್ ವೇಗವರ್ಧಕಗಳನ್ನು ಬಳಸುವುದರ ಜೊತೆಗೆ ಗ್ರಾಫ್ ಅಪ್ಟಿಮೈಜೇಷನ್‌ಗಳು ಮತ್ತು ಪರಿವರ್ತನೆಗಳಿಂದ ಅತ್ಯುತ್ತಮ ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ.

## ಜನರೇಟಿವ್ AI ಎಂದರೆ ಏನು?
ಜನರೇಟಿವ್ AI ಎಂದರೆ ಶಾಲಿತ AI ವ್ಯವಸ್ಥೆಗಳು, ಅವು ತರಬೇತಿಗೊಳಿಸಿದ ಡೇಟಾದ ಆಧಾರದ ಮೇಲೆ ಪಠ್ಯ, ಚಿತ್ರ ಅಥವಾ ಸಂಗೀತದಂತಹ ಹೊಸ ವಿಷಯವನ್ನು ಸೃಷ್ಟಿಸುವ ಸಾಮರ್ಥ್ಯವನ್ನು ಹೊಂದಿರುವವು. ಉದಾಹರಣೆಗೆ GPT-3 వంటి ಭಾಷಾ ಮಾದರಿಗಳು ಮತ್ತು ಸ್ಥಿರ ಸೋಮಗ್ಗು (Stable Diffusion) ಪ್ರಕಾರದ ಚಿತ್ರ ಸೃಷ್ಟಿ ಮಾದರಿಗಳು. ONNX ರನ್‌ಟೈಮ್ ಜನರೇಟಿವ್ AI ಲೈಬ್ರರಿಗೆ ONNX ಮಾದರಿಗಳಿಗಾಗಿ ಜನರೇಟಿವ್ AI ಲೂಪ್‌ನ್ನು, ಇನ್ಫರೆನ್ಸ್, ಲೋಗಿಟ್ ಪ್ರಕ್ರಿಯೆ, ಶೋಧನಾ ಮತ್ತು ನಮೂನಾದಾನ ಮತ್ತು KV ಕ್ಯಾಶೆ ನಿರ್ವಹಣೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ.

## GENAI ಗಾಗಿ ONNX ರನ್‌ಟೈಮ್
ONNX ರನ್‌ಟೈಮ್ GENAI ಗೆ ಜನರೇಟಿವ್ AI ಮಾದರಿಗಳನ್ನು ಬೆಂಬಲಿಸುವಂತೆ ತನ್ನ ಸಾಮರ್ಥ್ಯಗಳಿಗೆ ವಿಸ್ತಾರಮಾಡಿದೆ. ಕೆಲವು ಪ್ರಮುಖ ವೈಶಿಷ್ಟ್ಯಗಳು:

- **ವಿಶಾಲ ವೇದಿಕೆ ಬೆಂಬಲ:** ಇದು ವಿಂಡೋಸ್, ಲಿನಕ್ಸ್, ಮ್ಯಾಕ್‌ಒಎಸ್, ಆಂಡ್ರಾಯ್ಡ್ ಮತ್ತು iOS ಸೇರಿದಂತೆ ವಿವಿಧ ವೇದಿಕೆಗಳಲ್ಲಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತದೆ.
- **ಮಾದರಿ ಬೆಂಬಲ:** LLaMA, GPT-Neo, BLOOM ಮತ್ತು ಇತರ ಜನಪ್ರಿಯ ಜನರೇಟಿವ್ AI ಮಾದರಿಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತದೆ.
- **ಕಾರ್ಯಕ್ಷಮತೆ ಆಪ್ಟಿಮೈಜೇಶನ್:** NVIDIA GPU ಗಳು, AMD GPU ಗಳು ಮತ್ತು ಇತರೆ ಹಾರ್ಡ್‌ವೇರ್ ವೇಗವರ್ಧಕಗಳಿಗಾಗಿ ಆಪ್ಟಿಮೈಜೇಶನ್ಗಳನ್ನು ಒಳಗೊಂಡಿದೆ.
- **ಬಳಕೆ ಸರಳತೆ:** ಅನ್ವಯಿಕೆಗಳಲ್ಲಿ ಸುಲಭವಾಗಿ ಒಳಗೊಳ್ಳಲು API ಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ, ನೀವು ಗೊಂದಲವಿಲ್ಲದೆ ಪಠ್ಯ, ಚಿತ್ರ ಮತ್ತು ಇತರ ವಿಷಯವನ್ನು ಕಡಿಮೆ ಕೋಡ್ ಮೂಲಕ ಉತ್ಪಾದಿಸಬಹುದು.
- ಬಳಕೆದಾರರು generate() ಎಂಬ ಉನ್ನತ ಮಟ್ಟದ ವಿಧಾನವನ್ನು ಕರೆ ಮಾಡಬಹುದು ಅಥವಾ ಮಾದರಿಯ ಪ್ರತಿ ಹಂತವನ್ನು ಲೂಪಿನಲ್ಲಿ ಚಲಾಯಿಸಿ ಪ್ರತಿ ಬಾರಿ ಒಂದು ಟೋಕನ್ ಅನ್ನು ಸೃಷ್ಟಿಸಬಹುದು ಮತ್ತು ಆಯ್ಕೆಗೆ ತಕ್ಕಂತೆ ಲೂಪಿನ ಒಳಗೆ ಜನರೇಷನ್ ಪರಿಮಾಣಗಳನ್ನು ಬದಲಾಯಿಸಬಹುದು.
- ONNX ರನ್‌ಟೈಮ್ ಗ್ರೀಡಿ/ಬೀಮ್ ಶೋಧನೆ ಮತ್ತು TopP, TopK ನಮನದ ಮೂಲಕ ಟೋಕನ್ ಸರಣಿಗಳನ್ನು ರಚಿಸಲು, ಅವುಗಳ ನೈಜ ಲೋಗಿಟ್ ಪ್ರಕ್ರಿಯೆ (ಉದಾ: ಪುನರಾವೃತ್ತಿ ದಂಡಗಳು)ಗಳನ್ನು ಸಹ ಬೆಂಬಲಿಸುತ್ತದೆ. ನೀವು ಸುಲಭವಾಗಿ ಕಸ್ಟಮ್ ಸ್ಕೋರಿಂಗ್ ಕೂಡಾ ಸೇರಿಸಬಹುದು.

## ಪ್ರಾರಂಭಿಸುವುದು
GENAI ಗಾಗಿ ONNX ರನ್‌ಟೈಮ್ ಪ್ರಾರಂಭಿಸಲು, ಈ ಹಂತಗಳನ್ನು ಅನುಸರಿಸಬಹುದು:

### ONNX ರನ್‌ಟೈಮ್ ಅನ್ನು ಸ್ಥಾಪಿಸಿ:
```Python
pip install onnxruntime
```
### ಜನರೇಟಿವ್ AI ವಿಸ್ತರಣೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ:
```Python
pip install onnxruntime-genai
```

### ಮಾದರಿಯನ್ನು ಚಲಾಯಿಸಿ: ಪೈಥಾನ್‌ನಲ್ಲಿ ಸರಳ ಉದಾಹರಣೆ ಇಲ್ಲಿದೆ:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### ಡೆಮೊ: ONNX ರನ್‌ಟೈಮ್ GenAI ಬಳಸಿ Phi-3.5-Vision ಗೆ ಕರೆಗೆ


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```


**ಇತರರು**

ONNX ರನ್‌ಟೈಮ್, ಒಲ್ಲಾಮಾ ಮತ್ತು ಫೌಂಡ್ರಿ ಲೊಕಲ್ ಉಲ್ಲೇಖ ವಿಧಾನಗಳಿಗೆ ಹೆಚ್ಚುವರಿಯಾಗಿ, ನಾವು ವಿವಿಧ ತಯಾರಕರಿಂದ ಒದಗಿಸಲಾದ ಮಾದರಿ ಉಲ್ಲೇಖ ವಿಧಾನಗಳ ಮೇರೆಗೆ ಪ್ರಮಾಣೀಕೃತ ಮಾದರಿಗಳ ಉಲ್ಲೇಖವನ್ನೂ ಸಂಪೂರ್ಣಗೊಳಿಸಬಹುದು. ಉದಾ: Apple MLX ಫ್ರೆ임್ವರ್ಕ್ Apple Metal ನೊಂದಿಗೆ, Qualcomm QNN NPU ಜೊತೆಗೆ, Intel OpenVINO CPU/GPU ಜೊತೆ ಇತ್ಯಾದಿ. ಇನ್ನಷ್ಟು ವಿಷಯಗಳನ್ನು ನೀವು [Phi-3 ಕುಕ್ ಬುಕ್](https://github.com/microsoft/phi-3cookbook?WT.mc_id=academic-105485-koreyst) ನಲ್ಲಿ ಪಡೆಯಬಹುದು.
