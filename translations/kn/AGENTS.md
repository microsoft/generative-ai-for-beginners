- ಡೆವ್ ಕಂಟೈನರ್‌ಗಳು ಸಾಂಗತ್ಯದ ಅಭಿವೃದ್ಧಿ ಪರಿಸರಕ್ಕಾಗಿ

## ಸೆಟ್ ಅಪ್ ಕಮಾಂಡ್‌ಗಳು

### ಆರಂಭಿಕ ರಿಪೊಸಿಟರಿ ಸೆಟ್ ಅಪ್

```bash
# ರೆಪೊಸಿಟರಿಯನ್ನು ಕ್ಲೋನ್ ಮಾಡಿ
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# ಪರಿಸರ ಟೆಂಪ್ಲೇಟನ್ನು ನಕಲಿಸಿ
cp .env.copy .env
# ನಿಮ್ಮ API ಕೀಲಿಗಳು ಮತ್ತು ಎಂಡ್‌ಪಾಯಿಂಟ್‌ಗಳೊಂದಿಗೆ .env ಅನ್ನು ಸಂಪಾದಿಸಿ
```

### ಪೈಥಾನ್ ಪರಿಸರ ಸೆಟ್ ಅಪ್

```bash

python3 -m venv venv

# ವರ್ಚುವಲ್ ಪರಿಸರವನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ

venv\Scripts\activate

# ಅವಲಂಬನೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ
```

### Node.js/TypeScript ಸೆಟ್ ಅಪ್

### ಡೆವ್ ಕಂಟೈನರ್ ಸೆಟ್ ಅಪ್ (ಶಿಫಾರಸು)

1. GitHub Codespaces ಅಥವಾ VS Codeನಲ್ಲಿ ರಿಪೊಸಿಟರಿ ತೆರೆಯಿರಿ ಡೆವ್ ಕಂಟೈನರ್ ವಿಸ್ತರಣೆ ಸಹಿತ
2. ಡೆವ್ ಕಂಟೈನರ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ:

   - ಪೋಸ್ಟ್ ಕ್ರಿಯೇಟ್ ಸ್ಕ್ರಿಪ್ಟ್ ಅನ್ನು ರನ್ ಮಾಡುತ್ತದೆ (`.devcontainer/post-create.sh`)
   - ಜ್ಯೂಪಿಟರ್ ಕೆರ್ನೆಲ್ ಅನ್ನು ಸೆಟ್ ಅಪ್ ಮಾಡುತ್ತದೆ

## ಅಭಿವೃದ್ಧಿ ಕಾರ್ಯಪ್ರವಾಹ

### ಪರಿಸರ ಚರಗಳು

API ಪ್ರವೇಶ ಅಗತ್ಯವಿರುವ ಎಲ್ಲಾ ಪಾಠಗಳು `.env` ನಲ್ಲಿ განსDefined ಪರಿಸರ ಚರಗಳನ್ನು ಬಳಸುತ್ತವೆ:

- `OPENAI_API_KEY` - OpenAI APIಗಾಗಿ
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry (Azure OpenAI ಸೇವೆಯ ಭಾಗವಾಗಿದೆ: https://ai.azure.com)内 Azure OpenAIಗಾಗಿ
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI ಎಂಡ್ಪಾಯಿಂಟ್ URL (Foundry ಸಂಪನ್ಮೂಲ ಎಂಡ್ಪಾಯಿಂಟ್)
- `AZURE_OPENAI_DEPLOYMENT` - ಚಾಟ್ ಪೂರ್ಣತಾ ಮಾದರಿ ನಿಯೋಜನೆ ಹೆಸರು
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - ಎम्बೆಡ್ಡಿಂಗ್ ಮಾದರಿ ನಿಯೋಜನೆ ಹೆಸರು
- `AZURE_OPENAI_API_VERSION` - API ಆವೃತ್ತಿ (ಡೀಫಾಲ್ಟ್: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face ಮಾದರಿಗಳಿಗಾಗಿ
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry ಮಾದರಿಗಳ ಎಂಡ್ಪಾಯಿಂಟ್ (ಬಹು-ಪ್ರೊವೈಡರ್ ಮಾದರಿ ಕ್ಯಾಲಟಾಗ್)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry Models API ಕೀ (ನಿವೃತ್ತಿ ಆಗುತ್ತಿರುವ `GITHUB_TOKEN` ಬದಲಾವಣೆ)

## ಕೋಡ್ ಶೈಲಿ ಮಾರ್ಗಸೂಚಿಗಳು

### Python

- ಪರಿಸರ ಚರ ನಿರ್ವಹಣೆಗೆ `python-dotenv` ಬಳಸಿ
- API ಸಂವಾದಗಳಿಗಾಗಿ `openai` ಲೈಬ್ರರಿ ಆಮದು ಮಾಡಿ
- ಲಿಂಟಿಂಗ್ ಗೆ `pylint` ಬಳಸಿ (ಕೆಲವು ಉದಾಹರಣೆಗಳಲ್ಲಿ ಸರಳತೆಗೆ `# pylint: disable=all` ಇದೆ)
- PEP 8 ಹೆಸರುಕರಣ ನಿಯಮಗಳನ್ನು ಅನುಸರಿಸಿ
- API ಕ್ರೆಡೆನ್ಶಿಯಲ್ಸ್ `.env` ಫೈಲ್‌ನಲ್ಲಿ ಉಳಿಸಿ, ಕೋಡಿನಲ್ಲಿ ಎಂದಿಗೂ ಉಳಿಸಬೇಡಿ

### TypeScript

- ಪರಿಸರ ಚರಗಳಿಗೆ `dotenv` ಪ್ಯಾಕೇಜ್ ಬಳಸಿ
- ಪ್ರತಿಯೊಂದು ಆಪ್‌ಗೆ `tsconfig.json` ನಲ್ಲಿ TypeScript ಸಂರಚನೆ
- `openai` ಪ್ಯಾಕೇಜ್ ಅನ್ನು Azure OpenAI ಗೆ ಉಪಯೋಗಿಸಿ (`/openai/v1/` ಎಂಡ್ಪಾಯಿಂಟ್ ನೊಂದಿಗೆ ಕ್ಲೈಯಂಟ್ ಹಾಗೂ `client.responses.create` ಕರೆ); Microsoft Foundry Models ಗೆ `@azure-rest/ai-inference` ಬಳಸಿ
- ಸ್ವಯಂ-ರಿಲೋಡ್ ಜೊತೆ ಅಭಿವೃದ್ಧಿಗಾಗಿ `nodemon` ಉಪಯೋಗಿಸಿ
- ರನ್ ಮಾಡಲು ಮುಂಚೆ ನಿರ್ಮಿಸಿ: `npm run build` ನಂತರ `npm start`

### ಸಾಮಾನ್ಯ ನಿಯಮಗಳು

- ಕೋಡ್ ಉದಾಹರಣೆಗಳನ್ನು ಸರಳ ಮತ್ತು ಶಿಕ್ಷಣಾತ್ಮಕವಾಗಿ ಇಡಿ
- ಪ್ರಮುಖ ತತ್ವಗಳನ್ನು ವಿವರಣೆ ಮಾಡುವ ಟಿಪ್ಪಣಿಗಳನ್ನು ಸೇರಿಸಿ
- ಪ್ರತಿ ಪಾಠದ ಕೋಡ್ ಸ್ವತಂತ್ರವಾಗಿ ಮತ್ತು ನಡೆಸಬಹುದಾಗಿದೆ
- ಏಕರೂಪವಾದ ಹೆಸರುಕರಣವನ್ನು ಅನುಸರಿಸಿ: Azure OpenAI ಗಾಗಿ `aoai-` ಪೂರ್ವಪ್ರತ್ಯಯ, OpenAI API ಗಾಗಿ `oai-`, Microsoft Foundry Models ಗಾಗಿ `githubmodels-` (GitHub Models ಯುಗದಿಂದ ನಡೆದ ಪರಂಪರೆ)

### ಮಾರ್ಕ್‌ಡೌನ್ ಶೈಲಿ

- ಎಲ್ಲಾ URL ಗಳು `[text](../../url)` ನಮೂನೆಯಲ್ಲಿ ಇರಬೇಕು, ಹೆಚ್ಚುವರಿ ಖಾಲಿ ಇಲ್ಲದೆ

- ಚಿತ್ರಗಳು `./images` ಫೋಲ್ಡರ್‌ನಲ್ಲಿ ವಿವರಣಾತ್ಮಕ ಹೆಸರುಗಳೊಂದಿಗೆ ಸಂಗ್ರಹಿಸಲ್ಪಟ್ಟಿವೆ
- ಫೈಲ್ ಹೆಸರುಗಳಲ್ಲಿ ಇಂಗ್ಲಿಷ್ ಅಕ್ಷರಗಳು, ಸಂಖ್ಯೆ ಮತ್ತು ಡ್ಯಾಶ್ ಬಳಸಿರಿ

2. ** ಕೈಯಿಂದ ಪರೀಕ್ಷೆ**:
   - Python ಉದಾಹರಣೆಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ: ವರ್‌ಚಾರ್ಚುಯಲ್ ಪರಿಸರವನ್ನು ಸಕ್ರಿಯಗೊಳಿ ಮತ್ತು ಸ್ಕ್ರಿಪ್ಟ್‌ಗಳನ್ನು ರನ್ ಮಾಡಿ
   - TypeScript ಉದಾಹರಣೆಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ: `npm install`, `npm run build`, `npm start`
   - ಪರಿಸರ ಚರಗಳು ಸರಿಯಾಗಿ ಸಂರಚಿತವಾಗಿವೆ ಎಂದು ಪರಿಶೀಲಿಸಿ
   - API ಕೀಲಿಗಳು ಕೋಡ್ ಉದಾಹರಣೆಗಳ ಜೊತೆಗೆ ಕೆಲಸ ಮಾಡುತ್ತವೆಯೆ ಎಂದು ತಿಳಿಹಾಕಿ

3. **ಕೋಡ್ ಉದಾಹರಣೆಗಳು**:
   - ಎಲ್ಲಾ ಕೋಡ್ ದೋಷರಹಿತವಾಗಿ ರನ್ ಆಗಬೇಕು
   - ಆಗಾಗ್ಗೆ Azure OpenAI ಮತ್ತು OpenAI API ಜೊತೆಗೆ ಪರೀಕ್ಷಿಸಿ
   - ಒಪ್ಪಿಗೆಯಿದ್ದಲ್ಲಿ Microsoft Foundry Models ಜೊತೆ ಉದಾಹರಣೆಗಳು ಕೆಲಸ ಮಾಡುತ್ತವೆಯೆ ಎಂದು ಪರಿಶೀಲಿಸಿ

### ಸ್ವಯಂಚಾಲಿತ ಪರೀಕ್ಷಗಳಿಲ್ಲ

ಇದು ಟ್ಯುಟೋರಿಯಲ್ ಮತ್ತು ಉದಾಹರಣೆಗಳ ಮೇಲೆ ಕೇಂದ್ರೀಕೃತ ಶಿಕ್ಷಣಾತ್ಮಕ ರಿಪೊಸಿಟರಿ. ಯಾವುದೇ ಯುನಿಟ್ ಟೆಸ್ಟ್ ಅಥವಾ ಇಂಟಿಗ್ರೇಷನ್ ಟೆಸ್ಟ್ ಇಲ್ಲ. ಮಾನ್ಯತೆ ಮುಖ್ಯವಾಗಿ:
- ಕೋಡ್ ಉದಾಹರಣೆಗಳ ಕೈಯಿಂದ ಪರೀಕ್ಷೆ
- GitHub Actions ಮೂಲಕ ಮಾರ್ಕ್‌ಡೌನ್ ಮಾನ್ಯತೆ
- ಶಿಕ್ಷಣಾತ್ಮಕ ವಿಷಯಗಳ ಸಮುದಾಯ ಪರಿಶೀಲನೆ

## ಪಿ.ಆರ್ ಮಾರ್ಗಸೂಚಿಗಳು

### ಸಲ್ಲಿಸುವ ಮೊದಲು

1. ಸಂಭಾವ್ಯ Python ಮತ್ತು TypeScript ಕೋಡ್ ಬದಲಾವಣೆಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ
2. ಮಾರ್ಕ್‌ಡೌನ್ ಮಾನ್ಯತೆ ನಡೆಸಿ (ಸ್ವಯಂಚಾಲಿತವಾಗಿ PR ನಲ್ಲಿ ಟ್ರಿಗರ್ ಆಗುತ್ತದೆ)
3. ಎಲ್ಲಾ Microsoft URL ಗಳಲ್ಲಿ ಟ್ರ್ಯಾಕಿಂಗ್ IDs ಇರನಿರುತ್ತಿ
4. ಸಂಬಂಧಿಸಿದ ಲಿಂಕ್‌ಗಳು ಮಾನ್ಯವಾಗಿವೆ ಎಂದು ಹೇಳಿ
5. ಚಿತ್ರಗಳು ಸರಿಯಾಗಿ ಉಲ್ಲೇಖಿಸಲಾಗಿದೆ ಎಂದು ದೃಢೀಕರಿಸಿ

### PR ಶೀರ್ಷಿಕೆ ಸ್ವರೂಪ

- ವಿವರಣಾತ್ಮಕ ಶೀರ್ಷಿಕೆಗಳನ್ನು ಬಳಸಿ: `[Lesson 06] Fix Python example typo` ಅಥವಾ `Update README for lesson 08`
- ಅಗತ್ಯವಿದ್ದರೆ ಇಶ್ಯೂ ಸಂಖ್ಯೆಗಳನ್ನು ಸೂಚಿಸಿ: `Fixes #123`

### PR ವಿವರಣೆ

- ಏನನ್ನು ಬದಲಾಯಿಸಲಾಯಿತು ಮತ್ತು ಯಾಕೆ ಎಂಬುದನ್ನು ವಿವರಿಸಿ
- ಸಂಬಂಧಿತ ಇಶ್ಯೂಗಳಿಗೆ ಲಿಂಕ್ ನೀಡಿ
- ಕೋಡ್ ಬದಲಾವಣೆಗಳಿಗೆ ಪರಿಕ್ಷಿತ ಉದಾಹರಣೆಗಳನ್ನು ಸೂಚಿಸಿ
- ಭಾಷಾಂತರ PR ಗಳಿಗೆ ಪೂರ್ಣ ಭಾಷಾಂತರಕ್ಕಾಗಿ ಎಲ್ಲಾ ಫೈಲ್‌ಗಳನ್ನು ಸೇರಿಸಿ

### ಕೊಡುಗೆ ಅಗತ್ಯಗಳು

- Microsoft CLA ಸಹಿ ಮಾಡಬೇಕು (ಮೊದಲ PR ನಲ್ಲಿ ಸ್ವಯಂಚಾಲಿತ)
- ಬದಲಾವಣೆಮಾಡುವ ಮೊದಲು ಅಪ್‌ಚಾರ್ಜ್ ಮಾಡಿ
- ಪ್ರತಿ ಲಾಜಿಕಲ್ ಬದಲಾವಣೆಗೆ ಒಬ್ಬ PR (ಅಸಂಬಂಧಿತ ಫಿಕ್ಸ್‌ಗಳನ್ನು ಮಿಶ್ರಣ ಮಾಡಬೇಡಿ)
- ಸಾಧ್ಯವಾದಷ್ಟು PR ಗಳನ್ನು ಸಣ್ಣ ಹಾಗೂ ಕೇಂದ್ರಿತವಾಗಿರಿಸಿ

## ಸಾಮಾನ್ಯ ಕಾರ್ಯಪ್ರವಾಹಗಳು

### ಹೊಸ ಕೋಡ್ ಉದಾಹರಣೆಯನ್ನು ಸೇರಿಸುವುದು

1. ಸೂಕ್ತ ಪಾಠ ಡೈರೆಕ್ಟರಿಯನ್ನು ಪ್ರವೇಶಿಸಿ
2. `python/` ಅಥವಾ `typescript/` ಉಪ-ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಉದಾಹರಣೆ ಸೃಷ್ಟಿಸಿ
3. ಹೆಸರುಕರಣ ನಿಯಮ ಅನುಸರಿಸಿ: `{provider}-{example-name}.{py|ts|js}`
4. ನಿಜವಾದ API ಕ್ರೆಡೆನ್ಶಿಯಲ್ಸ್ ಬಳಸಿ ಪರೀಕ್ಷಿಸಿ
5. ಪಾಠ README ನಲ್ಲಿ ಯಾವುದೇ ಹೊಸ ಪರಿಸರ ಚರಗಳನ್ನು ದಾಖಲಿಸಿ

### ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ನವೀಕರಣ

1. ಪಾಠ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿನ README.md ಸಂಪಾದನೆ ಮಾಡಿ
2. ಮಾರ್ಕಡ್‌ನ ಮಾರ್ಗಸೂಚಿಗಳನ್ನು ಅನುಸರಿಸಿ (ಟ್ರ್ಯಾಕಿಂಗ್ IDಗಳು, ಸಂಬಂಧಿತ ಲಿಂಕ್‌ಗಳು)
3. GitHub Actions ಮೂಲಕ ಭಾಷಾಂತರ ನವೀಕರಣಗಳು ಮಾಡಲಾಗುತ್ತದೆ (ಮ್ಯಾನುಯಲ್ ಸಂಪಾದನೆ ಅಗತ್ಯವಿಲ್ಲ)
4. ಎಲ್ಲಾ ಲಿಂಕ್‌ಗಳು ಮಾನ್ಯವಾಗಿರುವುದನ್ನು ಪರೀಕ್ಷಿಸಿ

### ಡೆವ್ ಕಂಟೈನರ್‌ಗಳೊಂದಿಗೆ ಕೆಲಸ

1. ರಿಪೊಸಿಟರಿ `.devcontainer/devcontainer.json` ಒಳಗೊಂಡಿದೆ
2. ಪೋಸ್ಟ್-ಕ್ರಿಯೇಟ್ ಸ್ಕ್ರಿಪ್ಟ್ Python ಅವಶ್ಯಕತೆಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸ್ಥಾಪಿಸುತ್ತದೆ
3. Python ಮತ್ತು Jupyter ವಿಸ್ತರಣೆಗಳು ಪೂರ್ವ ಸಂರಚಿತವಾಗಿವೆ
4. ಪರಿಸರ `mcr.microsoft.com/devcontainers/universal:2.11.2` ಆಧಾರಿತವಾಗಿದೆ

## ನಿಯೋಜನೆ ಮತ್ತು ಪ್ರಕಟಣೆ

ಇದು ಕಲಿಕೆಗೆ ಸಂಬಂಧಿಸಿದ ರಿಪೊಸಿಟರಿ - ಯಾವುದೇ ನಿಯೋಜನೆ ಪ್ರಕ್ರಿಯೆಯಿಲ್ಲ. ಪಠ್ಯಕ್ರಮವನ್ನು ಉಪಯೋಗಿಸುತ್ತಾರೆ:

1. **GitHub ರಿಪೊಸಿಟರಿ**: ಕೋಡ್ ಮತ್ತು ಡಾಕ್ಯುಮೆಂಟೇಶನ್‌ಗೆ ನೇರ ಪ್ರವೇಶ
2. **GitHub Codespaces**: ಪೂರ್ವ ಸಂರಚಿತ ಸೆಟ್‌ಅಪ್ ಜೊತೆಗೆ ತಕ್ಷಣದ ಡೆವ್ ಪರಿಸರ
3. **Microsoft Learn**: ಅಧಿಕೃತ ಕಲಿಕೆ ವೇದಿಕೆಯಲ್ಲಿಗೂ ವಿಷಯ ಪಡೆದು ಕೊಳ್ಳಬಹುದು
4. **docsify**: Markdown ನಿಂದ ನಿರ್ಮಿತ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸೈಟ್ (ಗೋಚರವಾಗಿರುವ `docsifytopdf.js` ಮತ್ತು `package.json`)

### ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸೈಟ್ ನಿರ್ಮಾಣ

```bash
# ಡಾಕ್ಯುಮೆಂಟೇಶನ್‌ನಿಂದ PDF ಅನ್ನು ಸೃಷ್ಟಿಸಿ (ಅಗತ್ಯವಿದ್ದರೆ)
npm run convert
```

## ಸಮಸ್ಯೆ ಪರಿಹಾರ

### ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳು

**Python ಆಮದು ದೋಷಗಳು**:
- ವರ್ಚುವಲ್ ಪರಿಸರ ಸಕ್ರಿಯಗೊಂಡಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿ
- `pip install -r requirements.txt` ರನ್ ಮಾಡಿ
- Python ಆವೃತ್ತಿ 3.9+ ಆಗಿದೆ ಎಂದು ಪರಿಶೀಲಿಸಿ

**TypeScript ನಿರ್ಮಾಣ ದೋಷಗಳು**:
- ಅಗತ್ಯ ಆಪ್ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ `npm install` ರನ್ ಮಾಡಿ
- Node.js ಆವೃತ್ತಿ ಹೊಂದಿಕೊಳ್ಳುವ योग्य ಇದೆಯೇ ಎಂದು ಪರಿಶೀಲಿಸಿ
- `node_modules` ತೆರವು ಮಾಡಿ ಮತ್ತು ಮುಂಬರುವ ಸಂದರ್ಭದಲ್ಲಿ ಮರುಸ್ಥಾಪಿಸಿ

**API ದೃಢೀಕರಣ ದೋಷಗಳು**:
- `.env` ಫೈಲ್ ಇದೆ ಮತ್ತು ಸರಿಯಾದ ಮೌಲ್ಯಗಳುಳ್ಳದೆಯಾ ಎಂದು ಪರಿಶೀಲಿಸಿ
- API ಕೀಲಿಗಳು ಮಾನ್ಯವಾಗಿವೆ ಮತ್ತು ಅವಧಿ ಮುಗಿದಿಲ್ಲ ಎಂದು ಖಚಿತಪಡಿಸಿ
- ನಿಮ್ಮ ಪ್ರಾದೇಶಿಕ ಎಂಡ್ಪಾಯಿಂಟ್ URL ಗಳು ಸರಿಯಿವೆ ಎಂದು ಖಾತ್ರಿ ಮಾಡಿಕೊಳ್ಳಿ

**ಪರಿಸರ ಚರಗಳು ಕಾಣಿಸದೆ ಇರುವವೆಯಾ**:
- `.env.copy` ನಿಂದ `.env` ಗೆ ಪ್ರತಿಲಿಪಿ ಮಾಡಿ
- ನೀವು ಕೆಲಸ ಮಾಡುತ್ತಿರುವ ಪಾಠದ ಎಲ್ಲಾ ಅಗತ್ಯ ಮೌಲ್ಯಗಳನ್ನು ತುಂಬಿ
- `.env` ನವೀಕರಿಸಿದ ನಂತರ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಪುನರ್ ಪ್ರಾರಂಭ ಮಾಡಿ
