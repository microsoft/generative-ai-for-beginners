## ಅಭಿವೃದ್ಧಿ ಕಾರ್ಯಪ್ರವಾಹ

### ಪರಿಸರ 변수ಗಳು

API ಪ್ರವೇಶಕ್ಕೆ ಬೇಕಾದ ಎಲ್ಲಾ ಪಾಠಗಳು `.env` ನಲ್ಲಿ ನಿರ್ವಹಿಸಲ್ಪಡುವ ಪರಿಸರ ವ್ಯತ್ಯಾಸಗಳನ್ನು ಬಳಸುತ್ತದೆ:

- `OPENAI_API_KEY` - OpenAI API ಗಾಗಿ
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry ನಲ್ಲಿರುವ Azure OpenAI ಗಾಗಿ (Azure OpenAI ಸೇವೆ ಈಗ Microsoft Foundry ಭಾಗವಾಗುತ್ತಿದೆ: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI ಅಂತ್ಯಬಿಂದು URL (Foundry ಸಂಪನ್ಮೂಲ ಅಂತ್ಯಬಿಂದು)
- `AZURE_OPENAI_DEPLOYMENT` - ಚಾಟ್ ಪೂರ್ಣಗೊಳ್ಳುವ ಮಾದರಿ ನಿಯೋಜನೆಯ ಹೆಸರು (ಕೋರ್ಸ್ ಡೀಫಾಲ್ಟ್: `gpt-5-mini`)
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - ಎಂಬೆಡಿಂಗ್ಸ್ ಮಾದರಿ ನಿಯೋಜನೆಯ ಹೆಸರು (ಕೋರ್ಸ್ ಡೀಫಾಲ್ಟ್: `text-embedding-3-small`)
- `AZURE_OPENAI_API_VERSION` - API ಆವೃತ್ತಿ (ಡೀಫಾಲ್ಟ್: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - ಅಗತ್ಯವಿರುವ Hugging Face ಮಾದರಿಗಳಿಗಾಗಿ
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry Models ಅಂತ್ಯಬಿಂದು (ಬಹು-ಪೂರೈಕೆದಾರ ಮಾದರಿ ಕ್ಯಾಟಲಾಗ್)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry Models API ಕೀ (ನಿವೃತ್ತವಾಗುತ್ತಿರುವ `GITHUB_TOKEN` ಬದಲಿಗೆ)
- `AZURE_INFERENCE_CHAT_MODEL` - ಒಂದು ನಿರ್ಣಯವಿಲ್ಲದ ಮಾದರಿ (ಉದಾ: `Llama-3.3-70B-Instruct`) जिसे `temperature` ಉದಾಹರಣೆಗಳಲ್ಲಿ ಉಪಯೋಗಿಸಲಾಗುತ್ತದೆ, ಏಕೆಂದರೆ ನಿರ್ಣಯ ಮಾದರಿಗಳು ಸ್ಯಾಂಪ್ಲಿಂಗ್ ನಿಯಂತ್ರಣಗಳನ್ನು ಬೆಂಬಲಿಸುವುದಿಲ್ಲ

### ಮಾದರಿ ನಿಯಮಗಳು (ಮುಖ್ಯ)

- **ಡೀಫಾಲ್ಟ್ ಚಾಟ್ ಮಾದರಿ `gpt-5-mini` ಆಗಿದೆ** - ಪ್ರಸ್ತುತ, ನಿವೃತ್ತಿ ಇಲ್ಲದ **ನಿರ್ಣಯ** ಮಾದರಿ. 2026 ರಿಂದ ಹಳೆಯ `mini` ಮಾದರಿಗಳು(`gpt-4o-mini`, `gpt-4.1-mini`) ನಿವೃತ್ತಿಯಾಗುತ್ತಿವೆ, ಆದ್ದರಿಂದ ಪಠ್ಯ ಕ್ರಮವು GPT-5 ಕುಟುಂಬವನ್ನು ಸ್ಥಿರಮಾಡಿದೆ.
- **ನಿರ್ಣಯ ಮಾದರಿಗಳು `temperature` ಮತ್ತು `top_p` ಅನ್ನು ನಿರಾಕರಿಸುತ್ತವೆ**, ಮತ್ತು ಪ್ರತಿಕ್ರಿಯಾ API ನಲ್ಲಿ `max_output_tokens`/ಚಾಟ್ ಪೂರ್ಣಗೊಳ್ಳುವ `max_completion_tokens` ಬಳಕೆ ಮಾಡುತ್ತವೆ `max_tokens` ಬದಲು. `gpt-5-mini` ಕರೆಗೆಲ್ಲ `temperature`/`top_p`/`max_tokens` ಸೇರಿಸಬೇಡಿ.
- **`temperature` ಅನ್ನು ತೋರಿಸಲು**, ಉದಾಹರಣೆಗಳು **Llama** ಮಾದರಿ (`Llama-3.3-70B-Instruct`) ನು Microsoft Foundry Models ಅಂತ್ಯಬಿಂದು (`AZURE_INFERENCE_CHAT_MODEL`) ಮೂಲಕ ಬಳಸುತ್ತವೆ. ನಿರ್ಣಯ ಮಾದರಿಗಳನ್ನು ಪ್ರಾಂಪ್ಟ್ ಇಂಜಿನಿಯರಿಂಗ್ + ನಿರ್ಣಯ ನಿಯಂತ್ರಣಗಳೊಂದಿಗೆ ನಿಯಂತ್ರಿಸಿ, ಸ್ಯಾಂಪ್ಲಿಂಗ್ ನಿಯಂತ್ರಣಗಳಲ್ಲ.
- **ಫೈನ್-ಟುನಿಂಗ್ (ಪಾಠ 18)** `gpt-4.1-mini` ಅನ್ನು ಕಾಯ್ದುಕೊಳ್ಳುತ್ತದೆ: GPT-5 ಸಕ್ರೀಯ ತೃಪ್ತಿ ಫೈನ್-ಟುನಿಂಗ್ (RFT) ಮಾತ್ರ ಬೆಂಬಲಿಸುತ್ತದೆ, ಪರಿಶೀಲಿತ ಫೈನ್-ಟುನಿಂಗ್ (SFT) ಅಲ್ಲ.
- ಪಾಠ 20 (Mistral) ಮತ್ತು 21 (Meta) `temperature`/`max_tokens` ಅನ್ನು ಕಾಯ್ದುಕೊಳ್ಳುತ್ತವೆ ಏಕೆಂದರೆ ಅವು Mistral/Llama ಮಾದರಿಗಳನ್ನು ಗುರಿ ಮಾಡುತ್ತವೆ, ಅವುಗಳನ್ನು ಬೆಂಬಲಿಸುತ್ತವೆ.

- ಏಕರೂಪ ಹೆಸರೀಕರಣ ಬಳಸಿ: Azure OpenAI ಗಾಗಿ `aoai-` ಪೂರ್ವಪ್ರತ್ಯಯ, OpenAI API ಗಾಗಿ `oai-`, Microsoft Foundry Models ಗಾಗಿ `githubmodels-` (GitHub Models ಯುಗದಿಂದ ಉಳಿದ ಪ್ರಾಚೀನ ಪೂರ್ವಪ್ರತ್ಯಯ)

## ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಮಾರ್ಗದರ್ಶಿಗಳು

### ಮಾರ್ಕ್ಡೌನ್ ಶೈಲಿ

- ಎಲ್ಲಾ URL ಗಳು `[text](../../url)` ಸ್ವರೂಪದಲ್ಲಿ ಮತ್ತು ಹೆಚ್ಚುವರಿ ಖಾಲಿ ಜಾಗವಿಲ್ಲದೆ ಇರಬೇಕು
- ಆಪೇಕ್ಷಿಕ ಲಿಂಕ್ ಗಳು `./` ಅಥವಾ `../` ರಿಂದ ಪ್ರಾರಂಭವಾಗಬೇಕು
- ಎಲ್ಲಾ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಡೊಮೇನ್ ಗಳಿಗೆ ಟ್ರ್ಯಾಕ್ ಐಡಿ ಹಾಜರಿರಬೇಕು: `?WT.mc_id=academic-105485-koreyst`
- URL ಗಳಲ್ಲಿ ದೇಶ-ನಿರ್ದಿಷ್ಟ ಸ್ಥಳೀಯಿಕರಗಳನ್ನು (ಜೊತೆಗೆ `/en-us/`) ತಪ್ಪಿಸಬೇಕು
- ಚಿತ್ರಗಳು `./images` ಫೋಲ್ಡರ್‌ನಲ್ಲಿ ವಿವರಣಾತ್ಮಕ ಹೆಸರುಗಳೊಂದಿಗೆ ಸಂರಕ್ಷಿಸಲಾಗಿವೆ
- ಫೈಲ್ ಹೆಸರುಗಳಲ್ಲಿ ಇಂಗ್ಲೀಷ್ ಅಕ್ಷರಗಳು, ಸಂಖ್ಯೆಗಳು ಮತ್ತು ಡ್ಯಾಶ್ ಬಳಸಿ

### ಭಾಷಾಂತರ ಸಹಾಯಕತೆ

- ರೆಪೊಸಿಟರಿ 40+ ಭಾಷೆಗಳ 自動 GitHub Actions ಮೂಲಕ ಬೆಂಬಲಿಸುತ್ತದೆ
- ಭಾಷಾಂತರಗಳು `translations/` ಡೈರೆಕ್ಟರಿ ನಲ್ಲಿ ಸಂಗ್ರಹಿಸಲಾಗಿದೆ
- ಭಾಗಶಃ ಭಾಷಾಂತರಗಳನ್ನು ಸಲ್ಲಿಸಬೇಡಿ
- ಕಾರ್ಮಿಕ ಭಾಷಾಂತರಗಳನ್ನು ಅಂಗೀಕಾರ ಮಾಡಲಾಗುವುದಿಲ್ಲ
- ಭಾಷಾಂತರಿತ ಚಿತ್ರಗಳನ್ನು `translated_images/` ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಸಂಗ್ರಹಿಸಲಾಗಿದೆ

## ಪರೀಕ್ಷೆ ಮತ್ತು ಮಾನ್ಯತೆ

### ಸಲ್ಲಿಸುವ ಮೊದಲು ಪರಿಶೀಲನೆಗಳು

ಈ ರೆಪೊಸಿಟರಿ ಮಾನ್ಯತೆಗಾಗಿ GitHub Actions ಬಳಸುತ್ತದೆ. PR ಗಳನ್ನು ಸಲ್ಲಿಸುವ ಮೊದಲು:

1. **ಮಾರ್ಕ್ಡೌನ್ ಲಿಂಕ್ ಗಳ ಪರಿಶೀಲನೆ**:
   ```bash
   # validate-markdown.yml ಕಾರ್ಯಪ್ರವಾಹ ಪರಿಶೀಲಿಸುತ್ತದೆ:
   # - ಮುರಿದ ಸಂಬಂಧಿತ ಮಾರ್ಗಗಳು
   # - ಮಾರ್ಗಗಳಲ್ಲಿ ತಪ್ಪಿದ ಟ್ರ್ಯಾಕಿಂಗ್ ಐಡಿಗಳು
   # - URLಗಳಲ್ಲಿ ತಪ್ಪಿದ ಟ್ರ್ಯಾಕಿಂಗ್ ಐಡಿಗಳು
   # - ದೇಶೀಯ ಸ್ಥಳೀಯತೆ ಹೊಂದಿರುವ URLಗಳು
   # - ಮುರಿದ ಬಾಹ್ಯ URLಗಳು
   ```

2. **ಮ್ಯಾನುಯಲ್ ಪರೀಕ್ಷೆ**:
   - ಪೈಥಾನ್ ಉದಾಹರಣೆಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ: venv ಕೆಲಸದಲ್ಲಿ ಇನ್ನುನ್ನಿಸಿ ಮತ್ತು ಸ್ಕ್ರಿಪ್ಟ್ಗಳನ್ನು ಚಾಲನೆ ಮಾಡಿ
   - ಟೈಪ್ಸ್ಕ್ರಿಪ್ಟ್ ಉದಾಹರಣೆಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ: `npm install`, `npm run build`, `npm start`
   - ಪರಿಸರ ವ್ಯತ್ಯಾಸಗಳ ಸರಿಯಾಗಿ ಸಂರಚನೆ ಹೊಂದಿದೆಯೋ ಪರಿಶೀಲಿಸಿ
   - API ಕೀಲಿಗಳು ಕೋಡ್ ಮಾದರಿಗಳೊಡನೆ ಸರಿಯಾಗಿ ಕೆಲಸ ಮಾಡುತ್ತವೆಯೋ ಪರಿಶೀಲಿಸಿ

3. **ಕೋಡ್ ಉದಾಹರಣೆಗಳು**:
   - ಎಲ್ಲಾ ಕೋಡ್ ದೋಷವಿಲ್ಲದೆ ಚಾಲನೆಯಲ್ಲಿರಬೇಕು
   - ಅಗತ್ಯವಿದ್ದರೆ Azure OpenAI ಹಾಗೂ OpenAI API ಜೋಡಣೆಯಲ್ಲಿ ಪರೀಕ್ಷೆ ಮಾಡಿ
   - ಬೆಂಬಲವಿರುವ ಸ್ಥಳಗಳಲ್ಲಿ Microsoft Foundry Models ಜೊತೆಗೆ ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಪರಿಶೀಲಿಸಿ

### ಸ್ವಯಂ ಚಾಲಿತ ಪರೀಕ್ಷೆಗಳಿಲ್ಲ

ಇದು ಪಾಠ ಮತ್ತು ಉದಾಹರಣೆಗಳಿಗೆ ತರುವ ಶಿಕ್ಷಣಾತ್ಮಕ ರೆಪೊಸಿಟರಿ. ಹೀಗೆ, ಯಾವುದೇ ಘಟಕ ಪರೀಕ್ಷೆಗಳು ಅಥವಾ ಒಕ್ಕೂಟ ಪರೀಕ್ಷೆಗಳು ಇಲ್ಲ. ಮಾನ್ಯತೆ ಮುಖ್ಯವಾಗಿ:
- ಕೋಡ್ ಉದಾಹರಣೆಗಳ ಮ್ಯಾನುಯಲ್ ಟೆಸ್ಟಿಂಗ್
- ಮಾರ್ಕ್ಡೌನ್ ಮಾನ್ಯತೆಯಿಗಾಗಿ GitHub Actions
- ಶಿಕ್ಷಣ ಒಳಸಂಗ್ರಹದ ಸಮುದಾಯ ವಿಮರ್ಶೆ

## ಪುಲ್ ವಿನಂತಿ ಮಾರ್ಗದರ್ಶಿಗಳು

### ಸಲ್ಲಿಸುವ ಮೊದಲು

1. ಅಪ್ಲಿಕೇಶನ್ ಪೈಥಾನ್ ಮತ್ತು ಟೈಪ್ಸ್ಕ್ರಿಪ್ಟ್ ನಲ್ಲಿ ಪರೀಕ್ಷಿಸಿ ಅಗತ್ಯವಿದ್ದರೆ
2. ಮಾರ್ಕ್ಡೌನ್ ಮಾನ್ಯತೆ ನಡೆಸಿ (ಒಳಗೊಂಡಿರುವುದಾಗಿ PR ಮೇಲೆ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಚಾಲನೆಯಲ್ಲಿರುತ್ತದೆ)
3. ಎಲ್ಲಾ ಮೈಕ್ರೋಸಾಫ್ಟ್ URL ಗಳಲ್ಲಿ ಟ್ರ್ಯಾಕಿಂಗ್ ಐಡಿ ಇರುವುದನ್ನು ಖಾತ್ರಿ ಪಡಿಸಿ
4. ಸಂಬಂಧಿತ ಲಿಂಕ್ ಗಳು ಮಾನ್ಯವಾಗಿವೆ ಎಂದು ಪರಿಶೀಲಿಸಿ
5. ಚಿತ್ರಗಳಿಗೆ ಸೂಕ್ತ ಉಲ್ಲೇಖ ಇರುವುದನ್ನು ಖಾತ್ರಿ ಪಡಿಸಿ

### PR ಶೀರ್ಷಿಕೆ ಸ್ವರೂಪ

- ವಿವರಣಾತ್ಮಕ ಶೀರ್ಷಿಕೆಗಳ ಬಳಸಿ: `[Lesson 06] Fix Python example typo` ಅಥವಾ `Update README for lesson 08`
- ಸಂಬಂಧಿತ ಸಮಸ್ಯೆ ಸಂಖ್ಯೆಗಳ ಉಲ್ಲೇಖ ಕೊಡಿ: `Fixes #123`

### PR ವಿವರಣೆ

- ಯಾವ ಬದಲಾವಣೆ ಮಾಡಲಾಗಿದೆ ಮತ್ತು ಏಕೆ ಎಂದು ವಿವರಿಸಿ
- ಸಂಬಂಧಿತ ಸಮಸ್ಯೆಗಳ ಲಿಂಕ್ ಗಳನ್ನು ಸೇರಿಸಿ
- ಕೋಡ್ ಬದಲಾವಣೆಗಳಿಗೆ, ಪರೀಕ್ಷಿಸಿದ ಉದಾಹರಣೆಗಳನ್ನು ಸೂಚಿಸಿ
- ಭಾಷಾಂತರ PR ಗಳಿಗೆ, ಸಂಪೂರ್ಣ ಭಾಷಾಂತರಕ್ಕಾಗಿ ಎಲ್ಲಾ ಫೈಲ್ ಗಳನ್ನು ಸೇರಿಸಿ

### ಸಹಾಯದ ಅಗತ್ಯತೆಗಳು

- ಮೈಕ್ರೋಸಾಫ್ಟ್ CLA ಗೆ ಸಹಿ ಮಾಡಿ (ಮுதல் PR ಗೆ ಸ್ವಯಂಚಾಲಿತ)
- ಬದಲಾವಣೆ ಮಾಡಲು ಮೊದಲು ರೆಪೊಸಿಟರಿಯನ್ನು ನಕಲಿಸಿ
- ಪ್ರತಿ ಸಾಂದರ್ಭಿಕ ಬದಲಾವಣೆಗೆ ಒಂದು PR (ಸಂಬಂಧಿಲ್ಲದ ತಪಾಸಣೆಗಳನ್ನು ಸಂಯೋಜಿಸಬೇಡಿ)
- ಸಾಧ್ಯವಾದರೆ PR ಗಳನ್ನು ಗುರ್ತಿಸಿಲು ಮತ್ತು ಚಿಕ್ಕದಾಗಿ ಇಡಿ

## ಸಾಮಾನ್ಯ ಕಾರ್ಯಪ್ರವಾಹಗಳು

5. ಹೊಸ ಪರಿಸರ ವ್ಯತ್ಯಾಸಗಳನ್ನೂ ಪಾಠ README ರಲ್ಲಿ ದಾಖಲಿಸಿ

- `.env` ನ್ನು ನವೀಕರಿಸಿದ ನಂತರ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಮರುಪ್ರಾರಂಭಿಸಿ
