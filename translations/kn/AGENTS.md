# AGENTS.md

## ಪ್ರಾಜೆಕ್ಟ್ ಅವಲೋಕನ

ಈ ರೆಪೊಸಿಟರಿ ಜನರೇಟಿವ್ AI ಮೂಲಭೂತಗಳು ಮತ್ತು ಅಪ್ಲಿಕೇಶನ್ ಅಭಿವೃದ್ಧಿಯನ್ನು ಕಲಿಸುವ 21 ಪಾಠಗಳ ಸಮಗ್ರ ಪಠ್ಯಕ್ರಮವನ್ನು ಒಳಗೊಂಡಿದೆ. ಈ ಕೋರ್ಸ್ ಆರಂಭಿಕರಿಗಾಗಿ ವಿನ್ಯಾಸಗೊಳಿಸಲಾಗಿದೆ ಮತ್ತು ಮೂಲಭೂತ ತತ್ವಗಳಿಂದ ಉತ್ಪಾದನಾ-ತಯಾರ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುವವರೆಗೆ ಎಲ್ಲವನ್ನೂ ಒಳಗೊಂಡಿದೆ.

**ಪ್ರಮುಖ ತಂತ್ರಜ್ಞಾನಗಳು:**
- Python 3.9+ ಲೈಬ್ರರಿಗಳೊಂದಿಗೆ: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript Node.js ಮತ್ತು ಲೈಬ್ರರಿಗಳೊಂದಿಗೆ: `@azure/openai`, `@azure-rest/ai-inference`, `openai`
- Azure OpenAI ಸೇವೆ, OpenAI API, ಮತ್ತು GitHub ಮಾದರಿಗಳು
- ಇಂಟರಾಕ್ಟಿವ್ ಕಲಿಕೆಗೆ Jupyter ನೋಟ್ಬುಕ್‌ಗಳು
- ಸತತ ಅಭಿವೃದ್ಧಿ ಪರಿಸರಕ್ಕಾಗಿ Dev Containers

**ರೆಪೊಸಿಟರಿ ರಚನೆ:**
- 21 ಸಂಖ್ಯಿತ ಪಾಠ ಡೈರೆಕ್ಟರಿಗಳು (00-21) READMEs, ಕೋಡ್ ಉದಾಹರಣೆಗಳು ಮತ್ತು ಅಸೈನ್ಮೆಂಟ್‌ಗಳನ್ನು ಒಳಗೊಂಡಿವೆ
- Python, TypeScript ಮತ್ತು ಕೆಲವೊಮ್ಮೆ .NET ಉದಾಹರಣೆಗಳ ಅನೇಕ ಅನುಷ್ಠಾನಗಳು
- 40+ ಭಾಷಾ ಆವೃತ್ತಿಗಳೊಂದಿಗೆ ಅನುವಾದಗಳ ಡೈರೆಕ್ಟರಿ
- `.env` ಫೈಲ್ ಮೂಲಕ ಕೇಂದ್ರಿತ ಸಂರಚನೆ (ಟೆಂಪ್ಲೇಟಾಗಿ `.env.copy` ಬಳಸಿ)

## ಸೆಟಪ್ ಕಮಾಂಡ್‌ಗಳು

### ಪ್ರಾಥಮಿಕ ರೆಪೊಸಿಟರಿ ಸೆಟಪ್

```bash
# ರೆಪೊಸಿಟರಿಯನ್ನು ಕ್ಲೋನ್ ಮಾಡಿ
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# ಪರಿಸರ ಟೆಂಪ್ಲೇಟನ್ನು ನಕಲಿಸಿ
cp .env.copy .env
# ನಿಮ್ಮ API ಕೀಗಳು ಮತ್ತು ಎಂಡ್ಪಾಯಿಂಟ್‌ಗಳೊಂದಿಗೆ .env ಅನ್ನು ಸಂಪಾದಿಸಿ
```

### Python ಪರಿಸರ ಸೆಟಪ್

```bash
# ವರ್ಚುವಲ್ ಪರಿಸರವನ್ನು ರಚಿಸಿ
python3 -m venv venv

# ವರ್ಚುವಲ್ ಪರಿಸರವನ್ನು ಸಕ್ರಿಯಗೊಳಿಸಿ
# macOS/Linux ನಲ್ಲಿ:
source venv/bin/activate
# Windows ನಲ್ಲಿ:
venv\Scripts\activate

# ಅವಲಂಬನೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ
pip install -r requirements.txt
```

### Node.js/TypeScript ಸೆಟಪ್

```bash
# ರೂಟ್-ಮಟ್ಟದ ಅವಲಂಬನೆಗಳನ್ನು ಸ್ಥಾಪಿಸಿ (ದಾಖಲೆ ಸಾಧನಗಳಿಗಾಗಿ)
npm install

# ವೈಯಕ್ತಿಕ ಪಾಠ TypeScript ಉದಾಹರಣೆಗಳಿಗೆ, ನಿರ್ದಿಷ್ಟ ಪಾಠಕ್ಕೆ ನವಿಗೇಟ್ ಮಾಡಿ:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Dev Container ಸೆಟಪ್ (ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ)

ರೆಪೊಸಿಟರಿಯಲ್ಲಿ GitHub Codespaces ಅಥವಾ VS Code Dev Containers ಗಾಗಿ `.devcontainer` ಸಂರಚನೆ ಇದೆ:

1. GitHub Codespaces ಅಥವಾ VS Code ನಲ್ಲಿ Dev Containers ವಿಸ್ತರಣೆ ಜೊತೆಗೆ ರೆಪೊಸಿಟರಿ ತೆರೆಯಿರಿ
2. Dev Container ಸ್ವಯಂಚಾಲಿತವಾಗಿ:
   - `requirements.txt` ನಿಂದ Python ಅವಲಂಬನೆಗಳನ್ನು ಸ್ಥಾಪಿಸುತ್ತದೆ
   - ಪೋಸ್ಟ್-ಕ್ರಿಯೇಟ್ ಸ್ಕ್ರಿಪ್ಟ್ (`.devcontainer/post-create.sh`) ಅನ್ನು ಚಲಾಯಿಸುತ್ತದೆ
   - Jupyter ಕರ್ಣಲ್ ಅನ್ನು ಸೆಟ್ ಅಪ್ ಮಾಡುತ್ತದೆ

## ಅಭಿವೃದ್ಧಿ ಕಾರ್ಯಪ್ರವಾಹ

### ಪರಿಸರ ಚರಗಳು

ಎಲ್ಲಾ ಪಾಠಗಳು API ಪ್ರವೇಶಕ್ಕಾಗಿ `.env` ನಲ್ಲಿ ವ್ಯಾಖ್ಯಾನಿಸಲಾದ ಪರಿಸರ ಚರಗಳನ್ನು ಬಳಸುತ್ತವೆ:

- `OPENAI_API_KEY` - OpenAI API ಗಾಗಿ
- `AZURE_OPENAI_API_KEY` - Azure OpenAI ಸೇವೆಗಾಗಿ
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI ಎಂಡ್‌ಪಾಯಿಂಟ್ URL
- `AZURE_OPENAI_DEPLOYMENT` - ಚಾಟ್ ಪೂರ್ಣಗೊಳಿಸುವ ಮಾದರಿ ನಿಯೋಜನೆ ಹೆಸರು
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - ಎम्बೆಡ್ಡಿಂಗ್ ಮಾದರಿ ನಿಯೋಜನೆ ಹೆಸರು
- `AZURE_OPENAI_API_VERSION` - API ಆವೃತ್ತಿ (ಡೀಫಾಲ್ಟ್: `2024-02-01`)
- `HUGGING_FACE_API_KEY` - Hugging Face ಮಾದರಿಗಳಿಗಾಗಿ
- `GITHUB_TOKEN` - GitHub ಮಾದರಿಗಳಿಗಾಗಿ

### Python ಉದಾಹರಣೆಗಳನ್ನು ಚಾಲನೆ ಮಾಡುವುದು

```bash
# ಪಾಠ ಡೈರೆಕ್ಟರಿಗೆ ನಾವಿಗೇಟ್ ಮಾಡಿ
cd 06-text-generation-apps/python

# ಪೈಥಾನ್ ಸ್ಕ್ರಿಪ್ಟ್ ಅನ್ನು ಚಾಲನೆ ಮಾಡಿ

```

### TypeScript ಉದಾಹರಣೆಗಳನ್ನು ಚಾಲನೆ ಮಾಡುವುದು

cd 06-text-generation-apps/typescript/recipe-app

# ಟೈಪ್‌ಸ್ಕ್ರಿಪ್ಟ್ ಕೋಡ್ ಅನ್ನು ನಿರ್ಮಿಸಿ

## ಕೋಡ್ ಶೈಲಿ ಮಾರ್ಗಸೂಚಿಗಳು

### Python

- ಪರಿಸರ ಚರ ನಿರ್ವಹಣೆಗೆ `python-dotenv` ಬಳಸಿ
- API ಸಂವಹನಗಳಿಗೆ `openai` ಲೈಬ್ರರಿ ಆಮದುಮಾಡಿ
- ಲಿಂಟಿಂಗ್‌ಗೆ `pylint` ಬಳಸಿ (ಸರಳತೆಗೆ ಕೆಲವು ಉದಾಹರಣೆಗಳಲ್ಲಿ `# pylint: disable=all` ಇದೆ)
- PEP 8 ನಾಮಕರಣ ನಿಯಮಗಳನ್ನು ಅನುಸರಿಸಿ
- API ಪ್ರಮಾಣಪತ್ರಗಳನ್ನು `.env` ಫೈಲ್‌ನಲ್ಲಿ ಸಂಗ್ರಹಿಸಿ, ಕೋಡ್‌ನಲ್ಲಿ ಎಂದಿಗೂ ಇರಿಸಬೇಡಿ

### TypeScript

- ಪರಿಸರ ಚರಗಳಿಗೆ `dotenv` ಪ್ಯಾಕೇಜ್ ಬಳಸಿ
- ಪ್ರತಿ ಅಪ್ಲಿಕೇಶನ್‌ಗೆ `tsconfig.json` ನಲ್ಲಿ TypeScript ಸಂರಚನೆ
- Azure ಸೇವೆಗಳಿಗೆ `@azure/openai` ಅಥವಾ `@azure-rest/ai-inference` ಬಳಸಿ
- ಸ್ವಯಂಚಾಲಿತ ಪುನಃಲೋಡ್ ಜೊತೆಗೆ ಅಭಿವೃದ್ಧಿಗೆ `nodemon` ಬಳಸಿ
- ಚಾಲನೆಗೆ ಮೊದಲು ನಿರ್ಮಿಸಿ: `npm run build` ನಂತರ `npm start`

### ಸಾಮಾನ್ಯ ನಿಯಮಗಳು

- ಕೋಡ್ ಉದಾಹರಣೆಗಳನ್ನು ಸರಳ ಮತ್ತು ಶೈಕ್ಷಣಿಕವಾಗಿರಿಸಿ
- ಪ್ರಮುಖ ತತ್ವಗಳನ್ನು ವಿವರಿಸುವ ಕಾಮೆಂಟ್‌ಗಳನ್ನು ಸೇರಿಸಿ
- ಪ್ರತಿ ಪಾಠದ ಕೋಡ್ ಸ್ವತಂತ್ರವಾಗಿದ್ದು ಚಾಲನೆ ಮಾಡಬಹುದಾಗಿರಲಿ
- Azure OpenAI ಗಾಗಿ `aoai-`, OpenAI API ಗಾಗಿ `oai-`, GitHub ಮಾದರಿಗಳಿಗಾಗಿ `githubmodels-` ಎಂಬ ನಿಯಮಿತ ನಾಮಕರಣ ಬಳಸಿ

   - ಎಲ್ಲಾ ಕೋಡ್ ದೋಷರಹಿತವಾಗಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತವೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
   - ಸಾಧ್ಯವಾದರೆ Azure OpenAI ಮತ್ತು OpenAI API ಎರಡರಲ್ಲಿಯೂ ಪರೀಕ್ಷಿಸಿ
   - GitHub ಮಾದರಿಗಳೊಂದಿಗೆ ಕಾರ್ಯನಿರ್ವಹಿಸುವ ಉದಾಹರಣೆಗಳನ್ನು ಪರಿಶೀಲಿಸಿ

### ಸ್ವಯಂಚಾಲಿತ ಪರೀಕ್ಷೆಗಳು ಇಲ್ಲ

ಇದು ಪಾಠ ಮತ್ತು ಉದಾಹರಣೆಗಳ ಮೇಲೆ ಕೇಂದ್ರೀಕರಿಸಿದ ಶೈಕ್ಷಣಿಕ ರೆಪೊಸಿಟರಿ. ಯಾವುದೇ ಯೂನಿಟ್ ಅಥವಾ ಇಂಟಿಗ್ರೇಶನ್ ಪರೀಕ್ಷೆಗಳು ಇಲ್ಲ. ಮಾನ್ಯತೆ ಮುಖ್ಯವಾಗಿ:
- ಕೋಡ್ ಉದಾಹರಣೆಗಳ ಮ್ಯಾನುಯಲ್ ಪರೀಕ್ಷೆ
- ಮಾರ್ಕ್‌ಡೌನ್ ಮಾನ್ಯತೆಗಾಗಿ GitHub Actions
- ಶೈಕ್ಷಣಿಕ ವಿಷಯದ ಸಮುದಾಯ ವಿಮರ್ಶೆ

## ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್ ಮಾರ್ಗಸೂಚಿಗಳು

### ಸಲ್ಲಿಸುವ ಮೊದಲು

1. Python ಮತ್ತು TypeScript ಎರಡರಲ್ಲಿಯೂ ಕೋಡ್ ಬದಲಾವಣೆಗಳನ್ನು ಪರೀಕ್ಷಿಸಿ (ಸಾಧ್ಯವಾದರೆ)
2. ಮಾರ್ಕ್‌ಡೌನ್ ಮಾನ್ಯತೆ ನಡೆಸಿ (PR ನಲ್ಲಿ ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಚಾಲನೆ)
3. ಎಲ್ಲಾ Microsoft URL ಗಳಲ್ಲಿ ಟ್ರ್ಯಾಕಿಂಗ್ ID ಗಳಿರುವುದನ್ನು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
4. ಸಂಬಂಧಿತ ಲಿಂಕ್‌ಗಳು ಮಾನ್ಯವಾಗಿವೆ ಎಂದು ಪರಿಶೀಲಿಸಿ
5. ಚಿತ್ರಗಳು ಸರಿಯಾಗಿ ಉಲ್ಲೇಖಿಸಲಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ

### PR ಶೀರ್ಷಿಕೆ ಸ್ವರೂಪ

- ವಿವರಣಾತ್ಮಕ ಶೀರ್ಷಿಕೆಗಳನ್ನು ಬಳಸಿ: `[Lesson 06] Python ಉದಾಹರಣೆಯ ಟೈಪೋ ಸರಿಪಡಿಸಿ` ಅಥವಾ `lesson 08 ಗಾಗಿ README ನವೀಕರಿಸಿ`
- ಸಂಬಂಧಿತ ಸಮಸ್ಯೆ ಸಂಖ್ಯೆಗಳ ಉಲ್ಲೇಖ: `Fixes #123`

### PR ವಿವರಣೆ

- ಏನು ಬದಲಾಯಿಸಲಾಗಿದೆ ಮತ್ತು ಏಕೆ ಎಂಬುದನ್ನು ವಿವರಿಸಿ
- ಸಂಬಂಧಿತ ಸಮಸ್ಯೆಗಳಿಗೆ ಲಿಂಕ್ ನೀಡಿ
- ಕೋಡ್ ಬದಲಾವಣೆಗಳಿಗೆ ಯಾವ ಉದಾಹರಣೆಗಳನ್ನು ಪರೀಕ್ಷಿಸಲಾಗಿದೆ ಎಂದು ಸೂಚಿಸಿ
- ಅನುವಾದ PR ಗಾಗಿ ಸಂಪೂರ್ಣ ಅನುವಾದಕ್ಕಾಗಿ ಎಲ್ಲಾ ಫೈಲ್‌ಗಳನ್ನು ಸೇರಿಸಿ

### ಕೊಡುಗೆ ಅಗತ್ಯತೆಗಳು

- Microsoft CLA ಗೆ ಸಹಿ ಮಾಡಿ (ಮೊದಲ PR ನಲ್ಲಿ ಸ್ವಯಂಚಾಲಿತ)
- ಬದಲಾವಣೆಗಳ ಮೊದಲು ರೆಪೊಸಿಟರಿಯನ್ನು ನಿಮ್ಮ ಖಾತೆಗೆ ಫೋರ್ಕ್ ಮಾಡಿ
- ಪ್ರತಿ ತಾರ್ಕಿಕ ಬದಲಾವಣೆಗೆ ಒಂದು PR (ಅಸಂಬಂಧಿತ ಸರಿಪಡಣೆಗಳನ್ನು ಮಿಶ್ರಣ ಮಾಡಬೇಡಿ)
- PR ಗಳು ಸಾಧ್ಯವಾದಷ್ಟು ಕೇಂದ್ರೀಕೃತ ಮತ್ತು ಸಣ್ಣವಾಗಿರಲಿ

## ಸಾಮಾನ್ಯ ಕಾರ್ಯಪ್ರವಾಹಗಳು

### ಹೊಸ ಕೋಡ್ ಉದಾಹರಣೆಯನ್ನು ಸೇರಿಸುವುದು

1. ಸಂಬಂಧಿತ ಪಾಠ ಡೈರೆಕ್ಟರಿಗೆ ಹೋಗಿ
2. `python/` ಅಥವಾ `typescript/` ಉಪಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಉದಾಹರಣೆ ರಚಿಸಿ
3. ನಾಮಕರಣ ನಿಯಮ ಅನುಸರಿಸಿ: `{provider}-{example-name}.{py|ts|js}`
4. ನಿಜವಾದ API ಪ್ರಮಾಣಪತ್ರಗಳೊಂದಿಗೆ ಪರೀಕ್ಷಿಸಿ
5. ಪಾಠ README ನಲ್ಲಿ ಯಾವುದೇ ಹೊಸ ಪರಿಸರ ಚರಗಳನ್ನು ದಾಖಲಿಸಿ

### ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ನವೀಕರಣ

1. ಪಾಠ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ README.md ಸಂಪಾದಿಸಿ
2. ಮಾರ್ಕ್‌ಡೌನ್ ಮಾರ್ಗಸೂಚಿಗಳನ್ನು ಅನುಸರಿಸಿ (ಟ್ರ್ಯಾಕಿಂಗ್ ID ಗಳು, ಸಂಬಂಧಿತ ಲಿಂಕ್‌ಗಳು)
3. ಅನುವಾದಗಳನ್ನು GitHub Actions ನಿಂದ ನಿರ್ವಹಿಸಲಾಗುತ್ತದೆ (ಮ್ಯಾನುಯಲ್ ಸಂಪಾದನೆ ಮಾಡಬೇಡಿ)
4. ಎಲ್ಲಾ ಲಿಂಕ್‌ಗಳು ಮಾನ್ಯವಾಗಿವೆ ಎಂದು ಪರೀಕ್ಷಿಸಿ

### Dev Containers ಜೊತೆಗೆ ಕೆಲಸ ಮಾಡುವುದು

1. ರೆಪೊಸಿಟರಿಯಲ್ಲಿ `.devcontainer/devcontainer.json` ಇದೆ
2. ಪೋಸ್ಟ್-ಕ್ರಿಯೇಟ್ ಸ್ಕ್ರಿಪ್ಟ್ ಸ್ವಯಂಚಾಲಿತವಾಗಿ Python ಅವಲಂಬನೆಗಳನ್ನು ಸ್ಥಾಪಿಸುತ್ತದೆ
3. Python ಮತ್ತು Jupyter ವಿಸ್ತರಣೆಗಳು ಪೂರ್ವ ಸಂರಚಿತವಾಗಿವೆ
4. ಪರಿಸರ `mcr.microsoft.com/devcontainers/universal:2.11.2` ಆಧಾರಿತವಾಗಿದೆ

## ನಿಯೋಜನೆ ಮತ್ತು ಪ್ರಕಟಣೆ

ಇದು ಕಲಿಕೆಯ ರೆಪೊಸಿಟರಿ - ಯಾವುದೇ ನಿಯೋಜನೆ ಪ್ರಕ್ರಿಯೆ ಇಲ್ಲ. ಪಠ್ಯಕ್ರಮವನ್ನು ಕೆಳಗಿನ ಮೂಲಕ ಉಪಯೋಗಿಸಲಾಗುತ್ತದೆ:

1. **GitHub ರೆಪೊಸಿಟರಿ**: ಕೋಡ್ ಮತ್ತು ಡಾಕ್ಯುಮೆಂಟೇಶನ್‌ಗೆ ನೇರ ಪ್ರವೇಶ
2. **GitHub Codespaces**: ಪೂರ್ವ ಸಂರಚಿತ ಸೆಟಪ್ ಜೊತೆಗೆ ತಕ್ಷಣದ ಅಭಿವೃದ್ಧಿ ಪರಿಸರ
3. **Microsoft Learn**: ಅಧಿಕೃತ ಕಲಿಕಾ ವೇದಿಕೆಗೆ ವಿಷಯವನ್ನು ಪ್ರಸಾರ ಮಾಡಬಹುದು
4. **docsify**: ಮಾರ್ಕ್‌ಡೌನ್‌ನಿಂದ ನಿರ್ಮಿತ ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸೈಟ್ (ನೋಡಿ `docsifytopdf.js` ಮತ್ತು `package.json`)

### ಡಾಕ್ಯುಮೆಂಟೇಶನ್ ಸೈಟ್ ನಿರ್ಮಾಣ

```bash
# ಡಾಕ್ಯುಮೆಂಟೇಶನ್‌ನಿಂದ PDF ರಚಿಸಿ (ಅಗತ್ಯವಿದ್ದರೆ)
npm run convert
```

## ಸಮಸ್ಯೆ ಪರಿಹಾರ

### ಸಾಮಾನ್ಯ ಸಮಸ್ಯೆಗಳು

**Python ಆಮದು ದೋಷಗಳು**:
- ವರ್ಚುವಲ್ ಪರಿಸರ ಸಕ್ರಿಯವಾಗಿದೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ
- `pip install -r requirements.txt` ಅನ್ನು ಚಲಾಯಿಸಿ
- Python ಆವೃತ್ತಿ 3.9+ ಆಗಿದೆ ಎಂದು ಪರಿಶೀಲಿಸಿ

**TypeScript ನಿರ್ಮಾಣ ದೋಷಗಳು**:
- ನಿರ್ದಿಷ್ಟ ಅಪ್ಲಿಕೇಶನ್ ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ `npm install` ಅನ್ನು ಚಲಾಯಿಸಿ
- Node.js ಆವೃತ್ತಿ ಹೊಂದಿಕೆಯಾಗಿರುವುದನ್ನು ಪರಿಶೀಲಿಸಿ
- `node_modules` ಅನ್ನು ತೆರವುಗೊಳಿಸಿ ಮತ್ತು ಅಗತ್ಯವಿದ್ದರೆ ಮರುಸ್ಥಾಪಿಸಿ

**API ಪ್ರಮಾಣೀಕರಣ ದೋಷಗಳು**:
- `.env` ಫೈಲ್ ಇದೆ ಮತ್ತು ಸರಿಯಾದ ಮೌಲ್ಯಗಳಿವೆ ಎಂದು ಪರಿಶೀಲಿಸಿ
- API ಕೀಗಳು ಮಾನ್ಯವಾಗಿವೆ ಮತ್ತು ಅವಧಿ ಮುಗಿಯಿಲ್ಲ
- ನಿಮ್ಮ ಪ್ರದೇಶಕ್ಕೆ ಸರಿಯಾದ ಎಂಡ್‌ಪಾಯಿಂಟ್ URL ಗಳಿವೆ ಎಂದು ಖಚಿತಪಡಿಸಿಕೊಳ್ಳಿ

**ಪರಿಸರ ಚರಗಳು ಕಾಣೆಯಾದರೆ**:
- `.env.copy` ನ್ನು `.env` ಗೆ ನಕಲಿಸಿ
- ನೀವು ಕೆಲಸ ಮಾಡುತ್ತಿರುವ ಪಾಠಕ್ಕೆ ಅಗತ್ಯವಿರುವ ಎಲ್ಲಾ ಮೌಲ್ಯಗಳನ್ನು ಭರ್ತಿ ಮಾಡಿ
- `.env` ನವೀಕರಿಸಿದ ನಂತರ ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಮರುಪ್ರಾರಂಭಿಸಿ
