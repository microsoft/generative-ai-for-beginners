## ಪರಿಚಯ

AI ಏಜೆಂಟುಗಳು ಜನರೇಟಿವ್ AI ನಲ್ಲಿ ರೋಚಕ ಅಭಿವೃದ್ದಿಯನ್ನು ಪ್ರತಿನಿಧಿಸುತ್ತವೆ, LLM ಗಳನ್ನು ಸಹಾಯಕರಿಂದ ಕ್ರಿಯೆಗಳು ಕೈಗೊಂಡು ಸಾಧನೆ ಮಾಡಬಲ್ಲ ಏಜೆಂಟ್ ಗಳಾಗಿ ಪರಿವರ್ತಿಸುವುದಕ್ಕೆ ಅನುಮತಿ ನೀಡುತ್ತವೆ. AI ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್ ಗಳ ಮೂಲಕ ಡೆವಲಪರ್ ಗಳಿಗೆ LLM ಗಳಿಗೆ ಉಪಕರಣಗಳು ಮತ್ತು ಸ್ಥಿತಿ ನಿರ್ವಹಣೆಗೆ ಪ್ರವೇಶ ನೀಡುವ ಅಪ್ಲಿಕೇಶನ್ ಗಳನ್ನು ರಚಿಸುವುದಕ್ಕೆ ಸಾಧ್ಯವಾಗುತ್ತದೆ. ಈ ಫ್ರೇಮ್ವರ್ಕ್ ಗಳನ್ನು ಬಳಸಿ ಬಳಕೆದಾರರು ಮತ್ತು ಡೆವಲಪರ್ ಗಳು LLM ಗಳ ಯೋಜಿತ ಚಟುವಟಿಕೆಗಳನ್ನು ನಿಗಾ ವಹಿಸಬಹುದು, ಹೀಗಾಗಿ ಅನುಭವ ನಿರ್ವಹಣೆ ಸುಧಾರಿಸುವುದು ಸಾಧ್ಯವಾಗುತ್ತದೆ.

ಈ ಪಾಠವು ಕೆಳಗಿನ ಕ್ಷೇತ್ರಗಳನ್ನು ಒಳಗೊಂಡಿರುತ್ತದೆ:

- AI ಏಜೆಂಟ್ ಎಂದರೇನು ಎಂಬುದನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು - AI ಏಜೆಂಟ್ ಅಂದರೆ ನಿಜಕ್ಕೂ ಏನು?
- ಐದು ವಿಭಿನ್ನ AI ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್ ಗಳ ಬಗ್ಗೆ ಓದು ಮಾಡಿಕೊಳ್ಳುವುದು - ಅವುಗಳು ಏಕೆ ವಿಶೇಷ?
- ವಿವಿಧ ಬಳಕೆ ಪ್ರಕರಣಗಳಿಗೆ ಈ AI ಏಜೆಂಟ್ ಗಳನ್ನು ಅನ್ವಯಿಸುವುದು - ನಾವು ಯಾವಾಗ AI ಏಜೆಂಟ್ ಗಳನ್ನು ಬಳಸಬೇಕು?

## ಕಲಿಕೆಯ ಗುರಿಗಳು

ಈ ಪಾಠವನ್ನು ತೆಗೆದುಕೊಂಡ ನಂತರ, ನೀವು ಸಾಧ್ಯವಾಗುವುದು:

- AI ಏಜೆಂಟ್ ಗಳು ಏನು ಮತ್ತು ಅವುಗಳನ್ನು ಹೇಗೆ ಬಳಸಬಹುದು ಎಂಬುದನ್ನು ವಿವರಿಸಲು.
- ಕೆಲವು ಜನಪ್ರಿಯ AI ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್ ಗಳ ನಡುವಿನ ವ್ಯತ್ಯಾಸಗಳನ್ನು ಮತ್ತು ಅವುಗಳು ಹೇಗೆ ವಿಭಿನ್ನವೆಂಬುದನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು.
- AI ಏಜೆಂಟ್ ಗಳ ಕಾರ್ಯಾಚರಣೆಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಂಡು ಅವುಗಳೊಂದಿಗೆ ಅಪ್ಲಿಕೇಶನ್ ಗಳನ್ನು ನಿರ್ಮಿಸಲು.

## AI ಏಜೆಂಟ್ ಗಳು ಎಂದರೆ ಏನು?

AI ಏಜೆಂಟುಗಳು ಜನರೇಟಿವ್ AI ಜಗತ್ತಿನಲ್ಲಿ ಬಹಳ ರೋಚಕ ಕ್ಷೇತ್ರವಾಗಿದೆ. ಈ ರೋಚಕತೆಯೊಂದಿಗೆ ಕೆಲವೊಮ್ಮೆ ಪದಗಳ ಮತ್ತು ಅವುಗಳ ಅನ್ವಯಿಕೆಗಳ ಗೊಂದಲವೂ ಬರುತ್ತದೆ. AI ಏಜೆಂಟ್ ಗಳಿಗೆ ಸಂಬಂಧಿಸಿದ ಹೆಚ್ಚಿನ ಉಪಕರಣಗಳನ್ನು ಒಳಗೊಂಡಂತೆ ಸರಳ ಮತ್ತು ಒಳಗೊಂಡ ನಿರೂಪಣೆಯನ್ನು ಬಳಸಲು ನಾವು ಈ ವ್ಯಾಖ್ಯಾನವನ್ನು ಮಾಡೋಣ:

AI ಏಜೆಂಟ್ ಗಳು ಲಾರ್ಜ್ ಲ್ಯಾಂಗ್ವೇಜ್ ಮಾಡಲ್ಸ್ (LLMs) ಗೆ **ಸ್ಥಿತಿ** ಮತ್ತು **ಉಪಕರಣಗಳ** ಪ್ರವೇಶವನ್ನು ಒದಗಿಸುವ ಮೂಲಕ ಕಾರ್ಯಗಳನ್ನು ನಿರ್ವಹಿಸಲು ಅನುವು ಮಾಡಿಕೊಡುತ್ತವೆ.

![ಏಜೆಂಟ್ ಮಾದರಿ](../../../translated_images/kn/what-agent.21f2893bdfd01e6a.webp)

ಇದೇ ಪದಗಳನ್ನು ವ್ಯಾಖ್ಯಾನ ಮಾಡೋಣ:

**ಲಾರ್ಜ್ ಲ್ಯಾಂಗ್ವೇಜ್ ಮಾಡಲ್ಸ್** - ಈ ಪಾಠದಲ್ಲಿ ನಾವು ಉಲ್ಲೇಖಿಸುವ ಮಾದರಿಗಳು, ಉದಾಹರಣೆಗೆ GPT-3.5, GPT-4, Llama-2 ಮುಂತಾದವುಗಳು.

**ಸಸ್ಥಿತಿ** - LLM ಕಾರ್ಯನಡೆಯುತ್ತಿರುವ ಹಿನ್ನೆಲೆ ಅಥವಾ ಪರಿಸರವನ್ನು ಸೂಚಿಸುತ್ತದೆ. LLM ತನ್ನ ಪೂರ್ವ ಚಟುವಟಿಕೆಗಳ ಮತ್ತು ಪ್ರಸ್ತುತ ಹಿನ್ನೆಲೆಯ ಸಂದರ್ಭಗಳನ್ನು ಬಳಸಿಕೊಂಡು ಮುಂದಿನ ಕ್ರಮಗಳನ್ನು ನಿರ್ಧರಿಸುತ್ತದೆ. AI ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್ ಗಳು ಈ ಸಂದರ್ಭದಲ್ಲಿ ನಿರ್ವಹಣೆಯನ್ನು ಸುಲಭಮಾಡುತ್ತವೆ.

**ಉಪಕರಣಗಳು** - ಬಳಕೆದಾರರು ಕೇಳಿದ ಕಾರ್ಯವನ್ನು LLM ಯೋಜಿಸಿದಂತೆ ಪೂರೈಸಲು LLM ಗೆ ಉಪಕರಣಗಳ ಪ್ರವೇಶ ಬೇಕಾಗುತ್ತದೆ. ಕೆಲ ಉದಾಹರಣೆಗೆ ಡೇಟಾಬೇಸ್, API, ಹೊರಗಿನ ಅಪ್ಲಿಕೇಶನ್ ಅಥವಾ ಇನ್ನೊಮ್ಮೆ ಮತ್ತೊಬ್ಬ LLM ಆಗಿರಬಹುದು!

ಈ ವ್ಯಾಖ್ಯಾನಗಳು ಮುಂದುವರೆಸುವಾಗ ಆಹಿತವಾದ ಆಧಾರವನ್ನು ಒದಗಿಸಲಿದೆ. ಈಗ ನಾವು ಕೆಲವು ವಿಭಿನ್ನ AI ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್ ಗಳನ್ನು ಅನ್ವೇಷಿಸೋಣ:

## ಲಾಂಗ್ಚೈನ್ ಏಜೆಂಟ್ ಗಳು

[ಲಾಂಗ್‌ಚೈನ್ ಏಜೆಂಟ್ ಗಳು](https://python.langchain.com/docs/how_to/#agents?WT.mc_id=academic-105485-koreyst) ಮೇಲ್ಕಂಡ ವ್ಯಾಖ್ಯಾನಗಳನ್ನು ಕೃತ್ರಿಮವಾಗಿ ಅನುಷ್ಠಾನಗೊಳಿಸುವುದಾಗಿದೆ.

**ಸ್ಥಿತಿಯನ್ನು** ನಿರ್ವಹಿಸಲು, ಇದು `AgentExecutor` ಎಂಬ ಒಳಗೊಂಡ ಕಾರ್ಯವನ್ನು ಬಳಸುತ್ತದೆ. ಇದು ನಿರ್ಣಯಿಸಲ್ಪಟ್ಟ `agent` ಮತ್ತು ಲಭ್ಯವಿರುವ `tools` ಅನ್ನು ಅಂಗೀಕರಿಸುತ್ತದೆ.

`AgentExecutor` ಜೊತೆಗೆ ಸಂವಾದ ಇತಿಹಾಸವನ್ನು ಸಂಗ್ರಹಿಸಿ ಸಂವಾದದ ಸಂದರ್ಭವನ್ನು ಒದಗಿಸುತ್ತದೆ.

![ಲಾಂಗ್‌ಚೈನ್ ಏಜೆಂಟ್ ಗಳು](../../../translated_images/kn/langchain-agents.edcc55b5d5c43716.webp)

ಲಾಂಗ್‌ಚೈನ್ [ಉಪಕರಣಗಳ ಸೂಚಿಯನ್ನು](https://integrations.langchain.com/tools?WT.mc_id=academic-105485-koreyst) ಒದಗಿಸುತ್ತದೆ, ಇದು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಆಮದುಮಾಡಬಹುದು. ಇವು ಸಮುದಾಯ ಮತ್ತು ಲಾಂಗ್‌ಚೈನ್ ತಂಡದಿಂದ ರಚಿಸಲ್ಪಟ್ಟವು.

ನೀವು ಈ ಉಪಕರಣಗಳನ್ನು ಪರಿಗಣಿಸಿ `AgentExecutor` ಗೆ ಹಸ್ತಾಂತರಿಸಬಹುದು.

ದೃಶ್ಯಪಟವು ಇನ್ನೊಂದು ಪ್ರಮುಖ ಅಂಶವಾಗಿದೆ כאשר AI ಏಜೆಂಟ್ ಗಳು ಕುರಿತು ಮಾತನಾಡುತ್ತಿರುವಾಗ. ಅಪ್ಲಿಕೇಶನ್ ಡೆವಲಪರ್ ಗಳು ಯಾವ ಉಪಕರಣವನ್ನು LLM ಬಳಕೆಮಾಡುತ್ತಿದೆ ಮತ್ತು ಯಾಕೆ ಬಳಸುತ್ತಿದೆ ಎಂದು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಮಹತ್ವದ್ದು. ಅದಕ್ಕಾಗಿ ಲಾಂಗ್‌ಚೈನ್ ತಂಡವು LangSmith ಅನ್ನು ಅಭಿವೃದ್ಧಿಪಡಿಸಿದೆ.

## AutoGen

ಮುಂದಿನ AI ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್ ನಾವು ಚರ್ಚಿಸುವುದು [AutoGen](https://microsoft.github.io/autogen/?WT.mc_id=academic-105485-koreyst). AutoGen ನ ಪ್ರಮುಖ ಗಮನವು ಸಂಭಾಷಣೆಗಳ ಮೇಲೆ ಇದೆ. ಏಜೆಂಟ್ ಗಳು ಕೂಡಲೇ **ಸಂಭಾಷಣಾಯೋಗ್ಯ** ಮತ್ತು **ಕಸ್ಟಮೈಜ್ ಆಗಬಹುದಾದವು**.

**ಸಂಭಾಷಣಾಯೋಗ್ಯ -** LLM ಗಳು ಇನ್ನೊಂದು LLM ಜೊತೆಗೆ ಸಂಭಾಷಣೆ ಆರಂಭಿಸಿ ಮುಂದುವರಿಸಬಹುದು ಕಾರ್ಯ ಪೂರೈಸುವ ಉದ್ದೇಶದಿಂದ. ಇದನ್ನು `AssistantAgents` ಸೃಷ್ಟಿಸುವ ಮೂಲಕ ಮತ್ತು ಅವರಿಗೆ ನಿರ್ದಿಷ್ಟ ವ್ಯವಸ್ಥೆ ಸಂದೇಶವನ್ನು ನೀಡುವ ಮೂಲಕ ಮಾಡಲಾಗುತ್ತದೆ.

```python

autogen.AssistantAgent( name="Coder", llm_config=llm_config, ) pm = autogen.AssistantAgent( name="Product_manager", system_message="Creative in software product ideas.", llm_config=llm_config, )

```

**ಕಸ್ಟಮೈಜ್ ಆಗಬಹುದಾದ** - ಏಜೆಂಟ್ ಗಳನ್ನು ಕೇವಲ LLM ಗಳು ಎಂಬುದಾಗಿ ಮಾತ್ರವಲ್ಲ, ಬಳಕೆದಾರ ಅಥವಾ ಉಪಕರಣವನ್ನಾಗಿ ಕೂಡ ಪ್ರಕಟಿಸಬಹುದು. ಡೆವಲಪರ್ ಆಗಿ ನೀವು `UserProxyAgent` ಅನ್ನು ನಿರ್ಧರಿಸಬಹುದು, ಇದು ಕಾರ್ಯವನ್ನು ಪೂರೈಸುವಲ್ಲಿ ಪ್ರತಿಕ್ರಿಯೆಯಾಗಿ ಬಳಕೆದಾರನೊಂದಿಗೆ ಸಂವಹನ ಮಾಡುವುದು. ಆ ಪ್ರತಿಕ್ರಿಯೆ ಕಾರ್ಯದ ನಿರ್ವಹಣೆಯನ್ನು ಮುಂದುವರೆಸಬಹುದು ಅಥವಾ ನಿಲ್ಲಿಸಬಹುದು.

```python
user_proxy = UserProxyAgent(name="user_proxy")
```

### ಸ್ಥಿತಿ ಮತ್ತು ಉಪಕರಣಗಳು

ಪರಿಸ್ಥಿತಿಯನ್ನು ಬದಲಾಯಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು, ಸಹಾಯಕ ಏಜೆಂಟ್ ಪೈಥಾನ್ ಕೋಡ್ ರಚಿಸಿ ಕಾರ್ಯವನ್ನು ಪೂರೈಸುತ್ತದೆ.

ಪ್ರಕ್ರಿಯೆಯ ಉದಾಹರಣೆ ಇಲ್ಲಿದೆ:

![AutoGen](../../../translated_images/kn/autogen.dee9a25a45fde584.webp)

#### ಸಿಸ್ಟಮ್ ಸಂದೇಶದಿಂದ ನಿರ್ಧಾರಗೊಳ್ಳಲಾದ LLM

```python
system_message="For weather related tasks, only use the functions you have been provided with. Reply TERMINATE when the task is done."
```

ಈ ವ್ಯವಸ್ಥೆ ಸಂದೇಶವು ನಿರ್ದಿಷ್ಟ LLM ಗೆ ಯಾವುದೇ ಕಾರ್ಯಗಳಿಗೆ ಸಂಬಂಧಿಸಿದ ಫಂಕ್ಷನ್ ಗಳನ್ನು ಸೂಚಿಸುತ್ತದೆ. AutoGen ನಲ್ಲಿ ನೀವು ಬಗೆಯ ಬಗೆಯ AssistantAgents ಗೆ ವಿಭಿನ್ನ ಸಿಸ್ಟಮ್ ಸಂದೇಶಗಳನ್ನು ಕೊಡುವಿರಿ.

#### ಬಳಕೆದಾರರಿಂದ ಸಂವಾದ ಪ್ರಾರಂಭ

```python
user_proxy.initiate_chat( chatbot, message="I am planning a trip to NYC next week, can you help me pick out what to wear? ", )

```

ಈ ಸಂದೇಶವು user_proxy(ಮಾನವ) ಬರುವುದು ಏಜೆಂಟ್ ಕಾರ್ಯವನ್ನು ಆರಂಭಿಸಿ ಕಾರ್ಯಗತವಾಗಬೇಕಾದ ಸಾಧ್ಯ ಫಂಕ್ಷನ್ ಗಳನ್ನು ಅನ್ವೇಷಿಸಲು ಪ್ರೇರಣೆ ನೀಡುತ್ತದೆ.

#### ಕಾರ್ಯ ನಿರ್ವಹಣೆ

```bash
chatbot (to user_proxy):

***** Suggested tool Call: get_weather ***** Arguments: {"location":"New York City, NY","time_periond:"7","temperature_unit":"Celsius"} ******************************************************** --------------------------------------------------------------------------------

>>>>>>>> EXECUTING FUNCTION get_weather... user_proxy (to chatbot): ***** Response from calling function "get_weather" ***** 112.22727272727272 EUR ****************************************************************

```

ಪ್ರಾಥಮಿಕ ಸಂವಾದ ಪ್ರಕ್ರಿಯೆಯ ನಂತರ, ಏಜೆಂಟ್ ಕರೆ ಮಾಡಲು ಸೂಚಿಸಿದ ಉಪಕರಣವನ್ನು ಕಳುಹಿಸುತ್ತದೆ. ಈ ಪ್ರಕರಣದಲ್ಲಿ `get_weather` ಎಂಬ ಫಂಕ್ಷನ್ ಆಗಿದೆ. ನಿಮ್ಮ ಕಾನ್ಫಿಗರೇಶನ್ ಪ್ರಕಾರ, ಈ ಕಾರ್ಯವನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ರವಾನಿಸಬಹುದು ಅಥವಾ ಬಳಕೆದಾರನ ಇನ್ಪುಟ್ ಆಧಾರಿತವಾಗಿ ನಿರ್ವಹಿಸಬಹುದು.

ನೀವು ಇನ್ನಷ್ಟು ಆರಂಭಿಸಲು [AutoGen ಕೋಡ್ ನಿದರ್ಶನಗಳ](https://microsoft.github.io/autogen/docs/Examples/?WT.mc_id=academic-105485-koreyst) ಪಟ್ಟಿಯನ್ನು ಪರಿಶೀಲಿಸಬಹುದು.

## ಮೈಕ್ರೋಸಾಫ್ಟ್ ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್

[ಮೈಕ್ರೋಸಾಫ್ಟ್ ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್](https://learn.microsoft.com/agent-framework/?WT.mc_id=academic-105485-koreyst) ಮೈಕ್ರೋಸಾಫ್ಟ್ ನ ಹೊರಸೂತ್ರ SDK ಆಗಿದ್ದು Python ಮತ್ತು .NET ನಲ್ಲಿ AI ಏಜೆಂಟ್ ಮತ್ತು ಬಹು ಏಜೆಂಟ್ ವ್ಯವಸ್ಥೆಗಳ ನಿರ್ಮಾಣಕ್ಕೆ. ಇದು ಎರಡು ಹಿಂದಿನ ಮೈಕ್ರೋಸಾಫ್ಟ್ ಯೋಜನೆಗಳ ಶಕ್ತಿ ಸಂಯೋಜಿಸುತ್ತದೆ — **Semantic Kernel** ನ ಎಂಟರ್‌ಪ್ರೈಸ್ ವೈಶಿಷ್ಟ್ಯಗಳು ಮತ್ತು **AutoGen** ನ ಬಹು ಏಜೆಂಟ್ ಸಂಯೋಜನೆ — ಒಂದೇ, ಬೆಂಬಲಿತ ಫ್ರೇಮ್ವರ್ಕ್ ಆಗಿ. ನೀವು ಇಂದೇ ಹೊಸ ಏಜೆಂಟ್ ಯೋಜನೆ ಪ್ರಾರಂಭಿಸುತ್ತಿದ್ದರೆ, ಇದು AutoGen ನ ಶಿಫಾರಸು ಮಾಡಿದ ಮುಂದುವರಿಕೆ.

ಈ ಫ್ರೇಮ್ವರ್ಕ್ ಒಂದು ಏಜೆಂಟ್ ಸಂವಾದದಿಂದ ಬಹು ಏಜೆಂಟ್ ಕಾರ್ಯಪ್ರವಾಹಗಳ ತನಕ ವಿಸ್ತಾರಗೊಳ್ಳುತ್ತದೆ ಮತ್ತು સીಧಾಗ Microsoft Foundry, Azure OpenAI ಮತ್ತು OpenAI ಜೊತೆಗೆ ಸಂಯೋಜಿಸಲಾಗಿದೆ. ಇದು OpenTelemetry ಮೂಲಕ ನೇರಃಪರೀಕ್ಷಣೆಯನ್ನು ಒದಗಿಸುತ್ತದೆ, ಆದ್ದರಿಂದ ನಿಮ್ಮ ಏಜೆಂಟ್ ಗಳು ಏನು ಮಾಡುತ್ತಿರುವೆಯೋ ನಿಖರವಾಗಿ ಪತ್ತೆಹಚ್ಚಬಹುದು.

### ಸ್ಥಿತಿ ಮತ್ತು ಉಪಕರಣಗಳು

**ಸ್ಥಿತಿ** - ಫ್ರೇಮ್ವರ್ಕ್ ಸಂವಾದದ ಸಂದರ್ಭವನ್ನು **ಥ್ರೆಡ್‌ಗಳು** ಮೂಲಕ ನಿರ್ವಹಿಸುತ್ತದೆ. ಏಜೆಂಟ್ ಸಂದೇಶ ಇತಿಹಾಸವನ್ನು (ಬಳಕೆದಾರ ವಿನಂತಿಗಳು, ಉಪಕರಣ ಕರೆಗಳು, ಅವುಗಳ ಫಲಿತಾಂಶಗಳು) ಜನರೇಟ್ ಮಾಡುತ್ತದೆ, ಹೀಗಾಗಿ ಪ್ರತಿಯೊಂದು ತಿರುವು ಮುನ್ನವರ ಮೇಲೆ ಕಟ್ಟಿಕೊಳ್ಳುತ್ತದೆ. ಥ್ರೆಡ್‌ಗಳನ್ನು ಸಂರಕ್ಷಿಸಬಹುದು, ಸಂವಾದವನ್ನು ವಿರಾಮಮಾಡಿ ಹಿಂತಿರುಗಿ ಮುಂದುವರೆಸಬಹುದು.

**ಉಪಕರಣಗಳು** - ನೀವು ಏಜೆಂಟ್ ಗೆ ಸರಳ Python ಕಾರ್ಯಗಳನ್ನು ನೀಡುತ್ತೀರಿ. ಪ್ರಕಾರ ವರ್ಣಿಸಲಾಗಿದ ಪೆರാമೀಟರ್ಗಳು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಸಂದರ್ಶಿಸಲ್ಪಡುವ ರೂಪದಲ್ಲಿ ಪರಿವರ್ತಿಸಲಾಗುತ್ತದೆ, ಆದ್ದರಿಂದ ಮಾದರಿ ಅವುಗಳನ್ನು ಹೇಗೆ ಮತ್ತು ಯಾವಾಗ ಕರೆ ಮಾಡಬೇಕೆಂದು ಅರಿತುಕೊಳ್ಳುತ್ತದೆ (ಕಾರ್ಯ ಕರೆ). ಫ್ರೇಮ್ವರ್ಕ್ Model Context Protocol (MCP) ಸರ್ವರ್ ಗಳು ಮತ್ತು ಕೋಡ್ ಇಂಟರ್ಪ್ರೆಟರ್ ಹೋಲಿದ ಉಪಕರಣಗಳನ್ನು ಸಹ ಬೆಂಬಲಿಸುತ್ತದೆ.

ಇಲ್ಲಿ ವಿಶೇಷ ಉಪಕರಣವೊಂದೊಂದನ್ನು ಹೊಂದಿದ ಏಜೆಂಟ್ ನ ಉದಾಹರಣೆ ಇದೆ:

```python
import asyncio
from typing import Annotated

from pydantic import Field
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient


def get_weather(
    location: Annotated[str, Field(description="The location to get the weather for.")],
) -> str:
    """Get the weather for a given location."""
    return f"The weather in {location} is sunny with a high of 22°C."


async def main():
    agent = Agent(
        client=OpenAIChatClient(),
        instructions="You are a helpful assistant that can answer weather questions.",
        tools=[get_weather],
    )

    response = await agent.run("What's the weather in Amsterdam?")
    print(response)


asyncio.run(main())
```

ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡ್ರಿಯಲ್ಲಿ Azure OpenAI ಗೆ ಸಂಪರ್ಕಿಸಲು ನಿಮ್ಮ ಎಂಡ್‌ಪಾಯಿಂಟ್ ಮತ್ತು ಪ್ರಮಾಣಪತ್ರಗಳನ್ನು ಕ್ಲೈಂಟ್ ಗೆ ನೀಡಿ:

```python
from azure.identity.aio import AzureCliCredential
from agent_framework.openai import OpenAIChatClient

client = OpenAIChatClient(
    model="my-gpt-4o-deployment",
    azure_endpoint="https://my-resource.openai.azure.com",
    credential=AzureCliCredential(),
)
```

### ಬಹು ಏಜೆಂಟ್ ಕಾರ್ಯಪ್ರವಾಹಗಳು

ಫ್ರೇಮ್ವರ್ಕ್ ನ ವಿಶೇಷತೆ ಹಲವಾರು ಏಜೆಂಟ್ ಗಳನ್ನು ಒಟ್ಟಾಗಿ ಸಂಯೋಜಿಸುವುದಲ್ಲಿ ಇದೆ. ಉದಾಹರಣೆಗೆ, ನೀವು ಏಜೆಂಟ್ ಗಳನ್ನು ಸರಣಿಯಾಗಿ (ಪ್ರತಿಯೊಂದು ತಮ್ಮ ಸಂದರ್ಭವನ್ನು ಮುಂದಿನಗೆ ಹಂಚಿಕೊಂಡು) ಚಾಲನೆ ಮಾಡಬಹುದು ಅಥವಾ ಅನೇಕರ ಏಜೆಂಟ್ ಗಳಿಗೆ ಸಮಕಾಲೀನವಾಗಿ ಹಾರಿಸಿಕೊಂಡು ಫಲಿತಾಂಶಗಳನ್ನು ಸಂಗ್ರಹಿಸಬಹುದು:

```python
from agent_framework.orchestrations import SequentialBuilder, ConcurrentBuilder

# ಸರಣಿಯಲ್ಲಿ ಏಜೆಂಟ್‌ಗಳನ್ನು ಚಾಲನೆ ಮಾಡಿ, razgovvaartana ಸಂದರ್ಭವನ್ನು ಸರಣಿಯಲ್ಲಿ ಹಂಚಿಕೊಳ್ಳಿ
sequential = SequentialBuilder(participants=[researcher, writer, editor]).build()

# ಏಜೆಂಟ್‌ಗಳಿಗೆ ಪಾರллಿಯಲ್ಲಿ ವಿಸ್ತರಿಸಿ, ನಂತರ ಅವರಿಗೆಲ್ಲಾ ಉತ್ತರಗಳನ್ನು ಸಂಗ್ರಹಿಸಿ
concurrent = ConcurrentBuilder(participants=[analyst_a, analyst_b, analyst_c]).build()
```

ಫ್ರೇಮ್ವರ್ಕ್ ಇನ್ಸ್ಟಾಲ್ ಮಾಡಿ ಪ್ರಾರಂಭಿಸಲು:

```bash
pip install agent-framework-core
# ಐಚ್ಛಿಕ ಸಂಯೋಜನೆಗಳು
pip install agent-framework-openai       # ಓಪನ್‌ಎಐ ಮತ್ತು ಅಜ್ಯುರ್ ಓಪನ್‌ಎಐ
pip install agent-framework-foundry      # ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡ್ರಿ
```

ನೀವು [ಮೈಕ್ರೋಸಾಫ್ಟ್ ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್ ರೆಪೊ](https://github.com/microsoft/agent-framework?WT.mc_id=academic-105485-koreyst) ಮತ್ತು [ಅಧಿಕೃತ ಡಾಕ್ಯುಮೆಂಟೇಶನ್](https://learn.microsoft.com/agent-framework/?WT.mc_id=academic-105485-koreyst) ಇನ್ನುನಷ್ಟು ಅನ್ವೇಷಿಸಬಹುದು.

## ಟಾಸ್ಕ್ವೀವರ್

ಮುಂದಿನ ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್ ನಾವು ಅನ್ವೇಷಿಸುವುದು [Taskweaver](https://microsoft.github.io/TaskWeaver/?WT.mc_id=academic-105485-koreyst). ಇದನ್ನು "ಕೋಡ್-ಪ್ರಥಮ" ಏಜೆಂಟ್ ಎಂದು ಕರೆಸುತ್ತಾರೆ ಏಕೆಂದರೆ ಇದು ಕೇವಲ `strings` ನೊಂದಿಗೆ ಮಾತ್ರ ಕೆಲಸ ಮಾಡುವುದಿಲ್ಲ, Python ಡೇಟಾಫ್ರೇಮ್ ಗಳೊಂದಿಗೆ ಕೂಡ ಕೆಲಸ ಮಾಡುತ್ತದೆ. ಇದು ಡೇಟಾ ವಿಶ್ಲೇಷಣೆ ಮತ್ತು ಸೃಷ್ಟಿ ಕಾರ್ಯಗಳಲ್ಲಿ ಅತ್ಯಂತ ಉಪಯುಕ್ತವಾಗುತ್ತದೆ. ಉದಾಹರಣೆಗೆ ಗ್ರಾಫ್ ಅಥವಾ ಚಾರ್ಟ್ ರಚನೆ ಅಥವಾ ರ್ಯಾಂಡಮ್ ಸಂಖ್ಯೆ ಸೃಷ್ಟಿ.

### ಸ್ಥಿತಿ ಮತ್ತು ಉಪಕರಣಗಳು

ಸಂವಾದ ಸ್ಥಿತಿಯನ್ನು ನಿರ್ವಹಿಸಲು, TaskWeaver `Planner` ಎಂಬ ಸಿದ್ಧಾಂತವನ್ನು ಬಳಕೆಯಲ್ಲಿಡುತ್ತದೆ. `Planner` LLM ಆಗಿದ್ದು ಬಳಕೆದಾರರ ವಿನಂತಿಯನ್ನು ತೆಗೆದುಕೊಂಡು ಈ ವಿನಂತಿ ಪೂರೈಸಲು ಬೇಕಾಗುವ ಕಾರ್ಯಗಳನ್ನು ಹಂತ ಹಂತವಾಗಿ ಪ್ರಣಾಳಿಕೆಗೆ ಹಿಂತಿರುಗಿಸುತ್ತದೆ.

ಕಾರ್ಯಗಳನ್ನು ಪೂರ್ಣಗೊಳಿಸಲು `Planner` ಉಪಕರಣಗಳ ಸಂಕಲನ `Plugins` ಗೆ ಪ್ರವೇಶ ಇದೆ. ಇದು Python ತರಗತಿಗಳು ಅಥವಾ ಸಾಮಾನ್ಯ ಕೋಡ್ ಇಂಟರ್ಪ್ರೆಟರ್ ಆಗಿರಬಹುದು. ಈ ಪ್ಲಗಿನ್ ಗಳು ಎಂಬೆಡಿಂಗ್ ಗಳಾಗಿ ಸಂಗ್ರಹಿಸಲಾಗುತ್ತವೆ ಹೀಗಾಗಿ LLM ನಿಗೆ ಸರಿಯಾದ ಪ್ಲಗಿನ್ ಹುಡುಕಲು ಸಾಧ್ಯ.

![ಟಾಸ್ಕ್ವೀವರ್](../../../translated_images/kn/taskweaver.da8559999267715a.webp)

ಅನೋಮಲಿ ಪತ್ತೆಹಚ್ಚಲು ಪ್ಲಗಿನ್ ಒಂದು ಉದಾಹರಣೆ ಇಲ್ಲಿ:

```python
class AnomalyDetectionPlugin(Plugin): def __call__(self, df: pd.DataFrame, time_col_name: str, value_col_name: str):
```

ನಿರ್ವಹಿಸುವ ಮೊದಲು ಕೋಡ್ ಪರಿಶೀಲನೆ ಮಾಡಲಾಗುತ್ತದೆ. Taskweaver ನಲ್ಲಿ ಸಂವಾದದ ಹಿನ್ನೆಲೆಯನ್ನು ನಿರ್ವಹಿಸಲು ಮತ್ತೊಂದು ಲಕ್ಷಣವಾಗಿದೆ `ಅನುಭವ`. ಅನುಭವ YAML ಕಡತದಲ್ಲಿ ಸಂವಾದದ ಸಂದರ್ಭವನ್ನು ದೀರ್ಘಕಾಲ ಸಂಗ್ರಹಿಸಲು ಅನುಮತಿಸುತ್ತದೆ. ಇದು ಸಂರಚಿಸಲಾಗಬಹುದು ಹೀಗಾಗಿ LLM ಒಂದು ನಿರ್ದಿಷ್ಟ ಕಾರ್ಯಗಳಲ್ಲಿ ಕಾಲಕ್ರಮೇಣ ಸುಧಾರಣೆ ಹೊಂದಿಸುತ್ತದೆ ಹಿಂದಿನ ಸಂವಾದಗಳಿಂದ ತಿಳಿದುಕೊಳ್ಳುವ ಮೂಲಕ.

## ಜಾರ್‌ವಿಸ್

ಕೊನೆಗೆ ನಾವು ಅನ್ವೇಷಿಸುವ ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್ [JARVIS](https://github.com/microsoft/JARVIS?tab=readme-ov-file&WT.mc_id=academic-105485-koreyst). JARVIS ನ ವಿಶೇಷತೆಯಾಗಿದೆ ಇದು ಸಂವಾದದ `ಸ್ಥಿತಿ` ನಿರ್ವಹಿಸಲು LLM ಅನ್ನು ಬಳಸುವುದು ಮತ್ತು `ಉಪಕರಣಗಳು` ಇತರ AI ಮಾದರಿಗಳು. ಪ್ರತಿಯೊಂದು AI ಮಾದರಿ ನಿರ್ದಿಷ್ಟ ಕಾರ್ಯಗಳನ್ನು ನಿರ್ವಹಿಸುತ್ತವೆ, ಹೀಗೆ ವಸ್ತು ಪತ್ತೆ, ಲಿಪ್ಯಂತರಣೆ ಅಥವಾ ಚಿತ್ರ ವಿವರಣೆ.

![ಜಾರ್‌ವಿಸ್](../../../translated_images/kn/jarvis.762ddbadbd1a3a33.webp)

ಸಾಮಾನ್ಯ ಉದ್ದೇಶದ ಮಾದರಿ ಆಗಿರುವ LLM ಬಳಕೆದಾರನಿಂದ ವಿನಂತಿಯನ್ನು ಸ್ವೀಕರಿಸಿ ನಿರ್ದಿಷ್ಟ ಕಾರ್ಯ ಮತ್ತು ಪೂರ್ಣಗೊಳಿಸಲು ಅಗತ್ಯವಿರುವ ಯುಕ್ತಿಗಳು/ಡೇಟಾವನ್ನು ಗುರುತಿಸುತ್ತದೆ.

```python
[{"task": "object-detection", "id": 0, "dep": [-1], "args": {"image": "e1.jpg" }}]
```

ನಂತರ LLM ಆ ವಿನಂತಿಯನ್ನು ವಿಶೇಷ AI ಮಾದರಿ ಎಂದು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವ ರೀತಿಯಲ್ಲಿ ರೂಪಾಂತರಿಸುತ್ತದೆ, ಉದಾಹರಣೆಗೆ JSON ನಲ್ಲಿ. AI ಮಾದರಿ ಕಾರ್ಯದ ಆಧಾರದ ಮೇಲೆ ಪೂರ್ವಾನುಗುಣನೆಗೆ ಬನ್ನಿ ನಂತರ LLM ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ಸ್ವೀಕರಿಸುತ್ತದೆ.

ಒಂದು ಕಾರ್ಯವನ್ನು ಪೂರ್ಣಗೊಳಿಸಲು ಅನೇಕ ಮಾದರಿಗಳು ಬೇಕಾದರೆ, ಅವುಗಳ ಪ್ರತಿಕ್ರಿಯೆಗಳು ಸಹ ಅರ್ಥೈಸಿ ಬಳಕೆದಾರಿಗೆ ಪ್ರತಿಕ್ರಿಯೆಯನ್ನು ರಚಿಸುವುದಕ್ಕೆ ಒಂದಾಗಿದೆ.

ಕೆಳಗಿನ ಉದಾಹರಣೆಯು ಬಳಕೆದಾರ ಚಿತ್ರದಲ್ಲಿರುವ ವಸ್ತುಗಳ ವಿವರ ಮತ್ತು ಎಣಿಕೆ ಕೇಳುತ್ತಿರುವಾಗ ಹೇಗೆ ಕೆಲಸ ಮಾಡುತ್ತದೆ ಎಂಬುದನ್ನು ತೋರಿಸುತ್ತದೆ:

## ಹುದ್ದೆ

AI ಏಜೆಂಟ್ ಗಳ ಕಲಿಕೆಯನ್ನು ಮುಂದುವರೆಸಲು ನೀವು ಮೈಕ್ರೋಸಾಫ್ಟ್ ಏಜೆಂಟ್ ಫ್ರೇಮ್ವರ್ಕ್ ಬಳಸಿ ನಿರ್ಮಿಸಬಹುದು:

- ಶಿಕ್ಷಣ ಸ್ಟಾರ್ಟಪ್‌ನ ವಿಭಿನ್ನ ವಿಭಾಗಗಳೊಂದಿಗೆ ವ್ಯವಹಾರ ಸಭೆಯನ್ನು ಅನುಕರಿಸುವ ಅಪ್ಲಿಕೇಶನ್.
- LLM ಗಳಿಗೆ ವಿಭಿನ್ನ ವ್ಯಕ್ತಿತ್ವಗಳು ಮತ್ತು ಪ್ರಾಥಮ್ಯಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಮಾರ್ಗದರ್ಶಿಸುವ ವ್ಯವಸ್ಥೆ ಸಂದೇಶಗಳನ್ನು ರಚಿಸಿ ಬಳಕೆದಾರರನ್ನು ಹೊಸ ಉತ್ಪನ್ನದ ಸಂಪ್ರೇಷ್ಟನೆ ಮಾಡಲು ಅನುಮತಿಸುವುದು.
- ನಂತರ ಪ್ರತಿಯೊಂದು ವಿಭಾಗದಿಂದ ಹೆಚ್ಚಿನ ಪ್ರಶ್ನೆಗಳನ್ನು ರಚಿಸುವಂತೆ LLM ಕಾರ್ಯಗತಗೊಳಿಸಬೇಕು, ಹೀಗೆ ಸಂಪ್ರೇಷ್ಟನೆ ಮತ್ತು ಉತ್ಪನ್ನದ ಕಲ್ಪನೆಯನ್ನು ಸುಧಾರಿಸಲು.

## ಕಲಿಕೆಯು ಇಲ್ಲಿ ನಿಲ್ಲದು, ಪ್ರಯಾಣವನ್ನು ಮುಂದುವರೆಸಿರಿ

ಈ ಪಾಠ ಪೂರ್ಣಗೊಳಿಸಿದ ನಂತರ, ನಮ್ಮ [ಜನರೇಟಿವ್ AI ಕಲಿಕಾ ಸಂಗ್ರಹ](https://aka.ms/genai-collection?WT.mc_id=academic-105485-koreyst) ಅನ್ನು ಪರಿಶೀಲಿಸಿ ನಿಮ್ಮ ಜನರೇಟಿವ್ AI ಜ್ಞಾನವನ್ನು ಮತ್ತಷ್ಟು ಸುಧಾರಿಸಿ!
