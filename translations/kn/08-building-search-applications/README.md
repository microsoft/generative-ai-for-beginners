LLM ಗಳು ಚಾಟ್‌ಬಾಟ್‌ಗಳು ಮತ್ತು ಪಠ್ಯ ಉತ್ಪಾದನೆಗಿಂತ ಹೆಚ್ಚಿನದ್ದು. ಎಂಬೆಡಿಂಗ್‌ಗಳನ್ನು ಬಳಸಿ ಹುಡುಕುವ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುವುದನ್ನೂ ಸಾಧ್ಯ. ಎಂಬೆಡಿಂಗ್‌ಗಳು ಡೇಟಾವಿನ ಸಂಖ್ಯಾತ್ಮಕ ಪ್ರತಿನಿಧಾನಗಳು, ಇವರನ್ನು ವೆಕ್ಟರ್‌ಗಳೆಂದು ಕರೆಯಲಾಗುತ್ತದೆ ಮತ್ತು ಡೇಟಾ ಗೆ ಸೆಮಾಂಟಿಕ್ ಹುಡುಕಾಟಕ್ಕಾಗಿ ಬಳಸಬಹುದು.

ಈ ಪಾಠದಲ್ಲಿ, ನಾವು ನಮ್ಮ ಶಿಕ್ಷಣ ಸ್ಟಾರ್ಟ್‌ಅಪ್‌ಗೆ ಹುಡುಕುವ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ನಿರ್ಮಿಸುತ್ತೇವೆ. ನಮ್ಮ ಸ್ಟಾರ್ಟ್‌ಅಪ್ વિકાસಶೀಲ ದೇಶಗಳ ವಿದ್ಯಾರ್ಥಿಗಳಿಗೆ ಉಚಿತ ಶಿಕ್ಷಣವನ್ನು ನೀಡುವ ಲಾಭರಹಿತ ಸಂಘಟನೆ. ನಮ್ಮ ಸ್ಟಾರ್ಟ್‌ಅಪ್‌ಗೆ ವಿದ್ಯಾರ್ಥಿಗಳು AI ಕುರಿತು ಕಲಿತುಕೊಳ್ಳಲು ಬಳಸಬಹುದಾದ ಅನೇಕ YouTube ವೀಡಿಯೊಗಳಿವೆ. ವಿದ್ಯಾರ್ಥಿಗಳು ಪ್ರಶ್ನೆಯನ್ನು ಟೈಪ್‌ ಮಾಡಿ YouTube ವೀಡಿಯೊವನ್ನು ಹುಡುಕಲು ಅನುಮತಿಸುವ ಹುಡುಕುವ ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸಲು ನಮ್ಮ ಸ್ಟಾರ್ಟ್‌ಅಪ್ ಬಯಸುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ, ವಿದ್ಯಾರ್ಥಿ 'ಜ್ಯುಪಿಟರ್ ನೋಟ್ಬುಕ್ಸ್ ಎಂದ್ರೆ ಏನು?' ಅಥವಾ 'ಆಜೂರ್ ML ಎಂದರೆ ಏನು' ಎಂದು ಟೈಪ್ ಮಾಡಿದರೆ, ಹುಡುಕಾಟ ಅಪ್ಲಿಕೇಶನ್ ಅದಕ್ಕೆ ಸಾಂಕೇತಿಕ ಸಂಬಂಧಿತ YouTube ವೀಡಿಯೊಗಳ ಪಟ್ಟಿಯನ್ನು ಹಿಂತಿರುಗಿಸುತ್ತದೆ; ಇನ್ನಷ್ಟು ಉತ್ತಮವಾಗಿ, ಅದು ಪ್ರಶ್ನೆಯ ಉತ್ತರವಿರುವ ವೀಡಿಯೊಯಲ್ಲಿ ಸಂಬಂಧಿತ ಸ್ಥಳಕ್ಕೆ ಲಿಂಕ್ ನೀಡುತ್ತದೆ.

## ಪರಿಚಯ

ಈ ಪಾಠದಲ್ಲಿ ನಾವು ಒಳಗೊಂಡಿರುವವು:

- ಸೆಮಾಂಟಿಕ್ ಮತ್ತು ಕೀವರ್ಡ್ ಹುಡುಕಾಟದ ವ್ಯತ್ಯಾಸ.
- ಪಠ್ಯ ಎಂಬೆಡಿಂಗ್‌ಗಳು ಎಂದರೆ ಏನು.
- ಪಠ್ಯ ಎಂಬೆಡಿಂಗ್ ಸೂಚ್ಯಂಕವನ್ನು ರಚಿಸುವ ವಿಧಾನ.
- ಪಠ್ಯ ಎಂಬೆಡಿಂಗ್ ಸೂಚ್ಯಂಕದಲ್ಲಿ ಹುಡುಕಾಟ.

## ಅಧ್ಯಯನ ಗುರಿಗಳು

ಈ ಪಾಠವನ್ನು ಮುಗಿಸಿದ ಮೇಲೆ, ನೀವು ಸಾಧ್ಯವಾಗುವುದು:

- ಸೆಮಾಂಟಿಕ್ ಮತ್ತು ಕೀವರ್ಡ್ ಹುಡುಕಾಟದ ವ್ಯತ್ಯಾಸವನ್ನು ಹೇಳುವುದು.
- ಪಠ್ಯ ಎಂಬೆಡಿಂಗ್‌ಗಳೇನು ಎಂಬುದನ್ನು ವಿವರಿಸುವುದು.
- ಎम्बೆಡಿಂಗ್ ಬಳಸಿ ಡೇಟಾ ಹುಡುಕಲು ಅಪ್ಲಿಕೇಶನ್ ರಚಿಸುವುದು.

## ಹುಡುಕುವ ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸುವ ಕಾರಣವೇನು?

ಹುಡುಕುವ ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸುವುದು, ಎಂಬೆಡಿಂಗ್‌ಗಳನ್ನು ಉಪಯೋಗಿಸಿ ಡೇಟಾ ಹುಡುಕುವ ವಿಧಾನವನ್ನು ತಿಳಿದುಕೊಳ್ಳಲು ಸಹಾಯವಾಗುತ್ತದೆ. ನೀವು ಸಹ ವಿದ್ಯಾರ್ಥಿಗಳು ಮಾಹಿತಿಯನ್ನು ತ್ವರಿತವಾಗಿ ಹುಡುಕಲು ಬಳಸಬಹುದಾದ ಹುಡುಕಾಟ ಅಪ್ಲಿಕೇಶನ್ ರೂಪಿಸುವುದನ್ನು ಕಲಿಯುತ್ತೀರಿ.

ಈ ಪಾಠದಲ್ಲಿ, Microsoft [AI Show](https://www.youtube.com/playlist?list=PLlrxD0HtieHi0mwteKBOfEeOYf0LJU4O1) YouTube ಚಾನಲಿನ ವೀಡಿಯೊ ಭಾಷಾಂತರಗಳ ಎम्बೆಡಿಂಗ್ ಸೂಚ್ಯಂಕವೂ ಸೇರಿದೆ. AI Show ಒಂದು YouTube ಚಾನಲ್ ಆಗಿದ್ದು, AI ಮತ್ತು ಮಷೀನ್‌ ಲರ್ನಿಂಗ್ ಕುರಿತು ಬೋಧಿಸುತ್ತದೆ. ಎಂಬೆಡಿಂಗ್ ಸೂಚ್ಯಂಕವು 2023 ರ ಅಕ್ಟೋಬರ್ ವರೆಗೆ ಪ್ರತಿಯೊಂದು YouTube ಭಾಷಾಂತರ ವೀಡಿಯೊಗಳ ಎಂಬೆಡಿಂಗ್‌ಗಳನ್ನು ಒಳಗೊಂಡಿದೆ. ನೀವು ನಮ್ಮ ಸ್ಟಾರ್ಟ್‌ಅಪ್‌ಗಾಗಿ ಹುಡುಕಾಟ ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸಲು ಈ ಸೂಚ್ಯಂಕವನ್ನು ಬಳಸುತ್ತೀರಿ. ಹುಡುಕಾಟ ಅಪ್ಲಿಕೇಶನ್ ಪ್ರಶ್ನೆಯ ಉತ್ತರವಿರುವ ವೀಡಿಯೊನಲ್ಲಿನ ಸ್ಥಳಕ್ಕೆ ಲಿಂಕ್ ನೀಡುತ್ತದೆ. ಇದು ವಿದ್ಯಾರ್ಥಿಗಳಿಗೆ ತ್ವರಿತವಾಗಿ ಬೇಕಾದ ಮಾಹಿತಿಯನ್ನು ಹುಡುಕಲು ಉತ್ತಮ ಮಾರ್ಗ.

## ಸೆಮಾಂಟಿಕ್ ಹುಡುಕಾಟ ಎಂದರೆ ಏನು?

ಈಗ ನೀವು ಆಶ್ಚರ್ಯಪಡುವಿರೇನು, ಸೆಮಾಂಟಿಕ್ ಹುಡುಕಾಟ ಎಂದರೆ ಏನು? ಸೆಮಾಂಟಿಕ್ ಹುಡುಕಾಟವು ಒಂದು ತಂತ್ರಜ್ಞಾನವಾಗಿದ್ದು, ಅದು ಪ್ರಶ್ನೆಯಲ್ಲಿನ ಪದಗಳ ಅರ್ಥವನ್ನು ಉಪಯೋಗಿಸಿ ಸಂಬಂಧಿಸಿದ ಫಲಿತಾಂಶಗಳನ್ನು ಹಿಂತಿರುಗಿಸುತ್ತದೆ.

ಒಂದು ಉದಾಹರಣೆ ಇಲ್ಲಿದೆ. ನೀವು ಕಾರು ಖರೀದಿಸಲು ಹುಡುಕುತ್ತಿದ್ದರೆ, 'ನನ್ನ ಕನಸು ಕಾರು' ಎಂದು ಹುಡುಕಬಹುದು; ಸೆಮಾಂಟಿಕ್ ಹುಡುಕಾಟವು ನೀವು ಕಾರಿನ ಕನಸು ಕಾಣುತ್ತಿರುವುದಲ್ಲ, ನಿಮ್ಮ `ಆದರ್ಶ` ಕಾರು ಖರೀದಿಸಲು ಬಯಸುತ್ತಿರುವುದನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುತ್ತದೆ. ಸೆಮಾಂಟಿಕ್ ಹುಡುಕಾಟ ನಿಮ್ಮ ಉದ್ದೇಶ ತಿಳಿದುಕೊಂಡು ಸಂಬಂಧಿತ ಫಲಿತಾಂಶಗಳನ್ನು ನೀಡುತ್ತದೆ. ಪರ್ಯಾಯವಾಗಿ, 'ಕೀವರ್ಡ್ ಹುಡುಕಾಟ'ವು literal ಆಗಿ ಕನಸು ಕಾರುಗಳ ಬಗ್ಗೆ ಹುಡುಕಿ ಅನಗತ್ಯ ಫಲಿತಾಂಶಗಳನ್ನು ನೀಡಬಹುದು.

## ಪಠ್ಯ ಎಂಬೆಡಿಂಗ್‌ಗಳು ಎಂದರೆ ಏನು?

[ಪಠ್ಯ ಎಂಬೆಡಿಂಗ್‌ಗಳು](https://en.wikipedia.org/wiki/Word_embedding?WT.mc_id=academic-105485-koreyst) [ನೈಸರ್ಗಿಕ ಭಾಷಾ ಪ್ರಕ್ರಿಯೆ](https://en.wikipedia.org/wiki/Natural_language_processing?WT.mc_id=academic-105485-koreyst)ಯಲ್ಲಿ ಬಳಸುವ ಪಠ್ಯ ಪ್ರತಿನಿಧಾನದ ತಂತ್ರ. ಪಠ್ಯ ಎಂಬೆಡಿಂಗ್‌ಗಳು ಪಠ್ಯದ ಸಾಂಕೇತಿಕ ಸಂಖ್ಯಾತ್ಮಕ ಪ್ರತಿನಿಧಾನಗಳು. ಎಂಬೆಡಿಂಗ್‌ಗಳನ್ನು ಯಂತ್ರಗಳಿಗಾಗಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಸುಲಭವಾಗಿ ಡೇಟಾವನ್ನು ಪ್ರತಿನಿಧಿಸಲು ಉಪಯೋಗಿಸಲಾಗುತ್ತದೆ. ಪಠ್ಯ ಎಂಬೆಡಿಂಗ್‌ಗಳನ್ನು ನಿರ್ಮಿಸಲು ಹಲವಾರು ಮಾದರಿಗಳು ಇವೆ, ಈ ಪಾಠದಲ್ಲಿ ನಾವು OpenAI ಎಂಬೆಡಿಂಗ್ ಮಾದರಿಯನ್ನು ಉಪಯೋಗಿಸಿ ಎಂಬೆಡಿಂಗ್‌ಗಳನ್ನು ಉತ್ಪಾದಿಸುವುದರ ಮೇಲೆ ಗಮನಹರಿಸುತ್ತೇವೆ.

ಉದಾಹರಣೆಗೆ, ಕೆಳಗಿನ ಪಠ್ಯ AI Show YouTube ಚಾನಲಿನ ಒಂದು ಎಪಿಸೋಡ್‌ನ ಭಾಷಾಂತರದಲ್ಲಿ ಇರಬಹುದು ಎಂದು ಕಲ್ಪಿಸಿಕೊಳ್ಳಿ:

3. ನಂತರ ಭಾಷಾಂತರ ಪಠ್ಯವನ್ನು **3 ನಿಮಿಷಗಳ ಪಠ್ಯ ಭಾಗಗಳಾಗಿ** ವಿಭಜಿಸಲಾಗಿದೆ. ಭಾಗವು ಮುಂದೆಯದ ಭಾಗದಿಂದ ಸುಮಾರು 20 ಪದಗಳ ಅಡ್ಡಪದಗಳನ್ನು ಒಳಗೊಂಡಿರುತ್ತದೆ, ಇದರಿಂದ ಭಾಗದ ಎಂಬೆಡಿಂಗ್ ಕಡಿತವಾಗದೆ ಉತ್ತಮ ಹುಡುಕಾಟ ಸಾಂದರ್ಭಿಕತೆ ಒದಗಿಸುತ್ತದೆ.
4. ಪ್ರತಿ ಪಠ್ಯ ಭಾಗವನ್ನು OpenAI ಚಾಟ್ API ಗೆ ಹೋರೆದು 60 ಪದಗಳ ಸಾರಾಂಶವನ್ನು ಪಡೆಯಲಾಗಿದೆ. ಸಾರಾಂಶವು ಕೂಡ ಎಂಬೆಡಿಂಗ್ ಸೂಚ್ಯಂಕ `embedding_index_3m.json` ನಲ್ಲಿ ಸಂಗ್ರಹಿಸಲಾಗಿದೆ.
5. ಕೊನೆಗೆ, ಭಾಗದ ಪಠ್ಯವನ್ನು OpenAI ಎಂಬೆಡಿಂಗ್ API ಗೆ ಹೋರೆದು 1536 ಸಂಖ್ಯೆಗಳ ವೆಕ್ಟರ್‌ವನ್ನು ಪಡೆದಿದ್ದಾರೆ, ಇದು ಭಾಗದ ಸಾಂಕೇತಿಕ ಅರ್ಥವನ್ನು ಪ್ರತಿನಿಧಿಸುತ್ತದೆ. ಭಾಗ ಮತ್ತು ಅದರ OpenAI ಎಂಬೆಡಿಂಗ್ ವೆಕ್ಟರ್ `embedding_index_3m.json` ಎಂಬ ಎಂಬೆಡಿಂಗ್ ಸೂಚ್ಯಂಕದಲ್ಲಿ ಸಂಗ್ರಹಿಸಲಾಗಿದೆ.

### ವೆಕ್ಟರ್ ಡೇಟಾಬೇಸ್‌ಗಳು

ಪಾಠದ ಸರಳತೆಯಿಗಾಗಿ, ಎಂಬೆಡಿಂಗ್ ಸೂಚ್ಯಂಕವನ್ನು `embedding_index_3m.json` ಎಂಬ JSON ಕಡತದಲ್ಲಿ ಸಂಗ್ರಹಿಸಿ, ಪಾಂಡಾಸ್ ಡೇಟಾ ಫ್ರೇಮ್‌ಗೆ ಲೋಡ್ ಮಾಡಲಾಗಿದೆ. ಆದಾಗ್ಯೂ ಉತ್ಪಾದನಾ ಪರಿಸರದಲ್ಲಿ, ಎಂಬೆಡಿಂಗ್ ಸೂಚ್ಯಂಕವನ್ನು [Azure Cognitive Search](https://learn.microsoft.com/training/modules/improve-search-results-vector-search?WT.mc_id=academic-105485-koreyst), [Redis](https://cookbook.openai.com/examples/vector_databases/redis/readme?WT.mc_id=academic-105485-koreyst), [Pinecone](https://cookbook.openai.com/examples/vector_databases/pinecone/readme?WT.mc_id=academic-105485-koreyst), [Weaviate](https://cookbook.openai.com/examples/vector_databases/weaviate/readme?WT.mc_id=academic-105485-koreyst) ಮುಂತಾದ ವೆಕ್ಟರ್ ಡೇಟಾಬೇಸ್‌ಗಳಲ್ಲಿ ಸಂಗ್ರಹಿಸಲಾಗುತ್ತದೆ.

## ಕೋಸೈನ್ ಸಾದೃಶ್ಯದ ಅರ್ಥ

ನಾವು ಈಗ ಪಠ್ಯ ಎಂಬೆಡಿಂಗ್‌ಗಳನ್ನು ಕಲಿತಿದ್ದೇವೆ, ಮುಂದಿನ ಹಂತವೆಂದರೆ ಪಠ್ಯ ಎಂಬೆಡಿಂಗ್‌ಗಳನ್ನು ಬಳಸಿಕೊಂಡು ಡೇಟಾ ಹುಡುಕಾಟ ಮಾಡುವುದು ಮತ್ತು ವಿಶೇಷವಾಗಿ ಕೋಸೈನ್ ಸಾದೃಶ್ಯ ಬಳಸಿ ನಿರ್ದಿಷ್ಟ ಪ್ರಶ್ನೆಗೆ ಅತ್ಯಂತ ಸಾದೃಶ್ಯ ಎಂಬೆಡಿಂಗ್‌ಗಳನ್ನು ಹುಡುಕುವುದು.

### ಕೋಸೈನ್ ಸಾದೃಶ್ಯ ಎಂದರೆ ಏನು?

ಕೋಸೈನ್ ಸಾದೃಶ್ಯವು ಎರಡು ವೆಕ್ಟರ್‌ಗಳ ನಡುವೆ ಸಾದೃಶ್ಯವನ್ನು ಅಳೆಯುವ ಸಾಧನ, ಇದನ್ನು `ನಿಕಟ ನೆರೆಹೊರೆಯವರ ಹುಡುಕಾಟ` ಎಂದು ಸಹ ಕರೆಯಲಾಗುತ್ತದೆ. ಕೋಸೈನ್ ಸಾದೃಶ್ಯ ಹುಡುಕಾಟ ಮಾಡಲು ನೀವು OpenAI ಎಂಬೆಡಿಂಗ್ API ಬಳಸಿ ಪ್ರಶ್ನೆಯ ಪಠ್ಯವನ್ನು ವೆಕ್ಟರೈಸ್ ಮಾಡಬೇಕು. ನಂತರ ಪ್ರಶ್ನೆ ವೆಕ್ಟರ್ ಮತ್ತು ಪ್ರತಿಯೊಂದು ಎಂಬೆಡಿಂಗ್ ಸೂಚ್ಯಂಕ ವೆಕ್ಟರ್‌ಗಳ ನಡುವೆ ಕೋಸೈನ್ ಸಾದೃಶ್ಯವನ್ನು ಗಣನೆ ಮಾಡಬೇಕು. ಎಂಬೆಡಿಂಗ್ ಸೂಚ್ಯಂಕದಲ್ಲಿ ಪ್ರತಿಯೊಂದು YouTube ಭಾಷಾಂತರ ಪಠ್ಯ ಭಾಗಕ್ಕೆ ಒಂದು ವೆಕ್ಟರ್ ಬೇರ್ಪಡಿಸಲಾಗಿದೆ. ಕೊನೆಗೆ, ಫಲಿತಾಂಶಗಳನ್ನು ಕೋಸೈನ್ ಸಾದೃಶ್ಯದ ಯೋಗ್ಯತೆ ಅಂಕಗಳ ಆಧಾರದ ಮೇಲೆ ವಿಂಗಡಿಸಿ, ಅತ್ಯಧಿಕ ಸಾದೃಶ್ಯವುಳ್ಳ ಪಠ್ಯ ಭಾಗಗಳು ಪ್ರಶ್ನೆಗೆ ಅತ್ಯಂತ ಸಮಾನವಾಗಿರುವವು.

ಗಣಿತೀಯ ವೀಕ್ಷಣೆಯಿಂದ, ಕೋಸೈನ್ ಸಾದೃಶ್ಯವು ಬಹುಮಾನದ ಸ್ಥಳದಲ್ಲಿ ಬಿದ್ದಿರುವ ಇಬ್ಬರು ವೆಕ್ಟರ್‌ಗಳ ನಡುವಿನ ಕೋನದ ಕೋಸೈನ್ ಅನ್ನು ಅಳೆಯುತ್ತದೆ. ಇದು ಉಪಯುಕ್ತ, ಏಕೆಂದರೆ ಇಬ್ಬರು ದಸ್ತಾವೇಜುಗಳು ಎಷ್ಟು ದೂರದಲ್ಲಿದ್ದರೂ Euclidean ಅಂತರದಿಂದ, ಅವರಿಗೆ ಅವರ ನಡುವೆ ಅಲ್ಪ ಕೋನ ಇದ್ದರೆ ಅವರಿಗೆ ಹೆಚ್ಚು ಕೋಸೈನ್ ಸಾದೃಶ್ಯ ಇರುತ್ತದೆ. ಕೋಸೈನ್ ಸಾದೃಶ್ಯ ಸಮೀಕರಣಗಳ ಬಗ್ಗೆ ಹೆಚ್ಚು ಮಾಹಿತಿ ಪಡೆಯಲು, [Cosine similarity](https://en.wikipedia.org/wiki/Cosine_similarity?WT.mc_id=academic-105485-koreyst) ಅನ್ನು ನೋಡಿ.

## ನಿಮ್ಮ ಮೊದಲ ಹುಡುಕುವ ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸುವುದು
