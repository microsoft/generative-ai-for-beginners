# ಜನರೇಟಿವ್ AI-ಚಾಲಿತ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುವುದು

ಈಗ ನಾವು ಪಠ್ಯ-ಪೀಠಿಕೆ ಆಧಾರಿತ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಹೇಗೆ ನಿರ್ಮಿಸಬಹುದು ಎಂಬುದನ್ನು ನೋಡಿದ್ದೇವೆ, ಈಗ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನೋಡಿ.

ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು ನಮ್ಮ ದೈನಂದಿನ ಜೀವನದ ಅವಿಭಾಜ್ಯ ಭಾಗವಾಗಿವೆ, ಕೇವಲ ಹಗುರ ಮಾತುಕತೆಗಾಗಿ ಅಲ್ಲ. ಇವು ಗ್ರಾಹಕ ಸೇವೆ, ತಾಂತ್ರಿಕ ಸಹಾಯಕತೆ ಮತ್ತು ಸೂಕ್ಷ್ಮ ಸಲಹಾ ವ್ಯವಸ್ಥೆಗಳ ಪ್ರಮುಖ ಭಾಗಗಳಾಗಿವೆ. ನೀವು ಹಾಲಿ ಯಾವುದಾದರೂ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ನಲ್ಲಿ ಸಹಾಯ ಪಡೆದಿರಬಹುದು. ನಾವು ಜನರೇಟಿವ್ AI ಮುಂತಾದ ಹೆಚ್ಚಿನ ಸುಧಾರಿತ ತಂತ್ರಜ್ಞಾನಗಳನ್ನು ಈ ವೇದಿಕೆಗಳಿಗೆ ಸಂಯೋಜಿಸುವಂತೆ ಮಾಡಿಕೊಂಡಂತೆ, ಸಂಕೀರ್ಣತೆ ಮತ್ತು ಸವಾಲುಗಳು ಎರಡೂ ಹೆಚ್ಚಾಗುತ್ತವೆ.

ಕೆಲವು ಪ್ರಶ್ನೆಗಳು ನಾವು ಉತ್ತರಿಸಬೇಕಾಗಿದೆ:

- **ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಾಣ**. ನಾವು ಈ AI-ಚಾಲಿತ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ದಿಷ್ಟ ಬಳಕೆಯ ಪ್ರಕರಣಗಳಿಗೆ ಹೇಗೆ ಪರಿಣಾಮಕಾರಿಯಾಗಿ ನಿರ್ಮಿಸುತ್ತೇವೆ ಮತ್ತು ಸಹಜವಾಗಿ ಸಂಯೋಜಿಸುತ್ತೇವೆ?
- **ನಿಗಾ**. ನಿಯೋಜಿಸಿದ ನಂತರ, ಅಪ್ಲಿಕೇಶನ್‌ಗಳು ಕಾರ್ಯಕ್ಷಮತೆಯ ಹಾಗೂ [ಜವಾಬ್ದಾರಿಯುತ AI ಆರು ತತ್ವಗಳು](https://www.microsoft.com/ai/responsible-ai?WT.mc_id=academic-105485-koreyst)ಗಳಿಗೆ ಅನುಗುಣವಾಗುವಿಕೆಗೆ ಎಷ್ಟು ಉತ್ತಮ ಮಟ್ಟದಲ್ಲಿ ಕಾರ್ಯನಿರ್ವಹಿಸುತ್ತಿವೆ ಎಂಬುದನ್ನು ನಾವು ಹೇಗೆ ನಿಗನೋಡಬಹುದು?

ನಾವು ಸ್ವಯಂಚಾಲನೆ ಮತ್ತು ಅನುಸರಣೆಯುತ್ತ ಮಾನವ-ಯಂತ್ರ ಸಂವಹನ ಕಾಲಘಟ್ಟದಲ್ಲಿ ಮುಂದುವರಿಯುವಂತೆ, ಜನರೇಟಿವ್ AI ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳ ವ್ಯಾಪ್ತಿ, ಆಳ ಮತ್ತು ಹೊಂದಾಣಿಕೆಯು ಹೇಗೆ ಪರಿವರ್ತಿತವಾಗುತ್ತದೆ ಎಂಬುದನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಅವಶ್ಯಕವಾಗಿದೆ. ಈ ಪಾಠವು ಈ ಸಂಕೀರ್ಣ ವ್ಯವಸ್ಥೆಗಳ معماري ಅಂಶಗಳನ್ನು ಪರಿಶೀಲಿಸುವುದು, ನಿಖರ-ಕಾರ್ಯಗಳಿಗಾಗಿ ಅವುಗಳನ್ನು ಸೂಕ್ತಗೊಳಿಸುವ ವಿಧಾನಗಳನ್ನು ಗ್ರಹಿಸುವುದು ಮತ್ತು ಜವಾಬ್ದಾರಿಯುತ AI ನಿಯೋಗಕ್ಕೆ ಸಂಬಂಧಿಸಿದ ಮಾನದಂಡಗಳು ಮತ್ತು ಪರಿಗಣನೆಗಳನ್ನು ಮೌಲ್ಯಮಾಪನ ಮಾಡುವುದನ್ನು ಒಳಗೊಂಡಿದೆ.

## ಪರಿಚಯ

ಈ ಪಾಠದಲ್ಲಿ ಒಳಗೊಂಡಿರುವುದು:

- ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ನಿರ್ಮಿಸುವ ಮತ್ತು ಸಂಯೋಜಿಸುವ ತಂತ್ರಗಳು.
- ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಕಸ್ಟಮೈಸೇಶನ್ ಮತ್ತು ಸೂಕ್ಷ್ಮಸಂಯೋಜನೆ ಹೇಗೆ ಅನ್ವಯಿಸುವುದು.
- ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ನಿಗದಿಗೊಳಿಸಲು ತಂತ್ರಗಳು ಮತ್ತು ಪರಿಗಣನೆಗಳು.

## ಅಧ್ಯಯನ ಗುರಿಗಳು

ಈ ಪಾಠದ ಅಂತ್ಯಕ್ಕೆ, ನೀವು ಮುಂದಿನ ವಿಷಯಗಳನ್ನು ಮಾಡಬಲ್ಲಿರಿ:

- ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಅಸ್ತಿತ್ವದಲ್ಲಿರುವ ವ್ಯವಸ್ಥೆಗಳಿಗೆ ಸೇರಿಸುವ ಮತ್ತು ನಿರ್ಮಿಸಲು ಪರಿಗಣನೆಗಳನ್ನು ವಿವರಿಸಲು.
- ನಿರ್ದಿಷ್ಟ ಬಳಕೆಯ ಪ್ರಕರಣಗಳಿಗೆ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಕಸ್ಟಮೈಸ್ ಮಾಡಲು.
- AI ಚಾಲಿತ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳ ಗುಣಮಟ್ಟವನ್ನು ನಿಗದಿಗೊಳಿಸಲು ಮುಖ್ಯ ಮೆರಿಕೆಗಳು ಮತ್ತು ಪರಿಗಣನೆಗಳನ್ನು ಗುರುತಿಸಲು.
- ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು AI ಅನ್ನು ಜವಾಬ್ದಾರಿಯುತವಾಗಿ ಬಳಸುವುದನ್ನು ಖಚಿತಪಡಿಸಲು.

## ಜನರೇಟಿವ್ AI-ಯನ್ನು ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ಗಳಲ್ಲಿ ಸಂಯೋಜಿಸುವುದು

ಜನರೇಟಿವ್ AI ಮೂಲಕ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಅವುಗಳ ತಂತ್ರಸಿದ್ಧಾಂತ, ಕಾರ್ಯಕ್ಷಮತೆ ಮತ್ತು ಬಳಕೆದಾರ ಮುಖಮುಖಿ ಪರಿಷ್ಕರಿಸುವುದು ಮಾತ್ರವೇ ಅಲ್ಲ; ಇದರಿಂದ ಉತ್ತಮ ಬಳಕೆದಾರ ಅನುಭವವನ್ನು ನೀಡುವುದು ಗುರಿಯಾಗಿರುತ್ತದೆ. ಇದರಲ್ಲಿ معماري ಮೂಲಭೂತ ಅಂಶಗಳು, API ಸಂಯೋಜನೆಗಳು ಮತ್ತು ಬಳಕೆದಾರ ಮುಖಮುಖಿ ಪರಿಗಣನೆಗಳನ್ನು ಪರಿಶೀಲಿಸಲಾಗುತ್ತದೆ. ಈ ವಿಭಾಗವು ಈ ಸಂಕೀರ್ಣ ವಾತಾವರಣವನ್ನು ನವಿಗೇಟ್ ಮಾಡಲು ಸಂಪೂರ್ಣ ಮಾರ್ಗಸೂಚಿಯನ್ನು ಒದಗಿಸುವ ವಿಶ್ವಾಸಾರ್ಹ ಉದ್ದೇಶ ಹೊಂದಿದೆ, ನೀವು ಅವುಗಳನ್ನು ಇಂಟಿಗ್ರೇಟ್ ಮಾಡುತ್ತಿರುವೆಯಾದರೂ ಅಥವಾ ಸ್ವತಂತ್ರ ವೇದಿಕೆಗಳಾಗಿ ನಿರ್ಮಿಸುತ್ತಿರುವೆಯಾದರೂ.

ಈ ವಿಭಾಗದ ಕೊನೆಯಲ್ಲಿ, ನೀವು ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ಪರಿಣಾಮಕಾರಿಯಾಗಿ ನಿರ್ಮಿಸುವ ಮತ್ತು ಅಳವಡಿಸುವ ಪರಿಣತಿಯನ್ನು ಹೊಂದಿರುತ್ತೀರಿ.

### ಚಾಟ್‌ಬಾಟ್ ಅಥವಾ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್?

ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳನ್ನು ನಿರ್ಮಿಸುವ ಮುನ್ನ, 'ಚಾಟ್‌ಬಾಟ್' ಮತ್ತು 'AI ಚಾಲಿತ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳು' ಎಂಬ ವಿಭಿನ್ನ ಪಾತ್ರ ಮತ್ತು ಕಾರ್ಯತಂತ್ರಗಳನ್ನು ಹೊಂದಿರುವುದರಿಂದ ಅವುಗಳನ್ನು ಹೋಲಿಸೋಣ. ಚಾಟ್‌ಬಾಟ್‌ನ ಮುಖ್ಯ ಉದ್ದೇಶ ನಿರ್ದಿಷ್ಟ ಸಂವಾದ ಕಾರ್ಯಗಳನ್ನು ಸ್ವಯಂಚಾಲಿತಗೊಳಿಸುವುದು, ಉದಾಹರಣೆಗೆ ಸಾಮಾನ್ಯ ಪ್ರಶ್ನೆಗಳಿಗೆ ಉತ್ತರಿಸುವುದು ಅಥವಾ ಪ್ಯಾಕೇಜ್ ಟ್ರ್ಯಾಕಿಂಗ್ ಮಾಡುವುದಾಗಿದೆ. ಅದು ಸಾಮಾನ್ಯವಾಗಿ ನಿಯಮಾಧಾರಿತ ತರ್ಕ ಅಥವಾ ಸಂಕೀರ್ಣ AI ಆಲ್ಗರಿದಮ್ಗಳಿಂದ ನಿಯಂತ್ರಿಸಲ್ಪಡುತ್ತದೆ. ವಿಷಯವಾಗಿ, AI ಚಾಲಿತ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ ಬಹುಮಟ್ಟಿನ ಡಿಜಿಟಲ್ ಸಂವಹನಕ್ಕಾಗಿ ವಿನ್ಯಾಸಗೊಂಡಿದೆ, ಉದಾಹರಣೆಗೆ ಪಠ್ಯ, ಧ್ವನಿ ಮತ್ತು ದೃಶ್ಯ ಚಾಟ್‌ಗಳು ಮಾನವ ಬಳಕೆದಾರರ ನಡುವೆ. ಇದರ ವೈಶಿಷ್ಟ್ಯವೆಂದರೆ ಜನರೇಟಿವ್ AI ಮಾದರಿಯನ್ನು ಒಪ್ಪಿಕೊಳ್ಳುವುದಾದ್ದರಿಂದ, ಹೆಚ್ಚಿನ ಪ್ರಕಾರದ ಇನ್‌ಪುಟ್ ಮತ್ತು ಸನ್ನಿವೇಶ ಸಂಕೇತಗಳಿಂದ ಆಧರಿಸಿದ ಮನುಷ್ಯನಂತ ಸಮ್ವದನಾ ತಯಾರಿಕೆ. ಜನರೇಟಿವ್ AI ಚಾಲಿತ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ ಮುಕ್ತ-ಕ್ಷೇತ್ರ ಸಂವಾದಗಳಲ್ಲಿ ತೊಡಗಿಸಿಕೊಳ್ಳಬಹುದು, ಸಂವಾದ ಸನ್ನಿವೇಶ ಹದಗೆಡುತ್ತಿದ್ದಂತೆ ಹೊಂದಿಕೊಳ್ಳಬಹುದು, ಮತ್ತು ಸೃಜನಾತ್ಮಕ ಅಥವಾ ಸಂಕೀರ್ಣ ಸಂವಾದಗಳನ್ನು ಉತ್ಪಾದಿಸಬಹುದು.

ಕೆಳಗಿನ ತಳಿಕೆಯಲ್ಲಿ ಇವುಗಳ ಪ್ರಾಥಮಿಕ ಭೇದಗಳು ಮತ್ತು ಸಮಾನತೆಗಳನ್ನು ವಿವರಿಸಲಾಗಿದೆ, ಅವುಗಳ ವಿಶಿಷ್ಟ ಪಾತ್ರಗಳನ್ನು ಡಿಜಿಟಲ್ ಸಂವಹನದಲ್ಲಿ ಅರ್ಥಮಾಡಿಕೊಳ್ಳುವುದು ಸಹಾಯಕ.

### SDKಗಳು ಮತ್ತು APIಗಳೊಂದಿಗೆ ಪೂರ್ವನಿರ್ಮಿತ ಕಾರ್ಯಗಳು ಉಪಯೋಗಿಸುವುದು

ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸುವಾಗ, ದಾರಿತಪ್ಪದೆ ಮೊದಲೇ ಇರುವುದನ್ನು ಅಂದಾಜಿಸುವುದು ಉತ್ತಮ ಆರಂಭ. SDKಗಳು ಮತ್ತು APIಗಳನ್ನು ಉಪಯೋಗಿಸಿ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸುವುದು ಹಲವು ಕಾರಣಗಳಿಗಾಗಿ ಉಪಯುಕ್ತವಾದ ತಂತ್ರವಾಗಿದೆ. ಉತ್ತಮ ದಾಖಲಾಗಿರುವ SDKಗಳು ಮತ್ತು APIಗಳನ್ನು ಸಂಯೋಜಿಸುವ ಮೂಲಕ, ನೀವು ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ದೀರ್ಘಕಾಲಿಕ ಯಶಸ್ಸಿಗಾಗಿ ನಿರ್ವಹಣಾ ಮತ್ತು ವಿಸ್ತರಣಾ ಚಿಂತನೆಗಳನ್ನು ಎದುರಿಸಲು ನಿರ್ವಹಿಸಿದೆಯಾದಂತಾಗುತ್ತೀರಿ.

- **ಅಭಿವೃದ್ದಿ ಪ್ರಕ್ರಿಯೆಯನ್ನು ವೇಗಗೊಳಿಸುತ್ತದೆ ಮತ್ತು ಕಾಸ್ಟ್ ಕಡಿಮೆ ಮಾಡುತ್ತದೆ**: ಸ್ವಯಂ ನಿರ್ಮಾಣದ ಬದಲು ಪೂರ್ವನಿರ್ಮಿತ ಕಾರ್ಯಗಳನ್ನು ಉಪಯೋಗಿಸುವುದರಿಂದ, ನೀವು ಅಪ್ಲಿಕೇಶನ್‌ನ ಇತರ ಪ್ರಮುಖ ಅಂಶಗಳಾದ ವ್ಯಾಪಾರ ತರ್ಕವನ್ನು ಮೇಲೆ ಕೈ ಹಾಕಬಹುದು.
- **ಮೇಲ್ಮಟ್ಟದ ಕಾರ್ಯಕ್ಷಮತೆ**: ಕಾರ್ಯವನ್ನು ನೂತನದಾಗಿ ನಿರ್ಮಿಸುವಾಗ, "ಇದು ಹೇಗೆ ವಿಸ್ತಾರಗೊಳ್ಳುತ್ತದೆ? ಈ ಅಪ್ಲಿಕೇಶನ್ ತತ್ತ ಒಮ್ಮೆ ಹೆಚ್ಚುವರಿ ಬಳಕೆದಾರರನ್ನು ನಿರ್ವಹಿಸಬಹುದೇ?" ಎಂಬ ಪ್ರಶ್ನೆಗಳು ಬರುತ್ತವೆ. ಉತ್ತಮ ನಿರ್ವಹಿತ SDK ಮತ್ತು APIಗಳು ಈ ಗೊಂದಲಗಳಿಗೆ ಪರಿಹಾರಗಳನ್ನು ಹೊಂದಿರುತ್ತವೆ.
- **ಅನುವಂಶಿಕ ನಿರ್ವಹಣೆ ಸುಲಭ**: ಅಪ್‌ಡೇಟ್‌ಗಳು ಮತ್ತು ಸುಧಾರಣೆಗಳನ್ನು ನಿರ್ವಹಿಸುವುದು ಸುಲಭವಾಗಿದೆ ಏಕೆಂದರೆ ಹೆಚ್ಚಿನ APIಗಳು ಮತ್ತು SDKಗಳನ್ನು ಲೈಬ್ರರಿಯನ್ನು ನವೀಕರಿಸುವ ಮೂಲಕ ನಿರ್ವಹಿಸಬಹುದು.
- **ಕಟಿಂಗ್-ಎಜ್ ತಂತ್ರಜ್ಞಾನಕ್ಕೆ ಪ್ರವೇಶ**: ವಿಸ್ತೃತ ಡೇಟಾಸೆಟ್‌ಗಳ ಮೇಲೆ ಸೂಕ್ಷ್ಮತೆಯಿಂದ ತರಬೇತಿ ಪಡೆದ ಮಾದರಿಗಳನ್ನು ಉಪಯೋಗಿಸುವ ಮೂಲಕ, ನಿಮ್ಮ ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ನೈಸರ್ಗಿಕ ಭಾಷಾ ಸಾಮರ್ಥ್ಯಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ.

SDK ಅಥವಾ API ಕಾರ್ಯಕ್ಷಮತೆಯನ್ನು ಪ್ರವೇಶಿಸುವುದಕ್ಕೆ ಸಾಮಾನ್ಯವಾಗಿ ಸೇವೆಗಳನ್ನು ಬಳಸಲು ಅನುಮತಿಯು ಅಗತ್ಯ, ಅದು ಸಾಮಾನ್ಯವಾಗಿ ವಿಶೇಷ ಕೀ ಅಥವಾ ಪ್ರಮಾಣೀಕರಣ ಟೋಕನ್ ಮೂಲಕ. ನಾವು OpenAI Python ಲೈಬ್ರರಿ ಉಪಯೋಗಿಸಿ ಇದನ್ನು ಪರೀಕ್ಷಿಸುತ್ತೇವೆ. ನೀವು ಈ ಪಾಠಕ್ಕಾಗಿ ಈ ಕೆಳಗಿನ [OpenAI ನೋಟ್ಬುಕ್](./python/oai-assignment.ipynb?WT.mc_id=academic-105485-koreyst) ಅಥವಾ [ಅಜೂರ್ OpenAI ಸೇವೆಗಳ ನೋಟ್ಬುಕ್](./python/aoai-assignment.ipynb?WT.mc_id=academic-105485-koreys)ವನ್ನು ಸ್ವತಃ ಪ್ರಯತ್ನಿಸಬಹುದು.

```python
import os
from openai import OpenAI

API_KEY = os.getenv("OPENAI_API_KEY","")

client = OpenAI(
    api_key=API_KEY
    )

response = client.responses.create(model="gpt-4o-mini", input="Suggest two titles for an instructional lesson on chat applications for generative AI.", store=False)
print(response.output_text)
```

ಮೇಲಿನ ಉದಾಹರಣೆಯಲ್ಲಿ Responses APIನ್ನು ಪೂರ್ಣಗೊಳಿಸಲು GPT-4o ಮಿನಿ ಮಾದರಿಯನ್ನು ಉಪಯೋಗಿಸಲಾಗಿದೆ, ಆದರೆ API ಕೀ ಸೇವೆ ನಡೆಸುವ ಮೊದಲು ಸೆಟ್ ಮಾಡಲಾಗಿದೆ. ನೀವು ಕೀ ಸೆಟ್ ಇಲ್ಲದೆ ಇದ್ದರೆ ದೋಷ ಶುಂಕರಿಸಿತ್ತಿದ್ದೀರಿ.

## ಬಳಕೆದಾರ ಅನುಭವ (UX)

ಸಾಮಾನ್ಯ UX ತತ್ವಗಳು ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳಿಗೆ ಅನ್ವಯಿಸುತ್ತವೆ, ಆದರೆ ಇಲ್ಲಿ ಯಂತ್ರ-ಅಭ್ಯಾಸಗಳನ್ನು ಒಳಗೊಂಡಿರುವ ಕಾರಣ ಕೆಲ ಹೆಚ್ಚುವರಿ ಪರಿಗಣನೆಗಳು ವಿಶೇಷವಾಗಿ ಮಹತ್ವದ್ದಾಗುತ್ತವೆ.

- **ಅನಿಶ್ಚಿತತೆಗೆ ಪರಿಹಾರ ನೀಡುವ ಯಂತ್ರಗಳು**: ಜನರೇಟಿವ್ AI ಮಾದರಿಗಳು ಕೆಲವು ವೇಳೆ ಅಸ್ಪಷ್ಟ ಉತ್ತರಗಳನ್ನು ನೀಡಬಹುದು. ಬಳಕೆದಾರರು ಸ್ಪಷ್ಟತೆಯನ್ನು ಕೇಳಲು ಸೌಲಭ್ಯ ಸಿಗಬೇಕು.
- **ಸಂದರ್ಭ ಸ್ಥಿತಿಯನ್ನು ಕಾಯುವಿಕೆ**: ಉನ್ನತ ಮಟ್ಟದ ಜನರೇಟಿವ್ AI ಮಾದರಿಗಳನ್ನು ಸಂವಾದದ ಸನ್ನಿವೇಶವನ್ನು ನೆನಪಿಡುವ ಸಾಮರ್ಥ್ಯವಿದೆ. ಬಳಕೆದಾರರು ಸನ್ನಿವೇಶವನ್ನು ನಿಯಂತ್ರಿಸಲು ಮತ್ತು ನಿರ್ವಹಿಸಲು ಅವಕಾಶ ನೀಡಿದಾಗ ಉತ್ತಮ ಅನುಭವ ಸಿಗುತ್ತದೆ, ಆದರೆ ಸಂವೇದನಾಶೀಲ ಮಾಹಿತಿಯನ್ನು ಉಳಿಸುವ ಅಪಾಯ ಇದೆ. ಇದಕ್ಕಾಗಿ ಉಳಿಸುವ ಆವಧಿಯನ್ನು ನಿರ್ಧರಿಸುವ ಪರಿಸರ ಕಾರ್ಯ ನೀತಿ ಇತ್ಯಾದಿ ನೀತಿಗಳನ್ನು ಬದ್ಧಮಾಡುವುದು ಮುಖ್ಯ.
- **ವೈಯಕ್ತಿಕಗೊಳಿಸುವಿಕೆ**: ಕಲಿಯುವ ಮತ್ತು ಹೊಂದಿಕೊಳ್ಳುವ ಸಾಮರ್ಥ್ಯದಿಂದ AI ಮಾದರಿಗಳು ವ್ಯಕ್ತಿಗತ ಅನುಭವ ಒದಗಿಸುತ್ತವೆ. ಬಳಕೆದಾರರ ಪ್ರೊಫೈಲ್ ಮುಂತಾದ ವೈಶಿಷ್ಟ್ಯಗಳ ಮೂಲಕ ಅನುಭವ ಸ್ವರೂಪ ಮಾಡುವುದು ಬಳಕೆದಾರನಿಗೆ ಅವನು anlaşಿಕೊಂಡಿರುವ ಭಾವನೆ ನೀಡುತ್ತದೆ ಮತ್ತು ವಿಶೇಷ ಉತ್ತರಗಳನ್ನು ಕಂಡುಹಿಡಿಯಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ, ಪರಿಣಾಮಕಾರಿ ಮತ್ತು ತೃಪ್ತಿದಾಯಕ ಸಂವಹನಕ್ಕೆ ಸಹಾಯವಾಗುತ್ತದೆ.

ವೈಯಕ್ತಿಕಗೊಳಿಸುವಿಕೆಯ ಒಂದು ಉದಾಹರಣೆ OpenAI ChatGPT ನ "ಕಸ್ಟಮ್ ಸೂಚನೆಗಳು" ಸೆಟ್ಟಿಂಗ್‌ಗಳಾಗಿದೆ. ಇದು ನಿಮಗೆ ನಿಮ್ಮ ಬಗ್ಗೆ ಪೈಠಣಿಕ ಮಾಹಿತಿಯನ್ನು ಒದಗಿಸುವ ಅವಕಾಶ ನೀಡುತ್ತದೆ, ಅದು ನಿಮ್ಮ ಪ್ರಾಂಪ್ಟ್‌ಗಳಿಗೆ ಪ್ರಮುಖ ಸನ್ನಿವೇಶವಾಗಿರಬಹುದು. ಇಲ್ಲಿ ಒಂದು ಕಸ್ಟಮ್ ಸೂಚನೆಯ ಉದಾಹರಣೆ ಇದೆ.

### ಮೈಸ್ರೋಸಾಫ್ಟ್‌ನ ದೊಡ್ಡ ಭಾಷಾ ಮಾದರಿಗಳಿಗಾಗಿ ಸಿಸ್ಟಂ ಸಂದೇಶ ರಚನೆ ಪ್ರಕ್ರಿಯೆ

[Microsoft ಪ್ರಮುಖ ಮಾರ್ಗದರ್ಶನ](https://learn.microsoft.com/azure/ai-services/openai/concepts/system-message#define-the-models-output-format?WT.mc_id=academic-105485-koreyst) ಒದಗಿಸಿದೆ, ಅದು LLMಗಳಿಂದ ಉತ್ತರಗಳನ್ನು ಉತ್ಪಾದಿಸುವಾಗ ಪರಿಣಾಮಕಾರಿ ಸಿಸ್ಟಂ ಸಂದೇಶಗಳನ್ನು ಬರೆಯಲು 4 ಭಾಗಗಳಲ್ಲಿ ವಿಭಜಿಸಲಾಗಿದೆ:

1. ಮಾದರಿಯ ಬಳಕೆದಾರರು ಯಾರು ಮತ್ತು ಅದರ ಸಾಮರ್ಥ್ಯಗಳು ಮತ್ತು ಮಿತಿ ರೇಖೆಗಳನ್ನು ನಿರ್ದಿಷ್ಟಗೊಳಿಸುವುದು.
2. ಮಾದರಿಯ ಔಟ್‌ಪುಟ್ ಫಾರ್ಮಾಟ್ ನಿರ್ಧರಿಸುವುದು.
3. ಮಾದರಿಯ ನಿಗದಿತ ನಡವಳಿಕೆಯನ್ನು ತೋರಿಸುವ ನಿರ್ದಿಷ್ಟ ಉದಾಹರಣೆಗಳನ್ನು ಒದಗಿಸುವುದು.
4. ಹೆಚ್ಚುವರಿ ನಡವಳಿಕೆ ಗಾರ್ಡ್‌ರೈಲುಗಳನ್ನು ಒದಗಿಸುವುದು.

### ಪ್ರವೇಶಾತಿ

ಬಳಕೆದಾರರು ದೃಷ್ಟಿ, ಶ್ರವಣ, ಚಲಿಸುವಿಕೆ ಅಥವಾ ಜ್ಞಾನಾತ್ಮಕ ಅಡಚಣೆಗಳನ್ನು ಹೊಂದಿರಲಿ, ಒಟ್ಟು ಉತ್ತಮ ವಿನ್ಯಾಸವಾದ ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ ಎಲ್ಲರಿಗೂ ಬಳಸದಾರರಾಗಿರಬೇಕು. ವಿವಿಧ ಬಳಕೆದಾರ ಅক্ষমತೆಗಳಿಗೆ ವಿಶೇಷ ಪ್ರವೇಶಾತಿ ಸುಧಾರಣೆಗಳನ್ನು ಕೆಳಗಿನ ಪಟ್ಟಿ ವಿವರಿಸುವುದು.

- **ದೃಷ್ಟಿ ಅಡಚಣೆಗಳಿಗೆ ಸೌಲಭ್ಯಗಳು**: ಹೆಚ್ಚಿನ ಪ್ರತಿರೋಧಿತಗಳು ಮತ್ತು ಪಠ್ಯವನ್ನು ಮರುಗೊಳಿಸುವ ಸಾಮರ್ಥ್ಯ, ಸ್ಕ್ರೀನ್ ರೀಡರ್  ಅವಲಂಬನೆ.
- **ಶ್ರವಣ ಅಡಚಣೆಗಳಿಗೆ ಸೌಲಭ್ಯಗಳು**: ಪಠ್ಯ-ದಿಂದ-ಧ್ವನಿ ಮತ್ತು ಧ್ವನಿ-ದಿಂದ-ಪಠ್ಯ ಕಾರ್ಯಗಳು, ಧ್ವನಿ ಸೂಚನೆಗಳ ದೃಶ್ಯ ಸೂಚನೆಗಳು.
- **ಚಲನೆ ಅಡಚಣೆಗಳಿಗೆ ಸೌಲಭ್ಯಗಳು**: ಕೀಬೋರ್ಡ್ ನ್ಯಾವಿಗೇಶನ್ ಬೆಂಬಲ, ಧ್ವನಿ ಆಜ್ಞೆಗಳು.
- **ಜ್ಞಾನಾತ್ಮಕ ಅಡಚಣೆಗಳಿಗೆ ಸೌಲಭ್ಯಗಳು**: ಸರಳೀಕೃತ ಭಾಷಾ ಆಯ್ಕೆಗಳು.

## ಡೊಮೇನ್-ನಿರ್ದಿಷ್ಟ ಭಾಷಾ ಮಾದರಿಗಳಿಗಾಗಿ ಕಸ್ಟಮೈಸೇಶನ್ ಮತ್ತು ಸೂಕ್ಷ್ಮಸಂಯೋಜನೆ

ನಿಮ್ಮ ಕಂಪನಿಯ ನುಡಿ ಮತ್ತು ಅದರ ಬಳಕೆದಾರರ ಸಾಮಾನ್ಯ ಪ್ರಶ್ನೆಗಳನ್ನು ತಿಳುವಳಿಕೆಗೆ ಹೊಂದಿದ ಒಂದು ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ ಅನ್ನು ಕಲ್ಪಿಸಿ. ಕೆಲವು ವಿಧಾನಗಳನ್ನು ಸೂಚಿಸುವುದು:

- **DSL ಮಾದರಿಗಳನ್ನು ಉಪಯೋಗಿಸುವದು**. DSL ಅಂದರೆ ಡೊಮೇನ್ ನಿರ್ದಿಷ್ಟ ಭಾಷೆ. ನೀವು ನಿರ್ದಿಷ್ಟ ಕ್ಷೇತ್ರದಲ್ಲಿ ತರಬೇತಿಗೊಂಡ DSL ಮಾದರಿಯನ್ನು ಅದರ ಸಂಧರ್ಭ ಮತ್ತು ಸಂದರ್ಭಗಳನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಉಪಯೋಗಿಸಬಹುದು.
- **ಸೂಕ್ಷ್ಮಸಂಯೋಜನೆಯನ್ನು ಅನ್ವಯಿಸುವದು**. ಸೂಕ್ಷ್ಮಸಂಯೋಜನೆ ಎಂದರೆ ನಿಮ್ಮ ಮಾದರಿಯನ್ನು ನಿರ್ದಿಷ್ಟ ಡೇಟಾ ಸಹಿತ ಇನ್ನಷ್ಟು ತರಬೇತುಗೊಳಿಸುವ ಪ್ರಕ್ರಿಯೆ.

## ಕಸ್ಟಮೈಸೇಶನ್: DSL ಉಪಯೋಗಿಸುವುದು

ಡೊಮೇನ್-ನಿರ್ದಿಷ್ಟ ಭಾಷಾ ಮಾದರಿಗಳನ್ನು (DSL ಮಾದರಿಗಳನ್ನು) ಉಪಯೋಗಿಸುವುದರಿಂದ ಬಳಕೆದಾರರ ತೊಡಗಿಕೆ ಹೆಚ್ಚಿಸುತ್ತದೆ ಮತ್ತು ವಿಶೇಷ, ಸನ್ನಿವೇಶಕ್ಕೆ ಹೊಂದಿಕೊಂಡ ಸಂವಹನಗಳನ್ನು ಒದಗಿಸುತ್ತವೆ. ಇದು ನಿರ್ದಿಷ್ಟ ಕ್ಷೇತ್ರ, ಕೈಗಾರಿಕೆ ಅಥವಾ ವಿಷಯಕ್ಕೆ ಸಂಬಂಧಿಸಿದ ಪಠ್ಯವನ್ನು ಅರ್ಥಮಾಡಿಕೊಳ್ಳಲು ಅಥವಾ ತಯಾರಿಸಲು ತರಬೇತಿಗೊಂಡ ಅಥವಾ ಸೂಕ್ಷ್ಮಗೊಳಿಸಲ್ಪಟ್ಟ ಮಾದರಿ. DSL ಮಾದರಿಯನ್ನು ಆರ್ಜಿಸುವ ಆಯ್ಕೆಗಳು ಶೂನ್ಯದಿಂದ ಸೃಷ್ಠಿಸುವುದು ಅಥವಾ SDK ಮತ್ತು APIಗಳ ಮೂಲಕ ಪೂರ್ವನಿರ್ಮಿತ ಮಾದರಿಗಳನ್ನು ಉಪಯೋಗಿಸುವುದರಿಂದ ವ್ಯತ್ಯಾಸವಾಗಬಹುದು. ಮತ್ತೊಂದು ಆಯ್ಕೆಯಾದ ಸೂಕ್ಷ್ಮಸಂಯೋಜನೆ ಅಂದರೆ ಪೂರ್ವ ತರಬೇತಿಗೊಂಡ ಮಾದರಿಯನ್ನು ತೆಗೆದು ನಿರ್ದಿಷ್ಟ ಡೊಮೇನ್‌ಗಾಗಿ ಹೊಂದಿಕೊಳ್ಳುವುದು.

## ಕಸ್ಟಮೈಸೇಶನ್: ಸೂಕ್ಷ್ಮಸಂಯೋಜನೆ ಅನ್ವಯಿಸುವುದು

ಒಂದು ಪೂರ್ವ-ತರಬೇತಿಗೊಂಡ ಮಾದರಿ ನಿರ್ದಿಷ್ಟ ಕ್ಷೇತ್ರ ಅಥವಾ ಕಾರ್ಯದಲ್ಲಿ ಸಾಲದು ಎಂದು ಕಂಡರೆ ಸೂಕ್ಷ್ಮಸಂಯೋಜನೆಯನ್ನು ಪರಿಗಣಿಸಲಾಗುತ್ತದೆ.

ಉದಾಹರಣೆಗೆ, ವೈದ್ಯಕೀಯ ಪ್ರಶ್ನೆಗಳು ಸಂಕೀರ್ಣವಾಗಿವೆ ಮತ್ತು ಹೆಚ್ಚಿನ ಸನ್ನಿವೇಶವನ್ನೂ ಅಗತ್ಯ ಮಾಡಿಕೊಳ್ಳುತ್ತವೆ. ವೈದ್ಯಕೀಯ ತಜ್ಞನು ರೋಗಿಯನ್ನು ಖಚಿತಗೊಳಿಸುವಾಗ, ಜೀವನಶೈಲಿ ಅಥವಾ ಪೂರ್ವ-ಅವಸ್ಥೆಗಳಂತಹ ಹಲವಾರು ಅಂಶಗಳ ಮೇಲೆ ಅವಲಂಬಿಸಿರುತ್ತಾನೆ, ಹಾಗೆಯೇ ಇತ್ತೀಚಿನ ವೈದ್ಯಕೀಯ ಸಂಶೋಧನೆಗಳನ್ನೂ ಅವಲಂಬಿಸಬಹುದು. ಈ ರೀತಿಯ ಸೂಕ್ಷ್ಮ ಪರಿಸ್ಥಿತಿಗಳಲ್ಲಿ ಸಾಮಾನ್ಯ AI ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ ವಿಶ್ವಸನೀಯ ಸೇವೆ ಆಗಲು ಆಗುವುದಿಲ್ಲ.

### ಸನ್ನಿವೇಶ: ವೈದ್ಯಕೀಯ ಅಪ್ಲಿಕೇಶನ್

ವೈದ್ಯಕೀಯ ಪ್ರವರ್ತಕರಿಗೆ ತ್ವರಿತ ಮಾರ್ಗ ಸೂಚಿಸುವಿಕೆ, ಔಷಧಿ ಒಡಪುಗಳು ಅಥವಾ ಇತ್ತೀಚಿನ ಸಂಶೋಧನಾ ಫಲಿತಾಂಶಗಳನ್ನು ಒದಗಿಸುವ ಒಂದು ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್ ಅನ್ನು ಪರಿಗಣಿಸಿ.

ಸಾಮಾನ್ಯ ಮಾದರಿ ಸಾದಾರಣ ವೈದ್ಯಕೀಯ ಪ್ರಶ್ನೆಗಳಿಗೆ ಅಥವಾ ಸಾಮಾನ್ಯ ಸಲಹೆಗೆ ಸಾಕು ಆದರೆ ಕೆಳಗಿನ ವಿಷಯಗಳಲ್ಲಿ ತೊಂದರೆ ಕಾಣಬಹುದು:

- **ಅತ್ಯಂತ ವಿಶಿಷ್ಟ ಅಥವಾ ಸಂಕೀರ್ಣ ಪ್ರಕರಣಗಳು**. ಉದಾಹರಣೆಗೆ, ಕೈಗಾರಿಕಾ ತಜ್ಞನು ಕೇಳಬಹುದು, "ಮುಗಿದು ಪ್ರತಿರೋಧಕ ಎಪಿಲೆಪ್ಸಿಯ ನಿರ್ವಹಣೆಗೆ ರೌಲಿಪದ ಉತ್ತಮ ಕ್ರಮಗಳೇನು?"
- **ಇತ್ತೀಚಿನ ಸುಧಾರಣೆಗಳ ಕೊರತೆ**. ಸಾಮಾನ್ಯ ಮಾದರಿ neurology ಮತ್ತು ಔಷಧ ಶಾಸ್ತ್ರದ ಇತ್ತೀಚಿನ ಅಭಿವೃದ್ಧಿಗಳನ್ನು ಒಳಗೊಂಡಿರುವ ಪ್ರಸ್ತುತ ಉತ್ತರವನ್ನು ನೀಡಲು ತೊಂದರೆಪಡಬಹುದು.

ಈ ರೀತಿಯ ಸಂದರ್ಭಗಳಲ್ಲಿ, ವಿಶೇಷ ವೈದ್ಯಕೀಯ ಡೇಟಾಸೆಟ್‌ನೊಂದಿಗೆ ಮಾದರಿಯನ್ನು ಸೂಕ್ಷ್ಮಗೊಳಿಸುವುದು ಆ ಪ್ರಶ್ನೆಗಳನ್ನು ಹೆಚ್ಚು ನಿಖರವಾಗಿ ಮತ್ತು ವಿಶ್ವಸನೀಯವಾಗಿ ನಿಭಾಯಿಸುವ ಸಾಮರ್ಥ್ಯವನ್ನು ಬಹುಮಟ್ಟಿಗೆ ಹೆಚ್ಚಿಸುತ್ತದೆ. ಇದು ಸಂಬಂಧಿತ ದೊಡ್ಡ ಡೇಟಾಸೆಟ್‌ಗೆ ಪ್ರವೇಶ ಅಗತ್ಯವಿದೆ, ಅದು ಕ್ಷೇತ್ರ-ನಿರ್ದಿಷ್ಟ ಸಮಸ್ಯೆಗಳು ಮತ್ತು ಪ್ರಶ್ನೆಗಳನ್ನು ಪ್ರತಿಬಿಂಬಿಸುತ್ತದೆ.

## ಉತ್ತಮ ಗುಣಮಟ್ಟದ AI ಚಾಲಿತ ಚಾಟ್ ಅನುಭವಕ್ಕಾಗಿ ಪರಿಗಣನೆಗಳು

ಈ ವಿಭಾಗವು "ಅತ್ಯುತ್ತಮ ಗುಣಮಟ್ಟದ" ಚಾಟ್ ಅಪ್ಲಿಕೇಶನ್‌ಗಳ ಮಾನದಂಡಗಳನ್ನು ವಿವರಿಸುತ್ತದೆ, ಅದರೊಳಗೆ ಕಾರ್ಯಾನ್ವಯ ಮಾಡುವ ಮಾನದಂಡಗಳನ್ನು ಕ್ಯಾಪ್ಚರ್ ಮಾಡುವುದು ಮತ್ತು ಜವಾಬ್ದಾರಿಯುತ AI ತಂತ್ರಜ್ಞಾನ ಬಳಕೆಗೆ ಅನುಗುಣವಾಗಿದೆ.

### ಚಾಟ್ अनुप्रयोगಗಳಲ್ಲಿ ಜವಾಬ್ದಾರಿಯುತ AI ಅಭ್ಯಾಸಗಳನ್ನು ಜಾರಿಮಾಡುವುದು

Microsoft ನ ಜವಾಬ್ದಾರಿಯುತ AI ನ ದೃಷ್ಟಿಕೋನವು AI ಅಭಿವೃದ್ಧಿ ಮತ್ತು ಬಳಕೆಗೆ ಮಾರ್ಗದರ್ಶನ ನೀಡುವ ಆರು ತತ್ವಗಳನ್ನು ಗುರುತಿಸಿದೆ. ಕೆಳಗಿನವು ತತ್ವಗಳು, ಅವುಗಳ ವ್ಯಾಖ್ಯಾನ ಮತ್ತು ಚಾಟ್ ಡೆವಲಪರ್ ಪರಿಗಣಿಸಬೇಕಾದ ವಿಷಯಗಳು ಹಾಗೂ ಅವುಗಳನ್ನು ಗಂಭೀರವಾಗಿ ಯಾಕೆ ತೆಗೆದುಕೊಳ್ಳಬೇಕು ಎಂಬುದನ್ನು ನೀಡಿದೆ.
