# ಸುಧಾರಿತ ವೈಶಿಷ್ಟ್ಯಗಳು ಮತ್ತು ಸುಧಾರಣೆಗಳ ಮಾರ್ಗದರ್ಶಿ

ಈ ಡಾಕ್ಯುಮೆಂಟ್ ಜెనರೇಟಿವ್ AI for Beginners ಪಠ್ಯಕ್ರಮಕ್ಕಾಗಿ ಶಿಫಾರಸು ಮಾಡಿದ ಸುಧಾರಣೆಗಳು ಮತ್ತು ಅಭಿವೃದ್ಧಿಗಳನ್ನು ವಿವರಿಸುತ್ತದೆ, ಸಂಗ್ರಹಿತ ಕೋಡ್ ಪರಿಶೀಲನೆ ಮತ್ತು ಕೈಗಾರಿಕಾ ಉತ್ತಮ ಅಭ್ಯಾಸಗಳ ವಿಶ್ಲೇಷಣೆಯ ಆಧಾರದ ಮೇಲೆ.

## ಕಾರ್ಯನಿರ್ವಹಣಾ ಸಾರಾಂಶ

ಕೋಡ್‌ಬೇಸ್ ಅನ್ನು ಸುರಕ್ಷತೆ, ಕೋಡ್ ಗುಣಮಟ್ಟ ಮತ್ತು ಶೈಕ್ಷಣಿಕ ಪರಿಣಾಮಕಾರಿತ್ವದ ದೃಷ್ಟಿಯಿಂದ ವಿಶ್ಲೇಷಿಸಲಾಗಿದೆ. ಈ ಡಾಕ್ಯುಮೆಂಟ್ ತಕ್ಷಣದ ದೋಷನಿರ್ವಹಣೆ, ಹತ್ತಿರವಿರುವ ಭವನಗಳ ಸುಧಾರಣೆ ಮತ್ತು ಭವಿಷ್ಯದಲ್ಲಿ ಕಾರ್ಯಕ್ಷಮತೆಗಾಗಿ ಶಿಫಾರಸುಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ.

## 4. API ನವೀಕರಣ

### 4.1 ಪರಿಹರಿಸಲಾಗಿದdeprecated API ಮಾದರಿಗಳು ( ವರ್ಗಾವಣೆ ಪೂರ್ಣಗೊಂಡಿದೆ)

ಎಲ್ಲಾ ಪೈಥಾನ್ ಮತ್ತು ಟೈಪ್ಸ್ರಿಪ್ಟ್ **ಚಾಟ್** ಮಾದರಿಗಳು ಚಾಟ್ ಕಂಪ್ಲೀಷನ್ಸ್ API ನಿಂದ **ಪ್ರತಿಕ್ರಿಯೆಗಳು API** ಗೆ ವರ್ಗಾಯಿಸಲಾಗಿದೆ (`client.responses.create(...)` → `response.output_text`).

| ಹಳೆ ಮಾದರಿ | ಹೊಸ ಮಾದರಿ | ಸ್ಥಿತಿ |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (ಚಾಟ್) | `OpenAI(base_url="<endpoint>/openai/v1/")` (ಪ್ರತಿಕ್ರಿಯೆಗಳು API) | ಪೂರ್ಣಗೊಂಡಿದೆ |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | ಪೂರ್ಣಗೊಂಡಿದೆ |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (ಟೈಪ್ಸ್ರಿಪ್ಟ್) | `openai` ಪ್ಯಾಕೇಜ್ `client.responses.create()` → `response.output_text` | ಪೂರ್ಣಗೊಂಡಿದೆ |
| `df.append()` (ಪಾಂಡಸ್) | `pd.concat()` | ಪೂರ್ಣಗೊಂಡಿದೆ |

> **ಗುರುತಿಸಿ:** `azure-ai-inference` / `@azure-rest/ai-inference` SDK (`client.complete()`) ಬಳಸುವ Microsoft Foundry ಮಾದರಿಗಳು ಮಾದರಿ ಅನ್ವಯ API ಮೇಲೆ ಉಳಿದಿವೆ, ಇದು ಪ್ರತಿಕ್ರಿಯೆಗಳು API ಅನ್ನು ಬೆಂಬಲಿಸುವುದಿಲ್ಲ. `AzureOpenAI()` ಇನ್ನೂ ಮಾನ್ಯವಾಗಿರುವ ಸ್ಥಳಗಳಲ್ಲಿ (ಎಂಬೆಡಿಂಗ್‌ಗಳು ಮತ್ತು ಚಿತ್ರ ರಚನೆ) ಉದ್ದೇಶಿತವಾಗಿ ಉಳಿಸಲಾಗಿದೆ.

### 4.2 ಪ್ರದರ್ಶನಕ್ಕೆ ಹೊಸ API ವೈಶಿಷ್ಟ್ಯಗಳು

1. **ಸಂರಚಿತ ಔಟ್‌ಪುಟ್‌ಗಳು** (OpenAI)
   - JSON ಮೋಡ್
   - ಕಠಿಣ ಸ್ಕೀಮಾಗಳೊಂದಿಗೆ ಫಂಕ್ಷನ್ ಕರೆ

2. **ದೃಶ್ಯ ಸಾಮರ್ಥ್ಯಗಳು**
   - GPT-4o (ದೃಶ್ಯ) ಬಳಸಿ ಚಿತ್ರ ವಿಶ್ಲೇಷಣೆ
   - ಬಹು ಮಾಧ್ಯಮ ಪ್ರಾಂಪ್ಟ್‌ಗಳು

3. **ಪ್ರತಿಕ್ರಿಯೆಗಳು API ಒಳಗಡೆ ಟೂಲ್ಗಳು** (ಪೂರ್ವ Assistants API ದಾಟಿ ಹೋಗುತ್ತದೆ)
   - ಕೋಡ್ ಇಂಟರ್‌ಪ್ರಿಟರ್
   - ಫೈಲ್ ಹುಡುಕು
   - ವೆಬ್ ಹುಡುಕು ಮತ್ತು ಕಸ್ಟಮ್ ಟೂಲ್ಗಳು

## 5. ಮೂಲಸೌಕರ್ಯ ಸುಧಾರಣೆಗಳು

### 5.1 ಸಿಐ/ಸಿಡಿ ಸುಧಾರಣೆಗಳು

[`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml) ನಲ್ಲಿ ಜಾರಿಗೆ ತರುವಿಕೆಯಾಗಿದ್ದು: ಪೈಥಾನ್ ಲಿಂಟಿಂಗ್/ಫಾರ್ಮಾಟಿಂಗ್ (ರಫ್ + ಬ್ಲ್ಯಾಕ್) ಆಧಾರಿತ `shared/` ಉಪಯುಕ್ತತೆ ಮೋಡ್ಯೂಲ್‌ನಲ್ಲಿ ಕಡ್ಡಾಯವಾಗಿ ಜಾರಿಗೆ ಬರುತ್ತದೆ ಮತ್ತು ಪಠ್ಯಕ್ರಮದ ಉಳಿದ ಭಾಗಗಳಲ್ಲಿ ಸಲಹೆ ನೀಡುವಂತೆ, ಜೊತೆಗೆ ಜಾವಾಸ್ಕ್ರಿಪ್ಟ್/ಟೈಪ್ಸ್ರಿಪ್ಟ್‌ಗೆ ಸಲಹೆ ನೀಡುವ ESLint ಪಾಸ್ ಇದೆ. ಉದಾಹರಣೆಯ ಆಧಾರವಿತ್ತು:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### 5.2 ಸುರಕ್ಷತಾ ಸ್ಕ್ಯಾನಿಂಗ್

[`.github/workflows/security.yml`](../../../.github/workflows/security.yml) ನಲ್ಲಿ ಜಾರಿಗೆ ತರುವಿಕೆಯಾಗಿದ್ದು: ಪೈಥಾನ್ ಮತ್ತು ಜಾವಾಸ್ಕ್ರಿಪ್ಟ್/ಟೈಪ್ಸ್ರಿಪ್ಟ್ ಗೆ CodeQL ವಿಶ್ಲೇಷಣೆ (ಪುಶ್, ಪುಲ್ ರಿಕ್ವೆಸ್ಟ್ ಮತ್ತು ವಾರದ ವೇಳಾಪಟ್ಟಿಯಲ್ಲಿ) ಜೊತೆಗೆ ಡಿಪೆಂಡೆನ್ಸಿ ಪರಿಶೀಲನೆಗಳು. ಉದಾಹರಣೆಯ ಆಧಾರವಿತ್ತು:

```yaml
# .github/workflows/security.yml
name: Security Scan

on: [push, pull_request]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: javascript, python
      - uses: github/codeql-action/analyze@v3

  dependency-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
```

## 6. ಡೆವಲಪರ್ ಅನುಭವ ಸುಧಾರಣೆಗಳು

### 6.1 ಡೆವ್‌ಕಂಟೈನರ್ ಸುಧಾರಣೆಗಳು

[`.devcontainer/devcontainer.json`](../../../.devcontainer/devcontainer.json) ಮತ್ತು [`.devcontainer/post-create.sh`](../../../.devcontainer/post-create.sh) ಯಲ್ಲಿ ಜಾರಿಗೆ ತರುವಿಕೆಯಾಗಿದ್ದು: ಕಂಟೈನರ್ ಈಗ Pylance, Black ಫಾರ್ಮಾಟರ್, Ruff, ESLint, Prettier, ಮತ್ತು Copilot ವಿಸ್ತರಣೆಗಳನ್ನು ಒದಗಿಸುತ್ತದೆ, ಉಳಿಸುವಾಗ ಫಾರ್ಮಾಟ್-ಆನ್-ಸೇವ್ ಅನ್ನು ರೆಪೋದ Black/Prettier ಕಾಂಫಿಗರೇಶನ್ ಗೆ ಸಂಪರ್ಕಿಸುತ್ತದೆ, ಮತ್ತು ಡೆವಲಪರ್ ಟೂಲಿಂಗ್ (`ruff`, `black`, `mypy`, `pytest`) ಅನ್ನು ಇನ್ಸ್‌ಟಾಲ್ ಮಾಡುತ್ತದೆ ಹೀಗಾಗಿ [ಕೋಡ್-ಗುಣಮಟ್ಟ ವರ್ಕ್‌ಫ್ಲೋ](../../../.github/workflows/code-quality.yml) ಅನ್ನು ಸ್ಥಳೀಯವಾಗಿ ಪುನರ್‌ರಚಿಸಬಹುದು. `mcr.microsoft.com/devcontainers/universal` ಬೇಸ್ ಚಿತ್ರ ಈಗಾಗಲೇ ಪೈಥಾನ್ ಮತ್ತು ನೋಡ್ ಒದಗಿಸುತ್ತದೆ, ಆದ್ದರಿಂದ ಹೆಚ್ಚಿನ ವೈಶಿಷ್ಟ್ಯಗಳ ಅಗತ್ಯವಿಲ್ಲ. ಉದಾಹರಣೆಯ ಆಧಾರವಿತ್ತು:

```json

  "image": "mcr.microsoft.com/devcontainers/universal:2",
  "features": {
    "ghcr.io/devcontainers/features/python:1": {
      "version": "3.11"
    },
    "ghcr.io/devcontainers/features/node:1": {
      "version": "20"
    }
  },
  "customizations": {
    "vscode": {
      "extensions": [
        "ms-python.python",
        "ms-python.vscode-pylance",
        "ms-toolsai.jupyter",
        "dbaeumer.vscode-eslint",
        "esbenp.prettier-vscode",
        "github.copilot"
      ],
      "settings": {
        "python.formatting.provider": "black",
        "editor.formatOnSave": true
      }
    }
  },
  "postCreateCommand": "pip install -e .[dev] && npm install"
}
```

## 8. ಕಾರ್ಯಕ್ಷಮತಾ ಸುಧಾರಣೆಗಳು

### 8.1 ಕೋಡ್ ಮಟ್ಟದ ಸುಧಾರಣೆಗಳು

1. **ಆಸಿಂಕ್/ಅವೆಟ್ ಮಾದರಿಗಳು**
   - ಬ್ಯಾಚ್ ಪ್ರೊಸೆಸಿಂಗ್‌ಗಾಗಿ ಆಸಿಂಕ್ ಉದಾಹರಣೆಗಳನ್ನು ಸೇರಿಸಿ
   - ಸಮಕಾಲೀನ API ಕರೆಗಳನ್ನು ಪ್ರದರ್ಶನ ಮಾಡು

2. **ಕ್ಯಾಶಿಂಗ್ ತಂತ್ರಗಳು**
   - ಎಂಬೆಡಿಂಗ್ ಕ್ಯಾಶಿಂಗ್ ಉದಾಹರಣೆಗಳನ್ನು ಸೇರಿಸಿ
   - ಪ್ರತಿಕ್ರಿಯೆ ಕ್ಯಾಶಿಂಗ್ ಮಾದರಿಗಳನ್ನು ಪ್ರದರ್ಶನ

3. **ಟೋಕೆನ್ tốiಮೈಸೇಶನ್**
   - ಟಿಕ್‌ಟೋಕನ್ ಬಳಕೆ ಉದಾಹರಣೆಗಳನ್ನು ಸೇರಿಸಿ
   - ಪ್ರಾಂಪ್ಟ್ ಸಮಾತಲನ ತಂತ್ರಗಳನ್ನು ಪ್ರದರ್ಶನ

### 8.2 ವೆಚ್ಚ tốiಮೈಸೇಶನ್ ಉದಾಹರಣೆಗಳು

ಉದಾಹರಣೆಗಳನ್ನು ಸೇರಿಸಿ:
- ಕಾರ್ಯ ಸಂಕೀರ್ಣತೆ ಆಧರಿಸಿ ಮಾದರಿ ಆಯ್ಕೆ
- ಟೋಕೆನ್ ಕಾರ್ಯಕ್ಷಮತೆಯಿಗಾಗಿ ಪ್ರಾಂಪ್ಟ್ ಎಂಜಿನಿಯರಿಂಗ್
- ಬಹು ಪ್ರಮಾಣದ ಕಾರ್ಯಾಚರಣೆಗೆ ಬ್ಯಾಚ್ ಪ್ರೊಸೆಸಿಂಗ್

## 9. ಪ್ರವೇಶಸಾಧ್ಯತೆ ಮತ್ತು ಅಂತರರಾಷ್ಟ್ರೀಯೀಕರಣ

### 9.1 ಪ್ರಸ್ತುತ ಭಾಷಾಂತರ ಸ್ಥಿತಿ

ಎಲ್ಲಾ ಭಾಷಾಂತರಗಳು **ಪೂರ್ಣ** ಆಗಿವೆ ಮತ್ತು ಸ್ವಯಂಚಾಲಿತವಾಗಿ [Azure Co-op Translator](https://github.com/Azure/co-op-translator?WT.mc_id=academic-105485-koreyst) ಮೂಲಕ ಉತ್ಪಾದನೆಯಾಗಿವೆ, ಇದು ಪಠ್ಯಕ್ರಮದ 50+ ಭಾಷಾ ಆವೃತ್ತಿಗಳನ್ನು ಇಂಗ್ಲಿಷ್ ಮೂಲದೊಂದಿಗೆ ಸಮನ್ವಯಗೊಳಿಸುತ್ತದೆ. ಭಾಷಾಂತರಿತ ವಿಷಯ `translations/` ಅಡಿಯಲ್ಲಿ ಮತ್ತು ಸ್ಥಳೀಯ ಚಿತ್ರಗಳು `translated_images/` ಅಡಿಯಲ್ಲಿ ಇವೆ; ಲಭ್ಯವಿರುವ ಭಾಷೆಗಳ ಸಂಪೂರ್ಣ ಪಟ್ಟಿ ರೆಪೋಸ್‌ಟರಿ README ಶೀರ್ಷಿಕೆಯಲ್ಲಿ ಪ್ರಕಟಿಸಲಾಗಿದೆ.

| ಅಂಶ | ಸ್ಥಿತಿ |
|--------|--------|
| ಭಾಷಾಂತರ ವಿಸ್ತರಣೆ | ಸಂಪೂರ್ಣ — 50+ ಭಾಷೆಗಳು, ಎಲ್ಲಾ ಪಾಠಗಳು |
| ಭಾಷಾಂತರ ವಿಧಾನ | [Azure Co-op Translator](https://github.com/Azure/co-op-translator?WT.mc_id=academic-105485-koreyst) ಮೂಲಕ ಸ್ವಯಂಚಾಲಿತ |
| ಇಂಗ್ಲಿಷ್ ಮೂಲದೊಂದಿಗೆ ಸಮನ್ವಯದಲ್ಲಿ ಇಡಲಾಗಿದೆ | ಹೌದು — ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪುನರ್ಜನೆರಣೆ ಮಾಡಲಾಗಿದೆ |

### 9.2 ಪ್ರವೇಶಸಾಧ್ಯತೆ ಸುಧಾರಣೆಗಳು

1. ಎಲ್ಲಾ ಚಿತ್ರಗಳಿಗೆ alt ಪಠ್ಯ ಸೇರಿಸಿ
2. ಕೋಡ್ ಮಾದರಿಗಳಿಗೆ ಸರಿಯಾದ ಸಿಂಟ್ಯಾಕ್ಸ್ ಹೈಲೈಟಿಂಗ್ ಖಚಿತಪಡಿಸಿ
3. ಎಲ್ಲಾ ವೀಡಿಯೊ ವಿಷಯಗಳಿಗೆ ವೀಡಿಯೊ ಲಿಪ್ಯಂತರಗಳನ್ನು ಸೇರಿಸಿ
4. ಬಣ್ಣ ವೈರುದ್ಧ್ಯವು WCAG ಮಾರ್ಗಸೂಚಿಗಳನ್ನು ಪೂರೈಸುವಂತೆ ನಿರ್ಧರಿಸಿ
