3. **`.env` ಫೈಲ್ ಸಂಪಾದಿಸಿ**: `.env` ಫೈಲ್ ಅನ್ನು ಟೆಕ್ಸ್ಟ್ ಸಂಪಾದಕ (ಉದಾ: VS Code, ನೋಟ್ಪ್ಯಾಡ್++, ಅಥವಾ ಯಾವುದೇ ಸಂಪಾದಕ) ನಲ್ಲಿ ತೆರೆಯಿರಿ. ಫೈಲ್‌ಗೆ ಕೆಳಗಿನ ಸಾಲುಗಳನ್ನು ಸೇರಿಸಿ, ಪ್ರಾಸಂಗಿಕ Microsoft Foundry Models ಎಂಡ್‌ಪಾಯಿಂಟ್ ಮತ್ತು ಕೀ ಬದಲಾಯಿಸಿ (ಹಾಗೇ ಪಡೆಯಲು [`providers.md`](03-providers.md) ನೋಡಿ):

   > **ಗಮನಿಸಿ:** GitHub Models (ಮತ್ತು ಅದರ `GITHUB_TOKEN` ವ್ಯತ್ಯಯ) ಜುಲೈ 2026 ಕೊನೆಯಲ್ಲಿ ನಿವೃತ್ತಿಯಾಗುತ್ತದೆ. ಬದಲಿಗೆ [Microsoft Foundry Models](https://ai.azure.com/catalog/models?WT.mc_id=academic-105485-koreyst) ಬಳಸಿರಿ.

   ```env
   AZURE_INFERENCE_ENDPOINT=your_foundry_endpoint_here
   AZURE_INFERENCE_CREDENTIAL=your_foundry_api_key_here
   ```

4. **ಫೈಲ್ ಉಳಿಸಿ**: ಬದಲಾವಣೆಗಳನ್ನು ಉಳಿಸಿ ಮತ್ತು ಟೆಕ್ಸ್ಟ್ ಸಂಪಾದಕವನ್ನು փակಿಸಿ.

5. **`python-dotenv` ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಿ**: ನೀವು ಇನ್ನೂ ಮಾಡಿಲ್ಲದಿದ್ದರೆ, `.env` ಫೈಲ್‌ನಿಂದ ಪರಿಸರ ವ್ಯತ್ಯಯಗಳನ್ನು ನಿಮ್ಮ Python ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಲೋಡ್ ಮಾಡಲು `python-dotenv` ಪ್ಯಾಕೇಜನ್ನು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಬೇಕಾಗುತ್ತದೆ. ನೀವು `pip` ಬಳಸಿ ಇದನ್ನು ಇನ್‌ಸ್ಟಲ್ ಮಾಡಬಹುದು:

   ```bash
   pip install python-dotenv
   ```

6. **ನಿಮ್ಮ Python ಸ್ಕ್ರಿಪ್ಟ್‌ನಲ್ಲಿ ಪರಿಸರ ವ್ಯತ್ಯಯಗಳನ್ನು ಲೋಡ್ ಮಾಡಿ**: ನಿಮ್ಮ Python ಸ್ಕ್ರಿಪ್ಟ್‌ನಲ್ಲಿ `.env` ಫೈಲ್‌ನಿಂದ ಪರಿಸರ ವ್ಯತ್ಯಯಗಳನ್ನು ಲೋಡ್ ಮಾಡಲು `python-dotenv` ಪ್ಯಾಕೇಜನ್ನು ಉಪಯೋಗಿಸಿ:

   ```python
   from dotenv import load_dotenv
   import os

   # .env ಫೈಲಿನಿಂದ ಪರಿಸರ ಬದಲಾಗುವದೇರ್ಗಳನ್ನು ಲೋಡ್ ಮಾಡಿ
   load_dotenv()

   # Microsoft Foundry Models ಬದಲಾಗುವವೇರ್ಗಳನ್ನು ಪ್ರವೇಶಿಸಿ
   endpoint = os.getenv("AZURE_INFERENCE_ENDPOINT")
   token = os.getenv("AZURE_INFERENCE_CREDENTIAL")

   print(endpoint)
   ```

ಅದೇವು! ನೀವು ಯಶಸ್ವಿಯಾಗಿ `.env` ಫೈಲ್ ನಿರ್ಮಿಸಿದ್ದು, ನಿಮ್ಮ Microsoft Foundry Models ಪ್ರಮಾಣಪತ್ರಗಳನ್ನು ಸೇರಿಸಿದ್ದೀರಿ ಮತ್ತು ಅವುಗಳನ್ನು Python ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಲೋಡ್ ಮಾಡಿದ್ದೀರಿ.

## ಐಚ್ಛಿಕ алхೆಗಳು

### ಮಿನಿಕೊಂಡಾ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡುವುದು

[ಮಿನಿಕೊಂಡಾ](https://conda.io/en/latest/miniconda.html?WT.mc_id=academic-105485-koreyst) [ಕೊಂಡಾ](https://docs.conda.io/en/latest?WT.mc_id=academic-105485-koreyst), Python ಮತ್ತು ಕೆಲವು ಪ್ಯಾಕೇಜ್‌ಗಳನ್ನು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಲು ಲೈಟ್ ವೇಟ್ ಇನ್‌ಸ್ಟಾಲರ್ ಆಗಿದೆ.
ಕೊಂಡಾ ಸ್ವತಃ ಒಂದು ಪ್ಯಾಕೇಜ್ ಮ್ಯಾನ್‌ಜರ್ ಆಗಿದ್ದು, ಯಂತ್ರ ವರ್ಚುಯಲ್ ಪರಿಸರಗಳು ಮತ್ತು ಪ್ಯಾಕೇಜ್‌ಗಳ ನಡುವೆ ಸುಲಭವಾಗಿ ಸೆಟ್ ಅಪ್ ಮತ್ತು ಬದಲಾವಣೆಯನ್ನು ಮಾಡುತ್ತದೆ. ಜೊತೆಗೆ `pip` ನಲ್ಲಿ ಲಭ್ಯವಿಲ್ಲದ ಪ್ಯಾಕೇಜ್‌ಗಳಿಗೆ ಸಹ ಸೌಲಭ್ಯ ನೀಡುತ್ತದೆ.

ನೀವು [MiniConda ಇನ್‌ಸ್ಟಾಲೇಶನ್ ಗೈಡ್](https://docs.anaconda.com/free/miniconda/#quick-command-line-install?WT.mc_id=academic-105485-koreyst) ಪ್ರಕಾರ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಬಹುದು.

ಮಿನಿಕொண்டಾ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಿದ ನಂತರ, ನೀವು ರೆಪೊ ರೈತ (ಕ್ಲೋನ್) ಮಾಡಬೇಕಾಗುತ್ತದೆ (ನೀವು ಇನ್ನೂ ಮಾಡದಿದ್ದರೆ)

ನಂತರ, ನೀವು ವರ್ಚುಯಲ್ ಪರಿಸರವನ್ನು ಸೃಷ್ಟಿಸಬೇಕು. ಕೊಂಡಾ ಬಳಸಿ ಇದನ್ನು ಮಾಡಲು, ಹೊಸ ಪರಿಸರ ಫೈಲ್ (_environment.yml_) ರಚಿಸಿ. ನೀವು ಕೋಡ್ಸ್ಪೇಸಸ್‌ನಲ್ಲಿ ಇದ್ದರೆ, ಇದನ್ನು `.devcontainer` ಡೈರೆಕ್ಟರಿನೊಳಗೆ ರಚಿಸಿ, ಅಂದರೆ `.devcontainer/environment.yml`.

ಕೆಳಗಿನ ಸ್ಲಿಪ್‌ನೊಂದಿಗೆ ನಿಮ್ಮ ಪರಿಸರ ಫೈಲ್ ಅನ್ನು ತುಂಬಿಕೊಳ್ಳಿ:

```yml
name: <environment-name>
channels:
  - defaults
  - microsoft
dependencies:
  - python=<python-version>
  - openai
  - python-dotenv
  - pip
  - pip:
      - azure-ai-ml
```

ನೀವು ಕೊಂಡಾ ಬಳಸುವ ವೇಳೆ ದೋಷಗಳನ್ನು ಎದುರಿಸುತ್ತಿದ್ದರೆ, ಟರ್ಮಿನಲ್‌ನಲ್ಲಿ ಕೆಳಗಿನ ಕಮಾಂಡ್ ಮೂಲಕ Microsoft AI ಗ್ರಂಥಾಲಯಗಳನ್ನು ಕೈಯಿಂದ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಬಹುದು.

```
conda install -c microsoft azure-ai-ml
```

ಪರಿಸರ ಫೈಲ್ ನಾವು ಬೇಕಾದ ಡಿಪೆಂಡೆನ್ಸಿಗಳನ್ನು ಸೂಚಿಸುತ್ತದೆ. `<environment-name>` ಎಂಬುದು ನೀವು ಸೃಷ್ಟಿಸಲು ಬಯಸುವ ಕೊಂಡಾ ಪರಿಸರದ ಹೆಸರು ಮತ್ತು `<python-version>` ನೀವು ಬಳಸಲು ಬಯಸುವ Python ಆವೃತ್ತೆ (ಉದಾ: `3` ಇತ್ತೀಚಿನ ಪ್ರಮುಖ ಆವೃತ್ತೆ).

ಹೊರತುಪಡಿಸಿ, ನೀವು ಕೆಳಗಿನ ಕಮಾಂಡ್‌ಗಳನ್ನು ಪ್ರದಾನಿಸಿ ನಿಮ್ಮ ಕೊಂಡಾ ಪರಿಸರವನ್ನು ಸೃಷ್ಟಿಸಬಹುದು:

```bash
conda env create --name ai4beg --file .devcontainer/environment.yml # .devcontainer ಉಪಪಥವು ಕೇವಲ ಕೋಡ್ಸ್‌ಪೇಸ್ ಸೆಟ್‌ಅಪ್‌ಗಳಿಗೆ ಅನ್ವಯಿಸುತ್ತದೆ
conda activate ai4beg
```

ಯಾವುದೇ ಸಮಸ್ಯೆ ಎದುರಿಸಿದರೆ [Conda environments ಗೈಡ್](https://docs.conda.io/projects/conda/en/latest/user-guide/tasks/manage-environments.html?WT.mc_id=academic-105485-koreyst) ನೋಡಿ.

### Python ಬೆಂಬಲ ವಿಸ್ತರಣೆ ಒಳಗೊಂಡ Visual Studio Code ಉಪಯೋಗಿಸುವುದು

ಈ کور್ಸ್ ಗಾಗಿ ನಾವು [Visual Studio Code (VS Code)](https://code.visualstudio.com/?WT.mc_id=academic-105485-koreyst) ಸಂಪಾದಕ ಮತ್ತು ಅದಕ್ಕೆ ಸಂಬಂಧಿಸಿದ [Python ಬೆಂಬಲ ವಿಸ್ತರಣೆ](https://marketplace.visualstudio.com/items?itemName=ms-python.python&WT.mc_id=academic-105485-koreyst) ಬಳಸುವುದನ್ನು ಶಿಫಾರಸು ಮಾಡುತ್ತೇವೆ. ಇದೊಂದು ಶಿಫಾರಸು ಮಾತ್ರ ಮತ್ತು ನಿರ್ದಿಷ್ಟ ಅಗತ್ಯವಲ್ಲ.

> **ಗಮನಿಸಿ**: کور್ಸ್ ರೆಪೊವನ್ನು VS Code ನಲ್ಲಿ ತೆರೆಯುವುದರಿಂದ, ನೀವು ಪ್ರಾಜೆಕ್ಟ್ ಅನ್ನು ಕಂಟೈನರ್‌ನಲ್ಲಿ ಸೆಟ್ ಅಪ್ ಮಾಡುವ ಆಯ್ಕೆಯನ್ನು ಹೊಂದಿರುತ್ತೀರಿ. ಇದಕ್ಕಾಗಿ ವಿಶೇಷ `.devcontainer` ಡೈರೆಕ್ಟರಿ ಇರುತ್ತದೆ. ಇದರ ಬಗ್ಗೆ ನಂತರ ವಿವರವಾಗಿ.

> **ಗಮನಿಸಿ**: ನೀವು ರೆಪೊ ಕ್ಲೋನ್ ಮಾಡಿ VS Code ನಲ್ಲಿ ತೆರೆಯುತ್ತಿದ್ದಾಗ, ಇದು ಸ್ವಯಂಚಾಲಿತವಾಗಿ Python ಬೆಂಬಲ ವಿಸ್ತರಣೆ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಲು ಸೂಚಿಸುತ್ತದೆ.

> **ಗಮನಿಸಿ**: VS Code ನಿಮಗೆ ರೆಪೊವನ್ನು ಕಂಟೈನರ್‌ನಲ್ಲಿ ಮರುಮುಖ್ಯಗೊಳಿಸಲು ಸೂಚಿಸಿದರೆ, ಓಕೇ ಮಾಡಿ ಅಲ್ಲ ಎಂದು ನಿರಾಕರಿಸಿ, ಆಗ ನೀವು ಲೋಕಲ್ ಇನ್‌ಸ್ಟಾಲ್ಡ್ Python ಆವೃತ್ತೆ ಬಳಸಬಹುದು.

### ಬ್ರೌಸರ್‌ನಲ್ಲಿ Jupyter ಬಳಸುವುದು

ನೀವು ನಿಮ್ಮ ಬ್ರೌಸರ್‌ನಲ್ಲಿ ನೇರವಾಗಿ [Jupyter ಪರಿಸರ](https://jupyter.org?WT.mc_id=academic-105485-koreyst) ಬಳಸಿ ಪ್ರಾಜೆಕ್ಟ್ನಲ್ಲಿ ಕೆಲಸ ಮಾಡಬಹುದು. ಕ್ಲಾಸಿಕ್ Jupyter ಮತ್ತು [Jupyter Hub](https://jupyter.org/hub?WT.mc_id=academic-105485-koreyst) ಎರಡು ಅಭಿವೃದ್ಧಿಗೆ ಸುಲಭ ಕೈಗಾರಿಕೆಗಳನ್ನು ಒದಗಿಸುತ್ತವೆ.

ಲೋಕಲಿ Jupyter ಪ್ರಾರಂಭಿಸಲು, ಟರ್ಮಿನಲ್/ಕಮಾಂಡ್ ಲೈನ್‌ಗೆ ಹೋಗಿ, کور್ಸ್ ಡೈರೆಕ್ಟರಿಗೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡಿ, ಮತ್ತು ಹಾಕಿ:

```bash
jupyter notebook
```

ಅಥವಾ

```bash
jupyterhub
```

ಇದು Jupyter ಇನ್ಸ್ಟನ್ಸ್ ಅನ್ನು ಪ್ರಾರಂಭಿಸುತ್ತದೆ ಮತ್ತು ಪ್ರವೇಶಿಸಲು URL ಕಮಾಂಡ್ ಲೈನ್ ವಿಂಡೋದಲ್ಲಿ ತೆರೆದಿಡುತ್ತದೆ.

ನೀವು URL ಗೆ ಪ್ರವೇಶಿಸಿದ ನಂತರ, ನೀವು کور್ಸ್ ರೂಪರೇಖೆಯನ್ನು ನೋಡಬಹುದು ಮತ್ತು ಯಾವುದೇ `*.ipynb` ಫೈಲ್‌ಗೆ ನ್ಯಾವಿಗೇಟ್ ಮಾಡಬಹುದು. ಉದಾ: `08-building-search-applications/python/oai-solution.ipynb`.

### ಕಂಟೈನರ್‌ನಲ್ಲಿ ಕಾರ್ಯನಿರ್ವಹಿಸುವುದು

ನಿಮ್ಮ ಕಂಪ್ಯೂಟರ್ ಅಥವಾ ಕೋಡ್ಸ್ಪೇಸ್‌ನಲ್ಲಿ ಎಲ್ಲವನ್ನೂ ಸೆಟ್ ಅಪ್ ಮಾಡುವ ಬದಲಿ, [ಕಂಟೈನರ್](../../../00-course-setup/<https:/en.wikipedia.org/wiki/Containerization_(computing)?WT.mc_id=academic-105485-koreyst>) ಬಳಸಿ ಹೆಚ್ಚು ಸುಲಭ. ವಿಶೇಷ `.devcontainer` ಫೋಲ್ಡರ್ VS Codeಗೆ ಪ್ರಾಜೆಕ್ಟ್ ಅನ್ನು ಕಂಟೈನರ್‌ನಲ್ಲಿ ಸೆಟ್ ಅಪ್ ಮಾಡಲು ಸಾಧ್ಯವಾಗಿಸುತ್ತದೆ. Codespaces ಹೊರತುಪಡಿಸಿ, ಇದಕ್ಕಾಗಿ ಡೋಕರ್ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಬೇಕಾಗುತ್ತದೆ, ಮತ್ತು ಸಾಂಪ್ರದಾಯಿಕವಾಗಿ ಇದು ಸ್ವಲ್ಪ ಕಾರ್ಯಶೀಲವಾಗಿದೆ, ಆದ್ದರಿಂದ ಕಂಟೈನರ್‌ಗಳೊಂದಿಗೆ ಅನುಭವವಿರುವವರಿಗೆ ಮಾತ್ರ ಶಿಫಾರಸು ಮಾಡಲಾಗಿದೆ.

GitHub Codespaces ಬಳಸುವಾಗ ನಿಮ್ಮ API ಕಿಗಳನ್ನು ಸುರಕ್ಷಿತವಾಗಿಡಲು Codespace ರಹಸ್ಯಗಳನ್ನು ಬಳಸುವುದು ಒಂದು ಉತ್ತಮ ವಿಧಾನ. ಈ ಕುರಿತಾಗಿ [Codespaces ರಹಸ್ಯ ನಿರ್ವಹಣಾ](https://docs.github.com/en/codespaces/managing-your-codespaces/managing-secrets-for-your-codespaces?WT.mc_id=academic-105485-koreyst) ಮಾರ್ಗದರ್ಶಿಯನ್ನು ಅನುಸರಿಸಿ.
