## ಸಮಸ್ಯೆ ಪರಿಹಾರ

| ಲಕ್ಷಣ                                   | ಪರಿಹಾರ                                                             |

| VS Code “Dev container mounting…” ತೋರಿಸುತ್ತದೆ   | ಬ್ರೌಸರ್ ಟ್ಯಾಬ್ ರిఫ್ರೆಷ್ ಮಾಡಿ — Codespaces ಕೆಲವೊಮ್ಮೆ ಸಂಪರ್ಕವನ್ನು ಕಳೆದುಕೊಳ್ಳುತ್ತದೆ   |
| ನೋಟ್ಬುಕ್ ಕರ್ಣೆಲ್ ಇಲ್ಲ                  | ನೋಟ್ಬುಕ್ ಮೆನು ➜ **Kernel ▸ Select Kernel ▸ Python 3**           |

   ```bash

   ```

   ವಿಂಡೋಸ್:

   ```cmd
   echo . > .env
   ```

3. **`.env` ಫೈಲ್ ಅನ್ನು ಸಂಪಾದಿಸಿ**: `.env` ಫೈಲ್ ಅನ್ನು ಪಠ್ಯ ಸಂಪಾದಕ (ಉದಾಹರಣೆಗೆ, VS Code, ನೋಟ್ಪ್ಯಾಡ್++, ಅಥವಾ ಬೇರೆ ಯಾವುದೇ ಸಂಪಾದಕ) ನಲ್ಲಿ ಹೋಲಿಕೊಳ್ಳಿ. ಕೆಳಗಿನ ಸಾಲನ್ನು ಸೇರಿಸಿ, `your_github_token_here` ನಿಮ್ಮ ನಿಜವಾದ GitHub ಟೋಕನ್‌ದಿಂದ ಬದಲಾಯಿಸಿ:

   ```env
   GITHUB_TOKEN=your_github_token_here
   ```

4. **ಫೈಲ್ ಉಳಿಸಿ**: ಬದಲಾವಣೆಗಳನ್ನು ಉಳಿಸಿ ಮತ್ತು ಪಠ್ಯ ಸಂಪಾದಕವನ್ನು ಮುಚ್ಚಿ.

5. **`python-dotenv` ಇನ್ಸ್ಟಾಲ್ ಮಾಡಿ**: ನೀವು ಈಗಾಗಲೇ ಇದನ್ನು ಮಾಡಿದಿಲ್ಲದಿದ್ದರೆ, `.env` ಫೈಲ್‌ನಿಂದ ಪರಿಸರ ಪರಿವರ್ತನಗಳನ್ನು ನಿಮ್ಮ ಪೈಥಾನ್ ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಲೋಡ್ ಮಾಡಲು `python-dotenv` ಪ್ಯಾಕೇಜ್ needs to be ಇನ್ಸ್ಟಾಲ್ ಮಾಡಿಕೊಳ್ಳಬೇಕು. ನೀವು ಇದನ್ನು `pip` ಮೂಲಕ ಇನ್ಸ್ಟಾಲ್ ಮಾಡಬಹುದು:

   ```bash
   pip install python-dotenv
   ```

6. **ನಿಮ್ಮ ಪೈಥಾನ್ ಸ್ಕ್ರಿಪ್ಟ್‌ನಲ್ಲಿ ಪರಿಸರ ಪರಿವರ್ತನಗಳನ್ನು ಲೋಡ್ ಮಾಡಿ**: ನಿಮ್ಮ ಪೈಥಾನ್ ಸ್ಕ್ರಿಪ್ಟ್‌ನಲ್ಲಿ, `.env` ಫೈಲ್‌ನಿಂದ ಪರಿಸರ ಪರಿವರ್ತನಗಳನ್ನು ಲೋಡ್ ಮಾಡಲು `python-dotenv` ಪ್ಯಾಕೇಜ್ ಅನ್ನು ಬಳಸಿ:

   ```python
   from dotenv import load_dotenv
   import os

   # .env ಫೈಲ್‌ನಿಂದ ಪರಿಸರ ಚರಗಳನ್ನು ಲೋಡ್ ಮಾಡಿ
   load_dotenv()

   # GITHUB_TOKEN ಚರವನ್ನು ಪ್ರವೇಶಿಸಿ
   github_token = os.getenv("GITHUB_TOKEN")

   print(github_token)
   ```

ಅದಾಗಿದೆ! ನೀವು ಯಶಸ್ವಿಯಾಗಿ `.env` ಫೈಲ್ ರಚಿಸಿದ್ದೀರಿ, ನಿಮ್ಮ GitHub ಟೋಕನ್ ಸೇರಿಸಿದ್ದೀರಿ, ಮತ್ತು ನಿಮ್ಮ ಪೈಥಾನ್ ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಅದನ್ನು ಲೋಡ್ ಮಾಡಿದ್ದೀರಿ.

## ಐಚ್ಛಿಕ ಹಂತಗಳು

### ಮಿನಿಕೊಂಡಾ ಸ್ಥಾಪನೆ

[Miniconda](https://conda.io/en/latest/miniconda.html?WT.mc_id=academic-105485-koreyst) ಒಂದು ಲಘುಸ್ಥಾಪಕವಾಗಿದ್ದು, [Conda](https://docs.conda.io/en/latest?WT.mc_id=academic-105485-koreyst), ಪೈಥಾನ್ ಮತ್ತು ಕೆಲವು ಪ್ಯಾಕೇಜ್‌ಗಳನ್ನು ಸ್ಥಾಪಿಸಲು ಉಪಯೋಗಿಸುತ್ತಾರೆ.
Conda ಸ್ವತಃ ಒಂದು ಪ್ಯಾಕೇಜ್ ನಿರ್ವಾಹಕ, ಇದು ವಿವಿಧ Python [**ವರ್ಚುಯಲ್ ಪರಿಸರಗಳು**](https://docs.python.org/3/tutorial/venv.html?WT.mc_id=academic-105485-koreyst) ಮತ್ತು ಪ್ಯಾಕೇಜ್‌ಗಳನ್ನು ಸುಲಭವಾಗಿ ಸೆಟ್ ಅಪ್ ಮಾಡಿ ಬದಲಾಯಿಸಲು ಸಹಾಯ ಮಾಡುತ್ತದೆ. ಜೊತೆಗೆ, `pip` ಮೂಲಕ ಲಭ್ಯವಿಲ್ಲದ ಪ್ಯಾಕೇಜ್‌ಗಳನ್ನು ಸ್ಥಾಪಿಸಲು ಸಹ ಸಹಕರಿಸುತ್ತದೆ.

ನೀವು [MiniConda ಸ್ಥಾಪನೆ ಮಾರ್ಗದರ್ಶಿಯನ್ನ](https://docs.anaconda.com/free/miniconda/#quick-command-line-install?WT.mc_id=academic-105485-koreyst) ಪಡೆದು ಅದನ್ನು ಸೆಟ್ ಅಪ್ ಮಾಡಬಹುದು.

ಮಿನಿಕೊಂಡಾ ಸ್ಥಾಪಿಸಿದ ನಂತರ, ನೀವು ರೆಪೊಸಿಟೋರಿಯನ್ನು ಕ್ಲೋನ್ ಮಾಡಬೇಕು (ಹើಗೆಲಿ ನೀವು ಇನ್ನೂ ಮಾಡಿಲ್ಲವಾದರೂ)

ಮುಂದೆ, ನೀವು ವರ್ಚುಯಲ್ ಪರಿಸರವನ್ನು ರಚಿಸಬೇಕು. Conda ಬಳಸಿ ಇದನ್ನು ಮಾಡಲು, ಹೊಸ ಪರಿಸರ ಫೈಲ್ (_environment.yml_) ರಚಿಸಿ. ನೀವು Codespaces ಬಳಸುತ್ತಿದ್ದರೆ, ಇದನ್ನು `.devcontainer` ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಮಾಡಿ, ಹಾಗಾಗಿ `.devcontainer/environment.yml`.

ನಿಮ್ಮ ಪರಿಸರ ಫೈಲ್ ಅನ್ನು ಕೆಳಗಿನ ಸ್ನಿಪೆಟ್‌ನೊಂದಿಗೆ ತುಂಬಿಸಿ:

```yml
name: <environment-name>
channels:
  - defaults
  - microsoft
dependencies:
  - python=<python-version>
  - openai
  - python-dotenv
  - pip
  - pip:
      - azure-ai-ml
```

ನೀವು conda ಬಳಸುವಾಗ ದೋಷಗಳು ಎದುರಾಗುತ್ತಿವೆ ಎಂದು ಕಂಡರೆ, ಕೆಳಗಿನ ಕಮಾಂಡ್ ನೊಂದಿಗೆ Microsoft AI ಲೈಬ್ರರಿಗಳನ್ನು ಕೈಯಿಂದ ಇನ್ಸ್ಟಾಲ್ ಮಾಡಬಹುದು ಟರ್ಮಿನಲ್‌ನಲ್ಲಿ.

```
conda install -c microsoft azure-ai-ml
```

ಪರಿಸರ ಫೈಲ್ ನಾವು ಬೇಕಾದ ಅವಲಂಬನೆಗಳನ್ನು ನಿರ್ದಿಷ್ಟಪಡಿಸುತ್ತದೆ. `<environment-name>` ನಿಮ್ಮ Conda ಪರಿಸರಕ್ಕೆ ನೀವು ನೀಡಲು ಬಯಸುವ ಹೆಸರು ಮತ್ತು `<python-version>` ನೀವು ಬಳಸಲು ಬಯಸುವ ಪೈಥಾನ್ ಆವೃತ್ತಿ, ಉದಾಹರಣೆಗೆ, `3` ಪೈಥಾನ್‌ನ ನವೀನ ಪ್ರಮುಖ ಆವೃತ್ತಿ.

ಅದು ಮುಗಿದ ನಂತರ, ನಿಮ್ಮ Conda ಪರಿಸರವನ್ನು ಕೆಳಗಿನ ಕಮಾಂಡ್‌ಗಳನ್ನು ಟರ್ಮಿನಲ್/ಕಮಾಂಡೆಲೈನ್‌ನಲ್ಲಿ ರನ್ ಮಾಡಿ ರಚಿಸಬಹುದು

```bash
conda env create --name ai4beg --file .devcontainer/environment.yml # .devcontainer ಅನುಪಥವು ಕೇವಲ ಕೋಡ್ಸ್ಪೇಸ್ ಸೆಟ್ಅಪ್‌ಗಳಿಗೆ ಅನ್ವಯಿಸುತ್ತದೆ
conda activate ai4beg
```

ಯಾವುದೇ ಸಮಸ್ಯೆ ಎದುರಾದರೆ, [Conda ಪರಿಸರಗಳ ಮಾರ್ಗದರ್ಶಿ](https://docs.conda.io/projects/conda/en/latest/user-guide/tasks/manage-environments.html?WT.mc_id=academic-105485-koreyst) ನೋಡಿ.

### ಪೈಥಾನ್ ಬೆಂಬಲ ವಿಭಾಗದೊಂದಿಗೆ Visual Studio Code பயன்படுத்தುವುದು

ನಾವು ಈ ಕೋರ್ಸ್‌ಗೆ [Visual Studio Code (VS Code)](https://code.visualstudio.com/?WT.mc_id=academic-105485-koreyst) ಸಂಪಾದಕ ಮತ್ತು [Python ಬೆಂಬಲ ವಿಭಾಗ](https://marketplace.visualstudio.com/items?itemName=ms-python.python&WT.mc_id=academic-105485-koreyst) ಅನ್ನು ಇನ್ಸ್ಟಾಲ್ ಮಾಡಿಕೊಳ್ಳುವ ಸಲಹೆಯನ್ನು ನೀಡುತ್ತೇವೆ. ಇದೊಂದು ಶಿಫಾರಸು ಮಾತ್ರ, ಕಡ್ಡಾಯ ಅವಶ್ಯಕತೆ ಅಲ್ಲ.

> **ಗಮನಿಸಿ**: ಕೋರ್ಸ್ ರೆಪೊವನ್ನು VS Code ನಲ್ಲಿ ತೆರೆಯುವ ಮೂಲಕ, ನೀವು ಪ್ರಾಜೆಕ್ಟ್ ಅನ್ನು ಕಂಟೇನರ್ ಒಳಗೆ ಸಿದ್ಧಪಡಿಸಬಹುದು. ಇದಕ್ಕೆ ಕಾರಣವೆಂದರೆ ಕೋರ್ಸ್ ರೆಪೊರ್ಯೂಟರಿಯಲ್ಲಿ ಸಿಗುವ [ವಿಶೇಷ `.devcontainer`](https://code.visualstudio.com/docs/devcontainers/containers?itemName=ms-python.python&WT.mc_id=academic-105485-koreyst) ಡೈರೆಕ್ಟರಿ. ಇದಕ್ಕೆ ಮುಂದಿನ ಅಧ್ಯಾಯದಲ್ಲಿ ಮಾಹಿತಿ ಇದೆ.

> **ಗಮನಿಸಿ**: ನೀವು ರೆಪೊ ಕ್ಲೋನ್ ಮಾಡಿ VS Code ನಲ್ಲಿ ತೆರೆಯಿದಾಗ, Python ಬೆಂಬಲ ವಿಭಾಗವನ್ನು ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಇನ್ಸ್ಟಾಲ್ ಮಾಡಲು ಸೂಚಿಸುತ್ತದೆ.

> **ಗಮನಿಸಿ**: VS Code ನಿಮಗೆ ಈ ರೆಪೊವನ್ನು ಕಂಟೇನರ್‌ನಲ್ಲಿ ಮರುತೆರೆದುಕೊಳ್ಳಲು ಸೂಚಿಸಿದರೆ, ನೀವು ಸ್ಥಳೀಯವಾಗಿ ಇನ್ಸ್ಟಾಲ್ ಮಾಡಿದ ಪೈಥಾನ್ ಆವೃತ್ತಿಯನ್ನು ಬಳಸಲು ಈ ವಿನಂತಿಯನ್ನು ತಿರಸ್ಕರಿಸಿ.

### ಬ್ರೌಸರ್‌ನಲ್ಲಿ Jupyter ಬಳಸುವುದು

ನೀವು ಪ್ರಾಜೆಕ್ಟ್ ಮೇಲೆ ಕೆಲಸ ಮಾಡಲು [Jupyter ವಾತಾವರಣ](https://jupyter.org?WT.mc_id=academic-105485-koreyst) ಅನ್ನು ನಿಮ್ಮ ಬ್ರೌಸರ್‌ನಲ್ಲೇ ಬಳಸಬಹುದು. ಪಾರಂಪರಿಕ Jupyter ಮತ್ತು [Jupyter Hub](https://jupyter.org/hub?WT.mc_id=academic-105485-koreyst) ಸ್ವಯಂಚಾಲಿತ ಪೂರ್ಣಗೊಳಿಸುವಿಕೆ, ಕೋಡ್ ಹೈಲೈಟ್‌ಗಳು ಇತ್ಯಾದಿ ವೈಶಿಷ್ಟ್ಯಗಳನ್ನು ಹೊಂದಿರುವ ಆನಂದಕರ ಡೆವಲಪ್‌ಮೆಂಟ್ ವಾತಾವರಣಗಳನ್ನು ಒದಗಿಸುತ್ತವೆ.

Jupyter ಸ್ಥಳೀಯವಾಗಿ ಪ್ರಾರಂಭಿಸಲು, ಟರ್ಮಿನಲ್/ಕಮಾಂಡೆಲೈನ್‌ಗೆ ಹೋಗಿ, ಕೋರ್ಸ್ ಡೈರೆಕ್ಟರಿಗೆ ನವಿಗೇಟ್ ಮಾಡಿ, ಹಾಗೂ ಕೆಳಗಿನ ಆಜ್ಞೆಯನ್ನು ಚಾಲನೆ ಮಾಡಿರಿ:

```bash
jupyter notebook
```

ಅಥವಾ

```bash
jupyterhub
```

ಇದು Jupyter ಉದಾಹರಣೆಯನ್ನು ಪ್ರಾರಂಭಿಸುತ್ತದೆ ಮತ್ತು ಅದಕ್ಕೆ ಪ್ರವೇಶಿಸುವ URL ಕಮಾಂಡೆಲೈನ್ ವಿಂಡೋದಲ್ಲ göstərಿಸಲಾಗುತ್ತದೆ.

URL ಪ್ರವೇಶಿಸಿದ ಕೂಡಲೇ, ನೀವು ಕೋರ್ಸ್ ರೂಪರೇಖೆಯನ್ನು ನೋಡಬಹುದು ಮತ್ತು ಯಾವುದೇ `*.ipynb` ಫೈಲ್‌ಗೆ ನವಿಗೇಟ್ ಮಾಡಬಹುದು. ಉದಾಹರಣೆಗೆ, `08-building-search-applications/python/oai-solution.ipynb`.

### ಕಂಟೇನರ್‌ನಲ್ಲಿ ಚಾಲನೆ

ನಿಮ್ಮ ಕಂಪ್ಯೂಟರ್ ಅಥವಾ Codespace ನಲ್ಲಿ ಎಲ್ಲವನ್ನೂ ಸಿದ್ಧಪಡಿಸುವ ಬದಲು, ನೀವು [ಕಂಟೇನರ್](https://en.wikipedia.org/wiki/Containerization_%28computing%29?WT.mc_id=academic-105485-koreyst) ಅನ್ನು ಬಳಸಬಹುದು. ಕೋರ್ಸ್ ರೆಪೊರ್ಯೂಟರಿಯ ವಿಶೇಷ `.devcontainer` ಫೋಲ್ಡರ್ VS Code ಗೆ ಪ್ರಾಜೆಕ್ಟ್ ಅನ್ನು ಕಂಟೇನರ್ ಒಳಗೆ ಸಿದ್ಧಪಡಿಸಲು ಅನುಮತಿಸುತ್ತದೆ. Codespaces ಹೊರಗಡೆ, ಇದಕ್ಕಾಗಿ Docker ಸ್ಥಾಪನೆ ಅಗತ್ಯವಿದೆ ಮತ್ತು ಧೈರ್ಯದಿಂದ ಹೇಳುವುದಾದರೆ ಇದು ಸ್ವಲ್ಪ ಹೋಗಬಹುದಾದ ಕೆಲಸವಾಗಿದ್ದು, ಕಂಟೇನರ್‌ಗಳೊಂದಿಗೆ ಕೆಲಸ ಮಾಡುವ ಅನುಭವ ಹೊಂದಿರುವವರಿಗೆ ಮಾತ್ರ ನಾವು ಶಿಫಾರಸು ಮಾಡುತ್ತೇವೆ.

GitHub Codespaces ಬಳಸದಾಗ ನಿಮ್ಮ API ಕೀಲಿಗಳನ್ನು ಸುರಕ್ಷಿತವಾಗಿಡಲು Codespace Secrets ಬಳಕೆ ಅತ್ಯುತ್ತಮ ಮಾರ್ಗವಾಗಿದೆ. ದಯವಿಟ್ಟು [Codespaces ರಹಸ್ಯ ನಿರ್ವಹಣೆ](https://docs.github.com/en/codespaces/managing-your-codespaces/managing-secrets-for-your-codespaces?WT.mc_id=academic-105485-koreyst) ಮಾರ್ಗದರ್ಶಿಯನ್ನು ಅನುಸರಿಸಿ.
