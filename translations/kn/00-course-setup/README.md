# ಈ ಕೋರ್ಸ್‌ ಪ್ರಾರಂಭಿಸುವುದು

ನಾವು ನೀವು ಈ ಕೋರ್ಸ್ ಪ್ರಾರಂಭಿಸಲು ಮತ್ತು ಜನರೇಟಿವ್ AI ಬಳಸಿ ನಿರ್ಮಿಸಲು ಪ್ರೇರಿತರಾಗುವುದನ್ನು ನೋಡಲು ತುಂಬಾ ಉದ್ವೇಗಿತರಾಗಿದ್ದೇವೆ!

ನಿಮ್ಮ ಯಶಸ್ಸನ್ನು ಖಚಿತಪಡಿಸಲು, ಈ ಪುಟದಲ್ಲಿ ಸ್ಥಾಪನೆ ಹಂತಗಳು, ತಾಂತ್ರಿಕ ಅಗತ್ಯಗಳು ಹಾಗೂ ಸಹಾಯ ಬೇಕಾದಲ್ಲಿ ಎಲ್ಲಿಗೆ ಪಡೆಯಬೇಕೆಂಬುದನ್ನು ವಿವರಿಸಲಾಗಿದೆ.

## ಸಮಸ್ಯೆ ಪರಿಹಾರ

| `python: command not found`               | ಟರ್ಮಿನಲ್ ಜೋಡಿಸಲಾಗಿಲ್ಲ; **+** ➜ *bash* ಕ್ಲಿಕ್ ಮಾಡಿ                    |

| ನೋಟ್ಬುಕ್ ಕರ್ಮೆಲ್ ಇಲ್ಲ                    | ನೋಟ್ಬುಕ್ ಮೆನೂ ➜ **Kernel ▸ Select Kernel ▸ Python 3**           |

   ಯುನಿಕ್ಸ್ ಆಧಾರಿತ ವ್ಯವಸ್ಥೆಗಳು:

   ```bash
   touch .env
   ```

   ವಿಂಡೋನ್ಸ್:

   ```cmd
   echo . > .env
   ```

3. **`.env` ಫೈಲ್ ಸಂಪಾದಿಸಿ**: `.env` ಫೈಲ್ ಅನ್ನು ಟೆಕ್ಸ್ಟ್ ಎಡಿಟರ್ (ಉದಾ. VS Code, Notepad++, ಅಥವಾ ಮತ್ತೊಬ್ಬ ಎಡಿಟರ್) ನಲ್ಲಿ ತೆರೆಯಿರಿ. ನಿಮ್ಮ Microsoft Foundry Models ಎಂಡ್‌ಪಾಯಿಂಟ್ ಮತ್ತು ಕೀಲನ್ನು ನಿಜವಾದ ಸ್ಥಳಾಪನಗಳೊಂದಿಗೆ ಬದಲಾಯಿಸಿ ಈ ಕೆಳಗಿನ ಸಾಲುಗಳನ್ನು ಫೈಲಿಗೆ ಸೇರಿಸಿ ([`providers.md`](03-providers.md) ವೀಕ್ಷಿಸಿ ಈ ತರದಿಂದ ಪಡೆಯುವಿಕೆಗಾಗಿ):

   > **ಸೂಚನೆ:** GitHub Models (ಮತ್ತು ಅದರ `GITHUB_TOKEN` ವ್ಯತ್ಯಯ) ಜುಲೈ 2026 ಕೊನೆಯದಾಗಿ ನಿವೃತ್ತಿ ಪಡೆಯಲಿದೆ. ಅದಕ್ಕೆ ಬದಲಾಗಿ [Microsoft Foundry Models](https://ai.azure.com/catalog/models?WT.mc_id=academic-105485-koreyst) ಬಳಸಿ.

   ```env
   AZURE_INFERENCE_ENDPOINT=your_foundry_endpoint_here
   AZURE_INFERENCE_CREDENTIAL=your_foundry_api_key_here
   ```

4. **ಫೈಲ್ ಉಳಿಸಿ**: ಬದಲಾವಣೆಗಳನ್ನು ಉಳಿಸಿ ಮತ್ತು ಟೆಕ್ಸ್ಟ್ ಎಡಿಟರ್ ಅನ್ನು ಮುಚ್ಚಿ.

5. **`python-dotenv` ಇನ್ಸ್ಟಾಲ್ ಮಾಡಿ**: ನೀವು ಈಗಾಗಲೇ ಮಾಡಿಲ್ಲದಿದ್ದರೆ, `.env` ಫೈಲಿನಿಂದ ಪರಿಸರ ಚರಗಳನ್ನು ನಿಮ್ಮ Python ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಲೋಡ್ ಮಾಡಲು `python-dotenv` ಪ್ಯಾಕೇಜ್ ಅನ್ನು ಇನ್ಸ್ಟಾಲ್ ಮಾಡಬೇಕಾಗುತ್ತದೆ. ನೀವು `pip` ಬಳಸಿಕೊಂಡು ಇದನ್ನು ಇನ್ಸ್ಟಾಲ್ ಮಾಡಬಹುದು:

   ```bash
   pip install python-dotenv
   ```

6. **ನಿಮ್ಮ Python ಸ್ಕ್ರಿಪ್ಟ್‌ನಲ್ಲಿ ಪರಿಸರ ಚರಗಳನ್ನು ಲೋಡ್ ಮಾಡಿ**: ನಿಮ್ಮ Python ಸ್ಕ್ರಿಪ್ಟ್‌ನಲ್ಲಿ, `.env` ಫೈಲಿನಿಂದ ಪರಿಸರ ಚರಗಳನ್ನು ಲೋಡ್ ಮಾಡಲು `python-dotenv` ಪ್ಯಾಕೇಜ್ ಬಳಸಿರಿ:

   ```python
   from dotenv import load_dotenv
   import os

   # .env ಫೈಲ್‌ನಿಂದ ಪರಿಸರ ವ್ಯತ್ಯಾಸಗಳನ್ನು ಲೋಡ್ ಮಾಡು
   load_dotenv()

   # ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌಂಡ್‌ರಿ ಮಾದರಿಗಳ ವ್ಯತ್ಯಾಸಗಳಿಗೆ ಪ್ರವೇಶ ಪಡೆಯಿ
   endpoint = os.getenv("AZURE_INFERENCE_ENDPOINT")
   token = os.getenv("AZURE_INFERENCE_CREDENTIAL")

   print(endpoint)
   ```

ಅದೇದು! ನೀವು ಯಶಸ್ವಿಯಾಗಿ `.env` ಫೈಲ್ ಸೃಷ್ಟಿಸಿದ್ದೀರಿ, ನಿಮ್ಮ Microsoft Foundry Models ಕ್ರೆಡೆನ್ಷಿಯಲ್ಸ್ ಸೇರಿಸಿದ್ದೀರಿ, ಮತ್ತು ಅವನ್ನು ನಿಮ್ಮ Python ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಲೋಡ್ ಮಾಡಿದ್ದೀರಿ.

## ಐಚ್ಛಿಕ ಹಂತಗಳು

### ಮಿನಿಕಾಂಡಾ ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡುವುದು

[ಮಿನಿಕಾಂಡಾ](https://conda.io/en/latest/miniconda.html?WT.mc_id=academic-105485-koreyst) ಇದು [ಕಾಂಡಾ](https://docs.conda.io/en/latest?WT.mc_id=academic-105485-koreyst), ಪೈಥಾನ್ ಮತ್ತು ಕೆಲವೊಂದು ಪ್ಯಾಕೇಜುಗಳನ್ನು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡುವ ಲಘು ಸ್ಥಾಪಕ.
ಕಾಂಡಾ ತನ್ನದೇ package manager ಆಗಿದ್ದು, ವಿವಿಧ Python [**ವರ್ಚುವಲ್ ಎನ್ವಿರಾನ್ಮೆಂಟ್‌ಗಳು**](https://docs.python.org/3/tutorial/venv.html?WT.mc_id=academic-105485-koreyst) ಮತ್ತು ಪ್ಯಾಕೇಜುಗಳನ್ನು ಸೆಟ್‌ಅಪ್ ಮತ್ತು ಬದಲಾಯಿಸಲು ಸುಲಭ ಮಾಡುತ್ತದೆ. ಅದು `pip` ಮೂಲಕ ಲಭ್ಯವಿಲ್ಲದ ಪ್ಯಾಕೇಜುಗಳನ್ನು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಲು ಸಹ ಸಹಾಯಕ.

ನೀವು [MiniConda ಇನ್‌ಸ್ಟಾಲ್ ಗೈಡ್](https://docs.anaconda.com/free/miniconda/#quick-command-line-install?WT.mc_id=academic-105485-koreyst) ಅನುಸರಿಸಬಹುದು.

ಮಿನಿಕಾಂಡಾ ಇನ್‌ಸ್ಟಾಲ್ ಆದ ಮೇಲೆ, ನೀವು [ರೆಪೊ](https://github.com/microsoft/generative-ai-for-beginners/fork?WT.mc_id=academic-105485-koreyst) ಕ್ಲೋನ್ ಮಾಡಬೇಕು (ನೀವು ಈಗಾಗಲೇ ಮಾಡಿಲ್ಲದಿದ್ದರೆ)

ನಂತರ, ನೀವು ಒಂದು ವರ್ಚುವಲ್ ಎನ್ವಿರಾನ್ಮೆಂಟ್ ಸೃಷ್ಟಿಸಬೇಕು. ಇದಕ್ಕಾಗಿ ಕಾಂಡಾ ಬಳಸಿ, ಹೊಸ environment.yml ಫೈಲ್ ಸೃಷ್ಟಿಸಿ. ನೀವು Codespaces ಅನ್ನು ಬಳಸುತ್ತಿದ್ದರೆ, ಇದನ್ನು `.devcontainer` ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ನಿರ್ಮಿಸಿ, ಅಂದರೆ `.devcontainer/environment.yml`.

ಕೆಳಗಿನ ಕೇಡನ್ನು ನಿಮ್ಮ environment.yml ಫೈಲಿಗೆ ಸೇರಿಸಿ:

```yml
name: <environment-name>
channels:
  - defaults
  - microsoft
dependencies:
  - python=<python-version>
  - openai
  - python-dotenv
  - pip
  - pip:
      - azure-ai-ml
```

ಕಾಂಡಾ ಬಳಸಿ ಸಮಸ್ಯೆಗಳು ಬಂದರೆ, ನೀವು ಮ್ಯಾನ್ಯುವಲ್ ಆಗಿ ಕೆಳಗಿನ ಟರ್ಮಿನಲ್ ಕಮಾಂಡ್ ಬಳಸಿ Microsoft AI ಲೈಬ್ರರಿಗಳನ್ನು ಇನ್‌ಸ್ಟಾಲ್ ಮಾಡಬಹುದು.

```
conda install -c microsoft azure-ai-ml
```

environment.yml ಫೈಲ್ ನಿಂದ ಅವಲಂಬನೆಗಳನ್ನು ವಿವರಿಸಲಾಗಿದ್ದು, `<environment-name>` ನಿಮ್ಮ ಕಾಂಡಾ ವಾತಾವರಣದ ಹೆಸರು ಮತ್ತು `<python-version>` ನೀವು ಬಳಸಬೇಕಾದ ಪೈಥಾನ್ ಸಂಸ್ಕರಣೆಯ ಸಂಖ್ಯೆ, ಉದಾಹರಣೆಗೆ `3` ಎಂದು.

ಇದನ್ನೆಲ್ಲಾ ಮಾಡಿದ ಮೇಲೆ, ಕೆಳಗಿನ ಕಮಾಂಡ್‌ಗಳನ್ನು ಟರ್ಮಿನಲ್‌ನಲ್ಲಿ ಚಲಾಯಿಸಿ ನಿಮ್ಮ ಕಾಂಡಾ ವಾತಾವರಣ ರಚಿಸಿ

```bash
conda env create --name ai4beg --file .devcontainer/environment.yml # .devcontainer ಉಪಪಥವು ಕೇವಲ Codespace ಸೆಟัป್ಗಳಿಗೆ ಅನ್ವಯಿಸುತ್ತದೆ
conda activate ai4beg
```

ಸಮಸ್ಯೆಗಳಿದ್ದರೆ [ಕಾಂಡಾ ವಾತಾವರಣಗಳ ಗೈಡ್](https://docs.conda.io/projects/conda/en/latest/user-guide/tasks/manage-environments.html?WT.mc_id=academic-105485-koreyst) ನೋಡಿ.

### ಪೈಥಾನ್ ಬೆಂಬಲ ವಿಸ್ತಾರವನ್ನು ಹೊಂದಿದ ವಿಸುಯಲ್ ಸ್ಟುಡಿಯೋ ಕೋಡ್ ಬಳಕೆ

ಈ ಕೋರ್ಸ್‌ಗೆ ನಾವು [Visual Studio Code (VS Code)](https://code.visualstudio.com/?WT.mc_id=academic-105485-koreyst) ಎಡಿಟರ್ ಮತ್ತು [Python ಬೆಂಬಲ ವಿಸ್ತಾರ (extension)](https://marketplace.visualstudio.com/items?itemName=ms-python.python&WT.mc_id=academic-105485-koreyst) ಬಳಸುವುದನ್ನು ಶಿಫಾರಸು ಮಾಡುತ್ತೇವೆ. ಆದರೆ ಇದು ಕಡ್ಡಾಯವಲ್ಲ, ಶಿಫಾರಸು ಮಾತ್ರ.

> **ಸೂಚನೆ**: ಕೋರ್ಸ್ ರೆಪೊವನ್ನು VS Code ನಲ್ಲಿ ತೆರೆಯುವ ಮೂಲಕ, ನೀವು ಪ್ರಾಜೆಕ್ಟ್ ಅನ್ನು ಒಂದೊಂದು ಕಂಟೈನರ್ ಒಳಗೆ ಸೆಟ್ ಅಪ್ ಮಾಡುವ ಐಚ್ಛಿಕತೆಯನ್ನು ಹೊಂದಿರುತ್ತೀರಿ. ಇದು ಈ ರೆಪೊದಲ್ಲಿರುವ ವಿಶಿಷ್ಟ `.devcontainer` ಡೈರೆಕ್ಟರಿಯ ಸಿಗುವುದರಿಂದ ಸಾಧ್ಯ.

> **ಸೂಚನೆ**: ನೀವು ರೆಪೊ ಕ್ಲೋನ್ ಮಾಡಿ VS Code ನಲ್ಲಿ ತೆರೆಯುತ್ತಿದ್ದರೆ, ಸ್ವಯಂಚಾಲಿತವಾಗಿ ಪೈಥಾನ್ ಬೆಂಬಲ ವಿಸ್ತಾರವನ್ನು ಜೋಡಿಸಲು ಸೂಚಿಸಲಾಗುತ್ತದೆ.

ನೀವು ಪ್ರಾಜೆಕ್ಟ್ ಮೇಲೆ ಬ್ರೌಸರ್‌ನಲ್ಲಿ [ಜೂಪಿಟರ್ ಪರಿಸರ](https://jupyter.org?WT.mc_id=academic-105485-koreyst) ಬಳಸಿ ಕೆಲಸ ಮಾಡಬಹುದು. ಕ್ಲಾಸಿಕ್ ಜೂಪಿಟರ್ ಮತ್ತು [ಜೂಪಿಟರ್ ಹಬ್](https://jupyter.org/hub?WT.mc_id=academic-105485-koreyst) ಸ್ವಯಂಚಾಲಿತ ಪೂರ್ಣ, ಕೋಡ್ ಹೈಲೈಟಿಂಗ್ ಮೊದಲಾದ ಬೆಹುಲವಾದ ತಂತ್ರಗಳನ್ನು ಒದಗಿಸುತ್ತವೆ.

ಜೂಪಿಟರ್ ಸ್ಥಳೀಯವಾಗಿ ಪ್ರಾರಂಭಿಸಲು, ಟರ್ಮಿನಲ್ ಅಥವಾ ಕಮಾಂಡ್ ಲೈನ್ ಗೆ ಹೋಗಿ ಕೋರ್ಸ್ ಡೈರೆಕ್ಟರಿಗೆ ನವಿಗೇಟ್ ಮಾಡಿ ಮತ್ತು ಕೆಳಗಿನ ಕಮಾಂಡ್ ಚಲಾಯಿಸಿ:

```bash
jupyter notebook
```


### ಕಂಟೈನರ್‌ನಲ್ಲಿ ರನ್ ಮಾಡುವುದು

ನಿಮ್ಮ ಕಂಪ್ಯೂಟರ್ ಅಥವಾ ಕೋಡ್‌ಸ್ಪೇಸ್‌ನಲ್ಲಿ ಎಲ್ಲವನ್ನು ಸೆಟ್ ಅಪ್ ಮಾಡುವ ಬದಲು, ಒಂದು [ಕಂಟೈನರ್](../../../00-course-setup/<https:/en.wikipedia.org/wiki/Containerization_(computing)?WT.mc_id=academic-105485-koreyst>) ಬಳಸಬಹುದು. ಈ ಕೋರ್ಸ್ ರೆಪೊನಲ್ಲಿರುವ `.devcontainer` ಫೋಲ್ಡರ್ VS Code ಗೆ ಪ್ರಾಜೆಕ್ಟ್ ಅನ್ನು ಕಂಟೈನರ್ ಒಳಗೆ ಸೆಟ್-ಅಪ್ ಮಾಡಲು ಅವಕಾಶ ಮಾಡಿಕೊಡುತ್ತದೆ. Codespaces ಹೊರತು, ಡಾಕರ್ ಇನ್‌ಸ್ಟಾಲ್ ಬೇಕಾಗುತ್ತದೆ ಮತ್ತು ಇದು ಕೆಲವು ಶ್ರಮ ಬೇಕಾಗುತ್ತದೆ; ಅದಕ್ಕಾಗಿ, ಹೀಗೆ ಮಾಡುವವರಿಗೆ ಮಾತ್ರ ನಾವು ಶಿಫಾರಸು ಮಾಡುತ್ತೇವೆ.

GitHub Codespaces ಬಳಸದಾಗ ನಿಮ್ಮ API ಕೀಲಿಗಳನ್ನು ಸುರಕ್ಷಿತವಿಟ್ಟು ಇಟ್ಟುಕೊಳ್ಳಲು Codespace Secrets ಬಳಸುವುದು ಅತ್ಯುತ್ತಮ ಮಾರ್ಗ. ಇದಕ್ಕಾಗಿ ದಯವಿಟ್ಟು [Codespaces ಗುಪ್ತಕ್ಕಾಗಿ ನಿರ್ವಹಣೆ](https://docs.github.com/en/codespaces/managing-your-codespaces/managing-secrets-for-your-codespaces?WT.mc_id=academic-105485-koreyst) ಮಾರ್ಗಸೂಚಿ ಅನುಸರಿಸಿ.
