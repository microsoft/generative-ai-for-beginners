ಹೊಸ ಎನ್‌ವೈರನ್ಮೆಂಟ್ ಫೈಲ್ (*environment.yml*) ಸೃಷ್ಟಿಸಿ. ನೀವು Codespaces ಬಳಸುತ್ತಿರುವರೆ, ಇದನ್ನು `.devcontainer` ಡೈರೆಕ್ಟರಿಯಲ್ಲಿ ಸೃಷ್ಟಿಸಿ, ಅಂದರೆ `.devcontainer/environment.yml`.

### ಹಂತ 2 ನಿಮ್ಮ ಎನ್‌ವೈರನ್ಮೆಂಟ್ ಫೈಲ್ ತುಂಬಿಸಿ

ನಿಮ್ಮ `environment.yml`ಗೆ ಕೆಳಗಿನ ಸಂಕೇತವನ್ನು ಸೇರಿಸಿ


## 3. ನಿಮ್ಮ API ಕೀಲಿಗಳನ್ನು ಸೇರಿಸಿ

ಯಾವುದೇ ವಿನ್ಯಾಸದ ಆಪ್ಲಿಕೇಶನ್ ನಿರ್ಮಿಸುವಾಗ ನಿಮ್ಮ API ಕೀಲಿಗಳನ್ನು ಸುರಕ್ಷಿತವಾಗಿರಿಸುವುದು ಮುಖ್ಯ. ನಾವು ತ:")
`.env` ಫೈಲ್ ನ್ನು ನೇರವಾಗಿ ನಿಮ್ಮ ಕಡೆಯಲ್ಲಿಟ್ಟುಕೊಳ್ಳುವುದನ್ನು ಶಿಫಾರಸು ಮಾಡುತ್ತೇವೆ. ಪಬ್ಲಿಕ್ ರೆಪೋದಲ್ಲಿ ಈ ವಿವರಗಳನ್ನು ಕಂಪಿಟ್ ಮಾಡುವುದರಿಂದ ಭದ್ರತಾ ಸಮಸ್ಯೆಗಳು ಮತ್ತು ಅಪೇಕ್ಷಿತವಾಗದ ವೆಚ್ಚಗಳಾಗುವ ಅಪಾಯ ಉಂಟಾಗಬಹುದು.



1. **ನಿಮ್ಮ ಪ್ರಾಜೆಕ್ಟ್ ಡೈರೆಕ್ಟರಿಗೆ ನವಿಗೇಟ್ ಮಾಡಿ**: ನಿಮ್ಮ ಟರ್ಮಿನಲ್ ಅಥವಾ ಕಮಾಂಡ್ ಪ್ರಾಂಪ್ಟ್ ತೆರೆಯಿರಿ ಮತ್ತು `.env` ಫೈಲ್ ಸೃಷ್ಟಿಸಲು ನೀವು ಬಯಸುವ ಪ್ರಾಜೆಕ್ಟ್ ಮೂಲ ಡೈರೆಕ್ಟರಿಗೆ ನವಿಗೇಟ್ ಮಾಡಿ.

   ```bash
   cd path/to/your/project
   ```

2. **`.env` ಫೈಲ್ ಸೃಷ್ಟಿಸಿ**: ನಿಮ್ಮ ಇಚ್ಛಿತ ಪಠ್ಯ ಸಂಪಾದಕವನ್ನು ಬಳಸಿ `.env` ಎಂಬ ಹೆಸರಿನ ಹೊಸ ಫೈಲ್ ಸೃಷ್ಟಿಸಿ. ನೀವು ಕಮಾಂಡ್ ಲೈನ್ ಬಳಸುತ್ತಿರುವರೆ `touch` (ಯುನಿಕ್ಸ್ ಆಧಾರಿತ ಸಿಸ್ಟಮ್‌ಗಳು) ಅಥವಾ `echo` (ವಿಂಡೋಸ್):

   ಯುನಿಕ್ಸ್ ಆಧಾರಿತ ಸಿಸ್ಟಮ್‌ಗಳು:

   ```bash
   touch .env
   ```

   ವಿಂಡೋಸ್:

   ```cmd
   echo . > .env
   ```

3. **`.env` ಫೈಲ್ ಸಂಪಾದಿಸಿ**: `.env` ಫೈಲ್ ಅನ್ನು ಪಠ್ಯ ಸಂಪಾದಕದಲ್ಲಿ ತೆರೆಯಿರಿ (ಉದಾಹರಣೆಗೆ, VS ಕೋಡ್, ನೋಟ್ಪ್ಯಾಡ್++ ಅಥವಾ ಯಾವುದೇ ಇತರ ಸಂಪಾದಕ). ಹೀಗೆ, ನಿಮ್ಮ Microsoft Foundry ಪ್ರಾಜೆಕ್ಟ್ ಎಂಡ್‌ಪಾಯಿಂಟ್ ಮತ್ತು API ಕೀಲಿಯನ್ನು ಬದಲಾಯಿಸಿ ಕೆಳಗಿನ ಸಾಲುಗಳನ್ನು ಸೇರಿಸಿ:

   ```env
   AZURE_INFERENCE_ENDPOINT=your_foundry_endpoint_here
   AZURE_INFERENCE_CREDENTIAL=your_foundry_api_key_here
   ```

4. **ಫೈಲ್ ಉಳಿಸಿ**: ಬದಲಾವಣೆಗಳನ್ನು ಉಳಿಸಿ ಮತ್ತು ಪಠ್ಯ ಸಂಪಾದಕವನ್ನು ಮುಚ್ಚಿ.

5. **`python-dotenv` ಸ್ಥಾಪಿಸಿ**: ನೀವು ಈಗಾಗಲೇ ಸ್ಥಾಪಿಸದಿದ್ದರೆ, Python ಅಪ್ಲಿಕೇಶನ್‌ಗೆ `.env` ಫೈಲ್‌ನಿಂದ ಎನ್‌ವೈರನ್ಮೆಂಟ್ ವೇರಿಯಬಲ್‌ಗಳನ್ನು ಲೋಡ್ ಮಾಡಲು ನೀವು `python-dotenv` ಪ್ಯಾಕೇಜ್ ಅನ್ನು ಸ್ಥಾಪಿಸಬೇಕಾಗುತ್ತದೆ. ನೀವು `pip` ಬಳಸಿ ಈ ಕೆಳಗಿನಂತೆ ಸ್ಥಾಪಿಸಬಹುದು:

   ```bash
   pip install python-dotenv
   ```

6. **Python ಸ್ಕ್ರಿಪ್ಟ್‌ನಲ್ಲಿ ಎನ್‌ವೈರನ್ಮೆಂಟ್ ವೇರಿಯಬಲ್‌ಗಳನ್ನು ಲೋಡ್ ಮಾಡಿ**: ನಿಮ್ಮ ಪೈಥಾನ್ ಸ್ಕ್ರಿಪ್ಟ್‌ನಲ್ಲಿ, `.env` ಫೈಲಿನಿಂದ ವೇರಿಯಬಲ್‌ಗಳನ್ನು ಲೋಡ್ ಮಾಡಲು `python-dotenv` ಪ್ಯಾಕೇಜ್ ಅನ್ನು ಬಳಸಿ:

   ```python
   from dotenv import load_dotenv
   import os

   # .env ಕಡತದಿಂದ ಪರಿಸರ ಚರಗಳನ್ನು ಲೋಡ್ ಮಾಡಿ
   load_dotenv()

   # ಮೈಕ್ರೋಸಾಫ್ಟ್ ಫೌನ್ಡ್ರಿ ಮಾದರಿಗಳ ಚರಗಳನ್ನು ಪ್ರಾಪ್ತಗೊಳಿಸಿ
   endpoint = os.getenv("AZURE_INFERENCE_ENDPOINT")
   token = os.getenv("AZURE_INFERENCE_CREDENTIAL")

   print(endpoint)
   ```

ಆದುದರಿಂದ! ನೀವು ಯಶಸ್ವಿಯಾಗಿ `.env` ಫೈಲ್ ಸೃಷ್ಟಿಸಿದಿರಿ, ನಿಮ್ಮ Microsoft Foundry ಮಾದರಿಗಳ ಅರ್ಹತೆಗಳನ್ನು ಸೇರಿಸಿದ್ದೀರಿ, ಮತ್ತು ಅವನ್ನು ನಿಮ್ಮ Python ಅಪ್ಲಿಕೇಶನ್‌ಗೆ ಲೋಡ್ ಮಾಡಿದ್ದೀರಿ.
