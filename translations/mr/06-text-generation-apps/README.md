# टेक्स्ट जनरेशन ऍप्लिकेशन्स बनवणे

> _(या धड्याचा व्हिडिओ पाहण्यासाठी वरच्या प्रतिमेवर क्लिक करा)_

या अभ्यासक्रमात तुम्ही आतापर्यंत पाहिले आहे की प्रॉम्प्ट्स सारखी मुख्य संकल्पना आहेत आणि "प्रॉम्प्ट इंजिनिअरिंग" नावाची संपूर्ण शिस्त आहे. ChatGPT, Office 365, Microsoft Power Platform आणि अधिक सारखी अनेक साधने तुमच्यासाठी प्रॉम्प्ट्स वापरून काहीतरी साध्य करण्यासाठी समर्थन देतात.

तुमच्या ऍप्लिकेशनमध्ये अशा अनुभवाची भर घालण्यासाठी, तुम्हाला प्रॉम्प्ट्स, पूर्णता आणि काम करण्यासाठी लायब्ररी निवडण्यासारख्या संकल्पना समजून घेणे आवश्यक आहे. याच अध्यायात तुम्ही ते शिकणार आहात.

## परिचय

या अध्यायात, तुम्ही:

- openai लायब्ररी आणि त्याची मुख्य संकल्पना शिकाल.
- openai वापरून एक टेक्स्ट जनरेशन ऍप तयार कराल.
- टेक्स्ट जनरेशन ऍप तयार करण्यासाठी प्रॉम्प्ट, तापमान आणि टोकन्स सारख्या संकल्पना कशा वापरायच्या ते समजून घ्याल.

## शिकण्याची उद्दिष्टे

या धड्याच्या शेवटी, तुम्ही सक्षम असाल:

- टेक्स्ट जनरेशन ऍप म्हणजे काय हे स्पष्ट करा.
- openai वापरून एक टेक्स्ट जनरेशन ऍप तयार करा.
- तुमच्या ऍपला अधिक किंवा कमी टोकन्स वापरण्यासाठी कॉन्फिगर करा आणि विविध आउटपुटसाठी तापमान बदलू शकता.

## टेक्स्ट जनरेशन ऍप म्हणजे काय?

सामान्यतः जेव्हा तुम्ही ऍप तयार करता तेव्हा त्यामध्ये खालीलप्रमाणे काही प्रकारचे इंटरफेस असते:

- कमांड-आधारित. कन्सोल ऍप्स हे सामान्य ऍप्स आहेत जिथे तुम्ही कमांड टाइप करता आणि ते कार्य पूर्ण करते. उदाहरणार्थ, `git` हे कमांड-आधारित ऍप आहे.
- युजर इंटरफेस (UI). काही ऍप्समध्ये ग्राफिकल युजर इंटरफेस (GUIs) असतात जिथे तुम्ही बटणे क्लिक करता, मजकूर इनपुट करता, पर्याय निवडता आणि बरेच काही.

### कन्सोल आणि UI ऍप्स मर्यादित आहेत

तुम्ही कमांड टाइप करता अशा कमांड-आधारित ऍप्सच्या तुलनेत:

- **ते मर्यादित आहेत**. तुम्ही कोणतेही कमांड टाइप करू शकत नाही, फक्त ऍपला समर्थन देणारे कमांड टाइप करू शकता.
- **भाषा विशिष्ट**. काही ऍप्स अनेक भाषांना समर्थन देतात, परंतु डिफॉल्टनुसार ऍप विशिष्ट भाषेसाठी तयार केलेले आहे, जरी तुम्ही अधिक भाषा समर्थन जोडू शकता.

### टेक्स्ट जनरेशन ऍप्सचे फायदे

मग टेक्स्ट जनरेशन ऍप कसे वेगळे आहे?

टेक्स्ट जनरेशन ऍपमध्ये तुम्हाला अधिक लवचिकता असते, तुम्ही कमांड्सच्या सेट किंवा विशिष्ट इनपुट भाषेवर मर्यादित नसता. त्याऐवजी, तुम्ही नैसर्गिक भाषेचा वापर करून ऍपशी संवाद साधू शकता. आणखी एक फायदा म्हणजे तुम्ही आधीच माहितीच्या विस्तृत कॉर्पसवर प्रशिक्षित केलेल्या डेटा स्रोताशी संवाद साधत आहात, तर पारंपारिक ऍप्स डेटाबेसमध्ये मर्यादित असू शकतात.

### टेक्स्ट जनरेशन ऍपसह मी काय तयार करू शकतो?

तुम्ही अनेक गोष्टी तयार करू शकता. उदाहरणार्थ:

- **चॅटबॉट**. तुमच्या कंपनी आणि तिच्या उत्पादनांबद्दल प्रश्नांची उत्तरे देणारा चॅटबॉट एक चांगला पर्याय असू शकतो.
- **सहायक**. LLMs मजकूर संक्षेपित करणे, मजकुरातून अंतर्दृष्टी मिळवणे, रेज्युमेसारखा मजकूर तयार करणे आणि बरेच काही करण्यास चांगले आहेत.
- **कोड सहाय्यक**. तुम्ही वापरत असलेल्या भाषा मॉडेलवर अवलंबून, तुम्ही कोड लिहिण्यास मदत करणारा कोड सहाय्यक तयार करू शकता. उदाहरणार्थ, तुम्ही GitHub Copilot तसेच ChatGPT सारखा उत्पादन वापरू शकता ज्यामुळे तुम्हाला कोड लिहिण्यात मदत होईल.

## मी कसे सुरुवात करू शकतो?

ठीक आहे, तुम्हाला LLM सह एकत्रित करण्याचा मार्ग शोधणे आवश्यक आहे ज्यामध्ये सहसा खालील दोन पध्दती असतात:

- API वापरा. येथे तुम्ही तुमच्या प्रॉम्प्टसह वेब विनंत्या तयार करत आहात आणि तुम्हाला व्युत्पन्न मजकूर परत मिळतो.
- लायब्ररी वापरा. लायब्ररी API कॉल्स कॅप्सुलेट करण्यात मदत करतात आणि त्यांचा वापर सुलभ करतात.

## लायब्ररी/SDKs

LLMs सह काम करण्यासाठी काही प्रसिद्ध लायब्ररी आहेत जसे की:

- **openai**, ही लायब्ररी तुमच्या मॉडेलशी कनेक्ट करणे आणि प्रॉम्प्ट्स पाठवणे सोपे करते.

त्यानंतर उच्च स्तरावर कार्य करणाऱ्या लायब्ररी आहेत जसे की:

- **Langchain**. Langchain सुप्रसिद्ध आहे आणि Python ला समर्थन देते.
- **Semantic Kernel**. Semantic Kernel ही Microsoft ची लायब्ररी आहे जी C#, Python, आणि Java भाषांना समर्थन देते.

## openai वापरून पहिला ऍप

आम्ही आमचा पहिला ऍप कसा तयार करू शकतो, कोणत्या लायब्ररीची आवश्यकता आहे, किती आवश्यक आहे आणि इत्यादी पाहूया.

### openai स्थापित करा

OpenAI किंवा Azure OpenAI सह संवाद साधण्यासाठी अनेक लायब्ररी उपलब्ध आहेत. C#, Python, JavaScript, Java आणि अधिक भाषांचा वापर करणे शक्य आहे. आम्ही `openai` Python लायब्ररी वापरणे निवडले आहे, म्हणून आम्ही `pip` वापरून ते स्थापित करू.

### एक संसाधन तयार करा

तुम्हाला खालील चरण पूर्ण करणे आवश्यक आहे:

- Azure वर खाते तयार करा [https://azure.microsoft.com/free/](https://azure.microsoft.com/free/?WT.mc_id=academic-105485-koreyst).
- Azure OpenAI साठी प्रवेश मिळवा. [https://learn.microsoft.com/azure/ai-services/openai/overview#how-do-i-get-access-to-azure-openai](https://learn.microsoft.com/azure/ai-services/openai/overview#how-do-i-get-access-to-azure-openai?WT.mc_id=academic-105485-koreyst) वर जा आणि प्रवेशासाठी विनंती करा.

- Python स्थापित करा <https://www.python.org/>
- Azure OpenAI सेवा संसाधन तयार केले आहे. [संसाधन तयार कसे करावे](https://learn.microsoft.com/azure/ai-services/openai/how-to/create-resource?pivots=web-portal?WT.mc_id=academic-105485-koreyst) यासाठी हा मार्गदर्शक पहा.

### API की आणि एंडपॉइंट शोधा

या टप्प्यावर, तुम्हाला तुमच्या `openai` लायब्ररीला कोणती API की वापरायची ते सांगणे आवश्यक आहे. तुमची API की शोधण्यासाठी, तुमच्या Azure OpenAI संसाधनाच्या "Keys and Endpoint" विभागात जा आणि "Key 1" मूल्य कॉपी करा.

आता तुमच्याकडे ही माहिती कॉपी केली आहे, चला लायब्ररीला ते वापरण्यास सांगूया.

> [!NOTE]
> तुमची API की तुमच्या कोडपासून वेगळी ठेवणे योग्य आहे. तुम्ही पर्यावरणीय बदलांचा वापर करून हे करू शकता.
>
> - पर्यावरणीय बदल सेट करा `OPENAI_API_KEY` to your API key.
>   `export OPENAI_API_KEY='sk-...'`

## वेगवेगळ्या प्रकारचे प्रॉम्प्ट्स, वेगवेगळ्या गोष्टींसाठी

आता तुम्ही प्रॉम्प्ट वापरून टेक्स्ट कसे तयार करायचे ते पाहिले आहे. तुम्ही एक प्रोग्राम चालू आणि चालू आहे जो तुम्ही बदलू आणि बदलू शकता आणि वेगवेगळ्या प्रकारचा मजकूर तयार करू शकता.

प्रॉम्प्ट्स सर्व प्रकारच्या कार्यांसाठी वापरले जाऊ शकतात. उदाहरणार्थ:

- **मजकूराचा प्रकार तयार करा**. उदाहरणार्थ, तुम्ही कविता, क्विझसाठी प्रश्न इत्यादी तयार करू शकता.
- **माहिती शोधा**. तुम्ही 'वेब विकासामध्ये CORS म्हणजे काय?' सारखी माहिती शोधण्यासाठी प्रॉम्प्ट्स वापरू शकता.
- **कोड तयार करा**. तुम्ही कोड तयार करण्यासाठी प्रॉम्प्ट्स वापरू शकता, उदाहरणार्थ ईमेल्स सत्यापित करण्यासाठी वापरले जाणारे नियमित अभिव्यक्ती विकसित करणे किंवा संपूर्ण प्रोग्राम तयार करणे, जसे की वेब ऍप?

## अधिक व्यावहारिक उपयोग: एक रेसिपी जनरेटर

कल्पना करा की तुमच्याकडे घरी साहित्य आहेत आणि तुम्हाला काहीतरी शिजवायचे आहे. त्यासाठी तुम्हाला रेसिपीची आवश्यकता आहे. रेसिपी शोधण्याचा एक मार्ग म्हणजे सर्च इंजिन वापरणे किंवा तुम्ही LLM वापरू शकता.

तुम्ही असा प्रॉम्प्ट लिहू शकता:

> "चिकन, बटाटे आणि गाजरांसह डिशसाठी 5 रेसिपी दाखवा. प्रत्येक रेसिपीमध्ये वापरलेले सर्व साहित्य सूचीबद्ध करा"

वरील प्रॉम्प्ट दिल्यास, तुम्हाला अशा प्रकारे प्रतिसाद मिळू शकतो:

हे परिणाम छान आहेत, मला काय शिजवायचे आहे हे माहित आहे. या टप्प्यावर, काय उपयोगी सुधारणा आहेत:

- मला आवडत नसलेल्या किंवा ज्याला मी ऍलर्जी आहे अशा साहित्यांना फिल्टर करणे.
- जर माझ्याकडे सर्व साहित्य घरी नसतील तर खरेदीची यादी तयार करा.

वरील बाबींसाठी, चला एक अतिरिक्त प्रॉम्प्ट जोडूया:

> "कृपया लसूण असलेल्या रेसिपी काढून टाका कारण मला ऍलर्जी आहे आणि त्याऐवजी काहीतरी बदला. तसेच कृपया रेसिपीसाठी खरेदीची यादी तयार करा, विचारात घेऊन की माझ्याकडे चिकन, बटाटे आणि गाजर आधीच घरी आहेत."

आता तुम्हाला नवीन परिणाम मिळाला आहे, म्हणजे:

तुमच्या पाच रेसिपी आहेत, ज्यात लसूणचा उल्लेख नाही आणि तुम्ही आधीच घरी काय आहे हे विचारात घेऊन खरेदीची यादी देखील आहे.

### फिल्टर आणि शॉपिंग यादी जोडून सुधारणा करा

आता आमच्याकडे रेसिपी तयार करण्यास सक्षम असलेला कार्यरत ऍप आहे आणि ते लवचिक आहे कारण ते वापरकर्त्याकडून इनपुटवर अवलंबून आहे, रेसिपींच्या संख्येवर आणि वापरलेल्या साहित्यावर.

ते पुढे सुधारण्यासाठी, आम्हाला खालील गोष्टी जोडायच्या आहेत:

- **साहित्य फिल्टर करा**. आम्हाला आवडत नसलेले किंवा ज्याला आम्हाला ऍलर्जी आहे असे साहित्य फिल्टर करण्यास सक्षम व्हायचे आहे. हे बदल साध्य करण्यासाठी, आम्ही आमच्या विद्यमान प्रॉम्प्टचे संपादन करू शकतो आणि त्याच्या शेवटी फिल्टर अटी जोडू शकतो:

  वरील, आम्ही `{filter}` प्रॉम्प्टच्या शेवटी जोडतो आणि आम्ही वापरकर्त्याकडून फिल्टर मूल्य देखील पकडतो.

  प्रोग्राम चालवण्याचे उदाहरण इनपुट आता असे दिसू शकते:

  जसे तुम्ही पाहू शकता, त्यात दूध असलेल्या कोणत्याही रेसिपी फिल्टर केल्या आहेत. परंतु, जर तुम्ही लॅक्टोज असहिष्णु असाल तर तुम्हाला त्यात चीज असलेल्या रेसिपी देखील फिल्टर करायच्या असू शकतात, त्यामुळे स्पष्ट असणे आवश्यक आहे.

- **शॉपिंग यादी तयार करा**. आम्हाला घरी काय आहे हे विचारात घेऊन खरेदीची यादी तयार करायची आहे.

  या कार्यक्षमतेसाठी, आम्ही एक प्रॉम्प्टमध्ये सर्वकाही सोडवण्याचा प्रयत्न करू शकतो किंवा आम्ही ते दोन प्रॉम्प्ट्समध्ये विभाजित करू शकतो. चला नंतरची पध्दत वापरून पाहूया. येथे आम्ही एक अतिरिक्त प्रॉम्प्ट जोडण्याचे सुचवत आहोत, परंतु ते काम करण्यासाठी, आम्हाला पूर्वीच्या प्रॉम्प्टचा परिणाम नंतरच्या प्रॉम्प्टसाठी संदर्भ म्हणून जोडणे आवश्यक आहे.

  पहिल्या प्रॉम्प्टमधून परिणाम प्रिंट करण्याच्या कोडच्या भागाचा शोध घ्या आणि खालील कोड जोडा:

  खालील लक्षात ठेवा:

  1. आम्ही पहिल्या प्रॉम्प्टमधून परिणाम नवीन प्रॉम्प्टमध्ये जोडून नवीन प्रॉम्प्ट तयार करत आहोत:

  1. आम्ही नवीन विनंती करतो, परंतु आम्ही पहिल्या प्रॉम्प्टमध्ये विचारलेल्या टोकन्सच्या संख्येचा विचार करून, त्यामुळे यावेळी आम्ही म्हणतो `max_tokens` 1200 आहे.
