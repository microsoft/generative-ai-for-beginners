# सुधारित वैशिष्ट्ये आणि सुधारणा रोडमॅप

हा दस्तऐवज जनरेटिव्ह AI फॉर बिगिनर कोर्ससाठी शिफारस केलेल्या सुधारणा आणि सुधारणा यांचा आढावा देते, ज्याचे मुल्यमापन तांत्रिक पुनरावलोकन आणि उद्योगातील सर्वोत्तम पद्धतींच्या विश्लेषणावर आधारित आहे.

## कार्यकारी सारांश

कोडबेसचे सुरक्षा, कोड गुणवत्ता आणि शैक्षणिक प्रभावीता यासाठी विश्लेषण करण्यात आले आहे. हा दस्तऐवज तात्काळ दुरुस्त्या, जवळच्या काळातील सुधारणा आणि भविष्यातील सुधारणा यांसाठी शिफारसी प्रदान करतो.

## ४. API आधुनिकीकरण

### ४.१ जुनाट API पॅटर्न (मायग्रेशन पूर्ण)

सर्व Python आणि TypeScript **चॅट** नमुने Chat Completions API वरून **Responses API** (`client.responses.create(...)` → `response.output_text`) मध्ये स्थलांतरित झाले आहेत.

| जुना पॅटर्न | नवीन पॅटर्न | स्थिती |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (चॅट) | `OpenAI(base_url="<endpoint>/openai/v1/")` (Responses API) | पूर्ण |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | पूर्ण |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (TypeScript) | `openai` पॅकेज `client.responses.create()` → `response.output_text` | पूर्ण |
| `df.append()` (पांडा) | `pd.concat()` | पूर्ण |

> **टीप:** Microsoft Foundry Models चे नमुने जे `azure-ai-inference` / `@azure-rest/ai-inference` SDK (`client.complete()`) वापरतात, ते अजूनही Model Inference API वरच आहेत, ज्यात Responses API ची समर्थन नाही. `AzureOpenAI()` त्या ठिकाणी कायम आहे जिथे अजूनही वैध आहे (एम्बेडिंग्ज आणि प्रतिमा निर्मितीसाठी).

### ४.२ नवीन API वैशिष्ट्ये प्रदर्शित करणे

1. **संरचित आउटपुट्स** (OpenAI)
   - JSON मोड
   - कडक स्कीमांसह फंक्शन कॉलिंग

2. **व्हिजन क्षमता**
   - GPT-4o (व्हिजन) सह प्रतिमा विश्लेषण
   - मल्टी-मोडल प्रॉम्प्ट्स

3. **Responses API अंतर्बाह्य साधने** (जुन्या Assistants API ची जागा घेतली)
   - कोड इंटरप्रेटर
   - फाइल शोध
   - वेब शोध आणि सानुकूल साधने

## ५. पायाभूत सुविधा सुधारणा

### ५.१ CI/CD सुधारणा

[`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml) मध्ये राबवले गेले: Python लिंटिंग/फॉरमॅटिंग (Ruff + Black) हा `shared/` यूटिलिटीज मॉड्यूलसाठी **बांधील** आहे आणि इतर कोर्ससाठी **सलाहकार** स्वरूपात चालतो, तसेच JavaScript/TypeScript साठी ESLint सल्ला देणारा पास आहे. मूळ रेखाचित्र होते:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### ५.२ सुरक्षा स्कॅनिंग

[`.github/workflows/security.yml`](../../../.github/workflows/security.yml) मध्ये अंमलबजावणी: Python आणि JavaScript/TypeScript साठी CodeQL विश्लेषण (push, pull request, आणि साप्ताहिक वेळापत्रकावर) तसेच pull request वर अवलंबन पुनरावलोकन. मूळ रेखाचित्र होते:

```yaml
# .github/workflows/security.yml
name: Security Scan

on: [push, pull_request]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: javascript, python
      - uses: github/codeql-action/analyze@v3

  dependency-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
```
