# सुरुवातीसाठी जनरेटिव्ह AI साठी लहान भाषा मॉडेल्सची ओळख
जनरेटिव्ह AI ही कृत्रिम बुद्धिमत्तेची एक मनोरंजक क्षेत्र आहे जी नवीन सामग्री तयार करण्यास सक्षम प्रणाली तयार करण्यावर लक्ष केंद्रित करते. ही सामग्री मजकूर आणि प्रतिमा पासून संगीत आणि पूर्ण आभासी वातावरणांपर्यंत असू शकते. जनरेटिव्ह AI च्या सर्वात रोमांचक अनुप्रयोगांपैकी एक भाषा मॉडेल्सच्या क्षेत्रात आहे.

## लहान भाषा मॉडेल्स म्हणजे काय?

लहान भाषा मॉडेल (SLM) हा मोठ्या भाषा मॉडेल (LLM) चा एक कमी प्रमाणित प्रकार आहे, जो LLM च्या अनेक स्थापत्य तत्त्वे आणि तंत्र वापरतो, पण लक्षणीयरीत्या कमी संगणकीय जागा घेतो.

SLM हे माणसासारखा मजकूर तयार करण्यासाठी डिझाइन केलेले भाषा मॉडेल्सचे एक उपसमूह आहे. GPT-4 सारख्या मोठ्या मॉडेल्सच्या तुलनेत, SLM अधिक कॉम्पॅक्ट आणि कार्यक्षम आहेत, ज्यामुळे संगणकीय संसाधने कमी असलेल्या अनुप्रयोगांसाठी ते आदर्श आहेत. त्यांच्या लहान आकार असूनही, ते विविध कार्ये पार पाडू शकतात. सामान्यतः, SLMs चे बांधकाम LLMs चे संकुचन किंवा डिस्टिलेशन करून केले जाते, ज्यामुळे मूळ मॉडेलच्या कार्यशीलता आणि भाषिक क्षमतांचा एक महत्त्वाचा भाग ठेवण्याचा उद्देश असतो. मॉडेलच्या आकारात हा घट संपूर्ण जटिलता कमी करतो, ज्यामुळे SLM स्मृती वापर आणि संगणकीय आवश्यकतांच्या दृष्टीने अधिक कार्यक्षम होतात. या ऑप्टिमायझेशननंतरही, SLM अनेक नैसर्गिक भाषा प्रक्रिया (NLP) कार्ये पार पाडू शकतात:

- मजकूर निर्मिती: सुसंगत आणि संदर्भानुरूप वाक्ये किंवा परिच्छेद तयार करणे.
- मजकूर पूर्ण करणे: दिलेल्या प्रॉम्प्टनुसार वाक्यांची भविष्यवाणी आणि पूर्ण करणे.
- भाषांतर: एका भाषेतील मजकूर दुसऱ्या भाषेत रूपांतरित करणे.
- सारांश तयार करणे: लांब मजकूराचे संक्षिप्त, सोप्या स्वरूपात सारांश तयार करणे.

जरी त्यांच्या मोठ्या मॉडेल्सच्या तुलनेत कार्यक्षमता किंवा समजाच्या खोलाईत काही तोटा असला तरी.

## लहान भाषा मॉडेल कसे कार्य करतात?
SLMs मोठ्या प्रमाणावर मजकूर डेटावर प्रशिक्षित केले जातात. प्रशिक्षणादरम्यान, ते भाषेच्या नमुन्ये आणि संरचना शिकतात, जेणेकरून ते व्याकरणदृष्ट्या बरोबर आणि संदर्भानुरूप मजकूर तयार करू शकतात. प्रशिक्षण प्रक्रियेत खालील करणे समाविष्ट असते:

- डेटा संग्रहण: विविध स्रोतांमधून मोठ्या मजकूर डेटा संकलित करणे.
- पूर्वप्रक्रिया: डेटा स्वच्छ करणे आणि प्रशिक्षणासाठी योग्य प्रकारे संघटित करणे.
- प्रशिक्षण: मशीन लर्निंग अल्गोरिदम वापरून मॉडेलला मजकूर समजून घेणे आणि तयार करणे शिकवणे.
- फाईन-ट्यूनिंग: विशिष्ट कार्यांवर मॉडेलचे कार्य सुधारण्यासाठी समायोजने करणे.

SLM चा विकास संसाधने कमी असलेल्या वातावरणात तैनात करण्याच्या वाढती गरजेनुसार होत आहे, जसे की मोबाइल उपकरणे किंवा एज कंप्युटिंग प्लॅटफॉर्म्स, जिथे पूर्ण-आकाराचे LLMs त्यांच्या जड संसाधन मागण्यांमुळे अंमलबजावणीसाठी अव्यवहार्य असू शकतात. कार्यक्षमतेवर लक्ष केंद्रित करून, SLM कार्यक्षमता आणि प्रवेशयोग्यता यामध्ये समतोल राखतात, ज्यामुळे विविध क्षेत्रांमध्ये अधिक व्यापक अनुप्रयोग शक्य होतो.

## शिक्षण उद्दिष्टे

या धड्यात, आम्हाला SLM ची माहिती देऊन ती Microsoft Phi-3 सोबत जोडून मजकूर सामग्री, दृष्टी आणि MoE मधील भिन्न परिदृश्ये शिकवायची आहेत.

या धड्याच्या शेवटी, तुम्हाला पुढील प्रश्नांची उत्तरे देता यावी अशी अपेक्षा आहे:

- SLM म्हणजे काय?
- SLM आणि LLM मधील फरक काय आहे?
- Microsoft Phi-3/3.5 कुटुंब म्हणजे काय?
- Microsoft Phi-3/3.5 कुटुंबासह इनफरन्स कशी चालवायची?

तयार आहत? तर मग सुरुवात करूया.

## मोठे भाषा मॉडेल्स (LLMs) आणि लहान भाषा मॉडेल्स (SLMs) मधील फरक

LLMs आणि SLMs दोन्ही प्रायोरिटी मशीन लर्निंगच्या तत्त्वांवर आधारित आहेत, त्यांच्या स्थापत्य डिझाइन, प्रशिक्षण पद्धती, डेटा निर्मिती प्रक्रियेत तसेच मॉडेल मूल्यमापन तंत्रांमध्ये समान दृष्टिकोन वापरतात. मात्र या दोन्ही प्रकारच्या मॉडेल्समध्ये काही महत्त्वाचे फरक आहेत.

## लहान भाषा मॉडेल्सच्या अनुप्रयोग

SLMs चे विस्तृत अनुप्रयोग आहेत, ज्यात समाविष्ट आहे:

- चॅटबॉट्स: ग्राहक समर्थन देणे आणि वापरकर्त्यांशी संभाषणाद्वारे संवाद साधणे.
- सामग्री निर्मिती: लेखकांना कल्पना देणे किंवा पूर्ण लेख आकार देण्यात मदत करणे.
- शिक्षण: विद्यार्थ्यांना लेखन कामात किंवा नवीन भाषा शिकण्यात मदत करणे.
- प्रवेशयोग्यता: अशा व्यक्तीसाठी साधने तयार करणे ज्यांना अडचणी आहेत, जसे की मजकूर ते भाषण प्रणाली.

**आकार**
  
LLMs आणि SLMs मधील मुख्य फरक मॉडेलच्या आकारात आहे. LLMs, जसे ChatGPT (GPT-4), अंदाजे 1.76 ट्रिलियन पॅरामीटर्सचे असू शकतात, तर ओपन-सोर्स SLMs जसे Mistral 7B अत्यंत कमी पॅरामीटर्सने डिझाइन केलेले असतात — सुमारे 7 अब्ज. हा फरक मुख्यतः मॉडेल स्थापत्य आणि प्रशिक्षण प्रक्रियेमध्ये आहे. उदाहरणार्थ, ChatGPT एन्कोडर-डिकोडर फ्रेमवर्कमध्ये सेल्फ-अटेंशन मेकॅनिझम वापरतो, तर Mistral 7B स्लाइडिंग विंडो अटेंशन वापरतो, ज्यामुळे डिकोडर-ओन्ली मॉडेलमध्ये अधिक कार्यक्षम प्रशिक्षण शक्य होते. या स्थापत्यातील फरक मॉडेलची जटिलता आणि कार्यक्षमतेवर खोल परिणाम करतो.

**समजूतदारपणा**

SLM प्रामुख्याने विशिष्ट क्षेत्रांमध्ये कार्यक्षमतेसाठी ऑप्टिमाइझ केलेले असून ते खूप खासज्ञ आहेत, पण अनेक ज्ञान क्षेत्रांमध्ये व्यापक संदर्भ समजून घेण्यात मर्यादित असू शकतात. त्याउलट, LLMs मानवी सारखी बुद्धिमत्ता व्यापक स्तरावर अनुकरण करण्याचा प्रयत्न करतात. विशाल आणि विविध डेटासेटवर प्रशिक्षित केलेले LLMs विविध क्षेत्रांमध्ये चांगले प्रदर्शन करतात, ज्यामुळे त्यांना जास्त बहुमुखी आणि अनुकूलनीय बनते. परिणामी, LLMs नैसर्गिक भाषा प्रक्रिया आणि प्रोग्रामिंगसारख्या अनेक डाउनस्ट्रीम कार्यांसाठी अधिक योग्य आहेत.

**संगणकीय क्षमता**

LLMs चे प्रशिक्षण आणि तैनाती संसाधन-खर्चिक प्रक्रिया आहे, ज्यासाठी मोठ्या प्रमाणावरील GPU क्लस्टर्सची गरज असते. उदाहरणार्थ, ChatGPT सारखा मॉडेल सुरुवातीपासून प्रशिक्षित करण्यासाठी हजारों GPU बराच काळ लागतो. त्याउलट, SLMs, ज्यांच्या पॅरामीटर संख्या कमी आहे, त्यांना संगणकीय संसाधने कमी लागतात. Mistral 7B सारख्या मॉडेल्स मध्यम GPU क्षमतांसह स्थानिक संगणकावर प्रशिक्षण आणि चालवता येऊ शकतात, जरी प्रशिक्षणासाठी अनेक तास लागतात.

**पक्षपात**

पक्षपात हा LLMs मध्ये ज्ञात समस्या आहे, मुख्यतः प्रशिक्षण डेटाच्या सवयीमुळे. हे मॉडेल सामान्यतः इंटरनेटवरून उपलब्ध असलेल्या कच्च्या डेटावर अवलंबून असतात, ज्यामुळे काही गटांचा अपर्याप्त किंवा चुकीचा प्रतिनिधित्व होतो, चुकीच्या लेबलिंग होऊ शकते, किंवा बोलीभाषा, भौगोलिक फरक आणि व्याकरण नियमांमुळे भाषिक पक्षपात उद्भवू शकतो. तसेच, LLM स्थापत्यांच्या जटिलतेमुळे पक्षपात अधिक वाढू शकतो, जो सावधफटीने फाइन-ट्युनिंग न केल्यास लक्षात न येऊ शकतो. दुसरीकडे, SLMs अधिक मर्यादित, क्षेत्र-विशिष्ट डेटासेटवर प्रशिक्षित होतात, ज्यामुळे ते अशा पक्षपातांना तुलनेने कमी संवेदनशील असतात, तरीही ते पूर्णपणे मुक्‍त नाहीत.

**इन्फरन्स**

SLMs चा लहान आकार त्यांना इन्फरन्स वेगाच्या बाबतीत महत्त्वाचा फायदा देतो, ज्यामुळे ते स्थानिक हार्डवेअरवर विस्तृत समांतर प्रक्रिया न करता कार्यक्षमपणे आउटपुट तयार करू शकतात. त्याउलट, LLMs, त्यांच्या आकार आणि जटिलतेमुळे, स्वीकारार्ह इन्फरन्स वेळेसाठी मोठ्या संख्येने समांतर संगणकीय संसाधनांची गरज असते. अनेक वापरकर्ते एकाच वेळी वापरत असताना LLMs ची प्रतिसाद वेळ अधिक धीमी होते, विशेषतः मोठ्या प्रमाणावर तैनात केल्यास.

सारांश म्हणजे, LLMs आणि SLMs दोन्ही मशीन लर्निंगच्या मूलभूत तत्त्वांवर आधारित असले तरी, ते मॉडेल आकार, संसाधन गरजा, संदर्भ समज, पक्षपात संवेदनशीलता आणि इन्फरन्स वेग या बाबतीत लक्षणीय फरक दाखवतात. हे फरक त्यांच्या संबंधित उपयोगासाठी योग्यतेशी निगडीत आहेत, जिथे LLMs जास्त बहुमुखी पण संसाधन-खर्चिक आहेत, आणि SLMs अधिक क्षेत्र-विशिष्ट कार्यक्षमतेसह कमी संगणकीय मागणी करतात.

***टीप: या धड्यामध्ये, आम्ही Microsoft Phi-3 / 3.5 चा वापर करून SLM सादर करू.***

## Phi-3 / Phi-3.5 कुटुंबाची ओळख

Phi-3 / 3.5 कुटुंब मुख्यत्वे मजकूर, दृष्टी आणि एजंट (MoE) अनुप्रयोगांसाठी उद्दिष्ट आहे:

### Phi-3 / 3.5 इन्स्ट्रक्ट

मुख्यत्वे मजकूर निर्मिती, संवाद पूर्ण करणे आणि सामग्री माहिती काढणे इत्यादीसाठी.

**Phi-3-mini**

3.8B भाषा मॉडेल Microsoft Foundry, Hugging Face, आणि Ollama वर उपलब्ध आहे. Phi-3 मॉडेल्स समान किंवा जास्त आकाराच्या भाषा मॉडेल्सवर महत्त्वपूर्ण प्रकारे जास्त उत्तम कामगिरी करतात (खालील बेंचमार्क आकडे पहा, उच्च आकडे चांगले). Phi-3-mini त्याच्या आकाराच्या दुप्पट मॉडेल्सपेक्षा चांगले प्रदर्शन करतो, तर Phi-3-small आणि Phi-3-medium GPT-3.5 सह मोठ्या मॉडेल्स पेक्षा उत्तम आहेत.

**Phi-3-small आणि medium**

फक्त 7B पॅरामीटर्ससह, Phi-3-small GPT-3.5T च्या विविध भाषा, तर्कशक्ती, कोडिंग आणि गणिताच्या बेंचमार्कवर जास्त कार्यक्षम आहे.

14B पॅरामीटर्ससह Phi-3-medium हा प्रवाह सुरू ठेवतो आणि Gemini 1.0 Pro पेक्षा जास्त कामगिरी करतो.

**Phi-3.5-mini**

आपण याला Phi-3-mini चे एक अपग्रेड मानू शकतो. पॅरामीटर्स जरी अपरिवर्तित असले तरी, त्याने अनेक भाषा समर्थित केल्या आहेत (20+ भाषा: अरबी, चिनी, चेक, डॅनिश, डच, इंग्रजी, फिनिश, फ्रेंच, जर्मन, हिब्रू, हंगेरी, इटालियन, जपानी, कोरियन, नॉर्वेजियन, पोलिश, पोर्तुगीज, रशियन, स्पॅनिश, स्विडिश, थाई, टर्किश, युक्रेनियन) आणि लांब संदर्भासाठी बळकट समर्थन दिले आहे.

3.8B पॅरामीटर्ससह Phi-3.5-mini समान आकाराच्या भाषा मॉडेल्सपेक्षा चांगले कार्य करते आणि दुप्पट आकाराच्या मॉडेल्सशी समकक्ष असते.

### Phi-3 / 3.5 दृष्टी

Phi-3/3.5 चा इन्स्ट्रक्ट मॉडेल Phi ला समजण्याची क्षमता देतो, तर दृष्टी Phi ला जग समजून घेण्यासाठी डोळे प्रदान करते.


**Phi-3-विजन**

फक्त 4.2B पॅरामीटर्ससह Phi-3-विजन सामान्य दृश्य तर्कशक्तीच्या कार्यांमध्ये, OCR, तक्ते आणि आकृती समजण्याच्या कार्यांमध्ये Claude-3 Haiku आणि Gemini 1.0 Pro सारख्या मोठ्या मॉडेल्स पेक्षा उत्तम कामगिरी करतो.


**Phi-3.5-विजन**

Phi-3.5-विजन हे Phi-3-विजन चे एक अपग्रेड आहे, ज्यामध्ये अनेक प्रतिमा समर्थन केले आहे. आपण हे दृष्टीतील सुधारणा मानू शकतो, ज्यामुळे फक्त चित्रेच नव्हे तर व्हिडिओ देखील पाहू शकतो.

Phi-3.5-विजन OCR, तक्ता आणि चार्ट समजण्याच्या कार्यांमध्ये Claude-3.5 Sonnet आणि Gemini 1.5 Flash सारख्या मोठ्या मॉडेल्सपेक्षा चांगले आहे आणि सामान्य दृष्टी ज्ञान तर्कशक्तीच्या कार्यांमध्ये समकक्ष आहे. हे बहु-फ्रेम इनपुटला समर्थित करते, म्हणजेच अनेक इनपुट प्रतिमांवर तर्क करणे शक्य आहे.


### Phi-3.5-MoE

***Mixture of Experts (MoE)*** मॉडेल्सना कमी संगणकीय खर्चाने प्रीट्रेन करण्यास सक्षम करतो, ज्यामुळे आपण समान संगणकीय बजेटसह मॉडेल किंवा डेटासेटचा आकार मोठ्या प्रमाणावर वाढवू शकतो. विशेषतः, MoE मॉडेलने त्याच्या घन भागीदाराच्या तुलनेत प्रीट्रेनिंग दरम्यान जलद समान गुणवत्ता साध्य करावी.

Phi-3.5-MoE मध्ये 16x3.8B विशेषज्ञ मॉड्यूल्स आहेत. फक्त 6.6B सक्रिय पॅरामीटर्ससह Phi-3.5-MoE मोठ्या मॉडेल्सइतकीच तर्कशक्ती, भाषा समज आणि गणित करू शकतो.

आपण Phi-3/3.5 कुटुंब मॉडेल वेगवेगळ्या परिदृश्यांवर आधारित वापरू शकतो. LLM च्या तुलनेत, आपण Phi-3/3.5-mini किंवा Phi-3/3.5-विजन एज डिव्हायसेसवर तैनात करू शकतो.

**Hugging Face Transformer**
Hugging Face Transformers हे नैसर्गिक भाषा प्रक्रिया (NLP) आणि इतर मशीन लर्निंग कार्यांसाठी डिझाइन केलेले एक सामर्थ्यशाली लायब्ररी आहे. येथे त्याबद्दल काही महत्त्वाच्या बाबी आहेत:

1. **पूर्वप्रशिक्षित मॉडेल्स:** हे अनेक प्रशिक्षित मॉडेल्स प्रदान करते जे मजकूर वर्गीकरण, नामित घटक ओळख, प्रश्नोत्तरे, संक्षेपण, भाषांतर आणि मजकूर निर्मितीसाठी वापरले जाऊ शकतात.

2. **फ्रेमवर्क इंटरऑपरेबिलिटी:** लायब्ररी अनेक डीप लर्निंग फ्रेमवर्क्सना समर्थन देते, ज्यात PyTorch, TensorFlow, आणि JAX यांचा समावेश आहे. यामुळे तुम्ही एका फ्रेमवर्कमध्ये मॉडेल प्रशिक्षण देऊन दुसऱ्या फ्रेमवर्कमध्ये वापरू शकता.

3. **मल्टीमॉडल क्षमता:** NLP व्यतिरिक्त, Hugging Face Transformers संगणकीय दृष्टी (उदा., प्रतिमा वर्गीकरण, वस्तू शोध) आणि ऑडिओ प्रक्रिया (उदा., भाषण ओळख, ऑडिओ वर्गीकरण) यासाठी देखील समर्थन करते.

4. **वापरण्यास सुलभ:** लायब्ररीमध्ये API आणि साधने आहेत ज्यामुळे मॉडेल्स डाउनलोड करणे आणि फायनट्यून करणे सोपे होते, त्यामुळे ते नवशिक्या आणि तज्ञ दोघांसाठीही उपलब्ध आहे.

5. **समुदाय आणि संसाधने:** Hugging Face कडे एक सक्रिय समुदाय आहे आणि विस्तृत दस्तऐवजीकरण, मार्गदर्शक व ट्यूटोरियल्स आहेत जे वापरकर्त्यांना सुरुवात करण्यात आणि लायब्ररीचा जास्तीत जास्त फायदा घेण्यात मदत करतात.
[अधिकृत दस्तऐवजीकरण](https://huggingface.co/docs/transformers/index?WT.mc_id=academic-105485-koreyst) किंवा त्यांचा [GitHub संग्रह](https://github.com/huggingface/transformers?WT.mc_id=academic-105485-koreyst).

ONNX Runtime for Generative AI (GENAI) हा एक सामर्थ्यशाली साधन आहे जो तुम्हाला विविध प्लॅटफॉर्मवर जनरेटिव्ह AI मॉडेल्स कार्यक्षमतेने चालविण्यास मदत करतो.

## ONNX Runtime म्हणजे काय?
ONNX Runtime हा एक खुला स्रोत प्रकल्प आहे जो मशीन लर्निंग मॉडेल्सच्या उच्च-कार्यक्षमतेने इन्फरन्सला सक्षम करतो. तो Open Neural Network Exchange (ONNX) फॉरमॅटमधील मॉडेल्सना समर्थन करतो, जो मशीन लर्निंग मॉडेल्स सादर करण्यासाठी एक मानक आहे.ONNX Runtime इन्फरन्स ग्राहक अनुभव वेगवान करण्यासाठी आणि खर्च कमी करण्यासाठी मदत करू शकतो, PyTorch आणि TensorFlow/Keras सारख्या डीप लर्निंग फ्रेमवर्क्ससह तसेच scikit-learn, LightGBM, XGBoost सारख्या पारंपरिक मशीन लर्निंग लायब्ररीजशी सुसंगत आहे. ONNX Runtime वेगवेगळ्या हार्डवेअर, ड्रायव्हर्स, आणि ऑपरेटिंग सिस्टमसह सुसंगत असून, हार्डवेअर अ‍ॅक्सेलरेटरचे लवचिक वापर करत ग्राफ ऑप्टिमायझेशन्स आणि ट्रान्सफॉर्म्ससह सर्वोत्तम कार्यक्षमता प्रदान करतो.

## जनरेटिव्ह AI म्हणजे काय?
जनरेटिव्ह AI म्हणजे असे AI सिस्टम्स जे ठराविक प्रशिक्षण डेटावर आधारित नवीन सामग्री निर्माण करू शकतात, जसे मजकूर, प्रतिमा, किंवा संगीत. यामध्ये GPT-3 सारखे भाषा मॉडेल्स आणि Stable Diffusion सारखे प्रतिमा निर्मिती मॉडेल्स यांचा समावेश होतो. ONNX Runtime for GenAI लायब्ररी ONNX मॉडेल्ससाठी जनरेटिव्ह AI लूप प्रदान करते, ज्यात ONNX Runtime सह इन्फरन्स, लॉजिट्स प्रक्रिया, शोध आणि सॅम्पलिंग, आणि KV कॅश व्यवस्थापन यांचा समावेश आहे.

## ONNX Runtime for GENAI
ONNX Runtime for GENAI हे ONNX Runtime च्या क्षमतांना वाढवून जनरेटिव्ह AI मॉडेल्सस समर्थन करते. काही महत्त्वाच्या वैशिष्ट्यांमध्ये समाविष्ट आहे:

- **विस्तृत प्लॅटफॉर्म समर्थन:** Windows, Linux, macOS, Android, आणि iOS यांसह अनेक प्लॅटफॉर्मवर कार्य करते.
- **मॉडेल समर्थन:** LLaMA, GPT-Neo, BLOOM, आणि इतर लोकप्रिय जनरेटिव्ह AI मॉडेल्सना समर्थन करते.
- **कार्यक्षमता सुधारणा:** NVIDIA GPU, AMD GPU, आणि इतर हार्डवेअर अ‍ॅक्सेलरेटरसाठी ऑप्टिमायझेशन समाविष्ट आहे.
- **वापरण्यास सुलभ:** अनुप्रयोगांत सोप्या इंटीग्रेशनसाठी API प्रदान करते, ज्यामुळे थोड्या कोडच्या सहाय्याने मजकूर, प्रतिमा आणि अन्य सामग्री निर्माण करता येते.
- वापरकर्ते एक उच्चस्तरीय generate() मेथड कॉल करू शकतात, किंवा मॉडेलचे प्रत्येक पुनरावृत्ती लूपमध्ये चालवू शकतात, एक वेळेस एक टोकन निर्माण करत आणि पर्यायीपणे जनरेशन पॅरामीटर्स लूपमध्ये अद्ययावत करत.
- ONNX runtime मध्ये greedy/beam शोध आणि TopP, TopK सॅम्पलिंगद्वारे टोकन सिक्वेन्सेस निर्माण करण्यासाठी तसेच पुनरावृत्ती दंडांसारख्या अंतर्निर्मित लॉजिट्स प्रक्रियेस समर्थन आहे. तुम्ही सहज कस्टम स्कोरिंग देखील जोडू शकता.

## प्रारंभ कसे करावे
ONNX Runtime for GENAI शी सुरुवात करण्यासाठी, तुम्ही खालील पायऱ्या अनुसरू शकता:

### ONNX Runtime इंस्टॉल करा:
```Python
pip install onnxruntime
```
### जनरेटिव्ह AI एक्सटेंशन्स इंस्टॉल करा:
```Python
pip install onnxruntime-genai
```

### मॉडेल चालवा: येथे Python मध्ये एक सोपा उदाहरण आहे:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### डेमो: ONNX Runtime GenAI वापरून Phi-3.5-Vision कॉल करणे


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
