# जनरेटिव AI साठी लहान भाषा मॉडेल्सचा परिचय नवशिक्यांसाठी
जनरेटिव AI ही कृतिम बुद्धिमत्तेची एक आकर्षक क्षेत्र आहे जी नवीन सामग्री निर्माण करण्यास सक्षम असलेल्या प्रणालींवर लक्ष केंद्रित करते. ही सामग्री मजकूर, प्रतिमा, संगीत आणि अगदी संपूर्ण आभासी वातावरणापर्यंत वेगवेगळी असू शकते. जनरेटिव AI चा एक अत्यंत रोमांचक उपयोग भाषा मॉडेल्सच्या क्षेत्रात आहे.

## लहान भाषा मॉडेल म्हणजे काय?

लहान भाषा मॉडेल (SLM) हा मोठ्या भाषा मॉडेल (LLM) चा एक कमी प्रमाणात असलेला प्रकार आहे, जो LLM च्या बऱ्याच आर्किटेक्चरल तत्त्वे आणि तंत्रांचा उपयोग करतो, परंतु त्याचा संगणकीय प्रभाव लक्षणीयरीत्या कमी असतो.

SLM हे मानव-समान मजकूर तयार करण्यासाठी डिझाईन केलेले भाषा मॉडेल्सचा एक उपसमूह आहे. त्यांच्या मोठ्या समकक्षांपेक्षा, जसे की GPT-4, SLM अधिक कॉम्पॅक्ट आणि कार्यक्षम असतात, जे संगणकीय संसाधने मर्यादित असलेल्या अनुप्रयोगांसाठी आदर्श ठरतात. त्यांच्या लहान आकार असूनही, ते विविध कार्ये पार पाडू शकतात. सामान्यतः, SLM ला LLM चा संकुचन किंवा डिस्टिलेशन करून तयार केले जाते, ज्याचा उद्देश मूळ मॉडेलच्या कार्यक्षमतेचा आणि भाषिक क्षमतांचा मोठा भाग टिकवून ठेवणे असतो. मॉडेलचा हा आकार कमी होणे एकूण क्लिष्टता कमी करते, ज्यामुळे SLM मेमरी वापर आणि संगणकीय गरजांच्या बाबतीत अधिक कार्यक्षम होतात. या ऑप्टिमायझेशननंतरही, SLM नैसर्गिक भाषा प्रक्रिया (NLP) च्या विस्तृत कार्ये पार पाडू शकतात:

- मजकूर निर्माण: सुसंगत आणि संदर्भानुसार संबंधित वाक्ये किंवा परिच्छेद तयार करणे.
- मजकूर पूर्ण करणे: दिलेल्या संकेतावर आधारित वाक्ये भाकित करणे आणि पूर्ण करणे.
- भाषांतर: एका भाषेतील मजकूर दुसऱ्या भाषेत रूपांतरित करणे.
- संक्षेप करणे: लांब मजकुराला छोट्या आणि सोप्या सारांशात संक्षिप्त करणे.

त्यांच्या मोठ्या समकक्षांशी तुलनेत कार्यक्षमता किंवा समजण्याच्या खोलीत काही व्यापार-अनुबंधांसह.

## लहान भाषा मॉडेल कसे कार्य करतात?
SLM वेगवेगळ्या स्रोतांमधून मोठ्या प्रमाणात मजकूर डेटा वापरून प्रशिक्षण देण्यात येतात. प्रशिक्षणादरम्यान, ते भाषेच्या नमुन्यांचे आणि रचनेचे शिकल्यानंतर, व्याकरणसंगत आणि संदर्भास अनुरूप मजकूर तयार करण्यास सक्षम होतात. प्रशिक्षण प्रक्रियेत समाविष्ट आहे:

- डेटा संकलन: विविध स्रोतांमधून मोठा मजकूर संच गोळा करणे.
- पूर्वप्रक्रिया: प्रशिक्षणासाठी डेटा स्वच्छ करणे आणि संघटित करणे.
- प्रशिक्षण: मशीन लर्निंग अल्गोरिदम वापरून मॉडेलला मजकूर समजून घेणे आणि तयार करणे शिकवणे.
- सूक्ष्म-सुधारणा: विशिष्ट कार्यांसाठी मॉडेलची कार्यक्षमता वाढविण्यासाठी समायोजन करणे.

SLM चा विकास संसाधने मर्यादित असलेल्या वातावरणांत, जसे की मोबाइल उपकरणे किंवा एज कंम्प्युटिंग प्लॅटफॉर्म्समध्ये, जिथे पूर्ण LLM वापरणे अल्पस्वप्नीय असते, अशा गरजा पूर्ण करण्यासाठी झाला आहे. कार्यक्षमतेवर लक्ष केंद्रित करून, SLM कार्यक्षमता आणि प्रवेशयोग्यता यामध्ये संतुलन राखतात, ज्यामुळे विविध क्षेत्रांमध्ये अधिक व्यापक वापर शक्य होतो.

## शिकण्याचे उद्दिष्ट

या धड्यात, आम्ही SLM ची माहिती देण्याचा आणि Microsoft Phi-3 सह त्याचा वापर करून मजकूर सामग्री, दृष्टी आणि MoE मधील वेगवेगळ्या परिस्थिती शिकण्याचा प्रयत्न करू.

हा धडा संपल्यावर, तुम्हाला खालील प्रश्नांची उत्तरे देता येतील अशी अपेक्षा आहे:

- SLM म्हणजे काय?
- SLM आणि LLM मधील फरक काय आहे?
- Microsoft Phi-3/3.5 कुटुंब म्हणजे काय?
- Microsoft Phi-3/3.5 कुटुंबासह इन्फरन्स कसे चालवायचे?

तयार आहात का? चला सुरुवात करूया.

## मोठे भाषा मॉडेल (LLM) आणि लहान भाषा मॉडेल (SLM) यातील फरक

LLM आणि SLM दोन्ही संभाव्य मशीन लर्निंगच्या मूळ तत्त्वांवर आधारित आहेत, त्यांचे आर्किटेक्चरल डिझाइन, प्रशिक्षण पद्धती, डेटा निर्मिती प्रक्रिया आणि मॉडेल मूल्यांकन तंत्रांमध्ये समान दृष्टिकोन वापरतात. मात्र, या दोन्ही मॉडेल प्रकारांमध्ये काही महत्त्वाचे फरक आहेत.

## लहान भाषा मॉडेल्सचे अनुप्रयोग

SLM च्या अनुप्रयोगांची विस्तृत श्रेणी आहे, ज्यात समाविष्ट आहे:

- चॅटबॉट्स: ग्राहक समर्थन देणे आणि संवादात्मक पद्धतीने वापरकर्त्यांशी संवाद साधणे.
- सामग्री निर्मिती: लेखकांना कल्पना निर्माण करण्यात आणि लेखातील मसुदे तयार करण्यात मदत करणे.
- शिक्षण: विद्यार्थ्यांना लेखन कामे किंवा नवीन भाषा शिकण्यात मदत करणे.
- प्रवेशयोग्यता: अपंगत्व असलेल्या व्यक्तींसाठी साधने तयार करणे, जसे की मजकूर-ते-ध्वनी प्रणाली.

**आकार**
  
LLM आणि SLM मधील मुख्य फरक म्हणजे मॉडेलचा आकार. LLM, जसे ChatGPT (GPT-4), अंदाजे 1.76 ट्रिलियन पॅरामीटर्स असू शकतात, तर मुक्त-स्रोत SLM जसे Mistral 7B सुमारे 7 अब्ज पॅरामीटर्ससह डिझाईन केलेले आहेत. हा फरक मुख्यतः मॉडेल आर्किटेक्चर आणि प्रशिक्षण प्रक्रियेतील भिन्नतेमुळे आहे. उदाहरणार्थ, ChatGPT एन्कोडर-डिकोडर फ्रेमवर्कमध्ये सेल्फ-अटेंशन मेकॅनिझम वापरतो, तर Mistral 7B स्लायडिंग विंडो अटेंशन वापरतो, जे डिकोडर-केवळ मॉडेलमध्ये अधिक कार्यक्षम प्रशिक्षण सक्षम करते. या आर्किटेक्चरल भिन्नतेचा या मॉडेल्सच्या क्लिष्टता आणि कार्यक्षमतेवर खोल परिणाम होतो.

**समज**

SLM सामान्यतः विशिष्ट क्षेत्रांमध्ये कार्यक्षमतेसाठी ऑप्टिमाइझ केलेले असतात, ज्यामुळे ते अत्यंत विशेषज्ञ बनतात परंतु बहुविध ज्ञान क्षेत्रांमध्ये व्यापक संदर्भात्मक समज देण्यास मर्यादित असू शकतात. त्याउलट, LLM मानवी-समान बुद्धिमत्तेचे विस्तृत स्तरावर अनुकरण करण्याचा प्रयत्न करतात. विस्तृत आणि विविध डेटा संचावर प्रशिक्षण केलेले, LLM विविध क्षेत्रांत चांगली कामगिरी करण्यासाठी तयार केलेले आहेत, ज्यामुळे ते अधिक बहुमुखी आणि अनुकूल आहेत. परिणामी, LLM नैसर्गिक भाषा प्रक्रिया आणि प्रोग्रामिंगसारख्या विविध कार्यांसाठी अधिक योग्य आहेत.

**संगणकीय संसाधने**

LLM चे प्रशिक्षण आणि वितरण संसाधन-खूप गरज असलेली प्रक्रिया असते, ज्यासाठी मोठ्या प्रमाणावर GPU क्लस्टर्सची गरज असते. उदाहरणार्थ, ChatGPT सारखा मॉडेल पूर्णपणे शून्याहून प्रशिक्षण देण्यासाठी हजारो GPU आवश्यक असू शकतात जे दीर्घकाळ चालू शकते. त्याउलट, SLM कमी पॅरामीटर संख्या असल्यामुळे संगणकीय संसाधनांच्या दृष्टीने अधिक उपलब्ध आहे. Mistral 7B सारखे मॉडेल मध्यम GPU क्षमतेसह स्थानिक संगणकांवर प्रशिक्षण आणि चालवता येऊ शकतात, जरी प्रशिक्षणासाठी अनेक GPU वर काही तास लागू होतात.

**पक्षपात**

LLM मध्ये पक्षपात ही एक ज्ञात समस्या आहे, मुख्यतः प्रशिक्षण डेटाच्या स्वरूपामुळे. हे मॉडेल इंटरनेटवरील खुल्या डेटा वर अधिक अवलंबून असतात, ज्यामुळे काही गटांचे अंडर-प्रतीन किंवा चुकीचे प्रतिनिधित्व होऊ शकते, चुकीची लेबलिंग होऊ शकते, किंवा बोली, भौगोलिक भिन्नता, आणि व्याकरणीय नियमांमुळे भाषिक पक्षपात प्रतिबिंबित होऊ शकतो. याशिवाय, LLM आर्किटेक्चरची क्लिष्टता पक्षपात वाढवू शकते, ज्यासाठी काळजीपूर्वक सूक्ष्मसुधारणा आवश्यक असते. दुसरीकडे, SLM सीमित आणि विशिष्ट क्षेत्र आधारित डेटासेटवर प्रशिक्षण दिल्यामुळे अशा पक्षपातांना तुलनेत कमी संवेदनशील असतात, तरीही ते पूर्णपणे मुक्त नाहीत.

**इन्फरन्स**

SLM चा कमी आकार त्यांना इन्फरन्सच्या वेगात मोठा फायदा देतो, ज्यामुळे त्यांना स्थानिक हार्डवेअरवर प्रभावीपणे आउटपुट निर्माण करता येतो, विस्तृत समांतर प्रक्रिया आवश्यक नसते. त्याउलट, LLM त्यांचा आकार आणि क्लिष्टतेमुळे स्वीकारार्ह इन्फरन्स वेळ मिळवण्यासाठी महत्त्वाच्या समांतर संगणकीय संसाधनांची गरज असते. अनेक एकाच वेळी वापरकर्त्यांच्या उपस्थितीमध्ये LLM ची प्रतिक्रिया वेळ अधिक मंदावते, विशेषतः मोठ्या प्रमाणावर तैनात केल्यावर.

सारांश म्हणून, LLM आणि SLM दोन्ही मशीन लर्निंगच्या मूळ तत्त्वांवर आधारित असले तरी, त्यांनी मॉडेल आकार, संसाधन गरजा, संदर्भ समज, पक्षपातासंबंधी संवेदनशीलता आणि इन्फरन्स वेगाच्या बाबतीत लक्षणीय फरक दर्शविला आहे. हे भेद त्यांच्या वेगवेगळ्या वापर प्रकरणांसाठी योग्यतेचे प्रतिबिंब आहेत, जेथे LLM अधिक बहुमुखी पण संसाधन भार असणारे असतात आणि SLM विशिष्ट क्षेत्रांमध्ये कार्यक्षमतेसह कमी संगणकीय मागण्या सांगतात.

***टीप: या धड्यात, आम्ही Microsoft Phi-3 / 3.5 चा वापर करून SLM ची ओळख करून देऊ.***

## Phi-3 / Phi-3.5 कुटुंबाची ओळख

Phi-3 / 3.5 कुटुंब मुख्यतः मजकूर, दृष्टी, आणि एजंट (MoE) अनुप्रयोग परिस्थितींना लक्ष्य करतो:

### Phi-3 / 3.5 इन्स्ट्रक्ट

मुख्यतः मजकूर निर्माण, चॅट पूर्ण करणे, आणि सामग्रीच्या माहिती काढण्यासाठी इत्यादी.

**Phi-3-मिनी**

3.8B भाषा मॉडेल Microsoft Foundry, Hugging Face, आणि Ollama वर उपलब्ध आहे. Phi-3 मॉडेल मुख्य बेंचमार्कवर समान आणि मोठ्या आकाराच्या भाषा मॉडेल्सपेक्षा लक्षणीय चांगली कामगिरी करतात (खालील बेंचमार्क आकडे पहा, जास्त अंक चांगले). Phi-3-मिनी त्याच्या दुप्पट आकाराच्या मॉडेल्सपेक्षा अधिक चांगले आहे, तर Phi-3-लहान आणि Phi-3-मध्यम मोठ्या मॉडेल्ससह, ज्यात GPT-3.5 सुद्धा समाविष्ट आहे, चांगली कामगिरी करतात.

**Phi-3-लहान व मध्यम**

फक्त 7B पॅरामीटर्ससह, Phi-3-लहान GPT-3.5T ला विविध भाषा, तर्क, कोडिंग, आणि गणित बेंचमार्कवर पराभूत करते.

14B पॅरामीटर्ससह Phi-3-मध्यम ही प्रवृत्ती सुरू ठेवते आणि Gemini 1.0 Pro पेक्षा उच्च स्तरावर आहे.

**Phi-3.5-मिनी**

आपण त्याला Phi-3-मिनीचा अपग्रेड मानू शकतो. पॅरामीटर्स अपरिवर्तित असल्यानाही, हे एकापेक्षा जास्त भाषा समर्थित करते (20+ भाषा: अरबी, चीनी, झेक, डॅनिश, डच, इंग्रजी, फिनिश, फ्रेंच, जर्मन, हिब्रू, हंगेरियन, इटालियन, जपानी, कोरियन, नॉर्वेजीयन, पोलिश, पोर्तुगीज, रशियन, स्पॅनिश, स्विडिश, थाई, टर्किश, युक्रेनियन) आणि दीर्घ संदर्भासाठी अधिक मजबूत समर्थन देते.

3.8B पॅरामीटर्ससह Phi-3.5-मिनी समान आकाराच्या भाषा मॉडेल्सपेक्षा उत्तम आहे आणि दुप्पट आकारच्या मॉडेल्सशी तितकाच आहे.

### Phi-3 / 3.5 दृष्टी

आपण Phi-3/3.5 ची इन्स्ट्रक्ट मॉडेल Phi च्या समजूतदारपणासारखे समजू शकतो, आणि दृष्टी Phi ला जग समजून घेण्यासाठी डोळे देते.


**Phi-3-व्हिज़न**

फक्त 4.2B पॅरामीटर्ससह Phi-3-व्हिजन हि प्रवृत्ती सुरू ठेवतो आणि सामान्य दृष्टी तर्क, OCR, टेबल आणि आकृती समजण्याच्या कार्यांवर Claude-3 Haiku आणि Gemini 1.0 Pro सारख्या मोठ्या मॉडेल्सपेक्षा चांगली कामगिरी करतो.


**Phi-3.5-व्हिजन**

Phi-3.5-व्हिजन देखील Phi-3-व्हिजनचा अपग्रेड आहे, यात एकापेक्षा अधिक प्रतिमांच्या समर्थनाचा समावेश आहे. आपण हे दृष्टीतील सुधारणा म्हणून पाहू शकता, ज्यात आपण फक्त चित्र पाहू शकत नाही, तर व्हिडिओ देखील पाहू शकता.

Phi-3.5-व्हिजन OCR, टेबल आणि चार्ट समजण्याच्या कार्यांवर Claude-3.5 Sonnet आणि Gemini 1.5 Flash सारख्या मोठ्या मॉडेल्सपेक्षा चांगली कामगिरी करतो आणि सामान्य दृष्टी ज्ञान तर्क कार्यांवर समकक्ष आहे. अनेक प्रतिमा इनपुट (multi-frame input) वर तर्क करू शकतो.


### Phi-3.5-MoE

***Mixture of Experts (MoE)*** मॉडेल्सना कमी संगणकीय संसाधनांनी पूर्वप्रशिक्षित करण्याची क्षमता देते, ज्याचा अर्थ असा की तुम्ही एकाच संगणकीय बजेटने मॉडेल किंवा डेटासेट आकार मोठा पैमाण्यात वाढवू शकता. विशेषतः, MoE मॉडेल पूर्वप्रशिक्षणाच्या वेळी त्याच्या घन सक्षमता असलेल्या समकक्षासारखी गुणवत्ता जास्त वेगाने साधू शकतो.

Phi-3.5-MoE मध्ये 16x3.8B तज्ञ मॉड्यूल्स असतात. केवळ 6.6B अॅक्टिव्ह पॅरामीटर्ससह Phi-3.5-MoE मोठ्या मॉडेल्ससारखे तर्क, भाषा समज, आणि गणित क्षमता साध्य करतो.

**Hugging Face Transformer**
Hugging Face Transformers हा एक सामर्थ्यशाली लायब्ररी आहे जो नैसर्गिक भाषा प्रक्रिया (NLP) आणि इतर मशीन लर्निंग कामांसाठी डिझाइन केला आहे. त्याबद्दल काही महत्वाचे मुद्दे:

1. **पूर्वप्रशिक्षित मॉडेल्स**: हे हजारो पूर्वप्रशिक्षित मॉडेल्स प्रदान करते जे विविध कार्यांसाठी वापरले जाऊ शकतात जसे की मजकूर वर्गीकरण, नामित घटक ओळख, प्रश्न उत्तरे, सारांश, भाषांतर, आणि मजकूर निर्मिती.

2. **फ्रेमवर्क इंटरऑपरेबिलिटी:** या लायब्ररीमध्ये PyTorch, TensorFlow, आणि JAX सारख्या अनेक डीप लर्निंग फ्रेमवर्क्ससाठी समर्थन आहे. यामुळे तुम्ही एका फ्रेमवर्कमध्ये मॉडेल प्रशिक्षण देऊन ते दुसर्‍या फ्रेमवर्कमध्ये वापरू शकता.

3. **मल्टीमॉडल क्षमता:** NLP व्यतिरिक्त Hugging Face Transformers संगणकीय दृष्टी (उदा. प्रतिमा वर्गीकरण, ऑब्जेक्ट डिटेक्शन) आणि ऑडिओ प्रक्रिया (उदा. भाषण ओळख, ऑडिओ वर्गीकरण) कार्यांसाठी देखील समर्थन करते.

4. **सुलभ वापर:** लायब्ररी APIs आणि साधने ऑफर करते जी सहजपणे मॉडेल्स डाउनलोड आणि फाइन-ट्यून करता येतील, ज्यामुळे ते नवशिक्या आणि तज्ज्ञ दोघांसाठीही सोपे होते.

5. **समुदाय आणि संसाधने:** Hugging Face कडे एक सक्रिय समुदाय आणि विस्तृत दस्तऐवजीकरण, ट्यूटोरियल्स आणि मार्गदर्शक आहेत जे वापरकर्त्यांना सुरुवात करण्यास आणि लायब्ररीचा पूर्ण वापर करण्यास मदत करतात.
[अधिकृत दस्तऐवजीकरण](https://huggingface.co/docs/transformers/index?WT.mc_id=academic-105485-koreyst) किंवा त्यांच्या [GitHub रिपॉझिटरी](https://github.com/huggingface/transformers?WT.mc_id=academic-105485-koreyst) पहा.

हा सर्वात सामान्य वापरला जाणारा मार्ग आहे, पण हे GPU त्वरणाची गरज आहे. शेवटी, दृश्य (Vision) आणि MoE सारख्या परिस्थितींना खूप गणना करावी लागते, ज्यामुळे CPU वर ते क्वांटायज्ड नसल्यास फार मंद होऊ शकते.

**ONNX Runtime for GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) हा एक क्रॉस-प्लॅटफॉर्म अनुमान आणि प्रशिक्षण मशीन लर्निंग अक्‍सेलरेटर आहे. Generative AI (GENAI) साठी ONNX Runtime हा एक सामर्थ्यशाली साधन आहे जे विविध प्लॅटफॉर्मवर जनरेटिव AI मॉडेल्स कार्यक्षमतेने चालवायला मदत करते.

## ONNX Runtime म्हणजे काय?
ONNX Runtime हा एक खुला स्रोत प्रकल्प आहे जो मशीन लर्निंग मॉडेल्सचे उच्च-कार्यक्षमता अनुमान सक्षम करतो. तो Open Neural Network Exchange (ONNX) फॉर्मॅटमधील मॉडेल्सना समर्थन देतो, जो मशीन लर्निंग मॉडेल्स प्रतिनिधित्व करण्यासाठीचा एक मानक आहे. ONNX Runtime अनुमान ग्राहक अनुभव जलद बनवू शकतो आणि खर्च कमी करू शकतो, त्यात PyTorch आणि TensorFlow/Keras सारख्या डीप लर्निंग फ्रेमवर्क्स तसेच scikit-learn, LightGBM, XGBoost सारख्या पारंपरिक मशीन लर्निंग लायब्ररींचा समावेश आहे. ONNX Runtime विविध हार्डवेअर, ड्रायव्हर्स, आणि ऑपरेटिंग सिस्टिम्ससह सुसंगत आहे, आणि हार्डवेअर अक्‍सेलरेटर्सचा वापर करून तसेच ग्राफ ऑप्टिमायझेशन्स आणि ट्रान्सफॉर्मेशनसह सर्वोत्तम प्रदर्शन प्रदान करते.

## Generative AI म्हणजे काय?
जनरेटिव AI म्हणजे असे AI सिस्टम्स जे दिलेल्या प्रशिक्षण डेटावर आधारित नवीन मजकूर, प्रतिमा, किंवा संगीत यांसारखे नवीन कंटेंट तयार करू शकतात. उदाहरणार्थ GPT-3 सारखी भाषा मॉडेल्स आणि Stable Diffusion सारखी प्रतिमा निर्मिती मॉडेल्स. ONNX Runtime for GenAI लायब्ररी ONNX मॉडेल्ससाठी जनरेटिव AI लूप प्रदान करते, ज्यात ONNX Runtime सह अनुमान, logits प्रक्रिया, शोध आणि सॅम्पलिंग, तसेच KV कॅश व्यवस्थापन समाविष्ट आहे.

## ONNX Runtime for GENAI
ONNX Runtime for GENAI ONNX Runtime च्या क्षमता वाढवते ज्यामुळे जनरेटिव AI मॉडेल्सना समर्थन मिळते. काही मुख्य वैशिष्ट्ये खालीलप्रमाणे:

- **विस्तृत प्लॅटफॉर्म समर्थन:** हे Windows, Linux, macOS, Android, आणि iOS यांसारख्या विविध प्लॅटफॉर्मवर कार्य करते.
- **मॉडेल समर्थन:** हे LLaMA, GPT-Neo, BLOOM आणि इतर लोकप्रिय जनरेटिव AI मॉडेल्सना समर्थन देते.
- **कार्यक्षमता ऑप्टिमायझेशन:** यात NVIDIA GPUs, AMD GPUs आणि इतर हार्डवेअर अक्‍सेलरेटर्ससाठी ऑप्टिमायझेशन्स असतात.
- **सुलभ वापर:** हे अनुप्रयोगांमध्ये सुलभपणे एकत्रिकरणासाठी API प्रदान करते, ज्यामुळे तुम्ही कमी कोडने मजकूर, प्रतिमा आणि इतर कंटेंट तयार करू शकता.
- वापरकर्ते उच्च स्तरीय generate() मेथड कॉल करू शकतात, किंवा प्रत्येक आवृत्ती मॉडेलच्या लूपमध्ये चालवू शकतात, एकावेळी एक टोकन तयार करत आणि ऐच्छिकपणे लूपमध्ये जनरेशन पॅरामीटर्स अपडेट करू शकतात.
- ONNX रनटाइमgreedy/beam शोध आणि TopP, TopK सॅम्पलिंगसाठी समर्थन टिकवून ठेवतो ज्यायोगे टोकन सिक्वेन्स तयार करता येतात तसेच built-in logits प्रक्रिया जसे की पुनरावृत्ती शिक्षा (repetition penalties) देखील आहे. तुम्ही सहजपणे कस्टम स्कोरिंग देखील जोडू शकता.

## सुरुवात कशी करावी
ONNX Runtime for GENAI वापरून सुरुवात करण्यासाठी खालील पावले अनुसरा:

### ONNX Runtime इन्स्टॉल करा:
```Python
pip install onnxruntime
```
### Generative AI विस्तार इन्स्टॉल करा:
```Python
pip install onnxruntime-genai
```

### मॉडेल चालवा: Python मधील साधा उदाहरण:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### डेमो: ONNX Runtime GenAI वापरून Phi-3.5-Vision कॉल करणे


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
