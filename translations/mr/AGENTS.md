# AGENTS.md

## प्रकल्पाचा आढावा

हा रेपॉझिटरी जनरेटिव्ह AI मुलभूत तत्त्वे आणि अनुप्रयोग विकास शिकवण्याचा 21-धडा अभ्यासक्रम समाविष्ट करतो. कोर्स नवीन शिकणाऱ्यांसाठी डिझाइन केलेला असून मूलभूत संकल्पनांपासून उत्पादनासाठी तयार अनुप्रयोग तयार करण्यापर्यंत सर्वकाही समाविष्ट करतो.

**महत्त्वाच्या तंत्रज्ञानांमध्ये:**
- Python 3.9+ सह लायब्ररी: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- Node.js सह TypeScript/JavaScript आणि लायब्ररी: `openai` (Azure OpenAI v1 endpoint + Responses API द्वारे), `@azure-rest/ai-inference` (Microsoft Foundry Models)
- Azure OpenAI सेवा, OpenAI API, आणि Microsoft Foundry Models (GitHub Models जुलै २०२६ च्या अखेरीस बंद होत आहेत)
- परस्पर संवादासाठी Jupyter नोटबुक्स
- सुसंगत विकास वातावरणासाठी Dev कंटेनर्स

**रेपॉझिटरी संरचना:**
- 21 क्रमांकित धडा संचिका (00-21) ज्यामध्ये README, कोड उदाहरणे, आणि असाइनमेंट्स आहेत
- अनेक अंमलबजावण्या: Python, TypeScript, आणि कधीकधी .NET उदाहरणे
- 40+ भाषांमध्ये भाषांतर संचिका
- `.env` फाइलद्वारे केंद्रीकृत कॉन्फिगरेशन (टेम्प्लेटसाठी `.env.copy` वापरा)

## सेटअप आदेश

### प्रारंभिक रेपॉझिटरी सेटअप

```bash
# रिपॉझिटरी क्लोन करा
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# पर्यावरण साचा कॉपी करा
cp .env.copy .env
# तुमच्या API की आणि एंडपॉइंटसह .env संपादित करा
```

### Python वातावरण सेटअप

```bash
# आभासी वातावरण तयार करा
python3 -m venv venv

# आभासी वातावरण सक्रिय करा
# macOS/Linux वर:
source venv/bin/activate
# Windows वर:
venv\Scripts\activate

# अवलंबित्वे स्थापित करा
pip install -r requirements.txt
```

### Node.js/TypeScript सेटअप

```bash
# मूळ पातळीवरील अवलंबन प्रतिष्ठापित करा (दस्तऐवजीकरण साधनासाठी)
npm install

# वैयक्तिक धडा TypeScript उदाहरणांसाठी, विशिष्ट धड्याला भेट द्या:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

1. GitHub Codespaces किंवा VS Code मध्ये Dev Containers एक्सटेन्शनसह रेपॉझिटरी उघडा

## विकास कार्यप्रवाह

### वातावरणाच्या चल बदल

API प्रवेशासाठी आवश्यक असलेल्या सर्व धड्यांसाठी `.env` मध्ये वातावरणाच्या चल बदलांची व्याख्या करा:

- `OPENAI_API_KEY` - OpenAI API साठी
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry मधील Azure OpenAI साठी (Azure OpenAI सेवा आता Microsoft Foundry चा भाग: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI एंडपॉइंट URL (Foundry संसाधन एंडपॉइंट)
- `AZURE_OPENAI_DEPLOYMENT` - चॅट पूर्णता मॉडेल डिप्लॉयमेंट नाव
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - एम्बेडिंग मॉडेल डिप्लॉयमेंट नाव
- `AZURE_OPENAI_API_VERSION` - API आवृत्ती (डिफॉल्ट: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face मॉडेलसाठी
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry Models एंडपॉइंट (मल्टि-प्रोव्हायडर मॉडेल कॅटलॉग)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry Models API की (रिटायर होत असलेले `GITHUB_TOKEN` बदललेले)

## कोड शैली मार्गदर्शक तत्त्वे

### Python

- वातावरण चल व्यवस्थापनासाठी `python-dotenv` वापरा
- API संवादासाठी `openai` लायब्ररी आयात करा
- लिंटिंग साठी `pylint` वापरा (काही उदाहरणांमध्ये सुटसुटीतपणासाठी `# pylint: disable=all` वापरले आहे)
- PEP 8 नामांकन परंपरा पाळा
- API क्रेडेन्शियल्स `.env` फाईलमध्ये संग्रहित करा, कोडमध्ये कधीही नाही

### TypeScript

- वातावरण चलांसाठी `dotenv` पॅकेज वापरा
- प्रत्येक अॅपसाठी `tsconfig.json` मधील TypeScript कॉन्फिगरेशन
- Azure OpenAI साठी `openai` पॅकेज वापरा (क्लायंटला `/openai/v1/` एंडपॉइंटवर पॉइंट करा आणि `client.responses.create` कॉल करा); Microsoft Foundry Models साठी `@azure-rest/ai-inference` वापरा
- विकासासाठी `nodemon` वापरणे ज्यामुळे ऑटो-रिलोड होते
- चालवण्यापूर्वी बांधा: `npm run build` नंतर `npm start`

### सामान्य पद्धती

- कोड उदाहरणे सोपी आणि शैक्षिक ठेवा
- महत्त्वाच्या संकल्पनांसाठी टिप्पण्या समाविष्ट करा
- प्रत्येक धड्याचा कोड स्वतंत्र चालणारा आणि संरक्षित असावा
- नावांमध्ये सातत्य ठेवा: Azure OpenAI साठी `aoai-` प्रीफिक्स, OpenAI API साठी `oai-`, Microsoft Foundry Models साठी `githubmodels-` (GitHub Models काळातील जुना प्रीफिक्स राखलेला)

## दस्तऐवजीकरण मार्गदर्शक तत्त्वे

### Markdown शैली

- सर्व URL `[text](../../url)` या स्वरूपात असावेत, अतिरिक्त स्पेसशिवाय
- सापेक्ष दुवे `./` किंवा `../` ने सुरु होणे आवश्यक
- Microsoft डोमेनवरील सर्व दुव्यांसाठी ट्रॅकिंग ID असणे गरजेचे: `?WT.mc_id=academic-105485-koreyst`
- URL मध्ये देश-विशिष्ट लोकेल वापरू नका (उदा. `/en-us/` टाळा)
- `./images` फोल्डरमध्ये वर्णनात्मक नावांसह प्रतिमा साठवा
- फाईल नावांमध्ये इंग्रजी अक्षरे, संख्या, आणि डॅश वापरा

हे रेपॉझिटरी GitHub Actions चा वापर प्रमाणीकरणासाठी करते. PR सादर करण्यापूर्वी:

   - सर्व कोड चुका न करता चालवा याची खात्री करा
