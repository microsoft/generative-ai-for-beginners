# AGENTS.md

## प्रकल्पाचा आढावा

हा रिपॉझिटरी जनरेटिव्ह AI मूलतत्त्वे आणि अनुप्रयोग विकास शिकवणारा 21 धड्यांचा संपूर्ण अभ्यासक्रम समाविष्ट करतो. हा कोर्स नवशिक्यांसाठी डिझाइन केलेला असून मूलभूत संकल्पना पासून उत्पादनासाठी तयार अनुप्रयोग तयार करण्यापर्यंत सर्व काही समाविष्ट करतो.

**महत्त्वाच्या तंत्रज्ञान:**
- Python 3.9+ यासह लायब्ररी: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript Node.js सह आणि लायब्ररी: `openai` (Azure OpenAI v1 endpoint + Responses API), `@azure-rest/ai-inference` (Microsoft Foundry Models)
- Azure OpenAI सेवा, OpenAI API, आणि Microsoft Foundry Models (GitHub Models 2026 च्या जुलै अखेरीस बंद होत आहेत)
- इंटरऐक्टिव्ह लर्निंगसाठी Jupyter Notebooks
- सातत्यपूर्ण विकासासाठी Dev Containers

**रिपॉझिटरी रचना:**
- 21 क्रमांकाने दिलेल्या धडा संचिका (00-21) ज्यात READMEs, कोड उदाहरणे, आणि असाइनमेंट्स असतात
- अनेक अंमलबजावणी: Python, TypeScript, आणि कधी कधी .NET उदाहरणें
- 40+ भाषांमध्ये भाषांतर संचिका
- `.env` फाइलद्वारे केंद्रीकृत कॉन्फिगरेशन (टेम्पलेटसाठी `.env.copy` वापरा)

## सेटअप कमांड्स

### आरंभिक रिपॉझिटरी सेटअप

```bash
# रिपॉझिटरीची नक्कल करा
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# एन्व्हायरन्मेंट टेम्प्लेट कॉपी करा
cp .env.copy .env
# आपल्या API कीज आणि एन्डपॉईंटसह .env संपादित करा
```

### Python पर्यावरण सेटअप

```bash
# आभासी वातावरण तयार करा
python3 -m venv venv

# आभासी वातावरण सक्रिय करा
# macOS/Linux वर:
source venv/bin/activate
# Windows वर:
venv\Scripts\activate

# अवलंबित्वे स्थापित करा
pip install -r requirements.txt
```

```bash

# वैयक्तिक धडा TypeScript उदाहरणांसाठी, संबंधित धड्याकडे जा:
```

## विकास कार्यप्रवाह

### पर्यावरण व्हेरिएबल्स

सर्व धड्यांसाठी API प्रवेशासाठी `.env` मध्ये परिभाषित पर्यावरण व्हेरिएबल्स वापरल्या जातात:

- `OPENAI_API_KEY` - OpenAI API साठी
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry मधील Azure OpenAI साठी (Azure OpenAI सेवा आता Microsoft Foundry चा भाग आहे: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI endpoint URL (Foundry resource endpoint)
- `AZURE_OPENAI_DEPLOYMENT` - चॅट पूर्णत्व मॉडेल डिप्लॉयमेंट नाव (कोर्स डिफॉल्ट: `gpt-5-mini`)
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - एम्बेडिंग मॉडेल डिप्लॉयमेंट नाव (कोर्स डिफॉल्ट: `text-embedding-3-small`)
- `AZURE_OPENAI_API_VERSION` - API आवृत्ती (डिफॉल्ट: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face मॉडेलसाठी
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry Models endpoint (मल्टि-प्रोव्हायडर मॉडेल कॅटलॉग)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry Models API की (2026 मध्ये बंद होणाऱ्या `GITHUB_TOKEN` ची जागा घेतली आहे)
- `AZURE_INFERENCE_CHAT_MODEL` - एक नॉन-रिझनिंग मॉडेल (जसे की `Llama-3.3-70B-Instruct`), जे `temperature` उदाहरणांसाठी वापरले जाते, कारण रिझनिंग मॉडेल्स सॅंपलिंग नियंत्रणांना समर्थन करत नाहीत

### मॉडेल संहितीकरण (महत्त्वाचे)

- **डिफॉल्ट चॅट मॉडेल `gpt-5-mini` आहे** - वर्तमान, नॉन-डिप्रिकेटेड **रिझनिंग** मॉडेल. 2026 पासून तापमान-सक्षम "मिनी" मॉडेल्स (`gpt-4o-mini`, `gpt-4.1-mini`) *डिप्रिकेट होत आहेत*, त्यामुळे अभ्यासक्रम GPT-5 कुटुंबावर आधारित आहे.
- **रिझनिंग मॉडेल्स `temperature` आणि `top_p` नाकारतात**, आणि `max_output_tokens` (Responses API) / `max_completion_tokens` (चॅट पूर्णत्व) वापरतात `max_tokens` ऐवजी. `gpt-5-mini` कॉल करणाऱ्या नमुन्यांमध्ये `temperature`/`top_p`/`max_tokens` ऍड करू नका.
- **`temperature` दाखवण्यासाठी**, नमुन्यांमध्ये Microsoft Foundry Models endpoint (`AZURE_INFERENCE_CHAT_MODEL`) द्वारे **Llama** मॉडेल (`Llama-3.3-70B-Instruct`) वापरले जाते. रिझनिंग मॉडेल्सना प्रॉम्प्ट इंजिनियरिंग + रिझनिंग नियंत्रणांनी नियंत्रण करा, सॅंपलिंग नॉब्सने नव्हे.
- **फाईन-ट्युनिंग (धडा 18)** `gpt-4.1-mini` ठेवतो: GPT-5 फक्त रीनफोर्समेंट फाईन-ट्युनिंग (RFT) समर्थन करतो, येथे दाखवलेले सुपरवाइज्ड फाईन-ट्युनिंग (SFT) नाही.
- धडे 20 (मिस्ट्रल) आणि 21 (मेटा) `temperature`/`max_tokens` ठेवतात कारण ते मिस्ट्रल/लामा मॉडेल्सवर लक्ष केंद्रीत करतात, ज्यांना ते समर्थन आहे.

## कोड शैली मार्गदर्शक तत्त्वे

### Python

- पर्यावरण व्हेरिएबल व्यवस्थापनासाठी `python-dotenv` वापरा
- API संवादासाठी `openai` लायब्ररी इंपोर्ट करा
- `pylint` वापरा (काही उदाहरणांमध्ये सोप्या कारणासाठी `# pylint: disable=all` वापरले आहे)
- PEP 8 नामकरण तत्त्वांचे पालन करा
- API क्रेडेन्शियल्स `.env` फाईलमध्ये साठवा, कोडमध्ये कधीही नाही

### TypeScript

- पर्यावरण व्हेरिएबलसाठी `dotenv` पॅकेज वापरा
- प्रत्येक अॅपसाठी `tsconfig.json` मध्ये TypeScript कॉन्फिगरेशन
- Azure OpenAI साठी `openai` पॅकेज वापरा (क्लायंट `/openai/v1/` endpoint कडे इंगित करा आणि `client.responses.create` कॉल करा); Microsoft Foundry Models साठी `@azure-rest/ai-inference` वापरा
- विकासासाठी ऑटो रीलोड सह `nodemon` वापरा
- चालवण्यापूर्वी बनवा: `npm run build` नंतर `npm start`

### सामान्य संहितीकरणे

- कोड उदाहरणे साधी आणि शैक्षणिक ठेवा
- महत्त्वाच्या संकल्पनांचे स्पष्टीकरण देणाऱ्या टिप्पण्या समाविष्ट करा
- प्रत्येक धड्याचा कोड स्वावलंबी आणि चालवता येण्याजोगा असावा
- नावकरन एकसारखे ठेवा: Azure OpenAI साठी `aoai-` प्रीफिक्स, OpenAI API साठी `oai-`, Microsoft Foundry Models साठी `githubmodels-` (GitHub Models युगापासून पारंपरिक प्रीफिक्स राखले आहे)

## समस्या निराकरण

### सामान्य समस्या

**Python आयात त्रुटी**:
- आभासी वातावरण सक्रिय करा याची खात्री करा
- `pip install -r requirements.txt` चालवा
- Python आवृत्ती 3.9+ आहे का तपासा

**TypeScript बिल्ड त्रुटी**:
- विशिष्ट अॅप निर्देशिकेत `npm install` चालवा
- Node.js आवृत्ती सुसंगत आहे का तपासा
- `node_modules` साफ करा आणि पुन्हा स्थापित करा जर आवश्यक असेल तर

**API प्रमाणीकरण त्रुटी**:
- `.env` फाइल अस्तित्वात आणि योग्य मूल्यांसह आहे का तपासा
- API की वैध आहेत आणि कालबाह्य नाहीत याची खात्री करा
- आपल्या प्रदेशासाठी endpoint URLs बरोबर आहेत का तपासा

**पर्यावरण व्हेरिएबल गुम**:
- `.env.copy` ची `.env` मध्ये प्रती करा
- आपण काम करत असलेल्या धड्यासाठी सर्व आवश्यक मूल्ये भरा
- `.env` अद्ययावत केल्यानंतर आपला अनुप्रयोग पुनः सुरू करा
