# पुनर्प्राप्ती वृद्धीकरण जनरेशन (RAG) आणि व्हेक्टर डेटाबेस

[![पुनर्प्राप्ती वृद्धीकरण जनरेशन (RAG) आणि व्हेक्टर डेटाबेस](../../../translated_images/mr/15-lesson-banner.ac49e59506175d4f.webp)](https://youtu.be/4l8zhHUBeyI?si=BmvDmL1fnHtgQYkL)

शोध अनुप्रयोग धड्यात, आपण थोडक्यात आपले स्वतःचे डेटा मोठ्या भाषा मॉडेल्स (LLMs) मध्ये कसे समाकलित करायचे ते शिकले. या धड्यात, आपण आपले LLM अनुप्रयोगातील डेटा आधारभूत करण्याच्या संकल्पना, प्रक्रियेचे यांत्रिकी आणि डेटा संग्रहित करण्याच्या पद्धती, ज्यात एम्बेडिंग आणि मजकूर दोन्ही आहेत, यावर अधिक खोलवर चर्चा करू.

> **व्हिडिओ लवकरच उपलब्ध होईल**

## परिचय

या धड्यात आपण खालील गोष्टींचा आढावा घेऊ:

- RAG ची ओळख, ते काय आहे आणि AI (कृत्रिम बुद्धिमत्ता) मध्ये ते का वापरले जाते.

- व्हेक्टर डेटाबेस काय आहेत हे समजून घेणे आणि आमच्या अनुप्रयोगासाठी एक तयार करणे.

- RAG कसे समाकलित करायचे यावर व्यावहारिक उदाहरण.

## शिकण्याचे उद्दिष्टे

हा धडा पूर्ण केल्यानंतर, तुम्ही करू शकाल:

- डेटा पुनर्प्राप्ती आणि प्रक्रिया मध्ये RAG चे महत्त्व स्पष्ट करा.

- RAG अनुप्रयोग सेटअप करा आणि आपला डेटा LLM शी आधारित करा.

- LLM अनुप्रयोगांमध्ये RAG आणि व्हेक्टर डेटाबेसचे प्रभावी समाकलन.

## आमचे परिस्थिती: आमच्या स्वतःच्या डेटाने आमचे LLM सुधारणा

या धड्यासाठी, आपण शिक्षण स्टार्टअपमध्ये आपले स्वतःचे नोट्स जोडू इच्छितो, जेणेकरून चॅटबॉट वेगवेगळ्या विषयांवर अधिक माहिती मिळवू शकेल. आपल्या नोट्सचा वापर करून, शिकणाऱ्यांना चांगले अभ्यास करता येईल आणि वेगवेगळ्या विषयांना समजून घेणे सोपे होईल, जे त्यांच्या परीक्षांसाठी पुनरावलोकन करणे सोपे करेल. आमची परिस्थिती तयार करण्यासाठी, आपण वापरणार आहोत:

- `Azure OpenAI:` हा LLM आहे ज्याचा वापर आम्ही आपला चॅटबॉट तयार करण्यासाठी करू.

- `AI for beginners' lesson on Neural Networks`: हा डेटा आहे ज्यावर आम्ही आपला LLM आधारभूत करणार आहोत.

- `Azure AI Search` आणि `Azure Cosmos DB:` व्हेक्टर डेटाबेस ज्यात आपला डेटा संग्रहित करून शोध निर्देशांक तयार केला जाईल.

वापरकर्ते आपल्या नोट्समधून सराव क्विझ तयार करू शकतील, पुनरावलोकनासाठी फ्लॅश कार्ड्स आणि त्याचा सारांश संक्षिप्त आढावा रूपात करू शकतील. सुरुवात करण्यासाठी, आपण पाहूया RAG काय आहे आणि ते कसे कार्य करते:

## पुनर्प्राप्ती वृद्धीकरण जनरेशन (RAG)

एक LLM आधारित चॅटबॉट वापरकर्त्यांच्या प्रश्नांना प्रक्रिया करून उत्तर तयार करतो. हे संवादात्मक असते आणि वापरकर्त्यांसोबत विविध विषयांवर चर्चा करते. तथापि, त्याची उत्तरे प्रदान केलेल्या संदर्भ आणि त्याच्या आधारभूत प्रशिक्षण डेटावर मर्यादित असतात. उदाहरणार्थ, GPT-4 ची ज्ञानाच्या मर्यादेची तारीख सप्टेंबर 2021 आहे, म्हणजे त्या कालावधीनंतर घडलेल्या घटनांची त्याच्याकडे माहिती नाही. याशिवाय, LLM प्रशिक्षणासाठी वापरलेला डेटा गोपनीय माहिती जसे की वैयक्तिक नोट्स किंवा कंपनीचे उत्पादन मॅन्युअल समाविष्ट करत नाही.

### RAG कसे कार्य करतात

![RAG कसे कार्य करतात याचे चित्र](../../../translated_images/mr/how-rag-works.f5d0ff63942bd3a6.webp)

समजा, तुम्हाला तुमच्या नोट्सवर आधारित क्विझ तयार करणारा चॅटबॉट तैनात करायचा आहे, तर तुम्हाला ज्ञान आधाराशी संपर्क आवश्यक असेल. इथे RAG मदतीला येतो. RAG पुढीलप्रमाणे कार्य करतो:

- **ज्ञान आधार:** पुनर्प्राप्तीपूर्वी, या दस्तऐवजांना ग्रहण आणि पूर्वप्रक्रिया करणे आवश्यक असते, सामान्यतः मोठ्या दस्तऐवजांना लहान भागात विभाजित करून, त्यांना मजकूर एम्बेडिंग मध्ये रूपांतरित करून डेटाबेसमध्ये ठेवले जाते.

- **वापरकर्ता प्रश्न:** वापरकर्ता प्रश्न विचारतो

- **पुनर्प्राप्ती:** जेव्हा वापरकर्ता प्रश्न विचारतो, तेव्हा एम्बेडिंग मॉडेल आमच्या ज्ञान आधारातून संबंधित माहिती मिळवते जेणेकरून त्याचा संदर्भ प्रश्नात समाविष्ट केला जाईल.

- **वृद्धीकरण जनरेशन:** LLM आपल्या प्रतिसादाला मिळालेल्या माहितीच्या आधारावर सुधारतो. त्यामुळे उत्तर केवळ पूर्व-प्रशिक्षित डेटावर आधारित नसून जोडलेल्या संदर्भातून संबंधित माहितीवरही आधारित असते. मिळालेला डेटा LLM च्या उत्तरांना वृद्धिंगत करण्यात वापरला जातो. नंतर LLM वापरकर्त्याच्या प्रश्नाला उत्तर देते.

![RAG चे आर्किटेक्चर दाखवणारे चित्र](../../../translated_images/mr/encoder-decode.f2658c25d0eadee2.webp)

RAG चे आर्किटेक्चर ट्रान्सफॉर्मर वापरून तयार केले जाते ज्यात दोन भाग असतात: एक इनकोडर आणि एक डीकोडर. उदाहरणार्थ, जेव्हा वापरकर्ता प्रश्न विचारतो, तेव्हा इनपुट मजकूर 'वेक्टर' मध्ये 'इनकोड' केला जातो ज्यात शब्दांचा अर्थ कैद केला जातो आणि नंतर ते 'डीकोड' होऊन आमच्या दस्तऐवज निर्देशांकावर शोध घेतो व वापरकर्त्याच्या प्रश्नावर आधारित नवीन मजकूर तयार करतो. LLM उत्पादन तयार करण्यासाठी इनकोडर-डीकोडर मॉडेल दोन्ही वापरतो.

प्रस्तावित पत्रानुसार RAG अंमलबजावणीसाठी दोन पद्धती आहेत: [Retrieval-Augmented Generation for Knowledge intensive NLP Tasks](https://arxiv.org/pdf/2005.11401.pdf?WT.mc_id=academic-105485-koreyst):

- **_RAG-Sequence_** - वापरकर्त्याच्या प्रश्नाला सर्वोत्तम शक्य उत्तर भाकीत करणे साठी प्राप्त दस्तऐवजांचा वापर करणे

- **RAG-Token** - पुढील टोकन तयार करण्यासाठी दस्तऐवजांचा वापर करणे आणि नंतर ते वापरकर्त्याच्या प्रश्नाला उत्तर देण्यासाठी पुनर्प्राप्त करणे

### तुम्ही RAG का वापराल? 

- **माहिती समृद्धी:** मजकूर उत्तर अद्ययावत आणि वर्तमान ठेवते. त्यामुळे विशिष्ट क्षेत्रातील कामगिरी सुधारण्यासाठी अंतर्गत ज्ञान आधार वापरले जाते.

- तयार केलेले **सत्यापित डेटा** वापरून वापरकर्ता प्रश्नांवर संदर्भ पुरवून बनावट कमी करते.

- ते **खर्चवटींकडे दुर्लक्ष करणारे** आहेत कारण LLM ची फाइन-ट्यूनिंग करण्यापेक्षा अधिक किफायतशीर आहेत.

## ज्ञान आधार तयार करणे

आमचा अनुप्रयोग आमच्या वैयक्तिक डेटावर आधारित आहे म्हणजे AI For Beginners च्या न्यूरल नेटवर्क धड्याचा डेटा.

### व्हेक्टर डेटाबेस

पारंपरिक डेटाबेसच्या विपरीत, व्हेक्टर डेटाबेस एक विशेष डेटाबेस आहे जो एम्बेड केलेले व्हेक्टर संग्रहित, व्यवस्थापित आणि शोधतो. तो दस्तऐवजांचे संख्यात्मक प्रतिनिधित्व संग्रहित करतो. डेटाला संख्यात्मक एम्बेडिंगमध्ये रूपांतरित केल्याने आपल्या AI प्रणालीला डेटा समजून घेणे आणि प्रक्रिया करणे सोपे होते.

आम्ही आमचे एम्बेडिंग्स व्हेक्टर डेटाबेसमध्ये साठवतो कारण LLM ला प्राप्त होणाऱ्या टोकन्सची मर्यादा असते. तुम्ही संपूर्ण एम्बेडिंग्स LLM मध्ये देऊ शकत नाही, त्यासाठी त्यांना लहान भागांमध्ये विभागणे आवश्यक आहे आणि जेव्हा वापरकर्ता प्रश्न विचारतो, तेव्हा सर्वात संबंधित एम्बेडिंग्स प्रश्नासोबत प्राप्त होतात. विभागणी केल्याने टोकन्सच्या संख्येवर खर्चही कमी होतो.

काही लोकप्रिय व्हेक्टर डेटाबेसमध्ये Azure Cosmos DB, Clarifyai, Pinecone, Chromadb, ScaNN, Qdrant आणि DeepLake यांचा समावेश आहे. तुम्ही Azure CLI वापरून Azure Cosmos DB मॉडेल तयार करू शकता खालील कमांडने:

```bash
az login
az group create -n <resource-group-name> -l <location>
az cosmosdb create -n <cosmos-db-name> -r <resource-group-name>
az cosmosdb list-keys -n <cosmos-db-name> -g <resource-group-name>
```

### मजकूर ते एम्बेडिंगपर्यंत

डेटा साठवण्यापूर्वी, तो व्हेक्टर एम्बेडिंगमध्ये रूपांतरित करणे आवश्यक आहे. जर तुम्ही मोठ्या दस्तऐवजांसोबत काम करत असाल किंवा मोठा मजकूर असेल, तर तुम्ही प्रश्नानुसार तो विभागू शकता. विभागणी वाक्य पातळीवर किंवा परिच्छेद पातळीवर केली जाऊ शकते. विभागणी म्हणजे त्या शब्दांभोवतीच्या शब्दांचा अर्थ घेऊन भाग काढणे, त्यासाठी तुम्ही विभागांमध्ये अतिरिक्त संदर्भ जोडू शकता जसे दस्तऐवजाचे शीर्षक किंवा विभागापूर्वी किंवा नंतर काही मजकूर. तुम्ही डेटा खालीलप्रमाणे विभागू शकता:

```python
def split_text(text, max_length, min_length):
    words = text.split()
    chunks = []
    current_chunk = []

    for word in words:
        current_chunk.append(word)
        if len(' '.join(current_chunk)) < max_length and len(' '.join(current_chunk)) > min_length:
            chunks.append(' '.join(current_chunk))
            current_chunk = []

    # जर शेवटचा तुकडा किमान लांबीपर्यंत पोहोचला नसेल, तरीही तो जोडावा
    if current_chunk:
        chunks.append(' '.join(current_chunk))

    return chunks
```

विभागणी झाल्यावर, आम्ही वेगवेगळ्या एम्बेडिंग मॉडेल्स वापरून आपला मजकूर एम्बेड करू शकतो. काही मॉडेल्समध्ये word2vec, OpenAI चे ada-002, Azure Computer Vision आणि बरेच काही समाविष्ट आहेत. वापरायचे मॉडेल निवडणे वापरल्या जाणाऱ्या भाषा, एन्कोड केलेल्या सामग्रीचे प्रकार (मजकूर/प्रतिमा/ऑडिओ), त्याचा इनपुट आकार आणि एम्बेडिंग आउटपुटची लांबी यावर अवलंबून असेल.

OpenAI च्या `text-embedding-ada-002` मॉडेल वापरून तयार केलेल्या एम्बेडेड मजकूराचे उदाहरण:

## पुनर्प्राप्ती आणि व्हेक्टर शोध

जेव्हा वापरकर्ता प्रश्न विचारतो, तेव्हा पुनर्प्राप्ती यंत्रणा तो प्रश्न क्वेरी इनकोडर वापरून व्हेक्टरमध्ये रूपांतरित करते, नंतर ती आमच्या दस्तऐवज शोध निर्देशांकातून संबंधित व्हेक्टर शोधते जे इनपुटशी संबंधित असतात. नंतर तो दोन्ही इनपुट व्हेक्टर आणि दस्तऐवज व्हेक्टरला मजकूरात रूपांतरित करून LLM मध्ये पाठवतो.

### पुनर्प्राप्ती

पुनर्प्राप्ती म्हणजे प्रणाली वेगाने ते दस्तऐवज शोधते जे शोध निकष पूर्ण करतात. पुनर्प्राप्ती यंत्रणेचा उद्देश म्हणजे असे दस्तऐवज मिळवणे जे संदर्भ पुरवतील आणि LLM ला आपल्या डेटावर आधारभूत करतील.

आमच्या डेटाबेसमध्ये शोध करण्याचे अनेक मार्ग आहेत जसे की:

- **कीवर्ड शोध** - मजकूर शोधासाठी वापरले जाते

- **व्हेक्टर शोध** - दस्तऐवजांना एम्बेडिंग मॉडेल्स वापरून मजकूरातून व्हेक्टर प्रतिनिधित्वात रूपांतरित करते, ज्यामुळे शब्दांच्या अर्थाचा उपयोग करून **सामांतिक शोध** करता येतो. पुनर्प्राप्ती वापरकर्ता प्रश्नाशी सर्वात जवळचे व्हेक्टर असलेल्या दस्तऐवजांवर केली जाते.

- **हायब्रीड** - कीवर्ड आणि व्हेक्टर शोध यांचा संयोजन.

पुनर्प्राप्तीमध्ये एक अडचण म्हणजे जर डेटाबेसमध्ये प्रश्नाशी सुसंगत उत्तर नसेल, प्रणाली सर्वोत्तम माहिती देण्याचा प्रयत्न करेल, तरीसुद्धा तुम्ही संदर्भासाठी जास्तीत जास्त अंतर सेट करू शकता किंवा कीवर्ड आणि व्हेक्टर शोधाचा संयोजन करणारा हायब्रीड शोध वापरू शकता. या धड्यात आपण हायब्रीड शोध वापरणार आहोत. आपण आपल्या डेटाला एक डेटा फ्रेममध्ये संग्रहित करू ज्यामध्ये विभाग आणि एम्बेडिंग्स असतील.

### व्हेक्टर सादृश्यता

पुनर्प्राप्ती यंत्रणा ज्ञान डेटाबेसमध्ये जवळच्या एम्बेडिंग्ससाठी शोधते, कारण ते समान मजकूर असल्यामुळे ते जवळचे शेजारी मानले जातात. वापरकर्त्याने प्रश्न विचारल्यावर तो प्रथम एम्बेड होतो आणि नंतर समान एम्बेडिंगशी जुळवला जातो. विविध व्हेक्टर किती सादृश्य आहेत हे ओळखण्यासाठी सामान्य मोजमाप म्हणजे आक बिलीय सादृश्यता (cosine similarity) जी दोन व्हेक्टरमधल्या कोनावर आधारित आहे.

आपण इतर पर्यायही वापरू शकतो जसे की यु‍क्लिडियन अंतर ज्यामध्ये दोन व्हेक्टरच्या टोकांमधला सरळ रेषेचा अंतर मोजला जातो आणि डॉट उत्पाद ज्यात दोन व्हेक्टरच्या घटकांपैकी अनुक्रमे भागांच्या बेरीज मोजली जाते.

### शोध निर्देशांक

पुनर्प्राप्ती करताना, शोध करण्यापूर्वी आपल्याला आपल्या ज्ञान आधारासाठी शोध निर्देशांक तयार करणे आवश्यक आहे. एक निर्देशांक आमचे एम्बेडिंग्स संग्रहित करतो आणि मोठ्या डेटाबेसमध्ये सुद्धा सर्वात सादृश्य भाग लवकर शोधून आणू शकतो. आम्ही प्राथमिकरीत्या निर्देशांक स्थानिकरित्या तयार करू शकतो:

```python
from sklearn.neighbors import NearestNeighbors

embeddings = flattened_df['embeddings'].to_list()

# शोध निर्देशिका तयार करा
nbrs = NearestNeighbors(n_neighbors=5, algorithm='ball_tree').fit(embeddings)

# निर्देशिकेला क्वेरी करण्यासाठी, आपण kneighbors पद्धत वापरू शकता
distances, indices = nbrs.kneighbors(embeddings)
```

### पुन्हा क्रमवारी लावणे

एकदा तुम्ही डेटाबेस क्वेरी केल्यावर, तुम्हाला कदाचित सर्वात संबंधित परिणाम आजून चांगल्या प्रकारे क्रमवारीत लावावे लागेल. पुनःक्रमवारी लावणारा LLM मशीन लर्निंग वापरून शोध परिणामांची सुसंगतता सुधारतो आणि त्यांना सर्वात संबंधिततेच्या अनुक्रमे लावतो. Azure AI Search वापरून, हा पुनःक्रमवारी स्वतःहून सिमँटिक पुनःक्रमवारी वापरून केला जातो. न्यूनतम अंतर असलेल्या शेजाऱ्यांद्वारे पुनःक्रमवारी कशी होते याचे उदाहरण:

```python
# सर्वात जुळणारे दस्तऐवज शोधा
distances, indices = nbrs.kneighbors([query_vector])

index = []
# सर्वात जुळणारे दस्तऐवज छापा
for i in range(3):
    index = indices[0][i]
    for index in indices[0]:
        print(flattened_df['chunks'].iloc[index])
        print(flattened_df['path'].iloc[index])
        print(flattened_df['distances'].iloc[index])
    else:
        print(f"Index {index} not found in DataFrame")
```

```python
user_input = "what is a perceptron?"
```

## आमच्या अनुप्रयोगाचे मुल्यमापन

### मुल्यमापन मेट्रिक्स

- पुरवलेल्या प्रतिसादांची गुणवत्ता सैद्धांतिक, प्रवाही आणि मानवी सारखी वाटते का यांचे परीक्षण

- डेटाचा आधार: प्रतिसाद पुरवलेल्या दस्तऐवजांवर आधारित आहे का याचे मुल्यमापन

- संबंधितता: प्रतिसाद प्रश्नाशी सुसंगत आहे का याचे मुल्यमापन

- प्रवाहीपणा - प्रतिसाद व्याकरणिकदृष्ट्या अर्थपूर्ण आहे का

## RAG (पुनर्प्राप्ती वृद्धीकरण जनरेशन) आणि व्हेक्टर डेटाबेस वापरण्याच्या उपयोग केसेस

अनेक विविध उपयोग केसेस आहेत जिथे फंक्शन कॉल्स आपल्या अनुप्रयोगासाठी सुधारणा करू शकतात जसे की:

- प्रश्न आणि उत्तर: आपल्या कंपनीच्या डेटावर आधारित एक चॅट तयार करणे ज्याचा वापर कर्मचारी प्रश्न विचारण्यासाठी करू शकतात.

- शिफारस प्रणाली: जिथे तुम्ही अशी प्रणाली तयार करू शकता जी सर्वात सुसंगत मूल्यांशी जुळते जसे की चित्रपट, रेस्टॉरंट्स आणि बरेच काही.

- चॅटबॉट सेवा: तुम्ही चॅट इतिहास संग्रहित करू शकता आणि वापरकर्त्याच्या डेटावर आधारित संभाषण वैयक्तिकृत करू शकता.

- व्हेक्टर एम्बेडिंगवर आधारित प्रतिमा शोध, प्रतिमा ओळख आणि अपारंपरिक ओळखीसाठी उपयुक्त.

## सारांश

आम्ही RAG चे मूलभूत भाग कव्हर केले आहेत, आपल्या डेटासह अनुप्रयोग जोडणे, वापरकर्ता प्रश्न आणि आउटपुट. RAG तयार करणे सुलभ करण्यासाठी तुम्ही Semanti Kernel, Langchain किंवा Autogen सारखे फ्रेमवर्क वापरू शकता.
