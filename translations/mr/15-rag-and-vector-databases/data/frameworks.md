# न्यूरल नेटवर्क फ्रेमवर्क्स

जसे आपण आधीच शिकलो आहोत, न्यूरल नेटवर्क्सला प्रभावीपणे प्रशिक्षण देण्यासाठी आपल्याला दोन गोष्टी कराव्या लागतात:

* टेन्सर्सवर कार्य करणे, उदा. गुणाकार करणे, बेरीज करणे आणि काही फंक्शन्स जसे की सिग्मॉइड किंवा सॉफ्टमॅक्सची गणना करणे
* सर्व अभिव्यक्तींचे ग्रेडियंट्स गणना करणे, जेणेकरून ग्रेडियंट डिसेंट ऑप्टिमायझेशन करता येईल

जरी `numpy` लायब्ररी पहिला भाग करू शकते, तरीही आपल्याला ग्रेडियंट्सची गणना करण्यासाठी काहीतरी यंत्रणा आवश्यक आहे. मागील विभागात आपण विकसित केलेल्या फ्रेमवर्कमध्ये, आपल्याला `backward` पद्धतीमध्ये सर्व डेरिव्हेटिव्ह फंक्शन्स मॅन्युअली प्रोग्राम करावे लागले, जे बॅकप्रोपागेशन करते. आदर्शतः, एक फ्रेमवर्क आपल्याला *कोणत्याही अभिव्यक्तीचे* ग्रेडियंट्स गणना करण्याची संधी देईल.

आणखी एक महत्त्वाची गोष्ट म्हणजे GPU किंवा इतर कोणत्याही विशेषीकृत संगणकीय युनिट्स, जसे की TPU वर गणना करण्यास सक्षम असणे. डीप न्यूरल नेटवर्क प्रशिक्षणासाठी *खूप* गणना आवश्यक असते, आणि GPU वर त्या गणनांचा समांतरता करणे खूप महत्वाचे आहे.

> ✅ 'समांतरता' हा शब्द अनेक डिव्हाइसेसवर गणना वितरीत करणे याचा अर्थ आहे.

सध्या, दोन सर्वात लोकप्रिय न्यूरल फ्रेमवर्क्स आहेत: TensorFlow आणि PyTorch. दोन्ही CPU आणि GPU वर टेन्सर्ससह कार्य करण्यासाठी लो-लेव्हल API प्रदान करतात. लो-लेव्हल APIच्या वर, उच्च-स्तरीय API आहे, ज्याला अनुक्रमे Keras आणि PyTorch Lightning म्हणतात.

लो-लेव्हल API | TensorFlow| PyTorch
--------------|-------------------------------------|--------------------------------
उच्च-स्तरीय API| Keras| Pytorch

**लो-लेव्हल APIs** दोन्ही फ्रेमवर्क्समध्ये तथाकथित **गणनात्मक ग्राफ्स** तयार करण्यास अनुमती देतात. हा ग्राफ दिलेल्या इनपुट पॅरामीटर्ससह आउटपुट (सामान्यतः हानी फंक्शन) कसे गणना करायचे हे परिभाषित करतो, आणि GPU वर गणनेसाठी पाठविला जाऊ शकतो, जर तो उपलब्ध असेल. या गणनात्मक ग्राफला वेगळे करण्यासाठी आणि ग्रेडियंट्स गणना करण्यासाठी फंक्शन्स आहेत, जे नंतर मॉडेल पॅरामीटर्स ऑप्टिमाइझ करण्यासाठी वापरले जाऊ शकतात.

**उच्च-स्तरीय APIs** मुख्यतः न्यूरल नेटवर्क्सला **लेअर्सच्या अनुक्रम** म्हणून विचार करतात, आणि बहुतेक न्यूरल नेटवर्क्स तयार करणे खूप सोपे करतात. मॉडेल प्रशिक्षण सामान्यतः डेटाची तयारी करण्यास आवश्यक असते आणि नंतर काम करण्यासाठी `fit` फंक्शन कॉल करणे आवश्यक असते.

उच्च-स्तरीय API आपल्याला नेहमीचे न्यूरल नेटवर्क्स खूप वेगाने तयार करण्यास अनुमती देते, अनेक तपशीलांची काळजी न करता. त्याच वेळी, लो-लेव्हल API प्रशिक्षण प्रक्रियेवर अधिक नियंत्रण देते, आणि म्हणूनच ते संशोधनात खूप वापरले जाते, जेव्हा आपण नवीन न्यूरल नेटवर्क आर्किटेक्चर्सशी व्यवहार करत आहात.

हे देखील समजून घेणे महत्वाचे आहे की आपण दोन्ही APIs एकत्र वापरू शकता, उदा. आपण लो-लेव्हल API वापरून आपली स्वतःची नेटवर्क लेयर आर्किटेक्चर विकसित करू शकता, आणि नंतर ते उच्च-स्तरीय API सह तयार आणि प्रशिक्षित केलेल्या मोठ्या नेटवर्कमध्ये वापरू शकता. किंवा आपण लेअर्सच्या अनुक्रमणिका म्हणून उच्च-स्तरीय API वापरून नेटवर्क परिभाषित करू शकता, आणि नंतर आपला स्वतःचा लो-लेव्हल प्रशिक्षण लूप वापरून ऑप्टिमायझेशन करू शकता. दोन्ही APIs समान मूलभूत संकल्पनांचा वापर करतात, आणि ते एकत्र चांगले कार्य करण्यासाठी डिझाइन केलेले आहेत.

## शिक्षण

या कोर्समध्ये, आम्ही PyTorch आणि TensorFlow दोन्हीसाठी सामग्री ऑफर करतो. आपण आपल्या पसंतीच्या फ्रेमवर्कची निवड करू शकता आणि फक्त संबंधित नोटबुक्सद्वारे जाऊ शकता. जर आपण कोणता फ्रेमवर्क निवडायचा हे निश्चित नसल्यास, **PyTorch vs. TensorFlow** वर इंटरनेटवरील काही चर्चा वाचा. आपण दोन्ही फ्रेमवर्क्सवर देखील एक नजर टाकू शकता जेणेकरून चांगले समजून घेता येईल.

जेथे शक्य असेल, आम्ही सोप्या साठी उच्च-स्तरीय APIs वापरू. तथापि, आम्हाला असे वाटते की न्यूरल नेटवर्क्स कसे कार्य करतात हे तळापासून समजून घेणे महत्वाचे आहे, म्हणून सुरुवातीला आम्ही लो-लेव्हल API आणि टेन्सर्ससह काम करून सुरुवात करतो. तथापि, जर आपण वेगाने सुरू करू इच्छित असाल आणि या तपशीलांवर शिकण्यासाठी खूप वेळ खर्च करू इच्छित नसाल, तर आपण ते वगळू शकता आणि थेट उच्च-स्तरीय API नोटबुक्समध्ये जाऊ शकता.

लो-लेव्हल API | TensorFlow+Keras Notebook | PyTorch
--------------|-------------------------------------|--------------------------------
उच्च-स्तरीय API| Keras | *PyTorch Lightning*

# ओव्हरफिटिंग

ओव्हरफिटिंग हा मशीन लर्निंगमध्ये एक अत्यंत महत्वाचा संकल्पना आहे, आणि हे योग्य मिळवणे खूप महत्वाचे आहे!

खालील 5 बिंदूंच्या अनुमानाचा विचार करा (खालील ग्राफ्सवर `x` द्वारे दर्शविलेले):

!linear | overfit
-------------------------|--------------------------
**रेखीय मॉडेल, 2 पॅरामीटर्स** | **गैर-रेखीय मॉडेल, 7 पॅरामीटर्स**
प्रशिक्षण त्रुटी = 5.3 | प्रशिक्षण त्रुटी = 0
प्रमाणीकरण त्रुटी = 5.1 | प्रमाणीकरण त्रुटी = 20

* डावीकडे, आम्ही एक चांगली सरळ रेषा अनुमान पाहतो. कारण पॅरामीटर्सची संख्या पुरेशी आहे, मॉडेल बिंदू वितरणामागील कल्पना योग्य मिळवते.
* उजवीकडे, मॉडेल खूप शक्तिशाली आहे. कारण आमच्याकडे फक्त 5 बिंदू आहेत आणि मॉडेलमध्ये 7 पॅरामीटर्स आहेत, ते सर्व बिंदूंमधून जाण्यासारखे समायोजित करू शकते, ज्यामुळे प्रशिक्षण त्रुटी 0 होते. तथापि, हे मॉडेलला डेटामागील योग्य नमुना समजून घेण्यापासून रोखते, त्यामुळे प्रमाणीकरण त्रुटी खूप जास्त आहे.

मॉडेलची समृद्धता (पॅरामीटर्सची संख्या) आणि प्रशिक्षण नमुन्यांची संख्या यामध्ये योग्य संतुलन साधणे खूप महत्वाचे आहे.

## ओव्हरफिटिंग का होते

  * पुरेशी प्रशिक्षण डेटा नाही
  * खूप शक्तिशाली मॉडेल
  * इनपुट डेटामध्ये खूप आवाज

## ओव्हरफिटिंग कसे शोधावे

वरील ग्राफमधून आपण पाहू शकतो की, ओव्हरफिटिंग खूप कमी प्रशिक्षण त्रुटीने आणि उच्च प्रमाणीकरण त्रुटीने शोधले जाऊ शकते. सामान्यतः प्रशिक्षणादरम्यान आपण पाहू की प्रशिक्षण आणि प्रमाणीकरण त्रुटी दोन्ही कमी होऊ लागतात, आणि नंतर काही वेळाने प्रमाणीकरण त्रुटी कमी होणे थांबवू शकते आणि वाढू लागते. हे ओव्हरफिटिंगचे चिन्ह असेल, आणि आम्हाला कदाचित या बिंदूवर प्रशिक्षण थांबवायला हवे (किंवा किमान मॉडेलचे स्नॅपशॉट घ्यायला हवे).

ओव्हरफिटिंग

## ओव्हरफिटिंग कसे टाळावे

जर आपण पाहू शकता की ओव्हरफिटिंग होते आहे, तर आपण खालीलपैकी एक करू शकता:

 * प्रशिक्षण डेटाची मात्रा वाढवा
 * मॉडेलची गुंतागुंत कमी करा
 * काही नियमितरण तंत्र वापरा, जसे की ड्रॉपआउट, जे आपण नंतर विचार करू.

## ओव्हरफिटिंग आणि बायस-व्हेरियन्स ट्रेडऑफ

ओव्हरफिटिंग हा खरेतर आकडेवारीतील एक अधिक सामान्य समस्या आहे ज्याला बायस-व्हेरियन्स ट्रेडऑफ म्हणतात. जर आपण आपल्या मॉडेलमधील संभाव्य त्रुटींचे स्रोत विचारात घेतले तर आपण दोन प्रकारच्या त्रुटी पाहू शकतो:

* **बायस त्रुटी** आमच्या अल्गोरिदममुळे प्रशिक्षण डेटा दरम्यानचे नाते योग्यरित्या कॅप्चर करण्यात अक्षम होतात. हे आपल्या मॉडेल पुरेसे शक्तिशाली नाही यामुळे होऊ शकते (**अंडरफिटिंग**).
* **व्हेरियन्स त्रुटी**, जे मॉडेल इनपुट डेटामधील आवाजाचे अनुमान करण्यामुळे होतात, अर्थपूर्ण नातेसंबंधाऐवजी (**ओव्हरफिटिंग**).

प्रशिक्षणादरम्यान, बायस त्रुटी कमी होते (जसे की आमचे मॉडेल डेटा अनुमान शिकते), आणि व्हेरियन्स त्रुटी वाढते. ओव्हरफिटिंग टाळण्यासाठी प्रशिक्षण थांबवणे महत्वाचे आहे - मॅन्युअली (जेव्हा आपण ओव्हरफिटिंग शोधतो) किंवा स्वयंचलितपणे (नियमितरण सादर करून).

## निष्कर्ष

या धड्यात, आपण दोन सर्वात लोकप्रिय AI फ्रेमवर्क्स, TensorFlow आणि PyTorch साठी विविध APIs मधील फरकांबद्दल शिकलात. याव्यतिरिक्त, आपण एक अतिशय महत्वाचा विषय, ओव्हरफिटिंग बद्दल शिकलात.

## असाइनमेंट

या प्रयोगशाळेत, आपल्याला PyTorch किंवा TensorFlow वापरून एकल- आणि बहु-स्तरीय पूर्णपणे कनेक्टेड नेटवर्क्स वापरून दोन वर्गीकरण समस्या सोडवायला सांगितले जाते.

**अस्वीकरण**:  
हा दस्तऐवज AI अनुवाद सेवा [Co-op Translator](https://github.com/Azure/co-op-translator) वापरून अनुवादित केला गेला आहे. आम्ही अचूकतेसाठी प्रयत्नशील असलो तरी, कृपया लक्षात घ्या की स्वयंचलित अनुवादांमध्ये त्रुटी किंवा अशुद्धता असू शकतात. मूळ भाषेतील मूळ दस्तऐवज हा अधिकृत स्रोत मानला जावा. महत्त्वपूर्ण माहितीसाठी, व्यावसायिक मानवी अनुवादाची शिफारस केली जाते. या अनुवादाच्या वापरामुळे उद्भवणाऱ्या कोणत्याही गैरसमज किंवा चुकीच्या अर्थाबद्दल आम्ही जबाबदार नाही.
