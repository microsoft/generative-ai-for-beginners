## 我们如何获得生成式AI？

尽管最近生成式AI模型的发布引起了非凡的_热潮_，但这项技术已经历经数十年的发展，最早的研究努力可以追溯到60年代。现在，AI已经具备了人类的认知能力，例如对话，例如[OpenAI ChatGPT](https://openai.com/chatgpt)或[Bing Chat](https://www.microsoft.com/edge/features/bing-chat?WT.mc_id=academic-105485-koreyst)，它们也使用GPT模型进行网络搜索Bing对话。

往回追溯，最早的AI原型是打字机式聊天机器人，依赖于从专家组中提取的知识库并在计算机中表示。知识库中的答案是由输入文本中出现的关键词触发的。然而，很快就清楚这种使用打字机式聊天机器人的方法无法很好地扩展。

### AI的统计方法：机器学习

90年代出现了一个转折点，采用统计方法进行文本分析。这导致了新算法的发展——被称为机器学习——能够从数据中学习模式，而无需明确编程。这种方法使机器能够模拟人类语言理解：通过在文本-标签配对上训练统计模型，使模型能够用预定义标签分类未知输入文本，该标签代表消息的意图。

### 神经网络和现代虚拟助手

在最近的时间里，硬件技术的进步能够处理更大量的数据和更复杂的计算，促进了AI领域的研究，导致了高级机器学习算法的发展——称为神经网络或深度学习算法。

神经网络（尤其是循环神经网络——RNNs）显著增强了自然语言处理，能够以更有意义的方式表示文本的意义，重视句子中单词的上下文。

这项技术推动了新世纪第一个十年诞生的虚拟助手，非常擅长理解人类语言，识别需求并执行满足需求的操作——例如用预定义脚本回答或使用第三方服务。

### 现今，生成式AI

这就是我们今天如何来到生成式AI的，生成式AI可以被视为深度学习的一个子集。

![AI、ML、DL和生成式AI](../../../translated_images/AI-diagram.png?WT.e126d57e1a443697cd851d5d04d66753225b4d910f4aff65f9f28215b528471a.zh.mc_id=academic-105485-koreyst)

经过数十年的AI领域研究，一种新的模型架构——称为_Transformer_——克服了RNNs的限制，能够将更长的文本序列作为输入。Transformers基于注意力机制，使模型能够对其接收到的输入赋予不同的权重，‘更加关注’最相关信息集中的地方，而不考虑它们在文本序列中的顺序。

大多数最近的生成式AI模型——也称为大型语言模型（LLMs），因为它们处理文本输入和输出——确实基于这种架构。这些模型的有趣之处在于——在大量来自书籍、文章和网站的未标记数据上训练——它们可以适应各种任务，并生成具有创造性表象的语法正确的文本。因此，它们不仅极大地增强了机器‘理解’输入文本的能力，还使其能够生成以人类语言呈现的原创响应。

## 大型语言模型如何工作？

在下一章中，我们将探索不同类型的生成式AI模型，但现在让我们看看大型语言模型如何工作，重点关注OpenAI GPT（生成预训练变压器）模型。

- **Tokenizer，文本到数字**：大型语言模型接收文本作为输入，并生成文本作为输出。然而，作为统计模型，它们更擅长处理数字而不是文本序列。这就是为什么每个输入在被核心模型使用之前都由tokenizer处理。一个token是一个文本块——由可变数量的字符组成，因此tokenizer的主要任务是将输入拆分为token数组。然后，每个token被映射到一个token索引，即原始文本块的整数编码。

![Tokenization示例](../../../translated_images/tokenizer-example.png?WT.3b4be927057ceb39216ffc617cde2fd4d843e0d7557fc81d08a0018831f601ed.zh.mc_id=academic-105485-koreyst)

- **预测输出token**：给定n个token作为输入（最大n因模型而异），模型能够预测一个token作为输出。然后，该token被纳入下一次迭代的输入中，以扩展窗口模式，从而提供更好的用户体验，获得一个（或多个）句子作为答案。这解释了为什么如果你曾经玩过ChatGPT，你可能注意到有时它看起来像是在句子中间停下来。

- **选择过程，概率分布**：输出token由模型根据其在当前文本序列之后出现的概率选择。这是因为模型预测了所有可能‘下一个token’的概率分布，基于其训练进行计算。然而，并不总是选择具有最高概率的token作为结果分布的一部分。为了模拟创造性思维过程，这种选择中添加了一定程度的随机性，并且可以通过一个称为温度的模型参数进行调整。

## 我们的创业公司如何利用大型语言模型？

现在我们对大型语言模型的内部工作有了更好的理解，让我们看看一些它们可以很好地执行的最常见任务的实际示例，关注我们的业务场景。我们说大型语言模型的主要能力是_从头生成文本，从自然语言编写的文本输入开始_。

但是什么样的文本输入和输出呢？
大型语言模型的输入被称为提示，而输出被称为完成，这个术语指的是模型生成下一个token以完成当前输入的机制。我们将深入探讨什么是提示以及如何设计它以充分利用我们的模型。但现在，我们只需说提示可能包括：

- 指定我们期望模型输出类型的**指令**。这个指令有时可能包含一些示例或一些附加数据。

  1. 文章、书籍、产品评论等的总结，以及从非结构化数据中提取见解。

  2. 文章、论文、作业或更多内容的创造性构思和设计。
      
     ![创造性写作示例](../../../translated_images/creative-writing-example.png?WT.04b03c92f46ed96df1138828e37760ac81e06eaa8602de8c4b175a9c514c9a14.zh.mc_id=academic-105485-koreyst)

- 要完成的**文本块**，这隐含着寻求写作帮助。

以上示例相当简单，并不想成为大型语言模型功能的详尽演示。它们只是想展示使用生成式AI的潜力，尤其是在教育上下文中，但不限于此。

此外，生成式AI模型的输出并不完美，有时模型的创造性可能会适得其反，导致输出是人类用户可能解读为现实扭曲的词语组合，或者可能是冒犯性的。生成式AI不是智能的——至少在更全面的智能定义中，包括批判性和创造性推理或情感智能；它不是确定性的，也不值得信赖，因为虚构内容，例如错误的引用、内容和陈述，可能与正确信息结合在一起，并以一种令人信服和自信的方式呈现。在接下来的课程中，我们将处理所有这些限制，并看看我们可以做些什么来减轻它们。
