# LLM 미세조정

생성 AI 애플리케이션을 구축하기 위해 대형 언어 모델을 사용하는 데에는 새로운 도전 과제가 있습니다. 핵심 문제 중 하나는 주어진 사용자 요청에 대해 모델이 생성한 내용에서 응답의 품질(정확성과 적합성)을 보장하는 것입니다. 이전 강의에서는 기존 모델에 대한 _프롬프트 입력을 수정하는_ 기법인 프롬프트 엔지니어링과 검색 증강 생성과 같은 기법을 다루었습니다.

오늘 강의에서는 세 번째 기법인 <strong>미세조정</strong>에 대해 논의합니다. 이 기법은 _추가 데이터로 모델 자체를 재학습_하여 문제를 해결하려고 합니다. 자세히 살펴보겠습니다.

## 학습 목표

이 강의는 사전 학습된 언어 모델의 미세조정 개념을 소개하고, 이 접근법의 이점과 문제점을 탐구하며, 생성 AI 모델의 성능을 향상하기 위해 미세조정을 사용하는 시기와 방법에 대한 안내를 제공합니다.

- 언어 모델의 미세조정이란 무엇인가?
- 미세조정은 언제, 왜 유용한가?
- 사전 학습된 모델을 어떻게 미세조정할 수 있는가?
- 미세조정의 한계는 무엇인가?

## 언어 모델의 미세조정이란?

정의에 따르면, 대형 언어 모델은 인터넷을 포함한 다양한 출처의 대량 텍스트로 _사전 학습_됩니다. 이전 강의에서 배운 것처럼 모델이 사용자 질문("프롬프트")에 더 나은 응답을 하도록 하기 위해 _프롬프트 엔지니어링_과 _검색 증강 생성_과 같은 기법이 필요합니다.

인기 있는 프롬프트 엔지니어링 기법 중 하나는 모델에게 _명령_ (명시적 안내)이나 _몇 가지 예를 제공_ (암묵적 안내)하여 기대하는 응답 방향을 더 잘 알려주는 것입니다. 이를 _few-shot learning_이라 하지만, 두 가지 한계가 있습니다:

- 모델의 토큰 제한으로 인해 제공할 수 있는 예제 수가 제한되어 효과가 떨어질 수 있습니다.
- 모델 토큰 비용으로 인해 매 프롬프트마다 예시를 추가하는 것이 비용이 많이 들고 유연성이 제한됩니다.

미세조정은 기계학습 시스템에서 일반적으로 사용되는 방법으로, 기존 사전학습 모델을 새로운 데이터로 재학습시켜 특정 작업에서 성능을 향상시키는 것입니다. 언어 모델의 경우, 특정 작업이나 응용 분야에 대한 _선별된 예시 세트로_ 사전학습 모델을 미세조정하여 해당 작업/도메인에 더 정확하고 적합한 <strong>맞춤 모델</strong>을 생성할 수 있습니다. 미세조정의 부수적 이점으로, few-shot learning에 필요한 예시 개수를 줄여 토큰 사용과 관련 비용을 낮출 수 있습니다.

## 미세조정은 언제, 왜 해야 할까요?

여기서 말하는 미세조정은 원래 학습 데이터에 포함되지 않은 **새로운 데이터를 추가하여** 재학습하는 <strong>지도 학습 미세조정(supervised fine-tuning)</strong>을 의미합니다. 원래 데이터에 대해 하이퍼파라미터만 다르게 조정해 재학습하는 비지도 미세조정과는 다릅니다.

미세조정은 원하는 결과를 얻기 위해 상당한 전문성이 필요한 고급 기법임을 기억해야 합니다. 잘못 수행하면 기대한 개선이 없거나 오히려 모델 성능이 저하될 수도 있습니다.

따라서 언어 모델 미세조정의 "방법"을 배우기 전에, 왜 이 방식을 선택해야 하는지 "이유"와 언제부터 미세조정을 시작할지 "시기"를 알아야 합니다. 다음 질문들을 스스로에게 던져보세요:

- **사용 사례**: 미세조정의 _사용 사례_는 무엇인가? 현재 사전학습 모델의 어떤 부분을 개선하고 싶은가?
- <strong>대안</strong>: 원하는 결과를 얻기 위해 _다른 기법들_을 시도해 봤는가? 이를 기준점으로 삼아 비교해 보자.
  - 프롬프트 엔지니어링: 관련 프롬프트 응답 예시를 포함한 few-shot prompting 기법을 시도하고 응답 품질을 평가해 보자.
  - 검색 증강 생성: 데이터 검색 결과로 프롬프트를 보강하는 방법을 시도하고 응답 품질을 평가해 보자.
- <strong>비용</strong>: 미세조정 비용을 확인했는가?
  - 조정 가능성: 사전학습 모델이 미세조정 가능하게 제공되는가?
  - 노력: 학습 데이터 준비, 모델 평가 및 개선에 필요한 노력
  - 컴퓨팅: 미세조정 작업 실행과 미세조정 모델 배포에 필요한 컴퓨팅 리소스
  - 데이터: 미세조정에 영향을 줄 수 있는 충분한 품질의 예시 확보
- <strong>이점</strong>: 미세조정의 이점을 확인했는가?
  - 품질: 미세조정된 모델이 기준 모델을 능가했는가?
  - 비용: 프롬프트 간소화로 토큰 사용량이 줄었는가?
  - 확장성: 기본 모델을 새로운 도메인에 재활용할 수 있는가?

이 질문에 답함으로써, 미세조정이 여러분의 사용 사례에 적합한지 판단할 수 있습니다. 이상적으로, 이 접근법은 이점이 비용을 초과할 때만 유효합니다. 진행하기로 결정하면, 사전학습 모델을 _어떻게_ 미세조정할지 고민할 차례입니다.

## 사전 학습 모델을 어떻게 미세조정할 수 있나요?

사전 학습 모델을 미세조정하려면 다음이 필요합니다:

- 미세조정할 사전 학습 모델
- 미세조정을 위한 데이터셋
- 미세조정 작업을 실행할 학습 환경
- 미세조정된 모델을 배포할 호스팅 환경
