# LLM 미세 조정하기

대형 언어 모델을 사용해 생성 AI 애플리케이션을 구축할 때 새로운 과제가 따릅니다. 주된 이슈는 사용자 요청에 대해 모델이 생성하는 컨텐츠의 응답 품질(정확성과 관련성)을 보장하는 것입니다. 이전 강의에서 우리는 기존 모델에 대한 _프롬프트 입력을 수정하는_ 프롬프트 엔지니어링과 검색 증강 생성 같은 기법들을 논의했습니다.

오늘 강의에서는 세 번째 기법인 <strong>미세 조정(fine-tuning)</strong>에 대해 다루는데, 이는 추가 데이터를 활용해 _모델 자체를 재학습_ 시켜 이 문제에 접근하는 방식입니다. 자세히 살펴보겠습니다.

## 학습 목표

이 강의는 사전 훈련된 언어 모델의 미세 조정 개념을 소개하고, 이 접근법의 이점과 과제를 탐구하며, 생성 AI 모델의 성능 개선을 위해 언제 어떻게 미세 조정을 사용하는지 안내합니다.

강의를 마치면 다음 질문에 답할 수 있을 것입니다:

- 언어 모델의 미세 조정이란 무엇인가?
- 언제, 그리고 왜 미세 조정이 유용한가?
- 어떻게 사전 학습된 모델을 미세 조정할 수 있는가?
- 미세 조정의 한계는 무엇인가?

## 언어 모델의 미세 조정이란?

정의에 따르면, 대형 언어 모델은 인터넷을 포함한 다양한 소스에서 수집한 방대한 텍스트로 _사전훈련_ 되어 있습니다. 앞서 배운 것처럼, 우리는 모델의 사용자 질문("프롬프트")에 대한 응답 품질을 개선하기 위해 _프롬프트 엔지니어링_이나 _검색 증강 생성_ 같은 기법이 필요합니다.

널리 쓰이는 프롬프트 엔지니어링 기법에는 모델에 기대하는 응답에 대한 더 많은 안내를 제공하는데, 이는 _지시문(명시적 안내)_을 주거나 _몇 가지 예시를 보여주는(암시적 안내)_ 방식입니다. 이를 _few-shot learning(소수 예시 학습)_이라고 하지만 두 가지 한계가 있습니다:

- 모델 토큰 제한으로 인해 제공할 수 있는 예시 수에 제한이 있어 효과가 제한됩니다.
- 모델 토큰 비용 때문에 모든 프롬프트에 예시를 추가하는 것이 비용 부담이 되고 유연성이 떨어집니다.

미세 조정은 사전훈련 모델에 새로운 데이터를 사용해 특정 작업 성능을 개선하기 위해 재학습시키는 기계학습의 일반적인 관행입니다. 언어 모델에서는 특정 작업이나 애플리케이션 도메인에 맞는 예시 모음으로 사전훈련 모델을 미세 조정하여 <strong>맞춤형 모델</strong>을 만들 수 있습니다. 미세 조정의 부가적 장점으로 few-shot 학습에 필요한 예시 수를 줄여 토큰 사용량과 비용도 절감할 수 있습니다.

## 언제, 왜 모델을 미세 조정해야 하는가?

여기서 말하는 미세 조정은 원본 학습 데이터셋에 포함되지 않은 <strong>새로운 데이터를 추가하여 재학습하는 감독 학습 기반 미세 조정</strong>을 의미합니다. 이는 같은 데이터에 대해 다른 하이퍼파라미터로 재학습하는 비감독 학습 미세 조정과는 구별됩니다.

기억할 핵심은 미세 조정은 원하는 결과를 얻기 위해 일정 수준의 전문성이 필요한 고급 기법이라는 점입니다. 잘못 하면 기대한 개선을 얻지 못하거나 목표 도메인에서 모델 성능이 저하될 수 있습니다.

그러므로 언어 모델을 "어떻게" 미세 조정할지 배우기 전에, "왜" 이 방법을 선택해야 하는지, "언제" 미세 조정을 시작해야 하는지 알아야 합니다. 다음 질문을 통해 점검해보세요:

- **사용 사례**: 미세 조정의 _사용 사례_는 무엇인가? 현재 사전 훈련 모델의 어떤 부분을 개선하려고 하는가?
- <strong>대안</strong>: 원하는 결과를 달성하기 위해 _다른 기법_을 시도해보았는가? 그 결과를 비교 기준으로 활용하세요.
  - 프롬프트 엔지니어링: 관련된 프롬프트 응답 예시를 통해 few-shot 프롬프트 기법을 시도하고, 응답 품질을 평가합니다.
  - 검색 증강 생성: 데이터 검색 결과를 프롬프트에 추가하는 기법을 시도하고, 응답 품질을 평가합니다.
- <strong>비용</strong>: 미세 조정 비용을 파악했는가?
  - 조정 가능성 - 사전 훈련 모델이 미세 조정 가능한가?
  - 노력 - 학습 데이터 준비, 평가 및 모델 개선에 필요한 노력
  - 컴퓨팅 - 미세 조정 작업 실행 및 미세 조정 모델 배포에 필요한 컴퓨팅 자원
  - 데이터 - 미세 조정에 영향을 줄 충분한 품질의 예시 확보
- <strong>이점</strong>: 미세 조정의 이점을 확인했는가?
  - 품질 - 미세 조정 모델이 기준 모델보다 뛰어난가?
  - 비용 - 프롬프트 단순화로 토큰 사용량이 줄어드는가?
  - 확장성 - 기반 모델을 새로운 도메인으로 재활용할 수 있는가?

이 질문에 답하면 미세 조정이 사용 사례에 적합한 접근법인지 판단할 수 있습니다. 이상적으로는 이점이 비용을 상회할 때만 타당한 방법입니다. 진행하기로 결정했다면, 이제 _어떻게_ 사전 훈련 모델을 미세 조정할지 고민할 차례입니다.

결정 과정에 대한 추가 인사이트가 필요하다면 [미세 조정할까 말까](https://www.youtube.com/watch?v=0Jo-z-MFxJs)를 시청하세요.

## 사전 훈련 모델을 어떻게 미세 조정할 수 있나요?

사전 훈련 모델을 미세 조정하려면 다음이 필요합니다:

- 미세 조정할 사전 훈련 모델
- 미세 조정에 사용할 데이터셋
- 미세 조정 작업을 실행할 훈련 환경
- 미세 조정 모델을 배포할 호스팅 환경

## Microsoft Foundry에서 미세 조정

[Microsoft Foundry](https://ai.azure.com?WT.mc_id=academic-105485-koreyst)는 Azure 상에서 맞춤형 모델을 미세 조정, 배포, 관리할 수 있는 환경입니다(이전의 Azure OpenAI Studio와 Azure AI Studio를 통합한 플랫폼). 작업을 시작하기 전에 Foundry가 제공하는 선택지와 권장되는 모범 사례를 이해하는 것이 중요합니다. Foundry는 **LoRA(저차원 적응)** 기법으로 모델을 효율적으로 미세 조정하므로 모든 가중치를 재학습하는 것보다 빠르고 저렴합니다.

### 1단계: 훈련 기법 선택하기

Foundry는 세 가지 미세 조정 기법을 지원합니다. **SFT부터 시작하세요** - 가장 많은 시나리오를 포괄합니다.

| 기법 | 동작 방식 | 사용 시점 |
| --- | --- | --- |
| **감독 미세 조정(SFT)** | 입력/출력 예제 쌍을 학습시켜 원하는 응답을 생성하도록 합니다. | 대부분 작업에 기본값: 도메인 특화, 작업 성능, 스타일과 어조, 지시 따르기, 언어 적응 등 |
| **직접 선호 최적화(DPO)** | 선호/비선호 응답 쌍에서 학습해 출력 결과를 인간 선호에 맞게 조정합니다. | 비교 피드백이 있을 때 응답 품질, 안전성, 정렬 향상에 적합 |
| **강화학습 미세 조정(RFT)** | _평가자_의 보상 신호를 사용해 강화 학습으로 복잡한 행위를 최적화합니다. | 명확한 정답이 있는 객관적, 추론 중심 도메인(수학, 화학, 물리)에 적합. 더 높은 ML 전문성이 필요 |

### 2단계: 훈련 티어 선택하기

Foundry는 훈련이 어떻게 어디서 실행될지 선택할 수 있습니다:

- **표준(Standard)** - 리소스 지역 내에서 훈련되어 데이터 소재지 보장이 필요할 때 사용.
- **글로벌(Global)** - 지역 외 용량을 활용해 저렴하고 빠르게 큐에 등록 (데이터와 가중치가 훈련 지역으로 복사됨). 데이터 소재지 제한이 없을 때 권장.
- **개발자(Developer)** - 가장 저렴하며 유휴 용량 활용. 지연/서비스 수준 보장 없음(작업 중단·재개 가능). 실험에 이상적.

### 3단계: 기반 모델 선택하기

미세 조정 가능한 모델에는 OpenAI의 `gpt-4o-mini`, `gpt-4o`, `gpt-4.1`, `gpt-4.1-mini`, `gpt-4.1-nano` (SFT; 4o/4.1 계열은 DPO도 지원), 추론 모델 `o4-mini`, `gpt-5` (RFT), 그리고 오픈 소스 모델 `Ministral-3B`, `Qwen-32B`, `Llama-3.3-70B-Instruct`, `gpt-oss-20b` (Foundry 리소스 내 SFT) 등이 있습니다. 지원 메서드, 지역, 가용성은 항상 최신 [미세 조정 모델 목록](https://learn.microsoft.com/azure/ai-foundry/foundry-models/concepts/models-sold-directly-by-azure?WT.mc_id=academic-105485-koreyst#fine-tuning-models)을 확인하세요.

> Foundry는 두 가지 모드를 제공합니다: <strong>서버리스</strong> (소비 기반 과금, GPU 쿼터 없음, OpenAI 및 일부 모델)와 **관리형 컴퓨팅** (Azure Machine Learning을 통한 VM 직접 구성, 가장 넓은 모델 지원). 대부분은 서버리스부터 시작하는 것이 좋습니다.

### Foundry 권장 사례

- **기준선 먼저 측정.** 미세 조정 전에 프롬프트 엔지니어링과 RAG를 사용해 기본 모델 성능을 측정해 개선 효과를 입증하세요.
- **작게 시작해 점차 확장.** 50~100개의 고품질 예시로 접근법을 검증한 뒤 500개 이상으로 늘리세요. 품질이 양보다 중요하므로 저품질 예시는 제거하세요.
- **데이터 형식 정확히 맞추기.** 훈련 및 검증 파일은 JSONL, UTF-8 **BOM 포함**, 512MB 미만, 챗 완료 메시지 형식을 사용해야 하며, 과적합 감시용 검증 파일을 반드시 포함하세요.
- **추론 시 시스템 프롬프트 유지하기.** 훈련 시 사용한 시스템 메시지를 모델 호출 시에도 사용하세요.
- **체크포인트 평가 - 마지막 것만 맹신하지 말기.** Foundry는 마지막 세 에폭을 배포 가능 체크포인트로 저장하며, `train_loss` / `valid_loss`와 토큰 정확도로 가장 일반화가 잘 된 체크포인트를 선택하세요.
- **품질과 토큰 비용을 함께 측정.** 미세 조정 모델과 기준선 비교 시 토큰 비용도 확인하세요.
- **지속적 미세 조정으로 반복 개선.** 미세 조정된 모델에 새로운 데이터를 추가로 학습시킬 수 있습니다(OpenAI 모델 지원).
- **호스팅 비용에 주의.** 배포된 맞춤형 모델은 시간 단위로 과금되며, 비활성 배포는 15일 후 삭제됩니다. 불필요한 리소스는 정리하세요.

[미세 조정으로 모델 맞춤화하기](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning?WT.mc_id=academic-105485-koreyst) 전 과정을 숙지하고, 다른 기법은 준비되면 [DPO](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning-direct-preference-optimization?WT.mc_id=academic-105485-koreyst)와 [RFT](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/reinforcement-fine-tuning?WT.mc_id=academic-105485-koreyst) 가이드를 참고하세요.
