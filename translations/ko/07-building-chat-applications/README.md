자동화와 원활한 인간-기계 상호작용이 정의하는 시대가 다가오면서, 생성 AI가 채팅 애플리케이션의 범위, 깊이, 적응성을 어떻게 변화시키는지 이해하는 것이 필수적입니다. 이 수업에서는 이러한 복잡한 시스템을 지원하는 아키텍처 측면을 탐구하고, 도메인별 작업에 맞게 미세 조정하는 방법론을 살펴보며, 책임 있는 AI 배포를 보장하기 위한 측정항목과 고려 사항을 평가합니다.

## 생성 AI를 채팅 애플리케이션에 통합하기

생성 AI로 채팅 애플리케이션을 향상시키는 것은 단순히 더 똑똑하게 만드는 것만이 아니라, 아키텍처, 성능, 사용자 인터페이스를 최적화해 고품질 사용자 경험을 제공하는 것입니다. 여기에는 아키텍처 기반, API 통합, 사용자 인터페이스 고려 사항을 조사하는 작업이 포함됩니다. 이 섹션은 기존 시스템에 연결하든 독립 플랫폼으로 구축하든 이러한 복잡한 분야를 탐색하기 위한 종합 로드맵을 제공합니다.

이 섹션이 끝나면 채팅 애플리케이션을 효율적으로 구축하고 통합하는 전문 지식을 갖추게 될 것입니다.

### 챗봇 vs 채팅 애플리케이션?

채팅 애플리케이션 구축에 들어가기 전에, '챗봇'과 'AI 기반 채팅 애플리케이션'의 역할과 기능이 다름을 비교해 봅시다. 챗봇의 주된 목적은 FAQ에 답하거나 패키지 추적 등 특정 대화 작업을 자동화하는 것입니다. 보통 규칙 기반 로직이나 복잡한 AI 알고리즘으로 운영됩니다. 반면 AI 기반 채팅 애플리케이션은 인간 사용자 간 텍스트, 음성, 영상 채팅 등 다양한 디지털 커뮤니케이션을 지원하는 훨씬 더 광범위한 환경입니다. 생성 AI 모델을 통합해 복잡하고 인간 같은 대화를 시뮬레이션하며, 다양한 입력과 맥락 신호에 따라 반응을 생성하는 것이 특징입니다. 생성 AI 기반 채팅 애플리케이션은 개방형 주제 논의에 참여하고, 진화하는 대화 맥락에 적응하며, 창의적이고 복잡한 대화도 생산할 수 있습니다.

아래 표는 각자의 독특한 역할을 이해하는 데 도움이 되는 주요 차이점과 유사점을 요약합니다.

| 챗봇                               | 생성 AI 기반 채팅 애플리케이션                         |
| ------------------------------------- | -------------------------------------- |
| 작업 중심 및 규칙 기반               | 맥락 인식 가능                        |
| 종종 더 큰 시스템에 통합됨          | 하나 또는 여러 챗봇을 호스팅할 수 있음    |
| 프로그래밍된 기능에 국한됨          | 생성 AI 모델 포함                      |
| 전문적이고 구조화된 상호작용         | 개방형 주제 논의 가능                   |

### SDK 및 API로 이미 구축된 기능 활용하기

채팅 애플리케이션을 구축할 때, 이미 제공되는 기능을 평가하는 것이 좋은 첫걸음입니다. SDK와 API를 이용해 채팅 애플리케이션을 구축하는 것은 여러 가지 장점이 있는 전략입니다. 잘 문서화된 SDK와 API를 통합함으로써 장기적인 성공을 위한 전략적 위치를 확보하며, 확장성과 유지보수 문제를 다룰 수 있습니다.

- **개발 프로세스 가속 및 부담 경감**: 직접 구축하는 비용이 많이 드는 과정을 대신해 이미 구축된 기능을 사용함으로써 앱의 다른 중요한 측면, 예를 들어 비즈니스 로직에 집중할 수 있습니다.
- **더 나은 성능**: 기능을 처음부터 구축할 때 "규모 확장은 어떻게 되는가? 갑작스러운 사용자 폭주를 처리할 수 있는가?"라는 질문이 생기는데, 잘 관리되는 SDK와 API는 이러한 문제에 대한 내장 솔루션을 제공합니다.
- **유지보수 용이**: 대부분의 API와 SDK는 새 버전이 출시되면 라이브러리 업데이트만으로 변경 사항을 쉽게 관리할 수 있습니다.
- **최신 기술 접근**: 광범위한 데이터셋으로 미세 조정되고 훈련된 모델을 활용하면 앱에 자연어 처리 능력을 제공합니다.

SDK나 API 기능에 접근하려면 보통 고유 키나 인증 토큰을 통해 제공되는 서비스를 사용할 수 있는 권한을 얻어야 합니다. OpenAI Python 라이브러리를 사용해 이 과정을 살펴보겠습니다. 또한, 이 수업을 위해 다음 [OpenAI 노트북](./python/oai-assignment.ipynb?WT.mc_id=academic-105485-koreyst)이나 [Azure OpenAI 서비스 노트북](./python/aoai-assignment.ipynb?WT.mc_id=academic-105485-koreys)을 직접 시도해 볼 수 있습니다.

```python
import os
from openai import OpenAI

API_KEY = os.getenv("OPENAI_API_KEY","")

client = OpenAI(
    api_key=API_KEY
    )

response = client.responses.create(model="gpt-4o-mini", input="Suggest two titles for an instructional lesson on chat applications for generative AI.", store=False)
print(response.output_text)
```

위 예제는 Responses API로 GPT-4o mini 모델을 사용해 프롬프트를 완성하지만, API 키가 설정된 후에 요청된 점을 유의하세요. 키를 설정하지 않으면 오류가 발생합니다.

## 사용자 경험 (UX)

일반적인 UX 원칙이 채팅 애플리케이션에 적용되지만, 기계 학습 구성 요소 때문에 특히 중요한 추가 고려 사항이 있습니다.

- **모호성 처리 메커니즘**: 생성 AI 모델이 때때로 모호한 답변을 생성할 수 있습니다. 사용자가 명확한 답변을 요청할 수 있는 기능이 있으면 유용합니다.
- **맥락 유지**: 고급 생성 AI 모델은 대화 내에서 맥락을 기억할 수 있는데, 이는 사용자 경험에 필요한 자산입니다. 사용자가 맥락을 제어하고 관리할 수 있게 하면 경험이 향상되지만, 민감한 정보 유지 위험도 있습니다. 보존 정책 등 정보 저장 기간 관리는 맥락 필요성과 개인정보 보호의 균형을 맞출 수 있습니다.
- <strong>개인화</strong>: AI 모델은 학습하고 적응하는 능력으로 개별화된 경험을 제공합니다. 사용자 프로필 같은 기능을 통한 맞춤형 경험은 사용자가 이해받는 느낌을 줄 뿐 아니라, 특정 답변을 찾는 데도 효율적이고 만족스러운 상호작용을 만듭니다.

개인화의 한 예는 OpenAI의 ChatGPT에 있는 "Custom instructions" 설정입니다. 사용자에 관한 정보를 제공해 프롬프트에 중요한 맥락을 전달할 수 있습니다. 다음은 맞춤 지침 예시입니다.

### 접근성

시각, 청각, 운동 또는 인지 장애가 있는 사용자도 잘 설계된 채팅 애플리케이션을 사용할 수 있어야 합니다. 다음 목록은 다양한 장애 유형에 대한 접근성을 높이기 위한 구체적 기능을 분류한 것입니다.

- **시각 장애용 기능**: 고대비 테마 및 크기 조정 가능한 텍스트, 화면 읽기 기능 호환.
- **청각 장애용 기능**: 텍스트-음성 변환 및 음성-텍스트 변환 기능, 오디오 알림에 대한 시각적 신호.
- **운동 장애용 기능**: 키보드 내비게이션 지원, 음성 명령 지원.
- **인지 장애용 기능**: 단순화된 언어 옵션.

## 도메인별 언어 모델 커스터마이징과 미세 조정

여러분 회사의 용어를 이해하고 사용자들이 자주 묻는 구체적 질문을 예상하는 채팅 애플리케이션을 상상해 보세요. 몇 가지 접근법이 있습니다:

- **DSL 모델 활용**: DSL은 도메인 특화 언어를 뜻합니다. 특정 도메인에서 훈련된 DSL 모델을 활용해 그 개념과 시나리오를 이해하도록 할 수 있습니다.
- **미세 조정 적용**: 미세 조정은 특정 데이터를 이용해 모델을 추가적으로 훈련하는 과정입니다.

## 커스터마이징: DSL 사용하기

도메인 특화 언어 모델(DSL 모델)을 활용하면 전문적이고 맥락에 맞는 상호작용으로 사용자 참여를 높일 수 있습니다. 이는 특정 분야, 산업 또는 주제와 관련된 텍스트를 이해하고 생성하도록 훈련되거나 미세 조정된 모델입니다. DSL 모델을 사용하는 방법은 직접 훈련하거나, SDK와 API를 통해 기존 모델을 사용하는 것, 그리고 사전 훈련된 모델을 도메인별로 미세 조정하는 방법까지 다양합니다.

## 커스터마이징: 미세 조정 적용

미세 조정은 사전 훈련된 모델이 특정 도메인이나 작업에서 충분하지 않을 때 고려됩니다.

예를 들어, 의료 관련 문의는 복잡하고 많은 맥락이 필요합니다. 의료 전문가는 환자를 진단할 때 생활 습관이나 기존 질환, 최근 의료 저널 등을 바탕으로 합니다. 이런 섬세한 상황에서는 일반 AI 채팅 애플리케이션이 신뢰할 만한 출처가 되기 어렵습니다.

### 시나리오: 의료 애플리케이션

의료 전문가가 치료 지침, 약물 상호작용, 최신 연구 결과를 빠르게 참고할 수 있도록 돕는 채팅 애플리케이션을 생각해 봅시다.

일반 모델은 기본 의료 질문에 답하거나 일반적인 조언을 제공할 수 있지만, 다음과 같은 경우에는 어려움을 겪을 수 있습니다:

- **매우 구체적이거나 복잡한 사례**: 예를 들어, 신경과 전문의가 "소아 환자에서 난치성 간질 관리의 최신 권장 방법은 무엇인가요?"라고 묻는 상황.
- **최신 진보 반영 부족**: 일반 모델은 신경학과 약리학의 최신 발전 사항을 반영한 현재 답변을 제공하기 어려울 수 있습니다.

이러한 경우, 특화된 의료 데이터셋으로 모델을 미세 조정하면 복잡한 의료 질문을 훨씬 정확하고 신뢰성 있게 처리할 수 있습니다. 이를 위해서는 도메인별 문제와 질문을 대표하는 크고 적절한 데이터셋에 접근할 수 있어야 합니다.

## 고품질 AI 기반 채팅 경험을 위한 고려사항

이 섹션에서는 '고품질' 채팅 애플리케이션을 위한 기준, 실행 가능한 지표를 수집하는 방법과 AI 기술을 책임감 있게 활용하는 프레임워크 준수를 다룹니다.

### 핵심 지표

애플리케이션의 고품질 성능을 유지하려면 핵심 지표와 고려 사항을 꼼꼼히 추적하는 것이 중요합니다. 이러한 측정값은 단지 애플리케이션 기능뿐 아니라 AI 모델과 사용자 경험 품질을 평가하는 데도 도움이 됩니다. 다음은 기본, AI 및 사용자 경험 지표를 포함한 목록입니다.

| 지표                          | 정의                                                                                                                   | 채팅 개발자 고려 사항                                                  |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------- |
| **가동 시간(Uptime)**         | 애플리케이션이 운영되고 사용자 접근이 가능한 시간을 측정합니다.                                                        | 다운타임을 최소화하는 방법은?                                        |
| **응답 시간**                 | 사용자의 쿼리에 애플리케이션이 응답하는 데 걸리는 시간입니다.                                                            | 쿼리 처리 최적화로 응답 시간을 개선하는 방법은?                       |
| **정밀도(Precision)**         | 양성 예측 중 실제 양성 비율입니다.                                                                                       | 모델의 정밀도를 어떻게 검증할 것인가?                                |
| **재현율(Recall, 민감도)**    | 실제 양성 중 모델이 정확히 예측한 비율입니다.                                                                             | 재현율을 어떻게 측정하고 개선할 것인가?                              |
| **F1 점수**                  | 정밀도와 재현율의 조화평균으로 두 지표 간 균형을 나타냅니다.                                                          | 목표 F1 점수는 무엇인가? 정밀도와 재현율 간 균형을 어떻게 조절할 것인가? |
| **당혹도(Perplexity)**        | 모델이 예측한 확률 분포가 실제 데이터 분포와 얼마나 잘 일치하는지 측정합니다.                                           | 당혹도를 어떻게 최소화할 것인가?                                     |
| **사용자 만족도 지표**         | 사용자의 애플리케이션에 대한 인식을 측정하며, 주로 설문조사로 수집됩니다.                                              | 얼마나 자주 피드백을 수집하고 어떻게 반영할 것인가?                   |
| **오류율(Error Rate)**         | 모델이 이해나 출력에서 실수하는 비율입니다.                                                                             | 오류율을 줄이기 위한 전략은 무엇인가?                                |
| **재훈련 주기(Retraining Cycles)** | 모델이 새로운 데이터와 인사이트를 반영해 업데이트되는 빈도입니다.                                                        | 얼마나 자주 재훈련하고, 어떤 상황에서 재훈련을 결정할 것인가?          |

| **이상 탐지**               | 예상된 동작과 일치하지 않는 비정상적인 패턴을 식별하기 위한 도구와 기술.                            | 이상 상황에 어떻게 대응할 것인가?                                          |

### 채팅 애플리케이션에서 책임 있는 AI 관행 구현

마이크로소프트의 책임 있는 AI 접근 방식은 AI 개발 및 사용을 안내하는 여섯 가지 원칙을 제시했습니다. 아래에는 원칙, 정의, 채팅 개발자가 고려해야 할 사항 및 왜 이를 진지하게 받아들여야 하는지에 대해 설명합니다.

| 원칙                  | 마이크로소프트의 정의                                   | 채팅 개발자가 고려할 점                                              | 중요한 이유                                                                       |
| ---------------------- | ----------------------------------------------------- | ------------------------------------------------------------------ | -------------------------------------------------------------------------------- |
| 공정성                | AI 시스템은 모든 사람을 공정하게 대우해야 합니다.      | 채팅 애플리케이션이 사용자 데이터를 기반으로 차별하지 않도록 보장합니다. | 사용자 간 신뢰와 포용성을 형성하며 법적 문제를 방지하기 위해서입니다.            |
| 신뢰성 및 안전성      | AI 시스템은 신뢰할 수 있고 안전하게 작동해야 합니다.   | 오류와 위험을 최소화하기 위해 테스트 및 실패 방지 장치를 구현합니다.       | 사용자 만족도를 보장하고 잠재적 위험을 예방하기 위해서입니다.                   |
| 개인정보 보호 및 보안 | AI 시스템은 안전해야 하며 개인정보를 존중해야 합니다. | 강력한 암호화 및 데이터 보호 조치를 구현합니다.                      | 민감한 사용자 데이터를 보호하고 개인정보 보호법을 준수하기 위해서입니다.      |
| 포용성                | AI 시스템은 모두에게 권한을 부여하고 참여를 유도해야 합니다. | 다양한 대상이 접근하고 사용하기 쉬운 UI/UX를 설계합니다.               | 더 많은 사람이 애플리케이션을 효과적으로 사용할 수 있도록 보장합니다.         |
| 투명성                | AI 시스템은 이해할 수 있어야 합니다.                    | AI 응답에 대해 명확한 문서화와 근거를 제공합니다.                     | 사용자가 결정 방식 이해 시 시스템을 더 신뢰할 가능성이 높아집니다.             |
| 책임성                | AI 시스템에 대한 책임은 사람에게 있어야 합니다.         | AI 결정의 감사를 위한 명확한 프로세스를 수립하고 개선합니다.          | 지속적인 개선과 오류 발생 시 시정 조치를 가능하게 합니다.                     |
