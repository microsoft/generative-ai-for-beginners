## Sigurnost

Microsoft ozbiljno shvaća sigurnost naših softverskih proizvoda i usluga, što uključuje sve repozitorije izvornog koda kojima upravljaju naše GitHub organizacije, uključujući [Microsoft](https://github.com/microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet), [Xamarin](https://github.com/xamarin) i [naše GitHub organizacije](https://opensource.microsoft.com/).

Ako smatrate da ste pronašli sigurnosnu ranjivost u bilo kojem repozitoriju u vlasništvu Microsofta koja zadovoljava [Microsoftovu definiciju sigurnosne ranjivosti](https://aka.ms/opensource/security/definition), molimo da je prijavite kako je opisano u nastavku.

## Prijavljivanje sigurnosnih problema

**Molimo nemojte prijavljivati sigurnosne ranjivosti putem javnih GitHub problema.**

Umjesto toga, prijavite ih Microsoft Security Response Center (MSRC) na [https://msrc.microsoft.com/create-report](https://aka.ms/opensource/security/create-report).

Ako želite poslati prijavu bez prijave, pošaljite e-mail na [secure@microsoft.com](mailto:secure@microsoft.com). Ako je moguće, šifrirajte svoju poruku s našim PGP ključem; molimo preuzmite ga s [Microsoft Security Response Center PGP Key stranice](https://aka.ms/opensource/security/pgpkey).

Trebali biste primiti odgovor unutar 24 sata. Ako iz nekog razloga ne dobijete odgovor, molimo pratite putem e-maila kako biste osigurali da smo primili vašu izvornu poruku. Dodatne informacije možete pronaći na [microsoft.com/msrc](https://aka.ms/opensource/security/msrc).

Molimo uključite tražene informacije navedene u nastavku (koliko god možete pružiti) kako biste nam pomogli bolje razumjeti prirodu i opseg mogućeg problema:

  * Vrsta problema (npr. preljevanje međuspremnika, SQL injekcija, skriptiranje između stranica itd.)
  * Puni putovi izvornog fajla/fajlova povezanih s pojavom problema
  * Lokacija pogođenog izvornog koda (oznaka/grana/commit ili izravni URL)
  * Bilo koja posebna konfiguracija potrebna za reprodukciju problema
  * Korak-po-korak upute za reprodukciju problema
  * Dokaz koncepta ili iskorištavanje koda (ako je moguće)
  * Utjecaj problema, uključujući kako bi napadač mogao iskoristiti problem

Ove informacije pomoći će nam brže procijeniti vaš izvještaj.

Ako prijavljujete za bug bounty, potpuniji izvještaji mogu doprinijeti većoj nagradi. Molimo posjetite našu stranicu [Microsoft Bug Bounty Program](https://aka.ms/opensource/security/bounty) za više detalja o našim aktivnim programima.
