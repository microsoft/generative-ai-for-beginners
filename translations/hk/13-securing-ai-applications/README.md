## 在生成式人工智能背景下，安全意味著什麼？

隨著人工智能（AI）和機器學習（ML）技術越來越影響我們的生活，保護不僅是客戶數據，還包括人工智能系統本身變得至關重要。在一些錯誤決策可能帶來嚴重後果的行業中，AI/ML正越來越多地用於支持高價值決策過程。

以下是需要考慮的關鍵點：

- **AI/ML的影響**：AI/ML對日常生活有重大影響，因此保護它們已成為必要。
- **安全挑戰**：AI/ML的影響需要適當的關注，以應對保護基於AI的產品免受精密攻擊的需求，無論是由惡意網絡還是有組織的團體。
- **策略性問題**：科技行業必須主動應對策略性挑戰，以確保長期客戶安全和數據安全。

此外，機器學習模型大多無法分辨惡意輸入和良性異常數據。大量訓練數據來自未經策劃、未經審核的公共數據集，這些數據集開放給第三方貢獻者。攻擊者不需要破壞數據集，因為他們可以自由地對其進行貢獻。隨著時間的推移，如果數據結構/格式保持正確，低可信度的惡意數據會變成高可信度的可信數據。

這就是為什麼確保你的模型用於決策的數據存儲的完整性和保護至關重要。

## 理解人工智能的威脅和風險

在人工智能及相關系統方面，數據污染是當今最重要的安全威脅。數據污染是指有人故意更改用於訓練人工智能的信息，導致其出錯。這是由於缺乏標準化的檢測和減輕方法，加上我們依賴不受信任或未經策劃的公共數據集進行訓練。為了維持數據完整性並防止訓練過程中的缺陷，追蹤你的數據的來源和傳承至關重要。否則，古老的格言“垃圾進，垃圾出”會成真，導致模型性能受損。

以下是數據污染如何影響你的模型的例子：

1. **標籤翻轉**：在二元分類任務中，敵手故意翻轉一小部分訓練數據的標籤。例如，良性樣本被標記為惡意，導致模型學習錯誤的關聯。\
   **例子**：由於標籤被操縱，垃圾郵件過濾器將合法電子郵件錯誤分類為垃圾郵件。
2. **特徵污染**：攻擊者巧妙地修改訓練數據中的特徵以引入偏見或誤導模型。\
   **例子**：在產品描述中添加不相關的關鍵詞以操縱推薦系統。
3. **數據注入**：將惡意數據注入訓練集以影響模型的行為。\
   **例子**：引入虛假的用戶評論以偏斜情感分析結果。
4. **後門攻擊**：敵手在訓練數據中插入隱藏模式（後門）。模型學會識別這種模式，並在觸發時表現惡意。\
   **例子**：面部識別系統被訓練有後門的圖像，錯誤識別特定人物。

## 人工智能系統和LLMs的安全測試

人工智能（AI）正在改變各個領域和行業，為社會提供新的可能性和好處。然而，AI也帶來了重大挑戰和風險，例如數據隱私、偏見、缺乏可解釋性和潛在的濫用。因此，確保AI系統安全和負責任至關重要，意味著它們遵循倫理和法律標準，並且可以被用戶和利益相關者信任。

安全測試是評估AI系統或LLM安全性的方法，通過識別和利用其漏洞來進行。這可以由開發人員、用戶或第三方審核員進行，取決於測試的目的和範圍。AI系統和LLMs最常見的安全測試方法包括：

- **數據清理**：這是從AI系統或LLM的訓練數據或輸入中移除或匿名化敏感或私人信息的過程。數據清理可以通過減少機密或個人數據的暴露來防止數據泄露和惡意操縱。
- **對抗性測試**：這是生成和應用對抗性示例到AI系統或LLM的輸入或輸出，以評估其對抗攻擊的魯棒性和韌性。對抗性測試可以幫助識別和減輕AI系統或LLM可能被攻擊者利用的漏洞和弱點。
- **模型驗證**：這是驗證AI系統或LLM的模型參數或架構的正確性和完整性的過程。模型驗證可以通過確保模型受到保護和認證來幫助檢測和防止模型竊取。
- **輸出驗證**：這是驗證AI系統或LLM輸出的質量和可靠性的過程。輸出驗證可以通過確保輸出一致和準確來幫助檢測和糾正惡意操縱。

### AI安全

我們必須努力保護AI系統免受惡意攻擊、濫用或意外後果。這包括採取措施確保AI系統的安全性、可靠性和可信度，例如：

- 保護用於訓練和運行AI模型的數據和算法
- 防止未經授權的訪問、操縱或破壞AI系統
- 檢測和減輕AI系統中的偏見、歧視或倫理問題
- 確保AI決策和行動的責任、透明度和可解釋性
- 使AI系統的目標和價值觀與人類和社會的目標和價值觀保持一致

AI安全對於確保AI系統和數據的完整性、可用性和機密性至關重要。AI安全的一些挑戰和機會包括：

- 機會：將AI納入網絡安全策略中，因為它可以在識別威脅和提高響應速度方面發揮重要作用。AI可以幫助自動化和增強網絡攻擊的檢測和減輕，例如釣魚、惡意軟件或勒索軟件。
- 挑戰：AI也可以被對手用來發起複雜的攻擊，例如生成虛假或誤導性內容、冒充用戶或利用AI系統中的漏洞。因此，AI開發人員有責任設計出穩健且能抵禦濫用的系統。

### 數據保護

LLMs可能會對它們使用的數據的隱私和安全性造成風險。例如，LLMs可能會記住並泄露訓練數據中的敏感信息，例如個人姓名、地址、密碼或信用卡號碼。它們也可能被惡意行為者操縱或攻擊，這些行為者希望利用它們的漏洞或偏見。因此，重要的是要意識到這些風險並採取適當措施來保護使用LLMs的數據。你可以採取一些步驟來保護與LLMs一起使用的數據。這些步驟包括：

- **限制與LLMs共享的數據的數量和類型**：僅共享必要和相關的數據，並避免共享任何敏感、機密或個人數據。用戶還應匿名化或加密與LLMs共享的數據，例如通過移除或掩蓋任何識別信息，或使用安全通信渠道。
- **驗證LLMs生成的數據**：始終檢查LLMs生成的輸出的準確性和質量，以確保它們不包含任何不需要或不當的信息。
- **報告和警示任何數據泄露或事件**：警惕LLMs的任何可疑或異常活動或行為，例如生成不相關、不準確、冒犯或有害的文本。這可能是數據泄露或安全事件的跡象。

數據安全、治理和合規對於任何希望在多雲環境中利用數據和AI的力量的組織來說都是至關重要的。保護和治理所有你的數據是一項複雜且多方面的工作。你需要保護和治理不同類型的數據（結構化、非結構化和AI生成的數據）在多個雲中的不同位置，並且需要考慮現有和未來的數據安全、治理和AI法規。為了保護你的數據，你需要採用一些最佳做法和預防措施，例如：

- 使用提供數據保護和隱私功能的雲服務或平台。
- 使用數據質量和驗證工具檢查你的數據是否有錯誤、不一致或異常。
- 使用數據治理和倫理框架以確保你的數據以負責任和透明的方式使用。

### 模擬現實世界的威脅 - AI紅隊

模擬現實世界的威脅現在被視為建立有韌性的AI系統的標準做法，通過使用類似的工具、策略、程序來識別系統的風險並測試防禦者的響應。

1. **AI紅隊範圍的擴展**：
   AI紅隊現在涵蓋安全和負責任AI（RAI）結果。傳統上，紅隊側重於安全方面，將模型視為向量（例如，竊取底層模型）。然而，AI系統引入了新穎的安全漏洞（例如，提示注入、污染），需要特別關注。除了安全之外，AI紅隊還探測公平問題（例如，刻板印象）和有害內容（例如，美化暴力）。早期識別這些問題允許防禦投資的優先排序。
