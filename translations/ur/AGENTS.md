# AGENTS.md

## پروجیکٹ کا جائزہ

یہ ریپوزیٹری جنریٹو AI کی بنیادی باتیں اور ایپلیکیشن ڈیولپمنٹ سکھانے کے لیے جامع 21 اسباق پر مشتمل نصاب رکھتی ہے۔ یہ کورس ابتدائی افراد کے لیے بنایا گیا ہے اور بنیادی تصورات سے لے کر پیداوار کے لیے تیار ایپلیکیشنز بنانے تک سب کچھ شامل ہے۔

**اہم ٹیکنالوجیز:**
- Python 3.9+ لائبریریز کے ساتھ: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript Node.js اور لائبریریز کے ساتھ: `openai` (Azure OpenAI v1 اینڈپوائنٹ + Responses API کے ذریعہ)، `@azure-rest/ai-inference` (Microsoft Foundry Models)
- Azure OpenAI سروس، OpenAI API، اور Microsoft Foundry Models (GitHub Models جولائی 2026 کے آخر میں ریٹائر ہو رہے ہیں)
- انٹرایکٹو سیکھنے کے لیے Jupyter Notebooks
- مستقل ترقیاتی ماحول کے لیے Dev Containers

**ریپوزیٹری کا ڈھانچہ:**
- 21 عدد والے اسباق کے ڈائریکٹریز (00-21) جن میں README، کوڈ کے مثالیں، اور اسائنمنٹس شامل ہیں
- متعدد نفاذ: Python، TypeScript، اور کبھی کبھار .NET مثالیں
- ترجمہ جات کی ڈائریکٹری جس میں 40+ زبانوں کے ورژن شامل ہیں
- `.env` فائل کے ذریعہ مرکزی ترتیب (ٹیمپلیٹ کے طور پر `.env.copy` استعمال کریں)

## سیٹ اپ کمانڈز

### ابتدائی ریپوزیٹری سیٹ اپ

```bash
# ذخیرہ خانہ کلون کریں
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# ماحول کا سانچہ نقل کریں
cp .env.copy .env
# اپنی API چابیاں اور اختتامی نکات کے ساتھ .env کو ترمیم کریں
```

### Python ماحول کا سیٹ اپ

```bash
# ورچوئل ماحول بنائیں
python3 -m venv venv

# ورچوئل ماحول کو فعال کریں
# میک او ایس/لینکس پر:
source venv/bin/activate
# ونڈوز پر:
venv\Scripts\activate

# انحصاریات انسٹال کریں
pip install -r requirements.txt
```

### Node.js/TypeScript سیٹ اپ

```bash
# جڑ کی سطح کی انحصاریات انسٹال کریں (دستاویزات کے اوزار کے لیے)
npm install

# انفرادی سبق TypeScript کی مثالوں کے لیے، مخصوص سبق پر جائیں:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Dev Container سیٹ اپ (مشورہ دیا گیا)

ریپوزیٹری میں GitHub Codespaces یا VS Code Dev Containers کے لیے `.devcontainer` کنفیگریشن شامل ہے:

1. ریپوزیٹری کو GitHub Codespaces یا VS Code میں Dev Containers ایکسٹینشن کے ساتھ کھولیں
2. Dev Container خود بخود:
   - `requirements.txt` سے Python کی dependencies انسٹال کرے گا
   - پوسٹ-کریئٹ اسکرپٹ چلائے گا (`.devcontainer/post-create.sh`)
   - Jupyter کرنل سیٹ اپ کرے گا

## ترقیاتی ورک فلو

### ماحول کی متغیرات

تمام اسباق جنہیں API تک رسائی کی ضرورت ہے `.env` فائل میں تعریف شدہ ماحول کی متغیرات استعمال کرتے ہیں:

- `OPENAI_API_KEY` - OpenAI API کے لیے
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry میں Azure OpenAI کے لیے (Azure OpenAI سروس اب Microsoft Foundry کا حصہ ہے: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI اینڈپوائنٹ URL (Foundry resource endpoint)
- `AZURE_OPENAI_DEPLOYMENT` - چیٹ کمپلیشن ماڈل کی تعیناتی کا نام
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - ایمبیڈنگ ماڈل کی تعیناتی کا نام
- `AZURE_OPENAI_API_VERSION` - API ورژن (ڈیفالٹ: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face ماڈلز کے لیے
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry Models اینڈپوائنٹ (کئی فراہم کنندہ ماڈل کیٹلاگ)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry Models API کی کلید (ریٹائر ہونے والے `GITHUB_TOKEN` کی جگہ)

### Python مثالیں چلانا

```bash
# سبق کی ڈائریکٹری پر جائیں
cd 06-text-generation-apps/python

# ایک پائتھن اسکرپٹ چلائیں
python aoai-app.py
```

### TypeScript مثالیں چلانا

```bash
# ٹائپ اسکرپٹ ایپ ڈائریکٹری پر جائیں
cd 06-text-generation-apps/typescript/recipe-app

# ٹائپ اسکرپٹ کوڈ بنائیں
npm run build

# ایپلیکیشن چلائیں
npm start
```

### Jupyter Notebooks چلانا

```bash
# جیوپیٹر کو رپوزیٹری روٹ میں شروع کریں
jupyter notebook

# یا جیوپیٹر ایکسٹینشن کے ساتھ وی ایس کوڈ استعمال کریں
```

## کوڈ اسٹائل کے رہنما اصول

### Python

- ماحول کی متغیرات کے انتظام کے لیے `python-dotenv` کا استعمال کریں
- API تعاملات کے لیے `openai` لائبریری امپورٹ کریں
- لینٹنگ کے لیے `pylint` کا استعمال کریں (کچھ مثالوں میں آسانی کے لیے `# pylint: disable=all` شامل ہے)
- PEP 8 کے ناموں کے اصولوں کی پیروی کریں
- API کی معلومات `.env` فائل میں محفوظ کریں، کبھی کوڈ میں نہیں

### TypeScript

- ماحول کی متغیرات کے لیے `dotenv` پیکیج کا استعمال کریں
- ہر ایپ کے لیے `tsconfig.json` میں TypeScript کی ترتیب
- Azure OpenAI کے لیے `openai` پیکیج کا استعمال کریں (کلائنٹ کو `/openai/v1/` اینڈپوائنٹ کی طرف پوائنٹ کریں اور `client.responses.create` کال کریں)؛ Microsoft Foundry Models کے لیے `@azure-rest/ai-inference` استعمال کریں
- ترقی کے لیے `nodemon` کا استعمال کریں جو آٹو ری لوڈ کرتا ہے
- چلانے سے پہلے بنائیں: `npm run build` پھر `npm start`

### عمومی اصول

- کوڈ کی مثالیں سادہ اور تعلیمی رکھیں
- اہم تصورات کی وضاحت کرنے والے کمنٹس شامل کریں
- ہر سبق کا کوڈ خود مختار اور چلانے کے قابل ہونا چاہیے
- مستقل نام کاری کا استعمال کریں: Azure OpenAI کے لیے `aoai-` پری فکس، OpenAI API کے لیے `oai-`، Microsoft Foundry Models کے لیے `githubmodels-` (GitHub Models دور کا پرانا پری فکس برقرار رکھا گیا ہے)

## عام ورک فلو

### نیا کوڈ کی مثال شامل کرنا

1. متعلقہ سبق کی ڈائریکٹری میں جائیں
2. `python/` یا `typescript/` سب ڈائریکٹری میں مثال بنائیں
3. نام کاری کا اصول اپنائیں: `{provider}-{example-name}.{py|ts|js}`
4. حقیقی API کی چابیوں کے ساتھ ٹیسٹ کریں
5. سبق کے README میں نئے ماحول کی متغیرات دستاویزی شکل میں شامل کریں

### دستاویزات کو اپ ڈیٹ کرنا

1. سبق کی ڈائریکٹری میں README.md ترمیم کریں
2. مارک ڈاؤن رہنما اصول کی پیروی کریں (ٹریکنگ IDs، نسبتی لنکس)
3. ترجمہ جات خودکار GitHub Actions کے ذریعے ہینڈل ہوتے ہیں (دستی ترمیم نہ کریں)
4. تمام لنکس کی درستگی آزمائیں

### Dev Containers کے ساتھ کام کرنا

1. ریپوزیٹری میں `.devcontainer/devcontainer.json` شامل ہے
2. پوسٹ کریئٹ اسکرپٹ Python dependencies خود بخود انسٹال کرتا ہے
3. Python اور Jupyter کے ایکسٹینشنز پہلے سے ترتیب دیے گئے ہیں
4. ماحول `mcr.microsoft.com/devcontainers/universal:2.11.2` پر مبنی ہے

## مسئلے حل کرنا

### عام مسائل

**Python امپورٹ کی غلطیاں**:
- یقینی بنائیں کہ ورچوئل ماحول فعال ہے
- `pip install -r requirements.txt` چلائیں

**TypeScript کی تعمیر کی غلطیاں**:
- مخصوص ایپ ڈائریکٹری میں `npm install` چلائیں
- Node.js کا ورژن موافق ہو چیک کریں
- `node_modules` صاف کریں اور اگر ضرورت ہو دوبارہ انسٹال کریں

**API توثیقی غلطیاں**:
- تصدیق کریں کہ `.env` فائل موجود ہے اور درست اقدار رکھتی ہے
- API کی چابیاں درست اور میعاد ختم نہیں ہوئیں چیک کریں
- یقینی بنائیں کہ اینڈپوائنٹ URLs آپ کے علاقے کے لیے درست ہیں

**ماحول کی متغیرات غائب ہیں**:
- `.env.copy` کو `.env` میں کاپی کریں
- جس سبق پر کام کر رہے ہیں اس کے تمام ضروری اقدار بھر جائیں
