## 4. API جدید کاری

### 4.1 پرانی API پیٹرنز (مائیگریشن مکمل)

تمام پائتھن اور ٹائپ اسکرپٹ **چیٹ** سیمپلز کو Chat Completions API سے **Responses API** (`client.responses.create(...)` → `response.output_text`) پر منتقل کیا گیا ہے۔

| پرانا پیٹرن | نیا پیٹرن | حالت |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (چیٹ) | `OpenAI(base_url="<endpoint>/openai/v1/")` (Responses API) | مکمل شدہ |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | مکمل شدہ |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (ٹائپ اسکرپٹ) | `openai` پیکج `client.responses.create()` → `response.output_text` | مکمل شدہ |
| `df.append()` (pandas) | `pd.concat()` | مکمل شدہ |

> **نوٹ:** Microsoft Foundry Models سیمپلز جو `azure-ai-inference` / `@azure-rest/ai-inference` SDK (`client.complete()`) استعمال کرتے ہیں، وہ Model Inference API پر رہتے ہیں، جو Responses API کی حمایت نہیں کرتا۔ `AzureOpenAI()` جہاں اب بھی قابل اطلاق ہے وہاں برقرار رکھا گیا ہے (ایمبیڈنگز اور تصویر کی تخلیق کے لیے)۔

### 4.2 نئی API خصوصیات کی نمائش

1. **ساختہ آؤٹ پٹ** (OpenAI)
   - JSON موڈ
   - سخت اسکیمز کے ساتھ فنکشن کالنگ

2. **نظری صلاحیتیں**
   - GPT-4o (ویژن) کے ساتھ تصویر کا تجزیہ
   - کثیر الجہتی پرامپٹس

3. **Responses API کے بلٹ ان ٹولز** (پرانی Assistants API کی جگہ)
   - کوڈ انٹرپرٹر
   - فائل تلاش
   - ویب تلاش اور کسٹم ٹولز

## 5. انفراسٹرکچر کی بہتریاں

### 5.1 CI/CD بہتریاں

[`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml) میں نافذ: Python کی لنٹنگ/فارمیٹنگ (Ruff + Black) کو **shared/** یوٹیلٹیز ماڈیول پر لازمی بنایا گیا ہے اور باقی نصاب میں **مشورتی** طور پر چلایا جاتا ہے، اور جاوا اسکرپٹ/ٹائپ اسکرپٹ کے لیے ESLint کا مشورتی پاس بھی شامل ہے۔ تشریعی بنیاد یہ تھی:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### 5.2 سیکیورٹی اسکیننگ

[`.github/workflows/security.yml`](../../../.github/workflows/security.yml) میں نافذ: کوڈ کیول تجزیہ Python اور JavaScript/TypeScript کے لیے (push، pull request، اور ہفتہ وار شیڈول پر) اور pull requests پر انحصار کا جائزہ۔ تشریعی بنیاد یہ تھی:

```yaml
# .github/workflows/security.yml
name: Security Scan

on: [push, pull_request]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: javascript, python
      - uses: github/codeql-action/analyze@v3

  dependency-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
```
