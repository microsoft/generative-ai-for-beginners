# سرچ ایپلیکیشنز بنانا

> > _اس سبق کی ویڈیو دیکھنے کے لیے اوپر دی گئی تصویر پر کلک کریں_

ایل ایل ایمز صرف چیٹ بوٹس اور ٹیکسٹ جنریشن تک محدود نہیں ہیں۔ ایمبیڈنگز کا استعمال کرتے ہوئے سرچ ایپلیکیشنز بھی بنائی جا سکتی ہیں۔ ایمبیڈنگز ڈیٹا کی عددی نمائندگی ہوتی ہیں، جنہیں ویکٹر بھی کہا جاتا ہے، اور یہ ڈیٹا کی معنوی تلاش کے لیے استعمال کی جا سکتی ہیں۔

اس سبق میں، آپ ہمارے تعلیمی اسٹارٹ اپ کے لیے ایک سرچ ایپلیکیشن بنائیں گے۔ ہمارا اسٹارٹ اپ ایک غیر منافع بخش تنظیم ہے جو ترقی پذیر ممالک کے طلباء کو مفت تعلیم فراہم کرتا ہے۔ ہمارے اسٹارٹ اپ کے پاس بہت سے یوٹیوب ویڈیوز ہیں جنہیں طلباء اے آئی کے بارے میں سیکھنے کے لیے استعمال کر سکتے ہیں۔ ہمارا اسٹارٹ اپ ایک سرچ ایپلیکیشن بنانا چاہتا ہے جو طلباء کو سوال ٹائپ کرکے یوٹیوب ویڈیو تلاش کرنے کی اجازت دے۔

مثال کے طور پر، ایک طالب علم 'جیوپیٹر نوٹ بکس کیا ہیں؟' یا 'ایزور ایم ایل کیا ہے؟' ٹائپ کر سکتا ہے اور سرچ ایپلیکیشن سوال سے متعلق یوٹیوب ویڈیوز کی فہرست واپس کرے گی، اور مزید بہتر یہ کہ سرچ ایپلیکیشن ویڈیو میں اس جگہ کا لنک واپس کرے گی جہاں سوال کا جواب موجود ہے۔

## تعارف

اس سبق میں، ہم شامل کریں گے:

- معنوی بمقابلہ کی ورڈ سرچ۔
- ٹیکسٹ ایمبیڈنگز کیا ہیں۔
- ٹیکسٹ ایمبیڈنگز انڈیکس بنانا۔
- ٹیکسٹ ایمبیڈنگز انڈیکس میں تلاش کرنا۔

## سیکھنے کے مقاصد

اس سبق کو مکمل کرنے کے بعد، آپ قابل ہو جائیں گے:

- معنوی اور کی ورڈ سرچ کے درمیان فرق بتائیں۔
- ٹیکسٹ ایمبیڈنگز کیا ہیں وضاحت کریں۔
- ڈیٹا تلاش کرنے کے لیے ایمبیڈنگز کا استعمال کرتے ہوئے ایک ایپلیکیشن بنائیں۔

## سرچ ایپلیکیشن کیوں بنائیں؟

سرچ ایپلیکیشن بنانا آپ کو یہ سمجھنے میں مدد دے گا کہ ڈیٹا کی تلاش کے لیے ایمبیڈنگز کا استعمال کیسے کیا جاتا ہے۔ آپ یہ بھی سیکھیں گے کہ ایک ایسی سرچ ایپلیکیشن کیسے بنائی جائے جو طلباء کو جلدی معلومات حاصل کرنے میں مدد دے سکے۔

سبق میں مائیکروسافٹ [AI شو](https://www.youtube.com/playlist?list=PLlrxD0HtieHi0mwteKBOfEeOYf0LJU4O1) یوٹیوب چینل کے یوٹیوب ٹرانسکرپٹس کا ایمبیڈنگ انڈیکس شامل ہے۔ AI شو ایک یوٹیوب چینل ہے جو آپ کو اے آئی اور مشین لرننگ کے بارے میں سکھاتا ہے۔ ایمبیڈنگ انڈیکس میں اکتوبر 2023 تک کے ہر یوٹیوب ٹرانسکرپٹ کے ایمبیڈنگز شامل ہیں۔ آپ ہمارے اسٹارٹ اپ کے لیے ایک سرچ ایپلیکیشن بنانے کے لیے ایمبیڈنگ انڈیکس کا استعمال کریں گے۔ سرچ ایپلیکیشن ویڈیو میں اس جگہ کا لنک واپس کرے گی جہاں سوال کا جواب موجود ہے۔ یہ طلباء کے لیے جلدی معلومات حاصل کرنے کا ایک بہترین طریقہ ہے۔

ذیل میں 'کیا آپ آر اسٹوڈیو کو ایزور ایم ایل کے ساتھ استعمال کر سکتے ہیں؟' سوال کے لیے معنوی تلاش کی ایک مثال ہے۔ یوٹیوب یو آر ایل کو چیک کریں، آپ دیکھیں گے کہ یو آر ایل میں ایک ٹائم اسٹیمپ شامل ہے جو آپ کو ویڈیو میں اس جگہ لے جاتا ہے جہاں سوال کا جواب موجود ہے۔

## معنوی تلاش کیا ہے؟

اب آپ سوچ رہے ہوں گے کہ معنوی تلاش کیا ہے؟ معنوی تلاش ایک تلاش کی تکنیک ہے جو سوال میں الفاظ کے معنی یا سیمنٹکس کا استعمال کرتی ہے تاکہ متعلقہ نتائج واپس کیے جا سکیں۔

یہاں معنوی تلاش کی ایک مثال ہے۔ فرض کریں آپ ایک گاڑی خریدنا چاہتے ہیں، آپ 'میری خواب کی گاڑی' تلاش کر سکتے ہیں، معنوی تلاش سمجھتی ہے کہ آپ گاڑی کے بارے میں خواب نہیں دیکھ رہے، بلکہ آپ اپنی خواب کی گاڑی خریدنا چاہتے ہیں۔ معنوی تلاش آپ کی نیت کو سمجھتی ہے اور متعلقہ نتائج واپس کرتی ہے۔ متبادل یہ ہے کہ `keyword search` جو لفظی طور پر گاڑیوں کے بارے میں خوابوں کی تلاش کرے گا اور اکثر غیر متعلقہ نتائج واپس کرے گا۔

## ٹیکسٹ ایمبیڈنگز کیا ہیں؟

[ٹیکسٹ ایمبیڈنگز](https://en.wikipedia.org/wiki/Word_embedding?WT.mc_id=academic-105485-koreyst) قدرتی زبان کی پروسیسنگ میں استعمال ہونے والی ایک ٹیکسٹ نمائندگی کی تکنیک ہے۔ ٹیکسٹ ایمبیڈنگز ٹیکسٹ کی معنوی عددی نمائندگی ہوتی ہیں۔ ایمبیڈنگز ڈیٹا کو اس طرح سے ظاہر کرنے کے لیے استعمال کی جاتی ہیں جو مشین کے لیے سمجھنا آسان ہو۔ ٹیکسٹ ایمبیڈنگز بنانے کے بہت سے ماڈل ہیں، اس سبق میں ہم OpenAI ایمبیڈنگ ماڈل کا استعمال کرتے ہوئے ایمبیڈنگز پیدا کرنے پر توجہ مرکوز کریں گے۔

یہاں ایک مثال ہے، تصور کریں کہ مندرجہ ذیل ٹیکسٹ AI شو یوٹیوب چینل کے ایک ایپیسوڈ کے ٹرانسکرپٹ میں موجود ہے:

```text
Today we are going to learn about Azure Machine Learning.
```

ہم ٹیکسٹ کو OpenAI ایمبیڈنگ API کے حوالے کریں گے اور یہ 1536 نمبروں کا ایک ایمبیڈنگ واپس کرے گا جسے ویکٹر کہا جاتا ہے۔ ویکٹر میں ہر نمبر ٹیکسٹ کے ایک مختلف پہلو کی نمائندگی کرتا ہے۔ اختصار کے لیے، ویکٹر میں پہلے 10 نمبر یہ ہیں:

```python
[-0.006655829958617687, 0.0026128944009542465, 0.008792596869170666, -0.02446001023054123, -0.008540431968867779, 0.022071078419685364, -0.010703742504119873, 0.003311325330287218, -0.011632772162556648, -0.02187200076878071, ...]
```

## ایمبیڈنگ انڈیکس کیسے بنایا جاتا ہے؟

اس سبق کے لیے ایمبیڈنگ انڈیکس پائیتھون اسکرپٹس کی ایک سیریز کے ساتھ بنایا گیا تھا۔ آپ کو اس سبق کے 'اسکرپٹس' فولڈر میں [README](./scripts/README.md?WT.mc_id=academic-105485-koreyst) کے ساتھ اسکرپٹس اور ہدایات ملیں گی۔ آپ کو اس سبق کو مکمل کرنے کے لیے ان اسکرپٹس کو چلانے کی ضرورت نہیں ہے کیونکہ ایمبیڈنگ انڈیکس آپ کے لیے فراہم کیا گیا ہے۔

اسکرپٹس درج ذیل آپریشنز انجام دیتے ہیں:

1. [AI شو](https://www.youtube.com/playlist?list=PLlrxD0HtieHi0mwteKBOfEeOYf0LJU4O1) پلے لسٹ میں ہر یوٹیوب ویڈیو کے لیے ٹرانسکرپٹ ڈاؤن لوڈ کیا جاتا ہے۔
2. [OpenAI فنکشنز](https://learn.microsoft.com/azure/ai-services/openai/how-to/function-calling?WT.mc_id=academic-105485-koreyst) کا استعمال کرتے ہوئے، یوٹیوب ٹرانسکرپٹ کے پہلے 3 منٹ سے اسپیکر کا نام نکالنے کی کوشش کی جاتی ہے۔ ہر ویڈیو کے لیے اسپیکر کا نام ایمبیڈنگ انڈیکس `embedding_index_3m.json` میں محفوظ کیا جاتا ہے۔
3. پھر ٹرانسکرپٹ ٹیکسٹ کو **3 منٹ کے ٹیکسٹ حصوں** میں تقسیم کیا جاتا ہے۔ حصے میں اگلے حصے سے تقریباً 20 الفاظ شامل ہوتے ہیں تاکہ اس بات کو یقینی بنایا جا سکے کہ حصے کے لیے ایمبیڈنگ کو کاٹا نہ جائے اور بہتر تلاش کے سیاق و سباق فراہم کیے جا سکیں۔
4. پھر ہر ٹیکسٹ حصے کو OpenAI چیٹ API کے حوالے کیا جاتا ہے تاکہ ٹیکسٹ کو 60 الفاظ میں خلاصہ کیا جا سکے۔ خلاصہ بھی ایمبیڈنگ انڈیکس `embedding_index_3m.json` میں محفوظ کیا جاتا ہے۔
5. آخر میں، ٹیکسٹ حصے کو OpenAI ایمبیڈنگ API کے حوالے کیا جاتا ہے۔ ایمبیڈنگ API 1536 نمبروں کا ایک ویکٹر واپس کرتا ہے جو حصے کے معنوی معنی کی نمائندگی کرتا ہے۔ حصہ کے ساتھ OpenAI ایمبیڈنگ ویکٹر کو ایمبیڈنگ انڈیکس `embedding_index_3m.json` میں محفوظ کیا جاتا ہے۔

### ویکٹر ڈیٹا بیسز

سبق کی سادگی کے لیے، ایمبیڈنگ انڈیکس کو ایک JSON فائل `embedding_index_3m.json` میں محفوظ کیا جاتا ہے اور پانڈاس ڈیٹا فریم میں لوڈ کیا جاتا ہے۔ تاہم، پروڈکشن میں، ایمبیڈنگ انڈیکس کو ایک ویکٹر ڈیٹا بیس میں محفوظ کیا جائے گا جیسے کہ [Azure Cognitive Search](https://learn.microsoft.com/training/modules/improve-search-results-vector-search?WT.mc_id=academic-105485-koreyst)، [Redis](https://cookbook.openai.com/examples/vector_databases/redis/readme?WT.mc_id=academic-105485-koreyst)، [Pinecone](https://cookbook.openai.com/examples/vector_databases/pinecone/readme?WT.mc_id=academic-105485-koreyst)، [Weaviate](https://cookbook.openai.com/examples/vector_databases/weaviate/readme?WT.mc_id=academic-105485-koreyst)، چند کے نام۔

## کوسائن مماثلت کو سمجھنا

ہم نے ٹیکسٹ ایمبیڈنگز کے بارے میں سیکھا، اگلا قدم یہ سیکھنا ہے کہ ڈیٹا کی تلاش کے لیے ٹیکسٹ ایمبیڈنگز کا استعمال کیسے کیا جائے اور خاص طور پر کوسائن مماثلت کا استعمال کرتے ہوئے دیے گئے سوال کے لیے سب سے زیادہ مماثل ایمبیڈنگز کیسے تلاش کی جائیں۔

### کوسائن مماثلت کیا ہے؟

کوسائن مماثلت دو ویکٹرز کے درمیان مماثلت کی پیمائش ہے، آپ اسے `nearest neighbor search` کے نام سے بھی سنیں گے۔ کوسائن مماثلت کی تلاش کرنے کے لیے آپ کو سوال کے ٹیکسٹ کے لیے OpenAI ایمبیڈنگ API کا استعمال کرتے ہوئے _ویکٹرائز_ کرنے کی ضرورت ہے۔ پھر سوال ویکٹر اور ایمبیڈنگ انڈیکس میں موجود ہر ویکٹر کے درمیان _کوسائن مماثلت_ کا حساب لگائیں۔ یاد رکھیں، ایمبیڈنگ انڈیکس میں ہر یوٹیوب ٹرانسکرپٹ ٹیکسٹ حصے کے لیے ایک ویکٹر موجود ہے۔ آخر میں، نتائج کو کوسائن مماثلت کے حساب سے ترتیب دیں اور وہ ٹیکسٹ حصے جن کی کوسائن مماثلت سب سے زیادہ ہوتی ہے وہ سوال کے لیے سب سے زیادہ مماثل ہوتے ہیں۔

ریاضی کے نقطہ نظر سے، کوسائن مماثلت دو ویکٹرز کے درمیان زاویہ کے کوسائن کی پیمائش کرتی ہے جو کہ کثیر الجہتی خلا میں پروجیکٹ ہوتے ہیں۔ یہ پیمائش فائدہ مند ہے، کیونکہ اگر دو دستاویزات حجم کی وجہ سے یورپین فاصلہ سے دور ہیں، تو ان کے درمیان اب بھی ایک چھوٹا زاویہ ہو سکتا ہے اور اس طرح کوسائن مماثلت زیادہ ہو سکتی ہے۔ کوسائن مماثلت کے فارمولوں کے بارے میں مزید معلومات کے لیے [کوسائن مماثلت](https://en.wikipedia.org/wiki/Cosine_similarity?WT.mc_id=academic-105485-koreyst) دیکھیں۔

## اپنی پہلی سرچ ایپلیکیشن بنانا

اگلا، ہم سیکھیں گے کہ ایمبیڈنگز کا استعمال کرتے ہوئے ایک سرچ ایپلیکیشن کیسے بنائی جائے۔ سرچ ایپلیکیشن طلباء کو سوال ٹائپ کرکے ویڈیو تلاش کرنے کی اجازت دے گی۔ سرچ ایپلیکیشن سوال سے متعلق ویڈیوز کی فہرست واپس کرے گی۔ سرچ ایپلیکیشن ویڈیو میں اس جگہ کا لنک بھی واپس کرے گی جہاں سوال کا جواب موجود ہے۔

یہ حل ونڈوز 11، میک او ایس، اور اوبنٹو 22.04 پر Python 3.10 یا اس سے بعد کے ورژن کا استعمال کرتے ہوئے بنایا اور جانچا گیا ہے۔ آپ [python.org](https://www.python.org/downloads/?WT.mc_id=academic-105485-koreyst) سے Python ڈاؤن لوڈ کر سکتے ہیں۔

## اسائنمنٹ - طلباء کو فعال کرنے کے لیے سرچ ایپلیکیشن بنانا

ہم نے اس سبق کے آغاز میں اپنے اسٹارٹ اپ کا تعارف کرایا۔ اب وقت ہے کہ طلباء کو ان کے اسائنمنٹس کے لیے سرچ ایپلیکیشن بنانے کے قابل بنایا جائے۔

اس اسائنمنٹ میں، آپ Azure OpenAI سروسز بنائیں گے جو سرچ ایپلیکیشن بنانے کے لیے استعمال کی جائیں گی۔ آپ درج ذیل Azure OpenAI سروسز بنائیں گے۔ اس اسائنمنٹ کو مکمل کرنے کے لیے آپ کو ایک Azure سبسکرپشن کی ضرورت ہوگی۔

> [ماڈل دستیابی کی جدول](https://aka.ms/oai/models?WT.mc_id=academic-105485-koreyst) کو چیک کریں۔

```shell
az group create --name semantic-video-search --location eastus
```

#### ایک Azure OpenAI سروس ریسورس بنائیں

Azure کلاؤڈ شیل سے، Azure OpenAI سروس ریسورس بنانے کے لیے درج ذیل کمانڈ چلائیں۔

```shell
az cognitiveservices account create --name semantic-video-openai --resource-group semantic-video-search \
    --location eastus --kind OpenAI --sku s0
```

#### اس ایپلیکیشن میں استعمال کے لیے اینڈپوائنٹ اور کیز حاصل کریں

Azure کلاؤڈ شیل سے، Azure OpenAI سروس ریسورس کے لیے اینڈپوائنٹ اور کیز حاصل کرنے کے لیے درج ذیل کمانڈز چلائیں۔

```shell
az cognitiveservices account show --name semantic-video-openai \
   --resource-group  semantic-video-search | jq -r .properties.endpoint
az cognitiveservices account keys list --name semantic-video-openai \
   --resource-group semantic-video-search | jq -r .key1
```

#### OpenAI ایمبیڈنگ ماڈل کو تعینات کریں

Azure کلاؤڈ شیل سے، OpenAI ایمبیڈنگ ماڈل کو تعینات کرنے کے لیے درج ذیل کمانڈ چلائیں۔

```shell
az cognitiveservices account deployment create \
    --name semantic-video-openai \
    --resource-group  semantic-video-search \
    --deployment-name text-embedding-ada-002 \
    --model-name text-embedding-ada-002 \
    --model-version "2"  \
    --model-format OpenAI \
    --sku-capacity 100 --sku-name "Standard"
```
