تلاش کرنے والی ایپلیکیشنز کے سبق میں، ہم نے مختصراً سیکھا کہ اپنے خود کے ڈیٹا کو لارج لینگویج ماڈلز (LLMs) میں کیسے شامل کیا جاتا ہے۔ اس سبق میں، ہم اپنے LLM ایپلیکیشن میں اپنے ڈیٹا کو گراؤنڈ کرنے کے تصورات، عمل کی میکانکس اور ڈیٹا ذخیرہ کرنے کے طریقوں پر مزید تفصیل سے بات کریں گے، جس میں ایمبیڈنگز اور متن دونوں شامل ہیں۔

## تعارف

اس سبق میں ہم مندرجہ ذیل موضوعات کا احاطہ کریں گے:

- RAG کا تعارف، یہ کیا ہے اور اسے AI (مصنوعی ذہانت) میں کیوں استعمال کیا جاتا ہے۔

- ویکٹر ڈیٹا بیسز کیا ہوتے ہیں اور اپنے ایپلیکیشن کے لیے ایک بنانا۔

- ایک عملی مثال کہ RAG کو کسی ایپلیکیشن میں کیسے شامل کیا جاتا ہے۔

## تعلیمی مقاصد

اس سبق کو مکمل کرنے کے بعد، آپ قابل ہوں گے:

- RAG کی ڈیٹا بازیافت اور پراسیسنگ میں اہمیت کی وضاحت کریں۔

- RAG ایپلیکیشن سیٹ اپ کریں اور اپنے ڈیٹا کو LLM سے مربوط کریں۔

- LLM ایپلیکیشنز میں مؤثر طریقے سے RAG اور ویکٹر ڈیٹا بیسز کو یکجا کریں۔

## ہمارا منظر نامہ: اپنے LLMs کو اپنے خود کے ڈیٹا سے بہتر بنانا

- `Azure OpenAI:` وہ LLM جسے ہم اپنے چیٹ بوٹ بنانے کے لیے استعمال کریں گے۔

- `AI for beginners' lesson on Neural Networks:` یہ وہ ڈیٹا ہوگا جس پر ہم اپنا LLM گراؤنڈ کریں گے۔

- `Azure AI Search` اور `Azure Cosmos DB:` ویکٹر ڈیٹا بیس جہاں ہم اپنے ڈیٹا کو ذخیرہ کریں گے اور سرچ انڈیکس بنائیں گے۔

صارفین اپنے نوٹس سے عملی کوئزز بنا سکیں گے، ریویژن فلیش کارڈز تیار کریں گے اور انہیں مختصر جائزوں میں خلاصہ کریں گے۔ شروع کرنے کے لیے، آئیے دیکھتے ہیں کہ RAG کیا ہے اور کیسے کام کرتا ہے:

## بازیافت سے افزودہ نسل (RAG)

ایک LLM پر مبنی چیٹ بوٹ صارف کے اشارے کو پراسیس کر کے جوابات تیار کرتا ہے۔ یہ انٹرایکٹو ڈیزائن کیا گیا ہے اور مختلف موضوعات پر صارفین کے ساتھ بات چیت کرتا ہے۔ تاہم، اس کے جوابات فراہم کردہ سیاق و سباق اور اس کی بنیادی تربیتی ڈیٹا تک محدود ہوتے ہیں۔ مثال کے طور پر، GPT-4 کا علم کاٹ آف ستمبر 2021 ہے، یعنی اسے اس تاریخ کے بعد ہونے والے واقعات کا علم نہیں ہے۔ اضافی طور پر، LLMs کی تربیت میں استعمال ہونے والا ڈیٹا نجی معلومات جیسے ذاتی نوٹس یا کمپنی کے مصنوعات کے دستی کو شامل نہیں کرتا۔

### RAGs (بازیافت سے افزودہ نسل) کیسے کام کرتے ہیں

![تصویر جو دکھاتی ہے کہ RAGs کیسے کام کرتے ہیں](../../../translated_images/ur/how-rag-works.f5d0ff63942bd3a6.webp)

فرض کریں کہ آپ ایک چیٹ بوٹ تعینات کرنا چاہتے ہیں جو آپ کے نوٹس سے کوئزز تیار کرتا ہے، تو آپ کو علم کی بنیاد (knowledge base) سے رابطہ درکار ہوگا۔ یہاں RAG مدد کرتا ہے۔ RAGs اس طرح کام کرتے ہیں:

- **علم کی بنیاد:** بازیافت سے پہلے، ان دستاویزات کو انگریسٹ اور پری پروسیس کرنا چاہیے، عموماً بڑے دستاویزات کو چھوٹے حصوں میں تقسیم کرکے، انہیں ٹیکسٹ ایمبیڈنگ میں تبدیل کرکے اور ڈیٹا بیس میں محفوظ کیا جاتا ہے۔

- **صارف کا سوال:** صارف سوال پوچھتا ہے۔

- **بازیافت:** جب صارف سوال پوچھتا ہے، ایمبیڈنگ ماڈل ہمارے علم کی بنیاد سے متعلقہ معلومات بازیافت کرتا ہے تاکہ سیاق و سباق فراہم کرے جو پرومپٹ میں شامل کی جائے گی۔

- **افزودہ نسل:** LLM اپنے جواب کو بازیافت شدہ ڈیٹا کی بنیاد پر بہتر بناتا ہے۔ یہ جواب کو صرف پہلے سے تربیت یافتہ ڈیٹا کی بنیاد پر نہیں بلکہ اضافی سیاق و سباق سے متعلقہ معلومات کی مدد سے جنریٹ کرنے کی اجازت دیتا ہے۔ بازیافت شدہ ڈیٹا LLM کے جوابات کو افزودہ کرنے کے لیے استعمال ہوتا ہے۔ پھر LLM صارف کے سوال کا جواب دیتا ہے۔

![تصویر جو دکھاتی ہے کہ RAGs کا فن تعمیر](../../../translated_images/ur/encoder-decode.f2658c25d0eadee2.webp)

RAGs کا فن تعمیر ٹرانسفارمرز استعمال کرتے ہوئے نافذ کیا جاتا ہے جو دو حصوں پر مشتمل ہوتا ہے: ایک اینکوڈر اور ایک ڈیکوڈر۔ مثال کے طور پر، جب صارف سوال پوچھتا ہے، ان پٹ ٹیکسٹ کو ویکٹرز میں 'انکوڈ' کیا جاتا ہے جو الفاظ کے معانی کو پکڑتے ہیں، اور ویکٹرز کو ہمارے دستاویز انڈیکس میں 'ڈیکوڈ' کیا جاتا ہے اور صارف کے سوال کی بنیاد پر نیا متن تیار کرتے ہیں۔ LLM دونوں اینکوڈر-ڈیکوڈر ماڈل کا استعمال کرتا ہے تاکہ آؤٹ پٹ تیار کرے۔

RAG کو نافذ کرتے وقت دو طریقے تحقیقی مقالے [Retrieval-Augmented Generation for Knowledge intensive NLP (natural language processing software) Tasks](https://arxiv.org/pdf/2005.11401.pdf?WT.mc_id=academic-105485-koreyst) کے مطابق ہیں:

- **_RAG-Sequence_** بازیافت شدہ دستاویزات کا استعمال کرتے ہوئے صارف کے سوال کا بہترین ممکنہ جواب پیش گوئی کرنا۔

- **RAG-Token** دستاویزات کو اگلے ٹوکن بنانے کے لیے استعمال کرنا، پھر انہیں بازیافت کرنا تاکہ صارف کے سوال کا جواب دیا جا سکے۔

### آپ RAGs کیوں استعمال کریں؟

- **معلومات کی بھرپوریت:** یقینی بناتا ہے کہ متن کے جوابات تازہ ترین اور موجودہ ہوں۔ یہ خاص طور پر مخصوص شعبوں کے کاموں پر بہترین کارکردگی فراہم کرتا ہے کیونکہ یہ اندرونی علم کی بنیاد تک رسائی دیتا ہے۔

- جعلی معلومات کو کم کرتا ہے کیونکہ یہ علم کی بنیاد میں موجود **تصدیق شدہ ڈیٹا** کو صارف کے سوالات کے سیاق و سباق کے طور پر استعمال کرتا ہے۔

- یہ **اخلاقی طور پر مؤثر** ہے کیونکہ یہ LLM کی فائن ٹیوننگ کے مقابلے میں زیادہ معیشتی ہے۔

## علم کی بنیاد بنانا

ہمارا ایپلیکیشن ہمارے ذاتی ڈیٹا یعنی AI For Beginners کے نصاب پر نیورل نیٹ ورک کے سبق پر مبنی ہے۔

### ویکٹر ڈیٹا بیسز

ویکٹر ڈیٹا بیس، روایتی ڈیٹا بیسز کے برعکس، ایک مخصوص ڈیٹا بیس ہوتا ہے جو ایمبیڈڈ ویکٹرز کو ذخیرہ، منظم اور تلاش کرنے کے لیے بنایا گیا ہے۔ یہ دستاویزات کی عددی نمائندگی ذخیرہ کرتا ہے۔ ڈیٹا کو عددی ایمبیڈنگز میں توڑنا ہمارے AI سسٹم کے لیے ڈیٹا کو سمجھنا اور پراسیس کرنا آسان بناتا ہے۔

ہم اپنے ایمبیڈنگز ویکٹر ڈیٹا بیسز میں محفوظ کرتے ہیں کیونکہ LLMs کے پاس قبول کیے جانے والے ٹوکنز کی حد ہوتی ہے۔ چونکہ آپ مکمل ایمبیڈنگز کو ایک LLM کو نہیں دے سکتے، ہم انہیں چھوٹے حصوں میں تقسیم کریں گے اور جب صارف سوال پوچھے گا، سب سے مماثل ایمبیڈنگز کو پرومپٹ کے ساتھ واپس کیا جائے گا۔ چنکنگ سے LLM میں گزرنے والے ٹوکنز کی تعداد پر بھی لاگت کم ہوتی ہے۔

کچھ مشہور ویکٹر ڈیٹا بیسز میں Azure Cosmos DB، Clarifyai، Pinecone، Chromadb، ScaNN، Qdrant اور DeepLake شامل ہیں۔ آپ Azure CLI کے ذریعے Azure Cosmos DB ماڈل درج ذیل کمانڈ سے بنا سکتے ہیں:

```bash
az login
az group create -n <resource-group-name> -l <location>
az cosmosdb create -n <cosmos-db-name> -r <resource-group-name>
az cosmosdb list-keys -n <cosmos-db-name> -g <resource-group-name>
```

### متن سے ایمبیڈنگز تک

اپنے ڈیٹا کو ذخیرہ کرنے سے پہلے، ہمیں اسے ویکٹر ایمبیڈنگز میں تبدیل کرنا ہوگا۔ اگر آپ طویل دستاویزات یا لمبے متون پر کام کر رہے ہیں، تو انہیں ممکنہ سوالات کی بنیاد پر تقسیم کر سکتے ہیں۔ تقسیم جملوں کی سطح پر یا پیراگراف کی سطح پر کی جا سکتی ہے۔ چنکنگ الفاظ کے ارد گرد معانی نکالتی ہے، آپ کسی چنک میں کچھ اضافی سیاق و سباق بھی شامل کر سکتے ہیں، مثلاً دستاویز کا عنوان یا چنک سے پہلے یا بعد کا کچھ متن۔ آپ ڈیٹا کو مندرجہ ذیل طریقے سے چنک کر سکتے ہیں:

```python
def split_text(text, max_length, min_length):
    words = text.split()
    chunks = []
    current_chunk = []

    for word in words:
        current_chunk.append(word)
        if len(' '.join(current_chunk)) < max_length and len(' '.join(current_chunk)) > min_length:
            chunks.append(' '.join(current_chunk))
            current_chunk = []

    # اگر آخری ٹکڑا کم از کم لمبائی تک نہیں پہنچا تو بھی اسے شامل کریں
    if current_chunk:
        chunks.append(' '.join(current_chunk))

    return chunks
```

چنک ہونے کے بعد، ہم اپنے متن کو مختلف ایمبیڈنگ ماڈلز استعمال کرکے ایمبیڈ کر سکتے ہیں۔ کچھ ماڈلز جو آپ استعمال کر سکتے ہیں ان میں word2vec، OpenAI کا ada-002، Azure کمپیوٹر وژن اور بہت کچھ شامل ہیں۔ ماڈل کا انتخاب زبانوں، مواد کی قسم (متن/تصاویر/آڈیو)، ان پٹ کا حجم جسے ماڈل انکوڈ کر سکتا ہے اور ایمبیڈنگ آؤٹ پٹ کی لمبائی پر منحصر ہوگا۔

## بازیافت اور ویکٹر سرچ

جب صارف سوال پوچھتا ہے، تو ریٹریور اسے query encoder کے ذریعے ویکٹر میں تبدیل کرتا ہے، پھر ہمارے دستاویز سرچ انڈیکس سے متعلقہ ویکٹرز تلاش کرتا ہے جو ان پٹ سے متعلق ہوتے ہیں۔ مکمل کرنے کے بعد، یہ ان پٹ ویکٹر اور دستاویز ویکٹرز دونوں کو متن میں تبدیل کرتا ہے اور LLM کو دیتا ہے۔

### بازیافت

بازیافت اس وقت ہوتی ہے جب سسٹم انڈیکس سے جلدی سے وہ دستاویزات تلاش کرنے کی کوشش کرتا ہے جو تلاش کے معیار پر پورا اترتی ہوں۔ ریٹریور کا مقصد دستاویزات حاصل کرنا ہے جو سیاق و سباق فراہم کریں اور آپ کے ڈیٹا پر LLM کو گراؤنڈ کریں۔

ہماری ڈیٹا بیس میں تلاش کے کئی طریقے ہیں جیسا کہ:

- **کلیدی لفظ تلاش** - متن کے تلاش کے لیے استعمال ہوتا ہے۔

- **ویکٹر تلاش** - دستاویزات کو ایمبیڈنگ ماڈلز کے ذریعے متن سے ویکٹر نمائندگی میں تبدیل کرتا ہے، جو الفاظ کے معانی کی بنیاد پر **سیمانٹک تلاش** کی اجازت دیتا ہے۔ بازیافت ان دستاویزات کو تلاش کر کے کی جاتی ہے جن کے ویکٹر نمائندے صارف کے سوال سے سب سے قریب ہوتے ہیں۔

- **ہائبرڈ** - کلیدی لفظ اور ویکٹر تلاش دونوں کا امتزاج۔

بازیافت کے ساتھ چیلنج یہ ہے کہ جب ڈیٹا بیس میں سوال کی کوئی مشابہ جواب نہ ہو، تو سسٹم بہترین معلومات واپس دے گا، تاہم آپ مناسب فاصلہ سیٹ کر سکتے ہیں یا ہائبرڈ سرچ استعمال کر سکتے ہیں جو کلیدی الفاظ اور ویکٹر تلاش دونوں کو یکجا کرتا ہے۔ اس سبق میں ہم ہائبرڈ سرچ استعمال کریں گے، جو دونوں ویکٹر اور کلیدی الفاظ کی تلاش کا امتزاج ہے۔ ہم اپنا ڈیٹا ایک DataFrame میں محفوظ کریں گے جس کے کالمز میں چنک اور ایمبیڈنگ دونوں شامل ہوں گے۔

### ویکٹر مماثلت

ریٹریور علم کی بنیاد میں ایمبیڈنگز تلاش کرے گا جو ایک دوسرے کے قریب ہوں، سب سے قریب ترین ہمسایہ، کیونکہ یہ متن میں مشابہت ظاہر کرتے ہیں۔ منظر نامے میں، جب صارف سوال کرتا ہے، اسے پہلے ایمبیڈ کیا جاتا ہے پھر مشابہ ایمبیڈنگز کے ساتھ ملایا جاتا ہے۔ عام پیمائش جو مختلف ویکٹرز کی مشابہت معلوم کرنے کے لیے استعمال ہوتی ہے وہ cosine similarity ہے جو دو ویکٹرز کے زاویہ کی بنیاد پر ہے۔

ہم مشابہت کی پیمائش کے لیے دیگر متبادل استعمال کر سکتے ہیں جیسے Euclidean distance جو ویکٹر کے ابتدائی اور اختتامی نقاط کے درمیان سیدھی لائن ہے اور dot product جو دو ویکٹرز کے متعلقہ عناصر کے حاصل ضرب کا مجموعہ ناپتا ہے۔

### سرچ انڈیکس

بازیافت کرتے وقت، ہمیں اپنی علم کی بنیاد کے لیے سرچ انڈیکس بنانا ہوگا۔ انڈیکس ہمارے ایمبیڈنگز کو ذخیرہ کرتا ہے اور بڑی ڈیٹا بیس میں بھی سب سے مشابہ چنک جلدی بازیافت کر سکتا ہے۔ ہم اپنا انڈیکس مقامی طور پر بنا سکتے ہیں:

```python
from sklearn.neighbors import NearestNeighbors

embeddings = flattened_df['embeddings'].to_list()

# تلاش انڈیکس بنائیں
nbrs = NearestNeighbors(n_neighbors=5, algorithm='ball_tree').fit(embeddings)

# انڈیکس کو سوال کرنے کے لیے، آپ kneighbors طریقہ استعمال کر سکتے ہیں
distances, indices = nbrs.kneighbors(embeddings)
```

### ری-رینکنگ

جب آپ ڈیٹا بیس سے سوال کرتے ہیں، تو آپ کو نتیجوں کو سب سے زیادہ متعلقہ سے ترتیب دینا پڑ سکتا ہے۔ ایک re-ranking LLM میشن لرننگ استعمال کرتا ہے تاکہ تلاش کے نتائج کی مطابقت کو بہتر بنایا جا سکے۔ Azure AI Search استعمال کرتے وقت، رینکنگ خود بخود سیمانٹک ری-رینکر کے ذریعے ہوتی ہے۔ قریب ترین ہمسایہ کا استعمال کرتے ہوئے ری-رینکنگ کی مثال:

```python
# سب سے ملتے جلتے دستاویزات تلاش کریں
distances, indices = nbrs.kneighbors([query_vector])

index = []
# سب سے ملتے جلتے دستاویزات پرنٹ کریں
for i in range(3):
    index = indices[0][i]
    for index in indices[0]:
        print(flattened_df['chunks'].iloc[index])
        print(flattened_df['path'].iloc[index])
        print(flattened_df['distances'].iloc[index])
    else:
        print(f"Index {index} not found in DataFrame")
```

## سب کچھ ایک ساتھ لانا

آخری قدم یہ ہے کہ اپنے LLM کو شامل کریں تاکہ جوابات ہمارے ڈیٹا پر مبنی ہوں۔ ہم اسے درج ذیل طریقے سے نافذ کر سکتے ہیں:

```python
user_input = "what is a perceptron?"

    # سب سے زیادہ مماثل دستاویزات تلاش کریں
    distances, indices = nbrs.kneighbors([query_vector])

    # سیاق و سباق فراہم کرنے کے لیے دستاویزات کو سوال میں شامل کریں

    # تاریخ اور صارف کی ان پٹ کو یکجا کریں

    # ایک پیغام آبجیکٹ بنائیں
    messages=[
        {"role": "system", "content": "You are an AI assistant that helps with AI questions."},

    ]

    # ردعمل تیار کرنے کے لیے ریسپانسز API استعمال کریں
    response = client.responses.create(

chatbot(user_input)
```
