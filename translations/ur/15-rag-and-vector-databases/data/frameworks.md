# نیورل نیٹ ورک فریم ورک

جیسا کہ ہم نے پہلے ہی سیکھا ہے، نیورل نیٹ ورکس کو مؤثر طریقے سے تربیت دینے کے لئے ہمیں دو کام کرنے کی ضرورت ہے:

* ٹینسرز پر عمل کریں، مثلاً ضرب، جمع، اور کچھ فنکشنز جیسے سگموئڈ یا سوفٹ میکس کا حساب لگائیں
* تمام اظہارات کے گریڈینٹس کا حساب لگائیں، تاکہ گریڈینٹ ڈیسنٹ آپٹیمائزیشن کو انجام دیا جا سکے

`numpy` لائبریری پہلا کام کر سکتی ہے، ہمیں گریڈینٹس کا حساب لگانے کے لئے کچھ میکانزم کی ضرورت ہے۔ اس فریم ورک میں جو ہم نے پچھلے حصے میں تیار کیا تھا، ہمیں `backward` طریقہ کار کے اندر تمام مشتق فنکشنز کو دستی طور پر پروگرام کرنا پڑا، جو بیک پروپیگیشن کرتا ہے۔ مثالی طور پر، ایک فریم ورک ہمیں *کسی بھی اظہار* کے گریڈینٹس کا حساب لگانے کا موقع دینا چاہئے جو ہم تعریف کر سکتے ہیں۔

ایک اور اہم چیز یہ ہے کہ جی پی یو یا کسی دوسرے خاص کمپیوٹ یونٹس، جیسے ٹی پی یو پر حسابات انجام دینا ممکن ہو۔ ڈیپ نیورل نیٹ ورک کی تربیت کے لئے *بہت زیادہ* حسابات کی ضرورت ہوتی ہے، اور ان حسابات کو جی پی یوز پر متوازی بنانا بہت اہم ہے۔

> ✅ 'متوازی بنانا' کا مطلب ہے کہ حسابات کو متعدد آلات پر تقسیم کیا جائے۔

فی الحال، دو سب سے زیادہ مقبول نیورل فریم ورک ہیں: ٹینسر فلو اور پائی ٹورچ۔ دونوں سی پی یو اور جی پی یو پر ٹینسرز کے ساتھ کام کرنے کے لئے ایک لو لیول API فراہم کرتے ہیں۔ لو لیول API کے اوپر، ایک اعلی سطحی API بھی ہے، جسے بالترتیب کیرس اور پائی ٹورچ لائٹننگ کہا جاتا ہے۔

لو لیول API | ٹینسر فلو | پائی ٹورچ
-------------|-------------------------------------|-------------------------------
اعلی سطحی API | کیرس | پائی ٹورچ

**لو لیول APIs** دونوں فریم ورک میں آپ کو نام نہاد **کمپیوٹیشنل گراف** بنانے کی اجازت دیتی ہیں۔ یہ گراف اس بات کی وضاحت کرتا ہے کہ دیئے گئے ان پٹ پیرامیٹرز کے ساتھ آؤٹ پٹ (عام طور پر نقصان کا فنکشن) کو کیسے حساب کیا جائے، اور اگر دستیاب ہو تو جی پی یو پر حساب کے لئے بھیجا جا سکتا ہے۔ اس کمپیوٹیشنل گراف کو مختلف کرنے اور گریڈینٹس کا حساب لگانے کے لئے فنکشنز ہیں، جو پھر ماڈل کے پیرامیٹرز کو بہتر بنانے کے لئے استعمال کیے جا سکتے ہیں۔

**اعلی سطحی APIs** نیورل نیٹ ورکس کو **لیئرز کی ترتیب** کے طور پر سمجھتے ہیں، اور زیادہ تر نیورل نیٹ ورکس کی تعمیر کو بہت آسان بناتے ہیں۔ ماڈل کی تربیت عام طور پر ڈیٹا کی تیاری اور پھر `fit` فنکشن کو کال کرنے کی ضرورت ہوتی ہے۔

اعلی سطحی API آپ کو مخصوص نیورل نیٹ ورکس کو بہت جلدی تعمیر کرنے کی اجازت دیتا ہے بغیر بہت سی تفصیلات کے بارے میں فکر کئے بغیر۔ اسی وقت، لو لیول API تربیتی عمل پر بہت زیادہ کنٹرول فراہم کرتا ہے، اور اس طرح تحقیق میں بہت زیادہ استعمال ہوتا ہے، جب آپ نئے نیورل نیٹ ورک آرکیٹیکچرز کے ساتھ کام کر رہے ہوتے ہیں۔

یہ سمجھنا بھی ضروری ہے کہ آپ دونوں APIs کو ایک ساتھ استعمال کر سکتے ہیں، مثلاً آپ اپنی نیٹ ورک لیئر آرکیٹیکچر کو لو لیول API کا استعمال کرتے ہوئے تیار کر سکتے ہیں، اور پھر اسے بڑے نیٹ ورک کے اندر استعمال کر سکتے ہیں جو اعلی سطحی API کے ساتھ بنایا اور تربیت دیا گیا ہے۔ یا آپ اعلی سطحی API کا استعمال کرتے ہوئے نیٹ ورک کو لیئرز کی ترتیب کے طور پر تعریف کر سکتے ہیں، اور پھر اپنی کم سطحی تربیتی لوپ کا استعمال کرکے آپٹیمائزیشن کر سکتے ہیں۔ دونوں APIs ایک ہی بنیادی بنیادی تصورات کا استعمال کرتے ہیں، اور وہ ایک دوسرے کے ساتھ اچھی طرح کام کرنے کے لئے ڈیزائن کیے گئے ہیں۔

## سیکھنا

اس کورس میں، ہم زیادہ تر مواد دونوں کے لئے فراہم کرتے ہیں: پائی ٹورچ اور ٹینسر فلو۔ آپ اپنی پسندیدہ فریم ورک کا انتخاب کر سکتے ہیں اور صرف متعلقہ نوٹ بکس کے ذریعے جا سکتے ہیں۔ اگر آپ کو یقین نہیں ہے کہ کون سا فریم ورک منتخب کرنا ہے، تو **پائی ٹورچ بمقابلہ ٹینسر فلو** کے بارے میں انٹرنیٹ پر کچھ مباحثے پڑھیں۔ آپ دونوں فریم ورک پر بھی نظر ڈال سکتے ہیں تاکہ بہتر سمجھ حاصل ہو۔

جہاں ممکن ہو، ہم سادگی کے لئے اعلی سطحی APIs کا استعمال کریں گے۔ تاہم، ہمارا ماننا ہے کہ نیورل نیٹ ورکس کو بنیادی سطح سے سمجھنا ضروری ہے، لہذا شروع میں ہم لو لیول API اور ٹینسرز کے ساتھ کام کرنے سے آغاز کرتے ہیں۔ تاہم، اگر آپ تیزی سے آگے بڑھنا چاہتے ہیں اور ان تفصیلات پر زیادہ وقت نہیں گزارنا چاہتے ہیں، تو آپ انہیں چھوڑ سکتے ہیں اور سیدھے اعلی سطحی API نوٹ بکس میں جا سکتے ہیں۔

## ✍️ مشقیں: فریم ورک

اپنی سیکھنے کو مندرجہ ذیل نوٹ بکس میں جاری رکھیں:

لو لیول API | ٹینسر فلو + کیرس نوٹ بک | پائی ٹورچ
-------------|-------------------------------------|-------------------------------
اعلی سطحی API | کیرس | *پائی ٹورچ لائٹننگ*

فریم ورک میں مہارت حاصل کرنے کے بعد، آئیے اوورفٹنگ کے تصور کا جائزہ لیں۔

# اوورفٹنگ

اوورفٹنگ مشین لرننگ میں ایک انتہائی اہم تصور ہے، اور اسے صحیح طریقے سے سمجھنا بہت ضروری ہے!

مندرجہ ذیل مسئلے پر غور کریں جس میں 5 نقاط (جو `x` کے ذریعے گراف میں دکھائے گئے ہیں) کی تقریباً تخمینہ لگانی ہے:

!linear | overfit
-------------------------|--------------------------
**لکیری ماڈل، 2 پیرامیٹرز** | **غیر لکیری ماڈل، 7 پیرامیٹرز**
تربیتی غلطی = 5.3 | تربیتی غلطی = 0
تصدیقی غلطی = 5.1 | تصدیقی غلطی = 20

* بائیں جانب، ہم ایک اچھی سیدھی لائن کی تقریباً تخمینہ دیکھتے ہیں۔ چونکہ پیرامیٹرز کی تعداد مناسب ہے، ماڈل نقطہ تقسیم کے پیچھے کی تصور کو درست طور پر سمجھتا ہے۔
* دائیں جانب، ماڈل بہت زیادہ طاقتور ہے۔ چونکہ ہمارے پاس صرف 5 نقاط ہیں اور ماڈل کے پاس 7 پیرامیٹرز ہیں، یہ اس طرح ایڈجسٹ کر سکتا ہے کہ تمام نقاط سے گزرتا ہے، تربیتی غلطی کو 0 بنا دیتا ہے۔ تاہم، یہ ماڈل کو ڈیٹا کے پیچھے کے صحیح نمونہ کو سمجھنے سے روکتا ہے، اس لئے تصدیقی غلطی بہت زیادہ ہے۔

ماڈل کی دولت (پیرامیٹرز کی تعداد) اور تربیتی نمونوں کی تعداد کے درمیان صحیح توازن قائم کرنا بہت اہم ہے۔

## اوورفٹنگ کیوں ہوتی ہے

  * تربیتی ڈیٹا کی کمی
  * بہت زیادہ طاقتور ماڈل
  * ان پٹ ڈیٹا میں بہت زیادہ شور

## اوورفٹنگ کا پتہ کیسے لگائیں

جیسا کہ آپ اوپر دیئے گئے گراف سے دیکھ سکتے ہیں، اوورفٹنگ کا پتہ بہت کم تربیتی غلطی اور زیادہ تصدیقی غلطی سے لگایا جا سکتا ہے۔ عام طور پر تربیت کے دوران ہم دیکھیں گے کہ دونوں تربیتی اور تصدیقی غلطیاں کم ہونا شروع ہوتی ہیں، اور پھر کسی نقطہ پر تصدیقی غلطی کم ہونا بند کر سکتی ہے اور بڑھنا شروع کر سکتی ہے۔ یہ اوورفٹنگ کی علامت ہوگی، اور یہ اشارہ کہ ہمیں شاید اس وقت تربیت کو روک دینا چاہئے (یا کم از کم ماڈل کی اسنیپ شاٹ لے لینی چاہئے)۔

## اوورفٹنگ کو کیسے روکا جائے

اگر آپ دیکھ سکتے ہیں کہ اوورفٹنگ ہو رہی ہے، تو آپ مندرجہ ذیل میں سے کچھ کر سکتے ہیں:

 * تربیتی ڈیٹا کی مقدار بڑھائیں
 * ماڈل کی پیچیدگی کو کم کریں
 * کچھ ریگولرائزیشن تکنیک استعمال کریں، جیسے ڈراپ آؤٹ، جس پر ہم بعد میں غور کریں گے۔

## اوورفٹنگ اور بائس-ویریئنس ٹریڈ آف

اوورفٹنگ دراصل شماریات میں ایک زیادہ عمومی مسئلے کا ایک معاملہ ہے جسے بائس-ویریئنس ٹریڈ آف کہا جاتا ہے۔ اگر ہم اپنے ماڈل میں ممکنہ غلطیوں کے ذرائع پر غور کریں، تو ہم دو قسم کی غلطیوں کو دیکھ سکتے ہیں:

* **بائس غلطیاں** ہمارے الگورتھم کے تربیتی ڈیٹا کے درمیان تعلق کو صحیح طور پر سمجھنے میں ناکام ہونے کی وجہ سے ہوتی ہیں۔ یہ اس حقیقت سے پیدا ہو سکتی ہیں کہ ہمارا ماڈل کافی طاقتور نہیں ہے (**انڈر فٹنگ**).
* **ویریئنس غلطیاں**، جو ان پٹ ڈیٹا میں شور کو بامعنی تعلق کے بجائے قریب کرنے کی وجہ سے ہوتی ہیں (**اوورفٹنگ**).

تربیت کے دوران، بائس غلطی کم ہوتی ہے (جب ہمارا ماڈل ڈیٹا کو قریب کرنے کے لئے سیکھتا ہے)، اور ویریئنس غلطی بڑھتی ہے۔ اوورفٹنگ کو روکنے کے لئے تربیت کو روکنا اہم ہے - یا تو دستی طور پر (جب ہم اوورفٹنگ کا پتہ لگاتے ہیں) یا خودکار طور پر (ریگولرائزیشن کو متعارف کروا کر)۔

## نتیجہ

اس سبق میں، آپ نے دو سب سے زیادہ مقبول AI فریم ورک، ٹینسر فلو اور پائی ٹورچ کے مختلف APIs کے درمیان فرق کے بارے میں سیکھا۔ اس کے علاوہ، آپ نے ایک بہت اہم موضوع، اوورفٹنگ کے بارے میں سیکھا۔
