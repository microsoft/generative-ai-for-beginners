# ابتدائی افراد کے لیے جنریٹو AI کے لیے چھوٹے زبان کے ماڈلز کا تعارف
جنریٹو AI مصنوعی ذہانت کا ایک دلکش میدان ہے جو ایسے نظام تیار کرنے پر مرکوز ہے جو نئی مواد پیدا کرنے کے قابل ہوں۔ یہ مواد متن اور تصاویر سے لے کر موسیقی اور یہاں تک کہ مکمل ورچوئل ماحول تک ہو سکتا ہے۔ جنریٹو AI کی سب سے دلچسپ ایپلیکیشنز میں سے ایک زبان کے ماڈلز کی دنیا میں ہے۔

## چھوٹے زبان کے ماڈلز کیا ہیں؟

چھوٹا زبان کا ماڈل (SLM) بڑے زبان کے ماڈل (LLM) کا ایک چھوٹا شدہ ورژن ہے، جو LLM کے کئی فن تعمیراتی اصولوں اور تکنیکوں کو استعمال کرتا ہے، جبکہ نمایاں طور پر کم کمپیوٹیشنل قدم رکھتا ہے۔

SLMs زبان کے ماڈلز کا ایک ذیلی حصہ ہیں جو انسانی مانند متن پیدا کرنے کے لیے بنائے گئے ہیں۔ اپنے بڑے ہم منصبوں جیسے GPT-4 کے مقابلے میں، SLMs زیادہ کمپیکٹ اور موثر ہوتے ہیں، جو انہیں ایسے ایپلیکیشنز کے لیے مثالی بناتے ہیں جہاں کمپیوٹیشنل وسائل محدود ہوں۔ اپنی چھوٹی جسامت کے باوجود، یہ مختلف کام انجام دے سکتے ہیں۔ عام طور پر، SLMs کو LLMs کو کمپریس یا ڈسٹی ل کر بنایا جاتا ہے، تاکہ اصل ماڈل کی کارکردگی اور لسانی صلاحیتوں کا ایک بڑا حصہ برقرار رکھا جائے۔ ماڈل کے سائز میں یہ کمی مجموعی پیچیدگی کو کم کرتی ہے، جس سے SLMs میموری اور کمپیوٹیشنل دونوں حوالوں سے زیادہ موثر بنتے ہیں۔ ان اصلاحات کے باوجود، SLMs نیچرل لینگویج پروسیسنگ (NLP) کے وسیع کام انجام دے سکتے ہیں:

- متن کی تخلیق: مربوط اور سیاق و سباق کے لحاظ سے مناسب جملے یا پیراگراف تیار کرنا۔
- متن کی تکمیل: دیے گئے پرامپٹ کی بنیاد پر جملے پیشین گوئی اور مکمل کرنا۔
- ترجمہ: ایک زبان سے دوسری زبان میں متن کا تبادلہ۔
- خلاصہ: طویل متن کو مختصر، آسان سمجھنے والی خلاصوں میں بدلنا۔

اگرچہ کارکردگی یا گہرائیِ فہم میں کچھ فرق ہو سکتا ہے ان کے بڑے ہم منصبوں کے مقابلے میں۔

## چھوٹے زبان کے ماڈلز کیسے کام کرتے ہیں؟
SLMs وسیع مقدار میں متن کے ڈیٹا پر تربیت دیے جاتے ہیں۔ تربیت کے دوران، یہ زبان کے نمونوں اور ساختوں کو سیکھتے ہیں، جو انہیں گرامر کے لحاظ سے درست اور سیاق و سباق کے مطابق متن پیدا کرنے کے قابل بناتا ہے۔ تربیتی عمل میں شامل ہیں:

- ڈیٹا کا مجموعہ: مختلف ذرائع سے بڑے ڈیٹا سیٹس جمع کرنا۔
- پری پروسیسنگ: ڈیٹا کو صاف اور منظم کرنا تاکہ تربیت کے لیے موزوں بنایا جا سکے۔
- تربیت: مشین لرننگ الگورتھمز کا استعمال کرکے ماڈل کو متن سمجھنے اور پیدا کرنے کی تعلیم دینا۔
- فائن ٹیوننگ: مخصوص کاموں پر ماڈل کی کارکردگی کو بہتر بنانا۔

SLMs کی ترقی ان ماڈلز کی بڑھتی ہوئی ضرورت کے ساتھ مطابقت رکھتی ہے جنہیں محدود وسائل والے ماحول میں تعینات کیا جا سکتا ہے، جیسے موبائل ڈیوائسز یا ایج کمپیوٹنگ پلیٹ فارمز، جہاں مکمل حجم کے LLMs ان کے بھاری وسائل کی مانگ کی وجہ سے عملی نہیں ہو سکتے۔ کارکردگی پر توجہ مرکوز کرتے ہوئے، SLMs کارکردگی اور رسائی کے درمیان توازن رکھتے ہیں، جس سے مختلف شعبوں میں وسیع تر اطلاق ممکن ہوتا ہے۔

## تعلیمی مقاصد

اس سبق میں، ہم SLM کے علم کا تعارف کرانا چاہتے ہیں اور اسے Microsoft Phi-3 کے ساتھ ملا کر متن، وژن اور MoE میں مختلف منظرناموں کو سیکھنا چاہتے ہیں۔

اس سبق کے اختتام تک، آپ درج ذیل سوالات کے جواب دے سکیں گے:

- SLM کیا ہے؟
- SLM اور LLM میں کیا فرق ہے؟
- Microsoft Phi-3/3.5 فیملی کیا ہے؟
- Microsoft Phi-3/3.5 فیملی کے ساتھ انفرنس کیسے چلائی جائے؟

تیار ہیں؟ آئیے شروع کرتے ہیں۔

## بڑے زبان کے ماڈلز (LLMs) اور چھوٹے زبان کے ماڈلز (SLMs) کے درمیان امتیازات

LLMs اور SLMs دونوں احتمال پر مبنی مشین لرننگ کے بنیادی اصولوں پر مبنی ہیں، اور ان کی فن تعمیر، تربیت کے طریقے، ڈیٹا جنریشن کے عمل اور ماڈل کے جائزہ لینے کی تکنیکوں میں مشابہت رکھتے ہیں۔ تاہم، کئی اہم عوامل ان دونوں اقسام کے ماڈلز میں فرق کرتے ہیں۔

## چھوٹے زبان کے ماڈلز کی ایپلیکیشنز

SLMs کے وسیع رینج کے استعمالات ہیں، جن میں شامل ہیں:

- چیٹ بوٹس: صارفین کی مدد فراہم کرنا اور صارفین کے ساتھ گفت و شنید کی صورت میں بات چیت کرنا۔
- مواد کی تخلیق: مصنفین کی مدد کرنا خیالات پیدا کرکے یا مکمل مضامین کا مسودہ تیار کرکے۔
- تعلیم: طلباء کی تحریری اسائنمنٹس میں مدد یا نئی زبانیں سیکھنے میں مدد کرنا۔
- رسائی: معذور افراد کے لیے ٹولز بنانا، جیسے ٹیکسٹ سے اسپیس سسٹمز۔

**سائز**
  
LLMs اور SLMs کے درمیان بنیادی فرق ماڈل کے سائز میں ہے۔ LLMs، جیسے ChatGPT (GPT-4)، تقریباً 1.76 کھرب پیرامیٹرز پر مشتمل ہو سکتے ہیں، جبکہ اوپن سورس SLMs جیسے Mistral 7B کی ڈیزائن میں نمایاں طور پر کم پیرامیٹرز ہوتے ہیں—تقریباً 7 ارب۔ اس فرق کی بنیادی وجہ ماڈل کی فن تعمیر اور تربیتی عمل میں اختلافات ہیں۔ مثال کے طور پر، ChatGPT خود توجہ کے میکانزم کو اینکوڈر-ڈیکوڈر فریم ورک میں استعمال کرتا ہے، جبکہ Mistral 7B سلائیڈنگ ونڈو اٹینشن استعمال کرتا ہے، جو ڈیکوڈر واحد ماڈل میں زیادہ موثر تربیت کو ممکن بناتا ہے۔ اس فن تعمیر کے فرق کا ماڈلز کی پیچیدگی اور کارکردگی پر گہرا اثر ہوتا ہے۔

**فہم**

SLMs عام طور پر مخصوص شعبوں میں کارکردگی کے لیے بہتر کیے جاتے ہیں، جس سے وہ اعلیٰ تخصص کے حامل ہوتے ہیں لیکن کئی علمی شعبوں میں وسیع سیاق و سباق کی تفہیم محدود ہو سکتی ہے۔ اس کے برعکس، LLMs کی کوشش ہوتی ہے کہ وہ انسانی مانند ذہانت کو وسیع پیمانے پر نقل کریں۔ وسیع اور متنوع ڈیٹا سیٹس پر تربیت حاصل کرنے کے باعث، LLMs مختلف شعبوں میں عمدہ کارکردگی دکھانے کے لیے تیار کیے جاتے ہیں، جو زیادہ وسعت اور مطابقت پذیری فراہم کرتے ہیں۔ نتیجتاً، LLMs نیچرل لینگویج پروسیسنگ اور پروگرامنگ جیسے وسیع تر نیچے کی سطح کے کاموں کے لیے زیادہ موزوں ہوتے ہیں۔

**کمپیوٹنگ**

LLMs کی تربیت اور تعیناتی وسائل طلب عمل ہوتے ہیں، جو اکثر بڑی کمپیوٹیشنل انفراسٹرکچر، بشمول بڑے پیمانے پر GPU کلسٹرز کا تقاضا کرتے ہیں۔ مثال کے طور پر، ChatGPT جیسے ماڈل کو صفر سے تربیت دینے کے لیے ہزاروں GPUs کی طویل مدت درکار ہو سکتی ہے۔ اس کے مقابلے میں، SLMs، اپنی کم پیرامیٹر تعداد کی وجہ سے، کمپیوٹیشنل وسائل کے لحاظ سے زیادہ قابل رسائی ہوتے ہیں۔ Mistral 7B جیسے ماڈلز درمیانے درجے کی GPU صلاحیتوں والی مقامی مشینوں پر تربیت اور چلائے جا سکتے ہیں، اگرچہ تربیت میں بھی کئی گھنٹے اور متعدد GPUs کی ضرورت ہوتی ہے۔

**جانبداری**

جانبداری LLMs میں ایک معروف مسئلہ ہے، جو بنیادی طور پر تربیتی ڈیٹا کی نوعیت کی وجہ سے ہوتی ہے۔ یہ ماڈلز اکثر انٹرنیٹ سے کھلے عام دستیاب خام ڈیٹا پر انحصار کرتے ہیں، جو بعض گروپوں کی کم نمائندگی یا غلط نمائندگی کر سکتا ہے، غلط لیبلنگ شامل کر سکتا ہے، یا لہجے، جغرافیائی اختلافات اور گرامر کے قواعد کی وجہ سے لسانی جانبداریوں کی عکاسی کر سکتا ہے۔ مزید برآں، LLMs کی فن تعمیر کی پیچیدگی بلا ارادہ طور پر جانبداری کو بڑھا سکتی ہے، جو محتاط فائن ٹیوننگ کے بغیر نظر انداز ہو سکتی ہے۔ اس کے برعکس، SLMs، جو محدود، مخصوص شعبہ جاتی ڈیٹا سیٹس پر تربیت پاتے ہیں، ایسے جانبداریوں کے لیے فطری طور پر کم حساس ہوتے ہیں، لیکن وہ بھی اس سے مبرا نہیں ہیں۔

**انفرنس**

SLMs کے کم شدہ سائز انہیں انفرنس کی رفتار میں اہم فائدہ دیتا ہے، جس سے وہ مقامی ہارڈویئر پر موثر انداز میں نتائج پیدا کر پاتے ہیں بغیر وسیع پیمانے پر متوازی پروسیسنگ کی ضرورت کے۔ اس کے برعکس، LLMs، اپنی جسامت اور پیچیدگی کی بنا پر، بلند متوازی کمپیوٹیشنل وسائل کے بغیر قابل قبول انفرنس وقت فراہم کرنے میں مشکل ہوتے ہیں۔ متعدد ایک وقت میں صارفین کی موجودگی LLMs کے ردعمل کے وقت کو مزید سست کر دیتی ہے، خاص طور پر بڑے پیمانے پر تعینات ہونے پر۔

خلاصہ کے طور پر، اگرچہ LLMs اور SLMs دونوں مشین لرننگ کی بنیاد پر ہیں، لیکن وہ ماڈل کے سائز، وسائل کی ضروریات، سیاق و سباق کی سمجھ، جانبداری کی حساسیت، اور انفرنس کی رفتار میں نمایاں مختلف ہوتے ہیں۔ یہ امتیازات ان کی متعلقہ موزونیت کو مختلف استعمال کے کیسز کے لیے ظاہر کرتے ہیں، جس میں LLMs زیادہ ورسٹائل لیکن وسائل بھاری ہوتے ہیں، اور SLMs مختصر کمپیوٹیشنل مطالبات کے ساتھ زیادہ شعبہ جاتی کارکردگی فراہم کرتے ہیں۔

***نوٹ: اس سبق میں، ہم SLM کو Microsoft Phi-3 / 3.5 کی مثال کے طور پر متعارف کرائیں گے۔***

## Phi-3 / Phi-3.5 فیملی کا تعارف

Phi-3 / 3.5 فیملی بنیادی طور پر متن، وژن، اور ایجنٹ (MoE) ایپلیکیشنز کے منظرناموں کو ہدف بناتی ہے:

### Phi-3 / 3.5 انسٹرکٹ

زیادہ تر متن کی تخلیق، چیٹ کی تکمیل، اور مواد کی معلومات نکالنے کے لیے۔

**Phi-3-mini**

3.8B زبان کا ماڈل Microsoft Foundry، Hugging Face، اور Ollama پر دستیاب ہے۔ Phi-3 ماڈلز نمایاں طور پر برابر اور بڑے سائز کے زبان کے ماڈلز سے اہم بینچ مارکس پر بہتر کارکردگی دکھاتے ہیں (نیچے بینچ مارک اعداد و شمار دیکھیں، اعلیٰ نمبر بہتر ہوتے ہیں)۔ Phi-3-mini اپنے دوگنے سائز کے ماڈلز کو پیچھے چھوڑ دیتا ہے، جبکہ Phi-3-small اور Phi-3-medium بڑے ماڈلز بشمول GPT-3.5 سے بہتر ہیں۔

**Phi-3-small اور medium**

صرف 7B پیرامیٹرز کے ساتھ، Phi-3-small متعدد زبان، استدلال، کوڈنگ، اور ریاضی کے بینچ مارکس میں GPT-3.5T کو شکست دیتا ہے۔

14B پیرامیٹرز کے ساتھ Phi-3-medium اس رجحان کو جاری رکھتا ہے اور Gemini 1.0 Pro کو پیچھے چھوڑ دیتا ہے۔

**Phi-3.5-mini**

اسے Phi-3-mini کے اپگریڈ کے طور پر سمجھا جا سکتا ہے۔ جبکہ پیرامیٹرز تبدیل نہیں ہوئے، اس نے متعدد زبانوں کی حمایت کو بہتر بنایا ہے (20+ زبانیں سپورٹ کرتی ہے: عربی، چینی، چیک، ڈنمارکی، ڈچ، انگریزی، فِنش، فرانسیسی، جرمن، عبرانی، ہنگری، اطالوی، جاپانی، کوریائی، نارویجن، پولش، پرتگالی، روسی، ہسپانوی، سویڈش، تھائی، ترکی، یوکرائنی) اور لمبے سیاق و سباق کے لیے مضبوط تعاون شامل کیا ہے۔

3.8B پیرامیٹرز کے ساتھ Phi-3.5-mini اپنے برابر سائز کے زبان ماڈلز سے بہتر ہے اور دوگنے سائز کے ماڈلز کے برابر ہے۔

### Phi-3 / 3.5 وژن

ہم Phi-3/3.5 کے انسٹرکٹ ماڈل کو Phi کی سمجھنے کی صلاحیت کے طور پر سوچ سکتے ہیں، اور وژن وہ ہے جو Phi کو دنیا سمجھنے کے لیے آنکھیں دیتا ہے۔


**Phi-3-ویژن**

Phi-3-ویژن، صرف 4.2B پیرامیٹرز کے ساتھ، اس رجحان کو جاری رکھتا ہے اور کلاؤڈ-3 Haiku اور Gemini 1.0 Pro V جیسے بڑے ماڈلز کو عمومی بصری استدلال کے کاموں، OCR، اور جدول اور خاکہ سمجھنے کے کاموں میں پیچھے چھوڑ دیتا ہے۔


**Phi-3.5-ویژن**

Phi-3.5-ویژن بھی Phi-3-ویژن کا اپگریڈ ہے، جو متعدد تصاویر کی حمایت شامل کرتا ہے۔ آپ اسے وژن میں ایک بہتری کے طور پر سوچ سکتے ہیں، آپ نہ صرف تصاویر دیکھ سکتے ہیں بلکہ ویڈیوز بھی۔

Phi-3.5-ویژن بڑے ماڈلز جیسے کلاؤڈ-3.5 سونٹ اور Gemini 1.5 فلیش کو OCR، جدول اور چارٹ کی سمجھ کے کاموں میں پیچھے چھوڑ دیتا ہے اور عمومی بصری علم استدلال کے کاموں میں برابر ہے۔ یہ ملٹی فریم ان پٹ کی حمایت کرتا ہے، یعنی کئی ان پٹ تصاویر پر استدلال کرتا ہے۔


### Phi-3.5-MoE

***مخلوط ماہرین (Mixture of Experts - MoE)*** ماڈلز کو کم کمپیوٹیشن سے پہلے تربیت دینے کے قابل بناتا ہے، جس کا مطلب ہے کہ آپ برابر کمپیوٹیشن بجٹ کے ساتھ ماڈل یا ڈیٹا سیٹ کے سائز کو ڈرامائی طور پر بڑھا سکتے ہیں۔ خاص طور پر، ایک MoE ماڈل کو اپنے کثیف ہم منصب کے مقابلے میں پہلے تربیت کے دوران زیادہ تیزی سے اعلیٰ معیار حاصل کرنا چاہیے۔

Phi-3.5-MoE میں 16x3.8B ماہر ماڈیولز شامل ہیں۔ صرف 6.6B متحرک پیرامیٹرز کے ساتھ Phi-3.5-MoE ایسی سطح کی استدلال، زبان کی سمجھ، اور ریاضی حاصل کرتا ہے جو بہت بڑے ماڈلز کے برابر ہے۔

ہم Phi-3/3.5 فیملی ماڈل کو مختلف منظرناموں کی بنیاد پر استعمال کر سکتے ہیں۔ LLM کے برخلاف، آپ Phi-3/3.5-mini یا Phi-3/3.5-ویژن کو ایج ڈیوائسز پر تعینات کر سکتے ہیں۔

یہاں NVIDIA NIM کی کچھ کلیدی خصوصیات ہیں:

- **تعیناتی کی آسانی:** NIM ایک کمانڈ کے ساتھ AI ماڈلز کی تعیناتی کی اجازت دیتا ہے، جسے موجودہ ورک فلو میں شامل کرنا آسان ہے۔

- **بہتر کارکردگی:** یہ NVIDIA کے پری-آپٹمائزڈ انفیرنس انجنز جیسے کہ TensorRT اور TensorRT-LLM کا استعمال کرتا ہے تاکہ کم تاخیر اور زیادہ تھروپٹ یقینی بنایا جا سکے۔
- **وسعت پذیری:** NIM Kubernetes پر آٹوسکیلنگ کو سپورٹ کرتا ہے، جو اسے مختلف ورک لوڈز کو مؤثر انداز میں سنبھالنے کے قابل بناتا ہے۔
- **سیکیورٹی اور کنٹرول:** تنظیمیں اپنے ڈیٹا اور ایپلیکیشنز پر کنٹرول برقرار رکھ سکتی ہیں کیونکہ وہ NIM مائیکروسروسز کو اپنی مینیجڈ انفراسٹرکچر پر خود ہوسٹ کر سکتی ہیں۔
- **معیاری APIs:** NIM انڈسٹری معیاری APIs فراہم کرتا ہے، جو چیٹ بوٹس، AI معاونین، اور دیگر AI ایپلیکیشنز بنانے اور انٹیگریٹ کرنے کو آسان بناتا ہے۔

**Hugging Face Transformer**
Hugging Face Transformers ایک طاقتور لائبریری ہے جو قدرتی زبان کی پروسیسنگ (NLP) اور دیگر مشین لرننگ کے کاموں کے لیے ڈیزائن کی گئی ہے۔ یہاں اس کے بارے میں چند اہم نکات ہیں:

1. **پری ٹرینڈ ماڈلز:** یہ ہزاروں پری ٹرینڈ ماڈلز فراہم کرتا ہے جو مختلف کاموں کے لیے استعمال کیے جا سکتے ہیں جیسے کہ ٹیکسٹ کلاسیفیکیشن، نامیاتی شناخت، سوال کا جواب دینا، خلاصہ سازی، ترجمہ، اور متن کی تخلیق۔

2. **فریم ورک انٹرآپریبیلٹی:** یہ لائبریری متعدد ڈیپ لرننگ فریم ورکس کو سپورٹ کرتی ہے، جن میں PyTorch، TensorFlow، اور JAX شامل ہیں۔ اس سے آپ ایک فریم ورک میں ماڈل تربیت دے کر دوسرے میں استعمال کر سکتے ہیں۔

3. **ملٹی موڈل صلاحیتیں:** NLP کے علاوہ، Hugging Face Transformers کمپیوٹر ویژن (مثلاً، امیج کلاسیفیکیشن، آبجیکٹ ڈیٹیکشن) اور آڈیو پروسیسنگ (مثلاً، سپیچ ریکگنیشن، آڈیو کلاسیفیکیشن) کے کام بھی سپورٹ کرتا ہے۔

4. **استعمال میں آسانی:** یہ لائبریری APIs اور ٹولز فراہم کرتی ہے تاکہ ماڈلز کو آسانی سے ڈاؤن لوڈ اور فائن ٹیون کیا جا سکے، جس سے یہ ابتدائی اور ماہر دونوں کے لیے قابل رسائی بن جاتی ہے۔

5. **کمیونٹی اور وسائل:** Hugging Face کی ایک متحرک کمیونٹی ہے اور وسیع دستاویزات، ٹیوٹوریلز، اور رہنما دستیاب ہیں جو صارفین کو شروع کرنے اور لائبریری سے زیادہ سے زیادہ فائدہ اٹھانے میں مدد دیتی ہیں۔
[سرکاری دستاویزات](https://huggingface.co/docs/transformers/index?WT.mc_id=academic-105485-koreyst) یا ان کے [GitHub ذخیرے](https://github.com/huggingface/transformers?WT.mc_id=academic-105485-koreyst) کو دیکھیں۔

**ONNX Runtime برائے GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) ایک کراس پلیٹ فارم انفیرنس اور ٹریننگ مشین لرننگ ایکسیلیریٹر ہے۔ ONNX Runtime for Generative AI (GENAI) ایک طاقتور ٹول ہے جو آپ کو مختلف پلیٹ فارمز پر جنریٹو AI ماڈلز کو مؤثر طریقے سے چلانے میں مدد دیتا ہے۔

## ONNX Runtime کیا ہے؟
ONNX Runtime ایک اوپن سورس پروجیکٹ ہے جو مشین لرننگ ماڈلز کی ہائی پرفارمنس انفیرنس کو ممکن بناتا ہے۔ یہ Open Neural Network Exchange (ONNX) فارمیٹ میں ماڈلز کو سپورٹ کرتا ہے، جو مشین لرننگ ماڈلز کی نمائندگی کے لیے ایک معیار ہے۔ ONNX Runtime انفیرنس تیز تر کسٹمر تجربات اور کم لاگت ممکن بناتی ہے، PyTorch اور TensorFlow/Keras جیسے ڈیپ لرننگ فریم ورکس کے ماڈلز کے علاوہ classical مشین لرننگ لائبریریز جیسے scikit-learn، LightGBM، XGBoost وغیرہ کو بھی سپورٹ کرتی ہے۔ ONNX Runtime مختلف ہارڈویئر، ڈرائیورز، اور آپریٹنگ سسٹمز کے ساتھ ہم آہنگ ہے، اور ہارڈویئر ایکسیلیریٹرز کا فائدہ اٹھاتے ہوئے گراف آپٹیمائزیشنز اور ٹرانسفارمز کے ذریعے بہترین کارکردگی فراہم کرتا ہے۔

## جنریٹو AI کیا ہے؟
جنریٹو AI ایسے AI سسٹمز کو کہتے ہیں جو نئے مواد جیسے متن، تصاویر، یا موسیقی تیار کر سکتے ہیں جو ان ڈیٹا پر مبنی ہوتے ہیں جن پر انہیں تربیت دی گئی ہو۔ مثالوں میں زبان کے ماڈلز جیسے GPT-3 اور تصویر بنانے والے ماڈلز جیسے Stable Diffusion شامل ہیں۔ ONNX Runtime for GenAI لائبریری ONNX ماڈلز کے لیے جنریٹو AI لوپ فراہم کرتی ہے، جس میں ONNX Runtime کے ساتھ انفیرنس، لاجٹس پروسیسنگ، سرچ اور سیمپلنگ، اور KV کیش مینجمنٹ شامل ہیں۔

## ONNX Runtime برائے GENAI
ONNX Runtime for GENAI، ONNX Runtime کی صلاحیتوں کو بڑھا کر جنریٹو AI ماڈلز کی حمایت کرتا ہے۔ یہاں چند اہم خصوصیات ہیں:

- **وسیع پلیٹ فارم سپورٹ:** یہ کئی پلیٹ فارمز پر کام کرتا ہے، جن میں Windows، Linux، macOS، Android، اور iOS شامل ہیں۔
- **ماڈل سپورٹ:** یہ بہت سے مقبول جنریٹو AI ماڈلز کو سپورٹ کرتا ہے، جیسے LLaMA، GPT-Neo، BLOOM، اور مزید۔
- **کارکردگی کی بہتری:** یہ NVIDIA GPUs، AMD GPUs، اور دیگر ہارڈویئر ایکسیلیریٹرز کے لیے آپٹیمائزیشنز فراہم کرتا ہے۔
- **استعمال میں آسانی:** یہ ایپلیکیشنز میں آسانی سے انٹیگریٹ کرنے کے لیے APIs فراہم کرتا ہے، جس سے آپ کم کوڈ میں متن، تصاویر، اور دیگر مواد تیار کر سکتے ہیں۔
- صارفین اس کی اعلیٰ سطحی generate() میتھڈ کال کر سکتے ہیں، یا ماڈل کی ہر انٹرکیشن کو لوپ میں چلا کر ایک بار میں ایک ٹوکن پیدا کر سکتے ہیں، اور اختیاری طور پر جنریشن پیرامیٹرز لوپ کے اندر اپڈیٹ کر سکتے ہیں۔
- ONNX Runtime میں greedy/beam سرچ، TopP، TopK سیمپلنگ کا سپورٹ بھی ہے جو ٹوکن سیکوینسز بنانے کے لیے ہے، اور بلٹ-ان لاجٹس پروسیسنگ جیسے repetition penalties بھی شامل ہیں۔ آپ آسانی سے کسٹم اسکورنگ بھی شامل کر سکتے ہیں۔

## شروع کرنے کا طریقہ
ONNX Runtime for GENAI کے ساتھ شروع کرنے کے لیے آپ یہ اقدامات کر سکتے ہیں:

### ONNX Runtime انسٹال کریں:
```Python
pip install onnxruntime
```
### جنریٹو AI ایکسٹنشنز انسٹال کریں:
```Python
pip install onnxruntime-genai
```

### ماڈل چلائیں: یہاں Python میں ایک سادہ مثال ہے:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### ڈیمو: ONNX Runtime GenAI استعمال کرتے ہوئے Phi-3.5-Vision کو کال کرنا


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
