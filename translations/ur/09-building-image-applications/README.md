# امیج جنریشن ایپلیکیشنز بنانا

متن جنریشن کے علاوہ LLMs میں اور بھی بہت کچھ ہے۔ یہ ممکن ہے کہ متن کی تفصیلات سے تصاویر بھی بنائی جا سکیں۔ تصاویر کا ہونا مختلف شعبوں میں بہت مفید ہو سکتا ہے جیسے کہ میڈٹیک، آرکیٹیکچر، سیاحت، گیم ڈیولپمنٹ وغیرہ۔ اس باب میں، ہم دو سب سے مشہور امیج جنریشن ماڈلز، DALL-E اور Midjourney کا جائزہ لیں گے۔

## امیج جنریشن ایپلیکیشن کیوں بنائیں؟

امیج جنریشن ایپلیکیشنز جنریٹیو AI کی صلاحیتوں کو دریافت کرنے کا ایک عمدہ طریقہ ہیں۔ انہیں استعمال کیا جا سکتا ہے، مثلاً:

- **امیج ایڈیٹنگ اور سنتھیسز**۔ آپ مختلف استعمال کے لیے تصاویر بنا سکتے ہیں، جیسے امیج ایڈیٹنگ اور امیج سنتھیسز۔

- **مختلف صنعتوں میں اطلاق**۔ انہیں مختلف صنعتوں کے لیے تصاویر بنانے کے لیے بھی استعمال کیا جا سکتا ہے جیسے میڈٹیک، سیاحت، گیم ڈیولپمنٹ وغیرہ۔

## DALL-E اور Midjourney کیا ہیں؟

[DALL-E](https://openai.com/dall-e-2?WT.mc_id=academic-105485-koreyst) اور [Midjourney](https://www.midjourney.com/?WT.mc_id=academic-105485-koreyst) دو سب سے مشہور امیج جنریشن ماڈلز ہیں، یہ آپ کو پرامپٹس استعمال کر کے تصاویر بنانے کی اجازت دیتے ہیں۔

### DALL-E

آئیے DALL-E سے شروع کرتے ہیں، جو کہ ایک جنریٹیو AI ماڈل ہے جو متن کی تفصیلات سے تصاویر بناتا ہے۔

> [DALL-E دو ماڈلز، CLIP اور diffused attention کا مجموعہ ہے](https://towardsdatascience.com/openais-dall-e-and-clip-101-a-brief-introduction-3a4367280d4e?WT.mc_id=academic-105485-koreyst)۔

- **CLIP**، ایک ماڈل ہے جو امیجز اور متن سے عددی نمائندگی پیدا کرتا ہے۔

- **Diffused attention**، ایک ماڈل ہے جو ایمبیڈنگ سے تصاویر بناتا ہے۔ DALL-E تصاویر اور متن کے ڈیٹا سیٹ پر تربیت یافتہ ہے اور متن کی تفصیلات سے تصاویر بنانے کے لیے استعمال کیا جا سکتا ہے۔ مثال کے طور پر، DALL-E کو ایک ٹوپی میں بلی یا موہاک کے ساتھ کتا کی تصویر بنانے کے لیے استعمال کیا جا سکتا ہے۔

### Midjourney

Midjourney بھی DALL-E کی طرح کام کرتا ہے، یہ متن پرامپٹس سے تصاویر بناتا ہے۔ Midjourney کو بھی پرامپٹس استعمال کر کے تصاویر بنانے کے لیے استعمال کیا جا سکتا ہے جیسے "ٹوپی میں بلی" یا "موہاک کے ساتھ کتا"۔

## DALL-E اور Midjourney کیسے کام کرتے ہیں

پہلے، [DALL-E](https://arxiv.org/pdf/2102.12092.pdf?WT.mc_id=academic-105485-koreyst)۔ DALL-E ایک جنریٹیو AI ماڈل ہے جو ٹرانسفارمر آرکیٹیکچر پر مبنی ہے جس میں ایک _آٹو ریگریسیو ٹرانسفارمر_ ہے۔

ایک _آٹو ریگریسیو ٹرانسفارمر_ یہ وضاحت کرتا ہے کہ ماڈل کیسے متن کی تفصیلات سے تصاویر بناتا ہے، یہ ایک وقت میں ایک پکسل بناتا ہے، اور پھر بنائے گئے پکسلز کو اگلا پکسل بنانے کے لیے استعمال کرتا ہے۔ ایک نیورل نیٹ ورک کی متعدد پرتوں سے گزرتا ہے، یہاں تک کہ تصویر مکمل ہو جاتی ہے۔

اس عمل کے ساتھ، DALL-E، اس کی بنائی گئی تصویر میں خصوصیات، اشیاء، خصوصیات، اور مزید کو کنٹرول کرتا ہے۔ تاہم، DALL-E 2 اور 3 کے پاس بنائی گئی تصویر پر زیادہ کنٹرول ہے۔

## اپنی پہلی امیج جنریشن ایپلیکیشن بنانا

تو امیج جنریشن ایپلیکیشن بنانے کے لیے کیا درکار ہے؟ آپ کو درج ذیل لائبریریاں درکار ہیں:

- **python-dotenv**، آپ کو سختی سے سفارش کی جاتی ہے کہ آپ اپنی خفیہ معلومات کو کوڈ سے دور رکھنے کے لیے اس لائبریری کا استعمال کریں۔
- **openai**، یہ لائبریری وہ ہے جو آپ OpenAI API کے ساتھ بات چیت کرنے کے لیے استعمال کریں گے۔
- **pillow**، Python میں تصاویر کے ساتھ کام کرنے کے لیے۔
- **requests**، آپ کو HTTP درخواستیں کرنے میں مدد کرنے کے لیے۔

1. ایک فائل _.env_ درج ذیل مواد کے ساتھ بنائیں:

   ```text
   AZURE_OPENAI_ENDPOINT=<your endpoint>
   AZURE_OPENAI_API_KEY=<your key>
   ```

   Azure پورٹل میں اپنے وسائل کے "Keys and Endpoint" سیکشن میں یہ معلومات تلاش کریں۔

1. اوپر دی گئی لائبریریوں کو _requirements.txt_ نامی فائل میں جمع کریں:

   ```text
   python-dotenv
   openai
   pillow
   requests
   ```

1. اگلا، ورچوئل ماحول بنائیں اور لائبریریاں انسٹال کریں:

   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```

   ونڈوز کے لیے، اپنے ورچوئل ماحول کو بنانے اور فعال کرنے کے لیے درج ذیل کمانڈز استعمال کریں:

   ```bash
   python3 -m venv venv
   venv\Scripts\activate.bat
   ```

1. _app.py_ نامی فائل میں درج ذیل کوڈ شامل کریں:

   ```python
   import openai
   import os
   import requests
   from PIL import Image
   import dotenv

   # import dotenv
   dotenv.load_dotenv()

   # Get endpoint and key from environment variables
   openai.api_base = os.environ['AZURE_OPENAI_ENDPOINT']
   openai.api_key = os.environ['AZURE_OPENAI_API_KEY']

   # Assign the API version (DALL-E is currently supported for the 2023-06-01-preview API version only)
   openai.api_version = '2023-06-01-preview'
   openai.api_type = 'azure'


   try:
       # Create an image by using the image generation API
       generation_response = openai.Image.create(
           prompt='Bunny on horse, holding a lollipop, on a foggy meadow where it grows daffodils',    # Enter your prompt text here
           size='1024x1024',
           n=2,
           temperature=0,
       )
       # Set the directory for the stored image
       image_dir = os.path.join(os.curdir, 'images')

       # If the directory doesn't exist, create it
       if not os.path.isdir(image_dir):
           os.mkdir(image_dir)

       # Initialize the image path (note the filetype should be png)
       image_path = os.path.join(image_dir, 'generated-image.png')

       # Retrieve the generated image
       image_url = generation_response["data"][0]["url"]  # extract image URL from response
       generated_image = requests.get(image_url).content  # download the image
       with open(image_path, "wb") as image_file:
           image_file.write(generated_image)

       # Display the image in the default image viewer
       image = Image.open(image_path)
       image.show()

   # catch exceptions
   except openai.InvalidRequestError as err:
       print(err)

   ```

آئیے اس کوڈ کی وضاحت کریں:

- سب سے پہلے، ہم ان لائبریریوں کو امپورٹ کرتے ہیں جن کی ہمیں ضرورت ہے، بشمول OpenAI لائبریری، dotenv لائبریری، requests لائبریری، اور Pillow لائبریری۔

  ```python
  import openai
  import os
  import requests
  from PIL import Image
  import dotenv
  ```

- اگلا، ہم _.env_ فائل سے ماحول کے متغیرات لوڈ کرتے ہیں۔

  ```python
  # import dotenv
  dotenv.load_dotenv()
  ```

- اس کے بعد، ہم OpenAI API کے لیے اینڈ پوائنٹ، کلید، ورژن اور قسم مقرر کرتے ہیں۔

  ```python
  # Get endpoint and key from environment variables
  openai.api_base = os.environ['AZURE_OPENAI_ENDPOINT']
  openai.api_key = os.environ['AZURE_OPENAI_API_KEY']

  # add version and type, Azure specific
  openai.api_version = '2023-06-01-preview'
  openai.api_type = 'azure'
  ```

- پھر، ہم تصویر بناتے ہیں:

  ```python
  # Create an image by using the image generation API
  generation_response = openai.Image.create(
      prompt='Bunny on horse, holding a lollipop, on a foggy meadow where it grows daffodils',    # Enter your prompt text here
      size='1024x1024',
      n=2,
      temperature=0,
  )
  ```

  اوپر دیا گیا کوڈ ایک JSON آبجیکٹ کے ساتھ جواب دیتا ہے جس میں بنائی گئی تصویر کا URL ہوتا ہے۔ ہم URL کا استعمال کر کے تصویر کو ڈاؤن لوڈ کر سکتے ہیں اور اسے فائل میں محفوظ کر سکتے ہیں۔

- آخر میں، ہم تصویر کو کھولتے ہیں اور اسے معیاری امیج ویور کا استعمال کر کے دکھاتے ہیں:

  ```python
  image = Image.open(image_path)
  image.show()
  ```

### تصویر بنانے کی مزید تفصیلات

آئیے اس کوڈ کو دیکھیں جو تصویر بناتا ہے مزید تفصیل سے:

```python
generation_response = openai.Image.create(
        prompt='Bunny on horse, holding a lollipop, on a foggy meadow where it grows daffodils',    # Enter your prompt text here
        size='1024x1024',
        n=2,
        temperature=0,
    )
```

- **prompt**، وہ متن پرامپٹ ہے جو تصویر بنانے کے لیے استعمال ہوتا ہے۔ اس معاملے میں، ہم "خرگوش گھوڑے پر، لالی پاپ پکڑے ہوئے، دھندلی میدان میں جہاں ڈافوڈلز اگتے ہیں" پرامپٹ استعمال کر رہے ہیں۔
- **size**، بنائی گئی تصویر کا سائز ہے۔ اس معاملے میں، ہم 1024x1024 پکسلز کی تصویر بنا رہے ہیں۔
- **n**، بنائی گئی تصاویر کی تعداد ہے۔ اس معاملے میں، ہم دو تصاویر بنا رہے ہیں۔
- **temperature**، ایک پیرامیٹر ہے جو جنریٹیو AI ماڈل کے آؤٹ پٹ کی بے ترتیبی کو کنٹرول کرتا ہے۔ درجہ حرارت 0 اور 1 کے درمیان ایک قدر ہے جہاں 0 کا مطلب ہے کہ آؤٹ پٹ متعین ہے اور 1 کا مطلب ہے کہ آؤٹ پٹ بے ترتیب ہے۔ ڈیفالٹ قدر 0.7 ہے۔

امیجز کے ساتھ کرنے کے لیے اور بھی چیزیں ہیں جنہیں ہم اگلے سیکشن میں شامل کریں گے۔

## امیج جنریشن کی اضافی صلاحیتیں

آپ نے دیکھا ہے کہ ہم نے Python میں چند لائنوں کا استعمال کر کے ایک تصویر بنانے کے قابل ہو گئے۔ تاہم، امیجز کے ساتھ کرنے کے لیے اور بھی چیزیں ہیں۔

آپ یہ بھی کر سکتے ہیں:

- **ترمیمات انجام دیں**۔ ایک موجودہ تصویر، ماسک اور پرامپٹ فراہم کر کے، آپ تصویر میں تبدیلی کر سکتے ہیں۔ مثال کے طور پر، آپ تصویر کے ایک حصے میں کچھ شامل کر سکتے ہیں۔ ہماری خرگوش کی تصویر کا تصور کریں، آپ خرگوش کو ایک ٹوپی پہنا سکتے ہیں۔ آپ یہ کیسے کریں گے کہ تصویر، ماسک (تبدیلی کے لیے علاقے کے حصے کی نشاندہی کرنا) اور ایک متن پرامپٹ فراہم کر کے جو کہا جائے کہ کیا کیا جانا چاہیے۔

  ```python
  response = openai.Image.create_edit(
    image=open("base_image.png", "rb"),
    mask=open("mask.png", "rb"),
    prompt="An image of a rabbit with a hat on its head.",
    n=1,
    size="1024x1024"
  )
  image_url = response['data'][0]['url']
  ```

  بنیادی تصویر میں صرف خرگوش ہوتا لیکن حتمی تصویر میں خرگوش پر ٹوپی ہوتی۔

- **تغیرات بنائیں**۔ خیال یہ ہے کہ آپ ایک موجودہ تصویر لیتے ہیں اور پوچھتے ہیں کہ تغیرات بنائے جائیں۔ ایک تغیر بنانے کے لیے، آپ ایک تصویر اور ایک متن پرامپٹ فراہم کرتے ہیں اور کوڈ کچھ اس طرح ہے:

  ```python
  response = openai.Image.create_variation(
    image=open("bunny-lollipop.png", "rb"),
    n=1,
    size="1024x1024"
  )
  image_url = response['data'][0]['url']
  ```

  > نوٹ کریں، یہ صرف OpenAI پر معاونت یافتہ ہے

## اپنے ایپلیکیشن کے لیے میٹا پرامپٹس کے ساتھ حدود کیسے مقرر کریں

ہمارے ڈیمو کے ساتھ، ہم پہلے ہی اپنے کلائنٹس کے لیے تصاویر بنا سکتے ہیں۔ تاہم، ہمیں اپنی ایپلیکیشن کے لیے کچھ حدود بنانی ہوں گی۔

مثال کے طور پر، ہم ایسی تصاویر نہیں بنانا چاہتے جو کام کے لیے محفوظ نہ ہوں، یا جو بچوں کے لیے مناسب نہ ہوں۔

ہم یہ _میٹا پرامپٹس_ کے ساتھ کر سکتے ہیں۔ میٹا پرامپٹس وہ متن پرامپٹس ہیں جو جنریٹیو AI ماڈل کے آؤٹ پٹ کو کنٹرول کرنے کے لیے استعمال ہوتے ہیں۔ مثال کے طور پر، ہم میٹا پرامپٹس کا استعمال آؤٹ پٹ کو کنٹرول کرنے کے لیے کر سکتے ہیں، اور اس بات کو یقینی بناتے ہیں کہ بنائی گئی تصاویر کام کے لیے محفوظ ہوں، یا بچوں کے لیے مناسب ہوں۔

یہاں ایک ممکنہ حل ہے:

```python
import openai
import os
import requests
from PIL import Image
import dotenv

# import dotenv
dotenv.load_dotenv()

# Get endpoint and key from environment variables
openai.api_base = "<replace with endpoint>"
openai.api_key = "<replace with api key>"

# Assign the API version (DALL-E is currently supported for the 2023-06-01-preview API version only)
openai.api_version = '2023-06-01-preview'
openai.api_type = 'azure'

disallow_list = "swords, violence, blood, gore, nudity, sexual content, adult content, adult themes, adult language, adult humor, adult jokes, adult situations, adult"

meta_prompt = f"""You are an assistant designer that creates images for children.

The image needs to be safe for work and appropriate for children.

The image needs to be in color.

The image needs to be in landscape orientation.

The image needs to be in a 16:9 aspect ratio.

Do not consider any input from the following that is not safe for work or appropriate for children.
{disallow_list}"""

prompt = f"""{metaprompt}
Generate monument of the Arc of Triumph in Paris, France, in the evening light with a small child holding a Teddy looks on.
""""

try:
    # Create an image by using the image generation API
    generation_response = openai.Image.create(
        prompt=prompt,    # Enter your prompt text here
        size='1024x1024',
        n=2,
        temperature=0,
    )
    # Set the directory for the stored image
    image_dir = os.path.join(os.curdir, 'images')

    # If the directory doesn't exist, create it
    if not os.path.isdir(image_dir):
        os.mkdir(image_dir)

    # Initialize the image path (note the filetype should be png)
    image_path = os.path.join(image_dir, 'generated-image.png')

    # Retrieve the generated image
    image_url = generation_response["data"][0]["url"]  # extract image URL from response
    generated_image = requests.get(image_url).content  # download the image
    with open(image_path, "wb") as image_file:
        image_file.write(generated_image)

    # Display the image in the default image viewer
    image = Image.open(image_path)
    image.show()

# catch exceptions
except openai.InvalidRequestError as err:
    print(err)
```
