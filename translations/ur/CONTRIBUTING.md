# شراکت

جب آپ ایک پل ریکویسٹ جمع کراتے ہیں، تو ایک CLA بوٹ خود بخود یہ طے کرے گا کہ آیا آپ کو CLA فراہم کرنے کی ضرورت ہے اور PR کو مناسب طریقے سے سجائے گا (مثال کے طور پر، لیبل، تبصرہ)۔ بس بوٹ کی فراہم کردہ ہدایات پر عمل کریں۔ آپ کو یہ صرف ایک بار تمام ریپوزٹریز کے لئے کرنا ہوگا جو ہمارے CLA کا استعمال کرتے ہیں۔

## ٹائپوز، مسائل، بگز اور شراکتیں

جب بھی آپ "Generative AI for Beginners" ریپوزٹری میں کوئی تبدیلی جمع کرا رہے ہوں، براہ کرم ان سفارشات پر عمل کریں۔

* ہمیشہ اپنی ترامیم کرنے سے پہلے ریپوزٹری کو اپنے اکاؤنٹ میں فورک کریں
* ایک پل ریکویسٹ میں متعدد تبدیلیوں کو جمع نہ کریں۔ مثال کے طور پر، کسی بگ فکس اور دستاویزات کی تازہ کاریوں کو الگ PRs کے ذریعے جمع کریں
* اگر آپ کی پل ریکویسٹ میں مرج تنازعات دکھائی دیتے ہیں، تو یقینی بنائیں کہ آپ کا مقامی مین مین ریپوزٹری کے مین کا آئینہ ہے اس سے پہلے کہ آپ اپنی ترامیم کریں
* اگر آپ ترجمہ جمع کرا رہے ہیں، تو براہ کرم تمام ترجمہ شدہ فائلوں کے لئے ایک PR بنائیں کیونکہ ہم مواد کے لئے جزوی ترجمے قبول نہیں کرتے ہیں
* اگر آپ ٹائپو یا دستاویزات کی اصلاح جمع کرا رہے ہیں، تو آپ ایک PR میں مناسب جگہ پر ترامیم کو ملا سکتے ہیں

- یقینی بنائیں کہ کوئی بھی رشتہ دار لنک (یعنی ریپوزٹری میں دیگر فائلوں اور فولڈرز کے لنکس) ایک `./` سے شروع ہوتا ہے جو کہ موجودہ ورکنگ ڈائریکٹری میں واقع فائل یا فولڈر کی طرف اشارہ کرتا ہے یا ایک `../` جو کہ پیرنٹ ورکنگ ڈائریکٹری میں واقع فائل یا فولڈر کی طرف اشارہ کرتا ہے۔
- یقینی بنائیں کہ کوئی بھی رشتہ دار لنک (یعنی ریپوزٹری میں دیگر فائلوں اور فولڈرز کے لنکس) کے آخر میں ایک ٹریکنگ ID ہے (یعنی `?` یا `&` پھر `wt.mc_id=` یا `WT.mc_id=`)۔
- یقینی بنائیں کہ مندرجہ ذیل ڈومینز سے کوئی بھی URL _github.com, microsoft.com, visualstudio.com, aka.ms, اور azure.com_ کے آخر میں ایک ٹریکنگ ID ہے (یعنی `?` یا `&` پھر `wt.mc_id=` یا `WT.mc_id=`)۔
- یقینی بنائیں کہ آپ کے لنکس میں ملک کی مخصوص لوکیل نہیں ہے (یعنی `/en-us/` یا `/en/`)۔
- یقینی بنائیں کہ تمام تصاویر `./images` فولڈر میں محفوظ ہیں۔
- یقینی بنائیں کہ تصاویر کے نام انگریزی حروف، نمبروں، اور آپ کے تصویر کے نام میں ڈیشز کے ساتھ وضاحتی نام رکھتے ہیں۔

### چیک کریں ٹوٹے ہوئے رشتہ دار راستے

یہ ورک فلو یقینی بناتا ہے کہ آپ کی فائلوں میں کوئی بھی رشتہ دار راستہ کام کر رہا ہے۔ یہ ریپوزٹری GitHub صفحات پر تعینات ہے لہذا جب آپ لنکس ٹائپ کرتے ہیں جو سب کو جوڑتا ہے تو بہت محتاط رہنے کی ضرورت ہے تاکہ کسی کو غلط جگہ پر نہ لے جائے۔

یقینی بنانے کے لئے کہ آپ کے لنکس صحیح طریقے سے کام کر رہے ہیں، بس VS کوڈ کا استعمال کریں تاکہ اس کی جانچ کریں۔

مثال کے طور پر، جب آپ اپنی فائلوں میں کسی لنک پر ہور کرتے ہیں تو آپ کو **ctrl + click** پر کلک کر کے لنک کو فالو کرنے کی ہدایت دی جائے گی۔

اگر آپ کسی لنک پر کلک کرتے ہیں اور یہ مقامی طور پر کام نہیں کر رہا ہے تو، یقیناً یہ ورک فلو کو متحرک کرے گا اور GitHub پر کام نہیں کرے گا۔

اس مسئلے کو حل کرنے کے لئے، کوشش کریں کہ لنک کو VS کوڈ کی مدد سے ٹائپ کریں۔

جب آپ `./` یا `../` ٹائپ کرتے ہیں تو VS کوڈ آپ کو دستیاب اختیارات میں سے انتخاب کرنے کی ترغیب دے گا جو آپ نے ٹائپ کیا ہے۔

مطلوبہ فائل یا فولڈر پر کلک کر کے راستہ فالو کریں اور آپ کو یقین ہو جائے گا کہ آپ کا راستہ ٹوٹا ہوا نہیں ہے۔

ایک بار جب آپ صحیح رشتہ دار راستہ شامل کر لیں، تو اپنی تبدیلیاں محفوظ کریں اور پش کریں، ورک فلو آپ کی تبدیلیوں کی تصدیق کے لئے دوبارہ متحرک ہو جائے گا۔ اگر آپ چیک پاس کر لیتے ہیں تو آپ کو جانے کے لئے تیار ہیں۔
