# Фреймворки нейронних мереж

Як ми вже дізналися, для ефективного навчання нейронних мереж потрібно зробити дві речі:

* Працювати з тензорами, наприклад, множити, додавати та обчислювати деякі функції, такі як сигмоїда або softmax
* Обчислювати градієнти всіх виразів, щоб виконати оптимізацію методом градієнтного спуску

Хоча бібліотека `numpy` може виконувати першу частину, нам потрібен механізм для обчислення градієнтів. У нашій системі, яку ми розробили в попередньому розділі, нам доводилося вручну програмувати всі похідні функції в методі `backward`, який виконує зворотне поширення. Ідеально, якщо фреймворк надасть нам можливість обчислювати градієнти *будь-якого виразу*, який ми можемо визначити.

Інша важлива річ — це можливість виконувати обчислення на GPU або будь-яких інших спеціалізованих обчислювальних блоках, таких як TPU. Навчання глибоких нейронних мереж вимагає *багато* обчислень, і можливість паралелізації цих обчислень на GPU є дуже важливою.

> ✅ Терміни 'паралелізація' означають розподіл обчислень на декілька пристроїв.

На даний момент двома найпопулярнішими фреймворками нейронних мереж є: TensorFlow та PyTorch. Обидва надають низькорівневий API для роботи з тензорами як на CPU, так і на GPU. Крім низькорівневого API, існує також високорівневий API, відповідно, Keras та PyTorch Lightning.

Низькорівневий API | TensorFlow| PyTorch
------------------|-------------------------------------|--------------------------------
Високорівневий API| Keras| Pytorch

**Низькорівневі API** в обох фреймворках дозволяють будувати так звані **обчислювальні графи**. Цей граф визначає, як обчислювати вихід (зазвичай функцію втрат) з даними вхідними параметрами і може бути переданий для обчислення на GPU, якщо він доступний. Існують функції для диференціювання цього обчислювального графа та обчислення градієнтів, які потім можуть бути використані для оптимізації параметрів моделі.

**Високорівневі API** вважають нейронні мережі **послідовністю шарів** і значно спрощують створення більшості нейронних мереж. Навчання моделі зазвичай вимагає підготовки даних, а потім виклику функції `fit` для виконання роботи.

Високорівневий API дозволяє дуже швидко створювати типові нейронні мережі без занепокоєння про безліч деталей. Водночас низькорівневий API надає набагато більше контролю над процесом навчання, і тому їх часто використовують у дослідженнях, коли мають справу з новими архітектурами нейронних мереж.

Також важливо розуміти, що ви можете використовувати обидва API разом, наприклад, ви можете розробити власну архітектуру шару мережі за допомогою низькорівневого API, а потім використовувати її всередині більшої мережі, створеної та навченої за допомогою високорівневого API. Або ви можете визначити мережу за допомогою високорівневого API як послідовність шарів, а потім використовувати власний низькорівневий цикл навчання для виконання оптимізації. Обидва API використовують однакові базові концепції та розроблені для того, щоб добре працювати разом.

## Навчання

У цьому курсі ми пропонуємо більшість матеріалів як для PyTorch, так і для TensorFlow. Ви можете вибрати свій улюблений фреймворк і переглянути відповідні блокноти. Якщо ви не впевнені, який фреймворк вибрати, прочитайте деякі обговорення в інтернеті щодо **PyTorch vs. TensorFlow**. Ви також можете ознайомитися з обома фреймворками, щоб краще зрозуміти.

Де можливо, ми будемо використовувати високорівневі API для простоти. Однак ми вважаємо, що важливо зрозуміти, як працюють нейронні мережі з самого початку, тому на початку ми починаємо працювати з низькорівневим API і тензорами. Однак, якщо ви хочете швидко почати і не хочете витрачати багато часу на вивчення цих деталей, ви можете пропустити їх і перейти прямо до блокнотів з високорівневим API.

# Перенавчання

Перенавчання — це надзвичайно важливе поняття в машинному навчанні, і дуже важливо зрозуміти його правильно!

Розгляньмо наступну проблему апроксимації 5 точок (представлених `x` на графіках нижче):

!лінійний | перенавчання
-------------------------|--------------------------
**Лінійна модель, 2 параметри** | **Нелінійна модель, 7 параметрів**
Помилка навчання = 5.3 | Помилка навчання = 0
Помилка валідації = 5.1 | Помилка валідації = 20

* Зліва ми бачимо хорошу пряму лінію апроксимації. Оскільки кількість параметрів адекватна, модель правильно розуміє ідею розподілу точок.
* Справа модель занадто потужна. Оскільки у нас є лише 5 точок, а модель має 7 параметрів, вона може налаштуватися так, щоб пройти через усі точки, зводячи помилку навчання до 0. Однак це заважає моделі зрозуміти правильний шаблон даних, тому помилка валідації дуже висока.

Дуже важливо досягти правильного балансу між багатством моделі (кількість параметрів) і кількістю навчальних зразків.

## Чому виникає перенавчання

  * Недостатньо навчальних даних
  * Занадто потужна модель
  * Занадто багато шуму у вхідних даних

## Як виявити перенавчання

Як видно з графіка вище, перенавчання можна виявити за дуже низькою помилкою навчання та високою помилкою валідації. Зазвичай під час навчання ми бачимо, як помилки навчання та валідації починають зменшуватися, а потім у якийсь момент помилка валідації може перестати зменшуватися і почати зростати. Це буде ознакою перенавчання та вказівкою на те, що, мабуть, слід припинити навчання в цей момент (або принаймні зробити знімок моделі).

перенавчання

## Як запобігти перенавчанню

Якщо ви бачите, що відбувається перенавчання, ви можете зробити одне з наступного:

 * Збільшити кількість навчальних даних
 * Зменшити складність моделі
 * Використовувати якусь техніку регуляризації, таку як Dropout, яку ми розглянемо пізніше.

## Перенавчання та компроміс упередженості-варіації

Перенавчання насправді є випадком більш загальної проблеми в статистиці, яка називається компромісом упередженості-варіації. Якщо ми розглянемо можливі джерела помилки в нашій моделі, ми можемо побачити два типи помилок:

* **Помилки упередженості** викликані тим, що наш алгоритм не може правильно вловити взаємозв’язок між навчальними даними. Це може бути викликано тим, що наша модель недостатньо потужна (**недонавчання**).
* **Помилки варіації**, які викликані тим, що модель апроксимує шум у вхідних даних замість значущого зв’язку (**перенавчання**).

Під час навчання помилка упередженості зменшується (оскільки наша модель вчиться апроксимувати дані), а помилка варіації збільшується. Важливо припинити навчання – або вручну (коли ми виявляємо перенавчання), або автоматично (вводячи регуляризацію) – щоб запобігти перенавчанню.
