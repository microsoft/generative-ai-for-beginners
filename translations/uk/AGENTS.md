# AGENTS.md

## Огляд проекту

Цей репозиторій містить комплексну навчальну програму з 21 уроку, що навчає основам Генеративного ШІ та розробці застосунків. Курс призначений для початківців і охоплює все від базових понять до створення готових до виробництва застосунків.

**Ключові технології:**
- Python 3.9+ з бібліотеками: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript з Node.js та бібліотеками: `openai` (Azure OpenAI через v1 endpoint + Responses API), `@azure-rest/ai-inference` (Microsoft Foundry Models)
- Azure OpenAI Service, OpenAI API та Microsoft Foundry Models (GitHub Models буде припинено наприкінці липня 2026 року)
- Jupyter Notebooks для інтерактивного навчання
- Dev Containers для послідовного середовища розробки

**Структура репозиторію:**
- 21 каталог уроків з номерами (00-21), що містять README, приклади коду та завдання
- Кілька реалізацій: Python, TypeScript, та інколи приклади .NET
- Каталог перекладів із понад 40 мовними версіями
- Централізована конфігурація через файл `.env` (використовуйте `.env.copy` як шаблон)

## Команди налаштування

### Початкове налаштування репозиторію

```bash
# Клонуйте репозиторій
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# Скопіюйте шаблон середовища
cp .env.copy .env
# Відредагуйте .env з вашими API ключами та кінцевими точками
```

### Налаштування середовища Python

```bash
# Створити віртуальне середовище
python3 -m venv venv

# Активувати віртуальне середовище
# На macOS/Linux:
source venv/bin/activate
# На Windows:
venv\Scripts\activate

# Встановити залежності
pip install -r requirements.txt
```

### Налаштування Node.js/TypeScript

```bash
# Встановіть залежності на рівні root (для інструментів документації)
npm install

# Для окремих прикладів TypeScript з уроку перейдіть до конкретного уроку:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Налаштування Dev Container (рекомендується)

Репозиторій містить конфігурацію `.devcontainer` для GitHub Codespaces або VS Code Dev Containers:

1. Відкрийте репозиторій у GitHub Codespaces або VS Code з розширенням Dev Containers
2. Dev Container автоматично:
   - Встановить залежності Python з `requirements.txt`
   - Запустить постскрипт створення (`.devcontainer/post-create.sh`)
   - Налаштує ядро Jupyter

## Робочий процес розробки

### Змінні середовища

Усі уроки, що потребують доступу до API, використовують змінні середовища, визначені в `.env`:

- `OPENAI_API_KEY` - для OpenAI API
- `AZURE_OPENAI_API_KEY` - для Azure OpenAI у Microsoft Foundry (Azure OpenAI Service тепер частина Microsoft Foundry: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - URL кінцевої точки Azure OpenAI (кінцева точка ресурсу Foundry)
- `AZURE_OPENAI_DEPLOYMENT` - назва розгортання моделі для чат-завершення
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - назва розгортання моделі для embeddings
- `AZURE_OPENAI_API_VERSION` - версія API (за замовчуванням: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - для моделей Hugging Face
- `AZURE_INFERENCE_ENDPOINT` - кінцева точка Microsoft Foundry Models (каталог моделей з кількома провайдерами)
- `AZURE_INFERENCE_CREDENTIAL` - API ключ для Microsoft Foundry Models (замінює що припиняється `GITHUB_TOKEN`)

### Запуск прикладів Python

```bash
# Перейдіть до каталогу уроку

```

### Робота з різними типами уроків

- **Уроки "Learn"**: Основна увага на документацію README.md та концепції
- **Уроки "Build"**: Містять робочі приклади коду на Python та TypeScript
- Кожен урок має README.md із теорією, розбором коду та посиланнями на відеоконтент

## Керівництво зі стилю коду

### Python

- Використовуйте `python-dotenv` для керування змінними середовища
- Імпортуйте бібліотеку `openai` для роботи з API
- Використовуйте `pylint` для linting (деякі приклади мають `# pylint: disable=all` для спрощення)
- Дотримуйтесь конвенцій найменувань PEP 8
- Зберігайте ключі API у файлі `.env`, ніколи не в коді

### TypeScript

- Використовуйте пакет `dotenv` для змінних середовища
- Конфігурація TypeScript у файлі `tsconfig.json` для кожного застосунку
- Використовуйте пакет `openai` для Azure OpenAI (керуйте клієнтом через endpoint `/openai/v1/` і викликайте `client.responses.create`); використовуйте `@azure-rest/ai-inference` для Microsoft Foundry Models
- Використовуйте `nodemon` для розробки з автоперезапуском
- Спочатку збирайте: `npm run build`, потім запускайте: `npm start`

### Загальні конвенції

- Зберігайте приклади коду простими та навчальними
- Додавайте коментарі, що пояснюють ключові концепції
- Код кожного уроку повинен бути автономним і запускатись
- Використовуйте узгоджене найменування: префікс `aoai-` для Azure OpenAI, `oai-` для OpenAI API, `githubmodels-` для Microsoft Foundry Models (старий префікс з епохи GitHub Models збережений)

## Керівництво з документації

### Стиль Markdown

- Усі URL мають бути оформлені у вигляді `[text](../../url)` без зайвих пробілів
- Відносні посилання мають починатися з `./` або `../`
- Усі посилання на домени Microsoft повинні містити трекінговий ID: `?WT.mc_id=academic-105485-koreyst`
- Уникайте локалей, специфічних для країн, у URL (наприклад, `/en-us/`)
- Зображення зберігайте у папці `./images` з описовими іменами
- Використовуйте англійські символи, цифри та дефіси у назвах файлів

### Підтримка перекладів

- Репозиторій підтримує понад 40 мов за допомогою автоматизованих GitHub Actions
- Переклади зберігаються в каталозі `translations/`
- Не надсилайте часткові переклади
- Машинні переклади не приймаються
- Перекладені зображення зберігаються в каталозі `translated_images/`

## Тестування та валідація

### Перевірки перед відправленням

Цей репозиторій використовує GitHub Actions для валідації. Перед відправкою PR:

1. **Перевірка посилань Markdown**:
   ```bash
   # Робочий процес validate-markdown.yml перевіряє:
   # - Неправильні відносні шляхи
   # - Відсутні ідентифікатори відстеження на шляхах
   # - Відсутні ідентифікатори відстеження на URL-адресах
   # - URL-адреси з локалізацією країни
   # - Неправильні зовнішні URL-адреси
   ```

2. **Ручне тестування**:
   - Тестуйте приклади Python: Активуйте venv і запускайте скрипти
   - Тестуйте приклади TypeScript: `npm install`, `npm run build`, `npm start`
   - Перевірте правильність налаштувань змінних середовища
   - Переконайтеся, що ключі API працюють з прикладами коду

3. **Приклади коду**:
   - Переконайтеся, що весь код виконується без помилок
   - Тестуйте як Azure OpenAI, так і OpenAI API, якщо застосовно
   - Перевірте роботу з Microsoft Foundry Models, де підтримується

### Відсутність автоматизованих тестів

Це освітній репозиторій, орієнтований на навчальні матеріали та приклади. Немає юніт-тестів або інтеграційних тестів для запуску. Валідація в основному:
- Ручне тестування прикладів коду
- GitHub Actions для валідації Markdown
- Огляд освітнього контенту спільнотою

## Вирішення проблем

### Поширені проблеми

**Помилки імпорту Python**:
- Переконайтеся, що активовано віртуальне середовище
- Запустіть `pip install -r requirements.txt`
- Перевірте, що версія Python 3.9 або новіша

**Помилки збірки TypeScript**:
- Запустіть `npm install` у відповідному каталозі застосунку
- Перевірте сумісність версії Node.js
- Очистіть `node_modules` і перевстановіть за потреби

**Помилки автентифікації API**:
- Перевірте наявність файлу `.env` з правильними значеннями
- Переконайтеся, що ключі API дійсні і не прострочені
- Перевірте, що URL кінцевих точок правильні для вашого регіону

**Відсутні змінні середовища**:
- Скопіюйте `.env.copy` у `.env`
- Заповніть всі необхідні значення для поточного уроку
- Перезапустіть застосунок після оновлення `.env`
