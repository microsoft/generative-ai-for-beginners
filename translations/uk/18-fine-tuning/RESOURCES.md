# Ресурси для самостійного навчання

Урок був створений з використанням основних ресурсів OpenAI та Microsoft Foundry як довідкових матеріалів для термінології та навчальних посібників. Ось неповний список для ваших власних самостійних навчальних подорожей. Кожне посилання нижче веде до актуального, підтримуваного матеріалу.

## 1. Основні ресурси

| Назва/Посилання | Опис |
| :--- | :--- |
| [Тонке налаштування моделей OpenAI](https://platform.openai.com/docs/guides/fine-tuning?WT.mc_id=academic-105485-koreyst) | Тонке налаштування покращує навчання на кількох прикладах, тренуючись на набагато більшій кількості прикладів, ніж вміщується в підказку — економлячи кошти, покращуючи якість відповідей і зменшуючи затримки запитів. **Ознайомтеся з оглядом тонкого налаштування від OpenAI.** |
| [Коли використовувати тонке налаштування Microsoft Foundry](https://learn.microsoft.com/azure/ai-foundry/openai/concepts/fine-tuning-considerations?WT.mc_id=academic-105485-koreyst) | Зрозумійте **що таке тонке налаштування (концепція)**, навіщо воно потрібне, які дані використовувати і як вимірювати якість — а також коли підходять SFT, DPO або RFT. |
| [Налаштування моделі за допомогою тонкого налаштування](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning?WT.mc_id=academic-105485-koreyst) | Покрокова інструкція **як це зробити (процес)** у Microsoft Foundry за допомогою порталу, OpenAI / Foundry Python SDK або REST API — з покриттям підготовки даних, тренування, контрольних точок і розгортання. |
| [Безперервне тонке налаштування](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning?WT.mc_id=academic-105485-koreyst#perform-continuous-fine-tuning) | Ітеративний процес вибору вже тонко налаштованої моделі як базової та продовження її **тонкого налаштування** на нових наборах тренувальних прикладів. |
| [Тонке налаштування з викликом інструментів (функцій)](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning-functions?WT.mc_id=academic-105485-koreyst) | Тонке налаштування моделі **на прикладах виклику інструментів** покращує вихідні дані — робить відповіді точнішими, послідовними, однаково відформатованими з меншим числом токенів у підказці. |
| [Тонке налаштування моделей: керівництво Microsoft Foundry](https://learn.microsoft.com/azure/ai-foundry/foundry-models/concepts/models-sold-directly-by-azure?WT.mc_id=academic-105485-koreyst#fine-tuning-models) | Дізнайтеся, **які моделі можна тонко налаштовувати**, які методи вони підтримують (SFT / DPO / RFT) і в яких регіонах вони доступні. |
| [Огляд тонкого налаштування: техніки та модальності](https://learn.microsoft.com/azure/ai-foundry/concepts/fine-tuning-overview?WT.mc_id=academic-105485-koreyst) | Порівняйте три техніки навчання (SFT, DPO, RFT) і дві модальності (безсерверна та керована обчислювальна потужність), а також отримаєте рекомендації щодо вибору базової моделі та початку роботи. |
| **Посібник**: [Тонке налаштування моделі в Microsoft Foundry](https://learn.microsoft.com/azure/ai-foundry/openai/tutorials/fine-tune?WT.mc_id=academic-105485-koreyst) | Створіть приклад набору даних, підготуйтеся до тонкого налаштування, запустіть завдання тонкого налаштування на поточній підтримуваній моделі, наприклад `gpt-4.1-mini`, і розгорніть тонко налаштовану модель на Azure. |
| **Посібник**: [Тонке налаштування моделей із безсерверним розгортанням API](https://learn.microsoft.com/azure/ai-foundry/how-to/fine-tune-serverless?WT.mc_id=academic-105485-koreyst) | Налаштуйте відкриті та партнерські моделі (Phi, Llama, Mistral тощо) для ваших наборів даних _за допомогою low-code інтерфейсу_ в Microsoft Foundry. |
| **Посібник**: [Тонке налаштування моделей Hugging Face на Azure Databricks](https://learn.microsoft.com/azure/databricks/machine-learning/train-model/huggingface/fine-tune-model?WT.mc_id=academic-105485-koreyst) | Тонке налаштування моделі Hugging Face з бібліотекою `transformers` на одному GPU за допомогою Azure Databricks і Hugging Face Trainer. |
| **Навчання**: [Тонке налаштування фундаментальної моделі з Azure Machine Learning](https://learn.microsoft.com/training/modules/finetune-foundation-model-with-azure-machine-learning/?WT.mc_id=academic-105485-koreyst) | Каталог моделей Azure Machine Learning пропонує багато опенсорсних моделей для тонкого налаштування. Частина [Навчального шляху Azure ML Generative AI](https://learn.microsoft.com/training/paths/work-with-generative-models-azure-machine-learning/?WT.mc_id=academic-105485-koreyst). |
| **Посібник**: [Тонке налаштування Azure OpenAI з Weights & Biases](https://docs.wandb.ai/guides/integrations/azure-openai-fine-tuning?WT.mc_id=academic-105485-koreyst) | Відстежуйте й аналізуйте запуски тонкого налаштування на Azure за допомогою W&B. Розширює керівництво OpenAI тонкого налаштування з кроками, специфічними для Azure, і відстеженням експериментів. |

Додаток до [керівництва OpenAI з тонкого налаштування](https://platform.openai.com/docs/guides/fine-tuning?WT.mc_id=academic-105485-koreyst). |
| **OpenAI Cookbook**: [Тонке налаштування для Retrieval Augmented Generation (RAG) з Qdrant](https://cookbook.openai.com/examples/fine-tuned_qa/ft_retrieval_augmented_generation_qdrant?WT.mc_id=academic-105485-koreyst) | Всеосяжний приклад тонкого налаштування моделей OpenAI для RAG, інтеграції Qdrant і навчання на кількох прикладах для покращення продуктивності та зменшення вигадок. |
| **OpenAI Cookbook**: [Тонке налаштування GPT з Weights & Biases](https://cookbook.openai.com/examples/third_party/gpt_finetuning_with_wandb?WT.mc_id=academic-105485-koreyst) | Використовуйте W&B для відстеження тренування та тонкого налаштування моделей. Спочатку прочитайте їхнє [керівництво OpenAI Fine-Tuning](https://docs.wandb.ai/guides/integrations/openai-fine-tuning/?WT.mc_id=academic-105485-koreyst), а потім спробуйте вправу з Cookbook. |
