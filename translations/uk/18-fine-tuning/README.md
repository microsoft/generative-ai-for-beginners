# Налаштування вашої LLM

Використання великих мовних моделей для створення додатків генеративного штучного інтелекту супроводжується новими викликами. Основне питання полягає в забезпеченні якості відповіді (точність і релевантність) у контенті, що генерується моделлю на запит користувача. У попередніх уроках ми обговорювали такі техніки, як інженерія підказок і генерація з підсиленням вилученням, які намагаються вирішити проблему шляхом _модифікації вхідних даних підказки_ для існуючої моделі.

Сьогодні ми обговоримо третю техніку, **налаштування**, яка намагається вирішити цю проблему шляхом _перенавчання самої моделі_ з додатковими даними. Давайте заглибимося в деталі.

## Цілі навчання

Цей урок вводить концепцію налаштування для попередньо навчених мовних моделей, досліджує переваги та виклики цього підходу, а також надає рекомендації, коли і як використовувати налаштування для покращення продуктивності ваших генеративних моделей штучного інтелекту.

До кінця цього уроку ви повинні вміти відповісти на наступні питання:

- Що таке налаштування для мовних моделей?
- Коли і чому налаштування корисне?
- Як можна налаштувати попередньо навчену модель?
- Які обмеження налаштування?

Готові? Почнемо.

## Що таке налаштування для мовних моделей?

За визначенням, великі мовні моделі є _попередньо навченими_ на великих обсягах тексту, отриманих з різних джерел, включаючи інтернет. Як ми дізналися в попередніх уроках, нам потрібні такі техніки, як _інженерія підказок_ і _генерація з підсиленням вилученням_, щоб покращити якість відповідей моделі на запитання користувача ("підказки").

Популярна техніка інженерії підказок полягає в наданні моделі більшого керівництва щодо того, що очікується у відповіді, або шляхом надання _інструкцій_ (явне керівництво), або _наведенням кількох прикладів_ (неявне керівництво). Це називається _навчанням з кількома прикладами_, але воно має два обмеження:

- Обмеження токенів моделі можуть обмежити кількість прикладів, які ви можете надати, і обмежити ефективність.
- Вартість токенів моделі може зробити дорогим додавання прикладів до кожної підказки і обмежити гнучкість.

Налаштування - це загальноприйнята практика в системах машинного навчання, де ми беремо попередньо навчену модель і перенавчаємо її з новими даними для покращення її продуктивності на певному завданні. У контексті мовних моделей ми можемо налаштувати попередньо навчену модель _з підібраним набором прикладів для певного завдання або домену застосування_, щоб створити **індивідуальну модель**, яка може бути більш точною і релевантною для цього конкретного завдання або домену. Додатковою перевагою налаштування є те, що воно також може зменшити кількість прикладів, необхідних для навчання з кількома прикладами, зменшуючи використання токенів і пов'язані з цим витрати.

## Коли і чому ми повинні налаштовувати моделі?

У _цьому_ контексті, коли ми говоримо про налаштування, ми маємо на увазі **контрольоване** налаштування, де перенавчання здійснюється шляхом **додавання нових даних**, які не були частиною початкового набору даних для навчання. Це відрізняється від неконтрольованого підходу до налаштування, де модель перенавчається на початкових даних, але з іншими гіперпараметрами.

Головне, що потрібно пам'ятати, це те, що налаштування є передовою технікою, яка вимагає певного рівня експертизи для досягнення бажаних результатів. Якщо виконано неправильно, воно може не забезпечити очікуваних покращень, а може навіть погіршити продуктивність моделі для вашого цільового домену.

Отже, перш ніж ви дізнаєтеся "як" налаштовувати мовні моделі, вам потрібно знати "чому" ви повинні обрати цей шлях, і "коли" почати процес налаштування. Почніть, запитуючи себе ці питання:

- **Випадок використання**: Який ваш _випадок використання_ для налаштування? Який аспект поточної попередньо навченої моделі ви хочете покращити?
- **Альтернативи**: Чи пробували ви _інші техніки_ для досягнення бажаних результатів? Використовуйте їх для створення базового рівня для порівняння.
  - Інженерія підказок: Спробуйте техніки, такі як підказки з кількома прикладами, з прикладами релевантних відповідей на підказки. Оцініть якість відповідей.
  - Генерація з підсиленням вилученням: Спробуйте доповнити підказки результатами запитів, отриманих шляхом пошуку ваших даних. Оцініть якість відповідей.
- **Витрати**: Чи визначили ви витрати на налаштування?
  - Налаштованість - чи доступна попередньо навчена модель для налаштування?
  - Зусилля - для підготовки навчальних даних, оцінки та вдосконалення моделі.
  - Обчислення - для запуску завдань налаштування та розгортання налаштованої моделі.
  - Дані - доступ до достатньої кількості якісних прикладів для впливу налаштування.
- **Переваги**: Чи підтвердили ви переваги налаштування?
  - Якість - чи перевершила налаштована модель базовий рівень?
  - Вартість - чи зменшує вона використання токенів, спрощуючи підказки?
  - Розширюваність - чи можете ви перепрофілювати базову модель для нових доменів?

Відповівши на ці питання, ви повинні мати змогу вирішити, чи є налаштування правильним підходом для вашого випадку використання. Ідеально, підхід є дійсним, лише якщо переваги переважають витрати. Як тільки ви вирішите продовжити, настав час подумати про _як_ ви можете налаштувати попередньо навчену модель.

## Як ми можемо налаштувати попередньо навчену модель?

Щоб налаштувати попередньо навчену модель, вам потрібно мати:

- попередньо навчену модель для налаштування
- набір даних для використання при налаштуванні
- навчальне середовище для запуску завдання налаштування
- середовище хостингу для розгортання налаштованої моделі
