# Налаштування Вашої МОВ

Використання великих мовних моделей для створення генеративних AI-додатків пов’язане з новими викликами. Ключова проблема — це забезпечення якості відповіді (точності та релевантності) у контенті, який генерує модель за заданим запитом користувача. У попередніх уроках ми обговорювали такі техніки, як prompt engineering та retrieval-augmented generation, які намагаються вирішити проблему шляхом _зміни вхідного запиту_ до існуючої моделі.

В сьогоднішньому уроці ми розглянемо третю техніку — **тонке налаштування (fine-tuning)**, яка намагається вирішити виклик шляхом _перенавчання самої моделі_ за допомогою додаткових даних. Розглянемо деталі.

## Цілі навчання

У цьому уроці ми вводимо поняття тонкого налаштування для попередньо натренованих мовних моделей, розглядаємо переваги та виклики цього підходу, а також надаємо рекомендації, коли і як застосовувати тонке налаштування для покращення продуктивності ваших генеративних AI моделей.

До кінця цього уроку ви зможете відповісти на такі питання:

- Що таке тонке налаштування для мовних моделей?
- Коли і чому тонке налаштування є корисним?
- Як я можу тонко налаштувати попередньо натреновану модель?
- Які обмеження має тонке налаштування?

Готові? Почнімо.

## Що таке тонке налаштування для мовних моделей?

За визначенням, великі мовні моделі _попередньо навчені_ на великих обсягах тексту з різних джерел, включно з інтернетом. Як ми дізналися в попередніх уроках, для покращення якості відповідей моделі на запити користувачів ("prompt") потрібні такі методи, як _prompt engineering_ та _retrieval-augmented generation_.

Популярна техніка prompt engineering полягає в тому, щоб надати моделі більше вказівок щодо очікуваної відповіді: або через _інструкції_ (явні вказівки), або _подаючи кілька прикладів_ (неявні вказівки). Це називають _few-shot learning_, але у нього є два обмеження:

- Обмеження кількості токенів моделі може обмежити кількість прикладів і знизити ефективність.
- Вартість токенів моделі може зробити додавання прикладів до кожного запиту дорогим і знизити гнучкість.

Тонке налаштування — поширена практика у системах машинного навчання, де попередньо натреновану модель перенавчають на нових даних для покращення її продуктивності для конкретного завдання. У контексті мовних моделей можна тонко налаштувати попередньо натреновану модель _на відібраному наборі прикладів для певного завдання або домену застосування_, щоб створити **кастомну модель**, яка може бути точнішою та релевантнішою для цього завдання чи домену. Додатковою перевагою тонкого налаштування є те, що воно може зменшити кількість прикладів, необхідних для few-shot learning — знижуючи використання токенів та пов’язані витрати.

## Коли і чому слід тонко налаштовувати моделі?

У _цьому_ контексті, коли ми говоримо про тонке налаштування, ми маємо на увазі **кероване (supervised)** тонке налаштування, де перенавчання здійснюється шляхом **додавання нових даних**, які не входили до початкового навчального набору даних. Це відрізняється від некерованого (unsupervised) тонкого налаштування, де модель перенавчають на тих самих даних, але з іншими гіперпараметрами.

Важливо пам’ятати, що тонке налаштування — це просунута техніка, яка вимагає певного рівня експертизи для досягнення бажаних результатів. Якщо зробити це неправильно, воно може не принести очікуваних покращень, а навіть погіршити продуктивність моделі для вашого цільового домену.

Тож перш ніж вивчати "як" тонко налаштовувати мовні моделі, потрібно розуміти "чому" варто обрати цей шлях, і "коли" почати процес тонкого налаштування. Почніть із запитань:

- **Випадок використання:** Який ваш _випадок використання_ для тонкого налаштування? Який аспект поточної попередньо натренованої моделі ви хочете покращити?
- **Альтернативи:** Чи пробували ви _інші техніки_ для досягнення бажаних результатів? Використовуйте їх, щоб створити базовий рівень для порівняння.
  - Prompt engineering: Спробуйте техніки, як few-shot prompting з прикладами релевантних відповідей. Оцініть якість відповідей.
  - Retrieval Augmented Generation: Спробуйте доповнювати запити результатами пошуку у ваших даних. Оцініть якість відповідей.
- **Витрати:** Чи визначили ви витрати на тонке налаштування?
  - Можливість налаштування — чи доступна модель для тонкого налаштування?
  - Зусилля — на підготовку тренувальних даних, оцінку та доопрацювання моделі.
  - Обчислювальні ресурси — для запуску завдань тонкого налаштування та розгортання моделі.
  - Дані — доступ до достатньо якісних прикладів для ефективності тонкого налаштування.
- **Переваги:** Чи підтвердили ви переваги тонкого налаштування?
  - Якість — чи перевищила тонко налаштована модель базову?
  - Вартість — чи знижує це використання токенів за рахунок спрощення запитів?
  - Розширюваність — чи можна використовувати базову модель для нових доменів?

Відповідаючи на ці питання, ви зможете вирішити, чи є тонке налаштування правильним підходом для вашого випадку. Ідеально, якщо переваги переважають витрати. Коли ви вирішите продовжувати, настав час подумати, _як_ ви можете тонко налаштувати попередньо натреновану модель.

## Як ми можемо тонко налаштувати попередньо натреновану модель?

Щоб тонко налаштувати попередньо натреновану модель, вам потрібно мати:

- попередньо натреновану модель для налаштування
- набір даних для тонкого налаштування
- середовище для запуску завдання тонкого налаштування
- середовище для розгортання тонко налаштованої моделі

## Тонке налаштування на Microsoft Foundry

[Microsoft Foundry](https://ai.azure.com?WT.mc_id=academic-105485-koreyst) — це сервіс, де сьогодні ви тонко налаштовуєте, розгортаєте та керуєте кастомними моделями на Azure (його було створено на основі Azure OpenAI Studio та Azure AI Studio). Перш ніж розпочати завдання, корисно зрозуміти вибір, який надає Foundry, та найкращі практики, які рекомендує платформа. Під капотом Foundry використовує **LoRA (low-rank adaptation)** для ефективного тонкого налаштування моделей, що робить навчання швидшим і дешевшим, ніж перенавчання кожної ваги.

### Крок 1: Оберіть техніку навчання

Foundry підтримує три техніки тонкого налаштування. **Починайте з SFT** — вона охоплює найширше коло сценаріїв.

| Техніка | Що робить | Коли використовувати |
| --- | --- | --- |
| **Supervised Fine-Tuning (SFT)** | Навчає на парах вхід/вихід, щоб модель навчилася створювати бажані відповіді. | За замовчуванням для більшості завдань: спеціалізація домену, виконання завдань, стиль і тон, виконання інструкцій та адаптація мови. |
| **Direct Preference Optimization (DPO)** | Навчається на парах «бажана проти небажаної» відповіді для узгодження результатів з людськими вподобаннями. | Покращує якість відповіді, безпеку та узгодженість, коли є порівняльний зворотний зв’язок. |
| **Reinforcement Fine-Tuning (RFT)** | Використовує сигнали винагороди від _оцінювачів_ для оптимізації складних поведінкових моделей з підкріплювальним навчанням. | Об’єктивні, з високим рівнем логіки домени (математика, хімія, фізика) з чіткими правильними/неправильними відповідями. Вимагає більшої компетентності в машинному навчанні. |

### Крок 2: Оберіть рівень навчання

Foundry дозволяє обирати як і де відбувається навчання:

- **Стандартний** — навчає у вашому регіоні ресурсів та гарантує місце зберігання даних. Використовуйте, якщо дані мають залишатися у певному регіоні.
- **Глобальний** — дешевше і швидше в черзі, використовуючи ресурси поза вашим регіоном (дані і ваги копіюються до регіону навчання). Добрий стандарт, коли місце зберігання даних не критичне.
- **Розробник** — найнижча вартість, використання проста闲них ресурсів без гарантій затримки/доступності (завдання можна призупиняти й відновлювати). Ідеально для експериментів.

### Крок 3: Оберіть базову модель

Моделі, які можна тонко налаштовувати, включають OpenAI `gpt-4o-mini`, `gpt-4o`, `gpt-4.1`, `gpt-4.1-mini` та `gpt-4.1-nano` (SFT; сімейство 4o/4.1 також підтримує DPO), моделі для логічного мислення `o4-mini` та `gpt-5` (RFT), а також open-source моделі, такі як `Ministral-3B`, `Qwen-32B`, `Llama-3.3-70B-Instruct` та `gpt-oss-20b` (SFT на ресурсах Foundry). Завжди перевіряйте актуальний [список моделей для тонкого налаштування](https://learn.microsoft.com/azure/ai-foundry/foundry-models/concepts/models-sold-directly-by-azure?WT.mc_id=academic-105485-koreyst#fine-tuning-models) для підтримуваних методів, регіонів та наявності.

> Foundry пропонує два режими: **serverless** (оплата за споживання, немає потреби керувати квотою GPU, OpenAI та окремі моделі) та **managed compute** (власні віртуальні машини через Azure Machine Learning для найбільшого вибору моделей). Більшість користувачів слід починати з serverless.

### Найкращі практики Foundry

- **Спочатку базова лінія.** Оцініть базову модель за допомогою prompt engineering та RAG _перед_ тонким налаштуванням, щоб довести покращення.
- **Починайте з малої кількості, потім масштабуйте.** Спершу використовуйте 50-100 якісних прикладів для перевірки підходу, потім збільшуйте до 500+ для виробництва. Якість важливіша за кількість — видаляйте низькоякісні приклади.
- **Правильно форматувати дані.** Тренувальні та валідаційні файли мають бути у форматі JSONL, UTF-8 **з BOM**, розміром менше 512 МБ, використовувати формат повідомлень chat-completions. Завжди додавайте файл валідації для контролю перенавчання.
- **Зберігайте системні підказки для інференсу.** Використовуйте ту саму системну підказку, що й під час навчання.
- **Оцінюйте контрольні точки — не розгортайте сліпо останню.** Foundry зберігає останні три епохи як контрольні точки, які можна розгортати; обирайте ту, що найкраще узагальнює за `train_loss` / `valid_loss` та точністю токенів.
- **Вимірюйте вартість токенів разом із якістю** при порівнянні тонко налаштованої моделі із базовою.
- **Ітеруйте з безперервним тонким налаштуванням.** Можна тонко налаштовувати вже налаштовану модель новими даними (підтримується для моделей OpenAI).
- **Увага до вартості хостингу.** Розгорнута кастомна модель оплачується погодинно; неактивні розгортання видаляються через 15 днів — прибирайте непотрібне.

Пройдіть поетапний приклад у [Кастомізація моделі з тонким налаштуванням](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning?WT.mc_id=academic-105485-koreyst), а також дивіться посібники для [DPO](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning-direct-preference-optimization?WT.mc_id=academic-105485-koreyst) та [RFT](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/reinforcement-fine-tuning?WT.mc_id=academic-105485-koreyst), коли будете готові до інших технік.
