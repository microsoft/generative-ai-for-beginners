# Вступ до Малих Мовних Моделей для Генеративного ШІ для Початківців
Генеративний ШІ — це захоплююча область штучного інтелекту, яка зосереджена на створенні систем, здатних генерувати новий контент. Цей контент може варіюватися від тексту та зображень до музики і навіть цілих віртуальних середовищ. Одним з найцікавіших застосувань генеративного ШІ є сфера мовних моделей.

## Що таке Малі Мовні Моделі?

Мала Мовна Модель (SLM) — це зменшена варіація великої мовної моделі (LLM), що використовує багато архітектурних принципів і методик LLM, водночас демонструючи значно менше обчислювальне навантаження. 

SLM — це підмножина мовних моделей, призначених для генерації тексту, подібного до людського. На відміну від їх більших аналогів, таких як GPT-4, SLM компактніші та ефективніші, що робить їх ідеальними для застосувань з обмеженими обчислювальними ресурсами. Незважаючи на менший розмір, вони все ще можуть виконувати різноманітні завдання. Зазвичай SLM створюють шляхом стиснення або дистиляції LLM, прагнучи зберегти значну частину функціональності та лінгвістичних можливостей оригінальної моделі. Це зменшення розміру моделі знижує загальну складність, роблячи SLM ефективнішими як з точки зору використання пам’яті, так і обчислювальних вимог. Незважаючи на ці оптимізації, SLM все ще здатні виконувати широкий спектр завдань обробки природної мови (NLP):

- Генерація тексту: Створення зв’язних та контекстно релевантних речень або абзаців.
- Завершення тексту: Передбачення і доповнення речень на основі заданого підказування.
- Переклад: Перетворення тексту з однієї мови на іншу.
- Резюмування: Стислий виклад довгих текстів у коротші, більш зрозумілі огляди.

Хоча з певними компромісами у продуктивності або глибині розуміння порівняно з більшими аналогами. 

## Як Працюють Малі Мовні Моделі?
SLM тренуються на великих обсягах текстових даних. Під час навчання вони вивчають закономірності та структури мови, що дозволяє їм генерувати текст, який є граматично правильним і контекстно доречним. Процес навчання включає:

- Збір даних: Збір великих наборів текстів з різних джерел.
- Попередня обробка: Очищення та організація даних для підготовки до навчання.
- Навчання: Використання алгоритмів машинного навчання для навчання моделі розуміти й генерувати текст.
- Тонке налаштування: Коригування моделі для покращення її продуктивності з певних завдань.

Розробка SLM відповідає зростаючій потребі у моделях, які можуть розгортатися в середовищах з обмеженими ресурсами, таких як мобільні пристрої або платформи edge-комп’ютингу, де повномасштабні LLM можуть бути непрактичними через їх великі вимоги до ресурсів. Зосереджуючись на ефективності, SLM балансуюють між продуктивністю та доступністю, дозволяючи ширше застосування у різних сферах.

## Відмінності між Великими Мовними Моделями (LLM) та Малими Мовними Моделями (SLM)

І LLM, і SLM базуються на фундаментальних принципах ймовірнісного машинного навчання, слідуючи подібним підходам у архітектурному дизайні, методах навчання, процесах генерації даних і методах оцінки моделей. Однак є кілька ключових факторів, що відрізняють ці два типи моделей.

## Застосування Малих Мовних Моделей

SLM мають широкий спектр застосувань, серед яких:

- Чатботи: Надання підтримки клієнтам і спілкування з користувачами в розмовній формі.
- Створення контенту: Допомога авторам у генеруванні ідей або навіть написанні цілих статей.
- Освіта: Допомога студентам із письмовими завданнями або вивченням нових мов.
- Доступність: Створення інструментів для людей з обмеженими можливостями, таких як системи синтезу мовлення.

**Розмір**
  
Основною відмінністю між LLM і SLM є масштаб моделей. LLM, такі як ChatGPT (GPT-4), можуть містити приблизно 1,76 трильйона параметрів, тоді як відкриті SLM, як-от Mistral 7B, мають значно менше параметрів — приблизно 7 мільярдів. Ця різниця головним чином пов’язана з різницями в архітектурі моделей і процесах навчання. Наприклад, ChatGPT використовує механізм самоуваги в рамках енкодер-декодерної структури, тоді як Mistral 7B застосовує механізм ковзного вікна уваги, що дозволяє ефективніше навчатися в моделях лише декодерного типу. Ця архітектурна відмінність має суттєві наслідки для складності та продуктивності моделей.

**Розуміння**

SLM, як правило, оптимізовані для виконання завдань у певних доменах, що робить їх високоспеціалізованими, але потенційно обмеженими в здатності забезпечувати широке контекстуальне розуміння в різних сферах знань. Натомість LLM прагнуть імітувати інтелект, близький до людського, більш комплексно. Треновані на величезних і різноманітних наборах даних, LLM створені для хорошої роботи в різних доменах, пропонуючи більшу універсальність та адаптивність. Відповідно, LLM більш придатні для більш широкого спектра завдань, таких як обробка природної мови та програмування.

**Виконання обчислень**

Навчання та розгортання LLM є ресурсомісткими процесами, які часто потребують значної обчислювальної інфраструктури, включно з великими GPU-кластерами. Наприклад, навчання моделі на кшталт ChatGPT з нуля може вимагати тисячі GPU протягом тривалого часу. Натомість SLM, з меншим числом параметрів, більш доступні у плані обчислювальних ресурсів. Моделі, як Mistral 7B, можна навчати і запускати на локальних машинах з помірними можливостями GPU, хоча навчання все одно потребує кілька годин на декількох GPU.

**Упередження**

Упередження є відомою проблемою LLM, головним чином через природу навчальних даних. Ці моделі часто базуються на сирих, відкритих даних із інтернету, які можуть недостатньо або неправильно представляти певні групи, містити помилкове маркування або відображати лінгвістичні упередження, пов’язані з діалектом, географічними різновидами та граматичними правилами. Крім того, складність архітектури LLM може випадково посилювати упередження, які можуть залишатися непоміченими без ретельного тонкого налаштування. З іншого боку, SLM, які навчаються на більш обмежених, доменспецифічних наборах даних, априорно менш схильні до таких упереджень, хоча і не є повністю вільними від них.

**Висновки (Inference)**

Менший розмір SLM дає їм суттєву перевагу в швидкості висновків, дозволяючи ефективно генерувати результати на локальному обладнанні без необхідності інтенсивної паралельної обробки. Натомість LLM через свій розмір і складність часто потребують значних паралельних обчислювальних ресурсів для досягнення прийнятних часів відгуку. Наявність багатьох одночасних користувачів ще більше уповільнює час відгуку LLM, особливо при масштабному розгортанні.

Підсумовуючи, обидві LLM і SLM мають спільну основу у машинному навчанні, але суттєво відрізняються за розміром моделі, вимогами до ресурсів, контекстним розумінням, чутливістю до упереджень та швидкістю висновків. Ці відмінності відображають їх відповідну придатність для різних випадків використання: LLM є більш універсальними, але ресурсомісткими, тоді як SLM пропонують ефективність у певних доменах із зниженими обчислювальними потребами.

## Ознайомлення з сімейством Phi-3 / Phi-3.5

Сімейство Phi-3 / 3.5 в основному орієнтоване на сценарії застосування в тексті, зорі та агентах (MoE):

### Phi-3 / 3.5 Instruct

В основному для генерації тексту, завершення чатів, витягування інформації з контенту тощо.

**Phi-3-mini**

Мовна модель із 3.8 млрд параметрів доступна на Microsoft Foundry, Hugging Face і Ollama. Моделі Phi-3 значно перевершують мовні моделі рівного та більших розмірів на ключових бенчмарках (див. нижче; вищі числа означають кращий результат). Phi-3-mini перевершує моделі вдвічі більшого розміру, тоді як Phi-3-small і Phi-3-medium краще за більші моделі, включно з GPT-3.5.

**Phi-3-small і medium**

Із всього 7 млрд параметрів Phi-3-small перевершує GPT-3.5T на різних бенчмарках із мови, логіки, кодування та математики.

Phi-3-medium із 14 млрд параметрів продовжує цю тенденцію і краще за Gemini 1.0 Pro.

**Phi-3.5-mini**

Можна вважати це оновленням Phi-3-mini. Параметри залишаються незмінними, але покращено підтримку багатьох мов (підтримується 20+ мов: арабська, китайська, чеська, датська, голландська, англійська, фінська, французька, німецька, іврит, угорська, італійська, японська, корейська, норвезька, польська, португальська, російська, іспанська, шведська, тайська, турецька, українська) і додано більш потужну підтримку довгих контекстів.

Phi-3.5-mini з 3.8 млрд параметрів перевершує мовні моделі свого ж розміру і порівнянний із моделями вдвічі більшого розміру.

### Phi-3 / 3.5 Vision

Модель Instruct Phi-3/3.5 можна сприймати як здатність Phi розуміти, а Vision — це те, що дає Phi «очі» для розуміння світу.


**Phi-3-Vision**

Phi-3-vision, всього з 4.2 млрд параметрів, продовжує цю тенденцію і перевершує більші моделі, такі як Claude-3 Haiku і Gemini 1.0 Pro V, у загальних завданнях візуального мислення, OCR, а також у завданнях розуміння таблиць і діаграм.


**Phi-3.5-Vision**

Phi-3.5-Vision також є оновленням Phi-3-Vision, додаючи підтримку кількох зображень. Можна сказати, що це покращення зору: модель не лише бачить картинки, а й відео.

Phi-3.5-vision перевершує більші моделі, такі як Claude-3.5 Sonnet та Gemini 1.5 Flash у завданнях OCR, розумінні таблиць і графіків, а також є порівнянним за загальним візуальним знанням і логікою. Підтримує мультикадровий вхід, тобто виконує логічні операції над кількома вхідними зображеннями.


### Phi-3.5-MoE

***Мікс експертів (MoE)*** дозволяє моделям навчатися значно швидше, що означає, що можна суттєво масштабувати розмір моделі або набір даних з тим самим бюджетом обчислень, що й у звичайної повнозв’язної моделі. Зокрема, MoE моделі повинні досягати якості рівня своєї повнозв’язної альтернативи набагато швидше під час попереднього навчання.

Phi-3.5-MoE складається з 16 × 3.8 млрд експертних модулів. Phi-3.5-MoE з лише 6.6 млрд активних параметрів досягає рівня логіки, розуміння мови та математики, схожого на значно більші моделі.

Ми можемо використовувати моделі сімейства Phi-3/3.5 у різних сценаріях. На відміну від LLM, Phi-3/3.5-mini або Phi-3/3.5-Vision можна запускати на периферійних пристроях.

### Запуск Phi-3/3.5 локально
Інференс у контексті Phi-3, або будь-якої мовної моделі, як GPT-3, означає процес генерації відповідей або прогнозів на основі отриманого вводу. Коли ви надаєте підказку або питання Phi-3, він використовує свою натреновану нейронну мережу, щоб вивести найбільш ймовірну та релевантну відповідь, аналізуючи патерни та взаємозв’язки в даних, на яких він навчався.

**Hugging Face Transformer**
Hugging Face Transformers — це потужна бібліотека, розроблена для обробки природної мови (NLP) та інших завдань машинного навчання. Ось кілька ключових моментів:

1. **Попередньо навчені моделі**: Надає тисячі попередньо навчених моделей, які можна використовувати для різних задач, таких як класифікація тексту, виявлення іменованих сутностей, відповідь на запитання, резюмування, переклад та генерація тексту.

2. **Інтероперабельність фреймворків**: Бібліотека підтримує кілька фреймворків глибокого навчання, включно з PyTorch, TensorFlow і JAX. Це дає змогу тренувати модель в одному фреймворку та використовувати в іншому.

3. **Мультимодальні можливості**: Окрім NLP, Hugging Face Transformers підтримує завдання з комп’ютерного зору (наприклад, класифікацію зображень, виявлення об’єктів) та обробки аудіо (наприклад, розпізнавання мовлення, класифікація аудіо).

4. **Зручність використання**: Бібліотека пропонує API і інструменти для легкого завантаження та донавчання моделей, що робить її доступною як для початківців, так і для експертів.

5. **Спільнота та ресурси**: Hugging Face має активну спільноту та обширну документацію, навчальні матеріали й гіди, які допомагають користувачам розпочати роботу та максимально ефективно використовувати бібліотеку.

**ONNX Runtime для GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) — кросплатформений прискорювач інференсу та тренування моделей машинного навчання. ONNX Runtime для Generative AI (GENAI) — потужний інструмент, що допомагає ефективно запускати генеративні AI-моделі на різних платформах.

## Що таке ONNX Runtime?
ONNX Runtime — це проект з відкритим кодом, який забезпечує високопродуктивний інференс моделей машинного навчання. Він підтримує моделі у форматі Open Neural Network Exchange (ONNX), що є стандартом для представлення моделей машинного навчання. ONNX Runtime дозволяє прискорити обробку запитів користувачів і знизити витрати, підтримуючи моделі з фреймворків глибокого навчання, таких як PyTorch і TensorFlow/Keras, а також класичні бібліотеки машинного навчання, як scikit-learn, LightGBM, XGBoost тощо. ONNX Runtime сумісний з різним апаратним забезпеченням, драйверами та операційними системами, і забезпечує оптимальну продуктивність, використовуючи апаратні прискорювачі поряд з оптимізацією і трансформаціями графів.

## Що таке генеративний AI?
Генеративний AI — це системи штучного інтелекту, які можуть створювати новий контент, наприклад текст, зображення або музику, базуючись на даних, на яких вони були навчені. Прикладами є мовні моделі типу GPT-3 та моделі генерації зображень, як Stable Diffusion. Бібліотека ONNX Runtime для GenAI забезпечує генеративний AI цикл для ONNX-моделей, включно з інференсом за допомогою ONNX Runtime, обробкою логітів, пошуком та семплінгом, а також керуванням кешем KV.

## ONNX Runtime для GENAI
ONNX Runtime для GENAI розширює можливості ONNX Runtime для підтримки генеративних AI-моделей. Ось деякі ключові особливості:

- **Широка підтримка платформ:** Працює на різних платформах, включно з Windows, Linux, macOS, Android і iOS.
- **Підтримка моделей:** Підтримує багато популярних генеративних AI-моделей, таких як LLaMA, GPT-Neo, BLOOM та інші.
- **Оптимізація продуктивності:** Містить оптимізації для різних апаратних прискорювачів, таких як NVIDIA GPU, AMD GPU тощо.
- **Зручність використання:** Надає API для простої інтеграції в додатки, дозволяючи генерувати текст, зображення та інший контент з мінімумом коду.
- Користувачі можуть викликати високорівневий метод generate(), або виконувати кожну ітерацію моделі у циклі, генеруючи по одному токену за раз, і опціонально оновлювати параметри генерації всередині циклу.
- ONNX Runtime також підтримує жадібний/променевий пошук та семплінг TopP, TopK для генерування послідовностей токенів та вбудовану обробку логітів, як штрафи за повторення. Також легко додати власні оцінки.

### Запустіть модель: Ось простий приклад на Python:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### Демонстрація: Використання ONNX Runtime GenAI для виклику Phi-3.5-Vision


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
