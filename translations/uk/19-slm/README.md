# Вступ до малих мовних моделей для генеративного ШІ для початківців
Генеративний ШІ — це захоплююча галузь штучного інтелекту, що зосереджується на створенні систем, здатних генерувати новий контент. Цей контент може варіюватися від тексту та зображень до музики і навіть цілих віртуальних середовищ. Одним із найцікавіших застосувань генеративного ШІ є сферa мовних моделей.

## Що таке малі мовні моделі?

Мала мовна модель (SLM) — це спрощений варіант великої мовної моделі (LLM), який використовує багато архітектурних принципів і технік LLM, одночасно маючи значно менше обчислювальне навантаження.

SLM — це підмножина мовних моделей, розроблена для генерації тексту, схожого на людський. На відміну від їх більших аналогів, таких як GPT-4, SLM є компактнішими та ефективнішими, що робить їх ідеальними для застосувань із обмеженими обчислювальними ресурсами. Незважаючи на менший розмір, вони все ще можуть виконувати різноманітні завдання. Зазвичай SLM створюють шляхом стиснення або дистиляції LLM, прагнучи зберегти значну частину функціональності та мовних можливостей оригінальної моделі. Це зменшення розміру моделі знижує загальну складність, роблячи SLM більш ефективними як з точки зору використання пам’яті, так і обчислювальних вимог. Незважаючи на ці оптимізації, SLM можуть виконувати широкий спектр завдань обробки природної мови (NLP):

- Генерація тексту: Створення послідовних і контекстуально релевантних речень або абзаців.
- Завершення тексту: Передбачення і доповнення речень на основі заданого підказування.
- Переклад: Перетворення тексту з однієї мови на іншу.
- Резюмування: Скорочення довгих текстів до коротших, легших для сприйняття викладів.

Хоча й із певними компромісами у продуктивності чи глибині розуміння порівняно з їх більшими аналогами. 

## Як працюють малі мовні моделі?
SLM навчаються на величезних обсягах текстових даних. Під час навчання вони вивчають закономірності та структури мови, що дозволяє їм генерувати текст, який є граматично правильним і контекстуально доречним. Процес навчання включає:

- Збір даних: Збір великих наборів текстів із різних джерел.
- Попередня обробка: Очищення та організація даних для підготовки до навчання.
- Навчання: Використання алгоритмів машинного навчання для навчання моделі розуміти та генерувати текст.
- Тонке налаштування: Коригування моделі для покращення її роботи на конкретних завданнях.

Розробка SLM відповідає зростаючій потребі в моделях, які можна розгортати в умовах обмежених ресурсів, таких як мобільні пристрої чи платформи edge-комп’ютингу, де повномасштабні LLM можуть бути непрактичними через великі вимоги до ресурсів. Орієнтуючись на ефективність, SLM досягають балансу між продуктивністю та доступністю, забезпечуючи ширше застосування в різних галузях.

## Відмінності між великими мовними моделями (LLM) і малими мовними моделями (SLM)

І LLM, і SLM побудовані на основних принципах імовірнісного машинного навчання, слідуючи подібним підходам в архітектурі, методах навчання, процесах генерації даних і техніках оцінки моделей. Однак кілька ключових факторів відрізняють ці два типи моделей.

## Застосування малих мовних моделей

SLM мають широкий спектр застосувань, серед яких:

- Чатботи: Надання підтримки клієнтам і спілкування з користувачами у розмовному форматі.
- Створення контенту: Допомога письменникам у генерації ідей або навіть створенні повних статей.
- Освіта: Допомога студентам із письмовими завданнями або вивченням нових мов.
- Доступність: Створення інструментів для людей з інвалідністю, таких як системи перетворення тексту в мову.

**Розмір**
  
Основна відмінність між LLM та SLM полягає у масштабі моделей. LLM, як ChatGPT (GPT-4), може містити приблизно 1,76 трильйона параметрів, тоді як відкриті SLM, як Mistral 7B, мають значно менше параметрів — близько 7 мільярдів. Це розбіжність обумовлена переважно відмінностями в архітектурі моделі та процесах навчання. Наприклад, ChatGPT використовує механізм самоуваги в структурі енкодер-декодер, тоді як Mistral 7B покладається на ковзне вікно уваги, що дозволяє більш ефективно навчатися в моделі з виключно декодером. Ця архітектурна різниця має суттєві наслідки для складності й продуктивності моделей.

**Розуміння**

Зазвичай SLM оптимізовані для роботи в певних галузях, що робить їх дуже спеціалізованими, але потенційно обмеженими у здатності надавати широке контекстуальне розуміння у різних сферах знань. У свою чергу, LLM прагнуть наслідувати людський інтелект на більш комплексному рівні. Навчені на великих і різноманітних наборах даних, LLM розраховані на хорошу продуктивність у різних сферах, забезпечуючи більшу універсальність і адаптивність. Тому LLM підходять для ширшого кола завдань, таких як обробка природної мови і програмування.

**Обчислення**

Навчання і розгортання LLM є ресурсомісткими процесами, які часто вимагають значної обчислювальної інфраструктури, включно з великими кластерами GPU. Наприклад, навчання моделі, як ChatGPT, з нуля може зайняти тисячі GPU протягом тривалого часу. На відміну від них, SLM із меншою кількістю параметрів є більш доступними в плані обчислювальних ресурсів. Моделі, як Mistral 7B, можна навчати і запускати на локальних машинах із помірними можливостями GPU, хоча навчання все ж потребує кількох годин на кількох GPU.

**Упередженість**

Упередженість є відомою проблемою для LLM, головним чином через природу навчальних даних. Ці моделі часто використовують сирі, відкрито доступні дані з інтернету, які можуть недостатньо або неправильно представляти певні групи, містити хибне маркування або відображати лінгвістичні упередження, пов’язані з діалектами, географічними варіаціями та граматичними правилами. Крім того, складність архітектури LLM може випадково посилити упередженість, яка може залишатися непоміченою без ретельного тонкого налаштування. З іншого боку, SLM, які навчаються на більш обмежених, доменно-специфічних наборах даних, природно менш схильні до таких упереджень, хоча не є від них повністю вільними.

**Висновок**

Зменшений розмір SLM дає їм значну перевагу в швидкості висновку, дозволяючи ефективно генерувати результати на локальному обладнанні без потреби у великій паралельній обробці. Натомість LLM через свій розмір і складність часто потребують значних паралельних обчислювальних ресурсів для досягнення прийнятних часів висновку. Наявність багатьох одночасних користувачів додатково уповільнює час відгуку LLM, особливо при масштабному розгортанні.

Отже, хоча як LLM, так і SLM мають спільні основу в машинному навчанні, вони значно різняться за розміром моделей, вимогами до ресурсів, контекстуальним розумінням, схильністю до упереджень і швидкістю висновку. Ці відмінності відображають їхню відповідність для різних випадків використання: LLM є більш універсальними, але ресурсомісткими, тоді як SLM пропонують галузеву ефективність із меншими обчислювальними витратами.

## Представлення сімейства Phi-3 / Phi-3.5

Сімейство Phi-3 / 3.5 орієнтоване головним чином на текстові, візуальні та агентські (MoE) сценарії застосування:

### Phi-3 / 3.5 Instruct

Головним чином для генерації тексту, завершення діалогів і вилучення інформації з контенту тощо.

**Phi-3-mini**

Мовна модель з 3,8 млрд параметрів доступна на Microsoft Foundry, Hugging Face та Ollama. Моделі Phi-3 значно перевершують мовні моделі рівного та більшого розміру за ключовими бенчмарками (див. наведені нижче оцінки, вищі значення кращі). Phi-3-mini перевершує моделі вдвічі більшого розміру, тоді як Phi-3-small і Phi-3-medium перевершують більші моделі, включно з GPT-3.5.

**Phi-3-small & medium**

Phi-3-small з 7 млрд параметрів перевершує GPT-3.5T у різних мовних, логічних, кодувальних та математичних бенчмарках.

Phi-3-medium з 14 млрд параметрів продовжує цю тенденцію і перевершує Gemini 1.0 Pro.

**Phi-3.5-mini**

Його можна розглядати як оновлення Phi-3-mini. Хоч параметри залишилися незмінними, покращується підтримка кількох мов (підтримує більш як 20 мов: арабська, китайська, чеська, данська, голландська, англійська, фінська, французька, німецька, іврит, угорська, італійська, японська, корейська, норвезька, польська, португальська, російська, іспанська, шведська, тайська, турецька, українська) та додається потужніша підтримка довгого контексту.

Phi-3.5-mini з 3,8 млрд параметрів перевершує мовні моделі того ж розміру і на рівні моделей удвічі більшого розміру.

### Phi-3 / 3.5 Vision

Модель Instruct у Phi-3/3.5 можна вважати здатністю Phi розуміти, а Vision дає Phi очі для розуміння світу.


**Phi-3-Vision**

Phi-3-vision з лише 4,2 млрд параметрів продовжує цю тенденцію і перевершує більші моделі, як Claude-3 Haiku і Gemini 1.0 Pro V, у завданнях загального візуального мислення, OCR, а також у розумінні таблиць і діаграм.


**Phi-3.5-Vision**

Phi-3.5-Vision — також оновлення Phi-3-Vision, додає підтримку кількох зображень. Його можна вважати покращенням зору: не лише бачити картинки, а й відео.

Phi-3.5-vision перевершує більші моделі, такі як Claude-3.5 Sonnet і Gemini 1.5 Flash, у завданнях OCR, розумінні таблиць і графіків та на рівні із загальними завданнями візуального мислення. Підтримує багатокадровий вхід, тобто виконує мислення на основі кількох вхідних зображень.


### Phi-3.5-MoE

***Mixture of Experts (MoE)*** дозволяє моделям попередньо навчатися із значно меншими обчислювальними затратами, що означає можливість значного збільшення розміру моделі або набору даних з тим самим бюджетом обчислювальних ресурсів, що й щільна модель. Зокрема, MoE модель повинна досягати тієї ж якості, що й щільна модель, значно швидше під час попереднього навчання.

Phi-3.5-MoE складається з 16 експертних модулів по 3,8 млрд параметрів кожен. Phi-3.5-MoE з лише 6,6 млрд активних параметрів досягає схожого рівня логіки, розуміння мови і математики, що й значно більші моделі.

Модель сімейства Phi-3/3.5 можна використовувати в різних сценаріях. На відміну від LLM, Phi-3/3.5-mini або Phi-3/3.5-Vision можна розгортати на периферійних пристроях.

### Запуск Phi-3/3.5 локально
Інференс у контексті Phi-3 або будь-якої мовної моделі, такої як GPT-3, означає процес генерації відповідей або прогнозів на основі вхідних даних, які вона отримує. Коли ви надаєте запит або питання Phi-3, модель використовує своє навчене нейронне мережеве представлення для виведення найбільш ймовірної та релевантної відповіді, аналізуючи закономірності та зв’язки у навчальних даних.

**Hugging Face Transformer**
Hugging Face Transformers — це потужна бібліотека, призначена для обробки природної мови (NLP) та інших завдань машинного навчання. Ось кілька ключових моментів про неї:

1. **Попередньо навчені моделі**: Надає тисячі попередньо навчених моделей, які можна використовувати для різних завдань, таких як класифікація тексту, розпізнавання іменованих сутностей, відповіді на питання, резюмування, переклад та генерація тексту.

2. **Інтероперабельність фреймворків**: Бібліотека підтримує декілька фреймворків глибинного навчання, включно з PyTorch, TensorFlow та JAX. Це дозволяє тренувати модель в одному фреймворку та використовувати її в іншому.

3. **Мультимодальні можливості**: Окрім NLP, Hugging Face Transformers також підтримує завдання комп’ютерного зору (наприклад, класифікація зображень, виявлення об’єктів) та аудіообробки (наприклад, розпізнавання мови, класифікація аудіо).

4. **Простота використання**: Бібліотека пропонує API та інструменти для легкого завантаження й донастройки моделей, роблячи її доступною як для початківців, так і для експертів.

5. **Спільнота та ресурси**: Hugging Face має активну спільноту, а також обширну документацію, підручники та керівництва, які допомагають користувачам розпочати роботу та максимально ефективно використовувати бібліотеку.

**Ollama**
[Ollama](https://ollama.com/?WT.mc_id=academic-105485-koreyst) — це платформа, створена для спрощення запуску великих мовних моделей (LLM) локально на вашому пристрої. ollama run phi3.5

**ONNX Runtime для GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) є кросплатформним прискорювачем для інференсу та тренування моделей машинного навчання. ONNX Runtime для Generative AI (GENAI) — це потужний інструмент, що допомагає ефективно запускати генеративні AI-моделі на різних платформах.

## Що таке ONNX Runtime?
ONNX Runtime — це проєкт з відкритим кодом, що забезпечує високопродуктивний інференс моделей машинного навчання. Він підтримує моделі у форматі Open Neural Network Exchange (ONNX), який є стандартом для представлення моделей машинного навчання. ONNX Runtime дозволяє прискорити досвід користувачів та знизити витрати, підтримуючи моделі з фреймворків глибокого навчання, таких як PyTorch і TensorFlow/Keras, а також класичні бібліотеки машинного навчання, як-от scikit-learn, LightGBM, XGBoost тощо. ONNX Runtime сумісний з різним обладнанням, драйверами та операційними системами, забезпечуючи оптимальну продуктивність за рахунок використання апаратних акселераторів, а також оптимізацій та трансформацій графів.

## Що таке генеративний AI?
Генеративний AI — це системи штучного інтелекту, здатні створювати новий контент, такий як текст, зображення або музика, базуючись на даних, на яких вони були навчені. Прикладами є мовні моделі, як GPT-3, і моделі генерації зображень, такі як Stable Diffusion. Бібліотека ONNX Runtime для GenAI забезпечує цикл генеративного AI для ONNX-моделей, включаючи інференс з ONNX Runtime, обробку логітів, пошук і вибірку, а також управління кешем KV.

## ONNX Runtime для GENAI
ONNX Runtime для GENAI розширює можливості ONNX Runtime для підтримки генеративних AI-моделей. Ось деякі ключові особливості:

- **Широка підтримка платформ:** Працює на різних платформах, включаючи Windows, Linux, macOS, Android і iOS.
- **Підтримка моделей:** Підтримує багато популярних генеративних AI-моделей, таких як LLaMA, GPT-Neo, BLOOM та інші.
- **Оптимізація продуктивності:** Включає оптимізації для різних апаратних акселераторів, таких як NVIDIA GPUs, AMD GPUs тощо.
- **Простота використання:** Надає API для легкої інтеграції в додатки, дозволяючи генерувати текст, зображення та іншій контент з мінімальним кодом.
- Користувачі можуть викликати високорівневий метод generate(), або запускати кожну ітерацію моделі в циклі, генеруючи по одному токену за раз і, за потреби, оновлюючи параметри генерації всередині циклу.
- ONNX Runtime також підтримує жадібний/променевий пошук і вибірку TopP, TopK для генерації послідовностей токенів, а також вбудовану обробку логітів, як-от покарання за повторення. Ви також можете легко додавати власні оцінки.

## Початок роботи

```Python
pip install onnxruntime
```
### Встановіть розширення для генеративного AI:
```Python

```

### Запустіть модель: Ось простий приклад на Python:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### Демонстрація: Використання ONNX Runtime GenAI для виклику Phi-3.5-Vision


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
