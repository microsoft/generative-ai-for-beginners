# Налаштування у хмарі ☁️ – GitHub Codespaces

**Використовуйте цей посібник, якщо не хочете нічого встановлювати локально.**  
Codespaces надає безкоштовний браузерний екземпляр VS Code з усіма попередньо встановленими залежностями.

---

## 1. Чому Codespaces?

| Перевага | Що це означає для вас |
|---------|----------------------|
| ✅ Без встановлень | Працює на Chromebook, iPad, комп’ютерах комп’ютерного класу… |
| ✅ Попередньо налаштований контейнер розробника | Python 3, Node.js, .NET, Java вже всередині |
| ✅ Безкоштовна квота | Особисті акаунти отримують **120 ядер-годин / 60 ГБ-годин на місяць** |

> 💡 **Порада**  
> Тримайте вашу квоту в порядку, **зупиняючи** або **видаляючи** неактивні codespaces  
> (Перейдіть до ▸ Command Palette ▸ *Codespaces: Stop Codespace*).

---

## 2. Створення Codespace (одним кліком)

1. **Форкніть** цей репозиторій (кнопка **Fork** зверху справа).  
2. У вашому форку натисніть **Code ▸ Codespaces ▸ Create codespace on main**.  
   ![Діалогове вікно з кнопками для створення codespace](../../../translated_images/uk/who-will-pay.4c0609b1c7780f44.webp)

✅ Відкривається вікно VS Code у браузері, і починається збірка контейнера розробника.
Це займе **приблизно 2 хвилини** вперше.

## 3. Додайте свій API ключ (безпечний спосіб)

### Варіант A Секрети Codespaces — Рекомендовано

1. ⚙️ Іконка шестерні -> Command Palette -> Codespaces : Manage user secret -> Додати новий секрет.
2. Ім’я: OPENAI_API_KEY
3. Значення: вставте ваш ключ → Додати секрет

Ось і все — наш код автоматично його підхопить.

### Варіант B Файл .env (якщо дуже потрібно)

```bash
cp .env.copy .env
code .env         # заповніть OPENAI_API_KEY=your_key_here
```
