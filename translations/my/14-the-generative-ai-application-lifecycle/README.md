# Generative AI အပလီကေးရှင်း အသက်တာလမ်းကြောင်း

AI အပလီကေးရှင်းအားလုံးအတွက် အရေးကြီးသောမေးခွန်းမှာ AI လက္ခဏာများ၏ သက်ဆိုင်မှုဖြစ်ပြီး၊ AI သည် အလျင်အမြန်တိုးတက်နေသောနယ်ပယ်ဖြစ်သောကြောင့် သင့်အပလီကေးရှင်းသည် သက်ဆိုင်မှုရှိ၊ ယုံကြည်စိတ်ချရ၊ နှင့် ခိုင်မာမှုရှိနေစေရန် အဆက်မပြတ် စောင့်ကြည့်၊ အကဲဖြတ်၊ နှင့် တိုးတက်အောင်လုပ်ဆောင်ရမည်ဖြစ်သည်။ ဒီမှာ Generative AI အသက်တာလမ်းကြောင်း ပါဝင်သည်။

Generative AI အသက်တာလမ်းကြောင်းသည် Generative AI အပလီကေးရှင်းတစ်ခု ဖန်တီး၊ တပ်ဆင်၊ နှင့် ထိန်းသိမ်းခြင်းအဆင့်များကို လမ်းညွှန်ပေးသော ဖရိမ်ဝတ်ဖြစ်သည်။ သင့်ရည်မှန်းချက်များကို သတ်မှတ်ရန်၊ စွမ်းဆောင်ရည်ကို တိုင်းတာရန်၊ စိန်ခေါ်မှုများကို ဖော်ထုတ်ရန်၊ နှင့် ဖြေရှင်းချက်များကို အကောင်အထည်ဖော်ရန် ကူညီပေးသည်။ ထို့အပြင် သင့်အပလီကေးရှင်းကို သင့်နယ်ပယ်နှင့် ပါဝင်သူများ၏ ကျင့်ဝတ်နှင့် ဥပဒေရေးရာစံနှုန်းများနှင့် ကိုက်ညီစေရန်လည်း ကူညီပေးသည်။ Generative AI အသက်တာလမ်းကြောင်းကို လိုက်နာခြင်းဖြင့် သင့်အပလီကေးရှင်းသည် အမြဲတမ်း တန်ဖိုးပေးနေပြီး အသုံးပြုသူများကို စိတ်ကျေနပ်မှုရရှိစေသည်။

## နိဒါန်း

ဤအခန်းတွင် သင်သည် -

- MLOps မှ LLMOps သို့ Paradigm Shift ကို နားလည်မည်
- LLM အသက်တာလမ်းကြောင်း
- အသက်တာလမ်းကြောင်း ကိရိယာများ
- အသက်တာလမ်းကြောင်း တိုင်းတာခြင်းနှင့် အကဲဖြတ်ခြင်း

## MLOps မှ LLMOps သို့ Paradigm Shift ကို နားလည်ခြင်း

LLM များသည် Artificial Intelligence လက်နက်အစုအဝေးတွင် အသစ်သောကိရိယာဖြစ်ပြီး၊ အပလီကေးရှင်းများအတွက် ခွဲခြမ်းစိတ်ဖြာခြင်းနှင့် ဖန်တီးခြင်းလုပ်ငန်းများတွင် အလွန်အစွမ်းထက်သည်။ သို့သော် ဤအစွမ်းသည် AI နှင့် Classic Machine Learning လုပ်ငန်းများကို စနစ်တကျ လုပ်ဆောင်ရာတွင် အကျိုးဆက်တချို့ရှိသည်။

ဒါကြောင့် ဤကိရိယာကို တိကျသော အားပေးမှုများနှင့်အတူ ဒိုင်နမစ်စနစ်ဖြင့် လိုက်လျောညီထွေဖြစ်စေရန် Paradigm အသစ်တစ်ခု လိုအပ်သည်။ အဟောင်း AI အပလီကေးရှင်းများကို "ML Apps" ဟုခွဲခြားနိုင်ပြီး၊ အသစ် AI အပလီကေးရှင်းများကို "GenAI Apps" သို့မဟုတ် "AI Apps" ဟုခေါ်ဆိုနိုင်သည်၊ ထိုအချိန်တွင် အသုံးပြုသော နည်းပညာနှင့် နည်းလမ်းများကို ပြသသည်။ ၎င်းသည် ကျွန်ုပ်တို့၏ စကားဝိုင်းကို အမျိုးမျိုးနည်းဖြင့် ပြောင်းလဲစေသည်၊ အောက်ပါ နှိုင်းယှဉ်ချက်ကို ကြည့်ပါ။

![LLMOps vs. MLOps comparison](../../../translated_images/my/01-llmops-shift.29bc933cb3bb0080.png)

LLMOps တွင် အဓိကထားသည်မှာ အပလီကေးရှင်း ဖန်တီးသူများဖြစ်ပြီး၊ အဓိကအချက်အနေနှင့် ပေါင်းစည်းမှုများကို အသုံးပြုခြင်း၊ "Models-as-a-Service" ကို အသုံးပြုခြင်းနှင့် အောက်ပါအချက်များအတွက် တိုင်းတာမှုများကို စဉ်းစားခြင်းဖြစ်သည်။

- အရည်အသွေး: တုံ့ပြန်မှုအရည်အသွေး
- ထိခိုက်မှု: တာဝန်ရှိသော AI
- ရိုးသားမှု: တုံ့ပြန်မှု၏ အခြေခံမှု (အဓိပ္ပါယ်ရှိသလား? မှန်ကန်သလား?)
- ကုန်ကျစရိတ်: ဖြေရှင်းချက် ဘတ်ဂျက်
- နောက်ကျမှု: တုံ့ပြန်မှုအတွက် ပျမ်းမျှ အချိန်

## LLM အသက်တာလမ်းကြောင်း

ပထမဦးဆုံး အသက်တာလမ်းကြောင်းနှင့် ပြင်ဆင်မှုများကို နားလည်ရန် အောက်ပါ အချက်အလက်ပုံကို မှတ်သားပါ။

![LLMOps infographic](../../../translated_images/my/02-llmops.70a942ead05a7645.png)

သင်တွေ့မည်မှာ MLOps ၏ ပုံမှန် အသက်တာလမ်းကြောင်းများနှင့် ကွာခြားသည်။ LLM များတွင် Prompting, အရည်အသွေးတိုးတက်စေရန် နည်းလမ်းများ (Fine-Tuning, RAG, Meta-Prompts), တာဝန်ရှိသော AI နှင့် ပတ်သက်သော အကဲဖြတ်မှုများ၊ နောက်ဆုံးတွင် အသစ်သော တိုင်းတာမှုများ (အရည်အသွေး၊ ထိခိုက်မှု၊ ရိုးသားမှု၊ ကုန်ကျစရိတ်နှင့် နောက်ကျမှု) ပါဝင်သည်။

ဥပမာအားဖြင့်၊ ကျွန်ုပ်တို့ စဉ်းစားပုံကို ကြည့်ပါ။ Hypothesis မှန်ကန်နိုင်မလား စမ်းသပ်ရန် အမျိုးမျိုးသော LLM များနှင့် စမ်းသပ်ရန် prompt engineering ကို အသုံးပြုသည်။

ဤသည်သည် တန်းတူမဟုတ်ပဲ ပေါင်းစပ်ထားသော လည်ပတ်မှုများဖြစ်ပြီး၊ အကြိမ်ကြိမ် ပြန်လည်လုပ်ဆောင်ခြင်းနှင့် အထွေထွေ လည်ပတ်မှုတစ်ခုဖြစ်သည်။

အဆင့်များကို မည်သို့ ရှာဖွေမည်နည်း? အသက်တာလမ်းကြောင်း တည်ဆောက်ပုံကို အသေးစိတ် ကြည့်ကြရအောင်။

![LLMOps Workflow](../../../translated_images/my/03-llm-stage-flows.3a1e1c401235a6cf.png)

ဤသည်သည် အနည်းငယ်ရှုပ်ထွေးသည့်အတိုင်း မြင်ရနိုင်သော်လည်း ပထမဦးဆုံး အကြီးစား အဆင့်သုံးခုကို အာရုံစိုက်ကြည့်ပါ။

1. စဉ်းစားခြင်း/ရှာဖွေခြင်း: ရှာဖွေခြင်း၊ ဤနေရာတွင် စီးပွားရေးလိုအပ်ချက်များအရ ရှာဖွေနိုင်သည်။ Prototype ဖန်တီးခြင်း၊ [PromptFlow](https://microsoft.github.io/promptflow/index.html?WT.mc_id=academic-105485-koreyst) တည်ဆောက်ခြင်းနှင့် Hypothesis အတွက် ထိရောက်မှုရှိမရှိ စမ်းသပ်ခြင်း။
1. တည်ဆောက်ခြင်း/တိုးချဲ့ခြင်း: အကောင်အထည်ဖော်ခြင်း၊ ယခုအခါ၊ ကြီးမားသော ဒေတာစုများအတွက် အကဲဖြတ်ခြင်း၊ Fine-tuning နှင့် RAG ကဲ့သို့သော နည်းလမ်းများကို အသုံးပြု၍ ဖြေရှင်းချက်၏ ခိုင်မာမှုကို စစ်ဆေးသည်။ မအောင်မြင်ပါက ပြန်လည်တည်ဆောက်ခြင်း၊ လမ်းကြောင်းသစ်ထည့်ခြင်း သို့မဟုတ် ဒေတာကို ပြန်လည်စီမံခြင်းကူညီနိုင်သည်။ လမ်းကြောင်းနှင့် အရွယ်အစားကို စမ်းသပ်ပြီး Metrics များကို စစ်ဆေးပါက နောက်အဆင့်အတွက် ပြင်ဆင်ပြီးဖြစ်သည်။
1. လည်ပတ်မှုဆောင်ရွက်ခြင်း: ပေါင်းစည်းခြင်း၊ ယခုအခါ စနစ်တွင် စောင့်ကြည့်မှုနှင့် သတိပေးစနစ်များ ထည့်သွင်းခြင်း၊ တပ်ဆင်ခြင်းနှင့် အပလီကေးရှင်းနှင့် ပေါင်းစည်းခြင်း။

ထို့နောက် စီမံခန့်ခွဲမှု၏ အထွေထွေ လည်ပတ်မှုရှိပြီး လုံခြုံရေး၊ လိုက်နာမှုနှင့် အုပ်ချုပ်မှုကို အာရုံစိုက်သည်။

ဂုဏ်ပြုပါတယ်၊ ယခု သင့် AI အပလီကေးရှင်း အသင့်ဖြစ်ပြီး လည်ပတ်နိုင်ပါပြီ။ လက်တွေ့အတွေ့အကြုံအတွက် [Contoso Chat Demo](https://nitya.github.io/contoso-chat/?WT.mc_id=academic-105485-koreys) ကို ကြည့်ပါ။

ယခု၊ မည်သည့်ကိရိယာများကို အသုံးပြုနိုင်မလဲ?

## အသက်တာလမ်းကြောင်း ကိရိယာများ

ကိရိယာများအတွက် Microsoft သည် [Azure AI Platform](https://azure.microsoft.com/solutions/ai/?WT.mc_id=academic-105485-koreys) နှင့် [PromptFlow](https://microsoft.github.io/promptflow/index.html?WT.mc_id=academic-105485-koreyst) ကို ပံ့ပိုးပေးပြီး သင့်အသက်တာလမ်းကြောင်းကို လွယ်ကူစွာ အကောင်အထည်ဖော်နိုင်စေသည်။

[Azure AI Platform](https://azure.microsoft.com/solutions/ai/?WT.mc_id=academic-105485-koreys) သည် [AI Studio](https://ai.azure.com/?WT.mc_id=academic-105485-koreys) ကို အသုံးပြုခွင့်ပြုသည်။ AI Studio သည် မော်ဒယ်များ၊ နမူနာများနှင့် ကိရိယာများကို ရှာဖွေစူးစမ်းနိုင်သော ဝက်ဘ်ပေါ်တယ်ဖြစ်သည်။ သင့်ရင်းမြစ်များကို စီမံခန့်ခွဲခြင်း၊ UI ဖန်တီးမှု လမ်းကြောင်းများနှင့် SDK/CLI ရွေးချယ်မှုများဖြင့် Code-First ဖွံ့ဖြိုးတိုးတက်မှုကို ထောက်ပံ့သည်။

![Azure AI possibilities](../../../translated_images/my/04-azure-ai-platform.80203baf03a12fa8.png)

Azure AI သည် သင့် လည်ပတ်မှုများ၊ ဝန်ဆောင်မှုများ၊ စီမံကိန်းများ၊ vector ရှာဖွေရေးနှင့် ဒေတာဘေ့စ်လိုအပ်ချက်များကို စီမံခန့်ခွဲရန် အရင်းအမြစ်များစွာကို အသုံးပြုခွင့်ပြုသည်။

![LLMOps with Azure AI](../../../translated_images/my/05-llm-azure-ai-prompt.a5ce85cdbb494bdf.png)

Proof-of-Concept (POC) မှ စ၍ ကြီးမားသော အပလီကေးရှင်းများအထိ PromptFlow ဖြင့် တည်ဆောက်ပါ။

- VS Code မှ အပလီကေးရှင်းများကို ဒီဇိုင်းဆွဲပြီး ဖန်တီးပါ၊ မြင်သာသောနှင့် လုပ်ဆောင်နိုင်သော ကိရိယာများဖြင့်
- အရည်အသွေးမြင့် AI အတွက် အပလီကေးရှင်းများကို စမ်းသပ်ပြီး fine-tune လုပ်ပါ၊ လွယ်ကူစွာ
