# မူလက်န့်သင်ကြားမှုအတွက် ဗီးရှင်းစာပေးစစွမ်းရည်ရှိသည့် အသေးစားဘာသာစကားမော်ဒယ်များအား မိတ်ဆက်ခြင်း
မျိုးရိုး AI သည် အသစ်သော အကြောင်းအရာများ ဖန်တီးနိုင်သော စနစ်များ ဖန်တီးရေးတွင် အာရုံစိုက်ထားသည့် စိတ်ဝင်စားဖွယ် လူ့ဗဟုသုတများဖြစ်ပါတယ်။ ဒီအကြောင်းအရာတွေမှာ စာသား၊ ဓာတ်ပုံ၊ ဂီတနဲ့ စီးရီးအဖြစ် အပြည့်အစုံ ကျပြည့်သော အွန်လိုင်းပတ်ဝန်းကျင်များ ပါဝင်နိုင်ပါတယ်။ မျိုးရိုး AI ၏ အထူးစိတ်လှုပ်ရှားဖွယ် အသုံးချမှုများမှ တစ်ခုမှာ ဘာသာစကားမော်ဒယ်များဖြစ်ပါတယ်။

## အသေးစားဘာသာစကားမော်ဒယ်များ ဆိုတာဘာလဲ?

အသေးစားဘာသာစကားမော်ဒယ်တစ်ခု (SLM) သည် ကြီးမားသောဘာသာစကားမော်ဒယ် (LLM) ၏ အတိုချုပ်ဗားရှင်းတစ်ခုဖြစ်ပြီး LLM များ၏ အဆောက်အအုံသဘောတရားများနှင့် နည်းလမ်းများကို အသုံးပြုကာ ကွန်ပျူတာ တိုးတက်မှုအသုံးပြုမှုကို အလွန်လျော့ပါးစေသည်။

SLM များသည် လူ့စကားပြောဝါသနာတူ စာသားများ ဖန်တီးရန် ရည်ရွယ်ထားသော ဘာသာစကားမော်ဒယ်များဖြစ်သည်။ GPT-4 စသည့် ကြီးမားသော မော်ဒယ်များထက် လျှော့နည်းပြီး ထိထိရောက်ရောက် ဖြစ်သောကြောင့် ကွန်ပျူတာနိုင်ငံ့စွမ်းအားကန့်သတ်ထားသည့် အထူးလုပ်ဆောင်မှုပိုင်းတွင် သင့်တော်သည်။ မော်ဒယ်အသေးစား သော်လည်း မျိုးစုံသော အလုပ်များ ဆောင်ရွက်နိုင်သည်။ SLM များကို အသုံးပြုရန် LLM များကို စုတ်တောက်ခြင်း သို့မဟုတ် စုပေါင်းခြင်းဖြင့် တည်ဆောက်ပြီး မူလ မော်ဒယ်၏ လုပ်ဆောင်နိုင်မှုနှင့် ဘာသာစကားဆိုင်ရာ စွမ်းရည်များကို မျှတစွာ ထိန်းသိမ်းထားရန် ရည်ရွယ်သည်။ ဤမော်ဒယ်အသေးစားမှုသည် မော်ဒယ်ထဲတွင် တည်ရှိမှု အတိုင်းအလိမ်သက်သာစေပြီး မှတ်ဉာဏ်အသုံးပြုမှုနှင့် ကွန်ပျူတာလိုအပ်ချက်တို့တွင် ထိထိရောက်ရောက် ဖြစ်စေသည်။ ဤပြုပြင်ပြောင်းလဲမှုများပြီးနောက် SLM များသည် သဘာဝဘာသာစကားပြုပြင်ရေး (NLP) လုပ်ငန်းများ မျိုးစုံ ဆောင်ရွက်နိုင်ပါသည်။

- စာသားဖန်တီးခြင်း: ဆက်စပ်မှုရှိသော စာကြောင်းများ သို့မဟုတ် စာပိုဒ်များ ဖန်တီးခြင်း။
- စာသားပြည့်စုံခြင်း: ပေးအပ်ထားသည့် စာကြောင်းပေါ်တွင် ခန့်မှန်းပြီး ဖြည့်စွက်ခြင်း။
- ဘာသာပြောင်းခြင်း: တစ်ဘာသာမှ အခြားဘာသာသို့ စာသားပြောင်းခြင်း။
- အကျဥ်းချုပ်ရေးခြင်း: ရှည်လျားသော စာသားများကို ချုပ်ဆည်းပြီး နားလည်ရလွယ်သော အကျဥ်းချုပ်များ ဖန်တီးခြင်း။

ကြီးမားသော မော်ဒယ်များနှင့် နှိုင်းယှဉ်လျှင် လုပ်ဆောင်ချက်အကြီးအကျယ် သို့မဟုတ် နက်နဲသော နားလည်မှု ပြည့်စုံမှုတွင် တစ်ချို့ ကန့်သတ်ချက်များ ရှိနိုင်သည်။

## အသေးစားဘာသာစကားမော်ဒယ်များ ဘယ်လို အလုပ်လုပ်သလဲ?
SLM များကို စာသား ဒေတာအကြွင်းအကျန်များ အလွန်အမင်း စုဆောင်း၍ လေ့ကျင့်သည်။ လေ့ကျင့်မှုအတွင်း ဘာသာစကား၏ ဓလေ့စနစ်များနှင့် ဖွဲ့စည်းမှုများကို သင်ယူကာ၊ စာသားများကို အခြေအနေသင့်တော်ပြီး သရုပ်ဆောင်မှုပိုင်းတွင် မှန်ကန်စေရန် စွမ်းရည် ရရှိသည်။ လေ့ကျင့်ခြင်း လုပ်ငန်းစဉ်မှာ -

- ဒေတာစုဆောင်းခြင်း: အမျိုးမျိုးသောရင်းမြစ်များမှ စာသား ဒေတာစုဆောင်းခြင်း။
- ကြိုတင်ပြင်ဆင်ခြင်း: လေ့ကျင့်မှုအတွက် အသင့်တော်ဆုံးအခြေအနေဖြစ်စေရန် ဒေတာ သန့်ရှင်းခြင်းနှင့် စီမံခြင်း။
- လေ့ကျင့်ခြင်း: စက်မှုသင်ကြားမှုအယ်လဂိုရစ်များအသုံးပြုကာ မော်ဒယ်ကို စာသားနားလည်ခြင်းနှင့် ဖန်တီးခြင်း သင်ကြားရန်။
- လေ့ကျင့်မှုတိုးချဲ့ခြင်း: အထူးလုပ်ငန်းများတွင် လုပ်ဆောင်ချက်မြှင့်တင်ရန် မော်ဒယ်ကို ချိန်ညှိခြင်း။

SLM များ ဖန်တီးမှုသည် ရင်းမြစ်ကန့်သတ်သော ပတ်ဝန်းကျင်များ (အထူးသဖြင့် မိုဘိုင်းစက်များ သို့မဟုတ် နယ်နိမိတ်အွန်လိုင်း စနစ်များ) တွင် ကျယ်ပြန့်စွာ ထူထောင်ရန် လိုအပ်မှုတိုးပွားနေခြင်းနှင့် ကိုက်ညီသည်။ အကျိုးသက်ရောက်မှု အပြည့်အဝဖြစ်စေမှုကို ဦးစားပေး၍ SLM များသည် မျိုးစုံ အကွာအဝေးများတွင် အသုံးချနိုင်တယ်။

## သင်ယူရေးရာ ရည်မှန်းချက်များ

ဤသင်ခန်းစာတွင် SLM တွေကို မိတ်ဆက်ကာ Microsoft Phi-3 နှင့်ပေါင်းစပ်၍ စာသား၊ မြင်ကွင်းနှင့် MoE များဆိုင်ရာ အခြေအနေများသင်ယူမည်ဖြစ်သည်။

သင်ခန်းစာအဆုံးသတ်တွင် မေးခွန်းများကို ဖြေဆိုနိုင်သင့်သည်မှာ -

- SLM ဆိုတာဘာလဲ?
- SLM နှင့် LLM ကြားက မတူညီမှုများ ဘာတွေလဲ?
- Microsoft Phi-3/3.5 များကို ဘာလဲ?
- Microsoft Phi-3/3.5 များဖြင့် ဘယ်လို အမြင်အာရုံရယူမလဲ?

အသင့်ဖြစ်ပါပြီလား? စတင်ကြရအောင်။

## ကြီးမားသော ဘာသာစကားမော်ဒယ်များ (LLMs) နှင့် အသေးစားဘာသာစကားမော်ဒယ်များ (SLMs) ၏ ကွာခြားချက်များ

LLM များနှင့် SLM များစလုံးသည် ကုန်ကျစရိတ်မဲ့ စက်မှုသင်ကြားမှု နည်းစနစ်များအပေါ် အခြေခံပြီး တူညီသော အဆောက်အအုံဒီဇိုင်း၊ လေ့လာရေးနည်းလမ်းများ၊ ဒေတာဖန်တီးမှုနည်းလမ်းများနှင့် မော်ဒယ်အကဲဖြတ်နည်းများကို လိုက်နာသည်။ သို့သော် မော်ဒယ်နှစ်မျိုးတွင် အရေးကြီးသော ကွဲပြားမှု အချို့ ရှိသည်။

## အသေးစားဘာသာစကားမော်ဒယ်များ၏ သုံးစွဲမှုများ

SLM များနှင့် အများကြီးသော လုပ်ဆောင်ချက်များပါဝင်သည် -

- စကားပြောစနစ်များ: အသုံးပြုသူများနှင့် ဆက်သွယ်ဆက်ဆံခြင်းနှင့် ဝန်ဆောင်မှုမှုပေးခြင်း။
- အကြောင်းအရာ ဖန်တီးခြင်း: စာရေးသူများအား စိတ်ကူးများ စုဆောင်းကူညီခြင်း သို့မဟုတ် အပိုင်းအစများ ရေးသားပေးခြင်း။
- ပညာရေး: ကျောင်းသားများအား စာရေးအလုပ်များ သို့မဟုတ် အသစ်စကားများ သင်ယူရန် ကူညီခြင်း။
- ပေါ့ပါးမှု: မသန်စွမ်းသူများအတွက် စာသားမှ စကားပြောင်းစနစ်များ ဖန်တီးခြင်း။

**အရွယ်အစား**
  
LLM များနှင့် SLM များကြား အဓိက ကွာခြားချက်မှာ မော်ဒယ်၏ ပမာဏဖြစ်သည်။ ChatGPT (GPT-4) ကဲ့သို့ LLM များတွင် ၁.၇၆တရေသန်း parameters များရှိနိုင်ပြီး Mistral 7B ကဲ့သို့ ဖွင့်ထားသော SLM မော်ဒယ်များတွင် ဆယ်ပေါင်း သန်း ၇ ခန့်ရှိသော parameters သာ ပါဝင်သည်။ ဤကွာဟချက်သည် မော်ဒယ်၏ အဆောက်အအုံနှင့် လေ့ကျင့်မှု လုပ်ငန်းစဉ် ကွဲပြားမှုကြောင့် ဖြစ်သည်။ ဥပမာအားဖြင့် ChatGPT သည် encoder-decoder ဖွဲ့စည်းမှုတွင် self-attention ကို အသုံးပြုသော်လည်း Mistral 7B သည် decoder-only မော်ဒယ်ဖြစ်ပြီး sliding window attention ဖြင့် သင်ယူမှု ထိရောက်စေသည်။ ဤဖွဲ့စည်းမှုကွဲပြားမှုသည် မော်ဒယ် ရှုပ်ထွေးမှုနှင့် ဖျော်ဖြေပေးနိုင်မှု ပမာဏကို အလွန်သက်ရောက်စေသည်။

**နားလည်မှု**

SLM များသည် အထူးလုပ်ငန်းကဏ္ဍများအတွက် ထိရောက်မှုအတွက် အာရုံစိုက်ထားခြင်းကြောင့် အကျယ်ရပ်ဝန်းများကို စုံလင်စွာ နားလည်မှု ပေးနိုင်မှု ကန့်သတ်ပါသည်။ အခြားဖက်တွင် LLM များသည် လူ့ဗဟုသုတကို အလွန်ကျယ်ပြန့်စွာ နှိပ်စက်ရန် ရည်ရွယ်ခဲ့ပြီး အမျိုးမျိုးသော ဒေတာများဖြင့် လေ့ကျင့်ထားသည်။ ထိုကြောင့် LLM များသည် သဘာဝဘာသာစကားပြုပြင်ရေးနှင့် ပရိုဂရမ်မင်း ပြုလုပ်ခြင်း လုပ်ငန်းများတွင် ပိုမိုကောင်းမွန်ပြီး ပိုမို အဆင်ပြေအောင် ကြော်ငြာပြုလုပ်နိုင်သည်။

**ကွန်ပျူတာထောက်ပံ့မှု**

LLM များ၏ လေ့ကျင့်ခြင်းနှင့် အသုံးပြုခြင်းမှာ အရင်းအမြစ်များ ပြင်းထန်သော လုပ်ငန်းများ ဖြစ်ပြီး ကြီးမားသော GPU cluster များ လိုအပ်သည်။ ဥပမာအားဖြင့် ChatGPT မော်ဒယ်ကို မူလစတင်လေ့ကျင့်ရန် GPU များ အနှစ်ချုပ် ရှာဖွေထားရမည်။ SLM များမှာ parameters များနည်းသည့်ကြောင့် လုပ်ငန်းမြောက်က လျော့နည်းပြီး GPU အနည်းငယ်ပါသော စက်သုံး မော်ဒယ်များတွင် ထည့်သွင်း အသုံးပြုနိုင်သည်။ Mistral 7B ကို မည်သည့်အခါ training လုပ်လည်း GPU များအတော်များများနှင့် တစ်ချိန်တည်းတွင် နာရီအနည်းငယ် ဆောင်ရွက်ရမည်ဖြစ်သည်။

**ပိုင်းခြားမှု**

LLM များတွင် bias ဖြစ်ပေါ်မှုကို training data များ၏သဘာဝကြောင့် ဖြစ်ရှင်းသည်။ အင်တာနက်မှ အဆင့်မပြည့်ဝသော ဒေတာများ အသုံးပြုခြင်းကြောင့် အချိန်အခါ မှားယွင်းသည့် နာမည်မှတ်တိုင်များ ပါဝင်နိုင်ပြီး ဘာသာစကားနှင့် ဒေသဆိုင်ရာ ထူးခြားချက်များကိုလည်း ထိခိုက်စေသည်။ ထို့ပြင် LLM ၏ ရှုပ်ထွေးသော ဖွဲ့စည်းမှုသည် bias များ ပိုတိုးလေ့ ရှိပြီး ကြည့်ရှုလေ့လာမှုမရှိလျှင် မသိနိုင်ပါ။ တခြားဘက်တွင် SLM များသည် တိကျသော အပိုင်းအခြားဒေတာများ အပေါ်မှာ ရေရှည်လေ့လာမှုပြု၍ bias များ လျော့နည်းသည်၊ သို့သော် ကင်းလွတ်သည် မဟုတ်ပါ။

**ထင်မြင်ချက်ရယူခြင်း**

SLM များ၏ အသေးစားမှုကြောင့် inference အမြန်နှုန်းတွင် အရေးကြီးသော အားသာချက် ရရှိသည်။ ဒါကြောင့် ကိုယ်ပိုင်စက်များတွင် ထိထိရောက်ရောက် ထုတ်လွှတ်ခြင်း ပြုလုပ်နိုင်ပြီး များသောအချိန် ပဲလမ်းညွှန်ဆုံးမအဖြစ် အပြိုင် CPU/GPU များ မလိုအပ်ပါ။ LLM များသည် မော်ဒယ်အရွယ်အစားနဲ့ ရှုပ်ထွေးမှုကြောင့် များသော parallel computing အရင်းအမြစ် လိုအပ်ပြီး တစ်ချိန်တည်း များစွာသော အသုံးပြုသူများ ရှိလျှင် ပြန်စာလျှော့ ခဲ့သည်။

အနှစ်ချုပ်အားဖြင့် LLM များနှင့် SLM များသည် စက်ရုပ်သင်ယူမှု အခြေခံအခြေအနေ ပူးပေါင်းထားသော်လည်း မော်ဒယ်အရွယ်အစား၊ စွမ်းအားလိုအပ်ချက်၊ အကြောင်းအရာနားလည်မှု၊ bias ဖြစ်ပေါ်မှုနှင့် inference အလျင်အမြန်စွာမှာ ကြီးမားစွာ ကွဲပြားသည်။ ဤကွဲပြားချက်များသည် သုံးစွဲမှု ကွာဟချက်နှင့် ကိုက်ညီမှုတို့ကို ဖော်ပြသည်။ LLM များသည် အသုံးပြုနိုင်မှု ပြည့်စုံသော်လည်း အရင်းအမြစ်များ ခက်ခဲသော မော်ဒယ်များ ဖြစ်ပြီး SLM များသည် ဒေသဆိုင်ရာအထူးပြောစကားများတွင် ထိရောက်ပြီး ကွန်ပျူတာအသုံးစရိတ် သက်သာစေရန် ရည်ရွယ်သည်။

***မှတ်ချက်။ ဤသင်ခန်းစာတွင် Microsoft Phi-3 / 3.5 ကိုအသုံးပြုပြီး SLM များကို မိတ်ဆက်သွားမည်ဖြစ်သည်။***

## Phi-3 / Phi-3.5 မျိုးဆက် မိတ်ဆက်ခြင်း

Phi-3 / 3.5 မျိုးဆက်သည် စာသား၊ မြင်ကွင်းနှင့် Agent (MoE) အသုံးပြုမှုများအတွက် အဓိကရည်ရွယ်သည်။

### Phi-3 / 3.5 လမ်းညွှန်မှု

အဓိကအားဖြင့် စာသားဖန်တီးခြင်း၊ စကားပြောပြောင်မြောက်မှုနှင့် အကြောင်းအရာမှ အချက်အလက် ထုတ်ယူခြင်းများအတွက်။

**Phi-3-mini**

3.8B parameter ပါရှိသည့် ဘာသာစကားမော်ဒယ်သည် Microsoft Foundry၊ Hugging Face နှင့် Ollama တွင်ရရှိနိုင်သည်။ Phi-3 မော်ဒယ်များသည် အဓိက စမ်းသပ်ချက်များတွင် ကျယ်ပြန့်သော မော်ဒယ်များနှင့် ညီမျှ သို့မဟုတ် ထက်ပိုသာသည် (အောက်တွင် စမ်းသပ်ထားသော နံပါတ်များ ကြည့်ပါ၊ အမြင့်သော နံပါတ်များက ပိုကောင်းသည်။) Phi-3-mini သည် ၎င်း၏ အရွယ်အစား နှစ်ဆ ပမာဏရှိသော မော်ဒယ်များထက် ပိုကောင်းကြောင်း၊ Phi-3-small နှင့် Phi-3-medium များသည် GPT-3.5 ပါဝင်သော ကြီးမားသော မော်ဒယ်များထက် ပိုကောင်းသည်။

**Phi-3-small & medium**

7B parameter အတွင်း ကျားသုံးသည့် Phi-3-small သည် GPT-3.5T ကို ဘာသာစကား၊ အကြောင်းပြချက်ရှာဖွေရေး၊ ကုဒ်ရေးခြင်းနှင့် သင်္ချာ စမ်းသပ်ချက်များ၌ ရှုံးနိမ့်သည်။

14B parameter ပါရှိသည့် Phi-3-medium သည် ဆက်လက်ဘက်မှာ Gemini 1.0 Pro ကို ကျော်လွန်သည်။

**Phi-3.5-mini**

ပရမတ်တာများ မပြောင်းလဲပေမယ့် ဘာသာစကားစုံလင်စွာ ကူညီနိုင်မှု တိုးတက်စေပြီး (20+ ဘာသာစကားများထောက်ပံ့မှု: အာရဗီ, တရုတ်, ဈျက်, ဒိန်းမပ်ခ်, ဒတ်ချ်, အင်္ဂလိပ်, ဖင်လန်, ပြင်သစ်, ဂျာမဏီ, ဟီဗရူး, ဟန်ဂေရီ, အီတလီ, ဂျပန်, ကိုးရီးယား, နော်ဝေနီး, ပိုလန်, ပေါ်တူဂီ, ရုရှား, စပိန်, စွီဒင်, ထိုင်း, တူရကီ, ယူကရိန်း) ကွက်လပ်ကို ထောက်ပံ့ပြီး ရှည်လျားသောအကြောင်းအရာကို ပိုမိုကောင်းမွန်စွာ ထောက်ပံ့သည်။

3.8B parameter ပါရှိသည့် Phi-3.5-mini သည် တစ်ပမာဏသည်နှင့်ညီမျှ သို့မဟုတ် ဒါမှမဟုတ် အရွယ်အစားနှစ်ဆ ပမာဏရှိသော မော်ဒယ်များနှင့် ညီညွတ်သည်။

### Phi-3 / 3.5 မြင်ကွင်း

Phi-3/3.5 ၏ လမ်းညွှန်မော်ဒယ်ကို Phi ၏ နားလည်မှုစွမ်းရည် ဟု ယူဆနိုင်ပြီး မြင်ကွင်း မော်ဒယ်ဟာ Phi အရဲ့ ကမ္ဘာကို နားလည်ရန် မျက်လုံးထောက်ပံ့မှုဖြစ်သည်။


**Phi-3-Vision**

4.2B parameters ပါဝင်သော Phi-3-vision သည် Claude-3 Haiku နှင့် Gemini 1.0 Pro V ကဲ့သို့သောကြီးမားသော မော်ဒယ်များထက် ယေဘုယျ မြင်ကွင်းအဘိုးကေကွဲ၊ OCR နှင့် ဇယားနဲ့ဇယား နားလည်မှုအလုပ်များ၌ ပိုကောင်းစွာ ဆောင်ရွက်သည်။


**Phi-3.5-Vision**

Phi-3.5-Vision သည် Phi-3-Vision ၏ အဆင့်မြှင့်တင်မှုဖြစ်ပြီး ရုပ်ပုံအမျိုးမျိုး ထောက်ပံ့မှုရှိသည်။ ဤသည်ကို မြင်ကွင်းတိုးတက်မှုဟု ယူဆနိုင်ပြီး ဓာတ်ပုံသာမက ဗီဒီယိုများကိုလည်း မြင်နိုင်သည်။

Phi-3.5-vision သည် Claude-3.5 Sonnet နှင့် Gemini 1.5 Flash ကဲ့သို့သောကြီးမားသော မော်ဒယ်များထက် OCR၊ ဇယားနှင့် အကြမ်းဖျင်းပညာ တိုက်ရိုက်နားလည်မှုအလုပ်များ၌ အကောင်းဆုံးဖြစ်ပြီး ရှေ့တန်း အကြောင်းအရာ သေချာစွာ ပြုလုပ်ခြင်းကို အထောက်အပံ့ ပြုသည်။ များစွာသော ပုံများပေးပို့မှုကိုထောက်ပံ့သည်။


### Phi-3.5-MoE

***Mixture of Experts (MoE)*** သည် လေ့ကျင့်သည့် ကုန်ကျစရိတ်ကို လျော့ချစေပြီး မော်ဒယ် ဒေတာအရွယ်အစားကို တိုးမြှင့်နိုင်ခြင်းကို ဖြစ်စေသည်။ အထူးသဖြင့် MoE မော်ဒယ်သည် dense မော်ဒယ်ကြီး၏ အရည်အသွေးနှင့်အတူ ပိုမြန်သည့် မြန်ဆန်စေသည်။

Phi-3.5-MoE သည် 16x3.8B အထူးတီထွင်မှု မော်ဒူးများ ပါရှိသည်။ 6.6B active parameters ဖြင့် Phi-3.5-MoE သည် ဘာသာစကားနားလည်မှု၊ သင်္ချာ နှင့် အကြောင်းခံချက် ဆိုင်ရာ အရည်အသွေးများ ဖော်ဆောင်သည့် မော်ဒယ်ကြီးများနှင့် ကိုက်ညီသည်။

Phi-3/3.5 မျိုးဆက် မော်ဒယ်ကို အခြေအနေ အမျိုးမျိုးတွင် အသုံးပြုနိုင်သည်။ LLM များနှင့်မတူကာ Phi-3/3.5-mini သို့မဟုတ် Phi-3/3.5-Vision ကို edge စက်များတွင် စတင်အသုံးပြုနိုင်သည်။


## Phi-3/3.5 မျိုးဆက် မော်ဒယ်များကို ဘယ်လို အသုံးပြုမလဲ

**NVIDIA NIM**

မိုဃ်းထဲ Microsoft Foundry မော်ဒယ် catalog မဟုတ်ဘဲ [NVIDIA NIM](https://developer.nvidia.com/nim?WT.mc_id=academic-105485-koreyst) မှတဆင့်လည်း အပြုသဘောဆောင်သော API ခေါ်ယူမှုများ ပြုလုပ်နိုင်သည်။ NVIDIA NIM (NVIDIA Inference Microservices) သည် AI မော်ဒယ်များကို cloud၊ data center နှင့် workstation အပါအဝင် ပတ်ဝန်းကျင် မျိုးစုံတွင် ထိထိရောက်ရောက် တပ်ဆင်ကူညီရန် အရှိန်အဟုန်မြှင့် inference microservices များ စုစည်းမှု ဖြစ်သည်။

NVIDIA NIM ၏ အဓိက အင်္ဂါရပ်များမှာ -

- **တပ်ဆင်ရလွယ်ကူခြင်း:** NIM သည် တစ်ချက်တည်းသော command ဖြင့် AI မော်ဒယ်များ တပ်ဆင်နိုင်ပြီး လက်ရှိ workflow များသို့ လွယ်ကူစွာ ပေါင်းစပ်နိုင်သည်။

- **တိုးတက်မြှင့်တင်ထားသော စွမ်းဆောင်ရည်:** NVIDIA ၏ ကြိုတင် ပြင်ဆင်ထားသော inference engine များဖြစ်သော TensorRT နှင့် TensorRT-LLM ကို အသုံးပြုကာ လျှော့ချထားသော နောက်ကျမှုနှင့် မြင့်မားသော ထုတ်လုပ်မှုကို သေချာစေရန်။
- **တိုးချဲ့နိုင်မှု:** NIM သည် Kubernetes တွင် autoscaling ကို ထောက်ပံ့ပြီး အလုပ်အတော်များကို ထိရောက်စွာ ကိုင်တွယ်နိုင်စေသည်။
- **ဘေးကင်းလုံခြုံမှုနှင့် ထိန်းချုပ်နိုင်မှု:** အဖွဲ့အစည်းများသည် မိမိတို့ရဲ့ ဒေတာများနှင့် အက်ပ်လီကေးရှင်းများကို ကိုယ်ပိုင် စီမံခန့်ခွဲရာ တာဝန်ရှိသော အင်ဖရာစတာခ်ချာတွင် NIM microservices များကို မိမိတို့အလိုအလျောက် 호스팅ခြင်းဖြင့် ထိန်းချုပ်နိုင်သည်။
- **စံချိန် စံညွှန်း API များ:** NIM သည် စက်မှုလက်မှုစံချိန်စံညွှန်း API များကို ပံ့ပိုးပေးကာ chatbots, AI ကူညီသူများကဲ့သို့သော AI application များ တည်ဆောက်ရန် နှင့် ပေါင်းစည်းရန် လွယ်ကူစေသည်။

NIM သည် NVIDIA AI Enterprise ၏ အစိတ်အပိုင်းဖြစ်ပြီး AI မော်ဒယ်များကို NVIDIA GPU မ်ားပေါ်တွင် ထိရောက်စွာ လည်ပတ်စေရန် deployment နှင့် operationalization ကို လွယ်ကူစေသည်။

### Phi-3/3.5 ကို ကိုယ်ပိုင်စက်တွင် လည်ပတ်ခြင်း
Phi-3 သို့မဟုတ် GPT-3 ကဲ့သို့သော ဘာသာစကားမော်ဒယ်များနှင့် ဆက်နွယ်သော Inference ဆိုသည်မှာ ထည့်သွင်းသည့် ကန့်သတ်ချက်အရ ထုတ်လွှတ်သော တုံ့ပြန်ချက်များ သို့မဟုတ် ခန့်မှန်းချက်များကို ပြုလုပ်ခြင်း ဖြစ်သည်။ Phi-3 သို့ prompt သို့မဟုတ်မေးခွန်းတစ်ခု ပေးလိုက်သောအခါ၊ ၎င်းသည် သင်ကြားထားသော ဒေတာကွက် pattern နှင့် ဆက်နွယ်မှုများအား ခွဲခြမ်းစိတ်ဖြာ၍ အလားအလာအမြင့်ဆုံးနှင့် အသုံးဝင်ဆုံး တုံ့ပြန်ချက်ကို ယူဆသည်။

**Hugging Face Transformer**
Hugging Face Transformers သည် သဘာဝဘာသာစကား ပြုလုပ်ခြင်း (NLP) နှင့် ၎င်းနှင့်ဆိုင်သော မက်ရှင်လေ့လာမှု အလုပ်များအတွက် အကျိုးသက်ရောက်မှုမြင့်သော စာကြည့်တိုက်တစ်ခုဖြစ်သည်။ ၎င်းအကြောင်း အချက်အလက်အချို့如下:

1. **ကြိုတင်သင်ယူထားသော မော်ဒယ်များ**: စာသားတန်းသတ်မှတ်ခြင်း၊ အမည်ဖြစ်တယ် ခွဲခြားခြင်း၊ မေးခွန်းဖြေဆိုခြင်း၊ အကျဉ်းချုပ်ခြင်း၊ ဘာသာ ပြန်ခြင်းနှင့် စာသား ဖန်တီးခြင်း တို့အပါအဝင် အလုပ်အတော်များအတွက် အသုံးပြုနိုင်သော သန်းခေါင်များသော ကြိုတင်သင်ယူထားသော မော်ဒယ်များကို ဖ 제공သည်။

2. **ဖ্রိမ်းဝပ်များ အကြား အပြန်အလှန်သဘောတူမှု**: စာကြည့်တိုက်သည် PyTorch, TensorFlow, JAX ကဲ့သို့သော အနက်ရှိုင်းသော သင်ကြားမှု ဖ্রိမ်းဝပ် များစွာကို ထောက်ခံသည်။ ၎င်းကြောင့် မော်ဒယ်ကို ဖ্রိမ်းဝပ်တစ်ခုတွင် သင်ကြားပြီး အခြားဖ্রိမ်းဝပ်၌ အသုံးပြုနိုင်သည်။

3. **မော်ဒယ်အမျိုးမျိုး စွမ်းဆောင်ရည်များ**: NLP ဖြင့်ပင်မဟုတ်ဘဲ Hugging Face Transformers သည် ကွန်ပျူတာမြင်ကွင်း (ဥပမာ၊ ပုံတန်းသတ်မှတ်ခြင်း၊ အရာဝတ္ထု ရှာဖွေခြင်း) နှင့် အသံ စီမံခန့်ခွဲမှု (ဥပမာ၊ စကားသံ အသိအမှတ်ပြုခြင်း၊ အသံတန်းသတ်မှတ်ခြင်း) အလုပ်များကိုလည်း ထောက်ခံသည်။

4. **အသုံးပြုရလွယ်ကူမှု**: စာကြည့်တိုက်သည် မော်ဒယ်များကို လွယ်ကူစွာ ဒေါင်းလုဒ်လုပ်ပြီး fine-tune ပြုလုပ်နိုင်ရန် API များနှင့် ကိရိယာများ ပံ့ပိုးပေးပြီး စတင်သူများနှင့် ကျွမ်းကျင်သူများအတွက်လည်း အဆင်ပြေသည်။

5. **အသိုင်းအဝိုင်းနှင့် အရင်းအမြစ်များ**: Hugging Face သည် အသက်ဝင်ဝါကြီးသော အသိုင်းအဝိုင်းနှင့် ကျယ်ပြန့်သော အချက်အလက်စာတမ်းများ၊ သင်တန်းများ၊ လမ်းညွှန်များကို ပံ့ပိုးပေး၍ အသုံးပြုသူများအတွက် စတင်ခြင်းနှင့် စာကြည့်တိုက်ကို အပြည့်အဝ အသုံးချနိုင်စေသည်။
[ရုံးတော် Documentation](https://huggingface.co/docs/transformers/index?WT.mc_id=academic-105485-koreyst) သို့မဟုတ် ၎င်းတို့၏ [GitHub Repository](https://github.com/huggingface/transformers?WT.mc_id=academic-105485-koreyst)။

**ONNX Runtime for GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) သည် cross-platform inference နှင့် machine-learning training accelerator တစ်ခုဖြစ်သည်။ ONNX Runtime for Generative AI (GENAI) သည် မတူညီသော ပလက်ဖောင်းများပေါ်တွင် စွမ်းဆောင်ရည်မြင့် ပြုလုပ်နိုင်သော generative AI မော်ဒယ်များကို အထောက်အကူပြုသည့် စွမ်းအားကြီး ကိရိယာဖြစ်သည်။

## ONNX Runtime ဆိုတာဘာလဲ?
ONNX Runtime သည် machine learning မော်ဒယ်များ၏ မြန်ဆန်သော inference အသုံးပြုမှုများအတွက် ဖွင့်ဆိုင်းသည့် project ဖြစ်သည်။ ၎င်းသည် Open Neural Network Exchange (ONNX) ဖော်မတ်ရှိ မော်ဒယ်များကို ထောက်ခံပြီး Deep Learning frameworks ဖြစ်သော PyTorch နှင့် TensorFlow/Keras ပေါ်တွင် လည်ပတ်သည့် မော်ဒယ်များကို တိုးမြှင့်ကာ classical machine learning ပစ္စည်းဖြစ်သည့် scikit-learn, LightGBM, XGBoost များအပါအဝင် library များကို လည်း ထောက်ခံသည်။ ONNX Runtime သည် hardware, driver နှင့် operating system မျိုးစုံနှင့် ကိုက်ညီပြီး hardware accelerators ကိုအသုံးပြုခြင်းနှင့် graph optimizations လုပ်ခြင်းတို့ဖြင့် ထိရောက်မှုမြင့်မားစေသည်။

## Generative AI ဆိုတာဘာလဲ?
Generative AI သည် သင်ကြားထားသည့် ဒေတာအရ စာသား၊ ပုံများ သို့မဟုတ် ဂီတသစ်များကို ဖန်တီးနိုင်သော AI စနစ်များဖြစ်သည်။ ဥပမာများမှာ GPT-3 ကဲ့သို့သော ဘာသာစကား မော်ဒယ် များနှင့် Stable Diffusion ကဲ့သို့ ပုံဖန်တီး မော်ဒယ် များ ဖြစ်သည်။ ONNX Runtime for GenAI library သည် ONNX မော်ဒယ်များအတွက် generative AI loop ကို ပံ့ပိုးကာ ONNX Runtime နဲ့ inference, logits ကိုင်တွယ်ခြင်း, ရှာဖွေမှုနှင့် စမ်းသပ်မှု, KV cache စီမံခန့်ခွဲမှုတို့ကို ဖြည့်ဆည်းသည်။

## ONNX Runtime for GENAI
ONNX Runtime for GENAI သည္ ONNX Runtime ၏ စွမ်းဆောင်ရည်များကို ဇာတ်လိုက် AI မော်ဒယ်များ ထောက်ပံ့ရန် တိုးချဲ့ပေးသည်။ အောက်ပါ အဓိက အင်္ဂါရပ်များရှိသည်။

- **ပလက်ဖောင်းများ လက်ခံမှု ကျယ်ပြန့်မှု:** Windows, Linux, macOS, Android နှင့် iOS စသည့် မျိုးစုံသော ပလက်ဖောင်းများပေါ်တွင် လည်ပတ်နိုင်ပါသည်။
- **မော်ဒယ် များကို ထောက်ပံ့ခြင်း:** LLaMA, GPT-Neo, BLOOM စသည့် လူကြိုက်များသော Generative AI မော်ဒယ်များကို ထောက်ပံ့သည်။
- **စွမ်းဆောင်ရည် မြှင့်တင်ခြင်း:** NVIDIA GPU များ၊ AMD GPU များနှင့် အခြား hardware accelerators များအတွက် optimization များ ပါဝင်သည်။
- **အသုံးပြုရ လွယ်ကူမှု:** application များတွင် လွယ်ကူစွာ ပေါင်းစည်းအသုံးပြုနိုင်ရန် API များ ပံ့ပိုးပေးပြီး ဖြစ်စဉ်များစွာ စာသား၊ ပုံများနှင့် အခြားအကြောင်းအရာများ ကို ကိုးကားရန် လွယ်ကူစေသည်။
- အသုံးပြုသူများသည် generate() သို့မဟုတ် မော်ဒယ်၏ အသစ်ထွက် အပိုင်း တစ်ခုချင်းစီကို ဆက်လက် လည်ပတ်စေပြီး တစ်ချက်ချင်း token များ ဖန်တီးနိုင်သည်၊ လိုလားသလို function parameters များကို loop ထဲတွင် ပြင်ဆင်နိုင်သည်။
- ONNX runtime သည် greedy/beam search နှင့် TopP, TopK sampling များကို token sequences ဖန်တီးရန် ၊ repetition penalties ကဲ့သို့သော logits ကိုင်တွယ်မှု built-in ပါရှိသည်။ သင့်လျော်သော custom scoring ကိုလည်း လွယ်ကူစွာ ပေါင်းထည့်နိုင်သည်။

## စတင်အသုံးပြုခြင်း
ONNX Runtime for GENAI နဲ့ စတင်ရန် အောက်ပါအဆင့်များကို လိုက်နာနိုင်သည်-

### ONNX Runtime ထည့်သွင်းခြင်း:
```Python
pip install onnxruntime
```
### Generative AI Extensions ထည့်သွင်းခြင်း:
```Python
pip install onnxruntime-genai
```

### မော်ဒယ် တစ်ခု လည်ပတ်ခြင်း: Python ရာထူးရဲ့ ရိုးရိုး ဥပမာ:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### Demonstration: ONNX Runtime GenAI သုံးပြီး Phi-3.5-Vision ကို ခေါ်ဆိုခြင်း


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
