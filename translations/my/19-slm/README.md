# စတုတ္ထအကြောင်းအရာ မော်ဒယ်သင့္နေရာသစ်များအတွက် ဦးတည်ချက်
စတုတ္ထအကြောင်းအရာ AI သည် ဂျင်နေရတ်စ်စင်ဆာဗ်တင်ဖြစ်သော အတုအယောင်ကွန်ယက်စနစ်များ ဖန်တီးရန် အာရုံစိုက်ထားသည့် စိတ်ဝင်စားဖွယ် အင်္ဂလိပ်စကားဉာဏ်ရည်ကဏ္ဍဖြစ်သည်။ ၎င်းအကြောင်းအရာသည် စာပိုဒ်များ၊ ပုံများ၊ စာရေးစနစ်များနှင့် ပိုပြီး လက်တွေ့ဗirtual ပတ်ဝန်းကျင်များထိ ဖန်တီးနိုင်သည်။ စတုတ္ထအကြောင်းအရာ AI ၏ အထူးစိတ်ဝင်စားဖွယ် အသုံးချမှုတစ်ခုမှာ စကားပြော ရုပ်ရှင် မော်ဒယ်များ၏ ကဏ္ဍတွင် ဖြစ်သည်။

## စတုတ္ထအကြောင်းအရာ မော်ဒယ်သင့္နေရာသစ်များ ဘာလဲ?

စတုတ္ထအကြောင်းအရာ မော်ဒယ်သင့္နေရာသစ် (SLM) သည် သဘာဝ စကားပြော မော်ဒယ်ကြီး (LLM) ၏ အရွယ်အစားလျော့နည်းသည့် မော်ဒယ်တစ်မျိုးဖြစ်ပြီး၊ LLM များ၏ အဆောက်အအုံ အခြေခံနည်းပညာများကို အသုံးပြုပြီး မှန်းဆနိုင်သော ကွန်ပျူတာလုပ်ဆောင်မှု လျော့နည်းသည့် မော်ဒယ်တစ်မျိုးဖြစ်သည်။

SLM များသည် လူကိုဆန်ဆန် စာသားများကို အာရုံစိုက်ဖန်တီးရန် ရည်ရွယ်သော စကားပြောမော်ဒယ်များ၏ စုပုံတစ်ပုဒ်ဖြစ်သည်။ GPT-4 ကဲ့သို့သော မော်ဒယ်ကြီးများနှင့် ကွဲပြားချက်ရှိပြီး၊ SLM များသည် ပိုမိုသေးငယ်ပြည့်စုံ၍ လုပ်ဆောင်မှုပိုမိုထိရောက်စေသည်။ ထိုကြောင့် စွမ်းဆောင်ရည်အရ လွယ်ကူသော ပစ္စည်းပေးသောနေရာများတွင် အသုံးပြုရန် သင့်တော်သည်။ ထို့ကြောင့် SLM များသည် LLM မော်ဒယ်များမှ ဖိနှိပ်ခြင်း သို့မဟုတ် ပြန်ဆင့်ဟန်ခြင်းဖြင့် တည်ဆောက်ထားပြီး မူလမော်ဒယ်၏ လုပ်ဆောင်မှုနှင့် ဘာသာစကားနိဒါန်းမှုများကို အများပြား ထိန်းသိမ်းရန် ရည်ရွယ်သည်။ မော်ဒယ်အရွယ်အစားလျော့ခြင်းသည် စုစုပေါင်း တိုးတက်မှုကို ကန့်သတ်ပြီး စွမ်းဆောင်ရည်နှင့် သတိပြုမှုတို့တွင် ထိရောက်မှုမရှိစေရန်ပြုလုပ်သည်။ ထို့ကြောင့် SLM များသည် သဘာဝဘာသာစကား ပြုစုပျိုးထောင်မှု (NLP) အလုပ်ရပ်များစွာကို ဆောင်ရွက်နိုင်သည်။

- စာသားဖန်တီးခြင်း: သရုပ်မှန်တိကျပြီး အတည်ပြုမှုရှိသော စာပိုဒ်များ ဖန်တီးခြင်း။
- စာသားပြည့်စုံခြင်း: စာပိုဒ်များကို မူလစာသားအပေါ် အခြေပြု ကန့်သတ်ပြီး အတည်ပြုခြင်း။
- ဘာသာပြန်ခြင်း: စာသားကို တစ်ဘာသာမှ တစ်ဘာသာသို့ ပြောင်းလဲခြင်း။
- ချုပ်ဆိုခြင်း: ကျယ်ပြန့်သော စာသားများကို ကိုယ့်သဘောအတိုချုပ်များသို့ လျှော့ချခြင်း။

မော်ဒယ်ကြီးများနှင့် နှိုင်းယှဉ်တွဲကြည့်သောအခါ စွမ်းဆောင်ရည် သို့မဟုတ် နက်ရှိုင်းချက်တွင် အနည်းငယ် လျော့နည်းမှုရှိနိုင်သည်။

## စတုတ္ထအကြောင်းအရာ မော်ဒယ်သင့္နေရာသစ် မည်သို့ လုပ်ဆောင်သနည်း?
SLM များကို စာသားဒေတာ များစွာနှင့် သင်ကြားပေးကြသည်။ သင်ကြားသောအခါ စကား၏ ပုံစံနှင့် ဖွဲ့စည်းပုံများကို သင်ယူကာ သဒ္ဒါမှန်ပုံမှန်သာများဖြင့် ထုတ်ဖော်နိုင်သည်။ သင်ကြားရေး လုပ်ငန်းစဉ်တွင် အောက်ပါအချက်များ ပါဝင်သည်။

- ဒေတာ စုဆောင်းခြင်း: အမျိုးမျိုးသောရင်းမြစ်များမှ စာသားဒေတာအများကြီးစုဆောင်းခြင်း။
- အသုံးပြုရန် ပြင်ဆင်ခြင်း:  သင်ကြားရန်သင့်လျော်သောအနေအထားသို့ ဒေတာကို သန့်ရှင်းစီမံခြင်း။
- သင်ကြားခြင်း: စက်တင်ရှင်စနစ်များ ဖြင့် မော်ဒယ်ကို စကားလုံးများကို နားလည်စေရန် သင်ပေးခြင်း။
- အဆင့်မြှင့်ခြင်း: မော်ဒယ်၏ စွမ်းဆောင်ရည် တိုးတက်စေရန် ချိန်ညှိခြင်း။

SLM မော်ဒယ်များသည် စွမ်းအားကန့်သတ်သော စက်ပစ္စည်းများ၊ ဥပမာ မိုဘိုင်းစက်ပစ္စည်းများ သို့မဟုတ် edge computing ပလက်ဖောင်းများတွင် အဆင်ပြေစွာ ထုတ်လွှင့်အသုံးပြုရာတွင် ဆန်းသစ်တိုးတက်မှုအလိုက် တိုးတက်လာသည်။ ထိရောက်မှုအပေါ် ဦးတည်၍ SLM များသည် စွမ်းဆောင်မှုနှင့် လွယ်ကူမှုတို့ကို တန်ဖိုးထည့်ဆောင်ပြီး အမျိုးမျိုးသော ကဏ္ဍများတွင် အသုံးချနိုင်စေသည်။

## သင်ယူရမည့် ရည်ရွယ်ချက်များ

ဒီသင်ခန်းစာတွင် SLM ၏ အသိပညာကို မိတ်ဆက်ပြီး Microsoft Phi-3 နှင့် ပေါင်းပြီး စာသား အကြောင်းအရာ၊ ဗွီရှင်းနှင့် MoE အခြေအနေအမျိုးမျိုးကို သင်ယူသည်။

သင်ခန်းစာပြီးဆုံးချိန်တွင် အောက်ပါမေးခွန်းများကို ဖြေနိုင်သည်။

- SLM ဆိုတာဘာလဲ?
- SLM နှင့် LLM  အကြား ကွာခြားချက်များကဘာလဲ?
- Microsoft Phi-3/3.5 မျိုးဆက်ဆိုတာဘာလဲ?
- Microsoft Phi-3/3.5 မျိုးဆက်ဖြင့် မည်သို့ သုံးနိုင်သလဲ?

အဆင်သင့်ပါသလား? စတင်လိုက်ကြရအောင်။

## LLM များနှင့် SLM များအကြား ကွာခြားချက်များ

LLM နှင့် SLM နှစ်ခုလုံးသည် အခြေခံထားသည့် စက်မှုသင်ကြားမှုအခြေခံကျပြီး၊ နည်းပညာဖွဲ့စည်းမှု၊ သင်ကြားနည်းများ၊ ဒေတာဖန်တီးမှု နည်းလမ်းများ နှင့် မော်ဒယ် သုံးသပ်ခြင်းနည်းလမ်းများတွင် တူညီသည်။ သို့သော် အချက်အချာအချို့ကွာခြားချက်များ ရှိသည်။

## စတုတ္ထအကြောင်းအရာ မော်ဒယ်များ၏ အသုံးချမှုများ

SLM များတွင် အောက်ပါအသုံးချမှုများ ရှိသည်။

- စကားပြော စနစ်များ: ဖောက်သည်များအား ကူညီပံ့ပိုးပေးရန် နှင့် အသုံးပြုသူများနှင့် စကားပြော ပျော်စရာ ဆက်ဆံရန်။
- အကြောင်းအရာ ဖန်တီးမှု: စာရေးသူများအား စိတ်ကူးများ ပေးဆောင်ခြင်း သို့မဟုတ် စာများ စုစည်းရေးသားခြင်းတွင် ကူညီခြင်း။
- ပညာရေး: ကျောင်းသားများအား ရေးသားမှု အလုပ်ဆောင်လုပ်ငန်းများ သို့မဟုတ် အသစ်သော ဘာသာစကားများ သင်ယူရာတွင် ကူညီခြင်း။
- လက်လှမ်းမလွှတ်ခြင်း: ဖွဲ့စည်းပုံမှန်တေရေး စနစ်များ တည်ဆောက်ခြင်းသည် ထူးခြားသူများအတွက် အကူအညီပေးစနစ်ဖြစ်သည်။

**အရွယ်အစား**
  
LLM နှင့် SLM အကြား အဓိကကွာခြားချက်သည် မော်ဒယ်၏ အရွယ်အစားဖြစ်သည်။ ChatGPT (GPT-4) ကဲ့သို့သော LLM များသည် ၁.၇၆ တရီလီယံ ပုံစံများ ပါဝင်နိုင်ပြီး Mistral 7B ကဲ့သို့သော open-source SLM များမှာ ပို၍ သေးငယ်ပြီး ၇ ဘီလီယံ ပုံစံများသာ ပါဝင်သည်။ ကွာခြားချက်သည် မော်ဒယ်ဖွဲ့စည်းမှု နှင့် သင်ကြားမှု နည်းလမ်းကွဲပြားမှုကြောင့်ဖြစ်သည်။ ChatGPT သည် encoder-decoder ပေါ်တွင် self-attention နည်းဖြင့် စီမံထားပြီး Mistral 7B သည် sliding window attention ကို အသုံးပြုသော decoder-only မော်ဒယ်ဖြစ်သည်။ ဤနည်းပညာ ကွာခြားမှုများသည် မော်ဒယ်၏ ရိုးရှင်းမှု နှင့် စွမ်းဆောင်ရည်တွင် အရေးပါတဲ့ ထိခိုက်မှု ရှိသည်။

**နားလည်မှု**

SLM များသည် သီးသန့်နယ်ပယ်များတွင် ထူးခြားစွာ တိုးတက်အောင် အာရုံစိုက်ထားပြီး ပိုမို အထွေထွေအမြင်များတိုးတက်ရန် ကန့်သတ်မှုရှိနိုင်သည်။ LLM များသည် လူ့အင်္ဂါရပ်များခန့်မှန်းသလို အကျယ်ပြန့်သော ဒေတာများ သင်ကြားဖြင့် သီးသန့်ကဏ္ဍများတွင် အကျိုးရှိစေသည်။ ထို့ကြောင့် ဘာသာစကားဖြေရှင်းခြင်း နှင့် ပရိုဂရမ်ရေးခြင်းကဲ့သို့သော အလုပ်များတွင် ပိုမိုအသုံးဝင်သည်။

**ကွန်ပျူတာ အသုံးပြုမှု**

LLM များ၏ သင်ကြားမှုနှင့် စခန်းတည်ဆောက်မှုများသည် ကွန်ပျူတာ ဒါန့်စားမှုများ လိုအပ်ကြပြီး GPU cluster များလိုအပ်လွန်းသည်။ ဥပမာအားဖြင့် ChatGPT ကို စတင်သင်ကြားရန် သာသနာတော်ကြီး GPU များ လိုအပ်ပြီး ရက်များကာလတိုင်း အရမ်းကြာမြင့်သည်။ အနည်းငယ် အရွယ်အစားရဲ့ SLM များသည် GPU စွမ်းအားနည်းသော ဒက်ဇ်ပ်တွင်လည်း အသုံးပြုနိုင်သည်။ Mistral 7B သည် တိုက်ရိုက် သင်ကြားနိုင်ပြီး နည်းနည်း GPU များသာ အသုံးပြုကြသည်။ သို့သော် သင်ကြားမှုအချိန်ကလည်း နာရီအများကြီး များမည်ဖြစ်သည်။

**ဖွဲ့စည်းမှု မျက်မှောက်ခြင်း**

LLM များတွင် ဖွဲ့စည်းမှုသည် သင်ကြားထားသည့် ကြားဖြတ်ဒေတာနှင့် စပ်လျဉ်းသည်။ ၎င်းတို့တွင် အင်တာနက်မှ ဦးစားပေးမည့် မူရင်းဒေတာများရှိပြီး၊ အချို့အဖွဲ့များ ဘာမှတ်ချက် မရရှိ၊ မှားယွင်းစွာ တိကျမှု မရှိ၊ ဒိုင်ယလက်တ်၊ ဒေသဆိုင်ရာ ကွဲပြားချက်များနှင့် စကားပုံများမှ ခွဲခြားမှုများ ကျရောက်သည်။ LLM မော်ဒယ်၏ ဖွဲ့စည်းမှု ရှုပ်ထွေးမှုများသည် ဤဖွဲ့စည်းမှု မျက်မှောက်ခြင်းကို ပိုမိုပေါက်ဖောက်စေနိုင်ပြီး အပြည့်အစုံ ချိန်ညှိမှတစ်ပါး မတွေ့မြင်နိုင်သော အစိတ်အပိုင်းဖြစ်သည်။ SLM များသည် ပိုမို ကန့်သတ်ထားသည့် သီးသန့်ဒေတာများ ဖြင့် သင်ကြားထားသောကြောင့် ယင်းမည်သည့်စွက်ဖက်မှုမှလွဲကင်းပြီး ဖွဲ့စည်းမှု မျက်မှောက်ခြင်း အတွက် ပို၍ရည်းစားဖြစ်သည်။

**အနုတ်သတ်ခြင်း**

SLM များ၏ အနုတ်သတ်ခြင်း အပြောင်းအလဲသည် သက်သာမှု ပို၍ပိုပြီး အကျိုးရှိသည်၊ ဒေသတွင်းမူရင်းစက်ပစ္စည်းများဖြင့် ထုတ်လုပ်နိုင်သည့် အင်အားအပြည့်အဝကို မလိုအပ်ဘဲ မော်ဒယ်အအောင်မြင်မှုကို ရရှိစေနိုင်သည်။ LLM များသည် မော်ဒယ်အရွယ်အစားနှင့် ရုပ်ပိုင်းဆိုင်ရာရှုပ်ထွေးမှုကြောင့် အမြန်ဆုံးအတွက် အဆင့်မြင့် parallel ကွန်ပျူတာအားကောင်းမှုလိုအပ်သည်။ စုစည်းအသုံးပြုသူများပေါ်မူတည်၍ ထိုအချိန် အောက်မွန်းလျော့နည်းခြင်းတို့ကို ဖြစ်စေသည်။

အကျဉ်းချုပ်အားဖြင့် LLM နှင့် SLM များသည် စက်မှုသင်ကြားမှုအခြေခံနည်းလမ်းများကို ပိုင်ဆိုင်ထားသော်လည်း မော်ဒယ်အရွယ်အစား၊ ကွန်ပျူတာလိုအပ်ချက်များ၊ နားလည်မှုအတိအကျ၊ ဖွဲ့စည်းမှု မျက်မှောက်ခြင်းနှင့် အနုတ်သတ်မှုအရှိန်တို့တွင် ကွာခြားမှုများ ရှိသည်။ ၎င်းတို့သည် အသုံးပြုမှုများအတွက် မတူညီမှုရှိသော်လည်း၊ LLM များသည် ပိုမို အမျိုးမျိုးသုံးနိုင်သောကြောင့် မိုက်ထရိုဆာပရိုဆက်တာများကို လုပ်ဆောင်ရန် တောင်းဆိုမှုများရှိပြီး၊ SLM များသည် ကန့်သတ်ထားသည့် ကဏ္ဍများ၌ ထိရောက်မှု မြင့်မားစေသည်။

***မှတ်ချက်- ဒီသင်ခန်းစာတွင် Microsoft Phi-3 / 3.5 ကို အသုံးပြု၍ SLM များကို မိတ်ဆက်ပါမည်။***

## Phi-3 / Phi-3.5 မျိုးဆက် မိတ်ဆက်ခြင်း

Phi-3 / 3.5 မျိုးဆက်သည် အဓိကအားဖြင့် စာသား၊ ဗွီရှင်း နှင့် အေးဂျင့် (MoE) အသုံးချမှုအခြေအနေများကို ဦးတည်ထားပါသည်။

### Phi-3 / 3.5 Instruct

စာသားဖန်တီးခြင်း၊ စကားပြောပြီးစီးခြင်းနှင့် အကြောင်းအရာသိပ်သည်းမှုဖြုတ်ခြင်း အဓိကအတွက် ရည်ရွယ်သည်။

**Phi-3-mini**

3.8B ပုံစံ မော်ဒယ်သည် Microsoft Foundry, Hugging Face နှင့် Ollama တွင် ရနိုင်သည်။ Phi-3 မော်ဒယ်များသည် အညီပမာဏနှင့် ပိုကြီးသော မော်ဒယ်များထက် အကြီးပြင်းသော ချိန်ခွင်များတွင် ထိုက်တန်စွာ ကွဲပြားမှုရှိသည်။ Phi-3-mini သည် မော်ဒယ် အရွယ်အစားနှစ်ဆ၏ စွမ်းဆောင်ရည်ကို ကျော်လွန်ပြီး Phi-3-small နှင့် Phi-3-medium သည် GPT-3.5 ထက် ပို၍ စွမ်းဆောင်နိုင်သည်။

**Phi-3-small & medium**

7B ပုံစံနှင့် Phi-3-small သည် GPT-3.5T ကို ဘာသာစကား၊ ထောက်လှမ်းခြင်း၊ ကုဒ်ရေးခြင်းနှင့် သင်္ချာ benchmarks များတွင် အနိုင်ရသည်။

Phi-3-medium သည် 14B ပုံစံရှိပြီး Gemini 1.0 Pro ကို ကျော်လွန်စွာ ဆက်ထိန်းသည်။

**Phi-3.5-mini**

Phi-3-mini ၏ အဆင့်မြှင့်တင်ထားပြီး၊ ပုံစံအရေအတွက် မပြောင်းလဲဘဲ ဘာသာစကားအမျိုးမျိုးကို ပိုမိုကောင်းမွန်စေရန် (20+ ဘာသာစကား များ ပံ့ပိုးသည်၊ ဥပမာ - အာရပ်၊ တရုတ်၊ စီဇက်၊ ဒိန်းမတ်၊ ဒတ်ချ်၊ အင်္ဂလိပ်၊ ဖင်လန်၊ ပြင်သစ်၊ ဂျာမန်၊ ဟီဗရွူး၊ ဟန်ဂေရီ၊ အီတလီ၊ ဂျပန်၊ ကိုးရီးယား၊ နော်ဝေး၊ ပိုးလန်၊ ပေါ်တူဂီးส၊ ရုရှား၊ စပိန်၊ ဆွီဒင်၊ ထိုင်း၊ တူရကီ၊ ယူကရိန်း) နှင့် ရေရှည်အကြောင်းအရာ ပိုမိုအားကောင်းစေသည်။

3.8B ပုံစံနှင့် Phi-3.5-mini သည် တူညီသော အရွယ်အစားမော်ဒယ်များထက် ပိုမိုကောင်းမွန်ပြီး အရွယ်အစားနှစ်ဆသော မော်ဒယ်များနှင့် ၎င်း၏ စွမ်းဆောင်ရည်ညီမျှသည်။

### Phi-3 / 3.5 Vision

Phi-3/3.5 မှ Instruct မော်ဒယ်ကို Phi ၏ နားလည်မှုစွမ်းဆောင်ရည်အဖြစ် တွေးနိုင်ပြီး Vision သည် ကမ္ဘာသို့ ကြည့်မြင်နားလည်စေသော မျက်စိပေးသည်။


**Phi-3-Vision**

Phi-3-vision သည် 4.2B ပုံစံရှိပြီး Claude-3 Haiku နှင့် Gemini 1.0 Pro V ကဲ့သို့သော ပိုကြီးသော မော်ဒယ်များထက် အထွေထွေ ဗွီရှင်း ဆင်ခြင်ခွင့်များ၊ OCR နှင့် စာရင်းဇယားနားလည်မှု တိုက်ခိုက်မှုများတွင် ပိုမိုထက်မြတ်သည်။


**Phi-3.5-Vision**

Phi-3.5-Vision သည် Phi-3-Vision ၏ အဆင့်မြှင့်တင်မှုဖြစ်ပြီး ပုံရိပ်များစွာ ပံ့ပိုးသည်။ ၎င်းကို ဗွီရှင်းတိုးတက်မှုအနေဖြင့် တွေးနိုင်ပြီး ပုံများသာမက ဗွီဒီယိုများကိုလည်း ကြည့်မြင်နိုင်သည်။

Phi-3.5-vision သည် Claude-3.5 Sonnet နှင့် Gemini 1.5 Flash ကဲ့သို့ ပိုကြီးသော မော်ဒယ်များထက် OCR၊ စာရင်းဇယားနှင့် အချုပ်ချုပ်နားလည်မှု အလုပ်များတွင် သာယာကောင်းမွန်ပြီး၊ အထွေထွေ ဗွီရှင်း အသိပညာဆိုင်ရာ ထောက်ပံ့မှုမှာ ညီမျှသည်။ များစွာသော ပုံရိပ်များကို ဝင်ရောက်ပြီး ဆင်ခြင်မှု များ ပြုလုပ်နိုင်သည်။


### Phi-3.5-MoE

***Mixture of Experts(MoE)*** သည် မော်ဒယ်ကို ကြီးမားသော ကွန်ပျူတာစွမ်းအားမလိုဘဲ သင်ကြားနိုင်စေသည်။ ၎င်းဖြင့် မော်ဒယ်ဖြင့် ဒေတာအရွယ်အစားကို တိုးချဲ့ပေးပြီး ပုံမှန် မော်ဒယ်ထက် ပိုမြန်ဆန်စွာ သင်ကြားနိုင်မည်ဖြစ်သည်။

Phi-3.5-MoE သည် 16x3.8B ကျွမ်းကျင်သူအပိုင်းအစများနှင့် ဖွဲ့စည်းထားသည်။ 6.6B လှုပ်ရှားတဲ့ပုံစံများသာပါရှိသော်လည်း ဤ MoE မော်ဒယ်တွင် ရည်ရွယ်ချက်၊ ဘာသာစကားနားလည်မှုနှင့် သင်္ချာဆိုင်ရာ စွမ်းဆောင်ရည်ကို ပိုကြီးသော မော်ဒယ်များကဲ့သို့ ကောင်းမွန်စွာ ပြုလုပ်နိုင်သည်။

Phi-3/3.5 မော်ဒယ်ကို အခြေအနေအသေးစားအမျိုးမျိုးနှင့်အလိုက် အသုံးပြုနိုင်သည်။ LLM မဟုတ်ဘဲ Phi-3/3.5-mini သို့မဟုတ် Phi-3/3.5-Vision ကို edge device များတွင် ထည့်သွင်း အသုံးပြုနိုင်သည်။


## Phi-3/3.5 မျိုးဆက် မော်ဒယ်များကို မည်သို့ အသုံးပြုမည်နည်း

**NVIDIA NIM**

မိုဃ်းတံခါးပြင် Foundry Models စာရင်းအပြင် [NVIDIA NIM](https://developer.nvidia.com/nim?WT.mc_id=academic-105485-koreyst) မှတဆင့် Phi-3/3.5 မျိုးဆက် API ခေါ်ယူမှုများနှင့်လုပ်ဆောင်နိုင်သည်။ NVIDIA NIM (NVIDIA Inference Microservices) သည် မိုဃ်း၊ ဒေတာစင်တာများနှင့် ဝရာ့စတေးရှင်းများ လိုအပ်သည့် နေရာများတွင် AI မော်ဒယ်များ ပိုမို ထိရောက်စွာ ထည့်သွင်းရန် ဖန်တီးထားသည့် မြန်ဆန်သော inference microservices များအစုဖြစ်သည်။

NVIDIA NIM ၏ အဓိက အင်္ဂါရပ်များမှာ

- ** တင်သွင်းရအောင် လွယ်ကူခြင်း:** NIM သည် AI မော်ဒယ်တင်သွင်းမှုကို တစ်ချက် ကြောင့် လုပ်၍ ရသောကြောင့် လက်ရှိ လုပ်ငန်းစဉ်များတွင် လွယ်ကူစွာ ပေါင်းစည်းနိုင်သည်။

- **တိုးတက်မွန်ကန်သော ဆောင်ရွက်မှု:** NVIDIA ၏ ရှေ့ကောင်းစွာ ပြင်ဆင်ထားသော အယူအဆ အင်ဂျင်များဖြစ်သည့် TensorRT နှင့် TensorRT-LLM တို့ကိုအသုံးပြုပြီး နိမ့်သော အသံတုံ့ပြန်မှုနှုန်းနှင့် မြင့်မားသော ထွက်ရှိမှုကို သေချာစေသည်။
- **တိုးချဲ့နိုင်မှု:** NIM သည် Kubernetes ပေါ်တွင် autoscaling ကို ထောက်ပံ့ကာ လုပ်ဆောင်ပမာဏမျိုးစုံကို ထိရောက်စွာ ကိုင်တွယ်နိုင်စွမ်းရှိသည်။
- **လုံခြုံမှုနှင့် ထိန်းချုပ်မှု:** အဖွဲ့အစည်းများသည် မိမိတို့စီမံခန့်ခွဲသော အခြေခံအဆောက်အအုံပေါ်တွင် NIM microservices များကို မိမိတိုင် ငှားရမ်းထားခြင်းဖြင့် ဒေတာနှင့် အက်ပလီကေးရှင်းများကို ထိန်းချုပ်နိုင်သည်။
- **စံနှုန်း API များ:** NIM သည် စက်မှုလုပ်ငန်း စံ API များကို ပံ့ပိုးကာ chatbots၊ AI အကူအညီပေးသူများနှင့် အခြား AI အက်ပလီကေးရှင်းများ တည်ဆောက်ရန်နှင့် ပေါင်းစည်းရန် လွယ်ကူစေသည်။

NIM သည် NVIDIA AI Enterprise ၏ အစိတ်အပိုင်းတစ်ခု ဖြစ်ပြီး AI မော်ဒယ်များကို တပ်ဆင်ခြင်းနှင့် လည်ပတ်စေခြင်းကို ရိုးရှင်းစေရန် ရည်ရွယ်သည်၊ NVIDIA GPU များပေါ်တွင် ထိရောက်စွာ လည်ပတ်စေရန် သေချာစေသည်။

- ဒီမို: NVIDIA NIM ကို အသုံးပြုပြီး Phi-3.5-Vision-API ကို ခေါ်ယူခြင်း [[Click this link](./python/Phi-3-Vision-Nividia-NIM.ipynb?WT.mc_id=academic-105485-koreyst)]


### Phi-3/3.5 ကို ဒေသတွင်းတွင် လည်ပတ်ခြင်း
Phi-3 သို့မဟုတ် GPT-3 ကဲ့သို့သော ဘာသာစကား မော်ဒယ်များနှင့် ဆက်စပ်သည့် inference သည် input ကို အခြေခံ၍ စကားဝိုင်းများ သို့မဟုတ် ခန့်မှန်းချက်များ ထုတ်ပေးခြင်းဖြစ်သည်။ Phi-3 သို့ prompt သို့မဟုတ် မေးခွန်း ပေးသောအခါ၌ ၎င်း၏ လေ့ကျင့်ပြီးသော နယူးရယ်ကွန်ယက်ကို အသုံးပြု၍ သင်ကြားထားသော ဒေတာအပေါ် မျဉ်းစားမှုများနှင့် ဆက်ဆံမှုများကို 分析ပီး အလားအလာနှင့် သက်ဆိုင်မှုရှိသော တုံ့ပြန်ချက်ကို ခန့်မှန်းထုတ်ပေးသည်။

**Hugging Face Transformer**
Hugging Face Transformers သည် သဘာဝဘာသာစကား ပြုပြင်ထိန်းသိမ်းမှု (NLP) နှင့် အခြား စက်မှုလေ့လာမှု တာဝန်များအတွက် အင်အားကြီးသော စာကြည့်တိုက်တစ်ခု ဖြစ်သည်။ ၎င်းအကြောင်း အချက်အချို့မှာ-

1. **Pretrained Models**: စာသား အမျိုးမျိုးသုံးသော တာဝန်များအတွက် အသုံးပြုနိုင်သော သန်းကျော်သော လေ့ကျင့်ပြီး မော်ဒယ်များကို ပံ့ပိုးပေးသည်။ ဥပမာ- စာသားအမျိုးအစား ဖန်တီးခြင်း၊ အမည်ကွဲ အမှတ်အသား သတ်မှတ်ခြင်း၊ မေးခွန်းဖြေဆိုခြင်း၊ အကျဉ်းချုပ်ပြုလုပ်ခြင်း၊ ဘာသာပြန်ခြင်းနှင့် စာသား ထုတ်ပေးခြင်း စသဖြင့်။

2. **Framework Interoperability**: PyTorch၊ TensorFlow နှင့် JAX တို့အပါအဝင် သင့်လျော်သော လေ့လာမှု စနစ်များ များစွာကို ထောက်ပံ့သည်။ ဒါကြောင့် တစ်ဖန်မြောက် စနစ်တွင် မော်ဒယ်ကို လေ့လာပြီး၊ အခြားစနစ်တွင် အသုံးပြုနိုင်သည်။

3. **Multimodal Capabilities**: NLP အပြင် Hugging Face Transformers သည် ကွန်ပျူတာမြင်ကွင်း (ဥပမာ- ပုံအမျိုးအစား ခွဲခြားခြင်း၊ ပစ္စည်းကွဲထုတ်ခြင်း) နှင့် အသံလုပ်ဆောင်မှု (ဥပမာ- စကားသိရှိခြင်း၊ အသံအမျိုးအစား ခွဲခြားခြင်း) တာဝန်များကိုလည်း ပံ့ပိုးပေးသည်။

4. **လွယ်ကူစွာ အသုံးပြုနိုင်မှု**: မော်ဒယ်များကို လွယ်လင့်တကူ ဒေါင်းလုဒ်ဆွဲ၍ အတိုးအမြှင့် ပြုလုပ်ရန် API များနှင့် ကိရိယာများ ပံ့ပိုးပေးသည်၊ စတင်သင်ယူသူများနှင့် ကျွမ်းကျင်သူများ နှစ်ဦးလုံးအတွက် အဆင်ပြေစေသည်။

5. **အသိုင်းအဝိုင်းနှင့် အရင်းအမြစ်များ**: Hugging Face သည် ဖက်ရွံမှုကောင်းမွန်သော အသိုင်းအဝိုင်းနှင့် ကြီးမားသော စာရွက်စာတမ်းများ၊ သင်ခန်းစာများနှင့် လမ်းညွှန်များရှိပြီး အသုံးပြုသူများ အား အစပြုရန်နှင့် စာကြည့်တိုက်မှ အမြတ်ထုတ်ရန် အကူအညီပေးသည်။
[အတည်ပြုစာရွက်စာတမ်း](https://huggingface.co/docs/transformers/index?WT.mc_id=academic-105485-koreyst) သို့မဟုတ် ၎င်းတို့၏ [GitHub repository](https://github.com/huggingface/transformers?WT.mc_id=academic-105485-koreyst) ကို ကြည့်ရှုနိုင်ပါသည်။

**ONNX Runtime for GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) သည် ထပ်တူပလက်ဖောင်းများတွင် inference နှင့် လေ့လာမှုစက်ရုပ် အချက်အလက်မြန်စေသော ကိရိယာတစ်ခုဖြစ်သည်။ ONNX Runtime for Generative AI (GENAI) သည် ဖန်တီးနိုင်သော AI မော်ဒယ်များကို အကြောင်းမပြတ် ညီညာစွာ လည်ပတ်စေသည့် စွမ်းအင်မြင့် ကိရိယာဖြစ်သည်။

## ONNX Runtime ဆိုတာဘာလဲ?
ONNX Runtime သည် စက်မှုလေ့လာမှု မော်ဒယ်များ၏ မြင့်မားသော တုံ့ပြန်မှု တိုးမြှင့်ရေးအတွက် ဖွင့်လှစ် ဇယားတည်ဆောက်မှု ဖြစ်သည်။ ONNX ဖော်မတ်ဖြင့် မော်ဒယ်များကို ထောက်ပံ့သည်။ ONNX Runtime inference သည် ပိုမိုမြန်ဆန်သော အသုံးပြုသူ အတွေ့အကြုံများနှင့် ကုန်ကျစရိတ် လျော့ချမှုကို ပြုလုပ်နိုင်ပြီး PyTorch နှင့် TensorFlow/Keras ကဲ့သို့သော နက်နဲသောလေ့လာမှု စနစ်များအပါအဝင် scikit-learn၊ LightGBM၊ XGBoost ကဲ့သို့သော ယခင်က မော်ဒယ်များကိုလည်း ထောက်ပံ့သည်။ ONNX Runtime သည် hardware များ၊ မောင်းသူများ၊ အလုပ်လုပ်ပုံစံများနှင့် ကိုက်ညီပြီး hardware accelerator များကို ကိုင်တွယ်ကာ တိုးတက်မြင့်မားသော ပြောင်းလဲမှုများနှင့် ဇယား optimization များဖြင့် ထိရောက်မှုမြင့်စေသည်။

## Generative AI ဆိုတာဘာလဲ?
Generative AI သည် လေ့ကျင့်ထားသော ဒေတာကို အခြေခံ၍ စာသား၊ ပုံများ သို့မဟုတ် ဂီတများကဲ့သို့ အကြောင်းအရာအသစ်များ ဖန်တီးနိုင်သော AI စနစ်များကို ဆိုလိုသည်။ ဥပမာများမှာ GPT-3 ကဲ့သို့သော ဘာသာစကားမော်ဒယ်များနှင့် Stable Diffusion ကဲ့သို့သော ပုံဖန်တီးမော်ဒယ်များဖြစ်သည်။ ONNX Runtime for GenAI သည် ONNX မော်ဒယ်များအတွက် ဖန်တီးနိုင်သော AI လည်ပတ်မှု လုပ်ငန်းစဉ်အား ပံ့ပိုးသည်။ ၎င်းတွင် ONNX Runtime ။ inference၊ logits ကို စီမံခြင်း၊ ရှာဖွေရန်နှင့် နမူနာ ယူခြင်း၊ KV cache စီမံခန့်ခွဲမှုတို့ ပါဝင်သည်။

## ONNX Runtime for GENAI
ONNX Runtime for GENAI သည် generative AI မော်ဒယ်များ ထောက်ပံ့နိုင်ရန် ONNX Runtime ၏ စွမ်းဆောင်ရည်များကို ကျယ်ပြန့်စေသည်။ အဓိက အင်္ဂါရပ်များမှာ-

- **ပလက်ဖောင်း ကျယ်ပြန့်စွာ ထောက်ပံ့မှု:** Windows၊ Linux၊ macOS၊ Android နှင့် iOS တို့တွင် လည်ပတ်နိုင်သည်။
- **မော်ဒယ် ပံ့ပိုးမှု:** LLaMA၊ GPT-Neo၊ BLOOM စသည့် လူကြိုက်များသော generative AI မော်ဒယ်များကို ထောက်ပံ့သည်။
- **ဆောင်ရွက်မှု တိုးတက်မှု:** NVIDIA GPUs၊ AMD GPUs နှင့် အခြား hardware accelerator များအတွက် အချက်အလက်များ များစွာ တိုးတက်အောင် ပြုလုပ်ထားသည်။
- **အသုံးပြုရလွယ်ကူမှု:** သယ်ယူပို့ဆောင်မှု စနစ်များအတွက် API များ ပံ့ပိုးပေးကာ အနည်းငယ်သော ကုဒ်ဖြင့် စာသား၊ ပုံများနှင့် အခြားအကြောင်းအရာ များ ဖန်တီးနိုင်စေသည်။
- အသုံးပြုသူများသည် generate() အဆင့်မြင့် နည်းလမ်းကို ခေါ်နိုင်ပြီး မော်ဒယ်ကို တစ်ကြိမ်ချင်း စစ်ဆေးကာ တစ်တုံ့ပြန်ချက် အနေနှင့် token တစ်ခုစီ ဖန်တီးနိုင်ပြီး loop အတွင်းတွင် ဖန်တီးမှု ညှိနှိုင်းချက်များကို မလိုလားသလို ပြောင်းလဲနိုင်သည်။
- ONNX runtime သည် greedy/beam search နှင့် TopP, TopK sampling များကို ထောက်ပံ့ကာ token စီးရီးများ ဖန်တီးရန် အဖွဲ့အစည်းပါရှိသည်။ ထို့အပြင် ပြန်လည်ထပ်ခါခါ ထိန်းချုပ်မှုများကဲ့သို့ logits ကိုလည်း ထည့်သွင်းထားသည်။ သင်သည် စိတ်ကြိုက် အကဲဖြတ်ခြေကိုလည်း လှုပ်ရှားစွာ ထည့်သွင်းနိုင်သည်။

## စတင်ခြင်း
ONNX Runtime for GENAI နှင့် စတင်ရန်အတွက် အောက်ပါ ခြေလှမ်းများကို လိုက်နာနိုင်သည်။

### ONNX Runtime ကို ထည့်သွင်းပါ:
```Python
pip install onnxruntime
```
### Generative AI Extensions ကို ထည့်သွင်းပါ:
```Python
pip install onnxruntime-genai
```

### မော်ဒယ်တစ်ခု ပြေးရန်: Python တွင် ရိုးရှင်းသော ဥပမာ-
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### ဒီမို: ONNX Runtime GenAI ကို အသုံးပြုပြီး Phi-3.5-Vision ကို ခေါ်ယူခြင်း


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
