# စတင္မိတ္ဆက္ျခင္း - ကိုယ္ပိုင္ႏႊင့္ေခါင့္ ဗဟုသုတအတြက္ Generative AI အတြက္ အသက္ပိြဳးတရားေကာင္းေသာ ဘာသာစကား မော်ဒယ်ငယ်များ

Generative AI သည် အသစ္ေတြ႔သည့္ အကြာင္းအရာမ်ားကို ဖန္တီးတတ္သည့္ စနစ္မ်ား ဖန္တီးရာတွင္ အာရုံစူးစိုက္ေသာ အတုယူရခဲ႕ေသာ စက္ရုပ္ဗဟုသုတ၏ လယ္ကြင္းတစ္ခုျဖစ္သည္။ ၎င်းအကြောင်းအရာသည် စာသား၊ ပုံ၊ တေးဂီတ နှင့် အပြည့်အစုံသော အိမ်ရာမဟုတ်ဝယ်ပတ်ဝန်းကျင်များတိုင် ပိုမိုအကြမ်းဖျင်းဖန်တီးနိုင်သည်။ Generative AI ၏ အလွန်စိတ်ဝင်စားဖွယ် ရည်ရွယ်ချက်တစ်ခုမှာ ဘာသာစကား မော်ဒယ်များ၏ ဧရိယာဖြစ်သည်။

## ဘာသာစကား မော်ဒယ်ငယ်များက ဘာဖြစ်သနည်း?

ဘာသာစကား မော်ဒယ်ငယ် (Small Language Model, SLM) သည် အကြီးမဟုတ်သော ဘာသာစကား မော်ဒယ် (Large Language Model, LLM) ၏ ဖော်ပြချက်တစ်ခု ဖြစ်ပြီး၊ LLM များ၏ ဖွဲ့စည်းမှုပုံစံ နှင့် နည်းဗျူဟာများကို အများစုအသုံးပြုသော်လည်း ကွန်ပြူတာ အန္တရာယ်သေးသောနည်းလမ်းဖြင့် လုပ်ဆောင်သည်။ 

SLM များမှာ လူသားလို ဆက်လက်နေထိုင်မှုရှိသည့် စာသားများ ဖန်တီးရာတွင် မွန်သော ဘာသာစကား မော်ဒယ် အမျိုးအစားဖြစ်သည်။ GPT-4 ကဲ့သို့သော LLM များမှ ကွာခြား၍၊ SLM များ သည် ပိုမိုသေးငယ်ပြီး ထိရောက်မှုရှိသောကြောင့် ကွန်ပြူတာ အရင်းအမြစ်များ ကန့်သတ်ထားသည့် လုပ်ငန်းများတွင် သင့်တော်သည်။ သေးငယ်သော်လည်း ပုံမှန်အားဖြင့် များစွာသော တာဝန်များကို ဆောင်ရွက်နိုင်သည်။ ပုံမှန်အားဖြင့် SLM များသည် LLM များကို စုစည်းခြင်း သို့မဟုတ် ခြစ်ခြင်းဖြင့် ဖန်တီးကြပြီး မူရင်း မော်ဒယ်၏ လုပ်ဆောင်ချက်နှင့် ဘာသာစကား အရည်အသွေးများကို သိသိသာသာ သိမ်းဆည်းထားရန် ရည်ရွယ်သည်။ မော်ဒယ် အရွယ်အစား လျော့နည်းခြင်းသည် စုစည်းမှုခက်ခဲမှုကို လျော့နည်းစေပြီး၊ SLM များကို မှတ်ဉာဏ် သုံးစွဲမှုနှင့် ကွန်ပျူတာ လုပ်ငန်းဆောင်တာများတွင် ပိုမိုထိရောက်စေသည်။ ထို့သော် ဤ တိုးတက်မှုများဖြင့် SLM များသည် သဘာဝဘာသာစကား ဖြင့် ကိစ္စများကို စွမ်းဆောင်လို့ရသည် -

- စာသား ဖန်တီးခြင်း - ညီထွေညီထွေခံ စကားလုံးများ သို့မဟုတ် စာပိုဒ်များ ဖန်တီးခြင်း။
- စာသား ပြည့်စုံခြင်း - ပေးထားသော စကား တစ်ကြောင်းကို ခန့်မှန်းပြီး ပြည့်စုံစေခြင်း။
- ဘာသာပြန်ခြင်း - ဘာသာတစ်ခုမှ တစ်ခုသို့ စာသားကို ပြောင်းလဲခြင်း။
- အကျဉ်းချုပ်ရေးသားခြင်း - ရှည်လျားသော စာသားများကို အကျဉ်းချုပ်ပြီး နားလည်ရလွယ်ကူသည့် အကျဉ်းစိတ်များ ထုတ်ပေးခြင်း။

၎င်းတို့သည် ၎င်းတို့၏ နက်ရှိုင်းမှု သို့မဟုတ် နားလည်မှု အနည်းငယ် နှိမ့်ချရန် လိုအပ်ချက် အပြင် ထို အကြီးမားသော မော်ဒယ်များနှင့် နှိုင်းယှဉ်ပါက။

## ဘာသာစကား မော်ဒယ်ငယ်များ ဘယ်လို လုပ်ဆောင်သလဲ?

SLM များသည် စာသားအချက်အလက်များ တစုလုံး ပေါင်းစပ် စနစ်တကျ သင်ကြားခံရသည်။ သင်ကြားမှုတွင်၊ အဲဒီ မော်ဒယ်ဟာ ဘာသာစကား၏ မျဉ်းကြောင်းများ နှင့် ဖွဲ့စည်းပုံကို လေ့လာပြီး သင် အချိုးအစားတံတားများပါရှိသော စာသားကို ပြန်ဖန်တီးနိုင်သည်။ သင်ကြားမှုလုပ်ငန်း စဉ်သည် -

- ဒေတာ စုဆောင်းခြင်း - အမျိုးမျိုးသော အရင်းအမြစ်များမှ စာသား ဒေတာများကို စုဆောင်းခြင်း။
- အစီအစဉ်ပြင်ဆင်ခြင်း - သင်ကြားမှု အတွက် အသင့်ဖြစ်စေရန် ဒေတာကို ရှင်းလင်းပြီး စည်းကမ်းတကျ ချမှတ်ခြင်း။
- သင်ကြားခြင်း - စက်လေ့ကျင့်မှု အယ်လဂေါရင်းများ အသုံးပြု၍ မော်ဒယ်အား စာသား နားလည်ခြင်းနှင့် ဖန်တီးခြင်းကို သင်ကြားခြင်း။
- စနစ်တကျ ချိန်ညှိခြင်း - အထူးတာဝန်များတွင် မော်ဒယ်၏ ထိရောက်မှုကို ပြုပြင်တိုးတက်စေရန် ပြင်ဆင်ခြင်း။

SLM မော်ဒယ်များ ဖန်တီးမှုသည် ကန့်သတ်ထားသော အရင်းအမြစ်များပေါ်တွင် ထောက်ပံ့နိုင်ရန် လိုအပ်ချက်နှင့် ကိုက်ညီသည်။ ဥပမာအားဖြင့် မိုဘိုင်းစက်ပစ္စည်းများ သို့မဟုတ် အနားငါးကွန်ပျူတာများတွင် LLM များကို ထည့်သွင်းမရခြင်းကိုဖြတ်၊ SLM များသည် ထိရောက်မှုနှင့် လုံလောက်စွာ အသုံးပြုပြီး ကျယ်ပြန့်သော လုပ်ငန်းအသုံးပြုမှုများထဲတွင် အသုံးပြုနိုင်သည်။

## သင်ယူရမည့် ရည်ရွယ်ချက်များ

ဒီသင်ခန်းစာတွင် SLM ၏ အထွေထွေ ဗဟုသုတကို မိတ်ဆက်လေ့လာပြီး Microsoft Phi-3 နှင့် ပေါင်းစည်း၍ စာသား အကြောင်းအရာ၊ ရုပ်မြင်ကွင်း၊ MoE စသည့် ကိစ္စရပ်များကို သင်ကြားမည်ဖြစ်သည်။

ဒီသင်ခန်းစာပြီးဆုံးရင် အောက်ပါမေးခွန်းတွေကို ဖြေကြားနိုင်မှာ ဖြစ်တယ် -

- SLM ဆိုတာ ဘာလဲ?
- SLM နဲ့ LLM ကြားက တူညီချက်နဲ့ ကွာခြားချက် ဘာလဲ?
- Microsoft Phi-3/3.5 မျိုးဆက် သည် ဘာလဲ?
- Microsoft Phi-3/3.5 မျိုးဆက်နဲ့ inference ဘယ်လို ပြုလုပ်ရမလဲ?

စပြန်ကြပါစို့။

## အကြီးစား ဘာသာစကား မော်ဒယ်များ (LLM) နှင့် အသေးစား ဘာသာစကား မော်ဒယ်များ (SLM) ကြားက ကွာခြားချက်များ

LLM နှင့် SLM များသည် အခြေခံ ပညာရပ်များဖြစ်သဖြင့် သဘောတရားများနှင့် နည်းဗျူဟာများကို တူညီစွာ အသုံးပြုကြသည်။ အာကာသဖွဲ့စည်းပုံ၊ သင်ကြားချက် မော်ဒယ်တည်ဆောက်မှုပုံစံများ၊ ဒေတာ ဖန်တီးမှု၊ မော်ဒယ် ဖော်ထုတ်မှု စသည့် ကွာခြားချက်များရှိသည်။

## အသေးစား ဘာသာစကား မော်ဒယ်များ၏ အသုံးပြုမှုများ

SLM များသည် အောက်ပါဆိုင်ရာများတွင် အသုံးများသည် -

- စကားပြော စနစ်များ - ဖောက်သည် မေးခွန်းများကို ဖြေကြားပေးခြင်းနှင့် လူနှင့် ဆွေးနွေးပေါ်ပေါ် ဆက်သွယ်နိုင်ခြင်း။
- အကြောင်းအရာ ဖန်တီးခြင်း - စာရေးသူများအတွက် စိတ်ကူးများ ဖန်တီးပေးခြင်း သို့မဟုတ် ကဏ္ဍစာတမ်းများ အစပ်အပြက် အသုံးပြုခြင်း။
- ပညာရေး - ကျောင်းသားများကို စာရေးခြင်း အတန်းနှင့် အသစ်သော ဘာသာစကားများ သင်ယူရန် ကူညီပေးခြင်း။
- လေဆာအတွက် ကူညီမှု - သိပ္ပံနှင့်ပြီး စကားပြောစနစ်များကဲ့သို့ မတတ်နိုင်သူများအတွက် တိုးတက်တဲ့ကိရိယာများ ဖန်တီးခြင်း။

**အရွယ်အစား**  

LLM နှင့် SLM ယှဉ်ကြည့်ချက်အကြီးဆုံး။ ChatGPT (GPT-4) ကဲ့သို့သော LLM များတွင် ခန့်မှန်းသော 1.76 ထရီလီယံ ပါရာမီတာများပါရှိသော်လည်း Mistral 7B ကဲ့သို့သော SLM များတွင် အနည်းငယ်သာ ပါရာမီတာများ (အားလုံးတွင် 7 ဘီလီယံ ခန့်) ပါဝင်သည်။ ၎င်းကွာခြားချက်မှာ မော်ဒယ် ဖွဲ့စည်းပုံနှင့် သင်ကြားမှုဖြစ်စဉ်စွမ်းဆောင်ချက်တွင် အဓိက ကြားကနေ ဖြစ်သည်။ ဥပမာအားဖြင့် ChatGPT သည် encoder-decoder ဖွဲ့စည်းပုံတွင် self-attention နည်းပညာ အသုံးပြုသဖြင့် Mistral 7B သည် decoder-only မော်ဒယ် အတွင်း sliding window attention ကို အသုံးပြုသည်။ ၎င်းဖွဲ့စည်းပုံ ကွာခြားချက်များသည် မော်ဒယ်၏ ပြည့်စုံဆက်ခံနိုင်မှု နှင့်အဆင့်နှင့်ထိတွေ့သည်။

**နားလည်မှု**

SLM များသည် အကြောင်းအရာ အထူးပြု ကဏ္ဍများတွင် ထိရောက်စွာ လုပ်ဆောင်ရန် အထူးပြုလုပ်ထားသောကြောင့် ကျယ်ပြန့်သော နယ်ပယ်မြေပမာဏများတွင် နားလည်မှု ဆောင်ရွက်နိုင်မှု သည်အနည်းငယ် ကန့်သတ်ခံရနိုင်သည်။ LLM များသည် လူသားလို ဉာဏ်ရည်မြင့်မားမှုကို အပြည့်အဝ သင်ကြားထားပြီး အရင်းအမြစ်များအတော်များများနှင့် ဒီတာများ ပေါင်းစပ်ပြီး မျိုးစုံသော နယ်ပယ်များတွင် သင့်တော်မှုအမြင့်ဆုံး ဖြစ်စေသည်။ ထိုကြောင့် LLM များသည် သဘာဝဘာသာစကား ပြုလုပ်ခြင်းနှင့် ပြင်ဆင်ရေး လုပ်ငန်းကဏ္ဍများတွင် ပိုမို အသုံးပြုနိုင်သည်။

**မဟာသုံး**

LLM များ၏ သင်ကြားခြင်းနှင့် စွဲကပ်မှုများမှာ ကြီးမားသော သုံးစွဲမှု အင်ဖရာစတက်ချာများလိုအပ်ပြီး လုပ်ငန်းတည်ဆောက်မှုတွင် စွန့်ခွားမှုမြင့်သည်။ ဥပမာအားဖြင့် ChatGPT ကို ဂိုဏ်းပေါင်းများစွာ GPU များနှင့် တွဲဖက်၍ ရေရှည် သင်ကြားမှု ကြောင့် အများကြီးကုန်ကျသည်။ SLM များမှာ ပြောင်းလဲအသေးစားဖြစ်သောကြောင့် မိမိ မိသားစု လုပ်ငန်းအားဖြင့် GPU သာလျော့နည်းသော စက်ပစ္စည်းများပေါ်တွင် အသုံးပြုနိုင်သည်၊ သို့သော် သင်ကြားမှုအချိန်မှာ GPU များစွာလိုအပ်ပြီး များစွာသော နာရီများယူနိုင်သည်။

**တင်းကြပ်မှု**

LLM မော်ဒယ်များသည် သင်ကြားထားသော ဒေတာအရင်းအမြစ်များမှ ဖြစ်သလို အတားအဆီးများ၊ မမွန်ကန်မှု၊ မတူညီမှုများ၊ ဘာသာစကား မတူမှုများ အပါအဝင် တင်းကြပ်မှု ရှိနေနိုင်သည်။ မော်ဒယ်၏ ဖွဲ့စည်းမှု တင်းကြပ်မှုကြောင့် သင်ကြားမှုကိစ္စ စစ်ဆေးခြင်း မပြုမီ ဤတင်းကြပ်မှုများ ပို၍ ပိုမို ပိုမို ဆိုးရွားစေနိုင်သည်။ SLM များမှာ မျဉ်းစည်းထားသော ဒေတာများပေါ်တွင် သင်ကြားခြင်း အရ တင်းကြပ်မှု အနည်းငယ်ဖြစ်ပေါ်သည်၊ သို့သော် သူတို့သည် တင်းကြပ်မှုများကို မတင်နိုင်သေးပါဘူး။

**Inference**

SLM များသည် အရွယ်အစား ပိုသေးငယ်မှုကြောင့် inference အချိန်တွင် ထိရောက်မှုရှိပြီး ဒေသခံ စက်ပစ္စည်းများတွင် အချိန်နှစ်သက်စွာ အလုပ်လိုအပ်မှု မရှိဘဲဖြစ်နိုင်သည်။ LLM များသည် အရွယ်အစား ကြီးမားပြီး မော်ဒယ် အဆင့်အတန်းကြောင့် inference အချိန်မှာ ပြန်လည် စိတ်ရှည်မှုရှိပြီး၊ အခြားမော်ဒယ်အသုံးပြုသူများ၏ ကြိုးပမ်းမှုများတွင် ဖြေရှင်းမှု အားနည်းသည်။

အကျဉ်းချုပ် - LLM နှင့် SLM များသည် စက်ရုပ် သင်ယူမှုနည်းလမ်းများတွင် အခြေခံသဘောတူညီမှုရှိသော်လည်း မော်ဒယ်အိုးအစား၊ အရင်းအမြစ်လိုအပ်ချက်၊ နားလည်မှုကျယ်ပြန့်မှု၊ တင်းကြပ်မှု ဖြစ်ပေါ်မှုနှင့် inference အချိန်အပိုင်းများတွင် ကွာခြားမှု အထူးပြု သည်။ LLM များသည် ပိုမိုလွှမ်းမိုးနိင်သော်လည်း အရင်းအမြစ်များ များပြားသည်။ SLM များသည် အထူးပြု လုပ်ဆောင်နိုင်မှုနှင့် သက်သာတဲ့ ကွန်ပျူတာ လုပ်ငန်းများကို ပံ့ပိုးနိုင်သည်။

***သတိပြုရမည့်အချက် - ဒီသင်ခန်းစာတွင် Microsoft Phi-3/3.5 ကို ဥပမာအဖြစ် အသုံးပြုပြီး SLM ကို မိတ်ဆက်လေ့လာမည်။***

## Phi-3 / Phi-3.5 မျိုးဆက် မိတ်ဆက်ခြင်း

Phi-3 / 3.5 မျိုးဆက်သည် စာသား၊ ရုပ်မြင်ကွင်းနှင့် Agent (MoE) လုပ်ဆောင်မှုများနှင့် အထူးသင့်သော မော်ဒယ်များဖြစ်သည်-

### Phi-3 / 3.5 လမ်းညွှန်

စာသား ဖန်တီးခြင်း၊ စကားပြောပြီးစီးခြင်းနှင့် အကြောင်းအရာ သတင်းအချက်အလက် ထုတ်ယူခြင်းတို့အတွက် အဓိက ဖြစ်သည်။

**Phi-3-mini**

3.8B ပါရာမီတာမော်ဒယ်ကို Microsoft Azure AI Studio, Hugging Face နှင့် Ollama တွင် အသုံးပြုနိုင်သည်။ Phi-3 မော်ဒယ်များသည် စွမ်းဆောင်ရည် အလားတူ သို့မဟုတ် အသေးစားပါရာမီတာမော်ဒယ်များထက် အဓိက benchmark များတွင် ကျော်လွန်တင်စားထားသည်။ Phi-3-mini သည် ၎င်း၏ အရွယ်အစား နှစ်ဆမော်ဒယ်များထက်ပိုမိုကောင်းမွန်ပြီး Phi-3-small နှင့် Phi-3-medium များသည် GPT-3.5 ထက် ပိုမိုကောင်းမွန်သည်။

**Phi-3-small နှင့် medium**

7B ပါရာမီတာဖြင့် Phi-3-small သည် GPT-3.5T အား စကား, မှတ်ဉာဏ်, ကုတ်ရေးခြင်း နှင့် သင်္ချာညွှန်ပြမှုများတွင် ကျော်လွန်သည်။

Phi-3-medium သည် 14B ပါရာမီတာဖြင့် ဒီတန်းထဲကို ဆက်လက်တိုးတက်ကာ Gemini 1.0 Pro ကို ကျော်လွန်သည်။

**Phi-3.5-mini**

Phi-3-mini ၏ အဆင့်မြှင့်တစ်ခုအဖြစ် ယူဆနိုင်သည်။ ပါရာမီတာ မပြောင်းလဲသော်လည်း ဘာသာစကားအများကို ထောက်ပံ့မှု တိုးတက်စေပြီး (20+ ဘာသာစကား: အာရဗီ, တရုတ်, ဆယ်ချက်, ဒိန်းမတ်, ဒတ်ချ်, အင်္ဂလိပ်, ဖင်လန်, ပြင်သစ်, ဂျာမန်, ဟီဘရူး, ဟန်ဂေရီ, အီတလီ, ဂျပန်, ကိုရီးယား, နော်ဝေ, ပိုလိုး, ပေါ်တူဂီဖြစ်, ရုရှား, စပိန်, ဆွီဒင်, ထိုင်း, တူရကီ, ယူကရိန်း) နှင့် context ရှည်လျားမှု ထောက်ပံ့မှု ဂါဏ်တင်းစွမ်းအား မြှင့်တင်ထားသည်။

Phi-3.5-mini သည် 3.8B ပါရာမီတာဖြင့် အမျိုးအစားတူ မော်ဒယ်များကို ကျော်လွန်ပြီး နှစ်ဆအရွယ် အပြင်ပိုမိုကောင်းမွန်သော မော်ဒယ်များနှင့် ညီမျှသည်။

### Phi-3 / 3.5 Vision

Phi-3/3.5 Instruct မော်ဒယ်ကို Phi ၏ နားလည်မှုစွမ်းအားအဖြစ် ယူဆနိုင်ပြီး Vision သည် Phi သည် ပတ်ဝန်းကျင်ကို မြင်သည့်မျက်လုံးပေးသည်။

**Phi-3-Vision**

Phi-3-vision သည် 4.2B ပါရာမီတာသာရှိသော်လည်း Claude-3 Haiku နှင့် Gemini 1.0 Pro V ကဲ့သို့သော အကြီးစား မော်ဒယ်များ ထက် general visual reasoning၊ OCR၊ ကြားနှင့်ဇယား နားလည်မှု တာဝန်များတွင် ကျော်လွန်တင်စားထားသည်။

**Phi-3.5-Vision**

Phi-3.5-Vision သည် Phi-3-Vision ၏ အဆင့်မြှင့်တင်မှု ဖြစ်ပြီး ပုံများစွာ ထောက်ပံ့နိုင်သည်။ ၎င်း သည် ပုံများသာ မမြင်ဘဲ ဗီဒီယိုများကိုလည်း မြင်နိုင်သည်ဟု ယူဆနိုင်သည်။

Phi-3.5-vision သည် Claude-3.5 Sonnet နှင့် Gemini 1.5 Flash ကဲ့သို့သော မော်ဒယ်ကြီးများကို OCR၊ ဇယားနားလည်မှုတွင် ကျော်လွန်ပြီး general visual knowledge reasoning တွင် ညီမျှသည်။ multi-frame input ကို ထောက်ပံ့ထားပြီး အထာများစွာသည် ဂဏန်းပုံများပေါ်တွင် သက်ဆိုင်ရာ မှတ်ချက်ဆုများ ထုတ်ပေးနိုင်သည်။

### Phi-3.5-MoE

***Mixture of Experts (MoE)*** သည် မော်ဒယ် သင်ကြားမှုတွင် စွမ်းဆောင်မှုအနည်းငယ်သာအသုံးပြု၍ မော်ဒယ် သို့မဟုတ် ဒေတာအရွယ်အစားကို ထူးခြားစေသည်။ MoE မော်ဒယ်က Dense မော်ဒယ်နှင့် တူညီသော အရည်အသွေးကို Pretraining တိုင်း အဖျိတ်တွင် မြန်ဆန်စွာ ရရှိစေသည်။

Phi-3.5-MoE သည် 16x3.8B ကျွမ်းကျင်သူ မော်ဂျူးများပါရှိသည်။ 6.6B စက်တင်၏ active parameters ဖြင့် Phi-3.5-MoE သည် ကြီးမားသော မော်ဒယ်များနှင့်တလံတကြ မတက်ကူးသော စွမ်းဆောင်ရည်၊ ဘာသာစကားနားလည်မှု နှင့် သင်္ချာဆိုင်ရာလေ့လာမှု ပြုလုပ်နိုင်သည်။

**NVIDIA NIM**

Azure နှင့် GitHub က ကောင်းမွန်သော Model Catalog တည်ဆောက်မှုကို ပေးစွမ်းသော်လည်း NVIDIA NIM ကိုလည်း အသုံးပြု၍ Phi-3/3.5 မော်ဒယ် API ခေါ်သုံးနိုင်ပါသည်။ NVIDIA NIM (NVIDIA Inference Microservices) သည် clouds, data centers, workstations အပါအဝင် အသုံးပြုသူများအတွက် AI မော်ဒယ်များကို မြန်ဆန်ထိရောက်စွာ တပ်ဆင်နိုင်ရန် အကူအညီပြုသည့် လျှပ်စစ်အကျိုးဆောင် inference microservices စနစ်တစ်ခုဖြစ်သည်။

NVIDIA NIM ၏ အဓိက အင်္ဂါရပ်များ -

- **တပ်ဆင်ရလွယ်ကူမှု** - တစ်ချက်အလိုက် ပို့ကုန် ဖြင့် AI မော်ဒယ်များ တပ်ဆင်၍ လုပ်ငန်းဆောင်တာများ တွဲဖက် စနစ်တကျ ထည့်သွင်းနိုင်သည်။
- **စွမ်းဆောင်ရည် မြှင့်တင်မှု** - NVIDIA ၏ TensorRT နှင့် TensorRT-LLM ကဲ့သို့ pre-optimized inference engine များသုံးပြီး Latency နည်း၊ throughput မြင့်စေသည်။
- **တိုးချဲ့မှု** - Kubernetes ပေါ်တွင် autoscaling ထောက်ပံ့မှုရှိပြီး workload မျိုးစုံကို ထိရောက်စွာ ကိုင်တွယ်နိုင်သည်။
- **လုံခြုံမှုနှင့် ထိန်းချုပ်မှု:** အဖွဲ့အစည်းများသည် မိမိတို့၏ ပိုင်ဆိုင်မှုရှိသော တာဝန်ခံအောက်တွင် NIM မိုက်ခရိုဝန်ဆောင်မှုများကို ကိုယ်ပိုင် စီမံခန့်ခွဲမှုပေါ်တွင် မိမိတို့ ဒေတာနှင့် အက်ပ်လီကေးရှင်းများကို ထိန်းချုပ်နိုင်သည်။
- **စံချိန်တပ်သော API များ:** NIM သည် စက်မှုလုပ်ငန်းစံချိန် API များကို ပေးဆောင်ပြီး၊ chatbot များ၊ AI အကူအညီများ နှင့် အခြား AI အက်ပ်များ ဖန်တီးခြင်းနှင့် ပေါင်းစပ်ခြင်းကို လွယ်ကူစေသည်။

NIM သည် NVIDIA AI Enterprise ၏ အစိတ်အပိုင်းတစ်ခုဖြစ်ပြီး၊ AI မော်ဒယ်များကို ထည့်သွင်းအသုံးပြုခြင်းနှင့် စနစ်တကျ လည်ပတ်နိုင်ရေး ပြုလုပ်ပေးသည့်အတွက် NVIDIA GPU များ ပေါ်တွင် ထိရောက်စွာ လည်ပတ်စေရန် ရည်ရွယ်သည်။

- အစမ်းပြ လုပ်ငန်း: NVIDIA NIM ကို အသုံးပြု၍ Phi-3.5-Vision-API ကို ခေါ်ယူခြင်း [[Click this link](./python/Phi-3-Vision-Nividia-NIM.ipynb?WT.mc_id=academic-105485-koreyst)]


### Phi-3/3.5 ကို ဒေသတွင်းတွင် ဆောင်ရွက်ခြင်း
Phi-3 သို့မဟုတ် GPT-3 ကဲ့သို႔သော ဘာသာစကားမော်ဒယ်တိုင်း၏ နောက်ခံအဓိပ္ပါယ်သတ္တိဖြစ်သော Inference သည် ဝင်ရောက်လာသောအချက်အလက်အပေါ် မူတည်၍ တုံ့ပြန်ချက်များ သို့မဟုတ် ကြိုတင်ခန့်မှန်းချက်များ ဖန်တီးခဲဲသည့် လုပ်ငန်းစဉ်ကို ရည်ညွှန်းသည်။ သင် Phi-3 ထံတွင် စာမေးပွဲ သို့မဟုတ် မေးခွန်းပေးသောအခါ၊ ၎င်း၏ သင်ကြားထားသော နယ်သာလန်ကွန်ရက်ဖြင့် သင်္ချာဆိုင်ရာ ပုံစံများနှင့် ဆက်စပ်မှုများကို ချဲ့ထွင်၍ မျိုးဆက် သွားနိုင်သည့် အကြောင်းအရာ နှင့် အတူ စကားရပ်များကို တုံ့ပြန်ပေးသည်။

**Hugging Face Transformer**
Hugging Face Transformers သည် သဘာဝဘာသာစကားကို ကိုင်တွယ်နည်း (NLP) နှင့် စက်သင်ယူမှု လုပ်ငန်းများအတွက် ခိုင်မာသော စာကြည့်တိုက်တစ်ခုဖြစ်သည်။ အဓိက ဆောင်းပါးအချက်များမှာ-

1. **ကြိုသင်ကြားပြီးသား မော်ဒယ်များ** - စာသား ခွဲခြားခြင်း၊ အမည်တပ် သက်သေခံခြင်း၊ မေးလေရာျခြင်း၊ အကျဉ်းချုပ်ရေးခြင်း၊ ဘာသာပြန်ခြင်းနှင့် စာသား ဖန်တီးခြင်း စသည်တို့အတွက် အသုံးပြုနိုင်သော ကြိုသင်ကြားပြီးသား မော်ဒယ် စေရာင်းများရှိသည်။
2. **ဖရိမ်ဝတ်အမျိုးမျိုးမတူညီမှု** - ဒီစာကြည့်တိုက်သည် PyTorch၊ TensorFlow နှင့် JAX ကဲ့သို႔သော အမျိုးမျိုးသော နက်ရှိုင်းတဲ့ သင်ယူရေး ဖရိမ်ဝတ်များကို ပံ့ပိုးပေးသည်။ ဤကာလတွင် မော်ဒယ်ကို တစ်ဖရိမ်ဝတ်တွင် သင်ကြားပြီး၊ အခြားဖရိမ်ဝတ်တွင် အသုံးပြုနိုင်သည်။
3. **မျိုးစုံ မျက်နှာစာရင်းစွမ်းရည်များ** - NLP အပြင် Hugging Face Transformers သည် ဓာတ်ပုံ နှင့် အသံ ဆိုင်ရာ ဆောင်ရွက်မှုများကိုလည်း ပံ့ပိုးပေးသည် (ဥပမာ - ဓာတ်ပုံ ခွဲခြားခြင်း၊ ပစ္စည်း ရှာဖွေရေး၊ အသံ မှတ်မိခြင်း၊ အသံ ခွဲခြားခြင်း)။
4. **အသုံးပြုရ လွယ်ကူမှု** - မော်ဒယ်များကို လွယ်ကူစွာ ဒေါင်းလုပ် ဆွဲ၍ အပြုပြင် ပြောင်းလဲနိုင်ရန် API များနှင့် ကိရိယာများ ပေးထားသည်၊ စတင်သမားနှင့် ကျွမ်းကျင်သူနှစ်ဖက်စလုံးအတွက် လွယ်ကူသည်။
5. **အသိုင်းအဝိုင်းနှင့် ရင်းမြစ်များ** - Hugging Face သည် တက်ကြွသော အသိုင်းအဝိုင်းနှင့် ကျယ်ပြန့်သော စာရွက်စာတမ်းများ၊ သင်ခန်းစာများ၊ လမ်းညွှန်မှုများရှိပြီး အသုံးပြုသူများ အကောင်းဆုံး အသုံးချနိုင်ရန် ကူညီပေးသည်။
[အချိုးကျ စာရွက်စာတမ်းများ](https://huggingface.co/docs/transformers/index?WT.mc_id=academic-105485-koreyst) သို့မဟုတ် [GitHub ကိုယ်စားလှယ်သိုလှောင်တိုက်](https://github.com/huggingface/transformers?WT.mc_id=academic-105485-koreyst)။

**Ollama**
[Ollama](https://ollama.com/?WT.mc_id=academic-105485-koreyst) သည် မိမိစက်ပေါ် တွက်ချက်နိုင်ရန် အကြီးစား ဘာသာစကား မော်ဒယ်များ(Large Language Models) ကို ပို၍ လွယ်ကူစွာ ပြုလုပ်ရန် ဒီဇိုင်းလုပ်ထားသော မျက်မှောက်ပလပ်ဖောင်းတစ်ခုဖြစ်သည်။ Llama 3.1၊ Phi 3၊ Mistral၊ Gemma 2 စသည်တို့ကဲ့သို့သော မော်ဒယ် များကို ပံ့ပိုးပေးပြီး၊ မော်ဒယ် အလေးချိန်များ၊ ဖွဲ့စည်းမှု၊ ဒေတာများကို တစ်ခုတည်း ပက်ကေ့ချ်အဖြစ် ထည့်သွင်းပေးသည်။ ယင်းကူညီမှုကြောင့် မော်ဒယ်ကို အသုံးပြုသူများ အဆင်ပြေစွာ မိမိတို့ စိတ်ကြိုက်ပြင်ဆင်နိုင်ပြီး ကိုယ်ပိုင် မော်ဒယ်ဖန်တီးနိုင်သည်။ Ollama သည် macOS၊ Linux နှင့် Windows များပေါ်တွင် ရရှိနိုင်သည်။ cloud ဝန်ဆောင်မှုများ မပါဘဲ LLM များကို စမ်းသပ် သို့မဟုတ် ဖြန့်ဖြူးလိုသူများအတွက် လုပ်ဆောင်ရ အထူးသင့်တော်သော ကိရိယာဖြစ်သည်။ Ollama ကို တိုက်ရိုက် အသုံးပြုရန် အောက်ပါ command ကို အလုပ်ပေးရန် လိုအပ်သည်။

**ONNX Runtime for GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) သည် မတူညီသော ပလက်ဖောင်းများပေါ်တွင် နေရာတိုင်းတွင် inference နှင့် သင်ကြားမှု စက်သင်ယူမှု အမြန်ဆန်စေသူ တစ်ခုဖြစ်သည်။ ONNX Runtime for Generative AI (GENAI) သည် မတူညီသော ပလက်ဖောင်းများပေါ်တွင် generative AI မော်ဒယ်များကို ထိရောက်စွာ လည်ပတ်နိုင်စေရန် အားကောင်းသော ကိရိယာဖြစ်သည်။

## ONNX Runtime ဆိုတာ ဘာလဲ?
ONNX Runtime သည် စက်သင်ယူမှု မော်ဒယ်များ၏ အမြန်ဆန်သော inference ကို ပံ့ပိုးပေးသည့် အဖွဲ့အစည်းဗွီဒီယိုအရင်းအမြစ် စီမံကိန်းတစ်ခု ဖြစ်သည်။ ၎င်းသည် Open Neural Network Exchange (ONNX) ပုံစံဖြင့် မော်ဒယ်များကို ပံ့ပိုးပေးပြီး၊ စက်သင်ယူမှု မော်ဒယ်များကို တိုင်းတာရေးစံချိန်ထားသော ပုံစံဖြစ်သည်။ ONNX Runtime inference သည် ဖောက်သည်အတွေ့အကြုံ အမြန်ဆန်စေရန် နှင့် ကုန်ကျစရိတ်လျော့နည်းစေရန် အထောက်အကူပြုကာ PyTorch၊ TensorFlow/Keras ကဲ့သို့သော နက်ရှိုင်းသင်ယူမှုဖရိမ်၀တ်များနှင့် အတူတူ classic machine learning စာကြည့်တိုက်များဖြစ်သည့် scikit-learn၊ LightGBM၊ XGBoost စသည်တို့မှ မော်ဒယ်များကို ပံ့ပိုးပေးသည်။ ONNX Runtime သည် မတူညီသော စက်ပစ္စည်းများ၊ ဒိုင်ဗာများနှင့် စက်ရုပ်စနစ်များနှင့် ကိုက်ညီပြီး၊ စက်ပစ္စည်းအမြန်နှုန်းမြှင့်တင်ပေးသည့် accelerator များနှင့် ပေါင်းစပ်၍ ကောင်းမွန်စွာ ဒီဇိုင်းချထားသည်။

## Generative AI ဆိုတာဘာလဲ?
Generative AI သည် သင်ကြားမှု ရရှိထားသော ဒေတာမှ အခြေခံ၍ စာသား၊ ဓာတ်ပုံ၊ သီချင်း စသည်တို့ကို ဖန်တီးနိုင်သော AI စနစ်များဖြစ်သည်။ ဥပမာများတွင် GPT-3 ကဲ့သို့သော ဘာသာစကား မော်ဒယ်များနှင့် Stable Diffusion ကဲ့သို့သော ဓာတ်ပုံ ဖန်တီးရေး မော်ဒယ်များ ပါဝင်သည်။ ONNX Runtime for GenAI စာကြည့်တိုက်သည် ONNX မော်ဒယ်များ အတွက် generative AI ဝင်ရိုးကို ပံ့ပိုးပြီး၊ ONNX Runtime ဖြင့် inference ပြုလုပ်ခြင်း၊ logits ကို စီမံခန့်ခွဲခြင်း၊ ရှာဖွေရေး နဲ့ စမ်းသပ်မှုများ၊ KV cache စီမံခန့်ခွဲမှုစသဖြင့် ပါဝင်သည်။

## ONNX Runtime for GENAI
ONNX Runtime for GENAI သည် ONNX Runtime ၏ စွမ်းရည်ကို generative AI မော်ဒယ် ပံ့ပိုးနိုင်ရန် တိုးချဲ့ထားသည်။ အဓိက ဖူးစာများမှာ-

- **ကျယ်ပြန့်သော ပလက်ဖောင်း ပံ့ပိုးမှု** - Windows၊ Linux၊ macOS၊ Android နှင့် iOS စသည်တို့တွင် အလုပ်လုပ်သည်။
- **မော်ဒယ် ပံ့ပိုးမှု** - LLaMA ၊ GPT-Neo ၊ BLOOM နှင့် အခြား လူကြိုက်များသော generative AI မော်ဒယ်များကို ထောက်ပံ့သည်။
- **စွမ်းဆောင်ရည် တိုးတက်မှု** - NVIDIA GPU များ၊ AMD GPU များ စသဖြင့် hardware accelerator များအတွက် တိုးတက်မှုများပါရှိသည်။
- **အသုံးပြုရ လွယ်ကူမှု** - API များဖြင့် အက်ပ်လီကေးရှင်းများတွင် လွယ်ကူစွာ ပေါင်းစပ်ရန် ခွင့်ပြုသည့်အတွက် minimal ကိုဒ်ဖြင့် စာသား၊ ဓာတ်ပုံ နှင့် အခြား ပါဝင်သည့် အကြောင်းအရာများ ဖန်တီးနိုင်သည်။
- အသုံးပြုသူများသည် generate() ဆိုသည့် အဆင့်မြင့်လုပ်ဆောင်ချက်ကို ခေါ်ယူနိုင်ပြီး၊ မော်ဒယ်ကို တစ်ဆင့်စီ လည်ပတ်စေ၍ တစ်ကြိမ်လျှင် တိမ်ကွက် (token) တစ်ခု ဖန်တီးပေးနိုင်ပြီး၊ လည်ပတ်မှုအတွင်း စီမံခန့်ခွဲမှု parameter များပြင်ဆင်နိုင်သည်။
- ONNX runtime သည် greedy/beam search နှင့် TopP၊ TopK sampling ဖြင့် token စဉ်များဖန်တီးခြင်း နှင့် ပြန်လည်ထပ်ဖြေခြင်း အပြစ်ဒဏ်ကဲ့သို့သော logits စီမံခန့်ခွဲမှု ထည့်သွင်းထားသည်။ စိတ်ကြိုက် အမှတ်ပေးခြင်းကိုလည်း လွယ်ကူစွာ ထည့်သွင်းနိုင်သည်။

## စတင်အသုံးပြုရန်
ONNX Runtime for GENAI ဖြင့် စတင်လိုပါက အောက်ပါအဆင့်များကို လိုက်နာနိုင်သည်-

### ONNX Runtime ကို တပ်ဆင်ရန် :
```Python
pip install onnxruntime
```
### Generative AI Extension များတပ်ဆင်ရန်:
```Python
pip install onnxruntime-genai
```

### မော်ဒယ် လည်ပတ်မှု - Python အတွက် ရိုးရှင်းသော ဥပမာ :
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### အစမ်းပြ : ONNX Runtime GenAI ကို အသုံးပြု၍ Phi-3.5-Vision ကို ခေါ်ယူခြင်း


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
