# Prompt Engineering အခြေခံများ

[![Prompt Engineering Fundamentals](../../../translated_images/my/04-lesson-banner.a2c90deba7fedacd.webp)](https://youtu.be/GElCu2kUlRs?si=qrXsBvXnCW12epb8)

## နိဒါန်း
ဤ module တွင် generative AI မော်ဒယ်များအတွက် ထိထိရောက်ရောက် prompt များဖန်တီးရန် အရေးကြီးသော စကားလုံးများနှင့် နည်းဗျူဟာများကို ဖော်ပြပါသည်။ သင့်သည် LLM အတွက် ရေးသားသော prompt ဘယ်လို မဆို အရေးပါသည်။ သေချာစွာတည်းဖြတ်ထားသော prompt တစ်ခုက ပိုမိုအရည်အသွေးမြင့်သော တုံ့ပြန်မှုကို ရရှိစေနိုင်ပါတယ်။ ဒါပေမယ့် _prompt_ နဲ့ _prompt engineering_ လို့ခေါ်တဲ့ စကားလုံးတွေက တကယ်ဘာများကို ဖော်ပြတာလဲ? LLM ကို ပို့တဲ့ prompt _input_ ကို ဘယ်လိုတိုးတက်အောင် လုပ်နိုင်မလဲ? ဤအခန်းနှင့် နောက်ထပ်အခန်းတွင် ဝေဖန်ဖြေရှင်းရန် ကြိုးစားပါမည်။

_Generative AI_ သည် အသုံးပြုသူတောင်းဆိုချက်များအတွက် စာသား၊ ပုံများ၊ အသံများ၊ ကုဒ်များ စသည်ဖြင့် အသစ်ဖန်တီးနိုင်သည်။ ၎င်းသည် OpenAI ၏ GPT ("Generative Pre-trained Transformer") စီးရီးကဲ့သို့သော _Large Language Models_ များကို သဘာဝဘာသာစကားနှင့် ကုဒ်အသုံးပြုမှုအတွက် လေ့ကျင့်ထားခြင်းဖြင့် ဒါကို ဆောင်ရွက်ပါသည်။

အသုံးပြုသူများသည် ယခုအခါ နည်းပညာပိုင်းကျွမ်းကျင်မှု သို့မဟုတ် လေ့ကျင့်မှုမလိုဘဲ စကားပြောဆက်သွယ်ခြင်းလို ပုံစံများဖြင့် ဤမော်ဒယ်များနှင့် ဆက်သွယ်နိုင်ပါသည်။ မော်ဒယ်များသည် _prompt-based_ ဖြစ်ပြီး - အသုံးပြုသူသည် စာသားအရိပ်အမြွက် (prompt) ပေးပို့ပြီး AI ၏ တုံ့ပြန်ချက် (completion) ကို ရရှိပါသည်။ ထို့နောက် "AI နှင့် စကားပြောသည်" ကို များပြားသော အချိန်ခွဲစကားပြောခြင်းဖြင့် ဆက်လက်လုပ်ဆောင်နိုင်ပြီး တုံ့ပြန်ချက်သည် မျှော်မှန်းချက်များနှင့် ကိုက်ညီသည်ထိ prompt ကို တိုးတက်ပြုပြင်နိုင်ပါသည်။

"Prompts" သည် ယခု တွင် generative AI အက်ပ်များအတွက် အဓိက _ပရိုဂရမ်မင်း အင်တာဖေ့စ်_ ဖြစ်လာပြီး မော်ဒယ်များကို ဘာလုပ်ရမည်ကို ပြောပြီး ပြန်လာသော တုံ့ပြန်ချက်အရည်အသွေးကို သက်ရောက်စေပါသည်။ "Prompt Engineering" သည် prompt များ၏ _ဒီဇိုင်းနှင့် တိုးတက်အောင် လုပ်ဆောင်ခြင်း_ အတွက် အာရုံစိုက်သော လေ့လာမှု ခြေလှမ်းမြန်မြန် တိုးတက်လာသော နယ်ပယ်ဖြစ်သည်။

## သင်ယူရမည့် ရည်မှန်းချက်များ

ဤသင်ခန်းစာတွင် Prompt Engineering သည်ဘာလဲ၊ အရေးကြီးချိန်တွင် ဘာကြောင့်လိုအပ်သနည်း၊ မော်ဒယ်နှင့် အက်ပ်ဆိုင်ရာ ရည်မှန်းချက်အတွက် ပိုမိုထိရောက်သော prompt များကို ဘယ်လို ဖန်တီးနိုင်သနည်းကို သင်ယူပါမည်။ Prompt engineering အတွက် အဓိကအယူအဆများနှင့် အကောင်းဆုံး လေ့လာမှုများကို နားလည်ဦးမည် - ၎င်းတွင် interactive Jupyter Notebooks "sandbox" ပတ်ဝန်းကျင်ကို ကြည့်ရှုလုပ်ဆောင်ခြင်းဖြင့် ဗဟုသုတရရှိပါမည်။

သင်ခန်းစာအဆုံးတွင် ကျွန်တော်တို့ အောက်ပါအရာများကို ရနိုင်ပါမည် -

1. Prompt engineering သည်ဘာလဲ၊ ဘာကြောင့် အရေးကြီးသည်ကို ရှင်းပြနိုင်ခြင်း။
2. Prompt ၏ အစိတ်အပိုင်းများကို ဖော်ပြခြင်းနှင့် ၎င်းတို့ကို မည်သို့ အသုံးပြုသနည်း။
3. Prompt engineering အတွက် အကောင်းဆုံးနည်းလမ်းများနှင့် နည်းဗျူဟာများကို ကျင့်သုံးခြင်း။
4. OpenAI endpoint သုံးပြီး သင်ယူထားသည့် နည်းလမ်းများကို ရှေ့ပြေးနမူနာများတွင် အသုံးပြုခြင်း။

## အဓိကစကားလုံးများ

Prompt Engineering: AI မော်ဒယ်များကို မျှော်လင့်ထားသော ထွက်ရှိမှုများထုတ်လုပ်ရန် ဦးတည်စေဖို့ input များကို ဒီဇိုင်းဆွဲပြီး တိုးတက်အောင်လုပ်ခြင်းလုပ်ငန်းစဉ်။
Tokenization: စာသားကို အသေးအပိုင်းများဖြစ်သည့် tokens ဟုခေါ်သော အပိုင်းများသို့ ပြောင်းလဲခြင်းဖြင့် မော်ဒယ်တစ်ခုသိရှိနိုင်ပြီး လုပ်ဆောင်နိုင်သော လုပ်ငန်းစဉ်။
Instruction-Tuned LLMs: တိကျချဉ်းကပ်မှုများနှင့် သက်ဆိုင်သည့် တုံ့ပြန်မှုတိုးတက်အောင် အညွှန်းများဖြင့် ပြုပြင်ထားသော အကြီးစားဘာသာစကားမော်ဒယ်များ (LLMs)။

## သင်ယူမှု Sandbox

Prompt engineering သည် ယခုအခါ အတတ်ပညာပိုပြီး သိပ္ပံပိုအနည်းငယ် ဖြစ်သည်။ ၎င်းအတွက် အမြင်မြင်ရမှုပိုမိုတိုးတက်ရန် လေ့ကျင့်မှုများဆက်လုပ်ရန် နှင့် လျှောက်လွှဲမှုနယ်ပယ် ကျွမ်းကျင်မှုများ၊ အကြံပြုနည်းများနှင့် မော်ဒယ်ပိုင်ဆိုင်မှုများကို ပေါင်းစပ်၍ စမ်းသပ်ခြင်းလုပ်သင့်သည်။

ဤသင်ခန်းစာအတွက် Jupyter Notebook သည် သင်ယူထားသည့်အရာများကို စမ်းသပ်နိုင်သော _sandbox_ ပတ်ဝန်းကျင်ကို ပံ့ပိုးပေးသည်၊ သက်ဆိုင်ရာ လေ့ကျင့်ခန်းများဆောင်ရွက်နိုင်ရန် အောက်ပါ အရာများ လိုအပ်ပါမည် -

1. **Azure OpenAI API key** - တပ်ဆင်ထားသော LLM အတွက် ဝန်ဆောင်မှုပတ်ဝန်းကျင်။
2. **Python Runtime** - ထို Notebook ကို ဖျော်ဖြေလည်ပတ်နိုင်သော ပတ်ဝန်းကျင်။
3. **ပြည်တွင်း ပတ်ဝန်းကျင်အပြောင်းအလဲ (Local Env Variables)** - _အခု [SETUP](./../00-course-setup/02-setup-local.md?WT.mc_id=academic-105485-koreyst) အဆင့်များကို ပြီးစီးခြင်းဖြင့် ပြင်ဆင်ပါ_။

Notebook တွင် _စတင်ရန်_ လေ့ကျင့်ခန်းများ ပါဝင်၍ သင်၏ _Markdown_ (ဖော်ပြချက်) နှင့် _Code_ (prompt တောင်းဆိုမှု) ပိုင်းများကို ထပ်ဆောင်းထည့်သွင်းကာ နမူနာများ သို့မဟုတ် အကြံပြုချက်များကို စမ်းသပ်ပြုလုပ်၍ prompt ဒီဇိုင်းအတွက် အမြင် များတိုးပြောနိုင်ပါသည်။

## ပုံဆွဲ ဗဟုသုတ

ဤသင်ခန်းစာတွင် လေ့လာမည့် အကြောင်းအရာအကျဉ်းကို ကြည့်ရန် စိတ်ရှိပါသလော? ဤပုံဆွဲ ဗဟုသုတကြီးကို ကြည့်ပါ၊ ၎င်းသည် အဓိကခေါင်းစဉ်များနှင့် သင်အတွက် စဉ်းစားရန် အချက်များကို ဖော်ပြထားသည်။ သင်ခန်းစာလမ်းညွှန်သည် အခြေခံ အယူအဆများနှင့် စိန်ခေါ်မှုများကို နားလည်ပြီး ဒီထက်ပြဋ္ဌာန်းထားသော prompt engineering နည်းလမ်းများနှင့် အကောင်းဆုံးလေ့လာမှုများဖြင့် ဖြေရှင်းပေးနိုင်ပါသည်။ ထို့အပြင် ဤလမ်းညွှန်မှ "အဆင့်မြင့် နည်းလမ်းများ" အပိုင်းသည် မကြာမီအတွဲအခန်းတွင် စွဲသွင်းထားသော အကြောင်းအရာများကို ရည်ညွှန်းပါသည်။

## ကျွန်ုပ်တို့၏ စတားတပ်တည်နေရာ

ယခု မှတ်ချက်အတိုင်း _ဤခေါင်းစဉ်_ သည် [ပညာရေးတွင် AI ဆန်းသစ်တီထွင်မှုကို တင်ဆက်ခြင်း](https://educationblog.microsoft.com/2023/06/collaborating-to-bring-ai-innovation-to-education?WT.mc_id=academic-105485-koreyst) မစ်ရှင်နှင့် ဘယ်လိုဆက်နွယ်သနည်းဟု ပြောကြမယ်။ ကျွန်တော်တို့သည် _ပုဂ္ဂိုလ်ရေးသင်ယူမှု_ အတွက် AI အားဖြည့်ထားသော အက်ပ်များကို ဖန်တီးချင်သဖြင့် ကျွန်တော်တို့အက်ပ်အသုံးပြုသူများထဲမှာ မည်သူတွေ ဘယ်လို "prompt" ဖန်တီးနိုင်သလဲ ဆိုတာကို တွေးကြည့်ကြရအောင် -

- **အုပ်ချုပ်ရေးမှူးများ** သည် AI ကို _သင်ယူမှု အစီအစဉ်ဒေတာများအားလုံးကို ဖလှယ်အနေအထားများ ရှာဖွေရန် စိစစ်ရန်_ တောင်းနိုင်သည်။ AI သည် နမူနာများ သို့မဟုတ် ကုဒ်များဖြင့် အကျဉ်းချုပ် လုပ်နိုင်သည်။
- **ပညာရေးသမားများ** သည် AI ကို _ရည်မှန်းထားသော ပရိသတ်နှင့် ခေါင်းစဉ်အတွက် သင်ခန်းစာ အစီအစဉ် ဖန်တီးရန်_ တောင်းနိုင်သည်။ AI သည် သတ်မှတ်ထားသော ဖော်ပြချက်တွင် ပုဂ္ဂိုလ်ရေးအစီအစဉ်ကို တည်ဆောက်နိုင်သည်။
- **ကျောင်းသားများ** သည် AI ကို _ခက်ခဲသောဘာသာရပ်တွင် ကိုယ်ပိုင်သင်ကြားမှု စောင့်ကြည့်ရန်_ လိုအပ်သည်။ AI သည် အခုချိန်မှာ ကျောင်းသားများအတွက် အဆင့်တင့်မီသော သင်ခန်းစာများ၊ အကြံပြုချက်များနှင့် နမူနာများ ပေးသည်။

၎င်းသည် မှတ်တမ်းအတွက် အစိတ်အပိုင်းတစ်ခုသာ ဖြစ်ပါသည်။ [Prompts For Education](https://github.com/microsoft/prompts-for-edu/tree/main?WT.mc_id=academic-105485-koreyst) ကို ကြည့်ပါ - ပညာရေး ကျွမ်းကျင်သူများက စုစည်းထားသော အခမဲ့ prompt စာကြည့်တော် များဖြစ်ပြီး ဖြစ်နိုင်ခြေများကို စုပေါင်းကြည့်ရှုနိုင်သည်။ _sandbox တွင် prompt များကို အသုံးပြု၍ ဆက်စပ်မြင်ကွင်းများကို စမ်းသပ်လေ့လာပါ သို့မဟုတ် OpenAI Playground တွင် စမ်းသပ်ကြည့်ပါ!_

## Prompt Engineering ဆိုတာဘာလဲ?

ကျွန်တော်တို့ သင်ခန်းစာကို "Prompt Engineering" ဟူ၍ သတ်မှတ်တယ်၊ ပြီးတော့ _ဒီဇိုင်းဆွဲခြင်း နှင့် တိုးတက်အောင် လုပ်ခြင်း_ မြင်သာသော တုံ့ပြန်ချက်များ (completion) ကို ထုတ်ပေးရန် ချမှတ်ထားသော application ရည်မှန်းချက်နှင့် မော်ဒယ်အတွက် စာသား input များ (prompt) ဖြစ်သည်။ နှစ်ဆင့်လုပ်ငန်းစဉ်အနေနှင့် စဉ်းစားနိုင်သည် -

- အဓိက prompt ကို တစ်ခုချင်း မော်ဒယ်နှင့် ရည်မှန်းချက်အတွက် _ဒီဇိုင်းဆွဲခြင်း_။
- တုံ့ပြန်ချက်၏ အရည်အသွေး တိုးတက်စေရန် prompt ကို မကြာခဏ _ပြုပြင်တိုးတက်အောင်_လုပ်ခြင်း။

၎င်းသည် အသုံးပြုသူ၏ အမြင်နှင့် ကြိုးပမ်းအားထုတ်မှုကို လိုအပ်သော စမ်းသပ်ခန့်မှန်းခြေရေး လုပ်ငန်းစဉ်ဖြစ်သည်။ ထို့ကြောင့် ဘာကြောင့် လိုအပ်သနည်း ကို ဖြေဆိုရန်သင့်ပါတယ်။ ဤမေးခွန်းကို ဖြေရှင်းရန် ဆောင်ရွက်ရမည့် အယူအဆသုံးခု ရှိသည်။

- _Tokenization_ = မော်ဒယ် "prompt ကို ဘယ်လိုမြင်" သလဲ။
- _Base LLMs_ = မော်ဒယ် မူလကျောင်းကော "prompt ကို ဘယ်လို လုပ်ငန်းဆောင်တာလုပ်" သလဲ။
- _Instruction-Tuned LLMs_ = မော်ဒယ်သည် _"အလုပ်အတွက်"_ ဘယ်လိုမြင်နေလဲ။

### Tokenization

LLM သည် prompt များကို _tokens အစဉ်အလာ_ အဖြစ် မြင်ကွင်းအားဖြင့် ကြည့်ပါသည်၊ မော်ဒယ်အမျိုးအစား သို့မဟုတ် မော်ဒယ်ဗားရှင်းများက တူညီသည့် prompt ကိုကွဲပြားသည့် နည်းလမ်းများဖြင့် token အဖြစ် ခွဲနိုင်သည်။ LLM များကို tokens များဖြင့် လေ့ကျင့်ထားသည်၊ စာသားတိုက်ရိုက်မဟုတ်သောကြောင့် prompt များ token ဆိုသည်နည်းပညာ သိရှိမှုသည် ပြန်လာသော တုံ့ပြန်ချက်အရည်အသွေး ထိန်းသိမ်းမှုကို တိုက်ရိုက် သက်ရောက်စေသည်။

Tokenization အလုပ်လုပ်ပုံနားလည်ရန်အတွက် [OpenAI Tokenizer](https://platform.openai.com/tokenizer?WT.mc_id=academic-105485-koreyst) ကဲ့သို့သော ကိရိယာများကို စမ်းသပ်ကြည့်ပါ။ သင်၏ prompt ကို ကူးထည့်၍ စာလုံးပေါင်းများအနေနှင့် စာလုံးချွတ်များ (whitespace) နှင့် စာလုံးပေါင်းဗဟိုများကို ဘယ်လို စီစဉ်ထားသလဲ စူးစမ်းကြည့်ပါ။ ဤနမူနာတွင် LLM ဟောင်း (GPT-3) ကို ပြသထားခြင်းဖြစ်သောကြောင့် ဖြိုးတိုးလာသော မော်ဒယ်တစ်ခုဖြင့် စမ်းသပ်ခြင်းကွဲပြားခြင်း အရလည်း ရနိုင်သည်။

![Tokenization](../../../translated_images/my/04-tokenizer-example.e71f0a0f70356c5c.webp)

### အယူအဆ: Foundation Models

prompt သည် tokenized ဖြစ်သည်နောက် "[Base LLM](https://blog.gopenai.com/an-introduction-to-base-and-instruction-tuned-large-language-models-8de102c785a6?WT.mc_id=academic-105485-koreyst)" (Foundation model) ၏ အဓိကလုပ်ဆောင်ချက်မှာ မျဉ်းစဉ်တွင်း အနောက်ဆုံး token ကို ခန့်မှန်းခြင်း ဖြစ်သည်။ LLM များသည် မိမိစကားလုံးများအတွက် အကြီးမားဆုံး စာအုပ်အချက်အလက်များဖြင့် လေ့ကျင့်ထားပြီး token များအကြား စံချိန်ဆွေမျိုး ဆက်နွယ်မှုများကို တုံ့ပြန်ချက် ခန့်မှန်းနိုင်ပါသည်။ ၎င်းတို့သည် prompt သို့မဟုတ် token တွင် ပါဝင်သော စာလုံးများ၏ _အဓိပ္ပာယ်_ ကို မနားလည်ကြပေ။ စကားလုံးများကို _ပြီးမြောက်_ လုပ်ရန် စည်းကမ်း pattern တစ်ခုသာ မြင်ကြသည်။ အသုံးပြုသူ ရပ်တန့်ရန် သို့မဟုတ် ထိုကဲ့သို့ အခြေအနေများကြောင့် ပြီးဆုံးသည်အထိ ခန့်မှန်းမှု များဆက်လက်လုပ်ဆောင်နိုင်သည်။

prompt-based completion အလုပ်လုပ်ပုံကို မြင်လိုပါသလား? အထက်ပါ prompt ကို [Microsoft Foundry playground](https://ai.azure.com?WT.mc_id=academic-105485-koreyst) တွင် ပုံမှန်ပြင်ဆင်ထားသည့် settings ဖြင့် ထည့်ပါ။ စနစ်သည် prompt များကို အချက်အလက်တောင်းဆိုမှုအဖြစ် သတ်မှတ်ထားသောကြောင့် ၎င်း အတိုင်း ကိုက်ညီသော response ကိုမြင်ရမည်ဖြစ်သည်။

အသုံးပြုသူသည် တစ်စိတ်တစ်ပိုင်း သို့မဟုတ် အလုပ်တစ်ခုအတွက် သတ်မှတ်ချက်ကို ဖြည့်ဆည်းလိုပါက? ဤမှာ _instruction-tuned_ LLM များ ထင်ရှားလာသည်။

![Base LLM Chat Completion](../../../translated_images/my/04-playground-chat-base.65b76fcfde0caa67.webp)

### အယူအဆ: Instruction Tuned LLMs

[Instruction Tuned LLM](https://blog.gopenai.com/an-introduction-to-base-and-instruction-tuned-large-language-models-8de102c785a6?WT.mc_id=academic-105485-koreyst) သည် foundation model ကို အခါအားလျော်စွာ ဥပမာများ သို့မဟုတ် input/output ကျောပေါ်ဇယားများ (multi-turn "messages" ထည့်သွင်းခြင်းအပါအဝင်) အသုံးပြုပြီး ပြုပြင်တိုးတက်စေသည်။ ၎င်းတွင် ရိုးရှင်းသော်သော် _အညွှန်းများ_ ပါရှိပြီး AI ၏ တုံ့ပြန်ချက်က ပေးထားသော အညွှန်းကို လိုက်နာရန်ကြိုးစားသည်။

၎င်းသည် Reinforcement Learning with Human Feedback (RLHF) ကဲ့သို့ နည်းဗျူဟာများကို အသုံးပြုကာ မော်ဒယ်ကို _အညွှန်းလိုက်နာရန်_ နှင့် _တုံ့ပြန်ချက်မှ သင်ယူရန်_ လေ့ကျင့်သည်၊ ထို့ကြောင့် ၎င်းသည် လက်တွေ့ပေါ် applications များနှင့် အသုံးပြုသူရည်မှန်းချက်များအတွက် ပိုမိုသင့်တော်သော တုံ့ပြန်ချက်များ ထုတ်လုပ်နိုင်စေသည်။

ပြန်လည်ကြည့်ရန် - အထက်ပါ prompt ကို နောက်တစ်ခါ ပြင်ဆင်ပါ၊ သို့သော် _system message_ ကို အောက်ပါ အညွှန်းဖြင့် အခြေအနေပြုလုပ်ပါ -

> _သင်ပေးထားသော အကြောင်းအရာကို ဒုတိယတန်းကျောင်းသားတစ်ယောက်အတွက် အကျဉ်းချုပ်ပါ။ ရလဒ်ကို ပုဒ်မတစ်ပိုဒ်တွင် ၃ မှ ၅ ချက်အထိ bullet point များဖြင့် ထားပါ။_

ရလဒ်သည် လိုအပ်သော ရည်မှန်းချက်နှင့် ပုံစံကို ပြန်လည်ပြသသည်ကို တွေ့နိုင်ပါသည်။ ပညာရေးသမားတစ်ဦးကို ဤတုံ့ပြန်ချက်ကို ၎င်း၏ slide များတွင် တိုက်ရိုက်သုံးနိုင်ပါသည်။

![Instruction Tuned LLM Chat Completion](../../../translated_images/my/04-playground-chat-instructions.b30bbfbdf92f2d05.webp)

## Prompt Engineering လိုအပ်ခြင်းရဲ့ အကြောင်းရင်း

သင့်အား LLM များက prompt များကို မည်သို့ကုန်ကြမ်းလုပ်ဆောင်သည်ကို သိအပ်လျှင်၊ Prompt Engineering လိုအပ်သော အကြောင်းရင်းကို ဖော်ပြပါ။ ယနေ့ LLM များတွင် တုံ့ပြန်ချက်များကို _ယုံကြည်စိတ်ချရမှုနှင့် လျှောက်လွှာတိကျမှု_ တွေမြှင့်တင်သွားရန် prompt ဖန်တီးခြင်းနှင့် တိုးတက်အောင်လုပ်ခြင်းမရှိမဖြစ်လိုအပ်စေသော စိန်ခေါ်မှုများ များပြားသောကြောင့် ဖြစ်သည်။ ဥပမာအားဖြင့် -

1. **မော်ဒယ်တုံ့ပြန်ချက်များသည် stochastic ဖြစ်သည်။** _တူညီသော prompt_ သည် မတူညီသော မော်ဒယ်များ သို့မဟုတ် မော်ဒယ်ဗားရှင်းများဖြင့် မတူညီသော တုံ့ပြန်ချက်များ ထုတ်နိုင်သည်။ နောက်တစ်ကြိမ်တွင်လည်း _တူညီသောမော်ဒယ်_ ဖြင့် မတူညီသော ရလဒ်များ ပေးနိုင်သည်။ _Prompt engineering နည်းလမ်းများက ဤကွာခြားချက်များကို လျှော့ချရန် ကူညီပေးသည်_။

1. **မော်ဒယ်များသည် တုံ့ပြန်ချက်များကို ဖန်တီးနိုင်သည်။** မော်ဒယ်များကို _ကြီးမားပြီး၊ ဒုတိယတန်းအထိသာ_ တစ်ရပ်ကန့်သတ်ထားသည့် ဒေတာစုစည်းမှုဖြင့် သင်ကြားထားသည်၊ ထို့ကြောင့် သူတို့သည် လက်ရှိသိပံဇ္များနှင့် မကိုက်ညီသော သို့မဟုတ် မရှိသော အချက်အလက်များ ထုတ်လုပ်နိုင်သည်။ _Prompt engineering သည် အသုံးပြုသူများကို ထိုကဲ့သို့သော ဖန်တီးမှုများအား ဖော်ထုတ်ကာ လျှော့ချနိုင်ရန် ကူညီပေးသည်၊ ဥပမာ AI ကို ကိုးကားချက်များသို့မဟုတ် အကြောင်းပြချက်များ မေးခြင်းဖြင့်_။

1. **မော်ဒယ်နိုင်ငံတော်စွမ်းဆောင်ရည်များကွာခြားနိုင်သည်။** အသစ်ဖြစ်သော မော်ဒယ်များ သို့မဟုတ် မော်ဒယ်မျိုးဆက် အသစ်များတွင် စွမ်းရည်များပိုမိုကြီးမားသော်လည်း မြန်နှုန်းနှင့် ချေးနှုန်းတွင် ဆက်စပ်ထူးခြားချက်များကိုပါ ယူသွားနိုင်သည်။ _Prompt engineering က ကွဲပြားချက်များအား ဖုံးကွယ်ကာ မော်ဒယ်အပေါ်မူတည်သည့် လိုအပ်ချက်များအား ဖြေရေး အကောင်းဆုံး လမ်းစဉ်များနှင့် အလုပ်ခြင်းများ ဖန်တီးကူညီပေးသည်_။

OpenAI သို့မဟုတ် Azure OpenAI Playground တွင် ဤခြေလှမ်းများကို ကြည့်ရှုလိုက်ပါ။

- မတူညီသော LLM deployment များဖြင့် (ဥပမာ OpenAI၊ Azure OpenAI၊ Hugging Face) တူညီသော prompt ကို သုံးပါ - ကွာခြားချက်များကို တွေ့ခဲ့ပါသလား။
- တူညီသော LLM deployment ဖြင့် မကြာခဏ (ဥပမာ Azure OpenAI playground) တူညီသော prompt သုံးပါ - ၎င်းကွာခြားချက်များက ဘယ်လိုကွဲပြားသနည်း။

### Fabrications နမူနာ

ဤသင်တန်းတွင် _"fabrication"_ ဆိုသည်မှာ LLM များမှ အချိန်အခါအခါ လေ့ကျင့်မှု အကန့်အသတ်များ သို့မဟုတ် အခြား ကန့်သတ်ချက်များကြောင့် မှန်ကန်မှုမရှိသော အချက်အလက်များ ထုတ်ဖော်သည်ကို ရည်ညွှန်းသည်။ လူသိများသောဆောင်းပါးများ သို့မဟုတ် သုတေသနစာတမ်းများတွင် _"hallucinations"_ ဟုလည်း ခေါ်ကြသည်။ သို့သော် ထိုမှတ်သားချက်မှ ညွှန်ပြချက်သည် စက်ဖြစ်ပေါ်မှုအတွက် လူ့လက္ခဏာမျိုးစွမ်းအား ပေးခြင်း ဖြစ်လာနိုင်သဖြင့် ကျွန်တော်တို့ _"fabrication"_ ဆိုသော စကားလုံးကို အသုံးပြုရန် အတည်ပြုလိုက်သည်။ ဤသည်မှာ [Responsible AI လမ်းညွှန်ချက်များ](https://www.microsoft.com/ai/responsible-ai?WT.mc_id=academic-105485-koreyst) နှင့် ပါတ်သက်ပြီး သဘောတူညီချက်ဖြစ်ပြီး အချို့ ပတ်ဝန်းကျင်များ၌ ကိုယ်တော်ကျူးသော သို့မဟုတ် မပါဝင်သင့်သော စကားများမှ ရှောင်ရှားရန် ဖြစ်သည်။

*Fabrications* များဘယ်လိုဖြစ်ပေါ်ကြသည်ကို နားလည်ရန် လိုပါသလား? AI ကို မရှိသေးသော ခေါင်းစဉ်အတွက် အကြောင်းအရာဖန်တီးရန် အမိန့်ပေးသည့် prompt တစ်ခု ကို တွေးကြည့်ပါ (ခဲ့သော training dataset မရှိစေရန်)။ ဥပမာ - ငါ ဒီ prompt ကို စမ်းသပ်ခဲ့တယ် -

ဝက်ဘ်ရှာဖွေမှုတွင် 2076 ခုနှစ်မှာ မရှိသော်လည်း စွဲချက်၊ စစ်ပွဲပေါ်အသံတိတ်တွေ (e.g., တီဗီစီးရီးများ သို့မဟုတ် စာအုပ်များ) ရှိကြောင်း တွေ့ရှိခဲ့သည်။ ကျွမ်းကျင်မှုဖြင့်လည်း 2076 သည် _အနာဂတ်_ ဖြစ်သောကြောင့် အမှန်တကယ်ဖြစ်သော ဖြစ်ရပ်နှင့် မဆိုင်ပါ။

မျှော်မှန်းထားသလို၊ မော်ဒယ်တိုင်း (သို့မဟုတ် မော်ဒယ်ဗားရှင်းတိုင်း) နည်းနည်းကွဲပြားတဲ့တုံ့ပြန်ချက်တွေ ထုတ်ပေးပါတယ်၊ သဘာဝဖြစ်စဉ်နှင့် မော်ဒယ်စွမ်းရည်ကွဲပြားမှုတွေကြောင့်ပါ။ ဥပမာ၊ မော်ဒယ်တစ်ခုက ၈ တန်းကျောင်းသားတွေအတွက် ရည်ရွယ်ထားပြီး တစ်ခုကတော့ အထက်တန်းကျောင်းသားတစ်ယောက်လို ထင်မြင်ထားပါတယ်။ ဒါပေမယ့် မော်ဒယ်သုံးခုလုံးက အတုမရှိတဲ့အသုံးပြုသူကို အကောင်အထည်ဖော်ခဲ့တဲ့ ဖြစ်ရပ်တစ်ခုဟု ယုံကြည်စွာ ပြောနိုင်စေတဲ့ တုံ့ပြန်ချက်တွေ ထုတ်ပေးခဲ့ပါတယ်။

_metaprompting_ နဲ့ _temperature configuration_ လို prompt engineering နည်းလမ်းတွေက မော်ဒယ်က အတုဖန်တီးမှုကို တစ်စုံတစ်ရာလျော့နည်းစေနိုင်ပါတယ်။ အသစ်ထွက် prompt engineering _architectures_ တွေကလည်း အသစ် tools နဲ့ နည်းလမ်းတွေကို prompt flow ထဲထည့်သွင်းပေးပြီး ထို သက်ရောက်မှုတွေကို လျော့ချပေးသွားနိုင်ပါတယ်။

## Prompt တည်ဆောက်ခြင်း

Prompt engineering ဘာကြောင့် အရေးကြီးတယ်ဆိုတာ မြင်ကြပြီးပါပြီ - ယခု prompt များကို _တည်ဆောက်ပုံ_ နားလည်ပြီး ပိုထိရောက်တဲ့ prompt design အတွက် နည်းလမ်းများကို အကဲဖြတ်ကြရအောင်။

### အခြေခံ prompt

အခြေခံ prompt ကို စတင်ကြည့်မယ် - မခြုံကော context မပါဘဲ မော်ဒယ်သို့ စာသား input တစ်ခု အဖြစ်ပို့ပေးတာ။ ဥပမာ - အမေရိကန်နိုင်ငံတော်သံဂီတပေးသဲစကားလုံးအချို့ကို OpenAI [Completion API](https://platform.openai.com/docs/api-reference/completions?WT.mc_id=academic-105485-koreyst) ကို ပို့လိုက်တာနဲ့ တုံ့ပြန်ချက်အနေနဲ့ နောက်ထပ်လိုင်းအနည်းငယ်နဲ့ ပြီးဆုံးပေးတာကို မြင်တွေ့နိုင်ပြီး အခြေခံကာယနစ်ခြယ်မှုကို ဖော်ပြပါတယ်။

| Prompt (Input)     | Completion (Output)                                                                                                                        |
| :----------------- | :----------------------------------------------------------------------------------------------------------------------------------------- |
| Oh say can you see | It sounds like you're starting the lyrics to "The Star-Spangled Banner," the national anthem of the United States. The full lyrics are ... |

### ရင်းရှင်းတဲ့ prompt

အခု အခြေခံ prompt ကို context နဲ့ညွှန်းချက်တွေနဲ့ တိုးချဲ့လိုက်ပါ။ [Chat Completion API](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/chatgpt?WT.mc_id=academic-105485-koreyst) ကတော့ _messages_ အစုအဖွဲ့တစ်ခုအနေနဲ့ ရှုပ်၍ prompt တစ်ခုပြုလုပ်ဖို့ ပံ့ပိုးပေးပါတယ်။

- _user_ရဲ့ input နဲ့ _assistant_ ရဲ့ output တို့ကို ထည့်သွင်းထားသော input/output စုံများ။
- assistant ၏ အပြုအမူ သို့မဟုတ် ပုဂ္ဂိုလ်ရေး အချက်အလက်များ ရှေ့ပြေးပေးသော system message။

စိတ်တိုင်းကျတုံ့ပြန်မှုအရ စနစ် context ပြောင်းလဲခြင်းသည် user input ထက် ပိုအစွမ်းထက် လုပ်ဆောင်မှုရှိစေခြင်း ဖြစ်ပါတယ်။

```python
response = client.responses.create(
    model="gpt-5-mini",
    input=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Who won the world series in 2020?"},
        {"role": "assistant", "content": "The Los Angeles Dodgers won the World Series in 2020."},
        {"role": "user", "content": "Where was it played?"}
    ]
)
```

### ညွှန်ကြားချက် prompt

အထက်ပါ အတိုင်း၊ user prompt က ရိုးရှင်းသော ကိစ္စမေးခြင်းတစ်ခုဖြစ်ပြီး အချက်အလက်တောင်းဆိုခြင်းတွေအဖြစ် ဖော်ပြနိုင်ပါသည်။ _instruction_ prompt များဖြင့်တော့ စာသားကို ပိုမိုအသေးစိတ် ပြောပြ၍ AI ကို ပိုထိုက်တန်သော ညွှန်ကြားချက် ပေးနိုင်ပါသည်။ ဥပမာ:

| Prompt (Input)                                                                                                                                                                                                                         | Completion (Output)                                                                                                        | Instruction Type    |
| :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------- | :------------------ |
| Write a description of the Civil War                                                                                                                                                                                                   | _returned a simple paragraph_                                                                                              | Simple              |
| Write a description of the Civil War. Provide key dates and events and describe their significance                                                                                                                                     | _returned a paragraph followed by a list of key event dates with descriptions_                                             | Complex             |
| Write a description of the Civil War in 1 paragraph. Provide 3 bullet points with key dates and their significance. Provide 3 more bullet points with key historical figures and their contributions. Return the output as a JSON file | _returns more extensive details in a text box, formatted as JSON that you can copy-paste to a file and validate as needed_ | Complex. Formatted. |

## အဓိက အကြောင်းအရာ

အထက်ပါဥပမာများတွင် prompt သည် မျက်နှာဖုံး ဖွင့်ထားသေး၍ LLM ကို သင်ယူထားသော ဒေတာအပိုင်းရဲ့ ဘာကို သက်ဆိုင်ကြောင်းဆုံးဖြတ်ခွင့် ပေးထားသည်။ _primary content_ design pattern တွင် input စာသားကို အပိုင်းနှစ်ပိုင်းခွဲစိတ်သည်-

- ညွှန်ကြားချက် (လုပ်ဆောင်ရန်)
- ဆောင်ရွက်မှုအပေါ် သက်ရောက်မှုရှိသည့် သက်ဆိုင်ရာ အကြောင်းအရာ

ဥပမာတစ်ခုမှာ "ဤစာကို ၂ စာကြောင်းနဲ့အကျဉ်းချုပ်ပါ" ဖြစ်ပါတယ်။

| Prompt (Input)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      | Completion (Output)                                                                                                                                                                                                                                                                             |
| :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Jupiter သည် နေမှငါးကြိမ်မြောက် ကြယ်ဖိုလ် ဖြစ်ပြီး ဆောလားစနစ်တွင်း အကြီးဆုံး ကြယ်ဖိုလ်ဖြစ်သည်။ နေ၏ ဒဏ်ငွေတစ်ထောင်ပုံတစ်ပုံသာရှိသော်လည်း ဆောလားစနစ်ရှိ အခြားကြယ်ဖိုလ်များအားလုံး ထပ်ပေါင်းရင် နှစ်နှစ်ရက်ခြမ်းပိုသည်။ Jupiter သည် ညနေခင်းက မျက်လုံး တောက်ပြောင်အလင်းတစ်ခုဖြစ်ပြီး သမိုင်းမှတ်တမ်းမရှိခင်ကတည်းက ဓာတုအမျိုးမျိုးအား စွမ်းဆောင်ခဲ့သော အကြောင်း ရှိပြီး ရောမဘုရင် Jupiter အမည်ဖြင့် လူသိများသည်။ [19] မြေပြင်မှ ကြည့်သောအခါ Jupiter ၏ ထပ်တလဲလဲပြန်တောက် ထိုအလင်းသည် မျက်နှာသားအသွင်တူ မျက်စိနှင့် မြင်နိုင်သည်။ [20] နေရဲ့နှင့် Venus အခြားလည်း နေ့ဘက်မှာ တတိယအလင်းတောက်ဆုံး အရာတစ်ခုဖြစ်သည်။ <br/> **ဤကို ၂ စာကြောင်းနဲ့ အကြောင်းအရာ အကျဉ်းချုပ်ပါ** | Jupiter သည် နေနောက်ဆုံးငါးမြောက် ဂြမ်ငယ်တစ်ခုဖြစ်ပြီး ဆောလားစနစ်တွင် အကြီးဆုံးဖြစ်၍ ညနေခင်းအလင်းထက် တောက်ပဆုံး အရာတစ်ခုဖြစ်သည်။ ရောမ ဘုရင် Jupiter အမည်တပ်ထားသည့် ဤဂျက်ဆက်ကြယ်ဖိုလ်မှာ အခြား ကြယ်ဖိုလ်များ၏ ပေါင်းစုပမာဏ ထက်နှစ်နှစ်ပိုများသော ပမာဏရှိသော ဂယ်စ်ကြီးတစ်ခု ဖြစ်သည်။

အဓိက အကြောင်းအရာ အသီးသီးကို အကောင်းဆုံး ညွှန်ကြားချက်ပေးရန် အမျိုးမျိုး အတိုင်းအသုံးပြုနိုင်ပါသည်။

- **ဥပမာများ** - မော်ဒယ်ကို တိုက်ရိုက် လုပ်ဆောင်ရန်ကြေညာခြင်းမပြုဘဲ လုပ်ဆောင်ရန် ဥပမာများ ပေးပြီး မော်ဒယ်အနေ로 ပုံစံကို ခန့်မှန်းခွင့် ပေးခြင်း။
- **အကြောင်းပြချက်များ** - ညွှန်ကြားချက်ထက်နောက်တွင် "cue" တစ်ခု ထည့်သွင်းပေးခြင်းဖြင့် မော်ဒယ်ကို ပိုသက်ဆိုင်သော နေရာသို့ ဦးတည်စေခြင်း။
- **အက်စ်ပုံများ** - အတိအကျထည့်သွင်းနိုင်သည့် သင့်တော်မှုအတွက် ဒေတာဖြင့် တန်ဆာပလာများ ပြောင်းလဲနိုင်သော ပြန်ကြားချက်များ၏ မဟာဗျူဟာများ။

အဲ့ဒီနည်းလမ်းတွေကို လက်တွေ့သုံးကြည့်ရအောင်။

### ဥပမာများ အသုံးပြုခြင်း

အဓိက အကြောင်းအရာကို အသုံးပြုပြီး လိုအပ်သည့် အကျဉ်းချုပ် များ၏ ဥပမာတချို့ကို မော်ဒယ်ကို "အစားထိုး" ပေးပြီး လိုအပ်သည့် ဥပမာ ပုံစံကို မော်ဒယ်က ခန့်မှန်းခွင့်ပေးသည်။ ဥပမာပေါင်း အရေအတွက်အပေါ်မူတည်၍ zero-shot, one-shot, few-shot prompting စသည်ဖြင့် ခွဲခြားနိုင်သည်။

ယခု prompt တွင် အစိတ်အပိုင်း သုံးခု ပါဝင်သည်။

- တာဝန်ဖော်ပြချက်
- လိုအပ်သော output ၏ ဥပမာအနည်းငယ်
- နောက်အသစ် စတင်သော ဥပမာ (အလိုအလျောက် တာဝန် ဖော်ပြချက် ဖြစ်လာသည့်)

| သင်ကြားမှု မျိုး | Prompt (Input)                                                                                                                                        | Completion (Output)         |
| :------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------- | :-------------------------- |
| Zero-shot     | "The Sun is Shining". Translate to Spanish                                                                                                            | "El Sol está brillando".    |
| One-shot      | "The Sun is Shining" => ""El Sol está brillando". <br> "It's a Cold and Windy Day" =>                                                                 | "Es un día frío y ventoso". |
| Few-shot      | The player ran the bases => Baseball <br/> The player hit an ace => Tennis <br/> The player hit a six => Cricket <br/> The player made a slam-dunk => | Basketball                  |
|               |                                                                                                                                                       |                             |

zero-shot prompting မှာ တိုက်ရိုက်ညွှန်ကြားချက် ("Translate to Spanish") ကို ပေးရန် လိုအပ်ခဲ့သော်လည်း one-shot prompting တွင် အဲဒီညွှန်ကြားချက်ကို မော်ဒယ်က ထင်မှတ်နိုင်ခဲ့သည်။ few-shot ဥပမာကတော့ ဥပမာများ ပိုထည့်ခြင်းက မော်ဒယ်များအတွက် ပိုမှန်ကန်တဲ့ သဘောတူချက်များ ရရှိရန် ကူညီပေးသည်ကို ပြထားသည်။

### Prompt အကြောင်းပြချက်များ

အဓိက အကြောင်းအရာ အသုံးပြုရာတွင် နည်းလမ်းတစ်ခုမှာ ဥပမာ မဟုတ်ဘဲ _cue_ များပေးခြင်း ဖြစ်သည်။ ဤနည်းလမ်းတွင် မော်ဒယ်အား လိုအပ်သော ဖြေချက်ပုံစံကို ပြသသည့် အပိုင်းအစုဖြင့် _စတင်ပေး_ ရှိပြီး မော်ဒယ်သည် ထို cue ကို လက်ခံပြီး ဆက်လက်လုပ်ဆောင်ရန် ဖြစ်သည်။

| Cue အရေအတွက် | Prompt (Input)                                                                                                                                                                                                                                                                                                                                                                                                                                               | Completion (Output)                                                                                                                                                                                                                                                                                       |

### ဖော်ပြချက်ပုံစံများ

ဖော်ပြချက်ပုံစံတစ်ခုသည် _ဖော်ပြချက်အတွက် ကြိုတင်သတ်မှတ်ထားသော မုန့်ဟင်းခါး_ တစ်ခုဖြစ်ပြီး အသုံးပြုသူ အတွေ့အကြုံများကို ပိုမိုညီမျှစွာ ထိန်းသိမ်းပေးရန်အတွက် လိုအပ်သလို သိုလှောင်ထားပြီး ထပ်မံအသုံးပြုနိုင်သည်။ ၎င်း၏ အရိုးရှင်းဆုံး ပုံစံတွင် [OpenAI ထံမှ ဤပုံစံတစ်ခု](https://cookbook.openai.com/examples/gpt4-1_prompting_guide?WT.mc_id=academic-105485-koreyst) ကဲ့သို့ ရုပ်သိမ်းပေးသည့် ဖော်ပြချက် ဖန်တီးပုံတို့ (အသုံးပြုသူနှင့် စနစ်တင်ပြချက်များ) နှင့် API ဖြင့်တောင်းဆိုသော စနစ်အခြေခံပုံစံဖြစ်သည့် ပုံစံများပါဝင်သည်။

ပိုမိုရှုပ်ထွေးသော ပုံစံတစ်ခုမှာ [LangChain ထံမှ ဤဥပမာ](https://python.langchain.com/docs/concepts/prompt_templates/?WT.mc_id=academic-105485-koreyst) ကဲ့သို့ _နေရာဖြည့်_ ပိုင်းများကို ရှိပြီး ဤနေရာများအား အသုံးပြုသူ ထည့်သွင်းသော ဒေတာ၊ စနစ်အခြေအနေ၊ ပြင်ပဒေတာများ ကဲ့သို့သော မတူညီသော အရင်းအမြစ်များမှ အချက်အလက်များဖြင့် အစားထိုးနိုင်ပြီး ဖော်ပြချက်ကို စနစ်တကျ ဖန်တီးဖို့ အထောက်အကူပြုသည်။ ၎င်းက ကျွန်တော်တို့အတွက် ဖော်ပြချက်များကို စနစ်တကျ အသုံးပြုနိုင်သော စာကြည့်တိုက်တစ်ခု ဖန်တီးကာ အသုံးပြုသူအတွေ့အကြုံများကို **အစီအစဉ်တကျ** အဆင့်မြှင့်ရာတွင် အသုံးပြုနိုင်စေသည်။

နောက်ဆုံးတွင် ဖော်ပြချက်ပုံစံများ၏ အဖိုးတန်ဆုံး အကြောင်းအရာမှာ အသုံးပြုမှုဆိုင်ရာ ဧရိယာအလိုက် ဖော်ပြချက်စာကြည့်တိုက်များ ဖန်တီး၍ ထုတ်ဝေကာ ၎င်းဖော်ပြချက်ပုံစံသည် အသုံးပြုမည့် ဧရိယာအထူးအကြောင်းအရာ လိုအပ်ချက်များနှင့် ရည်ညွှန်းချက်များကို စူးစမ်းပြုပြင်ထားခြင်းဖြစ်သည်။ [Prompts For Edu](https://github.com/microsoft/prompts-for-edu?WT.mc_id=academic-105485-koreyst) စာကြည့်တိုက်သည် ပညာရေးဧရိယာအတွက် သင်ခန်းစာအစီအစဉ်ရေးဆွဲခြင်း၊ သင်ရင်းမြှင့်ရေး အစီအစဉ် တည်ဆောက်ခြင်း၊ ကျောင်းသားလေ့ကျင့်မှု စတာများအတွက် အဓိက ရည်ရွယ်ချက်များကို အာရုံစိုက်ကာ ဖော်ပြချက်များ စုစည်းထားသည့် ဥပမာကောင်းတစ်ခု ဖြစ်သည်။

## ထောက်ပံ့ပစ္စည်းများ

ဖော်ပြချက်ဖန်တီးခြင်းကို တာဝန်(supervised task) နှင့် ပစ်မှတ်(အဓိကအကြောင်းအရာ) ရှိခြင်းအနေဖြင့် စဉ်းစားမယ်ဆိုရင် _ဒုတိယအကြောင်းအရာ_ သည် အဖြေကို ရည်ညွှန်းမှုအတိုင်း ထိခိုက်အကျိုးသက်ရောက်စေနိုင်သော အပိုအခြေအနေတစ်ရပ်အဖြစ် အဓိပ္ပာယ်ရှိသည်။ ၎င်းသည်  ပြင်ဆင်ထားသည့် ပါရာမီတာများ၊ ပုံစံသတ်မှတ်ချက်များ၊ ခေါင်းစဉ် အမျိုးအစားများ စသည်ဖြင့် မော်ဒယ်၏ တုံ့ပြန်ချက် ကို သတ်မှတ်ထားသော အသုံးပြုသူရည်ရွယ်ချက်များနှင့် ကိုက်ညီအောင် _စိတ်ကြိုက်လုပ်ဆောင်နိုင်စေရန်_ အကူအညီပေးသည်။

ဥပမာ- သင်တန်းစာရင်းမှာ ပါဝင်သော သင်တန်းများအားလုံးအတွက် အဓိကအချက်အလက်များ (နာမည်၊ ဖော်ပြချက်၊ အဆင့်၊ မျိုးစုံဒေတာ တက်ဂ်များ၊ သင်ကြားသူများ စသည်) ပါဝင်သော သေးငယ်သော ဒေတာအချက်အလက်များ ပါဝင်သည်။

- သင်္ကေတအနေဖြင့် "2023 ရာသီဆောင်း သင်တန်းစာရင်းကို တိုတောင်းချက် တင်ပြပါ" ဟု သတ်မှတ်နိုင်သည်
- အဓိကအကြောင်းအရာအားဖြင့်  ရလဒ် နမူနာအနည်းငယ်ပေးနိုင်သည်
- ဒုတိယအကြောင်းအရာအားဖြင့် အကွာအဝေး အထိန်းထားသည့် "tags" ၅ ခုကို အထူးသတ်မှတ်နိုင်သည်။

ယခု မော်ဒယ်သည် နမူနာအနည်းငယ်ဖြင့် ပြရန် ဖော်ပြထားသည့် ပုံစံဖြင့် တိုတောင်းချက်တစ်ခုကို ဖန်တီးနိုင်သော်လည်း ရလဒ်ထဲတွင် tags များစွာပါဝင်ပါက ဒုတိယအကြောင်းအရာတွင် သတ်မှတ်ထားသည့် ၅ ခုကို ဦးစားပေးနိုင်မှု ရှိသည်။

## ဖော်ပြချက် အကောင်းဆုံးသော်လည်း အသုံးပြုပုံများ

ဖော်ပြချက်များကို _ဖန်တီးပုံ_ နားလည်ပြီးနောက် အကောင်းဆုံး အသုံးပြုနည်းများကို _ဒီဇိုင်းလုပ်နည်း_ တွင် ရည်မှန်းကြမည်။ အဓိကအားဖြင့် နှစ်ခုအပိုင်းဖြင့် စဉ်းစားနိုင်ပါသည်- တည်ရှိမှု _စိတ်ခံစားချက်_ နှင့် လုပ်ငန်း _နည်းလမ်း_။

### ဖော်ပြချက် အင်ဂျင်နီယာ စိတ်နေစိတ်ထား

ဖော်ပြချက် အင်ဂျင်နီယာလုပ်ငန်းသည် စမ်းသပ်၍ မီမှားခြင်းဖြစ်သောကြောင့် တောင့်တင်းသော သူးသားချက် သုံးခုကို မှတ်ထားပါ-

1. **ဧရိယာနားလည်မှု အရေးကြီးသည်။** တုံ့ပြန်ချက်တိကျမှုနှင့် သက်ဆိုင်မှုသည် အသုံးပြုသူ သို့မဟုတ် အက်ပ်လီကေးရှင်း လုပ်ဆောင်ရာ ဧရိယာတွင် ဆက်စပ်သည်။ သင်၏ အတွေးအခေါ်နှင့် ဧရိယာကျွမ်းကျင်မှုကို အသုံးပြုပြီး နည်းလမ်းများကို ထပ်မံ _သီးသန့်ထိန်းသိမ်း_ လုပ်ပါ။ ဥပမာ-စနစ်ဖော်ပြချက်များတွင် _ဧရိယာအလိုက် လူပုဂ္ဂိုလ်တစ်ဦး ထားရှိခြင်း_ သို့မဟုတ် အသုံးပြုသူ ဖော်ပြချက်များတွင် _ဧရိယာအလိုက် ပုံစံများ_ သုံးပါ။ ဒုတိယအကြောင်းအရာတွင် အဓိကထားရာဧရိယာဆိုင်ရာ အခြေအနေများ ဖော်ပြပါ၊ ဒါမှမဟုတ် _ဧရိယာအလိုက် ဖြေကြားမှုနည်းလမ်းများနှင့် ဥပမာ_ မော်ဒယ်ကို ညွှန်ပြရန် အသုံးပြုပါ။

2. **မော်ဒယ်နားလည်မှု အရေးကြီးသည်။** မော်ဒယ်များသည် လုပ်ဆောင်မှုမှာ စတိုကတ်စ်တစ် (stochastic) ဖြစ်ကြောင်း သိသည်။ သို့သော် မော်ဒယ်များ၏ ဖွင့်လှစ်မှုများမှာ အသုံးပြုသော သင်တန်းဒေတာစု (ကျွမ်းကျင်မှုရှိမှု), ပံ့ပိုးပေးသော စွမ်းရည်များ (API သို့မဟုတ် SDK ဖြင့်) နှင့် စတိုင်များ (ကုဒ်၊ ပုံရိပ်၊ စာသား) အလိုက် ခွဲခြားနိုင်သည်။ သင်အသုံးပြုမည့် မော်ဒယ်၏အားသာချက် နှင့် အားနည်းချက်များကို နားလည်ပြီး အလုပ်များ၏ ဦးစားပေးမှု သို့မဟုတ် မော်ဒယ်၏ စွမ်းရည်အလိုက် _သီးသန့်ပုံစံများ_ ဖန်တီးမှုအား အသုံးပြုပါ။

3. **အကြိမ်ပြန် စစ်ဆေးခြင်းနှင့် အတည်ပြုခြင်း အရေးကြီးသည်။** မော်ဒယ်များသည် အလျင်အမြန် တိုးတက်နေပြီး၊ ဖော်ပြချက်အင်ဂျင်နီယာနည်းများလည်း ထို့အတူတိုးတက်လျက်ရှိသည်။ သင့်အနေနှင့် ဧရိယာကျွမ်းကျင်သူတစ်ယောက်အနေဖြင့် သင်၏ အချက်အလက် သို့မဟုတ် သတ်မှတ်ချက်များကို မကြာသေးခင် အသုံးပြုမှုဖြင့် အခြေအနေထူးခြားစွာ ရှိနိုင်သည်။ ဖော်ပြချက်အင်ဂျင်နီယာကိရိယာများနှင့် နည်းလမ်းများ အသုံးပြု၍ ဖော်ပြချက် ဖန်တီးမှုကို "အခေါ်" ပေးပြီး ထပ်မံ ပြန်လည် စစ်ဆေး၍ သင့်ကွပ်ကဲမှုများနှင့် ဧရိယာကျွမ်းကျင်မှုများကို အသုံးပြုထားနိုင်သည်။ သင့်အမြင်များကို မှတ်တမ်းတင်ပြီး **အသိပညာအုပ်စု** (ဥပမာ-ဖော်ပြချက်စာကြည့်တိုက်များ) ဖန်တီးပါ။ ထိုကဲ့သို့ အသိပညာအုပ်စုမှ အခြေအနေအသစ်များအတွက် အခြေခံအချက်အလက် အဖြစ် ဆက်လက်အသုံးပြုနိုင်မည်ဖြစ်သည်။

## အလုပ်သတ်မှတ်ချက်

ဂုဏ်ပြုလိုက်ပါသည်! သင်ခန်းစာကုန်ခါးသွားပြီ! ၎င်းအကြောင်းအရာများနှင့် နည်းဗျူဟာများကို သင်ကြားမှုကနေ အကောင်အထည်ဖော်နိုင်ရန် အချိန်ရောက်ပါပြီ!

ကျွန်တော်တို့အလုပ်သတ်မှတ်ချက်အနေဖြင့် ကွေးစားသမား မေးခွန်းများ ပါဝင်သည့် Jupyter Notebook ကို အသုံးပြုပါမည်။ သင့်ကိုယ်တိုင် Markdown နှင့် Code cells များထည့်သွင်း၍ အတွေးအခေါ်များနှင့် နည်းဗျူဟာများကို လေ့လာနိုင်သည်။

### စတင်ရန် အတိုင်းအတာပေါ်ထားပြီး repo ကို အခြားခံယူပါ၊ ပြီးလျှင်

- (အကြံပြု) GitHub Codespaces ကို စတင်ဖွင့်ပါ
- (တခြားနည်း) repo ကို သင့် ဒေသခံ စက်သို့ ကလုံလုပ်ပြီး Docker Desktop အတွက် အသုံးပြုပါ
- (တခြားနည်း) သင်နှစ်သက်သော ပြီးဆုံးသည့် ပရိုဂရမ် ပတ်ဝန်းကျင်ဖြင့် Notebook ဖွင့်ပါ။

### နောက်တစ်ဆင့် သင့်ပတ်ဝန်းကျင် အချက်အလက်များ ညှိနှိုင်းပါ

- repo အမြစ်ဗဟိုတွင် ရှိနေသော `.env.copy` ဖိုင်ကို `.env` ဟု ကူးယူပြီး `AZURE_OPENAI_API_KEY`, `AZURE_OPENAI_ENDPOINT` နှင့် `AZURE_OPENAI_DEPLOYMENT` တန်ဖိုးများ ဖြည့်စွက်ပေးပါ။ ပြန်လာ၍ [Learning Sandbox ကို ဆက်လက်လေ့လာပါ](#သင်ယူမှု-sandbox)။

### နောက်တစ်ဆင့် Jupyter Notebook ကို ဖွင့်ပါ

- runtime kernel ကို ရွေးချယ်ပါ။ ရွေးချယ်မှု ၁ သို့ ၂ ကို အသုံးပြုပါက dev container မှ ပံ့ပိုးသော Python 3.10.x kernel ကို ရွေးချယ်ပါ။

မေးခွန်းများ စတင်ဖြေဆိုရန် စီစဉ်ထားပါပြီ။ အမှန်မှားမရှိဘဲ စမ်းသပ်ပြီး မော်ဒယ်နှင့် အသုံးပြုသူဧရိယာအလိုက် အသုံးဝင်မည့် နည်းလမ်းကို ရှာဖွေပါ။

_ဤအကြောင်းကြောင့် Code ဖြေရှင်းမှု အစိတ်အပိုင်း မပါရှိပါ။ သို့သော် Notebook တွင် "My Solution:" စာတန်းပါသော Markdown cells များ ပါပြီး တစ်ဥပမာအဖြေ တပြားကို ပြထားသည်။_
