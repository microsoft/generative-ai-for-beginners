# Retrieval Augmented Generation (RAG) နှင့် Vector Databases

[![Retrieval Augmented Generation (RAG) နှင့် Vector Databases](../../../translated_images/my/15-lesson-banner.ac49e59506175d4f.webp)](https://youtu.be/4l8zhHUBeyI?si=BmvDmL1fnHtgQYkL)

စာရင်းလေ့လာခြင်း (search applications) များသင်ခန်းစာတွင် သင်နွေးထုတ်မှုဖြင့် သင့်ဒေတာကို အကြီးစားဘာသာစကားမော်ဒယ်များ (LLMs) နှင့် ပေါင်းစည်းအသုံးပြုနည်းကို ဒဏ်ခံခဲ့ပြီးဖြစ်သည်။ ဒီသင်ခန်းစာမှာတော့ သင့်ဒေတာကို သင့် LLM လျှောက်လွှာထဲမှာ ဘယ်လိုအခြေခံပေးရမလဲ၊ အလုပ်လုပ်နည်းနဲ့ ဒေတာသိုလှောင်ခြင်းနည်းလမ်းများကိုရှင်းလင်းပြထားမှာဖြစ်ပြီး ဒါတွေမှာ embedding နဲ့ စာသားတိုတိုနှစ်မျိုးစလုံးပါဝင်သည်။

> **ဗီဒီယို မကြာမီထွက်ပါမည်**

## နိဒါန်း

ဒီသင်ခန်းစာမှာ အောက်ပါအချက်တွေကို ဖော်ပြသွားမှာဖြစ်သည် –

- RAG ဆိုတာဘာလဲ၊ ဘာကြောင့် AI (artificial intelligence) ထဲမှာအသုံးပြုကြတာလဲဆိုတာအကြောင်းရင်း

- Vector database ဆိုတာဘာလဲဆိုတာနားလည်ခြင်းနှင့် ကျွန်တော်တို့လျှောက်လွှာအတွက် တစ်ခုဖန်တီးခြင်း

- ဥပမာလက်တွေ့က RAG ကို လျှောက်လွှာထဲမှာ ဘယ်လိုပေါင်းစည်းရမလဲဆိုတာ

## သင်ယူရမည့်ရည်မှန်းချက်များ

ဒီသင်ခန်းစာပြီးဆုံးလျှင် သင်များမှာ အောက်ပါအရာများကို နားလည်ထားသွားပါလိမ့်မည် –

- ဒေတာ ရှာဖွေမှုနှင့် လုပ်ထုံးလုပ်နည်းထဲတွင် RAG ရဲ့ အရေးပါမှုကို ရှင်းပြနိုင်ရန်

- RAG လျှောက်လွှာကို တပ်ဆင်ပြီး သင့်ဒေတာကို LLM နဲ့ ချိတ်ဆက်နိုင်ရန်

- LLM လျှောက်လွှာများတွင် RAG နှင့် Vector Databases တို့ကို ထိရောက်စွာ ပေါင်းစည်းအသုံးချနည်း

## ကျွန်တော်တို့၏ ဖြစ်ရပ်မှန်ပုံစံ - ကျွန်တော်တို့၏ LLM များကို ကိုယ်ပိုင်ဒေတာဖြင့် တိုးတက်အောင်လုပ်ခြင်း

ဒီသင်ခန်းစာအတွက် ကျွန်တော်တို့က သူငယ်ချင်းတက်အစီအစဉ်ထဲမှာ ကိုယ်ပိုင်မှတ်ချက်များထည့်လိုပါတယ်၊ ဒါမှ တစ်ဆင့် chatbot က အကြောင်းအရာမျိုးစုံအပေါ်ပိုမိုသိရှိနိုင်အောင်လုပ်ဖို့ဖြစ်ပါတယ်။ ကျော်ကြားရာမှတ်ချက်တွေကို အသုံးပြု၍ လေ့လာသူများအား ပိုမိုကောင်းမွန်စွာ အနက်ကို နားလည်နိုင်ပြီး ချဉ်းကပ်မှုမှာလွယ်ကူစေခြင်းဖြစ်တယ်။ ကျွန်တော်တို့၏ဖြစ်ရပ်မှန်အတွက် အောက်ပါအရာများကို သုံးပါမည် –

- `Azure OpenAI:` ကျွန်တော်တို့ chatbot ဖန်တီးရန်အသုံးပြုမယ့် LLM

- `AI for beginners' lesson on Neural Networks`: ဒီဟာကတော့ ကျွန်တော်တို့ LLM အတွက် အခြေခံဒေတာဖြစ်ပါသည်

- `Azure AI Search` နဲ့ `Azure Cosmos DB:` vector database ကို ဒေတာသိုလှောင်ရန်နှင့် ရှာဖွေခြင်းအညွှန်းဇယားဖန်တီးရန်

အသုံးပြုသူများဟာ ကိုယ်ပိုင်မှတ်ချက်များမှ အခြေခံ၍ ပြန်လည်လေ့လာမှု ကွပ်ကဲမှု၊ အနှစ်ချုပ်ကတ်များ၊ အသေးစိတ်အနှစ်ချုပ်များ ပြုလုပ်နိုင်ပါသည်။ စတင်ရန် RAG ဆိုတာဘာလဲ နဲ့ ဘယ်လိုအလုပ်လုပ်တယ် ဆိုတာကို ကြည့်ကြပါစို့။

## Retrieval Augmented Generation (RAG)

LLM သုံး chatbot က အသုံးပြုသူစကားများကို လက်ခံပြီး ထွက်ရှိလာမည့်တုံ့ပြန်ချက်များ ဖန်တီးပေးသည်။ ဤ chatbot သည် သက်ဆိုင်ရာခေါင်းစဉ်များအပေါ် သက်ဝင်တုံ့ပြန်ကာ အသုံးပြုသူနှင့် ဆက်ဆံမှုရှိစေရန် ရည်ရွယ်သည်။ သို့သော် ၎င်း၏ တုံ့ပြန်ချက်များမှာ ပေးထားသော သရုပ်ဖော်မှုအကြောင်းအရာများနှင့် မူအခြေခံ သင်ကြားမှုဒေတာတွင် သက်ဆိုင်သည့် အနယ်နယ်အရပ်ရပ်မှသာ ကန့်သတ်ထားသည်။ ဥပမာ - GPT-4 ၏ သိမြင်မှုပိတ်ဆို့သည့်ရက်မှာ ၂၀၂၁ ခုနှစ် စက်တင်ဘာဖြစ်ပြီး၊ ၎င်းထွက်ပြီးနောက်ဖြစ်ပွားသော ဧည့်လုပ်ခင်းများကို မသိရှိပါ။ ထို့ပြင် LLM များသင်ကြားရန် အသုံးပြုသော ဒေတာသည် ကိုယ်ရေးထားမှတ်ချက်များ၊ ကုမ္ပဏီထုတ်ကုန်လက်စွဲစာအုပ်များကဲ့သို့ အကြောင်းမဖော်ပြရသည့် သတင်းအချက်အလက်များကို မပါဝင်ပါ။

### RAG (Retrieval Augmented Generation) များ ဘယ်လိုအလုပ်လုပ်တယ်

![drawing showing how RAGs work](../../../translated_images/my/how-rag-works.f5d0ff63942bd3a6.webp)

သင့်မှတ်ချက်တွေကနေ ကျောင်းစာမေးပွဲတွေ ဖန်တီးနိုင်တဲ့ chatbot တစ်ခုဖန်တီးချင်တယ်ဆိုပါစို့၊ နောက်ခံသိပ္ပံရင်းမြစ်နဲ့ ချိတ်ဆက်ထားဖို့လိုပါမယ်။ ဒီမှာ RAG က ကယ်တင်မှုလုပ်ပေးနိုင်ပါတယ်။ RAG များအလုပ်လုပ်ပုံမှာ –

- **သိပ္ပံရင်းမြစ်:** ရှာဖွေရခြင်းမပြု မီ၊ စာရွက်များကို လက်ခံပြီး ကြီးမားသော စာရွက်ကို အပိုင်းပိုင်းခွဲခြမ်း၍ စာသား embedding ပြောင်းလဲပြီး ဒေတာဘေ့စ်ထဲသိုလှောင်ထားရမည်။

- **အသုံးပြုသူ စုံစမ်းမှု:** အသုံးပြုသူသည် မေးခွန်းတစ်ခု မေးသည်။

- **ရှာဖွေရေး:** အသုံးပြုသူ မေးခွန်းမေးသည်နှင့် အမျှ embedding မော်ဒယ်က ကျွန်တော်တို့၏ သိပ္ပံရင်းမြစ်ထဲက သက်ဆိုင်ရာ သတင်းအချက်အလက်များကို ရှာဖွေ ထုတ်ယူပြီး ပိုမိုကောင်းမွန်သော အခြေအနေ ထည့်သွင်းရန်။

- **တိုးမြှင့်ထုတ်လုပ်မှု:** LLM သည် လျှောက်လွှာအတွက် ရရှိလာသည့် ဒေတာအပေါ် မှီတည့်ပြီး ဖြေရှင်းချက်ကို တိုးမြှင့်သည်။ ထူးခြားချက်မှာ မူလသင်ကြားထားသည့် ဒေတာအပေါ်မှသာ မူတည်ခြင်းမဟုတ်ဘဲ၊ ပြင်ပမှ ရရှိသော သက်ဆိုင်ရာ သတင်းအချက်အလက်များဖြင့် ဖြည့်ဆည်းပေးခြင်းဖြစ်သည်။ ထုတ်ယူပြီးတဲ့ ဒေတာများကို LLM ၏ ဖြေဆိုချက်တိုးမြှင့်ရန် အသုံးပြုသည်။ အဲ့ဒီနောက် LLM က အသုံးပြုသူ မေးခွန်းကို ဖြေတုံ့ပြန်ပေးသည်။

![drawing showing how RAGs architecture](../../../translated_images/my/encoder-decode.f2658c25d0eadee2.webp)

RAG များ၏ ပုံဆင်းစံအဆောက်အအုံမှာ transformer မော်ဒယ်နှစ်ပိုင်းပါဝင်ပြီး encoder နဲ့ decoder ဖြစ်သည်။ ဥပမာ၊ အသုံးပြုသူ မေးခွန်းမေးသောအခါ၊ input စာသားကို စကားလုံး၏ အဓိပ္ပါယ်ကို ဖော်ထုတ်သော vectors အဖြစ် 'encode' ပြီး vector များကို 'decode' ပြုလုပ်၍ ဒေတာဖိုင်အညွှန်းဇယားထဲကို ဆော့သွားပြီး အသုံးပြုသူ မေးခွန်းအပေါ် အခြေခံ၍ စာသားအသစ် ပြန်လုပ်ပေးသည်။ output ဖန်တီးရာတွင် LLM သည် encoder-decoder မော်ဒယ်ကို သုံးသည်။

RAG ကို အကောင်အထည်ဖော်သောနည်းလမ်း ၂ မျိုးကို [Retrieval-Augmented Generation for Knowledge intensive NLP (natural language processing software) Tasks](https://arxiv.org/pdf/2005.11401.pdf?WT.mc_id=academic-105485-koreyst) ဆိုတဲ့ ဆောင်းပါးက ဖော်ပြထားပါတယ်။

- **_RAG-Sequence_** ဟာ retrieved documents တွေကို သုံးပြီး အသုံးပြုသူ မေးခွန်းအတွက် အကောင်းဆုံးဖြေကြားချက်ခန့်မှန်းခြင်း။

- **RAG-Token** ဟာ စာရွက်တွေကို အသုံးပြုပြီး နောက်တစ်ချက် token ပြုလုပ်ပြီး အသုံးပြုသူမေးခွန်း ပြန်ဖြေပါတယ်။

### RAG များ ဘာကြောင့်အသုံးပြုသင့်လဲ။

- **သတင်းအချက်အလက် ပြည့်စုံမှု:** စာသားဖြေဆိုချက်များကို ယခုခေတ်နှင့်ညီအောင် သေချာစေသည်။ ဒါကြောင့် domain-specific ဝင်များတွင် အကျိုးသက်ရောက်မှုကို မြှင့်တင်ပေးသည်။

- အသုံးပြုသူ မေးခွန်းများမှန်ကန်စွာ ဖြေဆိုနိုင်ရန် **စစ်ဆေးနိုင်သော ဒေတာများ** ကိုသုံးသည့်အတွက် အတုများကို လျော့ချပေးသည်။

- LLM ကို fine-tuning ပြုလုပ်တဲ့အစား ပိုမို **ကုန်ကျမှုသက်သာ**ခြင်းရှိသည်။

## သိပ္ပံရင်းမြစ် ဖန်တီးခြင်း

ကျွန်တော်တို့ကျောင်းစာအား ကိုယ်ပိုင်ဒေတာအပေါ် အခြေခံထားသည့် AI For Beginners မှ Neural Network သင်ခန်းစာဖြစ်ပါသည်။

### Vector Databases

Vector database သည် ကောင်းစွာသိသာသော ဒေတာဘေ့စ်များနှင့် မတူကွဲပြားပြီး vector များကို သိမ်းဆည်း ဖန်တီး စီမံနှင့် ရှာဖွေတာရှာဖွေရာ အသုံးပြုသော database အမျိုးအစားဖြစ်သည်။ ဒေတာများကို အနကျပ်အရေအတွက်တန်ဖိုးအဖြစ် သိမ်းဆည်းထားသည်။ ဒေတာကို နံပါတ်ဖြစ်သော embedding များအဖြစ် ခွဲခြမ်းထားခြင်းက ကျွန်တော်တို့ AI စနစ်အတွက် နားလည်ခြင်းနှင့် ထိန်းချုပ်ခြင်း ပိုမိုလွယ်ကူစေသည်။

ကျွန်တော်တို့ရဲ့ embedding များကို vector database များထဲသိုလှောင်ပါမယ်၊ LLM များတွင် input token အရေအတွက်ကန့်သတ်ချက်ရှိသောကြောင့် အပြည့်အစုံ embedding များကို LLM ထဲတစ်ပြိုင်နက်မပို့နိုင်ပါ။ ထို့အစား embedding များကို အပိုင်းသို့ခွဲပြီး အသုံးပြုသူမေးခွန်းတစ်ခုစီအတွက် နီးစပ်သော embedding များကို ကောက်ယူပြီး prompt နှင့်တကွ ပြန်ပို့ပြီး စနစ်သုံးစွဲနိုင်စေရန်ဖြစ်ပါသည်။ ထို့အပြင် chunking ပြုလုပ်ခြင်းက LLM ထဲတွင် ပို့ပေးရမည့် token အရေအတွက်ကိုလည်း လျော့ချပေးနိုင်သည်။

လူသိများသော vector database များမှာ Azure Cosmos DB, Clarifyai, Pinecone, Chromadb, ScaNN, Qdrant နဲ့ DeepLake တို့ ပါဝင်သည်။ Azure CLI အသုံးပြု၍ Azure Cosmos DB မော်ဒယ်ကို အောက်ပါ command ဖြင့် ဖန်တီးနိုင်ပါသည် –

### စာသားမှ embedding များသို့

ကျွန်တော်တို့ဒေတာများကို သိမ်းဆည်းမည့်အစား embedding များအဖြစ် ပြောင်းလဲရမည်။ အကြီးစားစာရွက်စာတမ်းများ သို့မဟုတ် ရှည်လျားသောစာသားများဖြင့် အလုပ်လုပ်စဉ်တွင် မျှော်မှန်းထားသည့် မေးခွန်းအလိုက် အပိုင်းပိုင်းခွဲ၍ (chunking) ပြုလုပ်နိုင်သည်။ chunking ကို စာကြောင်းအတိုင်းသို့မဟုတ် ပိုဒအတိုင်း ခွဲခြားနိုင်သည်။ chunking သည် ကျေးလက်အနားရှိ စကားလုံးများမှ အဓိပ္ပါယ်ယူဆသဖြင့်၊ ယင်းအပိုင်း၌ အခြားနောက်ခံအကြောင်းအရာများကိုပါ ထည့်သွင်းပေးနိုင်သည်။ ဥပမာ၊ စာရွက်ခေါင်းစဉ်ထည့်သွင်းခြင်း သို့မဟုတ် chunk အကြို သို့မဟုတ်အပြီးအချပိုင်းစာသား ထည့်သွင်းပေးခြင်းစသည်ဖြင့် chunk များကို ဖန်တီးနိုင်သည်။ ဒါကို အောက်ပါအတိုင်း ပြုလုပ်နိုင်ပါသည် –

```python
def split_text(text, max_length, min_length):
    words = text.split()
    chunks = []
    current_chunk = []

    for word in words:
        current_chunk.append(word)
        if len(' '.join(current_chunk)) < max_length and len(' '.join(current_chunk)) > min_length:
            chunks.append(' '.join(current_chunk))
            current_chunk = []

    # နောက်ဆုံး ပိုင်းက မီနီမမ်အရှည်မရောက်သေးပါက အဲဒီကို မဖြုတ်ဘဲ ထည့်ပေးပါ။
    if current_chunk:
        chunks.append(' '.join(current_chunk))

    return chunks
```

chunking ပြီးနောက် ကျွန်တော်တို့စာသားများကို မတူညီသော embedding မော်ဒယ်များဖြင့် ဖန်တီးပေးနိုင်သည်။ အသုံးပြုနိုင်သော မော်ဒယ်များမှာ word2vec, OpenAI မှ ada-002, Azure Computer Vision စသဖြင့် များစွာ ရှိသည်။ မည်သည့်မော်ဒယ်ကိုရွေးမည်ဆိုသည်မှာ သင်သုံးမည့် ဘာသာစကားများ၊ Encode လုပ်မည့် အကြောင်းအရာအမျိုးအစား (စာသား/ပုံ/အသံ)၊ Encode နိုင်သည့် အင်ပुटအရေအတွက်နှင့် embedding output ၏ ရွေးချယ်ခြင်းမှာ တွဲဖက်ဆောင်ရွက်မှုတွင်မူတည်သည်။

OpenAI `text-embedding-ada-002` မော်ဒယ်ဖြင့် embedding ထားသည့် စာသားရဲ့ ဥပမာတစ်ခုမှာ –

## ရှာဖွေရေး နှင့် Vector ရှာဖွေရေး

အသုံးပြုသူ မေးခွန်းမေးသည်နှင့်အမျှ retriever က query encoder အသုံးပြုပြီး မေးခွန်းအား vector အဖြစ်ပြောင်းပြီး၊ ကျွန်တော်တို့ရဲ့ စာရွက်စာတမ်းရှာဖွေမှု အညွှန်းဇယားထဲက သက်ဆိုင်ရာ vector များကို ရှာဖွေရန် ဆောင်ရွက်သည်။ ပြီးလျှင် input vector နဲ့ စာရွက်စာတမ်း vector များကို စာသားဖြင့် ပြောင်းလဲပြီး LLM ထဲ ဖြတ်သွားစေသည်။

### ရှာဖွေရေး

ရှာဖွေမှုသည် စနစ်မှ ရှာဖွေရေးအညွှန်းဇယားထဲက၎င်း၊ ရှာဖွေရေး စံသတ်မှတ်ချက်နှင့် ကိုက်ညီသော စာရွက်စာတမ်းများကို အမြန် ရှာဖွေအောင် ကြိုးစားသည်။ retriever ရဲ့ ရည်ရွယ်ချက်မှာ စာရွက်စာတမ်းများကို ရှာဖွေပြီး အကြောင်းအရာကို ထောက်ပံ့ပေးရန်နှင့် LLM ကို သင့်ဒေတာအပေါ် အခြေခံစေခြင်းဖြစ်သည်။

ကျွန်တော်တို့ ဒေတာဘေ့စ်အတွင်း ရှာဖွေမှု ဆောင်ရွက်ရာတွင် နည်းလမ်းအတော်များစွာ ရှိသည်၊ ဥပမာ –

- **Keyword search** - စာတိုများကို ရှာဖွေရန် အသုံးပြု

- **Vector search** - စာသားများကို embedding မော်ဒယ်ဖြင့် vector ပုံစံပြောင်း၍ စိတ်ဓာတ်အရင်းအမြစ်ဖြင့် semantic ရှာဖွေမှုကို ခွင့်ပြုသည်။ အသုံးပြုသူမေးခွန်းနှင့် နီးစပ်သော vector များ ပြန်ရှာဖွေမှာဖြစ်သည်။

- **Hybrid** - keyword နှင့် vector ရှာဖွေမှုပေါင်းစပ်ထားခြင်း။

ရှာဖွေမှုတွင် ထပ်တူဖြေဆိုချက် မတွေ့ရှိပါက စနစ်က အကောင်းဆုံး သတင်းအချက်အလက်ကို ပြန်ပေးပို့တတ်သည်။ သို့သော် relevance အကွာအဝေးအမြင့်ဆုံးကို သတ်မှတ်ခြင်း သို့မဟုတ် keyword နှင့် vector ရှာဖွေမှုတို့ ပေါင်းစပ်ထားသည့် hybrid ရှာဖွေမှုကို သုံးနိုင်သည်။ ဒီသင်ခန်းစာမှာ hybrid ရှာဖွေမှုကို အသုံးပြုပါမည်။ ဒေတာများကို chunk နှင့် embedding တို့ပါဝင်သည့် dataframe ထဲသိုလှောင်ပါမည်။

### Vector similarity

Retriever သည် သိပ္ပံရင်းမြစ်ထဲမှ နီးစပ်သော embedding များကို ရှာဖွေသည်။ အသုံးပြုသူမေးခွန်းကို အရင် embed ပြီး နီးစပ်သော embedding များနှင့် ကိုက်ညီစေသည်။ vector များ ကြားရှိ အသီးသီးကွာခြားမှုကို တွက်ချက်ရာတွင် cosine similarity ကို အသုံးပြုကြသည်။ ၎င်းမှာ vectors နှစ်ခုအကြား ထောင့်အပေါ် အခြေခံသည်။

ဆင်တူမှုပမာဏကို တိုင်းတာရာတွင် အခြားရွေးချယ်စရာများမှာ Euclidean distance (vectors အထောင့်နှစ်ခုကြား တိုတောင်းသောလိုင်း) နှင့် dot product (vectors အဖြစ်နှစ်ခု၏ ကိုက်ညီသော အရာများ၏ ထွက်အတူတူပမာဏ) ဖြစ်သည်။

### ရှာဖွေမှု အညွှန်းဇယား

ရှာဖွေမှုအတွင်း စာရွက်စာတမ်းများကို မြန်မြန် ရှာဖွေဖို့အတွက် သိပ္ပံရင်းမြစ်အတွက် ရှာဖွေမှု အညွှန်းတစ်ခု ဖန်တီးရမည်။ အညွှန်းသည် embedding များကို သိမ်းဆည်းပြီး မကြီးမားသော ဒေတာဘေ့စ်မှာဖြစ်စေ နီးစပ်ဆုံး chunk များကိုမြန်ဆန်စွာ ရှာဖွေပြန်ပေးနိုင်သည်။ အောက်ပါတို့ကို အသုံးပြု၍ ဒေသတွင်း အညွှန်းဖန်တီးနိုင်သည် –

```python
from sklearn.neighbors import NearestNeighbors

embeddings = flattened_df['embeddings'].to_list()

# ရှာဖွေရေးညိွနှိုင်းချက်ကိုတည်ဆောက်ပါ
nbrs = NearestNeighbors(n_neighbors=5, algorithm='ball_tree').fit(embeddings)

# ညိွနှိုင်းချက်ကိုမေးမြန်းရန်၊ kneighbors နည်းလမ်းကိုအသုံးပြုနိုင်သည်
distances, indices = nbrs.kneighbors(embeddings)
```

### ပြန်တန်းခွဲခြင်း (Re-ranking)

ဒေတာဘေ့စ်ကို မေးမြန်းပြီးနောက်မှာ ရလဒ်အများဆုံးနီးစပ်သောအတိုင်း စီစဉ်လိုတယ်ဆိုရင်၊ reranking အထောက်အကူဖြစ်ပါတယ်။ reranking LLM သည် စနစ်ကို သင်ကြားပေးကာ ရှာဖွေမှုရလဒ်များကို ပိုနီးစပ်အောင် စီစဉ်ပေးသည်။ Azure AI Search အသုံးပြုသည်ဆို reranking ကို semantic reranker ဖြင့် အလိုအလျောက် ပြုလုပ်ပေးပါမယ်။ nearest neighbours ကိုအသုံးပြုပြီး reranking ယူနားပုံက –

```python
# အလွန်ဆင်တူသောစာရွက်စာတမ်းများကိုရှာပါ
distances, indices = nbrs.kneighbors([query_vector])

index = []
# အလွန်ဆင်တူသောစာရွက်စာတမ်းများကိုပုံနှိပ်ပါ
for i in range(3):
    index = indices[0][i]
    for index in indices[0]:
        print(flattened_df['chunks'].iloc[index])
        print(flattened_df['path'].iloc[index])

    else:
        print(f"Index {index} not found in DataFrame")
```

```python
        {"role": "user", "content": "\n\n".join(history) }
    ]

    # တုံ့ပြန်ချက်များ API ကို အသုံးပြု၍ တုံ့ပြန်ချက် တစ်ခု ဖန်တီးပါ
    response = client.responses.create(
        model="gpt-4o-mini",
        temperature=0.7,
        max_output_tokens=800,
        input=messages,
        store=False,
    )

    return response.output_text

chatbot(user_input)
```

## ကျွန်တော်တို့ လျှောက်လွှာ အကဲဖြတ်ခြင်း

### အကဲဖြတ် နည်းလမ်းများ

- ထောက်ပံ့သော ဖြေဆိုချက်များ၏ အရည်အသွေး၊ သဘာဝကျမှု၊ ပြေလည်ငြိမ်စာခြင်းနှင့် လူ့ဆန်မှု။

- ဒေတာအခြေခံမှု - ဖြေဆိုချက်သည် ပံ့ပိုးပေးသော စာရွက်စာတမ်းများမှ ဖြစ်ကြောင်း စစ်ဆေးခြင်း။

- ချိုးအကွာအဝေး - ဖြေဆိုချက်သည် မေးခွန်းနှင့် ကိုက်ညီမှုရှိသည်။

- ပြေလည်မှု - ဖြေဆိုချက် သဘောတရားပိုင်းမှ တိုင်းတာခြင်း။

## RAG (Retrieval Augmented Generation) နှင့် vector databases အသုံးပြုမှု များ

function calls နဲ့ သင့် app ကို တိုးတက်ကောင်းမွန်စေတတ်တဲ့ အသုံးပြုမှုများမှာ –

- မေးခွန်းဖြေဆိုခြင်း - ကုမ္ပဏီဒေတာကို chat နဲ့ချိတ်ဆက်၍ ဝန်ထမ်းများ လိုအပ်ချက်မေးမြန်းနိုင်စေရန်။

- အကြံပြုစနစ် - အတူညီဆုံး တန်ဖိုးများကို ကိုက်ညီစေရန် စနစ်တခု ဖန်တီးနိုင်သည်၊ ဥပမာ - ရုပ်ရှင်များ၊ စားသောက်ဆိုင်များ စသည်။

- Chatbot ဝန်ဆောင်မှုများ - စကားပြောမှုမှတ်တမ်းများကို သိမ်းဆည်းပြီး အသုံးပြုသူအချက်အလက်အပေါ် ကိုယ်ပိုင် စကားပြောမှု ပုံစံများ ဖန်တီးခြင်း။

- Vector embedding အခြေခံ ပုံရိပ်ရှာဖွေရေး၊ ပုံရိပ် အသိအမှတ် ပြုခြင်းများနှင့် အနည်းဆုံးလက္ခဏာ နေရာရှာဖွ်မှု ထောက်ပံ့မှု။

## အနှစ်ချုပ်

ကျွန်တော်တို့ RAG ၏ အခြေခံအချက်များကို ဆွေးနွေးခဲ့ပြီး၊ ဒေတာ ထည့်သွင်းခြင်း၊ အသုံးပြုသူ မေးခွန်းနှင့် ထွက်ရှိမှုတို့ကို ဖြတ်သန်းခဲ့သည်။ RAG ဖန်တီးမှုကို ပိုမိုလွယ်ကူစေရန် Semanti Kernel, Langchain သို့မဟုတ် Autogen စသည့် framework များကို အသုံးပြုနိုင်ပါသည်။

## သင်ခန်းစာအလုပ်

Retrieval Augmented Generation (RAG) ကို သင်ယူမှုကို ဆက်လက်တိုးတက်စေရန် ခဲ့ပါ –

- သင့်ရွေးချယ်သော framework ဖြင့် လျှောက်လွှာအတွက် front-end တစ်ခု ဖန်တီးပါ။

- LangChain သို့မဟုတ် Semantic Kernel ကို အသုံးပြုပြီး သင့်လျှောက်လွှာကို ပြန်လည်ဖန်တီးပါ။

သင်ခန်းစာပြီးစီးခဲ့တဲ့အတွက် အားပေးဂုဏ်ယူပါသည်👏။

## သင်ယူခြင်း ဒီမှာ မရပ်ပါ၊ ခရီးတစ်လျှောက် ဆက်လက်တိုးတက်ပါ။

သင်ခန်းစာပြီးဆုံးပြီးနောက် ကျွန်တော်တို့ရဲ့ [Generative AI Learning collection](https://aka.ms/genai-collection?WT.mc_id=academic-105485-koreyst) ကို စစ်ဆေးပြီး Generative AI သိပ္ပံဗဟုသုတ တိုးမြှင့်ပါ။
