# နယူးရယ် နက်ဝက် ဖရိမ်းဝက်များ

ကျွန်ုပ်တို့ အခုတင် သင်ယူထားသောအတိုင်း နယူးရယ် နက်ဝက်များကို ထိရောက်စွာ လေ့ကျင့်နိုင်ရန် အောက်ပါအရာ နှစ်ခု လိုအပ်ပါသည် -

* တန်ဆာများကို လုပ်ဆောင်ရန်၊ ဥပမာ - တိုးမြှင့်ခြင်း၊ ပေါင်းခြင်း၊ အချို့သော အလုပ်ဆောင်ချက်များကို တွက်ချက်ရန် (ဥပမာ - sigmoid သို့မဟုတ် softmax)
* အဆင့်ဆင့် လျှော့ချခြင်း (gradient descent optimization) လုပ်ဆောင်ရန် အတွက် အဆိုပါ အချက်အလက်များအားလုံး၏ ဂရေဒီအန့်များကို တွက်ချက်ရန်

`numpy` စာကြည့်တိုက်သည် ပထမပိုင်းကို လုပ်ဆောင်နိုင်သော်လည်း ဂရေဒီအန့်များကို တွက်ချက်ရန် အချို့သော ကိရိယာတစ်ခု လိုအပ်ပါသည်။ ယခင်ပိုင်းတွင် ဖွံ့ဖြိုးထားသော ဖရိမ်းဝက်တွင် `backward` နည်းလမ်းတွင် အနုတ်လက္ခဏာ လုပ်ဆောင်ချက်အားလုံးကို ကိုယ်တိုင်ရေးရပါသည်။ အကောင်းဆုံးဖြစ်ရန် ဖရိမ်းဝက်သည် ကျွန်ုပ်တို့ သတ်မှတ်နိုင်သည့် *မည်သည့် အချက်အလက်* ၏ ဂရေဒီအန့်ကို တွက်ချက်နိုင်စေရန် အခွင့်အလမ်း ပေးသင့်သည်။

အခြားအရေးကြီးသော အရာတစ်ခုမှာ GPU သို့မဟုတ် TPU ကဲ့သို့သော အထူးပြု ကွန်ပျူတာယူနစ်များတွင် တွက်ချက်မှုများကို လုပ်ဆောင်နိုင်စေရန်ဖြစ်သည်။ နက်ရှိုင်းသော နယူးရယ် နက်ဝက်လေ့ကျင့်ခြင်းတွင် *အလွန်များသော* တွက်ချက်မှုများ လိုအပ်သည်။ ထို့ကြောင့် ထိုတွက်ချက်မှုများကို GPU များတွင် ဆောင်ရွက်နိုင်ရန် အလွန်အရေးကြီးသည်။

> ✅ 'parallelize' ဆိုသော စကားလုံးသည် တွက်ချက်မှုများကို ကိရိယာများအပေါ်တွင် ဖြန့်ဝေခြင်းကို ဆိုလိုသည်။

လတ်တလောတွင် နယူးရယ် ဖရိမ်းဝက်များအနက် အလွန်လူကြိုက်များသော နှစ်ခုမှာ - TensorFlow နှင့် PyTorch ဖြစ်သည်။ နှစ်ခုစလုံးတွင် CPU နှင့် GPU နှစ်ခုလုံးတွင် တန်ဆာများဖြင့် လုပ်ဆောင်ရန် အနိမ့်အဆင့် API ကို ပေးပါသည်။ အနိမ့်အဆင့် API အပေါ်တွင် အမြင့်အဆင့် API ဖြစ်သော Keras နှင့် PyTorch Lightning တို့လည်း ရှိပါသည်။

Low-Level API | TensorFlow| PyTorch
--------------|-------------------------------------|--------------------------------
High-level API| Keras| Pytorch

**အနိမ့်အဆင့် API များ**သည် **တွက်ချက်မှု ဂရပ်များ** ဟုခေါ်သော အရာများကို တည်ဆောက်ရန် ခွင့်ပြုသည်။ ဤဂရပ်သည် ပေးထားသော အချက်အလက်များဖြင့် ထွက်ကုန်ကို (သာမန်အားဖြင့် ပျောက်ဆုံးမှုလုပ်ဆောင်ချက်) ဘယ်လို တွက်ချက်ရမည်ကို သတ်မှတ်ပေးပြီး GPU တွင် တွက်ချက်နိုင်ရန် ပို့နိုင်သည်။ ဤတွက်ချက်မှုဂရပ်ကို ကွဲပြားခြားနားစေရန် အလုပ်ဆောင်ချက်များ ရှိပြီး ဂရေဒီအန့်များကို တွက်ချက်နိုင်သည်။ ထို့နောက် မော်ဒယ်၏ အချက်အလက်များကို အကောင်းဆုံးဖြစ်စေရန် အသုံးပြုနိုင်သည်။

**အမြင့်အဆင့် API များ**သည် နယူးရယ်နက်ဝက်များကို **အလွှာများ၏ အစဉ်အလာ** အဖြစ် ပြန်လည်စဉ်းစားကာ နယူးရယ်နက်ဝက်များအများစုကို အလွယ်တကူ တည်ဆောက်နိုင်စေသည်။ မော်ဒယ်ကို လေ့ကျင့်ရန် အချက်အလက်များကို ပြင်ဆင်ပြီး အလုပ်ဆောင်ချက် `fit` ကို ခေါ်ဆိုရုံသာ လိုအပ်ပါသည်။

အမြင့်အဆင့် API သည် သာမန် နယူးရယ်နက်ဝက်များကို အလွန်မြန်မြန် ဆောက်နိုင်စေပြီး အသေးစိတ်များစွာကို မစိုးရိမ်ရပါ။ တစ်ချိန်တည်းမှာပင် အနိမ့်အဆင့် API သည် လေ့ကျင့်ခြင်း လုပ်ငန်းစဉ်ကို ထိန်းချုပ်နိုင်စေရန် ပိုမိုအထောက်အပံ့ ပေးနိုင်သောကြောင့် သုတေသနတွင် အသုံးပြုသည်။ အထူးသဖြင့် နယူး နက်ဝက်ဖွဲ့စည်းပုံအသစ်များကို ကိုင်တွယ်ရာတွင် အသုံးပြုသည်။

သင်သည် နှစ်ခုစလုံးကို တွဲဖက်အသုံးပြုနိုင်သည်ကို နားလည်ရပါသည်။ ဥပမာ - အနိမ့်အဆင့် API ကို အသုံးပြုပြီး သင်၏ကိုယ်ပိုင် နက်ဝက်လွှာ ဖွဲ့စည်းပုံကို ဖွံ့ဖြိုးနိုင်ပြီး အမြင့်အဆင့် API ဖြင့် တည်ဆောက်ပြီး လေ့ကျင့်ထားသော ကြီးမားသော နက်ဝက်တွင် အသုံးပြုနိုင်သည်။ သို့မဟုတ် အမြင့်အဆင့် API ကို အသုံးပြုပြီး အလွှာများအဖြစ် နက်ဝက်ကို သတ်မှတ်နိုင်ပြီး သင်၏ကိုယ်ပိုင် အနိမ့်အဆင့် လေ့ကျင့်မှု လှည့်လည်မှုကို အသုံးပြုပြီး အကောင်းဆုံးဖြစ်စေရန် လုပ်ဆောင်နိုင်သည်။ နှစ်ခုစလုံးသည် အခြေခံသဘောတရားများကို အသုံးပြုပြီး အချင်းချင်း ပေါင်းစပ်လုပ်ဆောင်နိုင်ရန် တည်ဆောက်ထားသည်။

## သင်ယူခြင်း

ဤသင်တန်းတွင် PyTorch နှင့် TensorFlow အတွက် အကြောင်းအရာများကို ပေးထားသည်။ သင့်အကြိုက်ဆုံး ဖရိမ်းဝက်ကို ရွေးချယ်ပြီး သက်ဆိုင်ရာ မှတ်စုများကိုသာ လေ့လာနိုင်ပါသည်။ သင်မည်သည့် ဖရိမ်းဝက်ကို ရွေးချယ်ရမည်ကို မသိပါက **PyTorch vs. TensorFlow** အကြောင်း အင်တာနက်တွင် ဆွေးနွေးချက်များကို ဖတ်ပါ။ ဖရိမ်းဝက်နှစ်ခုစလုံးကိုလည်း ကြည့်ပြီး နားလည်မှု ပိုမိုရနိုင်ပါသည်။

အခွင့်အလမ်း ရှိပါက တိုးတက်လွယ်ကူစေရန် အမြင့်အဆင့် API များကို အသုံးပြုပါမည်။ သို့သော် နယူးရယ်နက်ဝက်များ ဘယ်လို လုပ်ဆောင်သည့်ကို အခြေခံကနေ နားလည်ရသည်မှာ အရေးကြီးသည်ဟု ယုံကြည်ပါသည်။ ထို့ကြောင့် အစပိုင်းတွင် အနိမ့်အဆင့် API နှင့် တန်ဆာများကို သုံးပြီး အလုပ်လုပ်စတင်ပါမည်။ သို့သော် သင် အမြန်စတင်လိုပြီး အသေးစိတ်များကို သင်ယူရန် အချိန် မကုန်ချင်ပါက အမြင့်အဆင့် API မှတ်စုများသို့ တိုက်ရိုက် ကျော်သွားနိုင်ပါသည်။

## ✍️ လေ့ကျင့်ခန်းများ - ဖရိမ်းဝက်များ

အောက်ပါ မှတ်စုများတွင် သင့်လေ့လာမှုကို ဆက်လက်ဆောင်ရွက်ပါ -

Low-Level API | TensorFlow+Keras Notebook | PyTorch
--------------|-------------------------------------|--------------------------------
High-level API| Keras | *PyTorch Lightning*

ဖရိမ်းဝက်များကို ကျွမ်းကျင်ပြီးလျှင် အလွန်အရေးကြီးသော အကြောင်းအရာ - overfitting အကြောင်း ပြန်လည်သုံးသပ်ကြပါစို့။

# Overfitting

Overfitting သည် စက်ရုပ် သင်ယူမှုတွင် အလွန်အရေးကြီးသော အကြောင်းအရာတစ်ခုဖြစ်ပြီး သေချာသိထားရပါမည်။

အောက်ပါ ၅ ခုသော အချက်အလက်များကို (အောက်တွင် ရှိသော ဂရပ်တွင် `x` ဖြင့် ဖော်ပြထားသည်) ခန့်မှန်းရန် ပြဿနာကို စဉ်းစားပါ -

!linear | overfit
-------------------------|--------------------------
**Linear model, 2 parameters** | **Non-linear model, 7 parameters**
Training error = 5.3 | Training error = 0
Validation error = 5.1 | Validation error = 20

* ဘယ်ဘက်တွင် အချက်အလက်များကို ကောင်းစွာ ခန့်မှန်းထားသော တည့်ညီသော လိုင်းတစ်ခုကို မြင်ရသည်။ အချက်အလက်များ၏ လက်တွေ့မှုကို မှန်ကန်စွာ နားလည်နိုင်သည်။
* ညာဘက်တွင် မော်ဒယ်သည် အလွန်အစွမ်းထက်သည်။ အချက်အလက် ၅ ခုသာ ရှိပြီး မော်ဒယ်တွင် အချက်အလက် ၇ ခုရှိသောကြောင့် အချက်အလက်များအားလုံးကို ဖြတ်သွားရန် ပြင်ဆင်နိုင်သည်။ ထို့ကြောင့် လေ့ကျင့်မှု အမှားသည် ၀ ဖြစ်သည်။ သို့သော် အချက်အလက်များ၏ ပုံစံမှန်ကန်မှုကို နားလည်ရန် မဖြစ်စေပါ။ ထို့ကြောင့် အတည်ပြုမှု အမှားသည် အလွန်မြင့်သည်။

မော်ဒယ်၏ သာလွန်မှု (အချက်အလက် အရေအတွက်) နှင့် လေ့ကျင့်မှု အချက်အလက်များ၏ အရေအတွက်ကြား မှန်ကန်သော လိုက်လျောမှုကို ရှာဖွေရန် အလွန်အရေးကြီးသည်။

## Overfitting ဖြစ်ပေါ်ရခြင်း၏ အကြောင်းရင်းများ

  * လေ့ကျင့်မှု အချက်အလက် မလုံလောက်ခြင်း
  * အလွန်အစွမ်းထက်သော မော်ဒယ်
  * အချက်အလက်များတွင် ဆူညံမှု အများကြီး ရှိခြင်း

## Overfitting ကို ဘယ်လို ရှာဖွေမည်

အထက်ပါ ဂရပ်မှ ကြည့်ရင် overfitting ကို အလွန်နည်းသော လေ့ကျင့်မှု အမှားနှင့် အမြင့်သော အတည်ပြုမှု အမှားဖြင့် ရှာဖွေနိုင်သည်။ သာမန်အားဖြင့် လေ့ကျင့်မှုအတွင်း လေ့ကျင့်မှုနှင့် အတည်ပြုမှု အမှားများ သက်သာလာပြီး တစ်ချိန်တွင် အတည်ပြုမှု အမှားသည် သက်သာမှု ရပ်ပြီး မြင့်တက်လာနိုင်သည်။ ဤသည်မှာ overfitting ၏ လက္ခဏာဖြစ်ပြီး ဤအချိန်တွင် လေ့ကျင့်မှု ရပ်ရန် (သို့မဟုတ် အနည်းဆုံး မော်ဒယ်ကို မှတ်ထားရန်) လက္ခဏာဖြစ်သည်။

## Overfitting ကို ဘယ်လို ကာကွယ်မည်

Overfitting ဖြစ်ပေါ်နေသည်ကို မြင်ရလျှင် အောက်ပါအရာတစ်ခုခု လုပ်နိုင်သည် -

 * လေ့ကျင့်မှု အချက်အလက်များ ပမာဏ တိုးမြှင့်ပါ
 * မော်ဒယ်၏ စုစုပေါင်း အချက်အလက်များ လျှော့ချပါ
 * Dropout ကဲ့သို့သော ပုံမှန်လုပ်ဆောင်ချက်ကို အသုံးပြုပါ၊ ဤအကြောင်းကို နောက်ပိုင်းတွင် စဉ်းစားပါမည်။

## Overfitting နှင့် Bias-Variance Tradeoff

Overfitting သည် စာရင်းအင်းသင်္ချာတွင် Bias-Variance Tradeoff ဟုခေါ်သော ပိုမိုမျိုးစုံသော ပြဿနာ၏ အခြေအနေတစ်ခုဖြစ်သည်။ မော်ဒယ်တွင် ဖြစ်နိုင်သော အမှားရင်းမြစ်များကို စဉ်းစားပါက အမှားနှစ်မျိုးကို တွေ့နိုင်သည် -

* **Bias အမှားများ**သည် လေ့ကျင့်မှု အချက်အလက်များကြား သက်ဆိုင်မှုကို ကျွန်ုပ်တို့၏ အယ်လဂိုရီသမ်မှ မှန်ကန်စွာ ဖမ်းယူနိုင်ခြင်း မရှိခြင်းကြောင့် ဖြစ်ပေါ်သည်။ ဤသည်မှာ ကျွန်ုပ်တို့၏ မော်ဒယ်သည် အစွမ်းမထက်ခြင်း (**underfitting**) ကြောင့် ဖြစ်နိုင်သည်။
* **Variance အမှားများ**သည် အချက်အလက်များတွင် အဓိပ္ပါယ်ရှိသော သက်ဆိုင်မှုကို ဖမ်းယူခြင်း မလုပ်ဘဲ ဆူညံမှုကို ခန့်မှန်းခြင်းကြောင့် ဖြစ်ပေါ်သည် (**overfitting**)။

လေ့ကျင့်မှုအတွင်း bias အမှားသည် လျော့နည်းလာသည် (ကျွန်ုပ်တို့၏ မော်ဒယ်သည် အချက်အလက်များကို ခန့်မှန်းသင်ယူသောကြောင့်) နှင့် variance အမှားသည် တိုးလာသည်။ overfitting ကို ကာကွယ်ရန် လေ့ကျင့်မှုကို ရပ်တန့်ရန် - လက်ဖြင့် (overfitting ကို ရှာဖွေသောအခါ) သို့မဟုတ် အလိုအလျောက် (ပုံမှန်လုပ်ဆောင်ချက်ကို ထည့်သွင်းခြင်းဖြင့်) - အရေးကြီးသည်။

## နိဂုံးချုပ်

ဤသင်ခန်းစာတွင် AI ဖရိမ်းဝက်နှစ်ခုဖြစ်သော TensorFlow နှင့် PyTorch အတွက် အမျိုးမျိုးသော API များ၏ ကွာခြားချက်များအကြောင်း သင်ယူခဲ့ပါသည်။ ထို့အပြင် အလွန်အရေးကြီးသော အကြောင်းအရာ - overfitting အကြောင်းလည်း သင်ယူခဲ့ပါသည်။

## 🚀 စိန်ခေါ်မှု

ပါဝင်သော မှတ်စုများတွင် 'tasks' ဆိုသော အကြောင်းအရာများကို အောက်တွင် တွေ့ရပါမည်။ မှတ်စုများကို လေ့လာပြီး tasks များကို ပြီးမြောက်ပါ။

## ပြန်လည်ဆန်းစစ်ခြင်းနှင့် ကိုယ်ပိုင်လေ့လာမှု

အောက်ပါ အကြောင်းအရာများကို သုတေသန လုပ်ပါ -

- TensorFlow
- PyTorch
- Overfitting

အောက်ပါ မေးခွန်းများကို ကိုယ်တိုင် မေးပါ -

- TensorFlow နှင့် PyTorch ၏ ကွာခြားချက်က ဘာလဲ?
- overfitting နှင့် underfitting ၏ ကွာခြားချက်က ဘာလဲ?

## လုပ်ငန်းခွင်

ဤဓာတ်ခွင်တွင် PyTorch သို့မဟုတ် TensorFlow ကို အသုံးပြုပြီး အထပ်တန်းများနှင့် အများအပြားအလွှာများဖြင့် အပြည့်အဝ ဆက်သွယ်ထားသော နက်ဝက်များကို အသုံးပြု၍ အမျိုးအစားခွဲခြားမှု ပြဿနာနှစ်ခုကို ဖြေရှင်းရန် တိုက်တွန်းပါသည်။
