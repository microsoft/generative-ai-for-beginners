# နေရူရယ်ကွန်ရက်များကို မိတ်ဆက်ခြင်း။ မျိုးစုံအလွှာတွဲထားသော Perceptron

ယခင်ပိုင်းတွင် သင်သည် အလွန်ရိုးရှင်းသော နေရူရယ်ကွန်ရက်ပုံစံတစ်ခုဖြစ်သော - တစ်လွှာတည်းရှိသော perceptron၊ လျင်မြန်သော နှစ်ခွဲခွဲခြားခြင်းပုံစံကို လေ့လာခဲ့သည်။

ဤပိုင်းတွင် ဤပုံစံကို ပိုမိုစွမ်းဆောင်ရည်ရှိသော ဖွဲ့စည်းပုံတစ်ခုအဖြစ် တိုးချဲ့သွားမည်ဖြစ်ပြီး၊ ၎င်းသည် ကျွန်ုပ်တို့အား အောက်ပါအတိုင်း ခွင့်ပြုပါမည် -

* နှစ်ခွဲအပြင် **မျိုးစုံခွဲခြားခြင်း** ကို ဆောင်ရွက်နိုင်ခြင်း
* ခွဲခြားခြင်းအပြင် **ပြန်ကြားရေးပြဿနာများ** ကို ဖြေရှင်းနိုင်ခြင်း
* လျင်မြန်စွာခွဲခြားနိုင်သော မဟုတ်သော အတန်းများကို ခွဲခြားခြင်း

ကျွန်ုပ်တို့သည် Python တွင် မိမိတို့ရဲ့ အစိတ်အပိုင်းများဖြစ်သော ဖွဲ့စည်းပုံတစ်ခုကို ဖွံ့ဖြိုးတိုးတက်စေရန် လုပ်ဆောင်မည်ဖြစ်ပြီး၊ ၎င်းသည် ကျွန်ုပ်တို့အား ကွဲပြားသော နေရူရယ်ကွန်ရက် ဖွဲ့စည်းပုံများကို တည်ဆောက်နိုင်စေရန် ခွင့်ပြုပါမည်။

## စက်လေ့လာခြင်း၏ တရားဝင်ခြင်း

စက်လေ့လာခြင်းပြဿနာကို တရားဝင်ပြုလုပ်ခြင်းဖြင့် စတင်ကြစို့။ ကျွန်ုပ်တို့တွင် သင်ကြားမှုဒေတာအစုံ **X** နှင့် အမှတ်အသားများ **Y** ရှိသည်ဟု ချက်ချင်း ယူဆပါ၊ နှင့် ကျွန်ုပ်တို့သည် အတိအကျဆုံး ခန့်မှန်းချက်များ ပြုလုပ်နိုင်ရန် *f* မော်ဒယ်တစ်ခု တည်ဆောက်ရန် လိုအပ်သည်။ ခန့်မှန်းချက်များ၏ အရည်အသွေးကို **Loss function** ℒ ဖြင့် တိုင်းတာသည်။ အောက်ပါ loss functions များကို မကြာခဏ အသုံးပြုသည် -

* ပြန်ကြားရေးပြဿနာအတွက်၊ အရေအတွက်တစ်ခုကို ခန့်မှန်းရန် လိုအပ်သောအခါ၊ **absolute error** ∑<sub>i</sub>|f(x<sup>(i)</sup>)-y<sup>(i)</sup>| သို့မဟုတ် **squared error** ∑<sub>i</sub>(f(x<sup>(i)</sup>)-y<sup>(i)</sup>)<sup>2</sup> ကို အသုံးပြုနိုင်သည်
* ခွဲခြားခြင်းအတွက်၊ **0-1 loss** (မော်ဒယ်၏ **တိကျမှု** အဖြစ် အခြေခံအားဖြင့် အတူတူဖြစ်သည်) သို့မဟုတ် **logistic loss** ကို အသုံးပြုသည်။

တစ်လွှာတည်းရှိသော perceptron အတွက်၊ *f* function ကို လျင်မြန်သော function *f(x)=wx+b* အဖြစ် သတ်မှတ်ထားသည် (ဤနေရာတွင် *w* သည် အလေးချိန် matrix ဖြစ်ပြီး၊ *x* သည် input features များ၏ ဗက်တာဖြစ်ပြီး၊ *b* သည် bias vector ဖြစ်သည်)။ ကွဲပြားသော နေရူရယ်ကွန်ရက် ဖွဲ့စည်းပုံများအတွက်၊ ဤ function သည် ပိုမိုရှုပ်ထွေးသော ပုံစံယူနိုင်သည်။

> ခွဲခြားခြင်းအတွက်၊ သက်ဆိုင်ရာ အတန်းများ၏ အလားအလာများကို ကွန်ရက် output အဖြစ် ရရှိရန် အထူးလိုအပ်သည်။ သမားရိုးကျနံပါတ်များကို အလားအလာများသို့ ပြောင်းလဲရန် (ဥပမာ output ကို ပုံမှန်အဖြစ်ပြုလုပ်ရန်)၊ **softmax** function σ ကို မကြာခဏ အသုံးပြုသည်၊ နှင့် *f* function သည် *f(x)=σ(wx+b)* ဖြစ်လာသည်

* ပုံမှန် *f* ၏ အဓိပ္ပာယ်တွင်၊ *w* နှင့် *b* ကို **parameters** θ=⟨*w,b*⟩ ဟု ခေါ်သည်။ **X**,**Y** အစုံကို ထည့်သွင်းပါက၊ ကျွန်ုပ်တို့သည် ဒေတာအစုံအပေါ် အမှားတစ်ခုလုံးကို parameters θ ၏ function အဖြစ်တွက်ချက်နိုင်သည်။

> ✅ **နေရူရယ်ကွန်ရက် သင်ကြားမှု၏ ရည်မှန်းချက်မှာ parameters θ ကို ပြောင်းလဲခြင်းဖြင့် အမှားကို လျှော့ချရန် ဖြစ်သည်**

## Gradient Descent Optimization

function optimization ၏ နာမည်ကျော် နည်းလမ်းတစ်ခုမှာ **gradient descent** ဖြစ်သည်။ အကြောင်းအရာမှာ loss function ၏ gradient (မူလအနေဖြင့် derivative) ကို parameters အပေါ် တွက်ချက်ပြီး၊ အမှားကို လျှော့ချနိုင်သော နည်းလမ်းဖြင့် parameters များကို ပြောင်းလဲခြင်းဖြစ်သည်။ ဤကို အောက်ပါအတိုင်း တရားဝင်ပြုလုပ်နိုင်သည် -

* parameters များကို ကျပန်းတန်ဖိုး w<sup>(0)</sup>, b<sup>(0)</sup> ဖြင့် စတင်ပါ
* အောက်ပါအဆင့်ကို မကြာခဏ ထပ်ခါထပ်ခါ ပြုလုပ်ပါ -
    - w<sup>(i+1)</sup> = w<sup>(i)</sup>-η∂ℒ/∂w
    - b<sup>(i+1)</sup> = b<sup>(i)</sup>-η∂ℒ/∂b

သင်ကြားမှုအတွင်းတွင်၊ optimization အဆင့်များကို ဒေတာအစုံကို ထည့်သွင်းစဉ်းစားပြီး တွက်ချက်ရန် ရည်ရွယ်ထားသည် (loss သည် သင်ကြားမှုနမူနာများအားလုံးမှတဆင့် စုစုပေါင်းအဖြစ် တွက်ချက်သည်ကို မှတ်သားပါ။ သို့သော်၊ အမှန်တကယ်တွင် ကျွန်ုပ်တို့သည် **minibatches** ဟုခေါ်သော ဒေတာအစိတ်အပိုင်းများကို ယူပြီး၊ ဒေတာအစုစုတစ်ခုအပေါ် မူတည်၍ gradients များကို တွက်ချက်သည်။ အစုစုကို အကြိမ်တိုင်း ကျပန်းရွေးချယ်ထားသောကြောင့်၊ ဤနည်းလမ်းကို **stochastic gradient descent** (SGD) ဟု ခေါ်သည်။

## မျိုးစုံအလွှာတွဲထားသော Perceptrons နှင့် Backpropagation

တစ်လွှာတည်းရှိသော ကွန်ရက်သည် အထက်ပါအတိုင်း လျင်မြန်စွာခွဲခြားနိုင်သော အတန်းများကို ခွဲခြားနိုင်သည်။ ပိုမိုချမ်းသာသော ပုံစံတစ်ခုတည်ဆောက်ရန်၊ ကျွန်ုပ်တို့သည် ကွန်ရက်၏ အလွှာများစွာကို ပေါင်းစည်းနိုင်သည်။ သင်္ချာပိုင်းဆိုင်ရာ အနေနှင့်၊ function *f* သည် ပိုမိုရှုပ်ထွေးသော ပုံစံယူမည်ဖြစ်ပြီး၊ အဆင့်အတန်းများစွာတွင် တွက်ချက်မည်ဖြစ်သည် -

* z<sub>1</sub>=w<sub>1</sub>x+b<sub>1</sub>
* z<sub>2</sub>=w<sub>2</sub>α(z<sub>1</sub>)+b<sub>2</sub>
* f = σ(z<sub>2</sub>)

ဤနေရာတွင်၊ α သည် **မလျင်မြန်သော activation function** ဖြစ်ပြီး၊ σ သည် softmax function ဖြစ်သည်၊ နှင့် parameters θ=<*w<sub>1</sub>,b<sub>1</sub>,w<sub>2</sub>,b<sub>2</sub>*> ဖြစ်သည်။

gradient descent algorithm သည် အတူတူမည်ဖြစ်သော်လည်း၊ gradients များကို တွက်ချက်ရန် ပိုမိုခက်ခဲမည်ဖြစ်သည်။ chain differentiation rule အရ၊ derivatives များကို အောက်ပါအတိုင်း တွက်ချက်နိုင်သည် -

* ∂ℒ/∂w<sub>2</sub> = (∂ℒ/∂σ)(∂σ/∂z<sub>2</sub>)(∂z<sub>2</sub>/∂w<sub>2</sub>)
* ∂ℒ/∂w<sub>1</sub> = (∂ℒ/∂σ)(∂σ/∂z<sub>2</sub>)(∂z<sub>2</sub>/∂α)(∂α/∂z<sub>1</sub>)(∂z<sub>1</sub>/∂w<sub>1</sub>)

> ✅ chain differentiation rule သည် parameters များအပေါ် loss function ၏ derivatives များကို တွက်ချက်ရန် အသုံးပြုသည်။

ဤအထိမ်းအမှတ်များအားလုံး၏ ဘယ်ဘက်ဆုံး အပိုင်းသည် အတူတူဖြစ်ကြောင်း သတိပြုပါ၊ ထို့ကြောင့် ကျွန်ုပ်တို့သည် loss function မှစ၍ "နောက်ပြန်" တွက်ချက်ရေးပြုလုပ်နိုင်သည်။ ထို့ကြောင့် မျိုးစုံအလွှာတွဲထားသော perceptron သင်ကြားမှုနည်းလမ်းကို **backpropagation** သို့မဟုတ် 'backprop' ဟု ခေါ်သည်။
