# နယူးရယ် နက်ဝက်များအတွက် အကျဉ်းချုပ်: Perceptron

ခေတ်သစ် နယူးရယ် နက်ဝက်တစ်ခုနဲ့ ဆင်တူတဲ့ အရာတစ်ခုကို အကောင်အထည်ဖော်ဖို့ လုပ်ဆောင်မှုအထဲက တစ်ခုကတော့ 1957 ခုနှစ်မှာ Cornell Aeronautical Laboratory မှာ Frank Rosenblatt က လုပ်ခဲ့တာပါ။ ဒါက "Mark-1" လို့ခေါ်တဲ့ ဟာ့ဒ်ဝဲတစ်ခုဖြစ်ပြီး တြိဂံ၊ စတုဂံနဲ့ စက်ဝိုင်းလို အခြေခံ ဂျီဩမေရိပုံစံတွေကို မှတ်မိဖို့ ဖန်တီးထားတာပါ။

ထိတွေ့မယ့် ပုံရိပ်တစ်ခုကို 20x20 ဖိုတိုဆဲလ် အစုအဖွဲ့နဲ့ ကိုယ်စားပြုထားတာဖြစ်ပြီး၊ နယူးရယ် နက်ဝက်မှာ 400 ခုရောက်တဲ့ အထွက်နဲ့ အခြားတစ်ခုသော ဘိုင်နရီ အထွက်ရှိပါတယ်။ ရိုးရှင်းတဲ့ နက်ဝက်တစ်ခုမှာ နယူးရွန်တစ်ခုပါဝင်ပြီး၊ **threshold logic unit** လို့လည်း ခေါ်ပါတယ်။ နယူးရယ် နက်ဝက် အလေးချိန်တွေက လေ့ကျင့်မှု အဆင့်မှာ လက်ဖြင့် ပြုပြင်ရတဲ့ ပိုတီနိုမီတာလို လုပ်ဆောင်ခဲ့ပါတယ်။

> The New York Times က အဲဒီအချိန်မှာ perceptron အကြောင်းကို ရေးသားခဲ့ပါတယ်: *အဲဒီအခါ [ရေတပ်] က လမ်းလျှောက်နိုင်၊ စကားပြောနိုင်၊ မြင်နိုင်၊ ရေးသားနိုင်၊ ကိုယ့်ကိုယ်ကို ပြန်လည်ထုတ်လုပ်နိုင်ပြီး၊ ကိုယ့်ရဲ့ တည်ရှိမှုကို သိမြင်နိုင်တဲ့ အီလက်ထရွန်နစ် ကွန်ပျူတာရဲ့ မွေးကင်းစကလေးကို မျှော်လင့်ခဲ့ပါတယ်။*

## Perceptron မော်ဒယ်

ငါတို့ရဲ့ မော်ဒယ်မှာ N လက္ခဏာတွေရှိတယ်လို့ သတ်မှတ်ကြပါစို့၊ အဲဒီအခါ input ဗက်တာက N အရွယ်အစားရှိတဲ့ ဗက်တာဖြစ်မှာပါ။ Perceptron က **binary classification** မော်ဒယ်ဖြစ်ပြီး၊ input ဒေတာ အမျိုးအစားနှစ်မျိုးကို ခွဲခြားနိုင်ပါတယ်။ အောက်ပါ input ဗက်တာ x အတွက် အထွက်ဟာ အမျိုးအစားအပေါ်မူတည်ပြီး +1 ဒါမှမဟုတ် -1 ဖြစ်မယ်လို့ ခန့်မှန်းကြပါမယ်။ အထွက်ကို အောက်ပါနည်းလမ်းနဲ့ တွက်ချက်မှာပါ:

y(x) = f(w<sup>T</sup>x)

f ကတော့ အဆင့်သတ်မှတ် activation function ဖြစ်ပါတယ်

## Perceptron ကို လေ့ကျင့်ခြင်း

Perceptron ကို လေ့ကျင့်ဖို့ အထွက်တန်ဖိုးများကို အမှန်ဆုံး ခွဲခြားပေးနိုင်တဲ့ အလေးချိန်ဗက်တာ w ကို ရှာဖွေဖို့ လိုအပ်ပါတယ်။ ဒါကတော့ အနည်းဆုံး **အမှား** ဖြစ်ပေါ်စေဖို့ ဖြစ်ပါတယ်။ ဒီအမှားကို **perceptron criterion** နည်းလမ်းနဲ့ အောက်ပါအတိုင်း သတ်မှတ်ထားပါတယ်:

E(w) = -∑w<sup>T</sup>x<sub>i</sub>t<sub>i</sub>

ဒီမှာ:

* စုစုပေါင်းကို အမှားဖြစ်တဲ့ လေ့ကျင့်မှု ဒေတာအချက်အလက် i အပေါ်မှာ ယူထားတာပါ
* x<sub>i</sub> က input ဒေတာဖြစ်ပြီး၊ t<sub>i</sub> က အနုတ်လက္ခဏာနဲ့ အပေါင်းလက္ခဏာ နမူနာအတွက် -1 ဒါမှမဟုတ် +1 ဖြစ်ပါတယ်။

ဒီစံချိန်ကို အလေးချိန် w ရဲ့ အလုပ်လုပ်ပုံအဖြစ် သတ်မှတ်ပြီး၊ ဒါကို အနည်းဆုံးဖြစ်အောင် လုပ်ဖို့ လိုအပ်ပါတယ်။ အများအားဖြင့် **gradient descent** လို့ခေါ်တဲ့ နည်းလမ်းကို အသုံးပြုကြပြီး၊ အစပိုင်းအလေးချိန် w<sup>(0)</sup> နဲ့ စပြီး၊ နောက်ထပ်အဆင့်တစ်ခုချင်းစီမှာ အောက်ပါနည်းလမ်းနဲ့ အလေးချိန်တွေကို ပြုပြင်မှာပါ:

w<sup>(t+1)</sup> = w<sup>(t)</sup> - η∇E(w)

ဒီမှာ η က **learning rate** လို့ခေါ်တဲ့ အရာဖြစ်ပြီး၊ ∇E(w) က E ရဲ့ **gradient** ကို ဆိုလိုပါတယ်။ gradient ကို တွက်ချက်ပြီးနောက်မှာ

w<sup>(t+1)</sup> = w<sup>(t)</sup> + ∑ηx<sub>i</sub>t<sub>i</sub>

Python ကနေ အဲဒီအဆင့်ဆင့်နည်းလမ်းကို အောက်ပါအတိုင်းရေးထားပါတယ်:

```python
def train(positive_examples, negative_examples, num_iterations = 100, eta = 1):

    weights = [0,0,0] # Initialize weights (almost randomly :)
        
    for i in range(num_iterations):
        pos = random.choice(positive_examples)
        neg = random.choice(negative_examples)

        z = np.dot(pos, weights) # compute perceptron output
        if z < 0: # positive example classified as negative
            weights = weights + eta*weights.shape

        z  = np.dot(neg, weights)
        if z >= 0: # negative example classified as positive
            weights = weights - eta*weights.shape

    return weights
```

## နိဂုံးချုပ်

ဒီသင်ခန်းစာမှာ၊ ဘိုင်နရီ ခွဲခြားမှု မော်ဒယ်ဖြစ်တဲ့ perceptron အကြောင်း နဲ့ အလေးချိန်ဗက်တာကို အသုံးပြု၍ အဲဒါကို ဘယ်လိုလေ့ကျင့်ရမယ်ဆိုတာကို သင်ယူခဲ့ပါတယ်။
