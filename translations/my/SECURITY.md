## လုံခြုံရေး

Microsoft သည် ကျွန်ုပ်တို့၏ ဆော့ဖ်ဝဲ ထုတ်ကုန်များနှင့် ဝန်ဆောင်မှုများ၏ လုံခြုံရေးကို အလေးထားပါသည်၊ ဤသည်မှာ ကျွန်ုပ်တို့၏ GitHub အဖွဲ့အစည်းများမှ စီမံခန့်ခွဲသော အရင်းအမြစ်ကုဒ် ရာထူးများအားလုံး ပါဝင်သည်၊ [Microsoft](https://github.com/microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet), [Xamarin](https://github.com/xamarin) နှင့် [ကျွန်ုပ်တို့၏ GitHub အဖွဲ့အစည်းများ](https://opensource.microsoft.com/) ပါဝင်သည်။

Microsoft ပိုင်ရှင်ရာထူးများတွင် [Microsoft ၏ လုံခြုံရေး အခွင့်အရေးအဖွဲ့ အဓိပ္ပာယ်](https://aka.ms/opensource/security/definition) ကို ဖြည့်ဆည်းသည့် လုံခြုံရေး အခွင့်အရေးကို ရှာဖွေတွေ့ရှိခဲ့သည်ဟု သင် ယုံကြည်ပါက အောက်တွင် ဖော်ပြထားသော အတိုင်း ကျွန်ုပ်တို့ထံ အကြောင်းကြားပါ။

## လုံခြုံရေး အကြောင်းအရာများကို တင်ပြခြင်း

**GitHub အများသုံး အကြောင်းအရာများမှတဆင့် လုံခြုံရေး အခွင့်အရေးများကို မတင်ပြပါနှင့်။**

ထို့အစား Microsoft Security Response Center (MSRC) သို့ [https://msrc.microsoft.com/create-report](https://aka.ms/opensource/security/create-report) တွင် တင်ပြပါ။

လော့ဂ်အင်မဝင်ဘဲ တင်ပြလိုပါက [secure@microsoft.com](mailto:secure@microsoft.com) သို့ အီးမေးလ် ပေးပို့ပါ။  သင်၏ မက်ဆေ့ခ်ျကို ကျွန်ုပ်တို့၏ PGP key ဖြင့် စကားဝှက်ထိုးနိုင်ပါက ကျေးဇူးပြု၍ [Microsoft Security Response Center PGP Key page](https://aka.ms/opensource/security/pgpkey) မှ ဒေါင်းလုပ်ဆွဲပါ။

သင် ၂၄ နာရီအတွင်း တုံ့ပြန်မှု ရရှိသင့်သည်။ အကြောင်းမဲ့မရှိဘဲ မရရှိပါက သင်၏ မူရင်း မက်ဆေ့ခ်ျကို ကျွန်ုပ်တို့ရရှိခဲ့ကြောင်း သေချာစေရန် အီးမေးလ်မှတဆင့် နောက်ထပ်ဆက်သွယ်ပါ။ အပိုဆောင်းအချက်အလက်များကို [microsoft.com/msrc](https://aka.ms/opensource/security/msrc) တွင် ရှာဖွေနိုင်သည်။

အကြောင်းအရာ၏ လက်တွေ့အသုံးပြုမှုကို ပိုမိုနားလည်စေရန်အတွက် အောက်ပါ ဖော်ပြချက်များ (သင်ပေးနိုင်သည့်အတိုင်း) ပါဝင်စေရန် ကျေးဇူးပြုပါ:

  * အခွင့်အရေး အမျိုးအစား (ဥပမာ buffer overflow, SQL injection, cross-site scripting, စသည်)
  * အခွင့်အရေး၏ လက္ခဏာနှင့် ဆက်စပ်သော အရင်းအမြစ် ဖိုင်(များ)၏ လမ်းကြောင်းပြည့်စုံ
  * အကျိုးသက်ရောက်သော အရင်းအမြစ်ကုဒ်၏ တည်နေရာ (tag/branch/commit သို့မဟုတ် direct URL)
  * အခွင့်အရေးကို ထပ်မံဖျော်ဖြေရန် လိုအပ်သော အထူး configuration
  * အခွင့်အရေးကို ဖျော်ဖြေရန် လိုအပ်သော လှုပ်ရှားမှုအဆင့်ဆင့်
  * Proof-of-concept သို့မဟုတ် exploit code (ဖြစ်နိုင်ပါက)
  * အခွင့်အရေး၏ သက်ရောက်မှု၊ အခွင့်အရေးကို တော်လှန်ဖျော်ဖြေရန် တိုက်ဖျက်သူက ဘယ်လို exploit လုပ်နိုင်မည်ဆိုသည်ကို အပါအဝင်

ဤအချက်အလက်များသည် သင်၏ အကြောင်းကြားချက်ကို ပိုမိုလျင်မြန်စွာ စစ်ဆေးရန် ကူညီပါမည်။

သင် bug bounty အတွက် တင်ပြနေပါက၊ ပိုမိုပြည့်စုံသော အကြောင်းကြားချက်များသည် ပိုမိုမြင့်မားသော bounty ဆုချီးမြှင့်မှုကို အထောက်အကူဖြစ်စေပါသည်။ ကျွန်ုပ်တို့၏ လက်ရှိအစီအစဉ်များအကြောင်း အသေးစိတ်ကို [Microsoft Bug Bounty Program](https://aka.ms/opensource/security/bounty) စာမျက်နှာတွင် ကြည့်ရှုပါ။

## အကြိုက်ဆုံး ဘာသာစကားများ

ကျွန်ုပ်တို့သည် အီးမေးလ်ဆက်သွယ်မှုများအားလုံးကို အင်္ဂလိပ်ဘာသာဖြင့် ကြိုက်ပါသည်။

## မူဝါဒ
