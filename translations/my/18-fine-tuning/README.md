# သင့် LLM ကို ပိုမိုတိကျစွာ ထိန်းမြှုပ်ခြင်း

ကြီးမားသောဘာသာစကားမော်ဒယ်များကို အသုံးပြုထားသောဖန်တီး AI အပလီကေးရှင်းများတည်ဆောက်ခြင်းတွင် စိန်ခေါ်မှုအသစ်များ ရှိပါသည်။ အဓိကပြဿနာမှာ အသုံးပြုသူ၏ တောင်းဆိုမှုအတွက် မော်ဒယ်က ရေးထုတ်သည့် ဝေါဟာရ၏ တုံ့ပြန်မှုအရည်အသွေး (တိကျမှုနှင့် သက်ဆိုင်မှု) ကို သေချာစေရန် ဖြစ်သည်။ ယခင်သင်ခန်းစာများတွင်၊ ကျွန်ုပ်တို့သည် စောင့်ကြည့်ရန်အတိုင်း _prompt engineering_ နှင့် _retrieval-augmented generation_ ကဲ့သို့သော နည်းလမ်းများကို ပြောကြားခဲ့ပြီး၊ ရှိပြီးသား မော်ဒယ်သို့ _prompt input ကို ပြင်ဆင်ခြင်း_ ဖြင့် ပြဿနာကို ဖြေရှင်းရန် ကြိုးပမ်းသည်။

ယနေ့သင်ခန်းစာတွင် ကျွန်ုပ်တို့သည် မော်ဒယ်ကို _နည်းနည်းနဲ့ ထပ်မံသင်ကြားခြင်း_ဖြင့် စိန်ခေါ်မှုကို ဖြေရှင်းရန် ကြိုးပမ်းသော နည်းလမ်းတစ်ခု인 **fine-tuning** ကို ဆွေးနွေးပါမည်။ အသေးစိတ်သို့ ရောက်ကြပါစို့။

## သင်ယူရမည့်ရည်ရွယ်ချက်များ

ဤသင်ခန်းစာသည် အကြိုသင်ကြားထားသောဘာသာစကားမော်ဒယ်များအတွက် fine-tuning အကြောင်းအရာကို မိတ်ဆက်ပေးပြီး၊ ဤနည်းလမ်း၏ အကျိုးကျေးဇူးများနှင့် စိန်ခေါ်မှုများကို ရှင်းလင်းကာ သင့် generative AI မော်ဒယ်များ၏ စွမ်းဆောင်ရည်တိုးတက်စေရန် fine-tuning ကို ဘယ်အချိန်တွင်၊ ဘယ်လိုအသုံးပြုရမယ်ဆိုတာ လမ်းညွှန်ပေးပါမည်။

ဤသင်ခန်းစာပြီးဆုံးသည်အထိ၊ သင်သည် အောက်ပါမေးခွန်းများကို ဖြေကြားနိုင်ရမည်ဖြစ်သည်။

- ဘာသာစကားမော်ဒယ်များအတွက် fine-tuning ဆိုတာဘာလဲ?
- ဘယ်အချိန်တွင်၊ ဘာကြောင့် fine-tuning သုံးရတာ အကျိုးရှိသလဲ?
- မည့်သို့ pre-trained မော်ဒယ်ကို fine-tune လုပ်နိုင်မလဲ?
- fine-tuning ၏ ကန့်သတ်ချက်များ ဘာတွေလဲ?

ပြင်ဆင်ပြီးလား? စတင်လုပ်ဆောင်ကြစို့။

## ပုံဖော်ပြထားသော လမ်းညွှန်စာမျက်နှာ

ကျွန်ုပ်တို့ လုပ်ဆောင်မည့် အဓိကအကြောင်းအရာများကို စတင်မတွင်မြင်လိုပါသလား? ဤ fine-tuning ၏ အခြေခံအတိုင်းအတာနှင့် အကျိုးဖြစ်စဉ်၊ လုပ်ဆောင်ရန် အကောင်းဆုံးနည်းလမ်းများကို ဖော်ပြသည့် ပုံဖော်ပြထားသော လမ်းညွှန်စာမျက်နှာကို ကြည့်ပါ။ ဤကဏ္ဍမှာ စူးစမ်းလေ့လာရန် စိတ်ဝင်စားဖွယ်အကြောင်းအရာဖြစ်သည်၊ ထို့ကြောင့် [Resources](./RESOURCES.md?WT.mc_id=academic-105485-koreyst) စာမျက်နှာတွင် သင်၏ ကိုယ်ပိုင်လေ့လာမှု ခရီးစဉ်အတွက် ထောက်ပံ့မှုများအတွက် နောက်ထပ် လင့်ခ်များကို လည်း မမေ့ပဲ ကြည့်ရှုပါ။

## ဘာသာစကားမော်ဒယ်များအတွက် fine-tuning ဆိုတာဘာလဲ?

နိဒါန်းအရ၊ ကြီးမားသောဘာသာစကားမော်ဒယ်များကို အင်တာနက်မှ အမျိုးမျိုးသောအရင်းအမြစ်များမှ စကားလုံးများစွာဖြင့် _pre-trained_ လုပ်ထားသည်။ ယခင်သင်ခန်းစာများတွင် ကျွန်ုပ်တို့ ကြားဖူးသည့်အတိုင်း၊ _prompt engineering_ နှင့် _retrieval-augmented generation_ ကဲ့သို့နည်းလမ်းများသည် အသုံးပြုသူ၏မေးခွန်းများ ("prompt" များ) တွင် မော်ဒယ်၏ အကြောင်းပြန်မှုအရည်အသွေး တိုးတက်စေရန် လိုအပ်ပါသည်။

လူကြိုက်များသော prompt-engineering နည်းလမ်းမှာ မော်ဒယ်ကို တုံ့ပြန်ခံရမည့် အကြောင်းအရာများအတွက် _ညွှန်ကြားချက်များ_ (ရှင်းလင်းသောလမ်းညွှန်မှု) သို့မဟုတ် _ဥပမာအနည်းငယ်_ (အတိုကောက်လမ်းညွှန်မှု) ပေးခြင်းဖြင့် ပို၍ ညွှန်ကြားမှု ပြုခြင်း ဖြစ်သည်။ ဤနည်းလမ်းကို _few-shot learning_ ဟု ခေါ်ပြီး အကန့်အသတ် နှစ်မျိုး ရှိသည်-

- မော်ဒယ် token ကန့်သတ်ချက်များကြောင့် ပေးနိုင်သည့် ဥပမာအရေအတွက် လျှော့နည်းပြီး ထိရောက်မှု နည်းပါးနိုင်သည်။
- မော်ဒယ် token ကုန်ကျစရိတ်များကြောင့် နမူနာများကို အားလုံးသော prompt များတွင် ထည့်သွင်းရန် စျေးကြီးလာပြီး ပြောင်းလွယ်ပြင်လွယ်မှု လျော့နည်းနိုင်သည်။

Fine-tuning သည် ယခင်သင်ကြားထားသော မော်ဒယ်ကို အသစ်သော ဒေတာနှင့် ထပ်မံသင်ကြားခြင်းဖြင့် အထူးလုပ်ငန်းတစ်ခုခုတွင် စွမ်းဆောင်ရည်တိုးတက်စေရန် မော်ဒယ်ကို ဒေတာ အသစ်များဖြင့် ပြန်လည်သင်ကြားခြင်း ဖြစ်ပြီး၊ ဘာသာစကားမော်ဒယ်များ၏ ပတ်ဝန်းကျင်တွင် အတိအကျ လိုအပ်သော လုပ်ငန်း သို့မဟုတ် အက်ပလီကေးရှင်းဒိုမိန်းအတွက် ဥပမာများကို ရွေးချယ်ကာ ပြုလုပ်နိုင်သည်။  ၎င်းဖြင့် အတိအကျနှင့် သက်ဆိုင်မှု မြင့်မားသော **custom model** တစ်ခု ဖန်တီးနိုင်သည်။ Fine-tuning ၏ ထပ်တိုး အကျိုးကျေးဇူးမှာ few-shot learning အတွက် လိုအပ်သော ဥပမာများ အရေအတွက်လည်း လျော့နည်းစေနိုင်ပြီး token အသုံးပြုမှု နှင့် ကုန်ကျစရိတ်များကိုလည်း လျော့ချနိုင်သည်။

## ဘယ်အချိန်နှင့်ဘာကြောင့် မော်ဒယ်များကို fine-tune စေသင့်သလဲ?

ဒီအရာအရ fine-tuning ဆိုသည်မှာ မူလသင်ကြားမှု ဒေတာအသွင်အပြင် မပါဝင်သော ဒေတာအသစ်များဖြင့် **ပေါင်းထည့်၍** ထပ်မံသင်ကြားခြင်းဖြစ်သော **supervised** fine-tuning ဖြစ်သည်။ မူလဒေတာအပေါ် ဟိုက်ပါပါမားတွေ အသွင်ပြောင်း၍ ထပ်မံ သင်ကြားမှုလုပ်သော unsupervised fine-tuning နည်းလမ်းနှင့် ကွဲပြားသည်။

မေ့မလျော့ရန်မှာ fine-tuning သည် မိမိလိုချင်သည့်ရလဒ်ရရှိစေရန် ကျွမ်းကျင်မှု တစ်စိတ်တစ်ပိုင်း လိုအပ်သည့် နည်းစနစ်တစ်ခု ဖြစ်သည်။ မမှန်ကန်စွာ လုပ်ဆောင်ပါက စိတ်ချမှုပြသနာ ရှိ၍ မျှော်မှန်းသည့်တိုးတက်မှု မရရှိနိုင်သလို သတ်မှတ်ထားသော ဒိုမိန်းအတွက် မော်ဒယ်၏ စွမ်းဆောင်ရည် လျော့နည်းနိုင်ပါသည်။

ထို့ကြောင့် ဘာလို့ fine-tuning လုပ်သင့်သည့်အကြောင်းနှင့် ဘယ်အချိန်တွင် fine-tuning လုပ်မလဲဟု သင်သိရှိရန် မုန်းတရားမပေါ်ချိန် မီလေ့လာရန် လိုအပ်သည်။ မိတ်ဆွေအနေနဲ့ အောက်ပါမေးခွန်းများကို မေးကြည့်ပါ။

- **အသုံးပြုနိုင်မှု**: သင့် fine-tuning အတွက် _အသုံးပြုမှုအမျိုးအစား_ ဘာလဲ? လက်ရှိ သင်ကြားပြီးသား မော်ဒယ်၏ အချက်အလက် အပိုင်း ဘယ်တာကို တိုးတက်စေချင်ပါသလဲ?
- **အခြားနည်းလမ်းများ**: မိမိစိတ်တိုင်းကျ ရလဒ်ရရှိရန် အခြားနည်းလမ်းများကို ကြိုးစားတတ်ပြီးဖြစ်ပါသလား? ၎င်းအားများကို ဖြစ်စဉ်အခြေခံအဆင့် ချမှတ်ရန် အသုံးပြုပါ။
  - Prompt engineering: သက်ဆိုင်ရာ prompt များဖြင့် few-shot prompting နည်းလမ်းများကို စမ်းကြည့်ပါ။ တုံ့ပြန်မှု အရည်အသွေးကို သုံးသပ်ပါ။
  - Retrieval Augmented Generation: မိမိဒေတာမှ ရှာဖွေတွေ့ရှိထားသော query ရလဒ်များဖြင့် prompt များမြှင့်တင်ခြင်းကို စမ်းကြည့်ပါ။ တုံ့ပြန်မှု အရည်အသွေးကို သုံးသပ်ပါ။
- **ကုန်ကျစရိတ်များ**: fine-tuning အတွက် ကုန်ကျစရိတ်များကို သိရှိထားပါသလား?
  - Tunability - pre-trained မော်ဒယ်ကို fine-tuning လုပ်နိုင်ပါသလား?
  - ကြိုးပမ်းမှု - သင်ကြားမှု ဒေတာ ပြင်ဆင်ခြင်း၊ မော်ဒယ်မှာ အကဲဖြတ်ခြင်းနှင့် ပြုပြင်မွမ်းမံခြင်း။
  - ကွန်ပျူ့တာ - fine-tuning အလုပ်များ လုပ်ဆောင်ရေး နှင့် fine-tuned မော်ဒယ် ထုတ်လွှင့်ရေး။
  - ဒေတာ - fine-tuning အကျိုးသက်ရောက်မှုရှိစေရန် အရည်အသွေး ပြည့်စုံသော နမူနာများ ရရှိနိုင်ကူညီခြင်း။
- **အကျိုးအမြတ်များ**: fine-tuning ၏ အကျိုးကျေးဇူးများကို အတည်ပြုထားပါသလား?
  - အရည်အသွေး - fine-tuned မော်ဒယ်က ဘေ့(စ်)မော်ဒယ်ထက် ပို၍ ကောင်းမွန်သလား?
  - ကုန်ကျစရိတ် - prompt များလွယ်ကူစေရန် token အသုံးပြုမှု လျော့နည်းသလား?
  - တိုးချဲ့နိုင်မှု - ဗေ့(စ်)မော်ဒယ်ကို ဒိုမိန်းအသစ်များအတွက် အသုံးပြုနိုင်သလား?

ဤမေးခွန်းများကို ဖြေဆိုခြင်းဖြင့် fine-tuning သည် သင်၏ အသုံးပြုမှုအတွက် သင့်တော်သလားဟု ဆုံးဖြတ်နိုင်ပါသည်။ အကောင်းတစ်ခုမှာ ကုန်ကျစရိတ်ထက် သက်ဆိုင်သည့် အကျိုးများသာရှိပါက နည်းလမ်းသည် တရားဝင်ဖြစ်ပါသည်။ ဆက်လက်လုပ်ဆောင်ရန် ဆုံးဖြတ်ပါက အခုပဲ pre-trained မော်ဒယ်ကို ဘယ်လို fine-tune လုပ်မလဲဆိုတာ စဉ်းစားဖြစ်စေဖို့လိုပါသည်။

ဆုံးဖြတ်မှုလုပ်ငန်းစဉ်အပေါ် နက်ရှိုင်းသော သိရှိမှုများ ခံစားလိုပါသလား? [To fine-tune or not to fine-tune](https://www.youtube.com/watch?v=0Jo-z-MFxJs) ကို ကြည့်ရှုပါ။

## Pre-trained မော်ဒယ်တစ်ခုကို ဘယ်လို fine-tune လုပ်မလဲ?

ဖိုင်·တွေ fine-tune ဆွဲရန်လိုအပ်သည်-

- fine-tune လုပ်ရန် pre-trained မော်ဒယ်တစ်ခု
- fine-tune အတွက် အသုံးပြုမည့် ဒေတာစုစာရင်း တစ်ခု
- fine-tuning အလုပ် အဆောင်ရန် သင်ကြားရေး ပတ်ဝန်းကျင်
- fine-tuned မော်ဒယ် ထုတ်လွှင့်ရန် ဟိုစတင်ပတ်ဝန်းကျင်

## Microsoft Foundry တွင် Fine-Tuning လုပ်ခြင်း

[Microsoft Foundry](https://ai.azure.com?WT.mc_id=academic-105485-koreyst) သည် ယနေ့ Azure အပေါ် custom မော်ဒယ်များကို fine-tune, ထုတ်လွှင့်, စီမံခန့်ခွဲရာနေရာ ဖြစ်ပါသည်။ (အရင်က Azure OpenAI Studio နှင့် Azure AI Studio တို့ ကို ပေါင်းစပ်ထားသည်)။ အလုပ်စတင်မတိုင်ခင် Foundry သည်သင့်အား ရွေးချယ်စရာများနှင့် မော်ဒယ်ကို လွယ်ကူစွာ ထိရောက်စွာ fine-tune လုပ်နိုင်မည့် လမ်းညွှန်ချက်များကို နားလည်စေသည်။ ဖန်တီးမှုအတွက် Foundry က **LoRA (low-rank adaptation)** ကို အသုံးပြုကာ အလေးချိန်တိုင်းကို နောက်တစ်ဆင့် ပြန်လည်သင်ကြားခြင်းထက် လျင်မြန်ပြီး စျေးသက်သာစေပါသည်။

### ခြေလှမ်း ၁: သင်ကြားမှုနည်းလမ်း ရွေးချယ်ခြင်း

Foundry သည် fine-tuning နည်းလမ်းသုံးမျိုးကို ပံ့ပိုးသည်။ **SFT** သည် များစွာ အခြေအနေများအတွက် အသုံးပြုသည်။

| နည်းလမ်း | အလုပ်လုပ်ပုံ | ဘယ်အချိန်တွင် သုံးမလဲ |
| --- | --- | --- |
| **Supervised Fine-Tuning (SFT)** | input/output ဥပမာတွဲများပေါ်မှ သင်ကြားကာ အသင့်ရဲ့ တုံ့ပြန်ချက်များ ထုတ်ပေးရန် သင်ကြားသည်။ | များသော လုပ်ငန်းများတွင် default ဖြစ်သည်။ ဒိုမိန်းအထူးပြုခြင်း၊ စွမ်းဆောင်ရည်တိုးတက်မှု၊ စတိုင်နဲ့အသံ၊ လမ်းညွှန်ချက်လိုက်နာမှုနဲ့ဘာသာစကား ကူညီမှုများ။ |
| **Direct Preference Optimization (DPO)** | လူသား စိတ်ကြိုက်မှုနှင့် မစိတ်ကြိုက်မှု တုံ့ပြန်ချက်များမှ သင်ယူကာ ပြင်ဆင်ခြင်း။ | တုံ့ပြန်မှုအရည်အသွေး၊ လုံခြုံရေးနဲ့ လိုက်ဖက်မှု မြှင့်တင်ဖို့။ ယှဉ်ပြိုင်သောတုံ့ပြန်ချက်များ ရှိတဲ့အချိန်။ |
| **Reinforcement Fine-Tuning (RFT)** | _grader_ များမှ ဆုလာဘ်(signals) ကို အသုံးပြုကာ သင်ယူမှု အာရုံစိုက်ခြင်းဖြင့် အပြုအမူရှုပ်ထွေးမှုများကို အရမ်းတိုးတက်ဖို့ | ရည်ရွယ်ချက်ပြည့်သည့်၊ တွေးခေါ်မှုများလိုသော ဒိုမိန်းများ (သင်္ချာ၊ ဓာတုဗေဒ၊ ရုပ်ပိုင်းဗေဒ)။  ကြီးမားသော ML ကျွမ်းကျင်မှု လိုအပ်သည်။ |

### ခြေလှမ်း ၂: သင်ကြားမှု အဆင့်ရွေးချယ်ခြင်း

Foundry သည် သင်ကြားမှုကို မည်သို့ ၊ မည်နေရာတွင် ပြုလုပ်မလဲ ရွေးနိုင်စေသည်။

- **Standard** - သင့်အရင်းအမြစ်အရပ်တွင် သင်ကြားပြီး ဒေတာ နေရာယူမှု တည်ငြိမ်မှုကို အာမခံသည်။ ဒေတာ တစ်နေရာတည်းတွင်ထားရန်လိုသောအချိန် သုံးပါ။
- **Global** - ပိုသက်သာပြီး မြန်ဆန်စေရန် နိုင်ငံခြားအရပ်မှ ခြွင်းချက်တက်မြောက်မှုကို အသုံးပြုသည် (ဒေတာနှင့် အလေးချိန်များ သင်ကြားနေရာသို့ မိမိကူးယူထားသည်)။ ဒေတာ နေရာယူမှု မလိုအပ်သည့် အခြေအနေတွင် သုံးရန် ကောင်းသည်။
- **Developer** - အနည်းဆုံးကုန်ကျစရိတ်၊ စောင့်ကြည့်မှု (latency)/ SLA အာမခံချက်မပါ၊ အလုပ်များ မတည်ငြိမ်မှုရှိနိုင် (လုပ်အားများ ရပ်နားပြီး ပြန်စတင်နိုင်သည်)။ စမ်းသပ်လိုသူများအတွက် သင့်တော်သည်။

### ခြေလှမ်း ၃: အခြေခံမော်ဒယ်ကို ရွေးချယ်ခြင်း

Fine-tuning လုပ်နိုင်သော မော်ဒယ်များမှာ OpenAI `gpt-4o-mini`, `gpt-4o`, `gpt-4.1`, `gpt-4.1-mini`, `gpt-4.1-nano` (SFT; 4o/4.1 မျိုးဆက်မှာ DPO ကိုလည်း ပံ့ပိုးသည်), reasoning မော်ဒယ်များ `o4-mini` နှင့် `gpt-5` (RFT), ရုပ်သုံး အခမဲ့မော်ဒယ်များ အဖြစ် `Ministral-3B`, `Qwen-32B`, `Llama-3.3-70B-Instruct`, `gpt-oss-20b` တို့ ပါဝင်သည် (Foundry အရင်းအမြစ်ပေါ် SFT ဖြစ်သည်)။ ဆက်လက်အသုံးပြုရန် ပြည့်စုံသည့် နည်းလမ်းများ၊ ဒေသများနှင့် ရရှိနိုင်မှုကို ‌နောက်ဆုံး [Fine-tuning models list](https://learn.microsoft.com/azure/ai-foundry/foundry-models/concepts/models-sold-directly-by-azure?WT.mc_id=academic-105485-koreyst#fine-tuning-models) မှ စစ်ဆေးပါ။

> Foundry သည် နှစ်မျိုး modality ရှိသည်- **serverless** (စားသုံးမှုအပေါ်အခြေခံသတ်မှတ်စျေးနှုန်း, GPU quota မစိုးရိမ်, OpenAI နှင့် ရွေးချယ်ထားသော မော်ဒယ်များ) နှင့် **managed compute** (VMs ကို Azure Machine Learning မှ တာဝန်ယူသည့် server ပေါ်တွင် run တာ) ဖြစ်ပြီး၊ မော်ဒယ် အမျိုးအစား၍ ပိုမိုကျယ်ပြန့်သည်။ လူအများစုအတွက် serverless ဖြင့် စထည့်ရန် ပြောကြားသည်။

### Foundry ၏ အကောင်းဆံုး လုပ်ထုံးလုပ်နည်းများ

- **အခြေခံကို ရှိစေပါ။** အရင်းအမြစ် မော်ဒယ်ကို prompt engineering နဲ့ RAG နည်းလမ်းများ ဖြင့် တုံ့ပြန်မှုအရည်အသွေးကို သုံးသပ်ပြီး မိမိတိုးတက်မှုကို သက်သေပြပါ။
- **စတင်သိပ်စိုက်၊ ထို့နောက် တိုးချဲ့သည်။** ၅၀-၁၀၀ ရွေးချယ်ထားသော အရည်အသွေးမြင့် ဥပမာများဖြင့် လုပ်ငန်းစဉ်ကို အတည်ပြုပါ၊ ထို့နောက် ထုတ်လုပ်မှုအတွက် ၅၀၀ ကျော်သို့ တိုးပါ။ အရည်အသွေးသည် အရေအတွက်ထက် ထင်ရှားပါသည် - အရည်အသွေးနည်း ဥပမာများကို ဖယ်ရှားပါ။
- **ဒေတာအစီအစဉ်ကို မှန်ကန်စွာ ဖော်မြူလာ ပြုလုပ်ပါ။** သင်ကြားမှုနှင့် အတည်ပြုခြင်း ဖိုင်များသည် JSONL, UTF-8 **BOM နှင့်**, ၅၁၂ MB အောက် ဖြစ်ရမည်၊ chat-completions မက်ဆေ့ခ်ျ format ဖြင့် ဖြစ်ရမည်။ နောက်ထပ် overfitting ကို ကြည့်နိုင်ရန် validation ဖိုင် ပါဝင်ရန်လိုသည်။
- **training system prompt ကို inference တွင် သုံးပါ။** သင်ကြားသည့် အချိန်တွင် သုံးသော system message နှင့် အတူ မော်ဒယ်ကို ခေါ်ယူပါ။
- **checkpoint များကို ရိုက်ခတ်စဉ်းစားပါ - နောက်ဆုံး checkpoint ကို အလွယ်တကူ မdeploy လုပ်ပါနှင့်။** Foundry သည် နောက်ဆုံး epoch သုံးခုကို deploy မဖြစ်သေးသည့် checkpoint အဖြစ်သိမ်းဆည်းထားသည်။ `train_loss` / `valid_loss` နှင့် token တိကျမှုပြုမှုကို ကြည့်ကာ အကောင်းဆုံး ကောက်ချသည်။
- **token ကုန်ကျစရိတ် နှင့် အရည်အသွေးကို တပြိုင်နက်တိုင်းတိုင်း ဆန့်ဖက်တိုင်းတာပါ။** fine-tuned မော်ဒယ်နှင့် baseline ကို နှိုင်းယှဉ်စဉ်။
- **အဆက်မပြတ် fine-tuning ဖြင့် ပြုပြင်ကောင်းမွန်အောင် စတင်ပါ။** သင် ကြိုတင် fine-tuned မော်ဒယ်ကို ဒေတာအသစ်နဲ့ ထပ်မံ fine-tune လုပ်နိုင်သည်။ (OpenAI မော်ဒယ်များအတွက် ပံ့ပိုးသည်)
- **hosting ကုန်ကျစရိတ်ကို သတိပြုပါ။** ထုတ်လွှင့်ထားသော custom မော်ဒယ်သည် နာရီမီတာဖြင့် ကုတ်ချက် ပေးပြီး၊ အသုံးမပြုတော့သော deployment ကို ၁၅ ရက်အတွင်း ဖယ်ရှားပစ်လိုက်သည့်အတွက် မလိုအပ်သည်များကို ရှင်းပါ။

[Customize a model with fine-tuning](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning?WT.mc_id=academic-105485-koreyst) တွင် စတင်ပြီး ပြီးဆုံးသည်အထိ လမ်းညွှန်ထားသော walkthrough ကို လေ့လာပြီး၊ [DPO](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning-direct-preference-optimization?WT.mc_id=academic-105485-koreyst) နှင့် [RFT](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/reinforcement-fine-tuning?WT.mc_id=academic-105485-koreyst) နည်းလမ်းများကို အသုံးပြုရန် ပြင်ဆင်သည်အထိ ကြည့်ပါ။
