# သင့် LLM ကို ကြည့်ရှု၍ ပြင်ဆင်ခြင်း

ကြီးမားသောဘာသာစကားမော်ဒယ်များကို အသုံးပြု၍ ဖန်တီးမှု AI ဆိုင်ရာ အက်ပလီကေးရှင်းများ တည်ဆောက်ရာတွင် ကိစ္စအသစ်များ ရှိလာသည်။ အဓိကပြဿနာတစ်ခုမှာ မော်ဒယ်မှ ထုတ်ပေးသည့် အကြောင်းအရာ၏ တုံ့ပြန်မှုအရည်အသွေး (တိကျမှုနှင့် သက်ဆိုင်မှု) ကို သေချာစေရန်ဖြစ်သည်။ ယခင် သင်ခန်းစာများတွင် နည်းလမ်းများဖြစ်သည့် prompt engineering နှင့် retrieval-augmented generation တို့ အသုံးပြု၍ ရှိပြီးသား မော်ဒယ်အား _prompt input ကို ပြင်ဆင်ခြင်း_ ဖြင့် ပြဿနာကို ဖြေရှင်းရန် ကြိုးစားခဲ့သည်။

ယနေ့သင်ခန်းစာတွင် ငါးတစ်နည်းဖြစ်သည့် **fine-tuning** နည်းလမ်းကို ဆွေးနွေးမည်ဖြစ်ပြီး ယင်းသည် အပိုဒေတာများနှင့် _မော်ဒယ်ကို အထဲတွင်တန်းသင်ခန်းပြန်သင်ခြင်း_ ဖြင့် ပြဿနာကို ဖြေရှင်းခြင်းကို ရည်ရွယ်သည်။ အသေးစိတ် ထောက်လှမ်းကြည့်ကြပါစို့။

## သင်ယူရမည့်အချက်များ

ဒီသင်ခန်းစာအတွင်းမူ၊ ကြိုတင်လေ့လာပြီးသားဘာသာစကားမော်ဒယ်များအတွက် fine-tuning ၏အကြောင်းအရာကိုမိတ်ဆက်ပေးပြီး၊ ဤနည်းလမ်း၏ အကျိုးကျေးဇူးများ နှင့် စိန်ခေါ်မှုများကို ရှာဖွေကြည့်ပါမည်။ ထို့အတူ သင့် generative AI မော်ဒယ်များ၏ လုပ်ဆောင်ချက်များ တိုးတက်စေရန် fine tuning ကို ဘယ်အချိန်နှင့် မည်သို့အသုံးပြုရမလဲ ဆိုတာ အကြံပြုချက်များကို ဖော်ပြပါမည်။

ဒီသင်ခန်းစာပြီးဆုံးစဉ်မှာ၊ အောက်ပါမေးခွန်းများကို ဖြေကြားနိုင်ဖို့ဖြစ်ပါတယ်-

- ဘာသာစကားမော်ဒယ်များအတွက် fine tuning ဆိုတာ ဘာလဲ?
- ဘယ်အချိန်မှာ၊ ဘာကြောင့် fine tuning သုံးသင့်တာလဲ?
- ကြိုတင်လေ့လာပြီးသား မော်ဒယ်ကို မည်သို့ fine-tune လုပ်မလဲ?
- fine-tuning ၏ ကန့်သတ်ချက်များက ဘာများလဲ?

ပြင်ဆင်ပြီးပြီလား? စတင်ကြစို့။

## ပုံပေါ်လမ်းညွှန်

အစပြုခါစမှာ ဘာတွေကြုံတွေ့ရမလဲဆိုတာ အဓိကအချက်များကို ချဉ်းကပ်ရန်လိုချင်ပါသလား? ဒီပုံဖော်လမ်းညွှန်မှာ သင့်ကို fine-tuning ၏ အခြေခံအကြောင်းအရာနှင့် ရည်ရွယ်ချက်များကိုကာ ပိုမိုနက်နဲစွာ သဘောတရားနားလည်ရန်၊ ပြီးတော့ fine-tuning အလုပ်ကို မည်သို့ဆောင်ရွက်ရမလဲဆိုတာ အကောင်းဆုံးအလုပ်လုပ်နည်းများနဲ့ရှင်းပြထားသည်။ အမြဲတမ်း စူးစမ်းသိရှိရန်တွင်စိတ်ဝင်စားစရာပြည့်ဝ၍၊ သင်တို့ကိုယ်တိုင် လေ့လာလိုသည့် လမ်းကြောင်းပညာရပ်များအတွက် [Resources](./RESOURCES.md?WT.mc_id=academic-105485-koreyst) စာမျက်နှာကိုလည်း မမေ့ပဲ ကြည့်ရှုပါ။

## ဘာသာစကားမော်ဒယ်များအတွက် fine-tuning ဆိုတာ ဘာလဲ?

အဓိပ္ပါယ်အရ၊ ကြီးမားသောဘာသာစကားမော်ဒယ်များသည် အင်တာနက်ရှိ မတူကွဲပြားသောအရင်းအမြစ်များမှ စုဆောင်းထားသည့် စာသားများအပေါ် _ကြိုတင်လေ့လာ_ သွားပြီးဖြစ်ကြသည်။ ယခင်သင်ခန်းစာများတွင် သင်ယူခဲ့သည့် အတိုင်း၊ prompt engineering နှင့် retrieval-augmented generation ကဲ့သို့နည်းလမ်းများကိုအသုံးပြု၍ မော်ဒယ်၏ တုံ့ပြန်မှုအရည်အသွေးကို မြှင့်တင်ဖို့ လိုအပ်သည်။

လူကြိုက်များသော prompt engineering နည်းလမ်းတစ်ခုမှာ မော်ဒယ်ကို သတ်မှတ်ထားသော တုံ့ပြန်မှုတွင် ဘာမျှော်လင့်မလဲဆိုတာအကြောင်း _ညွှန်ကြားချက်များ_ (တိကျစွာ လမ်းညွှန်ခြင်း) သို့မဟုတ် _နမူနာအနည်းငယ်_ (အနည်းငယ် လမ်းညွှန်ခြင်း) ပေးခြင်းဖြင့် ပိုမိုညွှန်ကြားရန်ဖြစ်သည်။ ဤနည်းလမ်းကို _few-shot learning_ ဟု ခေါ်ကြပြီး အောက်ပါကန့်သတ်ချက် နှစ်ခုရှိသည်-

- မော်ဒယ်တို့တွင် token ကန့်သတ်ချက်ကြောင့် ပေးနိုင်သည့် နမူနာအရေအတွက်ကို ကန့်သတ်ပြီး ထိရောက်မှုကိုလည်း ကန့်သတ်ထားသည်။
- မော်ဒယ် token စျေးပေါမဟုတ်ပါက နမူနာများအား မည်သည့် prompt မှာမဆို ထည့်သွင်းခြင်းသည် စျေးကြီးပြီး လွယ်ကူမှုကို လျော့နည်းစေသည်။

Fine-tuning သည် အများဆုံး အသုံးဝင်သော စက်မောင်းထဲမှာ ကြိုတင်လေ့လာပြီးသား မော်ဒယ်တစ်ခုကို အသစ်ပြန်လေ့လာသင်ကြားရန် အသုံးပြုခြင်းဖြစ်ပြီး သတ်မှတ်ထားသည့် တာဝန်တွင် ပိုမိုကောင်းမွန်စွာ ဆောင်ရွက်နိုင်စေရန် ရည်ရွယ်သည်။ ဘာသာစကားမော်ဒယ်များသမိုင်းတွင် ပြောပါက၊ ပေးထားသော တာဝန် သို့မဟုတ် လျှောက်လွှာနယ်ပယ်အတွက် စီစစ်ဖျော်ဖြေထားသော နမူနာများအစုအဝေးဖြင့် ကြိုတင်လေ့လာပြီးသား မော်ဒယ်ကို ရှေးရှု fine-tune ပြုလုပ်၍ အတိအကျနဲ့ သက်ဆိုင်မှု ကောင်းမွန်စေရန် **သီးသန့်မော်ဒယ်** တစ်ခု ဖန်တီးနိုင်သည်။ fine-tuning ၏ ဘေးကျိုးတစ်ခုမှာ few-shot learning အတွက် လိုအပ်သော နမူနာအရေအတွက်ကိုလည်း လျော့နည်းစေနိုင်ခြင်းဖြစ်ပြီး token အသုံးပြုမှုနဲ့ သက်ဆိုင်မှုများကို လျော့ပါးစေသည်။

## ဘယ်အချိန်မှာနှင့် ဘာကြောင့် မော်ဒယ်များကို fine-tune လုပ်သင့်တာလဲ?

ဒီအခြေအနေအတွင်း၊ fine-tuning မှူး ကြောင့် က\ဲြပင်ပုံပြင်အသစ်ဖြင့် နှစ်ခြိုက် fine-tuning ကို ဆိုလိုသည်။ ဒါကတော့ ကြိုတင်လေ့လာသင်ယူမှု ဒေတာအစုတွင် မပါသော ဒေတာအသစ်တွေ ထည့်သွင်း၍ ပြန်လေ့လုပ်ခြင်း ဖြစ်သည်။ ဤနည်းလမ်းသည် original ဒေတာပေါ်တွင် hyperparameter မတူညီဘဲ ပြန်လေ့လာသင်ကြားခြင်းဖြင့် ပြုလုပ်သော unsupervised fine-tuning မှကွဲပြားသည်။

မှတ်သားရန် အဓိကအချက်မှာ fine-tuning သည် လုပ်နိုင်မှုအဆင့်မြင့် နည်းလမ်းတစ်ခုဖြစ်ပြီး ရရှိလိုသောရလဒ်အတွက် ကျွမ်းကျင်မှုတစ်မျိုးလိုအပ်သည်။ မှားယွင်းစွာ ကာစပ်လျှင် မမျှော်လင့်ထားသည့် တိုးတက်မှု မရနိုင်ဘဲ မော်ဒယ်၏ သင့်ရည်ရွယ်သည့်နယ်ပယ်အတွင်း လုပ်ဆောင်ချက်ပျက်ကွက်စေနိုင်သည်။

ဒါကြောင့် ဘာသာစကားမော်ဒယ်များကို fine-tune လုပ်နည်းကို သင်ယူမည့်အချိန်မှာ "ဘာကြောင့်" ဒီလမ်းကို သွားသင့်တာလဲဆိုတာနှင့် fine-tuning လုပ်စဉ်ကို ဘယ်အချိန်မှာ စတင်သင့်တာလဲကို သိဖို့လိုသည်။ အောက်ပါ မေးခွန်းများကို ကိုယ်တိုင်မေးကြည့်ပါ-

- **အသုံးပြုမှုအကွာအဝေး**: fine-tuning အတွက် သင့် _အသုံးပြုမှု_ ဟာ ဘာလဲ? ကြိုတင်လေ့လာထားသော မော်ဒယ်အတွက် ဘယ်လိုပိုမြင့်မားစေချင်သလဲ?
- **အစားထိုး နည်းလမ်းများ**: လိုသည့်ရလဒ်ကိုရဖို့အတွက် _နည်းလမ်းအခြားများ_ ကို ကြိုးစားလေ့လာခဲ့ပြီးသလား? ဒါကိုလည်း အခြေခံအနည်းငယ် အဖြစ် အသုံးပြုပါ။
  - Prompt engineering: သက်ဆိုင်ရာ prompt တုံ့ပြန်မှု နမူနာများနဲ့ few-shot prompting လို နည်းလမ်းတွေ ကိုကြိုးစားကြည့်ပါ။ တုံ့ပြန်မှု အရည်အသွေးကို ချဉ်းကပ်သုံးသပ်ပါ။
  - Retrieval Augmented Generation: သင့်ဒေတာကို ရှာဖွေပြီး ရရှိသော မေးခွန်း ရလဒ်များ ဖြင့် prompt များ ရှင်းလင်းတိုးတက်စေမှု ပြုလုပ်ကြည့်ပါ။ တုံ့ပြန်မှုအရည်အသွေးကို ချဉ်းကပ်သုံးသပ်ပါ။
- **ကုန်ကျစရိတ်များ**: fine-tuning လုပ်ရန်ကုန်ကျစရိတ်များကို တွက်ချက်ဖူးပြီလား?
  - Tunability - ကြိုတင်လေ့လာထားသော မော်ဒယ်ကို fine-tune လုပ်နိုင်ပါသလား?
  - ကြိုးစားမှု - လေ့လာသင်ကြားမှု ဒေတာ ပြင်ဆင်ခြင်း၊ မော်ဒယ်သုံးသပ်ခြင်းနှင့် ပြုပြင်မှုပြုလုပ်ခြင်းအတွက် ကြိုးပမ်းမှု။
  - ကွန်ပျူတာစွမ်းဆောင်ရည် - fine-tuning အလုပ်များ လုပ်ဆောင်ရန်နှင့် fine-tune လုပ်ပြီး မော်ဒယ်တင်ရန် မရှိမဖြစ်လိုအပ်သည်။
  - ဒေတာ - fine-tuning အတွက် လုံလောက်သော အရည်အသွေး နမူနာများ ရရှိနိုင်မှု။
- **အကျိုးကျေးဇူးများ**: fine-tuning ၏ အကျိုးကျေးဇူးများ ဘာတွေလဲ?
  - အရည်အသွေး - fine-tuned မော်ဒယ်သည် အခြေခံ မော်ဒယ်ထက် ပိုမိုကောင်းမွန်ခဲ့သလား?
  - ကုန်ကျစရိတ် - prompt များ လွယ်ကူစေရေး ဖြင့် token အသုံးပြုမှု လျော့ကာ ကျစရိတ် လျော့စေသလား?
  - တိုးချဲ့နိုင်မှု - မူရင်းမော်ဒယ်ကို နောက်ထပ် နယ်ပယ်အသစ်များအတွက် သုံးနိုင်သလား?

အဆိုပါမေးခွန်းများကို ဖြေဆိုရင် fine-tuning သည် သင့် အသုံးပြုမှုအတွက် မှန်ကန်တဲ့နည်းလမ်းဖြစ်မဖြစ် ဆုံးဖြတ်နိုင်သည်။ အကောင်းဆုံးမှာ အကျိုးအမြတ်များသည် ကုန်ကျစရိတ်များထက် များပေါ်လွင်သောအခါသာ လက်ခံရယူသင့်သည်။ ဆုံးဖြတ်ပြီးပါက ကြိုတင်လေ့လာထားသော မော်ဒယ်ကို fine-tune လုပ်မည့် နည်းလမ်းများကို တွေးခေါ်ရန် အချိန်ရောက်ပါပြီ။

ဆုံးဖြတ်မှုလုပ်ဆောင်ရာတွင် ပိုမိုသိရှိလိုသလား? [To fine-tune or not to fine-tune](https://www.youtube.com/watch?v=0Jo-z-MFxJs) ကို ကြည့်ရှုပါ။

## ကြိုတင်လေ့လာပြီးသား မော်ဒယ်ကို မည်သို့ fine-tune လုပ်မလဲ?

ကြိုတင်လေ့လာထားသော မော်ဒယ်ကို fine-tune လုပ်ရန် လိုအပ်သည်-

- fine-tune လုပ်ရန် ကြိုတင်လေ့လာထားသော မော်ဒယ်တစ်ခု
- fine-tuning အတွက် အသုံးပြုမည့် ဒေတာစု
- fine-tuning အလုပ်ကို စိတ်ကြိုက် ပြုလုပ်နိုင်မည့် လေ့လာသင်ကြားမှုပတ်ဝန်းကျင်
- fine-tune လုပ်ပြီး မော်ဒယ်အသစ်ကို တင်သွင်းရန် ပတ်ဝန်းကျင်

## လက်တွေ့ Fine-Tuning လုပ်ငန်း

> **အမှတ်တရ**: အောက်ပါ သင်ခန်းစာအချို့တွင် ကိုးကားထားသည့် `gpt-35-turbo` / `gpt-3.5-turbo` မော်ဒယ်များသည် မကြာသေးဘဲ တိုးတက်မှု မရရှိတော့သောကြောင့် inference နှင့် fine-tuning နှစ်ခုစလုံး မထောက်ပံ့တော့ပါ။ ယနေ့ fine-tuning အသစ်တစ်ခု စတင်ရန် ဥပမာအားဖြင့် `gpt-4o-mini` သို့မဟုတ် `gpt-4.1-mini` ကဲ့သို့ လက်ရှိထောက်ပံ့ထားသော မော်ဒယ်များကို ရွေးချယ်သင့်ပါသည်။ လက်ရှိ fine-tuning လုပ်နိုင်သော မော်ဒယ်စာရင်းများအတွက် [Fine-tuning models list](https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-models/concepts/models-sold-directly-by-azure?WT.mc_id=academic-105485-koreyst#fine-tuning-models) ကို ကြည့်ပါ။ ဤသင်ခန်းစာများတွင် ပါဝင်သော အကြောင်းအရာများနှင့် လုပ်ငန်းစဉ်များမှာ မူလအတိုင်း သက်ဆိုင်ပါသည်။

အောက်ပါ ရင်းမြစ်များတွင် မော်ဒယ်တစ်ခုနှင့် စီစစ်ဖျော်ဖြေရေး ဒေတာစုတစ်ခုကို အသုံးပြုပြီး လုပ်ငန်းစဥ်ဆက်တိုက် လေ့လာနိုင်သည့် လမ်းညွှန်ချက်များ ပါဝင်သည်။ ဤသင်ခန်းစာများကို လေ့လာရာတွင် သင့်တွင် ထူးခြားညွှန်ကြားသူ ရရှိသော ပံ့ပိုးပေးသူအားလုံးအကောင့်နှင့် သက်ဆိုင်သော မော်ဒယ်များ၊ ဒေတာအစုံများသို့ ဝင်ရောက်နိုင်မှုရှိရမည်။
