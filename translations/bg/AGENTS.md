# AGENTS.md

## Преглед на проекта

Това хранилище съдържа изчерпателна учебна програма от 21 урока, която преподава основите на Генеративния AI и разработката на приложения. Курсът е предназначен за начинаещи и обхваща всичко от основните концепции до създаването на приложения, готови за производство.

**Основни технологии:**
- Python 3.9+ с библиотеки: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript с Node.js и библиотеки: `openai` (Azure OpenAI чрез v1 endpoint + Responses API), `@azure-rest/ai-inference` (Microsoft Foundry Models)
- Azure OpenAI Service, OpenAI API и Microsoft Foundry Models (GitHub Models се закрива в края на юли 2026)
- Jupyter Notebooks за интерактивно учене
- Dev Containers за постоянна среда за разработка

**Структура на хранилището:**
- 21 директории с номерирани уроци (00-21), съдържащи README файлове, примери за код и задания
- Множество реализации: Python, TypeScript и понякога .NET примери
- Директория за преводи с 40+ езикови версии
- Централизирана конфигурация чрез `.env` файл (използвайте `.env.copy` като шаблон)

## Команди за настройка

### Първоначална настройка на хранилището

```bash
# Клонирайте хранилището
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# Копирайте шаблона за околната среда
cp .env.copy .env
# Редактирайте .env с вашите API ключове и крайни точки
```

### Настройка на Python среда

```bash
# Създаване на виртуална среда
python3 -m venv venv

# Активиране на виртуалната среда
# На macOS/Linux:
source venv/bin/activate
# На Windows:
venv\Scripts\activate

# Инсталиране на зависимости
pip install -r requirements.txt
```

### Настройка на Node.js/TypeScript

```bash
# Инсталирайте зависимости на root ниво (за инструменти за документация)
npm install

# За отделни примери на TypeScript от уроци, отидете до конкретния урок:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Настройка на Dev Container (препоръчително)

Хранилището включва конфигурация `.devcontainer` за GitHub Codespaces или VS Code Dev Containers:

1. Отворете хранилището в GitHub Codespaces или VS Code с разширението Dev Containers
2. Dev Container автоматично ще:
   - Инсталира Python зависимости от `requirements.txt`
   - Изпълни скрипт след създаване (`.devcontainer/post-create.sh`)
   - Настрои Jupyter kernel

## Процес на разработка

### Променливи на средата

Всички уроци, изискващи достъп до API, използват променливи на средата, дефинирани в `.env`:

- `OPENAI_API_KEY` - За OpenAI API
- `AZURE_OPENAI_API_KEY` - За Azure OpenAI в Microsoft Foundry (Azure OpenAI Service вече е част от Microsoft Foundry: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - URL на Azure OpenAI endpoint (Foundry resource endpoint)
- `AZURE_OPENAI_DEPLOYMENT` - Име на разгръщане на Chat completion модел
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - Име на разгръщане на модел за embeddings
- `AZURE_OPENAI_API_VERSION` - Версия на API (по подразбиране: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - За Hugging Face модели
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry Models endpoint (каталог с модели от множество доставчици)
- `AZURE_INFERENCE_CREDENTIAL` - API ключ за Microsoft Foundry Models (замества постепенно закриващия се `GITHUB_TOKEN`)

### Стартиране на Python примери

```bash
# Отидете до директорията на урока
cd 06-text-generation-apps/python

# Стартирайте Python скрипт
python aoai-app.py
```

### Стартиране на TypeScript примери

```bash
# Навигирайте до директорията на TypeScript приложението
cd 06-text-generation-apps/typescript/recipe-app

# Компилирайте TypeScript кода
npm run build

# Стартирайте приложението
npm start
```

### Стартиране на Jupyter Notebooks

```bash
# Стартирайте Jupyter в корена на хранилището
jupyter notebook

# Или използвайте VS Code с разширението Jupyter
```

### Работа с различни типове уроци

- **Уроци "Learn"**: Фокус върху документацията в README.md и концепциите
- **Уроци "Build"**: Включват работещи кодови примери на Python и TypeScript
- Всеки урок има README.md с теория, обяснения на кода и линкове към видео съдържание

## Насоки за стил на код

### Python

- Използвайте `python-dotenv` за управление на променливите на средата
- Импортирайте библиотеката `openai` за взаимодействия с API
- Използвайте `pylint` за проверка на код (някои примери включват `# pylint: disable=all` за опростяване)
- Следвайте именни конвенции според PEP 8
- Съхранявайте API данни за достъп в `.env` файл, никога не директно в кода

### TypeScript

- Използвайте пакета `dotenv` за променливи на средата
- Конфигурация на TypeScript в `tsconfig.json` за всяко приложение
- Използвайте пакета `openai` за Azure OpenAI (насочвайте клиента към `/openai/v1/` endpoint и извиквайте `client.responses.create`); използвайте `@azure-rest/ai-inference` за Microsoft Foundry Models
- Използвайте `nodemon` за разработка с автоматично презареждане
- Компилирайте преди стартиране: `npm run build` след това `npm start`

### Общи конвенции

- Дръжте кодовите примери прости и образователни
- Включвайте коментари, обясняващи ключови концепции
- Кодът на всеки урок трябва да е самостоятелен и изпълним
- Използвайте последователно именуване: префикс `aoai-` за Azure OpenAI, `oai-` за OpenAI API, `githubmodels-` за Microsoft Foundry Models (запазен префикс от епохата на GitHub Models)

## Тестване и валидация

### Проверки преди подаване

Това хранилище използва GitHub Actions за валидация. Преди да подадете PR:

1. **Проверете Markdown линковете**:
   ```bash
   # Работният процес validate-markdown.yml проверява:
   # - Счупени относителни пътища
   # - Липсващи ID-та за проследяване в пътищата
   # - Липсващи ID-та за проследяване в URL адресите
   # - URL адреси с локализация по държава
   # - Счупени външни URL адреси
   ```

2. **Ръчно тестване**:
   - Тествайте Python примерите: Активирайте venv и пуснете скриптовете
   - Тествайте TypeScript примерите: `npm install`, `npm run build`, `npm start`
   - Проверете дали променливите на средата са конфигурирани правилно
   - Уверете се, че API ключовете работят с кодовите примери

3. **Кодови примери**:
   - Уверете се, че всички кодови примери се изпълняват без грешки
   - Тествайте както с Azure OpenAI, така и с OpenAI API, когато е приложимо
   - Проверете дали примерите работят с Microsoft Foundry Models, където се поддържат

### Без автоматизирани тестове

Това е образователно хранилище, фокусирано върху уроци и примери. Няма да намерите юнит тестове или интеграционни тестове за изпълнение. Валидацията е основно:
- Ръчно тестване на кодовите примери
- GitHub Actions за валидация на Markdown
- Преглед от общността на образователното съдържание

## Отстраняване на проблеми

### Чести проблеми

**Грешки при импорт в Python**:
- Уверете се, че виртуалната среда е активирана
- Изпълнете `pip install -r requirements.txt`
- Проверете дали версията на Python е 3.9+

**Грешки при компилация на TypeScript**:
- Изпълнете `npm install` в конкретната директория на приложението
- Проверете дали версията на Node.js е съвместима
- Изчистете `node_modules` и преинсталирайте при нужда

**Грешки при автентикация към API**:
- Уверете се, че файлът `.env` съществува и съдържа правилните стойности
- Проверете дали API ключовете са валидни и не са изтекли
- Уверете се, че URL адресите на endpoint са правилни за вашия регион

**Липсващи променливи на средата**:
- Копирайте `.env.copy` като `.env`
- Попълнете всички необходими стойности за урока, върху който работите
- Рестартирайте приложението след актуализация на `.env`
