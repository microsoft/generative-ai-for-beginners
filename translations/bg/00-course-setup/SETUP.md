# Настройте вашата среда за разработка

Ние настроихме това хранилище и курс с [контейнер за разработка](https://containers.dev?WT.mc_id=academic-105485-koreyst), който има универсално изпълнение, поддържащо разработка с Python3, .NET, Node.js и Java. Свързаната конфигурация е дефинирана във файла `devcontainer.json`, намиращ се в папката `.devcontainer/` в корена на това хранилище.

За да активирате контейнера за разработка, стартирайте го в [GitHub Codespaces](https://docs.github.com/en/codespaces/overview?WT.mc_id=academic-105485-koreyst) (за изпълнение в облака) или в [Docker Desktop](https://docs.docker.com/desktop/?WT.mc_id=academic-105485-koreyst) (за локално изпълнение на устройството). Прочетете [тази документация](https://code.visualstudio.com/docs/devcontainers/containers?WT.mc_id=academic-105485-koreyst) за повече подробности относно това как работят контейнерите за разработка във VS Code.

> [!TIP]  
> Препоръчваме да използвате GitHub Codespaces за бърз старт с минимални усилия. Той предоставя щедра [безплатна квота за използване](https://docs.github.com/billing/managing-billing-for-github-codespaces/about-billing-for-github-codespaces#monthly-included-storage-and-core-hours-for-personal-accounts?WT.mc_id=academic-105485-koreyst) за лични акаунти. Конфигурирайте [таймаути](https://docs.github.com/codespaces/setting-your-user-preferences/setting-your-timeout-period-for-github-codespaces?WT.mc_id=academic-105485-koreyst) за спиране или изтриване на неактивни кодови пространства, за да максимизирате използването на вашата квота.

## 1. Изпълнение на задачи

Всяка лекция ще има _по желание_ задачи, които могат да бъдат предоставени на един или повече програмни езици, включително: Python, .NET/C#, Java и JavaScript/TypeScript. Този раздел предоставя общи насоки относно изпълнението на тези задачи.

### 1.1 Задачи на Python

Задачите на Python са предоставени или като приложения (файлове `.py`) или като Jupyter тетрадки (файлове `.ipynb`).
- За да стартирате тетрадката, отворете я във Visual Studio Code, след това кликнете _Изберете ядрото_ (горе вдясно) и изберете показаната по подразбиране опция Python 3. Сега можете да _изпълните всички_, за да стартирате тетрадката.
- За да стартирате Python приложения от командния ред, следвайте специфичните инструкции за задачите, за да сте сигурни, че сте избрали правилните файлове и сте предоставили необходимите аргументи.

### 2.1. Създайте файл `.env`

Предполагаме, че вече сте прочели горните насоки и сте се регистрирали с релевантния доставчик и сте получили необходимите идентификационни данни за удостоверяване (API_KEY или токен). В случая на Azure OpenAI, предполагаме, че имате и валидно разгръщане на услуга Azure OpenAI (крайна точка) с поне един GPT модел, разположен за завършване на чат.

Следващата стъпка е да конфигурирате вашите **локални променливи на средата** по следния начин:

1. Потърсете в основната папка файл `.env.copy`, който трябва да съдържа следното:

   ```bash
   # OpenAI Provider
   OPENAI_API_KEY='<add your OpenAI API key here>'

   ## Azure OpenAI
   AZURE_OPENAI_API_VERSION='2024-02-01' # Default is set!
   AZURE_OPENAI_API_KEY='<add your AOAI key here>'
   AZURE_OPENAI_ENDPOINT='<add your AOIA service endpoint here>'
   AZURE_OPENAI_DEPLOYMENT='<add your chat completion model name here>' 
   AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT='<add your embeddings model name here>'

   ## Hugging Face
   HUGGING_FACE_API_KEY='<add your HuggingFace API or token here>'
   ```

2. Копирайте този файл до `.env`, използвайки командата по-долу. Този файл е _gitignore-д_, за да запази тайните в безопасност.

   ```bash
   cp .env.copy .env
   ```

3. Попълнете стойностите (заменете запълнителите от дясната страна на `=`) както е описано в следващия раздел.

3. (Опция) Ако използвате GitHub Codespaces, имате възможност да запазите променливите на средата като _секрети на Codespaces_, свързани с това хранилище. В този случай няма да е необходимо да настройвате локален .env файл. **Все пак, имайте предвид, че тази опция работи само ако използвате GitHub Codespaces.** Все още ще трябва да настроите .env файл, ако използвате Docker Desktop.

### 2.2. Попълнете файл `.env`

Нека бързо разгледаме имената на променливите, за да разберем какво представляват:

| Променлива  | Описание  |
| :--- | :--- |
| HUGGING_FACE_API_KEY | Това е токенът за достъп на потребителя, който сте настроили в своя профил |
| OPENAI_API_KEY | Това е ключът за удостоверяване за използване на услугата за не-Azure OpenAI крайни точки |
| AZURE_OPENAI_API_KEY | Това е ключът за удостоверяване за използване на тази услуга |
| AZURE_OPENAI_ENDPOINT | Това е разположената крайна точка за ресурс Azure OpenAI |
| AZURE_OPENAI_DEPLOYMENT | Това е крайна точка за разгръщане на модел за _генериране на текст_ |
| AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT | Това е крайна точка за разгръщане на модел за _текстови вграждания_ |
| | |

Забележка: Последните две променливи на Azure OpenAI отразяват модел по подразбиране за завършване на чат (генериране на текст) и векторно търсене (вграждания) съответно. Инструкции за тяхното задаване ще бъдат дефинирани в съответните задачи.

### 2.3 Конфигурирайте Azure: От портала

Стойностите за крайна точка и ключ на Azure OpenAI ще бъдат намерени в [Azure портала](https://portal.azure.com?WT.mc_id=academic-105485-koreyst), така че нека започнем оттам.

1. Отидете в [Azure портала](https://portal.azure.com?WT.mc_id=academic-105485-koreyst)
1. Кликнете върху опцията **Ключове и крайна точка** в страничната лента (менюто вляво).
1. Кликнете **Показване на ключове** - трябва да видите следното: KEY 1, KEY 2 и Крайна точка.
1. Използвайте стойността на KEY 1 за AZURE_OPENAI_API_KEY
1. Използвайте стойността на Крайна точка за AZURE_OPENAI_ENDPOINT

След това се нуждаем от крайните точки за конкретните модели, които сме разположили.

1. Кликнете върху опцията **Разгръщания на модели** в страничната лента (ляво меню) за ресурс Azure OpenAI.
1. На целевата страница кликнете **Управление на разгръщанията**

Това ще ви отведе до уебсайта на Azure OpenAI Studio, където ще намерим останалите стойности, както е описано по-долу.

### 2.4 Конфигурирайте Azure: От студиото

1. Отидете в [Azure OpenAI Studio](https://oai.azure.com?WT.mc_id=academic-105485-koreyst) **от вашия ресурс**, както е описано по-горе.
1. Кликнете върху таба **Разгръщания** (странична лента, вляво), за да видите текущо разположените модели.
1. Ако желаният от вас модел не е разположен, използвайте **Създаване на ново разгръщане**, за да го разположите.
1. Ще ви трябва модел за _генериране на текст_ - препоръчваме: **gpt-35-turbo**
1. Ще ви трябва модел за _вграждане на текст_ - препоръчваме **text-embedding-ada-002**

Сега актуализирайте променливите на средата, за да отразяват _името на разгръщане_, което сте използвали. Това обикновено ще бъде същото като името на модела, освен ако не сте го променили изрично. Така че, като пример, може да имате:

```bash
AZURE_OPENAI_DEPLOYMENT='gpt-35-turbo'
AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT='text-embedding-ada-002'
```

**Не забравяйте да запазите файла .env, когато сте готови**. Сега можете да излезете от файла и да се върнете към инструкциите за стартиране на тетрадката.

### 2.5 Конфигурирайте OpenAI: От профила

Вашият OpenAI API ключ може да бъде намерен във вашия [OpenAI акаунт](https://platform.openai.com/api-keys?WT.mc_id=academic-105485-koreyst). Ако нямате такъв, можете да се регистрирате за акаунт и да създадете API ключ. След като имате ключа, можете да го използвате, за да попълните променливата `OPENAI_API_KEY` във файла `.env`.

### 2.6 Конфигурирайте Hugging Face: От профила

Вашият Hugging Face токен може да бъде намерен във вашия профил под [Токени за достъп](https://huggingface.co/settings/tokens?WT.mc_id=academic-105485-koreyst). Не ги публикувайте или споделяйте публично. Вместо това, създайте нов токен за използване на този проект и го копирайте във файла `.env` под променливата `HUGGING_FACE_API_KEY`. _Забележка:_ Технически това не е API ключ, но се използва за удостоверяване, така че запазваме тази наименователна конвенция за консистентност.
