## 4. Модернизация на API

### 4.1 Остарели API модели (Миграцията завършена)

Всички Python и TypeScript **чат** образци са мигрирани от Chat Completions API към **Responses API** (`client.responses.create(...)` → `response.output_text`).

| Стар модел | Нов модел | Статус |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (чат) | `OpenAI(base_url="<endpoint>/openai/v1/")` (Responses API) | Завършено |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | Завършено |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (TypeScript) | `openai` пакет `client.responses.create()` → `response.output_text` | Завършено |
| `df.append()` (pandas) | `pd.concat()` | Завършено |

> **Забележка:** Образците от Microsoft Foundry Models, които използват `azure-ai-inference` / `@azure-rest/ai-inference` SDK (`client.complete()`), остават на Model Inference API, който не поддържа Responses API. `AzureOpenAI()` се запазва умишлено, където още е валиден (embedding-и и генериране на изображения).

### 4.2 Нови функции на API за демонстриране

## 5. Подобрения на инфраструктурата

### 5.1 Подобрения на CI/CD

Реализирано в [`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml): Линтинг/форматиране на Python (Ruff + Black) е **задължително** за поддържания модул `shared/` и се изпълнява **консултативно** за останалата част от учебната програма, плюс консултативен ESLint пас за JavaScript/TypeScript. Илюстративният базов пример беше:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### 5.2 Сигурностно сканиране

Реализирано в [`.github/workflows/security.yml`](../../../.github/workflows/security.yml): Анализ с CodeQL за Python и JavaScript/TypeScript (при push, pull request и седмичен график) плюс преглед на зависимости при pull request-и. Илюстративният базов пример беше:

```yaml
# .github/workflows/security.yml
name: Security Scan

on: [push, pull_request]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: javascript, python
      - uses: github/codeql-action/analyze@v3

  dependency-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
```
