# AGENTS.md

## ទិដ្ឋភាពទូទៅរបស់គម្រោង

បណ្ណាល័យនេះមានវគ្គសិក្សា ២១ មេរៀនពេញលេញដែលបង្រៀនមូលដ្ឋាន និងការអភិវឌ្ឍកម្មវិធី AI បង្កើត។ វគ្គសិក្សាត្រូវបានរចនាសម្រាប់អ្នកចាប់ផ្តើម ហើយគ្របដណ្តប់ពីគំនិតមូលដ្ឋានដល់ការបង្កើតកម្មវិធីដែលរួចរាល់សម្រាប់ផលិតកម្ម។

**បច្ចេកវិទ្យាមេគុណ:**
- Python 3.9+ ជាមួយបណ្ណាល័យៈ `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript ជាមួយ Node.js និងបណ្ណាល័យៈ `openai` (Azure OpenAI តាមរយៈចំណុចចូល v1 + Responses API), `@azure-rest/ai-inference` (Microsoft Foundry Models)
- សេវា Azure OpenAI, OpenAI API, និង Microsoft Foundry Models (GitHub Models នឹងផ្តាច់នៅចុងខែមិថុនា ២០២៦)
- សៀវភៅ Jupyter សម្រាប់ការសិក្សាប្រតិបត្តិការ
- Dev Containers សម្រាប់បរិយាកាសអភិវឌ្ឍន៍មានភាពសំរួល

**រចនាសម្ព័ន្ធ Repository:**
- ថតមេរៀនលេខ ២១ (00-21) មានចំណាំ README, ឧទាហរណ៍កូដ និងការប្រលង
- ការអនុវត្តន៍ជាច្រើន: Python, TypeScript, និងខ្លះ .NET ឧទាហរណ៍
- មេរៀនបកប្រែ មានជាមួយភាសាមហាសំខាន់ ៤០+
- ការកំណត់ទម្រង់មួយតាម `.env` (ប្រើ `.env.copy` ជាគំរូ)

## ដំណើរការអភិវឌ្ឍន៍

### អថេរស៊ើបបត់បរិយាកាស

មេរៀនទាំងអស់ដែលតម្រូវឲ្យចូលប្រើ API ប្រើអថេរបរិយាកាសកំណត់នៅ `.env` ៖

- `OPENAI_API_KEY` - សម្រាប់ OpenAI API
- `AZURE_OPENAI_API_KEY` - សម្រាប់ Azure OpenAI ក្នុង Microsoft Foundry (Azure OpenAI Service ជាផ្នែកមួយនៃ Microsoft Foundry: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - URL ចំណុចចូល Azure OpenAI (ចំណុចចូល Foundry resource)
- `AZURE_OPENAI_DEPLOYMENT` - ឈ្មោះការតំឡើងម៉ូដែលការសប្តិ
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - ឈ្មោះការតំឡើងម៉ូដែល embeddings
- `AZURE_OPENAI_API_VERSION` - កំណែ API (លំនាំដើម: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - សម្រាប់ម៉ូដែល Hugging Face
- `AZURE_INFERENCE_ENDPOINT` - ចំណុចចូល Microsoft Foundry Models (ប្រតិបត្តិការម៉ូដែលពហុអ្នកផ្គត់ផ្គង់)
- `AZURE_INFERENCE_CREDENTIAL` - កូនសោ API របស់ Microsoft Foundry Models (ជំនួស `GITHUB_TOKEN` ដែលនឹងផ្អាក)

## វិធានកូដ Style

### Python

- ប្រើ `python-dotenv` សម្រាប់គ្រប់គ្រងអថេរបរិយាកាស
- យកបណ្ណាល័យ `openai` សម្រាប់ធ្វើប្រតិបត្តិការជាមួយ API
- ប្រើ `pylint` សម្រាប់ត្រួតពិនិត្យគុណភាពកូដ (ខ្លះមានប្រើ `# pylint: disable=all` សម្រាប់ភាពស្រួល)
- អនុវត្តតាមការនិមិត្តសញ្ញា PEP 8
- រក្សាទុកកាកូដ API ក្នុង `.env` មិននៅក្នុងកូដ

### TypeScript

- ប្រើកញ្ចប់ `dotenv` សម្រាប់អថេរបរិយាកាស
- កំណត់ត្រា TypeScript នៅក្នុង `tsconfig.json` សម្រាប់កម្មវិធីនីមួយៗ
- ប្រើកញ្ចប់ `openai` សម្រាប់ Azure OpenAI (បញ្ជូនម៉ាស៊ីនឈ្មោះទៅចំណុចចូល `/openai/v1/` និងហៅ `client.responses.create`); ប្រើ `@azure-rest/ai-inference` សម្រាប់ Microsoft Foundry Models
- ប្រើ `nodemon` សម្រាប់អភិវឌ្ឍន៍ជាមួយការផ្ទុកឡើងម្ដងម្កាល
- សង់មុនការដំណើរការ៖ `npm run build` បន្ទាប់មក `npm start`

### ប្រព័ន្ធទូទៅ

- រក្សាឧទាហរណ៍កូដឲ្យសាមញ្ញ និងអប់រំ
- មានការកត់ត្រាពន្យល់គំនិតសំខាន់ៗ
- កូដមេរៀនមួយៗគួរត្រូវមានទ្រង់ទ្រាយផ្ទាល់ខ្លួន ហើយអាចដំណើរការបាន
- ប្រើការនៅឈ្មោះម៉ាតច្បាស់៖ `aoai-` សម្រាប់ Azure OpenAI, `oai-` សម្រាប់ OpenAI API, `githubmodels-` សម្រាប់ Microsoft Foundry Models (prefix ល្បីពី GitHub Models)

១. សាកល្បងការផ្លាស់ប្តូរកូដក្នុង Python និង TypeScript នៅពេលត្រូវការ
២. ដំណើរការវាយតម្លៃ Markdown (ធ្វើដោយស្វ័យប្រវត្តិនៅលើ PR)
៣. បញ្ចាក់ថា tracking ID មានសម្រាប់ URL Microsoft ទាំងអស់
៤. ពិនិត្យតំណភ្ជាប់ទាក់ទងមានសុពលភាព
៥. បញ្ចាក់រូបភាពត្រូវបានយោងយ៉ាងត្រឹមត្រូវ

### ទ្រង់ទ្រាយចំណងជើង PR

- ប្រើចំណងជើងពិពណ៌នា៖ `[Lesson 06] ដំណោះស្រាយកំហុសឧទាហរណ៍ Python` ឬ `Update README សម្រាប់មេរៀន 08`
- ឱ្យយោងចំពោះលេខបញ្ហា (issue) នៅពេលត្រូវការ៖ `Fixes #123`

### សេចក្តីពិពណ៌នារបស់ PR

- ពន្យល់អំពីការផ្លាស់ប្តូរនិងហេតុផល
- ផ្ដល់តំណភ្ជាប់ទៅបញ្ហាដែលពាក់ព័ន្ធ
- សម្រាប់ការផ្លាស់ប្តូរកូដ ចុះបញ្ជីឧទាហរណ៍ដែលបានសាកល្បង
- សម្រាប់ PR បកប្រែ បានរាប់បញ្ចូលឯកសារទាំងអស់សម្រាប់ការប្រមូលបកប្រែពេញលេញ

### លក្ខខណ្ឌការរួមចំណែក

- ចុះហត្ថលេខាលើ Microsoft CLA (ស្វ័យប្រវត្តិលើ PR ដំបូង)
- Fork repository ទៅគណនីរបស់អ្នកមុនធ្វើការផ្លាស់ប្តូរ
- មួយ PR សម្រាប់ការផ្លាស់ប្តូរយោលដែលមិនទាក់ទងគ្នា
- រក្សា PR ឲ្យមានផ្នែកតូច និងផ្តោតជាក់លាក់ពេលអាចធ្វើបាន

## ដំណើរការធម្មតា

### បន្ថែមឧទាហរណ៍កូដថ្មី

១. ទៅកាន់ថតមេរៀនដែលត្រឹមត្រូវ
២. បង្កើតឧទាហរណ៍នៅក្នុងថតរង `python/` ឬ `typescript/`
៣. អនុវត្តតាមប្រព័ន្ធ​ឈ្មោះៈ `{provider}-{example-name}.{py|ts|js}`
៤. សាកល្បងជាមួយគណនី API ពិតប្រាកដ
៥. ឯកសារបរិយាកាសអថេរថ្មី ក្នុង README មេរៀន

### កែប្រែឯកសារ README

១. កែសម្រួល README.md នៅក្នុងថតមេរៀន
២. អនុវត្តតាមវិធានការ Markdown (tracking IDs, តំណភ្ជាប់ទាក់ទង)
៣. ភាពបកប្រែបានគេចាត់ទុកដោយ GitHub Actions (មិនត្រូវកែដោយដៃ)
៤. សាកល្បងឲ្យទាំងអស់តំណភ្ជាប់មានសុពលភាព

### ធ្វើការជាមួយ Dev Containers

១. Repository រួមបញ្ចូល `.devcontainer/devcontainer.json`
២. ស្គ្រីបក្រោយបង្កើតតំឡើងការទាមទារពី Python ដោយស្វ័យប្រវត្តិ
៣. បន្ថែមសម្រាប់ Python និង Jupyter កំណត់រួចរាល់
៤. បរិយាកាសមានមូលដ្ឋានលើ `mcr.microsoft.com/devcontainers/universal:2.11.2`

## ការតំឡើង និងការបោះពុម្ភផ្សាយ

នេះគឺជាបណ្ណាល័យសិក្សា - មិនមានដំណើរការតំឡើងនោះទេ។ វគ្គសិក្សា​ត្រូវបានប្រើប្រាស់ដោយ៖

១. **Repository GitHub**៖ ចូលដំណើរការកូដ និងឯកសារ
២. **GitHub Codespaces**៖ បរិយាកាសអភិវឌ្ឍន៍ភ្លាមៗជាមួយការតំឡើងកំណត់រួច
៣. **Microsoft Learn**៖ មាតិកាអាចត្រូវបានផ្សព្វផ្សាយទៅវេទិកាសិក្សាផ្លូវការ
៤. **docsify**៖ គេហទំព័រឯកសារដែលបានបង្កើតពី Markdown (មើល `docsifytopdf.js` និង `package.json`)

### ការសាងសង់គេហទំព័រឯកសារ

```bash
# បង្កើត PDF ពីឯកសារណែនាំ (ប្រសិនបើត្រូវការ)
npm run convert
```

## ការរកមូលហេតុបញ្ហា

### បញ្ហាទូទៅ

**កំហុស Import Python**:
- ធានាថាបរិយាកាស virtual មិនត្រូវបានបិទ
- រត់ពាក្យបញ្ជា `pip install -r requirements.txt`
- ពិនិត្យកំណែ Python 3.9 ឡើងទៅ

**កំហុស Build TypeScript**:
- រត់ `npm install` នៅក្នុងថតកម្មវិធីច្បាស់លាស់
- ពិនិត្យកំណែ Node.js សមរម្យ
- សម្អាត `node_modules` ហើយដំឡើងឡើងវិញ ប្រសិនបើចាំបាច់

**កំហុសផ្ទៀងផ្ទាត់ API**:
- ពិនិត្យឯកសារ `.env` មាន និងមានតម្លៃត្រឹមត្រូវ
- ពិនិត្យថាគី API មានសុពលភាព និងមិនផុតកំណត់
- ធានាថា URL ចំណុចចូលត្រឹមត្រូវសម្រាប់តំបន់របស់អ្នក

**អថេរបរិយាកាស ខ្វះខាត**:
- ចម្លង `.env.copy` ទៅ `.env`
- បំពេញទាំងអស់តម្លៃដែលត្រូវការសម្រាប់មេរៀនដែលអ្នកកំពុងធ្វើការ
- ចាប់ផ្តើមកម្មវិធីឡើងវិញបន្ទាប់ពីកែប្រែ `.env`
