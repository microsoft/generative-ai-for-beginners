# AGENTS.md

## ទិដ្ឋភាពគំរូគម្រោង

កន្លែងផ្ទុកនេះមានមេរៀនគោលការណ៍ចន្លោះ ២១ មេរៀន ដែលបង្រៀនពីមូលដ្ឋាន Generative AI និងការអភិវឌ្ឍកម្មវិធី។ មេរៀនត្រូវបានរៀបចំសម្រាប់អ្នកចាប់ផ្តើម និងគ្របដណ្តប់ពីយោងមូលដ្ឋានរហូតដល់ការបង្កើតកម្មវិធីដែលមានស្រែបម្រង់ធ្វើការផលិត។

**បច្ចេកវិទ្យាសំខាន់ៗ៖**
- Python 3.9+ ជាមួយបណ្ណាល័យ៖ `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript ជាមួយ Node.js និងបណ្ណាល័យ៖ `@azure/openai`, `@azure-rest/ai-inference`, `openai`
- សេវា Azure OpenAI, API OpenAI, និង GitHub Models
- Jupyter Notebooks សម្រាប់ការរៀនប្រតិបត្តិការ
- Dev Containers សម្រាប់បរិយាកាសអភិវឌ្ឍន៍ស្របតាមគ្នា

**រចនាសម្ព័ន្ធកន្លែងផ្ទុក៖**
- ថតមេរៀនលេខ ២១ (#០០-២១) មាន README, ឧទាហរណ៍កូដ និងការងារ
- មានការអនុវត្តន៍ច្រើន៖ Python, TypeScript ហើយរំពឹងឃើញ .NET នៅពេលខ្លះ
- ថតបកប្រែជាភាសាជាច្រើនជាង ៤០ ភាសា
- ការកំណត់នៅកណ្តាលតាមឯកសារ `.env` (ប្រើ `.env.copy` ជាគំរូ)

## ការកំណត់ត្រា កាមម៉ាន់

### ការកំណត់ត្រាកន្លែងផ្ទុកដំបូង

```bash
# ចម្លងហាងផ្ទុក
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# បម្រើគំរូបរិយាកាស
cp .env.copy .env
# កែសំរួល .env ជាមួយកូនសោ API និងចំណុចចេញរបស់អ្នក
```

### ការកំណត់បរិយាកាស Python

```bash
# បង្កើតបរិយាកាសជាក់ស្តែង
python3 -m venv venv

# បើកបរិយាកាសជាក់ស្តែង
# លើ macOS/Linux:
source venv/bin/activate
# លើ Windows:
venv\Scripts\activate

# ដំឡើងការពឹងផ្អែក
pip install -r requirements.txt
```

### ការកំណត់ Node.js/TypeScript

```bash
# តំឡើងភាសាដើមកម្រិត root-level dependencies (សម្រាប់ឧបករណ៍ឯកសារ)
npm install

# សម្រាប់ឧទាហរណ៍ TypeScript មេរៀនជាក់លាក់មួយៗ, ទៅកាន់មេរៀនដែលច្បាស់លាស់:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### កំណត់ Dev Container (ផ្តល់អនុសាសន៍)

កន្លែងផ្ទុកមានការកំណត់ `.devcontainer` សម្រាប់ GitHub Codespaces ឬ VS Code Dev Containers៖

1. បើកកន្លែងផ្ទុកក្នុង GitHub Codespaces ឬ VS Code ជាមួយកម្មវិធីបន្ថែម Dev Containers
2. Dev Container នឹងអូតូម៉ាទិក៖
   - ដំឡើងការពឹងផ្អែក Python ពី `requirements.txt`
   - បំពេញ script បន្ទាប់បង្កើត (`.devcontainer/post-create.sh`)
   - រចនាបថ kernel Jupyter

## ដំណើរការអភិវឌ្ឍន៍

### អថេរបរិយាកាស

មេរៀនទាំងអស់ដែលត្រូវការចូលប្រើ API ប្រើអថេរបរិយាកាសដែលកំណត់នៅ `.env`៖

- `OPENAI_API_KEY` - សម្រាប់ API OpenAI
- `AZURE_OPENAI_API_KEY` - សម្រាប់ សេវា Azure OpenAI
- `AZURE_OPENAI_ENDPOINT` - អាសយដ្ឋាន Azure OpenAI
- `AZURE_OPENAI_DEPLOYMENT` - ឈ្មោះ deployment ម៉ូដែលជជែក
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - ឈ្មោះ deployment ម៉ូដែល embeddings
- `AZURE_OPENAI_API_VERSION` - គំណូស API (លំនាំដើម៖ `2024-02-01`)
- `HUGGING_FACE_API_KEY` - សម្រាប់ម៉ូដែល Hugging Face
- `GITHUB_TOKEN` - សម្រាប់ GitHub Models

### ដំណើរការឧទាហរណ៍ Python

```bash
# ធ្វើដំណើរទៅកាន់ថតមេរៀន
cd 06-text-generation-apps/python

# ប្រតិបត្តិស្គ្រីប Python
```

### ដំណើរការឧទាហរណ៍ TypeScript

- មេរៀន **"បង្កើត"** ៖ រួមបញ្ចូលឧទាហរណ៍កូដ Python និង TypeScript

## រៀបចំស្ទាយកូដ

### Python

- ប្រើ `python-dotenv` សម្រាប់គ្រប់គ្រងអថេរបរិយាកាស
- នាំចូលបណ្ណាល័យ `openai` សម្រាប់អន្តរកម្ម API
- ប្រើ `pylint` សម្រាប់ត្រួតពិនិត្យ (ខ្លះមាន `# pylint: disable=all` ដើម្បីបង្រៀនសាមញ្ញ)
- ទាំងគោលការណ៍ PEP 8 សម្រាប់ដាក់ឈ្មោះ
- រក្សាទុកសំរបសំរួល API ក្នុង .env មិនត្រូវរក្សាទុកក្នុងកូដ

### TypeScript

- ប្រើកញ្ចប់ `dotenv` សម្រាប់អថេរបរិយាកាស
- ការកំណត់ TypeScript នៅ `tsconfig.json` សម្រាប់មួយម្ដុំកម្មវិធី
- ប្រើ `@azure/openai` ឬ `@azure-rest/ai-inference` សម្រាប់សេវា Azure
- ប្រើ `nodemon` សម្រាប់អភិវឌ្ឍន៍ជាមួយការផ្ទុកឡើងឡើងវិញស្វ័យប្រវត្តិ
- សាងសង់មុនរត់៖ `npm run build` បន្ទាប់ `npm start`

### ស្តង់ដារទូទៅ

- គោរពឧទាហរណ៍កូដឲ្យសាមញ្ញ និងឲ្យអប់រំ
- រួមបញ្ចូលមតិលម្អិតពន្យល់គំនិតសំខាន់ៗ
- គ្រប់មេរៀនកូដគួរតែទប់ទល់ដោយខ្លួនឯង និងអាចរត់បាន
- ប្រើឈ្មោះយ៉ាងឆ្លាតវៃ៖ `aoai-` សម្រាប់ Azure OpenAI, `oai-` សម្រាប់ OpenAI API, `githubmodels-` សម្រាប់ GitHub Models

## គោលការណ៍ឯកសារ

### ស្ទាយ Markdown

- ទាំងអស់ URL ត្រូវតែបិទក្នុងទម្រង់ `[text](../../url)` ដោយគ្មានចន្លោះបន្ថែម
- តំណភ្ជាប់ទំនាក់ទំនងចាប់ផ្តើមជាមួយ `./` ឬ `../`
- តំណភ្ជាប់ទៅតំបន់ Microsoft ត្រូវរួមបញ្ចូលអត្តសញ្ញាណតាមដាន៖ `?WT.mc_id=academic-105485-koreyst`
- មិនប្រើទីតាំងភាសាជាតិច្បាស់ក្នុង URL (ចៀសវាង `/en-us/`)
- រូបភាពរក្សានៅថត `./images` ជាមួយឈ្មោះពណ៌នាថ្លៃថ្លា
- ប្រើអក្សរអង់គ្លេស សញ្ញាស្រៈ និងសញ្ញាទាក់ទាញក្នុងឈ្មោះឯកសារ

### គាំទ្រការបកប្រែ

- កន្លែងផ្ទុកគាំទ្រភាសាជាង ៤០ តាមរយៈ GitHub Actions ស្វ័យប្រវត្តិ
- រូបមន្តបានរក្សាទុកនៅថត `translations/`
- មិនបានដាក់ស្នើការបកប្រែបែបផ្នែក
- មិនទទួលការបកប្រែម៉ាស៊ីន
- រូបភាពបកប្រែរក្សាទុកនៅ `translated_images/`

## ការធ្វើតេស្ត និងការផ្ទៀងផ្ទាត់

### ការត្រួតពិនិត្យមុនដាក់ស្នើ

កន្លែងផ្ទុកនេះប្រើ GitHub Actions សម្រាប់ការផ្ទៀងផ្ទាត់។ មុនដាក់ស្នើ PR៖

1. **ត្រួតពិនិត្យតំណ Markdown**:
   ```bash
   # ការងារ validate-markdown.yml ពិនិត្យ៖
   # - ផ្លូវស្របដែលខូច
   # - គ្មានអត្តសញ្ញាណតាមដានលើផ្លូវ
   # - គ្មានអត្តសញ្ញាណតាមដានលើ URL
   # - URL មានភាសាប្រទេស
   # - URL ខាងក្រៅដែលខូច
   ```

2. **តេស្តដោយដៃ**:
   - តេស្តឧទាហរណ៍ Python៖ បើក venv ហើយរត់ script
   - តេស្តឧទាហរណ៍ TypeScript៖ `npm install`, `npm run build`, `npm start`
   - ផ្ទៀងផ្ទាត់អថេរបរិយាកាសត្រូវការពិត
   - ពិនិត្យថា key API ដំណើរការជាមួយឧទាហរណ៍

3. **ឧទាហរណ៍កូដ**:
   - ប្រាកដថា កូដទាំងអស់រត់ដោយគ្មានកំហុស
   - តេស្តជាមួយសេវា Azure OpenAI និង OpenAI API ប្រសិនបើអាច
   - ផ្ទៀងផ្ទាត់ឧទាហរណ៍ជាមួយ GitHub Models នៅកន្លែងដែលគាំទ្រ

## ការបញ្ចូន និងការបោះពុម្ភផ្សាយ

នេះជាកន្លែងផ្ទុកសម្រាប់រៀន - មិនមានដំណើរការបញ្ចូនទេ។ មេរៀនត្រូវបានប្រើដោយ៖

1. **GitHub Repository**៖ ចូលដំណើរការកូដ និងឯកសារ
2. **GitHub Codespaces**៖ បរិយាកាសអភិវឌ្ឍន៍ភ្លាមៗជាមួយការកំណត់រួចហើយ
3. **Microsoft Learn**៖ មាតិការអាចត្រូវបានចែកចាយទៅវេទិកាសិក្សាផ្លូវការ
4. **docsify**៖ គេហទំព័រ​ឯកសារ​បង្កើត​ពី Markdown (មើល `docsifytopdf.js` និង `package.json`)

### ការសាងសង់គេហទំព័រ​ឯកសារ

```bash
# បង្កើត PDF ពីឯកសារ (ប្រសិនបើត្រូវការ)
npm run convert
```

## ការដោះស្រាយបញ្ហា

### ប្រធានបទទូទៅ

**កំហុសនាំចូល Python**:
- ធានាថាបរិយាកាស virtual បានបើក
- រត់ `pip install -r requirements.txt`
- ពិនិត្យកំណែ Python មិនតិចជាង 3.9

**កំហុសសាងសង់ TypeScript**:
- រត់ `npm install` នៅកម្មវិធីច្បាស់លាស់
- ពិនិត្យមើលកំណែ Node.js ត្រូវគ្នា
- លុប `node_modules` ហើយដំឡើងម្តងទៀតបើចាំបាច់

**កំហុសផ្ទៀងផ្ទាត់ API**:
- ពិនិត្យឯកសារ `.env` មានគម្លាតត្រឹមត្រូវ
- ពិនិត្យសោ API ត្រឹមត្រូវ និងមិនបានផុតកំណត់
- ធានាថា URL អ្នកផ្តល់សេវាត្រឹមត្រូវសម្រាប់តំបន់របស់អ្នក

**បាត់បង់អថេរបរិយាកាស**:
- តម្លើង `.env.copy` ទៅ `.env`
- បញ្ចូលតម្លៃទាំងអស់ត្រូវការសម្រាប់មេរៀនដែលក្នមួយកំពុងបម្រើ
- ចាប់ផ្តើមកម្មវិធីម្តងទៀតបន្ទាប់ពីបន្ទាន់សម័យ `.env`
