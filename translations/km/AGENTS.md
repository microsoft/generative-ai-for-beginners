## ទិដ្ឋភាពសង្ខេបគម្រោង

ឃ្លាំងនេះមានមេរៀនកម្មវិធីសិក្សាចំនួន ២១ មេរៀនពេញលេញគឹបង្ហាញគ្រឹះនិងការអភិវឌ្ឍន៌កម្មវិធី Generative AI។ វគ្គសិក្សានេះបានរចនាសម្រាប់អ្នកចាប់ផ្តើម ហើយគ្របដណ្តប់ពីមូលដ្ឋានដល់ការសំណង់កម្មវិធីដែលអាចប្រើប្រាស់បាន។

**បច្ចេកវិទ្យាសំខាន់ៗ៖**
- Python 3.9+ ជាមួយបណ្ណាល័យ៖ `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript ជាមួយ Node.js និងបណ្ណាល័យ៖ `openai` (Azure OpenAI តាមរយៈចំណុច v1 និង Responses API), `@azure-rest/ai-inference` (ម៉ូដែល Microsoft Foundry)
- សេវា Azure OpenAI, OpenAI API, និង ម៉ូដែល Microsoft Foundry (ម៉ូដែល GitHub កំពុងផ្អាកចាប់ពីចុងខែកក្កដា ២០២៦)
- សៀវភៅ Jupyter សម្រាប់ការសិក្សាផ្ទាល់  
- Dev Containers សម្រាប់បរិយាកាសអភិវឌ្ឍន៍ដែលមានភាពរឹងរូស

**រចនាសម្ព័ន្ធឃ្លាំង៖**
- បណ្ដាញថតមេរៀនលេខ ២១ (០០-២១) ដែលមាន README, ឧទាហរណ៍កូដ និងការងារផ្ទៀងផ្ទាត់
- ការអនុវត្តច្រើនប្រភេទ៖ Python, TypeScript, និងខ្លះៗ .NET
- ថតបំរែបំរួលភាសាជាមួយជាង ៤០ ភាសា

### ការរៀបចំបឋមសម្រាប់ឃ្លាំង

```bash

# យកគំរូបរិយាកាស

# កែប្រែ .env ជាមួយកូនសោ API និងចំណុចចេញដំណើរការរបស់អ្នក
```

### ការរៀបចំបរិយាកាស Python


## ជំហានអភិវឌ្ឍន៍

### អថេរបរិយាកាស

មេរៀនទាំងអស់ដែលតម្រូវឱ្យចូលប្រើ API ប្រើអថេរបរិយាកាសដែលកំណត់ក្នុង `.env`៖

- `OPENAI_API_KEY` - សម្រាប់ OpenAI API
- `AZURE_OPENAI_API_KEY` - សម្រាប់ Azure OpenAI ក្នុង Microsoft Foundry (សេវា Azure OpenAI គឺជាផ្នែកមួយនៃ Microsoft Foundry: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - URL ចំណុចចេញ Azure OpenAI (ចំណុចធនធាន Foundry)
- `AZURE_OPENAI_DEPLOYMENT` - ឈ្មោះការតំឡើងម៉ូដែលបញ្ចប់ជជែក (លំនាំដើមវគ្គ៖ `gpt-5-mini`)
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - ឈ្មោះការតំឡើងម៉ូដែល embeddings (លំនាំដើមវគ្គ៖ `text-embedding-3-small`)
- `AZURE_OPENAI_API_VERSION` - មុខងារ API (លំនាំដើម៖ `2024-10-21`)
- `HUGGING_FACE_API_KEY` - សម្រាប់ម៉ូដែល Hugging Face
- `AZURE_INFERENCE_ENDPOINT` - ចំណុចចេញម៉ូដែល Microsoft Foundry (កាតាឡុកម៉ូដែលពីអ្នកផ្គត់ផ្គង់ច្រើន)
- `AZURE_INFERENCE_CREDENTIAL` - កូនសោ API ម៉ូដែល Microsoft Foundry (ជំនួស `GITHUB_TOKEN` ដែលនឹងផ្អាក)
- `AZURE_INFERENCE_CHAT_MODEL` - ម៉ូដែលមិនមានការគិត (ឧ. `Llama-3.3-70B-Instruct`) ប្រើសម្រាប់ឧទាហរណ៍ `temperature` ព្រោះម៉ូដែលគិតមិនគាំទ្រការគ្រប់គ្រង sampling

### ប្រព័ន្ធម៉ូដែល (សំខាន់)

- **ម៉ូដែលជជែកលំនាំដើមគឺ `gpt-5-mini`** - ម៉ូដែល **គិត** បច្ចុប្បន្នដែលមិនលុបចោល។ ចាប់ពីឆ្នាំ ២០២៦ ម៉ូដែល "mini" ដែលមានសមត្ថភាព `temperature` ចាស់ៗ (`gpt-4o-mini`, `gpt-4.1-mini`) កំពុង *លុបចោល* ដូច្នេះវគ្គសិក្សានេះបានបញ្ជាក់លើក្រុម GPT-5។
- **ម៉ូដែលគិតដាក់ចោល `temperature` និង `top_p`** ហើយប្រើ `max_output_tokens` (Responses API) / `max_completion_tokens` (chat completions) ជំនួស `max_tokens`។ សូម **កុំ** បន្ថែម `temperature`/`top_p`/`max_tokens` លើឧទាហរណ៍ដែលហៅ `gpt-5-mini`។
- **ដើម្បីបង្ហាញពី `temperature`**, ឧទាហរណ៍ប្រើម៉ូដែល **Llama** (`Llama-3.3-70B-Instruct`) តាមរយៈចំណុចចេញម៉ូដែល Microsoft Foundry (`AZURE_INFERENCE_CHAT_MODEL`)។ គ្រប់គ្រងម៉ូដែលគិតដោយការតំលើងប៊ុល + ការគ្រប់គ្រងការគិត មិនមែនតាមលក្ខណៈ sampling។
- **ការកែលម្អម៉ូដែល (មេរៀន ១៨)** រក្សា `gpt-4.1-mini`: GPT-5 គាំទ្រតែការកែលម្អហ្វៃន៍-ធ្យួង (RFT) ប៉ុណ្ណោះ មិនគាំទ្រការកែលម្អតាមការបង្រៀន (SFT) ដែលបង្ហាញនៅទីនោះទេ។
- មេរៀន ២០ (Mistral) និង ២១ (Meta) រក្សា `temperature`/`max_tokens` ព្រោះគោលដៅទៅម៉ូដែល Mistral/Llama ដែលគាំទ្រពួកវា។

## មគ្គុទេសក្ដាប្រកួតកូដ

### Python

- ប្រើ `python-dotenv` សម្រាប់គ្រប់គ្រងអថេរបរិយាកាស
- នាំចូលបណ្ណាល័យ `openai` សម្រាប់ប្រតិបត្តិ API
- ប្រើ `pylint` សម្រាប់បញ្ឆេកកូដ (ឧទាហរណ៍ខ្លះមាន `# pylint: disable=all` សម្រាប់ភាពសាមញ្ញ)
- ធ្វើតាមក្រម PEP 8 សម្រាប់ការផ្ដាសំនួរឈ្មោះ
- រក្សាកូនសោ API ក្នុងឯកសារ `.env` មិនចេញក្នុងកូដ

### TypeScript

- ប្រើកញ្ចប់ `dotenv` សម្រាប់អថេរបរិយាកាស
- ការកំណត់ TypeScript នៅក្នុង `tsconfig.json` សម្រាប់កម្មវិធីនីមួយៗ
- ប្រើកញ្ចប់ `openai` សម្រាប់ Azure OpenAI (បញ្ជាក់អតិថិជននៅចំណុច `/openai/v1/` ហើយហៅ `client.responses.create`); ប្រើ `@azure-rest/ai-inference` សម្រាប់ម៉ូដែល Microsoft Foundry
- ប្រើ `nodemon` សម្រាប់អភិវឌ្ឍដោយមានការតំឡើងឡើងវិញស្វ័យប្រវត្តិ
- សំណង់មុនរត់៖ `npm run build` បន្ទាប់មក `npm start`

### ការត្រួតពិនិត្យមុនដាក់ស្នើ

ឃ្លាំងនេះប្រើ GitHub Actions សម្រាប់ផ្ទៀងផ្ទាត់។ មុនដាក់ស្នើ PR:

១. **ពិនិត្យតំណ Markdown**:
   ```bash
   # ការងារប្រតិបត្តិ validate-markdown.yml ពិនិត្យមើល៖
   # - ផ្លូវទាក់ទងដែលខូច
   # - ការខ្វះ ID តាមដានលើផ្លូវ
   # - ការខ្វះ ID តាមដានលើ URL
   # - URL ដែលមានភាសាទីតាំងប្រទេស
   # - URL ខាងក្រៅដែលខូច
   ```

២. **ការធ្វើតេស្តដោយដៃ**:
   - សាកល្បងឧទាហរណ៍ Python: បើក venv ហើយរត់ស្គ្រីប
   - សាកល្បងឧទាហរណ៍ TypeScript: `npm install`, `npm run build`, `npm start`
   - បញ្ជាក់ថាអថេរបរិយាកាសបានកំណត់ត្រឹមត្រូវ
   - ពិនិត្យថាកូនសោ API ប្រតិបត្តិសម្រួលជាមួយឧទាហរណ៍កូដ

៣. **ឧទាហរណ៍កូដ**:
   - ធានាថាកូដទាំងអស់ដំណើរការជោគជ័យដោយគ្មានកំហុស
   - សាកល្បងជាមួយ Azure OpenAI និង OpenAI API នៅពេលអាចបាន
   - បញ្ជាក់ឧទាហរណ៍ដំណើរការជាមួយ ម៉ូដែល Microsoft Foundry នៅពេលគាំទ្រ

នេះជាឃ្លាំងសិក្សាអំពីមេរៀននិងឧទាហរណ៍។ មិនមានតេស្តឯកតា ឬតេសត់ការរួមបញ្ចូល ។ ការផ្ទៀងផ្ទាត់គឺជាៈ

- GitHub Actions សម្រាប់ផ្ទៀងផ្ទាត់ Markdown
- ការពិនិត្យសមាជិកសហគមន៍លើមាតិកាសិក្សា

- សម្រាប់ PR ប្រែបម្រួល រួមបញ្ចូលឯកសារទាំងអស់សម្រាប់ការប្រែប្រួលពេញលេញ

### បញ្ហាទូទៅ

**កំហុសនាំចូល Python**:

- រត់ `pip install -r requirements.txt`
- ពិនិត្យកំណែ Python គឺ 3.9+

**កំហុសសាងសង់ TypeScript**:
- រត់ `npm install` ក្នុងថតកម្មវិធីជាក់លាក់
- ពិនិត្យកំណែ Node.js ត្រូវគ្នា
- សម្អាត `node_modules` ហើយដំឡើងឡើងវិញប្រសិនបើត្រូវការ

**កំហុសសំរាប់បញ្ចាក់ API**:
- ពិនិត្យឯកសារ `.env` មានរួចហើយមានតម្លៃត្រឹមត្រូវ
- ពិនិត្យកូនសោ API ត្រឹមត្រូវ ហើយមិនផុតកំណត់
- ធ្វើអោយប្រាកដថា URL ចំណុចចេញត្រឹមត្រូវសម្រាប់តំបន់របស់អ្នក

**អថេរបរិយាកាសខ្វះ**:
- ចម្លង `.env.copy` ទៅ `.env`
- បំពេញតម្លៃទាំងអស់ដែលត្រូវការសម្រាប់មេរៀនដែលអ្នកកំពុងធ្វើការ
- ចាប់ផ្តើមកម្មវិធីឡើងវិញបន្ទាប់ពីធ្វើបំលែង `.env`
