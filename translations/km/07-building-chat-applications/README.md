ឥឡូវនេះដែលយើងបានឃើញពីរបៀបដែលយើងអាចបង្កើតកម្មវិធីបង្កើតអត្ថបទបាន មកមើលកម្មវិធីជជែក។

កម្មវិធីជជែកបានបញ្ចូលចូលជាគ្រឿងផ្សំមួយក្នុងជីវិតប្រចាំថ្ងៃរបស់យើង មិនត្រឹមតែជារបៀបនៃការជជែកធម្មតា​ទេ។ វា​ជា​ផ្នែក​សំខាន់​របស់​សេវាកម្ម​អតិថិជន គាំទ្របច្ចេកទេស និងប្រព័ន្ធប្រឹក្សាជំនាញដែលមានស្មុគស្មាញ។ អ្នកប្រហែលជាបានទទួលជំនួយពីកម្មវិធីជជែកមួយមិនយូរប៉ុន្មានមុន។ ពេលយើងបញ្ចូលបច្ចេកវិទ្យាជាងសម័យដូចជា AI បង្កើតចូលក្នុងវេទិកាទាំងនេះ ការលំបាកនឹងកើនឡើងហើយការលំបាកផងដែរ។

សំណួរខ្លះដែលយើងត្រូវបានឆ្លើយតបមានដូចជា:

- **ការបង្កើតកម្មវិធី**។ តើយើងធ្វើដូចម្តេចដើម្បីបង្កើត និងបញ្ចូលកម្មវិធីដែលមានថាមពល AI ទៅប្រើប្រកបដោយប្រសិទ្ធភាពសម្រាប់ករណីប្រើប្រាស់ជាក់លាក់?
- **ការត្រួតពិនិត្យ**។ បន្ទាប់ពីដាក់ឲ្យដំណើរការ តើយើងអាចត្រួតពិនិត្យ និងធ្វើឲ្យប្រាកដថាកម្មវិធីដំណើរការដោយមានគុណភាពខ្ពស់បំផុតទេ ក្នុងជ្រុងនៃមុខងារ និងការអនុវត្តតាម [មូលដ្ឋានប្រាំមួយនៃ AI ទទួលខុសត្រូវ](https://www.microsoft.com/ai/responsible-ai?WT.mc_id=academic-105485-koreyst)?

ពេលយើងចូលទៅកាន់យុគសម័យដែលកំណត់ដោយស្វ័យប្រវត្តិ និងការប្រាស្រស់ល្អរវាងមនុស្សនិងម៉ាស៊ីន ការយល់ដឹងពីរបៀបដែល AI បង្កើតបន្តផ្លាស់ប្ដូរប្រភេទ ផ្ទៃខាងក្នុង និងភាពឆបគ្នារបស់កម្មវិធីជជែកក្លាយជារឿងសំខាន់។ មេរៀននេះនឹងស្រាវជ្រាវអំពីធាតុស្ថាបัตដែលគាំទ្រប្រព័ន្ធស្មុគស្មាញទាំងនេះ អនុវត្តវិធីសាស្ត្រសម្រាប់បង្រួបបង្រួមសម្រាប់ភាសាផ្ទាល់ខ្លួន ហើយវាយតម្លៃម៉េត្រនិងគន្លឹះដើម្បីធានាការចេញផ្សាយ AI ដោយទទួលខុសត្រូវ។

## ការណែនាំ

មេរៀននេះបង្ហាញពី៖

- វិធីសាស្ត្រសម្រាប់បង្កើត និងបញ្ចូលកម្មវិធីជជែកដោយប្រសិទ្ធិភាព។
- របៀបអនុវត្តការប្តូរតាមបំណង និងបង្រួបបង្រួមទៅលើកម្មវិធី។
- យុទ្ធសាស្ត្រនិងការពិចារណាដើម្បីត្រួតពិនិត្យកម្មវិធីជជែកប្រកបដោយប្រសិទ្ធភាព។

## គោលបំណងសិក្សា

នៅចប់មេរៀននេះ អ្នកនឹងអាច៖

- អធិប្បាយពីការពិចារណានៃការបង្កើត និងបញ្ចូលកម្មវិធីជជែកទៅក្នុងប្រព័ន្ធដែលមានរួច។
- ប្តូរកម្មវិធីជជែកឲ្យសាកសមនឹងករណីប្រើប្រាស់ជាក់លាក់។
- កំណត់ម៉េត្រសំខាន់ៗ និងការពិចារណាដើម្បីត្រួតពិនិត្យសន្ទុះតម្លៃនៃកម្មវិធីជជែក AI ដោយមានគុណភាព។ 
- ប្រាកដថាកម្មវិធីជជែកប្រើ AI ដោយទទួលខុសត្រូវ។

## ការបញ្ចូល AI បង្កើតចូលក្នុងកម្មវិធីជជែក

ការលើកកម្ពស់កម្មវិធីជជែកតាមរយៈ AI បង្កើត មិនត្រឹមតែធ្វើឲ្យវាពិចារណាល្អកាន់តែច្រើនទេ ប៉ុន្តែគឺជាការបង្រួបបង្រួមបណ្តាញស្ថាបត្យកម្ម ការសម្តែង និងចំណុចប្រទាក់អ្នកប្រើ ដើម្បីផ្ដល់បទពិសោធន៍ប្រើប្រាស់ដែលមានគុណភាព។ វារួមមានការស្រាវជ្រាវអំពីមូលដ្ឋានស្ថាបត្យកម្ម ការបញ្ចូល API និងការពិចារណាចំណុចប្រទាក់អ្នកប្រើ។ ផ្នែកនេះមានគោលបំណងផ្តល់ផែនទីពេញលេញសម្រាប់រុករកកន្លែងស្មុគស្មាញទាំងនេះ មិនថាអ្នកកំពុងចងភ្ជាប់ពួកវាជាមួយប្រព័ន្ធដែលមានរួច ឬក៏បង្កើតវាជាវេទិកាដាច់ដោយឡែក។

នៅចប់ផ្នែកនេះ អ្នកនឹងមានជំនាញដែលត្រូវការ ដើម្បីកសាងនិងបញ្ចូលកម្មវិធីជជែកបានយ៉ាងមានប្រសិទ្ធិភាព។

- **លឿនដំណើរការបង្កើត និងកាត់បន្ថយភាពធ្ងន់របស់ការងារ**: ដឹងច្បាស់ថាការប្រើមុខងារសង់រួចជំនួសការបង្កើតដោយខ្លួនឯងទៅទទួលអត្ថប្រយោជន៍ ដូចជាអនុវត្តមុខងារមុខងារអាជីវកម្មដែលអ្នកឃើញមានសារៈសំខាន់ជាង។
- **កំណត់មុខងារល្អប្រសើរជាងមុន**: ពេលបង្កើតមុខងារពីសូន្យ អ្នកនឹងសួរខ្លួនថា "តើវាត្រូវការការពង្រីកបានយ៉ាងដូចម្តេច? តើកម្មវិធីនេះអាចដោះស្រាយការចូលដំណើរការច្រើនក្នុងពេលខ្លីទេ?" SDK និង API ដែលបានថែរក្សាល្អជាញឹកញាប់មានដំណោះស្រាយសម្រាប់បញ្ហាទាំងនេះ។
- **ងាយស្រួលថែទាំ**: ការបច្ចុប្បន្នបច្ចេកវិទ្យា និងការកែលម្អទាន់សម័យមានភាពងាយស្រួលជាង ព្រោះភាគច្រើន API និង SDK ត្រូវការតែមួយកំណែបណ្ណាល័យបច្ចុប្បន្នប៉ុណ្ណោះនៅពេលមានកំណែថ្មី។
- **ចូលដំណើរការបច្ចេកវិទ្យាដ៏ទំនើបចុងក្រោយ**: ប្រើម៉ូដែលដែលបានបង្រៀនល្អ និងបង្រៀនលើទិន្នន័យធំធេងផ្តល់ឲ្យកម្មវិធីនូវភាពស្ទាត់ជំនាញភាសាធម្មជាតិ។

ការចូលប្រើមុខងារបណ្ណាល័យ SDK ឬ API គឺទាមទារពាក្យអនុញ្ញាតសម្រាប់ប្រើសេវាកម្មដែលផ្តល់ឲ្យ ដែលភាគច្រើនធ្វើតាមកូនសោឯកត្តភាព ឬសញ្ញាអះអាង។ យើងនឹងប្រើបណ្ណាល័យ OpenAI Python ដើម្បីស្វែងយល់ពីរូបរាង នេះ។ អ្នកក៏អាចសាកល្បងផងដោយខ្លួនអ្នកនៅក្នុង [សៀវភៅកំណត់ត្រាសម្រាប់ OpenAI](./python/oai-assignment.ipynb?WT.mc_id=academic-105485-koreyst) ឬ [សៀវភៅកំណត់ត្រាសម្រាប់ Azure OpenAI Services](./python/aoai-assignment.ipynb?WT.mc_id=academic-105485-koreys) សម្រាប់មេរៀននេះ។

```python

API_KEY = os.getenv("OPENAI_API_KEY","")
```

## បទពិសោធន៍អ្នកប្រើប្រាស់ (UX)

គោលការណ៍ UX ទូទៅអាប់ដេតសម្រាប់កម្មវិធីជជែក តែមានការពិចារណាពិសេសបន្ថែមចំពោះមុខងារបញ្ចូលបច្ចេកវិទ្យាសិក្សាម៉ាស៊ីនដែលពាក់ព័ន្ធ។

- **មេកានិចសម្រាប់ដោះស្រាយភាពចំរូងចំរាស**: ម៉ូដែល AI បង្កើតមើលម្តងមួយអាចបង្កើតចម្លើយមិនច្បាស់លាស់។ មុខងារមួយដែលអនុញ្ញាតឲ្យអ្នកប្រើសួរពន្យល់អាចមានប្រយោជន៍នៅពេលមានបញ្ហានេះ។
- **រក្សាភាពនៅក្នុងបរិបទ**: ម៉ូដែល AI បង្កើតកម្រិតខ្ពស់មានសមត្ថភាពចងចាំបរិបទក្នុងជជែក ដែលអាចជាទ្រព្យសម្បត្តិសម្រាប់បទពិសោធន៍អ្នកប្រើ។ ផ្តល់ឲ្យអ្នកប្រើអំណាចគ្រប់គ្រង និងគ្រប់គ្រងបរិបទធ្វើឲ្យទទួលបានបទពិសោធន៍ល្អ ប៉ុន្តែបង្កើតហានិភ័យនៃការរក្សាទុកព័ត៌មានលម្អិតរបស់អ្នកប្រើ។ ការពិចារណាអំពីរយៈពេលរក្សាទុកព័ត៌មាន ដូចជាការណែនាំគោលការណ៍រក្សាទុក អាចលំបាកចមေရការស្នាក់នៅក្នុងបរិបទតបស្នាក់ការពារការស៊ីវិលបច្ចេកទេស។
- **ផ្ទាល់ខ្លួន**: ជាមួយសមត្ថភាពរៀន និងអាស្រ័យម៉ូដែល AI ផ្តល់បទពិសោធន៍ផ្ទាល់ខ្លួនឲ្យអ្នកប្រើ។ ការប្តូរបទពិសោធន៍តាមបទលក្ខណ៍ដូចជាគណនីអ្នកប្រើមិនត្រឹមតែធ្វើឲ្យអ្នកប្រើមានអារម្មណ៍ថាត្រូវបានយល់ដឹងទេ ប៉ុន្តែគេហទំព័រក៏ជួយអ្នកក្នុងការស្វែងរកចម្លើយជាក់លាក់ កើតមានការប្រាស្រ័យទាក់ទងមានប្រសិទ្ធភាព និងពេញចិត្ត។

- **មុខងារ​សម្រាប់​ជម្ងឺ​ផ្លូវចិត្ដ**: ជម្រើសភាសាដែលបានសម្រួល។

សូមសន្និដ្ឋានកម្មវិធីជជែកមួយដែលយល់ពាក្យជំនួញរបស់ក្រុមហ៊ុនអ្នក និងរំពឹងទុកសំណួរពិសេសដែលអ្នកប្រើប្រាស់ភាគច្រើនមាន។ មានវិធីសាស្ត្រពីរផ្លូវដែលគួរអោយចាប់អារម្មណ៍ដូចជា៖

- **ប្រើម៉ូដែល DSL**។ DSL មានន័យថា ភាសាចំរុះជាក់លាក់។ អ្នកអាចប្រើម៉ូដែល DSL ដែលបានបណ្តុះបណ្តាលលើដែនជាក់លាក់ដើម្បីយល់ពីគន្លងនិងស្ថានការណ៍របស់វា។
- **អនុវត្តបង្រួម**។ បង្រួមគឺជាដំណើរការបណ្តុះបណ្តាលបន្ថែមម៉ូដែលរបស់អ្នកជាមួយទិន្នន័យជាក់លាក់។

## ការប្តូរតាមបំណង៖ ប្រើ DSL

ការប្រើម៉ូដែលភាសាដែនជាក់លាក់ (DSL Models) អាចបង្កើនការចូលរួមរបស់អ្នកប្រើដោយផ្តល់នូវការប្រាស្រ័យដែលជាការកំណត់ជាពិសេសនិងសមស្របជាមួយបរិបទ។ វាជាម៉ូដែលដែលបានបណ្តុះបណ្តាល ឬបង្រួបបង្រួមដើម្បីយល់ និងបង្កើតអត្ថបទដែលពាក់ព័ន្ធនឹងដែនជាក់លាក់ ឧស្សាហកម្ម ឬប្រធានបទ។ ចម្លែកនៃការប្រើម៉ូដែល DSL អាចខុសប្លែកចាប់ពីបណ្តុះបណ្តាលថ្មីទាំងមូល ដល់ការប្រើវាដើម្បីបាន SDKs និង APIs មួយទៀតគឺបង្រួម ដែលពាក់ព័ន្ធនឹងការយកម៉ូដែលដែលបានបណ្តុះបណ្តាលរួចមកកែប្រែក្នុងដែនជាក់លាក់។

## ការប្តូរតាមបំណង៖ អនុវត្តបង្រួម

បង្រួមតែងតែត្រូវបានគិតពេលម៉ូដែលដែលបានបណ្តុះបណ្តាលមុនមិនគ្រប់គ្រាន់ក្នុងដែនជាក់លាក់ ឬភារកិច្ចជាក់លាក់។

ឧទាហរណ៍ សំណួរពេទ្យមានសំណុំបែបបទស្មុគស្មាញ ហើយត្រូវការបរិបទច្រើន។ ពេលអ្នកជំនាញវេជ្ជសាស្ត្រធ្វើការពិនិត្យជម្ងឺ មូលហេតុគឺសម្រាប់ប៉ារ៉ាម៉ែត្រច្រើនដូចជារបៀបរស់នៅឬលក្ខខណ្ឌមានស្រាប់ ហើយវាអាចប្រើផងអត្ថបទពេលថ្មីៗក្នុងវិទ្យាសាស្ត្រសម្រាប់បញ្ជាក់ការវិភាគរបស់គាត់។ ក្នុងករណីចំរូងនេះ កម្មវិធីជជែក AI ទូទៅមិនអាចជាទ្រព្យសម្បត្តិត្រូវបានទុកចិត្តបានទេ។

### តំបន់វិភាគ៖ កម្មវិធីវេជ្ជសាស្ត្រ

សូមពិចារណាកម្មវិធីជជែកមួយដែលមានគោលបំណងជួយអ្នកជំនាញវេជ្ជសាស្ត្រដោយផ្តល់ការពន្យល់រហ័សរហួនពីគោលការណ៍ព្យាបាល ប្រតិកម្មថ្នាំ ឬការស្រាវជ្រាវថ្មីៗ។

ម៉ូដែលទូទៅអាចគ្រប់គ្រងបានសំណួរបែបឯកជន ឬផ្តល់ការណែនាំទូទៅ ប៉ុន្តែវាអាចប្រឈមមុខនឹង៖

- **ករណីជាក់លាក់ ឬស្មុគស្មាញខ្ពស់**។ ឧទាហរណ៍ វេជ្ជបណ្ឌិតខួរក្បាលអាចសំណួរថា "តើវិធីសាស្ត្រល្អបំផុតសម្រាប់គ្រប់គ្រងជំងឺធ្វើឲ្យវិលន្តសរសៃប្រសាទដែលធន់បំពង់ថ្នាំក្នុងកុមារមានអ្វីខ្លះ?"
- **ខ្វះនូវភាពទំនើបថ្មី**។ ម៉ូដែលទូទៅអាចជួបបញ្ហាគ្មានចម្លើយបច្ចុប្បន្នដែលរួមបញ្ចូលការអភិវឌ្ឍថ្មីៗក្នុងវិទ្យាសាស្ត្រខួរក្បាលនិងថ្នាំពេទ្យ។

ក្នុងករណីទាំងនេះ ការបង្រួមម៉ូដែលជាមួយនឹងទិន្នន័យវេជ្ជសាស្ត្រផ្ទាល់ខ្លួនអាចបង្កើនសមត្ថភាពដោះស្រាយសំណួរលំបាកទាំងនេះបានយ៉ាងច្បាស់លាស់ និងទុកចិត្តបាន។ នេះត្រូវការតម្រូវការចូលដំណើរការទិន្នន័យធំ និងពាក់ព័ន្ធ ដែលតំណាងឱ្យបញ្ហា និងសំណួរដែលត្រូវដោះស្រាយក្នុងដែនជាក់លាក់។
