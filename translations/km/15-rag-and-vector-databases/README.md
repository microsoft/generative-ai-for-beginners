# ការផលិតបន្ថែមដោយការសម្លឹងរក (RAG) និងទិន្នន័យវ៉ិចទ័រ

[![ការផលិតបន្ថែមដោយការសម្លឹងរក (RAG) និងទិន្នន័យវ៉ិចទ័រ](../../../translated_images/km/15-lesson-banner.ac49e59506175d4f.webp)](https://youtu.be/4l8zhHUBeyI?si=BmvDmL1fnHtgQYkL)

ក្នុងមេរៀនកម្មវិធីស្វែងរក យើងបានរៀនខ្លីពីរបៀបបញ្ចូលទិន្នន័យផ្ទាល់ខ្លួនទៅក្នុងគំរូភាសាធំ (LLMs)។ ក្នុងមេរៀននេះ យើងនឹងជ្រាបជាបូកបន្ថែមអំពីគំនិតនៃការដាក់ទិន្នន័យរបស់អ្នកក្នុងកម្មវិធី LLM របស់អ្នក ការប្រតិបត្តិការនៃដំណើរការ និងវិធីសាស្រ្តសម្រាប់រក្សាទុកទិន្នន័យ រួមទាំង embeddings និងអត្ថបទ។

> **វីដេអូអនាគតមកដល់**

## ការណែនាំ

ក្នុងមេរៀននេះ យើងនឹងគ្របដណ្តប់ដូចខាងក្រោម:

- ការណែនាំអំពី RAG តើវាជាអ្វី និងហេតុអ្វីបានជាវាត្រូវបានប្រើនៅក្នុង AI (បញ្ញាសិប្បនិម្មិត)។

- ការយល់ដឹងអំពីអ្វីទៅជាទិន្នន័យវ៉ិចទ័រ និងបង្កើតមួយសម្រាប់កម្មវិធីរបស់យើង។

- ឧទាហរណ៍ក្នុងអនុវត្តន៍ពីរបៀបបញ្ចូល RAG ទៅក្នុងកម្មវិធីមួយ។

## គោលបំណងការសិក្សា

បន្ទាប់ពីបញ្ចប់មេរៀននេះ អ្នកនឹងអាច:

- ពន្យល់សារៈសំខាន់នៃ RAG ក្នុងការស្វែងរក និងដំណើរការទិន្នន័យ។

- រៀបចំកម្មវិធី RAG និងដាក់ទិន្នន័យរបស់អ្នកទៅកាន់ LLM

- ការបញ្ចូល RAG និងទិន្នន័យវ៉ិចទ័រយ៉ាងមានប្រសិទ្ធភាពនៅក្នុងកម្មវិធី LLM។

## សន្តិសុខរបស់យើង៖ ការកែលម្អ LLMs របស់យើងជាមួយទិន្នន័យផ្ទាល់ខ្លួន

សម្រាប់មេរៀននេះ យើងចង់បន្ថែមកំណត់សម្គាល់ផ្ទាល់ខ្លួនទៅក្នុងសហគ្រាសអប់រំ ដែលអនុញ្ញាតឱ្យ chatbot ទទួលបានព័ត៌មានបន្ថែមអំពីមុខវិជ្ជាជាច្រើន។ ដោយប្រើកំណត់សម្គាល់ដែលយើងមាន អ្នករៀននឹងអាចសិក្សាបានល្អប្រសើរឡើង និងយល់ដឹងពីប្រធានបទខុសៗគ្នា ដែលធ្វើឱ្យមានភាពងាយស្រួលក្នុងការត្រៀមសម្រាប់ការប្រឡងរបស់ពួកគេ។ ដើម្បីបង្កើតសន្តិសុខរបស់យើង យើងនឹងប្រើ:

- `Azure OpenAI:` គំរូ LLM ដែលយើងនឹងប្រើសម្រាប់បង្កើត chatbot របស់យើង

- `មេរៀន AI សម្រាប់អ្នកចាប់ផ្តើមលើ បណ្ដាញប្រព័ន្ធប្រសិទ្ធិ`: នេះគឺជាទិន្នន័យដែលយើងដាក់ជាមូលដ្ឋានទៅលើ LLM របស់យើង

- `Azure AI Search` និង `Azure Cosmos DB:` ឃ្លាំងទិន្នន័យវ៉ិចទ័រសម្រាប់រក្សាទុកទិន្នន័យ និងបង្កើតសន្ទស្សន៍ស្វែងរក

អ្នកប្រើប្រាស់នឹងអាចបង្កើតវគ្គសំណួរប្រដាប់សម្រាប់ហាត់ពីកំណត់សម្គាល់របស់ពួកគេ លើកកាតត្រង់ការត្រៀម និងសង្ខេបវាទៅជាសង្ខេបខ្លីៗ។ ដើម្បីចាប់ផ្តើម សូមមើលថា RAG មានអ្វី និងធ្វើដូចម្តេច៖

## ការផលិតបន្ថែមដោយការសម្លឹងរក (RAG)

Chatbot ដែលបើកដំណើរការដោយ LLM ដំណើរការសំណើអ្នកប្រើដើម្បីបង្កើតចម្លើយ។ វាត្រូវបានរៀបចំឲ្យមានអន្តរកម្ម និងពាក់ព័ន្ធជាមួយអ្នកប្រើក្នុងប្រធានបទជាច្រើន។ ទោះជាយ៉ាងណា ចម្លើយរបស់វាត្រូវបានកំណត់ដោយបរិបទដែលផ្តល់ និងទិន្នន័យបណ្តាញបង្រៀនគំរូ។ ឧទាហរណ៍ GPT-4 មានកាលបរិច្ឆេទចំណុចដាក់ចំណេះដឹងនៅខែកញ្ញា ២០២១ ប្រាប់ថាវាពុំមានចំណេះដឹងអំពីព្រឹត្តិការណ៍ក្រោយពេលនោះ។ លើសពីនេះទៀត ទិន្នន័យដែលប្រើសម្រាប់បង្រៀន LLM មិនរួមមានព័ត៌មានទាក់ទងនឹងឯកជនដូចជា កំណត់សម្គាល់ផ្ទាល់ខ្លួន ឬមុខងារផលិតផលក្រុមហ៊ុន។

### របៀបដែល RAG (ការផលិតបន្ថែមដោយការសម្លឹងរក) ធ្វើការ

![គំនូរសំដៅរបៀបដែល RAGs ធ្វើការ](../../../translated_images/km/how-rag-works.f5d0ff63942bd3a6.webp)

សន្មត់ថាអ្នកចង់ដាក់បញ្ចូល chatbot ដែលបង្កើតវគ្គសំណួរពិចារណាពីកំណត់សម្គាល់របស់អ្នក អ្នកនឹងត្រូវការតំណភ្ជាប់ទៅឃ្លាំងចំណេះដឹង។ នេះជាគន្លងដែល RAG ជួយសង្គ្រោះ។ RAGs គឺដំណើរការតាមរយៈ៖

- **មូលដ្ឋានចំណេះដឹង:** មុនការសម្លឹងរក ត្រូវតែដាក់ទិន្នន័យនេះចូល និងដំណើរការមុន សាមញ្ញតែផែកឯកសារធំៗជាប្លុកតូចៗ បម្លែងជារូបភាពអត្ថបទembedding ហើយរក្សាទុកក្នុងឃ្លាំងទិន្នន័យ។

- **សំណួរអ្នកប្រើ:** អ្នកប្រើសួរសំណួរមួយ

- **ការសម្លឹងរក:** ពេលអ្នកប្រើសួរសំណួរ គំរូembedding នឹងយកព័ត៌មានពាក់ពន្ធន៍ពីមូលដ្ឋានចំណេះដឹងរបស់យើង ដើម្បីផ្តល់បរិបទបន្ថែមដូចជាការបញ្ចូលក្នុងសំណើ។

- **ការផលិតបន្ថែម:** LLM will enhance responses based on retrieved data. It enables the answer to be based not only on pre-trained information but also relevant information from contextual input. The retrieved data is used to augment the LLM's answers. The LLM then returns the answer to the user’s question.

![គំនូរបង្ហាញរចនាសម្ព័ន្ធរបស់ RAGs](../../../translated_images/km/encoder-decode.f2658c25d0eadee2.webp)

រចនាសម្ព័ន្ធសម្រាប់ RAGs ត្រូវបានអនុវត្តដោយប្រើ transformers ដែលមានពីរផ្នែក៖ encoder និង decoder។ ឧទាហរណ៍ ពេលអ្នកប្រើសួរសំណួរ អត្ថបទបញ្ចូលត្រូវបាន 'encode' ទៅវ៉ិចទ័រសម្រាប់ការបកន័យន័យនៃពាក្យ ហើយវ៉ិចទ័រនឹងត្រូវបាន 'decode' ទៅក្នុងសន្ទស្សន៍ឯកសារយើង ហើយបង្កើតអត្ថបទថ្មីមួយដោយផ្អែកលើសំណួរអ្នកប្រើ។ LLM ប្រើទាំង encoder-decoder គំរូសម្រាប់បង្កើតលទ្ធផលចេញ។

ចំនួនពីរប្រភេទនៃការអនុវត្ត RAG អាស្រ័យលើឯកសារដែលបានផ្ដល់៖ [Retrieval-Augmented Generation for Knowledge intensive NLP (natural language processing software) Tasks](https://arxiv.org/pdf/2005.11401.pdf?WT.mc_id=academic-105485-koreyst) មាន៖

- **_RAG-Sequence_** ប្រើឯកសារដែលបានយកមកវិញ ដើម្បីព្យាករណ៍ចម្លើយល្អបំផុតចំពោះសំណួរអ្នកប្រើ

- **RAG-Token** ប្រើឯកសារដើម្បីបង្កើត token បន្ទាប់ ហើយបន្ទាប់មកយកវាឡើងវិញដើម្បីឆ្លើយសំណួរអ្នកប្រើ

### ហេតុអ្វីអ្នកត្រូវប្រើ RAGs?

- **ភាពភ្អាក់ភ័ណ្ឌនៃព័ត៌មាន:** ធានាចម្លើយអត្ថបទមានភាពទាន់សម័យ និងសព្វថ្ងៃ។ ដូច្នេះ វាបង្កើនការអនុវត្តលើភារកិច្ចជាក់លាក់ដោយចូលដើមទៅមូលដ្ឋានចំណេះដឹងខាងក្នុង។

- កាត់បន្ថយការបង្កើតព័ត៌មានមិនពិតដោយប្រើ **ទិន្នន័យអាចផ្ទៀងផ្ទាត់** ក្នុងមូលដ្ឋានចំណេះដឹង ដើម្បីផ្តល់បរិបទចំពោះសំណួរអ្នកប្រើ។

- វាមាន **ថ្លៃថោក** ព្រោះវាសេវាបានថោកជាងការប្រើប្រាស់ fine-tune LLM

## ការបង្កើតមូលដ្ឋានចំណេះដឹង

កម្មវិធីរបស់យើងផ្អែកលើទិន្នន័យផ្ទាល់ខ្លួន គឺមេរៀនបណ្ដាញប្រសិទ្ធិលើ AI For Beginners។

### ទិន្នន័យវ៉ិចទ័រ

ទិន្នន័យវ៉ិចទ័រ គឺជាឃ្លាំងទិន្នន័យដែលមានចំណុចឯកទេសសម្រាប់រក្សាទុក គ្រប់គ្រង និងស្វែងរកវ៉ិចទ័របញ្ជាក់ទំហំ។ វារក្សាទុកតំណាងគណនាសម្រាប់ឯកសារ។ ការបំបែកទិន្នន័យទៅអត្ថបទembedding ធ្វើឱ្យប្រព័ន្ធ AI របស់យើងមានភាពងាយស្រួលក្នុងការយល់និងដំណើរការទិន្នន័យ។

### ពីអត្ថបទទៅ embeddings

មុនពេលយើងរក្សាទុកទិន្នន័យយើង ត្រូវបម្លែងវាទៅជាអត្ថបទ embedding មុនពេលវាត្រូវបានគេរក្សាទុកក្នុងឃ្លាំងទិន្នន័យ។ ប្រសិនបើអ្នកកំពុងបំលែងឯកសារធំ ឬអត្ថបទវែង អ្នកអាចបំបែកវាតាមសំណួរដែលអ្នករំពឹងទុក។ ការបំបែកអាចធ្វើនៅកម្រិតប្រយោគ ឬកម្រិតកថាខណ្ឌ។ ដោយសារការបំបែកប្លុកផ្អែកលើន័យនៃពាក្យ ជុំវិញ ពួកអ្នកអាចបន្ថែមបរិបទផ្សេងទៅកាន់ប្លុកមួយ ដូចជាការបន្ថែមចំណងជើងឯកសារ ឬរាប់បញ្ចូលអត្ថបទមុន ឬក្រោយប្លុក។ អ្នកអាចបំបែកទិន្នន័យដូចខាងក្រោម៖

```python
def split_text(text, max_length, min_length):
    words = text.split()
    chunks = []
    current_chunk = []

    for word in words:
        current_chunk.append(word)
        if len(' '.join(current_chunk)) < max_length and len(' '.join(current_chunk)) > min_length:
            chunks.append(' '.join(current_chunk))
            current_chunk = []

    # បើផ្នែកចុងក្រោយមិនបានឈានដល់ប្រវែងអប្បបរមា ក៏បញ្ចូលវាទៅម្តង
    if current_chunk:
        chunks.append(' '.join(current_chunk))

    return chunks
```

បន្ទាប់ពីបំបែកប្លុករួច យើងអាច embed អត្ថបទដោយប្រើម៉ូដែល embedding ផ្សេងៗ។ ម៉ូដែលខ្លះអ្នកអាចប្រើរួមមាន: word2vec, ada-002 របស់ OpenAI, Azure Computer Vision និងអ្នកផ្សេងទៀត។ ការជ្រើសរើសម៉ូដែលប្រើនឹងអាស្រ័យលើភាសាដែលអ្នកប្រើ ប្រភេទខ្លឹមសារ (អត្ថបទ/រូបភាព/សំឡេង) ទំហំបញ្ចូលដែលវាអាចបញ្ចូល និងប្រវែងបញ្ចូល។

### សភាពស្រដៀងវ៉ិចទ័រ

Retriever នឹងស្វែងរកតាមគេហទំព័រចំណេះដឹងសម្រាប់ embeddings ដែលនៅជិតគ្នា ជាមិត្តជិតស្និទ្ធ ព្រោះពួកវាជាអត្ថបទស្រដៀងគ្នា។ នៅក្នុងឧទាហរណ៍ដែលអ្នកប្រើសួរសំណួរ វាត្រូវបាន embedded ជាមុនហើយផ្គូរផ្គងជាមួយ embeddings ស្រដៀងគ្នា។ ការវាស់វែងធម្មតាសម្រាប់រកភាពស្រដៀងគ្នារវាងវ៉ិចទ័រផ្សេងគឺ cosine similarity ដែលផ្អែកលើមុំរវាងវ៉ិចទ័រ២។

យើងអាចវាស់វែងភាពស្រដៀងគ្នាផងដែរ ដោយប្រើវិធីជំរើសផ្សេងទៀត ដូចជា Euclidean distance ដែលជាសំណុំបន្ទាត់ត្រង់រវាងចំណុចចុងវ៉ិចទ័រនិង dot product ដែលវាស់សរុបផលគុណនៃធាតុបង្ករគ្នារបស់វ៉ិចទ័រពីរប្រភេទ។

### សន្ទស្សន៍ស្វែងរក

ពេលធ្វើការសម្លឹងរក យើងត្រូវតែបង្កើតសន្ទស្សន៍ស្វែងរកសម្រាប់មូលដ្ឋានចំណេះដឹងរបស់យើង មុនពេលធ្វើការស្វែងរក។ សន្ទស្សន៍នឹងរក្សាទុក embeddings របស់យើង ហើយអាចស្វែងរកប្លុកស្រដៀងគ្នាជាប់គ្នាបានយ៉ាងលឿន ទម្រាំច្រើនក្នុងធនាគារធំ។ យើងអាចបង្កើតសន្ទស្សន៍ក្នុងកន្លែងរបស់យើងដោយប្រើ៖

```python
from sklearn.neighbors import NearestNeighbors

embeddings = flattened_df['embeddings'].to_list()

# បង្កើតប្រមូលបណ្ដាញស្វែងរក
nbrs = NearestNeighbors(n_neighbors=5, algorithm='ball_tree').fit(embeddings)

# ដើម្បី​ស្វែងរកក្នុងប្រមូលបណ្ដាញ អ្នកអាចប្រើវិធី kneighbors
distances, indices = nbrs.kneighbors(embeddings)
```

ជំហានចុងក្រោយគឺបញ្ចូល LLM របស់យើងចូលជាមួយដើម្បីអាចទទួលបានចម្លើយដែលជារដ្ឋានលើទិន្នន័យរបស់យើង។ យើងអាចអនុវត្តវាដូចខាងក្រោម៖

```python
user_input = "what is a perceptron?"

def chatbot(user_input):
    # បម្លែងសំនួរទៅជាវ៉ែចទ័រ​សាកសម
    query_vector = create_embeddings(user_input)

    # ស្វែងរកឯកសារដែលស្រដៀងគ្នามากที่สุด
    distances, indices = nbrs.kneighbors([query_vector])

    # បន្ថែមឯកសារទៅកាន់សំណួរដើម្បីផ្តល់បរិបទ
    history = []
    for index in indices[0]:
        history.append(flattened_df['chunks'].iloc[index])

    # បញ្ចូលប្រវត្តិ និងការបញ្ចូលរបស់អ្នកប្រើ
    history.append(user_input)

    # បង្កើតអObjetសារ
    messages=[
        {"role": "system", "content": "You are an AI assistant that helps with AI questions."},
        {"role": "user", "content": "\n\n".join(history) }
    ]

    # ប្រើប្រាស់ API Responses ដើម្បីបង្កើតចម្លើយ
    response = client.responses.create(
        model="gpt-4o-mini",
        temperature=0.7,
        max_output_tokens=800,
        input=messages,
        store=False,
    )

    return response.output_text

chatbot(user_input)
```

## វាយតម្លៃកម្មវិធីរបស់យើង

### គន្លងវាយតម្លៃ

- គុណភាពចម្លើយផ្តល់ ត្រូវធានាថារសម្រួល មានភាពរលូន និងស្រដៀងមនុស្ស

- ការដាក់សម្រាប់ទិន្នន័យ: វាយតម្លៃថាចម្លើយដែលបានផ្តល់មកមានមូលដ្ឋានពីឯកសារផ្ដល់ឬអត់

- ភាពពាក់ព័ន្ធ: វាយតម្លៃថាចម្លើយផ្គូរផ្គងនិងទាក់ទងទៅនឹងសំណួរដែលបានសួរ

- ភាពរលូន - តើចម្លើយមានភាពនិយមន័យត្រឹមត្រូវបែបវេយ្យាករណ៍ឬអត់

## ករណីប្រើសម្រាប់ការប្រើ RAG (ការផលិតបន្ថែមដោយការសម្លឹងរក) និងទិន្នន័យវ៉ិចទ័រ

មានករណីប្រើជាច្រើនដែលការហៅមុខងារអាចធ្វើឱ្យកម្មវិធីរបស់អ្នកប្រសើរឡើងដូចជា៖

- សំណួរនិងចម្លើយ៖ ដាក់ទិន្នន័យក្រុមហ៊ុនរបស់អ្នកក្នុង chatbot ដែលអាចប្រើដោយបុគ្គលិកសម្រាប់សួរចម្លើយ។

- ប្រព័ន្ធផ្តល់អនុសាសន៍៖ ដែលអ្នកអាចបង្កើតប្រព័ន្ធផ្គូរផ្គងតម្លៃស្រដៀងគ្នាបំផុត ដូចជា រឿងភាគ ភោជនីយដ្ឋាន និងច្រើនទៀត។

- សេវាកម្ម chatbot៖ អ្នកអាចរក្សាទុកប្រវត្តិការជជែក និងផ្ទាល់ខ្លួនសន្ទនាជូនទៅតាមទិន្នន័យអ្នកប្រើ។

- ការស្វែងរករូបភាពដោយផ្អែកលើ embedding វ៉ិចទ័រ នៅពេលធ្វើការទទួលស្គាល់រូបភាព និងការរកឃើញករណីខុសប្រក្រតី។

## សង្ខេប

យើងបានគ្របដណ្តប់តំបន់មូលដ្ឋានរបស់ RAG ពីការបន្ថែមទិន្នន័យទៅកម្មវិធី សំណួរអ្នកប្រើ និងលទ្ធផលចេញ។ ដើម្បីធ្វើឱ្យការបង្កើត RAG សាមញ្ញ អ្នកអាចប្រើ Frameworks ដូចជា Semanti Kernel, Langchain ឬ Autogen។

## កិច្ចការ

ដើម្បីបន្តការសិក្សារបស់អ្នកអំពីការផលិតបន្ថែមដោយការសម្លឹងរក (RAG) អ្នកអាចសាងសង់៖

- បង្កើតផ្នែកមុខសម្រាប់កម្មវិធីដោយប្រើ Framework មួយដែលអ្នកចូលចិត្ត

- ប្រើ Framework មួយ ចំនួន LangChain ឬ Semantic Kernel ដើម្បីបង្កើតកម្មវិធីរបស់អ្នកឡើងវិញ។
