# ការបង្កើតបង្កើនការស្វែងរក (RAG) និងមូលដ្ឋានទិន្នន័យទិសវ៉ិចទ័រ

[![ការបង្កើតបង្កើនការស្វែងរក (RAG) និងមូលដ្ឋានទិន្នន័យទិសវ៉ិចទ័រ](../../../translated_images/km/15-lesson-banner.ac49e59506175d4f.webp)](https://youtu.be/4l8zhHUBeyI?si=BmvDmL1fnHtgQYkL)

ក្នុងមេរៀនកម្មវិធីស្វែងរក យើងបានរៀនខ្លីៗពីរបៀបបញ្ចូលទិន្នន័យផ្ទាល់ខ្លួនរបស់អ្នកទៅក្នុងគំរូភាសាធំនៅ (LLMs)។ ក្នុងមេរៀននេះ យើងនឹងពង្រីកជ្រាបច្រើនទៀតលើគំនិតនៃការជម្លែងទិន្នន័យរបស់អ្នកនៅក្នុងកម្មវិធី LLM របស់អ្នក គ្រឿងចក្រ​ដំណើរការ និងវិធីសាស្រ្តសម្រាប់រក្សាទុកទិន្នន័យ រួមមានទាំងការបញ្ចូលនិងអត្ថបទ។

> **វីដេអូឆាប់ៗនេះ**

## ការណែនាំ

ក្នុងមេរៀននេះ យើងនឹងគ្របដណ្តប់ដល់៖

- ការណែនាំអំពី RAG វាជាអ្វី និងហេតុអ្វីបានជាវាត្រូវបានប្រើនៅក្នុង AI (បញ្ញាសិប្បនិម្មិត)។

- បង្រៀនពីមូលដ្ឋានទិន្នន័យទិសវ៉ិចទ័រនិងបង្កើតមួយសម្រាប់កម្មវិធីរបស់យើង។

- ឧទាហរណ៍អន្ដរជាតិក្នុងការបញ្ចូល RAG ទៅក្នុងកម្មវិធីមួយ។

## គោលបំណងរៀន

បន្ទាប់ពីបញ្ចប់មេរៀននេះ អ្នកនឹងអាច:

- ពន្យល់អំពីសារៈសំខាន់នៃ RAG ក្នុងការស្វែងរកនិងដំណើរការទិន្នន័យ។

- រៀបចំកម្មវិធី RAG និងភ្ជាប់ទិន្នន័យរបស់អ្នកទៅឱ្យ LLM

- បញ្ចូលប្រសើរនៃ RAG និងមូលដ្ឋានទិន្នន័យទិសវ៉ិចទ័រនៅក្នុងកម្មវិធី LLM។

## ស្ថានភាពរបស់យើង៖ បង្កើត LLM របស់យើងជាមួយទិន្នន័យផ្ទាល់ខ្លួន

សម្រាប់មេរៀននេះ យើងចង់បន្ថែមកំណត់ត្រាផ្ទាល់ខ្លួនចូលទៅក្នុងស្តាតអាប់អប់រំ ដែលអនុញ្ញាតឱ្យ chatbot ទទួលបានព័ត៌មានបន្ថែមអំពីមុខវិជ្ជាផ្សេងៗ។ ដោយប្រើកំណត់ត្រាដែលយើងមាន អ្នករៀនអាចសិក្សាបានល្អប្រសើរនិងយល់ដឹងអំពីប្រធានបទផ្សេងៗ ដែលធ្វើឱ្យងាយស្រួលក្នុងការត្អូញត្អែរ សម្រាប់ការប្រឡងរបស់ពួកគេ។ ដើម្បីបង្កើតស្ថានភាពនេះ យើងនឹងប្រើ៖

- `Azure OpenAI:` LLM ដែលយើងប្រើសម្រាប់បង្កើត chatbot របស់យើង

- `មេរៀន AI សម្រាប់អ្នកចាប់ផ្ដើមនៅលើបណ្តាញប្រព័ន្ធប្រសាសន៍ខួរក្បាល`: នេះគឺជាទិន្នន័យដែលយើងជម្លែង LLM របស់យើង

- `Azure AI Search` និង `Azure Cosmos DB:` មូលដ្ឋានទិន្នន័យទិសវ៉ិចទ័រដើម្បីរក្សាទុកទិន្នន័យនិងបង្កើតសន្ទស្សន៍ស្វែងរក

អ្នកប្រើប្រាស់នឹងអាចបង្កើតសំណួរប្រូចត្រូវពីកំណត់ត្រារបស់ពួកគេ បង្កើតកាតចងចាំសង្ខេប និងសង្ខេបវាទៅជាសង្ខេបខ្លី។ ដើម្បីចាប់ផ្តើម យើងសូមមើលថា RAG ជាអ្វី និងវាដំណើរការយ៉ាងដូចម្តេច៖

## ការបង្កើតបង្កើនការស្វែងរក (RAG)

Chatbot ដែលមាន LLM ជាថាមពលដំណើរការតាមសំណើរបស់អ្នកប្រើដើម្បីបង្កើតចម្លើយ។ វាត្រូវបានរចនាឡើងដើម្បីអាចអន្តរកម្ម និងចូលរួមជាមួយអ្នកប្រើនៅលើប្រធានបទជាច្រើន។ ទោះយ៉ាងណា ចម្លើយរបស់វាត្រូវបានដាក់កំណត់ទៅតាមបរិបទដែលផ្តល់ដោយវានិងទិន្នន័យបណ្តុះបណ្តាលមូលដ្ឋានរបស់វា។ ឧទាហរណ៍ GPT-4 មានកំណត់ចំណេះដឹងដល់ខែកញ្ញា ២០២១ វាមានអត្ថន័យថា វាមិនមានចំណេះដឹងអំពីព្រឹត្តិការណ៍ដែលបានកើតមានក្រោយពីរយៈពេលនេះទេ។ លើសពីនេះទៀត ទិន្នន័យដែលបានប្រើសម្រាប់បណ្តុះបណ្តាល LLM មិនរួមបញ្ចូលព័ត៌មានរដ្ឋបាលដូចជាកំណត់ត្រាផ្ទាល់ខ្លួនឬមេនុយផលិតផលរបស់ក្រុមហ៊ុនឡើយ។

### របៀបនៃ RAGs (ការបង្កើតបង្កើនការស្វែងរក)

![សៀងបង្ហាញរបៀបដំណើរការរបស់ RAGs](../../../translated_images/km/how-rag-works.f5d0ff63942bd3a6.webp)

សូមសន្និដ្ឋានថាអ្នកចង់ដាក់ឱ្យប្រើ chatbot ដែលបង្កើតសំណួរប្រាជ្ញាពីកំណត់ត្រារបស់អ្នក អ្នកនឹងត្រូវការ​ភ្ជាប់ទៅមូលដ្ឋានចំណេះដឹង។ វាជាកន្លែងដែល RAG ចូលរួមជួយ។ RAGs ដំណើរការដូចខាងក្រោម៖

- **មូលដ្ឋានចំណេះដឹង:** មុនការស្វែងរក ឯកសារទាំងនេះត្រូវបានទទួលចូលនិងដំណើរការ ការបំលែងឯកសារទំហំធំទៅជាផ្នែកតូចៗ និងបំលែងជាផ្នែកបញ្ចូលអត្ថបទ​ និងរក្សាទុកក្នុងមូលដ្ឋានទិន្នន័យ។

- **សំណើរអ្នកប្រើ:** អ្នកប្រើសួរសំណួរ

- **ការស្វែងរក:** នៅពេលដែលអ្នកប្រើសួរសំណួរ ម៉ូឌែលបញ្ចូលអត្ថបទនឹងយកព័ត៌មានពាក់ព័ន្ធពីមូលដ្ឋានចំណេះដឹងរបស់យើង ដើម្បីផ្តល់បរិបទបន្ថែមដែលនឹងបញ្ចូលទៅក្នុងសំណើ។

- **ការបង្កើតបន្ថែម:** LLM ពង្រឹងចម្លើយរបស់វាដោយផ្អែកលើទិន្នន័យដែលបានយកមក។ វាអនុញ្ញាតឱ្យចម្លើយដែលបង្កើតមិនត្រឹមតែផ្អែកលើទិន្នន័យដែលបានបណ្តុះបណ្តាលមុនទេរួមបញ្ចូលព័ត៌មានពាក់ព័ន្ធពីបរិបទបន្ថែម។ ទិន្នន័យដែលបានយកមកនេះត្រូវបានប្រើសម្រាប់បង្កើនចម្លើយរបស់ LLM។ LLM បន្ទាប់មកនឹងផ្តល់ចម្លើយទៅសំណួរអ្នកប្រើ។

![សៀងបង្ហាញសំណង់របស់ RAG](../../../translated_images/km/encoder-decode.f2658c25d0eadee2.webp)

សំណង់សម្រាប់ RAGs ត្រូវបានអនុវត្តដោយប្រើ transformers ដែលរួមមានពីភាគ៖ encoder និង decoder។ ឧទាហរណ៍ នៅពេលដែលអ្នកប្រើសួរសំណួរ អត្ថបទបញ្ចូលត្រូវបាន 'encode' ទៅជាទិសវ៉ិចទ័រដែលចាប់អារម្មណ៍ន័យនៃពាក្យ ហើយទិសវ៉ិចទ័រត្រូវបាន 'decode' ទៅក្នុងសន្ទស្សន៍ឯកសាររបស់យើង ហើយបង្កើតអត្ថបទថ្មីផ្អែកលើសំណួរអ្នកប្រើ។ LLM ប្រើម៉ូឌែល encoder-decoder ទាំងពីរដើម្បីបង្កើតលទ្ធផល។

មានវិធីសាស្រ្តពីរក្នុងការអនុវត្ត RAG ទៅតាមអត្ថបទដែលបានផ្តល់៖ [Retrieval-Augmented Generation for Knowledge intensive NLP (natural language processing software) Tasks](https://arxiv.org/pdf/2005.11401.pdf?WT.mc_id=academic-105485-koreyst) គឺ:

- **_RAG-Sequence_** ប្រើឯកសារដែលបានយកមកផ្អែកដើម្បីប៉ាន់ស្មានចម្លើយល្អបំផុតសម្រាប់សំណួរអ្នកប្រើ

- **RAG-Token** ប្រើឯកសារដើម្បីបង្កើត token បន្ទាប់ បន្ទាប់មកយកពួកវាចេញសម្រាប់ឆ្លើយសំណួរអ្នកប្រើ

### ហេតុអ្វីបានជាអ្នកគួរប្រើ RAGs? 

- **ភាពសម្បូរបែបនៃព័ត៌មាន:** ប្រាកដថាចម្លើយអត្ថបទមានទាន់សម័យ និងទាន់ពេលវេលា។ ដូច្នេះ វាបង្កើតការកែទម្រង់លើការងារពិសេសតាមវិស័យដោយចូលប្រើមូលដ្ឋានចំណេះដឹងផ្ទៃក្នុង។

- កាត់បន្ថយការបង្កើតចម្លើយក្លែងក្លាយ តាមរយៈការប្រើ **ទិន្នន័យអាចបញ្ជាក់បាន** ក្នុងមូលដ្ឋានចំណេះដឹងដើម្បីផ្តល់បរិបទទៅសំណួរអ្នកប្រើ។

- វាជា **ការសន្សំចំណាយ** ព្រោះវាកាន់តែល្អសម្រាប់វេចខ្ចប់ជាមួយ LLM ទៅជាការតម្រឹមលំអិត។

### ពីអត្ថបទទៅជា embeddings

មុនពេលយើងរក្សាទុកទិន្នន័យយើង ត្រូវបម្លែងវាទៅជា vector embeddings មុនពេលវាត្រូវបានរក្សាទុកក្នុងមូលដ្ឋានទិន្នន័យ។ ប្រសិនបើអ្នកកំពុងដំណើរការ ឯកសារធំ ឬអត្ថបទវែង អ្នកអាចធ្វើការបំបែកវាចេញជាផ្នែកតាមសំណួរដែលអ្នករំពឹងថានឹងសួរជាមុន។ ការបំបែកអាចធ្វើនៅកម្រិតវេយ្យាករណ៍ ឬកម្រិតតួអត្ថបទ។ ព្រោះការបំបែក chunk យកន័យពីពាក្យជុំវិញ វាអាចបន្ថែមបរិបទផ្សេងៗទៅ chunk មួយ ដូចជា ការបញ្ចូលចំណងជើងឯកសារ ឬបញ្ចូលអត្ថបទមួយចំនួនមុន ឬបន្ទាប់ chunk។ អ្នកអាចបំបែកទិន្នន័យដូចជា៖

```python
def split_text(text, max_length, min_length):
    words = text.split()
    chunks = []
    current_chunk = []

    for word in words:
        current_chunk.append(word)
        if len(' '.join(current_chunk)) < max_length and len(' '.join(current_chunk)) > min_length:
            chunks.append(' '.join(current_chunk))
            current_chunk = []

    # ប្រសិនបើផ្នែកចុងក្រោយមិនបានដល់ប្រវែងដែលតិចបំផុត ក៏បន្ថែមវាទៅផង
    if current_chunk:
        chunks.append(' '.join(current_chunk))

    return chunks
```

បន្ទាប់ពីបានបំបែក chunk យើងអាចបញ្ចូលអត្ថបទរបស់យើងដោយប្រើម៉ូឌែល embedding ផ្សេងៗ។ ម៉ូឌែលខ្លះដែលអ្នកអាចប្រើរួមមាន៖ word2vec, ada-002 ដោយ OpenAI, Azure Computer Vision និងច្រើនទៀត។ ការជ្រើសរើសម៉ូឌែលប្រើប្រាស់នឹងអាស្រ័យទៅលើភាសាដែលអ្នកប្រើ ប្រភេទនៃមាតិកាដែលត្រូវ encode (អត្ថបទ/រូបភាព/សំឡេង) ទំហំបញ្ចូលដែលអាច encode ហើយប្រវែងលទ្ធផល embedding។

## ការស្វែងរកនិងស្វែងរកតាមទិសវ៉ិចទ័រ

នៅពេលដែលអ្នកប្រើសួរសំណួរ អ្នកយកពាក្យត្រូវបានបំលែងទៅជាទិសវ៉ិចទ័រដោយប្រើ query encoder បន្ទាប់មកវាស្វែងរកតាមសន្ទស្សន៍ឯកសាររបស់យើងសម្រាប់ទិសវ៉ិចទ័រដែលពាក់ព័ន្ធក្នុងឯកសារ។ បន្ទាប់ពីធ្វើរួច វាបំលែងទាំង input vector និង document vectors ទៅជាអត្ថបទ ហើយផ្ញើវាទៅ LLM។

### ការស្វែងរក

ការស្វែងរកកើតឡើងនៅពេលប្រព័ន្ធព្យាយាមរកឯកសារជាបន្ទាន់ពីសន្ទស្សន៍ដែលបំពេញលក្ខខណ្ឌស្វែងរក។ គោលបំណងនៃអ្នកយកព័ត៌មានគឺផ្តល់ឯកសារដែលអាចផ្តល់បរិបទ និងភ្ជាប់ LLM ទៅលើទិន្នន័យរបស់អ្នក។

មានវិធីជាច្រើនក្នុងការស្វែងរកនៅក្នុងមូលដ្ឋានទិន្នន័យរបស់យើងដូចជា៖

- **ស្វែងរកតាមពាក្យគន្លឹះ** - ប្រើសម្រាប់ស្វែងរកអត្ថបទ

- **ស្វែងរកតាមទិសវ៉ិចទ័រ** - បំលែងឯកសារពីអត្ថបទទៅជាតំណាងទិសវ៉ិចទ័រដោយប្រើម៉ូឌែល embedding ដែលអនុញ្ញាតឱ្យស្វែងរកអត្ថន័យតាមន័យនៃពាក្យ។ ការស្វែងរកនេះអាចអនុវត្តបានដោយសំណួរតាមឯកសារដែលមានតំណាងទិសវ៉ិចទ័រចម្ងាយជិតសំណួរអ្នកប្រើបំផុត។

- **ផ្សំគ្នា** - ការរួមបញ្ចូលចំណាស់នៃការស្វែងរកតាមពាក្យគន្លឹះ និងស្វែងរកតាមទិសវ៉ិចទ័រ។

បញ្ហាមួយនៅការស្វែងរកកើតមាននៅពេលមិនមានចម្លើយដូចគ្នាទៅនឹងសំណួរនៅក្នុងមូលដ្ឋានទិន្នន័យ ប្រព័ន្ធនឹងត្រលប់ព័ត៌មានល្អបំផុតដែលពួកគេអាចទទួលបាន ប៉ុន្តែអ្នកអាចប្រើយុទ្ធសាស្រ្តដូចជា កំណត់ចម្ងាយអតិបរមាសម្រាប់ការស្របគ្នារ ឬប្រើការស្វែងរកផ្សំច្រវ៉ាមរួមទាំងពាក្យគន្លឹះនិងស្វែងរកតាមទិសវ៉ិចទ័រ។ នៅក្នុងមេរៀននេះ យើងនឹងប្រើការស្វែងរកផ្សំច្រវ៉ាម ដែលជាការរួមបញ្ចូលនៃទិសវ៉ិចទ័រ និងពាក្យគន្លឹះ។ យើងនឹងរក្សាទុកទិន្នន័យរបស់យើងក្នុង dataframe ដែលមានជួរឈរទាំង្មាន chunk និង embedding ផងដែរ។

### ភាពស្រដៀងទិសវ៉ិចទ័រ

អ្នកយកព័ត៌មាននឹងស្វែងរកក្នុងមូលដ្ឋានចំណេះដឹងសម្រាប់ embedding ដែលនៅជិតក្បែរគ្នា ជាអ្នកជិតស្និទ្ធបំផុត ពីព្រោះវាជាអត្ថបទដែលស្រដៀងគ្នា។ ក្នុងស្ថានភាពនៃអ្នកប្រើសួរសំណួរ វាត្រូវបាន embed ជាមួយនឹង embedding ស្រដៀងគ្នា។ វិធីវាស់ស្ទង់ដ៏ទូទៅដែលប្រើសម្រាប់រកមើលភាពស្រដៀងរវាង vector ខុសគ្នាគឺ cosine similarity ដែលផ្អែកលើមុំរវាងទិសវ៉ិចទ័រក្នុងចំណោមពីររូប។

យើងអាចវាស់ភាពស្រដៀងជាមួយជម្រើសផ្សេងទៀតដូចជា Euclidean distance ដែលជាខ្សែស្រឡាយផ្ទាល់រវាងចំណុចប្រទាក់ vector និង dot product ដែលវាស់សរុបផលគុណនៃធាតុជាប់គ្នានៃ vector ទាំងពីរ។

### សន្ទស្សន៍ស្វែងរក

នៅពេលធ្វើការស្វែងរក យើងត្រូវការកសាងសន្ទស្សន៍ស្វែងរកសម្រាប់មូលដ្ឋានចំណេះដឹងរបស់យើង មុននឹងអនុវត្តការស្វែងរក។ សន្ទស្សន៍នឹងរក្សាទុក embedding របស់យើង ហើយអាចយក chunk ដែលស្រដៀងគ្នាជាច្រើនបានយ៉ាងលឿន ទោះបីនៅក្នុងមូលដ្ឋានទិន្នន័យធំ។ យើងអាចបង្កើតសន្ទស្សន៍របស់យើងក្នុងកន្លែង ដោយប្រើ៖

```python
from sklearn.neighbors import NearestNeighbors

embeddings = flattened_df['embeddings'].to_list()

# បង្កើតបញ្ជីស្វែងរក
nbrs = NearestNeighbors(n_neighbors=5, algorithm='ball_tree').fit(embeddings)

# ដើម្បីស្វែងរកក្នុងបញ្ជី អ្នកអាចប្រើវិធីសាស្រ្ត kneighbors
distances, indices = nbrs.kneighbors(embeddings)
```

### ការវាយតម្លៃឡើងវិញ

## ប្រមូលផ្តុំគ្នារួច

ជំហានចុងក្រោយគឺបញ្ចូល LLM របស់យើងដើម្បីអាចទទួលបានចម្លើយដែលភ្ជាប់ទិន្នន័យរបស់យើង។ យើងអាចអនុវត្តវាដូចខាងក្រោម៖

```python
user_input = "what is a perceptron?"

def chatbot(user_input):
    # បំលែងសំណួរឱ្យទៅជាវិចទ័រសំណួរ
    query_vector = create_embeddings(user_input)

    # ស្វែងរកឯកសារដែលមានលក្ខណៈស្រដៀងគ្នាបំផុត
    distances, indices = nbrs.kneighbors([query_vector])

    # បន្ថែមឯកសារទៅក្នុងសំណួរដើម្បីផ្តល់បរិបទ
    history = []
    for index in indices[0]:
        history.append(flattened_df['chunks'].iloc[index])

    # បញ្ចូលប្រវត្តិនិងការបញ្ចូលរបស់អ្នកប្រើ
    history.append(user_input)

    # បង្កើតវត្ថុកថា
    messages=[
        {"role": "system", "content": "You are an AI assistant that helps with AI questions."},
        {"role": "user", "content": "\n\n".join(history) }
    ]

    # ប្រើ API ផ្លាស្វីតតបតាមការឆ្លើយឆ្លើយ```
    response = client.responses.create(

    return response.output_text

chatbot(user_input)

### មាតិកាវាយតម្លៃ

- គុណភាពនៃចម្លើយដែលផ្តល់ ធានាថាវាសម្លេងធម្មជាតិ ល្អ និងដូចមនុស្ស

- ភាពភ្ជាប់នៃទិន្នន័យ៖ វាយតម្លៃថាចម្លើយដែលបានបានប្រមូលមកមានប្រភពពីឯកសារដែលផ្តល់ឬអត់

- សមរម្យភាព៖ វាយតម្លៃលទ្ធផលដែលឆ្លើយតបទៅនឹងសំណួរដែលបានគ្រោង

- ភាពរាបសារ - ថាតើចម្លើយមានអត្ថន័យត្រឹមត្រូវយ៉ាងដូចម្តេច

## ប្រើប្រាស់សម្រាប់ RAG (ការបង្កើតបង្កើនការស្វែងរក) និងមូលដ្ឋានទិន្នន័យទិសវ៉ិចទ័រ

មានករណីប្រើប្រាស់ជាច្រើនដែលការហៅមុខងារ function អាចកែលម្អកម្មវិធីរបស់អ្នកដូចជា៖

- ការឆ្លើយសំណួរ និងការផ្តល់ព័ត៌មាន៖ ភ្ជាប់ទិន្នន័យក្រុមហ៊ុនរបស់អ្នកទៅឆាតដែលអាចប្រើដោយបុគ្គលិកសម្រាប់សួរសំណួរ

- ប្រព័ន្ធផ្តល់អនុសាសន៍៖ ដែលអ្នកអាចបង្កើតប្រព័ន្ធដែលផ្គូរផ្គងតម្លៃដែលស្រដៀងគ្នា ដូចជា ភាពយន្ត ភោជនីយដ្ឋាន និងច្រើនទៀត

- សេវាកម្ម chatbot៖ អ្នកអាចរក្សាទុកប្រវត្តិការជជែក និងប្តូរជជែកឲ្យផ្ទាល់ខ្លួនដោយផ្អែកលើទិន្នន័យអ្នកប្រើ

- ស្វែងរករូបភាពដោយផ្អែកលើទិសវ៉ិចទ័រ embedding ប្រយោជន៍នៅពេលធ្វើការទទួលស្គាល់រូបភាព និងរកឃើញករណីកំហុស

## សង្ខេប

យើងបានគ្របដណ្តប់តំបន់មូលដ្ឋាននៃ RAG ចាប់ពីការបន្ថែមទិន្នន័យទៅកម្មវិធី សំណួរអ្នកប្រើ និងលទ្ធផល។ ដើម្បីសម្រួលការបង្កើត RAG អ្នកអាចប្រើប្រាស់ស៊ុមដូចជា Semanti Kernel, Langchain ឬ Autogen។

## ការតែងតាំងភារកិច្ច

ដើម្បីបន្តការរៀនរបស់អ្នកអំពីការបង្កើតបង្កើនការស្វែងរក (RAG) អ្នកអាចបង្កើត:

- បង្កើតផ្នែកមុខសម្រាប់កម្មវិធីដោយប្រើស៊ុមដែលអ្នកចូលចិត្ត

- ប្រើស៊ុម មួយក្នុងចំណោម LangChain ឬ Semantic Kernel ហើយបង្កើតឡើងវិញកម្មវិធីរបស់អ្នក។
