# ការណែនាំអំពីបណ្ដាញប្រសាទ។ Perceptron ពហុស្រទាប់

នៅក្នុងផ្នែកមុន អ្នកបានរៀនអំពីម៉ូឌែលបណ្ដាញប្រសាទសាមញ្ញបំផុត - perceptron មួយស្រទាប់ មូឌែលចាត់ថ្នាក់ពីរក្រុមតែមួយដែលមានបន្ទាត់។

នៅក្នុងផ្នែកនេះ យើងនឹងពង្រីកម៉ូឌែលនេះឱ្យទៅជាស៊ុមដែលបត់បែនបានច្រើនជាងនេះ ដែលអនុញ្ញាតិឱ្យយើង៖

* បំពេញការបែងចែកថ្នាក់ពហុថ្នាក់ លើសពីពីរក្រុម
* ដោះស្រាយបញ្ហាបង្រួចបញ្ច្រាស់ លើសពីការបែងចែកថ្នាក់
* បែងចែកថ្នាក់ដែលមិនអាចបែងចែកបានដោយបន្ទាត់

យើងក៏នឹងអភិវឌ្ឍស៊ុមម៉ូឌុលផ្ទាល់ខ្លួននៅក្នុង Python ដែលអនុញ្ញាតិឱ្យយើងបង្កើតស្ថាបត្យកម្មបណ្ដាញប្រសាទផ្សេងៗ។

## ការផ្លូវការរបស់ Machine Learning

ចាប់ផ្តើមដោយផ្លូវការបញ្ហា Machine Learning។ សន្មតថាយើងមានឈុតទិន្នន័យហ្វឹកហាត់ **X** ជាមួយនឹង ស្លាក **Y** ហើយយើងត្រូវបង្កើតម៉ូឌែល *f* ដែលនឹងធ្វើការព្យាករណ៍បានត្រឹមត្រូវបំផុត។ គុណភាពនៃការព្យាករណ៍វាស់ដោយ **Loss function** &lagran;។ Loss function ខាងក្រោមត្រូវបានប្រើជាញឹកញាប់៖

* សម្រាប់បញ្ហាបង្រួចបញ្ច្រាស់ ដែលយើងត្រូវព្យាករណ៍លេខ យើងអាចប្រើ **ភាពខុសត្រូវបរមា** &sum;<sub>i</sub>|f(x<sup>(i)</sup>)-y<sup>(i)</sup>| ឬ **ភាពខុសត្រូវពាក់កណ្ដាល** &sum;<sub>i</sub>(f(x<sup>(i)</sup>)-y<sup>(i)</sup>)<sup>2</sup>
* សម្រាប់បញ្ហាចាត់ថ្នាក់ យើងប្រើ **0-1 loss** (ដែលស្រដៀងនឹង **ភាពត្រឹមត្រូវ** នៃម៉ូឌែល) ឬ **logistic loss**។

សម្រាប់ perceptron មួយស្រទាប់, អនុគមន៍ *f* ត្រូវបានកំណត់ជា​អនុគមន៍របបបន្ទាត់ *f(x)=wx+b* (នៅទីនេះ *w* ជាមាត្រដ្ឋានទម្ងន់, *x* ជាវិចទ័រនៃលក្ខណៈបញ្ចូល, ហើយ *b* ជាវិចទ័រប្រេរ)។ សម្រាប់ស្ថាបត្យកម្មបណ្ដាញប្រសាទផ្សេងៗ អនុគមន៍នេះអាចមានរាងកាយស្មុគស្មាញជាងនេះ។

> សម្រាប់ករណីចាត់ថ្នាក់ វាទំនងជាចង់បានជាទូទៅក្នុងការទទួលបានប្រភពនៃថ្នាក់ដែលសម្របសម្រួលជាលទ្ធផលបណ្ដាញ។ ដើម្បីបម្លែងលេខដែលចៃដន្យទៅជាប្រភព (ឧ. ដើម្បីធម្មតា លទ្ធផល) យើងញឹកញាប់ប្រើអនុគមន៍ **softmax** &sigma; ហើយអនុគមន៍ *f* ក្លាយជារាង *f(x)=&sigma;(wx+b)*

ក្នុងការពិពណ៌នាពី *f* ខាងលើនេះ, *w* និង *b* ត្រូវបានហៅថា **parameters** &theta;=⟨*w,b*⟩។ នៅពេលដែលមានឈុតទិន្នន័យ ⟨**X**,**Y**⟩ យើងអាចគណនាខុសត្រូវសរុបលើឈុតទិន្នន័យជាអនុគមន៍នៃ​ parameters &theta;។

> ✅ **គោលដៅនៃការបណ្តុះបណ្តាលបណ្ដាញប្រសាទគឺបន្ថយខុសត្រូវដោយផ្លាស់ប្តូរ parameters &theta;**

## ការបង្កើនភាពល្អឡើង (Gradient Descent)

មានវិធីសាស្ត្រល្បីគឺ **gradient descent** សម្រាប់កំណត់អំពើអនុគមន៍។ គំនិតគឺ យើងអាចគណនាអនុគមន៍អាប់ (ក្នុងករណីច្រើនវិមាត្រត្រូវហៅថា **gradient**) នៃ loss function ទៅលើ parameters ហើយផ្លាស់ប្តូរ parameters ដើម្បីធ្វើឱ្យខុសត្រូវកើនយ៉ាងតិច។ វាអាចផ្លូវការជាចំណុចដូចជា៖

* បញ្ចូល parameters ជាតម្លៃចៃដន្យ w<sup>(0)</sup>, b<sup>(0)</sup>
* ធ្វើរំលងជំហានខាងក្រោមជាថ្មីៗ៖
    - w<sup>(i+1)</sup> = w<sup>(i)</sup>-&eta;&part;&lagran;/&part;w
    - b<sup>(i+1)</sup> = b<sup>(i)</sup>-&eta;&part;&lagran;/&part;b

បំរើការបណ្តុះបណ្តាល ជំហានអប្សរាត្រូវគណនាមើលពីឈុតទិន្នន័យទាំងមូល (ចងចាំថា loss គិតជាយូតាមគំរូទាំងអស់)។ ទោះយ៉ាងណា ក្នុងជីវិតពិត យើងយកផ្សំតូចៗមួយឈុតដែលហៅថា **minibatches** ហើយគណនាអាប់មួយដោយផ្អែកលើផ្នែកតូចនៃទិន្នន័យ។ នេះហៅថា **stochastic gradient descent** (SGD) ព្រោះផ្នែកតូចត្រូវបានយកចៃដន្យក្នុងមួយដង។

## Perceptron ពហុស្រទាប់ និង Backpropagation

បណ្ដាញមួយស្រទាប់ ដូចដែលយើងបានឃើញខាងលើ អាចចាត់ថ្នាក់ថ្នាក់ដែលអាចបែកបាច់ដោយបន្ទាត់បាន។ ដើម្បីបង្កើតម៉ូឌែលមានភាពសម្បូរបែប យើងអាចផ្សំស្រទាប់ជាច្រើនក្នុងបណ្ដាញ។ គណិតវិទ្យា មានន័យថា អនុគមន៍ *f* នឹងមានរាងត្រូវដែលស្មុគស្មាញនិងគណនាដោយជំហានច្រើន៖
* z<sub>1</sub>=w<sub>1</sub>x+b<sub>1</sub>
* z<sub>2</sub>=w<sub>2</sub>&alpha;(z<sub>1</sub>)+b<sub>2</sub>
* f = &sigma;(z<sub>2</sub>)

នៅទីនេះ &alpha; ជា **អនុគមន៍បាញ់បញ្ចូលមិនបន្ទាត់**, &sigma; ជា អនុគមន៍ softmax, និង parameters &theta;=<*w<sub>1</sub>,b<sub>1</sub>,w<sub>2</sub>,b<sub>2</sub>*>។

អាល់ហ្គរីធმი Gradient Descent នឹងនៅដដែល ប៉ុន្តែការគណនាអាប់នឹងជារឿងលំបាកជាងមុន។ ដោយប្រើច្បាប់ខ្សែសង្វាក់ differentiation យើងអាចគណនាអាប់ជាដូចខាងក្រោម៖

* &part;&lagran;/&part;w<sub>2</sub> = (&part;&lagran;/&part;&sigma;)(&part;&sigma;/&part;z<sub>2</sub>)(&part;z<sub>2</sub>/&part;w<sub>2</sub>)
* &part;&lagran;/&part;w<sub>1</sub> = (&part;&lagran;/&part;&sigma;)(&part;&sigma;/&part;z<sub>2</sub>)(&part;z<sub>2</sub>/&part;&alpha;)(&part;&alpha;/&part;z<sub>1</sub>)(&part;z<sub>1</sub>/&part;w<sub>1</sub>)

> ✅ ច្បាប់ខ្សែសង្វាក់ differentiation ត្រូវបានប្រើក្នុងការគណនាអាប់នៃ loss function ទៅលើ parameters។

ចំណាំ ថា ផ្នែកខាងឆ្វេងបំផុតនៃការបញ្ចេញទាំងអស់នោះដូចគ្នា ហើយដូច្នេះយើងអាចគណនាអាប់ដោយចាប់ផ្តើមពី loss function ហើយទៅតាមបំណាត់ "ត្រឡប់ក្រោយ" តាមរយៈក្រាបគណនា។ វិធីសាស្រ្តនៃការបណ្តុះបណ្តាល perceptron ពហុស្រទាប់នេះហៅថា **backpropagation**, ឬ 'backprop'។
