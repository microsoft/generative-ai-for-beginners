Phi-3.5-vision ល្អជាងម៉ូដែលធំៗដូចជា Claude-3.5 Sonnet និង Gemini 1.5 Flash លើ OCR តារាង និងការយល់ដឹងតារាងជាច្រើន និងសមស្របនៅលើការយល់ឃើញព័ត៌មានទូទៅ។ គាំទ្រការបញ្ចូលរូបភាពច្រើនខ្នាត (multi-frame input) ដែលអាចអនុវត្តហេតុផលលើរូបភាពច្រើន។


### Phi-3.5-MoE

***Mixture of Experts (MoE)*** អនុញ្ញាតឱ្យម៉ូដែលត្រូវបានបណ្ដុះបណ្ដាលដោយការប្រើប្រាស់កុំព្យូទ័រតិចជាង ដែលបើកវិញ ការពង្រីកទំហំម៉ូដែល ឬទិន្នន័យអាចធ្វើបានយ៉ាងច្រើនជាមួយថវិកាកុំព្យូទ័រដូចម៉ូដែលដង់ស៊ី។ ជាពិសេស ម៉ូដែល MoE គួរតែទទួលបានគុណភាពដូចម៉ូដែលដង់ស៊ីឆាប់រហ័សជាងនៅដំណាក់កាលបណ្ដុះបណ្ដាល។

Phi-3.5-MoE មានម៉ូឌុលជំនាញ 16x3.8B។ Phi-3.5-MoE ជាមួយ 6.6B ប៉ារ៉ាមែត្រ​សកម្ម សំរេចបានកម្រិតយល់ហេតុផល ភាសា និងគណិតវិទ្យាដូចម៉ូដែលធំពេញមួយទំហឹង។

យើងអាចប្រើម៉ូដែលក្រុម Phi-3/3.5 ដោយផ្អែកលើស្ថានការណ៍ផ្សេងៗ។ មិនដូចជា LLM អ្នកអាចដំឡើង Phi-3/3.5-mini ឬ Phi-3/3.5-Vision នៅលើឧបករណ៍ក្រៅដែនបាន។

**Hugging Face Transformer**
Hugging Face Transformers គឺជាបណ្ណាល័យមានអំណាច ដែលរួមបញ្ចូលសម្រាប់ការតម្រូវភាសាធម្មជាតិ (NLP) និងកិច្ចការម៉ាស៊ីនរៀនផ្សេងទៀត។ នេះគឺជាចំណុចសំខាន់ៗអំពីវា៖

1. **ម៉ូដែលបណ្តុះបណ្តាលមុន**៖ វាបង្ហាញម៉ូដែលបានបណ្តុះជាច្រើនដែលអាចប្រើសម្រាប់កិច្ចការផ្សេងៗ ដូចជា ការបែងចែកអក្សរ ការស្គាល់ឈ្មោះជា Entity ការឆ្លើយសំណួរ ការសង្ខេប ការប្រែសម្រួល និងការបង្កើតអត្ថបទ។

2. **ភាពអាចប្រើរួមនៃក្របខ័ណ្ឌ**៖ បណ្ណាល័យគាំទ្រក្របខ័ណ្ឌម៉ាស៊ីនរៀនជ្រៅជាច្រើន រួមមាន PyTorch, TensorFlow, និង JAX។ វាអាចអនុញ្ញាតិឱ្យអ្នកបណ្តុះម៉ូដែលនៅក្នុងក្របខ័ណ្ឌមួយ ហើយប្រើវានៅក្នុងក្របខ័ណ្ឌផ្សេងទៀត។

3. **សមត្ថភាព Multimodal**៖ មិនត្រឹមតែ NLP ប៉ុណ្ណោះទេ Hugging Face Transformers ក៏គាំទ្រកិច្ចការក្នុងការមើលឃើញកុំព្យូទ័រ (ឧទាហរណ៍ ការបែងចែករូបភាព ការស្វែងរកវត្ថុ) និងក្រុមសំឡេង (ឧទាហរណ៍ ការស្គាល់សំឡេង ការបែងចែកសំឡេង)។ 

4. **ងាយស្រួលប្រើ**៖ បណ្ណាល័យផ្តល់ API និងឧបករណ៍សម្រាប់ទាញយកនិងតំលក់ម៉ូដែល យ៉ាងងាយស្រួល ដែលអាចប្រើបានសម្រាប់អ្នកចាប់ផ្តើម និងអ្នកជំនាញទាំងអស់។

```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### ល្បង៖ ការប្រើប្រាស់ ONNX Runtime GenAI ដើម្បីហៅ Phi-3.5-Vision


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
