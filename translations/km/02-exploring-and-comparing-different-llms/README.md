# ការស្វែងយល់ និងប្រៀបធៀប LLMs ផ្សេងៗ

[![ការស្វែងយល់ និងប្រៀបធៀប LLMs ផ្សេងៗ](../../../translated_images/km/02-lesson-banner.ef94c84979f97f60.webp)](https://youtu.be/KIRUeDKscfI?si=8BHX1zvwzQBn-PlK)

> _ចុចរូបភាពខាងលើដើម្បីមើលវីដេអូបង្រៀននេះ_

ជាមួយមេរៀនមុននេះ យើងបានឃើញថា AI បង្កើតបានកំពុងផ្លាស់ប្តូរពិភពបច្ចេកវិទ្យា ដូចម្តេចដែលម៉ូដែលភាសាធំ (LLMs) ធ្វើការ និងក៏នៅរបៀបដែលអាជីវកម្ម—ដូចជា Startup របស់យើង—អាចអនុវត្តវាទៅករណីប្រើប្រាស់របស់ពួកគេ និងរីកចម្រើន! នៅក្នុងជាខ្សែភាពយន្តនេះ យើងកំពុងមើលឃើញការប្រៀបធៀប និងថ្លែងចិត្តពី LLMs ប្រភេទផ្សេងៗ ដើម្បីយល់ពីចំណុចប្រសើរនិងចំណុចខ្សោយរបស់ពួកវា។

ជំហានបន្ទាប់ក្នុងការធ្វើដំណើររបស់ startup របស់យើងគឺការស្វែងយល់ពីទស្សនៈបច្ចុប្បន្ននៃ LLMs និងយល់ថាណាដែលសមស្របសម្រាប់ករណីប្រើប្រាស់របស់យើង។

## ការណែនាំ

មេរៀននេះនឹងគ្របដណ្តប់៖

- ប្រភេទផ្សេងៗនៃ LLMs នៅក្នុងទស្សនៈបច្ចុប្បន្ន។
- ការប្រលង ការសាកល្បង និងការប្រៀបធៀបម៉ូដែលផ្សេងៗសម្រាប់ករណីប្រើប្រាស់របស់អ្នកនៅលើ Azure។
- របៀបបំពាក់ LLM។

## គោលបំណងការរៀន

បន្ទាប់ពីបញ្ចប់មេរៀននេះ អ្នកនឹងអាច៖

- ជ្រើសម៉ូដែលត្រឹមត្រូវសម្រាប់ករណីប្រើប្រាស់របស់អ្នក។
- យល់ពីរបៀបសាកល្បង កែលម្អ និងបង្កើតការប្រសើរឡើងនៃការសម្តែងម៉ូដែលរបស់អ្នក។
- ស្គាល់របៀបដែលអាជីវកម្មចាក់បញ្ចូលម៉ូដែល។

## យល់អំពីប្រភេទ LLMs ផ្សេងៗ

LLMs អាចមានចំណាត់ថ្នាក់ច្រើនផ្អែកលើរចនាសម្ព័ន្ធ ទិន្នន័យបណ្តុះបណ្តាល និងករណីប្រើប្រាស់របស់ពួកវា។ ការយល់អំពីភាពខុសគ្នាទាំងនេះនឹងជួយឱ្យ startup របស់យើងជ្រើសម៉ូដែលត្រឹមត្រូវសម្រាប់សេណារីយ៉ូ ហើយយល់ថាតើតើធ្វើដូចម្តេចក្នុងការសាកល្បង កែលម្អ និងធ្វើឲ្យការសម្តែងប្រសើរឡើង។

មានមូឌែល LLMs ផ្សេងៗជាច្រើន ការជ្រើសរើសម៉ូដែលរបស់អ្នកអាស្រ័យលើអ្វីដែលអ្នកចង់ប្រើវាសម្រាប់ ទិន្នន័យរបស់អ្នក របៀបដែលអ្នកត្រៀមខ្ចីប្រាក់ និងរឿងផ្សេងៗទៀត។

ផ្អែកលើថាតើអ្នកចង់ប្រើម៉ូដែលសម្រាប់អត្ថបទ សំឡេង វីដេអូ ការបង្កើតរូបភាព និងផ្សេងៗទៀត អ្នកអាចជ្រើសប្រភេទម៉ូដែលខុសៗគ្នា។

- **ការទទួលសំឡេងនិងការស្គាល់សំឡេង**។ ម៉ូដែលស្ទីល Whisper នៅតែមានប្រយោជន៍សម្រាប់ការស្គាល់សំឡេងប្រភេទទូទៅ ប៉ុន្តែជម្រើសផលិតកម្មឥឡូវនេះរួមមានម៉ូដែលសំឡេងទៅអត្ថបទថ្មីៗដូចជា `gpt-4o-transcribe`, `gpt-4o-mini-transcribe`, និងអត្ថប្រយោជន៍ចម្រៀង។ សូមវាយតម្លៃការគ្របដណ្ដប់ភាសា ការចម្រៀង ការគាំទ្រពេលវេលាពិត ប្រត្តិកម្ម និងថ្លៃចំណាយសម្រាប់សេណារីយ៉ូរបស់អ្នក។ សូមស្វែងយល់បន្ថែមក្នុង [ឯកសារសំឡេងទៅអត្ថបទ OpenAI](https://platform.openai.com/docs/guides/speech-to-text?WT.mc_id=academic-105485-koreyst)។

- **ការបង្កើតរូបភាព**។ DALL-E និង Midjourney គឺជាជម្រើសអស់ល្បីសម្រាប់ការបង្កើតរូបភាព ប៉ុន្តែ API រូបភាព OpenAI បច្ចុប្បន្នផ្តោតលើម៉ូដែលរូបភាព GPT ដូចជា `gpt-image-2` ខណៈដែល Stable Diffusion, Imagen, Flux និងគ្រួសារម៉ូដែលផ្សេងទៀតក៏ជាជម្រើសទូទៅផងដែរ។ ប្រៀបធៀបគោលការណ៍ សិក្សាអំពីការកែប្រែ ការគ្រប់គ្រងរចនាប័ទ្ម តម្រូវការសុវត្ថិភាព និងអាជ្ញាប័ណ្ណ។ សូមស្វែងយល់បន្ថែមនៅក្នុង [ការ​ណែនាំបង្កើត​រូបភាព OpenAI](https://platform.openai.com/docs/guides/images?WT.mc_id=academic-105485-koreyst) និងជំពូក ៩ នៃមាតិកាបង្រៀននេះ។

- **ការបង្កើតអត្ថបទ**។ ម៉ូដែលអត្ថបទឥឡូវនេះស្ថិតក្នុងម៉ូដែលដល់កម្រិតខ្ពស់ ម៉ូដែលហេតុកថា ម៉ូដែលតូច ទាបភ្លាមភ្លាម និងម៉ូដែលទំងន់បើកចំហ។ ឧទាហរណ៍បច្ចុប្បន្នរួមមានម៉ូដែល OpenAI GPT-5.x, ម៉ូដែល Anthropic Claude 4.x, ម៉ូដែល Google Gemini 3.x, ម៉ូដែល Meta Llama 4, និងម៉ូដែល Mistral។ សូមកុំជ្រើសឡើងតែដោយកាលបរិច្ឆេទចេញផ្សាយឬតម្លៃ; ប្រៀបធៀបគុណភាពភារកិច្ច ភ្លាមភ្លៅ ចង្អុល context ការប្រើឧបករណ៍ អាកប្បកិរិយាសុវត្ថិភាព ការចូលរួមតំបន់ និងថ្លៃសរុប។ [ទំនាក់ទំនងម៉ូដែល Microsoft Foundry](https://ai.azure.com/catalog?WT.mc_id=academic-105485-koreyst) គឺជាទីតាំងល្អសម្រាប់ប្រៀបធៀបម៉ូដែលដែលមាននៅលើ Azure។

- **បង្រួមសមត្ថភាពច្រើនប្រភេទ**។ ម៉ូដែលបច្ចុប្បន្នជាច្រើនអាចដំណើរការបានលើសពីអត្ថបទតែ១។ មានប្រភេទមួយទទួលបានការបញ្ចូលរូបភាព សំឡេង ឬវីដេអូ; មានអ្នកអាចហៅឧបករណ៍; ហើយម៉ូដែលជំនាញអាចបង្កើតរូបភាព សំឡេង ឬវីដេអូបាន។ ឧទាហរណ៍ម៉ូដែល OpenAI បច្ចុប្បន្នគាំទ្រការបញ្ចូលអត្ថបទ និងរូបភាព ម៉ូដែល Gemini អាចគាំទ្រអត្ថបទ កូដ រូបភាព សំឡេង និងវីដេអូនៅលើកំណត់ប្រភេទខុសគ្នា ហើយ Llama 4 Scout និង Maverick គឺជាម៉ូដែលទម្ងន់បើកដែលជាផ្លូវការច្រើនប្រភេទ។ តែងតែពិនិត្យផ្គត់ផ្គង់ការបញ្ចូល និងការបញ្ចេញលទ្ធផលមុនសាងសង់លំនាំការងារ។

ការជ្រើសម៉ូដែលមានន័យថាអ្នកទទួលបានសមត្ថភាពមូលដ្ឋានមួយចំនួន ដែលអាចមិនគ្រប់គ្រាន់។ ជាញឹកញាប់ អ្នកមានទិន្នន័យជាក់លាក់របស់ក្រុមហ៊ុនដែលអ្នកត្រូវប្រាប់ទៅ LLM។ មានជម្រើសខុសៗគ្នាអំពីរបៀបដោះស្រាយបញ្ហានេះ នៅខាងក្រោមមេរៀននេះ។

### Foundation Models ប្រៀបធៀបនឹង LLMs

ពាក្យ Foundation Model ត្រូវបាន [បង្កើតដោយអ្នកស្រាវជ្រាវ Stanford](https://arxiv.org/abs/2108.07258?WT.mc_id=academic-105485-koreyst) និងបានកំណត្តាថាជាម៉ូដែល AI ដែលបំពេញលក្ខណៈមួយចំនួន ដូចជា៖

- **ពួកវាត្រូវបានបណ្តុះបណ្តាលដោយការរៀនដោយខ្លួនឯងឬរៀនដោយខ្លួនឯងដោយគ្មានកាណ៍ណែនាំ** មានន័យថាពួកវាត្រូវបានបណ្តុះបណ្តាលលើទិន្នន័យមូលដ្ឋានចម្រូការពហុប្រភេទ ដែលមិនចាំបាច់មានការសម្គាល់ឬដាក់ផ្ទាំងទិន្នន័យសម្រាប់ដំណើរការបណ្តុះបណ្តាល។
- **ពួកវាជាម៉ូដែលធំមួយចំនួន** ដែលមានបណ្តាញសរសៃប្រសាទជ្រៅជាងមួយ បណ្ដុះឱ្យមានប៉ារ៉ាម៉ែត្រប្រមាណពាន់លាន។
- **ពួកវាត្រូវបានគេបង្កើតជាគ្រា ‘foundation’ សម្រាប់ម៉ូដែលផ្សេងទៀត** មានន័យថាអាចប្រើធ្វើជាទីតាំងចាប់ផ្តើមសម្រាប់ការបង្កើតម៉ូដែលផ្សេងទៀតដែលអាចធ្វើបានតាមរយៈការគ្រប់គ្រងលម្អិតនិងកែលម្អ។

### ម៉ូដែល Open-Weight/Open-Source ប្រៀបធៀបនឹងម៉ូដែលផ្ទាល់ខ្លួន

មុខងារផ្សេងទៀតសម្រាប់ចាត់ថ្នាក់ LLM គឺតើវាជា open-weight, open-source, ឬម៉ូដែលផ្ទាល់ខ្លួន។

ម៉ូដែល open-source និង open-weight ផ្តល់ឱ្យដំណើរការម៉ូដែលសម្រាប់ត្រួតពិនិត្យ ទាញយក ឬប្តូរតាមតម្រូវការ ប៉ុន្តែអាជ្ញាប័ណ្ណរបស់ពួកវាពិតជាផ្សេងគ្នា។ មានខ្លះពេញលេញ open-source ខណៈដែលផ្សេងទៀតជាម៉ូដែល open-weight ដែលមានកំណត់ការប្រើប្រាស់។ ពួកវាមានប្រយោជន៍នៅពេលអាជីវកម្មត្រូវការត្រួតពិនិត្យលើការចាក់ចេញ ព័ត៌មានតំបន់ ថ្លៃសេវា ឬការប្ដូរតាមតម្រូវការ។ ទោះជាយ៉ាងណា ក្រុមការងារត្រូវការត្រួតពិនិត្យលក្ខខណ្ឌអាជ្ញាប័ណ្ណ ថ្លៃសេវា សន្ដិសុខ ការថែទាំ និងគុណភាពសាកល្បងមុនប្រើប្រាស់ក្នុងផលិតកម្ម។ ឧទាហរណ៍រួមមាន [Meta Llama 4](https://ai.meta.com/blog/llama-4-multimodal-intelligence/?WT.mc_id=academic-105485-koreyst), ម៉ូដែល [Mistral មួយចំនួន](https://docs.mistral.ai/models/overview?WT.mc_id=academic-105485-koreyst), និងម៉ូដែលជាច្រើនដែលម៉ោននៅលើ [Hugging Face](https://huggingface.co/models?WT.mc_id=academic-105485-koreyst)។

ម៉ូដែលផ្ទាល់ខ្លួនពេញចិត្តនិងបម្រើដោយអ្នកផ្គត់ផ្គង់។ ម៉ូដែលទាំងនេះជាប់ក្នុងការបង្កើតឲ្យកាន់តែប្រសើរដោយគ្រប់គ្រងការប្រើប្រាស់លើផលិតកម្ម និងផ្តល់ជូននូវការគាំទ្រមានកម្លាំង ប្រព័ន្ធសុវត្ថិភាព ការរួមបញ្ចូលឧបករណ៍ និងធំធេងឆាប់។ ទោះជាយ៉ាងណា អតិថិជនជាវភាគច្រើនមិនអាចបញ្ជាក់ឬកែប្រែមាត្រាម៉ូដែល ហើយត្រូវតែលើកតម្កើងលក្ខខណ្ឌពីអ្នកផ្គត់ផ្គង់សម្រាប់ភាពឯកជន ការរក្សាទុក ការអនុវត្ត និងការប្រើប្រាស់ទទួលយកបាន។ ឧទាហរណ៍រួមមាន [ម៉ូដែល OpenAI](https://platform.openai.com/docs/models?WT.mc_id=academic-105485-koreyst), [Google Gemini](https://deepmind.google/models/gemini/pro/?WT.mc_id=academic-105485-koreyst), និង [Anthropic Claude](https://platform.claude.com/docs/en/about-claude/models/overview?WT.mc_id=academic-105485-koreyst)។

### Embedding ប្រៀប​ធៀប​នឹង Image generation ប្រៀប​ធៀប​នឹង Text and Code generation

LLMs ក៏អាចចាត់ថ្នាក់ទៅតាមលទ្ធផលដែលពួកវាបានបង្កើត។

Embeddings គឺជាសំណុំម៉ូដែលដែលអាចបម្លែងអត្ថបទទៅជារូបភាពលេខមួយហៅថា embedding ដែលជាការតំណាងជាលេខនៃអត្ថបទបញ្ចូល។ Embeddings ធ្វើឲ្យម៉ាស៊ីនយល់បានកាន់តែងាយស្រួលអំពីទំនាក់ទំនងរវាងពាក្យឬប្រយោគ ហើយអាចបរិភោគជាចូលដំណើរការម៉ូដែលផ្សេងៗ ដូចជា ម៉ូដែលចាត់ថ្នាក់ ឬម៉ូដែលក្រុម ដែលមានសមត្ថភាពល្អជាងលើទិន្នន័យលេខ។ ម៉ូដែល embedding ជាញឹកញាប់ប្រើសម្រាប់ការរៀនផ្ទេរ ដែលម៉ូដែលត្រូវបានបង្កើតសម្រាប់ភារកិច្ចជំនួសមួយដែលមានទិន្នន័យច្រើន ហើយបន្ទាប់មកវាអាចប្រើទំងន់ម៉ូដែល (embeddings) សម្រាប់ភារកិច្ចក្រោម។ ឧទាហរណ៍ក្នុងចំណោមនេះគឺ [OpenAI embeddings](https://platform.openai.com/docs/models/embeddings?WT.mc_id=academic-105485-koreyst)។

### Encoder-Decoder ប្រៀបធៀបនឹង Decoder-only

ដើម្បីនិយាយអំពីប្រភេទរចនាសម្ព័ន្ធផ្សេងៗនៃ LLMs មកប្រើរូបមន្តស្រដៀងគ្នា។

ស្រមៃថាមេដឹកនាំរបស់អ្នកបានផ្តល់ភារកិច្ចឲ្យអ្នកសរសេរប្រឡងសម្រាប់សិស្ស។ អ្នកមានមនុស្សរួមការងារពីរនាក់; ម្នាក់គ្រប់គ្រងការបង្កើតមាតិកា និងម្នាក់គ្រប់គ្រងពិនិត្យមើល។

អ្នកបង្កើតមាតិកាគឺដូចម៉ូដែល Decoder-only: ពួកគេចាំបាច់មើលប្រធានបទ មើលអ្វីដែលអ្នកបានសរសេរហើយបន្តបង្កើតមាតិកាឡើងវិញយោងតាម context នោះ។ ពួកគេមានជំនាញល្អក្នុងការសរសេរមាតិកាដែលទាក់ទាញនិងផ្តល់ព័ត៌មាន ប៉ុន្តែពុំមែនជាជម្រើសល្អជាងសម្រាប់ភារកិច្ចចាត់ថ្នាក់ យកឯកសារ ឬ encode ព័ត៌មានទេ។ ឧទាហរណ៍អ្នកគ្រប់គ្រងម៉ូដែល decoder-only រួមមានម៉ូដែល GPT និង Llama។

អ្នកពិនិត្យគឺដូចម៉ូដែល Encoder only ពួកគេចង់មើលមេរៀននិងចម្លើយ ដោយសង្កេតទំនាក់ទំនងរវាងពួកវា និងយល់ឆន្ទៈ ប៉ុន្តែនៅពួកគេមិនល្អក្នុងការបង្កើតមាតិកាទេ។ ឧទាហរណ៍ម៉ូដែល Encoder only គឺ BERT។

ស្រមៃថាយើងអាចមានមនុស្សម្នាក់ទៀតដែលអាចបង្កើតនិងពិនិត្យប្រឡងបាន នេះគឺជា ម៉ូដែល Encoder-Decoder។ ឧទាហរណ៍មួយចំនួនរួមមាន BART និង T5។

### សេវាកម្ម ប្រៀបធៀបនឹង ម៉ូដែល

ឥឡូវនេះ យើងនិយាយអំពីភាពខុសគ្នារវាងសេវាកម្ម និងម៉ូដែល។ សេវាកម្មគឺជាតម្លៃដែលផ្តល់ដោយអ្នកផ្គត់ផ្គង់សេវាកម្ម Cloud ហើយជាទូទៅជាការរួមបញ្ចូលម៉ូដែល ទិន្នន័យ និងគ្រឿងផ្សេងៗ។ ម៉ូដែលគឺជាប្រភេទមូលដ្ឋាននៃសេវាកម្ម ហើយជាទូទៅជាម៉ូដែលមូលដ្ឋាន ដូចជា LLM។

សេវាកម្មតែងតែត្រូវបានគេបង្កើតឲ្យមានប្រសិទ្ធភាពសម្រាប់ការប្រើប្រាស់ផលិតកម្ម ហើយងាយក្នុងការប្រើជាងម៉ូដែល តាមរយៈចំណុចប្រទាក់ប្រើប្រាស់ក្រាហ្វិក។ ទោះជាយ៉ាងណា សេវាកម្មមិនតែងតែមានដោយឥតគិតថ្លៃ ហើយអាចត្រូវការជាវឬការទូទាត់ក្នុងការប្រើ ដើម្បីប្រើប្រាស់ឧបករណ៍ និងធនធានរបស់ម្ចាស់សេវាកម្ម កែលម្អថ្លៃចំណាយ និងវិមាត្រងាយស្រួល។ ឧទាហរណ៍សេវាកម្មគឺ [Azure OpenAI Service](https://learn.microsoft.com/azure/ai-foundry/openai/overview?WT.mc_id=academic-105485-koreyst) ដែលផ្តល់ផែនការតម្លៃ pay-as-you-go មានន័យថាអ្នកប្រើទូទាត់តាមបរិមាណនៃការប្រើប្រាស់សេវាកម្ម។ Azure OpenAI Service ក៏ផ្តល់សុវត្ថិភាពថ្នាក់អាជីវកម្ម និងស៊ុម AI មានទំនួលខុសត្រូវលើសមត្ថភាពម៉ូដែល។

ម៉ូដែលគឺជាធាតុអាគូម៉ាញនេវើក: ប៉ារ៉ាម៉ែត្រ ទំងន់ រចនាសម្ព័ន្ធ កម្មវិធីបកស្រាយអត្ថបទ និងការកំណត់គាំទ្រ។ ប្រតិបត្តិម៉ូដែលក្នុងតំបន់បុគ្គល ឬបរិយាកាសឯកជនត្រូវការឧបករណ៍សមស្រប ការចាក់បម្រើគ្រឿងចក្រ ការត្រួតពិនិត្យ និងអាជ្ញាប័ណ្ណ open-source/open-weight ឬអាជ្ញាប័ណ្ណពាណិជ្ជកម្ម។ ម៉ូដែល open-weight ដូចជា Llama 4 ឬម៉ូដែល Mistral អាចបម្រើដោយអូខ្លួនឯង ប៉ុន្តែត្រូវការប្រើប្រាស់ថាមពលកុំព្យូទ័រ និងជំនាញប្រតិបត្តិការ។

## របៀបសាកល្បង និងកែលម្អជាមួយម៉ូដែលផ្សេងៗដើម្បីយល់ពីសមត្ថភាពលើ Azure


បន្ទាប់ពីក្រុមរបស់យើងបានស្ទង់មើលទេសភាព LLMs បច្ចុប្បន្ន និងទាញយកបេក្ខជនល្អៗមួយចំនួនសម្រាប់ស្ថានการณ์របស់ពួកគេ ជំហានបន្ទាប់គឺសាកល្បងពួកវាលើទិន្នន័យរបស់ពួកគេ និងលើបញ្ជីការងារ។ នេះគឺជាការប្រព្រឹត្តទៅវិញទៅមួយ ដែលធ្វើឡើងដោយការធ្វើប្រតិស៊ម និងវាស់វែង។
ម៉ូឌែលភាគច្រើនដែលយើងបាននិយាយក្នុងកថាខណ្ឌមុនៗ (ម៉ូឌែល OpenAI, ម៉ូឌែលសំរាប់ធ្វើការ open-weight ដូចជា Llama 4 និង Mistral, និងម៉ូឌែល Hugging Face) មាននៅក្នុង [Microsoft Foundry Models](https://learn.microsoft.com/azure/foundry/concepts/foundry-models-overview?WT.mc_id=academic-105485-koreyst)។

## ការកែលម្អលទ្ធផល LLM

យើងបានស្ទង់មើលជាមួយក្រុម startup របស់យើងនូវប្រភេទ LLMs ផ្សេងៗ និងវេទិកាកម្រងពពក (Microsoft Foundry) ដែលអនុញ្ញាតឲ្យយើងប្រៀបធៀបម៉ូឌែលផ្សេងៗ វាយតម្លៃលើទិន្នន័យសាកល្បង កែលម្អសមត្ថភាព និងចាក់បញ្ចូលទៅកាន់ចំណុចប្រទាក់វិភាគ។

តែពេលណាដែលពួកគេចង់ពិចារណា fine-tune ម៉ូឌែល មិនប្រើម៉ូឌែលដែលបានបណ្តុះបណ្តាលរួច? តើមានវិធីផ្សេងទៀតក្នុងការកែលម្អសមត្ថភាពម៉ូឌែលលើបញ្ជីការងារពិសេសទេ?

មានវិធីជាច្រើនដែលអាជីវកម្មអាចប្រើបានដើម្បីទទួលលទ្ធផលដែលពួកគេសម្រាប់ពី LLM។ អ្នកអាចជ្រើសរើសប្រភេទម៉ូឌែលផ្សេងៗ មានកម្រិតបណ្តុះបណ្តាលខុសគ្នាក្នុងការចាក់បញ្ចូល LLM ក្នុងផលិតកម្ម ដែលមានកម្រិតស្មុគស្មាញ ប្រាក់ចំណាយ និងគុណភាពខុសគ្នា។ នេះគឺជាវិធីផ្សេងៗមួយចំនួន៖

- **បច្ចេកវិទ្យា Prompt ជាមួយ Context**។ គំនិតគឺផ្តល់ context គ្រប់គ្រាន់ពេលលោកអ្នកបន្ទាន់បន្ថែម ដើម្បីធានាថាអ្នកទទួលបានចម្លើយដែលត្រូវការ។

- **Retrieval Augmented Generation, RAG**។ ទិន្នន័យរបស់អ្នកអាចមាននៅក្នុងមូលដ្ឋានទិន្នន័យ ឬចំណុចគេហទំព័រមួយ ដើម្បីធានាថាទិន្នន័យនេះ ឬផ្នែកមួយរបស់វា ត្រូវបានបញ្ចូលក្នុងពេលបញ្ចូល prompt អ្នកអាចយកទិន្នន័យពាក់ព័ន្ធនោះ ហើយបញ្ចូលវា​ជា​ផ្នែកនៃ prompt របស់អ្នកប្រើប្រាស់។

- **ម៉ូឌែលដែលបាន fine-tune**។ នៅទីនេះ អ្នកបានបណ្តុះបណ្តាលម៉ូឌែលបន្ថែមលើទិន្នន័យរបស់អ្នក ដែលនាំឲ្យម៉ូឌែលមានភាពត្រឹមត្រូវ និងឆាប់ឆន្ទៈទៅតម្រូវការ អាចមានការចំណាយខ្ពស់។

### បច្ចេកវិទ្យា Prompt ជាមួយ Context

LLMs ដែលបានបណ្តុះបណ្តាលរួចទៅហើយ ធ្វើការ​ល្អណាស់លើភារកិច្ចភាសាតាមធម្មជាតិទូទៅ ទោះបីទំរង់ prompt នោះខ្លីដូចជា វាក្យដើម្បីបញ្ចប់ឬសំណួរមួយដែរ — ដែលហៅថា “ការសិក្សាផ្លែសូន្យ” (zero-shot learning)។

ទោះយ៉ាងណា អ្នកប្រើប្រាស់អាចបង្ហាញសំណួររបស់ពួកគេជាលំអិត ជាមួយនឹងការស្នើរសុំ និងឧទាហរណ៍ ច្រើនប៉ុណ្ណោះ ការឆ្លើយតបនឹងច្បាស់ និងស្ទាត់ជាង តាមការរំពឹងទុករបស់អ្នកប្រើ។ ក្នុងករណីនេះ យើងនិយាយពី “ការសិក្សារជាឧទាហរណ៍មួយ” ប្រសិនបើ prompt មានតែឧទាហរណ៍តែមួយ និង “ការសិក្សារជាចំនួនពីរឬច្រើន” ប្រសិនបើវា មានឧទាហរណ៍ច្រើន។ បច្ចេកវិទ្យា Prompt ជាមួយ Context គឺជាវិធីដែលផ្តល់ភាពកាន់តែមានប្រសិទ្ធភាពក្នុងការចាប់ផ្តើម។




វិធីនេះអាចត្រូវបានដោះស្រាយតាមរយៈ RAG ដែលជាបច្ចេកទេសបន្ថែមទិន្នន័យខាងក្រៅជាជុំឯកសារចុងក្រោយចូលក្នុង prompt ដោយគិតទាំងកំណត់ប្រវែង prompt។ វាត្រូវបានគាំទ្រដោយឧបករណ៍មូលដ្ឋានទិន្នន័យ Vector (ដូចជា [Azure Vector Search](https://learn.microsoft.com/azure/search/vector-search-overview?WT.mc_id=academic-105485-koreyst)) ដែលយកជុំឯកសារដែលមានប្រយោជន៍ពីប្រភពទិន្នន័យកំណត់មុខនានា មកបញ្ចូលក្នុង Context នៃ prompt។
