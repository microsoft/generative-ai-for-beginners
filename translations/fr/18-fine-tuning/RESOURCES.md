# Ressources Pour Apprentissage Autonome

La leçon a été construite en utilisant des ressources principales de référence d'OpenAI et Microsoft Foundry pour la terminologie et les tutoriels. Voici une liste non exhaustive pour vos propres parcours d'apprentissage autonome. Chaque lien ci-dessous pointe vers du matériel actuel et supporté.

## 1. Ressources Principales

| Titre/Lien | Description |
| :--- | :--- |
| [Fine-tuning with OpenAI Models](https://platform.openai.com/docs/guides/fine-tuning?WT.mc_id=academic-105485-koreyst) | Le fine-tuning améliore l'apprentissage avec peu d’exemples en s’entraînant sur beaucoup plus d’exemples que ce qui peut tenir dans l’invite - réduisant les coûts, améliorant la qualité des réponses, et permettant des requêtes à latence plus faible. **Obtenez un aperçu du fine-tuning chez OpenAI.** |
| [When to use Microsoft Foundry fine-tuning](https://learn.microsoft.com/azure/ai-foundry/openai/concepts/fine-tuning-considerations?WT.mc_id=academic-105485-koreyst) | Comprenez **ce qu’est le fine-tuning (concept)**, pourquoi vous devriez l’envisager, quelles données utiliser, et comment mesurer la qualité - ainsi que quand SFT, DPO, ou RFT est le choix approprié. |
| [Customize a model with fine-tuning](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning?WT.mc_id=academic-105485-koreyst) | Le **guide complet (processus)** pour le fine-tuning dans Microsoft Foundry via le portail, le SDK Python OpenAI / Foundry, ou l’API REST - couvrant la préparation des données, l’entraînement, les points de contrôle, et le déploiement. |
| [Continuous fine-tuning](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning?WT.mc_id=academic-105485-koreyst#perform-continuous-fine-tuning) | Le processus itératif consistant à sélectionner un modèle déjà fine-tuné comme modèle de base et **à le fine-tuner davantage** sur de nouveaux ensembles d’exemples d’entraînement. |
| [Fine-tuning with tool (function) calling](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning-functions?WT.mc_id=academic-105485-koreyst) | Le fine-tuning de votre modèle **avec des exemples d’appel à des outils** améliore la sortie - des réponses plus précises, cohérentes, au format similaire en utilisant moins de tokens d’invite. |
| [Fine-tuning models: Microsoft Foundry guidance](https://learn.microsoft.com/azure/ai-foundry/foundry-models/concepts/models-sold-directly-by-azure?WT.mc_id=academic-105485-koreyst#fine-tuning-models) | Consultez **quels modèles peuvent être fine-tunés**, les méthodes qu’ils supportent (SFT / DPO / RFT), et les régions où ils sont disponibles. |
| [Fine-tuning overview: techniques and modalities](https://learn.microsoft.com/azure/ai-foundry/concepts/fine-tuning-overview?WT.mc_id=academic-105485-koreyst) | Comparez les trois techniques d’entraînement (SFT, DPO, RFT) et les deux modalités (serverless vs. calcul managé), avec des conseils pour choisir un modèle de base et démarrer. |
| **Tutoriel** : [Fine-tune a model in Microsoft Foundry](https://learn.microsoft.com/azure/ai-foundry/openai/tutorials/fine-tune?WT.mc_id=academic-105485-koreyst) | Créez un jeu de données exemple, préparez le fine-tuning, lancez une tâche de fine-tuning sur un modèle actuellement supporté comme `gpt-4.1-mini`, et déployez le modèle fine-tuné sur Azure. |
| **Tutoriel** : [Fine-tune models with serverless API deployments](https://learn.microsoft.com/azure/ai-foundry/how-to/fine-tune-serverless?WT.mc_id=academic-105485-koreyst) | Adaptez les modèles open et partenaires (Phi, Llama, Mistral, etc.) à vos jeux de données _grâce à un flux de travail à faible code, basé sur une interface utilisateur_ dans Microsoft Foundry. |
| **Tutoriel** : [Fine-tune Hugging Face models on Azure Databricks](https://learn.microsoft.com/azure/databricks/machine-learning/train-model/huggingface/fine-tune-model?WT.mc_id=academic-105485-koreyst) | Fine-tunez un modèle Hugging Face avec la bibliothèque `transformers` sur un GPU unique en utilisant Azure Databricks et le Hugging Face Trainer. |
| **Formation** : [Fine-tune a foundation model with Azure Machine Learning](https://learn.microsoft.com/training/modules/finetune-foundation-model-with-azure-machine-learning/?WT.mc_id=academic-105485-koreyst) | Le catalogue de modèles Azure Machine Learning offre de nombreux modèles open-source que vous pouvez fine-tuner. Partie du [parcours d’apprentissage Azure ML Générative AI](https://learn.microsoft.com/training/paths/work-with-generative-models-azure-machine-learning/?WT.mc_id=academic-105485-koreyst). |
| **Tutoriel** : [Azure OpenAI fine-tuning with Weights & Biases](https://docs.wandb.ai/guides/integrations/azure-openai-fine-tuning?WT.mc_id=academic-105485-koreyst) | Suivez et analysez les exécutions de fine-tuning sur Azure avec W&B. Prolonge le guide OpenAI fine-tuning avec des étapes spécifiques à Azure et le suivi des expériences. |

## 2. Ressources Secondaires

Cette section regroupe des ressources supplémentaires intéressantes que nous n’avons pas eu le temps de couvrir dans la leçon. Utilisez-les pour construire votre propre expertise sur ce sujet.

| Titre/Lien | Description |
| :--- | :--- |
| **OpenAI Cookbook** : [Préparation et analyse des données pour le fine-tuning des modèles de chat](https://cookbook.openai.com/examples/chat_finetuning_data_prep?WT.mc_id=academic-105485-koreyst) | Prétraitez et analysez un jeu de données de chat avant fine-tuning : vérifiez les erreurs de format, obtenez des statistiques de base, et estimez le nombre de tokens (et le coût). À utiliser avec le [guide OpenAI fine-tuning](https://platform.openai.com/docs/guides/fine-tuning?WT.mc_id=academic-105485-koreyst). |
| **OpenAI Cookbook** : [Fine-tuning pour Retrieval Augmented Generation (RAG) avec Qdrant](https://cookbook.openai.com/examples/fine-tuned_qa/ft_retrieval_augmented_generation_qdrant?WT.mc_id=academic-105485-koreyst) | Un exemple complet de fine-tuning des modèles OpenAI pour RAG, intégrant Qdrant et l’apprentissage avec peu d’exemples pour améliorer les performances et réduire les fabrications. |
| **OpenAI Cookbook** : [Fine-tuning GPT avec Weights & Biases](https://cookbook.openai.com/examples/third_party/gpt_finetuning_with_wandb?WT.mc_id=academic-105485-koreyst) | Utilisez W&B pour suivre l’entraînement et le fine-tuning des modèles. Lisez d’abord leur [guide OpenAI Fine-Tuning](https://docs.wandb.ai/guides/integrations/openai-fine-tuning/?WT.mc_id=academic-105485-koreyst), puis essayez l’exercice du Cookbook. |
| **Tutoriel Hugging Face** : [Comment Fine-Tuner des LLMs avec Hugging Face TRL](https://www.philschmid.de/fine-tune-llms-in-2024-with-trl?WT.mc_id=academic-105485-koreyst) | Fine-tunez des LLMs ouverts en utilisant Hugging Face TRL, Transformers et datasets : définissez un cas d’usage, configurez un environnement dev, préparez un jeu de données, fine-tunez, évaluez, et déployez. |
| **Hugging Face** : [AutoTrain Advanced](https://github.com/huggingface/autotrain-advanced?WT.mc_id=academic-105485-koreyst) | Une bibliothèque no-code / low-code de Hugging Face pour fine-tuner plusieurs types de modèles. Exécutez-la dans votre propre cloud, sur Hugging Face Spaces, ou localement via GUI, CLI, ou config YAML. |
| **Unsloth** : [Guide de Fine-Tuning des LLMs](https://docs.unsloth.ai/get-started/fine-tuning-llms-guide) | Un framework open-source qui facilite le fine-tuning local des LLMs et l’apprentissage par renforcement, avec des [notebooks](https://github.com/unslothai/notebooks?WT.mc_id=academic-105485-koreyst) prêts à l’emploi. |
