# AGENTS.md

## نظرة عامة على المشروع

يحتوي هذا المستودع على منهج شامل من 21 درسًا يُعلّم أساسيات الذكاء الاصطناعي التوليدي وتطوير التطبيقات. تم تصميم الدورة للمبتدئين وتغطي كل شيء من المفاهيم الأساسية إلى بناء تطبيقات جاهزة للإنتاج.

**التقنيات الرئيسية:**
- بايثون 3.9+ مع مكتبات: `openai`، `python-dotenv`، `tiktoken`، `azure-ai-inference`، `pandas`، `numpy`، `matplotlib`
- تايب سكريبت/جافا سكريبت مع Node.js والمكتبات: `openai` (Azure OpenAI عبر نقطة النهاية v1 + API الردود)، `@azure-rest/ai-inference` (نماذج Microsoft Foundry)
- خدمة Azure OpenAI، واجهة برمجة تطبيقات OpenAI، ونماذج Microsoft Foundry (نزول نماذج GitHub بحلول نهاية يوليو 2026)
- دفاتر Jupyter للتعلم التفاعلي
- حاويات التطوير لبيئة تطوير متسقة

**هيكل المستودع:**
- 21 مجلدًا للدرورس مرقمة (00-21) تحتوي على ملفات README، أمثلة للشفرة، وواجبات
- تنفيذات متعددة: أمثلة بايثون، تايب سكريبت، وأحيانًا .NET
- مجلد الترجمات مع أكثر من 40 نسخة لغوية
- إعداد مركزي عبر ملف `.env` (استخدم `.env.copy` كقالب)

## أوامر الإعداد

### إعداد المستودع الأولي

```bash
# استنساخ المستودع
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# نسخ قالب البيئة
cp .env.copy .env
# تحرير ملف .env بمفاتيح API ونقاط نهاية الخاصة بك
```

### إعداد بيئة بايثون

```bash
# إنشاء بيئة افتراضية
python3 -m venv venv

# تفعيل البيئة الافتراضية
# على macOS/Linux:
source venv/bin/activate
# على Windows:
venv\Scripts\activate

# تثبيت التبعيات
pip install -r requirements.txt
```


## سير عمل التطوير

### متغيرات البيئة

جميع الدروس التي تتطلب الوصول إلى API تستخدم متغيرات بيئة معرفة في `.env`:

- `OPENAI_API_KEY` - لمفتاح API الخاص بـ OpenAI
- `AZURE_OPENAI_API_KEY` - لمفتاح Azure OpenAI في Microsoft Foundry (خدمة Azure OpenAI جزء من Microsoft Foundry الآن: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - عنوان نهاية Azure OpenAI (نهاية مورد Foundry)
- `AZURE_OPENAI_DEPLOYMENT` - اسم نشر نموذج إكمال المحادثة
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - اسم نشر نموذج التضمين
- `AZURE_OPENAI_API_VERSION` - إصدار API (الافتراضي: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - لمفتاح نماذج Hugging Face
- `AZURE_INFERENCE_ENDPOINT` - نقطة نهاية نماذج Microsoft Foundry (كتالوج نماذج متعدد الموفرين)
- `AZURE_INFERENCE_CREDENTIAL` - مفتاح API لنماذج Microsoft Foundry (يحل محل `GITHUB_TOKEN` الذي تم تقاعده)

## إرشادات نمط الشفرة

### بايثون

- استخدم `python-dotenv` لإدارة متغيرات البيئة
- استورد مكتبة `openai` للتفاعل مع API
- استخدم `pylint` للتحقق من جودة الشفرة (بعض الأمثلة تستخدم `# pylint: disable=all` للبساطة)
- اتبع معايير تسمية PEP 8
- خزّن بيانات اعتماد API في ملف `.env`، وليس داخل الشفرة أبدا

### تايب سكريبت

- استخدم حزمة `dotenv` لمتغيرات البيئة
- تكوين تايب سكريبت في `tsconfig.json` لكل تطبيق
- استخدم حزمة `openai` لـ Azure OpenAI (وجه العميل إلى نقطة نهاية `/openai/v1/` واستدعي `client.responses.create`)؛ استخدم `@azure-rest/ai-inference` لنماذج Microsoft Foundry
- استخدم `nodemon` للتطوير مع إعادة تحميل تلقائية
- قم بعملية البناء قبل التشغيل: `npm run build` ثم `npm start`

### الاتفاقيات العامة

- اجعل أمثلة الشفرة بسيطة وتعليمية
- ضمن تعليقات تشرح المفاهيم الرئيسية
- يجب أن تكون شفرة كل درس مكتفية ذاتيًا وقابلة للتشغيل
- استخدم نظام تسمية متسق: بادئة `aoai-` لـ Azure OpenAI، `oai-` لـ OpenAI API، `githubmodels-` لنماذج Microsoft Foundry (احتفظ بالبادئة القديمة من عهد نماذج GitHub)

2. سكربت بعد الإنشاء يثبت تبعيات بايثون تلقائيًا
3. الإضافات لـ Python و Jupyter مهيأة مسبقًا
4. البيئة مبنية على `mcr.microsoft.com/devcontainers/universal:2.11.2`

## استكشاف الأخطاء وإصلاحها

### المشكلات الشائعة

**أخطاء استيراد بايثون**:
- تأكد من تفعيل البيئة الافتراضية
- شغل `pip install -r requirements.txt`
- تحقق من أن إصدار بايثون 3.9 أو أعلى

- شغل `npm install` في مجلد التطبيق المحدد

- تحقق من وجود ملف `.env` وقيمه صحيحة

- انسخ `.env.copy` إلى `.env`
