## التوليد المعزز بالاسترجاع (RAG)

يعالج روبوت المحادثة المعتمد على نموذج اللغة الكبير موجهات المستخدم لتوليد الردود. صمم ليكون تفاعليًا ويتعامل مع المستخدمين عبر مجموعة واسعة من المواضيع. ومع ذلك، ردوده محدودة بالسياق المقدم وبيانات التدريب الأساسية. على سبيل المثال، GPT-4 توقفت معرفته عند سبتمبر 2021، مما يعني أنه يفتقر إلى معرفة الأحداث التي وقعت بعد هذه الفترة. بالإضافة إلى ذلك، البيانات المستخدمة لتدريب نماذج اللغة الكبيرة لا تشمل المعلومات السرية مثل الملاحظات الشخصية أو دليل منتجات الشركات.

### كيف تعمل RAGs (التوليد المعزز بالاسترجاع)

![رسم يوضح كيفية عمل RAGs](../../../translated_images/ar/how-rag-works.f5d0ff63942bd3a6.webp)

افترض أنك تريد نشر روبوت محادثة ينشئ اختبارات من ملاحظاتك، ستحتاج إلى اتصال بقاعدة المعرفة. هذا هو دور RAG. تعمل RAGs كما يلي:

- **قاعدة المعرفة:** قبل الاسترجاع، يجب استهلاك هذه الوثائق ومعالجتها مسبقًا، عادةً بتقسيم الوثائق الكبيرة إلى قطع أصغر، وتحويلها إلى تضمينات نصية وتخزينها في قاعدة بيانات.

- **استعلام المستخدم:** يسأل المستخدم سؤالًا

- **الاسترجاع:** عندما يسأل المستخدم سؤالًا، يسترجع نموذج التضمين المعلومات ذات الصلة من قاعدة المعرفة لتوفير سياق إضافي يتم تضمينه في الموجه.

- **التوليد المعزز:** يعزز نموذج اللغة الكبير رده استنادًا إلى البيانات المسترجعة. مما يسمح بأن يكون الرد المولَّد ليس فقط مستندًا إلى البيانات المدربة مسبقًا بل أيضًا على المعلومات ذات الصلة من السياق المضاف. تُستخدم البيانات المسترجعة لتعزيز ردود النموذج. ثم يعيد النموذج إجابة على سؤال المستخدم.

![رسم يوضح هندسة RAGs](../../../translated_images/ar/encoder-decode.f2658c25d0eadee2.webp)

تُنفذ هندسة RAGs باستخدام المحولات المكونة من جزئين: المشفر والمفكك. على سبيل المثال، عندما يطرح المستخدم سؤالًا، يُرمز النص المدخل إلى متجهات تلتقط معنى الكلمات، ثم يتم فك تشفير المتجهات إلى فهرس المستندات لدينا وينشئ نصًا جديدًا استنادًا إلى سؤال المستخدم. يستخدم نموذج اللغة الكبير نموذج مشفر-مفكك لتوليد المخرجات.

هناك طريقتان عند تنفيذ RAG وفقًا للورقة المقترحة: [Retrieval-Augmented Generation for Knowledge intensive NLP Tasks](https://arxiv.org/pdf/2005.11401.pdf?WT.mc_id=academic-105485-koreyst):

- **_RAG-Sequence_** باستخدام المستندات المسترجعة للتنبؤ بأفضل إجابة ممكنة لاستعلام المستخدم

- **RAG-Token** باستخدام المستندات لتوليد الوحدة التالية، ثم استرجاعها للإجابة عن استعلام المستخدم

### لماذا تستخدم RAGs؟

- **غنى المعلومات:** تضمن أن ردود النصوص حديثة ومواكبة. وبالتالي، تعزز الأداء في المهام المتخصصة من خلال الوصول إلى قاعدة المعرفة الداخلية.

- تقلل من الاختلاق باستخدام **بيانات قابلة للتحقق** في قاعدة المعرفة لتوفير السياق لاستعلامات المستخدم.

- هي **فعالة من حيث التكلفة** لأنها أكثر اقتصادية مقارنة بضبط نموذج اللغة الكبير الدقيق.

## إنشاء قاعدة المعرفة

تطبيقنا يستند إلى بياناتنا الشخصية أي درس الشبكات العصبية في منهج الذكاء الاصطناعي للمبتدئين.

### قواعد بيانات المتجهات

قاعدة بيانات المتجهات، على عكس قواعد البيانات التقليدية، هي قاعدة بيانات متخصصة مصممة لتخزين وإدارة والبحث عن المتجهات المضمّنة. تخزن تمثيلات رقمية للوثائق. تقسيم البيانات إلى تضمينات رقمية يسهل على نظام الذكاء الاصطناعي فهم البيانات ومعالجتها.

نخزن التضمينات في قواعد بيانات المتجهات حيث أن نماذج اللغة الكبيرة لديها حد لعدد الرموز المقبولة كمدخلات. نظرًا لأنه لا يمكن تمرير التضمينات كاملة إلى نموذج لغة كبير، سنحتاج لتقسيمها إلى قطع وعندما يطرح المستخدم سؤالًا، يتم إرجاع التضمينات الأكثر تشابهًا مع السؤال مع الموجه. أيضًا، التقسيم يقلل من التكاليف بسبب عدد الرموز المرسلة للنموذج.

بعض قواعد بيانات المتجهات الشائعة تشمل Azure Cosmos DB, Clarifyai, Pinecone, Chromadb, ScaNN, Qdrant و DeepLake. يمكنك إنشاء نموذج Azure Cosmos DB باستخدام Azure CLI بالأمر التالي:

```bash
az login
az group create -n <resource-group-name> -l <location>
az cosmosdb create -n <cosmos-db-name> -r <resource-group-name>
az cosmosdb list-keys -n <cosmos-db-name> -g <resource-group-name>
```

### من النص إلى التضمينات

قبل تخزين بياناتنا، نحتاج لتحويلها إلى تضمينات رقمية قبل تخزينها في قاعدة البيانات. إذا كنت تتعامل مع وثائق كبيرة أو نصوص طويلة، يمكنك تقسيمها وفقًا للاستعلامات المتوقعة. يمكن التقسيم على مستوى الجملة أو الفقرة. نظرًا لأن التقسيم يعتمد على معاني الكلمات المحيطة، يمكنك إضافة سياق آخر للقطعة مثل عنوان الوثيقة أو بعض النص قبل أو بعد القطعة. يمكنك تقسيم البيانات كالتالي:

```python
def split_text(text, max_length, min_length):
    words = text.split()
    chunks = []
    current_chunk = []

    for word in words:
        current_chunk.append(word)
        if len(' '.join(current_chunk)) < max_length and len(' '.join(current_chunk)) > min_length:
            chunks.append(' '.join(current_chunk))
            current_chunk = []

    # إذا لم يصل الجزء الأخير إلى الطول الأدنى، فأضفه على أي حال
    if current_chunk:
        chunks.append(' '.join(current_chunk))

    return chunks
```

بعد التقسيم، يمكننا تضمين نصنا باستخدام نماذج تضمين مختلفة. بعض النماذج التي يمكنك استخدامها تشمل: word2vec, ada-002 من OpenAI، Azure Computer Vision والعديد غيرها. اختيار النموذج يعتمد على اللغات التي تستخدمها، نوع المحتوى المشفر (نص/صور/صوت)، حجم المدخلات التي يمكنه ترميزها وطول ناتج التضمين.

## الاسترجاع والبحث في المتجهات

عندما يطرح المستخدم سؤالًا، يحول المسترجع السؤال إلى متجه باستخدام مشفر الاستعلام، ثم يبحث في فهرس مستنداتنا عن المتجهات ذات الصلة في المستند المرتبطة بالإدخال. بعد ذلك، يحول كل من المتجه المدخل ومتجهات المستند إلى نص ويمرره للنموذج الكبير.

### الاسترجاع

يحدث الاسترجاع عندما يحاول النظام إيجاد الوثائق بسرعة من الفهرس التي تفي بمعايير البحث. هدف المسترجع هو الحصول على الوثائق التي ستستخدم لتوفير السياق وتأصيل النموذج على بياناتك.

هناك عدة طرق للبحث ضمن قاعدة البيانات مثل:

- **البحث بالكلمات المفتاحية** - يستخدم للبحث النصي

- **البحث بالمتجهات** - يحول الوثائق نصيًا إلى تمثيل متجه باستخدام نماذج التضمين، مما يسمح بالبحث الدلالي باستخدام معنى الكلمات. يتم الاسترجاع عن طريق استعلام الوثائق التي تمثل متجهاتها الأقرب إلى سؤال المستخدم.

- **مختلط** - مجموعة من البحث بالكلمات والبحث بالمتجهات.

تحدي الاسترجاع يظهر عندما لا يوجد رد مشابه للاستعلام في القاعدة، عندها يعيد النظام أفضل المعلومات المتوفرة، ولكن يمكنك استخدام استراتيجيات مثل تحديد أقصى مسافة للملاءمة أو استخدام البحث المختلط الذي يجمع بين البحث بالكلمات والبحث بالمتجهات. في هذا الدرس سنستخدم البحث المختلط، مزيج بين البحث بالمتجهات والكلمات المفتاحية. سنخزن بياناتنا في إطار بيانات يحتوي على الأعمدة للقطع والتضمينات.

### تشابه المتجهات

سيبحث المسترجع في قاعدة المعرفة عن التضمينات المتقاربة، أقرب الجيران، لأنها نصوص مشابهة. في السيناريو، عند طرح المستخدم استعلام، يُضمّن أولًا ثم يُطابق مع تضمينات مشابهة. القياس الشائع المستخدم لمعرفة تشابه المتجهات المختلفة هو التشابه الكوني القائم على الزاوية بين المتجهين.

يمكننا قياس التشابه باستخدام بدائل أخرى مثل المسافة الإقليدية وهي الخط المستقيم بين نهايات المتجه والمتجه الناتج الذي يقيس مجموع نواتج العناصر المناظرة للمتجهين.

### فهرس البحث

عند القيام بالاسترجاع، نحتاج لبناء فهرس بحث لقاعدة معرفتنا قبل البدء. يخزن الفهرس التضمينات ويمكنه استرجاع القطع الأكثر تشابهًا بسرعة حتى في قواعد بيانات كبيرة. يمكننا إنشاء الفهرس محليًا باستخدام:

```python
from sklearn.neighbors import NearestNeighbors

embeddings = flattened_df['embeddings'].to_list()

# إنشاء فهرس البحث
nbrs = NearestNeighbors(n_neighbors=5, algorithm='ball_tree').fit(embeddings)

# للاستعلام من الفهرس، يمكنك استخدام طريقة kneighbors
distances, indices = nbrs.kneighbors(embeddings)
```

### إعادة الترتيب

بمجرد الاستعلام من قاعدة البيانات، قد تحتاج إلى فرز النتائج من الأكثر ملاءمة. يستخدم نموذج إعادة الترتيب تقنيات تعلم الآلة لتحسين ملاءمة نتائج البحث بترتيبها من الأكثر صلة. باستخدام Azure AI Search، يتم إعادة الترتيب تلقائيًا لك باستخدام منظم معنوي. مثال على كيفية عمل إعادة الترتيب باستخدام أقرب الجيران:

```python
# ابحث عن المستندات الأكثر تشابهاً
distances, indices = nbrs.kneighbors([query_vector])

index = []
# اطبع المستندات الأكثر تشابهاً
for i in range(3):
    index = indices[0][i]
    for index in indices[0]:
        print(flattened_df['chunks'].iloc[index])
        print(flattened_df['path'].iloc[index])
        print(flattened_df['distances'].iloc[index])
    else:
        print(f"Index {index} not found in DataFrame")
```

## حالات استخدام RAG (التوليد المعزز بالاسترجاع) وقواعد بيانات المتجهات

هناك العديد من حالات الاستخدام حيث يمكن أن تحسن مكالمات الدوال تطبيقك مثل:

- الأسئلة والأجوبة: تأصيل بيانات شركتك إلى دردشة يمكن للموظفين استخدامها لطرح الأسئلة.
