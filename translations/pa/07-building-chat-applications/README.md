# ਜਨਰੇਟਿਵ ਏਆਈ-ਚਾਲਿਤ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਦਾ ਨਿਰਮਾਣ

ਹੁਣ ਜਦੋਂ ਅਸੀਂ ਵੇਖ ਲਿਆ ਹੈ ਕਿ ਕਿਵੇਂ ਅਸੀਂ ਟੈਕਸਟ-ਜਨਰੇਸ਼ਨ ਐਪਸ ਬਣਾਉਂਦੇ ਹਾਂ, ਚੱਲੋ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਵਿੱਚ ਵੱਡਾ ਦੇਖਦੇ ਹਾਂ।

ਚੈਟ ਐਪਲੀਕੇਸ਼ਨ ਸਾਡੇ ਰੋਜ਼ਾਨਾ ਜੀਵਨ ਦਾ ਹਿੱਸਾ ਬਣ ਗਈਆਂ ਹਨ, ਜੋ ਸਿਰਫ ਇਕ ਆਮ ਗੱਲ-ਬਾਤ ਦਾ ਜਰੀਆ ਹੀ ਨਹੀਂ, ਸਗੋਂ ਕਸਟਮਰ ਸਰਵਿਸ, ਤਕਨੀਕੀ ਸਹਾਇਤਾ ਅਤੇ ਇਨਮੀ ਸਲਾਹਕਾਰ ਪ੍ਰਣਾਲੀਆਂ ਦਾ ਭੀ ਅਹੰਕਾਰ ਹਨ। ਸੰਭਵ ਹੈ ਕਿ ਤੁਸੀਂ ਕੁਝ ਸਮਾਂ ਪਹਿਲਾਂ ਕਿਸੇ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨ ਤੋਂ ਮਦਦ ਪ੍ਰਾਪਤ ਕੀਤੀ ਹੋਵੇ। ਜਿਵੇਂ ਜਨਰੇਟਿਵ ਏਆਈ ਵਰਗੀਆਂ ਅੱਗੇ ਵਧੀਆਂ ਤਕਨਾਲੋਜੀਆਂ ਨੂੰ ਇਸ ਪਲੇਟਫਾਰਮਾਂ ਵਿੱਚ ਸ਼ਾਮਿਲ ਕਰਦੇ ਹਾਂ, ਜਟਿਲਤਾ ਵਧਦੀ ਹੈ ਅਤੇ ਚੁਣੌਤੀਆਂ ਭੀ ਇੰਨੀ ਹੀ ਵੱਧਦੀਆਂ ਹਨ।

ਕੁਝ ਸਵਾਲ ਜਿਨ੍ਹਾਂ ਦੇ ਜਵਾਬ ਲੱਭਣੇ ਜ਼ਰੂਰੀ ਹਨ:

- **ਐਪ ਬਣਾਉਣਾ**। ਕਿਸ ਤਰ੍ਹਾਂ ਅਸੀਂ ਇਨ੍ਹਾਂ ਏਆਈ-ਚਾਲਿਤ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਖਾਸ ਉਦੇਸ਼ਾਂ ਲਈ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਢੰਗ ਨਾਲ ਬਣਾਈਏ ਅਤੇ ਦਿੱਸਦੇ ਸਿਸਟਮਾਂ ਵਿੱਚ ਸਮਾਧਾਨ ਕਰੀਏ?
- **ਨਿਗਰਾਨੀ**। ਜਦੋਂ ਇਹ ਤਾਇਨਾਤ ਹੋ ਚੁੱਕੀਆਂ ਹੋਣ, ਕਿਵੇਂ ਅਸੀਂ ਇਹ ਯਕੀਨੀ ਬਣਾ ਸਕੀਏ ਕਿ ਐਪਲੀਕੇਸ਼ਨ ਸਫਲਤਾ ਅਤੇ [ਜ਼ਿੰਮੇਵਾਰ ਏਆਈ ਦੇ ਛੇ ਸਿਧਾਂਤਾਂ](https://www.microsoft.com/ai/responsible-ai?WT.mc_id=academic-105485-koreyst) ਦੀ ਪਾਲਣਾ ਕਰ ਰਹੀਆਂ ਹਨ?

ਜਿਵੇਂ ਅਸੀਂ ਇਕ ਐਸੇ ਯੁਗ ਵਿੱਚ ਪਹੁੰਚ ਰਹੇ ਹਾਂ ਜਿਸ ਦੀ ਪਰਿਭਾਸ਼ਾ ਆਟੋਮੇਸ਼ਨ ਅਤੇ ਬਿਨਾ ਕਿਸੇ ਰੁਕਾਵਟ ਵਾਲੇ ਮਨੁੱਖ-ਮਸ਼ੀਨ ਇੰਟਰਐਕਸ਼ਨਾਂ ਨਾਲ ਹੁੰਦੀ ਹੈ, ਜਨਰੇਟਿਵ ਏਆਈ ਕਿਵੇਂ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਦੇ ਪੈਮਾਨੇ, ਗਹਿਰਾਈ ਅਤੇ ਅਨੂਕੂਲਤਾ ਨੂੰ ਬਦਲਦਾ ਹੈ, ਇਹ ਸਮਝਣਾ ਆਵশ্যਕ ਹੈ। ਇਹ ਪਾਠ ਉਸ ਜਾਣਕਾਰੀ ਨੂੰ ਖੋਜੇਗਾ ਜੋ ਇਨ੍ਹਾਂ ਜਟਿਲ ਪ੍ਰਣਾਲੀਆਂ ਨੂੰ ਸਹਾਰਨ ਵਾਲੀ ਬਣਤਰ ਦੇ ਪੱਖਾਂ ਨੂੰ ਸਮਝਣ ਲਈ ਹੈ, ਖੇਤਰ-ਖਾਸ ਕੰਮਾਂ ਲਈ ਉਨ੍ਹਾਂ ਨੂੰ ਠੀਕ ਕਰਨ ਦੇ ਤਰੀਕੇ, ਅਤੇ ਜ਼ਿੰਮੇਵਾਰ ਏਆਈ ਦੇ ਤਾਇਨਾਤ ਲਈ ਲਾਗੂ ਮਾਪਦੰਡਾਂ ਅਤੇ ਵਿਚਾਰਾਂ ਦਾ ਮੁਲਾਂਕਣ ਕਰੇਗਾ।

## ਪਰਿਚਯ

ਇਸ ਪਾਠ ਵਿੱਚ ਸ਼ਾਮਲ ਹਨ:

- ਚੈਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਢੰਗ ਨਾਲ ਬਣਾਉਣ ਅਤੇ ਜੋੜਨ ਦੀਆਂ ਤਕਨੀਕਾਂ।
- ਐਪ ਦੀ ਵਿਸ਼ੇਸ਼ਤਾ ਅਤੇ ਫਾਈਨ-ਟਿਊਨਿੰਗ ਤੇ ਤਰੀਕੇ।
- ਚੈਟ ਐਪ ਦੀ ਨਿਗਰਾਨੀ ਲਈ ਸੂਝ-ਬੂਝ ਅਤੇ ਤਕਨੀਕਾਂ।

## ਸਿੱਖਣ ਦੇ ਹਦਫ਼

ਇਸ ਪਾਠ ਦੇ ਅੰਤ ਤੱਕ ਤੁਸੀਂ ਸਮਰੱਥ ਹੋਵੋਗੇ:

- ਚੈਟ ਐਪ ਨੂੰ ਮੌਜੂਦਾ ਸਿਸਟਮਾਂ ਵਿੱਚ ਜੋੜਨ ਅਤੇ ਉਸ ਦੀ ਨਿਰਮਾਣ ਲਈ ਵਿਚਾਰ ਕਰਨ।
- ਖਾਸ ਉਦੇਸ਼ਾਂ ਲਈ ਚੈਟ ਐਪ ਨੂੰ ਵਿਉਕਤਗਤ ਕਰਨ।
- ਚੈਟ ਏਆਈ-ਚਾਲਿਤ ਐਪ ਦੀ ਗੁਣਵੱਤਾ ਅਤੇ ਕਾਰਗੁਜ਼ਾਰੀ ਨਿਗਰਾਨੀ ਲਈ ਮੁੱਖ ਮਾਪਦੰਡਾਂ ਦੀ ਪਛਾਣ।
- ਜ਼ਿੰਮੇਵਾਰ ਢੰਗ ਨਾਲ ਏਆਈ ਦਾ ਲਾਭ ਉਠਾਉਣਾ ਯਕੀਨੀ ਬਣਾਉਣਾ।

## ਚੈਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਵਿੱਚ ਜਨਰੇਟਿਵ ਏਆਈ ਦੀ ਸ਼ਾਮਲਦਾਰੀ

ਜਨਰੇਟਿਵ ਏਆਈ ਨਾਲ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਸੁਧਾਰਨਾ ਸਿਰਫ ਉਨ੍ਹਾਂ ਨੂੰ ਜ਼ਿਆਦਾ ਸਮਾਰਟ ਬਣਾਉਣਾ ਨਹੀਂ ਹੈ; ਇਹ ਉਹਨਾਂ ਦੀ ਬਣਤਰ, ਕਾਰਗੁਜ਼ਾਰੀ ਅਤੇ ਯੂਜ਼ਰ ਇੰਟਰਫੇਸ ਨੂੰ ਸੰਵਾਰਨ ਬਾਰੇ ਹੈ ਤਾਂ ਜੋ ਉੱਤਮ ਯੂਜ਼ਰ ਅਨੁਭਵ ਦਿੱਤਾ ਜਾ ਸਕੇ। ਇਸ ਵਿੱਚ ਆਰਕੀਟੈਕਚਰ ਦੇ ਮੂਲ ਬੁਨਿਆਦਾਂ, API ਇਕਾਈਕਰਨ ਅਤੇ ਯੂਜ਼ਰ ਇੰਟਰਫੇਸ ਸਬੰਧੀ ਵਿਚਾਰ ਸ਼ਾਮਲ ਹਨ। ਇਹ ਭਾਗ ਤੁਹਾਨੂੰ ਇਨ੍ਹਾਂ ਜਟਿਲ ਖੇਤਰਾਂ ਵਿੱਚ ਮਾਰਗਦਰਸ਼ਨ ਦਿੰਦਾ ਹੈ, ਚਾਹੇ ਤੁਸੀਂ ਉਨ੍ਹਾਂ ਨੂੰ ਮੌਜੂਦਾ ਸਿਸਟਮਾਂ ਵਿੱਚ ਜੋੜ ਰਹੇ ਹੋ ਜਾਂ ਖੁਦ-ਮੁਖਤਿਆਰ ਪਲੇਟਫਾਰਮ ਵਜੋ ਬਣਾਉਂਦੇ ਹੋ।

ਇਸ ਭਾਗ ਦੇ ਅੰਤ ਤੱਕ, ਤੁਹਾਡੇ ਕੋਲ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਨੂੰ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਢੰਗ ਨਾਲ ਬਣਾਉਣ ਅਤੇ ਸ਼ਾਮਲ ਕਰਨ ਦੀ ਮਹਾਰਤ ਹੋਵੇਗੀ।

### ਚੈਟਬੋਟ ਜਾਂ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨ?

ਚੈਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਦਾ ਨਿਰਮਾਣ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ, ਆਓ 'ਚੈਟਬੋਟ' ਅਤੇ 'ਏਆਈ-ਚਾਲਿਤ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨ' ਦੀ ਤੁਲਨਾ ਕਰੀਏ ਜੋ ਵੱਖ-ਵੱਖ ਭੂਮਿਕਾਵਾਂ ਅਤੇ ਕਾਰਗੁਜ਼ਾਰੀ ਰੱਖਦੇ ਹਨ। ਚੈਟਬੋਟ ਦਾ ਮੁੱਖ ਮਕਸਦ ਖਾਸ ਗੱਲ-ਬਾਤ ਕੰਮਾਂ ਨੂੰ ਆਟੋਮੇਟ ਕਰਨਾ ਹੁੰਦਾ ਹੈ, ਜਿਵੇਂ ਪ੍ਰਸ਼ਨਾਂ ਦੇ ਜਵਾਬ ਦੇਣਾ ਜਾਂ ਪੈਕੇਜ ਟ੍ਰੈਕ ਕਰਨਾ। ਇਹ ਆਮ ਤੌਰ 'ਤੇ ਨਿਯਮ-ਆਧਾਰਿਤ ਤਰੱਕੀ ਨਾਲ ਜਾਂ ਜਟਿਲ ਏਆਈ ਅਲਗੋਰਿਦਮਾਂ ਨਾਲ ਚਲਾਇਆ ਜਾਂਦਾ ਹੈ। ਇਸਦੇ ਬਲੂੇ ਐਆਈ-ਚਾਲਿਤ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨ ਕਾਫੀ ਵੱਧ ਵਿਆਪਕ ਮਾਹੌਲ ਹੁੰਦੀ ਹੈ ਜੋ ਮਨੁੱਖੀ ਯੂਜ਼ਰਾਂ ਵਿਚਕਾਰ ਟੈਕਸਟ, ਆਵਾਜ਼ ਅਤੇ ਵੀਡੀਓ ਚੈਟ ਵਰਗੀਆਂ ਵੱਖਰੀਆਂ ਡਿਜੀਟਲ ਸੰਚਾਰ ਸਹੁਲਤਾਂ ਦਿੰਦੀ ਹੈ। ਇਸਦੀ ਮੁੱਖ ਵਿਸ਼ੇਸ਼ਤਾ ਜਨਰੇਟਿਵ ਏਆਈ ਮਾਡਲ ਦੀ ਸ਼ਾਮਲਦਾਰੀ ਹੈ ਜੋ ਬਹੁ-ਕਿਸਮ ਦੀ ਬੁਝਦਾਰ, ਮਨੁੱਖ-ਵੰਗਾਰਗ ਗੱਲਬਾਤ ਮੁਹੱਈਆ ਕਰਦਾ ਹੈ ਅਤੇ ਵੱਖ-ਵੱਖ ਇਨਪੁਟ ਤੇ ਸੰਦਰਭਾਂ ਦੇ ਆਧਾਰ 'ਤੇ ਜਵਾਬ ਬਣਾਉਂਦਾ ਹੈ। ਇਕ ਜਨਰੇਟਿਵ ਏਆਈ ਚੈਟ ਐਪ ਖੁੱਲੇ ਇਲਾਕੇ ਚਰਚਾ ਕਰ ਸਕਦਾ ਹੈ, ਗੱਲਬਾਤ ਦੇ ਸੰਦਰਭਾਂ ਦਾ ਅਨੁਕੂਲਨ ਕਰ ਸਕਦਾ ਹੈ ਅਤੇ ਕ੍ਰੀਏਟਿਵ ਜਾਂ ਜਟਿਲ ਸੰਵਾਦ ਵੀ ਤਿਆਰ ਕਰ ਸਕਦਾ ਹੈ।

ਹੇਠਾਂ ਦਿੱਤੀ ਸੀਮਾ ਸਾਨੂੰ ਉਨ੍ਹਾਂ ਦੇ ਵਿਸ਼ੇਸ਼ ਭੂਮਿਕਾਵਾਂ ਨੂੰ ਸਮਝਣ ਵਿੱਚ ਮਦਦ ਕਰਨ ਵਾਸਤੇ ਮੁੱਖ ਫਰਕ ਅਤੇ ਸੁਮਿਲਿਤੀਆਂ ਸਾਂਝੀਆਂ ਕਰਦੀ ਹੈ।

### SDK ਅਤੇ API ਨਾਲ ਪਹਿਲਾਂ ਤੋਂ ਬਣੇ ਫੰਕਸ਼ਨਲਿਟੀਜ਼ ਦਾ ਲਾਭ ਉਠਾਉਣਾ

ਚੈਟ ਐਪ ਬਣਾਉਂਦੇ ਸਮੇਂ, ਇਹ ਪਹਿਲਾ ਕਦਮ ਮਹਾਨ ਹੈ ਕਿ ਪਹਿਲਾਂ ਦੇ ਉਪਲਬਧ ਸਰੋਤਾਂ ਦਾ ਮੁੱਲਾਂਕਣ ਕੀਤਾ ਜਾਵੇ। SDK ਅਤੇ API ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਦੀ ਬਣਤਰ ਇੱਕ ਫਾਇਦੇਮੰਦ ਰਣਨੀਤੀ ਹੈ ਕਈ ਕਾਰਨਾਂ ਕਰਕੇ। ਚੰਗੀ ਤਰ੍ਹਾਂ ਦਸਤਾਵੇਜ਼ੀਬੱਧ SDK ਅਤੇ API ਨੂੰ ਸ਼ਾਮਿਲ ਕਰਕੇ, ਤੁਸੀਂ ਆਪਣੀ ਐਪ ਲਈ ਲੰਬੇ ਸਮੇਂ ਦੀ ਕਾਮਯਾਬੀ ਨੂੰ ਯਕੀਨੀ ਬਣਾ ਰਹੇ ਹੋ, ਸਕੇਲਬਿਲਟੀ ਅਤੇ ਸੰਭਾਲ ਦੀ ਚਿੰਤਾ ਕਰਦਿਆਂ।

- **ਡਿਵੈਲਪਮੈਂਟ ਪ੍ਰਕਿਰਿਆ ਨੂੰ ਤੇਜ਼ ਕਰਦਾ ਹੈ ਅਤੇ ਓਵਰਹੈਡ ਘਟਾਉਂਦਾ ਹੈ**: ਖੁਦ ਨਿਰਮਾਣ ਦੀ ਮਹਿੰਗੀ ਪ੍ਰਕਿਰਿਆ ਦੀ ਬਜਾਏ ਪਹਿਲਾਂ ਬਣੀ ਫੰਕਸ਼ਨਲਿਟੀ ਦੀ ਵਰਤੋਂ ਤੁਹਾਨੂੰ ਆਪਣੇ ਐਪ ਦੇ ਵਪਾਰਕ ਲਾਜਿਕ ਵਰਗੇ ਹੋਰ ਅਹਿਮ پہلوਆਂ 'ਤੇ ਧਿਆਨ ਕੇਂਦਰਿਤ ਕਰਨ ਦੀ ਆਜ਼ਾਦੀ ਦਿੰਦਾ ਹੈ।
- **ਵਧੀਆ ਕਾਰਗੁਜ਼ਾਰੀ**: ਜਦੋਂ ਤੁਸੀਂ ਫੰਕਸ਼ਨਲਿਟੀ ਖੁਦ ਬਣਾਉਂਦੇ ਹੋ, ਅੰਤ ਵਿੱਚ ਤੁਸੀਂ ਸੋਚਦੇ ਹੋ "ਇਹ ਕਿਵੇਂ ਸਕੇਲ ਕਰਦਾ ਹੈ? ਕੀ ਇਹ ਐਪ ਬੜੀ ਗਿਣਤੀ ਵਿੱਚ ਯੂਜ਼ਰਾਂ ਨੂੰ ਹੇਠਾਂ ਸੰਭਾਲ ਸਕਦਾ ਹੈ?" ਚੰਗੀ ਤਰ੍ਹਾਂ ਸੰਭਾਲਿਆ ਗਿਆ SDK ਅਤੇ API ਆਮ ਤੌਰ 'ਤੇ ਇਸ ਤਰ੍ਹਾਂ ਦੀਆਂ ਚਿੰਤਾਵਾਂ ਲਈ ਬਣੇ ਹੱਲ ਸ਼ਾਮਿਲ ਕਰਦੇ ਹਨ।
- **ਸੌਖਾ ਸੰਭਾਲਣਾ**: ਜਦੋਂ ਨਵੀਆਂ ਵਰਜਨਾਂ ਜਾਰੀ ਕੀਤੀਆਂ ਜਾਂਦੀਆਂ ਹਨ, ਅਪਡੇਟਸ ਅਤੇ ਸੁਧਾਰ ਅਸਾਨ ਹੁੰਦੇ ਹਨ ਕਿਉਂਕਿ ਜ਼ਿਆਦਾਤਰ API ਅਤੇ SDK ਲਾਇਬ੍ਰੇਰੀ ਅਪਡੇਟ ਕਰਨ ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ।
- **ਤਾਜ਼ਾ ਤਕਨਾਲੋਜੀ ਤੱਕ ਪਹੁੰਚ**: ਉਹ ਮਾਡਲ ਜਿਨ੍ਹਾਂ ਨੂੰ ਵੱਡੇ ਡੇਟਾਸੈਟ ਉੱਤੇ ਸਬਕਿਆ ਅਤੇ ਟਿਊਨ ਕੀਤਾ ਗਿਆ ਹੈ, ਤੁਹਾਡੇ ਐਪ ਲਈ ਕੁਦਰਤੀ ਭਾਸ਼ਾ ਸਕੀਲਾਂ ਮੁਹੱਈਆ ਕਰਵਾਉਂਦੇ ਹਨ।

SDK ਜਾਂ API ਦੀ ਫੰਕਸ਼ਨਲਿਟੀ ਤੱਕ ਪਹੁੰਚ ਪ੍ਰਾਪਤ ਕਰਨ ਦਾ ਤਰੀਕਾ ਆਮ ਤੌਰ 'ਤੇ ਇੱਕ ਵੱਖਰਾ ਕੁੰਜੀ ਜਾਂ ਪ੍ਰਮਾਣਿਕਤਾ ਟੋਕਨ ਦੇ ਕੇ ਹੁੰਦਾ ਹੈ। ਅਸੀਂ OpenAI ਪਾਇਥਨ ਲਾਇਬ੍ਰੇਰੀ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਇਸ ਦੀ ਵਿਵੇਚਨਾ ਕਰਾਂਗੇ। ਤੁਸੀਂ ਇਸਨੂੰ ਇਸ ਪਾਠ ਲਈ ਹੇਠ ਦਿੱਤੇ [OpenAI ਨੋਟਬੁੱਕ](./python/oai-assignment.ipynb?WT.mc_id=academic-105485-koreyst) ਜਾਂ [Azure OpenAI ਸੇਵਾਵਾਂ ਲਈ ਨੋਟਬੁੱਕ](./python/aoai-assignment.ipynb?WT.mc_id=academic-105485-koreys) ਵਿੱਚ ਖ਼ੁਦ ਵੀ ਕੋਸ਼ਿਸ਼ ਕਰ ਸਕਦੇ ਹੋ।

```python
import os
from openai import OpenAI

API_KEY = os.getenv("OPENAI_API_KEY","")

client = OpenAI(
    api_key=API_KEY
    )

response = client.responses.create(model="gpt-4o-mini", input="Suggest two titles for an instructional lesson on chat applications for generative AI.", store=False)
print(response.output_text)
```

ਉੱਪਰ ਦਿੱਤਾ ਉਦਾਹਰਨ GPT-4o ਮਿਨੀ ਮਾਡਲ ਅਤੇ Responses API ਨਾਲ ਪ੍ਰੰਪਟ ਨੂੰ ਪੂਰਾ ਕਰਨ ਲਈ ਵਰਤਦਾ ਹੈ, ਪਰ ਧਿਆਨ ਦਿਓ ਕਿ API ਕੁੰਜੀ ਇਸਨੂੰ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ ਸੈੱਟ ਕੀਤੀ ਗਈ ਹੈ। ਜੇ ਤੁਸੀਂ ਕੁੰਜੀ ਸੈੱਟ ਨਹੀਂ ਕਰੋਗੇ ਤਾਂ ਤੁਹਾਨੂੰ ਗਲਤੀ ਮਿਲੇਗੀ।

## ਯੂਜ਼ਰ ਅਨੁਭਵ (UX)

ਸਧਾਰਣ UX ਸਿਧਾਂਤ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨਾਂ 'ਤੇ ਲਾਗੂ ਹੁੰਦੇ ਹਨ, ਪਰ ਮਸ਼ੀਨ ਲਰਨਿੰਗ ਘਟਕਾਂ ਕਰਕੇ ਕੁਝ ਵੱਖਰੇ ਧਿਆਨਾਂ ਉਨੀਮੈਂ ਮਹੱਤਵਪੂਰਣ ਹੋ ਜਾਂਦੇ ਹਨ।

- **ਅਸਪਸ਼ਟਤਾ ਨੂੰ ਸੂਧਾਰਨ ਦਾ ਤਰੀਕਾ**: ਜਨਰੇਟਿਵ ਏਆਈ ਮਾਡਲ ਕਈ ਵਾਰ ਅਸਪਸ਼ਟ ਜਵਾਬ ਤਿਆਰ ਕਰਦੇ ਹਨ। ਇਕ ਐਸਾ ਫੀਚਰ ਜੋ ਯੂਜ਼ਰਾਂ ਨੂੰ ਸਪਸ਼ਟੀਕਰਨ ਮੰਗਣ ਦੀ ਪਹੁੰਚ ਦੇਵੇ, ਇਹ ਸਮੱਸਿਆ ਦੇ ਵਕਤ ਮਦਦਗਾਰ ਸਾਬਤ ਹੋ ਸਕਦਾ ਹੈ।
- **ਸੰਦਰਭ ਬਚਾਉਣਾ**: ਅੱਗੇ ਵਧੀਆਂ ਜਨਰੇਟਿਵ ਏਆਈ ਮਾਡਲ ਇੱਕ ਗੱਲਬਾਤ ਵਿੱਚ ਸੰਦਰਭ ਯਾਦ ਰੱਖ ਸਕਦੇ ਹਨ, ਜੋ ਯੂਜ਼ਰ ਅਨੁਭਵ ਲਈ ਜ਼ਰੂਰੀ ਹੈ। ਯੂਜ਼ਰਾਂ ਨੂੰ ਸੰਦਰਭ ਨਾਲ ਕੰਟਰੋਲ ਅਤੇ ਪ੍ਰਬੰਧਨ ਕਰਨ ਦੀ ਆਗਿਆ ਦੇਣਾ ਅਨੁਭਵ ਬਿਹਤਰ ਬਣਾਉਂਦਾ ਹੈ, ਪਰ ਇਸ ਨਾਲ ਸੰਵੇਦਨਸ਼ੀਲ ਜਾਣਕਾਰੀ ਸੰਭਾਲਣ ਦਾ ਖ਼ਤਰਾ ਵੀ ਹੁੰਦਾ ਹੈ। ਜਾਣਕਾਰੀ ਕਿੰਨੀ ਲੰਮੀ ਸਮੇਂ ਤੱਕ ਸੰਭਾਲੀ ਜਾਵੇ, ਜਿਵੇਂ ਰੀਟੈਨਸ਼ਨ ਨੀਤੀ ਲਾਗੂ ਕਰਨਾ, ਪ੍ਰਾਈਵੇਸੀ ਨਾਲ ਸੰਦਰਭ ਦੀ ਜਰूरत ਦਾ ਸੰਤੁਲਨ ਕਰ ਸਕਦਾ ਹੈ।
- **ਵਿਉਕਤਗਤਕਰਨ**: ਸਿੱਖਣ ਅਤੇ ਅਨੁਕੂਲਣ ਦੀ ਸਮਰੱਥਾ ਨਾਲ, ਏਆਈ ਮਾਡਲ ਯੂਜ਼ਰ ਲਈ ਵਿਅਕਤੀਗਤ ਅਨੁਭਵ ਪ੍ਰਦਾਨ ਕਰਦਾ ਹੈ। ਯੂਜ਼ਰ ਪ੍ਰੋਫਾਇਲ ਵਰਗੇ ਫੀਚਰਾਂ ਦੁਆਰਾ ਅਨੁਭਵ ਨੂੰ ਵਿਉਕਤਗਤ ਕਰਨਾ ਸਿਰਫ ਯੂਜ਼ਰ ਨੂੰ ਸਮਝਿਆ ਹੋਇਆ ਮਹਿਸੂਸ ਨਹੀਂ ਕਰਵਾਉਂਦਾ, ਸਗੋਂ ਉਨ੍ਹਾਂ ਦੀ ਖਾਸ ਜਾਣਕਾਰੀ ਲੱਭਣ ਦੀ ਤਾਕਤ ਨੂੰ ਵੀ ਬਹਾਲ ਕਰਦਾ ਹੈ, ਜਿਸ ਨਾਲ ਗੱਲਬਾਤ ਹੋਰ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਅਤੇ ਸੰਤੁਸ਼ਟਦਾਇਕ ਬਣਦੀ ਹੈ।

ਇੱਕ ਵਿਉਕਤਗਤਕਰਨ ਦਾ ਉਦਾਹਰਨ OpenAI ਦੇ ChatGPT ਵਿੱਚ "Custom instructions" ਸੈਟਿੰਗ ਹੈ। ਇਹ ਤੁਸੀਂ ਆਪਣੇ ਬਾਰੇ ਮਹੱਤਵਪੂਰਕ ਸੰਦਰਭ ਜਾਣਕਾਰੀ ਪ੍ਰਦਾਨ ਕਰ ਸਕਦੇ ਹੋ ਜੋ ਤੁਹਾਡੇ ਪ੍ਰੰਪਟਸ ਲਈ ਲਾਜ਼ਮੀ ਹੁੰਦੀ ਹੈ। ਇਹ ਇੱਕ ਕਸਟਮ ਨਿਰਦੇਸ਼ ਦਾ ਉਦਾਹਰਨ ਹੈ।

### ਬੜੇ ਭਾਸ਼ਾਈ ਮਾਡਲਾਂ ਲਈ ਮਾਈਕ੍ਰੋਸੌਫਟ ਦੀ ਸਿਸਟਮ ਮੇਸੇਜ਼ ਫਰੇਮਵਰਕ

[ਮਾਈਕ੍ਰੋਸੌਫਟ ਨੇ ਲਗਾਤਾਰ ਵਿੱਚ ਪ੍ਰਭਾਵਸ਼ਾਲੀ ਸਿਸਟਮ ਮੇਸੇਜ਼ ਲਿਖਣ ਲਈ ਮਦਦ ਦਿੱਤੀ ਹੈ](https://learn.microsoft.com/azure/ai-services/openai/concepts/system-message#define-the-models-output-format?WT.mc_id=academic-105485-koreyst) ਜੋ LLMs ਤੋਂ ਪ੍ਰਤਿਕ੍ਰਿਆ ਮੰਗਦੀਆਂ ਸਮੇਂ ਚਾਰ ਖੇਤਰਾਂ ਵਿੱਚ ਵੰਡਿਆ ਗਿਆ ਹੈ:

1. ਇਹ ਪਰਿਭਾਸ਼ਤ ਕਰਨਾ ਕਿ ਮਾਡਲ ਕਿਹੜਿਆਂ ਲਈ ਹੈ, ਅਤੇ ਇਸ ਦੀਆਂ ਸਮਰੱਥਾਵਾਂ ਅਤੇ ਸੀਮਾਵਾਂ।
2. ਮਾਡਲ ਦੇ ਆਉਟਪੁੱਟ ਫਾਰਮੈਟ ਨੂੰ ਪਰਿਭਾਸ਼ਤ ਕਰਨਾ।
3. ਖਾਸ ਉਦਾਹਰਣ ਪ੍ਰਦਾਨ ਕਰਨਾ ਜੋ ਮਾਡਲ ਦੇ ਉਦੇਸ਼ ਵਾਲੇ ਵਿਹਾਰ ਨੂੰ ਦਰਸਾਉਂਦੇ ਹਨ।
4. ਵਾਧੂ ਵਿਹਾਰਕ ਰੋਕਥਾਮ ਪ੍ਰਦਾਨ ਕਰਨਾ।

### ਪਹੁੰਚਯੋਗਤਾ

ਭਾਵੇਂ ਕਿਸੇ ਯੂਜ਼ਰ ਨੂੰ ਦਰਸ਼ਨ, ਸ੍ਰਵਣ, ਮੋਟਰ ਜਾਂ ਸੰਗਿਆਤਮਕ ਖ਼ਾਮੀਆਂ ਹੋਣ, ਇੱਕ ਵਧੀਆ ਡਿਜ਼ਾਈਨ ਕੀਤੀ ਚੈਟ ਐਪ ਸਭ ਲਈ ਵਰਤੀ ਜਾ ਸਕਣੀ ਚਾਹੀਦੀ ਹੈ। ਹੇਠਾਂ ਦਿੱਤੀ ਸੂਚੀ ਵੱਖ-ਵੱਖ ਖ਼ਾਮੀਆਂ ਵਾਲੇ ਯੂਜ਼ਰਾਂ ਲਈ ਪਹੁੰਚ ਯੋਗਤਾਵਾਂ ਨੂੰ ਬਿਹਤਰ ਬਣਾਉਣ ਵਾਲੇ ਖਾਸ ਫੀਚਰਾਂ ਦੀ ਜਾਣਕਾਰੀ ਦਿੰਦੀ ਹੈ।

- **ਦ੍ਰਿਸ਼ਟੀ ਖ਼ਾਮੀ ਲਈ ਫੀਚਰ**: ਉੱਚ ਕੰਟਰਾਸਟ ਥੀਮਾਂ ਅਤੇ ਪਾਢ ਬਦਲਣ ਯੋਗ ਲਿਪੀ, ਸਕਰੀਨ ਰੀਡਰ ਸਮਰਥਨ।
- **ਸ੍ਰਵਣ ਖ਼ਾਮੀ ਲਈ ਫੀਚਰ**: ਟੈਕਸਟ-ਟੂ-ਸਪੀਚ ਅਤੇ ਸਪੀਚ-ਟੂ-ਟੈਕਸਟ ਫੰਕਸ਼ਨ, ਆਡੀਓ ਨੋਟਿਫਿਕੇਸ਼ਨਾਂ ਲਈ ਵਿਜ਼ੂਅਲ ਸੁਚਕ।
- **ਮੋਟਰ ਖ਼ਾਮੀ ਲਈ ਫੀਚਰ**: ਕੀਬੋਰਡ ਨੈਵੀਗੇਸ਼ਨ ਸਹਾਇਤਾ, ਆਵਾਜ਼ ਹੁਕਮ।
- **ਸੰਗਿਆਤਮਕ ਖ਼ਾਮੀ ਲਈ ਫੀਚਰ**: ਸਧਾਰਣ ਭਾਸ਼ਾ ਵਿਕਲਪ।

## ਖੇਤਰ-ਖਾਸ ਭਾਸ਼ਾ ਮਾਡਲਾਂ ਲਈ ਵਿਉਕਤਗਤਕਰਨ ਅਤੇ ਫਾਈਨ-ਟਿਊਨਿੰਗ

ਸੋਚੋ ਇੱਕ ਐਸੀ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨ ਜਿਸਨੂੰ ਤੁਹਾਡੇ ਕੰਪਨੀ ਦੇ ਜਾਰਗਨ ਦੀ ਸਮਝ ਹੋਵੇ ਅਤੇ ਜੋ ਆਪਣੇ ਯੂਜ਼ਰ ਬੇਸ ਦੀ ਆਮ কਵੈਰੀਆਂ ਦੀ ਪੂਰਵ ਅੰਦਾਜ਼ਾ ਲਗਾਉਂਦੀ ਹੋਵੇ। ਇਸ ਲਈ ਕੁਝ ਤਰੀਕੇ ਜ਼ੁਕੜੇ ਜਾਣ ਯੋਗ ਹਨ:

- **DSL ਮਾਡਲਾਂ ਦੀ ਵਰਤੋਂ**। DSL ਦਾ ਅਰਥ ਖੇਤਰ-ਖਾਸ ਭਾਸ਼ਾ ਹੈ। ਤੁਸੀਂ ਖੇਤਰ-ਖਾਸ ਮਾਡਲ ਦੀ ਵਰਤੋਂ ਕਰ ਸਕਦੇ ਹੋ ਜੋ ਉਸ ਖੇਤਰ ਦੇ ਸੰਕਲਪਾਂ ਅਤੇ ਸੰਦਰਭਾਂ ਨੂੰ ਸਮਝ ਸਕਦਾ ਹੈ।
- **ਫਾਈਨ-ਟਿਊਨਿੰਗ ਲਾਗੂ ਕਰੋ**। ਫਾਈਨ-ਟਿਊਨਿੰਗ ਉਹ ਪ੍ਰਕਿਰਿਆ ਹੈ ਜਿਸ ਵਿੱਚ ਤੁਸੀਂ ਖਾਸ ਡੇਟਾ ਨਾਲ ਮਾਡਲ ਨੂੰ ਹੋਰ ਸਿਖਾਉਂਦੇ ਹੋ।

## ਵਿਉਕਤਗਤਕਰਨ: DSL ਦੀ ਵਰਤੋਂ ਕਰਨਾ

ਖੇਤਰ-ਖਾਸ ਭਾਸ਼ਾ ਮਾਡਲ (DSL ਮਾਡਲ) ਯੂਜ਼ਰ ਇੰਟਰਐਕਸ਼ਨ ਵਿੱਚ ਮਾਹਰ, ਸੰਦਰਭ-ਸੂਚੀਬੱਧ ਗੱਲਬਾਤ ਪ੍ਰਦਾਨ ਕਰਕੇ ਵਿਉਕਤ ਗ੍ਰਹਿਣਯੋਗਤਾ ਨੂੰ ਬਢ਼ਾ ਸਕਦੇ ਹਨ। ਇਹ ਉਹ ਮਾਡਲ ਹੈ ਜਿਨ੍ਹਾਂ ਨੂੰ ਕਿਸੇ ਖਾਸ ਖੇਤਰ, ਉਦਯੋਗ ਜਾਂ ਵਿਸ਼ੇ ਦੀ ਸਮਝ ਅਤੇ ਲਿਖਤ ਤਿਆਰ ਕਰਨ ਲਈ ਤਿਆਰ ਜਾਂ ਫਾਈਨ-ਟਿਊਨ ਕੀਤਾ ਜਾਂਦਾ ਹੈ। DSL ਮਾਡਲ ਦੀ ਵਰਤੋਂ ਲਈ ਵਿਕਲਪ ਵੱਖ-ਵੱਖ ਹਨ ਜੋ ਨਵੀਨਤਮ ਮਾਡਲ ਦੀ ਸਿਖਲਾਈ ਤੋਂ ਲੈ ਕੇ SDKs ਅਤੇ APIs ਦਾ ਉਪਯੋਗ ਕਰਨ ਤੱਕ ਜਾ ਸਕਦੇ ਹਨ। ਇੱਕ ਹੋਰ ਵਿਕਲਪ ਫਾਈਨ-ਟਿਊਨਿੰਗ ਹੈ, ਜੋ ਮੌਜੂਦਾ ਪ੍ਰੀ-ਟਰੇਨਡ ਮਾਡਲ ਨੂੰ ਖੇਤਰ-ਖਾਸ ਡੇਟਾ ਲਈ ਅਨੁਕੂਲਿਤ ਕਰਨ ਦਾ ਤਰੀਕਾ ਹੈ।

## ਵਿਉਕਤਗਤਕਰਨ: ਫਾਈਨ-ਟਿਊਨਿੰਗ ਲਾਗੂ ਕਰੋ

ਜਦੋਂ ਇੱਕ ਪ੍ਰੀ-ਟਰੇਨਡ ਮਾਡਲ ਕਿਸੇ ਖੇਤਰ ਜਾਂ ਖਾਸ ਕੰਮ ਵਿੱਚ ਅਪਰੇਫਾਇਟ ਸਾਬਿਤ ਹੁੰਦਾ ਹੈ ਤਾਂ ਫਾਈਨ-ਟਿਊਨਿੰਗ ਦੀ ਚਿੰਤਾ ਕੀਤੀ ਜਾਂਦੀ ਹੈ।

ਉਦਾਹਰਨ ਲਈ, ਮੈਡੀਕਲ ਸਵਾਲ ਜਟਿਲ ਹੁੰਦੇ ਹਨ ਅਤੇ ਕਾਫੀ ਸੰਦਰਭ ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ। ਜਦੋਂ ਇੱਕ ਡਾਕਟਰ ਕਿਸੇ ਮਰੀਜ਼ ਦਾ ਨਿਰਣਾ ਕਰਦਾ ਹੈ, ਉਹ ਜੀਵਨ ਸ਼ੈਲੀ ਜਾਂ ਮੌਜੂਦਾ ਬੀਮਾਰੀਆਂ ਵਰਗੇ ਕਈ ਕਾਰਕਾਂ 'ਤੇ ਭਰੋਸਾ ਕਰਦਾ ਹੈ ਅਤੇ ਤਾਜ਼ਾ ਮੈਡੀਕਲ ਜਰਨਲਾਂ ਵੀ ਇਸਦਾ ਫੈਸਲਾ ਅਦਾਇਗੀ ਲਈ ਵਰਤੀ ਜਾਂਦੀ ਹੈ। ਅਜਿਹੇ ਜਟਿਲ ਮਾਮਲਿਆਂ ਵਿੱਚ, ਇੱਕ ਆਮ ਇਰਾਦੇ ਵਾਲੀ ਏਆਈ ਚੈਟ ਐਪਲਿਕੇਸ਼ਨ ਭਰੋਸੇਯੋਗ ਸਰੋਤ ਨਹੀਂ ਹੋ ਸਕਦੀ।

### ਦਰਸ਼ਾ: ਇਕ ਮੈਡੀਕਲ ਐਪਲੀਕੇਸ਼ਨ

ਸੋਚੋ ਇੱਕ ਐਸੀ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨ ਜੋ ਮੈਡੀਕਲ ਪ੍ਰੈਕਟਿਸ਼ਨਰਾਂ ਨੂੰ ਇਲਾਜ ਦੀਆਂ ਪਹੁੰਚ, ਦਵਾਈਆਂ ਦੇ ਤੱਤੇ ਕ੍ਰਿਆਵਾਂ ਜਾਂ ਤਾਜ਼ਾ ਸ਼ੋਧ ਦੇ ਤੱਥ ਦਿੰਦੀ ਹੈ।

ਆਮ ਮਕਸਦ ਵਾਲਾ ਮਾਡਲ ਆਮ ਮੈਡੀਕਲ ਸਵਾਲਾਂ ਦਾ ਜਵਾਬ ਦੇ ਸਕਦਾ ਹੈ ਜਾਂ ਆਮ ਸਲਾਹ ਦੇ ਸਕਦਾ ਹੈ, ਪਰ ਇਹ ਹੇਠਾਂ ਦਿੱਤੀਆਂ ਚੀਜ਼ਾਂ ਵਿੱਚ ਮੁਸ਼ਕਲ ਦਾ ਸਾਹਮਣਾ ਕਰ ਸਕਦਾ ਹੈ:

- **ਬਹੁਤ ਖਾਸ ਜਾਂ ਜਟਿਲ ਮਾਮਲੇ**। ਉਦਾਹਰਨ ਲਈ, ਇੱਕ ਨ੍ਯੂਰੋਲੋਜਿਸਟ ਐਪ ਨੂੰ ਪੁੱਛ ਸਕਦਾ ਹੈ, "ਬੱਚਿਆਂ ਵਿੱਚ ਦਵਾਈ-ਰੋਧਕ ਮਿਰਗੀ ਲਈ ਸਭ ਤੋਂ ਵਧੀਆ ਪ੍ਰਬੰਧ ਕੀ ਹਨ?"
- **ਤਾਜ਼ਾ ਤਰੱਕੀਆਂ ਦੀ ਘਾਟ**। ਆਮ ਮਕਸਦ ਮਾਡਲ ਨਿਉਰੋਲੋਜੀ ਅਤੇ ਫਾਰਮਕੋਲੋਜੀ ਵਿੱਚ ਹੁਣ ਤੱਕ ਦੀਆਂ ਤਾਜ਼ਾ ਤਰੱਕੀਆਂ ਦੇ ਸਹੀ ਜਵਾਬ ਦੇਣ ਵਿੱਚ ਅਸਮਰੱਥ ਰਹਿ ਸਕਦਾ ਹੈ।

ਅਜਿਹੇ ਮਾਮਲਿਆਂ ਵਿੱਚ, ਖੇਤਰ-ਖਾਸ ਮੈਡੀਕਲ ਡੇਟਾਸੈਟ ਨਾਲ ਮਾਡਲ ਦੀ ਫਾਈਨ-ਟਿਊਨਿੰਗ ਇਸ ਦੀ ਸਮਰੱਥਾ ਨੂੰ ਕਾਫੀ ਬਿਹਤਰ ਬਣਾ ਸਕਦੀ ਹੈ, ਤਾਂ ਜੋ ਇਹ ਸ਼ਿਕਸਤਮਯ ਮੈਡੀਕਲ ਪ੍ਰਸ਼ਨਾਂ ਨੂੰ ਬਹੁਤ ਸਹੀ ਅਤੇ ਭਰੋਸੇਯੋਗ ਢੰਗ ਨਾਲ ਹੱਲ ਕਰ ਸਕੇ। ਇਸ ਲਈ ਇਕ ਵਿਸ਼ਾਲ ਅਤੇ ਲੋੜੀਂਦਾ ਡੇਟਾਸੈਟ ਦੀ ਲੋੜ ਹੈ ਜੋ ਖੇਤਰ ਖਾਸ ਚੁਣੌਤੀਆਂ ਅਤੇ ਸਵਾਲਾਂ ਦੀ ਪ੍ਰਤੀਨਿਧੀ ਕਰਦਾ ਹੋਵੇ।

## ਉੱਚ ਗੁਣਵੱਤਾ ਵਾਲੇ ਏਆਈ-ਚਾਲਿਤ ਚੈਟ ਅਨੁਭਵ ਲਈ ਵਿਚਾਰ

ਇਸ ਭਾਗ ਵਿੱਚ "ਉੱਚ ਗੁਣਵੱਤਾ" ਵਾਲੀਆਂ ਚੈਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਲਈ ਮਾਪਦੰਡ ਦਿੱਤੇ ਗਏ ਹਨ, ਜਿਨ੍ਹਾਂ ਵਿੱਚ ਕਿਰਿਆਸ਼ੀਲ ਮਾਪਦੰਡਾਂ ਦੀ хв្មਿਲ ਅਤੇ ਏਆਈ ਤਕਨਾਲੋਜੀ ਨੂੰ ਜ਼ਿੰਮੇਵਾਰ ਢੰਗ ਨਾਲ ਵਰਤਣ ਵਾਲੇ ਫਰੇਮਵਰਕ ਦੀ ਪਾਲਣਾ ਸ਼ਾਮਿਲ ਹੈ।

### ਚੈਟ ਐਪਲੀਕੇਸ਼ਨਾਂ ਵਿੱਚ ਜ਼ਿੰਮੇਵਾਰ AI ਅਭਿਆਸ ਲਾਗੂ ਕਰਨਾ

ਮਾਈਕ੍ਰੋਸਾਫਟ ਦੇ ਜ਼ਿੰਮੇਵਾਰ AI ਲਈ ਦ੍ਰਿਸ਼ਟੀਕੋਣ ਨੇ ਛੇ ਸਿਧਾਂਤ ਪਛਾਣੇ ਹਨ ਜੋ AI ਵਿਕਾਸ ਅਤੇ ਇਸਦੇ ਇਸਤੇਮਾਲ ਨੂੰ ਮਾਰਗਦਰਸ਼ਿਤ ਕਰਨੇ ਚਾਹੀਦੇ ਹਨ। ਹੇਠਾਂ ਇਹ ਸਿਧਾਂਤ, ਉਨ੍ਹਾਂ ਦੀ ਪਰਿਭਾਸ਼ਾ, ਅਤੇ ਚੈਟ ਡਿਵੈਲਪਰ ਲਈ ਵਿਚਾਰ ਕਰਨ ਵਾਲੀਆਂ ਗੱਲਾਂ ਹਨ ਅਤੇ ਉਹ ਇਹਨਾਂ ਨੂੰ ਗੰਭੀਰਤਾ ਨਾਲ ਕਿਉਂ ਲੈਣ।
