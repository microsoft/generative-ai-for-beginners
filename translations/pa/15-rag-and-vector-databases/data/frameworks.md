# ਨਿਊਰਲ ਨੈੱਟਵਰਕ ਫਰੇਮਵਰਕ

ਜਿਵੇਂ ਕਿ ਅਸੀਂ ਪਹਿਲਾਂ ਹੀ ਸਿੱਖ ਚੁੱਕੇ ਹਾਂ, ਨਿਊਰਲ ਨੈੱਟਵਰਕ ਨੂੰ ਕੁਸ਼ਲਤਾ ਨਾਲ ਟ੍ਰੇਨ ਕਰਨ ਲਈ ਸਾਨੂੰ ਦੋ ਚੀਜ਼ਾਂ ਕਰਨ ਦੀ ਲੋੜ ਹੈ:

* ਟੈਂਸਰਾਂ 'ਤੇ ਕੰਮ ਕਰਨ ਲਈ, ਜਿਵੇਂ ਕਿ ਗੁਣਾ, ਜੋੜਨਾ, ਅਤੇ ਕੁਝ ਫੰਕਸ਼ਨ ਜਿਵੇਂ ਸਿਗਮੋਇਡ ਜਾਂ ਸਾਫਟਮੈਕਸ ਦੀ ਗਣਨਾ ਕਰਨਾ
* ਸਾਰੇ ਐਕਸਪ੍ਰੈਸ਼ਨਾਂ ਦੇ ਗਰੇਡੀਅੰਟ ਦੀ ਗਣਨਾ ਕਰਨਾ, ਤਾਂ ਜੋ ਗਰੇਡੀਅੰਟ ਡਿਸੈਂਟ ਅਪਟੀਮਾਈਜ਼ੇਸ਼ਨ ਕੀਤਾ ਜਾ ਸਕੇ

ਜਦੋਂ ਕਿ `numpy` ਲਾਇਬ੍ਰੇਰੀ ਪਹਿਲੇ ਹਿੱਸੇ ਨੂੰ ਕਰ ਸਕਦੀ ਹੈ, ਸਾਨੂੰ ਗਰੇਡੀਅੰਟ ਦੀ ਗਣਨਾ ਕਰਨ ਲਈ ਕੁਝ ਮਕੈਨਿਜ਼ਮ ਦੀ ਲੋੜ ਹੈ। ਸਾਡੇ ਫਰੇਮਵਰਕ ਵਿੱਚ ਜੋ ਅਸੀਂ ਪਿਛਲੇ ਭਾਗ ਵਿੱਚ ਵਿਕਸਿਤ ਕੀਤਾ ਸੀ, ਸਾਨੂੰ ਸਾਰੇ ਡਿਰਿਵੇਟਿਵ ਫੰਕਸ਼ਨਾਂ ਨੂੰ ਹੱਥੋਂ ਹੱਥ ਪ੍ਰੋਗਰਾਮ ਕਰਨਾ ਪਿਆ ਸੀ `backward` ਵਿਧੀ ਦੇ ਅੰਦਰ, ਜੋ ਬੈਕਪ੍ਰੋਪਾਗੇਸ਼ਨ ਕਰਦੀ ਹੈ। ਆਦਰਸ਼ ਤੌਰ 'ਤੇ, ਇੱਕ ਫਰੇਮਵਰਕ ਸਾਨੂੰ *ਕਿਸੇ ਵੀ ਐਕਸਪ੍ਰੈਸ਼ਨ* ਦੇ ਗਰੇਡੀਅੰਟ ਦੀ ਗਣਨਾ ਕਰਨ ਦਾ ਮੌਕਾ ਦੇਣਾ ਚਾਹੀਦਾ ਹੈ ਜਿਸਨੂੰ ਅਸੀਂ ਪਰਿਭਾਸ਼ਿਤ ਕਰ ਸਕਦੇ ਹਾਂ।

ਇੱਕ ਹੋਰ ਮਹੱਤਵਪੂਰਨ ਗੱਲ ਇਹ ਹੈ ਕਿ GPU ਜਾਂ ਕਿਸੇ ਹੋਰ ਵਿਸ਼ੇਸ਼ਕ੍ਰਿਤ ਗਣਨਾ ਯੂਨਿਟਾਂ, ਜਿਵੇਂ TPU, 'ਤੇ ਗਣਨਾਵਾਂ ਕਰਨ ਦੇ ਯੋਗ ਹੋਣਾ। ਡੀਪ ਨਿਊਰਲ ਨੈੱਟਵਰਕ ਟ੍ਰੇਨਿੰਗ ਲਈ *ਬਹੁਤ ਸਾਰੀਆਂ* ਗਣਨਾਵਾਂ ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ, ਅਤੇ GPU 'ਤੇ ਉਹਨਾਂ ਗਣਨਾਵਾਂ ਨੂੰ ਪੈਰਲੇਲ ਕਰਨ ਦੇ ਯੋਗ ਹੋਣਾ ਬਹੁਤ ਮਹੱਤਵਪੂਰਨ ਹੈ।

> ✅ 'ਪੈਰਲੇਲਾਈਜ਼' ਦਾ ਅਰਥ ਹੈ ਕਿ ਗਣਨਾਵਾਂ ਨੂੰ ਕਈ ਡਿਵਾਈਸਾਂ 'ਤੇ ਵੰਡਿਆ ਜਾਵੇ।

ਵਰਤਮਾਨ ਵਿੱਚ, ਦੋ ਸਭ ਤੋਂ ਲੋਕਪ੍ਰਿਯ ਨਿਊਰਲ ਫਰੇਮਵਰਕ ਹਨ: TensorFlow ਅਤੇ PyTorch। ਦੋਵੇਂ CPU ਅਤੇ GPU ਦੋਵਾਂ 'ਤੇ ਟੈਂਸਰਾਂ ਨਾਲ ਕੰਮ ਕਰਨ ਲਈ ਇੱਕ ਨੀਵੀਂ ਪੱਧਰ ਦੀ API ਪ੍ਰਦਾਨ ਕਰਦੇ ਹਨ। ਨੀਵੀਂ ਪੱਧਰ ਦੀ API ਦੇ ਉੱਪਰ, ਇੱਕ ਉੱਚ ਪੱਧਰ ਦੀ API ਵੀ ਹੈ, ਜਿਸਨੂੰ ਮੁੜ ਕ੍ਰਮਵਾਰ Keras ਅਤੇ PyTorch ਲਾਇਟਨਿੰਗ ਕਿਹਾ ਜਾਂਦਾ ਹੈ।

ਨੀਵੀਂ ਪੱਧਰ ਦੀ API | TensorFlow | PyTorch
--------------|-------------------------------------|--------------------------------
ਉੱਚ ਪੱਧਰ ਦੀ API| Keras | PyTorch

**ਨੀਵੀਂ ਪੱਧਰ ਦੀਆਂ APIs** ਦੋਵੇਂ ਫਰੇਮਵਰਕਾਂ ਵਿੱਚ ਤੁਹਾਨੂੰ ਕਥਿਤ **ਗਣਨਾਤਮਕ ਗ੍ਰਾਫ** ਬਣਾਉਣ ਦੀ ਆਗਿਆ ਦਿੰਦੇ ਹਨ। ਇਹ ਗ੍ਰਾਫ ਪਰਿਭਾਸ਼ਿਤ ਕਰਦਾ ਹੈ ਕਿ ਦਿੱਤੇ ਗਏ ਇਨਪੁਟ ਪੈਰਾਮੀਟਰਾਂ ਨਾਲ ਆਉਟਪੁੱਟ (ਆਮ ਤੌਰ 'ਤੇ ਨੁਕਸਾਨ ਫੰਕਸ਼ਨ) ਕਿਵੇਂ ਗਣਨਾ ਕਰਨੀ ਹੈ, ਅਤੇ ਜੇਕਰ ਇਹ ਉਪਲਬਧ ਹੈ ਤਾਂ GPU 'ਤੇ ਗਣਨਾ ਲਈ ਧੱਕਿਆ ਜਾ ਸਕਦਾ ਹੈ। ਇਸ ਗਣਨਾਤਮਕ ਗ੍ਰਾਫ ਨੂੰ ਵੱਖਰਾ ਕਰਨ ਅਤੇ ਗਰੇਡੀਅੰਟ ਦੀ ਗਣਨਾ ਕਰਨ ਲਈ ਫੰਕਸ਼ਨ ਹਨ, ਜੋ ਫਿਰ ਮਾਡਲ ਪੈਰਾਮੀਟਰਾਂ ਨੂੰ ਅਪਟੀਮਾਈਜ਼ ਕਰਨ ਲਈ ਵਰਤੇ ਜਾ ਸਕਦੇ ਹਨ।

**ਉੱਚ ਪੱਧਰ ਦੀਆਂ APIs** ਅਕਸਰ ਨਿਊਰਲ ਨੈੱਟਵਰਕ ਨੂੰ **ਪਰਤਾਂ ਦੇ ਕ੍ਰਮ** ਵਜੋਂ ਮੰਨਦੇ ਹਨ, ਅਤੇ ਜ਼ਿਆਦਾਤਰ ਨਿਊਰਲ ਨੈੱਟਵਰਕਾਂ ਦਾ ਨਿਰਮਾਣ ਕਰਨਾ ਬਹੁਤ ਆਸਾਨ ਬਣਾ ਦਿੰਦੇ ਹਨ। ਮਾਡਲ ਨੂੰ ਟ੍ਰੇਨ ਕਰਨ ਲਈ ਆਮ ਤੌਰ 'ਤੇ ਡਾਟਾ ਤਿਆਰ ਕਰਨਾ ਅਤੇ ਫਿਰ `fit` ਫੰਕਸ਼ਨ ਨੂੰ ਕਾਲ ਕਰਨਾ ਲੋੜੀਂਦਾ ਹੈ।

ਉੱਚ ਪੱਧਰ ਦੀ API ਤੁਹਾਨੂੰ ਆਮ ਨਿਊਰਲ ਨੈੱਟਵਰਕ ਬਹੁਤ ਤੇਜ਼ੀ ਨਾਲ ਬਣਾਉਣ ਦੀ ਆਗਿਆ ਦਿੰਦੀ ਹੈ ਬਿਨਾਂ ਬਹੁਤ ਸਾਰੀਆਂ ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਦੀ ਚਿੰਤਾ ਕੀਤੇ। ਇਸਦੇ ਨਾਲ ਨਾਲ, ਨੀਵੀਂ ਪੱਧਰ ਦੀ API ਟ੍ਰੇਨਿੰਗ ਪ੍ਰਕਿਰਿਆ 'ਤੇ ਬਹੁਤ ਜ਼ਿਆਦਾ ਨਿਯੰਤਰਣ ਪ੍ਰਦਾਨ ਕਰਦੀ ਹੈ, ਅਤੇ ਇਸ ਲਈ ਇਹ ਖੋਜ ਵਿੱਚ ਬਹੁਤ ਵਰਤੀ ਜਾਂਦੀ ਹੈ, ਜਦੋਂ ਤੁਸੀਂ ਨਵੀਆਂ ਨਿਊਰਲ ਨੈੱਟਵਰਕ ਸਥਾਪਤੀਆਂ ਨਾਲ ਨਜਿੱਠ ਰਹੇ ਹੋ।

ਇਹ ਸਮਝਣਾ ਵੀ ਮਹੱਤਵਪੂਰਨ ਹੈ ਕਿ ਤੁਸੀਂ ਦੋਵੇਂ APIs ਨੂੰ ਇਕੱਠੇ ਵਰਤ ਸਕਦੇ ਹੋ, ਉਦਾਹਰਣ ਲਈ, ਤੁਸੀਂ ਨੀਵੀਂ ਪੱਧਰ ਦੀ API ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਆਪਣੀ ਆਪਣੀ ਨੈੱਟਵਰਕ ਪਰਤ ਸਥਾਪਤੀ ਦਾ ਵਿਕਾਸ ਕਰ ਸਕਦੇ ਹੋ, ਅਤੇ ਫਿਰ ਇਸਨੂੰ ਵੱਡੇ ਨੈੱਟਵਰਕ ਦੇ ਅੰਦਰ ਵਰਤ ਸਕਦੇ ਹੋ ਜੋ ਉੱਚ ਪੱਧਰ ਦੀ API ਨਾਲ ਬਣਾਇਆ ਗਿਆ ਹੈ ਅਤੇ ਟ੍ਰੇਨ ਕੀਤਾ ਗਿਆ ਹੈ। ਜਾਂ ਤੁਸੀਂ ਉੱਚ ਪੱਧਰ ਦੀ API ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਨੈੱਟਵਰਕ ਨੂੰ ਪਰਤਾਂ ਦੇ ਕ੍ਰਮ ਵਜੋਂ ਪਰਿਭਾਸ਼ਿਤ ਕਰ ਸਕਦੇ ਹੋ, ਅਤੇ ਫਿਰ ਆਪਣੇ ਨੀਵੀਂ ਪੱਧਰ ਦੀ ਟ੍ਰੇਨਿੰਗ ਲੂਪ ਦੀ ਵਰਤੋਂ ਕਰਕੇ ਅਪਟੀਮਾਈਜ਼ੇਸ਼ਨ ਕਰ ਸਕਦੇ ਹੋ। ਦੋਵੇਂ APIs ਇੱਕੋ ਜਿਹੇ ਬੁਨਿਆਦੀ ਅਧਾਰਭੂਤ ਧਾਰਨਾਵਾਂ ਦੀ ਵਰਤੋਂ ਕਰਦੇ ਹਨ, ਅਤੇ ਉਹਨਾਂ ਨੂੰ ਚੰਗੀ ਤਰ੍ਹਾਂ ਇਕੱਠੇ ਕੰਮ ਕਰਨ ਲਈ ਡਿਜ਼ਾਈਨ ਕੀਤਾ ਗਿਆ ਹੈ।

## ਸਿੱਖਣਾ

ਇਸ ਕੋਰਸ ਵਿੱਚ, ਅਸੀਂ PyTorch ਅਤੇ TensorFlow ਦੋਵਾਂ ਲਈ ਜ਼ਿਆਦਾਤਰ ਸਮੱਗਰੀ ਦੀ ਪੇਸ਼ਕਸ਼ ਕਰਦੇ ਹਾਂ। ਤੁਸੀਂ ਆਪਣਾ ਪਸੰਦੀਦਾ ਫਰੇਮਵਰਕ ਚੁਣ ਸਕਦੇ ਹੋ ਅਤੇ ਸਿਰਫ਼ ਸੰਬੰਧਿਤ ਨੋਟਬੁੱਕਾਂ ਵਿੱਚੋਂ ਗੁਜ਼ਰ ਸਕਦੇ ਹੋ। ਜੇ ਤੁਸੀਂ ਪੱਕਾ ਨਹੀਂ ਹੋ ਕਿ ਕਿਹੜਾ ਫਰੇਮਵਰਕ ਚੁਣਨਾ ਹੈ, ਤਾਂ **PyTorch ਵਿਰੁੱਧ TensorFlow** ਬਾਰੇ ਇੰਟਰਨੈਟ 'ਤੇ ਕੁਝ ਚਰਚਾਵਾਂ ਪੜ੍ਹੋ। ਤੁਸੀਂ ਦੋਵੇਂ ਫਰੇਮਵਰਕਾਂ 'ਤੇ ਵੀ ਇੱਕ ਨਜ਼ਰ ਮਾਰ ਸਕਦੇ ਹੋ ਤਾਂ ਜੋ ਵਧੇਰੇ ਸਮਝ ਪ੍ਰਾਪਤ ਹੋ ਸਕੇ।

ਜਿੱਥੇ ਸੰਭਵ ਹੋਵੇ, ਅਸੀਂ ਸਰਲਤਾ ਲਈ ਉੱਚ ਪੱਧਰ ਦੀਆਂ APIs ਦੀ ਵਰਤੋਂ ਕਰਾਂਗੇ। ਹਾਲਾਂਕਿ, ਅਸੀਂ ਮੰਨਦੇ ਹਾਂ ਕਿ ਇਹ ਸਮਝਣਾ ਮਹੱਤਵਪੂਰਨ ਹੈ ਕਿ ਨਿਊਰਲ ਨੈੱਟਵਰਕ ਜ਼ਮੀਨ ਤੋਂ ਕਿਵੇਂ ਕੰਮ ਕਰਦੇ ਹਨ, ਇਸ ਲਈ ਸ਼ੁਰੂ ਵਿੱਚ ਅਸੀਂ ਨੀਵੀਂ ਪੱਧਰ ਦੀ API ਅਤੇ ਟੈਂਸਰਾਂ ਨਾਲ ਕੰਮ ਕਰਕੇ ਸ਼ੁਰੂ ਕਰਦੇ ਹਾਂ। ਹਾਲਾਂਕਿ, ਜੇ ਤੁਸੀਂ ਤੇਜ਼ੀ ਨਾਲ ਅੱਗੇ ਵਧਣਾ ਚਾਹੁੰਦੇ ਹੋ ਅਤੇ ਇਹ ਵਿਸ਼ੇਸ਼ਤਾਵਾਂ ਸਿੱਖਣ 'ਤੇ ਬਹੁਤ ਜ਼ਿਆਦਾ ਸਮਾਂ ਨਹੀਂ ਬਿਤਾਉਣਾ ਚਾਹੁੰਦੇ, ਤਾਂ ਤੁਸੀਂ ਉਹਨਾਂ ਨੂੰ ਛੱਡ ਸਕਦੇ ਹੋ ਅਤੇ ਸਿੱਧੇ ਉੱਚ ਪੱਧਰ ਦੀ API ਨੋਟਬੁੱਕਾਂ ਵਿੱਚ ਜਾ ਸਕਦੇ ਹੋ।

## ✍️ ਅਭਿਆਸ: ਫਰੇਮਵਰਕ

ਹੇਠਾਂ ਦਿੱਤੀਆਂ ਨੋਟਬੁੱਕਾਂ ਵਿੱਚ ਆਪਣਾ ਸਿੱਖਣਾ ਜਾਰੀ ਰੱਖੋ:

ਨੀਵੀਂ ਪੱਧਰ ਦੀ API | TensorFlow+Keras ਨੋਟਬੁੱਕ | PyTorch
--------------|-------------------------------------|--------------------------------
ਉੱਚ ਪੱਧਰ ਦੀ API| Keras | *PyTorch Lightning*

ਫਰੇਮਵਰਕਾਂ ਵਿੱਚ ਮਾਹਰ ਬਣਨ ਦੇ ਬਾਅਦ, ਆਓ ਫਿਰ ਤੋਂ ਓਵਰਫਿਟਿੰਗ ਦੀ ਧਾਰਨਾ ਦਾ ਸਮੀਖਿਆ ਕਰੀਏ।

# ਓਵਰਫਿਟਿੰਗ

ਓਵਰਫਿਟਿੰਗ ਮਸ਼ੀਨ ਲਰਨਿੰਗ ਵਿੱਚ ਇੱਕ ਬਹੁਤ ਹੀ ਮਹੱਤਵਪੂਰਨ ਧਾਰਨਾ ਹੈ, ਅਤੇ ਇਸਨੂੰ ਸਹੀ ਤਰ੍ਹਾਂ ਸਮਝਣਾ ਬਹੁਤ ਮਹੱਤਵਪੂਰਨ ਹੈ!

ਹੇਠਾਂ ਦਿੱਤੇ ਸਮੱਸਿਆ ਨੂੰ ਸੋਚੋ ਜਿੱਥੇ 5 ਬਿੰਦੂਆਂ (ਗ੍ਰਾਫਾਂ ਦੇ ਉੱਤੇ `x` ਦੁਆਰਾ ਦਰਸਾਏ ਗਏ) ਦਾ ਅਨੁਮਾਨ ਲਗਾਉਣਾ ਹੈ:

!ਲਿਨੀਅਰ | ਓਵਰਫਿਟ
-------------------------|--------------------------
**ਲਿਨੀਅਰ ਮਾਡਲ, 2 ਪੈਰਾਮੀਟਰ** | **ਗੈਰ-ਲਿਨੀਅਰ ਮਾਡਲ, 7 ਪੈਰਾਮੀਟਰ**
ਟ੍ਰੇਨਿੰਗ ਗਲਤੀ = 5.3 | ਟ੍ਰੇਨਿੰਗ ਗਲਤੀ = 0
ਵੈਲੀਡੇਸ਼ਨ ਗਲਤੀ = 5.1 | ਵੈਲੀਡੇਸ਼ਨ ਗਲਤੀ = 20

* ਖੱਬੇ ਪਾਸੇ, ਅਸੀਂ ਇੱਕ ਵਧੀਆ ਸਿੱਧੀ ਲਾਈਨ ਅਨੁਮਾਨ ਵੇਖਦੇ ਹਾਂ। ਕਿਉਂਕਿ ਪੈਰਾਮੀਟਰਾਂ ਦੀ ਗਿਣਤੀ ਉਚਿਤ ਹੈ, ਮਾਡਲ ਬਿੰਦੂ ਵੰਡ ਦੇ ਪਿੱਛੇ ਵਾਸਤਵਿਕਤਾ ਨੂੰ ਸਹੀ ਤਰ੍ਹਾਂ ਸਮਝਦਾ ਹੈ।
* ਸੱਜੇ ਪਾਸੇ, ਮਾਡਲ ਬਹੁਤ ਜ਼ਿਆਦਾ ਸ਼ਕਤੀਸ਼ਾਲੀ ਹੈ। ਕਿਉਂਕਿ ਸਾਡੇ ਕੋਲ ਸਿਰਫ 5 ਬਿੰਦੂ ਹਨ ਅਤੇ ਮਾਡਲ ਵਿੱਚ 7 ਪੈਰਾਮੀਟਰ ਹਨ, ਇਹ ਇਸ ਤਰੀਕੇ ਨਾਲ ਸਮਾਂਜਿਤ ਹੋ ਸਕਦਾ ਹੈ ਕਿ ਸਾਰੇ ਬਿੰਦੂਆਂ ਵਿੱਚੋਂ ਲੰਘ ਸਕੇ, ਜਿਸ ਨਾਲ ਟ੍ਰੇਨਿੰਗ ਗਲਤੀ 0 ਹੋ ਜਾਂਦੀ ਹੈ। ਹਾਲਾਂਕਿ, ਇਹ ਮਾਡਲ ਨੂੰ ਡਾਟਾ ਦੇ ਪਿੱਛੇ ਸਹੀ ਪੈਟਰਨ ਨੂੰ ਸਮਝਣ ਤੋਂ ਰੋਕਦਾ ਹੈ, ਇਸ ਲਈ ਵੈਲੀਡੇਸ਼ਨ ਗਲਤੀ ਬਹੁਤ ਉੱਚੀ ਹੁੰਦੀ ਹੈ।

ਮਾਡਲ ਦੀ ਸੰਪੰਨਤਾ (ਪੈਰਾਮੀਟਰਾਂ ਦੀ ਗਿਣਤੀ) ਅਤੇ ਟ੍ਰੇਨਿੰਗ ਨਮੂਨਿਆਂ ਦੀ ਗਿਣਤੀ ਵਿਚਕਾਰ ਸਹੀ ਸੰਤੁਲਨ ਬਨਾਉਣਾ ਬਹੁਤ ਮਹੱਤਵਪੂਰਨ ਹੈ।

## ਕਿਉਂ ਓਵਰਫਿਟਿੰਗ ਹੁੰਦੀ ਹੈ

  * ਟ੍ਰੇਨਿੰਗ ਡਾਟਾ ਦੀ ਕਮੀ
  * ਬਹੁਤ ਜ਼ਿਆਦਾ ਸ਼ਕਤੀਸ਼ਾਲੀ ਮਾਡਲ
  * ਇਨਪੁਟ ਡਾਟਾ ਵਿੱਚ ਬਹੁਤ ਜ਼ਿਆਦਾ ਸ਼ੋਰ

## ਓਵਰਫਿਟਿੰਗ ਨੂੰ ਕਿਵੇਂ ਪਛਾਣਿਆ ਜਾਵੇ

ਜਿਵੇਂ ਤੁਸੀਂ ਉੱਪਰ ਦਿੱਤੇ ਗ੍ਰਾਫ ਤੋਂ ਦੇਖ ਸਕਦੇ ਹੋ, ਓਵਰਫਿਟਿੰਗ ਨੂੰ ਬਹੁਤ ਘੱਟ ਟ੍ਰੇਨਿੰਗ ਗਲਤੀ ਅਤੇ ਉੱਚੀ ਵੈਲੀਡੇਸ਼ਨ ਗਲਤੀ ਦੁਆਰਾ ਪਛਾਣਿਆ ਜਾ ਸਕਦਾ ਹੈ। ਆਮ ਤੌਰ 'ਤੇ ਟ੍ਰੇਨਿੰਗ ਦੌਰਾਨ ਅਸੀਂ ਦੇਖਾਂਗੇ ਕਿ ਟ੍ਰੇਨਿੰਗ ਅਤੇ ਵੈਲੀਡੇਸ਼ਨ ਦੋਵੇਂ ਗਲਤੀਆਂ ਘਟਣ ਲੱਗਦੀਆਂ ਹਨ, ਅਤੇ ਫਿਰ ਕਿਸੇ ਸਮੇਂ ਵੈਲੀਡੇਸ਼ਨ ਗਲਤੀ ਘਟਣ ਤੋਂ ਰੁਕ ਸਕਦੀ ਹੈ ਅਤੇ ਵੱਧਣ ਲੱਗ ਸਕਦੀ ਹੈ। ਇਹ ਓਵਰਫਿਟਿੰਗ ਦੀ ਨਿਸ਼ਾਨੀ ਹੋਵੇਗੀ, ਅਤੇ ਇਹ ਸੰਕੇਤਕਰਤਾ ਕਿ ਸਾਨੂੰ ਸ਼ਾਇਦ ਇਸ ਸਮੇਂ ਟ੍ਰੇਨਿੰਗ ਰੋਕ ਦੇਣੀ ਚਾਹੀਦੀ ਹੈ (ਜਾਂ ਘੱਟੋ-ਘੱਟ ਮਾਡਲ ਦਾ ਸਨੈਪਸ਼ਾਟ ਲੈ ਲੈਣਾ ਚਾਹੀਦਾ ਹੈ)।

## ਓਵਰਫਿਟਿੰਗ ਨੂੰ ਕਿਵੇਂ ਰੋਕਿਆ ਜਾਵੇ

ਜੇ ਤੁਸੀਂ ਦੇਖਦੇ ਹੋ ਕਿ ਓਵਰਫਿਟਿੰਗ ਹੁੰਦੀ ਹੈ, ਤਾਂ ਤੁਸੀਂ ਹੇਠਾਂ ਦਿੱਤਿਆਂ ਵਿੱਚੋਂ ਕੋਈ ਇੱਕ ਕਰ ਸਕਦੇ ਹੋ:

 * ਟ੍ਰੇਨਿੰਗ ਡਾਟਾ ਦੀ ਮਾਤਰਾ ਵਧਾਓ
 * ਮਾਡਲ ਦੀ ਜਟਿਲਤਾ ਘਟਾਓ
 * ਕੁਝ ਨਿਯਮਿਤਕਰਨ ਤਕਨੀਕ ਦੀ ਵਰਤੋਂ ਕਰੋ, ਜਿਵੇਂ ਕਿ ਡਰੌਪਆਉਟ, ਜਿਸਨੂੰ ਅਸੀਂ ਬਾਅਦ ਵਿੱਚ ਵਿਚਾਰਾਂਗੇ।

## ਓਵਰਫਿਟਿੰਗ ਅਤੇ ਬਾਇਸ-ਵੈਰੀਅੰਸ ਟਰੇਡਆਫ

ਓਵਰਫਿਟਿੰਗ ਅਸਲ ਵਿੱਚ ਅੰਕੜੇ ਵਿੱਚ ਇੱਕ ਹੋਰ ਆਮ ਸਮੱਸਿਆ ਦਾ ਮਾਮਲਾ ਹੈ ਜਿਸਨੂੰ ਬਾਇਸ-ਵੈਰੀਅੰਸ ਟਰੇਡਆਫ ਕਿਹਾ ਜਾਂਦਾ ਹੈ। ਜੇ ਅਸੀਂ ਆਪਣੇ ਮਾਡਲ ਵਿੱਚ ਗਲਤੀ ਦੇ ਸੰਭਾਵਿਤ ਸਰੋਤਾਂ ਨੂੰ ਵਿਚਾਰਦੇ ਹਾਂ, ਤਾਂ ਅਸੀਂ ਦੋ ਕਿਸਮ ਦੀਆਂ ਗਲਤੀਆਂ ਦੇਖ ਸਕਦੇ ਹਾਂ:

* **ਬਾਇਸ ਗਲਤੀਆਂ** ਸਾਡੀ ਐਲਗੋਰਿਦਮ ਦੁਆਰਾ ਸਹੀ ਤਰੀਕੇ ਨਾਲ ਟ੍ਰੇਨਿੰਗ ਡਾਟਾ ਦੇ ਸੰਬੰਧ ਨੂੰ ਕੈਪਚਰ ਕਰਨ ਵਿੱਚ ਅਸਮਰੱਥ ਹੋਣ ਕਾਰਨ ਹੁੰਦੀਆਂ ਹਨ। ਇਹ ਇਸ ਗੱਲ ਤੋਂ ਨਤੀਜਾ ਹੋ ਸਕਦਾ ਹੈ ਕਿ ਸਾਡਾ ਮਾਡਲ ਕਾਫੀ ਸ਼ਕਤੀਸ਼ਾਲੀ ਨਹੀਂ ਹੈ (**ਅੰਡਰਫਿਟਿੰਗ**).
* **ਵੈਰੀਅੰਸ ਗਲਤੀਆਂ**, ਜੋ ਇਨਪੁਟ ਡਾਟਾ ਵਿੱਚ ਸ਼ੋਰ ਦੀ ਬਜਾਏ ਅਰਥਪੂਰਨ ਸੰਬੰਧ ਨੂੰ ਅਨੁਮਾਨ ਕਰਨ ਕਾਰਨ ਹੁੰਦੀਆਂ ਹਨ (**ਓਵਰਫਿਟਿੰਗ**).

ਟ੍ਰੇਨਿੰਗ ਦੌਰਾਨ, ਬਾਇਸ ਗਲਤੀ ਘਟਦੀ ਹੈ (ਜਦੋਂ ਸਾਡਾ ਮਾਡਲ ਡਾਟਾ ਦਾ ਅਨੁਮਾਨ ਲਗਾਉਣ ਲਈ ਸਿੱਖਦਾ ਹੈ), ਅਤੇ ਵੈਰੀਅੰਸ ਗਲਤੀ ਵੱਧਦੀ ਹੈ। ਓਵਰਫਿਟਿੰਗ ਨੂੰ ਰੋਕਣ ਲਈ ਟ੍ਰੇਨਿੰਗ ਨੂੰ ਰੋਕਣਾ ਮਹੱਤਵਪੂਰਨ ਹੈ - ਜਾਂ ਤਾਂ ਮੈਨੂਅਲੀ (ਜਦੋਂ ਅਸੀਂ ਓਵਰਫਿਟਿੰਗ ਦਾ ਪਤਾ ਲਗਾਉਂਦੇ ਹਾਂ) ਜਾਂ ਆਪਣੇ ਆਪ (ਨਿਯਮਿਤਕਰਨ ਦਾਖਲ ਕਰਕੇ)।

## ਨਿਸ਼ਕਰਸ਼

ਇਸ ਪਾਠ ਵਿੱਚ, ਤੁਸੀਂ ਦੋ ਸਭ ਤੋਂ ਲੋਕਪ੍ਰਿਯ AI ਫਰੇਮਵਰਕ, TensorFlow ਅਤੇ PyTorch ਲਈ ਵੱਖ-ਵੱਖ APIs ਦੇ ਵਿਚਕਾਰ ਦੇ ਫਰਕਾਂ ਬਾਰੇ ਸਿੱਖਿਆ। ਇਸਦੇ ਨਾਲ ਨਾਲ, ਤੁਸੀਂ ਇੱਕ ਬਹੁਤ ਮਹੱਤਵਪੂਰਨ ਵਿਸ਼ਾ, ਓਵਰਫਿਟਿੰਗ ਬਾਰੇ ਸਿੱਖਿਆ।
