3. **`.env` ਫਾਈਲ ਸੋਧੋ**: ਕਿਸੇ ਟੈਕਸਟ ਐਡੀਟਰ (ਜਿਵੇਂ VS Code, Notepad++ ਜਾਂ ਕੋਈ ਹੋਰ ਐਡੀਟਰ) ਵਿੱਚ `.env` ਫਾਈਲ ਖੋਲ੍ਹੋ। ਹੇਠਾਂ ਦਿੱਤੇ ਲਾਈਨਾਂ ਨੂੰ ਫਾਈਲ ਵਿੱਚ ਜੋੜੋ, ਆਪਣੇ Microsoft Foundry Models ਦੇ endpoint ਅਤੇ ਕੁੰਜੀ ਨਾਲ placeholder ਬਦਲਦੇ ਹੋਏ (ਵੇਖੋ [`providers.md`](03-providers.md) ਕਿ ਇਹ ਕਿਵੇਂ ਪ੍ਰਾਪਤ ਕਰਨਗੇ):

   > **ਨੋਟ:** GitHub Models (ਅਤੇ ਇਸ ਦਾ `GITHUB_TOKEN` ਵੈਰੀਏਬਲ) ਜੁਲਾਈ 2026 ਦੇ ਅਖੀਰ ਵਿੱਚ ਬੰਦ ਹੋ ਰਿਹਾ ਹੈ। ਇਸ ਦੀ ਬਜਾਏ [Microsoft Foundry Models](https://ai.azure.com/catalog/models?WT.mc_id=academic-105485-koreyst) ਵਰਤੋਂ।

   ```env
   AZURE_INFERENCE_ENDPOINT=your_foundry_endpoint_here
   AZURE_INFERENCE_CREDENTIAL=your_foundry_api_key_here
   ```

4. **ਫਾਈਲ ਸੇਵ ਕਰੋ**: ਤਬਦੀਲੀਆਂ ਸੇਵ ਕਰੋ ਅਤੇ ਟੈਕਸਟ ਐਡੀਟਰ ਬੰਦ ਕਰੋ।

5. **`python-dotenv` ਇੰਸਟਾਲ ਕਰੋ**: ਜੇ ਤੁਸੀਂ ਪਹਿਲਾਂ ਇੰਸਟਾਲ ਨਹੀਂ ਕੀਤਾ, ਤਾਂ `.env` ਫਾਈਲ ਤੋਂ ਵਾਤਵਰਣ ਭਰ ਦੇ ਵੇਰੀਏਬਲਾਂ ਨੂੰ ਆਪਣੇ Python ਐਪਲੀਕੇਸ਼ਨ ਵਿੱਚ ਲੋਡ ਕਰਨ ਲਈ `python-dotenv` ਪੈਕੇਜ ਇੰਸਟਾਲ ਕਰੋ। ਤੁਸੀਂ ਇਸਨੂੰ `pip` ਨਾਲ ਇੰਸਟਾਲ ਕਰ ਸਕਦੇ ਹੋ:

   ```bash
   pip install python-dotenv
   ```

6. **ਆਪਣੇ Python ਸਕ੍ਰਿਪਟ ਵਿੱਚ ਵਾਤਵਰਣ ਭਰ ਦੇ ਵੇਰੀਏਬਲ ਲੋਡ ਕਰੋ**: ਆਪਣੇ Python ਸਕ੍ਰਿਪਟ ਵਿੱਚ `.env` ਫਾਈਲ ਵੱਲੋਂ ਵਾਤਵਰਣ ਭਰ ਦੇ ਵੇਰੀਏਬਲ ਲੋਡ ਕਰਨ ਲਈ `python-dotenv` ਪੈਕੇਜ ਦੀ ਵਰਤੋਂ ਕਰੋ:

   ```python
   from dotenv import load_dotenv
   import os

   # .env ਫ਼ਾਈਲ ਤੋਂ ਮਾਹੌਲ ਦੇ ਵੈਰੀਏਬਲ ਲੋਡ ਕਰੋ
   load_dotenv()

   # Microsoft Foundry Models ਦੇ ਵੈਰੀਏਬਲਾਂ ਤੱਕ ਪਹੁੰਚ ਪ੍ਰਾਪਤ ਕਰੋ
   endpoint = os.getenv("AZURE_INFERENCE_ENDPOINT")
   token = os.getenv("AZURE_INFERENCE_CREDENTIAL")

   print(endpoint)
   ```

ਬਸ ਹੋ ਗਿਆ! ਤੁਸੀਂ ਸਫਲਤਾਪੂਰਕ `.env` ਫਾਈਲ ਬਣਾਈ, ਆਪਣੇ Microsoft Foundry Models ਦੇ ਸਾਕਸ਼ੱਤ ਤੱਥ ਜੋੜੇ ਅਤੇ ਉਹਨਾਂ ਨੂੰ ਆਪਣੇ Python ਐਪਲੀਕੇਸ਼ਨ ਵਿੱਚ ਲੋਡ ਕੀਤਾ।

## ਵਿਕਲਪਤ ਕਦਮ

### Miniconda ਇੰਸਟਾਲ ਕਰਨਾ

[Miniconda](https://conda.io/en/latest/miniconda.html?WT.mc_id=academic-105485-koreyst) ਇੱਕ ਹਲਕੀ-ਫੁਲਕੀ ਇੰਸਟਾਲਰ ਹੈ ਜੋ [Conda](https://docs.conda.io/en/latest?WT.mc_id=academic-105485-koreyst), Python ਅਤੇ ਕੁਝ ਪੈਕੇਜ਼ ਇੰਸਟਾਲ ਕਰਨ ਲਈ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ।
Conda ਖੁਦ ਇੱਕ ਪੈਕੇਜ ਮੈਨੇਜਰ ਹੈ, ਜੋ ਤੁਹਾਨੂੰ ਵੱਖ-ਵੱਖ Python [**ਵਰਚੁਅਲ ਇਨਵਾਇਰਨਮੈਂਟਸ**](https://docs.python.org/3/tutorial/venv.html?WT.mc_id=academic-105485-koreyst) ਅਤੇ ਪੈਕੇਜਾਂ ਦੇ ਵਿਚਕਾਰ ਬਦਲਾਅ ਕਰਨਾ ਆਸਾਨ ਬਣਾ ਦਿੰਦਾ ਹੈ। ਇਹ ਉਹਨਾਂ ਪੈਕੇਜਾਂ ਨੂੰ ਵੀ ਇੰਸਟਾਲ ਕਰਨ ਵਿੱਚ ਮਦਦਗਾਰ ਹੁੰਦਾ ਹੈ ਜੋ `pip` ਦੁਆਰਾ ਉਪਲਬਧ ਨਹੀਂ ਹੁੰਦੇ।

ਤੁਸੀਂ [Miniconda ਇੰਸਟਾਲੇਸ਼ਨ ਗਾਈਡ](https://docs.anaconda.com/free/miniconda/#quick-command-line-install?WT.mc_id=academic-105485-koreyst) ਦਾ ਪਾਲਣ ਕਰ ਸਕਦੇ ਹੋ ਇਸਨੂੰ ਸੈਟਅਪ ਕਰਨ ਲਈ।

Miniconda ਇੰਸਟਾਲ ਹੋਣ ਤੋਂ ਬਾਅਦ, ਤੁਹਾਨੂੰ [ਰੈਪੋ](https://github.com/microsoft/generative-ai-for-beginners/fork?WT.mc_id=academic-105485-koreyst) ਕਲੋਨ ਕਰਨੀ ਪਵੇਗੀ (ਜੇ ਪਹਿਲਾਂ ਨਹੀਂ ਕੀਤੀ)

ਅਗਲਾ ਕਦਮ ਵਰਚੁਅਲ ਇਨਵਾਇਰਨਮੈਂਟ ਬਣਾਉਣਾ ਹੈ। Conda ਨਾਲ ਇਹ ਕਰਨ ਲਈ, ਇੱਕ ਨਵੀਂ ਇਨਵਾਇਰਨਮੈਂਟ ਫਾਈਲ (_environment.yml_) ਬਣਾਓ। ਜੇ ਤੁਸੀਂ Codespaces ਵਰਤ ਰਹੇ ਹੋ ਤਾਂ ਇਸਨੂੰ `.devcontainer` ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਬਣਾਓ, ਇਸ ਤਰ੍ਹਾਂ `.devcontainer/environment.yml`।

ਆਪਣੀ ਇਨਵਾਇਰਨਮੈਂਟ ਫਾਈਲ ਵਿੱਚ ਹੇਠਾਂ ਦਿਤਾ ਕੋਡ ਪੇਸਟ ਕਰੋ:

```yml
name: <environment-name>
channels:
  - defaults
  - microsoft
dependencies:
  - python=<python-version>
  - openai
  - python-dotenv
  - pip
  - pip:
      - azure-ai-ml
```

ਜੇ ਤੁਹਾਨੂੰ conda ਵਰਤਦੇ ਸਮੇਂ ਤਰੁੱਟੀਆਂ ਆ ਰਹੀਆਂ ਹਨ, ਤਾਂ ਤੁਸੀਂ ਮੈਨੁਅਲੀ ਤੌਰ 'ਤੇ Microsoft AI ਲਾਇਬ੍ਰੇਰੀਜ਼ ਨੂੰ ਟਰਮੀਨਲ ਵਿੱਚ ਹੇਠਾਂ ਦਿੱਤੇ ਕਮਾਂਡ ਨਾਲ ਇੰਸਟਾਲ ਕਰ ਸਕਦੇ ਹੋ.

```
conda install -c microsoft azure-ai-ml
```

ਇਨਵਾਇਰਨਮੈਂਟ ਫਾਈਲ ਉਹ ਨਿਰਭਰਤਾਵਾਂ ਦਿਖਾਉਂਦੀ ਹੈ ਜੋ ਸਾਨੂੰ ਚਾਹੀਦੀਆਂ ਹਨ। `<environment-name>` ਉਹ ਨਾਮ ਹੈ ਜੋ ਤੁਸੀਂ ਆਪਣੇ Conda ਇਨਵਾਇਰਨਮੈਂਟ ਲਈ ਰੱਖਣਾ ਚਾਹੁੰਦੇ ਹੋ, ਅਤੇ `<python-version>` ਉਹ Python ਦਾ ਵਰਜ਼ਨ ਹੈ ਜੋ ਤੁਸੀਂ ਵਰਤਣਾ ਚਾਹੁੰਦੇ ਹੋ, ਉਦਾਹਰਣ ਲਈ, `3` Python ਦਾ ਨਵਾਂ ਵੱਡਾ ਵਰਜ਼ਨ ਹੈ।

ਇਹ ਹੋ ਜਾਣ ਤੋਂ ਬਾਅਦ, ਤੁਸੀਂ ਹੇਠਾਂ ਦਿੱਤੇ ਕਮਾਂਡਾਂ ਨੂੰ ਚਲਾਕੇ ਆਪਣਾ Conda ਇਨਵਾਇਰਨਮੈਂਟ ਬਣਾਉਂਦੇ ਹੋ:

```bash
conda env create --name ai4beg --file .devcontainer/environment.yml # .devcontainer ਸਬ ਪਾਥ ਸਿਰਫ ਕੋਡਸਪੇਸ ਸੈਟਅਪਸ ਲਈ ਲਾਗੂ ਹੁੰਦਾ ਹੈ
conda activate ai4beg
```

ਗਰ ਕੋਈ ਸਮੱਸਿਆ ਹੋਵੇ ਤਾਂ [Conda ਇਨਵਾਇਰਨਮੈਂਟਸ ਦੀ ਗਾਈਡ](https://docs.conda.io/projects/conda/en/latest/user-guide/tasks/manage-environments.html?WT.mc_id=academic-105485-koreyst) ਨੂੰ ਦੇਖੋ।

### Python ਸਹਾਇਤਾ ਐਕਸਟੈਂਸ਼ਨ ਦੇ ਨਾਲ Visual Studio Code ਦੀ ਵਰਤੋਂ

ਅਸੀਂ ਇਸ ਕੋਰਸ ਲਈ [Visual Studio Code (VS Code)](https://code.visualstudio.com/?WT.mc_id=academic-105485-koreyst) ਐਡੀਟਰ ਨੂੰ [Python ਸਹਾਇਤਾ ਐਕਸਟੈਂਸ਼ਨ](https://marketplace.visualstudio.com/items?itemName=ms-python.python&WT.mc_id=academic-105485-koreyst) ਨਾਲ ਵਰਤਣ ਦੀ ਸਿਫਾਰਸ਼ ਕਰਦੇ ਹਾਂ। ਇਹ ਸਿਫਾਰਸ਼ ਹੈ, ਲਾਜ਼ਮੀ ਨਹੀਂ।

> **ਨੋਟ**: ਕੋਰਸ ਰੈਪੋਜ਼ ਨੂੰ VS Code ਵਿੱਚ ਖੋਲ੍ਹ ਕੇ, ਤੁਹਾਡੇ ਕੋਲ ਪ੍ਰੋਜੈਕਟ ਨੂੰ ਕੰਟੇਨਰ ਵਿੱਚ ਸੈਟਅਪ ਕਰਨ ਦਾ ਵਿਕਲਪ ਹੈ। ਇਹ [ਖਾਸ `.devcontainer`](https://code.visualstudio.com/docs/devcontainers/containers?itemName=ms-python.python&WT.mc_id=academic-105485-koreyst) ਡਾਇਰੈਕਟਰੀ ਦੀ ਕਾਰਨ ਹੈ ਜੋ ਕੋਰਸ ਰੈਪੋਜ਼ ਵਿੱਚ ਮਿਲਦੀ ਹੈ। ਇਸ ਬਾਰੇ ਬਾਅਦ ਵਿੱਚ ਹੋਰ ਜਾਣਕਾਰੀ ਦਿੱਤੀ ਜਾਵੇਗੀ।

> **ਨੋਟ**: ਜਦੋਂ ਤੁਸੀਂ ਰੈਪੋਜ਼ ਨੂੰ ਕਲੋਨ ਕਰਕੇ VS Code ਵਿੱਚ ਖੋਲ੍ਹੋਗੇ, ਤਾਂ ਇਹ ਆਪਣੇ ਆਪ ਤੁਹਾਨੂੰ Python ਸਹਾਇਤਾ ਐਕਸਟੈਂਸ਼ਨ ਇੰਸਟਾਲ ਕਰਨ ਦੀ ਸਿਫਾਰਸ਼ ਕਰੇਗਾ।

> **ਨੋਟ**: ਜੇ VS Code ਤੁਹਾਨੂੰ ਰੈਪੋਜ਼ ਨੂੰ ਕੰਟੇਨਰ ਵਿੱਚ ਮੁੜ ਖੋਲ੍ਹਣ ਲਈ ਕਹੇ, ਤਾਂ ਇਹ ਬੇਨਤੀ ਮਨਜ਼ੂਰ ਨਾ ਕਰੋ ਤਾਂ ਜੋ ਤੁਸੀਂ locally ਇੰਸਟਾਲ ਕੀਤਾ Python ਵਰਤ ਸਕੋ।

### ਬਰਾਊਜ਼ਰ ਵਿੱਚ Jupyter ਵਰਤਣਾ

ਤੁਸੀਂ ਪ੍ਰੋਜੈਕਟ 'ਤੇ ਆਪਣੇ ਬਰਾਊਜ਼ਰ ਵਿੱਚ ਸੀਧਾ [Jupyter ਮਾਹੌਲ](https://jupyter.org?WT.mc_id=academic-105485-koreyst) ਦੀ ਵਰਤੋਂ ਕਰ ਸਕਦੇ ਹੋ। ਕਲਾਸਿਕ Jupyter ਅਤੇ [Jupyter Hub](https://jupyter.org/hub?WT.mc_id=academic-105485-koreyst) ਦੋਹਾਂ ਇੱਕ ਸੁਖਦਾਈ ਵਿਕਾਸ ਮਾਹੌਲ ਦਿੰਦੇ ਹਨ ਜਿਸ ਵਿੱਚ ਆਟੋ-ਕంపਲੀਸ਼ਨ, ਕੋਡ ਹਾਈਲਾਈਟਿੰਗ ਵਰਗੀਆਂ ਸੁਵਿਧਾਵਾਂ ਸ਼ਾਮਲ ਹਨ।

ਸਥਾਨਕ ਤੌਰ 'ਤੇ Jupyter ਚਲਾਉਣ ਲਈ, ਟਰਮੀਨਲ/ਕਮਾਂਡ ਲਾਈਨ ਵਿੱਚ ਜਾਓ, ਕੋਰਸ ਡਾਇਰੈਕਟਰੀ ਨੂੰ ਖੋਲ੍ਹੋ ਅਤੇ ਇਹ ਚਲਾਓ:

```bash
jupyter notebook
```

ਜਾਂ

```bash
jupyterhub
```

ਇਹ ਇੱਕ Jupyter ਇੰਸਟੈਂਸ ਸ਼ੁਰੂ ਕਰੇਗਾ ਅਤੇ ਇਸ ਤੱਕ ਪਹੁੰਚ ਲਈ URL ਕਮਾਂਡ ਲਾਈਨ ਵਿੰਡੋ ਵਿੱਚ ਦਿਖਾਇਆ ਜਾਵੇਗਾ।

ਜਦੋਂ ਤੁਸੀਂ URL ਤੇ ਪਹੁੰਚੋਗੇ, ਤਾਂ ਤੁਹਾਨੂੰ ਕੋਰਸ ਦੀ ਢਾਂਚਾ ਅਤੇ ਕਿਸੇ ਵੀ `*.ipynb` ਫਾਈਲ ਤੱਕ ਨੈਵੀਗੇਟ ਕਰਨ ਦੀ ਸਮਰੱਥਾ ਮਿਲੇਗੀ। ਉਦਾਹਰਣ ਵਜੋਂ, `08-building-search-applications/python/oai-solution.ipynb`।

### ਕੰਟੇਨਰ ਵਿੱਚ ਚਲਾਉਣਾ

ਤੁਹਾਡੇ ਕਮਪਿਊਟਰ ਜਾਂ Codespace 'ਤੇ ਸਾਰਾ ਸੈਟਅਪ ਕਰਨ ਦੀ ਥਾਂ, ਤੁਸੀਂ [ਕੰਟੇਨਰ](../../../00-course-setup/<https:/en.wikipedia.org/wiki/Containerization_(computing)?WT.mc_id=academic-105485-koreyst>) ਦੀ ਵਰਤੋਂ ਕਰ ਸਕਦੇ ਹੋ। ਕੋਰਸ ਰੈਪੋਜ਼ ਦੀ ਖਾਸ `.devcontainer` ਫੋਲਡਰ VS Code ਨੂੰ ਪ੍ਰੋਜੈਕਟ ਨੂੰ ਕੰਟੇਨਰ ਵਿੱਚ ਸੈਟਅਪ ਕਰਨ ਦੀ ਆਗਿਆ ਦਿੰਦਾ ਹੈ। Codespaces ਦੇ ਬਾਹਰ, ਇਹ Docker ਇੰਸਟਾਲ ਕਰਨ ਦੀ ਲੋੜ ਹੋਵੇਗੀ, ਅਤੇ ਇਹ ਕਾਫੀ ਧਿਆਨ-ਮੰਗਦਾ ਕੰਮ ਹੈ, ਇਸ ਲਈ ਅਸੀਂ ਇਸਨੂੰ ਸਿਰਫ ਉਹਨਾਂ ਲਈ ਸੁਝਾਵਾਂ ਦੇਂਦੇ ਹਾਂ ਜਿਨ੍ਹਾਂ ਕੋਲ ਕੰਟੇਨਰਾਂ ਨਾਲ ਕੰਮ ਕਰਨ ਦਾ ਤਜ਼ਰਬਾ ਹੈ।

ਜਦੋਂ GitHub Codespaces ਵਰਤ ਰਹੇ ਹੋ, ਆਪਣੇ API ਕੁੰਜੀਆਂ ਨੂੰ ਸੁਰੱਖਿਅਤ ਰੱਖਣ ਦਾ ਸਭ ਤੋਂ ਵਧੀਆ ਤਰੀਕਾ Codespace Secrets ਦੀ ਵਰਤੋਂ ਕਰਨਾ ਹੈ। ਕਿਰਪਾ ਕਰਕੇ [Codespaces secrets management](https://docs.github.com/en/codespaces/managing-your-codespaces/managing-secrets-for-your-codespaces?WT.mc_id=academic-105485-koreyst) ਦੀ ਗਾਈਡ ਨੂੰ ਪਾਲਣ ਕਰੋ ਇਸ ਬਾਰੇ ਹੋਰ ਜਾਣਣ ਲਈ।


## ਪਾਠ ਅਤੇ ਤਕਨੀਕੀ ਜ਼ਰੂਰਤਾਂ

ਕੋਰਸ ਵਿੱਚ "ਸਿੱਖੋ" ਵਾਲੇ ਪਾਠ ਹਨ ਜੋ ਜਨਰੇਟਿਵ AI ਦੀਆਂ ਸੰਕਲਪਾਵਾਂ ਨੂੰ ਸਮਝਾਉਂਦੇ ਹਨ ਅਤੇ "ਬਣਾਉ" ਵਾਲੇ ਪਾਠ ਹਨ ਜਿਨ੍ਹਾਂ ਵਿੱਚ ਹੱਥੋਂ-ਹੱਥ ਕੋਡ ਉਦਾਹਰਨਾਂ ਹਨ, ਦੋਹਾਂ **Python** ਅਤੇ **TypeScript** ਵਿੱਚ ਜਿੱਥੇ ਸੰਭਵ ਹੋਵੇ।

ਕੋਡਿੰਗ ਪਾਠਾਂ ਲਈ, ਅਸੀਂ Microsoft Foundry ਵਿੱਚ Azure OpenAI ਦੀ ਵਰਤੋਂ ਕਰਦੇ ਹਾਂ। ਤੁਹਾਨੂੰ Azure ਦੀ ਸਬਸਕ੍ਰਿਪਸ਼ਨ ਅਤੇ API ਕੁੰਜੀ ਦੀ ਲੋੜ ਹੋਵੇਗੀ। ਐਕਸੈੱਸ ਖੁੱਲ੍ਹਾ ਹੈ - ਕੋਈ ਐਪਲੀਕੇਸ਼ਨ ਲੋੜੀਂਦੀ ਨਹੀਂ - ਇਸ ਲਈ ਤੁਸੀਂ [Microsoft Foundry ਰਿਸੋਰਸ ਬਣਾ ਕੇ ਅਤੇ ਮਾਡਲ ਨਿਯੁਕਤ ਕਰਕੇ](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/create-resource?pivots=web-portal&WT.mc_id=academic-105485-koreyst) ਆਪਣਾ endpoint ਅਤੇ ਕੁੰਜੀ ਪ੍ਰਾਪਤ ਕਰ ਸਕਦੇ ਹੋ।

ਹਰ ਕੋਡਿੰਗ ਪਾਠ ਵਿੱਚ ਇੱਕ `README.md` ਫਾਈਲ ਵੀ ਸ਼ਾਮਿਲ ਹੈ ਜਿੱਥੇ ਤੁਸੀਂ ਕੋਡ ਅਤੇ ਨਤੀਜੇ ਵਿਖ ਸਕਦੇ ਹੋ ਬਿਨਾਂ ਕੁਝ ਚਲਾਏ।
