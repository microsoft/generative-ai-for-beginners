## 2.  ਵਿਕਲਪ C – ਮਿਨੀਕੋੰਡਾ

[ਮਿਨੀਕੋੰਡਾ](https://conda.io/en/latest/miniconda.html?WT.mc_id=academic-105485-koreyst) ਇੱਕ ਹਲਕਾ ਭਾਰ ਇੰਸਟਾਲਰ ਹੈ ਜਿਸ ਨਾਲ [Conda](https://docs.conda.io/en/latest?WT.mc_id=academic-105485-koreyst), ਪਾਈਥਨ ਅਤੇ ਕੁਝ ਪੈਕੇਜਜ਼ ਇੰਸਟਾਲ ਕੀਤੇ ਜਾਂਦੇ ਹਨ।
Conda ਆਪਣੇ ਆਪ ਵਿੱਚ ਇੱਕ ਪੈਕੇਜ ਮੈਨੇਜਰ ਹੈ, ਜੋ ਵੱਖ-ਵੱਖ ਪਾਈਥਨ [**ਵਰਚੁਅਲ ਇਨਵਾਇਰਨਮੈਂਟਸ**](https://docs.python.org/3/tutorial/venv.html?WT.mc_id=academic-105485-koreyst) ਅਤੇ ਪੈਕੇਜਜ਼ ਦੇ ਵਿੱਚ ਸੈੱਟਅੱਪ ਅਤੇ ਬਦਲਣਾ ਆਸਾਨ ਬਣਾਉਂਦਾ ਹੈ। ਇਹ ਉਨ੍ਹਾਂ ਪੈਕੇਜਾਂ ਲਈ ਵੀ ਲਾਭਦਾਇਕ ਹੈ ਜੋ `pip` ਰਾਹੀਂ ਉਪਲਬਧ ਨਹੀਂ ਹਨ।

### ਕਦਮ 0  ਮਿਨੀਕੋੰਡਾ ਇੰਸਟਾਲ ਕਰੋ

ਮਿਨੀਕੋੰਡਾ ਸੈੱਟਅੱਪ ਕਰਨ ਲਈ [MiniConda ਇੰਸਟਾਲੇਸ਼ਨ ਗਾਈਡ](https://docs.anaconda.com/free/miniconda/#quick-command-line-install?WT.mc_id=academic-105485-koreyst) ਦਾ ਪਾਲਣ ਕਰੋ।

```bash
conda --version
```

### ਕਦਮ 1 ਇੱਕ ਨਵਾਂ ਵਾਤਾਵਰਣ ਬਣਾਓ

ਇੱਕ ਨਵੀਂ ਵਾਤਾਵਰਣ ਫਾਇਲ ਬਣਾਓ (*environment.yml*). ਜੇ ਤੁਸੀਂ ਕੋਡਸਪੇਸ ਦੇ ਨਾਲ ਅਗੇ ਵਧ ਰਹੇ ਹੋ, ਤਾਂ ਇਸਨੂੰ `.devcontainer` ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਬਣਾਓ, ਜਿਵੇਂ `.devcontainer/environment.yml`।

### ਕਦਮ 2 ਆਪਣੀ ਵਾਤਾਵਰਣ ਫਾਇਲ ਨੂੰ ਭਰੋ

ਹੇਠਾਂ ਦਿੱਤਾ ਟੁਕੜਾ ਆਪਣੀ `environment.yml` ਵਿੱਚ ਸ਼ਾਮਿਲ ਕਰੋ

```yml
name: <environment-name>
channels:
 - defaults
 - microsoft
dependencies:
- python=<python-version>
- openai
- python-dotenv
- pip
- pip:
    - azure-ai-ml

```

### ਕਦਮ 3 ਆਪਣਾ ਕੋੰਡਾ ਵਾਤਾਵਰਣ ਬਣਾਓ

ਹੇਠਾਂ ਦਿੱਤੇ ਕਮਾਂਡਾਂ ਆਪਣੇ ਕਮਾਂਡ ਲਾਈਨ/ਟਰਮੀਨਲ ਵਿੱਚ ਚਲਾਓ

```bash 
conda env create --name ai4beg --file .devcontainer/environment.yml # .devcontainer ਉਪ-ਪਥ ਸਿਰਫ਼ Codespace ਸੈਟਅੱਪ ਲਈ ਲਾਗੂ ਹੁੰਦਾ ਹੈ
conda activate ai4beg
```

ਜੇ ਕੋਈ ਸਮੱਸਿਆ ਆਵੇ ਤਾਂ [Conda ਵਾਤਾਵਰਣ ਗਾਈਡ](https://docs.conda.io/projects/conda/en/latest/user-guide/tasks/manage-environments.html?WT.mc_id=academic-105485-koreyst) ਨੂੰ ਵੇਖੋ।

## 3. ਆਪਣੀਆਂ API ਕੀਜ਼ ਜੋੜੋ

ਆਪਣੀਆਂ API ਕੀਜ਼ ਨੂੰ ਸੁਰੱਖਿਅਤ ਰੱਖਣਾ ਕਿਸੇ ਵੀ ਕਿਸਮ ਦੀ ਐਪਲੀਕੇਸ਼ਨ ਬਣਾਉਂਦੇ ਸਮੇਂ ਮਹੱਤਵਪੂਰਨ ਹੈ। ਅਸੀਂ ਸਿਫਾਰਸ਼ ਕਰਦੇ ਹਾਂ ਕਿ ਕੋਈ ਵੀ API ਕੀਜ਼ ਸਿੱਧਾ ਆਪਣੇ ਕੋਡ ਵਿੱਚ ਸਟੋਰ ਨਾ ਕਰੋ। ਜੇ ਇਹ ਵੇਰਵੇ ਕਿਸੇ ਪਬਲਿਕ ਰਿਪੋ ਵਿੱਚ ਦਾਖਲ ਹੋ ਜਾਣ, ਤਾਂ ਸੁਰੱਖਿਆ ਸਮੱਸਿਆਵਾਂ ਅਤੇ ਨੁਕਸਾਨਦਾਇਕ ਖਰਚ ਵਰਗੀਆਂ ਮੁਸ਼ਕਲਾਂ ਹੋ ਸਕਦੀਆਂ ਹਨ ਜੋ ਬੁਰੇ ਇਰਾਦੇ ਵਾਲੇ ਵਰਤੋਂਕਾਰ ਦਾ ਨਤੀਜਾ ਹੋ ਸਕਦੀਆਂ ਹਨ।
ਹੇਠਾਂ ਇੱਕ ਕਦਮ-ਬਾਈ-ਕਦਮ ਗਾਈਡ ਹੈ ਕਿ ਕਿਵੇਂ Python ਲਈ `.env` ਫਾਈਲ ਬਣਾਈਏ ਅਤੇ ਆਪਣੀਆਂ Microsoft Foundry Models ਕ੍ਰੈਡੈਂਸ਼ਲਜ਼ ਸ਼ਾਮਲ ਕਰੋ:

> **ਨੋਟ:** GitHub Models (ਅਤੇ ਇਸਦਾ `GITHUB_TOKEN` ਵੈਰੀਏਬਲ) ਜੁਲਾਈ 2026 ਦੇ ਅਖੀਰ ਵਿੱਚ ਰਿਟਾਇਰ ਹੋ ਰਹਿਆ ਹੈ। ਇਹ ਗਾਈਡ ਵਜੋਂ [Microsoft Foundry Models](https://ai.azure.com/catalog/models?WT.mc_id=academic-105485-koreyst) ਦੀ ਵਰਤੋਂ ਕਰਦੀ ਹੈ। ਜੇ ਤੁਸੀਂ ਪੂਰੀ ਤਰ੍ਹਾਂ ਆਫਲਾਈਨ ਕੰਮ ਕਰਨਾ ਚਾਹੁੰਦੇ ਹੋ, ਤਾਂ [Foundry Local](https://foundrylocal.ai?WT.mc_id=academic-105485-koreyst) ਵੇਖੋ।

1. **ਆਪਣੀ ਪ੍ਰੋਜੈਕਟ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਜਾਓ**: ਆਪਣਾ ਟਰਮੀਨਲ ਜਾਂ ਕਮਾਂਡ ਪ੍ਰਾਂਪਟ ਖੋਲ੍ਹੋ ਅਤੇ ਆਪਣੀ ਪ੍ਰੋਜੈਕਟ ਦੀ ਰੂਟ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਜਾਓ ਜਿੱਥੇ ਤੁਸੀਂ `.env` ਫਾਈਲ ਬਣਾਉਣਾ ਚਾਹੁੰਦੇ ਹੋ।

   ```bash
   cd path/to/your/project
   ```

2. **`.env` ਫਾਈਲ ਬਣਾਓ**: ਆਪਣੇ ਮਨਪਸੰਦ ਟੈਕਸਟ ਐਡੀਟਰ ਦੀ ਵਰਤੋਂ ਕਰਕੇ `.env` ਨਾਮਕ ਨਵੀਂ ਫਾਈਲ ਬਣਾਓ। ਜੇ ਤੁਹਾਨੂੰ ਕਮਾਂਡ ਲਾਈਨ ਵਰਤੀ ਜਾ ਰਹੀ ਹੈ, ਤਾਂ ਤੁਸੀਂ Unix-ਅਧਾਰਿਤ ਸਿਸਟਮਾਂ 'ਤੇ `touch` ਜਾਂ Windows 'ਤੇ `echo` ਵਰਤ ਸਕਦੇ ਹੋ:

   ਯੂਨਿਕਸ-ਅਧਾਰਿਤ ਸਿਸਟਮਾਂ:

   ```bash
   touch .env
   ```

   ਵਿਂਡੋਜ਼:

   ```cmd
   echo . > .env
   ```

3. **`.env` ਫਾਈਲ ਸੋਧੋ**: `.env` ਫਾਈਲ ਨੂੰ ਕਿਸੇ ਟੈਕਸਟ ਐਡੀਟਰ (ਜਿਵੇਂ VS ਕੋਡ, ਨੋਟਪੈਡ++, ਜਾਂ ਹੋਰ ਕਿਸੇ ਐਡੀਟਰ) ਵਿੱਚ ਖੋਲ੍ਹੋ। ਹੇਠਾਂ ਦਿੱਤੀਆਂ ਲਾਈਨਾਂ ਸ਼ਾਮਿਲ ਕਰੋ, ਆਪਣੀ ਅਸਲੀ Microsoft Foundry ਪ੍ਰੋਜੈਕਟ ਏਂਡਪੌਇੰਟ ਅਤੇ API ਕੁੰਜੀ ਨਾਲ ਪਲੇਸਹੋਲਡਰ ਬਦਲ ਕੇ:

   ```env
   AZURE_INFERENCE_ENDPOINT=your_foundry_endpoint_here
   AZURE_INFERENCE_CREDENTIAL=your_foundry_api_key_here
   ```

4. **ਫਾਈਲ ਸੇਵ ਕਰੋ**: ਬਦਲਾਅ ਸੇਵ ਕਰੋ ਅਤੇ ਟੈਕਸਟ ਐਡੀਟਰ ਬੰਦ ਕਰੋ।

5. **`python-dotenv` ਇੰਸਟਾਲ ਕਰੋ**: ਜੇ ਤੁਸੀਂ ਪਹਿਲਾਂ ਨਹੀਂ ਕੀਤਾ ਤਾਂ, ਤੁਹਾਨੂੰ `.env` ਫਾਈਲ ਤੋਂ ਵਾਤਾਵਰਣ ਚਲ ਬਣਾਉਣ ਵਾਸਤੇ `python-dotenv` ਪੈਕੇਜ ਇੰਸਟਾਲ ਕਰਨ ਦੀ ਲੋੜ ਹੈ। ਤੁਸੀਂ ਇਸਨੂੰ `pip` ਰਾਹੀਂ ਇੰਸਟਾਲ ਕਰ ਸਕਦੇ ਹੋ:

   ```bash
   pip install python-dotenv
   ```

6. **ਆਪਣੇ ਪਾਈਥਨ ਸਕ੍ਰਿਪਟ ਵਿੱਚ ਵਾਤਾਵਰਣ ਚਲ ਚਾਰਜ ਕਰੋ**: ਆਪਣੇ ਪਾਈਥਨ ਸਕ੍ਰਿਪਟ ਵਿੱਚ, `.env` ਫਾਈਲ ਤੋਂ ਵਾਤਾਵਰਣ ਚਲ ਲੋਡ ਕਰਨ ਲਈ `python-dotenv` ਪੈਕੇਜ ਦੀ ਵਰਤੋਂ ਕਰੋ:

   ```python
   from dotenv import load_dotenv
   import os

   # .env ਫਾਇਲ ਤੋਂ ਵਾਤਾਵਰਣ ਕਰਦੀਆਂ ਲੋਡ ਕਰੋ
   load_dotenv()

   # Microsoft Foundry ਮਾਡਲ ਵੈਰੀਏਬਲਾਂ ਨੂੰ ਪਹੁੰਚ ਕਰੋ
   endpoint = os.getenv("AZURE_INFERENCE_ENDPOINT")
   token = os.getenv("AZURE_INFERENCE_CREDENTIAL")

   print(endpoint)
   ```

ਹੋ ਗਿਆ! ਤੁਸੀਂ ਸਫਲਤਾਪੂਰਵਕ `.env` ਫਾਈਲ ਬਣਾਈ, ਆਪਣੀਆਂ Microsoft Foundry Models ਕ੍ਰੈਡੈਂਸ਼ਲਜ਼ ਜੋੜੀਆਂ, ਅਤੇ ਉਹਨਾਂ ਨੂੰ ਆਪਣੀ ਪਾਈਥਨ ਐਪਲੀਕੇਸ਼ਨ ਵਿੱਚ ਲੋਡ ਕੀਤਾ।

🔐 ਕਦੇ ਵੀ .env ਨੂੰ ਕਮਿਟ ਨਾ ਕਰੋ—ਇਹ ਪਹਿਲਾਂ ਹੀ .gitignore ਵਿੱਚ ਹੈ।
ਪੂਰੀ ਪ੍ਰਦਾਤਾ ਹਦਾਇਤਾਂ [`providers.md`](03-providers.md) ਵਿੱਚ ਮਿਲਦੀਆਂ ਹਨ।
