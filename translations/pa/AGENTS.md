## ਵਿਕਾਸ ਵਰਕਫਲੋ

### ਵਾਤਾਵਰਣ ਵੇਰੀਏਬਲ

ਸਾਰੇ ਪਾਠਾਂ ਨੂੰ API ਪਹੁੰਚ ਲਈ `.env` ਵਿੱਚ_DEFINED ਵਾਤਾਵਰਣ ਵੇਰੀਏਬਲ ਦੀ ਲੋੜ ਹੁੰਦੀ ਹੈ:

- `OPENAI_API_KEY` - OpenAI API ਲਈ
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry ਵਿੱਚ ਏਜ਼ਯੂਰ OpenAI ਲਈ (ਏਜ਼ਯੂਰ OpenAI ਸੇਵਾ ਹੁਣ Microsoft Foundry ਦਾ ਹਿੱਸਾ ਹੈ: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - ਏਜ਼ਯੂਰ OpenAI ਏਂਡਪੋਇੰਟ URL (Foundry ਸਰੋਤ ਏਂਡਪੋਇੰਟ)
- `AZURE_OPENAI_DEPLOYMENT` - ਚੈਟ ਸਮਾਪਤੀ ਮਾਡਲ ਡਿਪਲੋਇਮੈਂਟ ਨਾਮ (ਕੋਰਸ ਮੂਲ: `gpt-5-mini`)
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - ਐਮਬੈਡਿੰਗਸ ਮਾਡਲ ਡਿਪਲੋਇਮੈਂਟ ਨਾਮ (ਕੋਰਸ ਮੂਲ: `text-embedding-3-small`)
- `AZURE_OPENAI_API_VERSION` - API ਸੰਸਕਰਨ (ਮੂਲ: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face ਮਾਡਲਾਂ ਲਈ
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry ਮਾਡਲਸ ਏਂਡਪੋਇੰਟ (ਬਹੁ-ਪ੍ਰਦਾਤਾ ਮਾਡਲ ਕੈਟਾਲੌਗ)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry ਮਾਡਲਸ API ਕੀ (ਰਿਟਾਇਰ ਹੋ ਰਹੇ `GITHUB_TOKEN` ਦੀ ਥਾਂ)
- `AZURE_INFERENCE_CHAT_MODEL` - ਇਕ ਗੈਰ-ਵਿਚਾਰ ਮਾਡਲ (ਜਿਵੇਂ `Llama-3.3-70B-Instruct`) ਜੋ `temperature` ਉਦਾਹਰਣਾਂ ਵਿੱਚ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ, ਕਿਉਂਕਿ ਵਿਚਾਰਸ਼ੀਲ ਮਾਡਲ ਸੈਂਪਲਿੰਗ ਨਿਯੰਤਰਣਾਂ ਦਾ ਸਮਰਥਨ ਨਹੀਂ ਕਰਦੇ

### ਮਾਡਲ ਧਾਰਣਾਵਾਂ (ਮਹੱਤਵਪੂਰਨ)

- **ਮੂਲ ਚੈਟ ਮਾਡਲ `gpt-5-mini` ਹੈ** - ਇਕ ਮੌਜੂਦਾ, ਗੈਰ-ਹਟਾਏ ਗਏ **ਵਿਚਾਰਸ਼ੀਲ** ਮਾਡਲ। 2026 ਤੋਂ ਬਾਅਦ ਪੁਰਾਣੇ ਤਾਪਮਾਨ-ਸਮਰਥ “ਮੀਨੀ” ਮਾਡਲ (`gpt-4o-mini`, `gpt-4.1-mini`) *ਹਟਾਏ ਜਾਂਦੇ ਹਨ*, ਇਸ ਲਈ ਕੋਰਸ GPT-5 ਪਰਿਵਾਰ ਤੇ ਧਿਆਨ ਕੇਂਦਰਿਤ ਕਰਦਾ ਹੈ।
- **ਵਿਚਾਰਸ਼ੀਲ ਮਾਡਲ `temperature` ਅਤੇ `top_p` ਨੂੰ ਅਸਵੀਕਾਰ ਕਰਦੇ ਹਨ**, ਅਤੇ `max_tokens` ਦੀ ਥਾਂ `max_output_tokens` (Responses API) / `max_completion_tokens` (ਚੈਟ ਸਮਾਪਤੀਆਂ) ਵਰਤਦੇ ਹਨ। `gpt-5-mini` ਨੂੰ ਕਾਲ ਕਰਨ ਵਾਲੇ ਨਮੂਨਿਆਂ ਵਿੱਚ `temperature`/`top_p`/`max_tokens` ਨਾ ਜੋੜੋ।
- **`temperature` ਦੇ ਨਮੂਨੇ ਲਈ**, ਨਮੂਨੇ Microsoft Foundry ਮਾਡਲਸ ਏਂਡਪੋਇੰਟ (`AZURE_INFERENCE_CHAT_MODEL`) ਰਾਹੀਂ **Llama** ਮਾਡਲ (`Llama-3.3-70B-Instruct`) ਨੂੰ ਵਰਤਦੇ ਹਨ। ਵਿਚਾਰਸ਼ੀਲ ਮਾਡਲਾਂ ਨੂੰ ਸੈਂਪਲਿੰਗ ਨਿਯੰਤਰਣਾਂ ਦੀ ਥਾਂ ਪ੍ਰੰਪਟ ਇੰਜੀਨੀਅਰਿੰਗ + ਵਿਚਾਰਸ਼ੀਲ ਨਿਯੰਤਰਣ ਨਾਲ ਸਾਂਭੋ।
- **ਫਾਈਨ-ਟਿਊਨਿੰਗ (ਪਾਠ 18)** `gpt-4.1-mini` ਰੱਖਦਾ ਹੈ: GPT-5 ਸਿਰਫ਼ ਰੀਇੰਫੋਰਸਮੈਂਟ ਫਾਈਨ-ਟਿਊਨਿੰਗ (RFT) ਨੂੰ ਸਮਰਥਨ ਕਰਦਾ ਹੈ, ਨਾ ਕਿ ਉਸਪਰੀਵਾਜ਼ਡ ਫਾਈਨ-ਟਿਊਨਿੰਗ (SFT) ਜੋ ਉਥੇ ਦਿਖਾਇਆ ਗਿਆ ਹੈ।
- ਪਾਠ 20 (ਮਿਸਟ੍ਰਾਲ) ਅਤੇ 21 (ਮੇਟਾ) `temperature`/`max_tokens` ਰੱਖਦੇ ਹਨ ਕਿਉਂਕਿ ਉਹ ਮਿਸਟ੍ਰਾਲ/ਲਾਮਾ ਮਾਡਲਾਂ ਨੂੰ ਟਾਰਗੇਟ ਕਰਦੇ ਹਨ ਜੋ ਉਹਨਾਂ ਨੂੰ ਸਮਰਥਨ ਕਰਦੇ ਹਨ।

## ਕੋਡ ਸ਼ੈਲੀ ਦੀਆਂ ਰਹਿਣਿਯਾਂ

### ਪਾਇਥਨ

- ਵਾਤਾਵਰਣ ਵੇਰੀਏਬਲ ਪ੍ਰਬੰਧਨ ਲਈ `python-dotenv` ਦੀ ਵਰਤੋਂ ਕਰੋ
- API ਸੰਪਰਕ ਲਈ `openai` ਲਾਇਬ੍ਰੇਰੀ ਇੰਪੋਰਟ ਕਰੋ
- Linting ਲਈ `pylint` ਦੀ ਵਰਤੋਂ ਕਰੋ (ਕਈ ਉਦਾਹਰਣਾਂ ਵਿੱਚ ਸਾਦਗੀ ਲਈ `# pylint: disable=all` ਹੁੰਦਾ ਹੈ)
- PEP 8 ਨਾਂਕਰਨ ਰੀਤੀਆਂ ਦੀ ਪਾਲਣਾ ਕਰੋ
- API ਸਰਟੀਫਿਕੇਟ `.env` ਫ਼ਾਈਲ ਵਿੱਚ ਸਟੋਰ ਕਰੋ, ਕਦੇ ਵੀ ਕੋਡ ਵਿੱਚ ਨਹੀਂ

### ਟਾਈਪਸਕ੍ਰਿਪਟ

- ਵਾਤਾਵਰਣ ਵੇਰੀਏਬਲ ਲਈ `dotenv` ਪੈਕੇਜ ਦੀ ਵਰਤੋਂ ਕਰੋ
- ਹਰ ਐਪ ਲਈ `tsconfig.json` ਵਿੱਚ ਟਾਈਪਸਕ੍ਰਿਪਟ ਸੰਰਚਨਾ
- ਏਜ਼ਯੂਰ OpenAI ਲਈ `openai` ਪੈਕੇਜ ਦੀ ਵਰਤੋਂ ਕਰੋ (ਕਲਾਇੰਟ ਨੂੰ `/openai/v1/` ਏਂਡਪੋਇੰਟ ਸੈੱਟ ਕਰੋ ਅਤੇ `client.responses.create` ਕਾਲ ਕਰੋ); Microsoft Foundry ਮਾਡਲ ਲਈ `@azure-rest/ai-inference` ਦੀ ਵਰਤੋਂ ਕਰੋ
- ਆਟੋ-ਰੀਲੋਡ ਲਈ `nodemon` ਦੀ ਵਰਤੋਂ ਕਰੋ
- ਚਲਾਉਣ ਤੋਂ ਪਹਿਲਾਂ ਬਿਲਡ ਕਰੋ: `npm run build` ਫਿਰ `npm start`
