**ਮੁੱਖ ਤਕਨਾਲੋਜੀਆਂ:**
- Python 3.9+ ਲਾਇਬ੍ਰੇਰੀਜ਼ ਨਾਲ: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript Node.js ਅਤੇ ਲਾਇਬ੍ਰੇਰੀਜ਼ ਨਾਲ: `openai` (Azure OpenAI ਵੀਆਈ v1 endpoint + Responses API), `@azure-rest/ai-inference` (Microsoft Foundry ਮਾਡਲ)
- Azure OpenAI ਸਰਵਿਸ, OpenAI API, ਅਤੇ Microsoft Foundry ਮਾਡਲ (GitHub Models ਜੁਲਾਈ 2026 ਦੇ ਅੰਤ ਵਿੱਚ ਰਿਟਾਇਰ ਹੋ ਰਹੇ ਹਨ)
- ਇੰਟਰਐਕਟਿਵ ਸਿੱਖਣ ਲਈ Jupyter ਨੋਟਬੁੱਕਸ
- ਸਥਿਰ ਵਿਕਾਸ ਲਈ Dev Containers

**ਰਿਪੌਜ਼ਟਰੀ ਦੀ ਸੰਰਚਨਾ:**
- 21 ਨੰਬਰਵਾਲੇ ਪਾਠ ਡਾਇਰੈਕਟਰੀਜ਼ (00-21) ਜਿਨ੍ਹਾਂ ਵਿੱਚ README, ਕੋਡ ਉਦਾਹਰਨਾਂ, ਅਤੇ ਅਸਾਈਨਮੈਂਟ ਹਨ
- ਕਈ ਵਾਰੀ ਲਾਗੂ ਕੀਤੀਆਂ: Python, TypeScript ਅਤੇ ਕੁਝ ਸਮੇਂ .NET ਉਦਾਹਰਨਾਂ
- 40+ ਭਾਸ਼ਾਵਾਂ ਵਿੱਚ ਅਨੁਵਾਦ ਡਾਇਰੈਕਟਰੀ

## ਵਿਕਾਸ ਕਮਾਈ

### ਵਾਤਾਵਰਣ ਚਲਕ

ਸਾਰੇ ਪਾਠ ਜਿਨ੍ਹਾਂ ਨੂੰ API ਪ੍ਰਾਪਤੀ ਦੀ ਲੋੜ ਹੈ, ਉਹ `.env` ਵਿੱਚ ਪਰਿਭਾਸ਼ਿਤ ਵਾਤਾਵਰਣ ਚਲਕ ਵਰਤਦੇ ਹਨ:

- `OPENAI_API_KEY` - OpenAI API ਲਈ
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry ਵਿੱਚ Azure OpenAI ਲਈ (Azure OpenAI ਸਰਵਿਸ ਹੁਣ Microsoft Foundry ਦਾ ਹਿੱਸਾ ਹੈ: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI endpoint URL (Foundry ਸਰੋਤ endpoint)
- `AZURE_OPENAI_DEPLOYMENT` - ਚੈਟ ਪੂਰਾ ਕਰਨ ਵਾਲੇ ਮਾਡਲ ਦੀ ਤਾਇਨਾਤੀ ਨਾਮ
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - ਐਮਬੈਡਿੰਗ ਮਾਡਲ ਦੀ ਤਾਇਨਾਤੀ ਨਾਮ
- `AZURE_OPENAI_API_VERSION` - API ਵਰਜ਼ਨ (ਡਿਫਾਲਟ: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face ਮਾਡਲਾਂ ਲਈ
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry ਮਾਡਲਾਂ ਦਾ endpoint (ਮਲਟੀ-ਪ੍ਰੋਵਾਈਡਰ ਮਾਡਲ ਕੈਟਾਲੌਗ)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry ਮਾਡਲ API ਕੁੰਜੀ (`GITHUB_TOKEN` ਦੀ ਬਦਲੀ ਜੋ ਰਿਟਾਇਰ ਹੋ ਰਿਹਾ ਹੈ)

## ਕੋਡ ਸਟਾਈਲ ਮਾਰਗਦਰਸ਼ਕ

### Python

- ਵਾਤਾਵਰਣ ਚਲਕ ਪ੍ਰਬੰਧਨ ਲਈ `python-dotenv` ਦੀ ਵਰਤੋਂ ਕਰੋ
- API ਇੰਟਰਐਕਸ਼ਨਾਂ ਲਈ `openai` ਲਾਇਬ੍ਰੇਰੀ ਇੰਪੋਰਟ ਕਰੋ
- ਲਿੰਟਿੰਗ ਲਈ `pylint` ਵਰਤੋਂ ਕਰੋ (ਕੁਝ ਉਦਾਹਰਨਾਂ ਵਿਚ ਸਧਾਰਣਤਾ ਲਈ `# pylint: disable=all` ਸ਼ਾਮਿਲ ਹੈ)
- PEP 8 ਨਾਮਕਰਨ ਰਵਾਇਤਾਂ ਦੀ ਪਾਲਣਾ ਕਰੋ
- API ਸਾਕਾਂ `.env` ਫਾਇਲ ਵਿੱਚ ਰੱਖੋ, ਕੋਡ ਵਿੱਚ ਨਹੀਂ

### TypeScript

- ਵਾਤਾਵਰਣ ਚਲਕ ਲਈ `dotenv` ਪੈਕੇਜ ਦੀ ਵਰਤੋਂ ਕਰੋ
- ਹਰ ਐਪ ਲਈ `tsconfig.json` ਵਿੱਚ TypeScript ਸੰਰਚਨਾ
- Azure OpenAI ਲਈ `openai` ਪੈਕੇਜ ਵਰਤੋਂ (ਕਲਾਇੰਟ ਨੂੰ `/openai/v1/` endpoint ’ਤੇ ਦਿੱਖਾਓ ਅਤੇ `client.responses.create` ਕਾਲ ਕਰੋ); Microsoft Foundry ਮਾਡਲ ਲਈ `@azure-rest/ai-inference` ਵਰਤੋਂ
- ਵਿਕਾਸ ਲਈ `nodemon` ਵਰਤੋਂ ਕਰੋ ਜੋ ਆਟੋ-ਰੀਲੋਡ ਕਰਦਾ ਹੈ
- ਚਲਾਉਣ ਤੋਂ ਪਹਿਲਾਂ ਬਿਲਡ ਕਰੋ: `npm run build` ਫਿਰ `npm start`

- ਅਹੰ ਕੀ ਧਾਰਣਾਵਾਂ ਨੂੰ ਸਮਝਾਉਂਦੇ ਟਿੱਪਣੀਆਂ ਸ਼ਾਮਿਲ ਕਰੋ
- ਹਰ ਪਾਠ ਦਾ ਕੋਡ ਖੁਦ-ਮੁਖਤਾਰ ਅਤੇ ਚੱਲਣਯੋਗ ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ
- ਨਾਮਕਰਨ ਸਥਿਰ ਰੱਖੋ: Azure OpenAI ਲਈ `aoai-` ਪ੍ਰੀਫਿਕਸ, OpenAI API ਲਈ `oai-`, Microsoft Foundry Models ਲਈ `githubmodels-` (GitHub Models ਯੁੱਗ ਤੋਂ ਲਿਆਂਦਾ ਲੈਗੇਸੀ ਪ੍ਰੀਫਿਕਸ)

- ਕੋਡ ਉਦਾਹਰਨਾਂ ਦਾ ਮੈਨੂਅਲ ਟੈਸਟਿੰਗ
- ਮਾਰਕਡਾਊਨ ਵੈਧਤਾ ਲਈ GitHub Actions
- ਸਿੱਖਿਆ ਸਮੱਗਰੀ ਲਈ ਕਮਿਊਨਿਟੀ ਸਮੀਖਿਆ

## ਪੁਲ ਰਿਕਵੇਸਟ ਮਾਰਗਦਰਸ਼ਕ

### ਭੇਜਣ ਤੋਂ ਪਹਿਲਾਂ

1. ਜੇ ਲਾਗੂ ਹੁੰਦਾ ਹੈ ਤਾਂ Python ਅਤੇ TypeScript ਦੋਹਾਂ ਵਿੱਚ ਕੋਡ ਬਦਲਾਅ ਦੀ ਜਾਂਚ ਕਰੋ
2. ਮਾਰਕਡਾਊਨ ਵੈਧਤਾ ਚਲਾਓ (PR 'ਤੇ ਆਪਣੇ ਆਪ ਚੱਲਦੀ ਹੈ)
3. ਸਾਰੇ Microsoft URLs 'ਤੇ ਟ੍ਰੈਕਿੰਗ ID ਮੌਜੂਦ ਹੋਣ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ
4. ਸੰਬੰਧਤ ਲਿੰਕਾਂ ਦੀ ਸਹੀਤਾ ਜਾਂਚੋ
5. ਤਸਵੀਰਾਂ ਦੀ ਸਹੀ ਰਿਫਰੰਸਿੰਗ ਦੀ ਜਾਂਚ ਕਰੋ

### PR ਸਿਰਲੇਖ ਫਾਰਮੈਟ

- ਵਰਨਣਾਤਮਕ ਸਿਰਲੇਖ ਵਰਤੋਂ: `[Lesson 06] Fix Python example typo` ਜਾਂ `Update README for lesson 08`
- ਜੇ ਲਾਗੂ ਹੋਵੇ ਤਾਂ ਇਸ਼ੂ ਨੰਬਰ ਦਰਜ ਕਰੋ: `Fixes #123`

### PR ਵੇਰਵਾ

- ਕੀ ਬਦਲਾਅ ਕੀਤਾ ਗਿਆ ਅਤੇ ਕਿਉਂ, ਦਰਸਾਓ
- ਸੰਬੰਧਤ ਇਸ਼ੂਜ਼ ਨੂੰ ਲਿੰਕ ਕਰੋ
- ਕੋਡ ਬਦਲਾਅ ਲਈ, ਜਿਹੜੇ ਉਦਾਹਰਨ ਟੈਸਟ ਕੀਤੇ ਉਹ ਦਰਜ ਕਰੋ
- ਅਨੁਵਾਦ PR ਲਈ, ਪੂਰੇ ਅਨੁਵਾਦ ਲਈ ਸਾਰੇ ਫਾਇਲ ਸ਼ਾਮਿਲ ਕਰੋ

### ਯੋਗਦਾਨ ਦੀਆਂ ਲੋੜਾਂ

- Microsoft CLA 'ਤੇ ਸਾਈਨ ਕਰੋ (ਪਹਿਲੀ PR 'ਤੇ ਆਟੋਮੈਟਿਕ)
- ਬਦਲਾਅ ਕਰਨ ਤੋਂ ਪਹਿਲਾਂ ਰਿਪੌਜ਼ਟਰੀ ਨੂੰ ਫੋਰਕ ਕਰੋ
- ਹਰ ਲਾਜ਼ਮੀ ਬਦਲਾਅ ਲਈ ਇੱਕ PR (ਅਣਸੰਬੰਧਿਤ ਸੁਧਾਰਾਂ ਨੂੰ ਮਿਲਾਉਣਾ ਨਾ)
- ਜੇ ਸੰਭਵ ਹੋਵੇ ਤਾਂ PR ਛੋਟੇ ਅਤੇ ਕੇਂਦ੍ਰਿਤ ਰੱਖੋ

## ਆਮ ਵਰਕਫਲੋਜ਼

### ਨਵਾਂ ਕੋਡ ਉਦਾਹਰਨ ਜੋੜਨਾ

1. ਸਹੀ ਪਾਠ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਜਾਓ
2. `python/` ਜਾਂ `typescript/` ਉਪਡਾਇਰੈਕਟਰੀ ਵਿੱਚ ਉਦਾਹਰਨ ਬਣਾਓ
3. ਨਾਮਕਰਨ ਰਵਾਇਤ ਦੀ ਪਾਲਣਾ ਕਰੋ: `{provider}-{example-name}.{py|ts|js}`
4. ਅਸਲੀ API ਸਾਕਾਂ ਨਾਲ ਟੈਸਟ ਕਰੋ
5. ਕੋਈ ਵੀ ਨਵਾਂ ਵਾਤਾਵਰਣ ਚਲਕ ਪਾਠ README ਵਿੱਚ ਦਰਜ ਕਰੋ

### ਡੌਕਯੂਮੈਂਟੇਸ਼ਨ ਅਪਡੇਟ ਕਰਨਾ

1. ਪਾਠ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ README.md ਸੰਪਾਦਿਤ ਕਰੋ
2. ਮਾਰਕਡਾਊਨ ਮਾਰਗਦਰਸ਼ਕਾਂ ਦੀ ਪਾਲਣਾ ਕਰੋ (ਟ੍ਰੈਕਿੰਗ ID, ਸੰਬੰਧਤ ਲਿੰਕ)
3. ਅਨੁਵਾਦ GitHub Actions ਦੁਆਰਾ ਸੰਭਾਲੇ ਜਾਂਦੇ ਹਨ (ਹੱਥੋਂ ਸੋਧ ਨਹੀਂ ਕਰਨੇ)
4. ਸਾਰੇ ਲਿੰਕਾਂ ਦੀ ਮਾਨਤਾ ਹੋਣੀ ਚਾਹੀਦੀ ਹੈ

### Dev Containers ਨਾਲ ਕੰਮ ਕਰਨਾ

1. ਰਿਪੌਜ਼ਟਰੀ ਵਿੱਚ `.devcontainer/devcontainer.json` ਸ਼ਾਮਿਲ ਹੈ
2. ਪੋਸਟ-ਕ੍ਰੀਏਟ ਸਕ੍ਰਿਪਟ ਆਟੋਮੈਟਿਕ ਤੌਰ 'ਤੇ Python ਡਿਪੈਂਡੈਂਸੀਜ਼ ਇੰਸਟਾਲ ਕਰਦਾ ਹੈ
3. Python ਅਤੇ Jupyter ਐਕਸਟੈਂਸ਼ਨ ਪਹਿਲਾਂ ਤੋਂ ਸੰਰਚਿਤ ਹਨ
4. ਵਾਤਾਵਰਣ `mcr.microsoft.com/devcontainers/universal:2.11.2` 'ਤੇ ਅਧਾਰਿਤ ਹੈ

## ਡਿਪਲੋਇਮੈਂਟ ਅਤੇ ਪ੍ਰਕਾਸ਼ਨ

ਇਹ ਇੱਕ ਸਿੱਖਣ ਵਾਲੀ ਰਿਪੌਜ਼ਟਰੀ ਹੈ - ਇਸ ਵਿੱਚ ਕੋਈ ਡਿਪਲੋਇਮੈਂਟ ਪ੍ਰਕਿਰਿਆ ਨਹੀਂ ਹੈ। ਕੋਰਸ ਨੂੰ ਵਰਤਿਆ ਜਾਂਦਾ ਹੈ:

1. **GitHub ਰਿਪੌਜ਼ਟਰੀ**: ਕੋਡ ਅਤੇ ਡੌਕਯੂਮੈਂਟੇਸ਼ਨ ਤੱਕ ਸਿੱਧਾ ਪਹੁੰਚ
2. **GitHub Codespaces**: ਪ੍ਰੀ-ਕੰਫਿਗਰਡ ਸੈੱਟਅੱਪ ਨਾਲ ਤੁਰੰਤ ਵਿਕਾਸ ਵਾਤਾਵਰਣ
3. **Microsoft Learn**: ਸਮੱਗਰੀ ਅਧਿਕਾਰਿਕ ਸਿੱਖਣ ਪਲੇਟਫਾਰਮ ਤੇੋਂ ਪ੍ਰਸਾਰਿਤ ਕੀਤੀ ਜਾ ਸਕਦੀ ਹੈ
4. **docsify**: ਮਾਰਕਡਾਊਨ ਤੋਂ ਬਣਿਆ ਡੌਕਯੂਮੈਂਟੇਸ਼ਨ ਸਾਈਟ (ਦੇਖੋ `docsifytopdf.js` ਅਤੇ `package.json`)

### ਡੌਕਯੂਮੈਂਟੇਸ਼ਨ ਸਾਈਟ ਬਣਾਉਣਾ

```bash
# ਦਸਤਾਵੇਜ਼ ਤੋਂ PDF ਤਿਆਰ ਕਰੋ (ਜੇ ਲੋੜ ਹੋਵੇ)
npm run convert
```

## ਸਮੱਸਿਆ ਨਿਵਾਰਣ

### ਆਮ ਸਮੱਸਿਆਵਾਂ

**Python ਇੰਪੋਰਟ ਗਲਤੀਆਂ**:
- ਵਰਚੁਅਲ ਵਾਤਾਵਰਣ ਚਾਲੂ ਹੋਣ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ
- `pip install -r requirements.txt` ਚਲਾਓ
- Python ਸੰਸਕਰਨ 3.9+ ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ

**TypeScript ਬਿਲਡ ਗਲਤੀਆਂ**:
- ਖਾਸ ਐਪ ਡਾਇਰੈਕਟਰੀ ਵਿੱਚ `npm install` ਚਲਾਓ
- Node.js ਦਾ ਸੰਸਕਰਨ ਅਨੁਕੂਲ ਹੋਣਾ ਚਾਹੀਦਾ ਹੈ
- `node_modules` ਨੂੰ ਸਾਫ਼ ਕਰੋ ਅਤੇ ਲੋੜ ਹੋਣ 'ਤੇ ਦੁਬਾਰਾ ਇੰਸਟਾਲ ਕਰੋ

**API ਪ੍ਰਮਾਣਿਕਤਾ ਗਲਤੀਆਂ**:
- `.env` ਫਾਇਲ ਦੀ ਮੌਜੂਦਗੀ ਅਤੇ ਸਹੀ ਮੁੱਲਾਂ ਦੀ ਜਾਂਚ ਕਰੋ
- API ਕੁੰਜੀਆਂ ਨੂੰ ਸਹੀ ਅਤੇ ਮਿਆਦ ਵਿਚ ਹੋਣ ਦੀ ਪੁਸ਼ਟੀ ਕਰੋ
- ਆਪਣੇ ਖੇਤਰ ਲਈ endpoint URLs ਸਹੀ ਹਨ ਇਹ ਯਕੀਨੀ ਬਣਾਓ

**ਵਾਤਾਵਰਣ ਚਲਕ ਗੁੰਮ ਹੋਇਆ**:
- `.env.copy` ਨੂੰ `.env` ਵਿੱਚ ਨਕਲ ਕਰੋ
- ਜੇਹੜੇ ਪਾਠ 'ਤੇ ਕੰਮ ਕਰ ਰਹੇ ਹੋ ਉਸ ਦੇ ਲਈ ਸਾਰੇ ਲੋੜੀਂਦੇ ਮੁੱਲ ਪੂਰੇ ਕਰੋ
- `.env` ਅਪਡੇਟ ਕਰਨ ਤੋਂ ਬਾਅਦ ਆਪਣੀ ਐਪਲੀਕੇਸ਼ਨ ਨੂੰ ਦੁਬਾਰਾ ਚਾਲੂ ਕਰੋ
