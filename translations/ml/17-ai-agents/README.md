## പരിചയം

AI ഏജന്റുകൾ ജെനെറേറ്റീവ് AI യിൽ ഒരു രസകരമായ പുരോഗതിയാണ്, വലിയ ഭാഷാ മോഡലുകൾ (LLMs) അസിസ്റ്റന്റുകളിൽ നിന്നും നടപടികൾ കൈക്കൊള്ളാൻ കഴിവുള്ള ഏജന്റുകളായി വികസിക്കാൻ സഹായിക്കുന്നു. AI ഏജന്റ് ഫ്രെയിംവർക്കുകൾ വികസിപ്പകർക്ക് LLMs-നു ഉപകരണങ്ങൾക്കും സ്റ്റേറ്റ് മാനേജ്മെന്റിനും ആക്സസ് നൽകുന്ന അപ്ലിക്കേഷനുകൾ സൃഷ്ടിക്കാൻ അനുവദിക്കുന്നു. ഈ ഫ്രെയിംവർക്കുകൾ ദൃശ്യ വിന്യാസവും മെച്ചപ്പെടുത്തുന്നു, അതുവഴി ഉപയോക്താക്കളും ഡെവലപ്പർമാരും LLM സുപ്രധാനമായ പ്രവർത്തനങ്ങൾ നിരീക്ഷിക്കാൻ കഴിയും, അനുഭവ മാനേജ്മെന്റ് മെച്ചപ്പെടുത്തുന്നുണ്ട്.

പാഠം താഴെയുള്ള മേഖലകൾ ഉൾക്കൊള്ളുന്നു:

- AI ഏജന്റ് എന്താണ് എന്ന് മനസ്സിലാക്കൽ - AI ഏജന്റ് എന്തുകൊണ്ടാണ്?
- അഞ്ച് വ്യത്യസ്ഥ AI ഏജന്റ് ഫ്രെയിംവർക്കുകൾ പരിചയപ്പെടുത്തൽ - അവ എന്തുകൊണ്ടാണ് പ്രത്യേകതയുള്ളത്?
- വിവിധ ഉപയോഗകേസ്-കളിൽ AI ഏജന്റുകൾ ഉപയോഗിക്കുന്നത് - എപ്പോഴാണ് AI ഏജന്റുകൾ ഉപയോഗിക്കേണ്ടത്?

## പഠന ലക്ഷ്യങ്ങൾ

ഈ പാഠം കഴിഞ്ഞ ശേഷം, നിങ്ങൾക്ക് കഴിയും:

- AI ഏജന്റുകൾ എന്താണെന്നും അവ എങ്ങനെ ഉപയോഗിക്കാമെന്നുമെപ്പറ്റി വിവരിക്കുക.
- ചില ജനപ്രിയ AI ഏജന്റ് ഫ്രെയിംവർക്കുകളുടെ വ്യത്യാസങ്ങൾ മനസ്സിലാക്കുക.
- AI ഏജന്റുകൾ എങ്ങനെ പ്രവർത്തിക്കുന്നുവെന്ന് അറിയുക, അവ ഉപയോഗിച്ച് അപ്ലിക്കേഷനുകൾ നിർമ്മിക്കാൻ കഴിയും.

## AI ഏജന്റുകൾ എന്തൊക്കെയാണ്?

AI ഏജന്റുകൾ ജെനെറേറ്റീവ് AI ലോകത്ത് വളരെ രസകരമായ ഒരു മേഖലയാണ്. ഈ ആവേശത്തോടൊപ്പം ചിലപ്പോൾ പദങ്ങൾക്കും അവയുടെ പ്രയോഗത്തിനും ആശയക്കുഴപ്പം ഉണ്ടാകാം. പരമാവധി ഉപകരണങ്ങളെ ഉൾപ്പെടുത്തി ലളിതവും ഉൾക്കൊള്ളുന്നതുമായ ഒരു നിർവചനം ഉപയോഗിക്കുന്നു:

AI ഏജന്റുകൾ ലാർജ്ഭാഷാ മോഡലുകൾ (LLMs)ക്ക് **സ്റ്റേറ്റ്**ക്കും **ഉപകരണങ്ങൾ**ക്കും ആക്സസ് നൽകി ജോലി ചെയ്യാൻ അനുവദിക്കുന്നു.

![Agent Model](../../../translated_images/ml/what-agent.21f2893bdfd01e6a.webp)

ഈ പദങ്ങൾ നിർവ്വചിച്ചാൽ:

**വലിയ ഭാഷാ മോഡലുകൾ** - GPT-5, GPT-4o, Llama 3.3 തുടങ്ങിയവ ഈ കോഴ്സിൽ പരാമർശിക്കപ്പെട്ട മോഡലുകളാണ്.

**സ്റ്റേറ്റ്** - LLM പ്രവർത്തിക്കുന്ന പ്രেক্ষിതമാണ് ഇത്. മുൻനടപടികളുടെ പ്രസംഗവും നിലവിലെ പ്രസംഗവും ഉപയോഗിച്ച്, LLM അടുത്ത നടപടികൾക്ക് മാർഗ്ഗനിർദ്ദേശം നൽകുന്നു. AI ഏജന്റ് ഫ്രെയിംവർക്കുകൾ ഈ പ്രസംഗം എളുപ്പത്തിൽ കൈകാര്യം ചെയ്യുന്നതിന് ആധാരം നൽകുന്നു.

**ഉപകരണങ്ങൾ** - ഉപയോക്താവ് അപേക്ഷിച്ച ദൗത്യവും LLM ആസൂത്രണം ചെയ്തതും പൂർത്തിയാക്കാൻ ഉപകരണങ്ങൾ ആവശ്യമാണ്. ഒരു ഡാറ്റാബേസ്, API, ബാഹ്യ അപ്ലിക്കേഷൻ, മറ്റൊരു LLM-വുമൊക്കെ ഉദാഹരണങ്ങളാകാം!

ഈ നിർവചനങ്ങൾ മുമ്പോട്ടു പോകാൻ നല്ല ആധാരം നൽകും, ഇനി എന്തുപോലെ ഈ മോഡലുകൾ ഉപയോഗിക്കപ്പെടുന്നു എന്ന് പരിശോധിക്കാം. വ്യത്യസ്ത AI ഏജന്റ് ഫ്രെയിംവർക്കുകൾ കുറച്ച് പരിചയപ്പെടാം:

## LangChain ഏജന്റുകൾ

[LangChain ഏജന്റുകൾ](https://python.langchain.com/docs/how_to/#agents?WT.mc_id=academic-105485-koreyst) മുകളിൽ പറഞ്ഞ നിർവചനങ്ങൾ നടപ്പിലാക്കിയ ഒരു പ്രയോഗമാണ്.

**സ്റ്റേറ്റ്** നിയന്ത്രിക്കാൻ, ഇത് `AgentExecutor` എന്ന റെഡി ഫംഗ്ഷൻ ഉപയോഗിക്കുന്നു. ഇത് നിർവ്വചിച്ച `agent`വും ലഭ്യമായ `tools`-ഉം സ്വീകരിക്കുന്നു.

`Agent Executor` ചാറ്റ് ചരിത്രം സൂക്ഷിക്കുകയും ചാറ്റിന്റെ സാന്ദർഭ്യം നൽകുകയും ചെയ്യുന്നു.

![Langchain Agents](../../../translated_images/ml/langchain-agents.edcc55b5d5c43716.webp)

LangChain കമ്മ്യൂണിറ്റിയും ടീമും ഒരുക്കിയ [ഉപകരണങ്ങളുടെ ക്യാറ്റലോഗ്](https://integrations.langchain.com/tools?WT.mc_id=academic-105485-koreyst) നിങ്ങളുടെ അപ്ലിക്കേഷനിലേക്ക് ഇറക്കുമതി ചെയ്ത് LLM ആക്സസ് ചെയ്യാം.

നിങ്ങൾക്ക് ഈ ഉപകരണങ്ങൾ നിർവ്വചിച്ച് `Agent Executor`-ലേക്ക് നൽകാം.

ദൃശ്യമായ നിയന്ത്രണം AI ഏജന്റുകൾക്കുപറഞ്ഞപ്പോഴെല്ലാം പ്രധാനമാണ്. ഏജന്റുകൾ ഉപയോഗിക്കുന്ന ഉപകാരം ഏതെന്ന് അറിയാൻ ഡെവലപ്പർമാർക്കു കഴിയണം. അതിനായി LangChain ടീം LangSmith വികസിപ്പിച്ചിട്ടുണ്ട്.

## AutoGen

അടുത്ത AI ഏജന്റ് ഫ്രെയിംവർക്ക് ആണ് [AutoGen](https://microsoft.github.io/autogen/?WT.mc_id=academic-105485-koreyst). AutoGen-ന്റെ പ്രധാന ശ്രദ്ധ സംഭാഷണങ്ങളിലാണ്. ഏജന്റുകൾ ഒരുപോലെ **സംഭാഷണയോഗ്യവും** **ഇച്ഛാനുസൃതവുമാണ്**.

**സംഭാഷണയോഗ്യമായ -** LLMs മറ്റൊരു LLM-ഉടൊപ്പം സംഭാഷണം തുടങ്ങുകയും തുടർച്ചയായി നടത്തിയേക്കാം, ദൗത്യം പൂർത്തിയാക്കാനായി. ഇത് `AssistantAgents` സൃഷ്ടിച്ച് പ്രത്യേക സിസ്റ്റം സന്ദേശം കൊടുത്ത് ചെയ്യുന്നു.

```python

autogen.AssistantAgent( name="Coder", llm_config=llm_config, ) pm = autogen.AssistantAgent( name="Product_manager", system_message="Creative in software product ideas.", llm_config=llm_config, )

```

**ഇച്ഛാനുസൃതമായ** - ഏജന്റുകൾ LLM മാത്രമല്ല, ഉപയോക്താവ് അല്ലെങ്കിൽ ഉപകരണമായി നിർവ്വചിക്കാം. ഡെവലപ്പർ `UserProxyAgent` നിർവ്വചിച്ച് ഉപയോക്താവിനോടുള്ള പ്രതികരണത്തിനായി ഏജന്റിനെ ബാധകമാക്കി, ദൗത്യം തുടരുകയോ അവസാനിപ്പിക്കുകയോ ഉള്ളതിന്റെ തീരുമാനമെടുക്കാൻ കഴിയുന്നു.

```python
user_proxy = UserProxyAgent(name="user_proxy")
```

### സ്റ്റേറ്റ്‌ ഉം ഉപകരണങ്ങളും

സ്റ്റേറ്റ് മാറാനും നിയന്ത്രിക്കാനും, അസിസ്റ്റന്റ് ഏജന്റ് പൈത്തൺ കോഡ് സൃഷ്ടിച്ച് ദൗത്യം പൂർത്തിയാക്കുന്നു.

പ്രവർത്തനത്തെക്കുറിച്ചുള്ള ഒരു ഉദാഹരണം:

![AutoGen](../../../translated_images/ml/autogen.dee9a25a45fde584.webp)

#### സിസ്റ്റം സന്ദേശത്തോടെ LLM നിർവ്വചിക്കൽ

```python
system_message="For weather related tasks, only use the functions you have been provided with. Reply TERMINATE when the task is done."
```

ഈ സിസ്റ്റം സന്ദേശം ഈ LLM-ന് ദൗത്യത്തിനായുള്ള പരിപ്രേക്ഷ്യത്തിലുള്ള ഫംഗ്ഷനുകൾ ഏതാണ് എന്ന് നിർദ്ദേശിക്കുന്നു. AutoGen-ൽ പല വിഭാഗത്തിലുള്ള AssistantAgents വ്യത്യസ്ത സിസ്റ്റം സന്ദേശങ്ങളോടെ ഉണ്ടാകാവുന്നതാണ്.

#### ഉപയോക്താവ് ചേർത്ത് സംഭാഷണം ആരംഭിക്കുന്നു

```python
user_proxy.initiate_chat( chatbot, message="I am planning a trip to NYC next week, can you help me pick out what to wear? ", )

```

ഉപയോക്താവായ `user_proxy (Human)` എന്ന സന്ദേശം ഏജന്റിന്റെ ദൗത്യം നിർവ്വചിക്കാൻ തുടക്കം കുറിക്കുന്നു.

#### ഫംഗ്ഷൻ നടപ്പാക്കുന്നു

```bash
chatbot (to user_proxy):

***** Suggested tool Call: get_weather ***** Arguments: {"location":"New York City, NY","time_periond:"7","temperature_unit":"Celsius"} ******************************************************** --------------------------------------------------------------------------------

>>>>>>>> EXECUTING FUNCTION get_weather... user_proxy (to chatbot): ***** Response from calling function "get_weather" ***** 112.22727272727272 EUR ****************************************************************

```

ആദ്യത്തെ ചാറ്റ് പ്രോസസ് ചെയ്തതിനു ശേഷം, ഏജന്റ് നിർദ്ദേശിച്ച ഉപകരണമുറെ വിളിക്കും. ഈ ഉദാഹരണത്തിൽ `get_weather` എന്ന് വിളിക്കപ്പെടുന്ന ഫംഗ്ഷനാണ്. നിങ്ങളുടെ ക്രമീകരണത്തിന് അനുസരിച്ച്, ഈ ഫംഗ്ഷൻ ഓട്ടോമാറ്റിക്കായി മികവുറ്റ് ചുട്ടനടത്താം അല്ലെങ്കിൽ ഉപയോക്തൃ ഇൻപുട്ടിൽനിന്ന് ആയിരിക്കും നടപ്പാക്കുക.

നിങ്ങൾക്ക് കൂടുതൽ പഠിക്കാൻ [AutoGen കോഡ് സാമ്പിൾസ്](https://microsoft.github.io/autogen/docs/Examples/?WT.mc_id=academic-105485-koreyst) കാണാം.

## Microsoft Agent Framework

[Microsoft Agent Framework](https://learn.microsoft.com/agent-framework/?WT.mc_id=academic-105485-koreyst) മൈക്രോസോഫ്റ്റിന്റെ AI ഏജന്റുകളും ബഹുഉപകരണ സംവിധാനങ്ങളും Python-യും .NET-ഉം ഉപയോഗിച്ച് നിർമ്മിക്കുന്ന ഉത്ഘാടന കോഡാണ്. ഇത് രണ്ടു പഴയ മൈക്രോസോഫ്റ്റ് പ്രോജക്റ്റുകളുടെ ശക്തികൾ - **Semantic Kernel** എന്ന എന്റർപ്രൈസ് ഫീച്ചറുകളും **AutoGen**-ന്റെ ബഹുഉപകരണ ചിട്ടകളും - ഒരുമിച്ച് ഒരു പിന്തുണയുള്ള ഫ്രെയിംവർക്ക് ആയി ചേർക്കുന്നു. ഇന്ന് പുതിയ ഏജന്റ് പ്രോജക്ട് ആരംഭിക്കുന്നുവെങ്കിൽ, AutoGen-ന്റെ ശേഷിയുള്ളവ ഇവിടെ ആരാധിക്കപ്പെട്ടിരിക്കുന്നു.

ഫ്രെയിംവർക്ക് ഒരു ഏക ഏക ഏജന്റ് മുതൽ സങ്കീർണ്ണമായ ബഹുഉപകരണ വേർകിങ് വർക്ക് ഫ്ലോ വരെ സഹായിക്കുന്നു, ഇത് Microsoft Foundry, Azure OpenAI, OpenAI എന്നിവയുമായി നേരിട്ട് ഇന്റഗ്രേറ്റ് ചെയ്യുന്നു. കൂടാതെ OpenTelemetry വഴി ഉൾപ്പെട്ട ദൃശ്യമാക്കലും നൽകുന്നു, അതിലൂടെ ഏജന്റുകൾ ചെയ്യുന്നതെന്താണെന്ന് കൃത്യമായി ട്രേസ് ചെയ്യാം.

### സ്റ്റേറ്റ്‌ ഉം ഉപകരണങ്ങളും

**സ്റ്റേറ്റ്** - ഫ്രെയിംവർക്ക് ആശയവിനിമയംക്കുള്ള സാന്ദർഭ്യം **ത്രീഡുകൾ** മുഖേന കൈകാര്യം ചെയ്യുന്നു. ഏജന്റ് സന്ദേശ ചരിത്രം (ഉപയോക്തൃ അഭ്യർത്ഥനകൾ, ഉപകരണ വിളികൾ, ഫലങ്ങൾ) ശ്രദ്ധിക്കുന്നു, അതിലൂടെ ഓരോ തിരിയിലും മുമ്പത്തെത് വികസിപ്പിക്കുന്നു. ത്രീഡുകൾ നിലനിർത്താം, സംഭാഷണം നേരിട്ടോ രാവിലെ തുടർച്ചയായി നിന്നു നടത്താം.

**ഉപകരണങ്ങൾ** - സാദാരണ Python ഫംഗ്ഷനുകൾ വഴി ഏജന്റിന് ഉപകരണങ്ങൾ നൽകുന്നു. ടൈപ്പ്-അനോട്ടേഷൻ നൽകിയ പാരാമീറ്ററുകൾ സ്കീമയാക്കി മോഡൽ എപ്പോൾ വിളിക്കാമെന്ന് മനസ്സിലാക്കുന്നു (ഫംഗ്ഷൻ കോളിംഗ്). അംഗീകരിച്ച Model Context Protocol (MCP) സെർവറുകളും കോഡ് ഇന്റർപ്രറ്റർ പോലെയുള്ള ഹോസ്റ്റുചെയ്ത ഉപകരണങ്ങളും പിന്തുണയുണ്ട്.

ഒരു കസ്റ്റം ഉപകരണമുണ്ട് ഒരേ ഏജന്റിന്റെ ഉദാഹരണം ഇവിടെ:

```python
import asyncio
from typing import Annotated

from pydantic import Field
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient


def get_weather(
    location: Annotated[str, Field(description="The location to get the weather for.")],
) -> str:
    """Get the weather for a given location."""
    return f"The weather in {location} is sunny with a high of 22°C."


async def main():
    agent = Agent(
        client=OpenAIChatClient(),
        instructions="You are a helpful assistant that can answer weather questions.",
        tools=[get_weather],
    )

    response = await agent.run("What's the weather in Amsterdam?")
    print(response)


asyncio.run(main())
```

വിളയില്‍ Microsoft Foundry-യിൽ Azure OpenAI-യില് കണക്ട് ചെയ്യാൻ, ക്ലയന്റിനോട് എൻഡ്‌പോയിന്റും ക്രെഡെൻഷ്യൽസും നൽകുക:

```python
from azure.identity.aio import AzureCliCredential
from agent_framework.openai import OpenAIChatClient

client = OpenAIChatClient(
    model="my-gpt-5-mini-deployment",
    azure_endpoint="https://my-resource.openai.azure.com",
    credential=AzureCliCredential(),
)
```

### ബഹുഉപകരണ വേർകിങ് ഫ്ലോകൾ

ഫ്രെയിംവർക്ക് ഏറ്റവും പ്രാധാന്യമാർന്നത് ഒന്നിലധികം ഏജന്റുകൾ ചിട്ടപ്പെടുത്തിയെടുക്കുന്നതിലാണ്. ഉദാഹരണത്തിന്, ഏജന്റുകൾ ഒരു ശേഷം ഒരു പ്രവർത്തിക്കാം (ഓരോ കോണ്ടെക്സ്റ്റും അടുത്തതിലേക്ക് കൈമാറി) അല്ലെങ്കിൽ ഒന്നിച്ച് ഒന്നിലധികം ഏജന്റുകൾ പ്രവർത്തിപ്പിച്ച് ഫലങ്ങൾ സമാഹരിക്കാം:

```python
from agent_framework.orchestrations import SequentialBuilder, ConcurrentBuilder

# സംവാദ സാന്ദർഭ്യം ശൃംഖലയായി കടത്തുന്നുണ്ടാകെ ഏജന്റുകളെ പരമ്പരയിൽ പ്രവർത്തിപ്പിക്കുക
sequential = SequentialBuilder(participants=[researcher, writer, editor]).build()

# ഏജന്റുകളിലേക്ക് പരസ്പരം സമാന്തരമായി പ്രക്ഷേപണം ചെയ്ത്, തുടർന്ന് അവയുടെ പ്രതികരണങ്ങൾ സമാഹരിക്കുക
concurrent = ConcurrentBuilder(participants=[analyst_a, analyst_b, analyst_c]).build()
```

ഫ്രെയിംവർക്ക് ഇൻസ്റ്റാൾ ചെയ്ത് തുടങ്ങാനായി:

```bash
pip install agent-framework-core
# ঐচ্ছিক സംയോജനങ്ങൾ
pip install agent-framework-openai       # OpenAI and Azure OpenAI
pip install agent-framework-foundry      # Microsoft Foundry
```

[Microsoft Agent Framework റിപോസിറ്ററിയും](https://github.com/microsoft/agent-framework?WT.mc_id=academic-105485-koreyst) [അധികൃത ഡോക്യുമെന്റേഷനും](https://learn.microsoft.com/agent-framework/?WT.mc_id=academic-105485-koreyst) പരീക്ഷിക്കുക.

## Taskweaver

അടുത്ത ഏജന്റ് ഫ്രെയിംവർക്ക് ആണ് [Taskweaver](https://microsoft.github.io/TaskWeaver/?WT.mc_id=academic-105485-koreyst). ഇത് "കോഡ്-ഫസ്റ്റ്" ഏജന്റ് എന്നറിയപ്പെടുന്നു, കാരണം ഇത് `strings`-ൽകൂടാതെ Python-ൽ DataFrames-ഉം പ്രയോഗിക്കവచ్చు. ഈ കഴിവ് ഡാറ്റാ വിശകലനത്തിനും സൃഷ്ടിയ്ക്കും വളരെ ഉപയോഗപ്രദമാണ്. ഗ്രാഫുകളും ചാർട്ടുകളും സൃഷ്ടിക്കുന്നതടക്കമുള്ള കാര്യങ്ങൾക്കും, യാദൃച്ഛിക സംഖ്യകൾ സൃഷ്ടിക്കുന്നതിനും ഇത് സഹായിക്കുന്നു.

### സ്റ്റേറ്റ്‌ ഉം ഉപകരണങ്ങളും

സംഭാഷണത്തിന്റെ സ്റ്റേറ്റ് നിയന്ത്രിക്കാൻ TaskWeaver `Planner` എന്ന ആശയം ഉപയോഗിക്കുന്നു. `Planner` എൽഎൽഎം ഉപയോക്താക്കളുടെ അഭ്യർത്ഥന സ്വീകരിച്ച്, പൂർത്തിയാക്കേണ്ട പ്രവർത്തനങ്ങൾ സൂചിപ്പിക്കുന്നു.

പ്രവർത്തനങ്ങൾ പൂർത്തിയാക്കാനായി, `Planner`-ന് `Plugins` എന്ന ഉപകരണ സമാഹാരത്തിൽ ആക്സസ് ഉണ്ട്. Python ക്ലാസ്സുകളോ പൊതുവായ കോഡ് ഇന്റർപ്രറ്ററോ ആയിരിക്കും ഇത്. ഈ പ്ലഗിനുകൾ embeddings ആയി സൂക്ഷിക്കുന്നു, ഒറ്റത്തവണ തിരയലിൽ LLM-ന് ഉപകരണം കണ്ടെത്താൻ കുറഞ്ഞ സമയം വേണ്ടി വരും.

![Taskweaver](../../../translated_images/ml/taskweaver.da8559999267715a.webp)

അനോമലി ഡിസ്കവറി കൈകാര്യംചെയ്യുന്ന ഒരു പ്ലഗിൻ ഉദാഹരണം:

```python
class AnomalyDetectionPlugin(Plugin): def __call__(self, df: pd.DataFrame, time_col_name: str, value_col_name: str):
```

കോഡ് നടപ്പിലാക്കുന്നതിന് മുമ്പ് പരിശോധിക്കും. Taskweaver-ൽ മറ്റൊരു സവിശേഷത `experience` എന്നതാണ്, ഇത് സംഭാഷണത്തിന്റെ പരിസരം ദീർഘകാലം YAML ഫയലായി സൂക്ഷിക്കാം. മുൻ സംഭാഷണങ്ങൾ LLM മെച്ചപ്പെടുത്താൻ സഹായിക്കും.

## JARVIS

അവസാന ഏജന്റ് ഫ്രെയിംവർക്ക് ആണ് [JARVIS](https://github.com/microsoft/JARVIS?tab=readme-ov-file&WT.mc_id=academic-105485-koreyst). JARVIS-ന്റെ പ്രത്യേകത LLM സംഭാഷണത്തിന്റെ `സ്റ്റേറ്റ്` നിയന്ത്രിക്കുകയും `ഉപകരണങ്ങൾ` ആയി മറ്റ് AI മോഡലുകൾ ഉപയോഗിക്കുകയും ചെയ്യുന്നു. ഓരോ AI മോഡലും പ്രത്യേക ദൗത്യങ്ങൾ (വസ്‌തു കണ്ടെത്തൽ, ട്രാൻസ്ക്രിപ്ഷൻ, ചിത്ര വിവരണം എന്നിവ) നിർവഹിക്കുന്നു.

![JARVIS](../../../translated_images/ml/jarvis.762ddbadbd1a3a33.webp)

LLM പൊതുവായ മോഡലായതിനാൽ ഉപഭോക്തൃ അഭ്യർത്ഥന സ്വീകരിച്ച് പ്രത്യേക ദൗത്യം കണ്ടെത്തുകയും അതിന് വേണ്ട പാരാമീറ്ററുകൾ/ഡാറ്റ കണ്ടെത്തുകയും ചെയ്യുന്നു.

```python
[{"task": "object-detection", "id": 0, "dep": [-1], "args": {"image": "e1.jpg" }}]
```

പിന്നീട് LLM അഭ്യർത്ഥന പ്രത്യേക AI മോഡലുകൾക്ക് ഫോർമാറ്റ് ചെയ്യുന്നു, ഉദാഹരണത്തിന് JSON. AI മോഡലുകൾ പ്രവചനങ്ങൾ നൽകി LLM-ന് തിരിച്ചറിയുന്നു.

ദൗത്യം പൂർത്തിയാക്കാൻ വ്യത്യസ്ത മോഡലുകൾ ആവശ്യപ്പെട്ടാൽ, LLM അവയുടെ പ്രതികരണങ്ങൾ കൂടിച്ചേർത്ത് ഉപയോക്താവിന് മറുപടി സൃഷ്ടിക്കുന്നു.

താഴെ ഉദാഹരണത്തിൽ, ഉപയോക്താവ് ഒരു ചിത്രത്തിലെ വസ്തു വിവരണവും എണ്ണവും തേടുന്നു:

## അസൈൻമെന്റ്

AI ഏജന്റുകളെ Microsoft Agent Framework ഉപയോഗിച്ച് പഠനം തുടരാൻ, നിങ്ങൾക്ക് കഴിയും:

- വിദ്യാഭ്യാസ സ്റ്റാർട്ടപ്പിന്റെ വിവിധ വകുപ്പുകളുമായി ഒരു ബിസിനസ് മീറ്റിങ് സിമുലേറ്റ് ചെയ്യുന്നത്.
- വ്യത്യസ്ത വ്യക്തിത്വങ്ങളും മുൻഗണനകളും മനസ്സിലാകാൻ LLM-നു സഹായിക്കുന്ന സിസ്റ്റം സന്ദേശങ്ങൾ സൃഷ്ടിക്കുക, ഉപയോക്താവിന് പുതിയ ഉൽപ്പന്ന ആശയം അവതരിപ്പിക്കാൻ അനുവദിക്കുക.
- LLM ആകര്‍ഷകമായ ഉൽപ്പന്ന ആശയം മെച്ചപ്പെടുത്താനായി ഓരോ വകുപ്പുകാരോടും അനുസൃതമായ ഫോളോ-അപ്പ് ചോദ്യങ്ങൾ സൃഷ്ടിക്കും.
