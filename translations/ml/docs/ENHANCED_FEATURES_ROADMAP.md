# ഉയർത്തിപ്പെടുത്തിയ ഫീച്ചറുകളും മെച്ചപ്പെടുത്തലുകളും റോഡ്മാപ്

ഇൻഡസ്ട്രി മികച്ച പ്രാക്ടിസുകളുടെയും സമഗ്രമായ കോഡ് റിവ്യുവിന്റെയും അടിസ്ഥാനത്തിൽ, ജെനറേറ്റിവ് എഐ ഫോർ ബിഗിനേഴ്‌സ് പാഠ്യപദ്ധതിക്ക് ശുപാർശ ചെയ്ത മെച്ചപ്പെടുത്തലുകളും നവീകരണങ്ങളും ഈ രേഖ രേഖപ്പെടുത്തിയിട്ടുണ്ട്.

## ഏകോപിത സംക്ഷേപം

സുരക്ഷ, കോഡ് ഗുണമേന്മ, വിദ്യാഭ്യാസപ്രഭാവം എന്നിവയെ അടിസ്ഥാനമാക്കി കോഡ് ബേസ് വിശകലനം ചെയ്തു. ഈ രേഖ ഉടൻ പരിഹരിക്കേണ്ട തുണകളും, അടുത്തിടെ നടത്തേണ്ട മെച്ചപ്പെടുത്തലുകളും, ഭാവിയിലെ നവീകരണങ്ങളും ശുപാർശ ചെയ്യുന്നു.

## 4. API മോഡേൺമൈസേഷൻ

### 4.1 പഴയ എപി ഐ പാറ്റേണുകൾ (മൈഗ്രേഷൻ പൂർത്തിയായി)

എല്ലാ പൈതൺ, ടൈപ്പ്സ്ക്രിപ്റ്റ് **ചാറ്റ്** സാമ്പിളുകളും ചാറ്റ് കോംപ്ലീഷൻസ് APIയില്‍ നിന്നും **റസ്പോണ്‍സസ് API** (`client.responses.create(...)` → `response.output_text`) എന്നിലേക്ക് മാറ്റിയിട്ടുണ്ട്.

| പഴയ പാറ്റേൺ | പുതിയ പാറ്റേൺ | സ്ഥിതി |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (ചാറ്റ്) | `OpenAI(base_url="<endpoint>/openai/v1/")` (റസ്പോണ്‍സസ് API) | പൂർത്തിയായി |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | പൂർത്തിയായി |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (ടൈപ്പ്സ്ക്രിപ്റ്റ്) | `openai` പാക്കേജ് `client.responses.create()` → `response.output_text` | പൂർത്തിയായി |
| `df.append()` (pandas) | `pd.concat()` | പൂർത്തിയായി |

> **ശ്രദ്ധിക്കുക:** Microsoft Foundry Models സാമ്പിളുകൾ, `azure-ai-inference` / `@azure-rest/ai-inference` SDK (`client.complete()`) ഉപയോഗിക്കുന്നവ മോഡൽ ഇൻഫറൻസ് API-യിൽ തുടരുന്നു, ഇതു റെസ്പോൺസസ് API പിന്തുണയ്ക്കുന്നില്ല. `AzureOpenAI()` ഉദ്ദേശ്യത്തോടെ നിലനിർത്തപ്പെട്ടിരിക്കുന്നു (എംബെഡിംഗ്സ്, ഇമേജ് ജനറേഷൻ).

### 4.2 പുതിയ API ഫീച്ചറുകൾ പ്രദർശിപ്പിക്കുക

1. **സ്റ്റ്രക്ചർഡ് ഔട്ട്പുട്ടുകൾ** (OpenAI)
   - JSON മോഡ്
   - കർശന സ്കീമയോടെ ഫംഗ്ഷൻ കോളിംഗ്

2. **വിഷൻ കഴിവുകൾ**
   - GPT-4o (vision) ഉപയോഗിച്ച് ഇമേജ് വിശകലനം
   - മൾട്ടി-മോഡൽ പ്രാപംപ്റ്റുകൾ

3. **റസ്പോൺസസ് API ബിൽറ്റ്-ഇൻ ടൂളുകൾ** (പഴയ അസിസ്റ്റന്റ്സ് API യെ മറികടക്കും)
   - കോഡ് ഇന്റർപ്രിറ്റർ
   - ഫയൽ സെർച്ച്
   - വെബ് സെർച്ച്, ഇഷ്ടാനുസൃത ടൂളുകൾ

## 5. ഇൻഫ്രാസ്ട്രക്ചർ മെച്ചപ്പെടുത്തലുകൾ

### 5.1 CI/CD നവീകരണങ്ങൾ

[`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml)യിൽ നടപ്പാക്കി: പൈതൺ ലിൻറ്റിംഗ്/ഫോർമാറ്റിംഗ് (Ruff + Black) `shared/` യൂട്ടിലിറ്റീസ് മോഡ്യൂളിൽ നിർബന്ധിതമാണ്, ബാക്കി പാഠ്യപദ്ധതിയിൽ നിർദ്ദേശാത്മകമാണ്, ജാവാസ്ക്രിപ്റ്റ്/ടൈപ്പ്സ്ക്രിപ്റ്റിനും നിർദ്ദേശാത്മക ESLint പാസ്. ഉദാഹരണ അടിസ്ഥാന മാര്‍ക്കുണ്ട്:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### 5.2 സുരക്ഷാ സ്കാനിംഗ്

[`.github/workflows/security.yml`](../../../.github/workflows/security.yml)ൽ നടപ്പാക്കി: Python, ജാവാസ്ക്രിപ്റ്റ്/ടൈപ്പ്സ്ക്രിപ്റ്റ് കോഡ്QL വിശകലനം (push, pull request, ആഴ്ചക്കുറ്റി), പുൾ റിക്ക്വസ്റ്റ്‌സ് സമയത്ത് ഡിപ്പെൻഡൻസി റിവ്യൂ. ഉദാഹരണ അടിസ്ഥാന മാർക്കുണ്ട്:

## 8. പ്രകടന സുധാരങ്ങൾ

### 8.1 കോഡ്-നിലമുറ മെച്ചപ്പെടുത്തലുകൾ

1. **ആസിങ്ക്/അവെയ്റ്റ് പാറ്റേണുകൾ**
   - ബാച്ച് പ്രോസസിംഗിനായി ആസിങ്ക് ഉദാഹരണങ്ങൾ ചേർക്കുക
   - സമാന്തര API കോളുകൾ പ്രദർശിപ്പിക്കുക

2. **കാഷിംഗ് സ്ട്രാറ്റജികൾ**
   - എംബെഡിംഗ് കാഷിംഗ് ഉദാഹരണങ്ങൾ ചേർക്കുക
   - പ്രതികരണം കാഷിംഗ് പാറ്റേണുകൾ പ്രദർശിപ്പിക്കുക

3. **ടോക്കൺ മെച്ചപ്പെടുത്തൽ**
   - ടിക്ടോക്കൺ ഉപയോഗ ഉദാഹരണങ്ങൾ ചേർക്കുക
   - പ്രാംപ്റ്റ് കമ്പ്രഷൻ സാങ്കേതികതകൾ പ്രദർശിപ്പിക്കുക

### 8.2 ചെലവ് ലാഭത്തിന്റെ ഉദാഹരണങ്ങൾ

ഉദാഹരണങ്ങൾ ചേർക്കുക:
- സങ്കീർണ്ണതയുടെ അടിസ്ഥാനത്തിൽ മോഡൽ തിരഞ്ഞെടുക്കൽ
- ടോക്കൺ കാര്യക്ഷമതയ്ക്കുള്ള പ്രാദേശിക എഞ്ചിനീയറിംഗ്
- ബൾക്ക് ഓപ്പറേഷനുകൾക്കായുള്ള ബാച്ച് പ്രോസസിംഗ്

## 9. ആക്സസ് സുലഭതയും അന്താരാഷ്ട്രവത്കരണവും

### 9.1 നിലവിലെ വിവർത്തന സ്ഥിതി

എല്ലാ വിവർത്തനങ്ങളും **പൂർണ്ണമാണ്** [Azure Co-op Translator](https://github.com/Azure/co-op-translator?WT.mc_id=academic-105485-koreyst) ഉപയോഗിച്ച് യാന്ത്രികമായി നിർമിക്കപ്പെട്ടവയാണിത്, ഏത് ഇംഗ്ലീഷ് സ്രോതസുമായും പാഠ്യപദ്ധതിയുടെ 50+ ഭാഷാ പതിപ്പുകൾ സമന്വയിപ്പിക്കുന്നു. വിവർത്തനം ചെയ്ത ഉള്ളടക്കം `translations/` കീഴിൽ നിലനിൽക്കുന്നു, പ്രാദേശിക ചിത്രങ്ങൾ `translated_images/` കീഴിലാണ്; ലഭ്യമായ എല്ലാ ഭാഷകളുടെ സമഗ്ര പട്ടിക സംഭരിച്ചിട്ടുള്ള റീപ്പോടോപ്പിൽ മുകളിൽ പ്രസിദ്ധീകരിച്ചിരിക്കുന്നു.

| ഘടകം | സ്ഥിതി |
|--------|--------|
| വിവർത്തന പരിധി | പൂർണ്ണം — 50+ ഭാഷകൾ, എല്ലാ പാഠങ്ങൾ |
| വിവർത്തന രീതി | യാന്ത്രികം [Azure Co-op Translator](https://github.com/Azure/co-op-translator?WT.mc_id=academic-105485-koreyst) മുഖേന |
| ഇംഗ്ലീഷ് സ്രോതസുമായി സമന്വയിപ്പിച്ചിരിക്കുന്നു | അതെ — സ്വയം പുനഃസൃഷ്ടി പ്രവർത്തിക്കുന്നു |

### 9.2 ആക്സസ് സുലഭത മെച്ചപ്പെടുത്തലുകൾ

1. എല്ലാ ചിത്രങ്ങൾക്കും എൽട്ട് ടെക്സ്റ്റ് ചേർക്കുക
2. കോഡ് സാമ്പിളുകളുടെ ശരിയായ സിങ്ക്‌റ്റാക്സ് ഹൈലൈറ്റിംഗ് ഉറപ്പാക്കുക
3. എല്ലാ വീഡിയോ ഉള്ളടക്കത്തിനും വീഡിയോ ട്രാൻസ്‌ക്രിപ്റ്റുകൾ ചേർക്കുക
4. കളർ കോൺട്രാസ്റ്റ് WCAG മാർഗനിർദ്ദേശങ്ങൾ പാലിക്കുന്നത് ഉറപ്പാക്കുക
