# ജനറേറ്റീവ് എഐയ്ക്കുള്ള ചെറുതായ ഭാഷാ മോഡലുകളുടെ പരിചയം പുതിയവർക്കായി
ജനറേറ്റീവ് എഐ എന്നാൽ പുതിയ ഉള്ളടക്കം സൃഷ്ടിക്കാൻ ശേഷിയുള്ള സിസ്റ്റങ്ങൾ രൂപപ്പെടുത്തുന്നതിൽ കേന്ദ്രീകരിക്കുന്ന കൃത്രിമ ബുദ്ധിമുട്ടിന്റെ അതിശയകരമായ ഒരു ശാഖയാണ്. ഈ ഉള്ളടക്കം എഴുത്ത്, ചിത്രങ്ങൾ, സംഗീതം, അല്ലെങ്കിൽ മുഴുവൻ വെർച്വൽ പരിസരങ്ങൾ എന്നിവ വരെയുള്ളവ ആകാം. ഭാഷാ മോഡലുകളുടെ മേഖലയിൽ ജനറേറ്റീവ് എഐയുടെ ഏറ്റവും ആവേശകരമായ പ്രയോഗങ്ങളിലൊന്നാണ്.

## ചെറുതായ ഭാഷാ മോഡലുകൾ എന്തൊക്കെയാണ്?

ചെറുതായ ഭാഷാ മോഡൽ (SLM) ഒരു വലിയ ഭാഷാ മോഡലിന്റെ (LLM) ചുരുക്കപ്പെട്ട രൂപമാണ്, LLM-കളുടെ നിരവധി ശില്പശാസ്ത്ര സിദ്ധാന്തങ്ങളും സാങ്കേതിക വിദ്യകളും പ്രയോജനപ്പെടുത്തി, എന്നാൽ ഗണിതശാസ്ത്രപരമായ ഭാരം വളരെ കുറവ് വഹിക്കുന്നത്.

SLM-കൾ മനുഷ്യനുപോലെ ഉള്ള എഴുത്ത് സൃഷ്ടിക്കാൻ രൂപകൽപ്പന ചെയ്തിട്ടുള്ള ഭാഷാ മോഡലുകളുടെ ഒരു ഉപസമൂഹമാണ്. GPT-4 പോലുള്ള വലുത് മോഡലുകളോട് താരതമ്യപ്പെടുത്തുമ്പോൾ, SLM-കൾ ചെറുതും കാര്യക്ഷമവുമാണ്, അതിനാൽ കണക്കുകൂട്ടൽ വിഭവങ്ങൾ നിഷേധിക്കുന്ന സാദ്ധ്യതയുള്ള പ്രയോഗങ്ങൾക്ക് അനുയോജ്യമാണ്. ചെറുതായും ഇവയ്ക്ക് വ്യത്യസ്ത ഉപയോഗങ്ങൾ നടത്താൻ കഴിയും. സാധാരണയായി, SLM-കൾ LLM-കളെ സംക്ഷിപ്തമാക്കുകയോ അഥവാ സാരാംശമെടുക്കുകയോ ചെയ്യുന്നതിനാൽ രൂപീകരിക്കപ്പെടുന്നു, ഇത് യഥാർത്ഥ മോഡലിന്റെ പ്രവർത്തനശേഷിയും ഭാഷാശേഷിയും സൂക്ഷിക്കാൻ ലക്ഷ്യമിടുന്നു. മോഡലിന്റെ വലിപ്പം കുറവാകുന്നതിലൂടെ മൊത്തം സങ്കീർണത കുറയുന്നു, അതിനാൽ കൂടിയ മെമ്മറി ഉപയോഗവും കണക്കുകൂട്ടൽ ആവശ്യങ്ങളും കുറച്ച് SLM-കൾ കാര്യക്ഷമമാക്കുന്നു. ഈ തികഞ്ഞ മെച്ചപ്പെടുത്തലുകൾക്കിടയിലും, SLM-കൾ പ്രകൃതിഭാഷ പ്രോസസ്സിങ്ങിൽ (NLP) വ്യത്യസ്ത പ്രവർത്തനങ്ങൾ എളുപ്പത്തിൽ നിർവഹിക്കാനും കഴിയും:

- എഴുത്ത് സൃഷ്ടിക്കൽ: സുചിതവും സാന്ദർഭികവുമായ വാചകങ്ങളോ പാരഗ്രാഫുകളോ സൃഷ്ടിക്കുന്നു.
- എഴുത്ത് പൂർണ്ണത: നൽകിയ പ്രംപ്റ്റിന്റെ അടിസ്ഥാനത്തിൽ വാചകങ്ങൾ പ്രവചിച്ച് പൂർത്തിയാക്കുന്നു.
- ഭാഷಾಂತರണം: ഒരു ഭാഷയിൽ നിന്നുള്ള എഴുത്ത് മറ്റൊരു ഭാഷയിലേക്ക് മാറ്റുന്നു.
- സംക്ഷേപണം: നീണ്ട എഴുത്തുകൾ ചെറുതായി, സുലഭമായി ആക്കുന്നു.

വലുതായ മോഡലുകളുമായുള്ള പ്രകടനത്തിന്റെയും ആഴത്തിലുള്ള മനസ്സിലാക്കലിന്റെയും ചില എതിരാളിത്തങ്ങളോടെയാണ്.

## ചെറുതായ ഭാഷാ മോഡലുകൾ എങ്ങനെ പ്രവർത്തിക്കുന്നു?
SLM-കൾ വിശാലമായ എഴുത്ത് ഡാറ്റാ ഉപരിതലങ്ങളിൽ പരിശീലിപ്പിക്കപ്പെടുന്നു. പരിശീലന സമയത്ത്, ഭാഷയുടെ ഘടനകളും മാതൃകകളും നിർവചിക്കുന്നു, ഇതുവഴി വ്യാകരണപരവും സാന്ദർഭികവുമായ എഴുത്തുകൾ സൃഷ്ടിക്കാനാകും. പരിശീലന പ്രക്രിയയിൽ ഉൾപ്പെടുന്നത്:

- ഡാറ്റാ സമാഹരണം: വ്യത്യസ്ത ഉറവിടങ്ങളിൽ നിന്ന് വലിയ എഴുത്ത് ഡാറ്റാസെറ്റുകൾ ശേഖരിക്കുക.
- മുൻ-സംസ്‌കരണം: ഡാറ്റാ ശുദ്ധീകരിച്ച് പരിശീലനത്തിന് അനുയോജ്യമായ വിധത്തിൽ ക്രമീകരിക്കൽ.
- പരിശീലനം: യന്ത്രപഠന ആൾഗോരിത്‌മുകൾ ഉപയോഗിച്ച് മോഡലിന് എഴുത്ത് മനസ്സിലാക്കാനും സൃഷ്ടിക്കാനും പരിശീലിപ്പിക്കൽ.
- ഫൈൻ-ട്യൂണിംഗ്: പ്രത്യേക പ്രവർത്തനങ്ങളിൽ മോഡലിന്റെ പ്രകടനം മെച്ചപ്പെടുത്തുക.

SLM-കളുടെ വികസനം മൊബൈൽ ഉപകരണങ്ങൾ പോലുള്ള വിഭവപരിമിത സന്ധികളിലും എജ് കംപ്യൂട്ടിംഗ് പ്ലാറ്റ്ഫോമുകളിലും ഉപയോഗിക്കാൻ കഴിയുന്ന മോഡലുകളുടെ ഉയർന്ന ആവശ്യം പാലിക്കുന്നു, യഥാർത്ഥ LLM-കൾ ആവശ്യപ്പെടുന്ന വമ്പൻ വിഭവങ്ങൾ കാരണം അവയെ പ്രയോജനപ്പെടുത്താൻ പ്രയാസമുണ്ടാകുന്നത്. കാര്യക്ഷമതയിൽ മാത്രം ശ്രദ്ധിച്ചതുകൊണ്ടാണ് SLM-കൾ പ്രകടനവും ആക്സസിബിലിറ്റിയും സജീവമായി തുല്യപ്പെടുത്തുന്നത്, ഇത് വിവിധ മേഖലകളിൽ വ്യാപകമായ പ്രയോഗം പ്രാപ്തമാക്കുന്നു.

## വലിയ ഭാഷാ മോഡലുകളുടെയും ചെറുതായ ഭാഷാ മോഡലുകളുടെയും വ്യത്യാസങ്ങൾ

LLM-കളും SLM-കളും പ്രോഭാബിലിസ്റ്റിക് മെഷീൻ ലേണിങിന്റെ അടിസ്ഥാന സിദ്ധാന്തങ്ങളെ ആശ്രയിച്ചാണ് നിർമ്മിച്ചിരിക്കുന്നത്, അവർ ആർക്കിടെക്ചറൽ ഡിസൈൻ, പരിശീലന രീതി, ഡാറ്റാ ജനറേഷൻ പ്രക്രിയകൾ, മോഡൽ മൂല്യനിർണയം എന്നിവയിൽ സമാന സമീപനങ്ങൾ അനുസരിക്കുന്നു. എന്നാൽ, ഇവ രണ്ടും തമ്മിൽ ചില പ്രധാന ഘടകങ്ങളിൽ വ്യത്യാസം ഉണ്ട്.

## ചെറുതായ ഭാഷാ മോഡലുകളുടെ പ്രയോഗങ്ങൾ

SLM-കൾ സാധാരണയായി താഴെക്കാണുന്ന മേഖലകളിൽ വ്യാപകമായി പ്രയോഗിക്കപ്പെടുന്നു:

- ചാറ്റ്‌ബോട്ടുകൾ: ഉപഭോക്തൃ പിന്തുണ നൽകാനും ഉപയോക്താക്കളുമായി സംവദിക്കാനും.
- ഉള്ളടക്കം സൃഷ്ടിക്കൽ: എഴുത്തുകാരെ ആശയങ്ങൾ സൃഷ്ടിക്കാനോ മുഴുവൻ ലേഖനങ്ങൾ ഒരുക്കാനോ സഹായിക്കുന്നു.
- വിദ്യാഭ്യാസം: വിദ്യാര്‍ഥികൾക്ക് എഴുതൽ അസൈൻമെന്റുകളിലും പുതിയ ഭാഷകൾ പഠിക്കാനും സഹായിക്കുന്നു.
- ആക്‌സസിബിലിറ്റി: പകർച്ചവ്യാധികൾ ഉള്ള വ്യക്തികൾക്കായി ടെക്സ്റ്റ്-ടു-സ്പീച്ച് പോലുള്ള ഉപകരണങ്ങൾ സൃഷ്ടിക്കുന്നു.

**ആകൃതി**
  
LLM-കളും SLM-കളും തമ്മിൽ പ്രധാന വ്യത്യാസം മോഡലുകളുടെ വലിപ്പത്തിലും ആണ്. ChatGPT (GPT-4) പോലുള്ള LLM-കൾ ഏകദേശം 1.76 ട്രില്യൺ പാരാമീറ്ററുകൾ ഉൾക്കൊള്ളുന്നുവെന്നാണ് കണക്കുകൂട്ടുന്നത്, എന്നാൽ മിസ്ട്രൽ 7ബി പോലുള്ള ഓപ്പൺ സോഴ്‌സ് SLM-കൾ ഏകദേശം 7 ബില്യൺ പാരാമീറ്ററുകളുള്ളവയാണ്. ഈ വ്യത്യാസം പ്രധാനമായും മോഡൽ ആർക്കിടെക്ചറിലും പരിശീലന പ്രക്രിയകളിലും നിന്നാണ് ഉണ്ടാവുന്നത്. ഉദാഹരണത്തിന്, ChatGPT ഒരു എൻകോഡർ-ഡീകോഡർ ഘടനയിലുള്ള സെൽഫ്-അറ്റൻഷൻ മെക്കാനിസം പ്രയോജനപ്പെടുത്തുന്നു, എന്നാൽ മിസ്ട്രൽ 7ബി സ്ലൈഡിംഗ് വിൻഡോ അറ്റൻഷൻ ഉപയോഗിച്ച് ഡീകോഡർ-ഒൺലി മോഡലിൽ അധിക കാര്യക്ഷമമായ പരിശീലനം സാധ്യമാക്കുന്നു. ഈ ആർക്കിടെക്ചറലി വ്യത്യാസം മോഡലുകളുടെ സങ്കീർണ്ണതക്കും പ്രകടനത്തിനും ഗൗരവമുള്ള സ്വാധീനം ചെലുത്തുന്നു.

SLM-കളെ സാധാരണയായി പ്രത്യേക മേഖലകളിൽ മികച്ച പ്രകടനം നൽകാൻ വേണ്ടി മെച്ചപ്പെടുത്തിയിട്ടുണ്ട്, അതിനാൽ അവ വളരെ പ്രത്യേകമായി വികസിച്ചിട്ടുണ്ട്, പക്ഷെ വിവിധ മേഖലകളിൽ വ്യാപകമായ സാന്ദർഭിക ബോധം നൽകുന്നതിൽ കുറവായിരിക്കാം. മറിച്ച്, LLM-കൾ മനുഷ്യനുപോലെ ബുദ്ധിമുട്ട് കൂടുതൽ പൊതുവായ രീതിയിൽ അനുകരിക്കാൻ ഉദ്ദേശിച്ചിരിക്കുന്നു. വിപുലവും വൈവിധ്യമാർന്ന ഡാറ്റാസെറ്റുകളിലായിരുന്നു LLM-കളുടെ പരിശീലനം, അതിനാൽ അവ വിവിധ മേഖലയിലായി മികച്ച പ്രകടനം ക്ഷേമിക്കുന്നു, കൂടുതൽ വൈവിധ്യം കൊണ്ടും അനുകൂലിയുമായും. അതിനാൽ, LLM-കൾ പ്രകൃതിഭാഷ പ്രോസസ്സിങ്ങിലും പ്രോഗ്രാമിങ്ങിലും കൂടുതൽ വ്യാപകമായ ഒരു നിരത്തിലും ഉപയോഗിക്കാവുന്നതാണ്.

**കണക്കുകൂട്ടൽ**

LLM-കൾ പരിശീലിച്ചു പ്രവർത്തിപ്പിക്കൽ വലിയ കണക്കുകൂട്ടൽ വിഭവങ്ങൾ ആവശ്യപ്പെടുന്ന പ്രക്രിയയാണ്, സാധാരണയായി വലിയ GPU ക്ലസ്റ്ററുകൾ ഉൾപ്പെടുന്നു. ഉദാഹരണത്തിന്, ChatGPT പോലുള്ള മോഡലിനെ ആദ്യം മുതൽ പരിശീലിപ്പിക്കാൻ ആയിരക്കണക്കിന് GPU-കൾ ദീർഘകാലം വേണം. മറിച്ച്, SLM-കൾ വളരെ ചെറിയ പാരാമീറ്റർ സംഖ്യകൾ ഉള്ളതിനാൽ കണക്കുകൂട്ടൽ വിഭവങ്ങളിൽ കൂടുതൽ ഉപയോഗപ്രദമാണ്. മിസ്ട്രൽ 7ബി പോലുള്ള മോഡലുകൾ ശരാശരി GPU ശേഷിയുള്ള ലൊക്കൽ മെഷീനുകളിൽ പരിശീലിപ്പിക്കാനും പ്രവർത്തിക്കാനും കഴിയും, എന്നിരുന്നാലും പരിശീലനം പലഘട്ട GPU-കളിൽ നിരവധി മണിക്കൂറുകൾ ആവശ്യപ്പെടും.

**പരാജയം (ബയസ്)**

മറുപടി രണ്ട് വലിയ ലാംഗ്വേജ് മോഡലുകളിലുണ്ടാകുന്ന പ്രശ്നങ്ങളുടെ പ്രധാന ഭാഗമാണ് പരിശീലന ഡാറ്റയുടെ സ്വഭാവം. ഈ മോഡലുകൾ സാധാരണമായി ഇന്റർനെറ്റിൽ നിന്ന് ലഭിക്കുന്ന കച്ച ഡാറ്റകളിൽ ആശ്രയിക്കുന്നുവെങ്കിൽ, ചില സമൂഹങ്ങൾ കുറവായി പ്രതിനിധാനം ചെയ്യപ്പെടുകയോ തെറ്റായ അടയാളപ്പെടുത്തലുകൾ ഉണ്ടാകുകയോ ഭാഷാശൈലിയോ ഭൂമിശാസ്ത്ര വ്യത്യാസങ്ങളോ വ്യാകരണചട്ടങ്ങളോ മൂലം ഉണ്ടാകുന്ന വിവേചനപരമായ നവീകരണങ്ങൾ പ്രതിഫലിപ്പിക്കുകയോ ചെയ്യും. കൂടാതെ, LLM ആർക്കിടെക്ചറുകളുടെ സങ്കീർണ്ണത ഈ വിവേചനങ്ങളെ അനുഭവവേദനയാകാതെ ചൂഷണം ചെയ്യാം. മറുവശത്ത്, SLM-കൾ ശൃംഖലിതമായ, പ്രാദേശിക, സ്‌പെസിഫിക് ഡാറ്റാസെറ്റുകളിൽ പരിശീലിപ്പിക്കപ്പെടുന്നതിനാൽ ഈ ബയസുകൾക്ക് കുറവാണ് വഴങ്ങുന്നത്, എന്നാൽ അവസാനം ബയസ് ഇല്ലാതാക്കാൻ കഴിയാത്തതാണ്.

**ഇൻഫറൻസ്**

SLM-കളുടെ കുറവായ വലിപ്പം ഇൻഫറൻസ് വേഗത്തിൽ വലിയ മുൻഗണന നൽകുന്നു, ഇത് ലൊക്കൽ ഹാർഡ്‌വെയർ ഉപയോഗിച്ച് കൃത്യമായി കാര്യക്ഷമമായി ഫലം സൃഷ്ടിക്കാൻ അനുവദിക്കുന്നു, വലിയ സമാന്തര പ്രോസസ്സിംഗ് ആവശ്യപ്പെടാതെ. മറിച്ച്, LLM-കൾ വലിപ്പത്തിലും സങ്കീർണതയിലും കാരണം ഉയർന്ന സമാന്തര കണക്കുകൂട്ടൽ വിഭവങ്ങൾ ആവശ്യപ്പെടുന്നു, നിർബന്ധമായ സമയപരിധിയിൽ ഇൻഫറൻസ് പൂർത്തിയാക്കാൻ. സംഘട്ടിത ഉപയോക്താക്കളുടെ എണ്ണം കൂടുന്നതും LLM-കളുടെ പ്രതികരണ സമയം കുറക്കുന്നു, പ്രത്യേകിച്ച് വ്യാപകമായി വിനിയോഗിക്കുമ്പോൾ.

സംക്ഷേപത്തിൽ, LLM-കളും SLM-കളും മെഷീൻ ലേണിങ്ങിന്റെ അടിസ്ഥാനത്തിൽ സമാനമാണ്, എന്നാൽ മോഡലിന്റെ വലിപ്പം, വിഭവ ആവശ്യകത, സാന്ദർഭിക ബോധം, ബയസ് വലയ്ക്കുന്ന സാധ്യത, ഇൻഫറൻസ് വേഗം എന്നിവയിൽ ഗൗരവമുള്ള വ്യത്യാസങ്ങൾ ഉണ്ട്. ഈ വ്യത്യാസങ്ങൾ ഇവരൊക്കെയും വ്യത്യസ്ത പ്രയോഗങ്ങൾക്ക് അനുയോജ്യരാകുന്നതായി സൂചിപ്പിക്കുന്നു, LLM-കൾ കൂടുതലായി വൈവിധ്യമാർന്ന പ്രവർത്തനക്ഷമത നൽകുന്നു, എന്നാൽ ബഹുഭാര വിഭവം ആവശ്യപ്പെടുന്നു, SLM-കൾ കുറവായ കണക്കുകൂട്ടൽ ആവശ്യങ്ങളോടെ കൂടുതൽ പ്രത്യേകമാക്കപ്പെട്ട കാര്യക്ഷമത നൽകുന്നു.

***കുറിപ്പ്: ഈ പാഠത്തിൽ, മൈക്രോസോഫ്റ്റ് ഫൈ-3 / 3.5 പ്രയോഗം ഉദാഹരണമായി SLM പരിചയപ്പെടുത്തും.***

പ്രധാനമായും എഴുത്ത് സൃഷ്ടിക്കൽ, ചാറ്റ് പൂർത്തിയാക്കൽ, ഉള്ളടക്കം വിവരാവകാശം എന്നിവയ്‌ക്കു വേണ്ടി.

14B പാരാമീറ്ററുള്ള ഫൈ-3-മീഡിയം ഈ പ്രവണത തുടർച്ചാ തുടരുന്നു, Gemini 1.0 പ്രോയെ മറികടക്കുന്നു.

3.8B പാരാമീറ്ററുകളുള്ള ഫൈ-3.5-മിനി തുല്യവലുപ്പമുള്ള ഭാഷാ മോഡലുകളെ മറികടക്കുകയും, ഇരട്ടിയുള്ളവരോടും തുല്യനിലയിൽ പ്രവർത്തിക്കുകയും ചെയ്യുന്നു.

### ഫൈ-3 / 3.5 കാഴ്ച

ഫൈ-3/3.5 ഇൻസ്ട്രക്റ്റ് മോഡൽ ഫൈയെ മനസ്സിലാക്കാനുള്ള കഴിവായി കാണാം, കാഴ്ച ഫൈയ്ക്ക് ലോകം മനസ്സിലാക്കാൻ കണ്ണുകൾ നൽകുന്നതാണ്.


**ഫൈ-3-വിഷൻ**

വെറും 4.2B പാരാമീറ്ററുകളുള്ള ഫൈ-3-വിഷൻ ഈ പ്രവണത തുടരുന്നു കൂടാതെ Claude-3 Haiku, Gemini 1.0 പ്രോ போன்ற വലുത് മോഡലുകളെ സാധാരണ കാഴ്ചയുടെ യുക്തി, ഓസി‌ആർ, പട്ടികകളും ചിത്രങ്ങളും മനസ്സിലാക്കൽ പ്രവർത്തനങ്ങളിൽ മറികടക്കുന്നു.


**ഫൈ-3.5-വിഷൻ**

ഫൈ-3.5-വിഷൻ ഫൈ-3-വിഷന്റെ ഒരു അപ്ഗ്രേഡാണ്, ഇത് ബഹുദൃശ്യങ്ങൾക്കായുള്ള പിന്തുണ കൂടി നൽകുന്നു. ഇത് കാഴ്‌ച്ചയിൽ മെച്ചപ്പെടുത്തലായി കാണാം: ചിത്രങ്ങൾ മാത്രമല്ല, വീഡിയോകളും കാണാനും കഴിയും.

ഫൈ-3.5-വിഷൻ Claude-3.5 Sonnet, Gemini 1.5 Flash പോലുള്ള വലുത് മോഡലുകളെ ഓസി‌ആർ, പട്ടിക, പട്ടിക മനസ്സിലാക്കൽ പ്രവർത്തനങ്ങളിൽ മറികടക്കുന്നു കൂടാതെ സാധാരണ കാഴ്ചയുടെ ബോധം പരീക്ഷണ പ്രവർത്തനങ്ങളിൽ തുല്യനിലയിൽ ആണ്. ബഹുദൃശ്യ ഇൻപുട്ടുകളുടെ യുക്തിഗണന നടത്തുന്നു.


### ഫൈ-3.5-MoE

***മിക്സ്ചർ ഓഫ് എക്സ്പർട്ട്സ് (MoE)*** മോഡലുകൾ കുറവായ കണക്കുകൂട്ടലിൽ മുൻപരിശീലനം നൽകാനുവുന്നു, അതിനാൽ നിങ്ങൾക്ക് ഒരു ഡെൻസ് മോഡലിന്റെ അതേ കണക്കുകൂട്ടൽ ബജറ്റിൽ മോഡലിന്റെ വലിപ്പം അല്ലെങ്കിൽ ഡാറ്റാസെറ്റ് വലുപ്പം നാനുപാതികമായി കൂട്ടാൻ കഴിയും. പ്രത്യേകിച്ച്, MoE മോഡൽ സംബന്ധിച്ച ഡെൻസ് കൂട്ടുകാർക്ക് തുല്യമായ ഗുണമേന്മ ലഭിക്കാൻ കൂടുതലായി വേഗത്തിൽ സാധിക്കും.

ഫൈ-3.5-MoE 16x3.8B വിദഗ്ധ മാഡ്യൂളുകളുടെ കോമ്പോസിഷനാണ്. വെറും 6.6B സജീവ പാരാമീറ്ററുകളുള്ള ഫൈ-3.5-MoE വളരെ വലിയ മോഡലുകളുമായുള്ള യുക്തിചിന്തന, ഭാഷാ ബോധം, ഗണിതമേയുള്ള പ്രവർത്തനങ്ങളിൽ തുല്യവുമായി പ്രവർത്തിക്കുന്നു.

വ്യത്യസ്ത അവസ്ഥകളിൽ അടിസ്ഥാനമാക്കി ഫൈ-3/3.5 കുടുംബ മോഡലുകൾ ഉപയോഗിക്കാം. LLM-കളെ അപേക്ഷിച്ച്, ഫൈ-3/3.5-മിനി അല്ലെങ്കിൽ ഫൈ-3/3.5-വിഷൻ എജ് ഉപകരണങ്ങളിൽ വിനിയോഗിക്കാം.

2. **ഫ്രെയിംവർക്ക് ഇന്റർഓപ്പറബിൾറ്റി**: PyTorch, TensorFlow, JAX എന്നിവ ഉൾപ്പെടെയുള്ള നിരവധി ഡീപ്പ് ലേണിംഗ് ഫ്രെയിംവർക്കുകളെ ലൈബ്രറി പിന്തുണയ്ക്കുന്നു. ഇത് ഒരു ഫ്രെയിംവർക്കിൽ മോഡൽ ട്രെയിൻ ചെയ്ത് മറ്റൊന്നിൽ ഉപയോഗിക്കാനാകും.

3. **മൾട്ടിമോഡൽ കഴിവുകൾ**: NLPയ്ക്ക് പുറമെ, Hugging Face Transformers കമ്പ്യൂട്ടർ വിഷൻ (ഉദാ. ഇമേജ് ക്ലാസിഫിക്കേഷൻ, ഒബ്ജക്റ്റ് ഡിറ്റക്ഷൻ) ഓഡിയോ പ്രോസസ്സിംഗ് (ഉദാ. സ്പീച്ച് റെക്കഗ്നിഷൻ, ഓഡിയോ ക്ലാസിഫിക്കേഷൻ) എന്നിവയും പിന്തുണയ്ക്കുന്നു.

4. **ഉപയോഗ സൗകര്യം**: മോഡലുകൾ ഡൗൺലോഡ് ചെയ്ത് ഫൈൻ-ട്യൂൺ ചെയ്യാൻ എളുപ്പമുള്ള APIകളും ടൂളുകളും ലൈബ്രറി നൽകുന്നു, തുടക്കക്കാരോടും വിദഗ്ധരോടും ഒരുപോലെ സൗകര്യപ്രദമാണ്.

**GenAI-ക്കായുള്ള ONNX റൺടൈം**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) ക്രോസ്-പ്ലാറ്റ്ഫോം ഇൻഫറൻസ്, ട്രീനിംഗ് മെഷിൻ ലേണിംഗ് ആക്സിലറേറ്ററാണ്. GenAI-ക്കായുള്ള ONNX Runtime സവിശേഷമായി വ്യത്യസ്ത പ്ലാറ്റ്ഫോമുകളിൽ ജനറേറ്റീവ് AI മോഡലുകൾ ഫലപ്രദമായി പ്രവർത്തിപ്പിക്കാൻ സഹായിക്കുന്നു.

## ONNX Runtime എന്താണ്?
ONNX Runtime തുറന്ന സോഴ്‌സ് പ്രോജക്ടാണ്, ഉയർന്ന പ്രകടന ഇൻഫറൻസ് അനുവദിക്കുന്നു. ഇത് Open Neural Network Exchange (ONNX) ഫോർമാറ്റിലുള്ള മോഡലുകൾ പിന്തുണയ്ക്കുന്നു, 머신learning മോഡലുകൾ പ്രകടിപ്പിക്കുന്ന ഒരു മാനദണ്ഡമാണ്. പൈടോർച്ച്, ടെൻസർഫ്ലോ/കേരാസ് പോലുള്ള ഡീപ് ലേണിംഗ് ഫ്രെയിംവർകുകളും സ്കിക്കിറ്റ്-ലേൺ, ലൈറ്റ്‌ജിബിഎം, XGBoost പോലുള്ള പരമ്പരാഗത മെഷിൻ ലേണിംഗ് ലൈബ്രററികളും പിന്തുണയ്ക്കുന്നു. വിവിധ ഹാർഡ്‌വെയർ, ഡ്രൈവറുകൾ, ഓപ്പറേറ്റിങ്ങ് സിസ്റ്റങ്ങൾക്കൊപ്പം പ്രവർത്തിച്ച് മികച്ച പ്രകടനം ഉറപ്പാക്കാൻ ഹാർഡ്‌വെയർ ആക്സിലറേറ്ററുകളും ഗ്രാഫ് ഓപ്റ്റിമൈസേഷനുകളും ഉപയോഗിക്കുന്നു.

## Generative AI എന്താണ്?
Generative AI പുതിയ ഉള്ളടക്കം സൃഷ്ടിക്കാൻ പ്രാപ്തമായ AI സിസ്റ്റങ്ങളെയാണ് സൂചിപ്പിക്കുന്നത്, ഉദാഹരണത്തിന് ടെക്സ്റ്റ്, ചിത്രങ്ങൾ, സംഗീതം എന്നിവ, അവ പഠിച്ച ഡാറ്റ അടിസ്ഥാനമാക്കി. ഉദാഹരണങ്ങൾ: GPT-3 പോലുള്ള ഭാഷാ മോഡലുകളും Stable Diffusion പോലുള്ള ഇമേജ് ജനറേഷൻ മോഡലുകളും. ONNX Runtime for GenAI ലൈബ്രറി ONNX മോഡലുകൾക്കുള്ള ജനറേറ്റീവ് AI ലൂപ് നൽകുന്നു - ONNX Runtime ഇൻഫറൻസ്, ലോജിറ്റ്‌സ് പ്രോസസ്സിങ്, സെർച്ച്-ആൻഡ്-സാമ്പിൾ, KV കാഷ് മാനേജ്മെന്റ് എന്നിവ ഉൾപ്പെടുന്നു.

### ഡെമോ: ONNX Runtime GenAI ഉപയോഗിച്ച് Phi-3.5-Vision വിളിക്കൽ


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
