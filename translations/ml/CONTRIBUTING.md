## ടൈപ്പോകൾ, പ്രശ്നങ്ങൾ, ബഗുകൾ, സംഭാവനകൾ

Generative AI for Beginners റിപോസിറ്ററിയിൽ മാറ്റങ്ങൾ സമർപ്പിക്കുമ്പോൾ, ദയവായി ഈ നിർദ്ദേശങ്ങൾ പാലിക്കുക.

* മാറ്റങ്ങൾ ചെയ്യുന്നതിന് മുമ്പ് എപ്പോഴും റിപോസിറ്ററി നിങ്ങളുടെ അക്കൗണ്ടിലേക്ക് ഫോർക്ക് ചെയ്യുക
* ഒരേ പുൾ റിക്വസ്റ്റിൽ പല മാറ്റങ്ങളും ചേർക്കരുത്. ഉദാഹരണത്തിന്, ബഗ് ഫിക്സ്, ഡോക്യുമെന്റേഷൻ അപ്ഡേറ്റുകൾ വേർതിരിച്ച PR കളായി സമർപ്പിക്കുക
* നിങ്ങളുടെ പുൾ റിക്വസ്റ്റിൽ മർജ് കോൺഫ്ലിക്ടുകൾ കാണിച്ചാൽ, നിങ്ങളുടെ ലോക്കൽ മെയിൻ റിപോസിറ്ററി പ്രധാന റിപോസിറ്ററിയുടെ മിററായിരിക്കണമെന്ന് ഉറപ്പാക്കുക
* വിവർത്തനം സമർപ്പിക്കുമ്പോൾ, എല്ലാ വിവർത്തന ഫയലുകൾക്കും ഒരേ PR സൃഷ്ടിക്കുക, ഭാഗിക വിവർത്തനങ്ങൾ സ്വീകരിക്കില്ല
* ടൈപ്പോ അല്ലെങ്കിൽ ഡോക്യുമെന്റേഷൻ ഫിക്സ് സമർപ്പിക്കുമ്പോൾ, അനുയോജ്യമായിടത്ത് മാറ്റങ്ങൾ ഒരേ PR യിൽ ചേർക്കാം

## എഴുതുന്നതിനുള്ള പൊതുവായ മാർഗ്ഗനിർദ്ദേശങ്ങൾ

- നിങ്ങളുടെ എല്ലാ URLs സ്ക്വയർ ബ്രാക്കറ്റുകളിൽ ഉൾപ്പെടുത്തി, പിന്നെ പാരൻഥസിസിൽ ഇടുക, ഇടത്തും പുറത്തും അധിക സ്പേസുകൾ ഇല്ലാതെ `[](../..)` എന്ന രീതിയിൽ.
- റിപോസിറ്ററിയിലെ മറ്റ് ഫയലുകൾക്കും ഫോൾഡറുകൾക്കും ഉള്ള സാപേക്ഷ ലിങ്കുകൾ `./` (നിലവിലെ ഡയറക്ടറിയിലുള്ള ഫയൽ/ഫോൾഡർ) അല്ലെങ്കിൽ `../` (മുകളിൽ ഉള്ള ഡയറക്ടറിയിലുള്ള ഫയൽ/ഫോൾഡർ) എന്നതോടെ തുടങ്ങണം.
- സാപേക്ഷ ലിങ്കുകൾക്ക് ട്രാക്കിംഗ് ഐഡി (ഉദാ: `?` അല്ലെങ്കിൽ `&` പിന്നെ `wt.mc_id=` അല്ലെങ്കിൽ `WT.mc_id=`) അവസാനത്തിൽ ഉണ്ടായിരിക്കണം.
- _github.com, microsoft.com, visualstudio.com, aka.ms, azure.com_ എന്നീ ഡൊമെയ്ൻസിൽ നിന്നുള്ള URLs ട്രാക്കിംഗ് ഐഡി ഉണ്ടായിരിക്കണം.
- നിങ്ങളുടെ ലിങ്കുകളിൽ രാജ്യത്തെ പ്രത്യേക ലൊക്കൽ (ഉദാ: `/en-us/` അല്ലെങ്കിൽ `/en/`) ഉണ്ടായിരിക്കരുത്.
- എല്ലാ ചിത്രങ്ങളും `./images` ഫോൾഡറിൽ സൂക്ഷിക്കണം.
- ചിത്രങ്ങളുടെ പേരുകൾ ഇംഗ്ലീഷ് അക്ഷരങ്ങൾ, സംഖ്യകൾ, ഡാഷുകൾ ഉപയോഗിച്ച് വിവരണാത്മകമായിരിക്കണം.

## GitHub വർക്ക്‌ഫ്ലോകൾ

നിങ്ങൾ ഒരു പുൾ റിക്വസ്റ്റ് സമർപ്പിക്കുമ്പോൾ, മുൻപുള്ള നിയമങ്ങൾ പരിശോധിക്കാൻ നാല് വ്യത്യസ്ത വർക്ക്‌ഫ്ലോകൾ പ്രവർത്തിക്കും.
വർക്ക്‌ഫ്ലോ പരിശോധനകൾ കടക്കാൻ ഇവിടെ നൽകിയ നിർദ്ദേശങ്ങൾ പാലിക്കുക.

### Check Broken Relative Paths

ഈ വർക്ക്‌ഫ്ലോ നിങ്ങളുടെ ഫയലുകളിലെ സാപേക്ഷ പാതകൾ പ്രവർത്തിക്കുന്നുണ്ടോ എന്ന് ഉറപ്പാക്കുന്നു.
ഈ റിപോസിറ്ററി GitHub പേജുകളിൽ ഡിപ്ലോയുചെയ്യപ്പെട്ടതിനാൽ, ലിങ്കുകൾ തെറ്റായ സ്ഥലത്തേക്ക് നയിക്കാതിരിക്കാൻ ശ്രദ്ധിക്കണം.

ലിങ്കുകൾ ശരിയായി പ്രവർത്തിക്കുന്നുണ്ടോ എന്ന് ഉറപ്പാക്കാൻ VS കോഡ് ഉപയോഗിക്കുക.

ഉദാഹരണത്തിന്, നിങ്ങളുടെ ഫയലിലെ ഏതെങ്കിലും ലിങ്കിൽ ഹോവർ ചെയ്യുമ്പോൾ **ctrl + click** അമർത്തി ലിങ്ക് പിന്തുടരാൻ പ്രോംപ്റ്റ് കാണിക്കും.

![VS code follow links screenshot](../../translated_images/ml/vscode-follow-link.85520ab6a1237adc.png)

ലിങ്ക് ക്ലിക്ക് ചെയ്ത് ലോക്കലായി പ്രവർത്തിക്കാത്ത പക്ഷം, വർക്ക്‌ഫ്ലോ ട്രിഗർ ചെയ്ത് GitHub-ലും പ്രവർത്തിക്കില്ല.

ഈ പ്രശ്നം പരിഹരിക്കാൻ, VS കോഡിന്റെ സഹായത്തോടെ ലിങ്ക് ടൈപ്പ് ചെയ്യാൻ ശ്രമിക്കുക.

`./` അല്ലെങ്കിൽ `../` ടൈപ്പ് ചെയ്യുമ്പോൾ, VS കോഡ് നിങ്ങൾ ടൈപ്പ് ചെയ്തതനുസരിച്ച് ലഭ്യമായ ഓപ്ഷനുകളിൽ നിന്ന് തിരഞ്ഞെടുക്കാൻ പ്രോംപ്റ്റ് നൽകും.

![VS code select relative path screenshot](../../translated_images/ml/vscode-select-relative-path.3804eb73c3a9e5f2.png)

ആവശ്യമായ ഫയൽ അല്ലെങ്കിൽ ഫോൾഡർ ക്ലിക്ക് ചെയ്ത് പാത പിന്തുടരുക, പാത തകരാറില്ലെന്ന് ഉറപ്പാക്കാം.

ശരിയായ സാപേക്ഷ പാത ചേർത്ത ശേഷം, സേവ് ചെയ്ത് മാറ്റങ്ങൾ പുഷ് ചെയ്താൽ വർക്ക്‌ഫ്ലോ വീണ്ടും പ്രവർത്തിച്ച് പരിശോധിക്കും.
പരീക്ഷണം കടന്നാൽ, നിങ്ങൾക്ക് മുന്നോട്ട് പോകാം.

### Check Paths Have Tracking

ഈ വർക്ക്‌ഫ്ലോ സാപേക്ഷ പാതകളിൽ ട്രാക്കിംഗ് ഉണ്ടോ എന്ന് ഉറപ്പാക്കുന്നു.
GitHub പേജുകളിൽ ഡിപ്ലോയുചെയ്യപ്പെട്ടതിനാൽ, ഫയലുകൾക്കും ഫോൾഡറുകൾക്കുമിടയിലെ ചലനം ട്രാക്ക് ചെയ്യണം.

സാപേക്ഷ പാതകളിൽ `?wt.mc_id=` എന്ന ടെക്സ്റ്റ് അവസാനത്തിൽ ഉണ്ടോ എന്ന് പരിശോധിക്കുക.
ഉണ്ടെങ്കിൽ, ഈ പരിശോധന കടന്നുപോകും.

ഇല്ലെങ്കിൽ, താഴെ കാണുന്ന പിശക് വരാം.

![GitHub check paths missing tracking comment screenshot](../../translated_images/ml/github-check-paths-missing-tracking-comment.880d4afe03e898ff.png)

ഈ പ്രശ്നം പരിഹരിക്കാൻ, വർക്ക്‌ഫ്ലോ ഹൈലൈറ്റ് ചെയ്ത ഫയൽ പാത തുറന്ന് ട്രാക്കിംഗ് ഐഡി സാപേക്ഷ പാതകളുടെ അവസാനം ചേർക്കുക.

ട്രാക്കിംഗ് ഐഡി ചേർത്ത ശേഷം, സേവ് ചെയ്ത് മാറ്റങ്ങൾ പുഷ് ചെയ്താൽ വർക്ക്‌ഫ്ലോ വീണ്ടും പ്രവർത്തിച്ച് പരിശോധിക്കും.
പരീക്ഷണം കടന്നാൽ, നിങ്ങൾക്ക് മുന്നോട്ട് പോകാം.

### Check URLs Have Tracking

ഈ വർക്ക്‌ഫ്ലോ വെബ് URL കളിൽ ട്രാക്കിംഗ് ഉണ്ടോ എന്ന് ഉറപ്പാക്കുന്നു.
ഈ റിപോസിറ്ററി എല്ലാവർക്കും ലഭ്യമാണ്, അതിനാൽ ട്രാഫിക് എവിടുനിന്ന് വരുന്നു എന്ന് ട്രാക്ക് ചെയ്യണം.

URL കളിൽ `?wt.mc_id=` എന്ന ടെക്സ്റ്റ് അവസാനത്തിൽ ഉണ്ടോ എന്ന് പരിശോധിക്കുക.
