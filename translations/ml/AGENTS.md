# AGENTS.md

## Project Overview

ഈ റിപോസിറ്ററിയിൽ ജനറേറ്റീവ് AI അടിസ്ഥാനങ്ങൾക്കും അപ്ലിക്കേഷൻ ഡെവലപ്മെന്റിനും 21 പാഠങ്ങളുള്ള സമഗ്രമായ പാഠ്യപദ്ധതി ഉൾക്കൊള്ളുന്നു. കോഴ്സ് ആരംഭക്കാർക്കായി രൂപകൽപ്പന ചെയ്തതാണ്, അടിസ്ഥാന ആശയങ്ങളിൽ നിന്ന് പ്രൊഡക്ഷൻ റെഡി അപ്ലിക്കേഷനുകൾ നിർമ്മിക്കുന്നതുവരെ എല്ലാം ഉൾക്കൊള്ളുന്നു.

**പ്രധാന സാങ്കേതികവിദ്യകൾ:**
- Python 3.9+ ലൈബ്രറികളോടെ: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript Node.js-ഉം ലൈബ്രറികളുമായി: `@azure/openai`, `@azure-rest/ai-inference`, `openai`
- Azure OpenAI സർവീസ്, OpenAI API, GitHub മോഡലുകൾ
- ഇന്ററാക്ടീവ് പഠനത്തിനായി Jupyter നോട്ട്ബുക്കുകൾ
- സ്ഥിരതയുള്ള ഡെവലപ്മെന്റ് പരിസ്ഥിതിക്ക് Dev Containers

**റിപോസിറ്ററി ഘടന:**
- 21 നമ്പർ ചെയ്ത പാഠ ഡയറക്ടറികൾ (00-21) READMEകൾ, കോഡ് ഉദാഹരണങ്ങൾ, അസൈൻമെന്റുകൾ ഉൾക്കൊള്ളുന്നു
- Python, TypeScript, ചിലപ്പോൾ .NET ഉദാഹരണങ്ങൾ ഉൾപ്പെടുന്ന പല നടപ്പാക്കലുകൾ
- 40+ ഭാഷാ പതിപ്പുകളുള്ള വിവർത്തന ഡയറക്ടറി
- `.env` ഫയൽ വഴി കേന്ദ്രകൃതമായ കോൺഫിഗറേഷൻ (ടെംപ്ലേറ്റായി `.env.copy` ഉപയോഗിക്കുക)

## Setup Commands

### Initial Repository Setup

```bash
# റിപ്പോസിറ്ററി ക്ലോൺ ചെയ്യുക
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# പരിസ്ഥിതി ടെംപ്ലേറ്റ് പകർത്തുക
cp .env.copy .env
# നിങ്ങളുടെ API കീകളും എൻഡ്‌പോയിന്റുകളും ഉപയോഗിച്ച് .env എഡിറ്റ് ചെയ്യുക
```

### Python Environment Setup

```bash
# വെർച്വൽ എൻവയോൺമെന്റ് സൃഷ്ടിക്കുക
python3 -m venv venv

# വെർച്വൽ എൻവയോൺമെന്റ് സജീവമാക്കുക
# മാക്‌ഒഎസ്/ലിനക്സ്-ൽ:
source venv/bin/activate
# വിൻഡോസ്-ൽ:
venv\Scripts\activate

# ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്യുക
pip install -r requirements.txt
```

### Node.js/TypeScript Setup

```bash
# റൂട്ട്-നിലവാര ഡിപ്പൻഡൻസികൾ ഇൻസ്റ്റാൾ ചെയ്യുക (ഡോക്യുമെന്റേഷൻ ടൂളിംഗിനായി)
npm install

# വ്യക്തിഗത പാഠം ടൈപ്പ്‌സ്‌ക്രിപ്റ്റ് ഉദാഹരണങ്ങൾക്കായി, പ്രത്യേക പാഠത്തിലേക്ക് നാവിഗേറ്റ് ചെയ്യുക:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Dev Container Setup (Recommended)

റിപോസിറ്ററിയിൽ GitHub Codespaces അല്ലെങ്കിൽ VS Code Dev Containers-ക്കായി `.devcontainer` കോൺഫിഗറേഷൻ ഉൾക്കൊള്ളുന്നു:

1. GitHub Codespaces-ൽ അല്ലെങ്കിൽ Dev Containers എക്സ്റ്റൻഷൻ ഉള്ള VS Code-ൽ റിപോസിറ്ററി തുറക്കുക
2. Dev Container സ്വയം:
   - `requirements.txt`-ൽ നിന്നുള്ള Python ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്യും
   - പോസ്റ്റ്-ക്രിയേറ്റ് സ്ക്രിപ്റ്റ് (`.devcontainer/post-create.sh`) പ്രവർത്തിപ്പിക്കും
   - Jupyter കർണൽ സജ്ജമാക്കും

## Development Workflow

### Environment Variables

API ആക്‌സസ് ആവശ്യമായ എല്ലാ പാഠങ്ങൾക്കും `.env`-ൽ നിർവചിച്ച പരിസ്ഥിതി വ്യത്യാസങ്ങൾ ഉപയോഗിക്കുന്നു:

- `OPENAI_API_KEY` - OpenAI API-ക്കായി
- `AZURE_OPENAI_API_KEY` - Azure OpenAI സർവീസിനായി
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI എൻഡ്പോയിന്റ് URL
- `AZURE_OPENAI_DEPLOYMENT` - ചാറ്റ് പൂർത്തീകരണ മോഡൽ ഡിപ്ലോയ്മെന്റ് പേര്
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - എംബെഡ്ഡിംഗ്സ് മോഡൽ ഡിപ്ലോയ്മെന്റ് പേര്
- `AZURE_OPENAI_API_VERSION` - API പതിപ്പ് (ഡീഫോൾട്ട്: `2024-02-01`)
- `HUGGING_FACE_API_KEY` - Hugging Face മോഡലുകൾക്കായി
- `GITHUB_TOKEN` - GitHub മോഡലുകൾക്കായി

## Code Style Guidelines

### Python

- പരിസ്ഥിതി വ്യത്യാസങ്ങൾ കൈകാര്യം ചെയ്യാൻ `python-dotenv` ഉപയോഗിക്കുക
- API ഇടപെടലുകൾക്കായി `openai` ലൈബ്രറി ഇറക്കുമതി ചെയ്യുക
- ലിന്റിംഗിനായി `pylint` ഉപയോഗിക്കുക (സാധാരണ ചില ഉദാഹരണങ്ങളിൽ `# pylint: disable=all` ഉപയോഗിച്ചിരിക്കുന്നു)
- PEP 8 നാമകരണം പാലിക്കുക
- API ക്രെഡൻഷ്യലുകൾ `.env` ഫയലിൽ സൂക്ഷിക്കുക, കോഡിൽ ഒരിക്കലും സൂക്ഷിക്കരുത്

### TypeScript

- പരിസ്ഥിതി വ്യത്യാസങ്ങൾക്ക് `dotenv` പാക്കേജ് ഉപയോഗിക്കുക
- ഓരോ ആപ്പിനും `tsconfig.json`-ൽ TypeScript കോൺഫിഗറേഷൻ
- Azure സർവീസുകൾക്കായി `@azure/openai` അല്ലെങ്കിൽ `@azure-rest/ai-inference` ഉപയോഗിക്കുക
- ഓട്ടോ-റീലോഡിനായി `nodemon` ഉപയോഗിക്കുക
- പ്രവർത്തിപ്പിക്കുന്നതിന് മുമ്പ് നിർമ്മിക്കുക: `npm run build` തുടർന്ന് `npm start`

### General Conventions

- കോഡ് ഉദാഹരണങ്ങൾ ലളിതവും വിദ്യാഭ്യാസപരവുമാക്കുക
- പ്രധാന ആശയങ്ങൾ വിശദീകരിക്കുന്ന കമന്റുകൾ ഉൾപ്പെടുത്തുക
- ഓരോ പാഠത്തിന്റെയും കോഡ് സ്വയംപര്യാപ്തവും പ്രവർത്തനക്ഷമവുമാകണം
- സ്ഥിരതയുള്ള നാമകരണം പാലിക്കുക: Azure OpenAI-ക്കായി `aoai-` പ്രിഫിക്സ്, OpenAI API-ക്കായി `oai-`, GitHub മോഡലുകൾക്കായി `githubmodels-`

## Documentation Guidelines

### Markdown Style

- എല്ലാ URL-കളും `[text](../../url)` ഫോർമാറ്റിൽ, അധിക സ്പേസുകൾ ഇല്ലാതെ വേണം
- സാപേക്ഷ ലിങ്കുകൾ `./` അല്ലെങ്കിൽ `../`-ൽ തുടങ്ങണം
- Microsoft ഡൊമെയ്ൻ ലിങ്കുകൾക്ക് ട്രാക്കിംഗ് ഐഡി ഉൾപ്പെടുത്തണം: `?WT.mc_id=academic-105485-koreyst`
- URL-കളിൽ രാജ്യ-നിർദ്ദിഷ്ട ലോക്കലുകൾ ഒഴിവാക്കുക (ഉദാ: `/en-us/`)
- ചിത്രങ്ങൾ `./images` ഫോൾഡറിൽ വിവരണാത്മക നാമങ്ങളോടെ സൂക്ഷിക്കുക
- ഫയൽ നാമങ്ങളിൽ ഇംഗ്ലീഷ് അക്ഷരങ്ങൾ, സംഖ്യകൾ, ഡാഷുകൾ മാത്രം ഉപയോഗിക്കുക

### Translation Support

- റിപോസിറ്ററി 40+ ഭാഷകൾക്ക് GitHub Actions വഴി പിന്തുണ നൽകുന്നു
- വിവർത്തനങ്ങൾ `translations/` ഡയറക്ടറിയിൽ സൂക്ഷിക്കുന്നു
- ഭാഗിക വിവർത്തനങ്ങൾ സമർപ്പിക്കരുത്
- മെഷീൻ വിവർത്തനങ്ങൾ അംഗീകരിക്കപ്പെടുന്നില്ല
- വിവർത്തിച്ച ചിത്രങ്ങൾ `translated_images/` ഡയറക്ടറിയിൽ സൂക്ഷിക്കുന്നു

## Testing and Validation

### Pre-submission Checks

ഈ റിപോസിറ്ററി GitHub Actions ഉപയോഗിച്ച് സാധുത പരിശോധിക്കുന്നു. PR സമർപ്പിക്കുന്നതിന് മുമ്പ്:

1. **Markdown ലിങ്കുകൾ പരിശോധിക്കുക**:
   ```bash
   # validate-markdown.yml പ്രവൃത്തി പ്രവാഹം പരിശോധിക്കുന്നു:
   # - തകരാറുള്ള സാപേക്ഷ പാതകൾ
   # - പാതകളിൽ ട്രാക്കിംഗ് ഐഡികൾ കാണാനില്ല
   # - URLs-ൽ ട്രാക്കിംഗ് ഐഡികൾ കാണാനില്ല
   # - രാജ്യ ഭാഷാപരമായ URLs
   # - തകരാറുള്ള ബാഹ്യ URLs
   ```

2. **മാനുവൽ ടെസ്റ്റിംഗ്**:
   - Python ഉദാഹരണങ്ങൾ ടെസ്റ്റ് ചെയ്യുക: venv സജീവമാക്കി സ്ക്രിപ്റ്റുകൾ പ്രവർത്തിപ്പിക്കുക
   - TypeScript ഉദാഹരണങ്ങൾ ടെസ്റ്റ് ചെയ്യുക: `npm install`, `npm run build`, `npm start`
   - പരിസ്ഥിതി വ്യത്യാസങ്ങൾ ശരിയായി കോൺഫിഗർ ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക
   - API കീകൾ കോഡ് ഉദാഹരണങ്ങളുമായി പ്രവർത്തിക്കുന്നുണ്ടെന്ന് പരിശോധിക്കുക

3. **കോഡ് ഉദാഹരണങ്ങൾ**:
   - എല്ലാ കോഡും പിശകുകൾ കൂടാതെ പ്രവർത്തിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക
   - ആവശ്യമായപ്പോൾ Azure OpenAI, OpenAI API എന്നിവയുമായി ടെസ്റ്റ് ചെയ്യുക
   - GitHub മോഡലുകൾ പിന്തുണയ്ക്കുന്നിടത്ത് ഉദാഹരണങ്ങൾ പ്രവർത്തിക്കുന്നുണ്ടെന്ന് പരിശോധിക്കുക

### No Automated Tests

ഇത് ട്യൂട്ടോറിയലുകൾക്കും ഉദാഹരണങ്ങൾക്കും കേന്ദ്രീകരിച്ച വിദ്യാഭ്യാസ റിപോസിറ്ററിയാണ്. യുണിറ്റ് ടെസ്റ്റുകളും ഇന്റഗ്രേഷൻ ടെസ്റ്റുകളും ഇല്ല. സാധുത പരിശോധിക്കൽ പ്രധാനമായും:
- കോഡ് ഉദാഹരണങ്ങളുടെ മാനുവൽ ടെസ്റ്റിംഗ്
- Markdown സാധുതക്കായി GitHub Actions
- വിദ്യാഭ്യാസ ഉള്ളടക്കത്തിന് കമ്മ്യൂണിറ്റി അവലോകനം

## Pull Request Guidelines

### Before Submitting

1. Python, TypeScript-ൽ കോഡ് മാറ്റങ്ങൾ ടെസ്റ്റ് ചെയ്യുക (ആവശ്യമായപ്പോൾ)
2. Markdown സാധുത പരിശോധന നടത്തുക (PR-ൽ സ്വയം പ്രവർത്തിക്കും)
3. എല്ലാ Microsoft URL-കളിലും ട്രാക്കിംഗ് ഐഡികൾ ഉണ്ടെന്ന് ഉറപ്പാക്കുക
4. സാപേക്ഷ ലിങ്കുകൾ സാധുവാണെന്ന് പരിശോധിക്കുക
5. ചിത്രങ്ങൾ ശരിയായി റഫറൻസ് ചെയ്തിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക

### PR Title Format

- വിവരണാത്മക തലക്കെട്ടുകൾ ഉപയോഗിക്കുക: `[Lesson 06] Fix Python example typo` അല്ലെങ്കിൽ `Update README for lesson 08`
- ആവശ്യമായപ്പോൾ ഇഷ്യൂ നമ്പറുകൾ റഫറൻസ് ചെയ്യുക: `Fixes #123`

### PR Description

- എന്ത് മാറ്റം വരുത്തി, എന്തുകൊണ്ടെന്ന് വിശദീകരിക്കുക
- ബന്ധപ്പെട്ട ഇഷ്യൂകളിലേക്ക് ലിങ്ക് നൽകുക
- കോഡ് മാറ്റങ്ങൾക്കായി ടെസ്റ്റ് ചെയ്ത ഉദാഹരണങ്ങൾ വ്യക്തമാക്കുക
- വിവർത്തന PR-കൾക്കായി മുഴുവൻ ഫയലുകളും ഉൾപ്പെടുത്തുക

### Contribution Requirements

- Microsoft CLA-യിൽ ഒപ്പിടുക (ആദ്യ PR-ൽ സ്വയം പ്രവർത്തിക്കും)
- മാറ്റങ്ങൾ വരുത്തുന്നതിന് മുമ്പ് റിപോസിറ്ററി ഫോർക്ക് ചെയ്യുക
- ഓരോ ലൊജിക്കൽ മാറ്റത്തിനും ഒരു PR (അസംബന്ധിത പരിഹാരങ്ങൾ ഒന്നിച്ച് ചേർക്കരുത്)
- PR-കൾ ശ്രദ്ധ കേന്ദ്രീകരിച്ചും ചെറിയതും ആക്കുക

## Common Workflows

### Adding a New Code Example

1. അനുയോജ്യമായ പാഠ ഡയറക്ടറിയിലേക്ക് പോകുക
2. `python/` അല്ലെങ്കിൽ `typescript/` സബ് ഡയറക്ടറിയിൽ ഉദാഹരണം സൃഷ്ടിക്കുക
3. നാമകരണം പാലിക്കുക: `{provider}-{example-name}.{py|ts|js}`
4. യഥാർത്ഥ API ക്രെഡൻഷ്യലുകൾ ഉപയോഗിച്ച് ടെസ്റ്റ് ചെയ്യുക
5. പുതിയ പരിസ്ഥിതി വ്യത്യാസങ്ങൾ പാഠ README-യിൽ രേഖപ്പെടുത്തുക

### Updating Documentation

1. പാഠ ഡയറക്ടറിയിലെ README.md എഡിറ്റ് ചെയ്യുക
2. Markdown മാർഗനിർദ്ദേശങ്ങൾ പാലിക്കുക (ട്രാക്കിംഗ് ഐഡികൾ, സാപേക്ഷ ലിങ്കുകൾ)
3. വിവർത്തനങ്ങൾ GitHub Actions കൈകാര്യം ചെയ്യുന്നു (മാനുവലായി തിരുത്തരുത്)
4. എല്ലാ ലിങ്കുകളും സാധുവാണെന്ന് ടെസ്റ്റ് ചെയ്യുക

### Working with Dev Containers

1. റിപോസിറ്ററിയിൽ `.devcontainer/devcontainer.json` ഉണ്ട്
2. പോസ്റ്റ്-ക്രിയേറ്റ് സ്ക്രിപ്റ്റ് Python ആശ്രിതങ്ങൾ സ്വയം ഇൻസ്റ്റാൾ ചെയ്യും
3. Python, Jupyter എക്സ്റ്റൻഷനുകൾ മുൻകൂട്ടി കോൺഫിഗർ ചെയ്തിട്ടുണ്ട്
4. പരിസ്ഥിതി `mcr.microsoft.com/devcontainers/universal:2.11.2` അടിസ്ഥാനമാക്കിയുള്ളതാണ്

## Troubleshooting

### Common Issues

**Python Import Errors**:
- വെർച്വൽ എൻവയോൺമെന്റ് സജീവമാക്കിയിട്ടുണ്ടെന്ന് ഉറപ്പാക്കുക
- `pip install -r requirements.txt` പ്രവർത്തിപ്പിക്കുക
- Python പതിപ്പ് 3.9+ ആണെന്ന് പരിശോധിക്കുക

**TypeScript Build Errors**:
- പ്രത്യേക ആപ്പ് ഡയറക്ടറിയിൽ `npm install` പ്രവർത്തിപ്പിക്കുക
- Node.js പതിപ്പ് അനുയോജ്യമായതാണെന്ന് ഉറപ്പാക്കുക
- `node_modules` ക്ലിയർ ചെയ്ത് ആവശ്യമെങ്കിൽ വീണ്ടും ഇൻസ്റ്റാൾ ചെയ്യുക

**API Authentication Errors**:
- `.env` ഫയൽ നിലവിലുണ്ടെന്നും ശരിയായ മൂല്യങ്ങൾ ഉള്ളതുമാണെന്ന് പരിശോധിക്കുക
- API കീകൾ സാധുവും കാലഹരണപ്പെട്ടതുമല്ലെന്നും ഉറപ്പാക്കുക
- നിങ്ങളുടെ പ്രദേശത്തിനുള്ള എൻഡ്പോയിന്റ് URL-കൾ ശരിയാണെന്ന് ഉറപ്പാക്കുക

**Missing Environment Variables**:
- `.env.copy`-നെ `.env` ആയി പകർത്തുക
- നിങ്ങൾ പ്രവർത്തിക്കുന്ന പാഠത്തിനുള്ള എല്ലാ ആവശ്യമായ മൂല്യങ്ങളും പൂരിപ്പിക്കുക
- `.env` അപ്ഡേറ്റ് ചെയ്ത ശേഷം ആപ്പ് റീസ്റ്റാർട്ട് ചെയ്യുക
