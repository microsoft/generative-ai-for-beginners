### പ്രാരംഭ റിപോസിറ്ററി സെറ്റപ്പ്

```bash
# റെപ്പോസിറ്ററി ക്ലോണ്‍ ചെയ്യുക
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# പരിസര മാതൃക പകര്‍ക്കുക
cp .env.copy .env
# നിങ്ങളുടെ API കീകളും എൻഡ്‌പോയിന്റുകളും ഉപയോഗിച്ച് .env ഫയൽ സജ്ജമാക്കുക
```

### Python പരിസ്ഥിതി സെറ്റപ്പ്

```bash
# വർച്ച്വൽ എൻവയോൺമെന്റ് സൃഷ്ടിക്കുക
python3 -m venv venv

# വർച്ച്വൽ എൻവയോൺമെന്റ് സജീവമാക്കുക

source venv/bin/activate
# Windows-ൽ:
venv\Scripts\activate

# ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്യുക

```

### Node.js/TypeScript സെറ്റപ്പ്

```bash

# വ്യക്തിഗത പാഠം TypeScript ഉദാഹരണങ്ങൾക്ക്, പ്രത്യേക പാഠത്തിലേക്ക് നീങ്ങുക:
cd 06-text-generation-apps/typescript/recipe-app

### ഡെവ് കണ്ടെയ്‌നർ സെറ്റപ്പ് (ശിപാർശ ചെയ്‌തിരിക്കുന്നു)

GitHub കോഡ്സ്പേസിൽ അല്ലെങ്കിൽ VS Code ഡെവ് കണ്ടെയ്‌നർസ് എക്സ്റ്റൻഷൻ ഉപയോഗിച്ച് പ്രവർത്തിക്കാനുള്ള `.devcontainer` കോൺഫിഗറേഷൻ ഉൾക്കൊള്ളിച്ചിരിക്കുന്നു:

   - പോസ്റ്റ്-ക്രിയേറ്റ് സ്‌ക്രിപ്റ്റ് പ്രവർത്തിപ്പിക്കുക (`.devcontainer/post-create.sh`)

### പരിസ്ഥിതി വ്യത്യസ്തങ്ങൾ

API ആക്‌സസ് ആവശ്യമായ എല്ലാ പാഠങ്ങളും `.env`-ൽ നിർവചിച്ച പരിസ്ഥിതി വ്യത്യസ്ഥാനങ്ങൾ ഉപയോഗിക്കുന്നു:

- `OPENAI_API_KEY` - OpenAI API-ക്കായി
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry-ൽ Azure OpenAI (https://ai.azure.com ൽ Azure OpenAI സേവനം Microsoft Foundryയുടെ ഭാഗമാണ്)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI എന്റ്പോയിന്റ് URL (Foundry റിസോഴ്സ് എന്റ്പോയിന്റ്)
- `AZURE_OPENAI_DEPLOYMENT` - ചാറ്റ് പൂർത്തീകരണ മോഡൽ ഡീപ്ലോയ്മെന്റ് നാമം
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - എംബെഡ്ഡിംഗ്സ് മോഡൽ ഡീപ്ലോയ്മെന്റ് നാമം
- `AZURE_OPENAI_API_VERSION` - API പതിപ്പ് (ഡിഫോൾട്ട്: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face മോഡലുകൾക്കായി
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry മോഡലുകളുടെ എന്റ്പോയിന്റ് (മൾട്ടി-പ്രൊവൈഡർ മോഡൽ കാറ്റലോഗ്)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry മോഡലുകളുടെ API കീ (വിരമിക്കുന്ന `GITHUB_TOKEN`-ന് പകരം)

## കോഡ് സ്റ്റൈൽ മാർഗ്ഗരേഖകൾ

### Python

- പരിസ്ഥിതി വ്യത്യസ്ഥാനങ്ങൾ കൈകാര്യം ചെയ്യാൻ `python-dotenv` ഉപയോഗിക്കുക
- API ഇടപെടലുകൾക്കായി `openai` ലൈബ്രറി ഇറക്കുമതി ചെയ്യുക
- ലിന്റിങ്ങിനായി `pylint` ഉപയോഗിക്കുക (സുഖവായി ചില ഉദാഹരണങ്ങളിൽ `# pylint: disable=all` ഉപയോഗിച്ചിട്ടുണ്ട്)
- PEP 8 നാമനിർദ്ദേശങ്ങൾ പാലിക്കുക
- API ക്രെഡൻഷ്യലുകൾ `.env` ഫയലിൽ സൂക്ഷിക്കുക, കോഡിൽ ഒരിക്കലും സൂക്ഷിക്കരുത്

### TypeScript

- പരിസ്ഥിതി വ്യത്യസ്ഥാനങ്ങൾക്കായി `dotenv` പാക്കേജ് ഉപയോഗിക്കുക
- ഓരോ ആപ്പിനും `tsconfig.json` ലെ TypeScript കോൺഫിഗറേഷൻ
- Azure OpenAI-ക്കായി `openai` പാക്കേജ് ഉപയോഗിക്കുക (`/openai/v1/` എന്റ്പോയിന്റിൽ ക്ലയന്റ് പോയിൻറ് ചെയ്തു `client.responses.create` വിളിക്കുക); Microsoft Foundry മോഡലുകൾക്കായി `@azure-rest/ai-inference` ഉപയോഗിക്കുക
- ഡെവലപ്പ്മെന്റിനായി ഓട്ടോ-റീലോഡിംഗ് സജ്ജമാക്കുന്നത് `nodemon` ഉപയോഗിക്കുക
- പ്രവർത്തനത്തിന് മുമ്പ് ബിൽഡ് ചെയ്യുക: `npm run build` തുടർന്ന് `npm start`

   # - പൊട്ടിയ സംബദ്ധമായ പാതകൾ
   # - പാതകളിലെ ട്രാക്കിംഗ് ഐഡികൾ കാണുന്നില്ല
   # - URLs-ൽ ട്രാക്കിംഗ് ഐഡികൾ കാണുന്നില്ല
   # - രാജ്യത്തിന്റെ ലോക്കൽ ഉള്ള URLs
   # - പൊട്ടിയ ബാഹ്യ URLs
   ```

2. **മാനുവൽ ടെസ്റ്റിംഗ്:**
   - Python ഉദാഹരണങ്ങൾ ടെസ്റ്റ് ചെയ്യുക: venv സജീവമാക്കി സ്ക്രിപ്റ്റുകൾ പ്രവർത്തിപ്പിക്കുക
   - TypeScript ഉദാഹരണങ്ങൾ ടെസ്റ്റ് ചെയ്യുക: `npm install`, `npm run build`, `npm start`
   - പരിസ്ഥിതി വ്യത്യസ്ഥാനങ്ങൾ ശരിയായതാണെന്ന് പരിശോധിക്കുക
   - API കീകൾ കോഡ് ഉദാഹരണങ്ങൾക്കൊപ്പം ശരിയായി പ്രവർത്തിക്കുന്നുണ്ടോ എന്ന് പരിശോധിക്കുക

3. **കോട് ഉദാഹരണങ്ങൾ:**
   - എല്ലാ കോഡും പിശക്കില്ലാതെ പ്രവർത്തിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക
   - ബാധകമായപ്പോൾ Azure OpenAIയും OpenAI APIയും രണ്ടും ഉപയോഗിച്ച് പരിശോധന നടത്തുക
   - സാധിക്കുന്പോൾ Microsoft Foundry മോഡലുകളുമായി പ്രവർത്തിക്കുന്നുണ്ടെന്ന് ഉറപ്പാക്കുക

### സ്വയം നിർവഹിക്കുന്ന ടെസ്റ്റുകൾ ഇല്ല

ഇത് പഠനത്തിനും ഉദാഹരണങ്ങൾക്ക് കേന്ദ്രീകൃതമായ ഒരു റിപോസിറ്ററിയാണ്. ഒന്നും യൂണിറ്റ് ടെസ്റ്റുകളും ഇന്റഗ്രേഷൻ ടെസ്റ്റുകളും പ്രവർത്തിപ്പിക്കാൻ ഇല്ല. സാധുതാ മുൾപടയായി ആണ്:

- GitHub Actions മാർക്ക്‌ഡൗൺ സാധുത പരിശോധന
- വിദ്യാഭ്യാസ ഉള്ളടക്കത്തിന് കമ്മ്യൂണിറ്റി റിവ്യൂ

### PR ശീർഷക ഫോർമാറ്റ്

### PR വിവരണം

- എന്താണ് മാറ്റങ്ങൾ ചെയ്തതും നിർവാചിച്ചും വിശദീകരിക്കുക
- ബന്ധപ്പെട്ട തകരാറുകളിലേക്കുള്ള ലിങ്കുകൾ നൽകുക
- കോഡ് മാറ്റങ്ങൾക്ക് പരിശോധിച്ച ഉദാഹരണങ്ങൾ വ്യക്തമാക്കുക
- വിവർത്തന PR-കൾക്കായി, പൂർണ വിവർത്തനത്തിനുള്ള എല്ലാ ഫയലുകളും ഉൾപ്പെടുത്തുക

## പൊതു വർക്ക്‌ഫ്ലോകൾ

### പുതിയ കോഡ് ഉദാഹരണം ചേർക്കൽ

4. യഥാർത്ഥ API ക്രെഡൻഷ്യലുകൾ ഉപയോഗിച്ച് ടെസ്റ്റ് ചെയ്യുക

2. മാർക്ക്‌ഡൗൺ മാർഗ്ഗ നിർദ്ദേശങ്ങൾ പാലിക്കുക (ട്രാക്കിംഗ് ID-കൾ, ബന്ധപെട്ട ലിങ്കുകൾ)
3. വിവർത്തനങ്ങൾ GitHub Actions കൈകാര്യം ചെയ്യുന്നു (മാനുവൽ എഡിറ്റ് ചെയ്യരുത്)
4. എല്ലാ ലിങ്കുകളും സാധുവാണെന്ന് ടെസ്റ്റ് ചെയ്യുക

### ഡെവ് കണ്ടെയ്‌നറുകളുമായി ജോലി ചെയ്യൽ

1. റിപ്പോസിറ്ററിയിൽ `.devcontainer/devcontainer.json` ഉണ്ട്
2. പോസ്റ്റ്-ക്രിയേറ്റ് സ്‌ക്രിപ്റ്റ് Python ആശ്രിതങ്ങൾ സ്വയം ഇൻസ്റ്റോൾ ചെയ്യുന്നു
3. Python, Jupyter എക്സ്റ്റൻഷനുകൾ മുൻകൂർ കോൺഫിഗർ ചെയ്തിട്ടുണ്ട്
4. പരിസ്ഥിതി `mcr.microsoft.com/devcontainers/universal:2.11.2` അടിസ്ഥാനമാക്കി ആണ്

## പ്രശ്നപരിഹാരം

### പൊതുവായ പ്രശ്നങ്ങൾ

**Python ഇറക്കുമതി പിഴവുകൾ**:
- വേർച്ച്വൽ എൻവയറൺമെന്റ് സജീവമാക്കണമെങ്കിൽ ഉറപ്പാക്കുക
- `pip install -r requirements.txt` പ്രവർത്തിപ്പിക്കുക
- Python പതിപ്പ് 3.9+ ആണെന്ന് പരിശോധിക്കുക

**TypeScript ബിൽഡ് പിഴവുകൾ**:
- പ്രത്യേക ആപ്പ് ഡയറക്ടറിയിൽ `npm install` പ്രവർത്തിപ്പിക്കുക
- Node.js പതിപ്പ് അനുയോജ്യമായതാണെന്ന് പരിശോധിക്കുക
- `node_modules` നീക്കിയ ശേഷം ആവശ്യമെങ്കിൽ പുനഃസ്ഥാപിക്കുക

**API പ്രാമാണീകരണ പ്രശ്നങ്ങൾ**:
- `.env` ഫയൽ നിലവിലുണ്ട്, ശരിയായ മൂല്യങ്ങളോടെയാണ് എന്ന് സ്ഥിരീകരിക്കുക
- API കീകൾ സാധുവും കാലഹരണപ്പെടാത്തതും ആണെന്ന് പരിശോധിക്കുക
- നിങ്ങളുടെ പ്രദേശത്തിന് അനുയോജ്യമായ എന്റ്പോയിന്റ് URLs ആണെന്ന് ഉറപ്പാക്കുക

**പരിസ്ഥിതി വ്യത്യസ്ഥങ്ങൾ കാണാനില്ല**:
- `.env.copy`-നെ `.env` ആയി പകർത്തുക
- നിങ്ങൾ പ്രവർത്തിക്കുന്ന പാഠത്തിന് ആവശ്യമായ എല്ലാ മൂല്യങ്ങളും പൂരിപ്പിക്കുക
- `.env` അപ്ഡേറ്റ് ചെയ്ത് അപ്ലിക്കേഷൻ റീസ്റ്റാർട്ട് ചെയ്യുക
