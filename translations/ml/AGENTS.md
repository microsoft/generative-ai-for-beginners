## സജ്ജീകരണ കമാൻഡുകൾ

### തുടക്ക റിപ്പോസിറ്ററി സജ്ജീകരണം

```bash
# റിപ്പോസിറ്ററി ക്ലോൺ ചെയ്യുക
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# എൻവയോൺമെന്റ് ടെംപ്ലേറ്റ് കോപ്പി ചെയ്യുക
cp .env.copy .env
# നിങ്ങളുടെ API കീകൾക്കും എൻഡ്പോയിന്റുകൾക്കും അനുയോജ്യമായി .env എഡിറ്റ് ചെയ്യുക
```

### Python എൻവയോൺമെന്റ് സജ്ജീകരണം

```bash
# വെർച്വൽ എൻവയൺമെന്റ് സൃഷ്‌ടിക്കുക
python3 -m venv venv

# വെർച്വൽ എൻവയൺമെന്റ് സജീവമാക്കുക
# മാക്ഒഎസ്/ലിനക്സ്-ൽ:
source venv/bin/activate
# വിൻഡോസിൽ:
venv\Scripts\activate

# ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്യുക
pip install -r requirements.txt
```

### Node.js/TypeScript സജ്ജീകരണം

```bash
# മാതൃനില സുരക്ഷിതമായ ആശങ്കകൾ ഇൻസ്റ്റാൾ ചെയ്യുക (ഡോക്യുമെന്റേഷൻ ഉപകരണങ്ങൾക്കായി)
npm install

# വ്യക്തിഗത പാഠത്തിന്റെ TypeScript ഉദാഹരണങ്ങൾക്കായി, പ്രത്യേക പാഠത്തിലേക്ക് നാവിഗേറ്റ് ചെയ്യുക:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Dev Container സജ്ജീകരണം (അനുകൂലിക്കുന്നു)

GitHub Codespaces അല്ലെങ്കിൽ VS Code Dev Containers അണുബദ്ധിച്ച് `.devcontainer` കോൺഫിഗറേഷൻ ഉൾക്കൊള്ളുന്നു:

1. GitHub Codespaces അല്ലെങ്കിൽ VS Code తో Dev Containers എക്സ്റ്റൻഷൻ ഉപയോഗിച്ച് റിപ്പോസിറ്ററി തുറക്കുക
2. Dev Container സ്വയം:
   - `requirements.txt` ലെ Python ആശ്രിതങ്ങൾ ഇൻസ്റ്റാൾ ചെയ്യുക
   - പോസ്റ്റ്-സൃഷ്ടി സ്ക്രിപ്റ്റ് (`.devcontainer/post-create.sh`) പ്രവർത്തിപ്പിക്കുക
   - Jupyter കർണൽ ക്രമീകരിക്കുക

## വികസന പ്രവൃത്തി പ്രവാഹം

### പരിസ്ഥിതി വ്യത്യാസങ്ങൾ

API ആക്സസ് ആവശ്യമുള്ള എല്ലാ പാഠങ്ങളും `.env` ഫയലിൽ നിർവചിച്ച പരിസ്ഥിതി വ്യത്യാസങ്ങൾ ഉപയോഗിക്കുന്നു:

- `OPENAI_API_KEY` - OpenAI API ഉം
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry (Azure OpenAI സർവീസ് ഇപ്പോൾ ഇത് ഭാഗമാണ്: https://ai.azure.com) ൽ Azure OpenAI
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI എന്റ്പോയിന്റ് URL (Foundry റിസോർസ് എന്റ്പോയിന്റ്)
- `AZURE_OPENAI_DEPLOYMENT` - ചാറ്റ് പൂർത്തിയാക്കൽ മോഡൽ മോഡൽ ഡിപ്ലോയ്മെന്റ് പേര് (കോഴ്‌സ് ഡിഫോൾട്ട്: `gpt-5-mini`)
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - എംബെഡ്ഡിങ്സ് മോഡൽ ഡിപ്ലോയ്മെന്റ് പേര് (കോഴ്‌സ് ഡിഫോൾട്ട്: `text-embedding-3-small`)
- `AZURE_OPENAI_API_VERSION` - API പതിപ്പ് (ഡിഫോൾട്ട്: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face മോഡലുകൾക്കായി
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry മോഡലുകൾ എന്റ്പോയിന്റ് (മൾട്ടിപ്രൊവൈഡർ മോഡൽ കാറ്റലോഗ്)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry മോഡലുകളുടെ API കീ (പഴയ `GITHUB_TOKEN` മാറ്റി)
- `AZURE_INFERENCE_CHAT_MODEL` - ഒരു നിര്ണയമില്ലാത്ത മോഡൽ (ഉദാ: `Llama-3.3-70B-Instruct`), `temperature` ഉദാഹരണങ്ങളിൽ ഉപയോഗിക്കുന്നു, കാരണം നിര്ണയ മോഡലുകൾ സാമ്പിൾ നിയന്ത്രണങ്ങൾ പിന്തുണയ്ക്കുന്നില്ല

### മോഡൽ പരമ്പരാഗതങ്ങൾ (പ്രധാനമാണ്)

- **ഡിഫോൾട്ട് ചാറ്റ് മോഡൽ `gpt-5-mini` ആണ്** - നിലവിലുള്ള, അമാനമായ **നിയമനിർദ്ദേശം** മോഡൽ. 2026 മുതൽ പഴയ താപനില-സമ്പർക്കമുള്ള "മിനി" മോഡലുകൾ (`gpt-4o-mini`, `gpt-4.1-mini`) സ്തംഭിപ്പിക്കുന്നു, അതുകൊണ്ട് പാഠ്യക്രമം GPT-5 കുടുംബത്തിൽ സ്ഥിരീകരിക്കുന്നു.
- **നിയമനിർദ്ദേശ മോഡലുകൾ `temperature` & `top_p` സ്വീകരിക്കുന്നില്ല**, പകരം `max_output_tokens` (Responses API) / `max_completion_tokens` (ചാറ്റ് പൂർത്തീകരണങ്ങൾ) ഉപയോഗിക്കുന്നു `max_tokens` ന്റെ പകരം. `gpt-5-mini` വിളിക്കുന്ന സാമ്പിളുകളിൽ `temperature`/`top_p`/`max_tokens` ചേർക്കാതിരിക്കുക.
- **`temperature` പ്രദർശിപ്പിക്കാൻ** സാമ്പിളുകൾ Microsoft Foundry മോഡൽ എന്റ്പോയിന്റ് (`AZURE_INFERENCE_CHAT_MODEL`) വഴി **Llama** മോഡൽ (`Llama-3.3-70B-Instruct`) ഉപയോഗിക്കുന്നു. സാമ്പിൾ നിയന്ത്രണക്കണക്കുകൾക്കുപകരം പ്രോംപ്റ്റ് എഞ്ചിനീയറിംഗ് + നിയമനിർദ്ദേശ നിയന്ത്രണങ്ങളിലൂടെ നിയമനിർദ്ദേശ മോഡലുകൾ നിയന്ത്രിക്കുക.
- **ഫൈൻ-ട്യൂണിംഗ് (പാഠം 18)** `gpt-4.1-mini` നിലനിർത്തുന്നു: GPT-5 ആർഎഫ്‌ടി പിന്തുണയ്ക്കുന്നു, എന്നാൽ ഇവിടെ കാണിച്ച പ്രസക്തമായ എസ്‌എഫ്‌ടി പിന്തുണിക്കുന്നില്ല.
- പാഠങ്ങൾ 20 (Mistral) & 21 (Meta) `temperature`/`max_tokens` നിലനിർത്തുന്നു കാരണം അവ Mistral/Llama മോഡലുകളെയാണ് ലക്ഷ്യമിടുന്നത്, ഇവക്ക് പിന്തുണയുണ്ട്.

- ലിന്റിംഗിന് `pylint` ഉപയോഗിക്കുക (ചില ഉദാഹരണങ്ങളിൽ `# pylint: disable=all` ഉണ്ട് എളുപ്പത്തിനായി)

- API ക്രെഡൻഷ്യലുകൾ `.env` ഫയലിൽ സൂക്ഷിക്കുക, കോഡിൽ ഇല്ല

### TypeScript

- പരിസ്ഥിതി വ്യത്യാസങ്ങൾക്ക് `dotenv` പാക്കേജ് ഉപയോഗിക്കുക
- tsconfig.json ൽ TypeScript കോfiguration ഓരോ ആപ്പിനും
- Azure OpenAI用 `openai` പാക്കേജ് ഉപയോഗിക്കുക (`/openai/v1/` എന്റ്പോയിന്റ് പങ്കുവെച്ച്, `client.responses.create` വിളിക്കുക); Microsoft Foundry മോഡലുകൾക്ക് `@azure-rest/ai-inference` ഉപയോഗിക്കുക
- ഓട്ടോ-റിലോഡിനായി `nodemon` ഡെവലപ്പ്മെന്റിന് ഉപയോഗിക്കുക
- പ്രവർത്തിപ്പിക്കുന്നതിന് മുൻപ് നിർമ്മിക്കുക: `npm run build` പിന്നെ `npm start`

## ടെസ്റ്റിംഗും സ്ഥിരീകരണവും

### സമർപ്പിക്കാനുമുമ്പ് പരിശോധനകൾ

ഈ റിപ്പോസിറ്ററി GitHub Actions ഉപയോഗിച്ച് സ്ഥിരീകരിക്കപ്പെടുന്നു. പൂർണ്ണപ്പെട്ടു സമർപ്പിക്കുന്നതിന് മുമ്പ്:

1. **മാർക്ക്ഡൗൺ ലിങ്കുകൾ പരിശോധിക്കുക**:
   ```bash
   # validate-markdown.yml പ്രവൃത്തി പരിശോധിക്കുന്നു:
   # - തകർന്നിരിക്കുന്ന ആപേക്ഷിക പാതകൾ
   # - പാതകളിൽ ട്രാക്കിംഗ് ഐഡികൾ കാണാതിരിക്കുന്നു
   # - URL ൽ ട്രാക്കിംഗ് ഐഡികൾ കാണാതിരിക്കുന്നു
   # - രാജ്യഭാഷാ കോഡോടൊപ്പം ഉള്ള URLകൾ
   # - തകർന്നിരിക്കുന്ന ബാഹ്യ URLകൾ
   ```

2. **മാനുവൽ ടെസ്റ്റിംഗ്**:
   - Python ഉദാഹരണങ്ങൾ പരീക്ഷിക്കുക: venv സജീവമാക്കി സ്ക്രിപ്റ്റുകൾ ഓടിക്കുക
   - TypeScript ഉദാഹരണങ്ങൾ പരീക്ഷിക്കുക: `npm install`, `npm run build`, `npm start`
   - പരിസ്ഥിതി വ്യത്യാസങ്ങൾ ശരിയാവുന്നത് ഉറപ്പ് വരുത്തുക
   - API কীകൾ കോഡ് ഉദാഹരണങ്ങളിൽ പ്രവർത്തിക്കുന്നുവെന്ന് പരിശോധിക്കുക

3. **കോഡ് ഉദാഹരണങ്ങൾ**:
   - എല്ലാം പിഴവുകൾ nélkül പ്രവർത്തിക്കുന്നു എന്ന് ഉറപ്പാക്കുക
   - Azure OpenAI ഉം OpenAI API ഉം ഇടയിൽ ടെസ്റ്റ് ചെയ്യുക
   - Microsoft Foundry മോഡലുകളും പിന്തുണയുള്ള സ്ഥലങ്ങളിൽ ഉദാഹരണങ്ങൾ ശരിയാണെന്ന് ഉറപ്പാക്കുക

### സ്വയമാറ്റൻ ടെസ്റ്റുകൾ ഇല്ല

ഇത് ഒരു എഡ്യുക്കേഷനൽ റിപ്പോസിറ്ററിയാണ്, ട്യൂട്ടോറിയലുകൾക്കും ഉദാഹരണങ്ങൾക്കും. യൂണിറ്റ് ടെസ്റ്റുകളും ഇന്റഗ്രേഷൻ ടെസ്റ്റുകളും ഓടികൂടതില്ല. സ്ഥിരീകരണം പ്രധാനമായും:

**TypeScript നിർമ്മാണ പിശകുകൾ**:
- പ്രത്യേക ആപ്പ് ഡയറക്ടറിയിൽ `npm install` ഓടിക്കുക

- `node_modules` പിടിച്ചുവീഴ്ത്തി ആവശ്യമെങ്കിൽ വീണ്ടും ഇൻസ്റ്റാൾ ചെയ്യുക

- `.env` ഫയൽ ഉണ്ട് പ്ലസ് ശരിയായ മൂല്യങ്ങൾ

- ഓരോ പാഠവും സ്വയം പര്യാപ്തവും സ്വതന്ത്രമായി പൂർത്തിയാക്കാവുന്നതുമാണ്

- ഉള്ളടക്കം ബഹുഭാഷാരൂപത്തിൽ സ്വയമാറ്റം പ്രവർത്തനങ്ങളോടെ
- ചോദ്യങ്ങൾക്കും പിന്തുണയ്ക്കും Discord ൽ സജീവമായ സമുദായം
