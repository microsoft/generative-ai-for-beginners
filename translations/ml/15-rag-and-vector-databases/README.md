# വീണ്ടെടുക്കൽ വർദ്ധിപ്പിച്ച ജനറേഷൻ (RAG) және വെക്ടർ ഡേറ്റാബേസുകൾ

[![വീണ്ടെടുക്കൽ വർദ്ധിപ്പിച്ച ജനറേഷൻ (RAG) және വെക്ടർ ഡേറ്റാബേസുകൾ](../../../translated_images/ml/15-lesson-banner.ac49e59506175d4f.webp)](https://youtu.be/4l8zhHUBeyI?si=BmvDmL1fnHtgQYkL)

തിരയൽ അപ്ലിക്കേഷനുകളുടെ പാഠത്തിൽ, നിങ്ങളുടെ സ്വന്തം ഡേറ്റയെ Large Language Models (LLMs)യിലേക്കു എങ്ങനെ ഇന്റഗ്രേറ്റ് ചെയ്യാമെന്ന് നാം ചെറിയൊരു പരിചയം നേടി. ഈ പാഠത്തിൽ, നിങ്ങളുടെ LLM അപ്ലിക്കേഷനിൽ ഡേറ്റയുടെ അസ്ഥിരതയെ (grounding) കുറിച്ച്, പ്രക്രിയയുടെ സംവിധാനങ്ങളും എംബെഡിംഗുകളും വാചകവും ഉൾപ്പെടുന്ന ഡേറ്റ സംഭരണ രീതി സംബന്ധിച്ചും കൂടുതൽ അറിയാം.

> **വീഡിയോ ഉടൻ വരുന്നു**

## പരിചയം

ഈ പാഠത്തിൽ നമുക്ക് പഠിക്കാനിരിക്കുന്നവ:

- RAG ഇതെന്ത്, AI (കൃത്രിമ ബുദ്ധിമുട്ടുകളിൽ) ഇത് എന്തിനുവേണ്ടിയാണ് ഉപയോഗിക്കുന്നത് എന്നതിനെ കുറിച്ചുള്ള ഒരു പരിചയം.

- വെക്ടർ ഡേറ്റാബേസുകൾ എന്തൊക്കെയാണെന്ന് മനസിലാക്കുകയും നമ്മുടെ അപ്ലിക്കേഷനിൽ സൃഷ്ടിക്കുകയും ചെയ്യുക.

- ഒരു പ്രായോഗിക ഉദാഹരണത്തിൽ RAG എങ്ങനെ ഒരു അപ്ലിക്കേഷനിലേക്ക് ഇന്റഗ്രേറ്റ് ചെയ്യാമെന്ന് കാണുക.

## പഠന ലക്ഷ്യങ്ങൾ

ഈ പാഠം പൂർത്തിയാക്കിയശേഷം, നിങ്ങൾക്ക് കഴിയുന്നത്:

- ഡേറ്റാ വീണ്ടെടുക്കലിലും പ്രോസസ്സിംഗിലും RAGയുടെ പ്രാധാന്യം വിശദീകരിക്കുക.

- RAG അപ്ലിക്കേഷൻ സജ്ജീകരിക്കുകയും നിങ്ങളുടെ ഡേറ്റയെ LLM-നോട് ഭൂമികമിടുകയും ചെയ്യുക.

- LLM അപ്ലിക്കേഷനുകളിൽ RAGയും വെക്ടർ ഡേറ്റാബേസുകളും ഫലപ്രദമായി ഇന്റഗ്രേറ്റ് ചെയ്യുക.

## നമ്മുടെ സീനാർിയോ: നമ്മുടെ സ്വന്തം ഡേറ്റയിലൂടെ LLMs മെച്ചപ്പെടുത്തൽ

ഈ പാഠത്തിന്, വിദ്യാഭ്യാസ സ്റ്റാർട്ടപ്പ്‌സിൽ നമ്മളുടേതായ കുറിപ്പുകൾ ചേർക്കാൻ ആഗ്രഹിക്കുന്നു, അതിലൂടെ ചാറ്റ്ബോട്ട് കൂടുതൽ വിഷയങ്ങളിൽ വിവരങ്ങൾ നേടാൻ കഴിയും. നമുക്കുള്ള കുറിപ്പുകൾ ഉപയോഗിച്ച്, പഠിതാക്കൾക്ക് നല്ല പഠനം സാധിക്കാനും വിവിധ വിഷയങ്ങൾ മനസ്സിലാകാനും, പരീക്ഷകൾക്കായി പുനഃപരിശോധന ചെയ്യുന്നതിനും സഹായകരമാകും. നമ്മുടെ സീനാറിയോ സൃഷ്ടിക്കാൻ താഴെ പറയുന്നവ ഉപയോഗിക്കും:

- `Azure OpenAI:` ക്ലിപ്പാറ് സൃഷ്ടിക്കാൻ ഉപയോഗിക്കുന്ന LLM

- `AI for beginners' lesson on Neural Networks`: ഇത് നമ്മുടെ LLM-നുറ്ററിയിക്കുന്ന ഡേറ്റ ആയിരിക്കും

- `Azure AI Search` and `Azure Cosmos DB:` ഡേറ്റ സംഭരിക്കാനും തിരയൽ ഇൻഡക്സ് സൃഷ്ടിക്കാനും ഉപയോഗിക്കുന്ന വെക്ടർ ഡേറ്റാബേസുകൾ

ഉപയോക്താക്കൾക്ക് അവരുടെ കുറിപ്പുകളിൽനിന്ന് പരിശീലന ക്വിസുകളും പുനഃപരിശോധന ഫ്ലാഷ് കാർഡുകളും രൂപപ്പെടുത്താനാകും, കൂടാതെ അത് സംക്ഷിപ്തമായി സംഗ്രഹിക്കാനും കഴിയും. തുടങ്ങാൻ, RAG എന്ത് ആണെന്നും എങ്ങനെ പ്രവർത്തിക്കുന്നുവെന്നും നോക്കാം:

## വീണ്ടെടുക്കൽ വർദ്ധിപ്പിച്ച ജനറേഷൻ (RAG)

LLM അധിഷ്ഠിതം ഉള്ള ഒരു ചാറ്റ്ബോട്ട് ഉപയോക്താവിന്റെ പ്രോമ്പ്റ്റുകൾ പ്രോസസ് ചെയ്ത് മറുപടികൾ സൃഷ്ടിക്കുന്നു. ഇത് ഇന്ററാക്ടീവ് ആയി രൂപകല്‍പ്പനചെയ്തതാണ്, വ്യത്യസ്ത വിഷയങ്ങളിൽ ഉപയോക്താക്കളുമായി സംവദിക്കുന്നു. എങ്കിലും, മറുപടികൾ നൽകുന്നത് നൽകിയ കണ്ടекстിന്റെയും അടിസ്ഥാനത്തിലുള്ള പരിശീലന ഡേറ്റയുടെയും പരിധിയിൽ മാത്രമാണ്. ഉദാഹരണത്തിന്, GPT-4 ന്റെ അറിവ് അവസാനമុខം സെപ്റ്റംബർ 2021 ആണ്, അതിനുശേഷമുള്ള സംഭവങ്ങളിൽ അറിവില്ലാത്തതായിരിക്കും. കൂടാതെ, LLM-കളുടെ പരിശീലനത്തിന് ഉപയോഗിക്കുന്ന ഡേറ്റിൽ വ്യക്തിഗത കുറിപ്പുകൾ അല്ലെങ്കിൽ കമ്പനിയുടെ ഉൽപ്പന്ന മാനുവൽ പോലെയുള്ള രഹസ്യ വിവരങ്ങൾ ഉൾപ്പെടാറില്ല.

### RAG (Retrieval Augmented Generation) എങ്ങനെ പ്രവർത്തിക്കുന്നു

![RAG പ്രവർത്തനം കാണിക്കുന്ന ചിത്രരേഖ](../../../translated_images/ml/how-rag-works.f5d0ff63942bd3a6.webp)

നിങ്ങൾക്കു നിങ്ങളുടെ കുറിപ്പുകളിൽ നിന്നു ക്വിസുകൾ സൃഷ്ടിക്കുന്ന ഒരു ചാറ്റ്ബോട്ട് ഡിപ്ലോയ് ചെയ്യണമെങ്കിൽ, അറിവ് ആധാരത്തോടുള്ള ബന്ധം ആവശ്യമാണ്. ഇതുവഴി RAG സഹായിക്കുന്നു. RAGകൾ പ്രവർത്തിക്കുന്നത് ഇപ്രകാരം:

- **അറിവ് ആധാരം:** വീണ്ടെടുക്കലിന് മുമ്പ്, ഡോക്യുമെന്റുകൾ ഇൻജസ്റ്റ് ചെയ്ത് പ്രീപ്രോസസ് ചെയ്യണം, സാധാരണയായി വലിയ ഡോക്യുമെന്റുകൾ ചെറിയ ഭാഗങ്ങളായി വെട്ടി, അവയെ ടെക്സ്റ്റ് എംബെഡിങ്ങായി മാറ്റി ഡേറ്റാബേസിൽ സൂക്ഷിക്കുന്നു.

- **ഉപയോക്താവിന്റെ ചോദ്യം:** ഉപയോക്താവ് ഒരു ചോദ്യമുയർക്കുന്നു

- **വീണ്ടെടുക്കൽ:** ഉപയോക്താവ് ചോദ്യം ചോദിച്ചപ്പോൾ, എംബെഡിങ് മോഡൽ അറിവ് ആധാരത്തിൽ നിന്നും ബന്ധപ്പെട്ട വിവരങ്ങൾ വീണ്ടെടുക്കുന്നു, അത് പ്രോമ്പ്റ്റിലേക്ക് ഉൾപ്പെടുത്തപ്പെടും.

- **വർദ്ധിപ്പിച്ച ജനറേഷൻ:** LLM, വീണ്ടെടുത്ത ഡേറ്റ അടിസ്ഥാനമാക്കി മറുപടി മെച്ചപ്പെടുത്തുന്നു. ഇത് മുൻകൂട്ടി പരിശീലിപ്പിച്ച ഡേറ്റ മാത്രമല്ല, ചേർത്ത കണ്ടекстിൽ നിന്നുള്ള അനുയോജ്യമായ വിവരങ്ങൾ ഉൾപ്പെടുത്തുന്നതും ആണ്. വീണ്ടെടുത്ത ഡേറ്റ LLM-ന്റെ മറുപടികൾ പ്രോത്സാഹിപ്പിക്കാൻ ഉപയോഗിക്കുന്നു. പിന്നീട് LLM ഉപയോക്താവിന്റെ ചോദ്യത്തിന് ഉത്തരം നൽകുന്നു.

![RAG ആർകിടെക്ചർ കാണിക്കുന്ന ചിത്രം](../../../translated_images/ml/encoder-decode.f2658c25d0eadee2.webp)

RAG-കളുടെ ആർകിടെക്ചർ രണ്ട് ഭാഗങ്ങളായ എക്കോർഡറും ഡിക്കോർഡറും അടങ്ങിയ ട്രാൻസ്ഫോർമറുകൾ ഉപയോഗിച്ച് നടപ്പിലാക്കുന്നു. ഉദാഹരണത്തിന്, ഒരു ഉപയോക്താവ് ചോദ്യമുയർത്തുമ്പോൾ, ഇൻപുട്ട് വാചകം വേഡുകളുടെ അര്‍ത്ഥം പകർത്തുന്ന വെക്ടറുകളായി 'എങ്കോഡ്' ചെയ്യപ്പെടുകയും, ഈ വെക്ടറുകൾ ഡോക്യുമെന്റ് ഇൻഡക്സിൽ 'ഡിക്കോഡ്' ചെയ്ത് ഉപയോക്തൃ ചോദ്യം അടിസ്ഥാനമാക്കി പുതിയ വാചകം സൃഷ്ടിക്കുകയും ചെയ്യുന്നു. LLM ഔട്ട്‌പുട്ട് സൃഷ്ടിക്കാൻ എക്കോർഡർ-ഡിക്കോർഡർ മോഡൽ ഉപയോഗിക്കുന്നു.

RAG നടപ്പിലാക്കുമ്പോൾ പ്രമേയപത്രത്തിൽ നിർദ്ദേശിക്കുന്ന രണ്ടു സമീപനങ്ങൾ ഉണ്ട്: [Retrieval-Augmented Generation for Knowledge intensive NLP (natural language processing software) Tasks](https://arxiv.org/pdf/2005.11401.pdf?WT.mc_id=academic-105485-koreyst):

- **_RAG-Sequence_**: ഉപയോക്താവിന്റെ ചോദ്യത്തിന് ഏറ്റവും മികച്ച മറുപടി പ്രവചിക്കാൻ വീണ്ടെടുത്ത ഡോക്യുമെന്റുകൾ ഉപയോഗിക്കുന്നു

- **RAG-Token**: അടുത്ത ടോക്കൻ സൃഷ്ടിക്കാൻ ഡോക്യുമെന്റുകൾ ഉപയോഗിച്ച്, പിന്നീട് ഉപയോക്തൃ ചോദ്യത്തിന് മറുപടി നൽകാൻ അവ വീണ്ടെടുക്കുന്നു

### RAGകൾ എന്തിനായി ഉപയോഗിക്കുക? 

- **വിവര സമ്പുഷ്ടി:** ടെക്സ്റ്റ് മറുപടികൾ അപ്ഡേറ്റിലും നിലവാരത്തിലും ഉറപ്പുവരുത്തുന്നു. അതുവഴി, പ്രത്യേക ഡൊമെയ്ൻ ദൗത്യം മെച്ചപ്പെടുത്തുന്നു അറിവ് ആധാരത്തിലേക്ക് പ്രവേശനം വഴി.

- **സാക്ഷ്യം തെളിയിക്കാവുന്ന ഡേറ്റ** ഉപയോഗിച്ച് ഉപയോക്തൃ ചോദ്യങ്ങൾക്ക് കണ്ടекстു നൽകി കൃത്രിമ വിവര സൃഷ്ടി കുറയ്ക്കുന്നു.

- LLM ഫൈൻ-ട്യൂണിങിനെ അപേക്ഷിച്ച് ഉതകാത്ത വില വേണമെന്നതിനാൽ **വിലക്കുറവുള്ളതാണ്**.

## അറിവ് ആധാരം സൃഷ്ടിക്കൽ

നമ്മുടെ അപ്ലിക്കേഷൻ നമ്മുടെ വ്യക്തിഗത ഡേറ്റായ Neural Network പാഠഭാഗം AI For Beginners പാഠ്യപദ്ധതിയിലാണ് ആശ്രയിച്ചിരിക്കുന്നത്.

### വെക്ടർ ഡേറ്റാബേസുകൾ

ഒരു വെക്ടർ ഡേറ്റാബേസ് പരമ്പരാഗത ഡേറ്റാബേസുകളേക്കാളും വ്യത്യസ്തമാണ്; ഇത് സംജ്ഞാത വെക്ടറുകൾ സൂക്ഷിക്കാൻ, മാനേജുചെയ്യാനും തിരയാനും പ്രത്യേകിച്ചുള്ള ഡേറ്റാബേസാണ്. ഇത് ഡോക്യുമെന്റുകളുടെ സംഖ്യാത്മക പ്രതിനിധികളായ നമ്പരുകൾ സൂക്ഷിക്കുന്നു. ഡേറ്റ വെക്ടറുകളായി വിഭജിക്കുന്നത് നമ്മുടെ AI സംവിധാനം ആ ഡേറ്റ മനസിലാക്കാനും പ്രോസസ് ചെയ്യാനും എളുപ്പമാക്കുന്നു.

ഞങ്ങൾ LLMകൾക്ക് അനുവദിച്ചീടുന്ന ടോക്കൺ പരിധി പരിഗണിച്ച്, എംബെഡിംഗുകൾ വെക്ടർ ഡേറ്റാബേസുകളിൽ സൂക്ഷിക്കുന്നു. മുഴുവൻ എംബെഡിംഗുകളും LLM-ൽ പാസാക്കാൻ കഴിയാത്തതിനാൽ, അവ ചങ്കുകളിൽ വിഭജിച്ച് ഉപയോക്താവ് ചോദ്യം ചോദിച്ചപ്പോൾ ഏറ്റവും അനുയോജ്യമായ എംബെഡിംഗുകൾ പ്രോമ്പ്റ്റുമായി തിരികെ നൽകും. ചങ്കിംഗ് ടോക്കൺ ചർജ് കുറയ്ക്കാനും സഹായിക്കുന്നു.

പ്രസിദ്ധമായ വെക്ടർ ഡേറ്റാബേസുകൾ ആയ Azure Cosmos DB, Clarifyai, Pinecone, Chromadb, ScaNN, Qdrant, DeepLake എന്നിവ ഉൾപ്പെടുന്നു. Azure CLI ഉപയോഗിച്ച് Azure Cosmos DB മോഡൽ സൃഷ്ടിക്കാനുള്ള കമാൻഡ്:

```bash

az cosmosdb create -n <cosmos-db-name> -r <resource-group-name>
az cosmosdb list-keys -n <cosmos-db-name> -g <resource-group-name>
```

### ടെക്സ്റ്റിൽ നിന്ന് എംബെഡിംഗിലേക്ക്

ഡേറ്റ സ്റ്റോർ ചെയ്യുന്നതിനു മുമ്പ്, അത് വെക്ടർ എംബെഡിംഗുകളായി മാറ്റണം. വലിയ ഡോക്യുമെന്റുകളോ ദീർഘമായ വാചകങ്ങളോ അടുത്തുണ്ടെങ്കിൽ, നിങ്ങൾക്ക് പ്രതീക്ഷിക്കുന്ന ചോദ്യങ്ങളുടെ അടിസ്ഥാനത്തിൽ അവ ചങ്കിലാക്കാം. ചങ്കിംഗ് വാചക തലത്തിൽ അല്ലെങ്കിൽ പാരാഗ്രാഫ് തലത്തിലായിരിക്കാം. ചങ്കുകൾ അസ്പഷ്ടമായ ആശയം അർത്ഥമാക്കുമ്പോൾ ദിശാബോധം നൽകുന്നതിനായി, നിങ്ങൾക്ക് മറ്റ് കണ്ടെക്സ്റ്റുകൾ ചങ്കിൽ ചേർക്കാം, ഉദാഹരണത്തിന്, ഡോക്യുമെന്റ് തലക്കെട്ട് അല്ലെങ്കിൽ ചങ്കിനു മുമ്പോ പിറകിലും കുറിപ്പ് ചേർക്കാം. അല്ലെങ്കിൽ ചങ്ക് ചെയ്യാം:

```python
def split_text(text, max_length, min_length):
    words = text.split()
    chunks = []
    current_chunk = []

    for word in words:
        current_chunk.append(word)
        if len(' '.join(current_chunk)) < max_length and len(' '.join(current_chunk)) > min_length:
            chunks.append(' '.join(current_chunk))
            current_chunk = []

    # ലഭ്യമായതിനേക്കാളും കുറഞ്ഞ ദൈർഘ്യമുള്ള അവസാന ഭാഗം ഉണ്ടെങ്കിൽ പോലും അത് ചേർക്കുക
    if current_chunk:
        chunks.append(' '.join(current_chunk))

    return chunks
```

ചങ്ക് ചെയ്ത ശേഷം, വ്യത്യസ്ത എംബെഡിങ് മോഡലുകൾ ഉപയോഗിച്ച് ടെക്സ്റ്റ് എംബെഡ് ചെയ്യാം. ഉപയോഗിക്കാവുന്ന ചില മോഡലുകൾ: word2vec, OpenAI യുടെ ada-002, Azure Computer Vision എന്നിവ. നിങ്ങൾക്കു തിരഞ്ഞെടുക്കുന്നത് ഉപയോഗിക്കുന്ന ഭാഷകൾ, എൻകോഡ് ചെയ്യുന്ന ഉള്ളടക്ക തരം (ടെക്സ്റ്റ്/ഇമേജ്/ഓഡിയോ), എൻപുട്ടിന്റെ വലിപ്പം, ഔട്ട്പുട്ടിന്റെ നീളവും ആശ്രയിച്ചാണ്.

OpenAI-യുടെ `text-embedding-ada-002` മോഡൽ ഉപയോഗിച്ചുള്ള എംബെഡുചെയ്ത ടെക്സ്റ്റിന് ഉദാഹരണം:

## വീണ്ടെടുക്കൽ ಮತ್ತು വെക്ടർ തിരയൽ

ഉപയോക്താവ് ചോദ്യമുയർത്തുമ്പോൾ, റിട്ട്രീവർ ക്വറി എങ്കോഡർ ഉപയോഗിച്ച് അത് വെക്ടറാക്കി മാറ്റുന്നു, തുടർന്ന് ഡോക്യുമെന്റ് തിരയൽ ഇൻഡക്സ് വഴി ഉൾപ്പെട്ട ഡോക്യുമെന്റുകളിൽ നിന്ന് ബന്ധപ്പെട്ട വെക്ടറുകൾ തിരയുന്നു. ശേഷം ഇൻപുട്ടും ഡോക്യുമെന്റ് വെക്ടറുകളും ടെക്സ്റ്റായി മാറ്റി LLM-ൽ പാസാക്കുന്നു.

### വീണ്ടെടുക്കൽ

വീണ്ടെടുക്കൽ അതേസമയം സിസ്റ്റം തിരയൽ മാനദണ്ഡങ്ങളിൽച്ചെലവഴിച്ച് ഇൻഡക്സ്-ലിൽനിന്ന് ചുരുക്കിയ ഡോക്യുമെന്റുകൾ കണ്ടെത്താൻ ശ്രമിക്കുന്നു. റിട്ട്രീവറിന്റെ ലക്ഷ്യം LLM-യെ ഭൂമികമിടാൻ ഉപയോഗിക്കുന്ന ഡോക്യുമെന്റുകൾ ലഭ്യമാക്കുക ആണ്.

ഡേറ്റബേസിൽ തിരയലിനായി വിവിധ മാർഗ്ഗങ്ങൾ ഉണ്ട്:

- **കീവേഡ് തിരയൽ** - ടെക്സ്റ്റ് തിരയലിനായി

- **വെക്ടർ തിരയൽ** - ഡോക്യുമെന്റുകൾ ടെക്സ്റ്റിൽ നിന്ന് വെക്ടർ പ്രതിനിധികളായി മാറ്റി ഉപയോഗിക്കുന്ന എംബെഡിങ് മോഡലുകൾ കൊണ്ട്, വാക്കുകളുടെ അർത്ഥത്തിലൂടെ **സെമാന്റിക് തിരയൽ** അനുവദിക്കുന്നു. ഉപയോക്തൃ ചോദ്യത്തിന് ഏറ്റവും അടുത്ത് വെക്ടർ പ്രതിനിധികളുള്ള ഡോക്യുമെന്റുകൾ തിരയലിൽ ലഭിക്കും.

- **ഹൈബ്രിഡ്** - കീവേഡ് തിരയലും വെക്ടർ തിരയലും സംയുക്തം.

ഒരു വെല്ലുവിളി ഉണ്ട്, ചോദ്യത്തിനോർത്തുപോലൊരു മറുപടി ഡാറ്റാബേസിൽ ഇല്ലെങ്കിൽ, സിസ്റ്റം ഏറ്റവും നല്ല വിവരവും നൽകും. എന്നാൽ, പ്രാധാന്യത്തിനായി ഏറ്റവും വലിയ ദൂരം നിശ്ചയിക്കുക, അല്ലെങ്കിൽ കീവേഡ്-വെക്ടർ സംയോജനം നടത്തിയ ഹൈബ്രിഡ್ തിരയൽ ഉപയോഗിക്കാം. ഈ പാഠത്തിൽ, നിർദ്ദേശിച്ച പോലെ ഹീബ്രിഡ് തിരയൽ ഉപയോഗിക്കും — വെക്ടർ തിരയലും കീവേഡ് തിരയലും സംയുക്തം. ഡാറ്റ ഞങ്ങൾ ഡാറ്റാഫ്രെയിമിൽ സ്റ്റോർ ചെയ്യും, ജായി ചങ്കുകളും എംബെഡിങ്ങുകളും ഉള്ള കോളങ്ങളടങ്ങി.

### വെക്ടർ സാദൃശ്യം

റിട്ട്രീവർ അറിവ് ഡാറ്റാബേസിൽ സമാനമായ എംബെഡിങ്ങുകൾ തിരയുന്നു, ഏറ്റവും അടുത്ത അയൽക്കാരെ കണ്ടെത്തുന്നു, കാരണം അവ ടെക്സ്റ്റുകളും സമാനമാണ്. ഉപയോക്താവ് ചോദ്യമുയർത്തുമ്പോൾ, ആദ്യം അത് എംബെഡ് ചെയ്ത് സമാനമായ എംബെഡിങ്ങുകളുമായി താരതമ്യം ചെയ്യുന്നു. ഏറ്റവും സാധാരണമുള്ള അളവ് cosine similarity ആണ്, ഇത് രണ്ട് വെക്ടറുകൾക്കിടയിലെ കോണിന്റെ അടിസ്ഥാനത്തിലാണ്.

സാദൃശ്യ മനസ്സിലാക്കുവാനുള്ള മറ്റൊരു മാർഗ്ഗം Euclidean distance ആണ്, ഇത് വെക്ടറുകൾക്കിടയിലെ നേരിയ രേഖാ ദൂരം; മറ്റൊന്ന് dot product ആണ്, ഇത് രണ്ട് വെക്ടറുകളിലെ അനുബന്ധ ഘടകങ്ങളുടെ ഗുണഫലങ്ങളുടെ മൊത്തം നിരീക്ഷിക്കുന്നു.

### തിരയൽ ഇൻഡക്സ്

വീണ്ടെടുക്കുമ്പോഴ്, നമുക്ക് അറിവ് ആധാരത്തിന് തിരയൽ ഇൻഡക്സ് നിർമ്മിക്കേണ്ടി വരും. ഒരു ഇൻഡക്സ് നമ്മുടെ എംബെഡിങ്ങുകൾ സൂക്ഷിക്കുകയും വലിയ ഡാറ്റാബേസിലും വളരെ ഇളക്കിയ ചങ്കുകൾ പെട്ടെന്ന് കണ്ടെത്തുകയും ചെയ്യുന്നു. താഴെക്കൊടുത്തിരിക്കുന്ന കോഡ് ഉപയോഗിച്ച് ഇൻഡക്സ് ലൊക്കലി സൃഷ്ടിക്കാം:

```python
from sklearn.neighbors import NearestNeighbors

embeddings = flattened_df['embeddings'].to_list()

# സെർച്ച് ഇൻഡക്സുണ്ടാക്കുക
nbrs = NearestNeighbors(n_neighbors=5, algorithm='ball_tree').fit(embeddings)

# ഇൻഡക്സ് ക്വറി ചെയ്യാൻ, നിങ്ങൾക്ക് kneighbors മെത്തഡ് ഉപയോഗിക്കാം
distances, indices = nbrs.kneighbors(embeddings)
```

### പുന:ക്രമീകരണം

ഡാറ്റാബേസ് query ചെയ്തശേഷം, ഫലം ഏറ്റവും അനുയോജ്യമായ രീതിയിൽ ക്രമീകരിക്കേണ്ടി വരാം. പുനക്രമീകരണ LLM യന്ത്രം പ്രയോജനം കൊണ്ട് തിരയൽ ഫലങ്ങൾ ഏറ്റവും അനുയോജ്യമായത് ആദ്യം വരുന്നതായി ക്രമീകരിക്കുന്നു. Azure AI Search ഉപയോഗിക്കുമ്പോൾ, സെമാന്റിക് റീറാങ്കറിലൂടെ ഇത് സ്വയം നടക്കുന്നു. പിടിച്ചുകൊണ്ടുള്ള ലളിത ഉദാഹരണം അടുത്ത അയൽക്കാരെ ഉപയോഗിക്കുന്നതിന്റെ പ്രവർത്തനം:

```python
# ഏറ്റവും സമാനമായ രേഖകൾ കണ്ടെത്തുക
distances, indices = nbrs.kneighbors([query_vector])

index = []
# ഏറ്റവും സമാനമായ രേഖകൾ പ്രിന്റ് ചെയ്യുക
for i in range(3):
    index = indices[0][i]
    for index in indices[0]:
        print(flattened_df['chunks'].iloc[index])
        print(flattened_df['path'].iloc[index])
        print(flattened_df['distances'].iloc[index])
    else:
        print(f"Index {index} not found in DataFrame")
```

## എല്ലാം ഒന്നിച്ചുവെക്കുക

അവസാനഘട്ടം ഞങ്ങളുടെ LLM ചേർത്ത് നമ്മുടെ ഡേറ്റയിൽ ഭൂമികമിട്ട മറുപടികൾ നേടുന്നതാണ്. ഇത് ഇങ്ങനെ നടപ്പിലാക്കാം:

```python
user_input = "what is a perceptron?"

def chatbot(user_input):
    # ചോദ്യത്തെ ഒരു ക്വറി വെക്ടറായി മാറ്റുക
    query_vector = create_embeddings(user_input)

    # ഏറ്റവും സാമ്യം കുറിപ്പുകൾ കണ്ടെത്തുക
    distances, indices = nbrs.kneighbors([query_vector])

    # പ്രാധാന്യം നൽകാൻ കുറിപ്പുകൾ ക്വറിയിലേക്ക് ചേർക്കുക
    history = []
    for index in indices[0]:
        history.append(flattened_df['chunks'].iloc[index])

    # ചരിത്രവും ഉപയോക്തൃ ഇൻപുട്ടും സംയോജിപ്പിക്കുക
    history.append(user_input)

    # ഒരു സന്ദേശ объек്റെ তৈরি ചെയ്യുക
    messages=[
        {"role": "system", "content": "You are an AI assistant that helps with AI questions."},
        {"role": "user", "content": "\n\n".join(history) }
    ]

    # പ്രതികരണം സൃഷ്ടിക്കാൻ Responses API ഉപയോഗിക്കുക
    response = client.responses.create(
        model="gpt-4o-mini",
        temperature=0.7,
        max_output_tokens=800,
        input=messages,
        store=False,
    )

    return response.output_text

chatbot(user_input)
```

## നമ്മുടെ അപ്ലിക്കേഷൻ മൂല്യനിർണ്ണയം

### മൂല്യനിർണ്ണയ മാനദണ്ഡങ്ങൾ

- ലഭിക്കുന്ന മറുപടികളുടെ ഗുണനിലവാരം: സ്വാഭാവികമായും സുതാര്യവുമായ മാനവീകൃതമുള്ള ശബ്ദമുണ്ടാകണം

- ഡേറ്റയുടെ ഭൂമികത്വം: ലഭിച്ച മറുപടി നൽകിയ ഡോക്യുമെന്റുകളിൽ നിന്നുള്ളതാണെന്ന് വിലയിരുത്തൽ

- പ്രസക്തി: മറുപടി ചോദ്യവുമായി പൊരുത്തപ്പെടുന്നോ എന്ന് പരിശോധിക്കൽ

- പ്രവാഹം: മറുപടി വ്യാകരണപരമായി അർത്ഥവത്തായതാണോ എന്നത്

## RAG (Retrieval Augmented Generation) കൂടി വെക്ടർ ഡേറ്റാബേസുകൾ ഉപയോഗിക്കുന്ന ഉപയോഗ കേസുകൾ

നിങ്ങളുടെ ആപ്ലിക്കേഷൻ മെച്ചപ്പെടുത്താൻ ഫങ്ഷൻ കോൾസ് കരുതാവുന്ന പല വക പ്രദേശങ്ങളും ഉണ്ട്:

- ചോദ്യം-ഉത്തരം: നിങ്ങളുടെ കമ്പനിയുടെ ഡേറ്റ നിർമ്മിച്ച്, ജീവനക്കാർക്ക് ചാറ്റ് വഴി ചോദ്യങ്ങൾ ചോദിക്കാൻ സഹായിക്കുന്നു.

- ശിപാർശാ സിസ്റ്റങ്ങൾ: ഏറ്റവും സമാനമായ മൂല്യങ്ങൾ (ഉദാ: സിനിമകൾ, റെസ്റ്റൊറന്റുകൾ എന്നിവ) പൊരുത്തപ്പെടുത്തുന്ന സംവിധാനം സൃഷ്ടിക്കാം.

- ചാറ്റ്ബോട് സേവനങ്ങൾ: ചാറ്റ് ചരിത്രം സംഭരിച്ചു ഉപയോക്തൃ ഡേറ്റയെ അടിസ്ഥാനമാക്കി സംവാദം വ്യക്തിപരമാക്കുന്നു.

- വെക്ടർ എംബെഡിങ്ങുകളുടെ അടിസ്ഥാനത്തിലുള്ള ചിത്രം തിരയൽ, ചിത്രരൂപ തിരിച്ചറിവിലും അസാധാരണ കണ്ടെത്തലിലും ഉപകാരപ്രദം.

## സംഗ്രഹം

ഡാറ്റ അപ്ലിക്കേഷനിലേക്ക് ചേർക്കൽ, ഉപയോക്തൃ ക്വറിയും ഔട്ട്‌പുട്ടും ഉൾക്കൊള്ളുന്ന RAGയുടെ അടിസ്ഥാന പ്രദേശങ്ങൾ നാം പരിശോധിച്ചു. RAG സൃഷ്ടി ലളിതമാക്കാൻ Semanti Kernel, Langchain, Autogen പോലുള്ള ഫ്രെയിംവർക്കുകൾ ഉപയോഗിക്കാം.

## അസൈൻമെന്റ്

Retrieval Augmented Generation (RAG) പഠനം തുടർക്കൊണ്ടുപോകുന്നതിന്, നിങ്ങൾക്ക് സൃഷ്ടിക്കാവുന്നതുണ്ട്:

- നിങ്ങളുടെ ഇഷ്ടത്തിലുള്ള ഫ്രെയിംവർക് ഉപയോഗിച്ച് അപ്ലിക്കേഷനിന്റെ ഫ്രണ്ട്-എൻഡ് നിർമ്മിക്കുക

- LangChain അല്ലെങ്കിൽ Semantic Kernel പോലുള്ള ഒരു ഫ്രെയിംവർക്ക് ഉപയോഗിച്ച് നിങ്ങളുടെ അപ്ലിക്കേഷൻ പുനഃസൃഷ്ടിക്കുക.
