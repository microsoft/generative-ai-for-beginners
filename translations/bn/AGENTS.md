## প্রকল্পের পরিচিতি

এই রিপোজিটরি একটি ব্যাপক ২১-টিপাঠের কারিকুলাম সমৃদ্ধ যা জেনারেটিভ এআই এর মৌলিক বিষয়বস্ত এবং অ্যাপ্লিকেশন উন্নয়নের পাঠ দেয়। কোর্সটি নবীনদের জন্য ডিজাইন করা হয়েছে এবং প্রাথমিক ধারণা থেকে শুরু করে প্রোডাকশন-রেডি অ্যাপ্লিকেশন তৈরি পর্যন্ত সবকিছু অন্তর্ভুক্ত করে।

**প্রধান প্রযুক্তি:**
- পাইথন ৩.৯+ সহ লাইব্রেরিসমূহ: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- টাইপস্ক্রিপ্ট/জাভাস্ক্রিপ্ট সহ Node.js এবং লাইব্রেরিসমূহ: `openai` (Azure OpenAI v1 এন্ডপয়েন্ট + Responses API), `@azure-rest/ai-inference` (মাইক্রোসফট ফাউন্ড্রি মডেলস)
- Azure OpenAI সার্ভিস, OpenAI API, এবং Microsoft Foundry Models (GitHub Models জুলাই ২০২৬ শেষে অবসান হচ্ছে)
- Jupyter Notebooks ইন্টারেক্টিভ শেখার জন্য
- ডেভ কন্টেইনারস ধারাবাহিক উন্নয়নের পরিবেশের জন্য

**রিপোজিটরি কাঠামো:**
- ২১টি নম্বরকৃত লেসনের ডিরেক্টরি (০০-২১) যা README, কোড উদাহরণ, এবং অ্যাসাইনমেন্টগুলো অন্তর্ভুক্ত করে
- একাধিক বাস্তবায়ন: পাইথন, টাইপস্ক্রিপ্ট, এবং কখনও কখনও .NET উদাহরণ
- অনুবাদের ডিরেক্টরি ৪০+ ভাষা সংস্করণ সহ
- কেন্দ্রীভূত কনফিগারেশন `.env` ফাইল মাধ্যমে (টেমপ্লেট হিসেবে `.env.copy` ব্যবহার করুন)

## সেটআপ কমান্ডসমূহ

### প্রাথমিক রিপোজিটরি সেটআপ

```bash
# রেপোজিটরি ক্লোন করুন
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# পরিবেশ টেমপ্লেট কপি করুন
cp .env.copy .env
# আপনার API কী এবং এন্ডপয়েন্ট দিয়ে .env সম্পাদনা করুন
```

### পাইথন পরিবেশ সেটআপ

```bash
# ভার্চুয়াল পরিবেশ তৈরি করুন
python3 -m venv venv

# ভার্চুয়াল পরিবেশ সক্রিয় করুন
# ম্যাকওএস/লিনাক্স এ:
source venv/bin/activate
# উইন্ডোজ এ:
venv\Scripts\activate

# ডিপেন্ডেন্সি ইনস্টল করুন
pip install -r requirements.txt
```

### Node.js/TypeScript সেটআপ

```bash
# রুট-স্তরের নির্ভরশীলতাসমূহ ইনস্টল করুন (ডকুমেন্টেশন টুলিং এর জন্য)
npm install

# পৃথক পাঠ TypeScript উদাহরণগুলোর জন্য, নির্দিষ্ট পঠনে গমন করুন:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

## উন্নয়ন কার্যপ্রবাহ

### পরিবেশ পরিবর্তনশীলসমূহ

সব লেসনে API অ্যাক্সেসের জন্য `.env` ফাইলে পরিবেশ পরিবর্তনশীল সংজ্ঞায়িত করা হয়:

- `OPENAI_API_KEY` - OpenAI API এর জন্য
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry এর Azure OpenAI এর জন্য (Azure OpenAI Service এখন Microsoft Foundry এর অংশ: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI এন্ডপয়েন্ট URL (Foundry রিসোর্স এন্ডপয়েন্ট)
- `AZURE_OPENAI_DEPLOYMENT` - চ্যাট কমপ্লিশন মডেলের ডিপ্লয়মেন্ট নাম
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - এমবেডিং মডেলের ডিপ্লয়মেন্ট নাম
- `AZURE_OPENAI_API_VERSION` - API সংস্করণ (ডিফল্ট: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - হাগিং ফেস মডেলগুলোর জন্য
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry মডেলস এন্ডপয়েন্ট (মাল্টি-প্রোভাইডার মডেল ক্যাটালগ)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry Models API কী (`GITHUB_TOKEN` অবসান প্রতিস্থাপন)

## কোড স্টাইল নির্দেশিকা

### পাইথন

- পরিবেশ পরিবর্তনশীল ব্যবস্থাপনার জন্য `python-dotenv` ব্যবহার করুন
- API ইন্টারঅ্যাকশনের জন্য `openai` লাইব্রেরি ইমপোর্ট করুন
- লিন্টিং এর জন্য `pylint` ব্যবহার করুন (কিছু উদাহরণ সরলতার জন্য `# pylint: disable=all` অন্তর্ভুক্ত করে)
- PEP 8 নামকরণ নিয়ম অনুসরণ করুন
- API ক্রিডেনশিয়াল `.env` ফাইলে সংরক্ষণ করুন, কখনো কোডে নয়

### টাইপস্ক্রিপ্ট

- পরিবেশ পরিবর্তনশীলের জন্য `dotenv` প্যাকেজ ব্যবহার করুন
- প্রতি অ্যাপের জন্য `tsconfig.json` টাইপস্ক্রিপ্ট কনফিগারেশন
- Azure OpenAI এর জন্য `openai` প্যাকেজ ব্যবহার করুন (ক্লায়েন্টকে `/openai/v1/` এন্ডপয়েন্ট দেখান এবং `client.responses.create` কল করুন); Microsoft Foundry Models এর জন্য `@azure-rest/ai-inference` ব্যবহার করুন
- উন্নয়নের জন্য `nodemon` ব্যবহার করুন যা অটো-রিলোড সক্ষম
- চালানোর আগে বিল্ড করুন: `npm run build` তারপর `npm start`

### সাধারণ নিয়মাবলী

- কোড উদাহরণ সহজ এবং শিক্ষামূলক রাখুন
- প্রধান ধারণাগুলো ব্যাখ্যা করে মন্তব্য যোগ করুন
- প্রতিটি লেসনের কোড স্বতন্ত্র এবং চালানো যায় এমন হওয়া উচিত
- ধারাবাহিক নামকরণ ব্যবহার করুন: Azure OpenAI এর জন্য `aoai-` প্রিফিক্স, OpenAI API এর জন্য `oai-`, Microsoft Foundry Models এর জন্য `githubmodels-` (GitHub Models যুগের উত্তরাধিকার রক্ষিত প্রিফিক্স)

## ডকুমেন্টেশন নির্দেশিকা

### মার্কডাউন স্টাইল

- সব URL অবশ্যই `[text](../../url)` ফরম্যাটে রাখুন কোন অতিরিক্ত স্পেস ছাড়া
- আপেক্ষিক লিঙ্ক অবশ্যই `./` অথবা `../` দিয়ে শুরু হোক
- সব মাইক্রোসফট ডোমেইনের লিঙ্কে অবশ্যই ট্র্যাকিং আইডি অন্তর্ভুক্ত করুন: `?WT.mc_id=academic-105485-koreyst`
- দেশের-নির্দিষ্ট লোকেল URL এ ব্যবহার করবেন না (যেমন `/en-us/` এড়িয়ে চলুন)
- ইমেজগুলো `./images` ফোল্ডারে সংরক্ষিত এবং বর্ণনামূলক নাম থাকবে
- ফাইল নাম ইংরেজি অক্ষর, সংখ্যা, এবং ড্যাশ ব্যবহার করে লিখুন

- কি পরিবর্তন হয়েছে এবং কেন তা ব্যাখ্যা করুন
- সম্পর্কিত ইস্যু লিঙ্ক করুন
- কোড পরিবর্তনের জন্য কোন উদাহরণ পরীক্ষা করা হয়েছে তা উল্লেখ করুন
- অনুবাদ PR এর জন্য, সম্পূর্ণ অনুবাদের সব ফাইল অন্তর্ভুক্ত করুন

### অবদানের শর্তাবলী

- প্রথম PR এ স্বয়ংক্রিয়ভাবে Microsoft CLA সাইন ইন করুন
- পরিবর্তন করার আগে রিপোজিটরি আপনার অ্যাকাউন্টে ফর্ক করুন
- একটি যৌক্তিক পরিবর্তনের জন্য একটিমাত্র PR (অপ্রাসঙ্গিক ফিক্স একত্রিত করবেন না)
- সম্ভব হয়েছে PR গুলো ছোট এবং ফোকাসড রাখুন

## সাধারণ কার্যপ্রবাহ

### নতুন কোড উদাহরণ যোগ করা

১। প্রাসঙ্গিক লেসন ডিরেক্টরিতে যান

৪। প্রকৃত API ক্রিডেনশিয়াল দিয়ে পরীক্ষা করুন
৫। নতুন পরিবেশ পরিবর্তনশীল থাকলে লেসনের README তে ডকুমেন্ট করুন

### ডকুমেন্টেশন আপডেট

১। লেসন ডিরেক্টরির README.md সম্পাদনা করুন
২। মার্কডাউন নির্দেশিকা অনুসরণ করুন (ট্র্যাকিং ID, আপেক্ষিক লিঙ্ক)
৩। অনুবাদগুলোর আপডেট GitHub Actions দ্বারা পরিচালিত (ম্যানুয়ালি পরিবর্তন করবেন না)
৪। সব লিঙ্ক সঠিক আছে কিনা পরীক্ষা করুন

### ডেভ কন্টেইনার নিয়ে কাজ

১। রিপোজিটরিতে `.devcontainer/devcontainer.json` রয়েছে
২। পোস্ট-ক্রিয়েট স্ক্রিপ্ট স্বয়ংক্রিয়ভাবে পাইথন ডিপেন্ডেন্সি ইনস্টল করে
৩। পাইথন এবং জুপিটার এক্সটেনশন প্রি-কনফিগার করা
৪। পরিবেশটি `mcr.microsoft.com/devcontainers/universal:2.11.2` এর উপরে ভিত্তিক

## সমস্যা সমাধান

### সাধারণ সমস্যা

**পাইথন ইমপোর্ট এররস**:
- ভার্চুয়াল পরিবেশ সক্ষম হয়েছে কিনা নিশ্চিত করুন
- `pip install -r requirements.txt` চালান
- পাইথন ভার্সন ৩.৯+ কিনা নিশ্চিত করুন

**টাইপস্ক্রিপ্ট বিল্ড এররস**:
- নির্দিষ্ট অ্যাপ ডিরেক্টরিতে `npm install` চালান
- Node.js ভার্সন উপযুক্ত কিনা চেক করুন
- `node_modules` মুছে আবার ইনস্টল করুন প্রয়োজনে

**API Authentication এররস**:
- `.env` ফাইল আছে এবং সঠিক মান রয়েছে কিনা যাচাই করুন
- API কী বৈধ এবং মেয়াদোত্তীर्ण নয় কিনা পরীক্ষা করুন
- আপনার অঞ্চলের জন্য এন্ডপয়েন্ট URL সঠিক কিনা দেখুন

**পরিবেশ পরিবর্তনশীল মিসিং**:
- `.env.copy` থেকে `.env` কপি করুন
- আপনি যে লেসনে কাজ করছেন তার জন্য সব প্রয়োজনীয় মান পূরণ করুন
- `.env` আপডেট করার পর অ্যাপ্লিকেশন পুনরায় চালু করুন
