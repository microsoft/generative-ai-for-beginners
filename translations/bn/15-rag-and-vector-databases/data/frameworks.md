# নিউরাল নেটওয়ার্ক ফ্রেমওয়ার্ক

যেমন আমরা ইতিমধ্যেই শিখেছি, নিউরাল নেটওয়ার্ককে দক্ষতার সাথে প্রশিক্ষণ দেওয়ার জন্য আমাদের দুটি কাজ করতে হবে:

* টেন্সরগুলির উপর অপারেট করতে হবে, যেমন গুণ, যোগ এবং কিছু ফাংশন যেমন সিগময়েড বা সফটম্যাক্স গণনা করতে হবে
* সমস্ত এক্সপ্রেশনের গ্রেডিয়েন্ট গণনা করতে হবে, যাতে গ্রেডিয়েন্ট ডিসেন্ট অপ্টিমাইজেশন করা যায়

যদিও `numpy` লাইব্রেরি প্রথম অংশটি করতে পারে, আমাদের গ্রেডিয়েন্ট গণনা করার জন্য কিছু মেকানিজম দরকার। আমাদের ফ্রেমওয়ার্কে যা আমরা আগের সেকশনে তৈরি করেছি, আমাদের `backward` মেথডের ভিতরে সমস্ত ডেরিভেটিভ ফাংশন ম্যানুয়ালি প্রোগ্রাম করতে হয়েছিল, যা ব্যাকপ্রোপাগেশন করে। আদর্শভাবে, একটি ফ্রেমওয়ার্ক আমাদের যে কোনও এক্সপ্রেশনের গ্রেডিয়েন্ট গণনা করার সুযোগ দেওয়া উচিত যা আমরা সংজ্ঞায়িত করতে পারি।

আরেকটি গুরুত্বপূর্ণ বিষয় হল GPU বা অন্য কোনো বিশেষায়িত কম্পিউট ইউনিট, যেমন TPU, এ কম্পিউটেশন করতে সক্ষম হওয়া। ডিপ নিউরাল নেটওয়ার্ক প্রশিক্ষণ *অনেক* কম্পিউটেশন প্রয়োজন, এবং GPU-তে সেই কম্পিউটেশনগুলি প্যারালালাইজ করতে সক্ষম হওয়া খুবই গুরুত্বপূর্ণ।

> ✅ 'প্যারালালাইজ' শব্দটির অর্থ হল একাধিক ডিভাইসে কম্পিউটেশন বিতরণ করা।

বর্তমানে, দুটি সবচেয়ে জনপ্রিয় নিউরাল ফ্রেমওয়ার্ক হল: টেনসরফ্লো এবং পাইটর্চ। উভয়ই CPU এবং GPU উভয়ের টেন্সরগুলির সাথে কাজ করার জন্য একটি লো-লেভেল API প্রদান করে। লো-লেভেল API-এর উপরে, একটি উচ্চ-স্তরের API রয়েছে, যথাক্রমে কেরাস এবং পাইটর্চ লাইটনিং নামে।

লো-লেভেল API | টেনসরফ্লো | পাইটর্চ
--------------|-------------------------------------|--------------------------------
উচ্চ-স্তরের API | কেরাস | পাইটর্চ

**লো-লেভেল API** উভয় ফ্রেমওয়ার্কে আপনাকে তথাকথিত **কম্পিউটেশনাল গ্রাফ** তৈরি করতে দেয়। এই গ্রাফটি সংজ্ঞায়িত করে কিভাবে প্রদত্ত ইনপুট প্যারামিটার দিয়ে আউটপুট (সাধারণত লস ফাংশন) গণনা করতে হয় এবং GPU-তে কম্পিউটেশনের জন্য পুশ করা যেতে পারে, যদি এটি উপলব্ধ থাকে। এই কম্পিউটেশনাল গ্রাফটি পার্থক্য করার এবং গ্রেডিয়েন্ট গণনা করার জন্য ফাংশন রয়েছে, যা তখন মডেলের প্যারামিটার অপ্টিমাইজ করার জন্য ব্যবহার করা যেতে পারে।

**উচ্চ-স্তরের API** মূলত নিউরাল নেটওয়ার্ককে **লেয়ারের সিকোয়েন্স** হিসাবে বিবেচনা করে এবং বেশিরভাগ নিউরাল নেটওয়ার্ক তৈরি করা অনেক সহজ করে তোলে। মডেল প্রশিক্ষণ সাধারণত ডেটা প্রস্তুত করা এবং তারপর `fit` ফাংশন কল করার প্রয়োজন হয়।

উচ্চ-স্তরের API আপনাকে সাধারণ নিউরাল নেটওয়ার্কগুলি খুব দ্রুত তৈরি করতে দেয় অনেকগুলি বিশদ সম্পর্কে চিন্তা না করেই। একই সময়ে, লো-লেভেল API প্রশিক্ষণ প্রক্রিয়ার উপর অনেক বেশি নিয়ন্ত্রণ প্রদান করে, এবং তাই গবেষণায় অনেক ব্যবহার করা হয়, যখন আপনি নতুন নিউরাল নেটওয়ার্ক আর্কিটেকচারের সাথে কাজ করছেন।

এটি বোঝা গুরুত্বপূর্ণ যে আপনি উভয় API একসাথে ব্যবহার করতে পারেন, যেমন আপনি লো-লেভেল API ব্যবহার করে আপনার নিজস্ব নেটওয়ার্ক লেয়ার আর্কিটেকচার তৈরি করতে পারেন এবং তারপর এটি বৃহত্তর নেটওয়ার্কের ভিতরে ব্যবহার করতে পারেন যা উচ্চ-স্তরের API দিয়ে তৈরি এবং প্রশিক্ষিত হয়। অথবা আপনি লেয়ারের সিকোয়েন্স হিসাবে উচ্চ-স্তরের API ব্যবহার করে একটি নেটওয়ার্ক সংজ্ঞায়িত করতে পারেন এবং তারপর অপ্টিমাইজেশন করতে আপনার নিজস্ব লো-লেভেল প্রশিক্ষণ লুপ ব্যবহার করতে পারেন। উভয় API একই মৌলিক অন্তর্নিহিত ধারণাগুলি ব্যবহার করে এবং তারা একসাথে ভাল কাজ করার জন্য ডিজাইন করা হয়েছে।

## শেখা

এই কোর্সে, আমরা পাইটর্চ এবং টেনসরফ্লো উভয়ের জন্য বেশিরভাগ কন্টেন্ট অফার করি। আপনি আপনার পছন্দের ফ্রেমওয়ার্ক নির্বাচন করতে পারেন এবং শুধুমাত্র সংশ্লিষ্ট নোটবুকগুলি দেখতে পারেন। আপনি কোন ফ্রেমওয়ার্ক নির্বাচন করবেন তা নিশ্চিত না হলে, **পাইটর্চ বনাম টেনসরফ্লো** সম্পর্কে ইন্টারনেটে কিছু আলোচনা পড়ুন। আপনি আরও ভাল বোঝার জন্য উভয় ফ্রেমওয়ার্কের দিকে তাকাতে পারেন।

যেখানে সম্ভব, আমরা সরলতার জন্য উচ্চ-স্তরের API ব্যবহার করব। তবে, আমরা বিশ্বাস করি যে নিউরাল নেটওয়ার্কগুলি কীভাবে কাজ করে তা মাটি থেকে বোঝা গুরুত্বপূর্ণ, তাই শুরুতে আমরা লো-লেভেল API এবং টেন্সরের সাথে কাজ করে শুরু করি। তবে, আপনি যদি দ্রুত যেতে চান এবং এই বিশদগুলি শেখার জন্য অনেক সময় ব্যয় করতে না চান, আপনি সেগুলি এড়িয়ে যেতে পারেন এবং সরাসরি উচ্চ-স্তরের API নোটবুকগুলিতে যেতে পারেন।

## ✍️ অনুশীলন: ফ্রেমওয়ার্ক

নিচের নোটবুকগুলোতে আপনার শেখা চালিয়ে যান:

লো-লেভেল API | টেনসরফ্লো+কেরাস নোটবুক | পাইটর্চ
--------------|-------------------------------------|--------------------------------
উচ্চ-স্তরের API | কেরাস | *পাইটর্চ লাইটনিং*

ফ্রেমওয়ার্কগুলি আয়ত্ত করার পরে, ওভারফিটিং ধারণাটি পুনরায় পর্যালোচনা করা যাক।

# ওভারফিটিং

ওভারফিটিং মেশিন লার্নিং-এ অত্যন্ত গুরুত্বপূর্ণ একটি ধারণা, এবং এটি সঠিকভাবে বুঝতে পারা খুবই গুরুত্বপূর্ণ!

নিচের ৫টি বিন্দু (যা `x` দ্বারা প্রতিনিধিত্ব করা হয়েছে) আনুমানিক করার সমস্যা বিবেচনা করুন:

!লিনিয়ার | ওভারফিট
-------------------------|--------------------------
**লিনিয়ার মডেল, ২টি প্যারামিটার** | **নন-লিনিয়ার মডেল, ৭টি প্যারামিটার**
প্রশিক্ষণ ত্রুটি = ৫.৩ | প্রশিক্ষণ ত্রুটি = ০
ভ্যালিডেশন ত্রুটি = ৫.১ | ভ্যালিডেশন ত্রুটি = ২০

* বামে, আমরা একটি ভালো সরল রেখা আনুমানিক দেখতে পাই। কারণ প্যারামিটারগুলির সংখ্যা পর্যাপ্ত, মডেল বিন্দু বিতরণের ধারণাটি সঠিকভাবে পায়।
* ডানে, মডেলটি খুব শক্তিশালী। কারণ আমাদের মাত্র ৫টি বিন্দু আছে এবং মডেলের ৭টি প্যারামিটার আছে, এটি এমনভাবে সামঞ্জস্য করতে পারে যাতে সমস্ত বিন্দুর মধ্য দিয়ে যেতে পারে, প্রশিক্ষণের ত্রুটি ০ হয়। তবে, এটি মডেলকে ডেটার পিছনের সঠিক প্যাটার্ন বুঝতে বাধা দেয়, তাই ভ্যালিডেশন ত্রুটি খুব বেশি।

মডেলের সমৃদ্ধি (প্যারামিটার সংখ্যা) এবং প্রশিক্ষণ নমুনার সংখ্যার মধ্যে সঠিক ভারসাম্য বজায় রাখা খুবই গুরুত্বপূর্ণ।

## কেন ওভারফিটিং ঘটে

  * পর্যাপ্ত প্রশিক্ষণ ডেটা নেই
  * অত্যন্ত শক্তিশালী মডেল
  * ইনপুট ডেটায় খুব বেশি শব্দ

## কিভাবে ওভারফিটিং সনাক্ত করবেন

উপরে গ্রাফ থেকে আপনি দেখতে পাচ্ছেন, ওভারফিটিং একটি খুব কম প্রশিক্ষণ ত্রুটি এবং একটি উচ্চ ভ্যালিডেশন ত্রুটি দ্বারা সনাক্ত করা যেতে পারে। সাধারণত প্রশিক্ষণের সময় আমরা প্রশিক্ষণ এবং ভ্যালিডেশন উভয় ত্রুটিই কমতে দেখতে পাই, এবং তারপর এক পর্যায়ে ভ্যালিডেশন ত্রুটি কমা বন্ধ করতে পারে এবং বাড়তে শুরু করতে পারে। এটি ওভারফিটিং-এর একটি চিহ্ন হবে, এবং নির্দেশক যে আমাদের সম্ভবত এই পর্যায়ে প্রশিক্ষণ বন্ধ করা উচিত (অথবা অন্তত মডেলের একটি স্ন্যাপশট তৈরি করা উচিত)।

## কিভাবে ওভারফিটিং প্রতিরোধ করবেন

যদি আপনি দেখতে পান যে ওভারফিটিং ঘটছে, আপনি নিম্নলিখিতগুলির মধ্যে একটি করতে পারেন:

 * প্রশিক্ষণ ডেটার পরিমাণ বাড়ান
 * মডেলের জটিলতা কমান
 * কিছু নিয়মিতকরণ কৌশল ব্যবহার করুন, যেমন ড্রপআউট, যা আমরা পরে বিবেচনা করব।

## ওভারফিটিং এবং বায়াস-ভ্যারিয়েন্স ট্রেডঅফ

ওভারফিটিং আসলে পরিসংখ্যানের একটি আরও সাধারণ সমস্যা, যাকে বায়াস-ভ্যারিয়েন্স ট্রেডঅফ বলা হয়। যদি আমরা আমাদের মডেলে ত্রুটির সম্ভাব্য উৎসগুলি বিবেচনা করি, আমরা দুটি ধরনের ত্রুটি দেখতে পাই:

* **বায়াস ত্রুটি** আমাদের অ্যালগরিদম প্রশিক্ষণ ডেটার মধ্যে সম্পর্ক সঠিকভাবে ধরতে না পারার কারণে হয়। এটি আমাদের মডেল যথেষ্ট শক্তিশালী নয় (**আন্ডারফিটিং**) এই কারণে হতে পারে।
* **ভ্যারিয়েন্স ত্রুটি**, যা ইনপুট ডেটায় অর্থপূর্ণ সম্পর্কের পরিবর্তে শব্দ আনুমানিক করার কারণে হয় (**ওভারফিটিং**)।

প্রশিক্ষণের সময়, বায়াস ত্রুটি কমে যায় (যেহেতু আমাদের মডেল ডেটা আনুমানিক করতে শেখে), এবং ভ্যারিয়েন্স ত্রুটি বৃদ্ধি পায়। এটি গুরুত্বপূর্ণ যে প্রশিক্ষণ বন্ধ করা - হয় ম্যানুয়ালি (যখন আমরা ওভারফিটিং সনাক্ত করি) বা স্বয়ংক্রিয়ভাবে (নিয়মিতকরণ প্রবর্তন করে) - ওভারফিটিং প্রতিরোধ করার জন্য।

## উপসংহার

এই পাঠে, আপনি দুটি সবচেয়ে জনপ্রিয় AI ফ্রেমওয়ার্ক, টেনসরফ্লো এবং পাইটর্চ-এর বিভিন্ন API-এর মধ্যে পার্থক্য সম্পর্কে শিখেছেন। এছাড়াও, আপনি একটি খুব গুরুত্বপূর্ণ বিষয়, ওভারফিটিং সম্পর্কে শিখেছেন।

## পর্যালোচনা এবং স্ব-অধ্যয়ন

নিম্নলিখিত বিষয়গুলিতে কিছু গবেষণা করুন:

- টেনসরফ্লো
- পাইটর্চ
- ওভারফিটিং

নিজেকে নিম্নলিখিত প্রশ্নগুলি জিজ্ঞাসা করুন:

- টেনসরফ্লো এবং পাইটর্চের মধ্যে পার্থক্য কী?
- ওভারফিটিং এবং আন্ডারফিটিং-এর মধ্যে পার্থক্য কী?

## অ্যাসাইনমেন্ট

এই ল্যাবে, আপনাকে পাইটর্চ বা টেনসরফ্লো ব্যবহার করে একক- এবং মাল্টি-লেয়ারড সম্পূর্ণ সংযুক্ত নেটওয়ার্ক ব্যবহার করে দুটি শ্রেণীবিভাজন সমস্যা সমাধান করতে বলা হয়েছে।

**অস্বীকৃতি**:  
এই নথিটি AI অনুবাদ পরিষেবা [Co-op Translator](https://github.com/Azure/co-op-translator) ব্যবহার করে অনুবাদ করা হয়েছে। আমরা যথাসাধ্য সঠিকতার জন্য চেষ্টা করি, তবে দয়া করে সচেতন থাকুন যে স্বয়ংক্রিয় অনুবাদে ত্রুটি বা অসঙ্গতি থাকতে পারে। এর মূল ভাষায় থাকা নথিটিকেই প্রামাণিক উৎস হিসাবে বিবেচনা করা উচিত। গুরুত্বপূর্ণ তথ্যের জন্য, পেশাদার মানব অনুবাদ সুপারিশ করা হয়। এই অনুবাদের ব্যবহারের ফলে উদ্ভূত কোন ভুল বোঝাবুঝি বা ভুল ব্যাখ্যার জন্য আমরা দায়ী নই।
