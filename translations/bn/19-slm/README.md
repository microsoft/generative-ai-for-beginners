# শুরু করার জন্য ছোট ভাষার মডেলসমূহের পরিচিতি জেনারেটিভ AI-র জন্য
জেনারেটিভ AI হলো কৃত্রিম বুদ্ধিমত্তার একটি আকর্ষণীয় শাখা, যা নতুন কন্টেন্ট তৈরির ক্ষমতা সম্পন্ন সিস্টেম তৈরি করার উপর কেন্দ্রীভূত। এই কন্টেন্ট হতে পারে লেখা, ছবি, সঙ্গীত এমনকি পুরো ভার্চুয়াল পরিবেশ। জেনারেটিভ AI-এর সবচেয়ে উত্তেজনাপূর্ণ প্রয়োগগুলির একটি হলো ভাষার মডেলগুলোর ক্ষেত্রে।

## ছোট ভাষার মডেলগুলো কি?

ছোট ভাষার মডেল (SLM) হলো বড় ভাষার মডেল (LLM)-এর একটি সঙ্কুচিত রূপ, যা LLM-এর অনেক আর্কিটেকচারাল নীতি এবং কৌশল ব্যবহার করে, তবে উল্লেখযোগ্যভাবে কম প্রসেসিং ক্ষমতা ব্যবহার করে।

SLM হলো ভাষার মডেলগুলোর এক উপশ্রেণি যেগুলো মানুষের মত লেখনী তৈরি করতে সক্ষম। তাদের বড় সহপাঠীদের মতো যেমন GPT-4-এর তুলনায়, SLM গুলো বেশি সংক্ষিপ্ত এবং দক্ষ, যা কম্পিউটেশনাল সম্পদের সীমাবদ্ধতার মধ্যে ব্যবহার উপযোগী। যদিও ছোট, তারা বিভিন্ন কাজ সম্পাদন করতে পারে। সাধারণত, SLM তৈরি করা হয় LLM কে কম্প্রেস বা ডিস্টিল করে, যাতে মূল মডেলের কার্যকারিতা এবং ভাষাগত দক্ষতাগুলোর একটা বড় অংশ বজায় থাকে। এই মডেল সাইজ হ্রাস করার মাধ্যমে জটিলতা কমে যায়, যা স্মৃতি ব্যবহার এবং ক্যালকুলেশন সক্ষমতা উভয় ক্ষেত্রেই SLM কে বেশি দক্ষ করে তোলে। এই অপ্টিমাইজেশন সত্ত্বেও, SLM বিভিন্ন ধরনের প্রাকৃতিক ভাষা প্রক্রিয়াকরণ (NLP) কাজ করতে পারে:

- লেখা তৈরি: সামগ্রীগুলোকে পুনরায় সঙ্গতিপূর্ণ ও প্রসঙ্গানুযায়ী বর্ণনা করা।
- লেখনী সম্পূরক: প্রদত্ত প্রম্পট অনুযায়ী বাক্য পূরণ করা।
- অনুবাদ: একটি ভাষা থেকে অন্য ভাষায় লেখা রূপান্তর করা।
- সারসংক্ষেপ: দীর্ঘ লেখা সংক্ষিপ্ত, আরও সহজবোধ্য সারাংশে পরিণত করা।

যদিও তাদের বড় সংস্করণগুলোর তুলনায় কিছু পারফরম্যান্স বা বিষয়বস্তুর গভীরতায় প্রতিদান বা ছাড় দেওয়া যেতে পারে।

## ছোট ভাষার মডেলগুলি কীভাবে কাজ করে?
SLM গুলো প্রচুর পরিমাণে লেখা ডেটা দিয়ে প্রশিক্ষিত হয়। প্রশিক্ষণের সময়, তারা ভাষার প্যাটার্ন এবং গঠন শিখে, যা তাদের অভিধানগতভাবে সঠিক এবং প্রসঙ্গে গ্রহণযোগ্য লেখা তৈরি করতে সক্ষম করে। প্রশিক্ষণের প্রক্রিয়া হয়:

- ডেটা সংগ্রহ: বিভিন্ন উৎস থেকে বড় লেখা ডেটাসেট সংগ্রহ।
- প্রি-প্রসেসিং: ডেটা পরিষ্কার ও সংগঠিত করে প্রশিক্ষণের জন্য উপযুক্ত করা।
- প্রশিক্ষণ: মেশিন লার্নিং অ্যালগরিদম ব্যবহার করে মডেলকে লেখা বুঝতে ও তৈরি করতে শেখানো।
- ফাইন-টিউনিং: নির্দিষ্ট কাজের দক্ষতা বাড়ানোর জন্য মডেল সামঞ্জস্য করা।

SLM এর উন্নয়ন এমন মডেলের প্রয়োজনের সাথে সংগতি পূরণ করে যা সীমিত সম্পদ পরিবেশে যেমন মোবাইল ডিভাইস বা এজ কম্পিউটিং প্ল্যাটফর্মে কার্যকরী হতে পারে, যেখানে বড় LLM গুলো ভারী সম্পদ ব্যবহার ও অনুপযুক্ত হতে পারে। দক্ষতার ওপর ফোকাস করে, SLM পারফরম্যান্স ও প্রবেশগম্যতার মধ্যে ভারসাম্য বজায় রাখে, যা বিভিন্ন ক্ষেত্রে বিস্তৃত প্রয়োগের সুযোগ তৈরি করে।

## শিক্ষার উদ্দেশ্যসমূহ

এই পাঠে, আমরা আশা করি SLM-এর জ্ঞান পরিচয় করিয়ে দেব এবং Microsoft Phi-3 এর সাথে মিলিয়ে লেখা, ভিশন এবং MoE-এর বিভিন্ন পরিস্থিতি শিখব।

এই পাঠের শেষে, আপনি নিম্নলিখিত প্রশ্নের উত্তর দিতে সক্ষম হবেন:

- SLM কী?
- SLM এবং LLM-এর মধ্যে পার্থক্য কী?
- Microsoft Phi-3/3.5 পরিবার কী?
- Microsoft Phi-3/3.5 পরিবার দিয়ে কিভাবে ইনফারেন্স চালাবেন?

প্রস্তুত? চলুন শুরু করা যাক।

## বড় ভাষার মডেল (LLM) এবং ছোট ভাষার মডেল (SLM)-এর পার্থক্যসমূহ

LLM এবং SLM উভয়ই সম্ভাব্য মেশিন লার্নিং-এর মূলনীতি অনুসারে নির্মিত, যেখানে তাদের স্থাপত্য নকশা, প্রশিক্ষণ পদ্ধতি, ডেটা উৎপাদনের প্রক্রিয়া এবং মডেল মূল্যায়নের কৌশল অনুরূপ। তবে, কিছু মূল পার্থক্য এই দুই প্রকার মডেলের মধ্যে বিদ্যমান।

## ছোট ভাষার মডেলের প্রয়োগসমূহ

SLM-এর বহু ধরণের প্রয়োগ রয়েছে, যেমন:

- চ্যাটবট: গ্রাহক সমর্থন প্রদান ও ব্যবহারকারীদের সাথে কথোপকথনে জড়িত হওয়া।
- কন্টেন্ট সৃষ্টি: লেখকদের আইডিয়া জেনারেট বা পুরা আর্টিকেল খসড়া তৈরিতে সহায়তা করা।
- শিক্ষা: শিক্ষার্থীদের লেখার কাজ বা নতুন ভাষা শেখার ক্ষেত্রে সাহায্য করা।
- প্রবেশগম্যতা: প্রতিবন্ধী ব্যক্তিদের জন্য টুলস তৈরি, যেমন টেক্সট-টু-স্পিচ সিস্টেম।

**সাইজ**
  
LLM এবং SLM-এর মধ্যে প্রধান পার্থক্য হলো তাদের মডেলের আকার। LLM যেমন ChatGPT (GPT-4) প্রায় 1.76 ট্রিলিয়ন প্যারামিটার নিয়ে গঠিত হতে পারে, যখন ওপেন সোর্স SLM যেমন Mistral 7B প্রায় ৭ বিলিয়ন প্যারামিটার নিয়ে ডিজাইন করা। এই পার্থক্য প্রধানত মডেল স্থাপত্য ও প্রশিক্ষণ প্রক্রিয়ার পার্থক্যের কারণে। উদাহরণস্বরূপ, ChatGPT একটি এনকোডার-ডিকোডার ফ্রেমওয়ার্কের ভিতরে সেলফ-অ্যাটেনশন ব্যবহৃত হয়, যেখানে Mistral 7B স্লাইডিং উইন্ডো অ্যাটেনশন ব্যবহার করে, যা শুধুমাত্র ডিকোডার মডেলে আরও দক্ষ প্রশিক্ষণ দেয়। এই স্থাপত্যগত পার্থক্য মডেলের জটিলতা ও পারফরম্যান্সের ওপর গভীর প্রভাব ফেলে।

**বোঝাপড়া**

SLM সাধারণত নির্দিষ্ট ক্ষেত্রে পারফরম্যান্সের জন্য উন্নত করা হয়, যা তাদের অত্যন্ত বিশেষায়িত করে তোলে কিন্তু একাধিক বিষয়ের বিস্তৃত প্রসঙ্গ বোঝার ক্ষমতায় সীমাবদ্ধ করে। বিপরীতে, LLM মানুষ সদৃশ বুদ্ধিমত্তা অনুকরণ করার চেষ্টা করে বিস্তৃত মাত্রায়। ব্যাপক ও বৈচিত্র্যময় ডেটাসেটে প্রশিক্ষিত, LLM বিভিন্ন ক্ষেত্রে ভালো পারফরম্যান্স দেয়, যা তাদের বহুমুখী ও অভিযোজ্য করে তোলে। ফলস্বরূপ, LLM প্রাকৃতিক ভাষা প্রক্রিয়াকরণ ও প্রোগ্রামিং মত বহুমুখী নিচের কাজের জন্য উপযুক্ত।

**কম্পিউটিং**

LLM এর প্রশিক্ষণ ও মোতায়েন সম্পদ-নিবিষ্ট প্রক্রিয়া, যা বড় স্কেল GPU ক্লাস্টারের ব্যবহার প্রয়োজন। উদাহরণস্বরূপ, ChatGPT-এর মত মডেল শূন্য থেকে প্রশিক্ষণের জন্য হাজার হাজার GPU দীর্ঘ সময় ধরে চালাতে হতে পারে। অপরদিকে, SLM তাদের ছোট পাওয়ার কম্পিউটেশনাল প্রয়োজনীয়তার জন্য আরো প্রবেশযোগ্য। Mistral 7B মত মডেল মধ্যম ক্ষমতার GPU সহ স্থানীয় মেশিনে প্রশিক্ষিত এবং চালানো যায়, যদিও প্রশিক্ষণে ঘন্টা নয়েক GPU দরকার হয়।

**পক্ষপাত**

পক্ষপাত LLM-এ একটি পরিচিত সমস্যা, যা প্রশিক্ষণ ডেটার প্রকৃতির কারণে ঘটে। এই মডেলগুলো সাধারণত ইন্টারনেট থেকে সংগ্রহ করা কাঁচা, খোলা ডেটা ব্যবহার করে, যা কিছু গ্রুপকে অপর্যাপ্ত প্রতিনিধিত্ব করে অথবা ভুল উপস্থাপন করে, ভুল চিহ্নিতকরণ প্রবর্তন করে অথবা উপভাষা, ভৌগোলিক পরিবর্তন এবং ব্যাকরণগত নিয়ম দ্বারা প্রভাবিত ভাষাগত পক্ষপাত প্রদর্শন করে। তদুপরি, LLM আর্কিটেকচারের জটিলতা সঠিক ফাইন-টিউনিং ছাড়া পক্ষপাতকে বাড়িয়ে তুলতে পারে। অন্যদিকে, SLM সীমিত এবং নির্দিষ্ট ডেটাসেটে প্রশিক্ষিত হওয়ার কারণে এই পক্ষপাতগুলো কম প্রবণ, যদিও তারা সম্পূর্ণ মুক্ত নয়।

**ইনফারেন্স**

SLM-এর কম আকার তাদের একটি বড় সুবিধা দেয় ইনফারেন্স গতিতে, স্থানীয় হার্ডওয়্যারে দক্ষভাবে আউটপুট তৈরি করতে পারে বিস্তৃত সমান্তরাল প্রসেসিং এর প্রয়োজন ছাড়াই। বিপরীতভাবে, LLM তাদের আকার ও জটিলতার কারণে গ্রহণযোগ্য ইনফারেন্স সময়ে পৌঁছাতে মহান পরিমাণে সমান্তরাল কম্পিউটেশনাল সম্পদ প্রয়োজন। একাধিক ব্যবহারকারী উপস্থিত হলে LLM এর সাড়া সময় আরও ধীর হয়, বিশেষ করে স্কেলে মোতায়েনের সময়।

সংক্ষেপে, যদিও LLM ও SLM উভয়েই মেশিন লার্নিং-এর ভিত্তিতে নির্মিত, তারা মডেলের আকার, সম্পদ চাহিদা, প্রসঙ্গ বোঝা, পক্ষপাত প্রবণতা ও ইনফারেন্স গতিতে উল্লেখযোগ্য পার্থক্য রয়েছে। এই পার্থক্যগুলো তাদের আলাদা ব্যবহারের জন্য যোগ্যতা নির্ধারণ করে, LLM বেশি বহুমুখী কিন্তু সম্পদ-গুরুত্বপূর্ণ, আর SLM নির্দিষ্ট ক্ষেত্রের দক্ষতা দিয়ে কম কম্পিউটেশনের দাবি রাখে।

***নোট: এই পাঠে, আমরা Microsoft Phi-3 / 3.5 উদাহরণ হিসেবে ব্যবহার করে SLM পরিচয় করিয়ে দেব।***

## Phi-3 / Phi-3.5 পরিবার পরিচিতি

### Phi-3 / 3.5 নির্দেশনা (Instruct)

প্রধানত লেখা তৈরি, চ্যাট সম্পূরক এবং বিষয়বস্তু তথ্য আহরণের জন্য।

**Phi-3-মিনি**

3.8B প্যারামিটার বিশিষ্ট ভাষার মডেল Microsoft Foundry, Hugging Face এবং Ollama-তে পাওয়া যায়। Phi-3 মডেলগুলি মূলত সমান বা বড় আকারের ভাষা মডেলের তুলনায় গুরুত্বপূর্ণ বেঞ্চমার্কে উল্লেখযোগ্য পারফরম্যান্স দেখায় (নীচে বেঞ্চমার্ক নম্বর দেখুন, বড় সংখ্যা ভালো)। Phi-3-মিনি তার আকারের দ্বিগুণের তুলনায় মডেলগুলোকে পরাস্ত করে, এবং Phi-3-ছোট ও Phi-3-মধ্যম বড় মডেলগুলোর সাথে, যেমন GPT-3.5, তুলনীয় বা এগিয়ে থাকে।

**Phi-3-ছোট ও মধ্যম**

Phi-3-মধ্যম 14B প্যারামিটার নিয়ে এই ধারা বজায় রেখেছে এবং Gemini 1.0 Pro কে ছাড়িয়ে গেছে।

**Phi-3.5-মিনি**

এটি Phi-3-মিনির একটি আপগ্রেড হিসেবে ভাবা যেতে পারে। প্যারামিটার অপরিবর্তিত থাকলেও, এটি বহু ভাষার সহায়তা উন্নত করে (২০+ ভাষা সহায়তা করে: আরবি, চীনা, চেক, ড্যানিশ, ডাচ, ইংরেজি, ফিনিশ, ফরাসি, জার্মান, হিব্রু, হাঙ্গেরিয়ান, ইতালিয়ান, জাপানি, কোরিয়ান, নরওয়েজিয়ান, পোলিশ, পর্তুগিজ, রাশিয়ান, স্প্যানিশ, সুইডিশ, থাই, টার্কিশ, ইউক্রেনীয়) এবং দীর্ঘ প্রসঙ্গের জন্য শক্তিশালী সমর্থন যোগ করে।

Phi-3.5-মিনি 3.8B প্যারামিটার নিয়ে একই আকারের ভাষার মডেলের চেয়েও ভালো এবং তার দ্বিগুণ আকারের মডেলের সমতুল্য।

### Phi-3 / 3.5 ভিশন

**Phi-3-ভিশন**

Phi-3-ভিশন, মাত্র 4.2B প্যারামিটার সহ, এই প্রবণতা বজায় রেখেছে এবং Claude-3 Haiku ও Gemini 1.0 Pro V এর মত বড় মডেলগুলিকে সাধারণ ভিজ্যুয়াল যুক্তি, OCR, টেবিল ও চার্ট বোঝার কাজগুলিতে পরাস্ত করে।

### Phi-3.5-MoE

***মিশ্র বিশেষজ্ঞরা (MoE)*** কম কম্পিউট শক্তিতে মডেলকে প্রিট্রেন করতে সক্ষম করে, যার মানে আপনি একই কম্পিউট বাজেট দিয়ে মডেল বা ডেটাসেটের আকার নাটকীয়ভাবে বাড়াতে পারেন। বিশেষ করে, একটি MoE মডেল তার ঘন বিশেষজ্ঞ মডেলের সমতুল্য মান দ্রুত অর্জন করতে সক্ষম।

**Hugging Face Transformer**
Hugging Face Transformers একটি শক্তিশালী লাইব্রেরি যা প্রাকৃতিক ভাষা প্রক্রিয়াকরণ (NLP) এবং অন্যান্য মেশিন লার্নিং কাজের জন্য ডিজাইন করা। এখানে এর কিছু মূল দিক তুলে ধরা হলো:

১. **প্রি-ট্রেইনড মডেল:** হাজার হাজার প্রি-ট্রেইনড মডেল সরবরাহ করে যা বিভিন্ন কাজের জন্য ব্যবহার করা যায় যেমন টেক্সট ক্লাসিফিকেশন, নামকরণ সত্তা সনাক্তকরণ, প্রশ্নোত্তর, সারাংশ, অনুবাদ এবং টেক্সট জেনারেশন।

২. **ফ্রেমওয়ার্ক ইন্টারঅপারেবিলিটি:** এই লাইব্রেরি বিভিন্ন ডিপ লার্নিং ফ্রেমওয়ার্ক যেমন PyTorch, TensorFlow, এবং JAX সাপোর্ট করে। ফলে একটি ফ্রেমওয়ার্কে মডেল ট্রেইন করে অন্যত্র ব্যবহার করা যায়।

৩. **মাল্টিমোডাল সক্ষমতা:** NLP ছাড়াও, Hugging Face Transformers কম্পিউটার ভিশন (যেমন ইমেজ ক্লাসিফিকেশন, অবজেক্ট ডিটেকশন) এবং অডিও প্রোসেসিং (যেমন স্পিচ রেকগনিশন, অডিও ক্লাসিফিকেশন) কাজেও সমর্থন দেয়।

৪. **সুবিধাজনক ব্যবহার:** সহজে মডেল ডাউনলোড এবং ফাইন-টিউন করার API এবং সরঞ্জাম প্রদান করে, যা নবাগত ও বিশেষজ্ঞ উভয়ের জন্যই অ্যাক্সেসযোগ্য।

৫. **কমিউনিটি এবং রিসোর্স:** Hugging Face এর একটি সক্রিয় কমিউনিটি ও বিস্তৃত ডকুমেন্টেশন, টিউটোরিয়াল এবং গাইড রয়েছে, যা ব্যবহারকারীদের শুরু করতে এবং সর্বোচ্চ সুবিধা নিতে সাহায্য করে।
[অফিশিয়াল ডকুমেন্টেশন](https://huggingface.co/docs/transformers/index?WT.mc_id=academic-105485-koreyst) অথবা তাদের [GitHub রেপোজিটরি](https://github.com/huggingface/transformers?WT.mc_id=academic-105485-koreyst) দেখুন।

এটি সবচেয়ে প্রচলিত পদ্ধতি, তবে এটি GPU অ্যাক্সিলারেশনও প্রয়োজন। কারণ Vision এবং MoE এর মতো সিনারিও অনেক গণনা প্রয়োজন, যা CPU তে কুয়ান্টাইজ না করলে ধীর হবে।

**ONNX Runtime for GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) একটি ক্রস-প্ল্যাটফর্ম ইনফারেন্স এবং ট্রেনিং মেশিন লার্নিং অ্যাক্সিলারেটর। ONNX Runtime for Generative AI (GENAI) হলো একটি শক্তিশালী টুল যা বিভিন্ন প্ল্যাটফর্মে জেনারেটিভ AI মডেল দ্রুত ও দক্ষতায় চালাতে সাহায্য করে।

## ONNX Runtime কি?
ONNX Runtime একটি ওপেন সোর্স প্রকল্প যা মেশিন লার্নিং মডেলের উচ্চ কার্যকারিতা ইনফারেন্স সক্ষম করে। এটি Open Neural Network Exchange (ONNX) ফরম্যাটের মডেল সাপোর্ট করে, যা মেশিন লার্নিং মডেল উপস্থাপনের স্ট্যান্ডার্ড। ONNX Runtime ইনফারেন্স দ্রুত গ্রাহক অভিজ্ঞতা প্রদান এবং খরচ কমাতে সাহায্য করে, PyTorch, TensorFlow/Keras মত ডিপ লার্নিং ফ্রেমওয়ার্কের মডেল এবং scikit-learn, LightGBM, XGBoost মত ক্লাসিক্যাল মেশিন লার্নিং লাইব্রেরিগুলোকেও সাপোর্ট করে। ONNX Runtime বিভিন্ন হার্ডওয়্যার, ড্রাইভার, এবং অপারেটিং সিস্টেমের সাথে সামঞ্জস্যপূর্ণ এবং যেখানে সম্ভব হার্ডওয়্যার অ্যাক্সিলারেটর ব্যবহার করে গঠনের অপ্টিমাইজেশন ও ট্রান্সফর্মের মাধ্যমে সর্বোত্তম পারফরম্যান্স দেয়।

## জেনারেটিভ AI কি?
জেনারেটিভ AI হলো এমন AI সিস্টেম যা নতুন কন্টেন্ট যেমন টেক্সট, ছবি বা সঙ্গীত তৈরি করে, যা তারা প্রশিক্ষণ ডেটার ওপর ভিত্তি করে করে থাকে। যেমন GPT-3 মত ভাষা মডেল এবং Stable Diffusion মত ইমেজ জেনারেশন মডেল। ONNX Runtime for GenAI লাইব্রেরি ONNX মডেলের জন্য জেনারেটিভ AI লুপ প্রদান করে, যার মধ্যে রয়েছে ONNX Runtime ইনফারেন্স, লজিটস প্রক্রিয়াকরণ, সার্চ ও স্যাম্পলিং, এবং KV ক্যাশ ম্যানেজমেন্ট।

## ONNX Runtime for GENAI
ONNX Runtime for GENAI ONNX Runtime এর সক্ষমতাকে বাড়িয়ে জেনারেটিভ AI মডেল সাপোর্ট করে। এর কিছু মূল বৈশিষ্ট্য হলো:

- **বিস্তৃত প্ল্যাটফর্ম সাপোর্ট:** Windows, Linux, macOS, Android, ও iOS সহ বিভিন্ন প্ল্যাটফর্মে কার্যক্রম করে।
- **মডেল সাপোর্ট:** জনপ্রিয় অনেক জেনারেটিভ AI মডেল যেমন LLaMA, GPT-Neo, BLOOM ইত্যাদি সাপোর্ট করে।
- **পারফরম্যান্স অপ্টিমাইজেশন:** NVIDIA GPU, AMD GPU সহ বিভিন্ন হার্ডওয়্যার অ্যাক্সিলারেটর জন্য অপ্টিমাইজেশন অন্তর্ভুক্ত।
- **সুবিধাজনক ব্যবহার:** সহজ ইন্টিগ্রেশনের জন্য API প্রদান করে, যা কোড কম দিয়ে টেক্সট, ছবি এবং অন্যান্য কন্টেন্ট তৈরি করতে সক্ষম করে।
- ব্যবহারকারীরা উচ্চ স্তরের generate() মেথড কল করতে পারে, অথবা মডেলের প্রতিটি পুনরাবৃত্তি লুপে চালিয়ে একটি করে টোকেন জেনারেট করতে পারে এবং প্রয়োজনে লুপের ভিতরে জেনারেশন প্যারামিটার পরিবর্তন করতে পারে।
- ONNX Runtime গ্রিডি/বিম সার্চ এবং TopP, TopK স্যাম্পলিং সাপোর্ট করে টোকেন তালিকা তৈরি ও লজিটস প্রোসেসিং যেমন রিপিটিশন পেনাল্টির সুবিধা দেয়। এছাড়াও সহজে কাস্টম স্কোরিং যোগ করা যায়।

## শুরু করা যাক
ONNX Runtime for GENAI শুরু করতে নিচের ধাপগুলো অনুসরণ করুন:

### ONNX Runtime ইনস্টল করুন:
```Python
pip install onnxruntime
```
### জেনারেটিভ AI এক্সটেনশন ইনস্টল করুন:
```Python
pip install onnxruntime-genai
```

### মডেল চালান: Python এ একটি সহজ উদাহরণ:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### ডেমো: ONNX Runtime GenAI ব্যবহার করে Phi-3.5-Vision কল করা


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
