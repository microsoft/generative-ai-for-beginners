# উন্নত বৈশিষ্ট্য এবং উন্নতির রোডম্যাপ

এই ডকুমেন্টটি জেনারেটিভ AI ফর বিগিনার্স কারিকুলামের জন্য সুপারিশকৃত উন্নয়ন এবং উন্নতির রূপরেখা প্রদান করে, একটি বিস্তৃত কোড পর্যালোচনা এবং শিল্পের সেরা অনুশীলনের বিশ্লেষণের ভিত্তিতে।

## কার্যনির্বাহী সারাংশ

কোডবেসকে নিরাপত্তা, কোড গুণমান এবং শিক্ষামূলক কার্যকারিতার জন্য বিশ্লেষণ করা হয়েছে। এই ডকুমেন্টটি তাৎক্ষণিক সমাধান, নিকটবর্তী উন্নয়ন, এবং ভবিষ্যতের উন্নতির জন্য সুপারিশ প্রদান করে।

## ৪. API আধুনিকীকরণ

### ৪.১ অব্যবহৃত API প্যাটার্ন (মাইগ্রেশন সম্পন্ন)

সমস্ত পাইথন এবং টাইপস্ক্রিপ্ট **চ্যাট** নমুনাগুলো Chat Completions API থেকে **Responses API** এ স্থানান্তরিত হয়েছে (`client.responses.create(...)` → `response.output_text`)।

| পুরানো প্যাটার্ন | নতুন প্যাটার্ন | অবস্থা |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (চ্যাট) | `OpenAI(base_url="<endpoint>/openai/v1/")` (Responses API) | সম্পন্ন |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | সম্পন্ন |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (টাইপস্ক্রিপ্ট) | `openai` প্যাকেজের `client.responses.create()` → `response.output_text` | সম্পন্ন |
| `df.append()` (pandas) | `pd.concat()` | সম্পন্ন |

> **দ্রষ্টব্য:** Microsoft Foundry Models নমুনাগুলো যা `azure-ai-inference` / `@azure-rest/ai-inference` SDK (`client.complete()`) ব্যবহার করে, তারা Model Inference API তে রয়েছে, যা Responses API সমর্থন করে না। `AzureOpenAI()` ইচ্ছাকৃতভাবে অবশিষ্ট রাখা হয়েছে যেখানে তা এখনও বৈধ (এম্বেডিং এবং ইমেজ জেনারেশন)।

### ৪.২ প্রদর্শনের জন্য নতুন API বৈশিষ্ট্য

১. **স্ট্রাকচরড আউটপুট** (OpenAI)
   - JSON মোড
   - কঠোর স্কিমাসহ ফাংশন কলিং

২. **ভিশন ক্ষমতা**

৩. **Responses API বিল্ট-ইন টুলস** (পুরাতন Assistants API এর পরিবর্তে)
   - কোড ইন্টারপ্রিটার
   - ফাইল সার্চ
   - ওয়েব সার্চ এবং কাস্টম টুলস

## ৫. ইन्फ্রাস্ট্রাকচার উন্নতি

### ৫.১ CI/CD উন্নতি

[`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml) এ বাস্তবায়িত: Python linting/formatting (Ruff + Black) রক্ষণাবেক্ষণকৃত `shared/` ইউটিলিটিজ মডিউলে **প্রযোজ্য** এবং কারিকুলামের অন্যান্য স্থানে **সুপারিশমূলক**, সাথে জাভাস্ক্রিপ্ট/টাইপস্ক্রিপ্টের জন্য এসএলইনট অতিক্রম। প্রতীকী বেসলাইন ছিল:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### ৫.২ নিরাপত্তা স্ক্যানিং

[`.github/workflows/security.yml`](../../../.github/workflows/security.yml) এ বাস্তবায়িত: Python এবং JavaScript/TypeScript জন্য CodeQL বিশ্লেষণ (পুশ, পুল রিকোয়েস্ট এবং সাপ্তাহিক সময়সূচীতে) এবং পুল রিকোয়েস্টে ডিপেনডেন্সি রিভিউ। প্রতীকী বেসলাইন ছিল:

```yaml
# .github/workflows/security.yml
name: Security Scan

on: [push, pull_request]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: javascript, python
      - uses: github/codeql-action/analyze@v3

  dependency-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
```

```json

      "version": "3.11"
    },
    "ghcr.io/devcontainers/features/node:1": {
      "version": "20"
    }
  },
  "customizations": {
    "vscode": {
      "extensions": [
        "ms-python.python",
        "ms-python.vscode-pylance",
        "ms-toolsai.jupyter",
        "dbaeumer.vscode-eslint",
        "esbenp.prettier-vscode",
        "github.copilot"
      ],
      "settings": {
        "python.formatting.provider": "black",
        "editor.formatOnSave": true
      }
    }
  },
  "postCreateCommand": "pip install -e .[dev] && npm install"
```
