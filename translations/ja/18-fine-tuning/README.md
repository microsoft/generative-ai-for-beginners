# LLMのファインチューニング

大規模言語モデルを使って生成AIアプリケーションを構築する際には、新たな課題が伴います。重要な問題は、モデルが与えられたユーザーのリクエストに対して生成するコンテンツの応答品質（正確性と関連性）を確保することです。これまでのレッスンでは、既存のモデルへの_プロンプト入力を修正する_ことで問題を解決しようとするプロンプトエンジニアリングやリトリーバル強化生成といった手法を紹介しました。

本日のレッスンでは、<strong>ファインチューニング</strong>という三つ目の手法について解説します。これは追加データで_モデルそのものを再学習_させることで課題に対処しようとするものです。詳細を見ていきましょう。

## 言語モデルのファインチューニングとは？

定義では、大規模言語モデルはインターネットを含む多様なソースから収集された大量のテキストで_事前学習_されています。これまでのレッスンで学んだように、ユーザーの質問（プロンプト）に対する応答の質を改善するには、_プロンプトエンジニアリング_や_リトリーバル強化生成_のような技術が必要です。

人気のあるプロンプトエンジニアリング技法の一つは、応答に何を期待するかをモデルにより詳しく指示することで、_指示_（明示的ガイダンス）を与えたり、_いくつかの例を提供_（暗示的ガイダンス）したりすることです。これは_few-shot学習_と呼ばれますが、2つの制約があります：

- モデルのトークン制限により例の数が制限され、効果が制限される。
- モデルのトークンコストにより、すべてのプロンプトに例を追加するのが高額になり柔軟性が制限される。

ファインチューニングは、事前学習済みモデルを特定のタスクでの性能向上のために新たなデータで再学習させる機械学習の一般的手法です。言語モデルの場合、特定のタスクや適用領域に合わせて_厳選された例のセット_で事前学習済みモデルをファインチューニングし、そのタスクや領域に対してより正確で関連性の高い<strong>カスタムモデル</strong>を作成できます。ファインチューニングの副次的な利点は、few-shot学習で必要な例の数を減らすことができ、トークン使用量や関連コストを削減できることです。

## いつ、なぜモデルをファインチューニングすべきか？

_ここで_ファインチューニングと言うと、オリジナルの学習データセットに含まれていなかった<strong>新しいデータを追加して</strong>再学習を行う<strong>教師あり</strong>ファインチューニングを指します。これは、元のデータを使い異なるハイパーパラメータで再学習する非教師ありファインチューニングとは異なります。

覚えておくべき重要な点は、ファインチューニングは望ましい結果を得るために一定レベルの専門知識が必要な高度な技術であるということです。不適切に行うと期待通りの改善が得られず、対象の領域におけるモデル性能を悪化させることさえあります。

したがって、言語モデルのファインチューニングの「方法」を学ぶ前に、「なぜ」この方法を選ぶべきか、「いつ」ファインチューニングを開始すべきかを理解する必要があります。まず次の質問を自問してください：

- <strong>ユースケース</strong>：ファインチューニングの_ユースケース_は何か？現在の事前学習モデルのどの点を改善したいか？
- <strong>代替手段</strong>：望ましい結果を達成するために_他の技術_を試したか？それらを比較の基準とする。
  - プロンプトエンジニアリング：関連するプロンプト応答の例を使ったfew-shotプロンプト技法を試し、応答品質を評価する。
  - リトリーバル強化生成：データ検索で得たクエリ結果でプロンプトを強化し、応答品質を評価する。
- <strong>コスト</strong>：ファインチューニングのコストを把握しているか？
  - チューニング可能性 - 事前学習モデルはファインチューニング可能か？
  - 労力 - トレーニングデータの準備、モデル評価と改良にかかる労力。
  - 計算資源 - ファインチューニング作業の実行やファインチューニングモデルの展開に必要なコンピュートリソース。
  - データ - ファインチューニングの影響を与える十分な品質の例の入手。
- <strong>利点</strong>：ファインチューニングの利点を確認したか？
  - 品質 - ファインチューニングモデルはベースラインより優れているか？
  - コスト - プロンプトを簡素化することでトークン使用量を削減できるか？
  - 拡張性 - ベースモデルを新しいドメインに再利用できるか？

これらの質問に答えることで、ファインチューニングがユースケースに適しているかどうか判断できます。理想的には、利点がコストを上回る場合にのみ有効な方法です。実施の決定後は、事前学習済みモデルを_どのように_ファインチューニングできるかを考え始めます。

意思決定プロセスをより深く知りたい方は、[To fine-tune or not to fine-tune](https://www.youtube.com/watch?v=0Jo-z-MFxJs)をご覧ください。

## 事前学習モデルをどうやってファインチューニングするか？

事前学習済みモデルをファインチューニングするには、次が必要です：

- ファインチューニング対象の事前学習済みモデル
- ファインチューニング用データセット
- ファインチューニング作業を行うトレーニング環境
- ファインチューニング済みモデルを展開するホスティング環境
