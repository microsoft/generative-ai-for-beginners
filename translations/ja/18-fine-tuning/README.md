# LLMの微調整

大規模言語モデルを使用して生成AIアプリケーションを構築することには新しい課題が伴います。重要な問題は、特定のユーザーリクエストに対してモデルが生成するコンテンツの応答品質（正確性と関連性）を確保することです。前回のレッスンでは、既存のモデルへのプロンプト入力を修正することで問題を解決しようとするプロンプトエンジニアリングや検索強化生成といった技術について説明しました。

今日のレッスンでは、第三の技術である**微調整**について説明します。この技術は、追加のデータでモデル自体を再訓練することで課題に取り組むものです。詳細を見ていきましょう。

## 学習目標

このレッスンでは、事前訓練された言語モデルの微調整の概念を紹介し、このアプローチの利点と課題を探り、生成AIモデルの性能を向上させるための微調整の使用方法とタイミングについてのガイドを提供します。

- 事前訓練されたモデルをどのように微調整できますか？
- 微調整の限界は何ですか？

## 言語モデルの微調整とは？

定義上、大規模言語モデルはインターネットを含む多様なソースから得られた大量のテキストで事前訓練されています。前回のレッスンで学んだように、ユーザーの質問（「プロンプト」）へのモデルの応答の質を向上させるためには、プロンプトエンジニアリングや検索強化生成のような技術が必要です。

人気のあるプロンプトエンジニアリング技術は、モデルに応答で期待されることを指示（明示的なガイダンス）を提供するか、いくつかの例を示す（暗黙的なガイダンス）ことでより多くのガイダンスを与えることを含みます。これは「少数ショット学習」と呼ばれますが、二つの制限があります：

- モデルのトークン制限が、与えられる例の数を制限し、効果を制限する可能性があります。
- モデルのトークンコストが、各プロンプトに例を追加することを高価にし、柔軟性を制限する可能性があります。

微調整は、事前訓練されたモデルを取り、新しいデータで再訓練して特定のタスクの性能を向上させる機械学習システムで一般的な実践です。言語モデルのコンテキストでは、特定のタスクやアプリケーションドメインのために選定された例のセットで事前訓練されたモデルを微調整して、特定のタスクやドメインに対してより正確で関連性のある**カスタムモデル**を作成できます。微調整の副次的な利点は、少数ショット学習に必要な例の数を減らし、トークンの使用と関連コストを削減できることです。

## モデルを微調整するのはいつ、なぜ？

このコンテキストでは、微調整について話すとき、元の訓練データセットの一部ではなかった新しいデータを追加して再訓練する**監督付き**微調整を指します。これは、元のデータでモデルを再訓練するが異なるハイパーパラメータを使用する非監督付き微調整アプローチとは異なります。

覚えておくべき重要なことは、微調整は望ましい結果を得るためにある程度の専門知識が必要な高度な技術であるということです。誤って行うと、期待される改善が得られない可能性があり、ターゲットとするドメインに対するモデルの性能が低下する可能性もあります。

したがって、言語モデルを「どのように」微調整するかを学ぶ前に、「なぜ」このルートを取るべきか、「いつ」微調整のプロセスを開始するかを知る必要があります。まず、これらの質問を自問してみてください：

- **ユースケース**: 微調整のためのあなたの_ユースケース_は何ですか？現在の事前訓練モデルのどの側面を改善したいですか？
- **代替案**: 望ましい結果を達成するために_他の技術_を試しましたか？それらを比較のためのベースラインとして使用してください。
  - プロンプトエンジニアリング: 関連するプロンプト応答の例を用いた少数ショットプロンプトの技術を試してください。応答の質を評価します。
  - 検索強化生成: データを検索して取得したクエリ結果でプロンプトを強化してみてください。応答の質を評価します。
- **コスト**: 微調整のコストを特定しましたか？
  - 調整可能性 - 事前訓練モデルが微調整可能か？
  - 努力 - 訓練データの準備、モデルの評価と改良のための努力。
  - 計算 - 微調整ジョブの実行と微調整モデルのデプロイのための計算。
  - データ - 微調整の影響を与える十分な質の例へのアクセス。
- **利点**: 微調整の利点を確認しましたか？
  - 質 - 微調整されたモデルがベースラインを上回ったか？
  - コスト - プロンプトを簡素化することでトークン使用量を削減するか？
  - 拡張性 - ベースモデルを新しいドメインに再利用できるか？

これらの質問に答えることで、微調整があなたのユースケースにとって正しいアプローチかどうかを判断できるはずです。理想的には、利点がコストを上回る場合のみアプローチが有効です。進めることを決定したら、事前訓練モデルをどのように微調整できるかを考える時です。

意思決定プロセスについてもっと洞察を得たいですか？[微調整するかどうかの決定](https://www.youtube.com/watch?v=0Jo-z-MFxJs)を見てください。

## 事前訓練モデルをどのように微調整できますか？

事前訓練モデルを微調整するためには、次のものが必要です：

- 微調整する事前訓練モデル
- 微調整に使用するデータセット
- 微調整ジョブを実行する訓練環境
- 微調整モデルをデプロイするホスティング環境
