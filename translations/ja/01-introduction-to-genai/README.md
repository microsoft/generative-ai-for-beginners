# 生成AIと大規模言語モデルの紹介

_(上の画像をクリックして、このレッスンのビデオをご覧ください)_

生成AIは、テキスト、画像、その他の種類のコンテンツを生成することができる人工知能です。この技術が素晴らしいのは、AIを民主化する点であり、誰でも自然言語で書かれた一文のような簡単なテキストプロンプトで利用できることです。JavaやSQLのような言語を学ぶ必要はなく、自分の言葉を使って望むことを述べるだけで、AIモデルから提案が得られます。この技術の応用と影響は非常に大きく、報告書を書いたり理解したり、アプリケーションを作成したりすることが、ほんの数秒で可能です。

このカリキュラムでは、私たちのスタートアップが教育の世界で新しいシナリオを解き放つために生成AIを活用する方法と、その応用による社会的影響と技術的制限に関連する避けられない課題に対処する方法を探ります。

## シナリオ: 私たちの教育スタートアップ

生成人工知能 (AI) は、かつて不可能と思われていた限界を押し広げるAI技術の頂点を表しています。生成AIモデルにはいくつかの能力と応用がありますが、このカリキュラムでは架空のスタートアップを通じて教育を革新する方法を探ります。このスタートアップを_私たちのスタートアップ_と呼びます。私たちのスタートアップは教育分野で活動しており、その野心的な使命は

> _学習のアクセシビリティを向上させ、世界規模で公平な教育へのアクセスを確保し、学習者のニーズに応じた個別の学習体験を提供することです_。

私たちのスタートアップチームは、現代の最も強力なツールの一つである大規模言語モデル (LLM) を活用せずにはこの目標を達成できないことを認識しています。

生成AIは、今日の学習と教育の方法を革命的に変えることが期待されています。学生は24時間体制で膨大な情報と例を提供する仮想教師を利用でき、教師は革新的なツールを活用して学生を評価しフィードバックを与えることができます。

## 生成AIはどのようにして誕生したのか？

最近の生成AIモデルの発表によって生まれた驚異的な_ハイプ_にもかかわらず、この技術は数十年にわたって開発されており、最初の研究は60年代にまで遡ります。今では、AIは人間の認知能力を持ち、例えば[OpenAI ChatGPT](https://openai.com/chatgpt)や[Bing Chat](https://www.microsoft.com/edge/features/bing-chat?WT.mc_id=academic-105485-koreyst)のように、GPTモデルを使用してウェブ検索のBing会話を行うことができます。

少し振り返ると、AIの最初のプロトタイプはタイプライター式のチャットボットであり、専門家のグループから抽出された知識ベースに依存してコンピュータに表現されていました。知識ベースの回答は、入力テキストに現れるキーワードによってトリガーされました。しかし、すぐにタイプライター式チャットボットを使用するアプローチがうまくスケールしないことが明らかになりました。

### 統計的アプローチによるAI: 機械学習

90年代には、テキスト分析に統計的アプローチを適用することで転機が訪れました。これにより、データからパターンを学習する新しいアルゴリズム、つまり機械学習が開発され、明示的にプログラムされることなく機械が人間の言語理解をシミュレートすることができるようになりました。このアプローチにより、統計モデルはテキストとラベルのペアに基づいて訓練され、モデルは未知の入力テキストをメッセージの意図を表す事前定義されたラベルで分類することができるようになりました。

### ニューラルネットワークと現代の仮想アシスタント

近年、大量のデータとより複雑な計算を処理できるハードウェアの技術進化により、AIの研究が進展し、ニューラルネットワークまたはディープラーニングアルゴリズムと呼ばれる高度な機械学習アルゴリズムの開発が促進されました。

ニューラルネットワーク（特にリカレントニューラルネットワーク - RNN）は自然言語処理を大幅に向上させ、文中の単語のコンテキストを重視してテキストの意味をより意味のある方法で表現することを可能にしました。

これが新世紀の最初の10年間に誕生した仮想アシスタントを支える技術であり、人間の言語を解釈しニーズを特定し、事前定義されたスクリプトで回答したり第三者サービスを利用したりしてそれを満たす行動を取ることに非常に熟練しています。

### 現代、生成AI

これが今日の生成AIに到達した方法であり、ディープラーニングのサブセットと見ることができます。

AI分野での数十年にわたる研究の結果、新しいモデルアーキテクチャである_Transformer_がRNNの限界を克服し、より長いテキストシーケンスを入力として受け取ることができるようになりました。Transformersは注意機構に基づいており、モデルが受け取る入力に異なる重みを与え、テキストシーケンスの順序に関係なく最も関連性のある情報が集中している部分に「より注意を払う」ことを可能にします。

最近の生成AIモデルのほとんどは、テキスト入力と出力を扱うため、大規模言語モデル (LLM) としても知られており、実際にこのアーキテクチャに基づいています。これらのモデルは、本、記事、ウェブサイトなどの多様な情報源からの膨大な量のラベルなしデータで訓練されており、幅広いタスクに適応し、創造性のあるように見える文法的に正しいテキストを生成することができる点が興味深いです。したがって、これらのモデルは機械が入力テキストを「理解」する能力を驚異的に向上させただけでなく、人間の言語で独自の応答を生成する能力も可能にしました。

## 大規模言語モデルはどのように機能するのか？

次の章ではさまざまな種類の生成AIモデルを探りますが、今は大規模言語モデルがどのように機能するかを見てみましょう。特にOpenAI GPT（Generative Pre-trained Transformer）モデルに焦点を当てます。

- **トークナイザー、テキストから数字へ**: 大規模言語モデルはテキストを入力として受け取り、テキストを出力として生成します。しかし、統計モデルであるため、テキストシーケンスよりも数字での作業が得意です。そのため、モデルへのすべての入力はコアモデルに使用される前にトークナイザーによって処理されます。トークンは可変数の文字からなるテキストのチャンクであり、トークナイザーの主なタスクは入力をトークンの配列に分割することです。そして、各トークンはトークンインデックスとマッピングされ、元のテキストチャンクの整数エンコーディングとなります。

- **出力トークンの予測**: 最大nトークンを入力として受け取るモデル（モデルによって最大nは異なる）は、出力として1つのトークンを予測することができます。このトークンは次の反復の入力に組み込まれ、拡張ウィンドウパターンでユーザー体験を向上させ、回答として1つ（または複数）の文を得ることができます。これにより、ChatGPTを使用したことがある場合、時々文の途中で止まるように見える理由が説明されます。

- **選択プロセス、確率分布**: 出力トークンは現在のテキストシーケンスの後に発生する確率に基づいてモデルによって選ばれます。これは、モデルが訓練に基づいてすべての可能な「次のトークン」の確率分布を予測するためです。しかし、必ずしも結果として得られる分布から最も高い確率のトークンが選ばれるわけではありません。この選択にはランダム性が追加され、モデルが非決定論的に動作するようにし、同じ入力に対して同じ出力を得ることはありません。このランダム性の度合いは、創造的思考のプロセスをシミュレートするために追加され、温度というモデルパラメータを使用して調整できます。

## 私たちのスタートアップは大規模言語モデルをどのように活用できるか？

大規模言語モデルの内部動作について理解が深まったところで、私たちのビジネスシナリオに目を向けて、最も一般的なタスクの実用的な例を見てみましょう。大規模言語モデルの主な能力は、_自然言語で書かれたテキスト入力から始めて、ゼロからテキストを生成すること_です。

しかし、どのようなテキスト入力と出力なのでしょうか？
大規模言語モデルの入力はプロンプトと呼ばれ、出力は完了と呼ばれます。完了という用語は、現在の入力を完了するための次のトークンを生成するモデルのメカニズムを指します。プロンプトとは何か、そしてモデルから最大限の成果を得るためにどのように設計するかを詳しく探りますが、今はプロンプトが以下を含む可能性があることを述べておきます:

- モデルから期待される出力の種類を指定する**指示**。この指示には時々いくつかの例や追加のデータが含まれることがあります。

  1. 記事、本、製品レビューなどの要約と、非構造化データからの洞察の抽出。
    
  2. 記事、エッセイ、課題などの創造的なアイデアとデザイン。
      
- エージェントとの会話形式で尋ねられる**質問**。

- 暗黙的に書くための補助を求める**テキストの完成**のチャンク。

- 特定のタスクを実行するコードの生成を求めるコメントとともに、**コード**のチャンク。

上記の例は非常にシンプルであり、大規模言語モデルの能力の包括的なデモンストレーションを意図しているわけではありません。教育の文脈を含めて生成AIを使用する可能性を示すことを目的としています。

また、生成AIモデルの出力は完璧ではなく、モデルの創造性が逆効果をもたらし、人間のユーザーが現実の歪曲と解釈できる言葉の組み合わせや、攻撃的なものになることがあります。生成AIは知的ではありません - より包括的な知性の定義には批判的かつ創造的な推論や感情的知性が含まれます; 決定論的ではなく、信頼性がありません。誤った引用、コンテンツ、声明が正確な情報と組み合わされ、説得力のある自信を持って提示されることがあります。次のレッスンでは、これらの制限に対処し、それを緩和するためにできることを見ていきます。

## 知識チェック

大規模言語モデルについて何が正しいですか？

1. 毎回同じ応答が得られる。
1. 完璧に物事を行い、数を追加したり、動作するコードを生成したりするのが得意。
1. 同じプロンプトを使用しても応答が変わることがあります。また、テキストやコードの初稿を提供するのが得意ですが、結果を改善する必要があります。

A: 3、大規模言語モデルは非決定論的であり、応答は変わりますが、温度設定でその変動を制御できます。また、完璧に物事を行うことを期待すべきではなく、あなたのために重荷を軽減するために存在しており、しばしば改善が必要な良い最初の試みを提供します。
