# 貢獻

本專案歡迎貢獻和建議。大多數的貢獻需要您同意一份貢獻者許可協議 (CLA)，聲明您有權利，並實際授予我們使用您貢獻的權利。詳情請訪問 <https://cla.microsoft.com>。

> 重要：在翻譯此資料庫中的文本時，請確保不使用機器翻譯。我們將通過社群驗證翻譯，因此請僅在您精通的語言中自願進行翻譯。

當您提交拉取請求時，CLA-bot 會自動判斷您是否需要提供 CLA 並適當地裝飾 PR（例如，標籤、評論）。只需按照機器人提供的指示操作即可。您只需在所有使用我們 CLA 的資料庫中執行一次。

## 錯字、問題、錯誤和貢獻

每當您提交任何更改到 Generative AI for Beginners 資料庫時，請遵循以下建議。

* 在進行修改之前，始終將資料庫分叉到您自己的帳戶
* 不要將多個更改合併到一個拉取請求中。例如，使用單獨的 PR 提交任何錯誤修復和文件更新
* 如果您的拉取請求顯示合併衝突，請確保在進行修改之前更新本地 main 以鏡像主資料庫中的內容
* 如果您提交翻譯，請為所有翻譯的文件創建一個 PR，因為我們不接受內容的部分翻譯
* 如果您提交錯字或文件修正，您可以在適當的地方將修改合併到單個 PR 中

## 撰寫的一般指導

- 確保所有的 URL 都用方括號包裹，後面緊跟著沒有多餘空格的括號 `[](../..)`。
- 確保任何相對鏈接（即資料庫中其他文件和文件夾的鏈接）以 `./` 開頭，指向當前工作目錄中的文件或文件夾，或 `../`，指向父工作目錄中的文件或文件夾。
- 確保任何相對鏈接（即資料庫中其他文件和文件夾的鏈接）在末尾有跟蹤 ID（即 `?` 或 `&` 然後是 `wt.mc_id=` 或 `WT.mc_id=`）。
- 確保來自以下域名的任何 URL _github.com, microsoft.com, visualstudio.com, aka.ms, 和 azure.com_ 在末尾有跟蹤 ID（即 `?` 或 `&` 然後是 `wt.mc_id=` 或 `WT.mc_id=`）。
- 確保您的鏈接中沒有特定國家的語言區域（即 `/en-us/` 或 `/en/`）。
- 確保所有圖像都存儲在 `./images` 文件夾中。
- 確保圖像名稱具有描述性，使用英文字母、數字和短橫。

### 檢查損壞的相對路徑

此工作流程確保您文件中的任何相對路徑都能正常運作。此資料庫已部署到 GitHub 頁面，因此在輸入將所有內容連接在一起的鏈接時，您需要非常小心，以免將任何人引導到錯誤的地方。

為了確保您的鏈接正常運作，只需使用 VS code 檢查即可。

例如，當您將滑鼠懸停在文件中的任何鏈接上時，系統會提示您按 **ctrl + click** 跟隨該鏈接

![VS code 跟隨鏈接截圖](../../translated_images/vscode-follow-link.85520ab6a1237adcf01cc9cd8c228ce7b32ae685a034250bd5109e2682b9dfca.tw.png)

如果您點擊一個鏈接但在本地無法運作，那麼它肯定會觸發工作流程並且在 GitHub 上無法運作。

要解決此問題，請嘗試在 VS code 的幫助下輸入鏈接。

當您輸入 `./` 或 `../` 時，VS code 會根據您輸入的內容提示您從可用選項中進行選擇。

![VS code 選擇相對路徑截圖](../../translated_images/vscode-select-relative-path.3804eb73c3a9e5f2d345e3d3288f8173a9e584254d0e505d8bcbc6461dbf1f6c.tw.png)

通過點擊所需的文件或文件夾來跟隨路徑，您就可以確保您的路徑沒有損壞。

一旦您添加了正確的相對路徑，保存並推送您的更改，工作流程將再次觸發以驗證您的更改。如果您通過檢查，那麼您就可以繼續了。

### 檢查路徑是否有跟蹤

此工作流程確保任何相對路徑都有跟蹤。此資料庫已部署到 GitHub 頁面，因此我們需要跟蹤不同文件和文件夾之間的移動。

為了確保您的相對路徑中有跟蹤，只需檢查路徑末尾的以下文字 `?wt.mc_id=`。如果它被附加到您的相對路徑中，您將通過此檢查。

如果沒有，您可能會收到以下錯誤。

![GitHub 檢查路徑缺少跟蹤評論截圖](../../translated_images/github-check-paths-missing-tracking-comment.880d4afe03e898ffadeebe0f61f7fdea7525c25238bead9fecabc81a0a83b1c0.tw.png)

要解決此問題，請嘗試打開工作流程突出顯示的文件路徑，並在相對路徑的末尾添加跟蹤 ID。

一旦您添加了跟蹤 ID，保存並推送您的更改，工作流程將再次觸發以驗證您的更改。如果您通過檢查，那麼您就可以繼續了。

### 檢查 URL 是否有跟蹤

此工作流程確保任何網頁 URL 都有跟蹤。此資料庫對所有人開放，因此您需要確保跟蹤訪問以了解流量的來源。

為了確保您的 URL 中有跟蹤，只需檢查 URL 末尾的以下文字 `?wt.mc_id=`。如果它被附加到您的 URL 中，您將通過此檢查。

如果沒有，您可能會收到以下錯誤。

![GitHub 檢查 URL 缺少跟蹤評論截圖](../../translated_images/github-check-urls-missing-tracking-comment.1bd00d20b24a1e2e3179e59e1bd7d44f16637a1bb1ab265562565251166841ef.tw.png)

要解決此問題，請嘗試打開工作流程突出顯示的文件路徑，並在 URL 的末尾添加跟蹤 ID。

一旦您添加了跟蹤 ID，保存並推送您的更改，工作流程將再次觸發以驗證您的更改。如果您通過檢查，那麼您就可以繼續了。

### 檢查 URL 是否沒有區域

此工作流程確保任何網頁 URL 中沒有特定國家的語言區域。此資料庫對全球的每個人都開放，因此您需要確保不在 URL 中包含您國家的語言區域。

為了確保您的 URL 中沒有國家語言區域，只需檢查 URL 中的以下文字 `/en-us/` 或 `/en/` 或任何其他語言區域。如果它不存在於您的 URL 中，您將通過此檢查。

如果沒有，您可能會收到以下錯誤。

![GitHub 檢查國家語言區域評論截圖](../../translated_images/github-check-country-locale-comment.2f4fe93228161dee6ec8210f3d6ccc66af6864f6b178b8d96f30818498fba72a.tw.png)

要解決此問題，請嘗試打開工作流程突出顯示的文件路徑，並從 URL 中刪除國家語言區域。

一旦您刪除了國家語言區域，保存並推送您的更改，工作流程將再次觸發以驗證您的更改。如果您通過檢查，那麼您就可以繼續了。
