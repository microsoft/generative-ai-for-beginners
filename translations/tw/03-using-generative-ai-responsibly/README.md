# 負責任地使用生成式人工智慧

> _點擊上方圖片觀看本課程影片_

生成式人工智慧令人著迷，但你需要考慮如何負責任地使用它。你需要考慮如何確保輸出是公平的、不會造成傷害等。本章旨在為你提供相關背景資訊、應考慮的事項，以及如何採取積極措施改善你的AI使用方式。

## 介紹

本課程將涵蓋：

- 為何在構建生成式AI應用程式時應優先考慮負責任的AI。
- 負責任的AI核心原則及其與生成式AI的關係。
- 如何透過策略和工具將這些負責任的AI原則付諸實踐。

## 學習目標

完成本課程後，你將了解：

- 在構建生成式AI應用程式時負責任的AI的重要性。
- 何時在構建生成式AI應用程式時思考並應用負責任的AI核心原則。
- 你可用來實踐負責任AI概念的工具和策略。

## 負責任的AI原則

生成式AI的興奮度前所未有地高漲。這種興奮為這個領域帶來了大量的新開發者、關注和資金。雖然這對任何希望使用生成式AI構建產品和公司的人來說都是非常積極的，但我們也必須負責任地推進。

在整個課程中，我們專注於構建我們的初創企業和AI教育產品。我們將使用負責任的AI原則：公平性、包容性、可靠性/安全性、安全性與隱私、透明性和問責性。通過這些原則，我們將探討它們如何與我們在產品中使用生成式AI相關。

## 為什麼應優先考慮負責任的AI

在構建產品時，採取以人為本的方法，考慮用戶的最佳利益，將導致最佳結果。

生成式AI的獨特之處在於其為用戶創造有用答案、資訊、指導和內容的能力。這可以在沒有許多手動步驟的情況下完成，從而導致非常令人印象深刻的結果。沒有適當的計劃和策略，它也可能不幸地導致對用戶、產品和整個社會的某些有害結果。

讓我們來看看一些（但不是全部）這些潛在的有害結果：

### 幻覺

幻覺是用來描述當LLM產生的內容完全無意義或根據其他資訊來源明知是錯誤的情況。

例如，我們為初創企業構建了一個功能，允許學生向模型提出歷史問題。一名學生問了問題 `Who was the sole survivor of Titanic?`

這是一個非常自信且詳盡的答案。不幸的是，這是錯誤的。即使進行最少量的研究，也會發現鐵達尼號災難有不止一位生還者。對於剛開始研究這個主題的學生來說，這個答案可能足夠具有說服力而不被質疑並被視為事實。這可能導致AI系統不可靠並對我們初創企業的聲譽產生負面影響。

隨著每個給定LLM的迭代，我們看到在減少幻覺方面的性能改進。即使有這種改進，作為應用程式構建者和使用者，我們仍然需要保持對這些限制的認識。

### 有害內容

我們在前面的部分中討論了當LLM產生錯誤或無意義的回應時。另一個我們需要注意的風險是當模型回應有害內容時。

有害內容可以定義為：

- 提供自我傷害或傷害某些群體的指示或鼓勵。
- 仇恨或貶低的內容。
- 指導策劃任何形式的攻擊或暴力行為。
- 提供如何查找非法內容或進行非法行為的指示。
- 顯示露骨的性內容。

對於我們的初創企業，我們希望確保我們擁有合適的工具和策略，以防止此類內容被學生看到。

### 缺乏公平性

公平性被定義為“確保AI系統不帶偏見和歧視，並且公平地對待每個人。”在生成式AI的世界中，我們希望確保模型的輸出不會加強對邊緣化群體的排他性世界觀。

這些類型的輸出不僅對為用戶創造積極的產品體驗具有破壞性，還會造成進一步的社會傷害。作為應用程式構建者，我們在使用生成式AI構建解決方案時，應始終考慮廣泛且多樣化的用戶基礎。

## 如何負責任地使用生成式AI

現在我們已經確定了負責任的生成式AI的重要性，讓我們看看可以採取的4個步驟，以負責任地構建我們的AI解決方案：

### 衡量潛在危害

在軟體測試中，我們測試用戶在應用程式上的預期行為。同樣，測試用戶最有可能使用的多樣化提示集是衡量潛在危害的好方法。

由於我們的初創企業正在構建教育產品，準備一份與教育相關的提示清單會很好。這可以涵蓋某一主題、歷史事實以及有關學生生活的提示。

### 減少潛在危害

現在是時候找到方法來防止或限制模型及其回應可能造成的潛在危害了。我們可以從四個不同層面來看待這一點：

- **模型**。為合適的使用案例選擇合適的模型。較大且更複雜的模型如GPT-4在應用於較小且更具體的使用案例時，可能會帶來更大的有害內容風險。使用你的訓練資料進行微調也可以降低有害內容的風險。

- **安全系統**。安全系統是指在提供模型的平台上幫助減少危害的一組工具和配置。例如，Azure OpenAI服務上的內容過濾系統。系統還應檢測越獄攻擊和不想要的活動，如來自機器人的請求。

- **元提示**。元提示和基礎是我們可以根據某些行為和資訊指導或限制模型的方法。這可以使用系統輸入來定義模型的某些限制。此外，提供更相關於系統範圍或領域的輸出。

它還可以使用檢索增強生成（RAG）等技術來使模型僅從選擇的可信來源中提取資訊。本課程後面有一節課程專門講述[構建搜索應用程式](../08-building-search-applications/README.md?WT.mc_id=academic-105485-koreyst)

- **用戶體驗**。最後一層是用戶通過我們應用程式的介面以某種方式直接與模型互動。在這種方式中，我們可以設計UI/UX來限制用戶可以發送給模型的輸入類型，以及顯示給用戶的文本或圖像。在部署AI應用程式時，我們還必須透明地告知我們的生成式AI應用程式可以和不能做什麼。

我們有一整節課程專門講述[設計AI應用程式的用戶體驗](../12-designing-ux-for-ai-applications/README.md?WT.mc_id=academic-105485-koreyst)

- **評估模型**。與LLM合作可能具有挑戰性，因為我們不總是能控制模型訓練的資料。無論如何，我們應始終評估模型的性能和輸出。測量模型的準確性、相似性、基礎性和輸出的相關性仍然很重要。這有助於為利益相關者和用戶提供透明度和信任。

### 操作負責任的生成式AI解決方案

圍繞你的AI應用程式構建運營實踐是最後階段。這包括與我們初創企業的其他部門（如法律和安全）合作，以確保我們遵守所有監管政策。在啟動之前，我們還希望圍繞交付、處理事件和回滾建立計劃，以防止對我們用戶造成任何傷害。

## 工具

雖然開發負責任AI解決方案的工作看起來很多，但這是值得付出努力的工作。隨著生成式AI領域的增長，更多幫助開發者有效將責任融入工作流程的工具將會成熟。例如，[Azure AI內容安全](https://learn.microsoft.com/azure/ai-services/content-safety/overview?WT.mc_id=academic-105485-koreyst)可以通過API請求幫助檢測有害內容和圖像。
