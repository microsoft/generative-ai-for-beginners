# יסודות הנדסת פרומפטים

[![יסודות הנדסת פרומפטים](../../../translated_images/he/04-lesson-banner.a2c90deba7fedacd.webp)](https://youtu.be/GElCu2kUlRs?si=qrXsBvXnCW12epb8)

## מבוא
מודול זה מכסה מושגים וטכניקות חיוניות ליצירת פרומפטים יעילים במודלים של בינה מלאכותית גנרטיבית. האופן שבו אתם כותבים את הפרומפט למודל שפה גדול (LLM) גם הוא חשוב. פרומפט שנבנה בקפידה יכול להשיג איכות תגובה טובה יותר. אך מה בדיוק מתכוונים במונחים כמו _פרומפט_ ו־_הנדסת פרומפטים_? ואיך משפרים את קלט הפרומפט ששלחתי ל־LLM? שאלות אלו ונושאים נוספים ננסה לענות עליהם בפרק זה ובעוד פרק.

_בינה מלאכותית גנרטיבית_ מסוגלת ליצור תוכן חדש (למשל, טקסט, תמונות, קול, קוד וכו') בתגובה לבקשות משתמשים. היא משיגה זאת בעזרת _מודלים גדולים של שפה_ כמו סדרת GPT של OpenAI ("טרנספורמר מאומן מראש גנרטיבי") המאומנים לשימוש בשפה טבעית ובקוד.

כעת המשתמשים יכולים לקיים אינטראקציה עם מודלים אלה באמצעות שיטות מוכרות כמו צ׳אט, ללא צורך במומחיות טכנית או אימון. המודלים הם _מבוססי פרומפטים_ - המשתמשים שולחים קלט טקסטואלי (פרומפט) ומקבלים בחזרה תגובת בינה מלאכותית (השלמה). לאחר מכן ניתן "לדבר עם הבינה המלאכותית" באינטרקציות מרובות סבבים, ולשפר את הפרומפט עד שהתשובה מתאימה לציפיותיהם.

"הפרומפטים" הופכים כיום לממשק _תכנותי_ ראשי לאפליקציות בינה מלאכותית גנרטיבית, המספרות למודלים מה לעשות ומשפיעות על איכות התגובות המתקבלות. "הנדסת פרומפטים" היא תחום מחקר מתפתח במהירות המתמקד ב_עיצוב ואופטימיזציה_ של פרומפטים כדי לספק תגובות עקביות ואיכותיות בקנה מידה.

## מטרות הלמידה

בשיעור הזה נלמד מהי הנדסת פרומפטים, מדוע היא חשובה, ואיך נוכל ליצור פרומפטים יעילים יותר עבור מודל ומטרה אפליקטיבית מוגדרים. נבין מושגים מרכזיים ושיטות העבודה המומלצות בהנדסת פרומפטים - ונכיר סביבת "חול" אינטראקטיבית של Jupyter Notebooks שבה נוכל לראות את המושגים מיושמים על דוגמאות אמיתיות.

בסוף השיעור נוכל:

1. להסביר מהי הנדסת פרומפטים ולמה היא חשובה.
2. לתאר את רכיבי הפרומפט ואופן השימוש בהם.
3. ללמוד שיטות עבודה מומלצות וטכניקות בהנדסת פרומפטים.
4. ליישם את הטכניקות שנלמדו בדוגמאות אמיתיות, תוך שימוש בנקודת קצה של OpenAI.

## מונחים מרכזיים

הנדסת פרומפטים: תהליך עיצוב וטיוב הקלטים כדי להנחות מודלים של בינה מלאכותית לייצר פלט רצוי.  
טוקניזציה: תהליך המרת הטקסט ליחידות קטנות יותר, הנקראות טוקנים, שהמודל יכול להבין ולעבד.  
מודלי שפה גדולים מכווני הוראות: מודלים גדולים של שפה (LLMs) שעברו כוונון עדין עם הוראות ספציפיות לשיפור דיוק ורלוונטיות התגובה.

## סביבת חול למידה

הנדסת פרומפטים כיום היא יותר אומנות מאשר מדע. הדרך הטובה ביותר לשפר את האינטואיציה שלנו בה היא _לתרגל יותר_ ולאמץ גישה של ניסוי וטעיה שמשלבת מומחיות בתחום היישום עם שיטות מומלצות ואופטימיזציות ספציפיות למודל.

מחברת Jupyter המלווית לשיעור זה מספקת סביבת _חול_ בה תוכלו לנסות את מה שלמדתם - תוך כדי התקדמות או כחלק מאתגר הקוד בסוף. כדי להריץ את התרגילים, תזדקקו ל:

1. **מפתח API של Azure OpenAI** - נקודת הקצה של מודל LLM פרוס.  
2. **סביבת ריצת Python** - שבה ניתן להפעיל את המחברת.  
3. **משתני סביבה מקומיים** - _סיימו עכשיו את שלבי ה[התקנה](./../00-course-setup/02-setup-local.md?WT.mc_id=academic-105485-koreyst) כדי להתכונן_.

המחברת מגיעה עם תרגילים _התחלתיים_ - אך מומלץ להוסיף מקטעי _Markdown_ (תאור) ו־_Code_ (בקשות פרומפט) משלכם כדי לנסות דוגמאות ורעיונות נוספים ולבנות את האינטואיציה שלכם לעיצוב פרומפטים.

- **סטודנטים** עשויים לבקש שהבינה תלמד אותם נושא קשה. הבינה יכולה להנחות את התלמידים עם שיעורים, רמזים ודוגמאות המותאמים לרמתם.

## מהי הנדסת פרומפטים?

התחלנו את השיעור בהגדרת **הנדסת פרומפטים** כתהליך של _עיצוב ואופטימיזציה_ של קלטי טקסט (פרומפטים) כדי לספק תגובות עקביות ואיכותיות (השלמות) למטרה אפליקטיבית ולמודל נתון. נוכל לחשוב על כך כתהליך דו-שלבי:

- _עיצוב_ הפרומפט ההתחלתי עבור מודל ומטרה נתונים  
- _חידוד_ הפרומפט בסבבים חוזרים כדי לשפר את איכות התגובה

זהו בהכרח תהליך של ניסוי וטעיה שדורש אינטואיציה ומאמץ מצד המשתמש כדי להשיג תוצאות מיטביות. אז מדוע זה חשוב? כדי לענות על השאלה נצטרך קודם להבין שלושה מושגים:

- _טוקניזציה_ = איך המודל "רואה" את הפרומפט  
- _מודלי שפה בסיסיים_ = איך המודל הבסיסי "מעבד" פרומפט  
- _מודלי שפה מכווני הוראות_ = איך המודל מסוגל כעת לזהות "משימות"

### טוקניזציה

מודל שפה גדול רואה פרומפטים כרצף של _טוקנים_ כאשר מודלים שונים (או גרסאות שונות של מודל) יכולים לטוקנזט את אותו הפרומפט בדרכים שונות. מאחר ו־LLM מאומנים על טוקנים (ולא על טקסט גולמי), האופן שבו הפרומפטים מתפצלים לטוקנים משפיע ישירות על איכות התגובה שנוצרת.

כדי לקבל אינטואיציה לגבי אופן פעולת הטוקניזציה, נסו כלים כמו [OpenAI Tokenizer](https://platform.openai.com/tokenizer?WT.mc_id=academic-105485-koreyst) המוצג למטה. העתיקו את הפרומפט שלכם - וראו כיצד הוא מתורגם לטוקנים, תוך תשומת לב לאופן טיפול בדמויות רווח וסימני פיסוק. שימו לב שהדוגמה הזו משתמשת ב־LLM ישן יותר (GPT-3) - ולכן ניסיון עם דגם חדש יותר עשוי להניב תוצאות שונות.

![טוקניזציה](../../../translated_images/he/04-tokenizer-example.e71f0a0f70356c5c.webp)

### מושג: מודלים בסיסיים

ברגע שהפרומפט עובר טוקניזציה, התפקיד העיקרי של ה־["מודל בסיסי"](https://blog.gopenai.com/an-introduction-to-base-and-instruction-tuned-large-language-models-8de102c785a6?WT.mc_id=academic-105485-koreyst) (או מודל יסוד) הוא לחזות את הטוקן הבא ברצף. מכיוון ש־LLM מאומנים על מאגרי טקסט עצומים, יש להם הבנה טובה של הקשרים סטטיסטיים בין טוקנים ויכולת לחזות זאת עם רמת ביטחון מסוימת. שימו לב שהם לא "מבינים" את _משמעות_ המילים בפרומפט או בטוקן; הם פשוט מזהים דפוס שהם יכולים "להשלים" עם החיזוי הבא שלהם. הם יכולים להמשיך לחזות את הרצף עד שייגמר על ידי התערבות משתמש או תנאי שהוגדר מראש.

רוצים לראות איך השלמת פרומפט מתבצעת? הזינו את הפרומפט שלמעלה ב־Azure OpenAI Studio [_Chat Playground_](https://oai.azure.com/playground?WT.mc_id=academic-105485-koreyst) עם ההגדרות ברירת המחדל. המערכת מוגדרת להתייחס לפרומפטים כבקשות מידע - לכן תראו השלמה המתאימה להקשר זה.

אבל מה אם המשתמש רוצה לקבל משהו מסוים שעונה לקריטריון או למטרה מסוימת? כאן נכנסים לתמונה מודלי שפה _מכווני הוראות_.

![השלמת צ׳אט במודל בסיסי](../../../translated_images/he/04-playground-chat-base.65b76fcfde0caa67.webp)

### מושג: מודלי שפה מכווני הוראות

[מודלי שפה מכווני הוראות](https://blog.gopenai.com/an-introduction-to-base-and-instruction-tuned-large-language-models-8de102c785a6?WT.mc_id=academic-105485-koreyst) מתחילים עם המודל הבסיסי ומבצעים כיוונון עדין באמצעות דוגמאות או זוגות קלט/פלט (למשל, "הודעות" מרובות סבבים) שיכולות להכיל הוראות ברורות - והתשובה מהבינה המלאכותית מנסה לעקוב אחר ההוראות.

טכניקות אלו משתמשות בלמידת חיזוק עם משוב אנושי (RLHF) שמאפשרות לאמן את המודל _לעקוב אחר הוראות_ ו_ללמוד מהמשוב_ כך שהוא יפיק תגובות המתאימות יותר ליישומים מעשיים ורלוונטיות למטרות המשתמש.

בואו ננסה זאת - חזרו לפרומפט מעלה, אך כעת שנו את _הודעת המערכת_ כך שתספק את ההוראה הבאה כהקשר:

> _סכם את התוכן שאתה מקבל עבור תלמיד בכיתה ב'. שמור על התוצאה לפסקה אחת עם 3-5 נקודות עיקריות._

ראיתם כיצד התוצאה כעת מותאמת למטרה ולפורמט הרצוי? מחנך יכול כעת להשתמש ישירות בתגובה זו במצגות השיעור.

![השלמת צ׳אט במודל מכוון הוראות](../../../translated_images/he/04-playground-chat-instructions.b30bbfbdf92f2d05.webp)

## למה אנחנו צריכים הנדסת פרומפטים?

כעת כשאנחנו יודעים כיצד מודלי השפה מעבדים פרומפטים, נדבר על _למה_ אנחנו צריכים הנדסת פרומפטים. התשובה טמונה באתגרים הרבים שה־LLM הנוכחיים מציבים, מה שהופך את קבלת _השלמות אמינות ועקביות_ למאתגרות יותר ללא השקעת מאמץ בבניית הפרומפט ואופטימיזציה שלו. לדוגמה:

1. **תגובות המודל הן סטוכסטיות.** _אותו פרומפט_ צפוי לייצר תגובות שונות במודלים שונים או בגרסאות שונות של אותו מודל. ואפילו יתכן תוצאות משתנות עם _אותו המודל_ בזמנים שונים. _טכניקות הנדסת פרומפט יכולות לעזור למזער את השונות הללו על ידי מתן הגנות טובות יותר_.

1. **מודלים עשויים להמציא תגובות.** המודלים מאומנים על מאגרים _גדולים אך סופיים_, כלומר חסרה להם ידע על מושגים מחוץ להיקף האימון. כתוצאה מכך, הם עלולים לייצר השלמות לא מדויקות, דמיוניות או סותרות ידע ידוע. _טכניקות הנדסת פרומפט מסייעות למשתמשים לזהות ולהפחית המצאות כאלה, למשל בבקשת אזכורים או נימוקים מהבינה_.

1. **יכולות המודלים משתנות.** דגמים חדשים או דורות מודלים חדשים יכללו יכולות עשירות יותר, אך יביאו גם תכונות וחריגות ייחודיות, ועלויות ומורכבות שונות. _הנדסת פרומפטים יכולה לסייע בפיתוח שיטות עבודה וזרימות עבודה שמסתירות הבדלים ומסתגלות לדרישות ספציפיות של מודלים בצורה קלה ונגישה_.

בואו נבחן את זה בפעולה ב־OpenAI או Azure OpenAI Playground:

- השתמשו באותו הפרומפט בפריסות LLM שונות (למשל, OpenAI, Azure OpenAI, Hugging Face) - האם ראיתם הבדלים?  
- השתמשו באותו פרומפט שוב ושוב בפריסת LLM _זהה_ (למשל, ב־Azure OpenAI playground) - כיצד השונות הללו נבדלה?

### דוגמה להמצאות

בקורס זה, אנו משתמשים במונח **"המצאה"** כדי להתייחס לתופעה שבה מודלי שפה לעיתים מייצרים מידע שגוי עובדתית עקב מגבלות האימון או אילוצים אחרים. יתכן ששמעתם על כך בכינוי _"הלוצינציות"_ במאמרים פופולריים או מחקרים. עם זאת, אנו ממליצים בחום להשתמש במונח _"המצאה"_ כדי שלא לייחס בטעות התנהגות אנושית לתוצאה שמונעת על ידי מכונה. זה גם מחזק את [הקווים המנחים של בינה מלאכותית אחראית](https://www.microsoft.com/ai/responsible-ai?WT.mc_id=academic-105485-koreyst) מבחינת טרמינולוגיה, ומסיר מונחים שעלולים להיחשב לפוגעניים או לא מחוברים בהקשרים מסוימים.

רוצים להבין איך המצאות עובדות? חשבו על פרומפט שמורה לבינה לייצר תוכן על נושא שאינו קיים (כדי לוודא שהנושא לא נמצא במאגרי האימון). למשל - ניסיתי את הפרומפט הבא:

## בניית פרומפט

ראינו מדוע הנדסת פרומפטים חשובה – עכשיו נבין איך פרומפטים _בנויים_ כדי שנוכל להעריך טכניקות שונות לעיצוב פרומפטים יעיל יותר.

### פרומפט בסיסי

נתחיל בפרומפט הבסיסי: קלט טקסט שנשלח למודל ללא הקשר אחר. הנה דוגמה – כשנשלח את המילים הראשונות של ההמנון הלאומי של ארצות הברית ל-OpenAI [Completion API](https://platform.openai.com/docs/api-reference/completions?WT.mc_id=academic-105485-koreyst) הוא מיד _משלים_ את התגובה בשורות הבאות, ומדגים את התנהגות החיזוי הבסיסית.

| פרומפט (קלט)      | השלמה (פלט)                                                                                                                        |
| :----------------- | :---------------------------------------------------------------------------------------------------------------------------------- |
| Oh say can you see | זה נשמע שאתה מתחיל את מילות השיר "The Star-Spangled Banner", ההמנון הלאומי של ארצות הברית. מילות השיר המלאות הן ...             |

### פרומפט מורכב

כעת נוסיף הקשר והוראות לפרומפט הבסיסי. ה-[Chat Completion API](https://learn.microsoft.com/azure/ai-services/openai/how-to/chatgpt?WT.mc_id=academic-105485-koreyst) מאפשר לנו לבנות פרומפט מורכב כאוסף של _הודעות_ המכילות:

- זוגות קלט/פלט המשקפים קלט של _המשתמש_ ותגובה של _העוזר_.
- הודעת מערכת הקובעת את ההקשר להתנהגות או לאישיות של העוזר.

כעת הבקשה היא בצורה הבאה, כאשר ה-_טוקניזציה_ לתפוס בצורה אפקטיבית מידע רלוונטי מההקשר ומהשיחה. שינוי ההקשר של המערכת יכול להיות בעל השפעה משמעותית על איכות ההשלמות, כמו קלטי המשתמש שניתנים.

```python
response = openai.chat.completions.create(
    model="gpt-3.5-turbo",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Who won the world series in 2020?"},
        {"role": "assistant", "content": "The Los Angeles Dodgers won the World Series in 2020."},
        {"role": "user", "content": "Where was it played?"}
    ]
)
```

### פרומפט הוראות

בדוגמאות שלמעלה, פרומפט המשתמש היה שאלה פשוטה שניתן לפרש כהבקשה למידע. עם פרומפטים של _הוראות_, אנו יכולים להשתמש בטקסט זה כדי להגדיר משימה בפירוט רב יותר, ולספק הנחיה טובה יותר ל-AI. הנה דוגמה:

| פרומפט (קלט)                                                                                                                                                                                                                         | השלמה (פלט)                                                                                                        | סוג ההוראה       |
| :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------- | :---------------- |
| כתוב תיאור של מלחמת האזרחים                                                                                                                                                                                                          | _החזירה פסקה פשוטה_                                                                                                | פשוט              |
| כתוב תיאור של מלחמת האזרחים. ספק תאריכים ואירועים מרכזיים ותאר את משמעותם                                                                                                                                                           | _החזירה פסקה ואחריה רשימת תאריכי אירועים מרכזיים עם תיאורים_                                                     | מורכב              |
| כתוב תיאור של מלחמת האזרחים בפסקה אחת. ספק 3 נקודות עם תאריכים מרכזיים ומשמעותם. ספק 3 נקודות נוספות עם דמויות היסטוריות מרכזיות ותרומתן. החזר את הפלט בקובץ JSON | _מחזיר פרטים נרחבים יותר בתיבת טקסט, מעוצב כ-JSON שניתן להעתיק ולהדביק לקובץ ולוודא כנדרש_                        | מורכב. מעוצב.       |

## תוכן ראשי

בדוגמאות שלמעלה, הפרומפט היה עדיין פתוח יחסית, ואפשר ל-LLM להחליט איזה חלק מהמאגר הנתונים המאומן שלו רלוונטי. עם תבנית העיצוב _תוכן ראשי_, טקסט הקלט מחולק לשני חלקים:

- הוראה (פעולה)
- תוכן רלוונטי (שמשפיע על הפעולה)

הנה דוגמה שבה ההוראה היא "סכם זאת בשתי משפטים".

| פרומפט (קלט)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         | השלמה (פלט)                                                                                                                                                                                                                                                                             |
| :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| יופיטר הוא כוכב הלכת החמישי מהשמש והגדול במערכת השמש. זהו ענק גז עם מסת כאלפית מזו של השמש, אך פי שניים וחצי ממסת כל שאר כוכבי הלכת במערכת השמש יחד. יופיטר הוא אחת מהגופים המבריקים ביותר הנראים לעין בלתי מזוינת בשמי הלילה, והוא היה ידוע לציוויליזציות עתיקות מאז לפני ההיסטוריה המתועדת. הוא נקרא על שם האל הרומי יופיטר.[19] כאשר מתבוננים בו מהארץ, יופיטר יכול להיות מבריק מספיק על מנת שהאור המוחזר שלו יטיל צללים נראים לעין,[20] והוא בממוצע האובייקט הטבעי השלישי הבהיר ביותר בשמי הלילה אחרי הירח ונוגה. <br/> **סכם זאת בשני משפטים קצרים**                     | יופיטר, כוכב הלכת החמישי מהשמש, הוא הגדול ביותר במערכת השמש ונודע כאחד הגופים המבריקים ביותר בשמי הלילה. הוא נקרא על שם האל הרומי יופיטר, וענק גז שמסתו פי שניים וחצי ממסת כל שאר כוכבי הלכת במערכת השמש יחד. |

קטע התוכן הראשי יכול לשמש בדרכים שונות להנעת הוראות יעילות יותר:

- **דוגמאות** - במקום להגיד למודל מה לעשות עם הוראה מפורשת, תן לו דוגמאות למה לעשות ותן לו להסיק את הדפוס.
- **רמזים** - הוסף אחרי ההוראה "רמז" שמעורר את ההשלמה ומכוון את המודל לתגובות רלוונטיות יותר.
- **תבניות** - אלו מתכונים חוזרים עם תבניות (משתנים) שניתן להתאים עם נתונים לשימושים ספציפיים.

בואו נחקור את אלה בפעולה.

### שימוש בדוגמאות

זו גישה שבה משתמשים בתוכן הראשי כדי "להאכיל את המודל" עם דוגמאות לפלט הרצוי עבור הוראה נתונה, ותנו לו להסיק את הדגם לפלט הרצוי. על פי מספר הדוגמאות שניתנו, אפשר להשתמש ב-zero-shot prompting, one-shot prompting, few-shot prompting וכו'.

הפרומפט כעת מורכב משלושה מרכיבים:

- תיאור המשימה
- כמה דוגמאות לפלט הרצוי
- תחילת דוגמה חדשה (שהופכת לתיאור משימה מרומז)

| סוג הלמידה | פרומפט (קלט)                                                                                                                                        | השלמה (פלט)            |
| :------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------- | :----------------------- |
| Zero-shot     | "השמש זורחת". תרגם לספרדית                                                                                                                           | "El Sol está brillando". |
| One-shot      | "השמש זורחת" => "El Sol está brillando". <br> "זה יום קר ורוחי" =>                                                                                      | "Es un día frío y ventoso". |
| Few-shot      | השחקן רץ על הבסיסים => בייסבול <br/> השחקן ביצע אס => טניס <br/> השחקן הכה שש => קריקט <br/> השחקן עשה סלאם-דאנק =>                         | כדורסל                    |
|               |                                                                                                                                                       |                          |

שימו לב שעלינו לספק הוראה מפורשת ("תרגם לספרדית") בפרומפט ה-zero-shot, אך היא מוסקת אוטומטית בדוגמת ה-one-shot. דוגמת ה-few-shot מראה כיצד הוספת דוגמאות מאפשרת למודלים להסיק בצורה מדויקת יותר ללא הוראות נוספות.

### תבניות הנחיה

תבנית הנחיה היא _מתכון מוגדר מראש להנחיה_ שניתן לאחסן ולהשתמש בו שוב לפי הצורך, על מנת להניע חוויות משתמש עקביות בקנה מידה. בצורה הפשוטה ביותר, מדובר באוסף דוגמאות להנחה כמו [זו מאת OpenAI](https://cookbook.openai.com/examples/gpt4-1_prompting_guide?WT.mc_id=academic-105485-koreyst) שמספקת גם את רכיבי ההנחיה האינטראקטיביים (הודעות משתמש ומערכת) ואת פורמט הבקשה המונע על ידי API - לתמיכה בשימוש חוזר.

בצורה המורכבת יותר כמו [דוגמה זו מ-LangChain](https://python.langchain.com/docs/concepts/prompt_templates/?WT.mc_id=academic-105485-koreyst) היא מכילה _משתני מיקום_ שניתן להחליף בנתונים ממקורות שונים (קלט משתמש, הקשר מערכת, מקורות מידע חיצוניים וכו') ליצירת הנחיה באופן דינמי. זה מאפשר לנו ליצור ספרייה של הנחיות שניתן להשתמש בהן **תוך כדי תכנות** בקנה מידה.

לבסוף, הערך האמיתי של תבניות טמון ביכולת ליצור ולפרסם _ספריות הנחיות_ לתחומי יישום אנכיים - שבהם תבנית ההנחיה מוצמדת כעת להקשר ספציפי ליישום או דוגמאות המשפרות את הרלוונטיות והדיוק לקהל היעד. מאגר [Prompts For Edu](https://github.com/microsoft/prompts-for-edu?WT.mc_id=academic-105485-koreyst) הוא דוגמה מצוינת לגישה זו, המאגדת ספריה של הנחיות לתחום החינוך עם דגש על מטרות מרכזיות כמו תכנון שיעורים, עיצוב תכניות לימודים, הדרכת תלמידים וכו'.

## תוכן תומך

אם נחשוב על בניית הנחיה כהכילה של הוראה (משימה) ותוכן ראשי (תוכן עיקרי), אז התוכן המשני הוא כמו הקשר נוסף שאנו מספקים ל**להשפיע על הפלט בדרך כלשהי**. זה יכול להיות פרמטרים לכיוונון, הוראות עיצוב, טקסונומיות נושאים וכו' שיכולים לעזור למודל _לתפור_ את תגובתו כך שתתאים ליעדי המשתמש או לציפיותיו.

לדוגמה: בהינתן קטלוג קורסים עם מטא-נתונים נרחבים (שם, תיאור, רמה, תגי מטא, מרצה וכו') על כל הקורסים הזמינים בתכנית הלימודים:

- ניתן להגדיר הוראה "לסכם את קטלוג הקורסים לסתיו 2023"
- ניתן להשתמש בתוכן הראשי כדי לספק מספר דוגמאות לפלט הרצוי
- ניתן להשתמש בתוכן המשני כדי לזהות את 5 התגים המובילים של העניין.

כעת, המודל יכול לספק סיכום בפורמט שמוצג על ידי כמה דוגמאות - אך אם לתוצאה יש מספר תגיות, הוא יכול לתת עדיפות ל-5 התגים שהוגדרו בתוכן המשני.

## שיטות עבודה מומלצות להנחיה

כעת כאשר אנו יודעים כיצד ניתן _לבנות_ הנחיות, אנו יכולים להתחיל לחשוב כיצד לעצב אותן כדי לשקף שיטות עבודה מומלצות. ניתן לחשוב על כך בשני חלקים - להחזיק ב_מיינדסט_ נכון וליישם את ה_טכניקות_ הנכונות.

### מיינדסט להנדסת הנחיות

הנדסת הנחיות היא תהליך של ניסוי וטעייה ולכן שמרו על שלושה גורמים מנחים כלליים:

1. **הבנת תחום חשובה.** דיוק ורלוונטיות התגובה הם פונקציה של _התחום_ שבו האפליקציה או המשתמש פועלים. יש להשתמש באינטואיציה ומומחיות תחומית כדי **להתאים טכניקות** בצורה ממוקדת יותר. לדוגמה, להגדיר _אישיויות תחומיות_ בהנחיות המערכת, או להשתמש ב_תבניות תחומיות_ בהנחיות למשתמש. לספק תוכן משני המשקף הקשרים תחומיים, או להשתמש ב_רמזים ודוגמאות תחומיות_ כדי לכוון את המודל לדפוסי שימוש מוכרים.

2. **הבנת המודל חשובה.** אנו יודעים שמודלים הם סטוכסטיים במהותם. אך יישומי מודלים יכולים גם להשתנות במונחי מערך הנתונים עליו הם מאומנים (ידע מאולפן), היכולות שהם מספקים (למשל דרך API או SDK) וסוג התוכן שהם מותאמים לו (קוד מול תמונות מול טקסט). להבין את חוזקות וחולשות המודל שבו אתם משתמשים, ולנצל ידע זה כדי _לתעדף משימות_ או לבנות _תבניות מותאמות_ שמותאמות ליכולות המודל.

3. **איטרציה ואימות חשובים.** מודלים מתפתחים במהירות, וכך גם הטכניקות להנדסת הנחיות. כמומחה תחום, ייתכן שיש לכם הקשר או קריטריונים ייחודיים ליישום הספציפי שלכם, שאינם חלים על הקהילה הרחבה. השתמשו בכלי והנחיות הנדסת הנחיות ל"מתיחת זרוע" של בניית הנחיות, ואז בצעו איטרציה ואימות באמצעות האינטואיציה ומומחיות התחום שלכם. תעדו את התובנות וצרו **בסיס ידע** (למשל, ספריות הנחיות) שניתן להשתמש בו כקו בסיס חדש לאחרים, לאיטרציות מהירות יותר בעתיד.

## משימה

ברכות! הגעתם לסוף השיעור! הגיע הזמן לבחון כמה מהמושגים והטכניקות עם דוגמאות אמיתיות!

למשימתנו, נשתמש ב-Jupyter Notebook עם תרגילים שאפשר להשלים אינטראקטיבית. ניתן גם להרחיב את ה-Notebook עם תאי Markdown ו-Code משלכם כדי לחקור רעיונות וטכניקות באופן עצמאי.

### כדי להתחיל, עשו פיצול לרפוזיטורי, ואז

- (מומלץ) הפעלת GitHub Codespaces
- (אופציה נוספת) שיבטו את הרפוזיטורי למכשיר המקומי שלכם ושימוש בו עם Docker Desktop
- (אופציה נוספת) פתחו את ה-Notebook בסביבת הריצה המועדפת עליכם.

### לאחר מכן, הגדירו את משתני הסביבה שלכם

- העתיקו את הקובץ `.env.copy` בשורש הרפוזיטורי ל-`.env` ומלאו את הערכים `AZURE_OPENAI_API_KEY`, `AZURE_OPENAI_ENDPOINT` ואת `AZURE_OPENAI_DEPLOYMENT`. חזרו לקטע [סנדבוקס ללימוד](#סביבת-חול-למידה) כדי ללמוד כיצד.

### לאחר מכן, פתחו את Jupyter Notebook

- בחרו את קרנל הריצה. אם משתמשים באפשרות 1 או 2, פשוט בחרו את קרנל ברירת המחדל Python 3.10.x המסופק על ידי מכולת הפיתוח.

אתם מוכנים להפעיל את התרגילים. שימו לב שאין כאן תשובות _נכונות או שגויות_ - רק חקירת אפשרויות דרך ניסוי וטעייה ובניית אינטואיציה לגבי מה עובד עבור מודל ותחום יישום נתון.

_לסיבה זו אין קטעי פתרון קוד בשיעור זה. במקום זאת, ה-Notebook יכיל תאי Markdown שכותרתם "הפתרון שלי:" המציגים דוגמה אחת לפלט כהפניה._
