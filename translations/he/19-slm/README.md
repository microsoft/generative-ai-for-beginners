# מבוא למודלים שפה קטנים לבינה מלאכותית גנרטיבית למתחילים  
בינה מלאכותית גנרטיבית היא תחום מרתק בבינה מלאכותית המתמקד ביצירת מערכות היכולות ליצור תוכן חדש. תוכן זה יכול להיות טקסט, תמונות, מוזיקה ואפילו סביבות וירטואליות שלמות. אחת מהיישומים המרגשים ביותר של בינה מלאכותית גנרטיבית היא בתחום מודלי השפה.  

## מה הם מודלים שפה קטנים?  

מודל שפה קטן (SLM) מייצג גרסה מקוצרת של מודל שפה גדול (LLM), המנצל רבים מהעקרונות והטכניקות האדריכליות של מודלים גדולים, תוך כדי הצגת טביעת חישוב מופחתת משמעותית.  

SLM הם תת-קבוצה של מודלי שפה המיועדים ליצור טקסט הדומה לטקסט אנושי. בניגוד למקביליהם הגדולים יותר, כמו GPT-4, ה-SLM קומפקטיים ויעילים יותר, מה שהופך אותם לאידיאליים ליישומים שבהם משאבי החישוב מוגבלים. למרות הגודל הקטן שלהם, הם עדיין יכולים לבצע מגוון משימות. בדרך כלל, SLM נבנים על ידי דחיסה או זיקוק של LLM, במטרה לשמר חלק משמעותי מפונקציונליות המודל המקורי ויכולותיו הלשוניות. הקטנת גודל המודל מפחיתה את המורכבות הכוללת, מה שהופך את ה-SLM ליעילים יותר מבחינת השימוש בזיכרון ודרישות חישוב. למרות אופטימיזציות אלו, SLM יכולים לבצע מגוון רחב של משימות עיבוד שפה טבעית (NLP):  

- יצירת טקסט: יצירת משפטים או פסקאות קוהרנטיים ורלוונטיים הקשרית.  
- השלמת טקסט: חיזוי והשלמת משפטים בהתבסס על פרומפט נתון.  
- תרגום: המרת טקסט משפה אחת לאחרת.  
- סיכום: קיבוץ טקסטים ארוכים לסיכומים קצרים וברי עיכול יותר.  

אף על פי שקיימים פשרות בביצועים או בעומק ההבנה יחסית למקביליהם הגדולים יותר.  

## איך פועלים מודלים שפה קטנים?  
ה-SLM מאומנים על כמויות עצומות של נתוני טקסט. במהלך האימון הם לומדים את הדפוסים והמבנים של השפה, מה שמאפשר להם ליצור טקסט שהוא גם דקדוקי נכון וגם מתאים להקשר. תהליך האימון כולל:  

- איסוף נתונים: איסוף מאגרי טקסט גדולים ממקורות שונים.  
- עיבוד מקדים: ניקוי וארגון הנתונים כדי להתאים אותם לאימון.  
- אימון: שימוש באלגוריתמים של למידת מכונה ללמד את המודל כיצד להבין וליצור טקסט.  
- כוונון עדין: התאמת המודל לשיפור ביצועים במשימות ספציפיות.  

פיתוח ה-SLM מתיישב עם הצורך הגובר במודלים שניתן לפרוס בסביבות בעלות משאבים מוגבלים, כמו מכשירים ניידים או פלטפורמות מחשוב קצה, שבהן LLM מלאים עשויים להיות לא מעשיים בשל דרישות המשאבים הגבוהות שלהם. בהתמקדות ביעילות, SLM מאזנים ביצועים עם נגישות, ומאפשרים יישום רחב יותר בתחומים מגוונים.

## ההבדלים בין מודלי שפה גדולים (LLMs) ומודלי שפה קטנים (SLMs)  

שני סוגי המודלים, LLMs ו-SLMs, מבוססים על עקרונות יסודיים של למידת מכונה הסתברותית, ופועלים בשיטות דומות בעיצוב אדריכלי, מתודולוגיות אימון, תהליכי יצירת נתונים ושיטות הערכת מודל. עם זאת, מספר גורמים מרכזיים מבדילים בין שני סוגי המודלים.  

## יישומים של מודלי שפה קטנים  

ל-SLM יש מגוון רחב של יישומים, כולל:  

- צ'טבוטים: מתן תמיכה ללקוחות ואינטראקציה עם משתמשים באופן שיחתי.  
- יצירת תוכן: סיוע לכותבים ביצירת רעיונות ואפילו טיוטות של מאמרים שלמים.  
- חינוך: סיוע לתלמידים במשימות כתיבה או בלמידת שפות חדשות.  
- נגישות: יצירת כלים לאנשים עם מוגבלויות, כגון מערכות דיבור מטקסט.  

**גודל**  
  
ההבדל העיקרי בין LLM ל-SLM הוא בקנה המידה של המודלים. LLMs, כגון ChatGPT (GPT-4), עשויים לכלול כ-1.76 טריליון פרמטרים, בעוד שמודלי SLM בקוד פתוח כמו Mistral 7B מעוצבים עם פרמטרים בולים פחותים בהרבה - כ-7 מיליארד. הפער הזה נובע בעיקר מהבדלים באדריכלות המודל ובתהליכי האימון. לדוגמה, ChatGPT משתמש במנגנון תשומת לב עצמית בתוך מסגרת מקודד-מפענח, בעוד ש-Mistral 7B משתמש בתשומת לב חלונית משתלחת, מה שמאפשר אימון יעיל יותר במסגרת של מודל מפענח בלבד. שוני אדריכלי זה משפיע משמעותית על המורכבות והביצועים של המודלים.  

**הבנה**  

SLM בדרך כלל מותאמים לביצועים בתחום ספציפי, מה שהופך אותם למומחים מאוד אך עם יכולת מוגבלת לספק הבנה רחבה הקשרית במגוון תחומים. לעומת זאת, LLM נועדו לחקות אינטליגנציה בדומה לבני אדם ברמה מקיפה יותר. מאומנים על בסיס נתונים רחב ומגוון, LLMs מיועדים לבצע טוב במגוון תחומים, ומציעים גמישות והתאמה רבה יותר. לפיכך, LLM מתאימים יותר לטווח רחב של משימות מורכבות, כגון עיבוד שפה טבעית ותכנות.  

**חישוב**  

אימון ופריסה של LLM דורשים משאבים עצומים, לרוב עם תשתית חישובית כבדה הכוללת אשכולות GPU בקנה מידה גדול. לדוגמה, אימון מודל כמו ChatGPT מאפס עשוי לדרוש אלפי GPUs במשך תקופות ארוכות. לעומת זאת, SLM, עם מספר פרמטרים קטן יותר, נגישים יותר מבחינת המשאבים החישוביים. מודלים כמו Mistral 7B ניתנים לאימון והרצה במכונות מקומיות עם יכולות GPU ממוצעות, אם כי האימון עדיין דורש מספר שעות על פני כמה GPUs.  

**הטיה**  

הטיה היא בעיה מוכרת ב-LLM, בעיקר עקב טבע נתוני האימון. מודלים אלו מבוססים לעיתים קרובות על נתונים גלויים מהאינטרנט, שעשויים להכיל ייצוג חסר או מעוות של קבוצות מסוימות, תיוג שגוי או הטיות לשוניות המושפעות מדיאלקטים, שונות גאוגרפית וחוקי דקדוק. בנוסף, המורכבות של אדריכלות LLM יכולה להשפיע לרעה ולחזק הטיות אלו, שלעיתים אינן מתגלות ללא כוונון מוקפד. לעומת זאת, SLM, המאומנים על מאגרי נתונים מוגבלים וממוקדים תחום, פחות מועדים להטיות כאלה, אף שהם אינם חסינים מפניהן.  

**אינפרנס**  

הגודל המופחת של ה-SLM מעניק להם יתרון משמעותי במהירות האינפרנס, ומאפשר ייצור פלט ביעילות על חומרה מקומית ללא צורך בעיבוד מקבילי נרחב. לעומת זאת, LLM, עקב גודלם ומורכבותם, דורשים לעיתים משאבים חישוביים מקבילים משמעותיים כדי להשיג זמן תגובה מתקבל על הדעת. נוכחות של מספר משתמשים במקביל גם היא מאט את זמני התגובה של LLM, במיוחד בפריסה בקנה מידה גדול.  

לסיכום, בעוד של-LLM ו-SLM יש בסיס משותף בלמידת מכונה, הם נבדלים באופן משמעותי בגודל המודל, בדרישות המשאבים, בהבנה הקונטקסטואלית, ברגישות להטיה ובמהירות האינפרנס. הבדלים אלו משקפים את התאמתם למקרים שימוש שונים, כאשר LLM גמישים יותר אך כבדי משאבים, ו-SLM מציעים יעילות תחומית עם דרישות חישוב מופחתות.  

***הערה: בשיעור זה נציג את SLM באמצעות Microsoft Phi-3 / 3.5 כדוגמה.***  

## הצגת משפחת Phi-3 / Phi-3.5  

משפחת Phi-3 / 3.5 מיועדת בעיקר לתרחישי יישום בתחום הטקסט, הראייה והמודל המומחה (MoE):  

### Phi-3 / 3.5 Instruct  

בעיקר ליצירת טקסט, השלמת שיחות וחילוץ מידע מתוכן, ועוד.  

**Phi-3-mini**  

מודל השפה עם 3.8 מיליארד פרמטרים זמין ב-Microsoft Azure AI Studio, Hugging Face, ו-Ollama. מודלי Phi-3 מבצעים טוב בהרבה ממודלי שפה בגודל שווה או גדול יותר בבנצ'מרקים מרכזיים (ראו את מספרי הבנצ'מרק למטה, מספרים גבוהים יותר טובים יותר). Phi-3-mini מבצע טוב ממודלים בעלי גודל כפול, בעוד ש-Phi-3-small ו-Phi-3-medium מבצעים טובים יותר ממודלים גדולים יותר, כולל GPT-3.5.  

**Phi-3-small ו-medium**  

עם כ-7 מיליארד פרמטרים בלבד, Phi-3-small מקדים את GPT-3.5T במגוון מבחני שפה, היגיון, קידוד ומתמטיקה.  

Phi-3-medium עם 14 מיליארד פרמטרים ממשיך מגמה זו ומקדים את Gemini 1.0 Pro.  

**Phi-3.5-mini**  

ניתן לחשוב עליו כעדכון ל-Phi-3-mini. בעוד שמספר הפרמטרים נשאר ללא שינוי, הוא משפר את היכולת לתמוך בשפות רבות (תומך ב-20+ שפות: ערבית, סינית, צ'כית, דנית, הולנדית, אנגלית, פינית, צרפתית, גרמנית, עברית, הונגרית, איטלקית, יפנית, קוריאנית, נורווגית, פולנית, פורטוגזית, רוסית, ספרדית, שוודית, תאית, טורקית, אוקראינית) ומוסיף תמיכה חזקה יותר להקשר ארוך.  

Phi-3.5-mini עם 3.8 מיליארד פרמטרים מבצע טוב יותר ממודלי שפה בגודל זהה ועם ביצועים זהים למודלים בגודל כפול.  

### Phi-3 / 3.5 Vision  

ניתן לחשוב על מודל ה-Instruct של Phi-3/3.5 כיכולות ההבנה של Phi, ו-Vision הוא מה שנותן ל-Phi עיניים להבין את העולם.  


**Phi-3-Vision**  

Phi-3-vision, עם 4.2 מיליארד פרמטרים בלבד, ממשיך מגמה זו ומקדים מודלים גדולים יותר כגון Claude-3 Haiku ו-Gemini 1.0 Pro V במשימות כלליות של היגיון חזותי, OCR, והבנת טבלאות ודיאגרמות.  


**Phi-3.5-Vision**  

Phi-3.5-Vision הוא גם שדרוג של Phi-3-Vision, ומוסיף תמיכה בתמונות מרובות. ניתן לחשוב עליו כשיפור בראייה, לא רק שתוכל לראות תמונות, אלא גם וידאו.  

Phi-3.5-vision מקדים מודלים גדולים יותר כמו Claude-3.5 Sonnet ו-Gemini 1.5 Flash במשימות OCR, הבנת טבלאות וגרפים, ומשווה ביצועים במשימות היגיון חזותי כלליות. תומך בקלט רב-מסגרתי, כלומר נסיון היגיון על מספר תמונות קלט.  


### Phi-3.5-MoE  

***Mixture of Experts (MoE)*** מאפשר למודלים להיות מאומנים מראש עם הרבה פחות חישוב, מה שאומר שניתן להגדיל בחדות את גודל המודל או מאגר הנתונים עם אותו תקציב חישוב כמו במודל צפוף. בפרט, מודל MoE צריך להגיע לאותה איכות כמו מקבילו הצפוף מהר יותר במהלך האימון המקדים.  

Phi-3.5-MoE מורכב מ-16 מודולי מומחים של 3.8 מיליארד כל אחד. Phi-3.5-MoE עם רק 6.6 מיליארד פרמטרים פעילים משיג רמת היגיון, הבנה לשונית ומתמטית דומה למודלים גדולים בהרבה.  

ניתן להשתמש במודל משפחת Phi-3/3.5 בהתבסס על תרחישים שונים. בניגוד ל-LLM, ניתן לפרוס Phi-3/3.5-mini או Phi-3/3.5-Vision במכשירי קצה.

**NVIDIA NIM**  

בנוסף לפתרונות קטלוג המודלים מבוססי ענן המוצעים על ידי Azure ו-GitHub, ניתן גם להשתמש ב-[NVIDIA NIM](https://developer.nvidia.com/nim?WT.mc_id=academic-105485-koreyst) להשלמת הקריאות הרלוונטיות. ניתן לבקר ב-NVIDIA NIM כדי להשלים קריאות API של משפחת Phi-3/3.5. 1. **מודלים מאומנים מראש**: היא מספקת אלפי מודלים מאומנים מראש שניתן להשתמש בהם למגוון משימות כגון סיווג טקסט, זיהוי ישויות, מענה לשאלות, סיכום, תרגום ויצירת טקסט.

3. **יכולות מולטימודליות**: בנוסף לעיבוד שפה טבעית, Hugging Face Transformers תומכת גם במשימות בתחום ראייה ממוחשבת (למשל סיווג תמונות, זיהוי עצמים) ועיבוד קול (למשל זיהוי קולי, סיווג אודיו).

4. **קלות שימוש**: הספרייה מציעה ממשקי API וכלים להורדה וכיוונון קל של מודלים, מה שהופך אותה לנגישה למתחילים ולמומחים כאחד.

5. **קהילה ומשאבים**: ל-Hugging Face יש קהילה חיה ותיעוד נרחב, מדריכים והדרכות לעזור למשתמשים להתחיל ולהפיק את המרב מהספרייה.

**ONNX Runtime עבור GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) הוא מאיץ למידת מכונה רב-פלטפורמי להסקה ואימון. ONNX Runtime ל-Generative AI (GENAI) הוא כלי חזק שמסייע להריץ מודלים של AI גנרטיבי ביעילות על פני פלטפורמות שונות.

## מה זה ONNX Runtime?
ONNX Runtime הוא פרויקט קוד פתוח שמאפשר הסקה בעלת ביצועים גבוהים של מודלי למידת מכונה. הוא תומך במודלים בפורמט Open Neural Network Exchange (ONNX), שהוא תקן לייצוג מודלי למידה. היסקה עם ONNX Runtime יכולה לאפשר חוויות משתמש מהירות יותר ועלויות נמוכות יותר, ותומכת במודלים ממסגרות למידה עמוקה כמו PyTorch ו-TensorFlow/Keras וגם מספריות למידת מכונה קלאסיות כמו scikit-learn, LightGBM, XGBoost וכו'. ONNX Runtime תואם לחומרות, דרייברים ומערכות הפעלה שונות, ומספק ביצועים מיטביים על ידי ניצול מאיצי חומרה בהתאם, לצד אופטימיזציות וטרנספורמציות גרפיות.

## מה זה בינה מלאכותית גנרטיבית?
בינה מלאכותית גנרטיבית מתייחסת למערכות AI שיכולות ליצור תכנים חדשים, כגון טקסט, תמונות או מוזיקה, על בסיס הנתונים שעליהם אומנו. דוגמאות לכך הן מודלי שפה כמו GPT-3 ומודלי יצירת תמונות כמו Stable Diffusion. ספריית ONNX Runtime ל-GenAI מספקת את הלולאה הגנרטיבית למודלי ONNX, כולל הסקה עם ONNX Runtime, עיבוד לוגיטים, חיפוש ודגימה, וניהול מטמון KV.

## ONNX Runtime ל-GENAI
ONNX Runtime ל-GENAI מרחיב את יכולות ONNX Runtime לתמיכה במודלים של בינה מלאכותית גנרטיבית. הנה כמה תכונות מפתח:

- **תמיכה רחבה בפלטפורמות:** עובד על פלטפורמות שונות, כולל Windows, Linux, macOS, Android ו-iOS.
- **תמיכה במודלים:** תומך בהרבה מודלים פופולריים של בינה מלאכותית גנרטיבית, כגון LLaMA, GPT-Neo, BLOOM ועוד.
- **אופטימיזציה לביצועים:** כולל אופטימיזציות למאיצי חומרה שונים כמו NVIDIA GPUs, AMD GPUs ועוד.
- **קלות שימוש:** מספק ממשקי API לאינטגרציה קלה ליישומים, ומאפשר לך ליצור טקסט, תמונות ותוכן אחר עם מינימום קוד.
- משתמשים יכולים לקרוא למתודה ברמה גבוהה generate(), או להריץ כל איטרציה של המודל בלולאה, וליצור טוקן אחד בכל פעם, עם אפשרות לעדכן פרמטרים של ההפקה בתוך הלולאה.
- ל-ONNX runtime יש גם תמיכה בחיפוש greedy/beam ודגימות TopP, TopK ליצירת רצפי טוקנים ועיבוד לוגיטים מובנה כמו קנסות חזרה. ניתן גם להוסיף בקלות ניקוד מותאם אישית.

### הפעלת מודל: הנה דוגמה פשוטה בפייתון:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### דמו: שימוש ב-ONNX Runtime GenAI לקריאה ל-Phi-3.5-Vision


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
