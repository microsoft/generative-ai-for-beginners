אפליקציות צ'אט הפכו לחלק אינטגרלי בחיינו היומיומיים, ומציעות יותר מאשר רק אמצעי לשיחה מזדמנת. הן חלקים חיוניים בשירות לקוחות, תמיכה טכנית, ואפילו במערכות ייעוץ מתקדמות. סביר להניח שקיבלתם עזרה מאפליקציית צ'אט לא מזמן. ככל שאנו משלבים טכנולוגיות מתקדמות כמו בינה מלאכותית גנרטיבית בפלטפורמות אלו, המורכבות עולה וכך גם האתגרים.

- **ניטור**. לאחר הפריסה, איך נוכל לנטר ולהבטיח שהאפליקציות פועלות ברמת איכות הגבוהה ביותר, הן מבחינת פונקציונליות והן בהתאמה ל-[ששת עקרונות הבינה המלאכותית האחראית](https://www.microsoft.com/ai/responsible-ai?WT.mc_id=academic-105485-koreyst)?

ככל שאנו מתקדמים לעידן המוגדר על ידי אוטומציה ויחסי גומלין חלקים בין אדם למכונה, הבנת האופן שבו בינה מלאכותית גנרטיבית משנה את ההיקף, העומק והיכולת להסתגלויות באפליקציות צ'אט הופכת להיות חיונית. שיעור זה יחקור את היבטי הארכיטקטורה התומכים במערכות מורכבות אלו, יעמיק במתודולוגיות לכיול מיטבי למשימות בתחום מסוים, ויבחן את המדדים והשיקולים הרלוונטיים להבטחת פריסה אחראית של בינה מלאכותית.

## שילוב בינה מלאכותית גנרטיבית באפליקציות צ'אט

שדרוג אפליקציות צ'אט באמצעות בינה מלאכותית גנרטיבית הוא לא רק להפוך אותן לחכמות יותר; זה קשור לאופטימיזציה של המבנה, הביצועים וממשק המשתמש על מנת לספק חוויית משתמש באיכות גבוהה. זה כולל חקר יסודות הארכיטקטורה, אינטגרציות API, ושיקולי ממשק משתמש. חלק זה נועד להציע לכם מפת דרכים מקיפה לניווט בנופים המורכבים הללו, בין אם אתם משולבים במערכות קיימות או בונים פלטפורמות עצמאיות.

בסוף חלק זה, תצוידו במומחיות הדרושה לבנייה ושילוב יעיל של אפליקציות צ'אט.

### צ'אטבוט או אפליקציית צ'אט?

לפני שנכנסים לבניית אפליקציות צ'אט, נעשה השוואה בין 'צ'אטבוטים' ל-'אפליקציות צ'אט מונעות בינה מלאכותית', שמבצעים תפקידים ופונקציות מובחנות. המטרה העיקרית של צ'אטבוט היא לאוטומציה של משימות שיחה ספציפיות, כמו מענה על שאלות נפוצות או מעקב אחר חבילה. הוא נתון בדרך כלל לחוקים מבוססי כללים או אלגוריתמים מורכבים של בינה מלאכותית. לעומת זאת, אפליקציית צ'אט מונעת בינה מלאכותית היא סביבה רחבה בהרבה שנועדה להקל על צורות שונות של תקשורת דיגיטלית, כמו צ'אט טקסטואלי, קול ווידאו בין משתמשים אנושיים. התכונה המגדירה אותה היא האינטגרציה של מודל בינה מלאכותית גנרטיבי המדמה שיחות מאופיינות ודמויי-אנוש, ומייצר תגובות בהתבסס על מגוון רחב של קלט ורמזים הקשריים. אפליקציית צ'אט מונעת בינה מלאכותית גנרטיבית יכולה לנהל דיונים פתוחים, להסתגל להקשרים מתפתחים ואפילו לייצר דיאלוג יצירתי או מורכב.

הטבלה הבאה מפרטת את ההבדלים המרכזיים והדמיון כדי לסייע בהבנת התפקידים הייחודיים שלהם בתקשורת דיגיטלית.

| צ'אטבוט                               | אפליקציית צ'אט מונעת בינה מלאכותית גנרטיבית        |
| ------------------------------------- | ----------------------------------------------------- |
| ממוקד במשימות ומבוסס כללים            | מודע להקשר                                             |
| משולב לעיתים במערכות גדולות יותר       | עשוי לארח צ'אטבוט אחד או רבים                          |
| מוגבל לפונקציות מתוכנתות                | מכיל מודלים של בינה מלאכותית גנרטיבית                 |
| אינטראקציות מסודרות ומיוחדות            | יכול לקיים דיונים פתוחים                                |

### שימוש בפונקציונליות קיימת עם SDK ו-API

כאשר בונים אפליקציית צ'אט, צעד ראשון מצוין הוא להעריך מה קיים כבר. שימוש ב-SDK ו-API לבניית אפליקציות צ'אט הוא אסטרטגיה משתלמת ממגוון סיבות. באמצעות אינטגרציה של SDK ו-API מתועדים היטב, אתם ממקמים את האפליקציה שלכם באופן אסטרטגי להצלחה ארוכת טווח, ומתמודדים עם בעיות סקלביליות ותחזוקה.

- **מאיץ את תהליך הפיתוח ומפחית עלויות**: הסתמכות על פונקציונליות קיימת במקום בנייה יקרה בעצמכם מאפשרת לכם להתמקד בהיבטים אחרים של האפליקציה שחשובים לכם יותר, כגון לוגיקת עסקית.
- **ביצועים משופרים**: כשבונים פונקציונליות מאפס, בסופו של דבר תישאלו "איך זה מתרחב? האם האפליקציה מסוגלת לטפל בפתאום באלפי משתמשים?" SDK ו-API שמנוהלים היטב מציעים פתרונות מובנים לנושאים אלו.
- **תחזוקה קלה יותר**: עדכונים ושיפורים ניהוליים קלים יותר, שכן רוב ה-API וה-SDK דורשים פשוט עדכון ספרייה כשמתפרסמת גרסה חדשה.
- **גישה לטכנולוגיה מתקדמת**: שימוש במודלים שעברו כוונון ואומן על מאגרי נתונים נרחבים מספק לאפליקציה יכולות שפה טבעית.

גישה לפונקציונליות של SDK או API בדרך כלל כרוכה בקבלת הרשאה לשימוש בשירותים המסופקים, לרוב באמצעות מפתח ייחודי או אסימון אימות. נשתמש בספריית OpenAI בפייתון כדי לחקור איך זה נראה. תוכלו גם לנסות זאת בעצמכם ב-[מחברת OpenAI](./python/oai-assignment.ipynb?WT.mc_id=academic-105485-koreyst) או ב-[מחברת שירותי Azure OpenAI](./python/aoai-assignment.ipynb?WT.mc_id=academic-105485-koreys) לשיעור זה.

```python
import os
from openai import OpenAI

API_KEY = os.getenv("OPENAI_API_KEY","")

client = OpenAI(
    api_key=API_KEY
    )

response = client.responses.create(model="gpt-4o-mini", input="Suggest two titles for an instructional lesson on chat applications for generative AI.", store=False)
print(response.output_text)
```

הדוגמה שלמעלה משתמשת בדגם GPT-4o mini עם API לתגובות כדי להשלים את ההנחיה, אך שימו לב שמפתח ה-API מוגדר לפני כן. תקבלו שגיאה אם לא תגדירו את המפתח.

## חוויית משתמש (UX)

עקרונות UX כלליים חלים על אפליקציות צ'אט, אך הנה כמה שיקולים נוספים שחשובים במיוחד בשל רכיבי הלמידה המעשית המעורבים.

- **מנגנון לטיפול באמביוולנטיות**: מודלים של בינה מלאכותית גנרטיבית מייצרים לעיתים תשובות לא ברורות. תכונה שמאפשרת למשתמשים לבקש הבהרה יכולה לעזור במקרה בו הם נתקלים בבעיה זו.
- **שימור הקשר**: למודלים מתקדמים של בינה מלאכותית יש יכולת לזכור הקשר בתוך שיחה, דבר שיכול להיות נכס חיוני לחוויית המשתמש. מתן אפשרות למשתמשים לשלוט ולנהל את ההקשר משפר את חוויית המשתמש, אך מציב סיכון לשמירת מידע רגיש אודות המשתמש. יש לשקול כמה זמן מידע זה מאוחסן, למשל באמצעות מדיניות שמירת נתונים, כדי לאזן בין הצורך בהקשר לבין פרטיות.
- **התאמה אישית**: עם היכולת ללמוד ולהסתגל, מודלים של AI מציעים חוויה מותאמת אישית למשתמש. התאמת חוויית המשתמש דרך תכונות כמו פרופילי משתמש לא רק גורמת למשתמש להרגיש מובן, אלא גם עוזרת לו למצוא תשובות ספציפיות, וליצור אינטראקציה יעילה ומספקת יותר.

### נגישות

בין אם למשתמש יש לקויות ראייה, שמיעה, מוטוריות או קוגניטיביות, אפליקציית צ'אט מתוכננת היטב צריכה להיות ניתנת לשימוש על ידי כולם. הרשימה הבאה מפריטה תכונות ספציפיות שמטרתן לשפר את הנגישות עבור ליקויים שונים.

- **תכונות לליקויי ראייה**: נושאים בקונטרסט גבוה וטקסט ניתן להגדלה, תאימות לקורא מסך.
- **תכונות לליקויי שמיעה**: פונקציות טקסט לדיבור ודיבור לטקסט, רמזים חזותיים לתראות קוליות.
- **תכונות לליקויי מוטוריות**: תמיכה בניווט באמצעות מקלדת, פקודות קוליות.
- **תכונות לליקוי קוגניטיבי**: אפשרויות לשפה מפושטת.

## התאמה אישית וכיוונון למודלים של שפה לתחום ספציפי

דמיין אפליקציית צ'אט שמבינה את של השפה המקצועית של חברתך ומצפה לשאלות הספציפיות שמקור המשתמשים שלה בדרך כלל. יש כמה גישות שכדאי להזכיר:

- **שימוש במודלי DSL**. DSL משמעותה שפת תחום ספציפי. ניתן לנצל מודל DSL שמאומן בתחום מסוים כדי להבין את מושגיו ותסריטיו.
- **יישום כיוונון**. כיוונון הוא תהליך של אימון נוסף של המודל שלך עם נתונים ספציפיים.

## התאמה אישית: שימוש ב-DSL

שימוש במודלים לשפה של תחום ספציפי (מודלי DSL) יכול לשפר את מעורבות המשתמש על ידי מתן אינטראקציות מיוחדות, הקשריות ורלוונטיות. זהו מודל שאומן או כוונן להבין וליצור טקסט שקשור לתחום, תעשייה או נושא מסוים. אפשרויות השימוש במודל DSL משתנות מאימון חדש מאפס, לשימוש במודלים קיימים דרך SDK ו-API. אפשרות נוספת היא כיוונון, הכולל לקיחת מודל קיים שאומן מראש והתאמתו לתחום מסוים.

## התאמה אישית: יישום כיוונון

כיוונון נחשב לעיתים קרובות כאשר מודל שאומן מראש לא עומד בדרישות תחום ספציפי או משימה מסוימת.

לדוגמה, שאלות רפואיות מורכבות ודורשות הרבה הקשר. כאשר רופא מאבחן מטופל, זה מתבסס על מגוון גורמים כמו סגנון חיים או מחלות קיימות, ואפילו מסתמך על כתבי עת רפואיים עדכניים לאישור האבחנה. בסיטואציות מורכבות כאלה, אפליקציית צ'אט מונעת בינה מלאכותית כללית לא יכולה להיות מקור אמין.

### תרחיש: אפליקציה רפואית

שקול אפליקציית צ'אט שתוכננה לסייע לרופאים באמצעות מתן הפניות מהירות להנחיות טיפול, אינטראקציות בין תרופות או ממצאי מחקר אחרונים.

מודל כללי עלול להספיק למענה על שאלות רפואיות בסיסיות או מתן עצות כלליות, אך עלול להתקשות ב:

- **מקרים מאוד ספציפיים או מורכבים**. לדוגמה, נוירולוג יכול לשאול את האפליקציה: "מהן השיטות הטובות ביותר הנוכחיות לניהול אפילפסיה עמידה לתרופות בילדים?"
- **חוסר בעדכונים עדכניים**. מודל כללי עלול להתקשות לספק מענה עדכני שמשלב את ההתפתחויות האחרונות בנוירולוגיה ופרמקולוגיה.

במקרים כאלה, כיוונון מודל עם מאגר נתונים רפואי מיוחד יכול לשפר משמעותית את יכולתו להתמודד עם שאלות רפואיות מורכבות בצורה מדויקת ואמינה יותר. זה זקוק לגישה למסד נתונים גדול ורלוונטי שמייצג את האתגרים והשאלות הספציפיות של התחום.

## שיקולים לחוויית צ'אט מונעת בינה מלאכותית באיכות גבוהה

חלק זה מפרט את הקריטריונים לאפליקציות צ'אט "בעלי איכות גבוהה", שכוללים איסוף מדדים מעשיים ועמידה במסגרת שמנצלת את טכנולוגיית הבינה המלאכותית באחריות.

### מדדים מרכזיים

כדי לשמור על ביצועים איכותיים של האפליקציה, חשוב לעקוב אחר מדדים ושיקולים מרכזיים. מדידות אלו לא רק מבטיחות את פונקציונליות האפליקציה אלא גם מעריכות את איכות מודל הבינה המלאכותית וחוויית המשתמש. להלן רשימה של מדדים בסיסיים, AI וחוויית משתמש שיש לשקול.

| מדד                         | הגדרה                                                                                                               | שיקולים למפתח הצ'אט                                              |
| --------------------------- | -------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------- |
| **זמן פעילות (Uptime)**    | מודד את הזמן שהאפליקציה פעילה ונגישה למשתמשים.                                                                    | איך תפחית הפסקות פעילות?                                         |
| **זמן תגובה**              | הזמן שלוקח לאפליקציה להשיב לשאילתת משתמש.                                                                          | איך תייעל את עיבוד השאילתות לשיפור זמן התגובה?                   |
| **דיוק (Precision)**        | היחס בין תחזיות חיוביות נכונות לסך כל התחזיות החיוביות.                                                             | איך תאמת את הדיוק של המודל שלך?                                  |
| **יוזם (Recall / Sensitivity)** | היחס בין תחזיות חיוביות נכונות למספר החיוביים האמיתי.                                                                | איך תמדוד ותשפר את היוזם?                                        |
| **מדד F1**                  | הממוצע ההרמוני של הדיוק והיוזם, שמאזן בין שניהם.                                                                    | מהו מדד ה-F1 המיועד שלך? איך תאזן בין דיוק ליוזם?                |
| **פרפלקסיטי**              | מודד עד כמה התפלגות ההסתברויות שחזה המודל מסתדרת עם ההתפלגות המקורית של הנתונים.                                    | איך תקטין את הפרפלקסיטי?                                        |
| **מדדי שביעות רצון משתמש** | מודדים את תפיסת המשתמש כלפי האפליקציה. לרוב נאספים באמצעות סקרים.                                                    | באיזו תדירות תאסוף משוב ממשתמשים? איך תתאים את עצמך בהתאם?      |
| **שיעור טעויות**           | שיעור שבו המודל מבצע טעויות בהבנה או ביציאה.                                                                          | אילו אסטרטגיות יש לך כדי להקטין את שיעורי הטעויות?              |
| **מחזורי אימון מחדש**      | התדירות שבה המודל מתעדכן כדי לשלב נתונים ותובנות חדשות.                                                             | באיזו תדירות תאמן מחדש את המודל? מה גורם להתחלת מחזור אימון מחדש? |

| **זיהוי חריגות**         | כלים וטכניקות לזיהוי דפוסים לא שגרתיים שאינם תואמים את ההתנהגות הצפויה.                         | כיצד תגיב לחריגות?                                                       |

| פרטיות ואבטחה         | מערכות AI צריכות להיות מאובטחות ולכבד פרטיות.      | יישום הצפנה חזקה ואמצעי הגנה על נתונים.                              | לשמור על נתוני משתמש רגישים ולעמוד בחוקי פרטיות.                               |
