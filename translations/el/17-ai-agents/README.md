## Εισαγωγή

Οι AI Agents αποτελούν μια συναρπαστική εξέλιξη στην Παραγωγική Τεχνητή Νοημοσύνη, επιτρέποντας στα Μεγάλα Γλωσσικά Μοντέλα (LLMs) να εξελιχθούν από βοηθούς σε πράκτορες ικανούς να αναλαμβάνουν ενέργειες. Τα πλαίσια εργασίας για AI Agents δίνουν στους προγραμματιστές τη δυνατότητα να δημιουργούν εφαρμογές που παρέχουν στα LLMs πρόσβαση σε εργαλεία και διαχείριση κατάστασης. Αυτά τα πλαίσια επίσης βελτιώνουν την ορατότητα, επιτρέποντας στους χρήστες και προγραμματιστές να παρακολουθούν τις ενέργειες που σχεδιάζουν τα LLMs, βελτιώνοντας έτσι τη διαχείριση της εμπειρίας.

Το μάθημα θα καλύψει τις ακόλουθες περιοχές:

- Κατανόηση τι είναι ένας AI Agent - Τι ακριβώς είναι ένας AI Agent;
- Εξερεύνηση πέντε διαφορετικών πλαισίων εργασίας AI Agent - Τι τα κάνει μοναδικά;
- Εφαρμογή αυτών των AI Agents σε διάφορα σενάρια χρήσης - Πότε πρέπει να χρησιμοποιούμε AI Agents;

- Καταλάβετε πώς λειτουργούν οι AI Agents ώστε να μπορείτε να χτίζετε εφαρμογές με αυτούς.

## Τι Είναι οι AI Agents;

Οι AI Agents είναι ένας πολύ συναρπαστικός τομέας στον κόσμο της Παραγωγικής Τεχνητής Νοημοσύνης. Με αυτή τη συναρπαστική εξέλιξη εμφανίζεται μερικές φορές σύγχυση σχετικά με τους όρους και την εφαρμογή τους. Για να κρατήσουμε τα πράγματα απλά και περιλαμβάνοντας τα περισσότερα από τα εργαλεία που αναφέρονται σε AI Agents, θα χρησιμοποιήσουμε τον ακόλουθο ορισμό:

Οι AI Agents επιτρέπουν στα Μεγάλα Γλωσσικά Μοντέλα (LLMs) να εκτελούν εργασίες δίνοντάς τους πρόσβαση σε μια **κατάσταση** και **εργαλεία**.

![Μοντέλο Πράκτορα](../../../translated_images/el/what-agent.21f2893bdfd01e6a.webp)

Ας ορίσουμε αυτούς τους όρους:

**Μεγάλα Γλωσσικά Μοντέλα** - Αυτά είναι τα μοντέλα που αναφέρονται σε όλο το μάθημα, όπως το GPT-3.5, GPT-4, Llama-2, κ.ά.

**Κατάσταση** - Αναφέρεται στο πλαίσιο μέσα στο οποίο εργάζεται το LLM. Το LLM χρησιμοποιεί το πλαίσιο των προηγούμενων ενεργειών του και το τρέχον πλαίσιο, καθοδηγώντας τη λήψη αποφάσεων για τις επόμενες ενέργειες. Τα πλαίσια εργασίας για AI Agents διευκολύνουν τους προγραμματιστές να διατηρούν αυτό το πλαίσιο πιο εύκολα.

**Εργαλεία** - Για να ολοκληρωθεί η εργασία που έχει ζητήσει ο χρήστης και έχει σχεδιάσει το LLM, το LLM χρειάζεται πρόσβαση σε εργαλεία. Παραδείγματα εργαλείων μπορεί να είναι μια βάση δεδομένων, ένα API, μια εξωτερική εφαρμογή ή ακόμη και άλλο LLM!

Αυτοί οι ορισμοί θα σας δώσουν, ελπίζουμε, ένα καλό υπόβαθρο καθώς προχωράμε βλέποντας πώς υλοποιούνται. Ας εξερευνήσουμε μερικά διαφορετικά πλαίσια εργασίας AI Agent:

## AutoGen

Το επόμενο πλαίσιο εργασίας AI Agent που θα συζητήσουμε είναι το [AutoGen](https://microsoft.github.io/autogen/?WT.mc_id=academic-105485-koreyst). Η κύρια εστίαση του AutoGen είναι οι συνομιλίες. Οι πράκτορες είναι και **συναντίληπτοι** και **προσαρμόσιμοι**.

**Συναντίληπτοι -** Τα LLMs μπορούν να ξεκινήσουν και να συνεχίζουν μια συνομιλία με άλλο LLM για να ολοκληρώσουν μια εργασία. Αυτό γίνεται δημιουργώντας `AssistantAgents` και δίνοντάς τους ένα συγκεκριμένο μήνυμα συστήματος.

```python

autogen.AssistantAgent( name="Coder", llm_config=llm_config, ) pm = autogen.AssistantAgent( name="Product_manager", system_message="Creative in software product ideas.", llm_config=llm_config, )

```

**Προσαρμόσιμοι** - Οι πράκτορες μπορούν να οριστούν όχι μόνο ως LLM αλλά και ως χρήστης ή εργαλείο. Ως προγραμματιστής, μπορείτε να ορίσετε έναν `UserProxyAgent` που είναι υπεύθυνος για την αλληλεπίδραση με τον χρήστη για ανατροφοδότηση στην ολοκλήρωση μιας εργασίας. Αυτή η ανατροφοδότηση μπορεί είτε να συνεχίσει την εκτέλεση της εργασίας είτε να τη διακόψει.

```python
user_proxy = UserProxyAgent(name="user_proxy")
```

## Πλαίσιο Εργασίας Microsoft Agent

Το [Microsoft Agent Framework](https://learn.microsoft.com/agent-framework/?WT.mc_id=academic-105485-koreyst) είναι το ανοιχτό SDK της Microsoft για την κατασκευή AI Agents και πολυ-πρακτορικών συστημάτων τόσο σε **Python** όσο και σε **.NET**. Συνδυάζει τα πλεονεκτήματα δύο προηγούμενων έργων της Microsoft — τις επιχειρηματικές λειτουργίες του **Semantic Kernel** και την πολυ-πρακτορική ορχήστρωση του **AutoGen** — σε ένα ενιαίο, υποστηριζόμενο πλαίσιο. Αν ξεκινάτε ένα νέο project πράκτορα σήμερα, αυτό είναι ο συνιστώμενος διάδοχος του AutoGen.

Το πλαίσιο επεκτείνεται από έναν μόνο **πράκτορα συνομιλίας** μέχρι πολύπλοκα **πολυ-πρακτορικά ροή εργασιών**, και ενσωματώνεται απευθείας με το Microsoft Foundry, Azure OpenAI και OpenAI. Παρέχει επίσης ενσωματωμένη παρατηρησιμότητα μέσω OpenTelemetry ώστε να μπορείτε να παρακολουθείτε ακριβώς τι κάνουν οι πράκτορές σας.

### Κατάσταση και Εργαλεία

**Κατάσταση** - Το πλαίσιο διαχειρίζεται το πλαίσιο της συνομιλίας για εσάς μέσω **νημάτων**. Ένας πράκτορας παρακολουθεί το ιστορικό μηνυμάτων (αιτήματα χρήστη, κλήσεις εργαλείων και τα αποτελέσματά τους), ώστε κάθε γύρος να βασίζεται στους προηγούμενους. Τα νήματα μπορούν επίσης να διατηρηθούν, επιτρέποντας σε μια συνομιλία να παύσει και να συνεχιστεί αργότερα.

**Εργαλεία** - Δίνετε σε έναν πράκτορα εργαλεία περνώντας απλές συναρτήσεις Python. Οι παράμετροι με τύπους μετατρέπονται αυτόματα σε σχήμα, ώστε το μοντέλο να γνωρίζει πώς και πότε να τις καλεί (function calling). Το πλαίσιο υποστηρίζει επίσης διακομιστές Μεταγραφικού Πρωτοκόλλου Μοντέλου (Model Context Protocol - MCP) και φιλοξενούμενα εργαλεία όπως διερμηνέα κώδικα.

Ακολουθεί ένα παράδειγμα ενός μόνο πράκτορα με ένα προσαρμοσμένο εργαλείο:

```python
import asyncio
from typing import Annotated

from pydantic import Field
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient


def get_weather(
    location: Annotated[str, Field(description="The location to get the weather for.")],
) -> str:
    """Get the weather for a given location."""
    return f"The weather in {location} is sunny with a high of 22°C."


async def main():
    agent = Agent(
        client=OpenAIChatClient(),
        instructions="You are a helpful assistant that can answer weather questions.",
        tools=[get_weather],
    )

    response = await agent.run("What's the weather in Amsterdam?")
    print(response)


asyncio.run(main())
```

Για να συνδεθείτε στο Azure OpenAI στο Microsoft Foundry, περάστε το endpoint και τα διαπιστευτήρια στον πελάτη:

```python
from azure.identity.aio import AzureCliCredential

```

### Πολυ-πρακτορικά ροή εργασιών

Εκεί που το πλαίσιο πραγματικά ξεχωρίζει είναι η ορχήστρωση πολλών πρακτόρων μαζί. Για παράδειγμα, μπορείτε να εκτελέσετε πράκτορες ο ένας μετά τον άλλο (με κάθε πράκτορα να περνά το πλαίσιο του στον επόμενο) ή να διαχειριστείτε πολλούς παράλληλα και να συγκεντρώσετε τα αποτελέσματά τους:

```python
from agent_framework.orchestrations import SequentialBuilder, ConcurrentBuilder

# Εκτελέστε τους πράκτορες διαδοχικά, περνώντας το πλαίσιο της συνομιλίας κατά μήκος της αλυσίδας
sequential = SequentialBuilder(participants=[researcher, writer, editor]).build()

# Διαχωριστείτε σε παράλληλους πράκτορες και στη συνέχεια συγκεντρώστε τις απαντήσεις τους
concurrent = ConcurrentBuilder(participants=[analyst_a, analyst_b, analyst_c]).build()
```

## Taskweaver

Το επόμενο πλαίσιο πράκτορα που θα εξερευνήσουμε είναι το [Taskweaver](https://microsoft.github.io/TaskWeaver/?WT.mc_id=academic-105485-koreyst). Είναι γνωστό ως πράκτορας "με προτεραιότητα στον κώδικα" επειδή αντί να εργάζεται αυστηρά με `αλφαριθμητικά` , μπορεί να εργαστεί με DataFrames σε Python. Αυτό γίνεται εξαιρετικά χρήσιμο για εργασίες ανάλυσης και δημιουργίας δεδομένων. Αυτό μπορεί να περιλαμβάνει δημιουργία διαγραμμάτων και γραφημάτων ή παραγωγή τυχαίων αριθμών.

### Κατάσταση και Εργαλεία

Για να διαχειριστεί την κατάσταση της συνομιλίας, το TaskWeaver χρησιμοποιεί την έννοια του `Planner`. Ο `Planner` είναι ένα LLM που παίρνει το αίτημα από τους χρήστες και χαρτογραφεί τις εργασίες που πρέπει να ολοκληρωθούν για την ικανοποίηση αυτού του αιτήματος.

Για να ολοκληρώσει τις εργασίες, ο `Planner` έχει πρόσβαση στη συλλογή εργαλείων που ονομάζονται `Plugins`. Αυτά μπορεί να είναι κλάσεις Python ή ένας γενικός διερμηνέας κώδικα. Αυτά τα plugins αποθηκεύονται ως ενσωματώσεις (embeddings) ώστε το LLM να μπορεί να αναζητεί πιο αποτελεσματικά το σωστό plugin.

![Taskweaver](../../../translated_images/el/taskweaver.da8559999267715a.webp)

Ακολουθεί ένα παράδειγμα plugin για ανίχνευση ανωμαλιών:

```python
class AnomalyDetectionPlugin(Plugin): def __call__(self, df: pd.DataFrame, time_col_name: str, value_col_name: str):
```

Ο κώδικας ελέγχεται πριν εκτελεστεί. Ένα ακόμη χαρακτηριστικό για τη διαχείριση του πλαισίου στο Taskweaver είναι η `εμπειρία`. Η εμπειρία επιτρέπει την αποθήκευση του πλαισίου μιας συνομιλίας μακροπρόθεσμα σε ένα αρχείο YAML. Αυτό μπορεί να ρυθμιστεί ώστε το LLM να βελτιώνεται με τον καιρό σε ορισμένες εργασίες, δεδομένου ότι εκτίθεται σε προηγούμενες συνομιλίες.

## JARVIS

Το τελευταίο πλαίσιο πράκτορα που θα εξερευνήσουμε είναι το [JARVIS](https://github.com/microsoft/JARVIS?tab=readme-ov-file&WT.mc_id=academic-105485-koreyst). Αυτό που κάνει το JARVIS μοναδικό είναι ότι χρησιμοποιεί ένα LLM για να διαχειριστεί την `κατάσταση` της συνομιλίας και τα `εργαλεία` είναι άλλα μοντέλα Τεχνητής Νοημοσύνης. Κάθε ένα από τα μοντέλα AI είναι εξειδικευμένο μοντέλο που εκτελεί συγκεκριμένες εργασίες όπως ανίχνευση αντικειμένων, απομαγνητοφώνηση ή λεζαντογράφηση εικόνων.

![JARVIS](../../../translated_images/el/jarvis.762ddbadbd1a3a33.webp)

Το LLM, ως γενικό μοντέλο χρήσης, λαμβάνει το αίτημα από τον χρήστη και αναγνωρίζει την συγκεκριμένη εργασία και όποια επιχειρήματα/δεδομένα χρειάζονται για να ολοκληρωθεί η εργασία.

```python
[{"task": "object-detection", "id": 0, "dep": [-1], "args": {"image": "e1.jpg" }}]
```

Το LLM στη συνέχεια διαμορφώνει το αίτημα με τρόπο που μπορεί να ερμηνεύσει το εξειδικευμένο μοντέλο AI, όπως JSON. Μόλις το μοντέλο AI επιστρέψει την πρόβλεψή του βάσει της εργασίας, το LLM λαμβάνει την απάντηση.

Εάν απαιτούνται πολλά μοντέλα για να ολοκληρωθεί η εργασία, το LLM θα ερμηνεύσει επίσης τις απαντήσεις από αυτά τα μοντέλα πριν τα συνδυάσει για να παράγει την απάντηση προς τον χρήστη.
