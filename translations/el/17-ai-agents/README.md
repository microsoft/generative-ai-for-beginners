## Εισαγωγή

Οι Πράκτορες Τεχνητής Νοημοσύνης αντιπροσωπεύουν μια συναρπαστική εξέλιξη στην Παραγωγική Τεχνητή Νοημοσύνη, επιτρέποντας στα Μεγάλα Μοντέλα Γλώσσας (LLMs) να εξελιχθούν από βοηθούς σε πράκτορες ικανούς να αναλαμβάνουν δράση. Τα πλαίσια Πρακτόρων ΤΝ επιτρέπουν στους προγραμματιστές να δημιουργούν εφαρμογές που δίνουν στα LLM πρόσβαση σε εργαλεία και διαχείριση κατάστασης. Αυτά τα πλαίσια επίσης ενισχύουν την ορατότητα, επιτρέποντας σε χρήστες και προγραμματιστές να παρακολουθούν τις δράσεις που σχεδιάζουν τα LLM, βελτιώνοντας έτσι τη διαχείριση της εμπειρίας.

Το μάθημα θα καλύψει τους ακόλουθους τομείς:

- Κατανόηση του τι είναι ένας Πράκτορας ΤΝ - Τι ακριβώς είναι ένας Πράκτορας ΤΝ;
- Εξερεύνηση πέντε διαφορετικών Πλαισίων Πρακτόρων ΤΝ - Τι τα καθιστά μοναδικά;
- Εφαρμογή αυτών των Πρακτόρων ΤΝ σε διαφορετικές περιπτώσεις χρήσης - Πότε πρέπει να χρησιμοποιούμε Πράκτορες ΤΝ;

## Στόχοι μάθησης

Μετά την παρακολούθηση αυτού του μαθήματος, θα μπορείτε να:

- Κατανοήσετε τις διαφορές μεταξύ μερικών από τα δημοφιλή Πλαίσια Πρακτόρων ΤΝ και πώς διαφέρουν.
- Κατανοήσετε πώς λειτουργούν οι Πράκτορες ΤΝ για να δημιουργήσετε εφαρμογές με αυτούς.

## Τι είναι οι Πράκτορες ΤΝ;

Οι Πράκτορες ΤΝ είναι ένας πολύ συναρπαστικός τομέας στον κόσμο της Παραγωγικής Τεχνητής Νοημοσύνης. Με αυτή τη συναρπαστικότητα έρχεται και κάποιες φορές μια σύγχυση όρων και της εφαρμογής τους. Για να κρατήσουμε τα πράγματα απλά και περιλαμβάνοντας τα περισσότερα από τα εργαλεία που αναφέρονται στους Πράκτορες ΤΝ, θα χρησιμοποιήσουμε τον εξής ορισμό:

Οι Πράκτορες ΤΝ επιτρέπουν στα Μεγάλα Μοντέλα Γλώσσας (LLMs) να εκτελούν εργασίες δίνοντάς τους πρόσβαση σε μια **κατάσταση** και **εργαλεία**.

![Agent Model](../../../translated_images/el/what-agent.21f2893bdfd01e6a.webp)

Ας ορίσουμε αυτούς τους όρους:

**Μεγάλα Μοντέλα Γλώσσας** - Αυτά είναι τα μοντέλα που αναφέρονται σε όλο το μάθημα όπως το GPT-5, GPT-4o, και Llama 3.3, κ.ά.

**Κατάσταση** - Αυτό αναφέρεται στο πλαίσιο στο οποίο εργάζεται το LLM. Το LLM χρησιμοποιεί το πλαίσιο των προηγούμενων ενεργειών του και το τρέχον πλαίσιο, καθοδηγώντας τη λήψη αποφάσεων για τις επόμενες ενέργειες. Τα Πλαίσια Πρακτόρων ΤΝ επιτρέπουν στους προγραμματιστές να διαχειρίζονται αυτό το πλαίσιο πιο εύκολα.

**Εργαλεία** - Για να ολοκληρώσει την εργασία που έχει ζητήσει ο χρήστης και που έχει σχεδιάσει το LLM, το LLM χρειάζεται πρόσβαση σε εργαλεία. Παραδείγματα εργαλείων μπορεί να είναι μια βάση δεδομένων, ένα API, μια εξωτερική εφαρμογή ή ακόμη και ένα άλλο LLM!

Αυτοί οι ορισμοί, ελπίζουμε, να σας δώσουν καλή βάση καθώς θα δούμε πώς υλοποιούνται. Ας εξερευνήσουμε μερικά διαφορετικά πλαίσια Πρακτόρων ΤΝ:

## Πράκτορες LangChain

Οι [Πράκτορες LangChain](https://python.langchain.com/docs/how_to/#agents?WT.mc_id=academic-105485-koreyst) είναι μια υλοποίηση των ορισμών που δώσαμε παραπάνω.

Για τη διαχείριση της **κατάστασης**, χρησιμοποιούν μια ενσωματωμένη λειτουργία που ονομάζεται `AgentExecutor`. Αυτή αποδέχεται τον ορισμένο `agent` και τα `εργαλεία` που είναι διαθέσιμα σε αυτόν.

Ο `Agent Executor` επίσης αποθηκεύει το ιστορικό συνομιλίας για να παρέχει το πλαίσιο της συζήτησης.

![Langchain Agents](../../../translated_images/el/langchain-agents.edcc55b5d5c43716.webp)

Το LangChain προσφέρει έναν [κατάλογο εργαλείων](https://integrations.langchain.com/tools?WT.mc_id=academic-105485-koreyst) που μπορούν να εισαχθούν στην εφαρμογή σας και στα οποία το LLM μπορεί να έχει πρόσβαση. Αυτά είναι φτιαγμένα από την κοινότητα και από την ομάδα του LangChain.

Μπορείτε στη συνέχεια να ορίσετε αυτά τα εργαλεία και να τα περάσετε στον `Agent Executor`.

Η ορατότητα είναι επίσης μια σημαντική πτυχή όταν μιλάμε για Πράκτορες ΤΝ. Είναι σημαντικό για τους προγραμματιστές εφαρμογών να κατανοούν ποιο εργαλείο χρησιμοποιεί το LLM και γιατί. Γι' αυτό, η ομάδα στο LangChain ανέπτυξε το LangSmith.

## AutoGen

Το επόμενο πλαίσιο Πρακτόρων ΤΝ που θα συζητήσουμε είναι το [AutoGen](https://microsoft.github.io/autogen/?WT.mc_id=academic-105485-koreyst). Ο βασικός στόχος του AutoGen είναι οι συνομιλίες. Οι πράκτορες είναι ταυτόχρονα **συνομιλητικοί** και **προσαρμόσιμοι**.

**Συνομιλητικοί -** Τα LLM μπορούν να ξεκινήσουν και να συνεχίσουν μια συνομιλία με άλλο LLM για να ολοκληρώσουν μια εργασία. Αυτό γίνεται δημιουργώντας `AssistantAgents` και δίνοντάς τους ένα συγκεκριμένο μήνυμα συστήματος.

```python

autogen.AssistantAgent( name="Coder", llm_config=llm_config, ) pm = autogen.AssistantAgent( name="Product_manager", system_message="Creative in software product ideas.", llm_config=llm_config, )

```

**Προσαρμόσιμοι** - Οι πράκτορες μπορούν να οριστούν όχι μόνο ως LLM αλλά και ως χρήστης ή εργαλείο. Ως προγραμματιστής, μπορείτε να ορίσετε έναν `UserProxyAgent` που είναι υπεύθυνος για την αλληλεπίδραση με τον χρήστη για ανατροφοδότηση στην ολοκλήρωση μιας εργασίας. Αυτή η ανατροφοδότηση μπορεί είτε να συνεχίσει την εκτέλεση της εργασίας είτε να την σταματήσει.

```python
user_proxy = UserProxyAgent(name="user_proxy")
```

Εδώ είναι ένα παράδειγμα της διαδικασίας:

Ανάλογα με τη διαμόρφωσή σας, αυτή η λειτουργία μπορεί να εκτελεστεί αυτόματα και να διαβαστεί από τον Πράκτορα ή να εκτελεστεί βάσει εισόδου χρήστη.

## Πλαίσιο Πρακτόρων Microsoft

Το [Πλαίσιο Πρακτόρων Microsoft](https://learn.microsoft.com/agent-framework/?WT.mc_id=academic-105485-koreyst) είναι το ανοικτού κώδικα SDK της Microsoft για την κατασκευή Πρακτόρων ΤΝ και συστημάτων με πολλούς πράκτορες σε **Python** και **.NET**. Συνδυάζει τις δυνατότητες δύο προηγούμενων έργων της Microsoft — τα χαρακτηριστικά επιχείρησης του **Semantic Kernel** και τον συντονισμό πολλών πρακτόρων του **AutoGen** — σε ένα ενιαίο, υποστηριζόμενο πλαίσιο. Αν ξεκινάτε ένα νέο έργο πράκτορα σήμερα, αυτό είναι ο συνιστώμενος διάδοχος του AutoGen.

Το πλαίσιο κλιμακώνεται από έναν μεμονωμένο **πράκτορα συνομιλίας** μέχρι περίπλοκα **σενάρια εργασίας με πολλούς πράκτορες**, και ενσωματώνεται άμεσα με το Microsoft Foundry, το Azure OpenAI και το OpenAI. Παρέχει επίσης ενσωματωμένη παρακολούθηση μέσω OpenTelemetry ώστε να μπορείτε να εντοπίσετε ακριβώς τι κάνουν οι πράκτορές σας.

### Κατάσταση και Εργαλεία

**Κατάσταση** - Το πλαίσιο διαχειρίζεται το πλαίσιο συνομιλίας για εσάς μέσω **νημάτων**. Ένας πράκτορας παρακολουθεί το ιστορικό μηνυμάτων (αιτήματα χρήστη, κλήσεις εργαλείων και τα αποτελέσματά τους), ώστε κάθε βήμα να βασίζεται στα προηγούμενα. Τα νήματα μπορούν επίσης να αποθηκευτούν, επιτρέποντας την παύση και την συνέχιση μιας συνομιλίας αργότερα.

**Εργαλεία** - Δίνετε σε έναν πράκτορα εργαλεία περάζοντας απλές συναρτήσεις Python. Οι παράμετροι με τύπους μετατρέπονται αυτόματα σε σχήμα, ώστε το μοντέλο να γνωρίζει πώς και πότε να τις καλέσει (function calling). Το πλαίσιο υποστηρίζει επίσης διακομιστές Model Context Protocol (MCP) και φιλοξενούμενα εργαλεία όπως έναν διερμηνέα κώδικα.

Εδώ είναι ένα παράδειγμα ενός μεμονωμένου πράκτορα με ένα προσαρμοσμένο εργαλείο:

```python
import asyncio
from typing import Annotated

from pydantic import Field
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient


def get_weather(
    location: Annotated[str, Field(description="The location to get the weather for.")],
) -> str:
    """Get the weather for a given location."""
    return f"The weather in {location} is sunny with a high of 22°C."


async def main():
    agent = Agent(
        client=OpenAIChatClient(),
        instructions="You are a helpful assistant that can answer weather questions.",
        tools=[get_weather],
    )

    response = await agent.run("What's the weather in Amsterdam?")
    print(response)


asyncio.run(main())
```

### Σενάρια εργασίας με πολλούς πράκτορες

Εκεί όπου το πλαίσιο πραγματικά ξεχωρίζει είναι στον συντονισμό πολλών πρακτόρων μαζί. Για παράδειγμα, μπορείτε να τρέξετε πράκτορες ο ένας μετά τον άλλον (με κάθε έναν να περνά το πλαίσιο του στον επόμενο) ή να διαχωρίσετε σε πολλούς πράκτορες παράλληλα και να συγκεντρώσετε τα αποτελέσματά τους:

```python
from agent_framework.orchestrations import SequentialBuilder, ConcurrentBuilder

# Εκτελέστε πράκτορες διαδοχικά, μεταβιβάζοντας το περιβάλλον της συνομιλίας κατά μήκος της αλυσίδας
sequential = SequentialBuilder(participants=[researcher, writer, editor]).build()

# Διακλαδώστε προς τους πράκτορες παράλληλα και στη συνέχεια συγκεντρώστε τις απαντήσεις τους
concurrent = ConcurrentBuilder(participants=[analyst_a, analyst_b, analyst_c]).build()
```

## Taskweaver

Το επόμενο πλαίσιο πράκτορα που θα εξερευνήσουμε είναι το [Taskweaver](https://microsoft.github.io/TaskWeaver/?WT.mc_id=academic-105485-koreyst). Είναι γνωστό ως "code-first" πράκτορας επειδή αντί να δουλεύει αυστηρά με `strings`, μπορεί να δουλέψει με DataFrames στην Python. Αυτό γίνεται εξαιρετικά χρήσιμο για εργασίες ανάλυσης δεδομένων και παραγωγής. Αυτό μπορεί να είναι όπως η δημιουργία γραφημάτων και διαγραμμάτων ή η παραγωγή τυχαίων αριθμών.

### Κατάσταση και Εργαλεία

Για να διαχειριστεί την κατάσταση της συνομιλίας, το TaskWeaver χρησιμοποιεί την έννοια ενός `Planner`. Ο `Planner` είναι ένα LLM που λαμβάνει το αίτημα από τους χρήστες και χαρτογραφεί τις εργασίες που πρέπει να ολοκληρωθούν για να ικανοποιηθεί αυτό το αίτημα.

Για την ολοκλήρωση των εργασιών, ο `Planner` εκτίθεται στη συλλογή εργαλείων που ονομάζονται `Plugins`. Αυτά μπορεί να είναι κλάσεις Python ή ένας γενικός διερμηνέας κώδικα. αυτά τα plugins αποθηκεύονται ως embeddings ώστε το LLM να μπορεί να αναζητεί καλύτερα το σωστό plugin.

![Taskweaver](../../../translated_images/el/taskweaver.da8559999267715a.webp)

Εδώ είναι ένα παράδειγμα plugin για την ανίχνευση ανωμαλιών:

```python
class AnomalyDetectionPlugin(Plugin): def __call__(self, df: pd.DataFrame, time_col_name: str, value_col_name: str):
```

Ο κώδικας επαληθεύεται πριν εκτελεστεί. Μια άλλη λειτουργία για τη διαχείριση του πλαισίου στο Taskweaver είναι το `experience`. Η εμπειρία επιτρέπει το πλαίσιο μιας συνομιλίας να αποθηκεύεται μακροπρόθεσμα σε ένα αρχείο YAML. Αυτό μπορεί να διαμορφωθεί ώστε το LLM να βελτιώνεται με το χρόνο σε ορισμένες εργασίες αφού εκτίθεται σε προηγούμενες συνομιλίες.

## JARVIS

Το τελευταίο πλαίσιο πράκτορα που θα εξερευνήσουμε είναι το [JARVIS](https://github.com/microsoft/JARVIS?tab=readme-ov-file&WT.mc_id=academic-105485-koreyst). Αυτό που κάνει το JARVIS μοναδικό είναι ότι χρησιμοποιεί ένα LLM για να διαχειριστεί τη `κατάσταση` της συνομιλίας και τα `εργαλεία` είναι άλλα μοντέλα ΤΝ. Κάθε ένα από τα μοντέλα ΤΝ είναι εξειδικευμένο σε ορισμένες εργασίες όπως ανίχνευση αντικειμένων, απομαγνητοφώνηση ή δημιουργία λεζάντας εικόνας.

![JARVIS](../../../translated_images/el/jarvis.762ddbadbd1a3a33.webp)

Το LLM, ως μοντέλο γενικού σκοπού, λαμβάνει το αίτημα από τον χρήστη και αναγνωρίζει την συγκεκριμένη εργασία και οποιαδήποτε επιχειρήματα/δεδομένα χρειάζονται για να ολοκληρωθεί η εργασία.

```python
[{"task": "object-detection", "id": 0, "dep": [-1], "args": {"image": "e1.jpg" }}]
```

Το LLM στη συνέχεια μορφοποιεί το αίτημα με τρόπο που το εξειδικευμένο μοντέλο ΤΝ μπορεί να ερμηνεύσει, όπως JSON. Μόλις το μοντέλο ΤΝ επιστρέψει την πρόβλεψή του βάσει της εργασίας, το LLM λαμβάνει την απάντηση.

Αν απαιτούνται πολλά μοντέλα για να ολοκληρωθεί η εργασία, το LLM θα ερμηνεύσει επίσης τις απαντήσεις από αυτά τα μοντέλα πριν τις συνδυάσει για να δημιουργήσει την απάντηση προς τον χρήστη.
