# Δημιουργία Εφαρμογών Αναζήτησης

> > _Κάντε κλικ στην εικόνα παραπάνω για να δείτε το βίντεο αυτού του μαθήματος_

Υπάρχουν περισσότερα στα LLMs από chatbots και δημιουργία κειμένου. Είναι επίσης δυνατό να δημιουργήσετε εφαρμογές αναζήτησης χρησιμοποιώντας Ενσωματώσεις. Οι Ενσωματώσεις είναι αριθμητικές αναπαραστάσεις δεδομένων γνωστές και ως διανύσματα, και μπορούν να χρησιμοποιηθούν για σημασιολογική αναζήτηση δεδομένων.

Σε αυτό το μάθημα, θα δημιουργήσετε μια εφαρμογή αναζήτησης για την εκπαιδευτική μας νεοφυή επιχείρηση. Η νεοφυής μας επιχείρηση είναι ένας μη κερδοσκοπικός οργανισμός που παρέχει δωρεάν εκπαίδευση σε μαθητές σε αναπτυσσόμενες χώρες. Η νεοφυής μας επιχείρηση έχει μεγάλο αριθμό βίντεο στο YouTube που οι μαθητές μπορούν να χρησιμοποιήσουν για να μάθουν για την AI. Η νεοφυής μας επιχείρηση θέλει να δημιουργήσει μια εφαρμογή αναζήτησης που επιτρέπει στους μαθητές να αναζητούν ένα βίντεο στο YouTube πληκτρολογώντας μια ερώτηση.

Για παράδειγμα, ένας μαθητής μπορεί να πληκτρολογήσει 'Τι είναι τα Jupyter Notebooks;' ή 'Τι είναι το Azure ML' και η εφαρμογή αναζήτησης θα επιστρέψει μια λίστα με βίντεο στο YouTube που είναι σχετικά με την ερώτηση, και ακόμη καλύτερα, η εφαρμογή αναζήτησης θα επιστρέψει έναν σύνδεσμο στο σημείο του βίντεο όπου βρίσκεται η απάντηση στην ερώτηση.

## Εισαγωγή

Σε αυτό το μάθημα, θα καλύψουμε:

- Σημασιολογική έναντι Αναζήτησης με λέξεις-κλειδιά.
- Τι είναι οι Ενσωματώσεις Κειμένου.
- Δημιουργία ενός Δείκτη Ενσωματώσεων Κειμένου.
- Αναζήτηση σε έναν Δείκτη Ενσωματώσεων Κειμένου.

## Στόχοι Μάθησης

Αφού ολοκληρώσετε αυτό το μάθημα, θα μπορείτε να:

- Διακρίνετε τη διαφορά μεταξύ σημασιολογικής και αναζήτησης με λέξεις-κλειδιά.
- Εξηγήσετε τι είναι οι Ενσωματώσεις Κειμένου.
- Δημιουργήσετε μια εφαρμογή χρησιμοποιώντας Ενσωματώσεις για αναζήτηση δεδομένων.

## Γιατί να δημιουργήσετε μια εφαρμογή αναζήτησης;

Η δημιουργία μιας εφαρμογής αναζήτησης θα σας βοηθήσει να κατανοήσετε πώς να χρησιμοποιήσετε Ενσωματώσεις για αναζήτηση δεδομένων. Θα μάθετε επίσης πώς να δημιουργήσετε μια εφαρμογή αναζήτησης που μπορεί να χρησιμοποιηθεί από μαθητές για γρήγορη εύρεση πληροφοριών.

Το μάθημα περιλαμβάνει έναν Δείκτη Ενσωματώσεων των απομαγνητοφωνήσεων του YouTube για το κανάλι Microsoft [AI Show](https://www.youtube.com/playlist?list=PLlrxD0HtieHi0mwteKBOfEeOYf0LJU4O1). Το AI Show είναι ένα κανάλι YouTube που σας διδάσκει για την AI και τη μηχανική μάθηση. Ο Δείκτης Ενσωματώσεων περιέχει τις Ενσωματώσεις για κάθε μία από τις απομαγνητοφωνήσεις του YouTube έως τον Οκτώβριο 2023. Θα χρησιμοποιήσετε τον Δείκτη Ενσωματώσεων για να δημιουργήσετε μια εφαρμογή αναζήτησης για την νεοφυή μας επιχείρηση. Η εφαρμογή αναζήτησης επιστρέφει έναν σύνδεσμο στο σημείο του βίντεο όπου βρίσκεται η απάντηση στην ερώτηση. Αυτός είναι ένας εξαιρετικός τρόπος για τους μαθητές να βρίσκουν τις πληροφορίες που χρειάζονται γρήγορα.

Ακολουθεί ένα παράδειγμα σημασιολογικής ερώτησης για την ερώτηση 'μπορείτε να χρησιμοποιήσετε το rstudio με το azure ml;'. Δείτε το url του YouTube, θα δείτε ότι το url περιέχει μια χρονική σήμανση που σας οδηγεί στο σημείο του βίντεο όπου βρίσκεται η απάντηση στην ερώτηση.

## Τι είναι η σημασιολογική αναζήτηση;

Τώρα μπορεί να αναρωτιέστε, τι είναι η σημασιολογική αναζήτηση; Η σημασιολογική αναζήτηση είναι μια τεχνική αναζήτησης που χρησιμοποιεί τη σημασιολογία, ή το νόημα, των λέξεων σε μια ερώτηση για να επιστρέψει σχετικά αποτελέσματα.

Εδώ είναι ένα παράδειγμα σημασιολογικής αναζήτησης. Ας πούμε ότι ψάχνετε να αγοράσετε ένα αυτοκίνητο, μπορεί να αναζητήσετε 'το όνειρο αυτοκίνητό μου', η σημασιολογική αναζήτηση κατανοεί ότι δεν `dreaming` για ένα αυτοκίνητο, αλλά μάλλον ψάχνετε να αγοράσετε το `ideal` αυτοκίνητο. Η σημασιολογική αναζήτηση κατανοεί την πρόθεσή σας και επιστρέφει σχετικά αποτελέσματα. Η εναλλακτική είναι `keyword search` που θα έψαχνε κυριολεκτικά για όνειρα σχετικά με αυτοκίνητα και συχνά επιστρέφει άσχετα αποτελέσματα.

## Τι είναι οι Ενσωματώσεις Κειμένου;

[Οι ενσωματώσεις κειμένου](https://en.wikipedia.org/wiki/Word_embedding?WT.mc_id=academic-105485-koreyst) είναι μια τεχνική αναπαράστασης κειμένου που χρησιμοποιείται στην [επεξεργασία φυσικής γλώσσας](https://en.wikipedia.org/wiki/Natural_language_processing?WT.mc_id=academic-105485-koreyst). Οι ενσωματώσεις κειμένου είναι σημασιολογικές αριθμητικές αναπαραστάσεις κειμένου. Οι ενσωματώσεις χρησιμοποιούνται για την αναπαράσταση δεδομένων με τρόπο που είναι εύκολο για μια μηχανή να κατανοήσει. Υπάρχουν πολλά μοντέλα για την κατασκευή ενσωματώσεων κειμένου, σε αυτό το μάθημα, θα επικεντρωθούμε στη δημιουργία ενσωματώσεων χρησιμοποιώντας το Μοντέλο Ενσωμάτωσης της OpenAI.

Εδώ είναι ένα παράδειγμα, φανταστείτε ότι το ακόλουθο κείμενο βρίσκεται σε μια απομαγνητοφώνηση από ένα επεισόδιο στο κανάλι AI Show του YouTube:

```text
Today we are going to learn about Azure Machine Learning.
```

Θα περάσουμε το κείμενο στο OpenAI Embedding API και θα επιστρέψει την ακόλουθη ενσωμάτωση που αποτελείται από 1536 αριθμούς γνωστούς και ως διανύσματα. Κάθε αριθμός στο διάνυσμα αντιπροσωπεύει μια διαφορετική πτυχή του κειμένου. Για συντομία, εδώ είναι οι πρώτοι 10 αριθμοί στο διάνυσμα.

```python
[-0.006655829958617687, 0.0026128944009542465, 0.008792596869170666, -0.02446001023054123, -0.008540431968867779, 0.022071078419685364, -0.010703742504119873, 0.003311325330287218, -0.011632772162556648, -0.02187200076878071, ...]
```

## Πώς δημιουργείται ο Δείκτης Ενσωματώσεων;

Ο Δείκτης Ενσωματώσεων για αυτό το μάθημα δημιουργήθηκε με μια σειρά από σενάρια Python. Θα βρείτε τα σενάρια μαζί με οδηγίες στο [README](./scripts/README.md?WT.mc_id=academic-105485-koreyst) στον φάκελο 'scripts' για αυτό το μάθημα. Δεν χρειάζεται να εκτελέσετε αυτά τα σενάρια για να ολοκληρώσετε αυτό το μάθημα, καθώς ο Δείκτης Ενσωματώσεων παρέχεται για εσάς.

Τα σενάρια εκτελούν τις ακόλουθες λειτουργίες:

1. Η απομαγνητοφώνηση για κάθε βίντεο YouTube στη λίστα αναπαραγωγής [AI Show](https://www.youtube.com/playlist?list=PLlrxD0HtieHi0mwteKBOfEeOYf0LJU4O1) κατεβάζεται.
2. Χρησιμοποιώντας [Λειτουργίες OpenAI](https://learn.microsoft.com/azure/ai-services/openai/how-to/function-calling?WT.mc_id=academic-105485-koreyst), γίνεται προσπάθεια να εξαχθεί το όνομα του ομιλητή από τα πρώτα 3 λεπτά της απομαγνητοφώνησης του YouTube. Το όνομα του ομιλητή για κάθε βίντεο αποθηκεύεται στον Δείκτη Ενσωματώσεων με το όνομα `embedding_index_3m.json`.
3. Το κείμενο της απομαγνητοφώνησης στη συνέχεια τεμαχίζεται σε **τμήματα κειμένου 3 λεπτών**. Το τμήμα περιλαμβάνει περίπου 20 λέξεις που επικαλύπτονται από το επόμενο τμήμα για να εξασφαλιστεί ότι η Ενσωμάτωση για το τμήμα δεν κόβεται και να παρέχει καλύτερο πλαίσιο αναζήτησης.
4. Κάθε τμήμα κειμένου στη συνέχεια περνά στο OpenAI Chat API για να συνοψίσει το κείμενο σε 60 λέξεις. Η σύνοψη επίσης αποθηκεύεται στον Δείκτη Ενσωματώσεων `embedding_index_3m.json`.
5. Τέλος, το κείμενο του τμήματος περνά στο OpenAI Embedding API. Το Embedding API επιστρέφει ένα διάνυσμα 1536 αριθμών που αντιπροσωπεύουν το σημασιολογικό νόημα του τμήματος. Το τμήμα μαζί με το διάνυσμα Ενσωμάτωσης της OpenAI αποθηκεύεται σε έναν Δείκτη Ενσωματώσεων `embedding_index_3m.json`.

### Βάσεις Δεδομένων Διανυσμάτων

Για απλότητα μαθήματος, ο Δείκτης Ενσωματώσεων αποθηκεύεται σε ένα αρχείο JSON με το όνομα `embedding_index_3m.json` και φορτώνεται σε ένα Pandas DataFrame. Ωστόσο, στην παραγωγή, ο Δείκτης Ενσωματώσεων θα αποθηκευόταν σε μια βάση δεδομένων διανυσμάτων όπως το [Azure Cognitive Search](https://learn.microsoft.com/training/modules/improve-search-results-vector-search?WT.mc_id=academic-105485-koreyst), [Redis](https://cookbook.openai.com/examples/vector_databases/redis/readme?WT.mc_id=academic-105485-koreyst), [Pinecone](https://cookbook.openai.com/examples/vector_databases/pinecone/readme?WT.mc_id=academic-105485-koreyst), [Weaviate](https://cookbook.openai.com/examples/vector_databases/weaviate/readme?WT.mc_id=academic-105485-koreyst), για να αναφέρουμε μερικά.

## Κατανόηση της ομοιότητας συνημιτόνου

Έχουμε μάθει για τις ενσωματώσεις κειμένου, το επόμενο βήμα είναι να μάθουμε πώς να χρησιμοποιούμε τις ενσωματώσεις κειμένου για αναζήτηση δεδομένων και συγκεκριμένα να βρίσκουμε τις πιο όμοιες ενσωματώσεις σε μια δεδομένη ερώτηση χρησιμοποιώντας την ομοιότητα συνημιτόνου.

### Τι είναι η ομοιότητα συνημιτόνου;

Η ομοιότητα συνημιτόνου είναι ένα μέτρο ομοιότητας μεταξύ δύο διανυσμάτων, θα το ακούσετε επίσης να αναφέρεται ως `nearest neighbor search`. Για να εκτελέσετε μια αναζήτηση ομοιότητας συνημιτόνου, χρειάζεται να _διανυσματοποιήσετε_ για _κείμενο ερώτησης_ χρησιμοποιώντας το OpenAI Embedding API. Στη συνέχεια, υπολογίστε την _ομοιότητα συνημιτόνου_ μεταξύ του διανύσματος ερώτησης και κάθε διανύσματος στον Δείκτη Ενσωματώσεων. Θυμηθείτε, ο Δείκτης Ενσωματώσεων έχει ένα διάνυσμα για κάθε τμήμα κειμένου απομαγνητοφώνησης του YouTube. Τέλος, ταξινομήστε τα αποτελέσματα κατά ομοιότητα συνημιτόνου και τα τμήματα κειμένου με την υψηλότερη ομοιότητα συνημιτόνου είναι τα πιο όμοια με την ερώτηση.

Από μαθηματική άποψη, η ομοιότητα συνημιτόνου μετρά το συνημίτονο της γωνίας μεταξύ δύο διανυσμάτων που προβάλλονται σε έναν πολυδιάστατο χώρο. Αυτή η μέτρηση είναι ευεργετική, διότι αν δύο έγγραφα είναι μακριά από την Ευκλείδεια απόσταση λόγω μεγέθους, θα μπορούσαν ακόμα να έχουν μικρότερη γωνία μεταξύ τους και επομένως μεγαλύτερη ομοιότητα συνημιτόνου. Για περισσότερες πληροφορίες σχετικά με τις εξισώσεις ομοιότητας συνημιτόνου, δείτε [Ομοιότητα συνημιτόνου](https://en.wikipedia.org/wiki/Cosine_similarity?WT.mc_id=academic-105485-koreyst).

Η εφαρμογή αναζήτησης θα επιστρέφει επίσης έναν σύνδεσμο στο σημείο του βίντεο όπου βρίσκεται η απάντηση στην ερώτηση.

## Ανάθεση - δημιουργία εφαρμογής αναζήτησης, για να δώσετε δυνατότητα στους μαθητές

Εισαγάγαμε την νεοφυή μας επιχείρηση στην αρχή αυτού του μαθήματος. Τώρα είναι ώρα να δώσετε δυνατότητα στους μαθητές να δημιουργήσουν μια εφαρμογή αναζήτησης για τις αξιολογήσεις τους.

Σε αυτήν την ανάθεση, θα δημιουργήσετε τις Υπηρεσίες Azure OpenAI που θα χρησιμοποιηθούν για τη δημιουργία της εφαρμογής αναζήτησης. Θα δημιουργήσετε τις ακόλουθες Υπηρεσίες Azure OpenAI. Θα χρειαστείτε μια συνδρομή Azure για να ολοκληρώσετε αυτήν την ανάθεση.

### Έναρξη του Azure Cloud Shell

1. Συνδεθείτε στο [Azure portal](https://portal.azure.com/?WT.mc_id=academic-105485-koreyst).
2. Επιλέξτε το εικονίδιο Cloud Shell στην επάνω δεξιά γωνία του Azure portal.
3. Επιλέξτε **Bash** για τον τύπο περιβάλλοντος.

#### Δημιουργία ομάδας πόρων

> Για αυτές τις οδηγίες, χρησιμοποιούμε την ομάδα πόρων με το όνομα "semantic-video-search" στην Ανατολική ΗΠΑ.
> Μπορείτε να αλλάξετε το όνομα της ομάδας πόρων, αλλά όταν αλλάζετε την τοποθεσία για τους πόρους,
> ελέγξτε τον [πίνακα διαθεσιμότητας μοντέλων](https://aka.ms/oai/models?WT.mc_id=academic-105485-koreyst).

```shell
az group create --name semantic-video-search --location eastus
```

#### Δημιουργία πόρου υπηρεσίας Azure OpenAI

Από το Azure Cloud Shell, εκτελέστε την ακόλουθη εντολή για να δημιουργήσετε έναν πόρο υπηρεσίας Azure OpenAI.

```shell
az cognitiveservices account create --name semantic-video-openai --resource-group semantic-video-search \
    --location eastus --kind OpenAI --sku s0
```

#### Λήψη του τελικού σημείου και των κλειδιών για χρήση σε αυτήν την εφαρμογή

Από το Azure Cloud Shell, εκτελέστε τις ακόλουθες εντολές για να λάβετε το τελικό σημείο και τα κλειδιά για τον πόρο υπηρεσίας Azure OpenAI.

```shell
az cognitiveservices account show --name semantic-video-openai \
   --resource-group  semantic-video-search | jq -r .properties.endpoint
az cognitiveservices account keys list --name semantic-video-openai \
   --resource-group semantic-video-search | jq -r .key1
```

#### Ανάπτυξη του μοντέλου Ενσωμάτωσης της OpenAI

Από το Azure Cloud Shell, εκτελέστε την ακόλουθη εντολή για να αναπτύξετε το μοντέλο Ενσωμάτωσης της OpenAI.

```shell
az cognitiveservices account deployment create \
    --name semantic-video-openai \
    --resource-group  semantic-video-search \
    --deployment-name text-embedding-ada-002 \
    --model-name text-embedding-ada-002 \
    --model-version "2"  \
    --model-format OpenAI \
    --sku-capacity 100 --sku-name "Standard"
```
