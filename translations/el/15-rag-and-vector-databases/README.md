## Ανάκτηση Ενισχυμένης Γενιάς (RAG)

Ένα chatbot που υποστηρίζεται από LLM επεξεργάζεται τις εντολές των χρηστών για να δημιουργήσει απαντήσεις. Σχεδιάζεται να είναι διαδραστικό και να αλληλεπιδρά με τους χρήστες σε ένα ευρύ φάσμα θεμάτων. Ωστόσο, οι απαντήσεις του περιορίζονται στο πλαίσιο που παρέχεται και στα θεμελιώδη δεδομένα εκπαίδευσής του. Για παράδειγμα, η γνώση του GPT-4 είναι έως τον Σεπτέμβριο 2021, που σημαίνει ότι δεν γνωρίζει γεγονότα που συνέβησαν μετά από αυτή την περίοδο. Επιπλέον, τα δεδομένα που χρησιμοποιούνται για να εκπαιδεύσουν τα LLM αποκλείουν εμπιστευτικές πληροφορίες όπως προσωπικές σημειώσεις ή εγχειρίδιο προϊόντων μιας εταιρείας.

### Πώς λειτουργούν τα RAG (Retrieval Augmented Generation)

![σχέδιο που δείχνει πώς λειτουργούν τα RAG](../../../translated_images/el/how-rag-works.f5d0ff63942bd3a6.webp)

Ας υποθέσουμε ότι θέλετε να υλοποιήσετε ένα chatbot που δημιουργεί κουίζ από τις σημειώσεις σας, θα χρειαστείτε σύνδεση με τη βάση γνώσης. Εδώ έρχεται το RAG στη διάσωση. Τα RAG λειτουργούν ως εξής:

- **Βάση γνώσης:** Πριν από την ανάκτηση, αυτά τα έγγραφα πρέπει να εισαχθούν και να προεπεξεργαστούν, συνήθως διαχωρίζοντας μεγάλα έγγραφα σε μικρότερα κομμάτια, μετατρέποντας τα σε ενσωματώσεις κειμένου και αποθηκεύοντάς τα σε μια βάση δεδομένων.

- **Ερώτηση Χρήστη:** ο χρήστης κάνει μια ερώτηση

- **Ανάκτηση:** Όταν ο χρήστης κάνει μια ερώτηση, το μοντέλο ενσωμάτωσης ανακτά σχετικές πληροφορίες από τη βάση γνώσης μας για να παράσχει περισσότερο πλαίσιο που ενσωματώνεται στην εντολή.

- **Ενισχυμένη Γενιά:** το LLM βελτιώνει την απάντησή του βάσει των ανακτηθέντων δεδομένων. Αυτό επιτρέπει η παραγόμενη απάντηση να βασίζεται όχι μόνο σε προεκπαιδευμένα δεδομένα αλλά και σε σχετικές πληροφορίες από το πρόσθετο πλαίσιο. Τα ανακτηθέντα δεδομένα χρησιμοποιούνται για την ενίσχυση των απαντήσεων του LLM. Το LLM στη συνέχεια επιστρέφει απάντηση στην ερώτηση του χρήστη.

![σχέδιο που δείχνει την αρχιτεκτονική των RAG](../../../translated_images/el/encoder-decode.f2658c25d0eadee2.webp)

Η αρχιτεκτονική για τα RAG υλοποιείται χρησιμοποιώντας μετασχηματιστές που αποτελούνται από δύο μέρη: έναν κωδικοποιητή και έναν αποκωδικοποιητή. Για παράδειγμα, όταν ο χρήστης θέτει μια ερώτηση, το εισαγόμενο κείμενο 'κωδικοποιείται' σε διανύσματα που αιχμαλωτίζουν τη σημασία των λέξεων και τα διανύσματα 'αποκωδικοποιούνται' στο ευρετήριο εγγράφων μας και παράγουν νέο κείμενο βάσει της ερώτησης του χρήστη. Το LLM χρησιμοποιεί τόσο μοντέλο κωδικοποιητή-αποκωδικοποιητή για να δημιουργήσει το αποτέλεσμα.

Δύο προσεγγίσεις κατά την υλοποίηση του RAG σύμφωνα με την προτεινόμενη εργασία: [Retrieval-Augmented Generation for Knowledge intensive NLP (natural language processing software) Tasks](https://arxiv.org/pdf/2005.11401.pdf?WT.mc_id=academic-105485-koreyst) είναι:

- **_RAG-Sequence_** που χρησιμοποιεί ανακτημένα έγγραφα για να προβλέψει την καλύτερη δυνατή απάντηση σε μια ερώτηση χρήστη

- **RAG-Token** που χρησιμοποιεί έγγραφα για να παραγάγει τον επόμενο τόκεν, και στη συνέχεια τα ανακτά για να απαντήσει στην ερώτηση του χρήστη

### Γιατί να χρησιμοποιήσετε RAG;

- **Πλούτος πληροφοριών:** εξασφαλίζει ότι οι απαντήσεις κειμένου είναι ενημερωμένες και τρέχουσες. Επομένως, βελτιώνει την απόδοση σε συγκεκριμένες εργασίες τομέα με την πρόσβαση στη βάση γνώσης.

- Μειώνει την κατασκευή ψευδών απαντήσεων χρησιμοποιώντας **επαληθεύσιμα δεδομένα** στη βάση γνώσης για να παρέχει πλαίσιο στις ερωτήσεις των χρηστών.

- Είναι **οικονομικά αποδοτικό** αφού είναι πιο οικονομικό από το να κάνετε fine-tuning ενός LLM

Αποθηκεύει αριθμητικές αναπαραστάσεις εγγράφων. Η διάσπαση των δεδομένων σε αριθμητικές ενσωματώσεις διευκολύνει το σύστημα ΤΝ να κατανοήσει και να επεξεργαστεί τα δεδομένα.

### Από κείμενο σε ενσωματώσεις

Πριν αποθηκεύσουμε τα δεδομένα μας, θα χρειαστεί να τα μετατρέψουμε σε διανυσματικές ενσωματώσεις πριν αποθηκευτούν στη βάση δεδομένων. Αν εργάζεστε με μεγάλα έγγραφα ή μακροσκελή κείμενα, μπορείτε να τα χωρίσετε σε κομμάτια με βάση τις ερωτήσεις που αναμένετε. Η διαίρεση μπορεί να γίνει σε επίπεδο προτάσεως ή παραγράφου. Καθώς η διαίρεση αντλεί τις έννοιες από τις λέξεις γύρω τους, μπορείτε να προσθέσετε κάποιο άλλο πλαίσιο σε ένα κομμάτι, για παράδειγμα, προσθέτοντας τον τίτλο του εγγράφου ή συμπεριλαμβάνοντας κείμενο πριν ή μετά το κομμάτι. Μπορείτε να χωρίσετε τα δεδομένα ως εξής:

```python
def split_text(text, max_length, min_length):
    words = text.split()
    chunks = []
    current_chunk = []

    for word in words:
        current_chunk.append(word)
        if len(' '.join(current_chunk)) < max_length and len(' '.join(current_chunk)) > min_length:
            chunks.append(' '.join(current_chunk))
            current_chunk = []

    # Εάν το τελευταίο κομμάτι δεν έφτασε το ελάχιστο μήκος, προσθέστε το ούτως ή άλλως
    if current_chunk:
        chunks.append(' '.join(current_chunk))

    return chunks
```

Μόλις χωριστεί, μπορούμε στη συνέχεια να ενσωματώσουμε το κείμενό μας χρησιμοποιώντας διάφορα μοντέλα ενσωμάτωσης. Μερικά μοντέλα που μπορείτε να χρησιμοποιήσετε περιλαμβάνουν: word2vec, ada-002 της OpenAI, Azure Computer Vision και πολλά ακόμα. Η επιλογή μοντέλου εξαρτάται από τις γλώσσες που χρησιμοποιείτε, τον τύπο περιεχομένου που κωδικοποιείται (κείμενο/εικόνες/ήχος), το μέγεθος εισόδου που μπορεί να κωδικοποιήσει και το μήκος της εξόδου ενσωμάτωσης.

## Ανάκτηση και Αναζήτηση Διανυσμάτων

Όταν ένας χρήστης κάνει μια ερώτηση, ο αναζητητής τη μετατρέπει σε διάνυσμα χρησιμοποιώντας τον κωδικοποιητή ερωτήσεων, στη συνέχεια ψάχνει μέσα στο ευρετήριο αναζήτησης εγγράφων για σχετικά διανύσματα που συνδέονται με την είσοδο. Μόλις γίνει αυτό, μετατρέπει τόσο το εισαγόμενο διάνυσμα όσο και τα διανύσματα εγγράφων σε κείμενο και τα περνά στο LLM.

### Ανάκτηση

Η ανάκτηση συμβαίνει όταν το σύστημα προσπαθεί γρήγορα να βρει τα έγγραφα από το ευρετήριο που ικανοποιούν τα κριτήρια αναζήτησης. Ο στόχος του αναζητητή είναι να βρει έγγραφα που θα χρησιμοποιηθούν για να παρέχουν πλαίσιο και να θεμελιώσουν το LLM στα δεδομένα σας.

Υπάρχουν αρκετοί τρόποι για να πραγματοποιήσουμε αναζήτηση στη βάση δεδομένων μας, όπως:

- **Αναζήτηση με λέξεις-κλειδιά** - χρησιμοποιείται για αναζητήσεις κειμένου

- **Αναζήτηση διανυσμάτων** - μετατρέπει έγγραφα από κείμενο σε διανυσματικές αναπαραστάσεις με χρήση μοντέλων ενσωμάτωσης, επιτρέποντας **σημασιολογική αναζήτηση** χρησιμοποιώντας τη σημασία των λέξεων. Η ανάκτηση γίνεται ζητώντας τα έγγραφα των οποίων οι αναπαραστάσεις διανυσμάτων είναι πιο κοντά στην ερώτηση του χρήστη.

- **Υβριδική** - συνδυασμός τόσο αναζήτησης με λέξεις-κλειδιά όσο και αναζήτησης με διανύσματα.

Μια πρόκληση με την ανάκτηση είναι όταν δεν υπάρχει παρόμοια απάντηση στην ερώτηση στη βάση δεδομένων, το σύστημα τότε επιστρέφει τις καλύτερες πληροφορίες που μπορεί να βρει, ωστόσο, μπορείτε να χρησιμοποιήσετε τακτικές όπως να ορίσετε τη μέγιστη απόσταση για συνάφεια ή να χρησιμοποιήσετε υβριδική αναζήτηση που συνδυάζει αναζήτηση με λέξεις-κλειδιά και διανύσματα. Σε αυτό το μάθημα θα χρησιμοποιήσουμε υβριδική αναζήτηση, συνδυασμό και των δύο. Θα αποθηκεύσουμε τα δεδομένα μας σε dataframe με στήλες που περιέχουν τα κομμάτια καθώς και τις ενσωματώσεις.

### Ομοιότητα Διανυσμάτων

Ο αναζητητής θα αναζητήσει στη βάση γνώσης ενσωματώσεις που είναι κοντά μεταξύ τους, τον πιο κοντινό γείτονα, καθώς είναι κείμενα που μοιάζουν. Στο σενάριο που ένας χρήστης κάνει μια ερώτηση, αυτή πρώτα ενσωματώνεται και στη συνέχεια αντιστοιχίζεται με παρόμοιες ενσωματώσεις. Το κοινό μέτρο που χρησιμοποιείται για να βρεθεί πόσο παρόμοια είναι διάφορα διανύσματα είναι η σημασιολογική ομοιότητα (cosine similarity) που βασίζεται στη γωνία μεταξύ δύο διανυσμάτων.

Μπορούμε να μετρήσουμε την ομοιότητα χρησιμοποιώντας και άλλες εναλλακτικές, όπως η ευκλείδεια απόσταση που είναι η ευθεία γραμμή μεταξύ των άκρων των διανυσμάτων και το εσωτερικό γινόμενο που μετρά το άθροισμα των γινομένων των αντίστοιχων στοιχείων δύο διανυσμάτων.

### Ευρετήριο αναζήτησης

Κατά την ανάκτηση, θα χρειαστεί να δημιουργήσουμε ευρετήριο αναζήτησης για τη βάση γνώσης μας πριν πραγματοποιήσουμε την αναζήτηση. Ένα ευρετήριο θα αποθηκεύσει τις ενσωματώσεις μας και θα μπορεί γρήγορα να ανακτήσει τα πιο παρόμοια κομμάτια ακόμα και σε μια μεγάλη βάση δεδομένων. Μπορούμε να δημιουργήσουμε το ευρετήριό μας τοπικά χρησιμοποιώντας:

```python
from sklearn.neighbors import NearestNeighbors

embeddings = flattened_df['embeddings'].to_list()

# Δημιουργήστε το ευρετήριο αναζήτησης
nbrs = NearestNeighbors(n_neighbors=5, algorithm='ball_tree').fit(embeddings)

# Για να κάνετε ερώτηση στο ευρετήριο, μπορείτε να χρησιμοποιήσετε τη μέθοδο kneighbors
distances, indices = nbrs.kneighbors(embeddings)
```

### Ανακατάταξη (Re-ranking)

Μόλις κάνετε ερώτηση στη βάση δεδομένων, ίσως χρειαστεί να ταξινομήσετε τα αποτελέσματα ξεκινώντας από τα πιο σχετικά. Ένα reranking LLM χρησιμοποιεί Μηχανική Μάθηση για να βελτιώσει τη συνάφεια των αποτελεσμάτων αναζήτησης παραγγέλνοντάς τα από τα πιο σχετικά. Χρησιμοποιώντας το Azure AI Search, η ανακατάταξη γίνεται αυτόματα για εσάς χρησιμοποιώντας έναν σημασιολογικό επαναταξινομητή. Ένα παράδειγμα του πώς λειτουργεί η ανακατάταξη χρησιμοποιώντας τους πλησιέστερους γείτονες:

```python
# Βρείτε τα πιο παρόμοια έγγραφα
distances, indices = nbrs.kneighbors([query_vector])

index = []
# Εκτυπώστε τα πιο παρόμοια έγγραφα
for i in range(3):
    index = indices[0][i]
    for index in indices[0]:
        print(flattened_df['chunks'].iloc[index])
        print(flattened_df['path'].iloc[index])
        print(flattened_df['distances'].iloc[index])
    else:
        print(f"Index {index} not found in DataFrame")
```

- Ποιότητα των παρεχόμενων απαντήσεων που εξασφαλίζει ότι ακούγονται φυσικές, ρέουσες και ανθρώπινες

- Ρευστότητα - κατά πόσο η απάντηση έχει σωστή γραμματική

## Περιπτώσεις χρήσης για τη χρήση RAG (Retrieval Augmented Generation) και βάσεων δεδομένων διανυσμάτων

Υπάρχουν πολλές διαφορετικές περιπτώσεις χρήσης όπου οι κλήσεις λειτουργιών μπορούν να βελτιώσουν την εφαρμογή σας όπως:

- Ερωταπαντήσεις: θεμελιώνοντας τα δεδομένα της εταιρείας σας σε ένα chat που μπορούν να χρησιμοποιούν οι εργαζόμενοι για να κάνουν ερωτήσεις.

- Συστήματα σύστασης: όπου μπορείτε να δημιουργήσετε ένα σύστημα που ταιριάζει τις πιο παρόμοιες τιμές π.χ., ταινίες, εστιατόρια και πολλά άλλα.

- Υπηρεσίες chatbot: μπορείτε να αποθηκεύσετε το ιστορικό συνομιλίας και να προσωποποιήσετε τη συζήτηση βάσει των δεδομένων του χρήστη.

- Αναζήτηση εικόνων βασισμένη σε ενσωματώσεις διανυσμάτων, χρήσιμη για αναγνώριση εικόνων και ανίχνευση ανωμαλιών.

## Σύνοψη

Καλύψαμε τις βασικές περιοχές του RAG από την προσθήκη των δεδομένων μας στην εφαρμογή, την ερώτηση χρήστη και το αποτέλεσμα. Για να απλοποιήσετε τη δημιουργία του RAG, μπορείτε να χρησιμοποιήσετε frameworks όπως Semanti Kernel, Langchain ή Autogen.

Για να συνεχίσετε τη μάθησή σας για την Ανάκτηση Ενισχυμένης Γενιάς (RAG) μπορείτε να δημιουργήσετε:

- Χρησιμοποιήστε ένα framework, είτε LangChain είτε Semantic Kernel, και αναδημιουργήστε την εφαρμογή σας.
