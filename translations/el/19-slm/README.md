# Εισαγωγή στα Μικρά Γλωσσικά Μοντέλα για Γενετική Τεχνητή Νοημοσύνη για Αρχάριους
Η Γενετική Τεχνητή Νοημοσύνη είναι ένας συναρπαστικός τομέας της τεχνητής νοημοσύνης που εστιάζει στη δημιουργία συστημάτων ικανών να παράγουν νέο περιεχόμενο. Αυτό το περιεχόμενο μπορεί να κυμαίνεται από κείμενο και εικόνες μέχρι μουσική και ακόμη και ολόκληρα εικονικά περιβάλλοντα. Μία από τις πιο συναρπαστικές εφαρμογές της γενετικής τεχνητής νοημοσύνης είναι στον τομέα των γλωσσικών μοντέλων.

## Τι Είναι τα Μικρά Γλωσσικά Μοντέλα;

Ένα Μικρό Γλωσσικό Μοντέλο (SLM) αντιπροσωπεύει μια μειωμένη έκδοση ενός μεγάλου γλωσσικού μοντέλου (LLM), αξιοποιώντας πολλές από τις αρχιτεκτονικές αρχές και τεχνικές των LLMs, ενώ παρουσιάζει σημαντικά μειωμένο υπολογιστικό αποτύπωμα.

Τα SLM είναι ένα υποσύνολο γλωσσικών μοντέλων σχεδιασμένα για να παράγουν κείμενο που μοιάζει ανθρώπινο. Σε αντίθεση με τους μεγαλύτερους ομολόγους τους, όπως το GPT-4, τα SLM είναι πιο συμπαγή και αποδοτικά, καθιστώντας τα ιδανικά για εφαρμογές όπου οι υπολογιστικοί πόροι είναι περιορισμένοι. Παρά το μικρό τους μέγεθος, μπορούν ακόμα να εκτελέσουν ποικίλες εργασίες. Τυπικά, τα SLM κατασκευάζονται με συμπίεση ή απόσταξη των LLM, με στόχο να διατηρήσουν ένα σημαντικό μέρος της λειτουργικότητας και των γλωσσικών ικανοτήτων του αρχικού μοντέλου. Αυτή η μείωση στο μέγεθος του μοντέλου μειώνει τη συνολική πολυπλοκότητα, καθιστώντας τα SLM πιο αποδοτικά σε όρους χρήσης μνήμης και υπολογιστικών απαιτήσεων. Παρά αυτές τις βελτιστοποιήσεις, τα SLM μπορούν να εκτελέσουν ένα ευρύ φάσμα εργασιών επεξεργασίας φυσικής γλώσσας (NLP):

- Δημιουργία Κειμένου: Δημιουργία συνεκτικών και συμφραζόμενων προτάσεων ή παραγράφων.
- Συμπλήρωση Κειμένου: Πρόβλεψη και συμπλήρωση προτάσεων βάσει δοθέντος προτροπής.
- Μετάφραση: Μετατροπή κειμένου από μια γλώσσα σε άλλη.
- Περίληψη: Συμπύκνωση μεγάλων κειμένων σε πιο σύντομες, πιο εύπεπτες περιλήψεις.

Αν και με κάποιους συμβιβασμούς στην απόδοση ή στο βάθος της κατανόησης σε σύγκριση με τους μεγαλύτερους ομολόγους τους.

## Πώς Λειτουργούν τα Μικρά Γλωσσικά Μοντέλα;
Τα SLM εκπαιδεύονται σε τεράστια ποσά κειμενικών δεδομένων. Κατά την εκπαίδευση, μαθαίνουν τα μοτίβα και τις δομές της γλώσσας, επιτρέποντάς τους να παράγουν κείμενο που είναι τόσο γραμματικά σωστό όσο και συμφραζόμενο κατάλληλο. Η διαδικασία εκπαίδευσης περιλαμβάνει:

- Συλλογή Δεδομένων: Συγκέντρωση μεγάλων συνόλων δεδομένων κειμένου από διάφορες πηγές.
- Προεπεξεργασία: Καθαρισμός και οργάνωση των δεδομένων ώστε να είναι κατάλληλα για εκπαίδευση.
- Εκπαίδευση: Χρήση αλγορίθμων μηχανικής μάθησης για να διδάξουν στο μοντέλο πώς να κατανοεί και να παράγει κείμενο.
- Εξειδίκευση: Ρύθμιση του μοντέλου για βελτίωση της απόδοσής του σε συγκεκριμένες εργασίες.

Η ανάπτυξη των SLM ευθυγραμμίζεται με την αυξανόμενη ανάγκη για μοντέλα που μπορούν να αναπτυχθούν σε περιβάλλοντα με περιορισμένους πόρους, όπως φορητές συσκευές ή πλατφόρμες edge computing, όπου τα πλήρους κλίμακας LLM μπορεί να είναι ανεφάρμοστα λόγω των υψηλών απαιτήσεών τους σε πόρους. Εστιάζοντας στην αποδοτικότητα, τα SLM ισορροπούν την απόδοση με την προσβασιμότητα, επιτρέποντας ευρύτερη εφαρμογή σε διάφορους τομείς.

## Οι Διαφορές μεταξύ Μεγάλων Γλωσσικών Μοντέλων (LLMs) και Μικρών Γλωσσικών Μοντέλων (SLMs)

Τόσο τα LLM όσο και τα SLM βασίζονται σε θεμελιώδεις αρχές πιθανολογικής μηχανικής μάθησης, ακολουθώντας παρόμοιες προσεγγίσεις στον αρχιτεκτονικό σχεδιασμό, τις μεθοδολογίες εκπαίδευσης, τις διαδικασίες δημιουργίας δεδομένων και τις τεχνικές αξιολόγησης μοντέλων. Ωστόσο, αρκετοί βασικοί παράγοντες διαφοροποιούν αυτούς τους δύο τύπους μοντέλων.

## Εφαρμογές των Μικρών Γλωσσικών Μοντέλων

Τα SLM έχουν ένα ευρύ φάσμα εφαρμογών, συμπεριλαμβανομένων:

- Chatbots: Παροχή υποστήριξης πελατών και αλληλεπίδραση με χρήστες σε συνομιλιακή μορφή.
- Δημιουργία Περιεχομένου: Βοήθεια σε συγγραφείς με την παραγωγή ιδεών ή ακόμα και το σχεδιασμό ολόκληρων άρθρων.
- Εκπαίδευση: Βοήθεια σε μαθητές με γραπτές εργασίες ή με την εκμάθηση νέων γλωσσών.
- Προσβασιμότητα: Δημιουργία εργαλείων για άτομα με αναπηρίες, όπως συστήματα μετατροπής κειμένου σε ομιλία.

**Μέγεθος**
  
Η βασική διαφορά μεταξύ LLM και SLM έγκειται στο μέγεθος των μοντέλων. Τα LLM, όπως το ChatGPT (GPT-4), μπορεί να περιλαμβάνουν εκτιμώμενα 1,76 τρισεκατομμύρια παραμέτρους, ενώ τα ανοικτού κώδικα SLM όπως το Mistral 7B σχεδιάζονται με σημαντικά λιγότερες παραμέτρους — περίπου 7 δισεκατομμύρια. Αυτή η διαφορά οφείλεται κυρίως στις διαφορές στην αρχιτεκτονική μοντέλου και στις διαδικασίες εκπαίδευσης. Για παράδειγμα, το ChatGPT χρησιμοποιεί μηχανισμό αυτοπροσοχής μέσα σε ένα πλαίσιο κωδικοποιητή-αποκωδικοποιητή, ενώ το Mistral 7B χρησιμοποιεί προοδευτική προσοχή (sliding window attention), που επιτρέπει πιο αποδοτική εκπαίδευση μέσα σε μοντέλο αποκλειστικού αποκωδικοποιητή. Αυτή η αρχιτεκτονική διαφοροποίηση έχει σημαντικές επιπτώσεις στην πολυπλοκότητα και την απόδοση αυτών των μοντέλων.

**Κατανόηση**

Τα SLM τυπικά βελτιστοποιούνται για απόδοση σε συγκεκριμένους τομείς, καθιστώντάς τα ιδιαίτερα εξειδικευμένα αλλά πιθανώς περιορισμένα στην ικανότητά τους να παρέχουν ευρεία συμφραζόμενη κατανόηση σε πολλούς τομείς γνώσης. Αντίθετα, τα LLM στοχεύουν να προσομοιώσουν ανθρώπινη νοημοσύνη σε πιο ολοκληρωμένο επίπεδο. Εκπαιδευμένα σε τεράστια, ποικιλόμορφα σύνολα δεδομένων, τα LLM σχεδιάζονται να αποδίδουν καλά σε μια ποικιλία τομέων, προσφέροντας μεγαλύτερη ευελιξία και προσαρμοστικότητα. Συνεπώς, τα LLM είναι πιο κατάλληλα για ευρύτερο φάσμα εργασιών, όπως η επεξεργασία φυσικής γλώσσας και ο προγραμματισμός.

**Υπολογιστική Ικανότητα**

Η εκπαίδευση και ανάπτυξη των LLM είναι διεργασίες με μεγάλες απαιτήσεις σε πόρους, συχνά απαιτώντας εκτεταμένη υπολογιστική υποδομή, συμπεριλαμβανομένων μεγάλων συσσωρευμάτων GPU. Για παράδειγμα, η εκπαίδευση ενός μοντέλου όπως το ChatGPT από την αρχή μπορεί να απαιτήσει χιλιάδες GPUs για παρατεταμένες χρονικές περιόδους. Αντίθετα, τα SLM με τους μικρότερους αριθμούς παραμέτρων τους είναι πιο προσιτά από άποψη υπολογιστικών πόρων. Μοντέλα όπως το Mistral 7B μπορούν να εκπαιδευτούν και να τρέξουν σε τοπικούς υπολογιστές με μέτριες δυνατότητες GPU, αν και η εκπαίδευση εξακολουθεί να απαιτεί αρκετές ώρες σε πολλαπλές GPUs.

**Προκατάληψη**

Η προκατάληψη είναι ένα γνωστό ζήτημα στα LLM, κυρίως λόγω της φύσης των δεδομένων εκπαίδευσης. Αυτά τα μοντέλα συχνά βασίζονται σε ακατέργαστα, διαθέσιμα στο διαδίκτυο δεδομένα, που μπορεί να υποεκπροσωπούν ή να παραμορφώνουν ορισμένες ομάδες, να εισάγουν λανθασμένη ετικετοποίηση ή να αντανακλούν γλωσσικές προκαταλήψεις που επηρεάζονται από διάλεκτο, γεωγραφικές παραλλαγές και γραμματικούς κανόνες. Επιπλέον, η πολυπλοκότητα των αρχιτεκτονικών των LLM μπορεί να επιδεινώσει άθελά της την προκατάληψη, η οποία μπορεί να μην εντοπιστεί χωρίς προσεκτική εξειδίκευση. Από την άλλη πλευρά, τα SLM, επειδή εκπαιδεύονται σε πιο περιορισμένα, τομεο-ειδικά σύνολα δεδομένων, είναι εγγενώς λιγότερο επιρρεπή σε τέτοιες προκαταλήψεις, αν και δεν είναι αλώβητα από αυτές.

**Συμπερασματολογία**

Το μειωμένο μέγεθος των SLM τους δίνει σημαντικό πλεονέκτημα όσον αφορά την ταχύτητα συμπερασματολογίας, επιτρέποντάς τους να παράγουν αποτελέσματα αποδοτικά σε τοπικό υλικό χωρίς την ανάγκη για εκτεταμένη παράλληλη επεξεργασία. Αντίθετα, τα LLM, λόγω του μεγέθους και της πολυπλοκότητάς τους, συχνά απαιτούν σημαντικούς παράλληλους υπολογιστικούς πόρους για να επιτύχουν αποδεκτούς χρόνους απόκρισης. Η παρουσία πολλών ταυτόχρονων χρηστών επιβραδύνει ακόμα περισσότερο τους χρόνους απόκρισης των LLM, ειδικά όταν αναπτύσσονται σε μεγάλη κλίμακα.

Συνοπτικά, ενώ τόσο τα LLM όσο και τα SLM μοιράζονται θεμελιώδη βάση στη μηχανική μάθηση, διαφέρουν σημαντικά όσον αφορά το μέγεθος του μοντέλου, τις απαιτήσεις πόρων, την κατανόηση συμφραζομένων, την επιρροή από προκαταλήψεις και την ταχύτητα συμπερασματολογίας. Αυτές οι διαφορές αντικατοπτρίζουν την αντίστοιχη καταλληλότητα τους για διαφορετικές περιπτώσεις χρήσης, με τα LLM να είναι πιο ευέλικτα αλλά με υψηλές απαιτήσεις πόρων, και τα SLM να προσφέρουν αποδοτικότητα πιο τομεακού τύπου με μειωμένες υπολογιστικές απαιτήσεις.

## Παρουσίαση της Οικογένειας Phi-3 / Phi-3.5

Η οικογένεια Phi-3 / 3.5 στοχεύει κυρίως σε σενάρια εφαρμογών κειμένου, όρασης και Agent (MoE):

### Phi-3 / 3.5 Instruct

Κυρίως για δημιουργία κειμένου, συμπλήρωση διαλόγου και εξαγωγή πληροφοριών περιεχομένου, κλπ.

**Phi-3-mini**

Το γλωσσικό μοντέλο 3,8 δισεκατομμυρίων παραμέτρων είναι διαθέσιμο στο Microsoft Foundry, στο Hugging Face και την Ollama. Τα μοντέλα Phi-3 υπερέχουν σημαντικά σε γλωσσικά μοντέλα ισοδύναμου και μεγαλύτερου μεγέθους σε βασικά benchmarks (δείτε τους αριθμούς benchmarks παρακάτω, υψηλότεροι αριθμοί σημαίνουν καλύτερα). Το Phi-3-mini υπερτερεί μοντέλων διπλάσιου μεγέθους, ενώ τα Phi-3-small και Phi-3-medium υπερτερούν μεγαλύτερων μοντέλων, συμπεριλαμβανομένου του GPT-3.5.

**Phi-3-small & medium**

Με μόλις 7 δισεκατομμύρια παραμέτρους, το Phi-3-small ξεπερνά το GPT-3.5T σε διάφορα benchmarks γλώσσας, λογικής, κωδικοποίησης και μαθηματικών.

Το Phi-3-medium με 14 δισεκατομμύρια παραμέτρους συνεχίζει αυτήν την τάση και υπερέχει του Gemini 1.0 Pro.

**Phi-3.5-mini**

Μπορούμε να το θεωρήσουμε ως αναβάθμιση του Phi-3-mini. Ενώ οι παράμετροι παραμένουν αμετάβλητοι, βελτιώνει την ικανότητα υποστήριξης πολλαπλών γλωσσών (υποστηρίζει 20+ γλώσσες: Αραβικά, Κινέζικα, Τσέχικα, Δανέζικα, Ολλανδικά, Αγγλικά, Φινλανδικά, Γαλλικά, Γερμανικά, Εβραϊκά, Ουγγρικά, Ιταλικά, Ιαπωνικά, Κορεατικά, Νορβηγικά, Πολωνικά, Πορτογαλικά, Ρωσικά, Ισπανικά, Σουηδικά, Ταϊλανδέζικα, Τουρκικά, Ουκρανικά) και προσθέτει ισχυρότερη υποστήριξη για μεγάλα συμφραζόμενα.

Το Phi-3.5-mini με 3,8 δισεκατομμύρια παραμέτρους υπερτερεί σε γλωσσικά μοντέλα του ίδιου μεγέθους και είναι ισοδύναμο με μοντέλα διπλάσιου μεγέθους.

### Phi-3 / 3.5 Vision

Μπορούμε να θεωρήσουμε το μοντέλο Instruct του Phi-3/3.5 ως την ικανότητα του Phi να καταλαβαίνει, και το Vision είναι αυτό που δίνει στο Phi μάτια για να κατανοεί τον κόσμο.


**Phi-3-Vision**

Το Phi-3-vision, με μόλις 4,2 δισεκατομμύρια παραμέτρους, συνεχίζει την τάση αυτή και υπερέχει μεγαλύτερων μοντέλων όπως το Claude-3 Haiku και το Gemini 1.0 Pro V σε γενικά οπτικά προκείμενα εργασιών, OCR, καθώς και στην κατανόηση πινάκων και διαγραμμάτων.


**Phi-3.5-Vision**

Το Phi-3.5-Vision είναι επίσης αναβάθμιση του Phi-3-Vision, προσθέτοντας υποστήριξη για πολλαπλές εικόνες. Μπορείτε να το θεωρήσετε ως βελτίωση στην όραση, όχι μόνο μπορείτε να δείτε εικόνες αλλά και βίντεο.

Το Phi-3.5-vision υπερέχει μεγαλύτερων μοντέλων όπως το Claude-3.5 Sonnet και το Gemini 1.5 Flash σε εργασίες OCR, κατανόησης πινάκων και γραφημάτων και είναι ισοδύναμο σε γενικές εργασίες οπτικής γνώσης και λογικής. Υποστηρίζει πολλαπλά καρέ εισόδου, δηλαδή εκτέλεση λογικής σε πολλαπλές εισερχόμενες εικόνες.


### Phi-3.5-MoE

***Mixture of Experts (MoE)*** επιτρέπει στα μοντέλα να εκπαιδεύονται με πολύ λιγότερους υπολογιστικούς πόρους, πράγμα που σημαίνει ότι μπορείτε να κλιμακώσετε δραματικά το μέγεθος του μοντέλου ή του συνόλου δεδομένων με τον ίδιο προϋπολογισμό υπολογισμού όπως ένα πυκνό μοντέλο. Συγκεκριμένα, ένα μοντέλο MoE πρέπει να επιτυγχάνει την ίδια ποιότητα με τον αντίστοιχο πυκνό μοντέλο πολύ πιο γρήγορα κατά την προεκπαίδευση.

Το Phi-3.5-MoE αποτελείται από 16x3.8B εξειδικευμένα modules. Το Phi-3.5-MoE με μόλις 6,6 δισεκατομμύρια ενεργούς παραμέτρους επιτυγχάνει παρόμοιο επίπεδο λογικής, κατανόησης γλώσσας και μαθηματικών όπως πολύ μεγαλύτερα μοντέλα.

Μπορούμε να χρησιμοποιήσουμε το μοντέλο της οικογένειας Phi-3/3.5 βάσει διαφορετικών σεναρίων. Σε αντίθεση με τα LLM, μπορείτε να αναπτύξετε το Phi-3/3.5-mini ή το Phi-3/3.5-Vision σε συσκευές άκρου.

Μπορείτε να επισκεφτείτε το NVIDIA NIM για να ολοκληρώσετε τις κλήσεις API της οικογένειας Phi-3/3.5. Το NVIDIA NIM (NVIDIA Inference Microservices) είναι ένα σύνολο επιταχυνόμενων μικροϋπηρεσιών συμπερασματολογίας σχεδιασμένο για να βοηθάει τους προγραμματιστές να αναπτύσσουν AI μοντέλα αποδοτικά σε διάφορα περιβάλλοντα, συμπεριλαμβανομένων cloud, κέντρων δεδομένων και σταθμών εργασίας.

Εδώ είναι μερικά βασικά χαρακτηριστικά του NVIDIA NIM:

- **Ευκολία Ανάπτυξης:** Το NIM επιτρέπει την ανάπτυξη μοντέλων AI με μία μόνο εντολή, καθιστώντας εύκολη την ενσωμάτωση σε υπάρχοντα ροές εργασίας.

- **Βελτιστοποιημένη Απόδοση:** Αξιοποιεί τις προ-βελτιστοποιημένες μηχανές συμπερασμού της NVIDIA, όπως το TensorRT και το TensorRT-LLM, για να διασφαλίσει χαμηλή καθυστέρηση και υψηλό ρυθμό επεξεργασίας.
- **Κλιμάκωση:** Το NIM υποστηρίζει αυτόματη κλιμάκωση στο Kubernetes, επιτρέποντάς του να διαχειρίζεται αποτελεσματικά ποικίλα φορτία εργασίας.
- **Ασφάλεια και Έλεγχος:** Οι οργανισμοί μπορούν να διατηρήσουν τον έλεγχο των δεδομένων και των εφαρμογών τους με την αυτο-φιλοξενία των microservices του NIM στην δική τους διαχειριζόμενη υποδομή.
- **Τυπικά API:** Το NIM παρέχει API βιομηχανικού προτύπου, καθιστώντας εύκολη την κατασκευή και ενσωμάτωση εφαρμογών τεχνητής νοημοσύνης όπως chatbots, βοηθοί AI, και άλλα.

Το NIM αποτελεί μέρος του NVIDIA AI Enterprise, που στοχεύει στην απλοποίηση της ανάπτυξης και λειτουργιοποίησης μοντέλων AI, διασφαλίζοντας ότι τρέχουν αποδοτικά σε GPUs της NVIDIA.

**Hugging Face Transformer**
Το Hugging Face Transformers είναι μια ισχυρή βιβλιοθήκη σχεδιασμένη για επεξεργασία φυσικής γλώσσας (NLP) και άλλες εργασίες μηχανικής μάθησης. Ακολουθούν μερικά βασικά σημεία γι’ αυτήν:

1. **Προεκπαιδευμένα Μοντέλα**: Παρέχει χιλιάδες προεκπαιδευμένα μοντέλα που μπορούν να χρησιμοποιηθούν για διάφορες εργασίες όπως ταξινόμηση κειμένου, αναγνώριση ονομαστικών οντοτήτων, απάντηση σε ερωτήσεις, περίληψη, μετάφραση και δημιουργία κειμένου.

2. **Διαλειτουργικότητα Πλαισίων:** Η βιβλιοθήκη υποστηρίζει πολλά πλαίσια βαθιάς μάθησης, όπως PyTorch, TensorFlow και JAX. Αυτό σας επιτρέπει να εκπαιδεύσετε ένα μοντέλο σε ένα πλαίσιο και να το χρησιμοποιήσετε σε άλλο.

3. **Πολυτροπικές Δυνατότητες:** Εκτός από την NLP, το Hugging Face Transformers υποστηρίζει και εργασίες στην υπολογιστική όραση (π.χ., ταξινόμηση εικόνων, ανίχνευση αντικειμένων) και στην επεξεργασία ήχου (π.χ., αναγνώριση ομιλίας, ταξινόμηση ήχου).

4. **Ευκολία Χρήσης:** Η βιβλιοθήκη προσφέρει API και εργαλεία για εύκολο κατέβασμα και βελτιστοποίηση μοντέλων, καθιστώντας την προσιτή για αρχάριους και ειδικούς.

5. **Κοινότητα και Πόροι:** Το Hugging Face διαθέτει ζωντανή κοινότητα και εκτενή τεκμηρίωση, σεμινάρια και οδηγούς για να βοηθήσει τους χρήστες να ξεκινήσουν και να αξιοποιήσουν τη βιβλιοθήκη στο μέγιστο.

**ONNX Runtime για GenAI**

Το [ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) είναι ένας cross-platform επιταχυντής συμπερασμού και εκπαίδευσης μηχανικής μάθησης. Το ONNX Runtime για Γενετική AI (GENAI) είναι ένα ισχυρό εργαλείο που σας βοηθά να τρέχετε μοντέλα γενετικής AI αποδοτικά σε διάφορες πλατφόρμες.

## Τι είναι το ONNX Runtime;
Το ONNX Runtime είναι ένα έργο ανοιχτού κώδικα που επιτρέπει υψηλές επιδόσεις συμπερασμού μοντέλων μηχανικής μάθησης. Υποστηρίζει μοντέλα στο φορμάτ Open Neural Network Exchange (ONNX), που είναι ένα πρότυπο αναπαράστασης μοντέλων μηχανικής μάθησης. Ο συμπερασμός με ONNX Runtime μπορεί να επιταχύνει την εμπειρία πελατών και να μειώσει το κόστος, υποστηρίζοντας μοντέλα από πλαίσια βαθιάς μάθησης όπως PyTorch και TensorFlow/Keras καθώς και κλασικές βιβλιοθήκες μηχανικής μάθησης όπως scikit-learn, LightGBM, XGBoost κ.ά. Το ONNX Runtime είναι συμβατό με διάφορα υλικά, οδηγούς και λειτουργικά συστήματα, προσφέροντας βέλτιστη απόδοση αξιοποιώντας επιταχυντές υλικού όπου είναι διαθέσιμοι παράλληλα με βελτιώσεις και μετασχηματισμούς γραφήματος.

## Τι είναι η Γενετική AI;
Η Γενετική AI αναφέρεται σε συστήματα AI που μπορούν να δημιουργήσουν νέο περιεχόμενο, όπως κείμενο, εικόνες ή μουσική, βασισμένα στα δεδομένα με τα οποία έχουν εκπαιδευτεί. Παραδείγματα περιλαμβάνουν μοντέλα γλώσσας όπως το GPT-3 και μοντέλα δημιουργίας εικόνας όπως το Stable Diffusion. Η βιβλιοθήκη ONNX Runtime για GenAI παρέχει τον βρόχο γενετικής AI για τα μοντέλα ONNX, συμπεριλαμβανομένου του συμπερασμού με ONNX Runtime, της επεξεργασίας logits, της αναζήτησης και δειγματοληψίας, καθώς και της διαχείρισης cache KV.

## ONNX Runtime για GENAI
Το ONNX Runtime για GENAI επεκτείνει τις δυνατότητες του ONNX Runtime για να υποστηρίξει μοντέλα γενετικής AI. Ακολουθούν μερικά βασικά χαρακτηριστικά:

- **Ευρεία Υποστήριξη Πλατφορμών:** Λειτουργεί σε διάφορες πλατφόρμες, όπως Windows, Linux, macOS, Android και iOS.
- **Υποστήριξη Μοντέλων:** Υποστηρίζει πολλά δημοφιλή μοντέλα γενετικής AI, όπως LLaMA, GPT-Neo, BLOOM και άλλα.
- **Βελτιστοποίηση Απόδοσης:** Περιλαμβάνει βελτιστοποιήσεις για διάφορους επιταχυντές υλικού όπως GPUs της NVIDIA, GPUs της AMD, και άλλα2.
- **Ευκολία Χρήσης:** Παρέχει API για εύκολη ενσωμάτωση σε εφαρμογές, επιτρέποντας τη δημιουργία κειμένου, εικόνων και άλλου περιεχομένου με ελάχιστο κώδικα
- Οι χρήστες μπορούν να καλέσουν τη μέθοδο generate() υψηλού επιπέδου, ή να τρέξουν κάθε επανάληψη του μοντέλου σε βρόχο, δημιουργώντας έναν token τη φορά, και προαιρετικά να ενημερώνουν παραμέτρους δημιουργίας μέσα στον βρόχο.
- Το ONNX Runtime υποστηρίζει επίσης αναζήτηση greedy/beam και δειγματοληψία TopP, TopK για παραγωγή ακολουθιών token, καθώς και ενσωματωμένη επεξεργασία logits όπως ποινές επανάληψης. Μπορείτε επίσης να προσθέσετε εύκολα προσαρμοσμένη βαθμολόγηση.

```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### Παράδειγμα: Χρήση ONNX Runtime GenAI για κλήση Phi-3.5-Vision


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
