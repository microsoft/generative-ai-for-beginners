# Εισαγωγή σε Μικρά Γλωσσικά Μοντέλα για Γενετική Τεχνητή Νοημοσύνη για Αρχάριους
Η Γενετική Τεχνητή Νοημοσύνη είναι ένας συναρπαστικός τομέας της τεχνητής νοημοσύνης που εστιάζει στη δημιουργία συστημάτων ικανά να παράγουν νέο περιεχόμενο. Αυτό το περιεχόμενο μπορεί να κυμαίνεται από κείμενο και εικόνες έως μουσική και ακόμη και ολόκληρα εικονικά περιβάλλοντα. Μία από τις πιο συναρπαστικές εφαρμογές της γενετικής τεχνητής νοημοσύνης είναι στον τομέα των γλωσσικών μοντέλων.

## Τι είναι τα Μικρά Γλωσσικά Μοντέλα;

Ένα Μικρό Γλωσσικό Μοντέλο (SLM) αντιπροσωπεύει μια κλιμακωμένη εκδοχή ενός μεγάλου γλωσσικού μοντέλου (LLM), αξιοποιώντας πολλές από τις αρχιτεκτονικές αρχές και τεχνικές των LLM, ενώ παρουσιάζει σημαντικά μειωμένο υπολογιστικό αποτύπωμα.

Τα SLMs είναι μια υποκατηγορία γλωσσικών μοντέλων σχεδιασμένων να παράγουν κείμενο που μοιάζει με ανθρώπινο. Σε αντίθεση με τους μεγαλύτερους ομολόγους τους, όπως το GPT-4, τα SLMs είναι πιο συμπαγή και αποδοτικά, καθιστώντας τα ιδανικά για εφαρμογές όπου οι υπολογιστικοί πόροι είναι περιορισμένοι. Παρά το μικρότερο μέγεθός τους, μπορούν εξακολουθούν να εκτελούν διάφορα καθήκοντα. Συνήθως, τα SLM κατασκευάζονται μέσω συμπίεσης ή απόσταξης LLM, με στόχο να διατηρήσουν σημαντικό μέρος της λειτουργικότητας και των γλωσσικών ικανοτήτων του αρχικού μοντέλου. Αυτή η μείωση στο μέγεθος του μοντέλου μειώνει τη συνολική πολυπλοκότητα, κάνοντας τα SLM πιο αποδοτικά τόσο σε όρους χρήσης μνήμης όσο και υπολογιστικών απαιτήσεων. Παρά αυτές τις βελτιστοποιήσεις, τα SLM μπορούν να εκτελέσουν ένα ευρύ φάσμα εργασιών φυσικής γλώσσας (NLP):

- Δημιουργία Κειμένου: Δημιουργία συνεκτικών και συμφραζόμενα σχετικών προτάσεων ή παραγράφων.
- Ολοκλήρωση Κειμένου: Πρόβλεψη και συμπλήρωση προτάσεων βασισμένων σε ένα δεδομένο προτροπή.
- Μετάφραση: Μετατροπή κειμένου από μια γλώσσα σε άλλη.
- Περίληψη: Συμπύκνωση μεγάλων κειμένων σε συντομότερες, πιο εύπεπτες περίληψεις.

Αν και με μερικούς συμβιβασμούς στην απόδοση ή το βάθος κατανόησης σε σύγκριση με τους μεγαλύτερους ομολόγους τους.

## Πώς Λειτουργούν τα Μικρά Γλωσσικά Μοντέλα;
Τα SLM εκπαιδεύονται σε τεράστιες ποσότητες δεδομένων κειμένου. Κατά τη διάρκεια της εκπαίδευσης, μαθαίνουν τα πρότυπα και τις δομές της γλώσσας, επιτρέποντάς τους να παράγουν κείμενο που είναι τόσο γραμματικά σωστό όσο και συμφραζόμενα κατάλληλο. Η διαδικασία εκπαίδευσης περιλαμβάνει:

- Συλλογή Δεδομένων: Συγκέντρωση μεγάλων συνόλων δεδομένων κειμένου από διάφορες πηγές.
- Προεπεξεργασία: Καθαρισμός και οργάνωση των δεδομένων ώστε να είναι κατάλληλα για εκπαίδευση.
- Εκπαίδευση: Χρήση αλγορίθμων μηχανικής μάθησης για να διδάξουν στο μοντέλο πώς να κατανοεί και να παράγει κείμενο.
- Λεπτομερή Ρύθμιση: Προσαρμογή του μοντέλου για βελτίωση της απόδοσης σε συγκεκριμένα καθήκοντα.

Η ανάπτυξη των SLM ευθυγραμμίζεται με την αυξανόμενη ανάγκη για μοντέλα που μπορούν να αναπτυχθούν σε περιβάλλοντα με περιορισμένους πόρους, όπως φορητές συσκευές ή πλατφόρμες υπολογισμού άκρης, όπου τα πλήρους κλίμακας LLM μπορεί να είναι μη πρακτικά λόγω των μεγάλων απαιτήσεών τους. Εστιαζόμενα στην αποδοτικότητα, τα SLM εξισορροπούν την απόδοση με την προσβασιμότητα, επιτρέποντας ευρύτερη εφαρμογή σε διάφορους τομείς.

## Οι Διαφορές μεταξύ Μεγάλων Γλωσσικών Μοντέλων (LLMs) και Μικρών Γλωσσικών Μοντέλων (SLMs)

Τα LLM και SLM βασίζονται σε θεμελιώδεις αρχές πιθανοτικής μηχανικής μάθησης, ακολουθώντας παρόμοιες προσεγγίσεις στον αρχιτεκτονικό σχεδιασμό, τις μεθοδολογίες εκπαίδευσης, τις διαδικασίες δημιουργίας δεδομένων και τις τεχνικές αξιολόγησης μοντέλων. Ωστόσο, αρκετοί βασικοί παράγοντες διαχωρίζουν αυτούς τους δύο τύπους μοντέλων.

## Εφαρμογές των Μικρών Γλωσσικών Μοντέλων

Τα SLM έχουν ευρύ φάσμα εφαρμογών, όπως:

- Chatbots: Παροχή υποστήριξης πελατών και αλληλεπίδραση με χρήστες με συνομιλιακό τρόπο.
- Δημιουργία Περιεχομένου: Βοήθεια σε συγγραφείς μέσω δημιουργίας ιδεών ή ακόμη και σύνταξης ολόκληρων άρθρων.
- Εκπαίδευση: Βοήθεια σε μαθητές με εργασίες γραφής ή εκμάθηση νέων γλωσσών.
- Προσβασιμότητα: Δημιουργία εργαλείων για άτομα με αναπηρίες, όπως συστήματα μετατροπής κειμένου σε ομιλία.

**Μέγεθος**
  
Ένας βασικός διαχωρισμός μεταξύ LLM και SLM βρίσκεται στην κλίμακα των μοντέλων. Τα LLM, όπως το ChatGPT (GPT-4), μπορεί να περιλαμβάνουν περίπου 1,76 τρισεκατομμύρια παραμέτρους, ενώ τα ανοιχτού κώδικα SLM όπως το Mistral 7B έχουν σημαντικά λιγότερες παραμέτρους—περίπου 7 δισεκατομμύρια. Αυτή η διαφορά οφείλεται κυρίως σε διαφορές στην αρχιτεκτονική μοντέλου και τις διαδικασίες εκπαίδευσης. Για παράδειγμα, το ChatGPT χρησιμοποιεί μηχανισμό αυτοπροσοχής εντός πλαισίου κωδικοποιητή-αποκωδικοποιητή, ενώ το Mistral 7B χρησιμοποιεί προοδική προσοχή παραθύρου, που επιτρέπει πιο αποτελεσματική εκπαίδευση σε αποκωδικοποιητή μόνο. Αυτή η αρχιτεκτονική διαφορά έχει σημαντικές επιπτώσεις στην πολυπλοκότητα και απόδοση αυτών των μοντέλων.

**Κατανόηση**

Τα SLM συνήθως βελτιστοποιούνται για απόδοση σε συγκεκριμένους τομείς, καθιστώντας τα ιδιαίτερα εξειδικευμένα αλλά πιθανώς περιορισμένα στην ικανότητα παροχής ευρείας συμφραζόμενης κατανόησης σε πολλαπλά επιστημονικά πεδία. Αντίθετα, τα LLM στοχεύουν να προσομοιώσουν ανθρώπινη νοημοσύνη σε πιο ολοκληρωμένο επίπεδο. Εκπαιδευμένα σε τεράστια, ποικιλόμορφα σύνολα δεδομένων, τα LLM έχουν σχεδιαστεί να αποδίδουν καλά σε διάφορους τομείς, προσφέροντας μεγαλύτερη ευελιξία και προσαρμοστικότητα. Κατά συνέπεια, τα LLM είναι πιο κατάλληλα για ευρύτερο φάσμα εργασιών, όπως η επεξεργασία φυσικής γλώσσας και ο προγραμματισμός.

**Υπολογιστική Ισχύς**

Η εκπαίδευση και ανάπτυξη των LLM είναι διαδικασίες εντατικές σε πόρους, που συχνά απαιτούν σημαντική υπολογιστική υποδομή, συμπεριλαμβανομένων μεγάλων συστοιχιών GPU. Για παράδειγμα, η εκπαίδευση ενός μοντέλου σαν το ChatGPT από το μηδέν μπορεί να χρειαστεί χιλιάδες GPUs για παρατεταμένες περιόδους. Αντίθετα, τα SLM, με τον μικρότερο αριθμό παραμέτρων, είναι πιο προσβάσιμα σε όρους υπολογιστικών πόρων. Μοντέλα όπως το Mistral 7B μπορούν να εκπαιδευτούν και να τρέξουν σε τοπικούς υπολογιστές με μέτριες δυνατότητες GPU, αν και η εκπαίδευση εξακολουθεί να απαιτεί αρκετές ώρες σε πολλαπλές GPUs.

**Προκατάληψη**

Η προκατάληψη είναι γνωστό ζήτημα στα LLM, κυρίως λόγω της φύσης των δεδομένων εκπαίδευσης. Αυτά τα μοντέλα συχνά βασίζονται σε ακατέργαστα, ανοιχτά διαθέσιμα δεδομένα από το διαδίκτυο, τα οποία μπορεί να υποαντιπροσωπεύουν ή να παραμορφώνουν ορισμένες ομάδες, να εισάγουν λανθασμένες ετικέτες ή να αντανακλούν γλωσσικές προκαταλήψεις επηρεασμένες από διάλεκτο, γεωγραφικές παραλλαγές και γραμματικούς κανόνες. Επιπλέον, η πολυπλοκότητα της αρχιτεκτονικής των LLM μπορεί αθέλητα να επιδεινώσει την προκατάληψη, η οποία μπορεί να μην γίνει αντιληπτή χωρίς προσεκτική λεπτομερή ρύθμιση. Αντίθετα, τα SLM, που εκπαιδεύονται σε πιο περιορισμένα, ειδικά σύνολα δεδομένων, είναι εγγενώς λιγότερο ευάλωτα σε τέτοιες προκαταλήψεις, αν και δεν είναι άτρωτα σε αυτές.

**Συμπέρασμα**

Το μειωμένο μέγεθος των SLM τους παρέχει σημαντικό πλεονέκτημα ως προς την ταχύτητα συμπεράσματος, επιτρέποντάς τους να παράγουν αποτελέσματα αποδοτικά σε τοπικό υλικό χωρίς την ανάγκη εκτενούς παράλληλης επεξεργασίας. Αντίθετα, τα LLM, λόγω του μεγέθους και της πολυπλοκότητάς τους, συχνά απαιτούν σημαντικούς παράλληλους υπολογιστικούς πόρους για να επιτύχουν αποδεκτούς χρόνους συμπεράσματος. Η παρουσία πολλών ταυτόχρονων χρηστών επιβραδύνει περαιτέρω τους χρόνους απόκρισης των LLM, ειδικά όταν χρησιμοποιούνται σε ευρεία κλίμακα.

Συνοψίζοντας, ενώ και τα LLM και SLM μοιράζονται μια θεμελιώδη βάση στη μηχανική μάθηση, διαφέρουν σημαντικά ως προς το μέγεθος μοντέλου, τις απαιτήσεις πόρων, την κατανόηση συμφραζομένων, την ευπάθεια σε προκαταλήψεις και την ταχύτητα συμπεράσματος. Αυτές οι διαφορές αντικατοπτρίζουν την κατάλληλη χρήση τους σε διαφορετικές περιπτώσεις, με τα LLM να είναι πιο ευέλικτα αλλά με μεγάλες απαιτήσεις πόρων, και τα SLM να προσφέρουν μεγαλύτερη αποδοτικότητα σε συγκεκριμένους τομείς με μειωμένες υπολογιστικές απαιτήσεις.

## Εισαγωγή στην Οικογένεια Phi-3 / Phi-3.5

Η οικογένεια Phi-3 / 3.5 κυρίως απευθύνεται σε σενάρια εφαρμογών κειμένου, όρασης και Πράκτορα (MoE):

### Phi-3 / 3.5 Instruct

Κυρίως για δημιουργία κειμένου, συμπλήρωση συνομιλίας και εξαγωγή περιεχομένου πληροφοριών, κ.ά.

**Phi-3-mini**

Το γλωσσικό μοντέλο 3.8B είναι διαθέσιμο στο Microsoft Foundry, Hugging Face και Ollama. Τα μοντέλα Phi-3 αποδίδουν σημαντικά καλύτερα από γλωσσικά μοντέλα ίδιου ή μεγαλύτερου μεγέθους σε βασικά benchmarks (δείτε αριθμούς benchmark παρακάτω, υψηλότεροι αριθμοί είναι καλύτεροι). Το Phi-3-mini υπερέχει σε μοντέλα διπλάσιου μεγέθους, ενώ το Phi-3-small και το Phi-3-medium υπερέχουν σε μεγαλύτερα μοντέλα, συμπεριλαμβανομένου του GPT-3.5.

**Phi-3-small & medium**

Με μόλις 7B παραμέτρους, το Phi-3-small ξεπερνά το GPT-3.5T σε διάφορα benchmarks γλώσσας, λογικής, κωδικοποίησης και μαθηματικών.

Το Phi-3-medium με 14B παραμέτρους συνεχίζει αυτή την τάση και υπερέχει του Gemini 1.0 Pro.

**Phi-3.5-mini**

Μπορούμε να το θεωρήσουμε ως αναβάθμιση του Phi-3-mini. Ενώ οι παράμετροι παραμένουν αμετάβλητοι, βελτιώνει την ικανότητα υποστήριξης πολλαπλών γλωσσών (υποστηρίζει 20+ γλώσσες: Αραβικά, Κινέζικα, Τσέχικα, Δανέζικα, Ολλανδικά, Αγγλικά, Φινλανδικά, Γαλλικά, Γερμανικά, Εβραϊκά, Ουγγρικά, Ιταλικά, Ιαπωνικά, Κορεάτικα, Νορβηγικά, Πολωνικά, Πορτογαλικά, Ρωσικά, Ισπανικά, Σουηδικά, Ταϊλανδικά, Τουρκικά, Ουκρανικά) και προσθέτει ισχυρότερη υποστήριξη για μεγάλο συμφραζόμενο.

Το Phi-3.5-mini με 3.8B παραμέτρους υπερέχει από γλωσσικά μοντέλα ίδιου μεγέθους και ισοδυναμεί με μοντέλα διπλάσιου μεγέθους.

### Phi-3 / 3.5 Όραση

Μπορούμε να θεωρήσουμε το Instruct μοντέλο του Phi-3/3.5 ως την ικανότητα του Phi να κατανοεί, και η Όραση είναι που δίνει στα Phi μάτια για να κατανοεί τον κόσμο.


**Phi-3-Vision**

Το Phi-3-vision, με μόλις 4.2B παραμέτρους, συνεχίζει αυτή την τάση και υπερέχει σε μεγαλύτερα μοντέλα όπως τα Claude-3 Haiku και Gemini 1.0 Pro V σε γενικά εργασίες οπτικής λογικής, OCR, και κατανόησης πινάκων και διαγραμμάτων.


**Phi-3.5-Vision**

Το Phi-3.5-Vision αποτελεί επίσης αναβάθμιση του Phi-3-Vision, προσθέτοντας υποστήριξη για πολλαπλές εικόνες. Μπορείτε να το θεωρήσετε ως βελτίωση στην όραση, όχι μόνο μπορείτε να βλέπετε εικόνες αλλά και βίντεο.

Το Phi-3.5-vision υπερέχει από μεγαλύτερα μοντέλα όπως τα Claude-3.5 Sonnet και Gemini 1.5 Flash σε εργασίες OCR, κατανόησης πινάκων και γραφημάτων και ισοδυναμεί σε γενικές εργασίες οπτικής γνώσης και λογικής. Υποστηρίζει πολυ-καρέ είσοδο, δηλαδή εκτελεί λογική σε πολλές εισερχόμενες εικόνες.


### Phi-3.5-MoE

***Μείγμα Ειδικών (Mixture of Experts, MoE)*** επιτρέπει στα μοντέλα να εκπαιδεύονται με πολύ λιγότερη υπολογιστική ισχύ, που σημαίνει ότι μπορείτε δραματικά να κλιμακώσετε το μέγεθος του μοντέλου ή του συνόλου δεδομένων με τον ίδιο προϋπολογισμό υπολογισμών όπως ένα πυκνό μοντέλο. Συγκεκριμένα, ένα μοντέλο MoE θα πρέπει να επιτυγχάνει την ίδια ποιότητα με τον πυκνό αντίγραφό του πολύ πιο γρήγορα κατά την προεκπαίδευση.

Το Phi-3.5-MoE αποτελείται από 16x3.8B ειδικούς πυρήνες. Το Phi-3.5-MoE με μόλις 6.6B ενεργές παραμέτρους επιτυγχάνει παρόμοιο επίπεδο λογικής, κατανόησης γλώσσας και μαθηματικών με πολύ μεγαλύτερα μοντέλα.

Μπορούμε να χρησιμοποιήσουμε το μοντέλο από την οικογένεια Phi-3/3.5 με βάση διαφορετικά σενάρια. Σε αντίθεση με τα LLM, μπορείτε να αναπτύξετε το Phi-3/3.5-mini ή το Phi-3/3.5-Vision σε συσκευές άκρης.

**Hugging Face Transformer**
Το Hugging Face Transformers είναι μια ισχυρή βιβλιοθήκη σχεδιασμένη για επεξεργασία φυσικής γλώσσας (NLP) και άλλες εργασίες μηχανικής μάθησης. Εδώ είναι μερικά βασικά σημεία γι’ αυτήν:

1. **Προεκπαιδευμένα Μοντέλα**: Παρέχει χιλιάδες προεκπαιδευμένα μοντέλα που μπορούν να χρησιμοποιηθούν για διάφορες εργασίες όπως ταξινόμηση κειμένου, αναγνώριση οντοτήτων, απαντήσεις σε ερωτήσεις, περίληψη, μετάφραση και δημιουργία κειμένου.

2. **Διαλειτουργικότητα Πλατφορμών:** Η βιβλιοθήκη υποστηρίζει πολλαπλά πλαίσια βαθιάς μάθησης, όπως PyTorch, TensorFlow και JAX. Αυτό επιτρέπει την εκπαίδευση ενός μοντέλου σε ένα πλαίσιο και τη χρήση του σε άλλο.

3. **Πολυτροπικές Δυνατότητες:** Εκτός από την NLP, το Hugging Face Transformers υποστηρίζει εργασίες στην υπολογιστική όραση (π.χ. ταξινόμηση εικόνων, ανίχνευση αντικειμένων) και επεξεργασία ήχου (π.χ. αναγνώριση ομιλίας, ταξινόμηση ήχου).

4. **Ευκολία Χρήσης:** Η βιβλιοθήκη προσφέρει APIs και εργαλεία για εύκολη λήψη και προσαρμογή μοντέλων, κάνοντας την προσβάσιμη τόσο για αρχάριους όσο και για ειδικούς.

5. **Κοινότητα και Πόροι:** Το Hugging Face έχει μια ζωντανή κοινότητα και εκτενή τεκμηρίωση, σεμινάρια και οδηγούς για να βοηθήσει τους χρήστες να ξεκινήσουν και να αξιοποιήσουν στο έπακρο τη βιβλιοθήκη.

**ONNX Runtime για GenAI**

[Το ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) είναι ένας επιταχυντής μηχανικής μάθησης για συμπερασμό και εκπαίδευση πολλαπλών πλατφορμών. Το ONNX Runtime για Γενετική AI (GENAI) είναι ένα ισχυρό εργαλείο που βοηθά στην αποδοτική εκτέλεση γενετικών μοντέλων AI σε διάφορες πλατφόρμες. 

## Τι είναι το ONNX Runtime;
Το ONNX Runtime είναι ένα ανοιχτού κώδικα έργο που επιτρέπει αποδοτικό συμπερασμό μοντέλων μηχανικής μάθησης. Υποστηρίζει μοντέλα σε μορφή Open Neural Network Exchange (ONNX), που είναι ένα πρότυπο για την αναπαράσταση μοντέλων μηχανικής μάθησης. Ο συμπερασμός ONNX Runtime μπορεί να επιταχύνει την εμπειρία του πελάτη και να μειώσει το κόστος, υποστηρίζοντας μοντέλα από πλαίσια βαθιάς μάθησης όπως το PyTorch και το TensorFlow/Keras, καθώς και κλασικές βιβλιοθήκες μηχανικής μάθησης όπως scikit-learn, LightGBM, XGBoost κ.ά. Το ONNX Runtime είναι συμβατό με διαφορετικό υλικό, οδηγούς και λειτουργικά συστήματα και παρέχει βέλτιστη απόδοση αξιοποιώντας επιταχυντές υλικού όπου είναι διαθέσιμοι, παράλληλα με βελτιστοποιήσεις και μετασχηματισμούς γραφημάτων.

## Τι είναι η Γενετική AI;
Η Γενετική AI αναφέρεται σε συστήματα AI που μπορούν να δημιουργήσουν νέο περιεχόμενο, όπως κείμενο, εικόνες ή μουσική, βασισμένα στα δεδομένα με τα οποία έχουν εκπαιδευτεί. Παραδείγματα είναι γλωσσικά μοντέλα όπως το GPT-3 και μοντέλα δημιουργίας εικόνων όπως το Stable Diffusion. Η βιβλιοθήκη ONNX Runtime για GenAI παρέχει τον γενετικό κύκλο AI για μοντέλα ONNX, συμπεριλαμβανομένου του συμπερασμού με το ONNX Runtime, επεξεργασία λογάριθμων πιθανοτήτων (logits), αναζήτηση και δειγματοληψία και διαχείριση του προσωρινού αποθηκευτικού KV.

## ONNX Runtime για GENAI
Το ONNX Runtime για GENAI επεκτείνει τις δυνατότητες του ONNX Runtime ώστε να υποστηρίζει γενετικά μοντέλα AI. Εδώ είναι μερικά βασικά χαρακτηριστικά:

- **Ευρεία Υποστήριξη Πλατφορμών:** Λειτουργεί σε διάφορες πλατφόρμες, συμπεριλαμβανομένων των Windows, Linux, macOS, Android και iOS.
- **Υποστήριξη Μοντέλων:** Υποστηρίζει πολλά δημοφιλή γενετικά μοντέλα AI, όπως τα LLaMA, GPT-Neo, BLOOM και άλλα.
- **Βελτιστοποίηση Απόδοσης:** Περιλαμβάνει βελτιστοποιήσεις για διάφορους επιταχυντές υλικού όπως NVIDIA GPUs, AMD GPUs, και άλλα2.
- **Ευκολία Χρήσης:** Παρέχει APIs για εύκολη ενσωμάτωση σε εφαρμογές, επιτρέποντάς σας να παράγετε κείμενο, εικόνες και άλλο περιεχόμενο με ελάχιστο κώδικα
- Οι χρήστες μπορούν να καλέσουν μια υψηλού επιπέδου μέθοδο generate(), ή να τρέξουν κάθε επανάληψη του μοντέλου σε βρόχο, παράγοντας ένα token τη φορά, και προαιρετικά ενημερώνοντας παραμέτρους μέσα στον βρόχο.
- Το ONNX runtime υποστηρίζει επίσης greedy/beam search και δειγματοληψία TopP, TopK για τη δημιουργία ακολουθιών token και ενσωματωμένη επεξεργασία logits όπως ποινές επανάληψης. Μπορείτε επίσης εύκολα να προσθέσετε προσαρμοσμένη βαθμολόγηση.

```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### Demo: Χρήση του ONNX Runtime GenAI για κλήση του Phi-3.5-Vision


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
