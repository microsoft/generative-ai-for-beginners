# Ο Κύκλος Ζωής των Εφαρμογών Γεννητικής Τεχνητής Νοημοσύνης

Ένα σημαντικό ζήτημα για όλες τις εφαρμογές Τεχνητής Νοημοσύνης είναι η σχετικότητα των χαρακτηριστικών ΤΝ, καθώς η ΤΝ είναι ένα ταχέως εξελισσόμενο πεδίο, για να διασφαλίσετε ότι η εφαρμογή σας παραμένει σχετική, αξιόπιστη και ανθεκτική, πρέπει να την παρακολουθείτε, να την αξιολογείτε και να την βελτιώνετε συνεχώς. Εδώ έρχεται ο κύκλος ζωής της γεννητικής ΤΝ.

Ο κύκλος ζωής της γεννητικής ΤΝ είναι ένα πλαίσιο που σας καθοδηγεί μέσα από τα στάδια ανάπτυξης, ανάπτυξης και συντήρησης μιας εφαρμογής γεννητικής ΤΝ. Σας βοηθά να ορίσετε τους στόχους σας, να μετρήσετε την απόδοσή σας, να εντοπίσετε τις προκλήσεις σας και να εφαρμόσετε τις λύσεις σας. Σας βοηθά επίσης να ευθυγραμμίσετε την εφαρμογή σας με τα ηθικά και νομικά πρότυπα του τομέα σας και των εμπλεκόμενων μερών σας. Ακολουθώντας τον κύκλο ζωής της γεννητικής ΤΝ, μπορείτε να διασφαλίσετε ότι η εφαρμογή σας παρέχει πάντα αξία και ικανοποιεί τους χρήστες σας.

## Εισαγωγή

Σε αυτό το κεφάλαιο, θα:

- Κατανοήσετε την Αλλαγή Παραδείγματος από τα MLOps στα LLMOps
- Ο Κύκλος Ζωής των LLM
- Εργαλεία Κύκλου Ζωής
- Μέτρηση και Αξιολόγηση Κύκλου Ζωής

## Κατανοήστε την Αλλαγή Παραδείγματος από τα MLOps στα LLMOps

Τα LLMs είναι ένα νέο εργαλείο στο οπλοστάσιο της Τεχνητής Νοημοσύνης, είναι εξαιρετικά ισχυρά σε αναλυτικές και γενετικές εργασίες για εφαρμογές, ωστόσο αυτή η ισχύς έχει ορισμένες συνέπειες στον τρόπο που βελτιστοποιούμε τις εργασίες ΤΝ και Κλασικής Μηχανικής Μάθησης.

Με αυτό, χρειαζόμαστε ένα νέο Παράδειγμα για να προσαρμόσουμε αυτό το εργαλείο δυναμικά, με τα σωστά κίνητρα. Μπορούμε να κατηγοριοποιήσουμε παλαιότερες εφαρμογές ΤΝ ως "ML Apps" και νεότερες εφαρμογές ΤΝ ως "GenAI Apps" ή απλά "AI Apps", αντανακλώντας την κύρια τεχνολογία και τις τεχνικές που χρησιμοποιούνταν εκείνη την εποχή. Αυτό μετατοπίζει την αφήγησή μας με πολλούς τρόπους, δείτε την παρακάτω σύγκριση.

![Σύγκριση LLMOps vs. MLOps](../../../translated_images/el/01-llmops-shift.29bc933cb3bb0080.webp)

Παρατηρήστε ότι στα LLMOps, δίνουμε μεγαλύτερη έμφαση στους προγραμματιστές εφαρμογών, χρησιμοποιώντας ενσωματώσεις ως βασικό σημείο, χρησιμοποιώντας "Models-as-a-Service" και σκεφτόμαστε τα εξής σημεία για τις μετρήσεις.

- Ποιότητα: Ποιότητα Απάντησης
- Βλάβη: Υπεύθυνη ΤΝ
- Ειλικρίνεια: Βάση Απάντησης (Έχει νόημα; Είναι σωστή;)
- Κόστος: Προϋπολογισμός Λύσης
- Καθυστέρηση: Μέσος χρόνος απάντησης ενός token

## Ο Κύκλος Ζωής των LLM

Πρώτα, για να κατανοήσουμε τον κύκλο ζωής και τις τροποποιήσεις, ας δούμε το παρακάτω infographic.

![Infographic LLMOps](../../../translated_images/el/02-llmops.70a942ead05a7645.webp)

Όπως θα παρατηρήσετε, αυτό διαφέρει από τους συνήθεις Κύκλους Ζωής των MLOps. Τα LLM έχουν πολλές νέες απαιτήσεις, όπως Prompting, διαφορετικές τεχνικές για βελτίωση ποιότητας (Fine-Tuning, RAG, Meta-Prompts), διαφορετική αξιολόγηση και υπευθυνότητα με υπεύθυνη ΤΝ, τέλος, νέες μετρικές αξιολόγησης (Ποιότητα, Βλάβη, Ειλικρίνεια, Κόστος και Καθυστέρηση).

Για παράδειγμα, δούμε πώς κάνουμε ιδέες. Χρησιμοποιώντας μηχανική προτροπής για να πειραματιστούμε με διάφορα LLM για να εξερευνήσουμε τις δυνατότητες για να δοκιμάσουμε αν η υπόθεσή τους θα μπορούσε να είναι σωστή.

Σημειώστε ότι αυτό δεν είναι γραμμικό, αλλά ενσωματωμένοι κύκλοι, επαναληπτικοί και με έναν ευρύτερο κύκλο.

Πώς μπορούμε να εξερευνήσουμε αυτά τα βήματα; Ας δούμε λεπτομερώς πώς μπορούμε να κατασκευάσουμε έναν κύκλο ζωής.

![Ροές εργασίας LLMOps](../../../translated_images/el/03-llm-stage-flows.3a1e1c401235a6cf.webp)

Αυτό μπορεί να φαίνεται λίγο περίπλοκο, ας επικεντρωθούμε πρώτα στα τρία μεγάλα βήματα.

1. Ιδεασμός/Εξερεύνηση: Εξερεύνηση, εδώ μπορούμε να εξερευνήσουμε σύμφωνα με τις επιχειρηματικές μας ανάγκες. Δημιουργία πρωτοτύπου, δημιουργώντας ένα [PromptFlow](https://microsoft.github.io/promptflow/index.html?WT.mc_id=academic-105485-koreyst) και να δοκιμάσουμε αν είναι αρκετά αποδοτικό για την υπόθεσή μας.
1. Κατασκευή/Ενίσχυση: Υλοποίηση, τώρα, αρχίζουμε να αξιολογούμε για μεγαλύτερα σύνολα δεδομένων, εφαρμόζουμε τεχνικές, όπως Fine-tuning και RAG, για να ελέγξουμε την ανθεκτικότητα της λύσης μας. Αν δεν λειτουργεί, η επανυλοποίηση, η προσθήκη νέων βημάτων στη ροή μας ή η αναδιάρθρωση των δεδομένων μπορεί να βοηθήσει. Μετά τη δοκιμή της ροής και της κλίμακας μας, αν λειτουργεί και οι μετρήσεις μας είναι καλές, είναι έτοιμο για το επόμενο βήμα.
1. Ενεργοποίηση: Ενσωμάτωση, τώρα προσθέτουμε συστήματα Παρακολούθησης και Ειδοποιήσεις στο σύστημά μας, ανάπτυξη και ενσωμάτωση εφαρμογής στην Εφαρμογή μας.

Έπειτα, έχουμε τον ευρύτερο κύκλο της Διαχείρισης, εστιάζοντας στην ασφάλεια, τη συμμόρφωση και τη διακυβέρνηση.

Συγχαρητήρια, τώρα έχετε την εφαρμογή ΤΝ έτοιμη για χρήση και λειτουργία. Για μια πρακτική εμπειρία, ρίξτε μια ματιά στο [Contoso Chat Demo.](https://nitya.github.io/contoso-chat/?WT.mc_id=academic-105485-koreyst)

Τώρα, ποια εργαλεία μπορούμε να χρησιμοποιήσουμε;

## Εργαλεία Κύκλου Ζωής

Για εργαλεία, η Microsoft παρέχει την [Azure AI Platform](https://azure.microsoft.com/solutions/ai/?WT.mc_id=academic-105485-koreyst) και το [PromptFlow](https://microsoft.github.io/promptflow/index.html?WT.mc_id=academic-105485-koreyst) που διευκολύνουν και καθιστούν εύκολη την υλοποίηση και τη λειτουργία του κύκλου σας.

Η [Azure AI Platform](https://azure.microsoft.com/solutions/ai/?WT.mc_id=academic-105485-koreyst), σας επιτρέπει να χρησιμοποιήσετε το [Microsoft Foundry](https://ai.azure.com/?WT.mc_id=academic-105485-koreyst). Το Microsoft Foundry (πρώην Azure AI Studio) είναι μια διαδικτυακή πύλη που σας επιτρέπει να εξερευνήσετε μοντέλα, δείγματα και εργαλεία, να διαχειριστείτε τους πόρους σας και να χρησιμοποιήσετε ροές ανάπτυξης UI καθώς και επιλογές SDK/CLI για ανάπτυξη βάσει κώδικα.

![Δυνατότητες Azure AI](../../../translated_images/el/04-azure-ai-platform.80203baf03a12fa8.webp)

Το Azure AI, σας επιτρέπει να χρησιμοποιείτε πολλαπλούς πόρους, για να διαχειριστείτε τις επιχειρήσεις, τις υπηρεσίες, τα έργα, τις ανάγκες αναζήτησης vector και βάσεων δεδομένων.

![LLMOps με Azure AI](../../../translated_images/el/05-llm-azure-ai-prompt.a5ce85cdbb494bdf.webp)

Κατασκευάστε, από Proof-of-Concept (POC) μέχρι μεγάλες κλίμακες εφαρμογών με το PromptFlow:

- Σχεδιάστε και κατασκευάστε εφαρμογές από το VS Code, με οπτικά και λειτουργικά εργαλεία
- Δοκιμάστε και βελτιστοποιήστε τις εφαρμογές σας για ποιοτική ΤΝ, εύκολα.
- Χρησιμοποιήστε το Microsoft Foundry για να ενσωματώσετε και να επαναλάβετε με το cloud, Push και Deploy για γρήγορη ενσωμάτωση.

![LLMOps με PromptFlow](../../../translated_images/el/06-llm-promptflow.a183eba07a3a7fdf.webp)
