# Βελτιστοποίηση του LLM σας

Η χρήση μεγάλων γλωσσικών μοντέλων για την κατασκευή εφαρμογών γεννητικής τεχνητής νοημοσύνης συνοδεύεται από νέες προκλήσεις. Ένα βασικό ζήτημα είναι η διασφάλιση της ποιότητας της απόκρισης (ακρίβεια και σχετικότητα) στο περιεχόμενο που δημιουργεί το μοντέλο για μια συγκεκριμένη αίτηση χρήστη. Σε προηγούμενα μαθήματα, συζητήσαμε τεχνικές όπως η μηχανική εντολών και η αναζήτηση με ενισχυμένη παραγωγή που προσπαθούν να λύσουν το πρόβλημα με το _τροποποίηση της εισόδου ενεργοποίησης_ στο υπάρχον μοντέλο.

Στο σημερινό μάθημα, συζητάμε μια τρίτη τεχνική, την **βελτιστοποίηση (fine-tuning)**, που προσπαθεί να αντιμετωπίσει την πρόκληση με το _εκ νέου εκπαίδευση του μοντέλου_ με επιπλέον δεδομένα. Ας δούμε τις λεπτομέρειες.

## Στόχοι Μάθησης

Αυτό το μάθημα εισάγει την έννοια της βελτιστοποίησης για προεκπαιδευμένα γλωσσικά μοντέλα, εξερευνά τα οφέλη και τις προκλήσεις αυτής της προσέγγισης και παρέχει οδηγίες για το πότε και πώς να χρησιμοποιήσετε τη βελτιστοποίηση για να βελτιώσετε την απόδοση των γεννητικών μοντέλων AI σας.

Στο τέλος αυτού του μαθήματος, θα πρέπει να μπορείτε να απαντήσετε στις ακόλουθες ερωτήσεις:

- Τι είναι η βελτιστοποίηση για γλωσσικά μοντέλα;
- Πότε και γιατί είναι χρήσιμη η βελτιστοποίηση;
- Πώς μπορώ να βελτιστοποιήσω ένα προεκπαιδευμένο μοντέλο;
- Ποιες είναι οι περιορισμοί της βελτιστοποίησης;

Είστε έτοιμοι; Ας ξεκινήσουμε.

## Τι είναι η βελτιστοποίηση για γλωσσικά μοντέλα;

Με τον ορισμό, τα μεγάλα γλωσσικά μοντέλα είναι _προεκπαιδευμένα_ σε μεγάλες ποσότητες κειμένου που προέρχεται από διάφορες πηγές, συμπεριλαμβανομένου του διαδικτύου. Όπως έχουμε μάθει σε προηγούμενα μαθήματα, χρειαζόμαστε τεχνικές όπως η _μηχανική εντολών_ και η _αναζήτηση με ενισχυμένη παραγωγή_ για να βελτιώσουμε την ποιότητα των απαντήσεων του μοντέλου στις ερωτήσεις του χρήστη ("εντολές").

Μια δημοφιλής τεχνική μηχανικής εντολών περιλαμβάνει να δώσουμε στο μοντέλο περισσότερη καθοδήγηση για το τι αναμένεται στην απάντηση είτε μέσω παροχής _οδηγιών_ (ρητή καθοδήγηση) είτε _παρέχοντας μερικά παραδείγματα_ (έμμεση καθοδήγηση). Αυτό αναφέρεται ως _few-shot learning_ αλλά έχει δύο περιορισμούς:

- Τα όρια των token του μοντέλου μπορούν να περιορίσουν τον αριθμό των παραδειγμάτων που μπορείτε να δώσετε και να περιορίσουν την αποτελεσματικότητα.
- Το κόστος των token του μοντέλου μπορεί να κάνει ακριβό το να προσθέσετε παραδείγματα σε κάθε εντολή, περιορίζοντας την ευελιξία.

Η βελτιστοποίηση είναι μια κοινή πρακτική σε συστήματα μηχανικής μάθησης όπου παίρνουμε ένα προεκπαιδευμένο μοντέλο και το εκπαιδεύουμε εκ νέου με νέα δεδομένα για να βελτιώσουμε την απόδοσή του σε μια συγκεκριμένη εργασία. Στο πλαίσιο των γλωσσικών μοντέλων, μπορούμε να βελτιστοποιήσουμε το προεκπαιδευμένο μοντέλο _με ένα επιμελημένο σύνολο παραδειγμάτων για μια συγκεκριμένη εργασία ή τομέα εφαρμογής_ για να δημιουργήσουμε ένα **προσαρμοσμένο μοντέλο** που μπορεί να είναι πιο ακριβές και σχετικό για εκείνη την εργασία ή τομέα. Ένα παράπλευρο όφελος της βελτιστοποίησης είναι ότι μπορεί επίσης να μειώσει τον αριθμό των παραδειγμάτων που χρειάζονται για τη μάθηση με λίγα παραδείγματα - μειώνοντας τη χρήση token και τα σχετικά κόστη.

## Πότε και γιατί πρέπει να βελτιστοποιούμε τα μοντέλα;

Σε _αυτό_ το πλαίσιο, όταν μιλάμε για βελτιστοποίηση, αναφερόμαστε σε **εποπτευόμενη** βελτιστοποίηση όπου η εκπαίδευση εκ νέου γίνεται με **προσθήκη νέων δεδομένων** που δεν ήταν μέρος του αρχικού συνόλου δεδομένων εκπαίδευσης. Αυτό διαφέρει από μια μη εποπτευόμενη προσέγγιση βελτιστοποίησης όπου το μοντέλο εκπαιδεύεται εκ νέου στα αρχικά δεδομένα, αλλά με διαφορετικές υπερπαραμέτρους.

Το βασικό που πρέπει να θυμάστε είναι ότι η βελτιστοποίηση είναι μια προχωρημένη τεχνική που απαιτεί ένα ορισμένο επίπεδο εμπειρογνωμοσύνης για να επιτευχθούν τα επιθυμητά αποτελέσματα. Εάν γίνει λανθασμένα, μπορεί να μην προσφέρει τις αναμενόμενες βελτιώσεις και μπορεί ακόμη και να υποβαθμίσει την απόδοση του μοντέλου για το στοχευμένο σας πεδίο.

Έτσι, πριν μάθετε "πώς" να βελτιστοποιείτε γλωσσικά μοντέλα, πρέπει να ξέρετε "γιατί" πρέπει να ακολουθήσετε αυτή τη διαδρομή και "πότε" να ξεκινήσετε τη διαδικασία βελτιστοποίησης. Ξεκινήστε ρωτώντας τον εαυτό σας αυτές τις ερωτήσεις:

- **Περίπτωση Χρήσης**: Ποια είναι η _περίπτωσή_ σας για βελτιστοποίηση; Ποια πτυχή του τρέχοντος προεκπαιδευμένου μοντέλου θέλετε να βελτιώσετε;
- **Εναλλακτικές**: Έχετε δοκιμάσει _άλλες τεχνικές_ για να επιτύχετε τα επιθυμητά αποτελέσματα; Χρησιμοποιήστε τις για να δημιουργήσετε μια βάση σύγκρισης.
  - Μηχανική Εντολών: Δοκιμάστε τεχνικές όπως few-shot prompting με παραδείγματα σχετικών απαντήσεων εντολών. Αξιολογήστε την ποιότητα των απαντήσεων.
  - Αναζήτηση με Ενισχυμένη Παραγωγή: Δοκιμάστε να ενισχύσετε τις εντολές με αποτελέσματα ερωτημάτων που ανακτώνται μέσω αναζήτησης των δεδομένων σας. Αξιολογήστε την ποιότητα των απαντήσεων.
- **Κόστη**: Έχετε προσδιορίσει το κόστος για τη βελτιστοποίηση;
  - Δυνατότητα Βελτιστοποίησης - είναι το προεκπαιδευμένο μοντέλο διαθέσιμο για βελτιστοποίηση;
  - Προσπάθεια - για την προετοιμασία των δεδομένων εκπαίδευσης, την αξιολόγηση & βελτίωση του μοντέλου.
  - Υπολογιστική ισχύς - για την εκτέλεση των εργασιών βελτιστοποίησης και την ανάπτυξη του βελτιστοποιημένου μοντέλου
  - Δεδομένα - πρόσβαση σε επαρκή ποιοτικά παραδείγματα για τον αντίκτυπο της βελτιστοποίησης
- **Οφέλη**: Έχετε επιβεβαιώσει τα οφέλη για τη βελτιστοποίηση;
  - Ποιότητα - ξεπέρασε το βελτιστοποιημένο μοντέλο τη βάση σύγκρισης;
  - Κόστος - μειώνει τη χρήση token απλοποιώντας τις εντολές;
  - Επεκτασιμότητα - μπορείτε να επαναχρησιμοποιήσετε το βασικό μοντέλο για νέους τομείς;

Απαντώντας σε αυτές τις ερωτήσεις, θα πρέπει να μπορείτε να αποφασίσετε αν η βελτιστοποίηση είναι η σωστή προσέγγιση για την περίπτωσή σας. Ιδανικά, η προσέγγιση είναι έγκυρη μόνο αν τα οφέλη υπερτερούν των κόστους. Μόλις αποφασίσετε να προχωρήσετε, ήρθε η ώρα να σκεφτείτε _πώς_ μπορείτε να βελτιστοποιήσετε το προεκπαιδευμένο μοντέλο.

Θέλετε να πάρετε περισσότερες πληροφορίες σχετικά με τη διαδικασία λήψης αποφάσεων; Παρακολουθήστε [To fine-tune or not to fine-tune](https://www.youtube.com/watch?v=0Jo-z-MFxJs)

## Πώς μπορούμε να βελτιστοποιήσουμε ένα προεκπαιδευμένο μοντέλο;

Για να βελτιστοποιήσετε ένα προεκπαιδευμένο μοντέλο, πρέπει να έχετε:

- ένα προεκπαιδευμένο μοντέλο για βελτιστοποίηση
- ένα σύνολο δεδομένων για χρήση στη βελτιστοποίηση
- ένα περιβάλλον εκπαίδευσης για την εκτέλεση της εργασίας βελτιστοποίησης
- ένα περιβάλλον φιλοξενίας για την ανάπτυξη του βελτιστοποιημένου μοντέλου

## Βελτιστοποίηση στο Microsoft Foundry

Το [Microsoft Foundry](https://ai.azure.com?WT.mc_id=academic-105485-koreyst) είναι ο χώρος όπου βελτιστοποιείτε, αναπτύσσετε και διαχειρίζεστε προσαρμοσμένα μοντέλα στο Azure σήμερα (συνδυάζει αυτό που παλαιότερα ήταν το Azure OpenAI Studio και το Azure AI Studio). Πριν ξεκινήσετε μια εργασία, βοηθά να κατανοήσετε τις επιλογές που σας δίνει το Foundry - και τις βέλτιστες πρακτικές που προτείνει η πλατφόρμα. Στο παρασκήνιο, το Foundry χρησιμοποιεί **LoRA (low-rank adaptation)** για να βελτιστοποιεί τα μοντέλα αποτελεσματικά, διατηρώντας την εκπαίδευση ταχύτερη και πιο προσιτή σε σύγκριση με την εκπαίδευση εκ νέου όλων των βαρών.

### Βήμα 1: Επιλέξτε μια τεχνική εκπαίδευσης

Το Foundry υποστηρίζει τρεις τεχνικές βελτιστοποίησης. **Ξεκινήστε με το SFT** - καλύπτει το ευρύτερο φάσμα σεναρίων.

| Τεχνική | Τι κάνει | Πότε να την χρησιμοποιήσετε |
| --- | --- | --- |
| **Εποπτευόμενη βελτιστοποίηση (SFT)** | Εκπαιδεύει σε ζεύγη παραδειγμάτων εισόδου/εξόδου ώστε το μοντέλο να μάθει να παράγει τις απαντήσεις που θέλετε. | Η προεπιλογή για τις περισσότερες εργασίες: εξειδίκευση τομέα, απόδοση εργασίας, στυλ και τόνος, ακολουθία οδηγιών και προσαρμογή γλώσσας. |
| **Άμεση βελτιστοποίηση προτίμησης (DPO)** | Μαθαίνει από ζεύγη προτιμώμενων έναντι μη προτιμώμενων απαντήσεων για να ευθυγραμμίσει τις εξόδους με τις ανθρώπινες προτιμήσεις. | Βελτίωση ποιότητας απαντήσεων, ασφάλειας και ευθυγράμμισης όταν έχετε συγκριτική ανατροφοδότηση. |
| **Βελτιστοποίηση με ενίσχυση (RFT)** | Χρησιμοποιεί σήματα ανταμοιβής από _βαθμολογητές_ για να βελτιστοποιήσει σύνθετες συμπεριφορές με μάθηση ενίσχυσης. | Αντικειμενικοί, με βαριά λογική τομείς (μαθηματικά, χημεία, φυσική) με σαφείς σωστές/λάθος απαντήσεις. Απαιτεί περισσότερη εξειδίκευση στην ML. |

### Βήμα 2: Επιλέξτε ένα επίπεδο εκπαίδευσης

Το Foundry σας επιτρέπει να επιλέξετε πώς και πού θα τρέξει η εκπαίδευση:

- **Standard** - εκπαιδεύεται στην περιοχή των πόρων σας και εγγυάται τη διαμονή των δεδομένων. Χρησιμοποιήστε το όταν τα δεδομένα πρέπει να παραμείνουν σε συγκεκριμένη περιοχή.
- **Global** - φθηνότερο και ταχύτερο στην ουρά εκτέλεσης χρησιμοποιώντας χωρητικότητα πέρα από την περιοχή σας (τα δεδομένα και τα βάρη αντιγράφονται στην περιοχή εκπαίδευσης). Μια καλή προεπιλογή όταν η διαμονή των δεδομένων δεν είναι απαραίτητη.
- **Developer** - το χαμηλότερο κόστος, χρησιμοποιώντας αδρανή χωρητικότητα χωρίς εγγυήσεις καθυστέρησης/SLA (οι εργασίες μπορούν να διακοπούν και να συνεχιστούν). Ιδανικό για πειραματισμό.

### Βήμα 3: Επιλέξτε ένα βασικό μοντέλο

Τα μοντέλα που μπορούν να βελτιστοποιηθούν περιλαμβάνουν τα OpenAI `gpt-4o-mini`, `gpt-4o`, `gpt-4.1`, `gpt-4.1-mini`, και `gpt-4.1-nano` (SFT; η οικογένεια 4o/4.1 υποστηρίζει επίσης το DPO), τα μοντέλα λογικής `o4-mini` και `gpt-5` (RFT), καθώς και ανοικτού κώδικα μοντέλα όπως τα `Ministral-3B`, `Qwen-32B`, `Llama-3.3-70B-Instruct`, και `gpt-oss-20b` (SFT σε πόρους του Foundry). Ελέγξτε πάντα τη τρέχουσα [Λίστα μοντέλων βελτιστοποίησης](https://learn.microsoft.com/azure/ai-foundry/foundry-models/concepts/models-sold-directly-by-azure?WT.mc_id=academic-105485-koreyst#fine-tuning-models) για υποστηριζόμενες μεθόδους, περιοχές και διαθεσιμότητα.

> Το Foundry προσφέρει δύο τρόπους: **χωρίς διακομιστή** (τιμολόγηση βάσει κατανάλωσης, χωρίς διαχείριση GPU quota, OpenAI και επιλεγμένα μοντέλα) και **διαχειριζόμενη υπολογιστική** (bring-your-own VMs μέσω του Azure Machine Learning για το πιο ευρύ φάσμα μοντέλων). Οι περισσότεροι πρέπει να ξεκινήσουν με τον τρόπο χωρίς διακομιστή.

### Βέλτιστες πρακτικές Foundry

- **Ξεκινήστε με τη βάση.** Μετρήστε το βασικό μοντέλο με μηχανική εντολών και RAG _πριν_ βελτιστοποιήσετε, ώστε να μπορείτε να αποδείξετε το όφελος.
- **Ξεκινήστε μικρά, μετά κλιμακώστε.** Ξεκινήστε με 50-100 υψηλής ποιότητας παραδείγματα για να επαληθεύσετε την προσέγγιση, μετά αυξήστε σε 500+ για παραγωγή. Η ποιότητα υπερτερεί της ποσότητας - απομακρύνετε τα χαμηλής ποιότητας παραδείγματα.
- **Μορφοποιήστε σωστά τα δεδομένα.** Τα αρχεία εκπαίδευσης και επικύρωσης πρέπει να είναι JSONL, UTF-8 **με BOM**, κάτω από 512 MB, χρησιμοποιώντας τη μορφή μηνυμάτων chat-completions. Πάντα να περιλαμβάνετε ένα αρχείο επικύρωσης για να παρακολουθείτε πιθανή υπερεκπαίδευση.
- **Διατηρήστε το prompt συστήματος κατά την ερμηνεία.** Χρησιμοποιήστε το ίδιο μήνυμα συστήματος όταν καλείτε το μοντέλο όπως και κατά την εκπαίδευση.
- **Αξιολογήστε τα σημεία ελέγχου - μην αναπτύσσετε τυφλά το τελευταίο.** Το Foundry διατηρεί τις τελευταίες τρεις εποχές ως σημεία ελέγχου προς ανάπτυξη· επιλέξτε αυτό που γενικεύει καλύτερα παρακολουθώντας το `train_loss` / `valid_loss` και την ακρίβεια token.
- **Μετρήστε το κόστος token μαζί με την ποιότητα** κατά τη σύγκριση του βελτιστοποιημένου μοντέλου με τη βάση.
- **Επαναλάβετε με συνεχή βελτιστοποίηση.** Μπορείτε να βελτιστοποιήσετε ένα ήδη βελτιστοποιημένο μοντέλο με νέα δεδομένα (υποστηρίζεται για μοντέλα OpenAI).
- **Προσέξτε τα κόστη φιλοξενίας.** Ένα αναπτυγμένο προσαρμοσμένο μοντέλο χρεώνεται ανά ώρα, και μια ανενεργή ανάπτυξη αφαιρείται μετά από 15 ημέρες - καθαρίστε ό,τι δεν χρειάζεστε.

Διατρέξτε τον συνολικό οδηγό στο [Customize a model with fine-tuning](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning?WT.mc_id=academic-105485-koreyst) και δείτε τους οδηγούς για [DPO](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning-direct-preference-optimization?WT.mc_id=academic-105485-koreyst) και [RFT](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/reinforcement-fine-tuning?WT.mc_id=academic-105485-koreyst) όταν είστε έτοιμοι για τις άλλες τεχνικές.
