# Θεμελιώδεις Αρχές Μηχανικής Προτροπών

[![Θεμελιώδεις Αρχές Μηχανικής Προτροπών](../../../translated_images/el/04-lesson-banner.a2c90deba7fedacd.webp)](https://youtu.be/GElCu2kUlRs?si=qrXsBvXnCW12epb8)

## Εισαγωγή
Αυτό το μάθημα καλύπτει βασικές έννοιες και τεχνικές για τη δημιουργία αποτελεσματικών προτροπών σε γενετικά μοντέλα τεχνητής νοημοσύνης. Ο τρόπος με τον οποίο γράφετε την προτροπή σε ένα LLM έχει επίσης σημασία. Μια προσεκτικά σχεδιασμένη προτροπή μπορεί να επιτύχει καλύτερη ποιότητα απάντησης. Αλλά τι ακριβώς σημαίνουν οι όροι _προτροπή_ και _μηχανική προτροπής_; Και πώς βελτιώνω την προτροπή _εισόδου_ που στέλνω στο LLM; Αυτές είναι οι ερωτήσεις που θα προσπαθήσουμε να απαντήσουμε μέσα σε αυτό το κεφάλαιο και το επόμενο.

Η _γενετική τεχνητή νοημοσύνη_ είναι ικανή να δημιουργεί νέο περιεχόμενο (π.χ., κείμενο, εικόνες, ήχο, κώδικα κ.ά.) ως απάντηση σε αιτήματα χρήστη. Το επιτυγχάνει χρησιμοποιώντας _Μεγάλα Γλωσσικά Μοντέλα_ όπως η σειρά GPT της OpenAI ("Generative Pre-trained Transformer") που εκπαιδεύονται για χρήση φυσικής γλώσσας και κώδικα.

Οι χρήστες μπορούν πλέον να αλληλεπιδρούν με αυτά τα μοντέλα χρησιμοποιώντας γνώριμα πρότυπα όπως τη συνομιλία, χωρίς να χρειάζεται τεχνική εξειδίκευση ή εκπαίδευση. Τα μοντέλα είναι _με βάση προτροπή_ - οι χρήστες στέλνουν κείμενο (προτροπή) και λαμβάνουν την απάντηση της τεχνητής νοημοσύνης (συμπλήρωση). Μπορούν στη συνέχεια να "συνομιλούν με την AI" επαναληπτικά, σε πολυστροφικές συνομιλίες, βελτιώνοντας την προτροπή τους μέχρι η απάντηση να ανταποκρίνεται στις προσδοκίες τους.

Οι «Προτροπές» γίνονται πλέον η βασική _διεπαφή προγραμματισμού_ για εφαρμογές γενετικής AI, λέγοντας στα μοντέλα τι να κάνουν και επηρεάζοντας την ποιότητα των απαντήσεων που επιστρέφουν. Η «Μηχανική Προτροπής» είναι ένας ταχέως αναπτυσσόμενος τομέας που εστιάζει στο _σχεδιασμό και τη βελτιστοποίηση_ των προτροπών ώστε να παρέχουν σταθερές και ποιοτικές απαντήσεις σε μεγάλη κλίμακα.

## Στόχοι Μάθησης

Σε αυτό το μάθημα, μαθαίνουμε τι είναι η Μηχανική Προτροπής, γιατί έχει σημασία και πώς μπορούμε να δημιουργήσουμε πιο αποτελεσματικές προτροπές για ένα δεδομένο μοντέλο και στόχο εφαρμογής. Θα κατανοήσουμε βασικές έννοιες και βέλτιστες πρακτικές για τη μηχανική προτροπής και θα μάθουμε για ένα διαδραστικό περιβάλλον Jupyter Notebooks "sandbox" όπου μπορούμε να δούμε αυτές τις έννοιες εφαρμοσμένες σε πραγματικά παραδείγματα.

Στο τέλος του μαθήματος θα είμαστε σε θέση να:

1. Εξηγήσουμε τι είναι η μηχανική προτροπής και γιατί έχει σημασία.
2. Περιγράψουμε τα συστατικά μιας προτροπής και πώς χρησιμοποιούνται.
3. Μάθουμε βέλτιστες πρακτικές και τεχνικές για τη μηχανική προτροπής.
4. Εφαρμόσουμε τις τεχνικές που έχουμε μάθει σε πραγματικά παραδείγματα, χρησιμοποιώντας ένα σημείο πρόσβασης OpenAI.

## Βασικοί Όροι

Μηχανική Προτροπής: Η πρακτική του σχεδιασμού και της βελτίωσης εισόδων για την καθοδήγηση των μοντέλων AI προς την παραγωγή επιθυμητών εξόδων.  
Tokenization (Κατηγοριοποίηση σε tokens): Η διαδικασία μετατροπής κειμένου σε μικρότερα τμήματα, που ονομάζονται tokens, τα οποία μπορεί να κατανοεί και να επεξεργάζεται ένα μοντέλο.  
Instruction-Tuned LLMs: Μεγάλα Γλωσσικά Μοντέλα (LLMs) που έχουν βελτιστοποιηθεί με συγκεκριμένες οδηγίες για να βελτιώσουν την ακρίβεια και τη σχετικότητα των απαντήσεών τους.

## Περιβάλλον Εκμάθησης Sandbox

Η μηχανική προτροπής είναι προς το παρόν περισσότερο τέχνη παρά επιστήμη. Ο καλύτερος τρόπος να βελτιώσουμε την αντίληψή μας γι’ αυτήν είναι να _εξασκηθούμε περισσότερο_ και να υιοθετήσουμε μια διαδικασία δοκιμής και λάθους που συνδυάζει την τεχνογνωσία του τομέα εφαρμογής με προτεινόμενες τεχνικές και βελτιώσεις προσαρμοσμένες στο μοντέλο.

Το Jupyter Notebook που συνοδεύει αυτό το μάθημα παρέχει ένα περιβάλλον _sandbox_ όπου μπορείτε να δοκιμάσετε όσα μαθαίνετε - καθώς προχωράτε ή ως μέρος της άσκησης κώδικα στο τέλος. Για να εκτελέσετε τις ασκήσεις, θα χρειαστείτε:

Το notebook συνοδεύεται από _ασκήσεις εκκίνησης_ - αλλά σας ενθαρρύνουμε να προσθέσετε δικά σας τμήματα _Markdown_ (περιγραφή) και _Κώδικα_ (αιτήματα προτροπής) για να δοκιμάσετε περισσότερα παραδείγματα ή ιδέες - και να χτίσετε την αντίληψή σας για το σχεδιασμό προτροπών.

## Τι είναι η Μηχανική Προτροπών;

Ξεκινήσαμε αυτό το μάθημα ορίζοντας τη **Μηχανική Προτροπών** ως τη διαδικασία _σχεδιασμού και βελτιστοποίησης_ κειμενικών εισόδων (προτροπών) για την παροχή συνεπών και ποιοτικών απαντήσεων (συμπληρώσεων) για συγκεκριμένο στόχο εφαρμογής και μοντέλο. Μπορούμε να το σκεφτούμε ως μια διαδικασία δύο βημάτων:

- _σχεδιασμός_ της αρχικής προτροπής για ένα δεδομένο μοντέλο και στόχο  
- _βελτίωση_ της προτροπής επαναληπτικά για να βελτιωθεί η ποιότητα της απάντησης

Αυτή είναι αναπόφευκτα μια διαδικασία δοκιμής και λάθους που απαιτεί διαίσθηση και προσπάθεια από τον χρήστη για να επιτευχθούν βέλτιστα αποτελέσματα. Γιατί λοιπόν είναι σημαντική; Για να απαντήσουμε σε αυτήν την ερώτηση, χρειάζεται πρώτα να κατανοήσουμε τρεις έννοιες:

- _Κατηγοριοποίηση σε tokens_ = πώς το μοντέλο "βλέπει" την προτροπή  
- _Βασικά LLMs_ = πώς το βασικό μοντέλο "επεξεργάζεται" μια προτροπή  
- _Instruction-Tuned LLMs_ = πώς το μοντέλο μπορεί πλέον να βλέπει "ειδικές εργασίες"

### Κατηγοριοποίηση σε Tokens

Ένα LLM βλέπει τις προτροπές ως _σειρά tokens_ όπου διαφορετικά μοντέλα (ή εκδόσεις ενός μοντέλου) μπορούν να κατηγοριοποιήσουν την ίδια προτροπή με διαφορετικούς τρόπους. Εφόσον τα LLMs εκπαιδεύονται με tokens (και όχι με ακατέργαστο κείμενο), ο τρόπος που κατηγοριοποιούνται οι προτροπές επηρεάζει άμεσα την ποιότητα της παραγόμενης απάντησης.

Για να αποκτήσετε μια διαίσθηση του πώς λειτουργεί η κατηγοριοποίηση, δοκιμάστε εργαλεία όπως ο [OpenAI Tokenizer](https://platform.openai.com/tokenizer?WT.mc_id=academic-105485-koreyst) που φαίνεται παρακάτω. Αντιγράψτε την προτροπή σας – και δείτε πώς μετατρέπεται σε tokens, δίνοντας προσοχή στον τρόπο που χειρίζονται οι χαρακτήρες κενών και τα σημεία στίξης. Σημειώστε ότι αυτό το παράδειγμα δείχνει ένα παλαιότερο LLM (GPT-3) - οπότε η δοκιμή με νεότερο μοντέλο μπορεί να αποδώσει διαφορετικό αποτέλεσμα.

![Κατηγοριοποίηση σε Tokens](../../../translated_images/el/04-tokenizer-example.e71f0a0f70356c5c.webp)

### Έννοια: Βασικά Μοντέλα

Μόλις μια προτροπή κατηγοριοποιηθεί, η βασική λειτουργία του ["Base LLM"](https://blog.gopenai.com/an-introduction-to-base-and-instruction-tuned-large-language-models-8de102c785a6?WT.mc_id=academic-105485-koreyst) (ή Βασικού Μοντέλου) είναι να προβλέπει το token που ακολουθεί σε αυτή τη σειρά. Εφόσον τα LLMs εκπαιδεύονται σε τεράστια σύνολα κειμένων, έχουν καλή αίσθηση των στατιστικών σχέσεων μεταξύ των tokens και μπορούν να κάνουν αυτή την πρόβλεψη με κάποια εμπιστοσύνη. Σημειώστε ότι δεν κατανοούν το _νόημα_ των λέξεων στην προτροπή ή το token, βλέπουν απλώς ένα μοτίβο που μπορούν να "συμπληρώσουν" με την επόμενη πρόβλεψή τους. Μπορούν να συνεχίσουν να προβλέπουν τη σειρά μέχρι να διακοπούν από παρέμβαση χρήστη ή κάποιο προϋπάρχον όρο.

Θέλετε να δείτε πώς λειτουργεί η συμπλήρωση με βάση προτροπές; Εισάγετε την παραπάνω προτροπή στο Azure OpenAI Studio [_Chat Playground_](https://oai.azure.com/playground?WT.mc_id=academic-105485-koreyst) με τις προεπιλεγμένες ρυθμίσεις. Το σύστημα έχει ρυθμιστεί ώστε να μεταχειρίζεται τις προτροπές ως αιτήματα για πληροφορίες – οπότε θα δείτε μια συμπλήρωση που ικανοποιεί αυτό το πλαίσιο.

Αλλά τι γίνεται αν ο χρήστης θέλει να δει κάτι συγκεκριμένο που ανταποκρίνεται σε κάποια κριτήρια ή στόχο εργασίας; Εδώ μπαίνουν στο παιχνίδι τα _LLMs συντονισμένα με οδηγίες_.

![Συμπλήρωση Συνομιλίας με Βασικό LLM](../../../translated_images/el/04-playground-chat-base.65b76fcfde0caa67.webp)

### Έννοια: LLMs Συντονισμένα με Οδηγίες

Ένα [Instruction Tuned LLM](https://blog.gopenai.com/an-introduction-to-base-and-instruction-tuned-large-language-models-8de102c785a6?WT.mc_id=academic-105485-koreyst) ξεκινά με το βασικό μοντέλο και το βελτιστοποιεί περαιτέρω με παραδείγματα ή ζεύγη εισόδου/εξόδου (π.χ., πολλαπλών βημάτων "μηνύματα") που μπορεί να περιέχουν σαφείς οδηγίες – και η απάντηση της AI προσπαθεί να ακολουθήσει αυτές τις οδηγίες.

Αυτό χρησιμοποιεί τεχνικές όπως η Ενισχυτική Μάθηση με Ανθρώπινη Ανατροφοδότηση (RLHF) που μπορούν να εκπαιδεύσουν το μοντέλο να _ακολουθεί οδηγίες_ και να _μαθαίνει από την ανατροφοδότηση_ ώστε να παράγει απαντήσεις που είναι καλύτερα προσαρμοσμένες σε πρακτικές εφαρμογές και πιο σχετικές με τους στόχους του χρήστη.

Ας το δοκιμάσουμε – επιστρέψτε στην προτροπή παραπάνω, αλλά τώρα αλλάξτε το _συστημικό μήνυμα_ για να παρέχετε την παρακάτω οδηγία ως πλαίσιο:

> _Συνοψίστε το περιεχόμενο που σας παρέχεται για ένα μαθητή δεύτερης τάξης. Κρατήστε το αποτέλεσμα σε μια παράγραφο με 3-5 σημεία._

Βλέπετε πώς το αποτέλεσμα τώρα είναι ρυθμισμένο να αντανακλά τον επιθυμητό στόχο και μορφή; Ένας εκπαιδευτικός μπορεί πλέον να χρησιμοποιήσει άμεσα αυτήν την απάντηση στις διαφάνειές του για το συγκεκριμένο μάθημα.

![Συμπλήρωση Συνομιλίας με LLM Συντονισμένο με Οδηγίες](../../../translated_images/el/04-playground-chat-instructions.b30bbfbdf92f2d05.webp)

## Γιατί χρειαζόμαστε τη Μηχανική Προτροπής;

Τώρα που γνωρίζουμε πώς οι προτροπές επεξεργάζονται από τα LLMs, ας μιλήσουμε για το _γιατί_ χρειαζόμαστε τη μηχανική προτροπής. Η απάντηση βρίσκεται στο γεγονός ότι τα τρέχοντα LLMs παρουσιάζουν αρκετές προκλήσεις που καθιστούν την _αξιόπιστη και συνεπή παραγωγή συμπληρώσεων_ πιο δύσκολη χωρίς προσπάθεια στην κατασκευή και βελτιστοποίηση των προτροπών. Για παράδειγμα:

1. **Οι απαντήσεις του μοντέλου είναι στοχαστικές.** Η _ίδια προτροπή_ πιθανόν να παράγει διαφορετικές απαντήσεις με διαφορετικά μοντέλα ή εκδόσεις μοντέλου. Και μπορεί ακόμη να παράγει διαφορετικά αποτελέσματα με το _ίδιο μοντέλο_ σε διαφορετικές χρονικές στιγμές. _Οι τεχνικές μηχανικής προτροπών μπορούν να μας βοηθήσουν να ελαχιστοποιήσουμε αυτές τις διακυμάνσεις με το να παρέχουμε καλύτερους κανόνες_.

1. **Τα μοντέλα μπορεί να επινοήσουν απαντήσεις.** Τα μοντέλα έχουν εκπαιδευτεί με _μεγάλα αλλά πεπερασμένα_ σύνολα δεδομένων, πράγμα που σημαίνει ότι δεν έχουν γνώση για έννοιες εκτός αυτού του πλαισίου εκπαίδευσης. Ως αποτέλεσμα, μπορούν να παράγουν συμπληρώσεις που είναι ανακριβείς, φανταστικές ή και άμεσα αντίθετες με γνωστά γεγονότα. _Η μηχανική προτροπής βοηθά τους χρήστες να εντοπίζουν και να μετριάζουν τέτοιες επινοήσεις π.χ. ζητώντας αναφορές ή λογική_.

1. **Οι δυνατότητες των μοντέλων θα διαφέρουν.** Νεότερα μοντέλα ή γενιές μοντέλων θα έχουν πλουσιότερες δυνατότητες αλλά και ιδιαίτερα χαρακτηριστικά και συμβιβασμούς ως προς κόστος και πολυπλοκότητα. _Η μηχανική προτροπής μπορεί να μας βοηθήσει να αναπτύξουμε βέλτιστες πρακτικές και ροές εργασίας που αφαιρούν τις διαφορές και προσαρμόζονται σε απαιτήσεις συγκεκριμένων μοντέλων με κλιμακωτό και απρόσκοπτο τρόπο_.

Ας δούμε αυτά στην πράξη στο OpenAI ή Azure OpenAI Playground:

- Χρησιμοποιήστε την ίδια προτροπή με διαφορετικές αναπτύξεις LLM (π.χ., OpenAI, Azure OpenAI, Hugging Face) – είδατε τις διαφορές;  
- Χρησιμοποιήστε την ίδια προτροπή επανειλημμένα με την _ίδια_ ανάπτυξη LLM (π.χ., Azure OpenAI playground) – πώς διέφεραν αυτές οι παραλλαγές;

## Κατασκευή Προτροπής

Έχουμε δει γιατί η μηχανική προτροπών είναι σημαντική - τώρα ας κατανοήσουμε πώς _κατασκευάζονται_ οι προτροπές για να αξιολογήσουμε διαφορετικές τεχνικές για πιο αποτελεσματικό σχεδιασμό προτροπής.

### Βασική Προτροπή

Ας ξεκινήσουμε με την βασική προτροπή: μια κειμενική είσοδο που στέλνεται στο μοντέλο χωρίς επιπλέον συμφραζόμενα. Ιδού ένα παράδειγμα - όταν στέλνουμε τις πρώτες λέξεις του εθνικού ύμνου των ΗΠΑ στο OpenAI [Completion API](https://platform.openai.com/docs/api-reference/completions?WT.mc_id=academic-105485-koreyst) ολοκληρώνει αμέσως την απάντηση με τις επόμενες στροφές, και έτσι απεικονίζει τη βασική συμπεριφορά πρόβλεψης.

| Προτροπή (Είσοδος) | Ολοκλήρωση (Έξοδος)                                                                                                                         |
| :------------------ | :--------------------------------------------------------------------------------------------------------------------------------------------- |
| Oh say can you see  | Ακούγεται σαν να ξεκινάς τους στίχους του "The Star-Spangled Banner," του εθνικού ύμνου των Ηνωμένων Πολιτειών. Οι πλήρεις στίχοι είναι ... |

### Σύνθετη Προτροπή

Τώρα ας προσθέσουμε συμφραζόμενα και οδηγίες σε αυτήν τη βασική προτροπή. Το [Chat Completion API](https://learn.microsoft.com/azure/ai-services/openai/how-to/chatgpt?WT.mc_id=academic-105485-koreyst) μας επιτρέπει να κατασκευάσουμε μια σύνθετη προτροπή ως συλλογή _μηνυμάτων_ με:

- Ζεύγη εισόδου/εξόδου που αντανακλούν την είσοδο _χρήστη_ και την απάντηση του _βοηθού_.
- Μήνυμα συστήματος που ρυθμίζει τα συμφραζόμενα για τη συμπεριφορά ή προσωπικότητα του βοηθού.

Το αίτημα τώρα είναι με τη μορφή παρακάτω, όπου η _τοκενικοποίηση_ αποτυπώνει ουσιαστικά τις σχετικές πληροφορίες από τα συμφραζόμενα και τη συνομιλία. Η αλλαγή του συμφραζόμενου συστήματος μπορεί να επηρεάσει εξίσου την ποιότητα των ολοκληρώσεων, όσο και οι είσοδοι του χρήστη που παρέχονται.

```python
response = openai.chat.completions.create(
    model="gpt-3.5-turbo",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Who won the world series in 2020?"},
        {"role": "assistant", "content": "The Los Angeles Dodgers won the World Series in 2020."},
        {"role": "user", "content": "Where was it played?"}
    ]
)
```

## Κύριο Περιεχόμενο

Στα παραπάνω παραδείγματα, η προτροπή ήταν ακόμα αρκετά ανοικτή, επιτρέποντας στο LLM να αποφασίσει ποιο μέρος του προκαθορισμένου συνόλου δεδομένων του είναι σχετικό. Με το σχεδιαστικό μοτίβο του _κύριου περιεχομένου_, το κείμενο εισόδου χωρίζεται σε δύο μέρη:

- μια οδηγία (ενέργεια)
- σχετικό περιεχόμενο (που επηρεάζει την ενέργεια)

Ιδού ένα παράδειγμα όπου η οδηγία είναι «συνοψίστε αυτό σε 2 προτάσεις».

- **Παραδείγματα** - αντί να πούμε στο μοντέλο τι να κάνει με μια ρητή οδηγία, δίνουμε παραδείγματα του τι να κάνει και αφήνουμε το μοντέλο να συμπεράνει το μοτίβο.
- **Ερεθίσματα** - ακολουθούμε την οδηγία με ένα "ερέθισμα" που προετοιμάζει την ολοκλήρωση, καθοδηγώντας το μοντέλο προς πιο σχετικές απαντήσεις.
- **Πρότυπα** - αυτά είναι επαναλαμβανόμενες 'συνταγές' για προτροπές με καθορισμένες θέσεις (μεταβλητές) που μπορούν να προσαρμοστούν με δεδομένα για συγκεκριμένες χρήσεις.

### Χρήση Παραδειγμάτων

Αυτή είναι μια προσέγγιση όπου χρησιμοποιείς το κύριο περιεχόμενο για να "ταΐσεις" το μοντέλο με μερικά παραδείγματα της επιθυμητής εξόδου για μια δεδομένη οδηγία, και το αφήνεις να συμπεράνει το μοτίβο της επιθυμητής εξόδου. Βάσει του αριθμού των παραδειγμάτων που παρέχονται, μπορούμε να έχουμε zero-shot prompting, one-shot prompting, few-shot prompting κ.λπ.

- Μια περιγραφή εργασίας
- Μερικά παραδείγματα της επιθυμητής εξόδου
- Την έναρξη ενός νέου παραδείγματος (που γίνεται μια έμμεση περιγραφή εργασίας)

### Prompt Templates

Ένα πρότυπο prompt είναι μια _προκαθορισμένη συνταγή για ένα prompt_ που μπορεί να αποθηκευτεί και να επαναχρησιμοποιηθεί κατά βούληση, ώστε να προσφέρει πιο συνεπείς εμπειρίες χρήστη σε μεγάλη κλίμακα. Στην πιο απλή του μορφή, είναι απλά μια συλλογή παραδειγμάτων prompt όπως [αυτό από το OpenAI](https://cookbook.openai.com/examples/gpt4-1_prompting_guide?WT.mc_id=academic-105485-koreyst) που παρέχει τόσο τα διαδραστικά στοιχεία του prompt (μηνύματα χρήστη και συστήματος) όσο και τη μορφή του αιτήματος μέσω API - για να υποστηρίξει την επαναχρησιμοποίηση.

Σε πιο πολύπλοκη μορφή όπως [αυτό το παράδειγμα από το LangChain](https://python.langchain.com/docs/concepts/prompt_templates/?WT.mc_id=academic-105485-koreyst) περιέχει _θήκες_ που μπορούν να αντικατασταθούν με δεδομένα από διάφορες πηγές (είσοδος χρήστη, συμφραζόμενα συστήματος, εξωτερικές πηγές δεδομένων κ.ά.) για να παραχθεί δυναμικά ένα prompt. Αυτό μας επιτρέπει να δημιουργούμε μια βιβλιοθήκη επαναχρησιμοποιήσιμων prompts που μπορούν να χρησιμοποιηθούν για να παρέχουν συνεπείς εμπειρίες χρήστη **προγραμματιστικά** σε μεγάλη κλίμακα.

Τέλος, η πραγματική αξία των προτύπων βρίσκεται στην ικανότητα δημιουργίας και δημοσίευσης _βιβλιοθηκών prompt_ για κάθετες εφαρμογές - όπου το πρότυπο prompt είναι πλέον _βελτιστοποιημένο_ ώστε να αντανακλά ειδικό συμφραζόμενο εφαρμογών ή παραδείγματα που κάνουν τις απαντήσεις πιο σχετικές και ακριβείς για το στοχευμένο κοινό. Το αποθετήριο [Prompts For Edu](https://github.com/microsoft/prompts-for-edu?WT.mc_id=academic-105485-koreyst) είναι ένα εξαιρετικό παράδειγμα αυτής της προσέγγισης, το οποίο επιμελείται μια συλλογή prompt για τον τομέα της εκπαίδευσης με έμφαση σε βασικούς στόχους όπως ο σχεδιασμός μαθημάτων, το αναλυτικό πρόγραμμα, η διδασκαλία φοιτητών κ.ά.

## Υποστηρικτικό Περιεχόμενο

Αν σκεφτούμε τη δημιουργία prompt ως τη σύνθεση μιας οδηγίας (εργασίας) και του στόχου (κύριο περιεχόμενο), τότε το _δευτερεύον περιεχόμενο_ είναι σαν πρόσθετο πλαίσιο που παρέχουμε για να **επηρεάσουμε την έξοδο με κάποιον τρόπο**. Μπορεί να είναι παράμετροι ρύθμισης, οδηγίες μορφοποίησης, ταξινομίες θεμάτων κ.ά. που βοηθούν το μοντέλο να _προσαρμόσει_ την απάντησή του ώστε να ταιριάζει στους επιθυμητούς στόχους ή προσδοκίες του χρήστη.

Για παράδειγμα: Δεδομένου ενός καταλόγου μαθημάτων με εκτενή μεταδεδομένα (όνομα, περιγραφή, επίπεδο, ετικέτες μεταδεδομένων, διδάσκων κ.ά.) όλων των διαθέσιμων μαθημάτων στο πρόγραμμα σπουδών:

- μπορούμε να ορίσουμε μια οδηγία για "περίληψη του καταλόγου μαθημάτων για το Φθινόπωρο 2023"
- μπορούμε να χρησιμοποιήσουμε το κύριο περιεχόμενο για να παρέχουμε μερικά παραδείγματα της επιθυμητής εξόδου
- μπορούμε να χρησιμοποιήσουμε το δευτερεύον περιεχόμενο για να προσδιορίσουμε τις κορυφαίες 5 «ετικέτες» ενδιαφέροντος.

Τώρα, το μοντέλο μπορεί να παρέχει περίληψη στη μορφή που δείχνουν τα παραδείγματα - αλλά αν μια καταχώριση έχει πολλές ετικέτες, μπορεί να δώσει προτεραιότητα στις 5 ετικέτες που προσδιορίστηκαν στο δευτερεύον περιεχόμενο.

## Βέλτιστες Πρακτικές Προτροπών

Τώρα που γνωρίζουμε πώς μπορούν να _κατασκευαστούν_ τα prompts, μπορούμε να αρχίσουμε να σκεφτόμαστε πώς να τα _σχεδιάσουμε_ ώστε να αντανακλούν βέλτιστες πρακτικές. Μπορούμε να το σκεφτούμε σε δύο μέρη - έχοντας τη σωστή _νοοτροπία_ και εφαρμόζοντας τις κατάλληλες _τεχνικές_.

### Νοοτροπία Μηχανικής Προτροπών

Η μηχανική προτροπών είναι μια διαδικασία δοκιμής και λάθους, οπότε έχετε υπόψη σας τρεις βασικούς παράγοντες καθοδήγησης:

1. **Η Κατανόηση του Τομέα Μετράει.** Η ακρίβεια και η σχετικότητα της απάντησης εξαρτώνται από το _τομέα_ στον οποίο λειτουργεί η εφαρμογή ή ο χρήστης. Εφαρμόστε την διαίσθηση και την εξειδίκευσή σας στον τομέα για να **προσαρμόσετε περαιτέρω τις τεχνικές**. Για παράδειγμα, ορίστε _προσωπικότητες συγκεκριμένες για τον τομέα_ στις προτροπές του συστήματος, ή χρησιμοποιήστε _πρότυπα συγκεκριμένα για τον τομέα_ στα prompts του χρήστη. Παρέχετε δευτερεύον περιεχόμενο που αντανακλά ειδικά συμφραζόμενα του τομέα ή χρησιμοποιήστε _ενδείξεις και παραδείγματα συγκεκριμένα για τον τομέα_ για να καθοδηγήσετε το μοντέλο προς γνωστά μοτίβα χρήσης.

2. **Η Κατανόηση του Μοντέλου Μετράει.** Ξέρουμε πως τα μοντέλα είναι στοχαστικά από τη φύση τους. Αλλά οι υλοποιήσεις των μοντέλων μπορεί επίσης να διαφέρουν όσον αφορά το εκπαιδευτικό σύνολο δεδομένων που χρησιμοποιούν (προεκπαιδευμένη γνώση), τις δυνατότητες που παρέχουν (π.χ. μέσω API ή SDK) και τον τύπο περιεχομένου για τον οποίο είναι βελτιστοποιημένα (π.χ., κώδικας έναντι εικόνων έναντι κειμένου). Κατανοήστε τα προτερήματα και τους περιορισμούς του μοντέλου που χρησιμοποιείτε και χρησιμοποιήστε αυτή τη γνώση για να _ιεραρχήσετε εργασίες_ ή να δημιουργήσετε _προσαρμοσμένα πρότυπα_ βελτιστοποιημένα για τις ικανότητες του μοντέλου.

3. **Η Επανάληψη & Επικύρωση Μετράει.** Τα μοντέλα εξελίσσονται γρήγορα, όπως και οι τεχνικές μηχανικής προτροπών. Ως ειδικός στον τομέα, ίσως έχετε και άλλες απαιτήσεις ή κριτήρια _η_ συγκεκριμένη εφαρμογή σας, που ίσως δε ισχύουν για την ευρύτερη κοινότητα. Χρησιμοποιήστε εργαλεία και τεχνικές μηχανικής προτροπών για να κάνετε «γρήγορη εκκίνηση» στην κατασκευή prompts, στη συνέχεια επαναλάβετε και επικυρώστε τα αποτελέσματα βάσει της δικής σας διαίσθησης και εμπειρίας στον τομέα. Καταγράψτε τα συμπεράσματά σας και δημιουργήστε μια **βάση γνώσης** (π.χ. βιβλιοθήκες prompt) που μπορεί να χρησιμοποιηθεί ως νέο σημείο αναφοράς από άλλους, για γρηγορότερες επαναλήψεις στο μέλλον.
