# Θεμελιώδεις Αρχές Μηχανικής Προτροπής

[![Θεμελιώδεις Αρχές Μηχανικής Προτροπής](../../../translated_images/el/04-lesson-banner.a2c90deba7fedacd.webp)](https://youtu.be/GElCu2kUlRs?si=qrXsBvXnCW12epb8)

## Εισαγωγή
Αυτή η ενότητα καλύπτει βασικές έννοιες και τεχνικές για τη δημιουργία αποτελεσματικών προτροπών σε γενετικά μοντέλα τεχνητής νοημοσύνης. Ο τρόπος που γράφετε την προτροπή σας σε ένα LLM έχει επίσης σημασία. Μια προσεκτικά σχεδιασμένη προτροπή μπορεί να πετύχει καλύτερη ποιότητα απάντησης. Αλλά τι ακριβώς σημαίνουν οι όροι _προτροπή_ και _μηχανική προτροπής_; Και πώς μπορώ να βελτιώσω την _είσοδο_ προτροπής που στέλνω στο LLM; Αυτά είναι τα ερωτήματα που θα προσπαθήσουμε να απαντήσουμε σε αυτό το κεφάλαιο και το επόμενο.

_Γενετική Τεχνητή Νοημοσύνη_ είναι ικανή να δημιουργεί νέο περιεχόμενο (π.χ., κείμενο, εικόνες, ήχο, κώδικα κ.λπ.) σε απάντηση σε αιτήματα χρηστών. Το επιτυγχάνει αυτό χρησιμοποιώντας _Μεγάλα Μοντέλα Γλώσσας_ όπως η σειρά GPT της OpenAI ("Generative Pre-trained Transformer") που εκπαιδεύονται για χρήση φυσικής γλώσσας και κώδικα.

Τώρα οι χρήστες μπορούν να αλληλεπιδρούν με αυτά τα μοντέλα χρησιμοποιώντας οικεία παραδείγματα όπως η συνομιλία, χωρίς να απαιτείται τεχνική ειδίκευση ή εκπαίδευση. Τα μοντέλα είναι _με βάση τις προτροπές_ - οι χρήστες στέλνουν ένα κείμενο εισόδου (προτροπή) και λαμβάνουν την απάντηση της AI (ολοκλήρωση). Μπορούν στη συνέχεια να "συνομιλούν με την AI" επαναληπτικά, σε συζητήσεις πολλαπλών γύρων, βελτιώνοντας την προτροπή μέχρι η απάντηση να ανταποκρίνεται στις προσδοκίες τους.

Οι "Προτροπές" γίνονται τώρα η κύρια _διεπαφή προγραμματισμού_ για εφαρμογές γενετικής AI, λέγοντας στα μοντέλα τι να κάνουν και επηρεάζοντας την ποιότητα των απαντήσεων που επιστρέφονται. Η "Μηχανική Προτροπής" είναι ένας ταχέως αναπτυσσόμενος τομέας μελέτης που εστιάζει στο _σχεδιασμό και την βελτιστοποίηση_ των προτροπών για να παρέχει συνεπείς και ποιοτικές απαντήσεις σε κλίμακα.

## Στόχοι Μάθησης

Σε αυτό το μάθημα μαθαίνουμε τι είναι η Μηχανική Προτροπής, γιατί έχει σημασία, και πώς μπορούμε να δημιουργήσουμε πιο αποτελεσματικές προτροπές για ένα δεδομένο μοντέλο και στόχο εφαρμογής. Θα κατανοήσουμε βασικές έννοιες και καλές πρακτικές για τη μηχανική προτροπής - και θα μάθουμε για ένα διαδραστικό περιβάλλον Jupyter Notebooks "sandbox" όπου μπορούμε να δούμε αυτές τις έννοιες να εφαρμόζονται σε πραγματικά παραδείγματα.

Μέχρι το τέλος αυτού του μαθήματος θα είμαστε σε θέση να:

1. Εξηγήσουμε τι είναι η μηχανική προτροπής και γιατί έχει σημασία.
2. Περιγράψουμε τα συστατικά μιας προτροπής και πώς χρησιμοποιούνται.
3. Μάθουμε τις καλύτερες πρακτικές και τεχνικές για μηχανική προτροπής.
4. Εφαρμόσουμε τις τεχνικές που μάθαμε σε πραγματικά παραδείγματα, χρησιμοποιώντας ένα endpoint της OpenAI.

## Κύριοι Όροι

Μηχανική Προτροπής: Η πρακτική σχεδιασμού και βελτίωσης των εισόδων για να καθοδηγηθούν τα μοντέλα AI προς την παραγωγή επιθυμητών εξόδων.
Tokenization: Η διαδικασία μετατροπής του κειμένου σε μικρότερες μονάδες, που ονομάζονται tokens, τις οποίες ένα μοντέλο μπορεί να κατανοήσει και να επεξεργαστεί.
Instruction-Tuned LLMs: Μεγάλα Μοντέλα Γλώσσας (LLMs) που έχουν βελτιωθεί με συγκεκριμένες οδηγίες για να βελτιώσουν την ακρίβεια και σχετικότητα της απάντησής τους.

## Τι είναι η Μηχανική Προτροπής;

Ξεκινήσαμε αυτό το μάθημα ορίζοντας τη **Μηχανική Προτροπής** ως την διαδικασία του _σχεδιασμού και της βελτιστοποίησης_ των κειμενικών εισόδων (προτροπών) για να δώσουν συνεπείς και ποιοτικές απαντήσεις (ολοκληρώσεις) για έναν συγκεκριμένο στόχο εφαρμογής και μοντέλου. Μπορούμε να το σκεφτούμε ως μια διαδικασία 2 βημάτων:

- _σχεδιασμός_ της αρχικής προτροπής για ένα δεδομένο μοντέλο και στόχο
- _βελτίωση_ της προτροπής επαναληπτικά για να βελτιωθεί η ποιότητα της απάντησης

Αυτή είναι αναγκαστικά μια διαδικασία δοκιμής και λάθους που απαιτεί διαίσθηση και προσπάθεια από τον χρήστη για να επιτευχθούν βέλτιστα αποτελέσματα. Ποιος είναι λοιπόν ο λόγος που είναι σημαντική; Για να απαντήσουμε σε αυτό το ερώτημα, πρέπει πρώτα να κατανοήσουμε τρεις έννοιες:

- _Tokenization_ = πώς το μοντέλο "βλέπει" την προτροπή
- _Βασικά LLMs_ = πώς το θεμελιώδες μοντέλο "επεξεργάζεται" μια προτροπή
- _Instruction-Tuned LLMs_ = πώς το μοντέλο μπορεί τώρα να βλέπει "εργασίες"

### Tokenization

Ένα LLM βλέπει τις προτροπές ως _ακολουθία tokens_ όπου διαφορετικά μοντέλα (ή εκδόσεις μοντέλου) μπορούν να κάνουν tokenization της ίδιας προτροπής με διαφορετικούς τρόπους. Επειδή τα LLM εκπαιδεύονται με tokens (και όχι με ακατέργαστο κείμενο), ο τρόπος που οι προτροπές γίνονται tokens έχει άμεσο αντίκτυπο στην ποιότητα της παραγόμενης απάντησης.

Για να κατανοήσετε πώς λειτουργεί το tokenization, δοκιμάστε εργαλεία όπως το [OpenAI Tokenizer](https://platform.openai.com/tokenizer?WT.mc_id=academic-105485-koreyst) που φαίνεται παρακάτω. Αντιγράψτε την προτροπή σας - και δείτε πώς μετατρέπεται σε tokens, δίνοντας προσοχή στο πώς χειρίζονται οι χαρακτήρες κενών και τα σημεία στίξης. Σημειώστε ότι αυτό το παράδειγμα δείχνει ένα παλαιότερο LLM (GPT-3) - οπότε δοκιμάζοντας το με ένα νεότερο μοντέλο μπορεί να έχει διαφορετικό αποτέλεσμα.

![Tokenization](../../../translated_images/el/04-tokenizer-example.e71f0a0f70356c5c.webp)

### Έννοια: Θεμελιώδη Μοντέλα

Μόλις μια προτροπή μετατραπεί σε tokens, η κύρια λειτουργία του ["Βασικού LLM"](https://blog.gopenai.com/an-introduction-to-base-and-instruction-tuned-large-language-models-8de102c785a6?WT.mc_id=academic-105485-koreyst) (ή θεμελιώδους μοντέλου) είναι να προβλέπει το επόμενο token σε αυτήν την ακολουθία. Επειδή τα LLM έχουν εκπαιδευτεί σε τεράστια σύνολα δεδομένων κειμένου, έχουν καλή αντίληψη των στατιστικών σχέσεων μεταξύ των tokens και μπορούν να κάνουν αυτή την πρόβλεψη με κάποια αυτοπεποίθηση. Σημειώστε ότι δεν κατανοούν το _νόημα_ των λέξεων στην προτροπή ή το token· απλώς βλέπουν ένα μοτίβο που μπορούν να "ολοκληρώσουν" με την επόμενη πρόβλεψή τους. Μπορούν να συνεχίσουν να προβλέπουν την ακολουθία μέχρι να διακοπεί από παρέμβαση χρήστη ή κάποια προκαθορισμένη συνθήκη.

Θέλετε να δείτε πώς λειτουργεί η ολοκλήρωση με βάση την προτροπή; Εισάγετε την προτροπή παραπάνω στο [Microsoft Foundry playground](https://ai.azure.com?WT.mc_id=academic-105485-koreyst) με τις προεπιλεγμένες ρυθμίσεις. Το σύστημα είναι ρυθμισμένο να αντιμετωπίζει τις προτροπές ως αιτήματα για πληροφορίες - οπότε θα πρέπει να δείτε μια ολοκλήρωση που ικανοποιεί αυτό το πλαίσιο.

Αλλά τι γίνεται αν ο χρήστης ήθελε να δει κάτι συγκεκριμένο που πληροί κάποια κριτήρια ή στόχο εργασίας; Εδώ έρχονται οι _instruction-tuned_ LLMs.

### Έννοια: Instruction Tuned LLMs

Ένα [Instruction Tuned LLM](https://blog.gopenai.com/an-introduction-to-base-and-instruction-tuned-large-language-models-8de102c785a6?WT.mc_id=academic-105485-koreyst) ξεκινά με το θεμελιώδες μοντέλο και το βελτιώνει με παραδείγματα ή ζεύγη εισόδου/εξόδου (π.χ., μηνύματα πολλαπλών γύρων) που μπορεί να περιέχουν σαφείς οδηγίες - και η απάντηση από την AI προσπαθεί να ακολουθήσει αυτή την οδηγία.

Χρησιμοποιεί τεχνικές όπως η Ενισχυτική Μάθηση με Ανθρώπινη Ανατροφοδότηση (RLHF) που μπορούν να εκπαιδεύσουν το μοντέλο να _ακολουθεί οδηγίες_ και να _μαθαίνει από την ανατροφοδότηση_ έτσι ώστε να παράγει απαντήσεις που είναι πιο κατάλληλες για πρακτικές εφαρμογές και πιο σχετικές με τους στόχους του χρήστη.

Ας το δοκιμάσουμε - ξαναδείτε την παραπάνω προτροπή, αλλά τώρα αλλάξτε το _σύστημα μήνυμα_ ώστε να παρέχει την ακόλουθη οδηγία ως πλαίσιο:

> _Συνοψίστε το περιεχόμενο που σας παρέχεται για έναν μαθητή δευτέρου δημοτικού. Κρατήστε το αποτέλεσμα σε μια παράγραφο με 3-5 σημεία._

Δείτε πώς το αποτέλεσμα τώρα προσαρμόζεται για να αντανακλά τον επιθυμητό στόχο και μορφή; Ένας εκπαιδευτής μπορεί τώρα να χρησιμοποιήσει απευθείας αυτή την απάντηση στις διαφάνειες του για εκείνο το μάθημα.

## Γιατί χρειαζόμαστε Μηχανική Προτροπής;

Τώρα που ξέρουμε πώς τα LLM επεξεργάζονται τις προτροπές, ας μιλήσουμε για το _γιατί_ χρειαζόμαστε μηχανική προτροπής. Η απάντηση βρίσκεται στο γεγονός ότι τα τρέχοντα LLM θέτουν μια σειρά προκλήσεων που καθιστούν το _αξιόπιστο και συνεπές κλείσιμο_ πιο δύσκολο να επιτευχθεί χωρίς προσπάθεια στο στήσιμο και τη βελτιστοποίηση προτροπών. Για παράδειγμα:

1. **Οι απαντήσεις των μοντέλων είναι στοχαστικές.** Η _ίδια προτροπή_ πιθανώς θα παράγει διαφορετικές απαντήσεις με διαφορετικά μοντέλα ή εκδόσεις μοντέλου. Και μπορεί ακόμα να παράγει διαφορετικά αποτελέσματα με _το ίδιο μοντέλο_ σε διαφορετικές χρονικές στιγμές. _Οι τεχνικές μηχανικής προτροπής μπορούν να μας βοηθήσουν να ελαχιστοποιήσουμε αυτές τις διακυμάνσεις παρέχοντας καλύτερους οδηγούς_.

1. **Τα μοντέλα μπορούν να επινοούν απαντήσεις.** Τα μοντέλα είναι προεκπαιδευμένα με _μεγάλα αλλά πεπερασμένα_ σύνολα δεδομένων, που σημαίνει ότι στερούνται γνώσης για έννοιες εκτός αυτού του πεδίου εκπαίδευσης. Ως αποτέλεσμα, μπορεί να παράγουν απαντήσεις που είναι ανακριβείς, φανταστικές ή άμεσα αντίθετες σε γνωστές αλήθειες. _Οι τεχνικές μηχανικής προτροπής βοηθούν τους χρήστες να εντοπίζουν και να μετριάζουν τέτοιες επινοήσεις π.χ., ζητώντας από την AI παραπομπές ή λογική_.

1. **Οι ικανότητες των μοντέλων θα διαφέρουν.** Πιο νέα μοντέλα ή γενιές μοντέλων θα έχουν πλουσιότερες δυνατότητες αλλά φέρουν επίσης μοναδικές ιδιοτροπίες και συμβιβασμούς στο κόστος και την πολυπλοκότητα. _Η μηχανική προτροπής μπορεί να μας βοηθήσει να αναπτύξουμε βέλτιστες πρακτικές και ροές εργασίας που αφαιρούν τις διαφορές και προσαρμόζονται σε μοντέλο-ειδικές απαιτήσεις με κλιμακούμενους και ομαλούς τρόπους_.

Ας δούμε αυτό σε δράση στο OpenAI ή το Azure OpenAI Playground:

- Χρησιμοποιήστε την ίδια προτροπή με διαφορετικές αναπτύξεις LLM (π.χ., OpenAI, Azure OpenAI, Hugging Face) - παρατηρήσατε τις παραλλαγές;
- Χρησιμοποιήστε την ίδια προτροπή επανειλημμένα με _την ίδια_ ανάπτυξη LLM (π.χ., Azure OpenAI playground) - πώς διέφεραν αυτές οι παραλλαγές;

### Παράδειγμα Επινοήσεων

Μπορεί επίσης να έχετε ακούσει να αποκαλούνται _"παραληρήματα"_ σε δημοφιλή άρθρα ή επιστημονικές εργασίες. Ωστόσο, συστήνουμε έντονα τη χρήση του όρου _"επινόηση"_ ώστε να μην ανθρωπομορφίζουμε συμπτωματικά τη συμπεριφορά αποδίδοντας ανθρώπινα χαρακτηριστικά σε ένα αποτέλεσμα μηχανής. ## Κατασκευή Προτροπής

Έχουμε δει γιατί η μηχανική προτροπής είναι σημαντική - τώρα ας κατανοήσουμε πώς _κατασκευάζονται_ οι προτροπές ώστε να αξιολογήσουμε διαφορετικές τεχνικές για πιο αποτελεσματικό σχεδιασμό προτροπής.

### Απλή Προτροπή

Ας ξεκινήσουμε με την απλή προτροπή: μια κείμενη είσοδος που αποστέλλεται στο μοντέλο χωρίς άλλο πλαίσιο. Να ένα παράδειγμα - όταν στέλνουμε τις πρώτες λίγες λέξεις από τον εθνικό ύμνο των ΗΠΑ στο OpenAI [Completion API](https://platform.openai.com/docs/api-reference/completions?WT.mc_id=academic-105485-koreyst), ολοκληρώνει αμέσως την απάντηση με τις επόμενες αράδες, απεικονίζοντας τη βασική συμπεριφορά πρόβλεψης.

### Σύνθετη Προτροπή

Τώρα ας προσθέσουμε πλαίσιο και οδηγίες σε αυτήν την απλή προτροπή. Το [Chat Completion API](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/chatgpt?WT.mc_id=academic-105485-koreyst) μας επιτρέπει να κατασκευάσουμε μια σύνθετη προτροπή ως συλλογή _μηνυμάτων_ με:

- Ζεύγη εισόδου/εξόδου που αντανακλούν την είσοδο _χρήστη_ και την απάντηση _βοηθού_.
- Σύστημα μηνύματος που θέτει το πλαίσιο για τη συμπεριφορά ή προσωπικότητα του βοηθού.

Το αίτημα είναι τώρα στη μορφή που ακολουθεί, όπου η _τοκενιοποίηση_ καταγράφει αποτελεσματικά σχετικές πληροφορίες από το πλαίσιο και τη συνομιλία. Τώρα, η αλλαγή του πλαισίου συστήματος μπορεί να επηρεάσει την ποιότητα των ολοκληρώσεων όσο και οι είσοδοι που παρέχονται από τον χρήστη.

```python
response = client.responses.create(
    model="gpt-5-mini",
    input=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Who won the world series in 2020?"},
        {"role": "assistant", "content": "The Los Angeles Dodgers won the World Series in 2020."},
        {"role": "user", "content": "Where was it played?"}
    ]
)
```

- μια οδηγία (ενέργεια)

- **Παραδείγματα** - αντί να λέμε στο μοντέλο τι να κάνει με σαφή οδηγία, δίνουμε παραδείγματα για το τι να κάνει και αφήνουμε το μοντέλο να συμπεράνει το μοτίβο.

- **Πρότυπα** - είναι επαναλαμβανόμενες «συνταγές» για προτροπές με θέσεις (μεταβλητές) που μπορούν να προσαρμοστούν με δεδομένα για συγκεκριμένες χρήσεις.

### Χρήση Παραδειγμάτων

Αυτή είναι μια προσέγγιση όπου χρησιμοποιείτε το κύριο περιεχόμενο για να «τροφοδοτήσετε το μοντέλο» με παραδείγματα της επιθυμητής εξόδου για μια δεδομένη οδηγία, και να αφήσετε το μοντέλο να συμπεράνει το μοτίβο της επιθυμητής εξόδου. Βάσει του αριθμού των παρεχόμενων παραδειγμάτων, μπορούμε να έχουμε μηδενική προτροπή (zero-shot), προτροπή με ένα παράδειγμα (one-shot), προτροπή με λίγα παραδείγματα (few-shot) κλπ.

- Μια περιγραφή εργασίας
- Μερικά παραδείγματα της επιθυμητής εξόδου
- Η αρχή ενός νέου παραδείγματος (που γίνεται μια σιωπηρή περιγραφή εργασίας)

### Πρότυπα Προτροπών

Ένα πρότυπο προτροπής είναι μια _προκαθορισμένη συνταγή για μια προτροπή_ που μπορεί να αποθηκευτεί και να ξαναχρησιμοποιηθεί όπως χρειάζεται, για να οδηγήσει σε πιο συνεπείς εμπειρίες χρήστη σε μεγάλη κλίμακα. Στην απλούστερη μορφή του, είναι απλώς μια συλλογή παραδειγμάτων προτροπών όπως [αυτό από το OpenAI](https://cookbook.openai.com/examples/gpt4-1_prompting_guide?WT.mc_id=academic-105485-koreyst) που παρέχει τόσο τα αλληλεπιδραστικά στοιχεία της προτροπής (μηνύματα χρήστη και συστήματος) όσο και τη μορφή αιτήματος που γίνεται μέσω API - για να υποστηρικτεί η επανάχρηση.

Στην πιο σύνθετη μορφή του όπως [αυτό το παράδειγμα από το LangChain](https://python.langchain.com/docs/concepts/prompt_templates/?WT.mc_id=academic-105485-koreyst) περιέχει _κρατήσεις θέσης_ που μπορούν να αντικατασταθούν με δεδομένα από μια ποικιλία πηγών (είσοδος χρήστη, πλαίσιο συστήματος, εξωτερικές πηγές δεδομένων κ.ά.) για τη δυναμική δημιουργία προτροπής. Αυτό μας επιτρέπει να δημιουργήσουμε μια βιβλιοθήκη επαναχρησιμοποιήσιμων προτροπών που μπορούν να χρησιμοποιηθούν για να υποστηρίξουν συνεπείς εμπειρίες χρήστη **προγραμματιστικά** σε μεγάλη κλίμακα.

Τέλος, η πραγματική αξία των προτύπων έγκειται στην ικανότητα να δημιουργηθούν και να δημοσιευτούν _βιβλιοθήκες προτροπών_ για κάθετες εφαρμογές - όπου το πρότυπο προτροπής είναι πλέον _βελτιστοποιημένο_ ώστε να αντικατοπτρίζει το περιεχόμενο ή παραδείγματα χαρακτηριστικά της εφαρμογής που καθιστούν τις απαντήσεις πιο σχετικές και ακριβείς για το στοχευμένο κοινό χρηστών. Το αποθετήριο [Prompts For Edu](https://github.com/microsoft/prompts-for-edu?WT.mc_id=academic-105485-koreyst) είναι ένα εξαιρετικό παράδειγμα αυτής της προσέγγισης, συγκεντρώνοντας μια βιβλιοθήκη προτροπών για τον τομέα της εκπαίδευσης με έμφαση σε βασικούς στόχους όπως ο σχεδιασμός μαθημάτων, ο σχεδιασμός προγράμματος σπουδών, η διδασκαλία φοιτητών κ.ά.

## Υποστηρικτικό Περιεχόμενο

Αν σκεφτούμε τη δημιουργία προτροπής ως τη σύνθεση μιας οδηγίας (εργασία) και ενός στόχου (κύριο περιεχόμενο), τότε το _δευτερεύον περιεχόμενο_ είναι σαν πρόσθετο πλαίσιο που παρέχουμε για να **επηρεάσουμε με κάποιο τρόπο την έξοδο**. Μπορεί να είναι παράμετροι ρύθμισης, οδηγίες μορφοποίησης, ταξονομίες θεμάτων κ.ά. που βοηθούν το μοντέλο να _προσαρμόσει_ την απάντησή του ώστε να ταιριάζει στους επιθυμητούς στόχους ή προσδοκίες του χρήστη.

Για παράδειγμα: Δεδομένου ενός καταλόγου μαθημάτων με εκτεταμένα μεταδεδομένα (όνομα, περιγραφή, επίπεδο, ετικέτες μεταδεδομένων, εκπαιδευτής κ.ά.) για όλα τα διαθέσιμα μαθήματα στο πρόγραμμα σπουδών:

- μπορούμε να ορίσουμε μια οδηγία για "περίληψη του καταλόγου μαθημάτων για το Φθινόπωρο 2023"
- μπορούμε να χρησιμοποιήσουμε το κύριο περιεχόμενο για να παρέχουμε μερικά παραδείγματα της επιθυμητής εξόδου
- μπορούμε να χρησιμοποιήσουμε το δευτερεύον περιεχόμενο για να προσδιορίσουμε τις κορυφαίες 5 "ετικέτες" ενδιαφέροντος.

Τώρα, το μοντέλο μπορεί να παράσχει μια περίληψη στη μορφή που εμφανίζεται στα δείγματα - αλλά αν ένα αποτέλεσμα έχει πολλαπλές ετικέτες, μπορεί να δώσει προτεραιότητα στις 5 ετικέτες που προσδιορίστηκαν στο δευτερεύον περιεχόμενο.

## Καλές Πρακτικές στη Μηχανική Προτροπής

Τώρα που γνωρίζουμε πώς μπορούν να _συντεθούν_ οι προτροπές, μπορούμε να αρχίσουμε να σκεφτόμαστε πώς να τις _σχεδιάσουμε_ ώστε να αντανακλούν καλές πρακτικές. Μπορούμε να το εξετάσουμε σε δύο μέρη - έχοντας τη σωστή _νοοτροπία_ και εφαρμόζοντας τις σωστές _τεχνικές_.

### Νοοτροπία Μηχανικής Προτροπής

Η Μηχανική Προτροπής είναι μια διαδικασία με δοκιμές και λάθη, οπότε κρατήστε τρεις βασικούς οδηγούς παράγοντες στο μυαλό:

1. **Η Κατανόηση του Τομέα Μετράει.** Η ακρίβεια και η σχετικότητα των απαντήσεων εξαρτάται από τον _τομέα_ στον οποίο λειτουργεί η εφαρμογή ή ο χρήστης. Εφαρμόστε την διαίσθηση και την ειδίκευση σας στον τομέα για να **προσαρμόσετε περαιτέρω τις τεχνικές**. Για παράδειγμα, ορίστε _προσωπικότητες ειδικές στο τομέα_ στις προτροπές συστήματος σας ή χρησιμοποιήστε _πρότυπα ειδικά στον τομέα_ στις προτροπές χρήστη. Παρέχετε δευτερεύον περιεχόμενο που αντικατοπτρίζει πλαίσια ειδικά στον τομέα, ή χρησιμοποιήστε _ενδείξεις και παραδείγματα ειδικά στον τομέα_ για να κατευθύνετε το μοντέλο προς οικείες μορφές χρήσης.

2. **Η Κατανόηση του Μοντέλου Μετράει.** Γνωρίζουμε ότι τα μοντέλα είναι στοχαστικά από τη φύση τους. Αλλά οι υλοποιήσεις μοντέλων μπορεί επίσης να διαφέρουν όσον αφορά το σύνολο δεδομένων εκπαίδευσης που χρησιμοποιούν (προεκπαιδευμένη γνώση), τις δυνατότητες που προσφέρουν (π.χ. μέσω API ή SDK) και τον τύπο περιεχομένου για τον οποίο είναι βελτιστοποιημένα (π.χ. κώδικας έναντι εικόνων έναντι κειμένου). Κατανοήστε τα πλεονεκτήματα και τους περιορισμούς του μοντέλου που χρησιμοποιείτε, και χρησιμοποιήστε αυτή τη γνώση για να _δωρίσετε προτεραιότητες στις εργασίες_ ή να δημιουργήσετε _προσαρμοσμένα πρότυπα_ που είναι βελτιστοποιημένα για τις δυνατότητες του μοντέλου.

3. **Η Επανάληψη & Επαλήθευση Μετράει.** Τα μοντέλα εξελίσσονται ραγδαία, όπως και οι τεχνικές μηχανικής προτροπής. Ως ειδικός στον τομέα, μπορεί να έχετε άλλο πλαίσιο ή κριτήρια _για_ τη συγκεκριμένη εφαρμογή σας, που ίσως να μην ισχύουν για την ευρύτερη κοινότητα. Χρησιμοποιήστε εργαλεία και τεχνικές μηχανικής προτροπής για να «ξεκινήσετε γρήγορα» τη σύνθεση προτροπών, στη συνέχεια επαναλάβετε και επαληθεύστε τα αποτελέσματα χρησιμοποιώντας την δική σας διαίσθηση και ειδίκευση στον τομέα. Καταγράψτε τις γνώσεις σας και δημιουργήστε μια **βάση γνώσεων** (π.χ. βιβλιοθήκες προτροπών) που μπορούν να χρησιμοποιηθούν ως νέα βάση από άλλους, για πιο γρήγορες επαναλήψεις στο μέλλον.
