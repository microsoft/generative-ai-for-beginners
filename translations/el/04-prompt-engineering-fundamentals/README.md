# Θεμελιώδεις Αρχές Μηχανικής Προτροπών

[![Θεμελιώδεις Αρχές Μηχανικής Προτροπών](../../../translated_images/el/04-lesson-banner.a2c90deba7fedacd.webp)](https://youtu.be/GElCu2kUlRs?si=qrXsBvXnCW12epb8)

## Εισαγωγή
Αυτή η ενότητα καλύπτει βασικές έννοιες και τεχνικές για τη δημιουργία αποτελεσματικών προτροπών σε μοντέλα γενετικής τεχνητής νοημοσύνης. Ο τρόπος με τον οποίο γράφετε την προτροπή σας προς ένα LLM έχει επίσης σημασία. Μια προσεκτικά σχεδιασμένη προτροπή μπορεί να επιτύχει καλύτερη ποιότητα απάντησης. Αλλά τι ακριβώς σημαίνουν οι όροι _προτροπή_ και _μηχανική προτροπών_; Και πώς βελτιώνω την είσοδο της προτροπής που στέλνω στο LLM; Αυτές είναι οι ερωτήσεις που θα προσπαθήσουμε να απαντήσουμε σε αυτό το κεφάλαιο και το επόμενο.

_Η γενετική ΤΝ_ είναι ικανή να δημιουργεί νέο περιεχόμενο (π.χ. κείμενο, εικόνες, ήχο, κώδικα κ.λπ.) ως απάντηση σε αιτήματα χρηστών. Το πετυχαίνει αυτό χρησιμοποιώντας _Μεγάλα Γλωσσικά Μοντέλα_ όπως η σειρά GPT της OpenAI ("Generative Pre-trained Transformer") που έχουν εκπαιδευτεί για την χρήση φυσικής γλώσσας και κώδικα.

Οι χρήστες μπορούν τώρα να αλληλεπιδρούν με αυτά τα μοντέλα χρησιμοποιώντας οικείες παραδείγματα όπως chat, χωρίς να χρειάζεται τεχνική γνώση ή εκπαίδευση. Τα μοντέλα είναι _βασισμένα σε προτροπές_ – οι χρήστες στέλνουν ένα κείμενο εισόδου (προτροπή) και λαμβάνουν την απάντηση της ΤΝ (ολοκλήρωση). Μπορούν στη συνέχεια να "συνομιλήσουν με την ΤΝ" επαναληπτικά, σε διαδοχικές συνομιλίες, βελτιώνοντας την προτροπή τους μέχρι η απάντηση να ανταποκρίνεται στις προσδοκίες τους.

Οι "προτροπές" γίνονται πλέον η βασική _διεπαφή προγραμματισμού_ για εφαρμογές γενετικής ΤΝ, λέγοντας στα μοντέλα τι να κάνουν και επηρεάζοντας την ποιότητα των επιστρεφόμενων απαντήσεων. Η "μηχανική προτροπών" είναι ένας ταχέως αναπτυσσόμενος τομέας μελέτης που εστιάζει στο _σχεδιασμό και βελτιστοποίηση_ προτροπών για να παρέχει συνεπείς και ποιοτικές απαντήσεις σε μεγάλη κλίμακα.

## Στόχοι Μάθησης

Σε αυτό το μάθημα, θα μάθουμε τι είναι η μηχανική προτροπών, γιατί έχει σημασία και πώς μπορούμε να δημιουργήσουμε πιο αποτελεσματικές προτροπές για ένα δεδομένο μοντέλο και στόχο εφαρμογής. Θα κατανοήσουμε βασικές έννοιες και βέλτιστες πρακτικές για τη μηχανική προτροπών – και θα μάθουμε για ένα διαδραστικό περιβάλλον Jupyter Notebooks "sandbox" όπου μπορούμε να δούμε αυτές τις έννοιες να εφαρμόζονται σε πραγματικά παραδείγματα.

Μέχρι το τέλος αυτού του μαθήματος θα είμαστε σε θέση να:

1. Εξηγήσουμε τι είναι η μηχανική προτροπών και γιατί έχει σημασία.
2. Περιγράψουμε τα συστατικά μιας προτροπής και πώς χρησιμοποιούνται.
3. Μάθουμε βέλτιστες πρακτικές και τεχνικές για μηχανική προτροπών.
4. Εφαρμόσουμε τις τεχνικές που μάθαμε σε πραγματικά παραδείγματα, χρησιμοποιώντας ένα endpoint OpenAI.

## Βασικοί Όροι

Μηχανική Προτροπών: Η πρακτική του σχεδιασμού και της βελτίωσης των εισόδων για να καθοδηγούν τα μοντέλα ΤΝ προς την παραγωγή επιθυμητών εξόδων.
Τοκενιοποίηση: Η διαδικασία μετατροπής κειμένου σε μικρότερες μονάδες, που ονομάζονται τοκεν, που ένα μοντέλο μπορεί να κατανοήσει και να επεξεργαστεί.
Εξοικειωμένα Με Οδηγίες LLMs: Μεγάλα Γλωσσικά Μοντέλα που έχουν βελτιωθεί με συγκεκριμένες οδηγίες για να βελτιώσουν την ακρίβεια και τη σχετικότητα των απαντήσεών τους.

## Τι είναι η Μηχανική Προτροπών;

Ξεκινήσαμε αυτό το μάθημα ορίζοντας τη **Μηχανική Προτροπών** ως τη διαδικασία _σχεδιασμού και βελτιστοποίησης_ κειμενικών εισόδων (προτροπών) για να παρέχουν συνεπείς και ποιοτικές απαντήσεις (ολοκληρώσεις) για έναν δεδομένο στόχο εφαρμογής και μοντέλο. Μπορούμε να το σκεφτούμε ως μια διαδικασία 2 βημάτων:

- _σχεδιασμός_ της αρχικής προτροπής για ένα δεδομένο μοντέλο και στόχο
- _βελτίωση_ της προτροπής επαναληπτικά για να βελτιωθεί η ποιότητα της απόκρισης

Αυτή είναι αναγκαστικά μια διαδικασία δοκιμής και λάθους που απαιτεί διαίσθηση και προσπάθεια από τον χρήστη για να επιτύχει βέλτιστα αποτελέσματα. Γιατί λοιπόν είναι σημαντική; Για να απαντήσουμε σε αυτήν την ερώτηση, πρέπει πρώτα να κατανοήσουμε τρεις έννοιες:

- _Τοκενιοποίηση_ = πώς το μοντέλο "βλέπει" την προτροπή
- _Βασικά LLMs_ = πώς το θεμελιώδες μοντέλο "επεξεργάζεται" μια προτροπή
- _Εξοικειωμένα Με Οδηγίες LLMs_ = πώς το μοντέλο πλέον μπορεί να "δει εργασίες"

### Τοκενιοποίηση

Ένα LLM βλέπει τις προτροπές ως _σειρά τοκεν_ όπου διαφορετικά μοντέλα (ή εκδόσεις ενός μοντέλου) μπορούν να τοκενιοποιήσουν την ίδια προτροπή με διαφορετικούς τρόπους. Εφόσον τα LLMs εκπαιδεύονται με βάση τα τοκεν (και όχι το ακατέργαστο κείμενο), ο τρόπος που τοκενιοποιούνται οι προτροπές έχει άμεση επίδραση στην ποιότητα της παραγόμενης απάντησης.

Για να πάρετε μια αίσθηση του πώς λειτουργεί η τοκενιοποίηση, δοκιμάστε εργαλεία όπως τον [OpenAI Tokenizer](https://platform.openai.com/tokenizer?WT.mc_id=academic-105485-koreyst) που φαίνεται παρακάτω. Αντιγράψτε την προτροπή σας – και δείτε πώς αυτή μετατρέπεται σε τοκεν, δίνοντας προσοχή στον τρόπο που χειρίζονται τα κενά και τα σημεία στίξης. Σημειώστε ότι αυτό το παράδειγμα δείχνει ένα παλαιότερο LLM (GPT-3) – οπότε δοκιμάζοντας με νεότερο μοντέλο μπορεί να έχει διαφορετικό αποτέλεσμα.

![Τοκενιοποίηση](../../../translated_images/el/04-tokenizer-example.e71f0a0f70356c5c.webp)

### Έννοια: Θεμελιώδη Μοντέλα

Μόλις η προτροπή τοκενιοποιηθεί, η βασική λειτουργία του ["Βασικού LLM"](https://blog.gopenai.com/an-introduction-to-base-and-instruction-tuned-large-language-models-8de102c785a6?WT.mc_id=academic-105485-koreyst) (ή θεμελιώδους μοντέλου) είναι να προβλέψει το επόμενο τοκεν στη σειρά. Εφόσον τα LLMs εκπαιδεύονται σε τεράστια σύνολα δεδομένων κειμένου, έχουν καλή αίσθηση των στατιστικών σχέσεων μεταξύ τοκεν και μπορούν να κάνουν αυτή την πρόβλεψη με κάποια σιγουριά. Σημειώστε ότι δεν κατανοούν το _νόημα_ των λέξεων στην προτροπή ή το τοκεν· απλά βλέπουν ένα μοτίβο που μπορούν να "συμπληρώσουν" με την επόμενη πρόβλεψή τους. Μπορούν να συνεχίσουν να προβλέπουν τη σειρά μέχρι να διακοπεί από παρέμβαση χρήστη ή κάποια προϋπάρχουσα συνθήκη.

Θέλετε να δείτε πώς λειτουργεί η ολοκλήρωση βασισμένη σε προτροπές; Εισάγετε την παραπάνω προτροπή στο [Microsoft Foundry playground](https://ai.azure.com?WT.mc_id=academic-105485-koreyst) με τις προεπιλεγμένες ρυθμίσεις. Το σύστημα έχει διαμορφωθεί να αντιμετωπίζει τις προτροπές ως αιτήματα για πληροφορίες – οπότε θα πρέπει να δείτε μια ολοκλήρωση που ικανοποιεί αυτό το πλαίσιο.

Αλλά τι γίνεται αν ο χρήστης ήθελε να δει κάτι συγκεκριμένο που να πληροί κάποια κριτήρια ή στόχο εργασίας; Εδώ μπαίνουν στην εικόνα τα _εξοικειωμένα με οδηγίες_ LLMs.

### Έννοια: Εξοικειωμένα Με Οδηγίες LLMs

Ένα [Εξοικειωμένο με Οδηγίες LLM](https://blog.gopenai.com/an-introduction-to-base-and-instruction-tuned-large-language-models-8de102c785a6?WT.mc_id=academic-105485-koreyst) ξεκινά με το θεμελιώδες μοντέλο και το βελτιώνει ειδικά με παραδείγματα ή ζεύγη εισόδου/εξόδου (π.χ. πολυγνώσεις "μηνύματα") που μπορεί να περιέχουν σαφείς οδηγίες – και η απάντηση της ΤΝ προσπαθεί να ακολουθήσει αυτήν την οδηγία.

Αυτό χρησιμοποιεί τεχνικές όπως Ενισχυτική Μάθηση με Ανθρώπινη Ανατροφοδότηση (RLHF) που μπορούν να εκπαιδεύσουν το μοντέλο να _ακολουθεί οδηγίες_ και να _μαθαίνει από ανατροφοδότηση_ έτσι ώστε να παράγει απαντήσεις που είναι καλύτερα προσαρμοσμένες σε πρακτικές εφαρμογές και πιο σχετικές με τους στόχους των χρηστών.

Ας το δοκιμάσουμε – ξαναδείτε την παραπάνω προτροπή, αλλά τώρα αλλάξτε το _μήνυμα συστήματος_ για να παρέχετε την ακόλουθη οδηγία ως πλαίσιο:

> _Συνοψίστε το περιεχόμενο που λαμβάνετε για έναν μαθητή δεύτερης τάξης. Κρατήστε το αποτέλεσμα σε μια παράγραφο με 3-5 κουκκίδες._

Δείτε πώς το αποτέλεσμα προσαρμόστηκε για να αντανακλά τον επιθυμητό στόχο και μορφή; Ένας εκπαιδευτικός μπορεί τώρα να χρησιμοποιήσει απευθείας αυτήν την απάντηση στις διαφάνειές του για εκείνο το μάθημα.

## Γιατί χρειαζόμαστε Μηχανική Προτροπών;

Τώρα που ξέρουμε πώς τα μοντέλα LLM επεξεργάζονται τις προτροπές, ας μιλήσουμε για το _γιατί_ χρειαζόμαστε μηχανική προτροπών. Η απάντηση βρίσκεται στο γεγονός ότι τα τρέχοντα LLM παρουσιάζουν μια σειρά προκλήσεων που καθιστούν πιο δύσκολη την επίτευξη _αξιόπιστων και συνεπών ολοκληρώσεων_ χωρίς προσπάθεια στην κατασκευή και βελτιστοποίηση της προτροπής. Για παράδειγμα:

1. **Οι απαντήσεις του μοντέλου είναι στοχαστικές.** Η _ίδια προτροπή_ πιθανόν να παράγει διαφορετικές απαντήσεις με διαφορετικά μοντέλα ή εκδόσεις μοντέλου. Και μπορεί ακόμη και να παράγει διαφορετικά αποτελέσματα με το _ίδιο μοντέλο_ σε διαφορετικές χρονικές στιγμές. _Οι τεχνικές μηχανικής προτροπών μπορούν να μας βοηθήσουν να ελαχιστοποιήσουμε αυτές τις μεταβολές παρέχοντας καλύτερους κανόνες_.

1. **Τα μοντέλα μπορούν να επινοήσουν απαντήσεις.** Τα μοντέλα είναι προεκπαιδευμένα με _μεγάλα αλλά πεπερασμένα_ σύνολα δεδομένων, που σημαίνει ότι δεν έχουν γνώση για έννοιες εκτός αυτού του εύρους εκπαίδευσης. Ως αποτέλεσμα, μπορούν να παράγουν ολοκληρώσεις που είναι ανακριβείς, φανταστικές ή άμεσα αντιφατικές με γνωστά γεγονότα. _Οι τεχνικές μηχανικής προτροπών βοηθούν τους χρήστες να εντοπίζουν και να μετριάζουν τέτοιες επινοήσεις, π.χ. ζητώντας αναφορές ή λογική από την ΤΝ_.

1. **Οι δυνατότητες των μοντέλων διαφέρουν.** Νεότερα μοντέλα ή γενιές μοντέλων θα έχουν πλουσιότερες δυνατότητες αλλά φέρνουν επίσης ιδιόμορφα χαρακτηριστικά και συμβιβασμούς στο κόστος και την πολυπλοκότητα. _Η μηχανική προτροπών μπορεί να μας βοηθήσει να αναπτύξουμε βέλτιστες πρακτικές και ροές εργασίας που αφαιρούν τις διαφορές και προσαρμόζονται σε απαιτήσεις συγκεκριμένων μοντέλων με κλιμακωτούς και αβίαστους τρόπους_.

Ας το δούμε αυτό σε δράση στο OpenAI ή Azure OpenAI Playground:

- Χρησιμοποιήστε την ίδια προτροπή σε διαφορετικές αναπτύξεις LLM (π.χ. OpenAI, Azure OpenAI, Hugging Face) – είδατε τις διαφορές;
- Χρησιμοποιήστε την ίδια προτροπή επανειλημμένα με την _ίδια_ ανάπτυξη LLM (π.χ. Azure OpenAI playground) – πώς διέφεραν αυτές οι μεταβολές;

## Κατασκευή Prompt

Έχουμε δει γιατί ο σχεδιασμός prompt είναι σημαντικός - τώρα ας κατανοήσουμε πώς τα prompts _κατασκευάζονται_ ώστε να αξιολογήσουμε διαφορετικές τεχνικές για πιο αποτελεσματικό σχεδιασμό prompt.

### Βασικό Prompt

Ας ξεκινήσουμε με το βασικό prompt: μια κειμενική είσοδο που στέλνεται στο μοντέλο χωρίς άλλο πλαίσιο. Να ένα παράδειγμα - όταν στέλνουμε τις πρώτες λέξεις του εθνικού ύμνου των ΗΠΑ στο OpenAI [Completion API](https://platform.openai.com/docs/api-reference/completions?WT.mc_id=academic-105485-koreyst) ολοκληρώνει άμεσα την απάντηση με τις επόμενες γραμμές, παρουσιάζοντας τη βασική συμπεριφορά πρόβλεψης.

| Prompt (Είσοδος)  | Συμπλήρωση (Έξοδος)                                                                                                                        |
| :----------------- | :----------------------------------------------------------------------------------------------------------------------------------------- |
| Oh say can you see | Ακούγεται σαν να ξεκινάτε τους στίχους του "The Star-Spangled Banner," του εθνικού ύμνου των Ηνωμένων Πολιτειών. Οι πλήρεις στίχοι είναι ... |

### Σύνθετο Prompt

Τώρα ας προσθέσουμε πλαίσιο και οδηγίες σε αυτό το βασικό prompt. Το [Chat Completion API](https://learn.microsoft.com/azure/ai-services/openai/how-to/chatgpt?WT.mc_id=academic-105485-koreyst) μας επιτρέπει να κατασκευάσουμε ένα σύνθετο prompt ως συλλογή από _μηνύματα_ με:

- Ζεύγη εισόδου/εξόδου που αντανακλούν την είσοδο _χρήστη_ και την απάντηση _βοηθού_.
- Μήνυμα συστήματος που ορίζει το πλαίσιο για τη συμπεριφορά ή προσωπικότητα του βοηθού.

Το αίτημα είναι τώρα στη μορφή που ακολουθεί, όπου η _tokenization_ καταγράφει αποτελεσματικά σχετικές πληροφορίες από το πλαίσιο και τη συνομιλία. Τώρα, η αλλαγή του πλαισίου συστήματος μπορεί να είναι τόσο καθοριστική για την ποιότητα των συμπληρώσεων όσο και οι παρεχόμενες εισόδους χρήστη.

```python
response = client.responses.create(
    model="gpt-4o-mini",
    input=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Who won the world series in 2020?"},
        {"role": "assistant", "content": "The Los Angeles Dodgers won the World Series in 2020."},
        {"role": "user", "content": "Where was it played?"}
    ]
)
```

Με τα prompts _οδηγίες_, μπορούμε να χρησιμοποιήσουμε αυτό το κείμενο για να καθορίσουμε μια εργασία με περισσότερες λεπτομέρειες, παρέχοντας καλύτερη καθοδήγηση στην AI. Να ένα παράδειγμα:

| Prompt (Είσοδος)                                                                                                                                                                                                                         | Συμπλήρωση (Έξοδος)                                                                                                        | Τύπος Οδηγίας      |
| :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------- | :------------------ |
| Write a description of the Civil War                                                                                                                                                                                                   | _επέστρεψε μια απλή παράγραφο_                                                                                              | Απλό               |
| Write a description of the Civil War. Provide key dates and events and describe their significance                                                                                                                                     | _επέστρεψε μια παράγραφο ακολουθούμενη από λίστα με βασικές ημερομηνίες και περιγραφές γεγονότων_                           | Σύνθετο            |
| Write a description of the Civil War in 1 paragraph. Provide 3 bullet points with key dates and their significance. Provide 3 more bullet points with key historical figures and their contributions. Return the output as a JSON file | _επιστρέφει πιο εκτενή στοιχεία σε κουτί κειμένου, μορφοποιημένα σε JSON που μπορείτε να αντιγράψετε και να επαληθεύσετε αν χρειαστεί_ | Σύνθετο. Μορφοποιημένο. |

- μια οδηγία (ενέργεια)
- σχετικό περιεχόμενο (που επηρεάζει την ενέργεια)

### Χρήση Παραδειγμάτων

Αυτή είναι μια προσέγγιση όπου χρησιμοποιείτε το πρωτεύον περιεχόμενο για να «ταΐσετε το μοντέλο» με κάποια παραδείγματα της επιθυμητής εξόδου για μια δεδομένη οδηγία, και το αφήνετε να συμπεράνει το μοτίβο για την επιθυμητή έξοδο. Με βάση τον αριθμό των παραδειγμάτων που παρέχονται, μπορούμε να έχουμε zero-shot prompting, one-shot prompting, few-shot prompting κ.ά.

Το prompt τώρα αποτελείται από τρία στοιχεία:

- Μια περιγραφή εργασίας
- Μερικά παραδείγματα της επιθυμητής εξόδου
- Την αρχή ενός νέου παραδείγματος (που γίνεται ρητή περιγραφή εργασίας)

| Τύπος Μάθησης | Prompt (Είσοδος)                                                                                                                                        | Συμπλήρωση (Έξοδος)         |
| :------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------- | :-------------------------- |

| Few-shot      | Ο παίκτης έτρεξε τις βάσεις => Μπέιζμπολ <br/> Ο παίκτης έκανε άσο => Τένις <br/> Ο παίκτης έκανε εξάρι => Κρίκετ <br/> Ο παίκτης έκανε κάρφωμα =>         | Μπάσκετ                     |
|               |                                                                                                                                                       |                             |

Παρατηρήστε πώς έπρεπε να δώσουμε ρητή οδηγία («Μετάφραση στα Ισπανικά») στο zero-shot prompting, αλλά αυτή συμπεραίνεται στο παράδειγμα one-shot prompting. Το παράδειγμα few-shot δείχνει πώς η προσθήκη περισσότερων παραδειγμάτων επιτρέπει στα μοντέλα να κάνουν πιο ακριβείς συμπεράσεις χωρίς επιπλέον οδηγίες.

### Πρότυπα Προτροπών

Ένα πρότυπο προτροπής είναι μια _προκαθορισμένη συνταγή για μια προτροπή_ που μπορεί να αποθηκευτεί και να επαναχρησιμοποιηθεί όπως απαιτείται, για να οδηγήσει σε πιο συνεπείς εμπειρίες χρήστη σε μεγάλη κλίμακα. Στην απλούστερη μορφή του, είναι απλώς μια συλλογή παραδειγμάτων προτροπών όπως [αυτό από την OpenAI](https://cookbook.openai.com/examples/gpt4-1_prompting_guide?WT.mc_id=academic-105485-koreyst) που παρέχει τόσο τα διαδραστικά στοιχεία της προτροπής (μηνύματα χρήστη και συστήματος) όσο και τη μορφή αιτήματος που καθοδηγείται από το API - για να υποστηρίζει την επαναχρησιμοποίηση.

Στην πιο σύνθετη μορφή του, όπως [αυτό το παράδειγμα από τη LangChain](https://python.langchain.com/docs/concepts/prompt_templates/?WT.mc_id=academic-105485-koreyst), περιλαμβάνει _δείκτες θέσης_ που μπορούν να αντικατασταθούν με δεδομένα από διάφορες πηγές (εισροές χρήστη, συμφραζόμενα συστήματος, εξωτερικές πηγές δεδομένων κλπ.) για να δημιουργήσει μια δυναμική προτροπή. Αυτό μας επιτρέπει να δημιουργήσουμε μια βιβλιοθήκη επαναχρησιμοποιούμενων προτροπών που μπορούν να χρησιμοποιηθούν για να οδηγήσουν σε συνεπείς εμπειρίες χρήστη **προγραμματιστικά** σε μεγάλη κλίμακα.

Τέλος, η πραγματική αξία των προτύπων έγκειται στην ικανότητα να δημιουργούνται και να δημοσιεύονται _βιβλιοθήκες προτροπών_ για κάθετες εφαρμογές - όπου το πρότυπο προτροπής είναι πλέον _βελτιστοποιημένο_ ώστε να αντανακλά συγκεκριμένα συμφραζόμενα ή παραδείγματα της εφαρμογής που κάνουν τις απαντήσεις πιο σχετικές και ακριβείς για το στοχευμένο κοινό χρηστών. Το αποθετήριο [Prompts For Edu](https://github.com/microsoft/prompts-for-edu?WT.mc_id=academic-105485-koreyst) είναι ένα εξαιρετικό παράδειγμα αυτής της προσέγγισης, επιμελώντας μια βιβλιοθήκη προτροπών για τον τομέα της εκπαίδευσης με έμφαση σε βασικούς στόχους όπως ο σχεδιασμός μαθημάτων, το σχεδιασμό προγράμματος σπουδών, τη φροντιστηριακή υποστήριξη κλπ.

## Υποστηρικτικό Περιεχόμενο

Αν σκεφτούμε τη σύνθεση μιας προτροπής ως την ύπαρξη μιας οδηγίας (εργασίας) και ενός στόχου (κύριου περιεχομένου), τότε το _δευτερεύον περιεχόμενο_ είναι σαν το πρόσθετο συμφραζόμενο που παρέχουμε για να **επηρεάσουμε με κάποιον τρόπο το αποτέλεσμα**. Μπορεί να είναι παράμετροι ρύθμισης, οδηγίες μορφοποίησης, ταξινομίες θεμάτων κλπ. που βοηθούν το μοντέλο να _προσαρμόσει_ την απάντησή του ώστε να ταιριάζει με τους επιθυμητούς στόχους ή προσδοκίες του χρήστη.

Για παράδειγμα: Δεδομένου ενός καταλόγου μαθημάτων με εκτεταμένα μεταδεδομένα (όνομα, περιγραφή, επίπεδο, ετικέτες μεταδεδομένων, διδάσκων κλπ.) για όλα τα διαθέσιμα μαθήματα στο πρόγραμμα σπουδών:

- μπορούμε να ορίσουμε μια οδηγία για "περίληψη του καταλόγου μαθημάτων για το Φθινόπωρο 2023"
- μπορούμε να χρησιμοποιήσουμε το κύριο περιεχόμενο για να παρέχουμε μερικά παραδείγματα της επιθυμητής εξόδου
- μπορούμε να χρησιμοποιήσουμε το δευτερεύον περιεχόμενο για να εντοπίσουμε τις κορυφαίες 5 "ετικέτες" ενδιαφέροντος.

Τώρα, το μοντέλο μπορεί να παρέχει μια περίληψη στη μορφή που δείχνουν τα λίγα παραδείγματα - αλλά αν ένα αποτέλεσμα έχει πολλές ετικέτες, μπορεί να δώσει προτεραιότητα στις 5 ετικέτες που ορίζονται στο δευτερεύον περιεχόμενο.

## Καλές Πρακτικές στη Δημιουργία Προτροπών

Τώρα που γνωρίζουμε πώς μπορούν να _συντεθούν_ οι προτροπές, μπορούμε να αρχίσουμε να σκεφτόμαστε πώς να τις _σχεδιάσουμε_ ώστε να αντανακλούν τις βέλτιστες πρακτικές. Μπορούμε να το σκεφτούμε σε δύο μέρη - να έχουμε τη σωστή _νοοτροπία_ και να εφαρμόζουμε τις σωστές _τεχνικές_.

### Νοοτροπία Μηχανικής Προτροπών

Η Μηχανική Προτροπών είναι μια διαδικασία δοκιμής και λάθους, οπότε κρατήστε τρεις γενικούς καθοδηγητικούς παράγοντες στο μυαλό:

1. **Η Κατανόηση του Τομέα Μετράει.** Η ακρίβεια και η σχετικότητα της απάντησης εξαρτάται από τον _τομέα_ στον οποίο λειτουργεί η εφαρμογή ή ο χρήστης. Εφαρμόστε την διαίσθηση και την εμπειρογνωμοσύνη του τομέα για να **προσαρμόσετε περαιτέρω τις τεχνικές**. Για παράδειγμα, ορίστε _προσωπικότητες συγκεκριμένου τομέα_ στις προτροπές συστήματος, ή χρησιμοποιήστε _πρότυπα συγκεκριμένου τομέα_ στις προτροπές χρηστών. Παρέχετε δευτερεύον περιεχόμενο που αντανακλά συμφραζόμενα συγκεκριμένου τομέα ή χρησιμοποιήστε _ενδεικτικά σημάδια και παραδείγματα συγκεκριμένου τομέα_ για να κατευθύνετε το μοντέλο προς οικείες χρήσεις.

2. **Η Κατανόηση του Μοντέλου Μετράει.** Ξέρουμε ότι τα μοντέλα είναι στοχαστικά εκ φύσεως. Αλλά οι εφαρμογές μοντέλων μπορεί επίσης να διαφέρουν όσον αφορά το σύνολο δεδομένων εκπαίδευσης που χρησιμοποιούν (προεκπαιδευμένη γνώση), τις δυνατότητες που παρέχουν (π.χ., μέσω API ή SDK) και τον τύπο περιεχομένου για τον οποίο είναι βελτιστοποιημένα (π.χ., κώδικας έναντι εικόνων έναντι κειμένου). Κατανοήστε τα πλεονεκτήματα και τους περιορισμούς του μοντέλου που χρησιμοποιείτε και χρησιμοποιήστε αυτή τη γνώση για να _προτεραιοποιήσετε εργασίες_ ή να δημιουργήσετε _εξατομικευμένα πρότυπα_ που είναι βελτιστοποιημένα για τις δυνατότητες του μοντέλου.

3. **Η Επανάληψη & Επικύρωση Μετράει.** Τα μοντέλα εξελίσσονται γρήγορα, και το ίδιο και οι τεχνικές για τη μηχανική προτροπών. Ως ειδικός τομέα, μπορεί να έχετε άλλο συμφραζόμενο ή κριτήρια _για_ τη συγκεκριμένη σας εφαρμογή, που μπορεί να μην ισχύουν για την ευρύτερη κοινότητα. Χρησιμοποιήστε εργαλεία και τεχνικές μηχανικής προτροπών για να "ξεκινήσετε" τη σύνθεση προτροπών, μετά επαναλάβετε και επικυρώστε τα αποτελέσματα χρησιμοποιώντας τη δική σας διαίσθηση και εμπειρογνωμοσύνη. Καταγράψτε τις γνώσεις σας και δημιουργήστε μια **βάση γνώσεων** (π.χ., βιβλιοθήκες προτροπών) που μπορεί να χρησιμοποιηθεί ως νέα βάση από άλλους, για ταχύτερες επαναλήψεις στο μέλλον.
