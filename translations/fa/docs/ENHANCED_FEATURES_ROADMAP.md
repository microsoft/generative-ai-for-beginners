## 4. مدرن‌سازی API

### 4.1 الگوهای API منسوخ شده (مهاجرت تکمیل شد)

همه نمونه‌های پایتون و TypeScript **چت** از API تکمیل چت به **API پاسخ‌ها** مهاجرت داده شده‌اند (`client.responses.create(...)` → `response.output_text`).

| الگوی قدیمی | الگوی جدید | وضعیت |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (چت) | `OpenAI(base_url="<endpoint>/openai/v1/")` (API پاسخ‌ها) | تکمیل شد |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | تکمیل شد |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (TypeScript) | پکیج `openai` `client.responses.create()` → `response.output_text` | تکمیل شد |
| `df.append()` (pandas) | `pd.concat()` | تکمیل شد |

> **توجه:** نمونه‌های مدل‌های Microsoft Foundry که از SDK `azure-ai-inference` / `@azure-rest/ai-inference` (`client.complete()`) استفاده می‌کنند همچنان روی API استنتاج مدل قرار دارند که از API پاسخ‌ها پشتیبانی نمی‌کند. `AzureOpenAI()` به طور عمدی جایی که هنوز معتبر است حفظ شده است (ایمبدینگ و تولید تصویر).

## 5. بهبودهای زیرساختی

### 5.1 بهبودهای CI/CD

در [`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml) پیاده‌سازی شده است: لینتینگ/فرمت‌بندی پایتون (Ruff + Black) به صورت **اجباری** بر روی ماژول ابزارهای مشترک `shared/` اعمال می‌شود و به صورت **مشورتی** در باقی برنامه درسی اجرا می‌شود، علاوه بر یک اجرای مشورتی ESLint برای JavaScript/TypeScript. معیار نمونه به صورت زیر است:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### 5.2 اسکن امنیتی

در [`.github/workflows/security.yml`](../../../.github/workflows/security.yml) پیاده‌سازی شده است: تحلیل CodeQL برای پایتون و JavaScript/TypeScript (در ارسال، درخواست pull و برنامه هفتگی) به علاوه بازبینی وابستگی‌ها در درخواست‌های pull. معیار نمونه به صورت زیر است:

```yaml
# .github/workflows/security.yml
name: Security Scan

on: [push, pull_request]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: javascript, python
      - uses: github/codeql-action/analyze@v3

  dependency-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
```

```json

      "version": "3.11"

      "version": "20"
    }
  },
  "customizations": {
    "vscode": {
      "extensions": [
        "ms-python.python",
        "ms-python.vscode-pylance",
        "ms-toolsai.jupyter",
        "dbaeumer.vscode-eslint",
        "esbenp.prettier-vscode",
        "github.copilot"
      ],
      "settings": {
        "python.formatting.provider": "black",
        "editor.formatOnSave": true
      }
    }
  },
  "postCreateCommand": "pip install -e .[dev] && npm install"
}
```
