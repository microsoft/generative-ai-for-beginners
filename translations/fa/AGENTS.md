# AGENTS.md

## مرور پروژه

این مخزن شامل یک برنامه درسی جامع با ۲۱ درس است که اصول پایه هوش مصنوعی مولد و توسعه کاربردهای آن را آموزش می‌دهد. این دوره برای مبتدیان طراحی شده و همه چیز را از مفاهیم پایه تا ساخت برنامه‌های آماده تولید پوشش می‌دهد.

**فناوری‌های کلیدی:**
- پایتون ۳.۹+ با کتابخانه‌ها: `openai`، `python-dotenv`، `tiktoken`، `azure-ai-inference`، `pandas`، `numpy`، `matplotlib`
- تایپ‌اسکریپت/جاوااسکریپت با Node.js و کتابخانه‌ها: `openai` (Azure OpenAI از طریق نقطه پایانی v1 و API پاسخ‌ها)، `@azure-rest/ai-inference` (مدل‌های Microsoft Foundry)
- سرویس Azure OpenAI، API OpenAI، و مدل‌های Microsoft Foundry (مدل‌های GitHub تا پایان جولای ۲۰۲۶ بازنشسته می‌شوند)
- نوت‌بوک‌های Jupyter برای یادگیری تعاملی
- کانتینرهای توسعه برای محیط توسعه‌ی یکنواخت

**ساختار مخزن:**
- ۲۱ دایرکتوری شماره‌گذاری شده درس‌ها (۰۰-۲۱) شامل فایل‌های README، مثال‌های کد، و تمرین‌ها
- پیاده‌سازی‌های متعدد: نمونه‌های پایتون، تایپ‌اسکریپت، و گاهی نمونه‌های .NET
- دایرکتوری ترجمه‌ها با نسخه‌های بیش از ۴۰ زبان
- تنظیمات مرکزی از طریق فایل `.env` (از `.env.copy` به عنوان قالب استفاده کنید)

## دستورات راه‌اندازی

### راه‌اندازی اولیه مخزن

```bash
# مخزن را کلون کنید
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# قالب محیط را کپی کنید
cp .env.copy .env
# فایل .env را با کلیدها و نقاط پایانی API خود ویرایش کنید
```

### راه‌اندازی محیط پایتون

```bash
# ایجاد محیط مجازی
python3 -m venv venv

# فعال‌سازی محیط مجازی
# در مک‌اواس/لینوکس:
source venv/bin/activate
# در ویندوز:
venv\Scripts\activate

# نصب وابستگی‌ها
pip install -r requirements.txt
```

### راه‌اندازی Node.js/TypeScript

```bash
# نصب وابستگی‌های سطح ریشه (برای ابزار مستندسازی)
npm install

# برای مثال‌های TypeScript هر درس به صورت جداگانه، به درس خاص مراجعه کنید:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### راه‌اندازی کانتینر توسعه (توصیه شده)

مخزن شامل پیکربندی `.devcontainer` برای GitHub Codespaces یا VS Code Dev Containers است:

۱. مخزن را در GitHub Codespaces یا VS Code با افزونه Dev Containers باز کنید
۲. کانتینر توسعه به صورت خودکار:
   - وابستگی‌های پایتون را از `requirements.txt` نصب می‌کند
   - اسکریپت پس از ایجاد را اجرا می‌کند (`.devcontainer/post-create.sh`)
   - کرنل Jupyter را تنظیم می‌کند

## روند توسعه

### متغیرهای محیطی

تمام درس‌هایی که نیاز به دسترسی API دارند از متغیرهای محیطی تعریف شده در `.env` استفاده می‌کنند:

- `OPENAI_API_KEY` - برای API OpenAI
- `AZURE_OPENAI_API_KEY` - برای Azure OpenAI در Microsoft Foundry (سرویس Azure OpenAI اکنون بخشی از Microsoft Foundry است: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - آدرس نقطه پایانی Azure OpenAI (نقطه پایانی منبع Foundry)
- `AZURE_OPENAI_DEPLOYMENT` - نام استقرار مدل چت کامل
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - نام استقرار مدل امبدینگ‌ها
- `AZURE_OPENAI_API_VERSION` - نسخه API (پیش‌فرض: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - برای مدل‌های Hugging Face
- `AZURE_INFERENCE_ENDPOINT` - نقطه پایانی مدل‌های Microsoft Foundry (کاتالوگ مدل چند ارائه‌دهنده)
- `AZURE_INFERENCE_CREDENTIAL` - کلید API مدل‌های Microsoft Foundry (جایگزین `GITHUB_TOKEN` که بازنشسته می‌شود)

### اجرای مثال‌های پایتون

```bash
# رفتن به پوشه درس
cd 06-text-generation-apps/python

python aoai-app.py
```

### اجرای نوت‌بوک‌های Jupyter


## دستورالعمل‌های سبک کد

### پایتون

- از `python-dotenv` برای مدیریت متغیرهای محیطی استفاده کنید
- کتابخانه `openai` را برای تعاملات API وارد کنید
- برای بررسی کد از `pylint` استفاده کنید (برخی مثال‌ها شامل `# pylint: disable=all` برای سادگی هستند)
- از قرارداد نام‌گذاری PEP 8 پیروی کنید
- اعتبارنامه‌های API را در فایل `.env` ذخیره کنید، هرگز در کد

### تایپ‌اسکریپت

- از بسته `dotenv` برای متغیرهای محیطی استفاده کنید
- پیکربندی تایپ‌اسکریپت در `tsconfig.json` برای هر برنامه
- از بسته `openai` برای Azure OpenAI (کلاینت را روی نقطه پایانی `/openai/v1/` تنظیم کنید و `client.responses.create` را فراخوانی کنید)؛ از `@azure-rest/ai-inference` برای مدل‌های Microsoft Foundry استفاده کنید
- برای توسعه با بارگذاری خودکار از `nodemon` استفاده کنید
- قبل از اجرا بسازید: `npm run build` سپس `npm start`

### قواعد عمومی

- مثال‌های کد را ساده و آموزشی نگه دارید
- شامل توضیحات کلیدی باشید
- کد هر درس باید مستقل و قابل اجرا باشد
- نام‌گذاری یکنواخت را رعایت کنید: پیشوند `aoai-` برای Azure OpenAI، `oai-` برای API OpenAI، `githubmodels-` برای مدل‌های Microsoft Foundry (پیشوند قدیمی از دوران مدل‌های GitHub حفظ شده)
