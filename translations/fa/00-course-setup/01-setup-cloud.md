# راه‌اندازی فضای ابری ☁️ – GitHub Codespaces

**اگر نمی‌خواهید چیزی را روی کامپیوتر خود نصب کنید، از این راهنما استفاده کنید.**  
Codespaces یک نمونه رایگان VS Code مبتنی بر مرورگر در اختیار شما می‌گذارد که همه وابستگی‌ها از پیش نصب شده‌اند.

---

## ۱. چرا Codespaces؟

| مزیت | معنای آن برای شما |
|---------|----------------------|
| ✅ بدون نیاز به نصب | روی کروم‌بوک، آیپد، کامپیوترهای آزمایشگاه مدرسه و ... کار می‌کند |
| ✅ کانتینر توسعه از پیش ساخته شده | پایتون ۳، Node.js، .NET، جاوا از قبل داخل آن است |
| ✅ سهمیه رایگان | حساب‌های شخصی **۱۲۰ ساعت پردازنده / ۶۰ گیگابایت ساعت در ماه** دریافت می‌کنند |

> 💡 **نکته**  
> با **متوقف کردن** یا **حذف** کد‌اسپیس‌های بدون استفاده، سهمیه خود را سالم نگه دارید  
> (از View ▸ Command Palette ▸ *Codespaces: Stop Codespace* استفاده کنید).

---

## ۲. ایجاد یک Codespace (با یک کلیک)

1. این مخزن را **فورک** کنید (دکمه **Fork** بالا سمت راست).  
2. در فورک خود، روی **Code ▸ Codespaces ▸ Create codespace on main** کلیک کنید.  
   ![پنجره‌ای که دکمه‌هایی برای ایجاد یک کداسپیس نشان می‌دهد](../../../translated_images/fa/who-will-pay.4c0609b1c7780f44.webp)

✅ یک پنجره VS Code در مرورگر باز می‌شود و کانتینر توسعه شروع به ساخت می‌کند.
این کار در اولین بار تقریباً **۲ دقیقه** طول می‌کشد.

## ۳. کلید API خود را اضافه کنید (روش ایمن)

### گزینه‌ی الف: مخفی‌گاه‌های Codespaces — توصیه‌شده

1. روی آیکون ⚙️ -> Command Palette -> Codespaces : Manage user secret -> افزودن یک مخفی‌گاه جدید کلیک کنید.
2. نام: OPENAI_API_KEY
3. مقدار: کلید خود را وارد کنید → افزودن مخفی‌گاه

همین بود — کد ما به طور خودکار آن را خواهد خواند.

### گزینه‌ی ب: فایل .env (اگر واقعاً نیاز است)

```bash
cp .env.copy .env
code .env         # مقدار OPENAI_API_KEY=کلید_شما_اینجا را وارد کنید
```
