# چارچوب‌های شبکه عصبی

همان‌طور که قبلاً یاد گرفته‌ایم، برای اینکه بتوانیم شبکه‌های عصبی را به‌طور مؤثر آموزش دهیم، باید دو کار انجام دهیم:

* کار با تنسورها، مانند ضرب، جمع و محاسبه برخی توابع مانند سیگموید یا سافت‌مکس
* محاسبه گرادیان‌های تمام عبارات، برای انجام بهینه‌سازی نزول گرادیان

در حالی که کتابخانه `numpy` می‌تواند بخش اول را انجام دهد، ما به مکانیزمی برای محاسبه گرادیان‌ها نیاز داریم. در چارچوبی که در بخش قبلی توسعه داده‌ایم، مجبور بودیم تمام توابع مشتق را به‌صورت دستی درون متد `backward` برنامه‌نویسی کنیم که وظیفه بازپس‌پراکنی را دارد. در حالت ایده‌آل، یک چارچوب باید به ما این امکان را بدهد که گرادیان‌های *هر عبارتی* را که می‌توانیم تعریف کنیم، محاسبه کنیم.

نکته مهم دیگر این است که بتوانیم محاسبات را بر روی GPU یا هر واحد محاسباتی تخصصی دیگر مانند TPU انجام دهیم. آموزش شبکه‌های عصبی عمیق به *محاسبات زیادی* نیاز دارد و توانایی موازی‌سازی این محاسبات بر روی GPUها بسیار مهم است.

> ✅ اصطلاح 'موازی‌سازی' به معنای توزیع محاسبات بر روی چندین دستگاه است.

در حال حاضر، دو چارچوب عصبی محبوب عبارتند از: TensorFlow و PyTorch. هر دو یک API سطح پایین برای کار با تنسورها بر روی CPU و GPU فراهم می‌کنند. علاوه بر API سطح پایین، یک API سطح بالا نیز وجود دارد که به ترتیب Keras و PyTorch Lightning نامیده می‌شود.

API سطح پایین | TensorFlow| PyTorch
--------------|-------------------------------------|--------------------------------
API سطح بالا| Keras| PyTorch

**API‌های سطح پایین** در هر دو چارچوب به شما اجازه می‌دهند تا به اصطلاح **گراف‌های محاسباتی** بسازید. این گراف تعریف می‌کند که چگونه خروجی (معمولاً تابع هزینه) با پارامترهای ورودی داده‌شده محاسبه شود و می‌تواند برای محاسبه بر روی GPU ارسال شود، اگر در دسترس باشد. توابعی برای تفکیک این گراف محاسباتی و محاسبه گرادیان‌ها وجود دارد که می‌توانند برای بهینه‌سازی پارامترهای مدل استفاده شوند.

**API‌های سطح بالا** عمدتاً شبکه‌های عصبی را به‌عنوان **توالی لایه‌ها** در نظر می‌گیرند و ساخت بیشتر شبکه‌های عصبی را بسیار آسان‌تر می‌کنند. آموزش مدل معمولاً نیاز به آماده‌سازی داده‌ها دارد و سپس با فراخوانی یک تابع `fit` کار انجام می‌شود.

API سطح بالا به شما اجازه می‌دهد تا شبکه‌های عصبی معمولی را به‌سرعت بسازید بدون اینکه نگران جزئیات زیادی باشید. در عین حال، API سطح پایین کنترل بیشتری بر فرآیند آموزش ارائه می‌دهد و به همین دلیل در تحقیقات، زمانی که با معماری‌های جدید شبکه عصبی سر و کار دارید، بسیار مورد استفاده قرار می‌گیرد.

همچنین مهم است که بفهمید می‌توانید هر دو API را با هم استفاده کنید، مثلاً می‌توانید معماری لایه شبکه خود را با استفاده از API سطح پایین توسعه دهید و سپس از آن در شبکه بزرگ‌تری که با API سطح بالا ساخته و آموزش داده شده است، استفاده کنید. یا می‌توانید شبکه‌ای را با استفاده از API سطح بالا به‌عنوان توالی لایه‌ها تعریف کنید و سپس از حلقه آموزش سطح پایین خود برای انجام بهینه‌سازی استفاده کنید. هر دو API از مفاهیم پایه‌ای مشابه استفاده می‌کنند و برای کار با یکدیگر طراحی شده‌اند.

## یادگیری

در این دوره، بیشتر محتوای خود را هم برای PyTorch و هم برای TensorFlow ارائه می‌دهیم. می‌توانید چارچوب مورد علاقه خود را انتخاب کرده و فقط از دفترچه‌های مربوطه عبور کنید. اگر مطمئن نیستید که کدام چارچوب را انتخاب کنید، برخی از بحث‌های اینترنتی در مورد **PyTorch در مقابل TensorFlow** را بخوانید. همچنین می‌توانید به هر دو چارچوب نگاهی بیندازید تا درک بهتری پیدا کنید.

در صورت امکان، از API‌های سطح بالا برای سادگی استفاده خواهیم کرد. با این حال، ما معتقدیم که درک نحوه کار شبکه‌های عصبی از ابتدا بسیار مهم است، بنابراین در ابتدا با API سطح پایین و تنسورها کار می‌کنیم. با این حال، اگر می‌خواهید سریع پیش بروید و نمی‌خواهید زمان زیادی را صرف یادگیری این جزئیات کنید، می‌توانید آن‌ها را نادیده بگیرید و مستقیماً به دفترچه‌های API سطح بالا بروید.

## ✍️ تمرینات: چارچوب‌ها

به یادگیری خود در دفترچه‌های زیر ادامه دهید:

API سطح پایین | دفترچه TensorFlow+Keras | PyTorch
--------------|-------------------------------------|--------------------------------
API سطح بالا| Keras | *PyTorch Lightning*

پس از تسلط بر چارچوب‌ها، مفهوم بیش‌برازش را مرور کنیم.

# بیش‌برازش

بیش‌برازش یک مفهوم بسیار مهم در یادگیری ماشین است و بسیار مهم است که به درستی آن را درک کنیم!

به مسئله زیر در تقریب ۵ نقطه (نمایش داده‌شده توسط `x` در نمودارهای زیر) توجه کنید:

!linear | overfit
-------------------------|--------------------------
**مدل خطی، ۲ پارامتر** | **مدل غیرخطی، ۷ پارامتر**
خطای آموزش = ۵.۳ | خطای آموزش = ۰
خطای اعتبارسنجی = ۵.۱ | خطای اعتبارسنجی = ۲۰

* در سمت چپ، تقریب یک خط مستقیم خوب را می‌بینیم. زیرا تعداد پارامترها مناسب است، مدل ایده پشت توزیع نقاط را به‌درستی می‌فهمد.
* در سمت راست، مدل بیش از حد قدرتمند است. زیرا ما فقط ۵ نقطه داریم و مدل ۷ پارامتر دارد، می‌تواند به‌گونه‌ای تنظیم شود که از تمام نقاط عبور کند و خطای آموزش را به ۰ برساند. با این حال، این مدل را از درک الگوی صحیح داده‌ها بازمی‌دارد و در نتیجه خطای اعتبارسنجی بسیار بالاست.

بسیار مهم است که بین غنای مدل (تعداد پارامترها) و تعداد نمونه‌های آموزشی تعادل صحیحی برقرار کنیم.

## چرا بیش‌برازش رخ می‌دهد

  * داده‌های آموزشی کافی نیست
  * مدل بیش از حد قدرتمند است
  * نویز بیش از حد در داده‌های ورودی

## چگونه بیش‌برازش را تشخیص دهیم

همان‌طور که از نمودار بالا می‌بینید، بیش‌برازش می‌تواند با یک خطای آموزشی بسیار کم و یک خطای اعتبارسنجی بالا تشخیص داده شود. معمولاً در طول آموزش، هر دو خطای آموزشی و اعتبارسنجی شروع به کاهش می‌کنند و سپس در نقطه‌ای خطای اعتبارسنجی ممکن است متوقف شود و شروع به افزایش کند. این نشانه‌ای از بیش‌برازش خواهد بود و نشانگر این است که احتمالاً باید در این نقطه آموزش را متوقف کنیم (یا حداقل یک عکس از مدل بگیریم).

## چگونه از بیش‌برازش جلوگیری کنیم

اگر می‌بینید که بیش‌برازش رخ می‌دهد، می‌توانید یکی از موارد زیر را انجام دهید:

 * مقدار داده‌های آموزشی را افزایش دهید
 * پیچیدگی مدل را کاهش دهید
 * از تکنیک‌های منظم‌سازی مانند Dropout استفاده کنید که بعداً به آن خواهیم پرداخت.

## بیش‌برازش و تعادل بایاس-واریانس

بیش‌برازش در واقع یک مورد از مشکل عمومی‌تری در آمار به نام تعادل بایاس-واریانس است. اگر منابع احتمالی خطا در مدل خود را در نظر بگیریم، می‌توانیم دو نوع خطا را ببینیم:

* **خطاهای بایاس** ناشی از عدم توانایی الگوریتم ما در به‌درستی گرفتن رابطه بین داده‌های آموزشی است. این می‌تواند ناشی از این واقعیت باشد که مدل ما به اندازه کافی قدرتمند نیست (**کم‌برازش**).
* **خطاهای واریانس** که ناشی از تقریب مدل به نویز در داده‌های ورودی به جای رابطه معنادار است (**بیش‌برازش**).

در طول آموزش، خطای بایاس کاهش می‌یابد (زیرا مدل ما یاد می‌گیرد که داده‌ها را تقریب بزند) و خطای واریانس افزایش می‌یابد. مهم است که آموزش را متوقف کنیم - چه به‌صورت دستی (زمانی که بیش‌برازش را تشخیص می‌دهیم) یا به‌صورت خودکار (با معرفی منظم‌سازی) - تا از بیش‌برازش جلوگیری کنیم.
