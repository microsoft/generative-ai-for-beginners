## تولید تقویت‌شده با بازیابی (RAG)

یک چت‌بات مبتنی بر LLM دستورات کاربر را پردازش می‌کند تا پاسخ‌ها را تولید کند. این سیستم به‌صورت تعاملی طراحی شده و با کاربران درباره موضوعات متنوعی گفتگو می‌کند. با این حال، پاسخ‌های آن محدود به زمینه ارائه شده و داده‌های آموزش پایه آن است. برای مثال، محدوده دانش GPT-4 تا سپتامبر ۲۰۲۱ است، یعنی از رویدادهای پس از این تاریخ مطلع نیست. علاوه بر این، داده‌های آموزش LLM شامل اطلاعات محرمانه‌ای مانند یادداشت‌های شخصی یا دفترچه راهنمای محصولات شرکت‌ها نمی‌شود.

### نحوه کار RAGها (تولید تقویت‌شده با بازیابی)

![نموداری که نحوه کار RAGها را نشان می‌دهد](../../../translated_images/fa/how-rag-works.f5d0ff63942bd3a6.webp)

فرض کنید می‌خواهید یک چت‌بات راه‌اندازی کنید که از یادداشت‌هایتان آزمون ایجاد می‌کند، در این صورت به ارتباط با پایگاه دانش نیاز خواهید داشت. اینجا جایی است که RAG به کمک می‌آید. عملکرد RAGها به شرح زیر است:

- **پایگاه دانش:** قبل از بازیابی، این اسناد باید وارد و پیش‌پردازش شوند، معمولاً تجزیه اسناد بزرگ به بخش‌های کوچکتر، تبدیل آنها به تعبیه‌های متنی و ذخیره‌سازی در پایگاه داده.

- **پرسش کاربر:** کاربر یک سؤال می‌پرسد

- **بازیابی:** وقتی کاربر سؤال می‌پرسد، مدل تعبیه، اطلاعات مرتبط را از پایگاه دانش ما بازیابی می‌کند تا متن بیشتری برای گنجاندن در پرسش فراهم کند.

- **تولید تقویت‌شده:** LLM پاسخ خود را بر اساس داده‌های بازیابی‌شده بهبود می‌بخشد. این امکان را می‌دهد که پاسخ تولید شده نه تنها بر اساس داده‌های آموزش دیده باشد بلکه شامل اطلاعات مرتبط از متن اضافه شده نیز باشد. داده‌های بازیابی‌شده برای تقویت پاسخ‌های LLM استفاده می‌شود و سپس LLM پاسخ سؤال کاربر را برمی‌گرداند.

![نموداری که معماری RAGها را نشان می‌دهد](../../../translated_images/fa/encoder-decode.f2658c25d0eadee2.webp)

معماری RAGها با استفاده از ترنسفورمرها که از دو بخش تشکیل شده است: انکودر و دیکودر پیاده‌سازی می‌شود. برای مثال، وقتی کاربر سؤالی می‌پرسد، متن ورودی به بردارهایی تبدیل می‌شود که معنای کلمات را در خود دارند و این بردارها به ایندکس اسناد ما دیکود می‌شوند و بر اساس پرسش کاربر متن جدیدی تولید می‌کنند. LLM از هر دو مدل انکودر-دیکودر برای تولید خروجی استفاده می‌کند.

دو روش برای پیاده‌سازی RAG طبق مقاله پیشنهادی: [Retrieval-Augmented Generation for Knowledge intensive NLP (natural language processing software) Tasks](https://arxiv.org/pdf/2005.11401.pdf?WT.mc_id=academic-105485-koreyst) وجود دارد:

- **_RAG-Sequence_** از مدارک بازیابی‌شده برای پیش‌بینی بهترین پاسخ ممکن به سؤال کاربر استفاده می‌کند

- **RAG-Token** با استفاده از مدارک برای تولید توکن بعدی، سپس بازیابی آن‌ها برای پاسخ به سوال کاربر

### چرا باید از RAGها استفاده کنید؟

- **غنای اطلاعات:** تضمین می‌کند که پاسخ‌های متنی به‌روز و جاری باشند. بنابراین عملکرد در وظایف خاص حوزه‌ای را با دسترسی به پایگاه دانش داخلی بهبود می‌بخشد.

- کاهش ساختگی‌بودن پاسخ‌ها با استفاده از **داده‌های قابل تأیید** در پایگاه دانش برای ارائه زمینه به پرسش‌های کاربران.

- **اقتصادی بودن** زیرا در مقایسه با تنظیم دقیق مدل LLM مقرون به صرفه‌تر است

## ایجاد پایگاه دانش

برنامه ما بر اساس داده‌های شخصی ما است، یعنی درس شبکه عصبی در برنامه آموزشی AI برای مبتدیان.

### پایگاه‌های داده برداری

پایگاه داده برداری، برخلاف پایگاه‌های داده سنتی، پایگاه داده تخصصی طراحی شده برای ذخیره، مدیریت و جستجوی بردارهای تعبیه‌شده است. این پایگاه داده نمایش‌های عددی اسناد را ذخیره می‌کند. تجزیه داده‌ها به تعبیه‌های عددی باعث می‌شود سیستم هوش مصنوعی ما راحت‌تر داده‌ها را درک و پردازش کند.

ما تعبیه‌های خود را در پایگاه‌های داده برداری ذخیره می‌کنیم زیرا LLMها محدودیت تعداد توکنی که به عنوان ورودی می‌پذیرند دارند. از آنجا که نمی‌توان تمام تعبیه‌ها را به LLM ارسال کرد، باید آنها را به بخش‌های کوچکتر تقسیم کنیم و وقتی کاربر سوالی می‌پرسد، تعبیه‌هایی که بیشترین تطابق با سوال دارند همراه با دستور ارسال می‌شوند. بخش‌بندی همچنین هزینه‌ها را در تعداد توکن ارسال‌شده به LLM کاهش می‌دهد.

برخی از پایگاه‌های داده برداری محبوب شامل Azure Cosmos DB، Clarifyai، Pinecone، Chromadb، ScaNN، Qdrant و DeepLake هستند. می‌توانید یک مدل Azure Cosmos DB را با استفاده از Azure CLI با فرمان زیر ایجاد کنید:

```bash
az login
az group create -n <resource-group-name> -l <location>
az cosmosdb create -n <cosmos-db-name> -r <resource-group-name>
az cosmosdb list-keys -n <cosmos-db-name> -g <resource-group-name>
```

### تبدیل متن به تعبیه‌ها

قبل از ذخیره داده، باید آن را به تعبیه برداری تبدیل کنیم قبل از اینکه در پایگاه داده ذخیره شود. اگر با اسناد بزرگ یا متن‌های طولانی کار می‌کنید، می‌توانید آنها را براساس پرسش‌هایی که انتظار دارید بخش‌بندی کنید. بخش‌بندی می‌تواند در سطح جمله یا پاراگراف انجام شود. چون بخش‌بندی معانی را از کلمات اطراف می‌گیرد، می‌توانید زمینه دیگری به بخش اضافه کنید، مثلاً اضافه کردن عنوان سند یا متنی قبل یا بعد از بخش. می‌توانید داده‌ها را به صورت زیر بخش‌بندی کنید:

```python
def split_text(text, max_length, min_length):
    words = text.split()
    chunks = []
    current_chunk = []

    for word in words:
        current_chunk.append(word)
        if len(' '.join(current_chunk)) < max_length and len(' '.join(current_chunk)) > min_length:
            chunks.append(' '.join(current_chunk))
            current_chunk = []

    # اگر بخش آخر به حداقل طول نرسیده بود، به هر حال آن را اضافه کن
    if current_chunk:
        chunks.append(' '.join(current_chunk))

    return chunks
```

پس از بخش‌بندی، می‌توانیم متن خود را با استفاده از مدل‌های تعبیه مختلف جاسازی کنیم. برخی مدل‌هایی که می‌توانید استفاده کنید عبارتند از: word2vec، ada-002 از OpenAI، Azure Computer Vision و دیگر مدل‌ها. انتخاب مدل بستگی به زبان‌های مورد استفاده، نوع محتوایی که کدگذاری می‌شود (متن/تصویر/صدا)، اندازه ورودی قابل کدگذاری و طول خروجی تعبیه دارد.

## بازیابی و جستجوی برداری

وقتی کاربر سوالی می‌پرسد، بازیابی‌کننده آن را با استفاده از انکودر پرسش به بردار تبدیل می‌کند، سپس در ایندکس جستجوی اسناد به دنبال بردارهای مرتبط می‌گردد. پس از آن، بردار ورودی و بردارهای سند را به متن تبدیل کرده و آن را از طریق LLM عبور می‌دهد.

### بازیابی

بازیابی اتفاق می‌افتد وقتی سیستم سعی می‌کند اسنادی را از ایندکس به سرعت پیدا کند که معیارهای جستجو را برآورده کنند. هدف بازیابی‌کننده این است که اسنادی را به دست آورد که برای ارائه زمینه و پایه‌گذاری LLM بر داده‌های شما مورد استفاده قرار گیرد.

روش‌های مختلفی برای جستجو در پایگاه داده ما وجود دارد مانند:

- **جستجوی کلیدواژه** - برای جستجوهای متنی استفاده می‌شود

- **جستجوی برداری** - اسناد را از متن به نمایش برداری با استفاده از مدل‌های تعبیه تبدیل می‌کند، امکان یک **جستجوی معنایی** با استفاده از معنای کلمات را فراهم می‌کند. بازیابی با پرس و جو از اسنادی انجام می‌شود که نمایش برداری آنها به سؤال کاربر نزدیک‌تر است.

- **ترکیبی** - ترکیبی از جستجوی کلیدواژه و برداری.

چالشی که در بازیابی وجود دارد این است که اگر پاسخی مشابه پرسش در پایگاه داده وجود نداشته باشد، سیستم بهترین اطلاعات موجود را ارائه می‌دهد، اما می‌توانید روش‌هایی مانند تنظیم حداکثر فاصله برای ارتباط و استفاده از جستجوی ترکیبی که هم کلیدواژه و هم جستجوی برداری را با هم دارد، استفاده کنید. در این درس از جستجوی ترکیبی، ترکیبی از جستجوی برداری و کلیدواژه، استفاده خواهیم کرد. داده‌های خود را در یک دیتافریم با ستون‌هایی که شامل بخش‌ها و همچنین تعبیه‌ها هستند ذخیره خواهیم کرد.

### شباهت برداری

بازیابی‌کننده در پایگاه دانش به دنبال تعبیه‌هایی می‌گردد که نزدیک به هم باشند، نزدیک‌ترین همسایه، زیرا این متن‌ها مشابه هستند. در سناریو، وقتی کاربر یک سؤال می‌پرسد، اول آن را به تعبیه تبدیل می‌کنیم سپس با تعبیه‌های مشابه تطبیق می‌دهیم. اندازه‌گیری معمول استفاده‌شده برای تعیین میزان شباهت بردارهای مختلف، شباهت کسینوسی است که بر اساس زاویه بین دو بردار است.

ما می‌توانیم شباهت را با روش‌های دیگری نیز اندازه‌گیری کنیم، مانند فاصله اقلیدسی که خط مستقیم بین نقاط نهایی بردارها است و ضرب داخلی که مجموع حاصل‌ضرب عناصر متناظر دو بردار را محاسبه می‌کند.

### ایندکس جستجو

هنگام انجام بازیابی، باید قبل از انجام جستجو یک ایندکس جستجو برای پایگاه دانش خود بسازیم. ایندکس تعبیه‌های ما را ذخیره می‌کند و می‌تواند به سرعت بخش‌های مشابه را حتی در یک پایگاه داده بزرگ بازیابی کند. می‌توانیم ایندکس خود را به صورت محلی با دستور زیر ایجاد کنیم:

```python
from sklearn.neighbors import NearestNeighbors

embeddings = flattened_df['embeddings'].to_list()

# ساخت نمایه جستجو
nbrs = NearestNeighbors(n_neighbors=5, algorithm='ball_tree').fit(embeddings)

# برای پرس‌وجو در نمایه، می‌توانید از روش kneighbors استفاده کنید
distances, indices = nbrs.kneighbors(embeddings)
```

### بازرتبه‌بندی

پس از پرسش پایگاه داده، ممکن است نیاز داشته باشید نتایج را از مرتبط‌ترین مرتب کنید. یک LLM بازرتبه‌بندی با استفاده از یادگیری ماشین مرتبط بودن نتایج جستجو را بهبود می‌بخشد با مرتب‌کردن آن‌ها از بیشترین ارتباط. در Azure AI Search، بازرتبه‌بندی به‌صورت خودکار با استفاده از یک بازرتبه‌بند معنایی انجام می‌شود. نمونه‌ای از نحوه کار بازرتبه‌بندی با استفاده از نزدیک‌ترین همسایگان:

```python
# پیدا کردن اسناد مشابه‌ترین
distances, indices = nbrs.kneighbors([query_vector])

index = []
# چاپ اسناد مشابه‌ترین
for i in range(3):
    index = indices[0][i]
    for index in indices[0]:
        print(flattened_df['chunks'].iloc[index])
        print(flattened_df['path'].iloc[index])
        print(flattened_df['distances'].iloc[index])
    else:
        print(f"Index {index} not found in DataFrame")
```

## همه چیز را با هم جمع کردن

گام نهایی افزودن LLM به فرآیند برای توانایی گرفتن پاسخ‌هایی است که بر داده‌های ما پایه‌گذاری شده‌اند. می‌توانیم آن را به صورت زیر پیاده‌سازی کنیم:

```python
user_input = "what is a perceptron?"

def chatbot(user_input):
    # سوال را به یک بردار پرس و جو تبدیل کنید
    query_vector = create_embeddings(user_input)

    # یافتن مستندات مشابه‌ترین
    distances, indices = nbrs.kneighbors([query_vector])

    # افزودن مستندات به پرس و جو برای فراهم کردن زمینه
    history = []
    for index in indices[0]:
        history.append(flattened_df['chunks'].iloc[index])

    # ترکیب تاریخچه و ورودی کاربر
    history.append(user_input)

    # ایجاد یک شیء پیام
    messages=[
        {"role": "system", "content": "You are an AI assistant that helps with AI questions."},
        {"role": "user", "content": "\n\n".join(history) }
    ]

    # استفاده از API پاسخ‌ها برای تولید پاسخ
    response = client.responses.create(
        model="gpt-4o-mini",
        temperature=0.7,
        max_output_tokens=800,
        input=messages,
        store=False,
    )

    return response.output_text

chatbot(user_input)
```

- مرتبط بودن: ارزیابی اینکه پاسخ با سؤال پرسیده شده مطابقت دارد و مرتبط است

- روانی - اینکه پاسخ از نظر دستوری معنی دارد یا خیر

## موارد استفاده از RAG (تولید تقویت‌شده با بازیابی) و پایگاه‌های داده برداری

موارد استفاده متنوعی وجود دارد که فراخوانی توابع می‌تواند برنامه شما را بهبود بخشد مانند:

- پرسش و پاسخ: پایه‌گذاری داده شرکت شما در چتی که کارمندان می‌توانند از آن سؤال بپرسند.

- سیستم‌های توصیه‌گر: جایی که می‌توانید سیستمی بسازید که بیشترین مقادیر مشابه را تطبیق دهد، مثلاً فیلم‌ها، رستوران‌ها و موارد دیگر.

- خدمات چت‌بات: می‌توانید سابقه چت را ذخیره کرده و مکالمه را بر اساس داده‌های کاربر شخصی‌سازی کنید.

- جستجوی تصویر بر اساس تعبیه‌های برداری، مفید برای شناسایی تصویر و تشخیص ناهنجاری.

## خلاصه

مباحث اساسی RAG، از افزودن داده‌های خود به برنامه، پرسش کاربر و خروجی را پوشش دادیم. برای ساده‌تر کردن ایجاد RAG، می‌توانید از چارچوب‌هایی مانند Semanti Kernel، Langchain یا Autogen استفاده کنید.

## تمرین

برای ادامه یادگیری تولید تقویت‌شده با بازیابی (RAG) می‌توانید بسازید:

- یک رابط کاربری برای برنامه با استفاده از چارچوب دلخواه خود بسازید

- از چارچوبی مانند LangChain یا Semantic Kernel استفاده کنید و برنامه خود را بازسازی کنید.
