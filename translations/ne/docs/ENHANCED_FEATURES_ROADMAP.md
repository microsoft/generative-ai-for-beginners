## ४. API आधुनिकीकरण

### ४.१ अप्रचलित API ढाँचाहरू (स्थानान्तरण पूरा भयो)

सबै Python र TypeScript **च्याट** नमूनाहरू चैट कम्प्लीशन API बाट **प्रतिक्रिया API** (`client.responses.create(...)` → `response.output_text`) मा स्थानान्तरण गरिएका छन्।

| पुरानो ढाँचा | नयाँ ढाँचा | स्थिति |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (च्याट) | `OpenAI(base_url="<endpoint>/openai/v1/")` (प्रतिक्रिया API) | पूरा भयो |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | पूरा भयो |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (TypeScript) | `openai` प्याकेज `client.responses.create()` → `response.output_text` | पूरा भयो |
| `df.append()` (pandas) | `pd.concat()` | पूरा भयो |

> **सूचना:** Microsoft Foundry Models का नमूनाहरू जुन `azure-ai-inference` / `@azure-rest/ai-inference` SDK (`client.complete()`) प्रयोग गर्छन्, मोडेल इन्फरेन्स API मा रहन्छन्, जुन प्रतिक्रिया API लाई समर्थन गर्दैन। `AzureOpenAI()` त्यहाँ जानाजानी संरक्षित गरिएको छ जहाँ अझै मान्य छ (एम्बेडिङ र छवि सिर्जना)।

### ४.२ प्रदर्शनका लागि नयाँ API सुविधाहरू

१. **संरचित आउटपुटहरू** (OpenAI)
   - JSON मोड
   - कडा स्किमासँग फंक्शन कलिङ

२. **भिजन क्षमताहरू**

३. **प्रतिक्रिया API मा बिल्ट-इन उपकरणहरू** (पुरानो असिस्टेन्ट API लाई प्रतिस्थापन गर्दछ)

   - फाइल खोज
   - वेब खोज र अनुकूलित उपकरणहरू

## ५. पूर्वाधार सुधारहरू

### ५.१ CI/CD सुधारहरू

[`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml) मा लागू गरिएको: Python लिन्टिङ/स्वरूप (Ruff + Black) लाई `shared/` उपयोगिताहरुमा अनिवार्य गरिएको छ र पाठ्यक्रमका बाँकी भागहरूमा परामर्शात्मक रूपमा चलाइएको छ, साथै JavaScript/TypeScript को लागि ESLint को परामर्शात्मक पास छ। उदाहरणमूलक आधाररेखा यस्तो थियो:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### ५.२ सुरक्षा स्क्यानिङ

[`.github/workflows/security.yml`](../../../.github/workflows/security.yml) मा लागू गरिएको: Python र JavaScript/TypeScript कोडQL विश्लेषण (पुष, पुल रिक्वेस्ट, र साप्ताहिक तालिका अनुसार) साथै डिपेन्डेन्सी समीक्षा पुल रिक्वेस्टमा। उदाहरणमूलक आधाररेखा यस्तो थियो:

```yaml
# .github/workflows/security.yml
name: Security Scan

on: [push, pull_request]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: javascript, python
      - uses: github/codeql-action/analyze@v3

  dependency-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
```
