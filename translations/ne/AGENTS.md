**मुख्य प्रविधिहरू:**

- TypeScript/JavaScript Node.js सहित र पुस्तकालयहरू: `openai` (Azure OpenAI v1 endpoint + Responses API मार्फत), `@azure-rest/ai-inference` (Microsoft Foundry मोडेलहरू)
- Azure OpenAI सेवा, OpenAI API, र Microsoft Foundry मोडेलहरू (GitHub मोडेलहरू जुलाई २०२६ को अन्त्यमा समाप्त हुँदैछ)
- अन्तरक्रियात्मक सिकाइका लागि Jupyter नोटबुकहरू
- लगातार विकास वातावरणका लागि Dev Containers

**रिपोजिटरी संरचना:**
- २१ नम्बर गरिएका पाठ डाइरेक्टरीहरू (00-21) जसमा README हरू, कोड उदाहरणहरू, र असाइन्मेन्टहरू समावेश छन्

## सेटअप कमाण्डहरू

### प्रारम्भिक रिपोजिटरी सेटअप

```bash
# रिपोजिटरी क्लोन गर्नुहोस्
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# वातावरण टेम्प्लेट प्रति गर्नुहोस्
cp .env.copy .env
# तपाईंका API कुञ्जीहरू र अन्त बिन्दुहरू सहित .env सम्पादन गर्नुहोस्
```

### Python वातावरण सेटअप

```bash
# भर्चुअल परिवेश सिर्जना गर्नुहोस्
python3 -m venv venv

# भर्चुअल परिवेश सक्रिय गर्नुहोस्
# macOS/Linux मा:
source venv/bin/activate
# Windows मा:
venv\Scripts\activate

# निर्भरताहरू स्थापना गर्नुहोस्
pip install -r requirements.txt
```

### Node.js/TypeScript सेटअप

```bash
# मूल-संस्थान स्तरका निर्भरता स्थापना गर्नुहोस् (कागजात उपकरणका लागि)
npm install

# व्यक्तिगत पाठको TypeScript उदाहरणहरूको लागि, निर्दिष्ट पाठमा जानुहोस्:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Dev Container सेटअप (सिफारिस गरिएको)

रिपोजिटरीमा GitHub Codespaces वा VS Code Dev Containers को लागि `.devcontainer` कन्फिगरेसन समावेश छ:

१. GitHub Codespaces वा VS Code Dev Containers एक्सटेन्सन सहित रिपोजिटरी खोल्नुहोस्
२. Dev Container स्वतः:
   - `requirements.txt` बाट Python निर्भरता स्थापना गर्नेछ
   - post-create स्क्रिप्ट (`.devcontainer/post-create.sh`) चलाउनेछ
   - Jupyter कर्नेल सेट अप गर्नेछ

## विकास वर्कफ्लो

### वातावरण भेरिएबलहरू

एपीआई पहुँच आवश्यक पर्ने सबै पाठहरू `.env` मा परिभाषित वातावरण भेरिएबलहरू प्रयोग गर्छन्:

- `OPENAI_API_KEY` - OpenAI API का लागि
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry को Azure OpenAI का लागि (Azure OpenAI सेवा अब Microsoft Foundry को भाग हो: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI endpoint URL (Foundry स्रोत endpoint)
- `AZURE_OPENAI_DEPLOYMENT` - च्याट कम्प्लीसन मोडेल तैनाती नाम
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - इम्बेडिङ मोडेल तैनाती नाम
- `AZURE_OPENAI_API_VERSION` - API संस्करण (पूर्वनिर्धारित: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face मोडेलहरूका लागि
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry मोडेलहरूको endpoint (मल्टि-प्रोभाइडर मोडेल क्याटलग)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry मोडेलहरू API कुञ्जी (रेर्टाइर हुँदै गरेको `GITHUB_TOKEN` को सट्टा)

## कोड शैली मार्गदर्शनहरू

### Python

- वातावरण भेरिएबल व्यवस्थापनका लागि `python-dotenv` प्रयोग गर्नुहोस्
- API अन्तरक्रियाहरूका लागि `openai` पुस्तकालय आयात गर्नुहोस्
- लिंटिङको लागि `pylint` प्रयोग गर्नुहोस् (केही उदाहरणहरूमा सजिलोको लागि `# pylint: disable=all` प्रयोग गरिन्छ)
- PEP 8 नामाङ्कन सम्मेलनहरू पालना गर्नुहोस्
- API प्रमाणपत्रहरू `.env` फाइलमा भण्डारण गर्नुहोस्, कहिल्यै कोडमा होइन

### TypeScript

- वातावरण भेरिएबलहरूको लागि `dotenv` प्याकेज प्रयोग गर्नुहोस्
- प्रत्येक एपको लागि `tsconfig.json` मा TypeScript कन्फिगरेसन
- Azure OpenAI का लागि `openai` प्याकेज प्रयोग गर्नुहोस् (क्लाइन्टलाई `/openai/v1/` endpoint मा संकेत गर्नुहोस् र `client.responses.create` कल गर्नुहोस्); Microsoft Foundry मोडेलहरूको लागि `@azure-rest/ai-inference` प्रयोग गर्नुहोस्
- स्वतः रीलोड सहित विकासका लागि `nodemon` प्रयोग गर्नुहोस्
- चलाउनुअघि बिल्ड गर्नुहोस्: `npm run build` त्यसपछि `npm start`

### सामान्य सम्मेलनहरू

- कोड उदाहरणहरू सरल र शैक्षिक राख्नुहोस्
- मुख्य अवधारणाहरू व्याख्या गर्ने टिप्पणीहरू समावेश गर्नुहोस्
- प्रत्येक पाठको कोड स्व-सम्पूर्ण र चलाउन योग्य हुनु पर्छ
- Azure OpenAI का लागि `aoai-` उपसर्ग, OpenAI API का लागि `oai-`, Microsoft Foundry मोडेलहरूको लागि `githubmodels-` नामाकरणमा सुसंगत रहनुहोस् (GitHub मोडेल युगबाट पूर्व उपसर्ग कायम)

## समस्या निवारण

### सामान्य समस्याहरू

**Python आयात त्रुटिहरू**:
- भर्चुअल वातावरण सक्रिय छ कि छैन जाँच गर्नुहोस्
- `pip install -r requirements.txt` चलाउनुहोस्
- Python संस्करण ३.९+ छ कि छैन जाँच गर्नुहोस्

**TypeScript बिल्ड त्रुटिहरू**:
- सम्बन्धित एप डाइरेक्टरीमा `npm install` चलाउनुहोस्
- Node.js संस्करण उपयुक्त छ कि छैन जाँच गर्नुहोस्
- `node_modules` खाली गरेर पुनःस्थापन गर्नुहोस् यदि आवश्यक छ भने

**API प्रमाणीकरण त्रुटिहरू**:
- `.env` फाइल छ र सही मानहरू छन् कि छैनन् जाँच गर्नुहोस्
- API कुञ्जीहरू मान्य छन् र समय सीमा सकिएको छैन जाँच्नुहोस्
- तपाईंको क्षेत्रका लागि endpoint URLs सही छन् कि छैनन् सुनिश्चित गर्नुहोस्

**वातावरण भेरिएबल हराएका छन्**:
- `.env.copy` बाट `.env` मा कपी गर्नुहोस्
- तपाईंले काम गरिरहेको पाठका लागि आवश्यक सबै मानहरू भर्नुहोस्
- `.env` अपडेट गरेपछि आफ्नो एप पुनः सुरु गर्नुहोस्
