## அறிமுகம்

AI முகவர்கள் உருவாக்கும் AIல் ஒரு பரபரப்பான முன்னேற்றத்தை குறிக்கின்றனர், பெரிய மொழி மாதிரிகள் (LLMs) உதவியாளர்களாக இருந்து செயல்களை மேற்கொள்ளக்கூடிய முகவர்களாக மாறுகின்றன. AI முகவர் கட்டமைப்புகள் LLMக்களுக்கு கருவிகள் மற்றும் நிலை மேலாண்மை அணுகலை வழங்கும் பயன்பாடுகளை உருவாக்க உருவாக்குநர்களுக்கு உதவுகின்றன. இந்த கட்டமைப்புகள் மேலும் காணொளி மேம்படுத்தலும் கொடுக்கும், அது பயனர் மற்றும் உருவாக்குநர்கள் LLMகள் திட்டமிட்ட செயல்களை கண்காணிக்க உதவுகிறது, இதனால் அனுபவ மேலாண்மை மேம்படுகிறது.

பாடம் பின்வரும் பகுதிகளை கையாளும்:

- AI முகவர் என்ன என்பது புரிந்துகொள்வது - AI முகவர் என்றால் என்ன?
- ஐந்து வெவ்வேறு AI முகவர் கட்டமைப்புகளை ஆராய்வு - அவற்றின் தனித்துவம் என்ன?
- இந்த AI முகவர்களை வெவ்வேறு பயன்பாடுகளில் பயன்படுத்துதல் - எப்போது AI முகவர்களை பயன்படுத்த வேண்டும்?

## கற்றல் இலக்குகள்

இந்த பாடத்தை முடித்த பிறகு, நீங்கள் செய்யக்கூடியவை:

- AI முகவர்கள் என்ன மற்றும் அவற்றை எவ்வாறு பயன்படுத்த முடியும் என்பதை விளக்கவும்.
- சில பிரபலமான AI முகவர் கட்டமைப்புகளுக்கு இடையேயான வேறுபாடுகளை புரிந்து கொள்வது.
- AI முகவர்கள் எப்படி செயல்படுகின்றன என்பதை புரிந்து கொண்டு, அவற்றுடன் பயன்பாடுகள் உருவாக்குதல்.

## AI முகவர்கள் என்றால் என்ன?

AI முகவர்கள் உருவாக்கும் AI உலகில் ஒரு மிகப் பரபரப்பான துறையாகும். இந்த பரபரப்புடன் சில சமயங்களில் சொற்கள் மற்றும் அவற்றின் பயன்பாட்டில் குழப்பம் ஏற்படுகிறது. AI முகவர்களாக குறிப்பிடப்படும் பெரும்பாலான கருவிகளுக்கு இலகுவாகவும் உட்புகுமாறு இந்த வரையறையை நாம் பயன்படுத்தப் போகின்றோம்:

AI முகவர்கள் பெரிய மொழி மாதிரிகள் (LLMs) செயல்கள் செய்ய **நிலை** மற்றும் **கருவிகளுக்கு** அணுகலை அளிக்கின்றன.

![Agent Model](../../../translated_images/ta/what-agent.21f2893bdfd01e6a.webp)

இச்சொற்களை வரையறுக்கலாம்:

**பெரிய மொழி மாதிரிகள்** - இந்த பாடத்தில் குறிப்பிடப்படும் மாதிரிகள், உதாரணத்திற்கு GPT-5, GPT-4o மற்றும் Llama 3.3 போன்றவை.

**நிலை** - இது LLM செயல்படுகின்ற சூழலை குறிக்கிறது. LLM அதன் கடந்த செயற்பாடுகளின் சூழலும் தற்போதைய சூழலும் பயன்படுத்தி எதிர்கால செயற்பாடுகளுக்கான முடிவுகளை வழிநடத்துகிறது. AI முகவர் கட்டமைப்புகள் உருவாக்குநர்களுக்கு இந்த சூழலை எளிதாக பராமரிக்க உதவுகின்றன.

**கருவிகள்** - பயனர் கோரிய மற்றும் LLM திட்டமிட்ட பணியை நிறைவேற்ற, LLMக்கு கருவிகளுக்கு அணுகல் தேவை. சில கருவி உதாரணங்கள் காலத்தை шка хранения дерекқор, API, வெளிப்புற பயன்பாடுகள் அல்லது இன்னொரு LLM ஆக இருக்கலாம்!

இந்த வரையறைகள் அடுத்த கட்டங்களுக்கான நல்ல அடித்தளத்தை அளிக்கும் என்று நம்புகிறோம். சில வெவ்வேறு AI முகவர் கட்டமைப்புகளை ஆராய்ப்போம்:

## LangChain முகவர்கள்

[LangChain முகவர்கள்](https://python.langchain.com/docs/how_to/#agents?WT.mc_id=academic-105485-koreyst) மேலே வழங்கிய வரையறைகளின் ஒரு செயல்பாடு ஆகும்.

**நிலையை** நிர்வகிக்க, `AgentExecutor` என்ற அடிப்படை முகாமை பயன்படுத்துகிறது. இது நிறுவிய `agent` மற்றும் அந்தக் கருவிகள் (tools) உடன் செயல்படுகிறது.

`Agent Executor` உரையாடல் வரலாற்றையும் சேமித்து உரையாடல் சூழலைக் கொடுக்கிறது.

![Langchain Agents](../../../translated_images/ta/langchain-agents.edcc55b5d5c43716.webp)

LangChain உங்கள் பயன்பாட்டுக்குள் இறக்கும் ஒரு [கருவி தொகுப்பை](https://integrations.langchain.com/tools?WT.mc_id=academic-105485-koreyst) வழங்குகிறது. இவை சமூகத்தால் மற்றும் LangChain குழுவினால் உருவாக்கப்பட்டவை.

பின்னர் உங்கள் கருவிகளை வரையறுத்து `Agent Executor` க்கு அனுப்பலாம்.

AI முகவர்களைப் பற்றி பேசும் போது, பார்வை மற்றொரு முக்கிய அம்சமாகும். எந்த கருவியை LLM பயன்படுத்துகிறது மற்றும் ஏன் என்பதை பயன்பாடு உருவாக்குநர்களுக்கு புரிந்து கொள்வது அவசியம். அதற்காக LangChain குழு LangSmith ஐ உருவாக்கியுள்ளனர்.

## AutoGen

அடுத்த AI முகவர் கட்டமைப்பு [AutoGen](https://microsoft.github.io/autogen/?WT.mc_id=academic-105485-koreyst). AutoGen இன் முக்கிய கவனம் உரையாடல்களில் உள்ளது. முகவர்கள் இரண்டும் **உரையாடக்கூடிய** மற்றும் **தனிப்பயனாக்கக்கூடிய** ஆவார்கள்.

**உரையாடக்கூடிய -** LLMகள் வேறு LLM உடன் உரையாடலைத் தொடங்கி தொடரலாம் ஒரு பணியை நிறைவேற்ற. இது `AssistantAgents` உருவாக்கி, அவைகளுக்கு ஒரு குறிப்பிட்ட சிஸ்டம் செய்தி கொடுத்து செய்யப்படுகிறது.

```python

autogen.AssistantAgent( name="Coder", llm_config=llm_config, ) pm = autogen.AssistantAgent( name="Product_manager", system_message="Creative in software product ideas.", llm_config=llm_config, )

```

**தனிப்பயனாக்கக்கூடிய** - முகவர்கள் LLM களாக மட்டுமல்ல, பயனர் அல்லது கருவியாகவும் வரையறுக்கப்படலாம். ஒரு உருவாக்குநராக, நீங்கள் ஒரு `UserProxyAgent` ஐ வரையறுக்கலாம், அது பயனருடன் பரிமாற்றி கருத்து பெறும் பணிக்கு பொறுப்பானது. இந்த கருத்து பணியை தொடரவோ நிறுத்தவோ செய்யலாம்.

```python
user_proxy = UserProxyAgent(name="user_proxy")
```

### நிலை மற்றும் கருவிகள்

நிலையை மாற்ற மற்றும் பராமரிக்க, உதவி முகவர் Python குறியீட்டை உருவாக்கி பணியை நிறைவேற்றுகிறது.

இது ஒரு செயல்முறை எடுத்துக்காட்டு:

![AutoGen](../../../translated_images/ta/autogen.dee9a25a45fde584.webp)

#### சிஸ்டம் செய்தியுடன் வரையறுக்கப்பட்ட LLM

```python
system_message="For weather related tasks, only use the functions you have been provided with. Reply TERMINATE when the task is done."
```

இந்த சிஸ்டம் செய்தி இந்த குறிப்பிட்ட LLM அதை செய்ய தேவையான செயல்பாடுகளை அறிவுறுத்துகிறது. AutoGen உடன் நீங்கள் பல AssistantAgents-ஐ வெவ்வேறு சிஸ்டம் செய்திகளுடன் அமைக்கலாம் என்று நினைவில் வையுங்கள்.

#### உரையாடல் பயனர் தொடக்கம்

```python
user_proxy.initiate_chat( chatbot, message="I am planning a trip to NYC next week, can you help me pick out what to wear? ", )

```

பயனர் ப்ராக்ஸி (மனிதர்) இலிருந்து வரும் இந்த செய்தி முகவரின் செயல்முறையைத் துவக்கும், அது எது செயல்படுத்த வேண்டும் என ஆராய்ச்சி செய்யும்.

#### செயல்பாடு நிறைவேற்றப்பட்டது

```bash
chatbot (to user_proxy):

***** Suggested tool Call: get_weather ***** Arguments: {"location":"New York City, NY","time_periond:"7","temperature_unit":"Celsius"} ******************************************************** --------------------------------------------------------------------------------

>>>>>>>> EXECUTING FUNCTION get_weather... user_proxy (to chatbot): ***** Response from calling function "get_weather" ***** 112.22727272727272 EUR ****************************************************************

```

ஆரம்ப உரையாடலும் முடிந்தவுடன், முகவர் பரிந்துரைக்கப்பட்ட கருவியை அழைக்க அனுப்பும். இந்த எடுத்துக்காட்டில் `get_weather` என்ற செயல்பாடு. உங்கள் கட்டமைப்புக்கு ஏற்பட்டு, இந்த செயல்பாடு தானாக இயங்கலாம் அல்லது பயனர் உள்ளீட்டின் அடிப்படையில் செயல்படுத்தலாம்.

மேலும் ஆரம்பிக்க [AutoGen குறியீட்டு எடுத்துக்காட்டுகளை](https://microsoft.github.io/autogen/docs/Examples/?WT.mc_id=academic-105485-koreyst) காணலாம்.

OpenTelemetry மூலம் உள்ளக கண்காணிப்பு வசதியை வழங்கி, உங்கள் முகவர்கள் என்ன செய்கிறார்கள் என்பதை நேரடியாக வரையறுக்க முடியும்.

### நிலை மற்றும் கருவிகள்

**நிலை** - உரையாடல் சூழலை **தொடர்கள்** மூலம் கட்டமைப்பு நிர்வகிக்கிறது. ஒரு முகவர் செய்தி வரலாற்றைக் கவனித்து, ஒவ்வொரு முறை கடந்தவரை மேம்படுத்துகிறது. தொடர்களை நிறுத்தி மீண்டும் தொடங்கலாம்.

**கருவிகள்** - நீங்கள் ஒரு முகவருக்கு சாதாரண Python செயல்களை வழங்கலாம். வகை குறியீட்டோடு கூடிய அளவுருக்கள் தானாகவே ஒரு திட்டத்தில் மாற்றப்பட்டு, மாதிரி அவற்றை எப்போது எப்படி அழைக்க வேண்டும் என்பதை அறிவது. கட்டமைப்பு Model Context Protocol (MCP) சேவையகங்களையும், குறியீடு விளக்குபவர் போன்ற ஹோஸ்ட் செய்யப்பட்ட கருவிகளையும்தான் ஆதரிக்கிறது.

இது ஒரு தனிப்பயன் கருவியுடன் ஒரே முகவரின் எடுத்துக்காட்டு:

```python
import asyncio
from typing import Annotated

from pydantic import Field
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient


def get_weather(
    location: Annotated[str, Field(description="The location to get the weather for.")],
) -> str:
    """Get the weather for a given location."""
    return f"The weather in {location} is sunny with a high of 22°C."


async def main():
    agent = Agent(
        client=OpenAIChatClient(),
        instructions="You are a helpful assistant that can answer weather questions.",
        tools=[get_weather],
    )

    response = await agent.run("What's the weather in Amsterdam?")
    print(response)


asyncio.run(main())
```

Microsoft Foundryவில் Azure OpenAIக்கு தொடர்பு கொள்வதற்கு, உங்கள் முடிவு புள்ளி மற்றும் சான்றுகளை கிளையன்டிற்கு கொடுங்கள்:

```python
from azure.identity.aio import AzureCliCredential
from agent_framework.openai import OpenAIChatClient

client = OpenAIChatClient(
    model="my-gpt-5-mini-deployment",
    azure_endpoint="https://my-resource.openai.azure.com",
    credential=AzureCliCredential(),
)
```

### பல முகவர் பணிச்சூழல்கள்

கட்டமைப்பு உண்மையில் சிறப்பாக செயல்படுவது பல முகவர்களை ஒருங்கிணைப்பதில். உதாரணமாக, முகவர்கள் ஒன்றை அடுத்தொன்றாக இயக்கலாம் (ஒவ்வொன்றும் தன் சூழலை அடுத்து பகிர்கிறது) அல்லது பல முகவர்களை ஒரே நேரத்தில் இயக்கி, முடிவுகளை ஒன்றிணைக்கலாம்:

```python
from agent_framework.orchestrations import SequentialBuilder, ConcurrentBuilder

# உரையாடல் சூழலை சங்கிலியில் கடத்தி, ajanகளை வரிசைப்படுத்திச் செயல்படுத்துக
sequential = SequentialBuilder(participants=[researcher, writer, editor]).build()

# ajanகளுக்கு ஒரே நேரத்தில் பரவச் செய்து, பின்னர் அவர்களின் பதில்களை ஒருங்கிணைக்கவும்
concurrent = ConcurrentBuilder(participants=[analyst_a, analyst_b, analyst_c]).build()
```


## Taskweaver

அடுத்த முகவர் கட்டமைப்பான [Taskweaver](https://microsoft.github.io/TaskWeaver/?WT.mc_id=academic-105485-koreyst) "குறியீடு முதலில்" முகவராக அறியப்படுகிறது, ஏனென்றால் அது கட்டாயமாக `strings` உடன் செயல்படாமல், Python இல் DataFrames உடன் செயல்பட முடியும். இது தரவுத்தள பகுப்பாய்வு மற்றும் உருவாக்க பணிகளுக்கு மிகவும் பயனுள்ளது. இது வரைபடங்கள் மற்றும் பட்டியல்கள் உருவாக்குதல் அல்லது சீரற்ற எண்களை உருவாக்குதல் போன்று இருக்கலாம்.

### நிலை மற்றும் கருவிகள்

உரையாடல் நிலையை நிர்வகிக்க TaskWeaver `Planner` என்ற கருத்தை பயன்படுத்துகிறது. `Planner` என்பது பயனர் கோரிக்கையை எடுத்து, அதை நிறைவேற்ற வேண்டிய பணிகளை வரைபடப்படுத்தும் ஒரு LLM ஆகும்.

பணிகளை முடிக்க `Planner` கு `Plugins` என்ற கருவி தொகுப்பு காட்டப்படுகிறது. இது Python வகுப்புகள் அல்லது பொதுவான குறியீடு விளக்குபவர் ஆக இருக்கலாம். இந்த பிளக்கின்கள் ஏற்றுமான பக்கங்களாக சேமிக்கப்படுகின்றன, ஆகவே LLM சரியான பிளக்கினை தேட எளிதாயிருக்கும்.

![Taskweaver](../../../translated_images/ta/taskweaver.da8559999267715a.webp)

இது அசாதாரணம் கண்டறிதல் (anomaly detection) கையாள ஒரு பிளக்கின் எடுத்துக்காட்டு:

```python
class AnomalyDetectionPlugin(Plugin): def __call__(self, df: pd.DataFrame, time_col_name: str, value_col_name: str):
```

குறியீடு செயல்படுத்துவதற்கு முன் சரிபார்க்கப்படுகிறது. Taskweaverல் சூழல் நிர்வகரிக்க மற்றொரு அம்சம் `experience`. "அனுபவம்" என்பது உரையாடல் சூழலை நீண்டகாலமாக YAML கோப்பில் சேமிக்கும். இது கண்டு LLM சில பணிகளில் நேரத்திற்குட்பட்டு மேம்படும் வகையில் அமைக்கலாம்.

## JARVIS

கடைசி முகவர் கட்டமைப்பு [JARVIS](https://github.com/microsoft/JARVIS?tab=readme-ov-file&WT.mc_id=academic-105485-koreyst). JARVIS தனிச்சிறப்பு என்னவென்றால் உரையாடல் `நிலையை` நிர்வகிப்பதில் LLM ஐ பயன்படுத்துகிறது மற்றும் `கருவிகள்` வேறு AI மாதிரிகள். ஒவ்வொரு AI மாதிரியும் குறிப்பிட்ட பணிகளை செய்ய சிறப்பாக்கப்பட்ட மாதிரிகள், உதாரணமாக பொருள் கண்டறிதல், உரையின் பதிப்பு அல்லது படம் விளக்கம் போன்றவை.

![JARVIS](../../../translated_images/ta/jarvis.762ddbadbd1a3a33.webp)

தானாக வகை பரப்பப்படும் மாதிரி LLM பயனரிடமிருந்து கோரிக்கையை பெற்று, குறிப்பிட்ட பணியை மற்றும் பணியை நிறைவேற்ற தேவையான தகவல்களை அடையாளம் காண்கிறது.

```python
[{"task": "object-detection", "id": 0, "dep": [-1], "args": {"image": "e1.jpg" }}]
```

LLM பின்னர் JSON போன்ற வடிவத்தில் கோரிக்கையை மாற்றி, சிறப்பு AI மாதிரி அதை உணரக்கூடியவாறு செய்கிறது. AI மாதிரி அதன் கணிப்புகளை வழங்கியதும், LLM பதில்களை பெறும்.

பணியை நிறைவேற்ற பல மாதிரிகள் தேவைப்பட்டால், LLM அவற்றின் பதில்களையும் விளக்கி பின்னர் ஒன்றிணைத்து பயனருக்கு பதிலளிக்கும்.

கீழே ஒரு எடுத்துக்காட்டு உள்ளது; பயனர் ஒரு படத்தில் உள்ள பொருட்களின் விளக்கம் மற்றும் எண்ணிக்கை கோரும்போது இது எப்படி செயல்படும் என்பதைக் காட்டுகிறது:

## பணிப்பணி

Microsoft முகவர் கட்டமைப்போடு உங்கள் AI முகவர் கற்றலை தொடர:

- கல்வி துவக்கம் நிறுவன பல துறைவரிசையாக ஒரு வணிக கூட்டத்தை புனைக.
- LLMகளை வெவ்வேறு நபர் அல்லது முன்னுரிமைகளைப் புரிந்துகொள்ளும் சிஸ்டம் செய்திகளை உருவாக்கி, பயனருக்கு புதிய தயாரிப்பை விளக்கவும்.
- பின்னர் LLM ஒவ்வொரு துறையிடமிருந்து தொடர்ச்சி கேள்விகளை உருவாக்கி, விளக்கத்தையும் தயாரிப்பு யோசனையும் மேம்படுத்தும்.
