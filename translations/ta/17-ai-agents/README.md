## அறிமுகம்

AI முகவர்கள் ஜெனரட்டிவ் AI-வில் ஒரு மிக சுவாரசியமான முன்னேற்றமாகும், பெரும் மொழி மாதிரிகள் (LLMs) உதவியாளர்களாக இருந்து செயல்களைச் செய்யக்கூடிய முகவர்களாக மாறுவதற்கான வழிகளை திறக்கின்றன. AI முகவர் கட்டமைப்புகள், LLM-க்களுக்கு கருவிகள் மற்றும் நிலை மேலாண்மைக்கு அணுகலை வழங்கும் செயலிகளைக் கட்டியமைக்க டெவலப்பர்களுக்கு உதவுகின்றன. இந்த கட்டமைப்புகள் காணப்படுதல் திறனையும் மேம்படுத்துகின்றன, பயனாளர்களும் டெவலப்பர்களும் LLM-கள் திட்டமிட்டுள்ள செயல்களை கண்காணிக்க முடிகிறது, மேலும் அனுபவ மேலாண்மையை மேம்படுத்துகின்றன.

இந்த பாடம் பின்வரும் பகுதிகளை உள்ளடக்கியதாக இருக்கும்:

- AI முகவர் என்றால் என்ன என்பதை புரிந்துகொள்ளுதல் - AI முகவர்கள் என்னவென்று சரியாக என்ன?
- ஐந்து விதமான AI முகவர் கட்டமைப்புகளை ஆராய்தல் - அவைகள் ஏன் தனித்துவமானவை?
- இந்த AI முகவர்களைக் குழப்பங்களாக பயன்படுத்துவது - எப்போது AI முகவர்களை பயன்படுத்த வேண்டும்?

## கற்றல் இலக்குகள்

இந்த பாடத்தை எடுத்துக்கொண்ட பிறகு நீங்கள் செய்ய இயலும்:

- AI முகவர்கள் எவை மற்றும் அவற்றை எப்படிக் பயன்படுத்துவதென்று விளக்குதல்.
- பாபுலர் AI முகவர் கட்டமைப்புகளுக்கு இடையேயான வேறுபாடுகளை புரிந்து கொள்ளுதல்.
- AI முகவர்கள் எப்படி செயல்படுகின்றன என்பதைப் புரிந்து கொண்டு அவற்றுடன் செயலிகள் கட்டியமைத்தல்.

## AI முகவர்கள் என்றால் என்ன?

Generative AI உலகத்தில் AI முகவர்கள் மிகவும் சுவாரசியமான துறையாக உள்ளன. இந்த அசத்தல் சில நேரங்களில் விளக்கங்கள் மற்றும் அவற்றின் பயன்பாட்டில் குழப்பத்தையும் ஏற்படுத்துகிறது. AI முகவர்களை குறிப்பு செய்யும் பெரும்பான்மையான கருவிகளை உட்படுத்த எளிமையாகவும் உள்ளடக்கமாகவும் இருக்க, நாம் இந்த வரையறையைப் பயன்படுத்தப் போகிறோம்:

AI முகவர்கள் பெரும் மொழி மாதிரிகள் (LLMs) ஒரு **நிலை** மற்றும் **கருவிகள்** கொண்டு செயல்களைச் செய்ய உதவுகின்றன.

![Agent Model](../../../translated_images/ta/what-agent.21f2893bdfd01e6a.webp)

இந்த சொற்களைக் குறியாக்கலாம்:

**பெரும் மொழி மாதிரிகள்** - GPT-3.5, GPT-4, Llama-2 போன்ற இந்த பாடத்தில் குறிப்பிடப்படும் மாதிரிகள்.

**நிலை** - இது LLM பணியாற்றும் சூழலை குறிக்கிறது. LLM தன் கடந்த செயல்களின் மற்றும் தற்போதைய சூழலைப் பயன்படுத்தி அடுத்த செயல்களுக்கான முடிவுகளை வழிநடத்துகிறது. AI முகவர் கட்டமைப்புகள் இந்த சூழலை பராமரிக்க সহজமாகச் செய்யும்.

**கருவிகள்** - பயனர் கேட்ட செயலையும் LLM திட்டமிடுவதை நிறைவேற்ற, LLM-க்கு கருவிகளுக்கான அணுகல் தேவை. கருவிகளின் சில உதாரணங்கள் தரவுத்தளம், API, வெளிப்புற செயலி அல்லது இன்னொரு LLM இருக்க முடியும்!

இந்த வரையறைகள், அவை எப்படி செயல்படுத்தப்படுகின்றன என்பதைப் பார்ப்பதில் நமக்குக் கூடிய அறிவுத் தளத்தை அளிக்கும் என்று நம்புகிறோம். சில வித்தியாசமான AI முகவர் கட்டமைப்புகளை ஆய்வோம்:

## LangChain முகவர்கள்

[LangChain முகவர்கள்](https://python.langchain.com/docs/how_to/#agents?WT.mc_id=academic-105485-koreyst) மேலேத் தரப்பட்ட வரையறைகளின் ஒரு செயல்படுத்தலாகும்.

**நிலை**-ஐ நிர்வகிக்க, இது `AgentExecutor` என்ற நுழைவாயில் பண்பைக் கொண்டுள்ள இயல்புநிலை செயற்கூறை பயன்படுத்துகிறது. இது வரையறுக்கப்பட்ட `agent` மற்றும் கிடைக்கும் `tools`ஐ ஏற்றுக்கொள்கிறது.

`Agent Executor` உரையாடல் வரலாற்றையும் சேமித்து உரையாடல் சூழலை வழங்குகிறது.

![Langchain Agents](../../../translated_images/ta/langchain-agents.edcc55b5d5c43716.webp)

LangChain ஒரு [கருவி பட்டியலை](https://integrations.langchain.com/tools?WT.mc_id=academic-105485-koreyst) வழங்குகிறது, இதைப் உங்கள் செயலியில் இறக்குமதி செய்து LLM-க்கு அணுகல் அளிக்க முடியும். இவை சமுதாயத்தினரால் மற்றும் LangChain குழுவினரால் உருவாக்கப்பட்டவை.

பின்னர், இந்த கருவிகளை வரையறுத்து `Agent Executor`-க்கு வழங்க முடியும்.

AI முகவர்களைப் பற்றி பேசும்போது காணப்படுதல் திறன் முக்கிய அம்சம். LLM எந்த கருவியை எதற்காகப் பயன்படுத்துகிறது என்பதைக் காண application டெவலப்பர்கள் புரிந்துகொள்ள வேண்டும். இதற்காக LangChain குழு LangSmith-ஐ உருவாக்கியுள்ளது.

## AutoGen

அடுத்த AI முகவர் கட்டமைப்பு [AutoGen](https://microsoft.github.io/autogen/?WT.mc_id=academic-105485-koreyst) ஆகும். AutoGen இன் முக்கிய கவனம் உரையாடலில் உள்ளது. முகவர்கள் இரண்டும் **உரையாடக்கூடியமும்** மற்றும் **இயல்புத்தன்மையுடைய**வுமாக இருக்கின்றன.

**உரையாடக்கூடிய -** LLM இரண்டு LLM-களுடன் உரையாடத் தொடங்கி, தொடர முடியும், ஒரு பணியை நிறைவேற்ற. இது `AssistantAgents` உருவாக்கி அவர்களுக்கு ஒரு குறிப்பிட்ட சிஸ்டம் செய்தியைக் கொடுப்பதன் மூலம் செய்கிறது.

```python

autogen.AssistantAgent( name="Coder", llm_config=llm_config, ) pm = autogen.AssistantAgent( name="Product_manager", system_message="Creative in software product ideas.", llm_config=llm_config, )

```

**இயல்புத்தன்மையுடைய** - முகவர்கள் LLM-களாக மட்டுமின்றி பயனாளர் அல்லது கருவியாகவும் வரையறுக்கப்படலாம். டெவலப்பராக, `UserProxyAgent` எனும் முகவரைக் வரையறுக்கலாம், இது பயனாளருடன் தொடர்பு கொண்டு பணியை நிறைவேற்ற கருத்துக்களைப் பெற பொறுப்பாக இருக்கிறது. அந்த கருத்து பணியை தொடரவோ நிறுத்தவோ செய்யலாம்.

```python
user_proxy = UserProxyAgent(name="user_proxy")
```

### நிலையும் கருவிகளும்

நிலையை மாற்றவும் நிர்வகிக்கவும், ஒரு உதவியாளர் முகவர் Python குறியீட்டை உருவாக்கி பணியை நிறைவேற்றுகிறது.

செயல்முறை எடுத்துக்காட்டு இதோ:

![AutoGen](../../../translated_images/ta/autogen.dee9a25a45fde584.webp)

#### ஒரு சிஸ்டம் செய்தியுடன் வரையறுக்கப்பட்ட LLM

```python
system_message="For weather related tasks, only use the functions you have been provided with. Reply TERMINATE when the task is done."
```

இந்த சிஸ்டம் செய்தி குறிப்பிட்ட LLM-க்கு எந்த செயல்பாடுகள் அதன் பணிக்குத் தொடர்புடையவை என்பதைக் கண்காணிக்கிறது. AutoGen-ல் பல AssistantAgents வேறுபட்ட சிஸ்டம் செய்திகளுடன் வரையறுக்கப்படலாம் என்றதை நினைவில் கொள்க.

#### உரையாடல் பயனாளரால் துவங்கப்படுகிறது

```python
user_proxy.initiate_chat( chatbot, message="I am planning a trip to NYC next week, can you help me pick out what to wear? ", )

```

இதுவே user_proxy (மனிதர்) மூலம் வரும் செய்தி, இது முகவரின் செயல்முறையை துவக்குவதற்கான வழியைக் காண்கிறது.

#### செயல்பாடு இயங்குகிறது

```bash
chatbot (to user_proxy):

***** Suggested tool Call: get_weather ***** Arguments: {"location":"New York City, NY","time_periond:"7","temperature_unit":"Celsius"} ******************************************************** --------------------------------------------------------------------------------

>>>>>>>> EXECUTING FUNCTION get_weather... user_proxy (to chatbot): ***** Response from calling function "get_weather" ***** 112.22727272727272 EUR ****************************************************************

```

ஆரம்ப உரையாடல் செயலாக்கப்பட்ட பிறகு, முகவர் அழைக்க வேண்டிய கருவியை பரிந்துரைக்கிறது. இங்கு `get_weather` என்ற செயல்பாடாகும். உங்கள் கட்டமைப்பு மேல், இந்த செயல்பாடு தானாக இயக்கப்பட்டு முகவரால் வாசிக்கப்படலாம் அல்லது பயனர் உள்ளீட்டின் அடிப்படையில் இயக்கப்படலாம்.

[AutoGen குறியீடு மாதிரிகள்](https://microsoft.github.io/autogen/docs/Examples/?WT.mc_id=academic-105485-koreyst) பட்டியலை மேலும் அறிந்து கட்ட அமைக்க முயற்சிக்கலாம்.

## Microsoft Agent கட்டமைப்பு

[Microsoft Agent கட்டமைப்பு](https://learn.microsoft.com/agent-framework/?WT.mc_id=academic-105485-koreyst) Microsoft ஐக்கிய மொழி SDK ஆகும், Python மற்றும் .NET இரண்டிலும் AI முகவர்களையும் பன்முக முகவர் அமைப்புகளையும் உருவாக்க உதவுகிறது. இது முன்னர் இரண்டு Microsoft திட்டங்களின் பலங்களை ஒன்றிணைக்கிறது — **Semantic Kernel** இன் நிறுவன அம்சங்கள் மற்றும் **AutoGen** இன் பன்முக முகவர்களின் ஒத்துழைப்பு — ஒரே கட்டமைப்பாக. இன்று புதிய முகவர் திட்டம் துவங்கினால், இது AutoGen-க்கு பரிந்துரைக்ககூடிய மாற்று ஆகும்.

இந்த கட்டமைப்பு ஒரு **உரையாடல் முகவரிலிருந்து** தொடங்கி சிக்கலான **பன்முக முகவர் வேலைபடிகளுக்கு** வரை விரிவடைகிறது, மேலும் Microsoft Foundry, Azure OpenAI மற்றும் OpenAI உடன் நேரடியாக இணைக்கிறது. OpenTelemetry மூலம் கட்டமைப்பு முன்னோட்டத்தையும் வழங்குகிறது, உங்கள் முகவர்கள் என்ன செய்கிறார்கள் என்பதைக் கவனிக்க முடியும்.

### நிலையும் கருவிகளும்

**நிலை** - உரையாடல் சூழலை நீங்கள் நிர்வகிக்கின்றீர்கள், **துணைகள்** மூலமாக. ஒரு முகவர் செய்தி வரலாறினை (பயனர் கோரிக்கைகள், கருவி அழைப்புகள் மற்றும் பலன்கள்) கண்காணிக்கிறது, ஆகவே ஒவ்வொரு சுற்றம் முந்தையவற்றின் அடிப்படையில் கட்டப்படுகிறது. துணைகள் பாதுகாக்கப்படலாம்; உரையாடலை நிறுத்தி பிறகு தொடரலாம்.

**கருவிகள்** - நீங்கள் முகவருக்கு கருவிகளை சலாஸ்பாரியான Python செயல்பாடுகளை வழங்கி கொடுக்கின்றீர்கள். அங்கு வகை குறிப்பு கொண்ட அளவுருக்கள் தானாகவே ஒரு கட்டமைப்பாக மாறும், ஆகவே மாதிரி அவற்றை எப்போது எப்படிப் பயன்படுத்துவது என்பதைக் கற்றுக் கொள்ளும் (செயல்பாடு அழைப்புக் குறியீடு). கட்டமைப்பு Model Context Protocol (MCP) சேவைகள் மற்றும் ஒரு குறியீடு விளக்கி போன்ற அடங்கிய கருவிகளையும் ஆதரிக்கிறது.

கீழே ஒரு தனியுரிமை கருவியுடன் ஒரே முகவரின் எடுத்துக்காட்டு:

```python
import asyncio
from typing import Annotated

from pydantic import Field
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient


def get_weather(
    location: Annotated[str, Field(description="The location to get the weather for.")],
) -> str:
    """Get the weather for a given location."""
    return f"The weather in {location} is sunny with a high of 22°C."


async def main():
    agent = Agent(
        client=OpenAIChatClient(),
        instructions="You are a helpful assistant that can answer weather questions.",
        tools=[get_weather],
    )

    response = await agent.run("What's the weather in Amsterdam?")
    print(response)


asyncio.run(main())
```

Microsoft Foundry-இல் Azure OpenAI இணைக்க, உங்கள் முடிவுநிலை மற்றும் சான்றுகளை client-க்கு கொடுக்கவும்:

```python
from azure.identity.aio import AzureCliCredential
from agent_framework.openai import OpenAIChatClient

client = OpenAIChatClient(
    model="my-gpt-4o-deployment",
    azure_endpoint="https://my-resource.openai.azure.com",
    credential=AzureCliCredential(),
)
```

### பன்முக முகவர் வேலைபடிகள்

இந்த கட்டமைப்பு முக்கியமாக பல முகவர்களை ஒருங்கிணைப்பதில் சிறக்கிறது. உதாரணமாக, முகவர்கள் ஒன்றுக்குப்பிறகு ஓர் முகவர் இயக்கலாம் (ஒவ்வொருவரும் தமது சூழலை அடுத்தவருக்கு வழங்கி) அல்லது பல முகவர்களை ஒன்றே நேரத்தில் இயக்கி அவற்றின் முடிவுகளை கூட்டலாம்:

```python
from agent_framework.orchestrations import SequentialBuilder, ConcurrentBuilder

# உரையாடல் контекстை சங்கிலியில் கொண்டு செல்லும் பயணிகளைக் காலவரிசையில் இயக்கவும்
sequential = SequentialBuilder(participants=[researcher, writer, editor]).build()

# ஒரே நேரத்தில் பல பயணிகளுக்கு பரவல் செய்து, பின்னர் அவர்களின் பதில்களை கூட்டவும்
concurrent = ConcurrentBuilder(participants=[analyst_a, analyst_b, analyst_c]).build()
```

```bash
pip install agent-framework-openai       # OpenAI மற்றும் Azure OpenAI
pip install agent-framework-foundry      # Microsoft Foundry
```

## Taskweaver

அடுத்த முகவர் கட்டமைப்பு [Taskweaver](https://microsoft.github.io/TaskWeaver/?WT.mc_id=academic-105485-koreyst). இது "முந்தைய குறியீடு" முகவராக அறியப்படுகிறது, ஏனெனில் இது `strings` உடன் சிக்காமல், Python-இல் DataFrames உடன் பணியாற்ற முடியும். இது தரவு பகுப்பாய்வு மற்றும் உருவாக்க பணிகளுக்கு மிகவும் பயனுள்ளது. இது வரைபடங்கள் மற்றும் வரைபுக்களை உருவாக்குதல் அல்லது சீரற்ற எண்களை உருவாக்குதல் போன்றவை ஆகும்.

### நிலையும் கருவிகளும்

உரையாடல் நிலையை நிர்வகிக்க TaskWeaver `Planner` என்ற கருத்தை பயன்படுத்துகிறது. `Planner` என்பது பயனர்களின் கோரிக்கையை எடுத்து, அதை நிறைவேற்ற தேவையான பணிகளை வரைபடமாக்கும் LLM ஆகும்.

பணிகளை நிறைவேற்ற `Planner`-க்கு `Plugins` என்ற கருவி தொகுப்பை அணுக அனுமதிக்கப்படுகிறது. இவை Python வகுப்புகளோ அல்லது ஒரு பொதுவான குறியீடு விளக்கியல்புருவாக்கமும் ஆகலாம். இந்த பிளகின்கள் முத்திரைகளாக சேமிக்கப்படுகின்றன, ஆகவே LLM சரியான பிளகின் தேட முடியும்.

![Taskweaver](../../../translated_images/ta/taskweaver.da8559999267715a.webp)

அசாதாரணம் கண்டறிதல் நிர்வகிக்கக் கூடிய ஒரு பிளகின் எடுத்துக்காட்டு இதோ:

```python
class AnomalyDetectionPlugin(Plugin): def __call__(self, df: pd.DataFrame, time_col_name: str, value_col_name: str):
```

குறியீடு இயக்குவதற்கு முன் சரிபார்க்கப்படுகிறது. Taskweaver-இல் மேலதிக உள்ளடக்கம் நிர்வாகக் கருவி `experience`. Experience உரையாடல் சூழலை நீண்ட காலம் YAML கோப்பில் சேமிக்க உதவுகிறது. இது LLM சில குறிப்பிட்ட பணிகளில் பழைய உரையாடல்களைப் பார்க்கும்போது முன்னேற்றம் அடைய வழி செய்யும்.

## JARVIS

கடைசியில் ஆய்வு செய்யவுள்ள முகவர் கட்டமைப்பு [JARVIS](https://github.com/microsoft/JARVIS?tab=readme-ov-file&WT.mc_id=academic-105485-koreyst). JARVISயின் தனித்துவம் LLM உரையாடல் `நிலை`-ஐ நிர்வகிக்க பயன்படுவதாகவும், பிற AI மாதிரிகள் `கருவிகள்`ஆக பயன்படுவதாகவும் இருக்கிறது. ஒவ்வொரு AI மாதிரியும் குறிப்பிட்ட பணிகளைச் செய்யும் சிறப்பு மாதிரிகள், যেমন பொருள் கண்டறிதல், ஒலி உரைத்தல் அல்லது படம் விளக்கம்.

![JARVIS](../../../translated_images/ta/jarvis.762ddbadbd1a3a33.webp)

பொதுவான நோக்குடன் இருக்கும் LLM பயனாளரின் கோரிக்கையை பெறுகிறது, குறிப்பிட்ட பணியை மற்றும் அதை நிறைவேற்ற தேவையான தரவுகளை அடையாளம் காண்கிறது.

```python
[{"task": "object-detection", "id": 0, "dep": [-1], "args": {"image": "e1.jpg" }}]
```

LLM பிறகு JSON போன்ற வடிவத்தில் கோரிக்கையை வடிவமைக்கிறது, சிறப்பான AI மாதிரிகள் அதனை புரிந்து செயல்பட. AI மாதிரி பணியின் அடிப்படையில் முன்கூட்டிய முன்னறிவிப்பை வழங்கி, LLM பதிலைக் பெறும்.

ஒரே செயலுக்குக் பல மாதிரிகள் தேவைப்பட்டால், அவற்றின் பதில்களையும் புரிந்து தக்கவாறு ஒன்று சேர்த்து பயனாளருக்கான பதிலை உருவாக்கும்.

கீழே எடுத்துக்காட்டு, பயனர் படத்தில் உள்ள பொருட்களின் விளக்கம் மற்றும் எண்ணிக்கையை கேட்கும்போது இது எப்படிச் செயல்படுகிறது என்பதை காட்டுகிறது:

## பணிமுறை

AI முகவர்கள் பற்றிய உங்கள் கற்றலை தொடர Microsoft Agent கட்டமைப்புடன் கட்டமைக்கலாம்:

- கல்வி துவக்க நிறுவனத்தின் வெவ்வேறு துறை உறுப்பினர்களுடன் ஒரு வணிக கூட்டத்தை நகலெடுக்க ஒரு செயலியை உருவாக்குக.
- LLM-களை வெவ்வேறு பண்புகள் மற்றும் முன்னுரிமைகளைப் புரிந்துகொள்ள வழிநடத்தும் சிஸ்டம் செய்திகள் உருவாக்கவும், பயனாளருக்கு புதிய தயாரிப்பு யோசனையை முன்மொழிய அனுமதி வழங்கவும்.
- பின் ஒவ்வொரு துறையிடமிருந்து கேள்விகள் உருவாக்கப்பட்டு யோசனையை சீரமைக்கவும் மேம்படுத்தவும் செய்யப்பட வேண்டும்.
