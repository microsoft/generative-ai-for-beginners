# தொடக்கம் குழந்தை மொழி மாதிரிகளுக்கான உருவாக்கும் AI க்கான அறிமுகம்
உருவாக்கும் AI என்பது புதிய உள்ளடக்கங்களை உருவாக்கும் திறன் கொண்ட கணினி அமைப்புகளை உருவாக்குவதை முக்கியமாக கவனிக்கும் கைவினைக் கலைதுறையைச் சேர்ந்த ஒரு ஆர்வமூட்டும் பகுதி ஆகும். இந்த உள்ளடக்கம் உரை மற்றும் படங்கள் முதல் இசை மற்றும் முழு வர்த்தக சூழல்களைப் பொருந்தக்கூடும். உருவாக்கும் AI இன் மிகவும் சுவாரஸ்யமான பயன்பாடுகளில் ஒன்று மொழி மாதிரிகள் ஆகும்.

## குழந்தை மொழி மாதிரிகள் என்றால் என்ன?

ஒரு குழந்தை மொழி மாதிரி (SLM) என்பது பெரிய மொழி மாதிரி (LLM) ஐ குறைந்த அளவில் கண்டிப்பதற்கான ஒரு வடிவமாகும், இது LLM களின் பல கட்டமைப்பு 원리 மற்றும் தொழில்நுட்பங்களை பயன்படுத்துகின்றது, ஆனால் கணினி Footprint-ஐ மிகுந்த அளவு குறைத்துள்ளது.

SLM கள் மனிதர் போன்ற உரையை உருவாக்குவதற்காக வடிவமைக்கப்பட்ட மொழி மாதிரி ஒரு துணை அடுக்கு ஆகும். அவை GPT-4 போன்ற பெரிய மாதிரிகளுக்கு மாறாக, குறைந்த அளவு மற்றும் திறமையானவையாக இருந்தால், கணினி வளங்கள் குறைந்த இடங்களில் பயன்பாட்டிற்கு சிறந்தவை. அவை சிறிய அளவில் இருந்தாலும் பலவகை பணிகளுக்கு இயல்பாக செயல்படக்கூடும். பொதுவாக, SLM கள் LLM களை சுருக்கி அல்லது சுருக்கி உருவாக்கப்படுகின்றன, முதன்மை மாதிரி செயல்திறன் மற்றும் பாஷை திறன்களை பெரும்பாலான அளவிலும் பாதுகாக்க முயற்சித்து. மாதிரி அளவு குறைப்பால் மொத்தக் குழப்பம் குறைய, நினைவகம் பயன்படுத்தல் மற்றும் கணினி தேவைகளில் மேம்படுத்தப்படுகின்றது. இவ்வாறான மேம்பாடுகளுடன் கூட, SLM கள் பல விலக்கு பாஷை செயலாக்க (NLP) பணிகளை செய்யலாம்:

- உரை உருவாக்கம்: தெளிவான மற்றும் சூழல் பொருத்தமான வாக்கியங்கள் அல்லது பத்திகளை உருவாக்குதல்.
- உரை முடித்தல்: கொடுக்கப்பட்ட தொடக்கத்திற்குத் தகுந்த வாக்கியங்களை கணிக்கவும் மற்றும் முடிக்கவும்.
- மொழி மாற்றம்: ஒரு மொழியிலிருந்து மற்றொரு மொழிக்கு உரையை மாற்றுதல்.
- சுருக்கல்: நீண்ட உரையை குறைந்த, சுளுக்கமான சுருக்கங்களில் மாற்றுதல்.

பெரிய மாதிரிகளுடன் ஒப்பிடும்போது செயல்திறன் அல்லது ஆழமான புரிதலில் சில மாற்றங்களுடன்.

## குழந்தை மொழி மாதிரிகள் எப்படி செயல்படுகின்றன?
SLM கள் பெரிதளவில் உரை தகவல்களைப் பயிற்சியில் கற்றுக்கொள்கின்றன. பயிற்சியின் போது, மொழியின் முறைப்படிகள் மற்றும் கட்டமைப்புகளை கற்றுக்கொண்டு, வாய்ப்பான மற்றும் பொருத்தமான உரையை உருவாக்கத் திறனாகின்றன. பயிற்சி செயல்முறை:

- தரவு சேகரிப்பு: பல்வேறு மூலங்களிலிருந்து பெரிய உரை தரவுத்தொகுப்புகளை சேர்ந்தல்.
- முன்பதிவு: பயிற்சிக்கு ஏற்றவாறு தரவுகளை சுத்தம் செய்யும் மற்றும் ஒழுங்குபடுத்துதல்.
- பயிற்சி: இயந்திரக்கற்றல் ஆல்காரிதம்களை பயன்படுத்தி மாதிரியை உரையை புரிந்து உருவாக்க கற்றுத்தருதல்.
- நுட்பமுடுக்கம்: குறிப்பிட்ட பணிகளில் செயல்திறனை மேம்படுத்த மாதிரியை சரிசெய்தல்.

SLM களின் முன்னேற்றம் கணினி வளங்கள் குறைவான சூழல்களில், உதாரணத்திற்கு மொபைல் சாதனங்கள் அல்லது எட்ஜ் கணினி தளங்களில், முழுமையான LLMகள் அதிக வள தேவைக்காணல் காரணமாக செயல்படும் வகையில் உள்ளது. திறமையை கவனித்து, SLM கள் செயல்திறன் மற்றும் அணுகுமுறையை சமநிலை செய்யின்றன, பல்வேறு துறைகளில் பரவலான பயன்பாட்டிற்கு வழி செய்கின்றன.

## கற்றல் குறிக்கோள்கள்

இந்த பாடத்தின் முடிவில், நீங்கள் பின்வரும் கேள்விகளுக்கு பதிலளிக்க முடியும்:

- SLM என்பது என்ன?

தயார்? ஆரம்பிக்கலாம்.

## பெரிய மொழி மாதிரிகள் (LLM) மற்றும் குழந்தை மொழி மாதிரிகள் (SLM) இடையேயான வேறுபாடுகள்

LLM மற்றும் SLM இரண்டும் வாய்ப்புக்களை அடிப்படையாகக் கொண்ட இயந்திரக் கற்றல் 원리-களின் அடிப்படையில் கட்டப்பட்டவை, அதே கட்டமைப்பு வடிவமைப்பு, பயிற்சி முறைகள், தரவு உருவாக்க செயல்முறைகள் மற்றும் மாதிரி மதிப்பீடு தொழில்நுட்பங்களில் ஒத்துமையை பின்பற்றுகின்றன. எனினும், முக்கியமான சில காரணி இவ்விரு மாதிரிகளுக்கு வேறுபாடு ஏற்படுத்துகின்றன.

## குழந்தை மொழி மாதிரிகள் பயன்பாடுகள்

SLM கள் பலவகை பயன்பாடுகளை கொண்டுள்ளன, உதாரணமாக:

- சந்தை சந்தாதாரர்கள்: வாடிக்கையாளர் ஆதரவு வழங்குதல் மற்றும் உரையாடல் முறையில் பயனர்களுடன் தொடர்பு கொள்ளுதல்.
- உள்ளடக்க உருவாக்கம்: எழுத்தாளர்களைக் கையாள உதவுதல், ஒருமட்டும் கட்டுரைகள் வரை உருவாக்குதல்.
- கல்வி: மாணவர்களுக்கு எழுத்து பணிகளிலும் புதிய மொழிகள் கற்றலிலும் உதவுதல்.
- அணுகல் நேர்முகம்: மாற்றுத்திறனாளிகளுக்கான கருவிகள் உருவாக்குதல், உதாரணத்துக்காக உரையைக் கற்று பேசும் அமைப்புகள்.

**அளவு**
  
LLM மற்றும் SLM இடையேயான முதன்மை வேறுபாடு மாதிரிகளின் அட்டவணை அளவிலுள்ளது. ChatGPT (GPT-4) போன்ற LLM கள் சுமார் 1.76 டிரில்லியன் பரிமாணங்களைக் கொண்டிருக்கலாம், ஆனால் திறந்த மூல SLM கள் போன்ற Mistral 7B மாதிரி சுமார் 7 பில்லியன் பரிமாணங்களை கொண்டுள்ளன. இந்த வேறுபாடு மாதிரி கட்டமைப்பு மற்றும் பயிற்சி செயல்முறைகளில் இருந்து வருகின்றது. உதாரணமாக, ChatGPT சுய கவனிப்பு இயந்திரத்தை இணைக்கும் எங்கோடர்-டிகோடர் வடிவமைப்பில் பயன்படுத்தினால், Mistral 7B ஸ்லைடிங் விண்டோ கவனிப்பை பயன்படுத்தி டிகோடர் மட்டும் மாதிரியில் திறமையான பயிற்சியை அனுமதிக்கிறது. இவ்வித கட்டமைப்பு வேறுபாடு இந்த மாதிரிகளின் குழப்பத்திலும் செயல்திறனிலும் மிகப்பெரிய தாக்கத்தை ஏற்படுத்துகிறது.

**புரிதல்**

SLM கள் குறிப்பிட்ட துறைகளில் செயல்திறனை மேம்படுத்த நீரோட்டப்படுத்தப்படுகின்றன, அதனால் அவை மிகவும் நிபுணத்துவம் வாய்ந்தவை ஆகின்றன, ஆனால் பல துறைகளின் பரவலான புரிதலை வழங்குவதில் சில கோரிக்கைகள் இருக்கலாம். அதற்கு மாறாக, LLM கள் மனித போல் அறிவை அதிகமேம்படுத்துவதற்கு நோக்கக்கோள் ஆகும். பரவலான, வேறுபடும் தரவுத்தொகுப்புகள் பயிற்சியில், LLM கள் அதிக துறைகளில் சிறப்பாக செயல்படும்படி வடிவமைக்கப்பட்டுள்ளன, மேலும் அவை அதிக நெருக்கியதையும் தனிமைப்படுத்தலையும் அளிக்கின்றன. எனவே, LLM கள் இயற்கை மொழி செயலாக்கம் மற்றும் நிரல் எழுதுதல் போன்ற பல பணிகளுக்கு அதிக பொருத்தமானவை.

**கணினி சக்தி**

LLM கள் பயிற்சி மற்றும் செயல்படுத்துதல் கணினி வளங்களை அதிகமாகக் கேட்கும் செயல்முறைகள், பெரும்பாலும் பெரிய அளவிலான GPU குழுக்களை பயன்படுத்த வேண்டும். உதாரணமாக, ChatGPT போன்ற மாதிரியை முழுதாய் தொடக்கம் இருந்து பயிற்சி செய்ய குறைந்தது ஆயிரக்கணக்கான GPU கள் நீண்ட காலம் தேவையானதாகலாம். மாற்றாக, SLM கள் குறைந்த பரிமாணங்களைக் கொண்டதால், கணினி வளங்களுக்கு ஒப்பிடுகையில் எளிதில் கிடைக்கக்கூடியவை. Mistral 7B மாதிரி போன்றவை, நடுத்தர GPU திறன்கள் கொண்ட உள்ளூர் கணினிகளில் பயிற்சி செய்யவும் இயங்கவும் முடியும், ஆனால் பயிற்சி இன்னும் பல கணினிகளுக்கு பல மணி நேரம் தேவைபடலாம்.

**பாய்ஸ்**

பாய்ஸ் LLM களில் தெரிந்த ஒரு பிரச்சினை ஆகும், இதற்கான காரணம் பயிற்சி தரவின் இயல்பு. இம்மாதிரிகள் பொதுவாக இணையத்தில் திறந்தவெளியிலிருந்து கிடைக்கும் خام தரவை சார்ந்திருப்பதால், சில குழுக்களை குறைவு அல்லது தவறான முறையில் பிரதிபலிக்கலாம், தவறான குறிச்சொற்களை உட்படுத்தலாம் அல்லது மொழி சார்ந்த பாய்ஸ்கள், பகுதிச் சூழல் மாறுதல்கள் மற்றும் இலக்கணச்சீர்முதல்களை பிரதிபலிக்கக்கூடும். கூடுதலாக, LLM கட்டமைப்புகளின் குழப்பம் பாய்ஸை இன்னும் அதிகப்படுத்தும், கவனமாக நுட்பமுடுக்கத்தின் இல்லாமையில் இது வெளிப்படாமலும் இருக்கலாம். அதற்கு மாறாக, SLM கள் தரவுத் தொகுப்புகளுக்கு மிகவும் கட்டுப்பட்ட, துறை-சார்ந்த தரவுகளைப் பயிற்சி செய்யப்படுவதால், இவ்வித பாய்ஸ்களுக்கு குறைவாக பாதிக்கப்படுகின்றன, ஆனால் அவை முழுமையாக விடுவிக்கப்படவில்லை.

**இன்ஃபெரன்ஸ்**

SLM களின் குறைந்த அளவு அவற்றுக்கு இன்ஃபெரன்ஸ் வேகத்தில் முக்கிய பலன் அளிக்கிறது, அவை உள்ளூர் உபகரணங்களில் விரைவாக வெளியீடுகளை உருவாக்க முடியும், மிகப்பெரிய இணை செயலாக்கத்திறன் வேண்டாம். அதனால், LLM கள் பெரும் அளவில் மற்றும் குழப்பமானருக்கும், ஏற்ற சிறந்த இன்ஃபெரன்ஸ் நேரத்திற்கு பெரும் இணை கணினி வளங்கள் தேவை. பல பயனர்கள் ஒரே நேரத்தில் பயன்படுத்தும் போது, LLM கள் பதில் நேரம் மேலும் குறையும், குறிப்பாக பெரிய அளவில் பயன்படுத்தப்படும் போது.

சுருக்கமாக, LLM மற்றும் SLM இரண்டும் இயந்திரக் கற்றலின் அடிப்படையில் இருப்பினும், மாதிரி அளவு, வள தேவைகள், சூழல் புரிதல், பாய்ஸ் தாக்கம், மற்றும் இன்ஃபெரன்ஸ் வேகம் ஆகியவற்றில் மிகப்பெரிய வேறுபாடுகள் உள்ளன. இவ்வாறான வேறுபாடுகள் அவற்றின் முறையான பயன்பாடு நோக்கங்களைக் குறிக்கின்றன, LLM கள் பல்துறைகளில் பொருத்தமானவை, ஆனால் வளம் அதிகமாக தேவைப்படுகின்றன, SLM கள் குறிப்பிட்ட துறைகள் மற்றும் குறைந்த கணினி தேவைகளுக்கு மேலான திறமையை வழங்குகின்றன.

***குறிப்பு: இந்த பாடத்தில், நாம் Microsoft Phi-3 / 3.5 ஐ உதாரணமாகக் கொண்டு SLM ஐ அறிமுகப்படுத்துவோம்.***

## Phi-3 / Phi-3.5 குடும்பத்தை அறிமுகப்படுத்துக

Phi-3 / 3.5 குடும்பம் பிரதானமாக உரை, பார்வை மற்றும் முகவர் (MoE) பயன்பாட்டு சூழல்களை குறிக்கிறது:

### Phi-3 / 3.5 வழிகாட்டுதல்

முக்கியமாக உரை உருவாக்கம், உரையாடல் முடித்தல், மற்றும் உள்ளடக்க தகவல் சுருக்குதல் போன்றவைக்கு.

**Phi-3-mini**

இந்த 3.8B மொழி மாதிரி Microsoft Foundry, Hugging Face மற்றும் Ollama இல் கிடைக்கின்றது. Phi-3 மாதிரிகள் சம அளவு மற்றும் பெரிய மொழி மாதிரிகளுக்கு முக்கிய அடிக்கடி வலைப்பூ சோதனைகளில் (benchmark) முறையீடு செய்தும் வெற்றி பெற்றுள்ளன (கீழே உள்ள benchmark எண்களை பார்க்கவும், அதிக எண்கள் சிறந்தவை). Phi-3-mini தனது அளவின் இரட்டை அளவு மாதிரிகளைவிட மேல் செயல்படுகிறது, மேலும் Phi-3-small மற்றும் Phi-3-medium பெரிய மாதிரிகளையும் உட்பட மிகச்சிறந்த செயல்திறனை வெளிப்படுத்துகின்றன, GPT-3.5 உட்பட.

**Phi-3-small மற்றும் medium**

Phi-3-small 7B பரிமாணங்களுடன் பல மொழிப் பணிகள், காரணம் முறை, குறியீடு மற்றும் கணித அடிப்படையில் GPT-3.5T ஐத் தள்ளுகிறது.

14B பரிமாணங்களுடைய Phi-3-medium இதை தொடர்கிறது மற்றும் Gemini 1.0 Pro ஐ வென்று வெளிக்கின்றது.

**Phi-3.5-mini**

இதை Phi-3-mini யின் மேம்பாடாகப் பார்க்கலாம். பரிமாணங்கள் மாறாத போதும், பல மொழிகளுக்கு ஆதரவு (20+ மொழிகள்: அரபிகு, சீன, செக், டேனிஷ், டச்சு, ஆங்கிலம், பினிஷ், பிரஞ்சு, ஜெர்மன், இப்ரேல், ஹங்கேரியன், இத்தாலிய, ஜப்பானிய, கொரியன், நோர்வே, போலிஷ், போர்ச்சுகீஸ், ரஷியன், ஸ்பானிஷ், சொய்டிஷ், தாய், டுர்க்கிஷ், உக்ரைனியன்) ஐ மேம்படுத்தி, நீண்ட சூழல் ஆதரவையும் கூட்டுகிறது.

3.8B பரிமாண Phi-3.5-mini அதே அளவுள்ள மற்ற மொழி மாதிரிகளுக்கு மேல் செயல்படுகிறது மற்றும் இரட்டை அளவுள்ள மாதிரிகளுடன் சமம்.

### Phi-3 / 3.5 பார்வை

Phi-3/3.5 இன் வழிகாட்டும் மாதிரியை Phi இலுள்ள புரிதலாகக் கருதலாம், பார்வை என்பது Phi க்கு உலகத்தைப் புரிந்துகொள்ளக் கண் கொடுக்கும்.


**Phi-3-பார்வை**

Phi-3-பார்வை, 4.2B பரிமாணங்களுடன், இந்த தொடரைக் கொண்டு கிளோடு-3 ஹாய்கூ மற்றும் Gemini 1.0 Pro V போன்ற பெரிய மாதிரிகளை மும்புவழிப்படுத்தும் பொதுவான காட்சிச் காரணம், OCR மற்றும் அட்டவணை மற்றும் விளக்கப்பட விவரிப்பு பணிகளில் சிறப்பாக செயல்படுகிறது.


**Phi-3.5-பார்வை**

Phi-3.5-பார்வை Phi-3-பார்வையின் மேம்பாடாகவும், பல படங்களுக்கு ஆதரவு சேர்க்கிறது. இது பார்வை மேம்பாட்டாகக் கருதலாம், நீங்கள் படங்களை மட்டும் அல்லாமல் வீடியோக்களையும் பார்க்க முடியும்.

Phi-3.5-பார்வை Claude-3.5 சோனெட் மற்றும் Gemini 1.5 ஃபிளாஷ் போன்ற பெரிய மாதிரிகளுக்கு OCR, அட்டவணை மற்றும் வரைவுப் புரிதல் பணிகளில் முன்னிலை வகிக்கிறது. பொதுவான காட்சி அறிவு காரண பணி மரியாதையாக உள்ளது. பல பட உள்ளீடு ஆதரவை (multi-frame input support) வழங்குகிறது, அதாவது பல படங்களில் காரணம் செய்யக்கூடும்.


### Phi-3.5-MoE

***கல்லூரியின் கலவை (Mixture of Experts - MoE)*** மாதிரிகள் மிகவும் குறைந்த கணினி வளங்கள் கொண்டு முன் பயிற்சி செய்யப்பட முடியும், இது அதே கணினி பட்ஜெட்டில் மாதிரியின் அல்லது தரவுத்தொகுப்பின் அளவை மிகப்பெரிதாக அதிகரிக்க அனுமதிக்கிறது. குறிப்பாக, MoE மாதிரி அதன் கன்டென்ஸ் மாதிரியின் சம அளவுள்ள தரத்தைக் குறைந்த காலத்தில் அடைகிறது.

Phi-3.5-MoE 16x3.8B நிபுண மாதிரிகளைக் கொண்டது. 6.6B செயலில் உள்ள பரிமாணங்களோடு Phi-3.5-MoE அதிக பெரிய மாதிரிகளுக்கு சமமான காரணம், மொழி புரிதல் மற்றும் கணித திறனை அடையிறது.

Phi-3/3.5 குடும்பத்தை வெவ்வேறு சூழல்களைப் பொறுத்து பயன்படுத்தலாம். LLM போல அல்லாததால், Phi-3/3.5-mini அல்லது Phi-3/3.5-பார்வை இன் பயன்பாடு எட்ஜ் சாதனங்களில் முடியும்.


## Phi-3/3.5 குடும்ப மாதிரிகளை எப்படி பயன்படுத்துவது

Phi-3/3.5 ஐ வெவ்வேறு சூழல்களில் பயன்படுத்தத் திட்டமிட்டுள்ளோம். அடுத்ததாக, நாம் Phi-3/3.5 ஐ வெவ்வேறு சூழல்பிரிவுகளில் பயன்படுத்துவோம்.

NVIDIA NIM (NVIDIA Inference Microservices) என்பது பல்வேறு சூழல்களில், மேகம், தரவு மையங்கள் மற்றும் வேலைநிறுத்திகள் போன்ற இடங்களில் உருவாக்குநர்களுக்கு AI மாதிரிகளை திறமையாக பயன்படுத்த உதவும் வேகமான இன்ஃபெரன்ஸ் மைக்ரோசேவை சேவைகள் தொகுப்பு ஆகும்.

NVIDIA NIM இன் முக்கிய அம்சங்கள் சில:

- **பதிவேற்றுதல் சுலபம்:** NIM எந்த AI மாதிரிகளையும் ஒரே கட்டளையுடன் பதிவேற்ற அனுமதிக்கிறது, இதனால் தற்போதைய பணிச்சுழற்சிகளில் எளிதில் இணைக்க முடியும்.

- **சம்பந்தப்பட்ட செயல்திறன்:** NVIDIA இன் முன்கூட்டியே மேம்படுத்தப்பட்ட நிரூபண இயந்திரங்கள், TensorRT மற்றும் TensorRT-LLM போன்றவை பயன்படுத்தப்பட்டு குறைந்த தாமதம் மற்றும் உயர் திறனுடன் செயல்படுகிறது.
- **வெளிப்படை திறன்:** NIM குபெர்நிடீஸ் மேல் தானாக அளவுகோல்கூறும் செயல்பாட்டை ஆதரிக்கிறது, இது மாறும் பணிகளைக் திறம்பட கையாள உதவுகிறது.
- **பாதுகாப்பு மற்றும் கட்டுப்பாடு:** நிறுவனங்கள் தங்கள் தரவுகள் மற்றும் பயன்பாடுகளுக்கு முழு கட்டுப்பாட்டை பெற்றுக்கொள்ள NIM மைக்ரோசேவைகளை தன் சொந்த நிர்வகிக்கப்படும் உட்சாத்திரத்தில் இயக்கலாம்.
- **நிலையான API கள்:** NIM தொழில்துறைக்கேற்ற API களை வழங்குகிறது, இது பேசுவர்களை, AI உதவிகள் மற்றும் பிற AI செயலிகளை எளிதில் உருவாக்க மற்றும் ஒருங்கிணைக்க உதவுகிறது.

NIM என்பது NVIDIA AI நிறுவனத்தின் ஒரு பகுதியாகும், இது AI மாதிரிகள் செயல்படுத்தலையும் பணியாற்றுதலையும் எளிதாக்குவதற்கு நோக்கமாக கொள்ளப்பட்டுள்ளது, NVIDIA GPU களில் அவை திறம்பட ஓடுவதை உறுதிப்படுத்துகிறது.

**Hugging Face Transformer**
Hugging Face Transformers என்பது இயற்கை மொழி செயலாக்கம் (NLP) மற்றும் பிற இயந்திரக் கற்றல் பணிகளுக்கான சக்திவாய்ந்த நூலகமாகும். இதைப் பற்றிய முக்கிய அம்சங்கள்:

1. **முன்னேற்பயிற்சியமைக்கப்பட்ட மாதிரிகள்**: உரை வகைப்படுத்தல், பெயர் அறிமுகம், கேள்விக்குப் பதில், சுருக்கம், மொழிபெயர்ப்பு மற்றும் உரை உருவாக்கம் போன்ற பல பணிகளுக்கு ஆயிரக்கணக்கான முன்னேற்பயிற்சியடைந்த மாதிரிகள் கிடைக்கின்றன.

2. **மேடை இடைமுகத் தரம்:** PyTorch, TensorFlow மற்றும் JAX போன்ற பல ஆழ்ந்த கற்றல் மேடைகள் ஆதரிக்கப்படுகின்றன. ஒரே மாதிரியை ஒரே மேடையில் பயிற்றுவித்து, இன்னொரு மேடையில் பயன்படுத்த முடியும்.

3. **பலவழி திறன்கள்:** NLP தவிர Hugging Face Transformers கணினி பார்வை (பட வகைப்படுத்தல், பொருள் கண்டறிதல்) மற்றும் ஆடியோ செயலாக்க (சொலி அங்கீகாரம், ஆடியோ வகைப்படுத்தல்) பணிகளையும் ஆதரிக்கிறது.

4. **பயன்பாட்டில் எளிமை:** மாதிரிகளை எளிதாக பதிவிறக்கம் செய்து நுணுக்கமாக சீரமைக்க API களையும் கருவிகளையும் வழங்குகிறது, இது தொடக்க நிலை மற்றும் நிபுணர்களுக்கும் அணுகுமுறை.

5. **சமூகமும் வளமும்:** Hugging Face ஒரு பரபரப்பான சமூகத்தைக் கொண்டுள்ளது, விரிவான ஆவணங்கள், பயிற்சிக் குறிப்புகள் மற்றும் வழிகாட்டுதல்களை வழங்குகிறது.

**ONNX Runtime for GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) என்பது ஒரு பல மேடைகளிலும் இயங்கக்கூடிய inference மற்றும் பயிற்சி இயந்திரக் கற்றல் அதிவேகப்படுத்தி ஆகும். Generative AI (GENAI) க்கான ONNX Runtime ஒரு சக்திவாய்ந்த கருவி, இது பல மேடைகளில் உருவாக்கும் AI மாதிரிகளை திறம்பட இயக்க உதவுகிறது.

## ONNX Runtime என்றால் என்ன?
ONNX Runtime என்பது இயந்திரக் கற்றல் மாதிரிகளுக்கான உயர் செயல்திறன் inference ஐ இயலுமைப்படுத்தும் திறந்த மூல திட்டம் ஆகும். இது Open Neural Network Exchange (ONNX) வடிவத்தில் உள்ள மாதிரிகளைக் ஆதரிக்கிறது, இது இயந்திரக் கற்றல் மாதிரிகளை பிரதிநிதித்துவப்படுத்தும் ஒரு நிலையாகும்.ONNX Runtime வாடிக்கையாளர்களுக்கு விரைவான அனுபவங்களையும் குறைந்த செலவுகளையும் வழங்குகிறது, PyTorch மற்றும் TensorFlow/Keras போன்ற ஆழ்ந்த கற்றல் மேடைகளுடன் கூடிய மாதிரிகள் மற்றும் scikit-learn, LightGBM, XGBoost போன்ற பாரம்பரிய இயந்திரக் கற்றல் நூலகங்களையும் ஆதரிக்கிறது. ONNX Runtime பலவகை இருதரப்பு இயந்திரங்கள், இயக்கிகள் மற்றும் இயக்க முறைமைகளுக்கு இணக்கமாக இருக்கிறது மேலும், தேவையான இடங்களில் ஹார்ட்வேர்த் அதிவேகப்படுத்திகளை பயன்படுத்தி, கிராப் மேம்பாடுகளையும் மாற்றங்களையும் ஒருங்கிணைத்து சிறந்த செயல்திறனை வழங்குகிறது.

## Generative AI என்றால் என்ன?
உருவாக்கும் AI என்பது பயிற்சியளிக்கப்பட்ட தரவுகளின் அடிப்படையில் புதிய உள்ளடக்கங்களை உருவாக்கக்கூடிய AI அமைப்புகள் ஆகும், உதாரணமாக உரை, படங்கள் அல்லது இசை உருவாக்குதல். எடுத்துக்காட்டுகள்: GPT-3 போன்ற மொழி மாதிரிகள் மற்றும் Stable Diffusion போன்ற பட உருவாக்கும் மாதிரிகள். ONNX Runtime for GenAI நூலகம் ONNX மாதிரிகளுக்கு generative AI வளியாக செயல்படும், இதில் ONNX Runtime மூலம் inference, logits சிகிச்சை, தேடல் மற்றும் மாதிரித்திறன் எடுத்துக்கொள்ளுதல் மற்றும் KV cache மேலாண்மை அடங்கும்.

## ONNX Runtime for GENAI
ONNX Runtime for GENAI ONNX Runtime இன் திறன்களை உருவாக்கும் AI மாதிரிகளுக்கு விரிவாக்கி வழங்குகிறது. முக்கிய அம்சங்கள்:

- **வெளிப்பட்ட மேடை ஆதரவு:** Windows, Linux, macOS, Android மற்றும் iOS உட்பட பல மேடைகளில் இயங்குகிறது.
- **மாதிரி ஆதரவு:** LLaMA, GPT-Neo, BLOOM மற்றும் பல பிரபலமான உருவாக்கும் AI மாதிரிகளை ஆதரிக்கிறது.
- **செயல்திறன் மேம்பாடு:** NVIDIA GPU கள், AMD GPU கள் மற்றும் பிற ஹார்ட்வேர்த் அதிவேகப்படுத்திகளுக்கான மேம்பாடுகளை கொண்டுள்ளது.
- **பயன்பாட்டுக்கு எளிது:** குறைந்த குறியீடுடன் உரை, படங்கள் மற்றும் பிற உள்ளடக்கங்களை உருவாக்க API களை வழங்குகிறது.
- பயனர்கள் generate() என்ற உயர்தரம் முறைமையை அழைக்கலாம், அல்லது மாதிரியின் ஒவ்வொரு திரும்பும் சுற்றிலும் மறுபடி இயக்கி, ஒவ்வொரு முறையும் ஒரு சொற்களை உருவாக்கி, விருப்பத்தேர்வு படி உருவாக்கு அளவுருக்களை புதுப்பிக்கவும் முடியும்.
- ONNX runtime வெற்றிகர / கதிர் தேடல் மற்றும் TopP, TopK மாதிரித்திறன் மூலம் சொற்தொடர்களை உருவாக்குவதற்கும் உள்ளடக்க தொகுப்பு வழங்கும் logits சிகிச்சைகளையும் உட்பட மீண்டும் மீண்டும் வரிசைகள் போன்றவற்றிற்கான தண்டனைகளையும் ஆதரிக்கிறது. இவ்வாறே தனிப்பயன் மதிப்பிடுதல்களை எளிதில் சேர்க்கலாம்.

## துவக்கம்
ONNX Runtime for GENAI உடன் துவங்க, பின்வரும் படிகளை பின்பற்றலாம்:

### ONNX Runtimeஐ நிறுவுக:
```Python
pip install onnxruntime
```
### உருவாக்கும் AI நீட்டிப்புகளை நிறுவுக:
```Python
pip install onnxruntime-genai
```

### மாதிரியை இயக்குக: Python இல் ஒரு எளிய எடுத்துக்காட்டு இதோ:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### டெமோ: ONNX Runtime GenAI ஐ பயன்படுத்தி Phi-3.5-Vision ஐ அழைவேண்டும்


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
