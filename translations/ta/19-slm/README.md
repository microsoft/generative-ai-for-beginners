# தொடக்கத்திற்கான சிறிய மொழி மாதிரிகள் குறித்து அறிமுகம் - உருவாக்கும் ஏஐ beginners காக
உருவாக்கும் ஏஐ என்பது புதிய உள்ளடக்கங்களை உருவாக்கும் திறன் கொண்ட அமைப்புகளை உருவாக்குவதில் கவனம் செலுத்தும் கலை சோஃப்ட்வேர் புலம் ஆகும். இந்த உள்ளடக்கம் உரை மற்றும் படங்கள் முதல் இசை மற்றும் முழுமையான மெய்நிகர் சூழல்கள் வரை பரவலாம். உருவாக்கும் ஏஐயின் மிக ரசகரமான பயன்பாடுகளில் ஒன்று மொழி மாதிரிகள் ஆகும்.

## சிறிய மொழி மாதிரிகள் என்றால் என்ன?

ஒரு சிறிய மொழி மாதிரி (SLM) என்பது பெரிய மொழி மாதிரியின் (LLM) குறைக்கப்பட்ட வடிவமாகும், LLMகளின் நிர்மாணக் கொள்கைகள் மற்றும் நுட்பங்களை பயன்படுத்தி, கணக்கீட்டு பரிசோதனையின் அளவை மிகவும் குறைத்துள்ளது.

SLMகள் மனிதன் போன்று உரையாற்றும் உரையை உருவாக்க வடிவமைக்கப்பட்டுள்ள மொழி மாதிரிகளின் துணையினத்தில் உள்ளன. அவை GPT-4 போன்ற பெரிய மாதிரிகளுக்கு மாறாக, சிறிய மற்றும் திறமையானவை, கணினி வளங்கள் குறைந்த சமயங்களில் பயன்படுத்துவதற்குத் தகுந்தவைகள். அதன் சிறிய அளவினிடமும் பல செயல்களை செய்யக்கூடியவை. பெரும்பாலும், SLMகள் LLMகளை சுருக்கி அல்லது சுரண்டி உருவாக்கப்படுகின்றன, அசல் மாதிரியின் செயல்திறன் மற்றும் மொழி திறனைப் பெரிதும் காப்பாற்ற முனைவர். இந்த மாதிரி அளவு குறைக்கப்படுவதால் பொதுவாக சிக்கலை குறைத்து, நினைவக பயன்பாட்டிலும் கணக்கீட்டு தேவைகளிலும் திறமையானதாக இருக்கிறது. இந்த ஒழுங்குமுறை மற்றும் சிறப்பான செயல்திறனை வைத்திருந்தாலும், SLMகள் பல்வேறு இயற்கை மொழி செயலாக்க (NLP) பணிகளை still செய்ய முடியும்:

- உரை உருவாக்கம்: சம்பந்தப்பட்ட மற்றும் பொருத்தமான வாக்கியங்கள் அல்லது பத்திகளைக் கட்டவிழ்க்கிறது.
- உரை பூர்த்தி: உரைந்த ஒரு தொடக்க வார்த்தையை அடிப்படையாக கொண்டு சாத்தியமான உரையை முன்னறிவிக்கும்.
- மொழிபெயர்ப்பு: ஒரு மொழியிலிருந்து மற்றொரு மொழிக்கு உரையை மாற்றுதல்.
- சுருக்கம்: நீண்ட உரைகளை சுருக்கி, சுலபமாக உணரக்கூடிய சுருக்கங்களை உருவாக்குதல்.

பெரிய மாதிரிகளுடன் ஒப்பிடுகையில் செயல்திறன் அல்லது ஆழமான புரிதலில் சில வரம்புகளுடன்.

## சிறிய மொழி மாதிரிகள் எப்படி செயல்படுகின்றன?
SLMகள் மிக அதிக உரை தரவுகளில் பயிற்சி பெறுகின்றன. பயிற்சி காலத்தில் அவை மொழியின் முறை மற்றும் கட்டமைப்புகளை கற்றுக்கொள்கின்றன, இதனால், நியமகரமான மற்றும் தொடர்புடைய உரைகளை உருவாக்கக்கூடியவை ஆகின்றன. பயிற்சி செயல்முறை:

- தரவு சேகரிப்பு: பல்வேறு ஆதாரங்களில் இருந்து பெரிய அளவு உரை தரவுகளை சேகரித்தல்.
- முன்னமைவு: தரவை சுத்தமாக்கி மற்றும் ஒழுங்கமைத்து பயிற்சிக்கு உடையேற்றுக் கொள்வது.
- பயிற்சி: மெஷின் லெர்னிங் அல்காரிதம்களைப் பயன்படுத்தி மாதிரிக்கு உரை புரிந்து கொண்டு உருவாக்க கற்றல்.
- நுட்ப தன்மை கூர்மை: குறிப்பிட்ட பணிகளில் செயல்திறனை மேம்படுத்த மாதிரியை சரிசெய்தல்.

SLMகளின் உருவாகுதல் வளங்கள் குறைந்த சூழலான பிளாட்ஃபாம்களில் (மொபைல் சாதனங்கள் அல்லது எட்ஜ் கணினி) பயன்படுத்த சாத்தியமாகும் மாதிரிகளை தேவையாகியதற்கு ஏற்ப பொருந்துகிறது. திறமைக்கு முக்கியத்துவம் கொடுத்து SLMகள் தொழில்நுட்பத்தில் திறமையுடன் செயல்படும் வகையில் வடிவமைக்கப்பட்டுள்ளன, இவை பல துறைகளிலும் பரவலான பயன்பாட்டிற்கு அனுமதி தருகின்றன.

## கற்றல் நோக்கங்கள்

இந்த பாடத்தில், நாம SLM பற்றி அறிந்து அதை Microsoft Phi-3 உடன் இணைத்து உரை உள்ளடக்கம், காட்சி மற்றும் MoE இல் பல்வேறு சூழ்நிலைகளில் கற்றுக்கொள்ள முயற்சிப்போம்.

இந்த பாடத்திற்குப் பிறகு நீங்கள் பின்வரும் கேள்விகளுக்கு பதிலளிக்க முடிந்திருக்க வேண்டும்:

- SLM என்றால் என்ன?
- SLM மற்றும் LLM இல் என்ன வேறுபாடு உள்ளது?
- Microsoft Phi-3/3.5 குடும்பம் என்றால் என்ன?
- Microsoft Phi-3/3.5 குடும்பத்துடன் எப்படி inference இயக்குவது?

தயாரா? தொடங்குவோம்.

## பெரிய மொழி மாதிரிகள்(LLMs) மற்றும் சிறிய மொழி மாதிரிகள்(SLMs) இடையேயான வேறுபாடுகள்

LLMகும் SLMகளும் சாத்திய முன்னோக்கி இயந்திர கற்றல் அடிப்படைக் கொள்கைகளையே அங்கீகாரம் செய்து, நிர்மாண வடிவமைப்பிலும் பயிற்சி முறைகளிலும், தரவு உருவாக்க செயல்முறைகளிலும், மாதிரி மதிப்பீட்டிலும் ஒரே மாதிரி அணுகுமுறைகளை பின்பற்றுகின்றன. இருப்பினும், சில முக்கிய அம்சங்கள் இரண்டு மாதிரிகளை வேறுபடுத்துகின்றன.

## சிறிய மொழி மாதிரிகளின் பயன்பாடுகள்

SLMகளின் பல்வேறு பயன்பாடுகள் உள்ளன, அவை:

- பேச்சு கருவிகள்: வாடிக்கையாளர் ஆதரவுக்கு உரையாடலில் ஈடுபடுதல்.
- உள்ளடக்கம் உருவாக்கல்: எழுத்தாளர்களுக்கு கருத்துக்கள் வழங்குதல் அல்லது முழு கட்டுரைகளை வடிவமைத்தல்.
- கல்வி: மாணவர்களுக்கு எழுத்து பணிகளிலும் புதிய மொழிகள் கற்றலிலும் உதவி.
- அணுகல் திறன்:, எனது ஊடாடல் போன்ற ஆதரவு கருவிகள் உருவாக்குதல்.

**அளவு**
  
LLMகளுக்கும் SLMகளுக்கும் இடையேயான முக்கிய வேறுபாடு மாதிரியின் அளவிலேயே உள்ளது. ChatGPT (GPT-4) போன்ற LLMகள் 1.76 ட்ரில்லியன் பரிமாணங்களைக் கொண்டிருக்கலாம், Mistral 7B போன்ற திறந்த மூல SLMகளில் சுமார் 7 பில்லியன் பரிமாணங்கள் மட்டுமே உள்ளன. இந்த வேறுபாடு மாதிரி கட்டமைப்பு மற்றும் பயிற்சி முறைகளில் உள்ள வேறுபாட்டினால் ஏற்படுகிறது. உதாரணத்திற்கு, ChatGPT ஒரு self-attention செயலியை encoder-decoder வடிவத்தில் பயன்படுத்துவதில், Mistral 7B sliding window attention ஐ decoder-ஓரே மாதிரியில் திறமையாக பயிற்சிக்க பயன்படுத்துகிறது. இந்த கட்டமைப்பு வேறுபாடு மாதிரிகளின் சிக்கல் மற்றும் செயல்திறனுக்கு பெரும் விளைவைக் கொண்டுள்ளது.

**புரிதல்**

SLMகள் சாதாரணமாக குறிப்பிட்ட துறைகளைச் சமாளிப்பதற்காக செயல்திறனில் மேம்படுத்தப்படுகின்றன, ஆகவே அவை மிகவும் சிறப்பாக இருக்கின்றன, ஆனால் பல துறைகளை உள்ளடக்கிய பரந்த புரிதலை வழங்குவதில் குறைப்பாக இருக்கலாம். அதே நேரத்தில், LLMகள் மனித உணர்வை போன்ற அறிவாற்றலை விரிவாகக் காட்சி அளிக்க உருவாக்கப்படுகின்றன. பெரிய மற்றும் வெவ்வேறு தரவு தொகுப்புகளில் பயிற்சி பெற்ற LLMகள் பல துறைகளில் சிறந்த செயல்திறனைக் காட்டுகின்றன, மேலும் இவை அதிக துறைகளில் பயன்படுத்தக்கூடிய மற்றும் தகுதியானதாக உள்ளன. அதனால், LLMகள் இயற்கை மொழி செயலாக்கம் மற்றும் நிரலாக்கம் போன்றவற்றிற்கு ஏற்றவையாக இருக்கின்றன.

**கணினி வளங்கள்**

LLMகளின் பயிற்சி மற்றும் பயன்பாடு வளம் அதிகம் தேவைப்படுகின்றன, பெரும் கணினி வளங்கள் வேண்டும் என்பது பொதுவானது, பெரும்பாலும் பெரிய GPU குழுக்களைச் சேர்த்து இயக்க வேண்டியிருக்கும். உதாரணமாக, ChatGPT எனும் மாதிரியை ஆரம்பத்திலிருந்து உருவாக்க, ஆயிரக்கணக்கான GPUகளில் நீண்ட காலமாக பயிற்சி செய்யவேண்டும். அதற்கு மாறாக, SLMகள் சிறிய பரிமாணங்களைக் கொண்டதால் கணினி வள சேமிப்பில் வசதியானவை. Mistral 7B மாதிரி போன்றவை மிதமான GPU உள்ளடக்கிய உள்ளூர் கணினிகளில் பயிற்சி மற்றும் ஓட்டம் செய்யக்கூடியவை, பயிற்சி பல மணி நேரம் மற்றும் பல GPUகளில் நிகழ்ந்து இருப்பினும்.

**பாகுபாடு**

பாகுபாடு என்பது LLMகளில் இசைவான பிரச்சனை, அது பயிற்சி தரவின் இயற்கையால் உருவாகிறது. இவ்வாறு மாதிரிகள் பொதுவாக இணையத்திலிருந்து கிடைக்கும் குறைந்த பிராமாணிக தரவு மீது பொறுத்திருக்கின்றன, இது சில குழுக்களின் குறைவான பிரதிநிதித்துவம் அல்லது தவறான குறிக்கோள்களை உள்ளடக்கக்கூடும், மொழி மாறுபாடுகள் மற்றும் இலக்கண விதிகளால் பாகுபாடுகளை உருவாக்கக்கூடும். மேலும, LLMகளின் சிக்கலான கட்டமைப்புகள் பாகுபாட்டை வலுப்படுத்தும் வாய்ப்பு உண்டு, இது கவனமாக நுட்பமாக சரிசெய்யப்படாமல் இருந்தால் தெரியாது. மாறாக, SLMகள் கட்டுப்படுத்தப்பட்ட மற்றும் துறையை குறிப்பிட்ட தரவுகளில் பயிற்சிபெற்றதால் இவை இதுபோன்ற பாகுபாட்டுக்கு குறைவாக உட்படுகின்றன, ஆனால் இவை முழுமையாக பாகுபாடு இல்லாதவைகள் அல்ல.

**முடிவெடுப்பு**

SLMகளின் குறைந்த அளவு அவற்றிற்கான முக்கியமான முன்னேற்றத்தை வழங்குகின்றது, inference வேகத்தில் பெரிய தாக்கம். இவை உள்ளூரில் கிடைக்கும் கருவிகளில் அதிநிகர்வு தேவையின்றி விரைவில் முடிவுகளை தர முடியும். அதன்போல LLMகள் பெரிய பரிமாணம் மற்றும் சிக்கலின் காரணமாக நிறைய கூடுதல் ஒரே நேரத்தில் செயலாக்க வளங்கள் தேவையாகும். ஒரே நேரத்தில் பல பயனர்கள் பயன்படுத்தும் போது LLMகளின் பதில் நேரம் மேலும் மெதுவாகும்.

சுருக்கமாகச் சொல்லும்போது, LLMகளும் SLMகளும் மெஷின் லெர்னிங் அடிப்படையில் இருக்கும் போதிலும், மாதிரி அளவு, வளப்படுத்தல் தேவைகள், உரையாழி புரிதல், பாகுபாடு மற்றும் முடிவெடுப்பு வேகத்தில் பெரும் வேறுபாடுகள் உள்ளன. இந்த வேறுபாடுகள் அவற்றின் பயன்பாட்டு துறைக்கு ஏற்ப LLMகளை அதிக பன்முகத்தன்மையுடன் தரவுகள் மற்றும் வளங்களை அதிகமாக தேவைப்படும் விதத்தில் மேலும் SLMகள் துறையின் தேவைகளுக்கு ஏற்ப குறைந்த கணினி வளத்தில் திறமையாக செயல்படுத்துவதற்கு தகுந்தவை ஆவதாக சுட்டிக்காட்டு தெரிய வருகின்றன.

***குறிப்பு: இந்த பாடத்தில், Microsoft Phi-3 / 3.5 அழகு SLM உதாரணமாக அறிமுகப்படுத்தப்படும்.***

## Phi-3 / Phi-3.5 குடும்பத்தை அறிமுகம் செய்கிறோம்

Phi-3 / 3.5 குடும்பம் பெரும்பாலும் உரை, காட்சி மற்றும் முகவர் (MoE) பயன்பாட்டு சூழ்நிலைகளை இலக்காகக் கொண்டுள்ளது:

### Phi-3 / 3.5 Instruct

முதன்மையாக உரை உருவாக்கம், உரையாடல் பூர்த்தி, மற்றும் உள்ளடக்கம் தகவல் பெறுதல் போன்றவற்றிற்கு.

**Phi-3-mini**

3.8B பரிமாண மொழி மாதிரி Microsoft Foundry, Hugging Face மற்றும் Ollama இல் கிடைக்கிறது. Phi-3 மாதிரிகள் முக்கியமான தரைப் பரிசோதனைகளில் சமமான அல்லது பெரியதான மொழி மாதிரிகளைக் கடந்து செயல்திறனை காண்பிக்கின்றன (கீழே தரவு எண்களை பார்க்கவும், அதிக எண்கள் சிறந்தது). Phi-3-mini தன் அளவுக்கு இருவேறு மடங்கு மாதிரிகளைச் சமாளிக்கக்கூடியது, Phi-3-small மற்றும் Phi-3-medium எனும் மாதிரிகள் GPT-3.5 உட்பட பெரிய மாதிரிகள் காட்டியுள்ளதை மீறுகின்றன.

**Phi-3-small & medium**

சுமாரான 7B பரிமாணங்களுடன், Phi-3-small பல மொழி, வழிகாட்டுதல், குறியீடு, மற்றும் கணித பரிசோதனைகளில் GPT-3.5T-ஐ முந்துகிறது.

14B பரிமாணங்களோடு Phi-3-medium இந்த போக்கு தொடர்கிறது மற்றும் Gemini 1.0 Pro ஐ மீறுகிறது.

**Phi-3.5-mini**

இதை Phi-3-mini-இன் மேம்படுத்தல் என்று நாம் நினைக்கலாம். பரிமாணங்கள் அதே இருப்பினும், இது 20+ மொழிகளுக்கு (அரபுக், சீன, செஷ், டேனிஷ், டச்சு, ஆங்கிலம், பின்லிஷ், பிரஞ்சு, ஜெர்மன், ஹீப்ரூ, ஹங்கேரியன், இத்தாலியன், ஜப்பானியன், கொரியன், நோர்வேஜியன், போலிஷ், போர்ச்சுகீசு, ரஷ்யன், ஸ்பானிஷ், சுவீடிஷ், தாய், துருக்கிஷ், உருக்கிரேனியன்) ஆதரவு வழங்கும் திறனையும் நீண்ட அர்த்த மதிப்புக் கட்டுரைக்கும் வலுவான ஆதரவும் சேர்க்கிறது.

Phi-3.5-mini 3.8B பரிமாணங்களுடன் ஒரே அளவு மொழி மாதிரிகளை மீறி இருமடக்கும் மாதிரிகளுடன் சமமாக செயல்படுகிறது.

### Phi-3 / 3.5 Vision

Phi-3/3.5 இன் Instruct மாதிரியை Phi இன் புரிதல் திறன் என நினைக்கலாம், Vision என்பது Phi-க்கு கண்ணோட்டத்தை கொடுத்து உலகத்தை புரிந்துகொள்ள உதவுகிறதா.


**Phi-3-Vision**

Phi-3-vision, 4.2B பரிமாணங்கள் கொண்டது, இந்த போக்கை தொடர்கிறது மற்றும் பொதுவான காட்சியியல் தர்க்கக் கல்லாய்வுகள், OCR, அட்டவணை மற்றும் வரைபட புரிதல் பணிகளில் Claude-3 Haiku மற்றும் Gemini 1.0 Pro V போன்ற பெரிய மாதிரிகளை மீறுகிறது.


**Phi-3.5-Vision**

Phi-3.5-Vision என்பது Phi-3-Vision இன் மேம்படுத்தல், பல படங்களை ஆதரிக்கிறது. இதை கண்களுக்கான மேம்பாடு என்று நினைத்து, படங்களை மட்டுமல்லாமல் வீடியோக்களையும் பார்க்க முடியும்.

Phi-3.5-vision OCR, அட்டவணை மற்றும் விளக்கப்பட பகுப்பாய்வு பணிகளில் Claude-3.5 Sonnet மற்றும் Gemini 1.5 Flash போன்ற பெரிய மாதிரிகளை மீறுகிறது மற்றும் பொதுவான காட்சியியல் அறிவு தர்க்க பணிகளில் சம நீளம் காட்டுகிறது. பன்முக படங்கள் உள்ளீட்டைக் கையாளவும், அதாவது பல படம் உள்ளீடுகளில் தர்க்கம் செய்யவும் ஆதரவு.


### Phi-3.5-MoE

***Mixture of Experts(MoE)*** மாதிரிகள் மிகக் குறைந்த கணினி வளத்தில் முன்பயிற்சி செய்யப்படுவதாகும், இதனால் அதே கணினி பொருள்களுடன் பெரிய மாதிரிகளோ அல்லது பெரிய தரவுத்தொகுப்புகளோ உருவாக்கப்பட முடியும். குறிப்பாக, MoE மாதிரி அதே தரத்தை அடைவதில் என்னென்ன அதிக வேகத்துடன் dense மாதிரிக்கு போட்டியாக நடக்கிறது.

Phi-3.5-MoE 16x3.8B நிபுணர் தொகுதிகளைக் கொண்டுள்ளது. 6.6B செயல்படும் பரிமாணங்களுடன் Phi-3.5-MoE பெரிய மாதிரிகளோடு ஒப்பிடுகையில் அதே நிலை தர்க்கம், மொழி புரிதல் மற்றும் கணித திறன்களை காட்டுகிறது.

நம்மால் Phi-3/3.5 குடும்ப மாதிரிகளை பல்வேறு சூழ்நிலைகளில் பயன்படுத்த முடியும். LLM போன்றது அல்லாமல், Phi-3/3.5-mini அல்லது Phi-3/3.5-Vision இவற்றை எட்ஜ் சாதனங்களில் பயன்படுத்தலாம்.

- **நடைமுறை அமர்:** NIM ஒரே கட்டளையால் AI மாதிரிகளைக் குறியீடு செய்ய அனுமதிக்கிறது, இதனால் தற்போதைய பணிவழிகளில் எளிதாக இணைக்கலாம்.

- **முன்னேற்றமான செயல்திறன்:** இது குறைந்த தாமதம் மற்றும் உயர் ஊக்கத்தை உறுதிப்படுத்த NVIDIA இன் முன்-ஆப்டிமைச செய்யப்பட்ட முன்னறிவிப்பு இன்ஜின்களை, உதாரணமாக TensorRT மற்றும் TensorRT-LLM ஐ பயன்படுத்துகிறது.
- **தொகுத்தி அளவைப் பெருக்கல் (Scalability):** NIM குபேர்நெட்டீஸில் தானாக அளவைப் பெருக்கலை ஆதரிக்கிறது, இது மாறும் பணிகளைக் கையாள மிக நன்று.
- **பாதுகாப்பு மற்றும் கட்டுப்பாடு:** நிறுவனங்கள் தங்கள் தரவுகளையும் செயலிகளையும் தங்களுடைய நிர்வகிக்கப்படும் பின்னணியில் NIM மைக்ரோசெர்வீசுகளை சுய-பிரதிநிதித்துவம் செய்ய மூலம் கட்டுப்பாட்டை பராமரிக்கலாம்.
- **தரநிலை APIகள்:** NIM தொழில்துறை தரநிலை APIகளை வழங்குகிறது, இதனால் சாட்பாட்கள், AI உதவியாளர்கள் மற்றும் பிற AI செயலிகளை எளிதாக உருவாக்கி ஒருங்கிணைக்கலாம்.

NIM என்பது NVIDIA AI நிறுவனத்தின் ஒரு பகுதி, இது AI மாதிரிகளை நிரல்படுத்தி செயல்படுத்ததை எளிதாக்கி NVIDIA GPUகளில் சரியான செயல்பாட்டுடன் இயங்க உதவுகிறது.

**Hugging Face Transformer**
Hugging Face Transformers என்பது இயற்கை மொழி செயலாக்கம் (NLP) மற்றும் பிற இயந்திரக் கற்றல் பணிகளுக்கான சக்திவாய்ந்த நூலகமாகும். இதன் முக்கிய அம்சங்கள் சில:

1. **முன்னைமையாக பயிற்றப்பட்ட மாதிரிகள்**: உரை வகைப்பாடு, பெயர் பொருள் அடையாளம், கேள்வி பதில், சுருக்கம், மொழிபெயர்ப்பு, உரை உருவாக்கம் போன்ற பணிகளுக்கான ஆயிரக்கணக்கான பயிற்றப்பட்ட மாதிரிகள் கிடைக்கின்றன.

2. **அமைப்பு இடைமுகத்தன்மை**: இந்த நூலகம் PyTorch, TensorFlow மற்றும் JAX உள்ளிட்ட பல தீவிர காதல் கற்றல் சூழல்களை ஆதரிக்கிறது. இதனால் ஒரு சூழலில் பயிற்சி பெற்ற மாதிரியைக் மற்றொரு சூழலில் பயன்படுத்தலாம்.

3. **பலவகை திறன்கள்**: NLP தவிர, Hugging Face Transformers கணினி பார்வை (உதா: படம் வகைப்படுத்தல், பொருள் கண்டறிதல்) மற்றும் ஒலி செயலாக்கம் (உதா: தொலைபேசி அடையாளம், ஒலி வகைப்படுத்தல்) பணிகளையும் ஆதரிக்கிறது.

4. **இயல்திறன் வசதிகள்**: இது மாதிரிகளை எளிதாக பதிவிறக்கம் செய்து நுணுக்கமாக படிப்படியாக பராமரிக்க APIகளையும் கருவிகளையும் வழங்குகிறது, தொடக்கர்கள் மற்றும் வல்லுநர்களுக்கு ஏற்றது.

5. **சமூகம் மற்றும் வளங்கள்**: Hugging Faceக்கு விரிவான சமூகமும் விரிவான ஆவணங்கள், பயிற்சிகட்டுரைகள், வழிகாட்டுதல்கள் உள்ளன, பயனர்களுக்கு உதவுவதற்காக.
[அதிகாரப்பூர்வ ஆவணங்கள்](https://huggingface.co/docs/transformers/index?WT.mc_id=academic-105485-koreyst) அல்லது அவற்றின் [GitHub நிலையகம்](https://github.com/huggingface/transformers?WT.mc_id=academic-105485-koreyst).

**GenAI க்கான ONNX Runtime**

## ONNX Runtime என்றால் என்ன?
ONNX Runtime என்பது உயர் செயல்திறன் இயந்திரக் கற்றல் மாதிரி முன்னறிவிப்பை இயக்கும் திறந்த மூலத் திட்டமாகும். இது Open Neural Network Exchange (ONNX) வடிவமைப்பில் உள்ள மாதிரிகளை ஆதரிக்கிறது, இது இயந்திரக் கற்றல் மாதிரிகளைச் சட்டரீதியாக பிரதிநிதித்துவம் செய்யும் தரநிலையாகும். ONNX Runtime முன்னறிவிப்பு வாடிக்கையாளர் அனுபவங்களை வேகப்படுத்தி செலவுகளை குறைக்க உதவுகிறது, PyTorch மற்றும் TensorFlow/Keras போன்ற தீவிரக் கற்றல் வடிவமைப்புகளையும் scikit-learn, LightGBM, XGBoost போன்ற சாஸ்திரக் கற்றல் நூலகங்களையும் ஆதரிக்கிறது. ONNX Runtime வெவ்வேறு உபகரணங்கள், இயக்கிகள் மற்றும் இயக்க முறைமைகளுடன் பொருந்துகின்றது மற்றும் உட்பொறிகளைச் செயல்படுத்துதல், கிராஃப் மேம்பாடுகள் மற்றும் மாற்றங்களைக் கொண்டு சிறந்த செயல்திறனை வழங்குகிறது.

## Generative AI என்றால் என்ன?
Generative AI என்பது பயிற்சி பெற்ற தரவின் அடிப்படையில் புதிய உள்ளடக்கங்களை உருவாக்கும் AI அமைப்புகளை குறிக்கின்றது, உதாரணமாக உரை, படங்கள், இசை போன்றவை. உதாரணமாக GPT-3 போன்ற மொழி மாதிரிகள் மற்றும் Stable Diffusion போன்ற படம் உருவாக்கும் மாதிரிகள். ONNX Runtime பண்பலுக்கான GenAI நூலகம் ONNX மாதிரிகளுக்கான உருவாக்கும் AI சுழற்சியைப் பெறுவதையும், ONNX Runtime மூலம் முன்னறிவிப்பைச் செய்யவும், லாஜிட்கள் செயலாக்கம், தேடல் மற்றும் மாதிரிப்பங்களை உள்ளடக்குகிறது.

- **மாதிரி ஆதரவு:** LLaMA, GPT-Neo, BLOOM மற்றும் பிற பிரபல உருவாக்கும் AI மாதிரிகளை ஆதரிக்கிறது.
- **செயல்திறன் மேம்பாடு:** NVIDIA GPUகள், AMD GPUகள் மற்றும் பிற பொருட்களை ஆதரிக்கும் வெவ்வேறு உபகரண வேகமிடல்கள் கொண்டுள்ளது.
- **எளிதான பயன்பாடு:** பயன்பாடுகளுக்கு எளிதில் ஒருங்கிணைக்க APIகளை வழங்கி குறைந்த குறியீட்டுடன் உரை, படம் மற்றும் பிற உள்ளடக்கங்களை உருவாக்க ஏதுவாக்குகிறது.
- பயனர்கள் உயர் நிலை generate() முறையை அழைக்கவோ அல்லது மாதிரியின் ஒவ்வொரு சுற்றத்தையும் ஒரு டோக்கனை ஒரே நேரத்தில் உருவாக்க(loop) இயக்கவோ, விருப்பத்திற்கேற்றவாறு சுற்றի உள்ளே உருவாக்கும் அளவுருக்களை புதுப்பிக்கவோ முடியும்.
- ONNX Runtime பாகமன / கதிர் தேடல் மற்றும் TopP, TopK மாதிரிப்புகளை உள் வளர்ப்பு, மேலும் அடுக்கு லாஜிட் செயலாக்கம் போன்றவை உள்ளன. தனிப்பயன் மதிப்பீடு சேர்க்கவும் எளிது.

## துவங்குதல்

```Python
pip install onnxruntime
```
### உருவாக்கும் AI நீட்சிகளை நிறுவுதல்:
```Python
pip install onnxruntime-genai
```

### மாதிரி இயக்குதல்: Python இல் ஒரு எளிய உதாரணம்:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### டெமோ: ONNX Runtime GenAI பயன்படுத்தி Phi-3.5-Vision அழைப்பு


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
