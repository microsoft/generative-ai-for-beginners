# Prompt Engineering அடிப்படைகள்

[![Prompt Engineering அடிப்படைகள்](../../../translated_images/ta/04-lesson-banner.a2c90deba7fedacd.webp)](https://youtu.be/GElCu2kUlRs?si=qrXsBvXnCW12epb8)

## அறிமுகம்
இந்த பகுதியில் உருவாக்கும் AI மாதிரிகளில் பயனுள்ள prompt-களை உருவாக்குவதற்கான அடிப்படை கருத்துக்கள் மற்றும் தொழில்நுட்பங்களை கையாளப்படுகிறது. LLM-க்கு நீங்கள் எழுதும் prompt-வின் மூலமும் முக்கியம். கவனமாக வடிவமைக்கப்பட்ட prompt ஒன்று சிறந்த பதிலை பெற செய்யக்கூடும். ஆனால் _prompt_ மற்றும் _prompt engineering_ என்ன அர்த்தம்? நான் LLM-க்கு அனுப்பும் prompt _input_-ஐ எப்படி மேம்படுத்துவது? இவை போன்ற கேள்விகளுக்கு இப்பகுதி மற்றும் அடுத்த பகுதியில் பதில் காண்போம்.

_உருவாக்கும் AI_ பயனர் வேண்டுதலுக்கு புதிய உள்ளடக்கங்களை (உதா., உரை, படங்கள், ஒலி, குறியீடு அல்லது பிற) உருவாக்கும் திறனுடையது. இது OpenAI-வின் GPT ("Generative Pre-trained Transformer") போன்ற _பெரிய மொழி மாதிரிகளை_ பயன்படுத்தி இயங்குகிறது, அவை இயற்கை மொழி மற்றும் குறியீட்டை பயிற்றுவிக்கப்பட்டவை.

பயனர்கள் இப்போது உரையாடல் போன்ற பரிச்சயமான முறைகளை பயன்படுத்தி இந்த மாதிரிகளுடன் தொடர்பு கொள்ளலாம், எந்த தொழில்நுட்ப நிபுணத்துவமும் பயிற்சியும் தேவையில்லை. மாதிரிகள் _prompt-க்களைக்_ கொண்டு இயங்குகின்றன - பயனர்கள் ஒரு உரை உள்ளீட்டை (prompt) அனுப்பி AI பதிலை (completion) பெறுகிறார்கள். பின்னர் பன்முறை உரையாடலில் "AI-னுடன் உரையாடி" தங்கள் prompt-ஐ தேவைப்படி மேம்படுத்தலாம், பதில் எதிர்பார்ப்புக்கு ஏற்ப செல்லும் வரை.

இப்போது "Prompt" கள் உருவாக்கும் AI செயலிகளுக்கான முதன்மை _ப்ரோகிராமிங் இடைமுகமாக_ மாறிவிட்டன, மாதிரிகளுக்கு என்ன செய்ய வேண்டும் என்பது கூறி, கிடைக்கும் பதில்களின் தரத்தையும் தாக்குகின்றன. "Prompt Engineering" என்பது prompt-களை வடிவமைத்து மேம்படுத்தும் ஆராய்ச்சி துறை ஆக விரைவில் வளர்ந்து வருகிறது, இதனால் மிகுந்த அளவிலான தரமான பதில்களை வழங்க முடிகிறது.

## பயிற்சி நோக்குகள்

இந்த பாடத்தில், Prompt Engineering என்ன, அது ஏன் முக்கியம், மற்றும் ஒரு மாதிரி மற்றும் பயன்பாட்டிற்கான சிறந்த prompt-ஐ உருவாக்குவது எப்படி என்பதைக் கற்போம். prompt engineering-க்கான அடிப்படை கருத்துகள் மற்றும் சிறந்த நடைமுறைகள் பற்றி அறிந்து, அவற்றை செயல்படுத்த Jupyter Notebooks-ன் "sandbox" சுற்றுச்சூழல் பற்றி கற்றுக்கொள்வோம்.

இந்த பாட முடிவில், நாங்கள்:

1. Prompt engineering என்பது என்ன மற்றும் அது ஏன் தேவையென விளக்கியிட முடியும்.
2. ஒரு prompt-ன் பகுதிகள் என்ன மற்றும் அவை எப்படி பயன்படுத்தப்படுகின்றன என்பதைக் கூறலாம்.
3. prompt engineering-க்கான சிறந்த நடைமுறைகள் மற்றும் தொழில்நுட்பங்களை கற்கலாம்.
4. கற்ற தொழில்நுட்பங்களை உண்மையான எடுத்துக்காட்டுகளில் OpenAI endpoint பயன்படுத்தி செயல்படுத்தலாம்.

## முக்கிய சொற்கள்

Prompt Engineering: AI மாதிரிகளை தேவையான உள்ளீடுகளை வடிவமைத்து மேம்படுத்தி, வேண்டிய முடிவுகளை பெறுவதற்கான நடைமுறை.

Tokenization: உரையை சிறிய பகுதிகளாக (tokens என்று அழைக்கப்படும்) மாற்றும் செயல்முறை, மாதிரி அவற்றை புரிந்து செயலாக்கி முடிவுகள் இயற்ற வருகிறது.

Instruction-Tuned LLMs: குறிப்பிட்ட ஆலோசனைகளுடன் தனிப்பயிற்றுவிக்கப்பட்ட பெரிய மொழி மாதிரிகள், அவை பதில் சரியானதும் பொருத்தமானதும் இருக்க செய்ய உதவுகின்றன.

## பயிற்சி மைதானம்

Prompt engineering தற்போது உனசியில் கைவினை கலையும் அறிவியலும் ஒன்றாகும். அதை விரிவாக புரிந்து கொள்ள சிறந்த வழி _பயிற்சி செய்ய_ மற்றும் ஆராய்ச்சி முனைவுடன் செயல்பாடுகளை இணைத்த முயற்சி முறையை பின்பற்றுவது.

இந்த பாடத்துடன் இணைக்கப்பட்டுள்ள Jupyter Notebook-ல் நீங்கள் கற்றதை முயற்சி செய்ய ஒரு _sandbox_ சூழலும் உள்ளது. பயிற்சிகளை செய்வதற்கு உங்கள் கையில்:

1. **Azure OpenAI API முக்கியம்** - செயல்படுத்தப்பட்ட LLM கோரிக்கை இடைமுகர்.
2. **Python Runtime** - Notebook இயங்க தேவையான சுற்றுச்சூழல்.
3. **உள்ளூர் சூழ்நிலை மாறிலிகள்** - இப்போது [SETUP](./../00-course-setup/02-setup-local.md?WT.mc_id=academic-105485-koreyst) படிகளை கடந்தே தயாராகுங்கள்.

Notebook-க்கு ஆரம்ப பயிற்சிகள் அமைந்திருக்கின்றன, ஆனால் நீங்கள் உங்கள் சொந்த _Markdown_ (விபரம்) மற்றும் _Code_ (prompt வேண்டுகோள்) பிரிவுகளை சேர்த்து கூடுதல் எடுத்துக்காட்டுகளை முயற்சி செய்து உங்கள் அறிவை வளர்க்க encouraged க்கப்படுகிறீர்கள்.

## விளக்கப்படக் கையேடு

இந்த பாடத்தில் என்ன கற்போம் என்பதற்கு ஒரு பொதுவான கண்ணோட்டம் வேண்டுமா? இந்த விளக்கப்படக் கையேட்டை பாருங்கள், இது முக்கிய தலைப்புகளை மற்றும் ஒவ்வொன்றிலும் நீங்கள் பரிசீலிக்க வேண்டிய முக்கிய அம்சங்களை உங்களுக்கு காட்டும். பாடத்திட்ட ஏங்கும் கோரு கருத்துகளையும் சவால்களையும் புரிந்து, அவற்றை தொடர்புடைய prompt engineering தொழில்நுட்பங்களும் சிறந்த நடைமுறைகளும் கொண்டு எவ்வாறு கையாள்வது என்பதற்கு வழிகாட்டும். "மேம்பட்ட தொழில்நுட்பங்கள்" பகுதி அடுத்த பாடத்தில் விரிவாக உண்டாகும்.

## எங்கள் தொடக்க நிறுவனம்

இந்த தலைப்பு எங்கள் தொடக்க நிறுவனத்தின் [கல்விக்கான AI புதுமைகளை கொண்டு வருகிறோம்](https://educationblog.microsoft.com/2023/06/collaborating-to-bring-ai-innovation-to-education?WT.mc_id=academic-105485-koreyst) என்ற பணி தொடர்பாக எப்படி பொருந்துகிறது என்பதைக் பார்க்கலாம். _தனிப்பட்ட கற்றல்_ AI செயலிகள் உருவாக்க விரும்புகிறோம் - அதனால் எங்கள் பயன்பாட்டின் வேறுபட்ட பயனர்கள் எப்படி "prompt" வடிவமைக்கலாம் என்று எண்ணுங்கள்:

- **நிர்வாகிகள்** AI-க்கு _பாடத்திட்ட தரவுகளை பகுப்பாய்வு செய்து கோரிக்கைகள் எவை என்பதை கண்டறிய_ கேட்கலாம். AI முடிவுகளை சுருக்கம்செய்யவும் குறியீடு கொண்டு காட்சி படைப்பதையும் செய்யக்கூடும்.
- **ஆசிரியர்கள்** AI-க்கு _குறிப்பிடப்பட்ட பார்வையாளர்களுக்கும் தலைப்பிற்குமான பாடத் திட்டத்தை உருவாக்க_ கேட்கலாம். AI குறிப்பிட்ட வடிவத்தில் தனிப்பட்ட திட்டத்தை உருவாக்கும்.
- **மாணவர்கள்** AI-யை _ஒரு கடினமான பாடத்தில் வழிகாட்டி_ கேட்கலாம். AI தற்போது அந்த மாணவரின் நிலைக்கு ஏற்ப பாடங்களும், குறிப்பு மற்றும் எடுத்துக்காட்டுகளும் வழங்கும்.

இது தொடக்கம் மட்டும்தான். கல்வி நிபுணர்களால் தொகுக்கப்பட்ட [Prompts For Education](https://github.com/microsoft/prompts-for-edu/tree/main?WT.mc_id=academic-105485-koreyst) - ஒரு திறந்த மூல prompt நூலகத்தை பாருங்கள் - இதில் எழும்பக்கூடிய கூடுதல் வாய்ப்புகளைக் காணலாம்! _sandbox-ல் அல்லது OpenAI Playground-இல் அவற்றைப் பயன்படுத்தி முயற்சி செய்யுங்கள்!_

## Prompt Engineering என்றால் என்ன?

இந்த பாடத்தை துவக்கும்போது **Prompt Engineering** என்றதனை ஒரு செயலியின் குறிக்கோள் மற்றும் மாதிரிக்கு ஏற்ப _prompt-களை வடிவமைக்கவும் மற்றும் மேம்படுத்தவும்_ செய்வதே என்று விளக்கினோம். இதை 2 கட்டங்களாகவே கருதலாம்:

- கொடுக்கப்பட்ட மாதிரி மற்றும் குறிக்கோளுக்கு ஆரம்ப prompt-ஐ _வடிவமைத்தல்_
- பதிலின் தரத்தை மேம்படுத்த _prompt-ஐ முறையாக மீண்டும் திருத்துதல்_

இது அவசியமாக முயற்சி மற்றும் பிழைசொல்வது ஆகும், சிறந்த முடிவுகளுக்கா பயனர் உணர்வு மற்றும் உழைப்பு தேவை. ஏன் இது முக்கியமெனக் கேட்கிறீர்களா? பதிலாக கவனிக்க வேண்டியது மூன்று கருத்துக்களைப் புரிந்து கொள்ள வேண்டும்:

- _Tokenization_ = மாதிரி prompt-ஐ எவ்வாறு "பார்க்கிறது"
- _அடிப்படை LLMs_ = அடிப்படை மாதிரி prompt-ஐ எப்படி "செயலாக்குகிறது"
- _Instruction-Tuned LLMs_ = மாதிரி "பணிகளை" எப்படி பார்க்க முடியும் என்பதன் விளக்கம்

### Tokenization

ஒரு LLM prompt-களை _token-களின் வரிசையாக_ பார்க்கிறது, வெவ்வேறு மாதிரிகள் (அல்லது அவற்றின் பதிப்புகள்) அதே prompt-ஐ வேறு முறைகளில் tokenize செய்யலாம். LLM-கள் token-களில் பயிற்றுவிக்கப்படுகின்றன (உரையே அல்ல), ஆகவே prompt-களை tokenize செய்வது மூலமே உருவாக்கப்படும் பதிலை பாதிக்கும்.

tokenization எப்படி நடக்கும் என்பதற்கான உணர்வுக்காக, கீழே கொடுக்கப்பட்ட [OpenAI Tokenizer](https://platform.openai.com/tokenizer?WT.mc_id=academic-105485-koreyst) போன்ற கருவிகளை முயற்சியுங்கள். உங்கள் prompt-ஐ நகலெடுத்து அவை எவ்வாறு tokens ஆக மாறுகின்றன என்பதை பாருங்கள், வெற்று இடங்கள் மற்றும் знак கொள்க்களும் எப்படி கையாளப்படுகிறது என கவனியுங்கள். இந்த எடுத்துக்காட்டு பழைய LLM (GPT-3) ஐ காட்டுகிறது - புதிய மாதிரிகளுடன் செய்யும் முயற்சி வேறு முடிவுகளை கொடுக்கலாம்.

![Tokenization](../../../translated_images/ta/04-tokenizer-example.e71f0a0f70356c5c.webp)

### கருத்து: அடிப்படை மாதிரிகள்

ஒரு prompt tokenize ஆகிய பின், ["அடிப்படை LLM"](https://blog.gopenai.com/an-introduction-to-base-and-instruction-tuned-large-language-models-8de102c785a6?WT.mc_id=academic-105485-koreyst) அல்லது அடிப்படை மாதிரியின் முக்கியப் பணி அந்த வரிசையில் அடுத்த token-ஐ கணிக்க தான். LLM-கள் பெரும் உரைத் தொகுப்புகளில் பயிற்றுவிக்கப்பட்டதால், அவை token-களுக்கு இடையேயான புள்ளியியல் தொடர்புகளை நன்கு அறிந்து, நம்பிக்கை உடைய கணிக்க முடியும். அவை prompt அல்லது token-ல் உள்ள வார்த்தைகளின் _அர்த்தத்தை_ புரியவில்லை; pattern-ஐ மட்டும் பார்த்து அடுத்த முன்னொட்டு முனைப்பில் அதை "முழுமையாக்க" செய்கின்றன. பயனர் இடையீடு அல்லது ஏதாவது முன்னதாக நிர்ணயிக்கப்பட்ட நிபந்தனை நிறைவடையும்வரை வரிசையை தொடர்ந்துதான் கணிக்கும்.

prompt அடிப்படையிலான completion-ஐ எப்படி நடக்கும் என்பதைக் காண விரும்பினால், மேல் உள்ள prompt-ஐ Azure OpenAI Studio [_Chat Playground_](https://oai.azure.com/playground?WT.mc_id=academic-105485-koreyst) இல் இயல்புநிலை அமைப்புகளுடன் உள்ளிடவும். இந்த நிரல் prompt-களை தகவல் கோரிக்கை என்றும் கருதி பதிலளிக்கும்.

ஆனா பயனர் குறிப்பிட்ட பணிக்கான சூழலை பூர்த்தி செய்வதைப் பார்க்க விரும்பினால்? அதற்கேற்ப _instruction-tuned_ LLM-கள் உதவி செய்யும்.

![Base LLM Chat Completion](../../../translated_images/ta/04-playground-chat-base.65b76fcfde0caa67.webp)

### கருத்து: Instruction Tuned LLMs

[Instruction Tuned LLM](https://blog.gopenai.com/an-introduction-to-base-and-instruction-tuned-large-language-models-8de102c785a6?WT.mc_id=academic-105485-koreyst) என்பது அடிப்படை மாதிரியை எடுத்துக் கொண்டு, தெளிவான வழிமுறைகள் கொண்ட எடுத்துக்காட்டுகள் அல்லது உள்ளீடு/வெளியேற்று ஜோடியை (உதா., பன்முறை "செய்திகள்") கொண்டு சிறுகூர்ந்து தயார் செய்த மாதிரி ஆகும், இது AI பதில் அந்த வழிமுறையை பின்பற்ற முயலுகிறது.

இது மனித பின்விளைவுடன் கட்டாயப்படுத்தும் கற்றலுடன் (RLHF) போன்ற தொழில்நுட்பங்களைப் பயன்படுத்தி, மாதிரியை _வழிமுறைகளை பின்பற்ற_ மற்றும் _பின்விளைவுகளிலிருந்து கற்க_ வகுத்து, பயனுள்ள செயலிகளுக்கு ஏற்ப சிறந்த பதில் கொடுக்க உதவுகிறது.

இதைச் செய்ய, மேல் உள்ள prompt-ஐ மீண்டும் பார்க்க, ஆனால் இப்போது _system message_-இல் கீழ்காணும் வழிமுறைகளை கொடுத்து பாருங்கள்:

> _உங்களிடம் கொடுக்கப்பட்ட உள்ளடக்கத்தை இரண்டாம் வகுப்பு மாணவனுக்குத் தொகுக்கவும். முடிவை ஒரு ஒற்றை பத்தி மற்றும் 3-5 புத்தாக்க அட்டவணைகளில் வைக்கவும்._

இதனால் முடிவு எந்த வகையில் இனிமேல் தேர்ந்தெடுக்கப்பட்டு அமைக்கப்படுகிறது என்பதைப்பாருங்கள்? ஆசிரியர் இப்பதிலை நேரடியாக தங்கள் வகுப்புக் கோப்புகளில் பயன்படுத்தலாம்.

![Instruction Tuned LLM Chat Completion](../../../translated_images/ta/04-playground-chat-instructions.b30bbfbdf92f2d05.webp)

## ஏன் Prompt Engineering தேவை?

prompt-களை LLM-கள் செயலாக்குவதற்குப் பிறகு, ஏன் prompt engineering அவசியமென கேட்டால், இதற்கு பதில் தற்போது உள்ள LLM-கள் _நம்பகமான மற்றும் ஒருங்கிணைந்த பதில்களை_ உருவாக்குவதில் சில சவால்களை கொண்டுள்ளன என்பதுதான். இதனால் prompt உருவாக்கம் மற்றும் மேம்பாடு மீது முயற்சி செலுத்தாமல் விளைவுகள் சரியாக வராமல் போகும். உதாரணமாக:

1. **மாதிரி பதில்கள் மாற்றத்தன்மையுடையவை.** _அதே prompt_ வெவ்வேறு மாதிரிகள் அல்லது அதே மாதிரியின் வெவ்வேறு பதிப்புகளுடன் வெவ்வேறு பதில்களை உருவாக்கும். ஒரே மாதிரியுடன் கூட வெவ்வேறு நேரங்களில் வேறுபட்ட முடிவுகள் வரும். _Prompt engineering முறைகள் இந்த வேறுபாடுகளை குறைத்து சிறந்த கட்டுப்பாடுகளை வழங்க உதவும்_.

1. **மாதிரிகள் பதில்களை கற்பனை செய்கின்றன.** மாதிரிகள் _பெரிய ஆனால் வரம்பான_ தரவுத்தொகுப்புகளில் பயிற்றுவிக்கப்பட்டதால், தொகுப்பின் வெளிப்புற கருத்துக்களைப் பற்றி அறிவில்லை. ஆகவே தவறான, கற்பனையோ அல்லது தெரியாத உண்மைகளை எதிர்க்கும் பதில்களை உருவாக்கும். _Prompt engineering பயனர்கள் இல்லாத அல்லது விடய சேர்க்கப்பட்டது என்பதை தடுக்கும் வழிகளை கற்றுக்கொள்ள உதவும், என்று AI-இல் மேற்கோள் கேட்டு செவிவிழுங்க உதவும்_.

1. **மாதிரி திறன்கள் மாறுபடும்.** புதிய மாதிரிகள் அல்லது தலைமுறைகள் மேலான திறன்களோடு வரும் ஆனாலும் தனித்துவமான குறைகள் மற்றும் செலவு மற்றும் சிக்கலுடன் வரலாம். _Prompt engineering சிறந்த நடைமுறைகளை உருவாக்கியும், மாதிரிக் குறிப்பிட்ட தேவைகளை ஏற்றாற்போல பராமரிக்கும் வேலைமுறையையும் உதவும்_.

இதனை OpenAI அல்லது Azure OpenAI Playground-ல் காட்சி படுத்தலாம்:

- வெவ்வேறு LLM அமைப்புகளில் (OpenAI, Azure OpenAI, Hugging Face) அதே prompt பயன்படுத்தி பார்க்கவும் - வேறுபாடுகள் தோன்றினரா?
- ஒரே LLM அமைப்பை பல முறைகள் (உதா., Azure OpenAI playground) கொண்டு அதே prompt-ஐ மீண்டும் மீண்டும் பயன்படுத்திய போது வேறுபாடுகள் எப்படி இருந்தன?

### கற்பனை(பணிகள்) எடுத்துக்காட்டு

இந்த பாடத்தில், LLM-கள் சில நேரங்களில் பயிற்றுவிப்பில் சீரான விளக்கமற்ற காரணத்தால் உண்மையற்ற தகவலை உருவாக்கும் நிகழ்வை **"fabrication"** என்று குறிப்பிடுகிறோம். பொதுவாக இதை _"hallucinations"_ என்று அழைக்கப்பட்டாலும், இயந்திரம் மனிதத்தன்மை கொண்டது என்று தவறாகப் பொருள் கொள்ளாமல் _fabrication_ என்ற சொற்றொடரை பயன்படுத்த பரிந்துரைக்கபடுகிறது. இது [பொறுப்புடைய AI வழிகாட்டுதல்கள்](https://www.microsoft.com/ai/responsible-ai?WT.mc_id=academic-105485-koreyst) அடிப்படையில் மனிதத்தை அண்மைகாக்கும் வார்த்தைகள் வெளியேற்றப்படுகிறது, இது சில சூழல்களில் முதிர்ச்சியற்ற அல்லது விரோதமானதாகவும் இருக்கக்கூடும்.

fabrication எப்படி நடக்கும் என்பதை எண்ண விரும்பினால், பயிற் தரவுத்தொகுப்பில் இல்லாத தலைப்புக்கான உள்ளடக்கத்தை உருவாக்க AI-ஐ உத்தரவிடும் prompt எண்ணுங்கள். உதா - நான் இந்த prompt-ஐ முயற்சி செய்தேன்:

> **Prompt:** 2076-ஆம் ஆண்டு மார்டியன் போருக்கு பாடத் திட்டம் ஒன்றை உருவாக்கவும்.
ஒரு வலை தேடல் எனக்கு காட்டியது மார்டியன் போருகளுக்கு (உதாரணத்திற்கு, தொலைக்காட்சி தொடர்களோ அல்லது புத்தகங்களோ) கற்பனையான கணக்குகள் இருந்தன - ஆனால் 2076 இல் ஒன்று இல்லை. பொதுக்கருத்தும் 2076 என்பது _எதிர்கால_ ஆகும் என்பதால், அது உண்மையான நிகழ்வுடன் தொடர்புடையதாக இருக்க முடியாது என்று சொல்லுகிறது.

_மெட்டா பிராம்டிங்_ மற்றும் _தென்பரேச்சர் அமைப்பு_ போன்ற விருப்பக் கேள்வி பொறியியல்துறை தொழில்நுட்பங்கள் சில அளவுக்கு மாதிரி கற்பனைகளை குறைக்கலாம். புதிய விருப்பக் கேள்வி பொறியியல் _வடிவமைப்புகள்_ புதிய கருவிகள் மற்றும் தொழில்நுட்பங்களை தானாகவே விருப்பக் கேள்வி ஓட்டத்துக்கு ஒருங்கிணைத்து, இவற்றுள் சில செயல்களை தணிக்க அல்லது குறைக்க உதவுகின்றன.

## விருப்பக் கேள்வி கட்டமைப்பு

நாம் விருப்பக் கேள்வி பொறியியல் ஏன் முக்கியமோ என்று பார்த்தோம் - இப்போது விருப்பக் கேள்விகள் _எவ்வாறு கட்டமைக்கப்படுகின்றன_ என்பதை புரிந்து கொண்டு பலவிதமான தொழில்நுட்பங்களை மதிப்பாய்வு செய்வோம், மேலும் பயனுள்ள விருப்பக் கேள்வி வடிவமைப்பிற்கு.

### அடிப்படை விருப்பக் கேள்வி

அடிப்படையான விருப்பக் கேள்வி என்றால் மாதிரிக்கு அனுப்பும் உரை உள்ளீடு மட்டுமே. உதாரணமாக, அமெரிக்க தேசிய கீதத்தின் (US national anthem) தொடக்க சில வார்த்தைகளை OpenAI [Completion API](https://platform.openai.com/docs/api-reference/completions?WT.mc_id=academic-105485-koreyst) இற்கு அனுப்பினால், அது உடனே அடுத்த சில வரிகளுடன் பதிலளிப்பதை காணலாம், இது அடிப்படை முன்னறிவிப்புக் குணத்தைக் காட்டுகிறது.

| Prompt (Input)     | Completion (Output)                                                                                                                        |
| :----------------- | :----------------------------------------------------------------------------------------------------------------------------------------- |
| Oh say can you see | It sounds like you're starting the lyrics to "The Star-Spangled Banner," the national anthem of the United States. The full lyrics are ... |

### சிக்கலான விருப்பக் கேள்வி

இப்போது இந்த அடிப்படைக் கேள்விக்கு உள்ளமைவும் அறிவுறுத்தல்களும் சேர்க்கலாம். [Chat Completion API](https://learn.microsoft.com/azure/ai-services/openai/how-to/chatgpt?WT.mc_id=academic-105485-koreyst) இடையே பின்வரும் வகையில் _செய்திகள்_ தொகுப்பாக சிக்கலான விருப்பக் கேள்வியை அமைக்க அனுமதிக்கிறது:

- உள்ளீடு/வெளியீடு жұ不得ள் _பயனர்_ உள்ளீடும் _உதவியாளன்_ பதிலோடு பொருந்துவதாக.
- அமைப்பு செய்தி உதவியாளரின் நடத்தை அல்லது குணங்காட்டலில் உள்ளமைவு அமைக்கிறது.

கோரிக்கை இப்போது கீழ்காணும் வடிவில் இருக்கும், இங்கு _Tokenization_ தொடர்புடைய தகவல்களை ஊடுருவல் மற்றும் உரையாடலிலிருந்து சிறப்பாக எடுத்துக்கொள்கிறது. அமைப்புப் பின்னணி மாற்றும் போது, வழங்கப்பட்ட பயனர் உள்ளீடுகளுக்கு போன்றும்தான் பதில்களின் தரத்திலும் பெரும் தாக்கம் ஏற்படும்.

```python
response = openai.chat.completions.create(
    model="gpt-3.5-turbo",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Who won the world series in 2020?"},
        {"role": "assistant", "content": "The Los Angeles Dodgers won the World Series in 2020."},
        {"role": "user", "content": "Where was it played?"}
    ]
)
```

### அறிவுறுத்தல் விருப்பக் கேள்வி

மேலுள்ள எடுத்துக்காட்டுகளில், பயனரின் விருப்பக் கேள்வி என்பது தகவல் கோரிக்கை என்ற அர்த்தத்தில் இருக்கும் எளிய உரை கேள்வியாக இருந்தது. _அறிவுறுத்தல்_ விருப்பக் கேள்விகளில், இந்தக் கேள்வியை பயன்படுத்தி AIக்குக் காரியம் குறித்து விரிவாக கூறி, சிறந்த வழிகாட்டுதல் வழங்கலாம். உதாரணம் இங்கே:

Provide key dates and events and describe their significance                                                                                                                                     | _returned a paragraph followed by a list of key event dates with descriptions_                                             | Complex             |

## பிரதான உள்ளடக்கம்

மேலுள்ள எடுத்துக்காட்டுகளில், விருப்பக் கேள்வி இன்னும் திறந்திருப்புமாக இருந்தது, LLM எந்த ஒரு முன் பயிற்சி தரவுத்தொகுதியில் தொடர்புடையது என்று தீர்மானிக்க முடிந்தது. _பிரதான உள்ளடக்கம்_ வடிவமைப்பில், உள்ளீட்டு உரை இரண்டு பகுதிகளாக பிரிக்கப்படுகிறது:

- ஒரு அறிவுறுத்தல் (செயல்)
- தொடர்புடைய உள்ளடக்கம் (செயலுக்கு பங்குபற்றுவது)

உதாரணமாக, அறிவுறுத்தல் "இதை 2 வாக்கியங்களில் சுருக்கி" என்றதாக இருக்கிறது.

| Prompt (Input)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      | Completion (Output)                                                                                                                                                                                                                                                                             |
| :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Jupiter is the fifth planet from the Sun and the largest in the Solar System. It is a gas giant with a mass one-thousandth that of the Sun, but two-and-a-half times that of all the other planets in the Solar System combined. Jupiter is one of the brightest objects visible to the naked eye in the night sky, and has been known to ancient civilizations since before recorded history. Named after the Roman god Jupiter, it's a gas giant whose mass is two-and-a-half times that of all other planets in the Solar System combined. |

பிரதான உள்ளடக்க பாகத்தை பலவிதமாக உபயோகித்து சிறந்த அறிவுறுத்தல்களை இயக்கலாம்:

- **மாதிரிகள்** - மாதிரிக்கு என்ன செய்ய வேண்டும் என்பதை தெளிவாக கூறுவது பதிலாக, செய்ய வேண்டியவை எடுக்கப்பட்ட மாதிரிகளை வழங்கி, மாதிரி வழிமுறையை கண்டறிய விடுங்கள்.
- **கியூஸ்** - அறிவுறுத்தலுக்கு பின்பு ஒரு "கியூ" வழங்கி நிறைவு திசையை கொண்டு செல்லுங்கள், மாதிரியை பொருத்தமான பதில்களுக்கு வழிநடத்துங்கள்.
- **வொர்க்குகள்** - இவை மாற்றும் வகையான 'செய்முறைகள்' ஆகும், வடிவமைப்புகளில் (மாறி) கையாளக்கூடியவை, குறிப்பிட்ட பயன்பாடுகளுக்கு தரவால் தனிப்பயனாக்கப்படக்கூடியவை.

இவற்றை செயல்பாட்டில் பார்ப்போம்.

### மாதிரிகள் பயன்பாடு

இது பிரதான உள்ளடக்கத்தில் கட்டமைப்புக்கேற்ற மாதிரிகளை வழங்கி, விரும்பிய வெளியீட்டின் மாதிரியை மாதிரி தெரிந்துகொள்ள அனுமதிக்கும் ஒரு முறையாகும். கொடுக்கப்பட்ட மாதிரிகளின் எண்ணிக்கையைப் பொருத்து, சீரற்ற ஒற்றை எழுத்துப்்படுத்தல், ஒரு-ஒற்றை எழுத்துப்்படுத்தல், சில-ஒற்றை எழுத்துப்்படுத்தல் என வகைப்படுத்தலும் செய்யலாம்.

### Prompt Templates

ஒரு prompt வார்ப்புரு என்பது ஒரு _முன்னிருக்கும் குறிப்பிட்ட விதி_ ஆகும், அது தேவையானபோது சேமித்து மீண்டும் பயன்படுத்தக்கூடியது, மிகுந்த ஒரேசமமாக பயனர் அனுபவங்களை உருவாக்குவதற்கு. மிகவும் எளிமையான வடிவில், இது [OpenAIஇல் இதுவரை உள்ள உதாரணம்](https://cookbook.openai.com/examples/gpt4-1_prompting_guide?WT.mc_id=academic-105485-koreyst) போன்று சில prompt எடுத்துக்காட்டுகளின் தொகுப்பாகும், இது தொடர்புடைய prompt கூறுகள் (பயனர் மற்றும் அமைப்பு செய்திகள்) மற்றும் API வழங்கும் கோரிக்கை வடிவம் ஆகியவற்றைக் கொண்டுள்ளது – இது மீண்டும் பயன்படுத்த உதவும்.

அது மிகவும் சிக்கலான வடிவத்தில் [LangChainஇல் இந்த உதாரணம்](https://python.langchain.com/docs/concepts/prompt_templates/?WT.mc_id=academic-105485-koreyst) போல் உள்ளது, அது பல்வேறு மூலங்களிலிருந்து தரவை (பயனர் உள்ளீடு, அமைப்பு சூழல், வெளிப்புற தரவுகள் போன்றவை) பயன்படுத்தி _placeholderஐ_ நிரப்பும் வகையில் உள்ளது, அதனால் prompt ஐ இயக்க முடியும். இதனால் நாம் மீண்டும் பயன்படுத்தக்கூடிய prompt நூலகங்களை உருவாக்க முடியும், அவை **நிகழ்ச்சி வழியாக** ஒரே மாதிரியான பயனர் அனுபவங்களை உருவாக்க உதவுகின்றன.

இறுதியில், வார்ப்புருக்களின் உண்மையான மதிப்பெண் வரம்பு செயல்துறைகளுக்கான _prompt நூலகங்களை_ உருவாக்கி வெளியிடும் திறனில் உள்ளது - அங்கு prompt வார்ப்புரு தற்போது செயல்துறைக்கு உருவாக்கப்பட்ட சூழலை அல்லது எடுத்துக்காட்டுகளை எடுத்துக்காட்டி பதில்கள் இணக்கமானவையும் துல்லியமானவையும் ஆக இருக்கும். [Prompts For Edu](https://github.com/microsoft/prompts-for-edu?WT.mc_id=academic-105485-koreyst) அடையேற்று, கல்வி துறைக்கான prompts நூலகத்தை திரட்டி, பாட திட்டம், பாடத் திட்ட வடிவமைப்பு, மாணவர் வழிகாட்டுதல் போன்ற முக்கிய நோக்கங்களை வலுப்படுத்துகிறது.

## ஆதரவு உள்ளடக்கம்

நாம் prompt கட்டமைப்பை ஒரு படிவுறுதி (பணி) மற்றும் இலக்கு (முதன்மை உள்ளடக்கம்) எனக் கருத்து கொண்டால், _இரண்டாம் நிலை உள்ளடக்கம்_ என்பது வெளியீட்டை **சில வகைகளில் செம்மைப்படுத்த** நாம் அளிக்கும் கூடுதல் சூழல் போன்றது. இது அமைப்புக் கொள்கைகள், வடிவ அமைப்புக் கொள்கைகள், பொருள் வர்க்கங்கள் போன்றவையாக இருக்க முடியும், இது மாதிரியை தனக்கேற்றவாறு பதிலளிக்க வழிகாட்டும்.

உதாரணத்திற்கு: ஒரு பாடக் குறிப்பு புத்தகத்தில் உள்ள அனைத்துப் பாடங்களின் விரிவான மெட்டாடேட்டா (பெயர், விளக்கம், அளவு, மெட்டாடேட்டா குறிச்சொற்கள், பயிற்பவர், முதலியன) இருந்தால்:

- நமது நடவடிக்கை "Fall 2023 பாடக் குறிப்பு 요약" ஆக வரையறுக்கலாம்
- முதன்மை உள்ளடக்கம் ஆக விருப்பப்பட்டது உள்ளடக்கங்களை சில எடுத்துக்காட்டுகளுடன் வழங்கலாம்
- இரண்டாம் நிலை உள்ளடக்கம் ஆக சிறு 5 "குறிச்சொற்களை" அடையாளம் காணலாம்.

இப்போது மாதிரி கொடுக்கப்பட்ட சில எடுத்துக்காட்டுகள் போன்று 요약 வழங்கும் - ஆனால் முடிவில் பல குறிச்சொற்கள் இருந்தால், இரண்டாம் நிலை உள்ளடக்கம் அளித்த 5 குறிச்சொற்கள் முதன்மையாக்கப்படும்.

## Prompting சிறந்த நடைமுறைகள்

இப்போது நாம் prompt-களை எப்படி _கட்டமைக்கலாம்_ என்பதை அறிவதால், அவற்றை சிறந்த நடைமுறைகளுக்கு ஏற்ப எப்படி _வடிவமைக்கலாம்_ என்று நினைக்கலாம். இதைக்கே இரண்டு பகுதிகளாக பார்க்கலாம் - சரியான _மனப்பான்மையை_ கொண்டிருத்தல் மற்றும் சரியான _தொழில்நுட்பங்களை_ பயன்படுத்துதல்.

### Prompt Engineering மனப்பான்மை

Prompt Engineering என்பது முயற்சி-தவறு சுவாசமாகும், ஆகையால் மூன்று பொதுவான வழிகாட்டும் அம்சங்களை மனதில் வைக்கவும்:

1. **துறை புரிதல் முக்கியம்.** பதிலின் துல்லியம் மற்றும் பொருத்தம் அந்த செயலி அல்லது பயனர் இயங்கும் _துறையின்_ ஒரு செயல்திறன் ஆகும். உளறல் மற்றும் துறை நிபுணத்துவத்தைப் பயன்படுத்தி **துறைக்கேற்ற நுட்பங்களை** மேலும் தனிப்பயனாக்கவும். உதாரணத்திற்கு, உங்கள் அமைப்பு promptகளில் _துறைக்கு தனிப்பட்ட குணாதிசயங்கள்_ அமைக்கவும், அல்லது பயனர் promptகளில் _துறைக்கு சிறந்த வார்ப்புருக்கள்_ பயன்படுத்து. துறைக்கு உட்பட்ட சூழல்களை பிரதிபலிக்கும் இரண்டாம் நிலை உள்ளடக்கத்தை வழங்கவும், அல்லது _துறைக்கு உட்பட்ட குறிகள் மற்றும் எடுத்துக்காட்டுகளால்_ மாதிரியை வழிநடத்தவும்.

2. **மாதிரி புரிதல் முக்கியம்.** மாதிரிகள் இயல்பாக ஸ்டோகாஸ்டிக் என்பதை நாம் அறிந்தோம். ஆனால் மாதிரி நடைமுறைகள் பயன்படுத்தும் பயிற்சி தரவுத்தொகுப்பு (முன்னேற்பு அறிவு), வழங்கும் திறன்கள் (API அல்லது SDK மூலம்) மற்றும் கவனிக்கப்பட்ட உள்ளடக்கம் வகை (உதா., குறியீடு, படங்கள், உரை) போன்றவை மாறுபடலாம். நீங்கள் பயன்படுத்தும் மாதிரியின் வலிமைகளும் அவற்றின் கூர்மைகளையும் புரிந்துகொண்டு, அவற்றின் திறனுக்கு ஏற்ற வகையில் ஏற்றுக் கொள்ளும் பணிகள் மற்றும் தனிப்பயன் வார்ப்புருக்களை முன்னுரிமை அளிக்கவும்.

3. **மறுபிரதி மற்றும் சரிபார்ப்பு முக்கியம்.** மாதிரிகள் விரைவாக வளர்ந்து வருகின்றன, அதே போல prompt engineering நுட்பங்களும். ஒரு துறை நிபுணராக, உங்களுக்கு உங்கள் தனிப்பட்ட செயலிக்கான கூடுதல் சூழல் அல்லது அளவுகோல்களுடன் கூட இருக்க வாய்ப்பு உள்ளது, பொதுவான சமூகத்தில் பொருந்த கூடாது. prompt engineering கருவிகள் மற்றும் நுட்பங்களை பயன்படுத்தி prompt கட்டமைப்பை "ஆரம்பிக்கவும்", பிறகு உங்கள் பாதுகாப்பான எண்ணங்கள் மற்றும் துறை நிபுணத்துவத்தைப் பயன்படுத்தி மறுபிரதி மற்றும் சரிபார்ப்பு செய்யவும். உங்கள் கருத்துக்களை பதிவு செய்து ஒரு **அறிவு தளம்** (உதா., prompt நூலகங்கள்) உருவாக்கவும், இது மற்றவர்கள் விரைவான மறுபிரதிகளுக்கு பயன்படுத்தக்கூடிய புதிய அடிப்படையாக இருக்கும்.

எங்களது பணிக்காக, Jupyter குறிப்பு புத்தகத்தில் உள்ள பயிற்சிகளை நேரடி முறையில் முடிக்கலாம். நீங்கள் ஒற்றுமையற்ற உங்கள் சொந்த Markdown மற்றும் குறியீடு புள்ளிகளுடன் குறிப்பு புத்தகத்தை விரிவாக்கி உங்களுக்கான எண்ணங்களை முயற்சிக்கலாம்.

## அறிவு சரிபார்ப்பு

பின்வரும்களில் எது சிறந்த prompt ஆகும்?

1. எனக்கு சிவப்பு காரின் படத்தை காட்டுக

3. எனக்கு Volvo வகை XC90 என்ற மாதிரி கொண்ட சிவப்பு காரின் படத்தை காட்டுக

பதில்: 2, இது சிறந்த prompt, ஏனெனில் "என்ன" என்பதையும், குறிப்பிட்ட மாதிரி மற்றும் வகை என்பதையும் மற்றும் சூழலை விவரிக்கும். 3 அடுத்து ஏற்கனவே விரிவான விளக்கம் கொண்டதாகும்.
