## பொருளடக்கத் தேடு என்பது என்ன?

இப்போது நீங்கள் பொருளடக்கத் தேடு என்னவென்று கேள்வி கேட்கலாம். பொருளடக்கத் தேடு என்பது, ஒரு கேள்வியில் உள்ள சொற்களின் பொருளைக் (semantics) பயன்படுத்தி பொருத்தமான முடிவுகளை வழங்கும் தேடல் தொழில்நுட்பம்.

இதோ ஒரு பொருளடக்கத் தேடு உதாரணம். நீங்கள் ஒரு கார் வாங்க விரும்பினால், 'என் கனவு கார்' என்று தேடினீர்கள் எனக் கொள்ளுங்கள். பொருளடக்கத் தேடு, நீங்கள் ஒரு காரைக் கனவுக் காண்கிறதாக அல்ல; நீங்கள் வாங்கவேண்டிய 'உங்களுக்கான சிறந்த' கார் என்று புரிந்து கொள்ளும். இதற்கு பதிலாக முக்கியச்சொல் தேடு (keyword search) கனவு மற்றும் கார்கள் போன்ற சொற்களைத் literal-ஆக தேடி தொடர்பற்ற முடிவுகளை தரும்.

## Text Embeddings என்றால் என்ன?

[Text embeddings](https://en.wikipedia.org/wiki/Word_embedding?WT.mc_id=academic-105485-koreyst) என்பது [நிலையான மொழி செயலாக்கத்தில்](https://en.wikipedia.org/wiki/Natural_language_processing?WT.mc_id=academic-105485-koreyst) பயன்படுத்தப்படும் ஒரு உரை பிரதிநிதித்துவ தொழில்நுட்பம். Text embeddings என்பது உரையின் எண்ணியல் பொருளடக்க பிரதிநிதித்துவங்கள். Embeddings என்பது கணினிக்கு எந்த உண்மையான தரவு என்பதை எளிதில் புரியவைக்கும் விதத்தில் பிரதிநிதித்துவப்படுத்தப்படுகிறது. பல மாதிரிகள் உள்ளன, இந்த பாடத்தில், OpenAI Embedding மாதிரியைப் பயன்படுத்தி embeddings உருவாக்குவதில் கவனம் செலுத்துகிறோம்.

இதோ ஒரு உதாரணம், AI Show YouTube சேனலின் ஒரு பகுதியின் உரைக்கு இடைக்காலமாக கீழ்காணுமாறு இருக்கலாம்:

```text
Today we are going to learn about Azure Machine Learning.
```

உரையை OpenAI Embedding API-க்கு அனுப்பினால், 1536 எண்கள் கொண்ட ஒரு embedding (வரிசை) தரப்படும். வரிசையில் உள்ள ஒவ்வொரு எண் உரையின் வெவ்வேறு அம்சத்தைக் குறிக்கிறது. சுருக்கமாக முதலில் 10 எண்களை காண்க.

## Embedding குறியீடு எப்படி உருவாக்கப்படுகிறது?

இந்த பாடத்திற்கான Embedding குறியீடு பல Python ஸ்கிரிப்டுகளைக் கொண்டு உருவாக்கப்பட்டது. இந்த பாடத்திற்கான 'scripts' foldரில் உள்ள [README](./scripts/README.md?WT.mc_id=academic-105485-koreyst) யில் ஸ்கிரிப்ட்கள் மற்றும் հրահանգங்களைக் காணலாம். இந்த ஸ்கிரிப்ட்களை இயக்குவதன் அவசியம் இல்லை, ஏனென்றால் Embedding குறியீடு உங்களுக்கு வழங்கப்பட்டுள்ளது.

ஸ்கிரிப்ட்கள் பின்வரும் செயல்களைச் செய்கின்றன:

1. [AI Show](https://www.youtube.com/playlist?list=PLlrxD0HtieHi0mwteKBOfEeOYf0LJU4O1) பிளேலிஸ்டில் உள்ள ஒவ்வொரு YouTube வீடியோவின் உரையை பதிவிறக்கம் செய்கின்றது.
2. [OpenAI Functions](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/function-calling?WT.mc_id=academic-105485-koreyst) பயன்படுத்தி, YouTube உரையின் முதலாவது 3 நிமிடங்களில் பேச்சாளரை அகற்ற முயற்சிக்கிறது. ஒவ்வொரு வீடியோவுக்கும் பேச்சாளர் பெயர் 'embedding_index_3m.json' என்ற Embedding குறியீட்டில் சேமிக்கப்படுகிறது.
3. உரை 3 நிமிட உரை பகுதிய்களாக பகுக்கப்படுகிறது. அடுத்த பகுதியிலிருந்து சுமார் 20 சொற்கள் மோதிக் கொண்டிருக்கின்றன, அதனால் பகுதி embedding குறைவு அடையாது மற்றும் தேடல் பொருள் Context மேம்படும்.
4. ஒவ்வொரு உரை பகுதியும் OpenAI Chat API-க்கு அனுப்பி, 60 சொற்களுக்குள் சுருக்கப்படுகிறது. சுருக்கமும் 'embedding_index_3m.json' என்ற Embedding குறியீட்டில் சேர்க்கப்படுகிறது.
5. இறுதியாக, உரை பகுதி OpenAI Embedding API-க்கு அனுப்பப்படுகிறது. Embedding API பகுதியின் பொருள் உடைய 1536 எண்களின் வெக்டர் (vector) தருகிறது. பகுதி மற்றும் அது உடைய வெக்டர் 'embedding_index_3m.json' என்ற Embedding குறியீட்டில் சேமிக்கப்படுகின்றன.

### வெக்டர் தரவுத்தளங்கள்

எளிமைக்காக, Embedding குறியீடு `embedding_index_3m.json` என்ற JSON கோப்பில் சேமிக்கப்பட்டு Pandas DataFrame-க்கு ஏற்றப்படுகிறது. ஆனால் உற்பத்தியில் Embedding குறியீடு [Azure Cognitive Search](https://learn.microsoft.com/training/modules/improve-search-results-vector-search?WT.mc_id=academic-105485-koreyst), [Redis](https://cookbook.openai.com/examples/vector_databases/redis/readme?WT.mc_id=academic-105485-koreyst), [Pinecone](https://cookbook.openai.com/examples/vector_databases/pinecone/readme?WT.mc_id=academic-105485-koreyst), [Weaviate](https://cookbook.openai.com/examples/vector_databases/weaviate/readme?WT.mc_id=academic-105485-koreyst) போன்ற வெக்டர் தரவுத்தளங்களில் சேமிக்கப்படும்.

## கோசைன் ஒத்திசைவு புரிதல்

Text embeddings பற்றியதைக் கற்றுக்கொண்டோம், அடுத்து text embeddings-ஐத் தரவுக்கான தேடலில் எவ்வாறு பயன்படுத்துவது மற்றும் குறிப்பாக கொடுத்த கேள்விக்கான மிகச் சமான embedding-ஐ கோசைன் ஒத்திசைவு (cosine similarity) மூலம் எப்படி கண்டுபிடிப்பது என்பதைக் கற்றுக்கொள்ளப் போகிறோம்.

### கோசைன் ஒத்திசைவு என்றால் என்ன?

கோசைன் ஒத்திசைவு என்பது இரு வெக்டர்களுக்கு இடையேயான ஒத்திசைவின் அளவீடு, இதைக் `சரியான அடுத்த அயல் தேடல்` (nearest neighbor search) என்றும் கூறுவர். கோசைன் ஒத்திசைவு தேடலை செய்ய, OpenAI Embedding API-ஐப் பயன்படுத்தி கேள்வி உரையை வெக்டர் ஆக்க வேண்டும். பிறகு அந்த கேள்வி வெக்டரும் Embedding குறியீட்டில் உள்ள ஒவ்வொரு வெக்டருடனுமுள்ள கோசைன் ஒத்திசைவையும் கணக்கிட வேண்டும். வாழ்த்துகளுக்கு, Embedding குறியீடு ஒவ்வொரு YouTube உரை பகுதிக்கும் வெக்டரைக் கொண்டுள்ளது. கடைசியாக, முடிவுகளை கோசைன் ஒத்திசைவு அளவைக் கொண்டு வரிசைப்படுத்தி, மிக அதிக கோசைன் ஒத்திசைவு கொண்ட உரை பகுதிகள் கேள்விக்குள் மிகச் சரியானவை.

கணித ரீதியிலாக, கோசைன் ஒத்திசைவு என்பது பலஅளவிலான இடத்தினுள் இரண்டு வெக்டர்களுக்கிடையேயான கோணத்தின் கோசைனை அளவிடுகிறது. இது பயனுள்ளதாகும், ஏனென்றால் இரண்டு ஆவணங்கள் Euclidean தூரம் மூலம் பெரிய விதிவிலக்குடன் இருக்கிறாய் இருந்தாலும், அவற்றுக்கிடையேயான கோணம் சிறியதாகவும், ஆகவே கோசைன் ஒத்திசைவு அதிகமாக இருக்க முடியும். கோசைன் ஒத்திசைவு சமன்பாடுகள் பற்றி விரிவான தகவல் காண [Cosine similarity](https://en.wikipedia.org/wiki/Cosine_similarity?WT.mc_id=academic-105485-koreyst).

## உங்கள் முதலாவது தேடு பயன்பாட்டை உருவாக்குதல்

அடுத்து, Embeddings பயன்படுத்தி ஒரு தேடு பயன்பாட்டை உருவாக்க எவ்வாறு செய்வது என்று கற்றுக்கொள்வோம். தேடு பயன்பாடு மாணவர்கள் கேள்வி எழுத்து மூலம் வீடியோ தேட உதவும். கேள்விக்கு சம்பந்தப்பட்ட வீடியோ பட்டியலை வழங்கும். மேலும் கேள்விக்கு பதில் உள்ள வீடியோ இடத்திற்கு இணைப்பையும் வழங்கும்.

இந்த தீர்வு Windows 11, macOS, மற்றும் Ubuntu 22.04-இல் Python 3.10 அல்லது அதற்குப் பிறகான பதிப்புகள் பயன்படுத்தி கட்டப்பட்டு சோதிக்கப்பட்டது. Python-ஐ [python.org](https://www.python.org/downloads/?WT.mc_id=academic-105485-koreyst) இல் இருந்து தரவிறக்கம் செய்யலாம்.

## பணியமைப்பு - மாணவர்கள் பயன்படுத்தச் தேடு பயன்பாட்டை உருவாக்குதல்

பாடத்திற்குத் தொடக்கத்தில் நமது ஸ்டார்ட்அப்பை அறிமுகப்படுத்தினோம். இப்போது மாணவர்கள் தங்கள் மதிப்பீடுகளுக்கான தேடு பயன்பாட்டை உருவாக்கக்கூடிய நேரம்.

இந்த பணியமைப்பில், தேடு பயன்பாட்டை உருவாக்க பயன்படுத்தப்படும் Azure OpenAI சேவைகளை நீங்கள் உருவாக்குவீர்கள். பின்வரும் Azure OpenAI சேவைகளை உருவாக்க வேண்டும். இந்த பணியமைப்பைப் முழுமையாக செய்ய Azure சந்தா ஒன்றை வைத்திருக்க வேண்டியது அவசியம்.

### Azure Cloud Shell-ஐ துவங்குக

1. [Azure போர்டல்](https://portal.azure.com/?WT.mc_id=academic-105485-koreyst) இல் சாய் செய்யவும்.
2. Azure போர்டலின் மேல் வலது மூல பகுதியில் உள்ள Cloud Shell ஐகானைத் தேர்ந்தெடுக்கவும்.
3. சூழலை வகை ஆக **Bash** ஐ தேர்வு செய்யவும்.

#### ஒரு resource group உருவாக்குதல்

> இந்த வழிகாட்டுதல்களுக்கு, East US இல் உள்ள "semantic-video-search" என்ற resource group-ஐ நாம் பயன்படுத்துகிறோம்.
> நீங்கள் resource group பெயரை மாற்றலாம், ஆனால் கோர்சிகள் இருப்பிடத்தை மாற்றும்போது,
> [மாதிரி கிடைக்கும் அட்டவணை](https://aka.ms/oai/models?WT.mc_id=academic-105485-koreyst) ஐச் சரிபார்க்கவும்.

```shell
az group create --name semantic-video-search --location eastus
```

#### Azure OpenAI சேவை resource உருவாக்குதல்

Azure Cloud Shell-இல், பின்வரும் கட்டளையை இயக்கி Azure OpenAI சேவை resource உருவாக்கவும்.

```shell
az cognitiveservices account create --name semantic-video-openai --resource-group semantic-video-search \
    --location eastus --kind OpenAI --sku s0
```

#### இந்த பயன்பாட்டில் பயன்படுத்த வேண்டும் என endpoint மற்றும் விசைகளை பெறுதல்

Azure Cloud Shell-இல், பின்வரும் கட்டளைகளை இயக்கி Azure OpenAI சேவை resource-க்கான endpoint மற்றும் விசைகளைப் பெறவும்.

```shell
az cognitiveservices account show --name semantic-video-openai \
   --resource-group  semantic-video-search | jq -r .properties.endpoint
az cognitiveservices account keys list --name semantic-video-openai \
   --resource-group semantic-video-search | jq -r .key1
```

#### OpenAI Embedding மாதிரியை இயக்குதல்

Azure Cloud Shell-இல், OpenAI Embedding மாதிரியை இயக்க பின்வரும் கட்டளையை இயக்கவும்.

```shell
az cognitiveservices account deployment create \
    --name semantic-video-openai \
    --resource-group  semantic-video-search \
    --deployment-name text-embedding-ada-002 \
    --model-name text-embedding-ada-002 \
    --model-version "2"  \
    --model-format OpenAI \
    --sku-capacity 100 --sku-name "Standard"
```
