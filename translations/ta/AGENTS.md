# AGENTS.md

## திட்ட கண்ணோட்டம்

இந்த சேமிப்பகம் ஸ்ரீஜன AI அடிப்படைகள் மற்றும் பயன்பாட்டு மேம்பாடு பற்றி 21 பாடங்களைக் கொண்ட விரிவான பாடத்திட்டத்தை கொண்டுள்ளது. இந்த பாடநெறி தொடக்க நிலை பயனாளர்களுக்காக வடிவமைக்கப்பட்டு, அடிப்படை கருத்துக்களிலிருந்து தயாரிப்பு தயாராகும் செயலிகளுக்குப் போதுமான அனைத்தையும் உள்ளடக்கியது.

**முக்கிய தொழில்நுட்பங்கள்:**
- Python 3.9+ மற்றும் பைபிரரிகள்: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript with Node.js மற்றும் பைபிரரிகள்: `openai` (Azure OpenAI வி1 எண்ட்பாயிண்ட் மற்றும் Responses API வழியாக), `@azure-rest/ai-inference` (Microsoft Foundry மாதிரிகள்)
- Azure OpenAI சேவை, OpenAI API மற்றும் Microsoft Foundry மாதிரிகள் (GitHub மாதிரிகள் 2026 ஜூலை முடிவில் இழக்கப்படுகிறது)
- Jupyter நோட்புக்குகள் இன்டர்ஐக்டிவ் கல்விக்காக
- ஒரே மாதிரியான மேம்பாட்டு சூழலுக்கான Dev Containers

**சேமிப்பக அமைப்பு:**
- 21 எண்கள் கொண்ட பாடத் துறைகள் (00-21) இதில் READMEகள், குறியீட்டு எடுத்துக்காட்டுகள் மற்றும் பணிகள் உள்ளன
- பல அமலாக்கங்கள்: Python, TypeScript, சில நேரங்களில் .NET எடுத்துக்காட்டுகள்
- 40+ மொழி பதிப்புகளுடன் மொழிபெயர்ப்புகள் அடைவு
- `.env` கோப்பின் மைய அமைப்புச் செயல் (`.env.copy` மாதிரியாக பயன்படுத்தவும்)

## செட்டப் கட்டளைகள்

### ஆரம்ப சேமிப்பக அமைப்பு

```bash
# ரிப்பாசிடரியை கிளோன் செய்யவும்
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# சூழல் வார்ப்புருவை நகலெடுக்கவும்
cp .env.copy .env
# உங்கள் API விசைகளையும் எண்ட்பாயின்ட்களையும் கொண்டு .env ஐத் தொகுக்கவும்
```

### பைதான் சூழல் அமைப்பு

```bash
# மெய்நிகர் சூழலை உருவாக்குக
python3 -m venv venv

# மெய்நிகர் சூழலை இயக்குக
# macOS/Linux இல்:
source venv/bin/activate
# Windows இல்:
venv\Scripts\activate

# சாரா அம்சங்களை நிறுவுக
pip install -r requirements.txt
```

### Node.js/TypeScript அமைப்பு

```bash
# ரூட் நிலை சார்புகள் (ஆவண வேலை கருவிகளுக்கு) நிறுவவும்
npm install

# தனிப்பட்ட பாடம் TypeScript உதாரணங்களுக்கு, குறிப்பிட்ட பாடத்திற்கு செல்லவும்:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Dev Container அமைப்பு (பரிந்துரைக்கப்படுகிறது)

இந்த சேமிப்பகத்தில் GitHub Codespaces அல்லது VS Code Dev Containers க்கான `.devcontainer` அமைப்பு உள்ளது:

1. GitHub Codespaces அல்லது VS Code உடன் Dev Containers விரிவாக்கம் திறக்கவும்
2. Dev Container தானாகவே:
   - `requirements.txt` லிருந்து Python சார்புகளை நிறுவும்
   - பிந்தைய-உருவாக்கும் ஸ்கிரிப்டை இயக்கும் (`.devcontainer/post-create.sh`)
   - Jupyter கர்னலை அமைக்கும்

## மேம்பாட்டு வேலைநடை

### சூழல் மாறிகள்

API அணுகலை தேவைக்கும் அனைத்து பாடங்களும் `.env` இல் வரையறுக்கப்பட்ட சூழல் மாறிகளை பயன்படுத்துகின்றன:

- `OPENAI_API_KEY` - OpenAI API க்கானது
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry இல் Azure OpenAI க்கானது (Azure OpenAI சேவை இப்போது Microsoft Foundry இன் ஒரு பகுதி: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI முடிவுப் புள்ளி URL (Foundry வள முடிவுப் புள்ளி)
- `AZURE_OPENAI_DEPLOYMENT` - உரையாடல் நிறைவு மாதிரி பணியிடப்பெயர் (பாடநெறி இயல்பு: `gpt-5-mini`)
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - எம்பெட்டிங்ஸ் மாதிரி பணியிடப்பெயர் (பாடநெறி இயல்பு: `text-embedding-3-small`)
- `AZURE_OPENAI_API_VERSION` - API பதிப்பு (இயல்பு: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - ஹக்கிங் முகப்பு மாதிரிகளுக்காக
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry மாதிரிகள் முடிவுப் புள்ளி (பல்வேர் வழங்குநர் மாதிரி அடைவு)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry மாதிரிகள் API விசை (`GITHUB_TOKEN` ஓய்வுக்குச் செல்லும் போது மாற்றம்)
- `AZURE_INFERENCE_CHAT_MODEL` - ஒரு காரணமற்ற மாதிரி (உதா: `Llama-3.3-70B-Instruct`) `temperature` எடுத்துக்காட்டுகளுக்குப் பயன்படுத்தப்படுகிறது, ஏனெனில் காரணமுள்ள மாதிரிகள் மாதிரிகட்டுப்பாடுகளை ஆதரிக்காது

### மாதிரி அருவைகள் (முக்கியம்)

- **இயல்புநிலை உரையாடல் மாதிரி `gpt-5-mini`** - தற்போதைய, ஓய்வீடு செய்யப்படாத **காரண** மாதிரி. 2026 ஆம் ஆண்டின் பின்னர் பழைய "மினி" மாதிரிகள் (`gpt-4o-mini`, `gpt-4.1-mini`) ஓய்வு பெற விருப்பியால், பாடத்திட்டம் GPT-5 குடும்பத்தில் ஒருமித்தமாக்கப்பட்டது.
- **காரண மொடில்கள் `temperature` மற்றும் `top_p` ஐ நிராகரிக்கின்றன**, மற்றும் `max_output_tokens` (Responses API)/ `max_completion_tokens` (உரையாடல் நிறைவுகள்) ஐ `max_tokens` க்கு பதிலாக பயன்படுத்துகின்றன. `gpt-5-mini` அழைக்கும் எடுத்துக்காட்டுகளில் `temperature`/`top_p`/`max_tokens` சேர்க்க வேண்டாம்.
- **`temperature`ஐ காட்ட, Microsoft Foundry Models முடிவுப் புள்ளி (`AZURE_INFERENCE_CHAT_MODEL`) மூலம் ஒரு **Llama** மாதிரி (`Llama-3.3-70B-Instruct`) பயன்படுத்தப்படுகின்றது. கருத்து மாதிரிகளை மாதிரிகாட்டி கிரகிப்பின் மாற்றாக கட்டளை பொறியியலுடன் வழிநடத்துங்கள்.
- **நுண்ணறிவுத் திசைமை (பாடம் 18)** `gpt-4.1-mini` ஐ வைத்திருக்கிறது: GPT-5 மட்டும் பலமடங்கு நுண்ணறிவு திசைமை (RFT) ஆதரிக்கிறது, காட்டப்பட்ட மேற்பார்வை நுண்ணறிவு திசைமை (SFT) இல்லை.
- பாடம் 20 (Mistral) மற்றும் பாடம் 21 (Meta) `temperature`/`max_tokens` பயன்படுத்துகின்றன ஏனெனில் அவை Mistral/Llama மாதிரிகளை இலக்கிடுகின்றன.

### மார்க்டவுன் பாணி

- எல்லா URLகளும் `[text](../../url)` வடிவில் இருக்க வேண்டும் மற்றும் கூடுதல் இடங்கள் இல்லை

- படங்கள் `./images` அடைவில் விவரமான பெயர்களுடன் சேமிக்கப்பட்டுள்ளன

- மொழி பெயர்க்கப்பட்ட படங்கள் `translated_images/` அடைவில் சேமிக்கப்படுகின்றன

## சோதனை மற்றும் சரிபார்ப்புகள்

### சமர்ப்பிப்பதற்கு முன் சரிபார்ப்புகள்

இந்த சேமிப்பகம் சரிபார்ப்புகளுக்காக GitHub Actions பயன்படுத்துகிறது. PRகள் சமர்ப்பிப்பதற்கு முன்:

1. **மார்க்டவுன் இணைப்புகளை சரி பார்க்கவும்**:
   ```bash
   # validate-markdown.yml வேலைப்பாட்டு சரிபார்க்கிறது:
   # - உடைந்த தாச்சிய பாதைகள்
   # - பாதைகளில் காணாமல் போன கண்காணிப்பு IDகள்
   # - URLs இல் காணாமல் போன கண்காணிப்பு IDகள்
   # - நாடு மொழிபெயர்ப்பு கொண்ட URLs
   # - உடைந்த வெளிப்புற URLs
   ```

2. **கைவினைப் பரிசோதனை**:
   - பைதான் எடுத்துக்காட்டுகளை சோதியுங்கள்: venv செயல்படுத்தி ஸ்கிரிப்ட்களை இயக்குக
   - TypeScript எடுத்துக்காட்டுகளைச் சோதியுங்கள்: `npm install`, `npm run build`, `npm start`
   - சூழல் மாறிகள் சரியாக வரையறுக்கப்பட்டுள்ளதா என உறுதிப்படுத்துக
   - API விசைகள் குறியீட்டு எடுத்துக்காட்டுகளுடன் செயல்படுகிறதா என பரிசோதிக்கவும்

3. **குறியீட்டு எடுத்துக்காட்டுகள்**:
   - அனைத்து குறியீடும் பிழை இல்லாமல் இயங்குவதை உறுதிசெய்யவும்
   - குறியீடுகள் Azure OpenAI மற்றும் OpenAI API இரண்டிலும் செயல்படுகிறதா பரிசோதியுங்கள்
   - Microsoft Foundry Models ஆதரிக்கப்படுவிடங்களில் எடுத்துக்காட்டுகள் வேலை செய்கிறதா என சரிபார்க்கவும்

### தானாக நடைபெறும் சோதனைகள் இல்லை

இது கல்வி நோக்கத்துக்கு உள்ள சேமிப்பகம் - பாடங்களும் எடுத்துக்காட்டுகளும் உள்ளன. ஒன்றிய சோதனைகள், ஒருங்கிணைப்பு சோதனைகள் செயல்படாமல் உள்ளன. சரிபார்ப்பு பெரும்பாலும்:
- குறியீட்டு எடுத்துக்காட்டுகளின் கைவினைப் பரிசோதனை
- மார்க்டவுன் விவரங்களுக்கான GitHub Actions
- கல்வி உள்ளடக்கத்தின் சமூக மதிப்பீடு

## புல் கோரிக்கை வழிகாட்டிகள்

### சமர்ப்பிப்பதற்கு முன்

1. Python மற்றும் TypeScript இல் குறியீடு மாற்றங்களை சோதியுங்கள் (பயன்பாட்டுக்கு ஏற்ப)
2. மார்க்டவுன் சரிபார்ப்பை இயக்கவும் (புல் கோரிக்கை மீது தானாகவே செயற்படும்)
3. Microsoft ஐ URLகளில் கண்காணிப்பு IDs உள்ளன என உறுதி செய்யவும்
4. சார்ந்த இணைப்புகள் சரியானவையாக உள்ளதா என பரிசோதிக்கவும்
5. படங்கள் சரியாக குறிக்கப்படுகின்றன என்று உறுதிப்படுத்தவும்

### புல் கோரிக்கை தலைப்பு வடிவம்

- விவரமான தலைப்புகளைப் பயன்படுத்தவும்: `[பாடம் 06] பைதான் எடுத்துக்காட்டு பிழையை சரிசெய்` அல்லது `பாடம் 08 க்கான README புதுப்பிக்கவும்`
- தேவையான போது பிரச்சினை எண்களை குறிப்பிடவும்: `Fixes #123`

### புல் கோரிக்கை விளக்கம்

- மாற்றப்பட்டவை மற்றும் காரணங்களை விளக்கவும்
- தொடர்புடைய பிரச்சனைகளுக்கு இணைப்புகளை கொடுக்கவும்
- குறியீடு மாற்றங்களுக்கு எந்த எடுத்துக்காட்டுகள் சோதிக்கப்பட்டன என்பதை குறிப்பிடவும்
- மொழிபெயர்ப்பு புல் கோரிக்கைகளில் அனைத்து கோப்புகளும் முழுமையான மொழிபெயர்ப்புக்கு சேர்க்கப்பட வேண்டும்

### பங்குச் சேர்க்கும் தேவைகள்

- Microsoft CLA க்கு கையெழுத்திடவும் (முதல் PR ஆல் தானாகவே)
- மாற்றங்களை செய்ய முன் சேமிப்பகத்தை உங்கள் கணக்கில் fork செய்யவும்
- ஒவ்வொரு விவரமான மாற்றத்திற்கும் தனித்தனியான புல் கோரிக்கைகள் (இணைக்கப்பட்ட திருத்தங்களைச் சேர்க்க வேண்டாம்)
- சாத்தியமான போது புல் கோரிக்கைகளை சிக்கலில்லாமல் மற்றும் சிறியதாக வைத்திருங்கள்

## பொதுவான வேலைநடைகள்

### புதிய குறியீட்டு எடுத்துக்காட்டைச் சேர்த்தல்

1. உரிய பாடத்துறையை திறக்கவும்
2. `python/` அல்லது `typescript/` உபஅடைவுகளிலில் எடுத்துக்காட்டை உருவாக்கவும்
3. பெயரிடல் நடைமுறையை பின்பற்றவும்: `{provider}-{example-name}.{py|ts|js}`
4. உண்மையான API அடையாளங்களுடன் சோதனை செய்யவும்
5. பாடம் README இல் எந்த புதிய சூழல் மாறிகளையும் ஆவணப்படுத்தவும்

### ஆவணங்களைப் புதுப்பிப்பது

1. பாட துறையின் README.md ஐ மாற்றவும்
2. மார்க்டவுன் வழிகாட்டிகளை பின்பற்றவும் (கண்காணிப்பு IDs, சார்ந்த இணைப்புகள்)
3. மொழிபெயர்ப்புகளை GitHub Actions கையாள்கிறது (கைவினையாக திருத்த வேண்டாம்)
4. எல்லா இணைப்புகளும் சரியானவையாக உள்ளனவா என சோதிக்கவும்

1. சேமிப்பகத்தில் `.devcontainer/devcontainer.json` உள்ளது

## வெளியீடு மற்றும் வெளியிடல்

இது ஒரு கற்றல் சேமிப்பகம் - வெளியீட்டு செயல்முறை இல்லை. பாடத்திட்டம் கீழ்கண்டவற்றால் கையாளப்படுகிறது:

2. **GitHub Codespaces**: முன்தயார்ப்பான விரிவாக்க சூழலுடன் உடனடி மேம்பாட்டு சூழல்
3. **Microsoft Learn**: உள்ளடக்கம் அதிகாரப் பயிற்சி வலைத்தளத்திற்கு பகிரப்படலாம்
4. **docsify**: மார்க்டவுன் மூலம் கட்டிய ஆவண தளம் (நோக்கு `docsifytopdf.js` மற்றும் `package.json`)

```bash
npm run convert
```

## பிரச்சனைகள் தீர்க்கும் போது

### பொதுவான பிரச்சனைகள்

**Python இறக்குமதி பிழைகள்**:
- பாவனைச் சூழல் செயல்படுத்தப்பட்டிருக்க வேண்டும்

- Python பதிப்பு 3.9+ ஆக இருக்க வேண்டும் என சரிபார்க்கவும்

**TypeScript கட்டுமான பிழைகள்**:
- குறிப்பிட்ட செயலியில் `npm install` இயக்கவும்
- Node.js பதிப்பு பொருந்துமா என சரிபார்க்கவும்
- `node_modules` அழித்து மறுபதிப்பிக்கவும் தேவையெனில்

**API அங்கீகாரம் பிழைகள்**:
- `.env` கோப்பு உள்ளது மற்றும் சரியான மதிப்புகளுடன் இருக்கிறது என பரிசோதிக்கவும்
- API விசைகள் செல்லுபடியாகும் மற்றும் காலாவதியாகவில்லை என்பதை உறுதிசெய்யவும்
- உங்கள் பிராந்தியத்திற்கான எண்ட்பாயிண்ட் URLகள் சரியானதா என சரிபார்க்கவும்

**மாற்றி மதிப்புகள் இன்றி**:
- `.env.copy` ஐ `.env` என நகலெடுக்கவும்
- நீங்கள் பணியாற்றும் பாடத்திற்கான அனைத்து தேவை மதிப்புகளையும் நிரப்பவும்
- `.env` புதுப்பித்த பின் உங்கள் செயலியை மறுதொடக்கம் செய்யவும்
