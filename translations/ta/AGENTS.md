# AGENTS.md

## திட்டத்துக்கான அவலோகனம்

இந்த களவியல் தொகுப்பு ஜெனரேட்டிவ் AI அடிப்படைகள் மற்றும் பயன்பாடு மேம்பாட்டை கற்பிக்கும் 21 பாடங்களோடும் கூடிய முழுமையான பாடத்திட்டத்தை கொண்டுள்ளது. இந்த பாடநிகழ்ச்சி ஆரம்ப பயனர்களுக்காக வடிவமைக்கப்பட்டு, அடிப்படை கருத்துக்களிலிருந்து தயாரிப்பு தயாரிப்பு பயன்பாடுகளை கட்டமைப்பதற்கான அனைத்தையும் உள்ளடக்கியது.

**முக்கிய தொழிற்நுட்பங்கள்:**
- Python 3.9+ மற்றும் நூலகங்கள்: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript மற்றும் Node.js நூலகங்கள்: `openai` (Azure OpenAI v1 endpoint மற்றும் Responses API மூலம்), `@azure-rest/ai-inference` (Microsoft Foundry மாதிரிகள்)
- Azure OpenAI சேவை, OpenAI API மற்றும் Microsoft Foundry மாதிரிகள் (GitHub Models ஜூலை 2026 முடிய விலகும்)
- இடைக்கால கற்றலைக்கான Jupyter நோட்புக்குகள்
- இடைநிலை மேம்பாட்டுக்கான Dev Containers

**களவியல் அமைப்பு:**
- 21 எண்ணிக்கை கொண்ட பாடம் கோப்புறைகள் (00-21), README கோப்புகள், குறியீடு உதாரணங்கள் மற்றும் பணிகள் கொண்டவை
- பல செயலாக்கங்கள்: Python, TypeScript மற்றும் சில நேரங்களில் .NET உதாரணங்கள்
- 40+ மொழி பதிப்புகளுடன் மொழிபெயர்ப்புகள் கோப்புறை
- `.env` கோப்பில் மையப்படுத்தப்பட்ட கட்டமைப்பு (`.env.copy` டெம்ப்ளேட் ஆக பயன்படுத்தவும்)

## அமைப்பு கட்டளைகள்

### ஆரம்ப களவியல் அமைப்பு

```bash
# களஞ்சியத்தை நகலெடுக்கவும்
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# சுற்றுச்சூழல் சுருக்கத்தை நகலெடுக்கவும்
cp .env.copy .env
# உங்கள் API விசைகள் மற்றும் இடைமுகங்களை கொண்டு .env ஐ தொகுக்கவும்
```

### Python சூழல் அமைப்பு

```bash
# மெய்நிகர் சூழலை உருவாக்கவும்
python3 -m venv venv

# மெய்நிகர் சூழலை இயக்கவும்
# macOS/Linux இல்:
source venv/bin/activate
# Windows இல்:
venv\Scripts\activate

# சாரா பணிகளை நிறுவவும்
pip install -r requirements.txt
```

### Node.js/TypeScript அமைப்பு

```bash
# ரூட்-நிலை சார்பு பொருட்களை நிறுவவும் (பத்திரிகை சாதனங்களுக்காக)
npm install

# தனிப்பட்ட பாட TypeScript உதாரணங்களுக்கு, குறிப்பிட்ட பாடத்திற்கு செல்லவும்:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Dev Container அமைப்பு (பரிந்துரை செய்யப்படுகிறது)

இந்தக் களவியல் தொகுப்பில் GitHub Codespaces அல்லது VS Code Dev Containers க்கான `.devcontainer` கட்டமைப்பு உள்ளது:

1. GitHub Codespaces அல்லது VS Code உடன் Dev Containers நீட்சியை திறக்கவும்
2. Dev Container தானாகவே:
   - `requirements.txt` இலிருந்து Python சார்புகளை நிறுவும்
   - உருவாக்கப்பட்ட பிறகு ஸ்கிரிப்ட் (`.devcontainer/post-create.sh`) இயங்கும்
   - Jupyter கர்னல் அமைக்கும்

## மேம்பாட்டு பணிச்சூழல்

### சூழல் மாறிகள்

API அணுகல் தேவைப்படும் அனைத்து பாடங்களும் `.env`ல் வரையறுக்கப்பெற்ற சூழல் மாறிகள் பயன்படுத்துகின்றன:

- `OPENAI_API_KEY` - OpenAI API க்கான
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry உள்ள Azure OpenAIக்கான (`Azure OpenAI சேவை இப்போ Microsoft Foundry ஒரு பகுதியாக உள்ளது: https://ai.azure.com`)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI இடைமுக URL (Foundry வள இடைமுகம்)
- `AZURE_OPENAI_DEPLOYMENT` - பேச்சு முடித்தல் மாதிரி நிலுவைப் பெயர்
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - இடைநிறைத்தல் மாதிரி நிலுவைப் பெயர்
- `AZURE_OPENAI_API_VERSION` - API பதிப்பு (இயல்புநிலை: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face மாதிரிகள் க்கான
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry மாதிரிகள் இடைமுகம் (பல வழங்குநர்கள் மாதிரி பட்டியல்)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry மாதிரிகள் API முக்கியம் (மறைந்துவரும் `GITHUB_TOKEN` ஐ மாற்றும்)

## குறியீடு பாணி வழிகாட்டுதல்கள்

### Python

- சூழல் மாறி மேலாண்மைக்கு `python-dotenv` ஐ பயன்படுத்தவும்
- API தொடர்பிற்கான `openai` நூலகத்தை இறக்குமதி செய்வது
- குறியீடு ஆய்விற்காக `pylint` ஐ பயன்படுத்தவும் (சில உதாரணங்களில் எளிமைக்காக `# pylint: disable=all` உள்ளது)
- PEP 8 பெயரிடும் நடைமுறைகளை பின்பற்றவும்
- API கடவுச்சொற்களை `.env` கோப்பில் சேமிக்கவும், குறியீட்டில் ஒருபோதும் பதிக்க வேண்டாம்

### TypeScript

- சூழல் மாறிகளுக்காக `dotenv` தொகுப்பைப் பயன்படுத்தவும்
- ஒவ்வொரு செயலியில் `tsconfig.json` இல் TypeScript அமைப்பு
- Azure OpenAIக்கான `openai` தொகுப்பைப் பயன்படுத்தவும் (கிளையண்டை `/openai/v1/` இடைமுகத்தில் நோக்கி `client.responses.create` கையடக்கப்படு); Microsoft Foundry மாதிரிகளுக்காக `@azure-rest/ai-inference` ஐ பயன்படுத்தவும்
- auto-reload உடன் மேம்பாட்டுக்கு `nodemon` ஐப் பயன்படுத்தவும்
- இயக்குவதற்கு முன் கட்டமைக்க: `npm run build` பின்னர் `npm start`

### பொது நடைமுறைகள்

- குறியீடு உதாரணங்களை எளிமையாகவும் கல்விப்பண்புடன் வைக்கவும்
- முக்கிய கருத்துக்களை விளக்கும் கருத்துக்களை சேர்க்கவும்
- ஒவ்வொரு பாடத்தின் குறியீடு தனித்துவமாகவும் இயக்கக்கூடியதாகவும் இருக்க வேண்டும்
- ஒரேபோன்ற பெயரிடல்: Azure OpenAI க்கு `aoai-`, OpenAI API க்கு `oai-`, Microsoft Foundry மாதிரிகள் க்கு `githubmodels-` (GitHub Models காலத்திலிருந்து பழக்கப்படுத்திய முன்னோடியைக் கையாளும்)

## ஆவணப்படுத்தல் வழிகாட்டுதல்கள்

### மார்க்டவுன் பாணி

- அனைத்து URLகளும் `[text](../../url)` வடிவத்தில் கூடுதல் இடைவெளி இல்லாமல் கொண்டிருக்க வேண்டும்
- சார்ந்த இணைப்புகள் `./` அல்லது `../` மூலம் தொடங்க வேண்டும்
- Microsoft டொமைன்களுக்கு அனைத்து இணைப்புகளும் தடுப்பமுடைய ஐடியுடன் இருக்க வேண்டும்: `?WT.mc_id=academic-105485-koreyst`
- URLகளில் நாட்டின் குறிப்பிட்ட மொழி படி பகுதிகள் இருக்கக்கூடாது (எ.கா., `/en-us/` தவிர்க்கவும்)
- படங்கள் `./images` கோப்புறையில் விளக்கமுள்ள பெயர்களுடன் சேமிக்கப்படுகின்றன
- கோப்புப் பெயர்களில் ஆங்கில எழுத்துக்கள், எண்கள் மற்றும் கோடுகள் பயன்படுத்தவும்

### மொழிபெயர்ப்பு ஆதரவு

- களவியல் 40+ மொழிகளில் தானியங்கி GitHub Actions மூலம் ஆதரிக்கிறது
- மொழிபெயர்ப்புகள் `translations/` கோப்புறையில் சேமிக்கப்படுகின்றன
- பகுதி மொழிபெயர்ப்புகளை சமர்ப்பிக்க வேண்டாம்
- மெஷின் மொழிபெயர்ப்புகள் ஏற்றுக்கொள்ளப்பட மாட்டாது
- மொழிபெயர்க்கப்பட்ட படங்கள் `translated_images/` கோப்புறையில் சேமிக்கப்படுகின்றன

### சமர்ப்பிக்க முந்தைய சரிபார்ப்புகள்

இக்களவியல் GitHub Actions மூலம் சரிபார்ப்பு செய்கிறது. PRகளை சமர்ப்பிக்கும் முன்:

1. **மார்க்டவுன் இணைப்புகளை சரிபார்க்கவும்**:
   ```bash
   # validate-markdown.yml வேலைப்பாடு சரிபார்க்கிறது:
   # - உடைந்த தொடர்புடைய பாதைகள்
   # - பாதைகளில் காணாமல் போன கண்காணிப்பு IDகள்
   # - URLகளில் காணாமல் போன கண்காணிப்பு IDகள்
   # - நாட்டின் காணொலி உள்ள URLகள்
   # - உடைந்த வெளிப்புற URLகள்
   ```

2. **மற்றும் கைமுறை சோதனை**:
   - Python உதாரணங்களைச் சோதிக்க: venv இயக்கி ஸ்கிரிப்ட்களை இயக்கு

   - API திறவுகோல்கள் குறியீடு உதாரணங்களோடு வேலை செய்வதா என பார்க்கவும்

- கல்வி உள்ளடக்கத்திற்கு சமூகவியல் விமர்சனம்

## புள்ளிவிவர கோரிக்கைகள்

### சமர்ப்பிப்பதற்கு முன்

1. Python மற்றும் TypeScript இரண்டு மொழிகளிலும் குறியீடு மாற்றங்களை சோதிக்கவும், பொருந்துமானால்
2. மார்க்டவுன் சரிபார்ப்பை இயக்கவும் (PRக்கு தானாக இயக்கப்படும்)
3. Microsoft URL அனைவரிலும் தடிப்பு ஐடியுடன் இருப்பதைக் உறுதி செய்யவும்
4. சார்ந்த இணைப்புக்களும் செல்லுபடியாக இருக்க வேண்டும் என்று சரிபார்
5. படங்கள் சரியாகக் குறிக்கப்பட்டுள்ளனவா என உறுதி செய்க

### PR தலைப்பு நடைமுறை

- விளக்கான தலைப்புகளைப் பயன்படுத்தவும்: `[Lesson 06] Python உதாரணத்தில் அச்சுப்பிழை திருத்து` அல்லது `lesson 08க்கான README இன் புதுப்பிப்பு`
- பொருந்துமானால் பிழை எண்களை குறிப்பிடவும்: `Fixes #123`

### PR விளக்கம்

- மாற்றியவையும் காரணங்களையும் விளக்கவும்
- தொடர்புடைய பிழைகளுக்கு இணைப்பு கொடுக்கவும்
- குறியீடு மாற்றங்களுக்கு எந்த உதாரணங்கள் சோதிக்கப்பட்டன என்பதை குறிப்பிடவும்
- மொழிபெயர்ப்பு PRக்காக, முழுமையான மொழிபெயர்ப்புக்கு அனைத்து கோப்புகளும் சேர்க்கப்பட வேண்டும்

### பங்களிப்பு தேவைகள்

- Microsoft CLA இல் கையொப்பமிடவும் (முதற்கட்ட PRக்கு தானாக)
- மாற்றங்கள் செய்யும் முன் களவியலை உங்களின் கணக்கில் Fork செய்யவும்
- ஒவ்வொரு கருத்துக்கு ஒரு PR மட்டும் (பொருளற்ற திருத்தங்களை ஒன்றிணைக்க வேண்டாம்)
- PRகளை சுருக்கமாகவும் கவனம் சென்றவையாகவும் வைக்கவும்

## பொது பணிச்சூழல்கள்

### புதிய குறியீடு உதாரணம் சேர்த்தல்

1. பொருத்தமான பாட கோப்புறைக்கு செல்லவும்
2. `python/` அல்லது `typescript/` உப்கோப்புறையில் உதாரணத்தை உருவாக்கவும்
3. பெயரிடும் நடைமுறையை பின்பற்றவும்: `{provider}-{example-name}.{py|ts|js}`
4. உண்மையான API சொன்னியைப் பயன்படுத்தி சோதனை செய்யவும்
5. பாட READMEல் புதிய சூழல் மாறிகள் குறித்து ஆவணம் செய்யவும்

### ஆவணங்களை புதுப்பித்தல்

1. பாட கோப்புறையில் README.md ஐ தொகுக்கவும்
2. மார்க்டவுன் வழிகாட்டுதலை பின்பற்றவும் (தடிப்பு ஐடியுடன், சார்ந்த இணைப்புகள்)
3. மொழிபெயர்ப்புகள் GitHub Actions மூலம் தானாக சரியாக்கப்படுகிறது (கைமுறை திருத்தம் செய்வதே இல்லை)
4. அனைத்து இணைப்புகளும் செல்லுபடியாக உள்ளனவா என்பதை சோதிக்கவும்

### Dev Containers உடன் பணிபுரிதல்

1. களவியல் `.devcontainer/devcontainer.json` சேர்க்கப்பட்டுள்ளது
2. post-create ஸ்கிரிப்ட் Python சார்புகளை தானாக நிறுவுகிறது
3. Python மற்றும் Jupyter நீட்சிகள் முன்போல் கட்டமைக்கப்பட்டுள்ளன
4. சூழல் `mcr.microsoft.com/devcontainers/universal:2.11.2` அடிப்படையிலுள்ளது

## பொருத்தம் மற்றும் வெளியீடு

இது ஒரு கற்றல் களவியல், வெளியீடு பணிவிளக்கம் இல்லை. பாடத்திட்டம் பயன்படுத்தப்படுவது:

1. **GitHub Repository**: குறியீடு மற்றும் ஆவணங்களுக்கான நேரடி அணுகல்
2. **GitHub Codespaces**: முன்கட்டமைக்கப்பட்டுள்ள உடனடி மேம்பாட்டு சுற்றுச்சூழல்
3. **Microsoft Learn**: உள்ளடக்கம் அதிகாரப்பூர்வ கற்றல் தளத்திற்கு பகிரப்படலாம்
4. **docsify**: மார்க்டவுன் மூலம் கட்டப்பட்ட ஆவண தளம் (`docsifytopdf.js` மற்றும் `package.json` பாருங்கள்)

### ஆவணம் தளத்தை உருவாக்குதல்

```bash
# தேவையானால் ஆவணத்திலிருந்து PDF உருவாக்கவும்
npm run convert
```

## பிரச்சனைகள் தீர்க்கல்

### பொதுவான பிரச்சனைகள்

**Python இறக்குமதி பிழைகள்**:
- மெய்நிகர் சூழல் இயங்குகிறதா என்பதை உறுதி செய்க
- `pip install -r requirements.txt` இயக்கவும்
- Python பதிப்பு 3.9+ இருக்கும்படி சரிபார்

**TypeScript கட்டமைப்பு பிழைகள்**:
- குறிப்பிட்ட செயலியில் `npm install` இயக்கவும்
- Node.js பதிப்பு பொருந்துமா என சரிபார்
- `node_modules` நீக்கி மறுபடியும் நிறுவவும், அவசியமெனில்

**API அங்கீகார பிழைகள்**:
- `.env` கோப்பு இருப்பதும் சரியான மதிப்புகளுடன் இருப்பதும் உறுதி செய்க
- API திறவுகோல்கள் செல்லுபடியாக உள்ளனவா, காலாவதியா என்பதை சரிபார்
- உங்கள் பிராந்தியத்திற்கான இடைமுக URLகள் சரியாக உள்ளனவா என உறுதி செய்க

**சூழல் மாறிகள் இல்லை**:
- `.env.copy` ஐ `.env` ஆக நகலெடுக்கவும்
- நீங்கள் பணியாற்றும் பாடத்திற்கு தேவையான அனைத்து மதிப்புகளையும் நிரப்பவும்
- `.env` تازه செய்த பிறகு உங்கள் பயன்பாட்டை மீட்டெழுதவும்
