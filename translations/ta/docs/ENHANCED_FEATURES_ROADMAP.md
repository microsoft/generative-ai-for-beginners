இந்த ஆவணம், தொழிற்துறை சிறந்த நடைமுறைகளை விரிவான குறியீடு மதிப்பாய்வு மற்றும் பகுப்பாய்வின் அடிப்படையில், தொடக்கத்தரக்கமான செயற்கை நுண்ணறிவு பாடத்திட்டத்துக்கான பரிந்துரைக்கப்பட்ட மேம்பாடுகள் மற்றும் முன்னேற்றங்களை விளக்குகிறது.

## 4. API நவீனம்

### 4.1 பழைய API முறைகள் (மைக்ரேஷன் முடிந்தது)

அனைத்து Python மற்றும் TypeScript **chat** மாதிரிகள் Chat Completions API இருந்து **Responses API** க்கு மாற்றப்பட்டுள்ளன (`client.responses.create(...)` → `response.output_text`).

| பழைய முறை | புதிய முறை | நிலை |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (chat) | `OpenAI(base_url="<endpoint>/openai/v1/")` (Responses API) | முடிந்தது |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | முடிந்தது |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (TypeScript) | `openai` பக்கேஜ் `client.responses.create()` → `response.output_text` | முடிந்தது |
| `df.append()` (pandas) | `pd.concat()` | முடிந்தது |

> **குறிப்பு:** Microsoft Foundry Models மாதிரிகள் `azure-ai-inference` / `@azure-rest/ai-inference` SDK (`client.complete()`) பயன்படுத்தி இருப்பதால் Model Inference API இல் இருந்து Responses API ஆதரவு இல்லை. `AzureOpenAI()` இன்னும் செல்லுபடியாகும் இடங்களில் (embeddings மற்றும் பட உருவாக்கம்) உள்ளதால் பாதுகாக்கப்பட்டது.

### 4.2 புதிய API அம்சங்களை காட்ட

1. **அமைக்கப்பட்ட வெளியீடுகள்** (OpenAI)
   - JSON முறைமை
   - கடுமையான திட்டங்கள் கொண்ட செயல்பாட்டு அழைப்புகள்

2. **தோம்புகள் திறன்கள்**
   - GPT-4o (vision) மூலம் பட ஆய்வு
   - பன்முறை ஊக்கங்கள்

3. **Responses API இல் உள்ள கருவிகள்** (பழைய Assistants API க்குப் பதிலாக)
   - குறியீடு மொழிபெயர்ப்பாளர்
   - கோப்பு தேடல்
   - வலை தேடல் மற்றும் தனிப்பயன் கருவிகள்

## 5. பிணையமைப்பு முன்னேற்றங்கள்

### 5.1 CI/CD மேம்பாடுகள்

[`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml) இல் செயல்படுத்தப்பட்டது: Python linting/formatting (Ruff + Black) `shared/` உதவிக்கருவி தொகுதியில் கட்டாயம் செயல்படுத்து மற்றும் மீதமுள்ள பாடத்திட்டத்தில் ஆலோசனை விதமாக இயங்கும், கூட JavaScript/TypeScript க்கான ESLint ஆலோசனை ஓடும். எடுத்துக்காட்டு அடித்தளம்:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### 5.2 பாதுகாப்பு ஸ்கேன்

[`.github/workflows/security.yml`](../../../.github/workflows/security.yml) இல் செயல்படுத்தப்பட்டது: Python மற்றும் JavaScript/TypeScript க்கான CodeQL பகுப்பாய்வு (push, pull request மற்றும் வாரந்தோறும்) மற்றும் pull request களின் சார்பு ஆய்வும். எடுத்துக்காட்டு அடித்தளம்:

```yaml
# .github/workflows/security.yml
name: Security Scan

on: [push, pull_request]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: javascript, python
      - uses: github/codeql-action/analyze@v3

  dependency-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
```
