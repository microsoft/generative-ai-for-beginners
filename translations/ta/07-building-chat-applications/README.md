# உருவாக்கும் AI ஆற்றலான அதிகபட்ச உரையாடல் பயன்பாடுகள் கட்டமைத்தல்

நாம் உரை-உருவாக்கும் செயலிகள் எப்படி கட்டமைக்கப்படுவதை பார்த்துவிட்டோம், இப்போது உரையாடல் பயன்பாடுகளைக் காண்போம்.

உரையாடல் பயன்பாடுகள் எங்கள் அன்றாட வாழ்விலும் இணைந்துள்ளன, சாதாரண உரையாடலுக்கு மட்டுமே அல்ல. அவை வாடிக்கையாளருக்கு சேவை, தொழில்நுட்ப ஆதரவு மற்றும் தூர்வாரியான ஆலோசனைக் க Sistemandukல் பகுதிகளாகவும் மாறியுள்ளன. நீங்கள் உடனடியாக ஒரு உரையாடல் பயன்பாட்டிலிருந்து உதவி பெற்றிருப்பதாக இருக்க முடியும். உருவாக்கும் AI போன்ற மேம்பட்ட தொழில்நுட்பங்களை இப்பிளாட்ஃபாரங்கள் ஒன்றுகூடச் சேர்த்த போது, சிக்கலானவை அதிகரிக்கும் மற்றும் சவால்களும் அதிகரிக்கும்.

எந்தவென்ற சில கேள்விகள் பதில் தேவை:

- **பயன்பாட்டை கட்டமைத்தல்**. குறிப்பிட்ட பயன்பாடுகளுக்காக இந்த AI ஆற்றலான செயலிகளை எப்படி திறமையான முறையில் கட்டமைத்து ஒட்டிக்கொள்வது?
- **மேற்காணுதல்**. ஒருமுறை வெளியிடப்பட்ட பிறகு, செயலிகள் மிக உயர்ந்த தரத்தில் இயங்குவதை எப்படி கண்காணித்து உறுதிப்படுத்தலாம், செயல்பாடுகளில் மற்றும் [பாதுகாப்பான AI ஐட்டைகள் 6 கொள்கைகள்](https://www.microsoft.com/ai/responsible-ai?WT.mc_id=academic-105485-koreyst) பின்பற்றுவதில்?

ஆட்டோமேஷன் மற்றும் அரைத்தடை மனித-யந்திர பரிமாற்றங்கள் நிர்ணயிக்கும் காலத்திற்கு நாங்கள் அருகில் செல்லும் போது, உருவாக்கும் AI உரையாடல் பயன்பாடுகளின் வரம்பை, ஆழத்தையும், மாறுபாடுகளையும் மாற்றும் விதத்தைப் புரிந்துகொள்வது அவசியமாகிறது. இந்த பாடம் இந்த சிக்கலான அமைப்புகளை ஆதரிக்கும் கட்டமைப்புக் கூறுகளை ஆராய்ந்து, குறிப்பிட்ட துறையில் சிறப்பாக மறு பயிற்சி செய்யும் முறைகளை அடைவதும், பாதுகாப்பான AI பயன்முறை தொடர்பான அளவுகோல்களையும் பரிசீலிக்கும்.

## அறிமுகம்

இந்த பாடம் விவரிக்கும்:

- உரையாடல் பயன்பாடுகளை திறமையாக கட்டமைக்கும் மற்றும் ஒருங்கிணைக்கும் நுட்பங்கள்.
- செயலிகளுக்கு தனிப்பயனாக்குதல் மற்றும் சிறப்புக் குழாய்ப்பாடு (fine-tuning) எப்படி செய்து கொள்ளலாம்.
- உரையாடல் செயலிகளை திறமையாக கண்காணிப்பதற்கான யோசனைகள் மற்றும் உத்திகள்.

## கற்றல் குறிக்கோள்கள்

இந்த பாடம் முடிந்த பிறகு, நீங்கள் முடியும்:

- உரையாடல் பயன்பாடுகளை கட்டமைக்கும் மற்றும் உள்ளமைப்புடன் ஒருங்கிணைக்கும் பொறுப்புக்களை விவரிக்க.
- குறிப்பிட்ட பயன்பாடுகளுக்காக உரையாடல் செயலிகளை தனிப்பயனாக்க.
- AI ஆற்றலான உரையாடல் செயலிகளின் தரத்தைக் கண்காணிக்க முக்கியமான அளவுகோல்கள் மற்றும் கவனிப்புக்களை அடையாளப்படுத்த.
- உரையாடல் செயலிகள் AI ஐ பொறுப்பான முறையில் பயன்படுத்துகிறதா என்பதைக் உறுதிப்படுத்த.

## உரையாடல் பயன்பாடுகளில் உருவாக்கும் AI ஐ ஒருங்கிணைத்தல்

உரையாடல் செயலிகளை உருவாக்கும் AI மூலம் மேம்படுத்துவது அவற்றை மட்டுமல்லாமல் கவனிப்பதற்கான கட்டமைப்பு, செயல்திறன் மற்றும் பயனர் இடைமுகத்தை ஒட்டுமொத்தமான தரத்தை வழங்க சிறந்த முறையில் வடிவமைக்கிறது. இது கட்டமைப்பு அடிப்படைகள், API ஒருங்கிணைப்புகள் மற்றும் பயனர் இடைமுக கவனிப்புகளை ஆராய்வதை உள்ளடக்கியது. இந்த பகுதி உங்களுக்கு இவ்வாறு சிக்கலான சூழல்களை சரியாக அடைவதற்கான முழுமையான வழிகாட்டியை வழங்குவதை நோக்கமாகக் கொண்டது, நீங்கள் இவற்றை உள்ளமைக்கு பொருந்தும் அல்லது தனித்தளங்களாக கட்டமைக்க விரும்பினாலும்.

இந்த பகுதி முடிவில், நீங்கள் உரையாடல் பயன்பாடுகளை திறமையாக உருவாக்கி ஒருங்கிணைக்கும் அறிவைப் பெறுவீர்கள்.

### உரையாடல் பொம்மையா அல்லது உரையாடல் பயன்பாட்டா?

உரையாடல் பயன்பாடுகளை கட்டமைக்க நடவடிக்கை எடுக்க முன்பு, 'உரையாடல் பொம்மைகள்' மற்றும் 'AI ஆற்றலான உரையாடல் பயன்பாடுகள்' என்றவைகளை ஒப்பிடுவோம்; இவை வேறு வேறு பங்கு மற்றும் செயல்பாடுகளைக் கொண்டவை. ஒரு உரையாடல் பொம்மையின் முக்கிய நோக்கம் குறிப்பிட்ட உரையாடல் பணி தானாகச் செயல் படுத்துவதாகும், உதாரணமாக அடிக்கடி கேட்கப்படும் கேள்விகளுக்கு பதில் அளித்தல் கையடக்கப் பொருள் பாடி கண்காணித்தல் போன்றவை. இது பெரும்பாலும் ஒழுங்குமுறை அடிப்படையிலான தற்காலிக சிந்தனை அல்லது சிக்கலான AI கணக்கீடுகள் மூலம் நிர்வகிக்கப்படுகிறது. அதே சமயம், AI ஆற்றலான உரையாடல் பயன்பாடு ஒரு விரிவான சூழலும், உரை, குரல் மற்றும் காணொளி உரையாடலை மனித பயனாளர்களுக்கு வழங்கும் இடமாக இருக்கிறது. இதன் சிறப்பம்சம் ஒரு உருவாக்கும் AI மொடலை ஒருங்கிணைப்பதில் உள்ளது, இது மனிதனோடு ஒப்பிடத்தக்கவாறு நுட்பமான உரையாடல்களை உருவாக்குகிறது, பலவிதமான உள்ளீட்டிலும் சூழல் குறிகளிலும் அடிப்படையாகக் கொண்டு பதில்கள் உருவாக்குகிறது. உருவாக்கும் AI ஆற்றலான உரையாடல் பயன்பாடு திறந்த துறையான உரையாடல்களில் ஈடுபட முடியும், மாறும் உரையாடல் சூழல்களுக்கு தக்கவாறு ஏற்படுத்திக் கொள்ளலாம் மற்றும் evenகடத்துக்கூடிய அல்லது சிக்கலான உரையாடல்களை உருவாக்கலாம்.

கீழே உள்ள அட்டவணை முக்கிய வேறுபாடுகள் மற்றும் ஒத்த தன்மைகளை விளக்குகிறது, அவற்றின் தனித்துவமான பங்குகளை புரிந்துகொள்ள உதவுகிறது.

### SDK கள் மற்றும் API களை கொண்டு முன் கட்டப்பட்டுள்ளது செயல்பாடுகளை பயன்படுத்துதல்

ஒரு உரையாடல் செயலியை கட்டமைக்கும் போது, நீங்கள் முதலில் ஏற்கனவே உள்ளவை என்ன என்று மதிப்பாய்வு செய்தல் சிறந்த ஆரம்பமாகும். SDK கள் மற்றும் API களை பயன்படுத்தி உரையாடல் செயலிகளை கட்டமைப்பது பல காரணங்களுக்காக நல்ல அகற்றுகலான உத்தி. நற்களிக்கப்பட்ட SDK கள் மற்றும் API களை ஒருங்கிணைப்பதன் மூலம் உங்கள் செயலியை நீண்ட கால வெற்றிக்காக இலந்திருப்பதற்கான இடத்தை அமைக்கிறீர்கள், விரிவாக்கத்தன்மையும் பராமரிப்புப் பிரச்சனைகளும் உயர்கின்றன.

- **அமைக்கும் செயலி செயல்முறையை விரைவுபடுத்தி மேலதிக செலவு குறைகிறது**: மேம்படுத்தப்பட்ட செயல்பாடுகளை தராதவாறு நேர்த்தியான முறையில் நீங்கள் கட்டும் செலவை தவிர்க்க முடியும், உங்கள் செயலியின் மற்ற முக்கிய அம்சங்கள், போன்ற வணிக நுண்ணறிவுகள் போன்றவை கவனிக்கலாம்.
- **சிறந்த செயல்திறன்**: செயல்பாடுகளைத் தனது முறையில் கட்டும்போது, நீங்கள் இறுதியில் கேட்கலாம் "இது எவ்வாறு பரந்தளவிலானது? இந்த செயலி பயனர் அதிரடியான வளர்ச்சியை எதிர்கொள்ள முடியும்?" நன்றாக பராமரிக்கப்பட்ட SDK மற்றும் API களில் பல அவசரங்கள் விடயங்களுக்கு முன்ன ஓர் தீர்வுகள் உள்ளன.
- **எளிதான பராமரிப்பு**: புதுப்பிப்புகளுக்கு மற்றும் மேம்படுத்தல்களுக்கு எளிதாக செயல்படக்கூடியவை, பெரும்பாலான API களுக்கும் SDK களுக்கும் புதிய பதிப்பு வெளியிடும் போது ஒரு நூலகத்தை புதுப்பிப்பது மட்டுமே தேவையாகும்.
- **சூட்சுமமான தொழில்நுட்ப அணுகல்**: பன்முகிக் கோப்புகளை மீறி நுண்ணறிவுத்தன்மை உள்ள இயல்பான மொழி திறன்களுடன் மொடல்களைப் பயன்படுத்துதல் உங்கள் செயலிக்கு வழங்குகிறது.

SDK அல்லது API செயல்பாடுகளை அணுகுதலுக்கு பொதுவாக வழங்கப்படும் சேவைகளைப் பயன்படுத்த அனுமதி பெற வேண்டும், இது பொதுவாக தனித்துவமான விசை அல்லது அங்கீகார டோக்கன் மூலம் நிகழும். இதைக் காண்பதற்கு OpenAI Python நூலகத்தை பயன்படுத்துவோம். இந்த பாடத்திற்கு [OpenAIக்கு நோட்புக்](./python/oai-assignment.ipynb?WT.mc_id=academic-105485-koreyst) அல்லது [Azure OpenAI சேவைகளுக்கான நோட்புக்](./python/aoai-assignment.ipynb?WT.mc_id=academic-105485-koreys) உங்கள் தேவைக்கேற்ப முயற்சி செய்யலாம்.

```python
import os
from openai import OpenAI

API_KEY = os.getenv("OPENAI_API_KEY","")

client = OpenAI(
    api_key=API_KEY
    )

chat_completion = client.chat.completions.create(model="gpt-3.5-turbo", messages=[{"role": "user", "content": "Suggest two titles for an instructional lesson on chat applications for generative AI."}])
```

மேலே கொடுக்கப்பட்ட உதাহரம் GPT-3.5 Turbo மொடலை பயன்படுத்தி விசாரணையை முடிக்கிறது, ஆனால் API விசை முன்பே அமைக்கப்பட்டது என்பது கவனிக்கவும். விசையை அமைக்காவிட்டால் பிழை வரும்.

## பயனர் அனுபவம் (UX)

பொதுவான UX கொள்கைகள் உரையாடல் பயன்பாடுகளுக்கு பொருந்தும், ஆனால் இயந்திர கற்றல் கூறுகள் தொடர்பால் குறிப்பாக முக்கியமாகும் கூடுதல் அம்சங்கள் உள்ளன.

- **பிரச்சனை தொலைக்காட்சிக்கு வழிமுறை**: உருவாக்கும் AI மொடல்கள் சில சமயங்களில் பொதுவான பதில்களை உண்டாக்குகின்றன. பயனர்கள் குழப்பப்பட்டால் தெளிவு கேட்க நினைப்பதற்காக அம்சம் உதவும்.
- **சூழலின் நினைவூட்டல்**: மேம்பட்ட உருவாக்கும் AI மொடல்கள் உரையாடல் உள்ளடக்கத்தில் சூழலை நினைவுசெய்யும் திறன் கொண்டவை, இது பயனர் அனுபவத்திற்கு தேவையானது. பயனர்களுக்கு சூழலை கட்டுப்படுத்தும் மற்றும் நிர்வகிக்கும் வாய்ப்பை வழங்குவது UX-ஐ மேம்படுத்துகிறது, ஆனால் அதனுடன் நுண்ணறிவு தகவல்களை அதிகமாக வைத்திருக்கவும் ஆபத்து உள்ளது. இந்த தகவல் எவ்வளவு பராமரிக்கப்படும் என கருத்துக்களை கொண்டிருப்பது, பின்வட்டச் சீர்திருத்தக் கொள்கையை அறிமுகப்படுத்துதல் போன்றவை தனியுரிமைக்கு இன்ப ஏற்படுத்தும்.
- **தனிப்பயனாக்கல்**: கற்றல் மற்றும் பொருந்துதல் திறன் வாய்ந்த AI மொடல்கள் பயனர்களுக்கு தனிப்பட்ட அனுபவத்தை வழங்குகின்றன. பயனர் சுயவிவரம் போன்ற அம்சங்கள் மூலம் UX-ஐ தனிப்பயனாக்குதல் பயனரை புரிந்துகொள்வதாக உணர வைக்கும் மட்டும் அல்லாமல், குறிப்பிட்ட பதில்களை தேடும் முயற்சியில் சிறந்தவை மற்றும் திருப்திகரமான தொடர்பை உருவாக்குகிறது.

இதற்கு ஒரு மாதிரியாக OpenAI-இன் ChatGPT இல் உள்ள "தனிப்பயன் வழிமுறைகள்" அமைப்புகளை எடுத்துக்கொள்ளலாம். இது உங்கள் ப்ராம்ப்ஸ் (prompt) க்கான முக்கியமான சூழலைப் பற்றி தகவலை வழங்க அனுமதிக்கிறது. இதோ ஒரு தனிப்பயனாக்கிய வழிமுறை.

### பெரிய மொழி மொடல்களுக்கு Microsoft-இன் அமைப்பு செய்தி காட்சிக்கோவை

[Microsoft வழங்கிய வழிகாட்டல்](https://learn.microsoft.com/azure/ai-services/openai/concepts/system-message#define-the-models-output-format?WT.mc_id=academic-105485-koreyst) LLM களில் பதில்களை உருவாக்கும்போது விளங்கும் அமைப்பு செய்திகளை எழுதுவதற்கான 4 பிரிவுகளுக்கு கீழ்க்காணும் வகையில்:

1. இது மாதிரி யாருக்கு, அதன் சக்தி மற்றும் வரம்புகளை வரையறுத்தல்.
2. மாதிரியின் வெளியீட்டு வடிவத்தை வரையறுத்தல்.
3. மாதிரியின் எதிர்பார்க்கப்படும் நடத்தையை வெளிப்படுத்தும் குறிப்பிட்ட எடுத்துக்காட்டுகளை வழங்கல்.
4. கூடுதல் நடத்தைக் கட்டுப்பாடுகளை வழங்கல்.

### அணுகுதன்மை

ஒரு பயனாளர் பார்வை, கேளிக்கை, இயக்க 능력 அல்லது அறிவியல் குறைபாடுகளைக் கொண்டிருந்தாலும், நன்றாக வடிவமைக்கப்பட்ட உரையாடல் பயன்பாடு அனைவராலும் பயன்படுத்தக்கூடியதாக இருக்க வேண்டும். கீழே பயனாளர் குறைபாடுகளுக்கு சிறப்பாக அணுகுதன்மையை மேம்படுத்தும் அம்சங்கள் விவரிக்கப்பட்டுள்ளன.

- **பார்வை குறைபாடு உள்ளோருக்கான அம்சங்கள்**: உயர் முரணான தீம் மற்றும் மாற்றக்கூடிய எழுத்துப்படங்கள், திரை வாசிப்பாளர் ஒத்துழைப்பு.
- **கேளிக்கை குறைபாடு உள்ளோருக்கான அம்சங்கள்**: உரை-பேச்சு மற்றும் பேச்சு-உரை செயல்பாடுகள், ஒலி அறிவிப்புகளுக்கு காட்சி குறிகள்.
- **இயக்க குறைபாடு உள்ளோருக்கான அம்சங்கள்**: விசைப்பலகை வழிசெலுத்தல் ஆதரவு, குரல் கட்டளைகள்.
- **அறிவியல் குறைபாடு உள்ளோருக்கான அம்சங்கள்**: எளிமைப்படுத்தப்பட்ட மொழி தேர்வுகள்.

## துறையெனும் மொழி மொடல்களின் தனிப்பயனாக்கலும் சிறப்பு குழாய்ப்பாடும்

உங்கள் நிறுவனத்தின் சொந்தக் குழப்ப அகராதியைக் புரிந்து கொண்டு அதன் பயனாளர்களின் பொதுவான கேள்விகளை முன்னறிவிக்கும் ஒரு உரையாடல் பயன்பாட்டை கற்பனை செய்யுக. இங்கே பேச வேண்டிய சில முயற்சிகள் இருக்கின்றன:

- **DSL மொடல்களை பயன்படுத்துதல்**. DSL என்பது துறைக்கு சிறப்பு மொடலை குறிக்கும். குறிப்பிட்ட துறையைப் புரிந்து கொள்ள அது பயிற்சி பெற்ற DSL மொடலைப் பயன்படுத்தலாம்.
- **சிறப்புக் குழாய்ப்பாடு பயன்பாடு**. உங்கள் மொடலை குறிப்பிட்ட தரவு கொண்டு மேலதிக பயிற்சி அளிப்பதையே சிறப்புக் குழாய்ப்பாடு குறிப்பிடுகிறது.

## தனிப்பயனாக்கல்: DSL பயன்படுத்துதல்

துறை சிறப்பான மொழி மொடல்களை (DSL மொடல்கள்) பயன்படுத்துவதால் பயனர் ஈடுபாட்டை அதிகரித்து, சிறப்பான, சூழல் சம்பந்தப்பட்ட உரையாடல்களை வழங்க முடியும். இது குறிப்பிட்ட துறை, தொழில் அல்லது பொருளை தொடர்புடைய உரைக்குரிய தகவல்களை புரிந்து கொள்ளவும் உருவாக்கவும் பயின்று வளர்ந்த மொடல் ஆகும். DSL மொடலை உருவாக்குதல், SDK மற்றும் API களைப் பயன்படுத்தி ஏற்கனவே உள்ளவைகளைப் பயன்படுத்துதல் ஆகியவை பல வாய்ப்புகள். இன்னும் ஒன்று சிறப்புக் குழாய்ப்பாடு என்பது ஏற்கனவே பயிற்சி பெற்ற மொடலை எடுத்துக் கொண்டு ஒரு குறிப்பிட்ட துறைக்கு உகந்ததாக மாற்றுவது ஆகும்.

## தனிப்பயனாக்கல்: சிறப்புக் குழாய்ப்பாடு

ஒரு முன்பயிற்சி பெற்ற மொடல் குறிப்பிட்ட துறையில் அல்லது பணியில் குறைபாடாக இருந்தால் சிறப்புக் குழாய்ப்பாடு பரிந்துரைக்கப்படுகிறது.

எடுத்துக்காட்டாக, மருத்துவ கேள்விகள் சிக்கலானவை மற்றும் மிகவும் சூழல் சார்ந்தவைகள். ஒரு மருத்துவ நிபுணர் நோயாளியை கண்டறியும்போது அது வாழ்க்கை நெறிகள், முன் வரலாறு போன்ற பல காரணிகளைக் கருத்தில் கொண்டு மற்றும் அண்மைய மருத்துவ ஆய்வுக் கட்டுரைகளை மேற்கோள் காட்டி இருக்க முடியும். இத்தகைய நுட்பமான சூழல்களில், ஒரு பொதுவான நோன்போர் AI உரையாடல் பயன்பாடு நம்பகமான ஆதாரமாக இருப்பது அரிது.

### சாரணி: மருத்துவ பயன்பாடு

மருத்துவப் பயிற்சியாளர்களுக்கு சிகிச்சைக் வழிகாட்டிகள், மருந்துகளின் இடைச்சேர்க்கைகள் அல்லது அண்மைய ஆய்வுத் தகவல்களை விரைவாகக் கொடுக்க ஒரு உரையாடல் பயன்பாட்டைப் பரிசீலிக்கவும்.

ஒரு பொதுவான மொடல் அடிப்படையான மருத்துவ கேள்விகளுக்கு அல்லது பொதுவான ஆலோசனைகளுக்கு இருப்பது போதுமானதாக இருக்கலாம், ஆனால் இதோ சில சிக்கல்கள்:

- **மிகச் சிறப்பான அல்லது சிக்கலான வழக்கு கண்**கள். உதாரணமாக, ஒரு நரம்பியல் நிபுணர் இந்த பயன்பாட்டிடம் கேட்கலாம், "குழந்தை கொசை மருந்துக்கு எதிரான மின்மினிக்குரிய ஈபிலேப்சியை கையாள்வதில் அண்மைய சிறந்த நடைமுறைகள் என்ன?"
- **அண்மைய முன்னேற்றங்கள் இல்லாமை**. பொதுவான மொடல் அண்மைய நரம்பியல் மற்றும் மருந்தியல் முன்னேற்றங்களுடன் பதில் வழங்குவதில் சிரமமாகலாம்.

இத்தகைய சூழல்களில், மருத்துவத் தரவுத்தளத்துடன் சிறப்புக் குழாய்ப்பாடு செய்தால், இந்த சிக்கலான மருத்துவக் கேள்விகள் மீதான பதிலளிப்பு திறனும் நம்பகத்தன்மையும் பெரிதும் மேம்படும். இதற்கு பெரிய மற்றும் தொடர்புடைய தரவுத்தள அணுகல் தேவை, அது துறைக்கு சம்பந்தப்பட்ட சவால்கள் மற்றும் கேள்விகளை பிரதிபலிக்க வேண்டும்.

## உயர் தர AI இயக்கங்களுடன் உரையாடல் அனுபவத்துக்கான பரிசீலனைகள்

இந்த பகுதி "உயர் தர" உரையாடல் செயலிகளுக்கான அளவுகோல்கள் மற்றும் AI தொழில்நுட்பத்தை பொறுப்புடன் பயன்படுத்தும் கட்டமைப்பை விளக்குகிறது.
