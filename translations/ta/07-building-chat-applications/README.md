# உருவாக்கி வரும் செயற்கை நுண்ணறிவு இயங்கும் சந்தை பயன்பாடுகள்

இப்போது நாம் உரை உருவாக்கும் பயன்பாடுகள் எப்படி கட்டமைக்கப்படுகிறதென பார்த்துள்ளோம், சந்தை பயன்பாடுகளைக் கவனிப்போம்.

சந்தை பயன்பாடுகள் எங்கள் தினசரி வாழ்க்கையில் ஒருங்கிணைக்கப்பட்டு, சாதாரண உரையாடல் வழிமுறையாக மட்டுமல்லாமல் எல்லா துறைகளிலும் அவசியமான பகுதியாகிவிட்டது. அவை வாடிக்கையாளர் சேவை, தொழில்நுட்ப ஆதரவு, மற்றும் மேலும் நுணுக்கமான ஆலோசனை முறைமைகள் போன்றவற்றின் முக்கிய பகுதியாக உள்ளன. நீங்களும் அடுத்த காலத்து சந்தை பயன்பாட்டிலிருந்து உதவி பெற்றிருக்க வாய்ப்பு உள்ளது. நாங்கள் உருவாக்கி வரும் செயற்கை நுண்ணறிவைப் போன்ற முன்னோக்கிய தொழில்நுட்பங்களை இந்த தளங்களில் இணைக்கும் போது சிக்கலான தன்மையும், சவால்களும் அதிகரிக்கும்.

சில கேள்விகளுக்குப் பதில் தேவை:

- **பயன்பாட்டைக் கட்டமைத்தல்**. குறிப்பிட்ட பயன்படுத்தும் வழிகளுக்கான இந்த செயற்கை நுண்ணறிவு இயங்கும் பயன்பாடுகளை எவ்வாறு திறம்பட கட்டமைத்து ஒருங்கிணைப்பு செய்வது?
- **கண்காணித்தல்**. தொழில்நுட்பத்தை அமல்படுத்திய பின், செயல்பாடின் தரத்தை மற்றும் [பொறுப்பான செயற்கை நுண்ணறிவு ஆறு 원칙ங்களைக்](https://www.microsoft.com/ai/responsible-ai?WT.mc_id=academic-105485-koreyst) கடைப்பிடிப்பதை எப்படி கண்காணித்து உறுதி செய்வது?

தானியங்கி மற்றும் மனித-யந்திர இடையுறுப்புகளால் வரையறுக்கப்பட்ட காலத்திற்குள், உருவாக்கி வரும் செயற்கை நுண்ணறிவு சந்தை பயன்பாடுகளின் பரிமாணம், ஆழம் மற்றும் ஏற்பாட்டை மாற்றுவதில் எப்படி மாற்றம் உண்டாகிறது என்பதைப் புரிந்து கொள்ளுதல் அவசியம். இந்த பாடம் இவற்றைக் கூடும் வியூக அம்சங்களை ஆராய்ந்து, சுயேச்சமிக்க பணிகளுக்கான முறைகளையும் மதிப்பிடும், பொறுப்பான செயற்கை நுண்ணறிவு பிரயோகத்திற்கான மனவரிசைகள் மற்றும் பரிசீலனைகளை கவனிக்கும்.

## அறிமுகம்

இந்த பாடம் அமைந்துள்ளது:

- சந்தை பயன்பாடுகளைக் கட்டமைத்து ஒருங்கிணைக்கும் திறமையான நுட்பங்கள்.
- பயன்பாடுகளை தனிப்பயனாக்கி நுணுக்கமாக மாற்றுவது எப்படி என்பது.
- சந்தை பயன்பாடுகளை திறம்பட கண்காணிக்கும் நடைமுறைகள் மற்றும் பரிசீலனைகள்.

## கற்றல் இலக்குகள்

இந்த பாடத்தின் முடிவில், நீங்கள் செய்யக்கூடியவை:

- செயல்படுத்தப்பட்ட அமைப்புகளில் சந்தை பயன்பாடுகளை கட்டமைத்து ஒருங்கிணைக்கும் பரிசீலனைகளை விரிவாக விவரிக்க.
- குறிப்பிட்ட பயன்படுத்தும் வழிகளுக்கான சந்தை பயன்பாடுகளை தனிப்பயனாக்கவும்.
- செயற்கை நுண்ணறிவு இயங்கும் சந்தை பயன்பாடுகளின் தரத்தை திறம்பட கண்காணிக்க முக்கியமான அளவுகோல்கள் மற்றும் பரிசீலனைகளை கண்டறி.
- சந்தை பயன்பாடுகள் செயற்கை நுண்ணறிவை பொறுப்புடன் பயன்படுத்துவதை உறுதி செய்.

## உருவாக்கி வரும் செயற்கை நுண்ணறிவை சந்தை பயன்பாடுகளில் ஒருங்கிணைத்தல்

சந்தை பயன்பாடுகளை உருவாக்கி வரும் செயற்கை நுண்ணறிவால் மேம்படுத்துவது, அவற்றை சுலபமாகச் செய்யும் மட்டும் அல்ல; தரமான பயனர் அனுபவத்தை வழங்கக் கட்டமைப்பு, செயல்திறன் மற்றும் பயனர் இடைமுகத்தை முறையாக சிறப்புப்படுத்துவது ஆகும். இதில் கட்டமைப்பு அடித்தளங்கள், API ஒருங்கிணைப்புகள் மற்றும் பயனர் இடைமுக பரிசீலனைகள் அடங்கும். இந்த பகுதி உங்கள் பயன்பாடுகளை கையாள்வதற்கான விரிவான வழிகாட்டுதலை வழங்க விரும்புகின்றது, நீங்கள் அவற்றை ஏற்கனவே உள்ள அமைப்புகளில் இணைக்கிறீர்களோ அல்லது தனியான தளங்களாக உருவாக்குகிறீர்களோ.

இந்த பகுதியில் முடிவில், சந்தை பயன்பாடுகளை திறம்பட கட்டமைத்து ஒருங்கிணைக்க தேவையான திறமைகளை நீங்கள் பெறுவீர்கள்.

### சாட்பாட்டோ அல்லது சந்தை பயன்பாடோ?

சந்தை பயன்பாடுகளை கட்டமைக்க முன், 'சாட்பாட்' மற்றும் 'செயற்கை நுண்ணறிவு இயங்கும் சந்தை பயன்பாடுகள்' என்பவற்றின் வேறுபாடுகளை நோக்குவோம். சாட்பாட்டின் முதன்மை நோக்கம் அடிக்கடி கேட்கப்படும் கேள்விகளுக்கு பதில் சொல்லுதல் அல்லது ஒரு பார்சல் சுட்டிக்காட்டல் போன்ற உரையாடல் பணிகளை தானியங்கி செய்யும் உதவி ஆகும். அது பெரும்பாலும் விதி அடிப்படைக் கொள்கைகள் அல்லது சிக்கலான செயற்கை நுண்ணறிவு தத்துவமைப்புகளால் நிர்வகிக்கப்படும். மாற்றாக, செயற்கை நுண்ணறிவு இயங்கும் சந்தை பயன்பாடுகள் மனித பயர்களுக்கு உரை, குரல் மற்றும் காணொலி உதவிகள் போன்ற பல்வேறு வடிவில் டிஜிட்டல் தொடர்பு வசதிகளை வழங்க கவனமாக வடிவமைக்கப்பட்டுள்ளன. அவற்றின் பிரதான அம்சம் உருவாக்கி வரும் செயற்கை நுண்ணறிவை கோர்வையை அடிப்படையாகக் கொண்டு மனிதத்துக்கு ஒத்தைப்படுத்தப்பட்ட உரையாடல்களை உருவாக்குதல். ஒரு உருவாக்கி வரும் AI இயங்கும் சந்தை பயன்பாடு திறந்த பரிமாண உரைபுரிதல், மாற்றம் அடைந்த உரையாடல் சூழலை பிடித்து தமிழா மற்றும் சிக்கலான உரையாடல்களையும் உருவாக்க முடியும்.

கீழே உள்ள அட்டவணை அவர்களின் தனிப்பட்ட பங்குகளையும் வேறுபாடுகளையும் விளக்கும், டிஜிட்டல் தொடர்பில் அவற்றின் தனிப்பட்ட பங்கு புரிந்து கொள்ள உதவும்.

### SDK மற்றும் API களை பயன்படுத்தி முன்கூட்டியாக உள்ள செயல்பாடுகளை பயன்படுத்துதல்

சந்தை பயன்பாடுகளை கட்டமைக்கும் போது, முதலில் ஏற்கனவே உள்ளதை மதிப்பாய்வு செய்வது சிறந்த முறையாகும். SDK மற்றும் API களை பயன்படுத்தி சந்தை பயன்பாடுகளை உருவாக்குவது பல காரணங்களால் பயனுள்ளது. நன்கு விளக்கப்பட்ட SDK மற்றும் API களை இணைப்பதனால் உங்கள் பயன்பாடு நீண்ட கால வெற்றிக்கு இடம் பெற்று விரிவாக்கம் மற்றும் பராமரிப்பு சிக்கல்களை சமாளிக்க இது உதவும்.

- **உருவாக்கும் செயல்முறையை வேகப்படுத்தி சுமையை குறைத்து விடுகிறது**: வசதிகள் உருவாக்குவதற்கு செலவு நிறைந்த செயல்முறையை தவிர்க்க SDK மற்றும் API களை பயன்படுத்துவதால் நீங்கள் உங்கள் பயன்பாட்டின் மற்ற முக்கிய அம்சங்களுக்கு, எ.கா., வணிகத்துக்கான தர்க்கங்களுக்கு கவனம் செலுத்தலாம்.
- **நிறைவேற்றலுக்கு மேம்பாடு**: செயல்பாடுகளை ஆரம்பத்தில் உருவாக்கும்போது நீங்கள் "இது எவ்வாறு விரிவடையும்? இது 갑작스러운 பயனர் பெருக்கத்தை கையாள முடியுமா?" எனக்கேள்வி எழுப்புவீர்கள். நன்கு பராமரிக்கப்பட்ட SDK மற்றும் API களில் இந்த சிக்கல்களுக்கு உள்ளே தீர்வுகள் அடங்கியுள்ளன.
- **சுலபமான பராமரிப்பு**: புதிய பதிப்புகள் வெளியிடப்படும் போது நூலகத்தை புதுப்பிப்பதன் மூலம் பராமரிக்கலும் மேம்பாடுகளும் சுலபமாக நடக்கிறது.
- **நவீன தொழில்நுட்ப அணுகல்**: விரிவான தரவுத்தொகுப்புகளில் பயிற்சி பெற்ற உருவாக்கி வரும் மாதிரிகளை பயன்படுத்தி உங்கள் பயன்பாட்டுக்கு இயற்கை மொழி திறன்களை வழங்கும்.

SDK அல்லது API செயல்பாடுகளைப் பயன்படுத்த அனுமதி பெறுவது பொதுவாக வழங்கும் சேவைகளுக்கான தனித்திறன் விசையோ அங்கீகார குறியீட்டோ арқылы நடக்கிறது. இதைப் புரிந்துகொள்ள நாம் OpenAI Python நூலகத்தை பயன்படுத்தி பார்ப்போம். நீங்கள் தனி முறையிலும் இந்த பாடத்திற்கான [OpenAI நோட்புக்](./python/oai-assignment.ipynb?WT.mc_id=academic-105485-koreyst) அல்லது [Azure OpenAI சேவைகள் நோட்புக்](./python/aoai-assignment.ipynb?WT.mc_id=academic-105485-koreys) பயில முயற்சித்து பார்வையிடலாம்.

```python
import os
from openai import OpenAI

API_KEY = os.getenv("OPENAI_API_KEY","")

client = OpenAI(
    api_key=API_KEY
    )

response = client.responses.create(model="gpt-5-mini", input="Suggest two titles for an instructional lesson on chat applications for generative AI.", store=False)
print(response.output_text)
```

மேற்படி எடுத்துக்காட்டில் GPT-5 மினி மாதிரியை Responses API மூலம் பயன்படுத்தி கருத்தினை முடிக்கிறது, ஆனால் API விசை முன்பே அமைக்கப்பட்டுள்ளது என்பதை கவனியுங்கள். விசை அமைக்காவிட்டால் பிழை வரும்.

## பயனர் அனுபவம் (UX)

பொதுவான UX கொள்கைகள் சந்தை பயன்பாடுகளுக்கு பொருந்தும், ஆனால் இயந்திர கற்றல் கூறுகள் காரணமாக கூடுதல் பரிசீலனைகள் முக்கியமாகின்றன.

- **மாயங்கும் பதில்களை சீர்செய்வதற்கு அடுக்குகள்**: உருவாக்கி வரும் AI மாதிரிகள் சிலசமயம் குழப்பமான பதில்களை வழங்கலாம். பயனர்கள் விளக்கக்கேள்வி கேட்கும் வசதி இதற்கு உதவும்.
- **சூழல் தகவல் ஏமாற்றல்**: மேம்பட்ட உருவாக்கி வரும் AI மாதிரிகளுக்கு உரையாடலில் உள்ள சூழலை நினைவில் வைக்கவிருப்பது பயனுக்கேற்றது. பயனர்கள் சூழலை கட்டுப்படுத்தி சமாளிக்கத் தமக்கான அதிகாரம் கொடுத்து அனுபவத்தை மேம்படுத்துவது அவசியம் ஆனால் முக்கியமான பயனர் தகவல்களை பாதுகாப்பது முன்வருவதாகும். தகவல் எவ்வளவு நேரம் வைக்கப்படும் என்பதற்கான கொள்கைகளை முன்மொழிவது தனியுரிமையை போற்றும் கட்டமைப்பிற்கான வினை சீரமைப்பாக இருக்கும்.
- **தனிமைப்படுத்தல்**: கற்றல் மற்றும் ஏற்றுக்கொள்ளல் திறன் கொண்ட AI மாதிரிகள் தனிப்பட்ட அனுபவத்தை வழங்குகின்றன. பயனர் சுயவிவரங்கள் போன்ற அம்சங்கள் பயனர் உணர்வை மேம்படுத்துவதோடு குறிப்பிட்ட பதில்கள் தேடுதலை எளிதாக்குகின்றன, செயல்திறன் மற்றும் திருப்தியான தொடர்பை ஏற்படுத்துகின்றன.

தனிமைப்படுத்தலாக OpenAI ChatGPT இல் உள்ளது "Custom instructions" அமைப்புகள். இது உங்கள் பாரம்பரியமான விவரங்களை அளித்து உங்கள் கருத்துக்களுக்கு சூழல் அளிக்க உதவுகிறது. கீழே ஒரு தனிப்பயன் கட்டளை எடுத்துக்காட்டு உள்ளது.

### மைக்ரோசாஃப்ட் மொத்த மொழி மாதிரிகளுக்கான சிஸ்டம் மெசேஜ் வடிவமைப்பு

[மைக்ரோசாஃப்ட் வழங்கிய வழிகாட்டுதல்](https://learn.microsoft.com/azure/ai-foundry/openai/concepts/system-message#define-the-models-output-format?WT.mc_id=academic-105485-koreyst) LLM க்கான பதில்கள் உருவாக்கும் போது சிஸ்டம் மெசேஜ்களைச் சிறப்பாக எழுத चार பகுதியா் வகுத்துள்ளது:

1. மாதிரி யாருக்காக என்று, அதன் திறன்கள் மற்றும் வரம்புகளை குறிப்பிடுதல்.
2. மாதிரி உருவாக்கும் வெளியீடு வடிவத்தை வரையறுத்தல்.
3. மாதிரியின் செயல்பாட்டை விளக்கும் குறிப்பிட்ட உதாரணங்களை வழங்குதல்.
4. மேலதிக நடத்தை காப்புக் கருவிகளை வழங்குதல்.

### அணுகல்

பயனர் பார்வை, கேள்வித்திறன், மோட்டார், அல்லது அறிவாற்றல் குறைபாடுகள் இருந்தாலும்கூட, நன்றாக வடிவமைக்கப்பட்ட சந்தை பயன்பாடு எல்லாரும் பயன்படுத்த தீக்கானதாக வேண்டியது அவசியம். கீழே கொடுக்கப்பட்ட பட்டியல் வளரும் பயனர் குறைபாடுகளுக்கு அணுகலை மேம்படுத்த முதன்மை அம்சங்களை விளக்குகிறது.

- **பார்வை குறைபாடு அம்சங்கள்**: உயர் எதிரிவன தாளம் மற்றும் மாற்றக்கூடிய எழுத்துகள், திரை வாசிப்பி உடன் இணக்கம்.
- **கேள்வித்திறன் குறைபாடு அம்சங்கள்**: உரை-சொல் மற்றும் சொல்-உரை செயல்பாடுகள், ஒலி அறிவிப்புகளுக்கான காணொலி குறியீடுகள்.
- **மோட்டார் குறைபாடு அம்சங்கள்**: விசைபலகை வழிசெலுத்தல், குரல் கட்டளைகள்.
- **அறிவாற்றல் குறைபாடு அம்சங்கள்**: எளிமையான மொழி விருப்புகள்.

## தனிப்பயனாக்கல் மற்றும் கனி-திருத்தல் குறித்த துறையெனிய மொழி மாதிரிகள்

உங்கள் நிறுவனத்தின் சம்பவங்களையும் பயனர் அடிப்படையில் பொதுவான கேள்விகளையும் புரியும் சந்தை பயன்பாடைக் கற்பனை செய்து கொள்ளுங்கள். சில அணுகுமுறைகள் குறிப்பிடத்தக்கவை:

- **DSL மாதிரிகளைப் பயன்படுத்தல்**. DSL என்பது துறை-சார்ந்த மொழி என பொருள். குறிப்பிட்ட துறைக்கு பயிற்சி பெற்ற DSL மாதிரியைப் பயன்படுத்தி அதன் தத்துவங்களையும் சூழல்களையும் புரியவைக்கலாம்.
- **கனித்திறன் திருத்தம் செய்யவும்**. கனித்திறன் திருத்தம் என்பது உங்கள் மாதிரியை குறிப்பிட்ட தரவுகளுடன் மேலும் பயிற்சி செய்வதற்கான செயல்முறை.

## தனிப்பயனாக்கல்: DSL பயன்பாடு

துறையெனிய மொழி மாதிரிகள் (DSL Models) பயனர் ஈடுபாட்டை மேம்படுத்தி சிறப்பு மற்றும் சூழல் பொருந்திய தொடர்புகளை வழங்கவு சாத்தியம் உள்ளது. இது குறிப்பிட்ட துறை, தொழில், அல்லது பொருளுடன் தொடர்புடைய உரைகளை புரிந்துகொண்டு உருவாக்க பயிற்சி கொண்ட மாதிரி ஆகும். DSL மாதிரியை பயன்படுத்துவது, அதை ஆரம்பத்திலிருந்து பயிற்சி செய்வது அல்லது SDK மற்றும் API வழியாக உள்ளது நடைமுறைப்படுத்துவது போன்ற பல வாய்ப்புக்களுமே உள்ளன. மற்றொரு வழி கனித்திறன் திருத்தம், அதாவது ஏற்கனவே பயிற்சி பெற்ற மாதிரியை எடுத்து, அதனை ஒரு குறிப்பிட்ட துறைக்கானதாக மாற்றும் செயல்முறை.

## தனிப்பயனாக்கல்: கனித்திறன் திருத்தம் செய்தல்

கனித்திறன் திருத்தம் பெரும்பாலும் ஒரு பயிற்சி பெற்ற மாதிரி குறிப்பிட்ட துறை அல்லது பணிக்கான தேவைகளை பூர்த்தி செய்யாத போது கருதப்படுகிறது.

எடுத்துக்காட்டாக, மருத்துவ கேள்விகள் சிக்கலானவை மற்றும் பல சூழல்கள் தேவைப்படுகின்றன. மருத்துவர் ஒரு நோயாளியை கண்டறியும் போது வாழ்க்கை முறை, முன்கூட்டிய நோய்கள் போன்ற பல காரணிகளைக் கருத்தில் கொண்டு அண்மைய மருத்துவ அறிக்கைகளுடன் ஒப்பிட்டு தொற்று உறுதி செய்யலாம். இத்தகைய நுணுக்கமான சூழல்களில் பொதுவான AI சந்தை பயன்பாடுகள் நம்பகமான ஆதாரமாக இருக்க முடியாது.

### சூழல்: ஒரு மருத்துவ பயன்பாடு

மருத்துவ நிபுணர்களுக்கு மருத்துவ வழிகாட்டிகள், மருந்து தொடர்புகள் அல்லது அண்மைய ஆய்வு தகவல்களை விரைவாக வழங்கும் சந்தை பயன்பாடாக திட்டமிடப்பட்டது என்று நினைத்துக்கொள்ளுங்கள்.

அடிப்படைக் மருத்துவ கேள்விகளுக்கு அல்லது பொதுவான ஆலோசனைகளுக்கு பொதுத்துறைக் மாதிரி போதுமானதே, ஆனால் கீழ்காணும் பிரச்சினைகள் உண்டு:

- **மிகவும் குறிப்பிட்ட அல்லது சிக்கலான வழக்குகள்**. எடுத்துக்காட்டாக, ஒரு அறிவுத்துறை மருத்துவர் "குழந்தைகள் பகுதியின் மருந்து எதிர்ப்பு எபிலப்ப்சி மேலாண்மையில் தற்போதைய சிறந்த நடைமுறைகள் என்ன?" என கேட்கிறார்.
- **அண்மைய முன்னேற்றங்கள் குறைவாக உள்ளவை**. ஒரு பொதுத்துறை மாதிரி நல்லண்மைய மருத்துவ மற்றும் மருந்தாய்வுத் தகவல்களை சரியான முறையில் உள்ளடக்க முடியாமல் இருக்க வாய்ப்பு உள்ளது.

இத்தகைய சூழலில், குறிப்பிட்ட மருத்துவ தரவுத்தொகுப்புடன் மாதிரியை கனித்திறன் திருத்தம் செய்தல் இந்த தொழில்நுட்ப கேள்விகளைக் கவனமாகவும் நம்பகமாகவும் கையாள்வதற்கு பெரிதும் உதவும். இது வாய்ந்த, பொருந்தக்கூடிய மற்றும் துறை சார்ந்த சவால் மற்றும் கேள்விகளை கொண்ட ஒரு பெரிய தரவுத்தொகுப்பை அன்றாட ஆதரவைச் தேவைப்படுத்தும்.

## உயர்தர செயற்கை நுண்ணறிவு இயங்கும் சந்தை அனுபவத்துக்கான பரிசீலனைகள்

இத்துறைச் சொகுசான, செயற்கை நுண்ணறிவு தொழில்நுட்பத்தை பொறுப்புடன் பயன்படுத்துவதற்கான கட்டமைப்பையும் செயற்பாட்டிலான அளவுகோல்களையும் உட்படுத்திய “உயர்தர” சந்தை பயன்பாடுகளுக்கான கொள்கைகளை விவரிக்கிறது.
