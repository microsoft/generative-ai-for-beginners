## 2. விருப்பம் C – மினிகொண்டா

[Miniconda](https://conda.io/en/latest/miniconda.html?WT.mc_id=academic-105485-koreyst) என்பது [Conda](https://docs.conda.io/en/latest?WT.mc_id=academic-105485-koreyst), பைதான் மற்றும் சில தொகுப்புகளை நிறுவ சிறிய இன்ஸ்டாலர் ஆகும்.
கான்டா என்பது தொகுப்பு மேலாளர் ஆகும், இது பல பைதான் [**வாச்சை சுற்றுச்சூழல்களை**](https://docs.python.org/3/tutorial/venv.html?WT.mc_id=academic-105485-koreyst) மற்றும் தொகுப்புகளை எளிதில் அமைக்கவும் மாறவும் உதவுகிறது. இது `pip` மூலம் கிடைக்காத தொகுப்புகளை நிறுவும் போது உதவுகிறது.

### படி 0 மினிகொண்டாவை நிறுவுக

[MiniConda நிறுவல் வழிகாட்டியைக்](https://docs.anaconda.com/free/miniconda/#quick-command-line-install?WT.mc_id=academic-105485-koreyst) பின்பற்றவும்.

```bash
conda --version
```

### படி 1 ஒரு சுற்றுச்சூழலை உருவாக்கு

ஒரு புதிய சுற்றுச்சூழல் கோப்பை (*environment.yml*) உருவாக்குக. Codespaces பயன்படுத்தினால் `.devcontainer` அடைவுக்குள், அதாவது `.devcontainer/environment.yml` உள்ளே அதை உருவாக்கவும்.

### படி 2 சுற்றுச்சூழல் கோப்பை நிரப்புக

உங்கள் `environment.yml` இல் கீழ்கண்ட குறியீட்டைக் கூட்டுக

```yml
name: <environment-name>
channels:
 - defaults
 - microsoft
dependencies:
- python=<python-version>
- openai
- python-dotenv
- pip
- pip:
    - azure-ai-ml

```

### படி 3 உங்கள் கான்டா சுற்றுச்சூழலை உருவாக்குக

கீழேயுள்ள கட்டளைகளை உங்கள் கட்டளைத்தாள் / டெர்மினலில் இயக்குக

```bash 
conda env create --name ai4beg --file .devcontainer/environment.yml # .devcontainer துணை பாதை Codespace அமைப்புகளுக்கு மட்டுமே பொருந்தும்
conda activate ai4beg
```

எந்த சிக்கலும் வந்தால் [Conda சுற்றுச்சூழல்கள் வழிகாட்டியை](https://docs.conda.io/projects/conda/en/latest/user-guide/tasks/manage-environments.html?WT.mc_id=academic-105485-koreyst) பார்க்கவும்.

## 3. உங்கள் API விசைகளைச் சேர்க்கவும்

எந்தவொரு பயன்பாட்டை உருவாக்கும்போது உங்கள் API விசைகளை பாதுகாப்பாக வைத்திருப்பது அவசியம். எந்த API விசைகளையும் நேரடியாக உங்கள் குறியீட்டில் சேமிக்காததை பரிந்துரைக்கிறோம். அவற்றை பொதுவான ரிப்போசிடரியில் சேர்வது பாதுகாப்பு பிரச்சனைகளை ஏற்படுத்தக்கூடும் மற்றும் தீங்கு செய்யும் வியத்தகு ஒருவர் பயன்படுத்தினால் தேவையற்ற செலவுகளும் ஏற்படும்.
Python கோப்புகளுக்கான `.env` கோப்பினை உருவாக்கும் மற்றும் Microsoft Foundry Models அங்கீகாரங்களைச் சேர்க்கும் படி இது ஒரு படிநிலை வழிகாட்டி:

> **குறிப்பு:** GitHub Models (அதன் `GITHUB_TOKEN` மாறி) 2026 ஜூலை இறுதியில் நிறுத்தப்பட உள்ளது. இந்த வழிகாட்டி [Microsoft Foundry Models](https://ai.azure.com/catalog/models?WT.mc_id=academic-105485-koreyst) பயன்படுத்துகிறது. முழுவதும் ஆஃப்லைனில் செயல்பட விரும்பினால் [Foundry Local](https://foundrylocal.ai?WT.mc_id=academic-105485-koreyst) பாருங்கள்.

1. **உங்கள் திட்ட அடைவுக்கு செல்லவும்**: உங்கள் டெர்மினல் அல்லது கட்டளைத்தாளை திறந்து, `.env` கோப்பை உருவாக்க வேண்டிய திட்டத்தின் ரூட் அடைவிற்கு செல்லவும்.

   ```bash
   cd path/to/your/project
   ```

2. **`.env` கோப்பை உருவாக்கவும்**: உங்கள் விருப்பமான உரை தொகுப்பியில் `.env` என்ற பெயரில் புதிய கோப்பை உருவாக்குக. கட்டளைத்தாளில் இருந்தால் Unix அடிப்படையிலான அமைப்புகளில் `touch` அல்லது விண்டோஸில் `echo` பயன்படுத்தலாம்:

   Unix அடிப்படையிலான அமைப்புகள்:

   ```bash
   touch .env
   ```

   விண்டோஸ்:

   ```cmd
   echo . > .env
   ```

3. **`.env` கோப்பை திருத்தவும்**: `.env` கோப்பை ஒரு உரை தொகுப்பியில் திறக்கவும் (உதா: VS Code, Notepad++, அல்லது வேறு தொகுப்பி). கீழ்க்கண்ட வரிகளைக் கூட்டவும், உங்கள் Microsoft Foundry திட்ட முகவரி மற்றும் API விசையை மாற்றிகளுக்கு பதிலாக சேர்க்கவும்:

   ```env
   AZURE_INFERENCE_ENDPOINT=your_foundry_endpoint_here
   AZURE_INFERENCE_CREDENTIAL=your_foundry_api_key_here
   ```

4. **கோப்பை சேமித்து மூடு**: மாற்றங்களைச் சேமித்து உரை தொகுப்பியை மூடவும்.

5. **`python-dotenv` ஐ நிறுவுக**: நீங்கள் இன்னும் நிறுவவில்லை என்றால், `.env` கோப்பிலிருந்து சுற்றுச்சூழல் மாறிகளை உங்கள் பைதான் பயன்பாட்டுக்கு ஏற்ற `python-dotenv` தொகுப்பை நிறுவ வேண்டும். `pip` பயன்படுத்தி நிறுவலாம்:

   ```bash
   pip install python-dotenv
   ```

6. **தானியங்கி நிரலில் சுற்றுச்சூழல் மாறிகளை ஏற்றுக**: உங்கள் பைதான் நிரலில், `.env` கோப்பிலிருந்து சுற்றுச்சூழல் மாறிகளை ஏற்ற `python-dotenv` தொகுப்பை பயன்படுத்துக:

   ```python
   from dotenv import load_dotenv
   import os

   # .env கோப்பில் இருந்து சூழல் மாறிலிகளைக் โหลด செய்க
   load_dotenv()

   # Microsoft Foundry Models மாறிலிகளை அணுகுக
   endpoint = os.getenv("AZURE_INFERENCE_ENDPOINT")
   token = os.getenv("AZURE_INFERENCE_CREDENTIAL")

   print(endpoint)
   ```

முடிந்தது! நீங்கள் வெற்றி பெற்று `.env` கோப்பை உருவாக்கி, உங்கள் Microsoft Foundry Models அங்கீகாரங்களைச் சேர்த்து, அதை உங்கள் பைதான் பயன்பாட்டில் ஏற்றியுள்ளீர்கள்.

## 5. சிக்கல் தீர்வு

| அறிகுறி                                   | சரிசெய்                                                             |
|-------------------------------------------|-----------------------------------------------------------------|
| `python காணவில்லை`                        | Python ஐ PATH இல் சேர்க்கவும் அல்லது நிறுவிய பிறகு டெர்மினலை மீண்டும் திறக்கவும்  |
| `pip` அடுக்கு சக்கரங்களை உருவாக்க முடியவில்லை (விண்டோஸ்)       | `pip install --upgrade pip setuptools wheel` பிறகு மீண்டும் முயற்சிக்கவும்.        |
| `ModuleNotFoundError: dotenv`             | `pip install -r requirements.txt` இயக்கவும் (சுற்றுச்சூழல் நிறுவப்படவில்லை).   |
| டோக்கர் கட்டுமானம் தோல்வி *இடம் இல்லை*        | Docker Desktop ▸ *அமைப்புகள்* ▸ *வளங்கள்* → வட்டி அளவை அதிகரிக்கவும். |
| VS Code மீண்டும் திறக்கும்படி கூருவதைக் கைகூறும்         | இரு விருப்பங்களும் செயல்பாடு நிலையில் இருக்கலாம்; ஒன்றைத் தேர்ந்தெடுக்கவும் (venv **அல்லது** கண்டெய்னர்)|
| OpenAI 401 / 429 பிழைகள்                   | `OPENAI_API_KEY` மதிப்பை சரிபார்க்கவும் / கோரிக்கை விகித வரம்புகளை கவனிக்கவும்.             |
| கான்டா பயன்படுத்தும் பிழைகள்                        | Microsoft AI நூலகங்களை நிறுவ `conda install -c microsoft azure-ai-ml` பயன்படுத்தவும் |
