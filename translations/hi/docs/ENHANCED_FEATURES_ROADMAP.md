## 4. API आधुनिकीकरण

### 4.1 अप्रचलित API पैटर्न (माइग्रेशन पूरा हुआ)

सभी Python और TypeScript **चैट** नमूने Chat Completions API से **Responses API** (`client.responses.create(...)` → `response.output_text`) में माइग्रेट कर दिए गए हैं।

| पुराना पैटर्न | नया पैटर्न | स्थिति |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (चैट) | `OpenAI(base_url="<endpoint>/openai/v1/")` (Responses API) | पूर्ण |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | पूर्ण |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (TypeScript) | `openai` पैकेज `client.responses.create()` → `response.output_text` | पूर्ण |
| `df.append()` (pandas) | `pd.concat()` | पूर्ण |

> **ध्यान दें:** Microsoft Foundry Models नमूने जो `azure-ai-inference` / `@azure-rest/ai-inference` SDK (`client.complete()`) का उपयोग करते हैं, वे Model Inference API पर ही बने रहते हैं, जो Responses API का समर्थन नहीं करता। जहां अभी भी वैध है वहां `AzureOpenAI()` को जानबूझकर रखा गया है (एम्बेडिंग्स और इमेज जनरेशन के लिए)।

### 4.2 प्रदर्शित करने के लिए नए API फीचर्स

1. **संरचित आउटपुट** (OpenAI)
   - JSON मोड
   - सख्त स्कीमाओं के साथ फंक्शन कॉलिंग

2. **दृश्य क्षमताएँ**
   - GPT-4o (vision) के साथ छवि विश्लेषण
   - मल्टी-मोडल प्रॉम्प्ट्स

3. **Responses API में अंतर्निर्मित टूल्स** (पुराने Assistants API का प्रतिस्थापन)
   - कोड इंटरप्रेटर
   - फ़ाइल खोज
   - वेब खोज और कस्टम टूल्स

## 5. इन्फ्रास्ट्रक्चर सुधार

### 5.1 CI/CD सुधार

[`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml) में कार्यान्वित: Python लिंटिंग/फॉर्मेटिंग (Ruff + Black) में बनाए गए `shared/` यूटिलिटीज मॉड्यूल पर **कड़ाई से लागू** किया गया है और बाकी पाठ्यक्रम पर **सलाहकार** रूप से चलता है, साथ ही JavaScript/TypeScript के लिए ESLint का सलाहकार पास। मापदंड था:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### 5.2 सुरक्षा स्कैनिंग

[`.github/workflows/security.yml`](../../../.github/workflows/security.yml) में कार्यान्वित: Python और JavaScript/TypeScript के लिए CodeQL विश्लेषण (पुश, पुल रिक्वेस्ट, और साप्ताहिक अनुसूची पर) और पुल रिक्वेस्ट पर निर्भरता समीक्षा। मापदंड था:

```yaml
# .github/workflows/security.yml
name: Security Scan

on: [push, pull_request]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: javascript, python
      - uses: github/codeql-action/analyze@v3

  dependency-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
```
