# AGENTS.md

## परियोजना अवलोकन

यह रिपॉजिटरी स्वयं-निर्मित AI के मूल सिद्धांतों और अनुप्रयोग विकास को सिखाने वाले 21-पाठ्यक्रम का एक व्यापक पाठ्यक्रम शामिल करता है। यह कोर्स शुरुआती लोगों के लिए डिज़ाइन किया गया है और बुनियादी अवधारणाओं से लेकर उत्पादन-तैयार अनुप्रयोगों के निर्माण तक सब कुछ कवर करता है।

**मुख्य तकनीकें:**
- Python 3.9+ पुस्तकालयों के साथ: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript Node.js और पुस्तकालयों के साथ: `openai` (Azure OpenAI v1 एपीआई एंडपॉइंट + Responses API), `@azure-rest/ai-inference` (Microsoft Foundry मॉडल)
- Azure OpenAI सेवा, OpenAI API, और Microsoft Foundry मॉडल (GitHub मॉडल  जुलाई 2026 के अंत में बंद हो रहे हैं)
- इंटरैक्टिव सीखने के लिए Jupyter नोटबुक
- निरंतर विकास पर्यावरण के लिए Dev कंटेनर

**रिपॉजिटरी संरचना:**
- 21 क्रमांकित पाठ डिरेक्टरी (00-21) जिनमें README, कोड उदाहरण, और असाइनमेंट शामिल हैं
- विभिन्न कार्यान्वयन: Python, TypeScript, और कभी-कभी .NET उदाहरण
- 40+ भाषाओं में अनुवाद डिरेक्टरी
- केंद्रीकृत कॉन्फ़िगरेशन `.env` फ़ाइल के माध्यम से (टेम्प्लेट के रूप में `.env.copy` का उपयोग करें)

## सेटअप कमांड्स

### प्रारंभिक रिपॉजिटरी सेटअप

```bash
# रिपॉजिटरी क्लोन करें
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# एनवायरनमेंट टेम्प्लेट कॉपी करें
cp .env.copy .env
# अपनी API कुंजी और एंडपॉइंट्स के साथ .env संपादित करें
```

### Python पर्यावरण सेटअप

```bash
# वर्चुअल पर्यावरण बनाएं
python3 -m venv venv

# वर्चुअल पर्यावरण सक्रिय करें
# macOS/Linux पर:
source venv/bin/activate
# Windows पर:
venv\Scripts\activate

# निर्भरताएं स्थापित करें
pip install -r requirements.txt
```

### Node.js/TypeScript सेटअप

```bash
# रूट-स्तरीय निर्भरताएँ स्थापित करें (डॉक्यूमेंटेशन टूलिंग के लिए)
npm install

# व्यक्तिगत पाठ TypeScript उदाहरणों के लिए, विशिष्ट पाठ पर जाएँ:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Dev कंटेनर सेटअप (सिफारिश की गई)

रिपॉजिटरी में GitHub Codespaces या VS Code Dev Containers के लिए `.devcontainer` कॉन्फ़िगरेशन शामिल है:

1. GitHub Codespaces या Dev Containers एक्सटेंशन के साथ VS Code में रिपॉजिटरी खोलें
2. Dev कंटेनर स्वचालित रूप से:
   - `requirements.txt` से Python निर्भरताएँ इंस्टॉल करेगा
   - पोस्ट-क्रिएट स्क्रिप्ट (`.devcontainer/post-create.sh`) चलाएगा
   - Jupyter कर्नल सेटअप करेगा

## विकास कार्यप्रवाह

### पर्यावरण चर

सभी पाठ जो API एक्सेस की आवश्यकता रखते हैं, में `.env` में परिभाषित पर्यावरण चर उपयोग किए जाते हैं:

- `OPENAI_API_KEY` - OpenAI API के लिए
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry में Azure OpenAI के लिए (Azure OpenAI सेवा अब Microsoft Foundry का हिस्सा है: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI एंडपॉइंट URL (Foundry रिसोर्स एंडपॉइंट)
- `AZURE_OPENAI_DEPLOYMENT` - चैट पूर्णता मॉडल परिनियोजन नाम
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - एम्बेडिंग्स मॉडल परिनियोजन नाम
- `AZURE_OPENAI_API_VERSION` - API संस्करण (डिफ़ॉल्ट: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face मॉडल के लिए
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry मॉडल्स एंडपॉइंट (मल्टी-प्रोवाइडर मॉडल कैटलॉग)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry मॉडल्स API कुंजी (सेवानिवृत्त हो रहे `GITHUB_TOKEN` की जगह)

## कोड शैली दिशानिर्देश

### Python

- पर्यावरण चर प्रबंधन के लिए `python-dotenv` का उपयोग करें
- API इंटरैक्शन के लिए `openai` पुस्तकालय को इम्पोर्ट करें
- लिंटिंग के लिए `pylint` का उपयोग करें (कुछ उदाहरणों में सरलता के लिए `# pylint: disable=all` शामिल है)
- PEP 8 नामकरण सम्मेलन का पालन करें
- API क्रेडेंशियल्स को `.env` फ़ाइल में संग्रहीत करें, कभी भी कोड में नहीं

### TypeScript

- पर्यावरण चर के लिए `dotenv` पैकेज का उपयोग करें
- प्रत्येक ऐप के लिए `tsconfig.json` में TypeScript कॉन्फ़िगरेशन
- Azure OpenAI के लिए `openai` पैकेज का उपयोग करें (क्लाइंट को `/openai/v1/` एंडपॉइंट पर पॉइंट करें और `client.responses.create` कॉल करें); Microsoft Foundry मॉडल्स के लिए `@azure-rest/ai-inference` का उपयोग करें
- विकास के लिए `nodemon` का उपयोग करें जो ऑटो-रिलोड करता है
- चलाने से पहले बिल्ड करें: `npm run build` फिर `npm start`

### सामान्य सम्मेलन

- कोड उदाहरणों को सरल और शैक्षिक रखें
- प्रमुख अवधारणाओं को समझाने वाले टिप्पणी शामिल करें
- प्रत्येक पाठ का कोड स्वदेशी और चलाने योग्य होना चाहिए
- लगातार नामकरण का उपयोग करें: Azure OpenAI के लिए `aoai-` उपसर्ग, OpenAI API के लिए `oai-`, Microsoft Foundry मॉडल्स के लिए `githubmodels-` (GitHub मॉडल युग से परंपरागत उपसर्ग)

**पर्यावरण चर गायब हैं**:

- उस पाठ के लिए आवश्यक सभी मान भरें जिस पर आप काम कर रहे हैं
- `.env` अपडेट करने के बाद अपने एप्लिकेशन को पुनः आरंभ करें
