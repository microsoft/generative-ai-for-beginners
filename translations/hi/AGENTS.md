- सुसंगत विकास पर्यावरण के लिए Dev Containers

## विकास कार्यप्रवाह

### पर्यावरण चर

API एक्सेस की आवश्यकता वाले सभी पाठ `.env` में परिभाषित पर्यावरण चर का उपयोग करते हैं:

- `OPENAI_API_KEY` - OpenAI API के लिए
- `AZURE_OPENAI_API_KEY` - Microsoft Foundry में Azure OpenAI के लिए (Azure OpenAI सेवा अब Microsoft Foundry का हिस्सा है: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI endpoint URL (Foundry संसाधन endpoint)
- `AZURE_OPENAI_DEPLOYMENT` - चैट पूर्णता मॉडल डिप्लॉयमेंट नाम (कोर्स डिफ़ॉल्ट: `gpt-5-mini`)
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - एम्बेडिंग मॉडल डिप्लॉयमेंट नाम (कोर्स डिफ़ॉल्ट: `text-embedding-3-small`)
- `AZURE_OPENAI_API_VERSION` - API संस्करण (डिफ़ॉल्ट: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face मॉडल्स के लिए
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry मॉडल्स endpoint (मल्टी-प्रोवाइडर मॉडल कैटलॉग)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry मॉडल्स API key (रिटायर हो रहे `GITHUB_TOKEN` की जगह)
- `AZURE_INFERENCE_CHAT_MODEL` - एक गैर-तर्कसंगत मॉडल (जैसे `Llama-3.3-70B-Instruct`) जिसे `temperature` उदाहरणों में उपयोग किया जाता है, क्योंकि तर्कसंगत मॉडल नमूना नियंत्रण का समर्थन नहीं करते

### मॉडल कन्वेंशंस (महत्वपूर्ण)

- **डिफ़ॉल्ट चैट मॉडल है `gpt-5-mini`** - एक वर्तमान, अप्रचलित नहीं **तर्कसंगत** मॉडल। 2026 तक पुराने temperature-सक्षम "मिनी" मॉडल (`gpt-4o-mini`, `gpt-4.1-mini`) *अप्रचलित* हो रहे हैं, इसलिए पाठ्यक्रम GPT-5 परिवार पर आधारित है।
- **तर्कसंगत मॉडल `temperature` और `top_p` को अस्वीकार करते हैं**, और `max_output_tokens` (Responses API) / `max_completion_tokens` (चैट पूर्णताओं) का उपयोग करते हैं बजाय `max_tokens` के। `gpt-5-mini` कॉल करने वाले नमूनों में `temperature`/`top_p`/`max_tokens` न जोड़ें।
- **`temperature` दिखाने के लिए**, नमूने **Llama** मॉडल (`Llama-3.3-70B-Instruct`) का उपयोग करते हैं Microsoft Foundry Models endpoint (`AZURE_INFERENCE_CHAT_MODEL`) के माध्यम से। तर्कसंगत मॉडल को नमूना नियंत्रण के बजाय प्रांप्ट इंजीनियरिंग + तर्क नियंत्रण से नियंत्रित करें।
- **फाइन-ट्यूनिंग (पाठ 18)** `gpt-4.1-mini` को बनाए रखता है: GPT-5 केवल रिइंफोर्समेंट फाइन-ट्यूनिंग (RFT) का समर्थन करता है, न कि सुपरवाइज्ड फाइन-ट्यूनिंग (SFT) जैसा कि वहां दिखाया गया है।
- पाठ 20 (Mistral) और 21 (Meta) `temperature`/`max_tokens` बनाए रखते हैं क्योंकि वे Mistral/Llama मॉडल्स को लक्षित करते हैं, जो इन्हें सपोर्ट करते हैं।

## कोड शैली दिशानिर्देश

### Python

- पर्यावरण चर प्रबंधन के लिए `python-dotenv` का उपयोग करें
- API इंटरैक्शन के लिए `openai` लाइब्रेरी आयात करें
- लिंटिंग के लिए `pylint` का उपयोग करें (कुछ उदाहरण में सरलता के लिए `# pylint: disable=all` शामिल है)
- PEP 8 नामकरण कन्वेंशंस का पालन करें
- API क्रेडेंशियल्स को `.env` फ़ाइल में स्टोर करें, कोड में कभी नहीं

### TypeScript

- पर्यावरण चर के लिए `dotenv` पैकेज का उपयोग करें
- प्रत्येक ऐप के लिए `tsconfig.json` में TypeScript कॉन्फ़िगरेशन
- Azure OpenAI के लिए `openai` पैकेज का उपयोग करें (क्लाइंट को `/openai/v1/` endpoint पर पॉइंट करें और `client.responses.create` कॉल करें); Microsoft Foundry मॉडल्स के लिए `@azure-rest/ai-inference` का उपयोग करें
- ऑटो-रिलोड के साथ विकास के लिए `nodemon` का उपयोग करें
- चलाने से पहले बिल्ड करें: `npm run build` फिर `npm start`

### सामान्य कन्वेंशंस

- कोड उदाहरण सरल और शैक्षिक रखें
- मुख्य अवधारणाओं को समझाने वाले टिप्पणियां शामिल करें

## समस्या निवारण

### सामान्य समस्याएं

**Python इम्पोर्ट त्रुटियां**:
- सुनिश्चित करें कि वर्चुअल वातावरण सक्रिय है
- `pip install -r requirements.txt` चलाएं
- Python संस्करण 3.9+ है यह जांचें

**TypeScript निर्माण त्रुटियां**:
- संबंधित ऐप निर्देशिका में `npm install` चलाएं
- Node.js संस्करण संगत है यह जांचें
- `node_modules` क्लियर करें और आवश्यक होने पर पुनः इंस्टॉल करें

**API प्रमाणीकरण त्रुटियां**:
- सुनिश्चित करें कि `.env` फ़ाइल मौजूद है और सही मान हैं
- API keys वैध और एक्सपायर्ड नहीं हैं यह जांचें
- अपने क्षेत्र के लिए endpoint URLs सही हैं यह सुनिश्चित करें

**पर्यावरण चर गायब हैं**:
- `.env.copy` को `.env` में कॉपी करें
- जिस पाठ पर काम कर रहे हैं उसके लिए सभी आवश्यक मान भरें
- `.env` अपडेट करने के बाद अपना एप्लिकेशन पुनः आरंभ करें
