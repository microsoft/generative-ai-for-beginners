## 4. Модернизация API

### 4.1 Устаревшие паттерны API (Миграция завершена)

Все Python и TypeScript **чат**-примеры были мигрированы с Chat Completions API на **Responses API** (`client.responses.create(...)` → `response.output_text`).

| Старый паттерн | Новый паттерн | Статус |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (чат) | `OpenAI(base_url="<endpoint>/openai/v1/")` (Responses API) | Выполнено |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | Выполнено |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (TypeScript) | пакет `openai` `client.responses.create()` → `response.output_text` | Выполнено |
| `df.append()` (pandas) | `pd.concat()` | Выполнено |

> **Примечание:** Образцы Microsoft Foundry Models, использующие SDK `azure-ai-inference` / `@azure-rest/ai-inference` (`client.complete()`), остаются на Model Inference API, который не поддерживает Responses API. `AzureOpenAI()` намеренно сохранён там, где это актуально (эмбеддинги и генерация изображений).

### 4.2 Новые возможности API для демонстрации

1. **Структурированные ответы** (OpenAI)
   - Режим JSON
   - Вызов функций с жёсткими схемами

2. **Визуальные возможности**
   - Анализ изображений с GPT-4o (vision)

3. **Встроенные инструменты Responses API** (заменяет устаревший Assistants API)

## 5. Улучшения инфраструктуры

### 5.1 Улучшения CI/CD

Внедрено в [`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml): линтинг и форматирование Python (Ruff + Black) **обязательны** для поддерживаемого модуля `shared/` и запускаются **рекомендательно** по всему остальному курсу, плюс рекомендательный ESLint проход для JavaScript/TypeScript. Иллюстративный эталон:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### 5.2 Сканирование безопасности

Внедрено в [`.github/workflows/security.yml`](../../../.github/workflows/security.yml): анализ CodeQL для Python и JavaScript/TypeScript (при пуше, pull request и еженедельно) плюс проверка зависимостей при pull request. Иллюстративный эталон:

```yaml
# .github/workflows/security.yml
name: Security Scan

on: [push, pull_request]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: javascript, python
      - uses: github/codeql-action/analyze@v3

  dependency-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
```
