# Введение в маленькие языковые модели для генеративного ИИ для начинающих
Генеративный ИИ — это увлекательная область искусственного интеллекта, которая фокусируется на создании систем, способных генерировать новый контент. Этот контент может варьироваться от текста и изображений до музыки и даже целых виртуальных сред. Одним из самых захватывающих приложений генеративного ИИ являются языковые модели.

## Что такое маленькие языковые модели?

Маленькая языковая модель (SLM) представляет собой уменьшенную версию большой языковой модели (LLM), использующую многие архитектурные принципы и техники LLM, при этом имеющую значительно меньший вычислительный след. 

SLM являются подмножеством языковых моделей, предназначенных для генерации текста, похожего на человеческий. В отличие от их больших аналогов, таких как GPT-4, SLM более компактны и эффективны, что делает их идеальными для приложений с ограниченными вычислительными ресурсами. Несмотря на меньший размер, они могут выполнять широкий спектр задач. Как правило, SLM создаются путём сжатия или дистилляции LLM с целью сохранить значительную часть функциональности и лингвистических возможностей исходной модели. Сокращение размера модели уменьшает общую сложность, делая SLM более эффективными с точки зрения использования памяти и вычислительных требований. Несмотря на эти оптимизации, SLM всё ещё способны выполнять широкий спектр задач обработки естественного языка (NLP):

- Генерация текста: Создание связных и контекстно релевантных предложений или абзацев.
- Завершение текста: Предсказание и дополнение предложений на основе заданного запроса.
- Перевод: Перевод текста с одного языка на другой.
- Суммаризация: Сокращение длинных текстов до более коротких, удобоваримых резюме.

Хотя с некоторыми компромиссами в производительности или глубине понимания по сравнению с их более крупными аналогами. 

## Как работают маленькие языковые модели?
SLM обучаются на огромных объёмах текстовых данных. Во время обучения они изучают паттерны и структуры языка, что позволяет им генерировать текст, который одновременно грамматически правильный и контекстно уместный. Процесс обучения включает:

- Сбор данных: Сбор больших наборов текстов из различных источников.
- Предобработка: Очистка и организация данных для подготовки к обучению.
- Обучение: Использование алгоритмов машинного обучения для обучения модели понимать и генерировать текст.
- Тонкая настройка: Корректировка модели для улучшения её работы на конкретных задачах.

Разработка SLM соответствует растущей потребности в моделях, которые можно развертывать в условиях ограниченных ресурсов, таких как мобильные устройства или платформы edge-компьютинга, где полноразмерные LLM могут быть непрактичны из-за их высоких требований к ресурсам. Сосредоточившись на эффективности, SLM обеспечивают баланс между производительностью и доступностью, распространяя применение моделей на различные области.

## Отличия больших языковых моделей (LLM) и маленьких языковых моделей (SLM)

И LLM, и SLM построены на основе фундаментальных принципов вероятностного машинного обучения, следуя схожим подходам в архитектуре, методах обучения, процессах генерации данных и методах оценки моделей. Тем не менее, существует несколько ключевых факторов, отличающих эти два типа моделей.

## Применения маленьких языковых моделей

SLM находят широкое применение, включая:

- Чатботы: Обеспечение поддержки клиентов и взаимодействие с пользователями в формате разговора.
- Создание контента: Помощь авторам с генерацией идей или даже написанием целых статей.
- Образование: Помощь студентам с письменными заданиями или изучение новых языков.
- Доступность: Создание инструментов для людей с инвалидностью, таких как системы преобразования текста в речь.

**Размер**
  
Главное отличие между LLM и SLM заключается в масштабе моделей. LLM, такие как ChatGPT (GPT-4), могут содержать примерно 1,76 триллиона параметров, в то время как открытые SLM, например Mistral 7B, имеют значительно меньше параметров — примерно 7 миллиардов. Это различие обусловлено в первую очередь архитектурой модели и процессом обучения. Например, ChatGPT использует механизм самовнимания в энкодер-декодерной структуре, тогда как Mistral 7B применяет скользящее окно внимания, что позволяет более эффективно обучаться в модели типа только декодер. Эти архитектурные различия сильно влияют на сложность и производительность моделей.

**Понимание**

SLM обычно оптимизированы для работы в конкретных предметных областях, делая их высокоспециализированными, но потенциально ограниченными в способности обеспечивать широкое контекстное понимание в различных областях знаний. В отличие от них, LLM стремятся симулировать интеллект, похожий на человеческий, на более всеобъемлющем уровне. Обученные на огромных и разнообразных наборах данных, LLM предназначены для эффективной работы в различных областях, предлагая большую универсальность и адаптивность. Следовательно, LLM более подходят для широкого спектра downstream-задач, таких как обработка естественного языка и программирование.

**Вычисления**

Обучение и развертывание LLM — ресурсоёмкие процессы, часто требующие значительной вычислительной инфраструктуры, включая крупномасштабные GPU-кластеры. Например, обучение такой модели, как ChatGPT с нуля, может потребовать тысячи GPU в течение длительного времени. В отличие от них, SLM с меньшим числом параметров более доступны с точки зрения вычислительных ресурсов. Модели, такие как Mistral 7B, можно обучать и запускать на локальных машинах с умеренными возможностями GPU, хотя обучение всё равно требует несколько часов на нескольких GPU.

**Смещение**

Смещение — известная проблема LLM, связанная в первую очередь с характером обучающих данных. Эти модели часто используют необработанные, общедоступные данные из интернета, которые могут недостаточно репрезентировать или неправильно представлять отдельные группы, вводить ошибочную маркировку или отражать лингвистические предубеждения, обусловленные диалектами, географическими вариациями и грамматическими правилами. Кроме того, комплексность архитектур LLM может непреднамеренно усугублять смещение, которое остаётся незамеченным без тщательной тонкой настройки. С другой стороны, SLM, обучающиеся на более ограниченных, предметно-специфичных наборах данных, изначально менее подвержены таким смещениям, хотя и не полностью застрахованы от них.

**Вывод**

Меньший размер SLM даёт им значительное преимущество по скорости вывода, позволяя эффективно генерировать результаты на локальном оборудовании без необходимости масштабной параллельной обработки. В отличие от этого, LLM из-за своего размера и сложности часто требуют значительных параллельных вычислительных ресурсов для достижения приемлемого времени вывода. Наличие множества одновременных пользователей дополнительно замедляет время отклика LLM, особенно при их масштабном развертывании.

В итоге, хотя и LLM, и SLM имеют общую базу в машинном обучении, они значительно различаются по размеру модели, требованиям к ресурсам, контекстному пониманию, подверженности смещениям и скорости вывода. Эти отличия отражают их соответствующую пригодность для разных случаев использования: LLM более универсальны, но ресурсозатратны, тогда как SLM предлагают более предметно-специализированную эффективность с уменьшенными вычислительными затратами.

## Представление семейства Phi-3 / Phi-3.5

Семейство Phi-3 / 3.5 в основном ориентировано на сценарии применения в тексте, визуальном восприятии и агентах (MoE):

### Phi-3 / 3.5 Instruct

В основном для генерации текста, дополнения чата и извлечения информации из контента и прочего.

**Phi-3-mini**

Языковая модель с 3,8 млрд параметров доступна на Microsoft Foundry, Hugging Face и Ollama. Модели Phi-3 значительно превосходят языковые модели равного и большего размера по основным бенчмаркам (см. показатели ниже, чем выше — тем лучше). Phi-3-mini превосходит модели вдвое большего размера, тогда как Phi-3-small и Phi-3-medium обгоняют более крупные модели, включая GPT-3.5.

**Phi-3-small и medium**

С всего 7 млрд параметров Phi-3-small превосходит GPT-3.5T по ряду языковых, логических, кодировочных и математических бенчмарков.

Phi-3-medium с 14 млрд параметров продолжает эту тенденцию и превосходит Gemini 1.0 Pro.

**Phi-3.5-mini**

Его можно рассматривать как улучшенную версию Phi-3-mini. Хотя количество параметров остаётся прежним, улучшена способность поддерживать множество языков (более 20 языков: арабский, китайский, чешский, датский, голландский, английский, финский, французский, немецкий, иврит, венгерский, итальянский, японский, корейский, норвежский, польский, португальский, русский, испанский, шведский, тайский, турецкий, украинский) и добавлена более мощная поддержка длинного контекста.

Phi-3.5-mini с 3,8 млрд параметров превосходит языковые модели того же размера и сравним с моделями вдвое большего размера.

### Phi-3 / 3.5 Vision

Модель Instruct Phi-3/3.5 можно рассматривать как способность Phi к пониманию, а Vision — как "глаза" Phi для понимания мира.


**Phi-3-Vision**

Phi-3-vision с 4,2 млрд параметров продолжает эту тенденцию и превосходит более крупные модели такие, как Claude-3 Haiku и Gemini 1.0 Pro V по общим задачам визуального рассуждения, OCR и понимания таблиц и диаграмм.


**Phi-3.5-Vision**

Phi-3.5-Vision также является улучшенной версией Phi-3-Vision, добавляющей поддержку нескольких изображений. Можно считать это улучшением зрения — теперь модель может видеть не только картинки, но и видео.

Phi-3.5-vision превосходит более крупные модели, такие как Claude-3.5 Sonnet и Gemini 1.5 Flash в задачах OCR, понимания таблиц и графиков и сравним по общим задачам визуального рассуждения. Поддерживает ввод из нескольких кадров, то есть выполняет рассуждения по множественным входным изображениям.


### Phi-3.5-MoE

***Mixture of Experts (MoE)*** позволяет моделям обучаться с гораздо меньшими вычислительными затратами, что означает, что можно значительно увеличить размер модели или набора данных при том же бюджете вычислений, что и у плотной модели. В частности, модель MoE должна быстрее достигать того же качества, что и её плотный аналог, во время предварительного обучения.

Phi-3.5-MoE состоит из 16 модулей экспертов по 3,8 млрд параметров. Phi-3.5-MoE с всего 6,6 млрд активных параметров достигает схожего уровня рассуждений, понимания языка и математики с гораздо более крупными моделями.

Мы можем использовать модель семейства Phi-3/3.5 в различных сценариях. В отличие от LLM, Phi-3/3.5-mini или Phi-3/3.5-Vision можно развернуть на устройствах на периферии.

### Запуск Phi-3/3.5 локально
Инференс в отношении Phi-3 или любой другой языковой модели, такой как GPT-3, относится к процессу генерации ответов или предсказаний на основе полученного ввода. Когда вы вводите запрос или вопрос в Phi-3, он использует свою обученную нейросеть, чтобы вывести наиболее вероятный и релевантный ответ, анализируя паттерны и связи в данных, на которых он обучался.

**Hugging Face Transformer**
Hugging Face Transformers — это мощная библиотека, предназначенная для обработки естественного языка (NLP) и других задач машинного обучения. Вот несколько ключевых моментов о ней:

1. **Предварительно обученные модели**: Она предоставляет тысячи предварительно обученных моделей, которые можно использовать для различных задач, таких как классификация текста, распознавание именованных объектов, ответы на вопросы, суммирование, перевод и генерация текста.

2. **Совместимость с фреймворками:** Библиотека поддерживает несколько фреймворков глубокого обучения, включая PyTorch, TensorFlow и JAX. Это позволяет обучать модель в одном фреймворке и использовать ее в другом.

3. **Мультимодальные возможности:** Кроме NLP, Hugging Face Transformers также поддерживает задачи компьютерного зрения (например, классификацию изображений, обнаружение объектов) и обработки аудио (например, распознавание речи, классификация аудио).

4. **Удобство использования:** Библиотека предоставляет API и инструменты для легкого скачивания и тонкой настройки моделей, делая её доступной как для новичков, так и для экспертов.

5. **Сообщество и ресурсы:** Hugging Face имеет активное сообщество и обширную документацию, учебники и руководства, которые помогают пользователям начать работу и максимально использовать библиотеку.

**ONNX Runtime для GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) — это кроссплатформенный ускоритель инференса и обучения моделей машинного обучения. ONNX Runtime для Generative AI (GENAI) — мощный инструмент, который помогает эффективно запускать модели генеративного ИИ на различных платформах.

## Что такое ONNX Runtime?
ONNX Runtime — это проект с открытым исходным кодом, который обеспечивает высокопроизводительный инференс моделей машинного обучения. Он поддерживает модели в формате Open Neural Network Exchange (ONNX), являющемся стандартом для представления моделей машинного обучения. Инференс с помощью ONNX Runtime может обеспечить более быстрый опыт пользователей и снизить затраты, поддерживая модели из фреймворков глубокого обучения, таких как PyTorch и TensorFlow/Keras, а также классические библиотеки машинного обучения, такие как scikit-learn, LightGBM, XGBoost и др. ONNX Runtime совместим с различным оборудованием, драйверами и операционными системами, предоставляя оптимальную производительность за счет использования аппаратных ускорителей, где это возможно, вместе с оптимизациями и преобразованиями графов.

## Что такое генеративный ИИ?
Генеративный ИИ относится к системам ИИ, которые могут создавать новый контент, например текст, изображения или музыку, на основе данных, на которых они обучены. Примерами являются языковые модели, такие как GPT-3, и модели генерации изображений, такие как Stable Diffusion. Библиотека ONNX Runtime для GenAI обеспечивает цикл генеративного ИИ для моделей ONNX, включая инференс с помощью ONNX Runtime, обработку логитов, поиск и сэмплинг, а также управление кешем KV.

## ONNX Runtime для GENAI
ONNX Runtime для GENAI расширяет возможности ONNX Runtime, чтобы поддерживать генеративные модели ИИ. Вот некоторые ключевые особенности:

- **Широкая поддержка платформ:** Работает на различных платформах, включая Windows, Linux, macOS, Android и iOS.
- **Поддержка моделей:** Поддерживает многие популярные модели генеративного ИИ, такие как LLaMA, GPT-Neo, BLOOM и другие.
- **Оптимизация производительности:** Включает оптимизации для различных аппаратных ускорителей, таких как GPU NVIDIA, GPU AMD и др.
- **Удобство использования:** Предоставляет API для простой интеграции в приложения, позволяя генерировать текст, изображения и другой контент с минимальным количеством кода.
- Пользователи могут вызывать высокий уровень метода generate() или запускать каждую итерацию модели в цикле, генерируя по одному токену за раз и при необходимости обновляя параметры генерации внутри цикла.
- ONNX Runtime также поддерживает жадный/лучевой поиск и сэмплинг TopP, TopK для генерации последовательностей токенов, а также встроенную обработку логитов, такую как штрафы за повторения. Вы также можете легко добавить собственную оценку.

## Начало работы
Чтобы начать работу с ONNX Runtime для GENAI, вы можете выполнить следующие шаги:

### Установка ONNX Runtime:
```Python
pip install onnxruntime
```
### Установка расширений для генеративного ИИ:
```Python
pip install onnxruntime-genai
```

### Запуск модели: Вот простой пример на Python:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### Демонстрация: Использование ONNX Runtime GenAI для вызова Phi-3.5-Vision


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
