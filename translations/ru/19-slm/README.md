# Введение в небольшие языковые модели для генеративного ИИ для начинающих
Генеративный ИИ — это захватывающая область искусственного интеллекта, которая сосредоточена на создании систем, способных генерировать новый контент. Этот контент может варьироваться от текста и изображений до музыки и даже целых виртуальных сред. Одно из самых интересных применений генеративного ИИ — в области языковых моделей.

## Что такое небольшие языковые модели?

Небольшая языковая модель (SLM) представляет собой уменьшенную версию большой языковой модели (LLM), использующую многие архитектурные принципы и методы LLM, но при этом обладающую значительно меньшими вычислительными затратами.

SLM являются подмножеством языковых моделей, предназначенных для генерации текста, похожего на человеческий. В отличие от своих больших аналогов, таких как GPT-4, SLM более компактны и эффективны, что делает их идеальными для применения в условиях ограниченных вычислительных ресурсов. Несмотря на меньший размер, они способны выполнять множество задач. Обычно SLM создаются путем сжатия или дистилляции LLM с целью сохранить значительную часть функций и языковых возможностей исходной модели. Это снижение размера модели уменьшает общую сложность, делая SLM более эффективными как с точки зрения использования памяти, так и вычислительных требований. Несмотря на эти оптимизации, SLM могут выполнять широкий спектр задач обработки естественного языка (NLP):

- Генерация текста: создание связанных и контекстуально релевантных предложений или абзацев.
- Дополнение текста: предсказание и завершение предложений на основе заданного запроса.
- Перевод: преобразование текста с одного языка на другой.
- Суммирование: сокращение длинных текстов до более коротких и удобоваримых резюме.

Хотя и с некоторыми компромиссами в производительности или глубине понимания по сравнению с их большими аналогами.

## Как работают небольшие языковые модели?
SLM обучаются на огромных объемах текстовых данных. Во время обучения они изучают паттерны и структуры языка, что позволяет им генерировать текст, который одновременно грамматически правильный и контекстуально уместный. Процесс обучения включает:

- Сбор данных: накопление больших наборов текстов из различных источников.
- Предобработка: очистка и организация данных для подготовки к обучению.
- Обучение: использование алгоритмов машинного обучения для обучения модели пониманию и генерации текста.
- Тонкая настройка: корректировка модели для улучшения ее работы в конкретных задачах.

Разработка SLM соответствует растущей потребности в моделях, которые можно развертывать в условиях ограниченных ресурсов, таких как мобильные устройства или периферийные вычислительные платформы, где полные масштабные LLM могут быть непрактичны из-за высоких ресурсов. Сосредотачиваясь на эффективности, SLM балансируют производительность с доступностью, что позволяет шире использовать их в различных областях.

## Отличия между большими языковыми моделями (LLM) и небольшими языковыми моделями (SLM)

И LLM, и SLM строятся на фундаментальных принципах вероятностного машинного обучения, следуя схожим подходам в архитектуре, методах обучения, процессах генерации данных и методах оценки моделей. Однако существуют несколько ключевых факторов, которые различают эти два типа моделей.

## Применение небольших языковых моделей

SLM находят применение в самых разных областях, включая:

- Чат-боты: предоставление поддержки клиентам и взаимодействие с пользователями в диалоговом режиме.
- Создание контента: помощь писателям в генерации идей или даже создании целых статей.
- Образование: помощь студентам с письменными заданиями или изучением новых языков.
- Доступность: создание инструментов для людей с ограниченными возможностями, таких как системы преобразования текста в речь.

**Размер**
  
Основное отличие между LLM и SLM заключается в масштабе моделей. LLM, такие как ChatGPT (GPT-4), могут насчитывать порядка 1,76 триллиона параметров, тогда как открытые SLM, например Mistral 7B, имеют значительно меньше параметров — около 7 миллиардов. Такое различие объясняется, в первую очередь, разницей в архитектуре моделей и процессах обучения. Например, ChatGPT использует механизм самовнимания в рамках архитектуры энкодер-декодер, тогда как Mistral 7B применяет скользящее внимание, что позволяет более эффективно обучаться в модели только с декодером. Эта архитектурная разница имеет глубокие последствия для сложности и производительности моделей.

**Понимание**

SLM обычно оптимизируются для работы в конкретных областях, что делает их высокоспециализированными, но потенциально ограниченными в способности обеспечивать широкое контекстуальное понимание множества сфер знаний. Напротив, LLM стремятся симулировать человеческий интеллект на более всеобъемлющем уровне. Обученные на огромных, разнообразных наборах данных, LLM предназначены для эффективной работы в различных областях, обеспечивая большую универсальность и адаптивность. Следовательно, LLM более подходят для выполнения широкого спектра задач, таких как обработка естественного языка и программирование.

**Вычислительные ресурсы**

Обучение и развёртывание LLM требуют значительных ресурсов, часто включая мощные GPU-кластеры большого масштаба. К примеру, обучение модели вроде ChatGPT с нуля может требовать тысячи GPU в течение продолжительного времени. В отличие от этого, SLM с меньшим числом параметров более доступны с точки зрения вычислительных ресурсов. Модели типа Mistral 7B могут обучаться и запускаться на локальных машинах с умеренной GPU-вычислительной мощностью, хотя процесс обучения всё равно занимает несколько часов на нескольких GPU.

**Смещение (Bias)**

Проблема смещения известна в LLM, главным образом из-за особенностей обучающих данных. Эти модели часто используют сырые, открытые данные из интернета, которые могут недостаточно или искажённо представлять определённые группы, содержать ошибочную маркировку или отражать языковые предубеждения, основанные на диалектах, географических различиях и грамматических правилах. Кроме того, сложность архитектуры LLM может непреднамеренно усиливать смещение, которое остаётся незамеченным без тщательной настройки. С другой стороны, SLM, обучаемые на более ограниченных, специализированных наборах данных, менее подвержены таким смещениям, хоть и не полностью застрахованы от них.

**Инференс**

Меньший размер SLM даёт им значительное преимущество в скорости вывода, позволяя быстро генерировать результаты на локальном оборудовании без необходимости масштабной параллельной обработки. В противоположность этому, LLM из-за их размера и сложности часто требуют значительных параллельных вычислительных ресурсов для достижения приемлемого времени ответа. Наличие множества одновременных пользователей дополнительно замедляет реакции LLM, особенно при масштабном развертывании.

В итоге, хотя LLM и SLM основываются на общих принципах машинного обучения, они существенно отличаются по размеру моделей, требованиям к ресурсам, уровню контекстуального понимания, восприимчивости к смещениям и скорости вывода. Эти различия отражают их соответствующую пригодность для разных случаев использования: LLM более универсальны, но требовательны к ресурсам, а SLM предлагают более высокую эффективность в специализированных областях при сниженных вычислительных затратах.

## Введение в семейство Phi-3 / Phi-3.5

Семейство Phi-3 / 3.5 нацелено в основном на сценарии применения с текстом, визуалом и агентами (MoE):

### Phi-3 / 3.5 Instruct

В основном для генерации текста, дополнения диалогов и извлечения информации из контента и т.д.

**Phi-3-mini**

Языковая модель с 3.8 миллиардами параметров доступна в Microsoft Foundry, Hugging Face и Ollama. Модели Phi-3 существенно превосходят языковые модели равного или большего размера по ключевым тестам производительности (см. приведённые ниже цифры бенчмарков, где выше — лучше). Phi-3-mini превосходит модели вдвое большего размера, а Phi-3-small и Phi-3-medium превосходят более крупные модели, включая GPT-3.5.

**Phi-3-small & medium**

С 7 миллиардами параметров Phi-3-small превосходит GPT-3.5T по различным языковым, логическим, программным и математическим тестам.

Phi-3-medium, с 14 миллиардами параметров, продолжает эту тенденцию и превосходит Gemini 1.0 Pro.

**Phi-3.5-mini**

Можно рассматривать как улучшение Phi-3-mini. При том же числе параметров она расширяет поддержку нескольких языков (поддержка более 20 языков: арабский, китайский, чешский, датский, голландский, английский, финский, французский, немецкий, иврит, венгерский, итальянский, японский, корейский, норвежский, польский, португальский, русский, испанский, шведский, тайский, турецкий, украинский) и добавляет более мощную поддержку длинного контекста.

Phi-3.5-mini с 3.8 миллиардами параметров превосходит языковые модели того же размера и сравнима с моделями вдвое большего размера.

### Phi-3 / 3.5 Vision

Можно считать модель Instruct Phi-3/3.5 способностью Phi к пониманию, а Vision — глазами Phi для понимания мира.


**Phi-3-Vision**

Phi-3-Vision с 4.2 миллиардами параметров продолжает тенденцию и превосходит более крупные модели такие, как Claude-3 Haiku и Gemini 1.0 Pro V, в задачах общего визуального рассуждения, OCR и понимания таблиц и диаграмм.


**Phi-3.5-Vision**

Phi-3.5-Vision — это также улучшенная версия Phi-3-Vision, добавляющая поддержку множества изображений. Можно считать это улучшением зрения — теперь модель видит не только отдельные картинки, но и видео.

Phi-3.5-Vision превосходит более крупные модели, такие как Claude-3.5 Sonnet и Gemini 1.5 Flash, по задачам OCR, понимания таблиц и графиков и сопоставима с ними по задачам общего визуального рассуждения. Поддерживает ввод нескольких кадров, т.е. выполнение рассуждений на множестве входных изображений.


### Phi-3.5-MoE

***Mixture of Experts (MoE)*** позволяет обучать модели с гораздо меньшими вычислительными затратами, что позволяет значительно масштабировать размер модели или набор данных при том же бюджете вычислений, что и плотная модель. В частности, модель MoE должна достигать того же качества, что и ее плотный аналог, гораздо быстрее во время предобучения.

Phi-3.5-MoE состоит из 16 модулей по 3.8 миллиарда параметров каждый. Phi-3.5-MoE с лишь 6.6 миллиардами активных параметров достигает похожего уровня рассуждений, понимания языка и математики, как гораздо более крупные модели.

Мы можем использовать модели семейства Phi-3/3.5 в разных сценариях. В отличие от LLM, Phi-3/3.5-mini или Phi-3/3.5-Vision можно разворачивать на периферийных устройствах.

### Запуск Phi-3/3.5 локально
Инференс в контексте Phi-3 или любой языковой модели, такой как GPT-3, относится к процессу генерации ответов или предсказаний на основе полученного ввода. Когда вы предоставляете запрос или вопрос Phi-3, он использует свою обученную нейронную сеть для вывода наиболее вероятного и релевантного ответа, анализируя паттерны и связи в данных, на которых был обучен.

**Hugging Face Transformer**
Hugging Face Transformers — мощная библиотека, разработанная для обработки естественного языка (NLP) и других задач машинного обучения. Вот некоторые ключевые моменты о ней:

1. **Предобученные модели**: Она предоставляет тысячи предобученных моделей, которые можно использовать для различных задач, таких как классификация текста, распознавание именованных сущностей, ответы на вопросы, суммирование, перевод и генерация текста.

2. **Совместимость с фреймворками:** Библиотека поддерживает несколько фреймворков глубокого обучения, включая PyTorch, TensorFlow и JAX. Это позволяет обучать модель в одном фреймворке и использовать её в другом.

3. **Мультимодальные возможности:** Помимо NLP, Hugging Face Transformers также поддерживает задачи в области компьютерного зрения (например, классификация изображений, обнаружение объектов) и аудиообработки (например, распознавание речи, классификация аудио).

4. **Простота использования:** Библиотека предлагает API и инструменты для легкой загрузки и дообучения моделей, делая её доступной как для новичков, так и для экспертов.

5. **Сообщество и ресурсы:** Hugging Face имеет активное сообщество и обширную документацию, учебники и руководства, которые помогают пользователям начать работу и максимально эффективно использовать библиотеку.

**ONNX Runtime для GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) — это кроссплатформенный ускоритель инференса и обучения машинного обучения. ONNX Runtime для Generative AI (GENAI) — мощный инструмент, который помогает эффективно запускать модели генеративного ИИ на различных платформах.

## Что такое ONNX Runtime?
ONNX Runtime — проект с открытым исходным кодом, обеспечивающий высокопроизводительный инференс моделей машинного обучения. Он поддерживает модели в формате Open Neural Network Exchange (ONNX), который является стандартом для представления моделей машинного обучения. Инференс в ONNX Runtime может обеспечивать более быстрый пользовательский опыт и снижать затраты, поддерживая модели из фреймворков глубокого обучения, таких как PyTorch и TensorFlow/Keras, а также классические библиотеки машинного обучения, такие как scikit-learn, LightGBM, XGBoost и другие. ONNX Runtime совместим с различным оборудованием, драйверами и операционными системами и обеспечивает оптимальную производительность, используя аппаратные акселераторы, где это возможно, а также применяя оптимизации и преобразования графа.

## Что такое генеративный ИИ?
Генеративный ИИ — это системы ИИ, способные создавать новый контент, такой как текст, изображения или музыка, на основе данных, на которых они были обучены. Примерами являются языковые модели, такие как GPT-3, и модели генерации изображений, такие как Stable Diffusion. Библиотека ONNX Runtime для GenAI предоставляет цикл генеративного ИИ для моделей ONNX, включая инференс с ONNX Runtime, обработку логитов, поиск и выборку, а также управление кешем KV.

## ONNX Runtime для GENAI
ONNX Runtime для GENAI расширяет возможности ONNX Runtime для поддержки моделей генеративного ИИ. Вот некоторые ключевые функции:

- **Широкая поддержка платформ:** Работает на различных платформах, включая Windows, Linux, macOS, Android и iOS.
- **Поддержка моделей:** Поддерживает множество популярных моделей генеративного ИИ, таких как LLaMA, GPT-Neo, BLOOM и другие.
- **Оптимизация производительности:** Включает оптимизации для различных аппаратных акселераторов, таких как GPU NVIDIA, GPU AMD и другие.
- **Простота использования:** Предоставляет API для легкой интеграции в приложения, позволяя генерировать текст, изображения и другой контент с минимальным кодом.
- Пользователи могут вызывать высокоуровневый метод generate() или запускать каждую итерацию модели в цикле, генерируя по одному токену за раз и при необходимости обновляя параметры генерации внутри цикла.
- ONNX Runtime также поддерживает жадный поиск/поиск по лучу (greedy/beam search) и методы выборки TopP, TopK для генерации последовательностей токенов, а также встроенную обработку логитов, например, штрафы за повторения. Вы также можете легко добавить пользовательскую оценку.

## Начало работы
Чтобы начать работу с ONNX Runtime для GENAI, выполните следующие шаги:

### Установите ONNX Runtime:
```Python
pip install onnxruntime
```
### Установите расширения для генеративного ИИ:
```Python
pip install onnxruntime-genai
```

### Запустите модель: Вот простой пример на Python:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### Демонстрация: Использование ONNX Runtime GenAI для вызова Phi-3.5-Vision


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
