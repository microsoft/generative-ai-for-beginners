# Тонкая настройка вашего LLM

Использование больших языковых моделей для создания приложений генеративного ИИ связано с новыми вызовами. Ключевая проблема — обеспечение качества ответа (точности и релевантности) в содержании, сгенерированном моделью для заданного запроса пользователя. В предыдущих уроках мы обсуждали такие техники, как инженерия запросов и генерация с поддержкой поиска, которые пытаются решить проблему путем _модификации входного запроса_ для существующей модели.

В сегодняшнем уроке мы рассмотрим третью технику — **тонкую настройку**, которая пытается решить задачу посредством _дообучения самой модели_ на дополнительных данных. Давайте углубимся в детали.

## Что такое тонкая настройка для языковых моделей?

По определению, большие языковые модели _предобучены_ на больших объемах текста из разнообразных источников, включая интернет. Как мы узнали в предыдущих уроках, нам нужны техники, такие как _инженерия запросов_ и _генерация с поддержкой поиска_, чтобы улучшить качество ответов модели на вопросы пользователя ("промпты").

Популярная техника инженерии запросов предполагает предоставление модели дополнительного руководства о том, чего ожидают в ответе, либо через _инструкции_ (явное руководство), либо _путем предоставления нескольких примеров_ (неявное руководство). Это называется _обучением с малым числом примеров_, но у него есть два ограничения:

- Ограничения на количество токенов модели могут ограничить число примеров, которые вы можете дать, и эффективность.
- Стоимость токенов модели может сделать дорогостоящим добавление примеров в каждый промпт и ограничить гибкость.

Тонкая настройка — широко применяемая практика в машинном обучении, когда мы берем предобученную модель и дообучаем её на новых данных для повышения производительности на конкретной задаче. В контексте языковых моделей можно тонко настроить предобученную модель _на специально подобранном наборе примеров для конкретной задачи или области применения_, чтобы создать **кастомную модель**, которая может быть более точной и релевантной для этой задачи или области. Дополнительным плюсом тонкой настройки является уменьшение количества примеров, необходимых для обучения с малым числом примеров — что снижает использование токенов и связанные с этим затраты.

## Когда и почему стоит выполнять тонкую настройку моделей?

В _этом_ контексте, когда мы говорим о тонкой настройке, мы имеем в виду **контролируемую** тонкую настройку, при которой дообучение происходит за счет **добавления новых данных**, которые не входили в исходный обучающий набор. Это отличается от неконтролируемой тонкой настройки, когда модель дообучается на оригинальных данных, но с другими гиперпараметрами.

Главное помнить, что тонкая настройка — это продвинутая техника, требующая определенного уровня экспертизы для получения желаемых результатов. Если её выполнять неправильно, можно не получить ожидаемых улучшений, а даже ухудшить производительность модели для выбранной области.

Поэтому прежде чем учиться «как» тонко настраивать языковые модели, нужно понять «почему» стоит пойти этим путем и «когда» начинать процесс тонкой настройки. Начните с вопросов:

- **Сценарий использования:** Какова ваша _сценарий использования_ для тонкой настройки? Какой аспект текущей предобученной модели вы хотите улучшить?
- **Альтернативы:** Пробовали ли вы _другие техники_ для достижения желаемых результатов? Используйте их для создания базового уровня для сравнения.
  - Инженерия запросов: попробуйте техники вроде few-shot с примерами релевантных ответов. Оцените качество ответов.
  - Генерация с поддержкой поиска: попробуйте дополнить запросы результатами поиска по вашим данным. Оцените качество ответов.
- **Затраты:** Определили ли вы затраты на тонкую настройку?
  - Возможность настройки — доступна ли предобученная модель для тонкой настройки?
  - Усилия — подготовка обучающих данных, оценка и доработка модели.
  - Вычислительные ресурсы — запуск задач тонкой настройки и деплой тонко настроенной модели.
  - Данные — доступ к достаточному количеству качественных примеров для ощутимого эффекта тонкой настройки.
- **Преимущества:** Подтвердили ли вы выгоды от тонкой настройки?
  - Качество — превзошла ли тонко настроенная модель базовый уровень?
  - Затраты — снижает ли она использование токенов за счёт упрощения запросов?
  - Расширяемость — можно ли использовать базовую модель для новых областей?

Ответив на эти вопросы, вы сможете решить, подходит ли тонкая настройка для вашего сценария. Идеально, если выгоды превышают затраты. После решения начать, пора думать о том, _как_ тонко настроить предобученную модель.

## Как мы можем тонко настроить предобученную модель?

Чтобы тонко настроить предобученную модель, вам нужно иметь:

- предобученную модель для настройки
- набор данных для тонкой настройки
- среду для запуска задачи тонкой настройки
- среду для развертывания тонко настроенной модели

## Тонкая настройка на Microsoft Foundry

[Microsoft Foundry](https://ai.azure.com?WT.mc_id=academic-105485-koreyst) — это платформа, где вы можете выполнять тонкую настройку, развертывание и управление кастомными моделями на Azure (она объединила Azure OpenAI Studio и Azure AI Studio). Перед началом работы полезно понять, какие возможности предоставляет Foundry, и лучшие практики, рекомендованные платформой. В основе Foundry лежит технология **LoRA (адаптация низкого ранга)**, позволяющая эффективно тонко настраивать модели, что ускоряет обучение и удешевляет его по сравнению с полным дообучением всех весов.

### Шаг 1: Выберите технику обучения

Foundry поддерживает три техники тонкой настройки. **Начните с SFT** — она охватывает самый широкий спектр сценариев.

| Техника | Что она делает | Когда использовать |
| --- | --- | --- |
| **Контролируемая тонкая настройка (SFT)** | Обучается на парах вход/выход, чтобы модель училась генерировать нужные ответы. | По умолчанию для большинства задач: специализация по домену, эффективность задачи, стиль и тон, следование инструкциям и адаптация к языку. |
| **Оптимизация предпочтений напрямую (DPO)** | Учится на парах _предпочитаемый/непредпочитаемый_ ответ для выравнивания выходных данных под человеческие предпочтения. | Для улучшения качества ответов, безопасности и выравнивания, когда есть сравнительная обратная связь. |
| **Реинфорсмент тонкая настройка (RFT)** | Использует сигналы вознаграждения от _оценщиков_ для оптимизации сложных поведений с помощью обучения с подкреплением. | Объективные, требующие рассуждений области (математика, химия, физика) с четкими правильными/неправильными ответами. Требует больше опыта в машинном обучении. |

### Шаг 2: Выберите уровень обучения

Foundry позволяет выбирать, где и как будет проходить обучение:

- **Стандартный** — обучение в регионе вашего ресурса с гарантией локализации данных. Используйте, когда данные должны оставаться в конкретном регионе.
- **Глобальный** — дешевле и быстрее за счет использования мощностей за пределами вашего региона (данные и веса копируются в регион обучения). Хороший вариант, если локализация данных не требуется.
- **Разработчик** — самая низкая стоимость, использование простаивающих ресурсов без гарантий задержек и SLA (задачи могут быть прерваны и возобновлены). Идеально для экспериментов.

### Шаг 3: Выберите базовую модель

Модели, поддерживающие тонкую настройку: OpenAI `gpt-4o-mini`, `gpt-4o`, `gpt-4.1`, `gpt-4.1-mini` и `gpt-4.1-nano` (SFT; семейство 4o/4.1 также поддерживает DPO), модели для рассуждений `o4-mini` и `gpt-5` (RFT), а также open-source модели, такие как `Ministral-3B`, `Qwen-32B`, `Llama-3.3-70B-Instruct` и `gpt-oss-20b` (SFT на ресурсах Foundry). Всегда проверяйте актуальный [список моделей для тонкой настройки](https://learn.microsoft.com/azure/ai-foundry/foundry-models/concepts/models-sold-directly-by-azure?WT.mc_id=academic-105485-koreyst#fine-tuning-models) для поддерживаемых методов, регионов и доступности.

> Foundry предлагает два варианта: **без серверов** (плата по потреблению, нет необходимости управлять квотами GPU, поддержка OpenAI и выбранных моделей) и **управляемые вычисления** (использование своих VMs через Azure Machine Learning для наибольшего выбора моделей). Большинству пользователей стоит начать с безсерверного варианта.

### Лучшие практики Foundry

- **Сначала оцените базу.** Оцените базовую модель, используя инженерию запросов и RAG _до_ тонкой настройки, чтобы доказать эффект.
- **Начинайте с малого, затем масштабируйте.** Начните с 50-100 качественных примеров для проверки подхода, затем увеличьте до 500+ для продакшена. Качество важнее количества — убирайте низкокачественные примеры.
- **Правильно форматируйте данные.** Файлы для обучения и валидации должны быть в формате JSONL, UTF-8 **с BOM**, размером менее 512 MB, используя формат сообщений chat-completions. Всегда включайте файл валидации для отслеживания переобучения.
- **Сохраняйте системный промпт во время инференса.** Используйте ту же системную инструкцию при вызове модели, что и при обучении.
- **Оценивайте контрольные точки — не деплойте последнюю бездумно.** Foundry сохраняет последние три эпохи как готовые к деплою контрольные точки; выбирайте ту, которая лучше всего обобщается, анализируя `train_loss` / `valid_loss` и точность токенов.
- **Сравнивайте стоимость токенов вместе с качеством** при сравнении тонко настроенной модели с базовой.
- **Итеративно улучшайте с помощью непрерывной тонкой настройки.** Можно дообучать уже тонко настроенную модель на новых данных (поддерживается для моделей OpenAI).
- **Учтите расходы на хостинг.** Развернутая кастомная модель тарифицируется почасово, а неактивное развертывание удаляется через 15 дней — удаляйте то, что не нужно.

Пройдите полное руководство в [Настройка модели с помощью тонкой настройки](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning?WT.mc_id=academic-105485-koreyst), а также ознакомьтесь с руководствами по [DPO](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/fine-tuning-direct-preference-optimization?WT.mc_id=academic-105485-koreyst) и [RFT](https://learn.microsoft.com/azure/ai-foundry/openai/how-to/reinforcement-fine-tuning?WT.mc_id=academic-105485-koreyst), когда будете готовы к другим техникам.
