# Тонкая настройка вашего LLM

Использование больших языковых моделей для создания приложений с генеративным ИИ связано с новыми вызовами. Ключевая проблема — обеспечение качества ответов (точности и релевантности) в контенте, генерируемом моделью по запросу пользователя. В предыдущих уроках мы обсуждали такие техники, как проектирование запросов и генерация с дополнением извлечения, которые пытаются решить проблему путем _модификации входного запроса_ к существующей модели.

В сегодняшнем уроке мы обсудим третью технику — **тонкую настройку**, которая пытается решить задачу путем _переподготовки самой модели_ с дополнительными данными. Давайте углубимся в детали.

## Что такое тонкая настройка для языковых моделей?

По определению, большие языковые модели _предварительно обучены_ на больших объемах текста, взятых из разнообразных источников, включая интернет. Как мы узнали в предыдущих уроках, нам нужны такие техники, как _проектирование запросов_ и _генерация с дополнением извлечения_, чтобы улучшить качество ответов модели на вопросы пользователя ("запросы").

Популярная техника проектирования запросов включает в себя предоставление модели более четкого руководства о том, что ожидается в ответе, либо путем предоставления _инструкций_ (явное руководство), либо _предоставлением нескольких примеров_ (неявное руководство). Это называется _обучением с несколькими примерами_, но у него есть два ограничения:

- Ограничения на количество токенов модели могут ограничивать количество примеров, которые вы можете предоставить, и снижать эффективность.
- Стоимость токенов модели может сделать добавление примеров к каждому запросу дорогим и ограничивать гибкость.

Тонкая настройка — это распространенная практика в системах машинного обучения, когда мы берем предварительно обученную модель и переподготавливаем ее с новыми данными, чтобы улучшить ее производительность для конкретной задачи. В контексте языковых моделей мы можем провести тонкую настройку предварительно обученной модели _с набором примеров для конкретной задачи или области применения_, чтобы создать **пользовательскую модель**, которая может быть более точной и релевантной для этой конкретной задачи или области. Побочным преимуществом тонкой настройки является то, что она может также уменьшить количество примеров, необходимых для обучения с несколькими примерами — сокращая использование токенов и связанные с этим расходы.

## Когда и почему мы должны проводить тонкую настройку моделей?

В _этом_ контексте, когда мы говорим о тонкой настройке, мы имеем в виду **управляемую** тонкую настройку, где переподготовка осуществляется путем **добавления новых данных**, которые не были частью исходного набора данных для обучения. Это отличается от подхода неуправляемой тонкой настройки, где модель переподготавливается на исходных данных, но с различными гиперпараметрами.

Ключевое, что нужно помнить, — это то, что тонкая настройка является сложной техникой, требующей определенного уровня экспертизы для достижения желаемых результатов. Если она выполнена неправильно, она может не обеспечить ожидаемых улучшений и даже ухудшить производительность модели для вашей целевой области.

Итак, прежде чем изучать "как" проводить тонкую настройку языковых моделей, вы должны знать "почему" вы должны выбрать этот путь и "когда" начать процесс тонкой настройки. Начните с ответа на эти вопросы:

- **Сценарий использования**: Каков ваш _сценарий использования_ для тонкой настройки? Какой аспект текущей предварительно обученной модели вы хотите улучшить?
- **Альтернативы**: Пробовали ли вы _другие техники_ для достижения желаемых результатов? Используйте их для создания базового уровня для сравнения.
  - Проектирование запросов: Попробуйте техники, такие как запросы с несколькими примерами с примерами релевантных ответов на запросы. Оцените качество ответов.
  - Генерация с дополнением извлечения: Попробуйте дополнить запросы результатами поиска в ваших данных. Оцените качество ответов.
- **Расходы**: Определили ли вы расходы на тонкую настройку?
  - Настраиваемость — доступна ли предварительно обученная модель для тонкой настройки?
  - Усилия — для подготовки данных для обучения, оценки и уточнения модели.
  - Вычислительные ресурсы — для выполнения задач тонкой настройки и развертывания модели с тонкой настройкой.
  - Данные — доступ к достаточному количеству качественных примеров для влияния тонкой настройки.
- **Преимущества**: Подтвердили ли вы преимущества тонкой настройки?
  - Качество — превзошла ли модель с тонкой настройкой базовый уровень?
  - Расходы — сокращает ли она использование токенов, упрощая запросы?
  - Расширяемость — можете ли вы перепрофилировать базовую модель для новых областей?

Ответив на эти вопросы, вы должны быть в состоянии решить, является ли тонкая настройка правильным подходом для вашего сценария использования. Идеально, если подход оправдан только в том случае, если преимущества перевешивают расходы. Как только вы решите продолжить, пора подумать о _том, как_ вы можете провести тонкую настройку предварительно обученной модели.

## Как мы можем провести тонкую настройку предварительно обученной модели?

Чтобы провести тонкую настройку предварительно обученной модели, вам нужно иметь:

- предварительно обученную модель для тонкой настройки
- набор данных для использования при тонкой настройке
- среду обучения для выполнения задачи тонкой настройки
- среду хостинга для развертывания модели с тонкой настройкой
