# Настройка вашей LLM

Использование больших языковых моделей для создания генеративных AI-приложений несет с собой новые вызовы. Ключевой проблемой является обеспечение качества ответов (точности и релевантности) в контенте, сгенерированном моделью для данного пользовательского запроса. В предыдущих уроках мы обсуждали такие техники, как инженерия подсказок и генерация с дополнением за счет поиска, которые пытаются решить проблему путем _изменения входной подсказки_ для существующей модели.

В сегодняшнем уроке мы рассмотрим третью технику — **тонкую настройку**, которая пытается решить задачу путем _дообучения самой модели_ на дополнительных данных. Давайте погрузимся в детали.

## Что такое тонкая настройка для языковых моделей?

По определению, большие языковые модели _предварительно обучены_ на больших объемах текста, взятого из различных источников, включая интернет. Как мы узнали в предыдущих уроках, нам нужны такие техники, как _инженерия подсказок_ и _генерация с дополнением за счет поиска_, чтобы улучшить качество ответов модели на вопросы пользователя ("подсказки").

Популярная техника инженерии подсказок включает предоставление модели более точного руководства о том, что ожидается в ответе — либо через _инструкции_ (явное руководство), либо _путем предоставления нескольких примеров_ (неявное руководство). Это называется _обучением с несколькими примерами_ (few-shot learning), но у нее есть два ограничения:

- Ограничения на количество токенов модели могут ограничивать число примеров, которые вы можете дать, и снижать эффективность.
- Стоимость токенов модели может сделать добавление примеров к каждой подсказке дорогим и ограничить гибкость.

Тонкая настройка — распространенная практика в системах машинного обучения, при которой мы берем предварительно обученную модель и дообучаем ее на новых данных, чтобы улучшить ее производительность по конкретной задаче. В контексте языковых моделей мы можем тонко настроить предварительно обученную модель _на специально подобранном наборе примеров для конкретной задачи или предметной области_, чтобы создать **пользовательскую модель**, которая может быть более точной и релевантной для этой задачи или области. Дополнительным преимуществом тонкой настройки является возможность уменьшить количество примеров, необходимых для обучения с несколькими примерами — что сокращает использование токенов и связанные с этим затраты.

## Когда и почему следует тонко настраивать модели?

В _этом_ контексте, когда мы говорим о тонкой настройке, мы имеем в виду **контролируемую** тонкую настройку, при которой дообучение происходит за счет **добавления новых данных**, которые не входили в исходный обучающий набор. Это отличается от неконтролируемого подхода тонкой настройки, где модель переобучается на исходных данных, но с другими гиперпараметрами.

Главное, что нужно помнить — тонкая настройка является продвинутой техникой, требующей определенного уровня экспертизы для достижения желаемых результатов. Если делать это неправильно, она может не дать ожидаемых улучшений и даже ухудшить производительность модели для вашей целевой области.

Поэтому, прежде чем учиться "как" тонко настраивать языковые модели, нужно понять "почему" стоит выбрать этот путь и "когда" начинать процесс тонкой настройки. Начните с вопросов:

- **Сценарий использования**: Каков ваш _сценарий использования_ для тонкой настройки? Какой аспект текущей предварительно обученной модели вы хотите улучшить?
- **Альтернативы**: Пробовали ли вы _другие техники_ для достижения нужных результатов? Используйте их для создания базового уровня сравнения.
  - Инженерия подсказок: попробуйте техники, такие как обучение с несколькими примерами с релевантными ответами на подсказки. Оцените качество ответов.
  - Генерация с дополнением за счет поиска: попробуйте дополнять подсказки результатами поиска по вашим данным. Оцените качество ответов.
- **Затраты**: Определили ли вы затраты на тонкую настройку?
  - Возможность настройки — доступна ли модель для тонкой настройки?
  - Усилия — подготовка обучающих данных, оценка и доработка модели.
  - Вычислительные ресурсы — для запуска задач тонкой настройки и развертывания настроенной модели.
  - Данные — наличие достаточного количества качественных примеров для значимого эффекта тонкой настройки.
- **Преимущества**: Подтвердили ли вы преимущества тонкой настройки?
  - Качество — превзошла ли тонко настроенная модель базовый уровень?
  - Затраты — снижает ли она использование токенов за счет упрощения подсказок?
  - Расширяемость — можно ли использовать базовую модель для новых доменов?

Ответив на эти вопросы, вы сможете решить, подходит ли тонкая настройка для вашего сценария. Идеально, если подход оправдан, то есть преимущества превышают затраты. Когда вы решите приступить, пора подумать о _том, как_ вы можете тонко настроить предварительно обученную модель.

## Как тонко настроить предварительно обученную модель?

Для тонкой настройки предварительно обученной модели вам понадобится:

- предварительно обученная модель для настройки
- набор данных для тонкой настройки
- среда обучения для запуска задачи тонкой настройки
- среда размещения для развертывания настроенной модели
