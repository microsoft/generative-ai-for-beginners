### Чатбот или чат-приложение?

Прежде чем перейти к созданию чат-приложений, давайте сравним понятия «чатбот» и «чат-приложение с ИИ», которые выполняют различные функции и роли. Основная задача чатбота — автоматизация конкретных разговорных задач, таких как ответы на часто задаваемые вопросы или отслеживание посылок. Обычно он управляется на основе правил или сложных ИИ-алгоритмов. Напротив, чат-приложение с ИИ — это гораздо более обширная среда для различных форм цифрового общения, таких как текстовые, голосовые и видеочаты между пользователями. Его ключевая особенность — интеграция генеративной модели ИИ, которая моделирует нюансированные, подобные человеческим разговоры, генерируя ответы, основываясь на разнообразных входных данных и контекстуальных подсказках. Чат-приложение с генеративным ИИ может участвовать в обсуждениях открытого типа, адаптироваться к меняющемуся контексту и даже создавать креативные или сложные диалоги.

В таблице ниже приведены ключевые различия и сходства, которые помогут понять их уникальные роли в цифровой коммуникации.

| Чатбот                               | Чат-приложение на базе генеративного ИИ |
| ------------------------------------- | -------------------------------------- |
| Ориентирован на задачи и основан на правилах           | Осведомлен о контексте                          |
| Часто интегрирован в более крупные системы  | Может содержать один или несколько чатботов      |
| Ограничен запрограммированным функционалом       | Включает генеративные модели ИИ      |
| Специализированные и структурированные взаимодействия | Способен вести обсуждения открытого типа     |

### Использование встроенных функций с помощью SDK и API

При создании чат-приложения отличным первым шагом будет оценка доступных решений. Использование SDK и API для создания чат-приложений — выгодная стратегия по множеству причин. Интегрируя хорошо документированные SDK и API, вы стратегически обеспечиваете вашему приложению долгосрочный успех, учитывая вопросы масштабируемости и поддержки.

- **Ускоряет процесс разработки и снижает нагрузку**: Использование готовых функций вместо затратной разработки с нуля позволяет сосредоточиться на других важных аспектах приложения, таких как бизнес-логика.
- **Лучше производительность**: При разработке с нуля вы в итоге зададите себе вопрос «Как это масштабируется? Способно ли приложение справиться с резким наплывом пользователей?» Хорошо поддерживаемые SDK и API обычно включают решения этих вопросов.
- **Удобство поддержки**: Обновления и улучшения проще управлять, так как большинство API и SDK требуют лишь обновления библиотеки при выпуске новой версии.
- **Доступ к передовым технологиям**: Использование моделей, дообученных и натренированных на обширных данных, обеспечивает вашему приложению возможности обработки естественного языка.

Доступ к функционалу SDK или API обычно подразумевает получение разрешения на использование предоставляемых сервисов, часто через уникальный ключ или токен аутентификации. Мы используем библиотеку OpenAI для Python, чтобы показать, как это выглядит. Вы также можете попробовать самостоятельно в следующих [тетрадках для OpenAI](./python/oai-assignment.ipynb?WT.mc_id=academic-105485-koreyst) или [тетрадках для Azure OpenAI Services](./python/aoai-assignment.ipynb?WT.mc_id=academic-105485-koreys) для этого урока.

## Пользовательский опыт (UX)

Общие принципы UX применимы к чат-приложениям, но здесь есть дополнительные аспекты, которые становятся особенно важны из-за компонентов машинного обучения.

- **Механизм для устранения неоднозначности**: Модели генеративного ИИ иногда создают неопределенные ответы. Функция, позволяющая пользователям запрашивать уточнения, может помочь при возникновении такой проблемы.
- **Сохранение контекста**: Продвинутые модели генеративного ИИ умеют запоминать контекст в разговоре, что является важным активом для пользовательского опыта. Предоставление пользователям контроля и управления контекстом улучшает опыт, но несет риск сохранения конфиденциальной информации. Нужно продумать, как долго хранится эта информация, например, внедряя политику хранения, чтобы сбалансировать необходимость контекста и конфиденциальность.
- **Персонализация**: Благодаря способности учиться и адаптироваться, ИИ-модели обеспечивают индивидуальный опыт пользователя. Настройка UX через функции, например, пользовательские профили, не только создает ощущение понимания, но и помогает эффективнее находить нужные ответы, делая взаимодействие более продуктивным и приятным.

Независимо от того, есть ли у пользователя нарушения зрения, слуха, моторики или когнитивные нарушения, хорошо спроектированное чат-приложение должно быть доступно для всех. В следующем списке приведены конкретные функции, направленные на повышение доступности для разных типов нарушений.

## Кастомизация и донастройка моделей языка для конкретных доменов

Представьте чат-приложение, которое понимает терминологию вашей компании и предугадывает типичные вопросы пользователей. Есть несколько подходов, которые стоит упомянуть:

- **Использование DSL-моделей**. DSL — это доменно-специфичный язык. Можно использовать так называемую DSL-модель, тренированную на конкретном домене, чтобы понимать его концепции и сценарии.
- **Применение донастройки**. Донастройка — это процесс дополнительного обучения модели на специфических данных.

## Кастомизация: использование DSL

Использование доменно-специфичных моделей языка (DSL-моделей) может повысить вовлеченность пользователей, обеспечивая специализированные, контекстуально релевантные взаимодействия. Это модель, натренированная или дообученная для понимания и генерации текста, связанного с конкретной областью, отраслью или темой. Варианты использования DSL-моделей варьируются: от обучения новой модели с нуля до использования уже существующих через SDK и API. Другой вариант — донастройка, которая подразумевает адаптацию уже предварительно натренированной модели под конкретный домен.

## Кастомизация: применение донастройки

Донастройку часто рассматривают, когда предварительно обученная модель недостаточно подходит для специализированного домена или задачи.

Например, медицинские запросы сложны и требуют большого контекста. Медицинский профессионал ставит диагноз, опираясь на множество факторов, таких как образ жизни, сопутствующие заболевания, и может даже ссылаться на последние медицинские публикации для подтверждения диагноза. В таких тонких случаях универсальное чат-приложение с ИИ не может быть надежным источником.

### Сценарий: медицинское приложение

Рассмотрим чат-приложение, призванное помогать медикам, предоставляя быстрые ссылки на руководства по лечению, взаимодействию лекарств или последние исследования.

Универсальная модель может быть достаточна для ответов на базовые медицинские вопросы или общие рекомендации, но может испытывать трудности с:

- **Очень специфическими или сложными случаями**. Например, невролог может спросить: «Какие современные лучшие практики управления лекарственно-резистентной эпилепсией у детей?»
- **Отсутствием последних достижений**. Универсальная модель может не предложить актуальный ответ с учетом последних достижений в неврологии и фармакологии.

В таких случаях донастройка модели с помощью специализированного медицинского набора данных может значительно повысить её способность более точно и надежно отвечать на сложные медицинские запросы. Для этого нужен большой и релевантный набор данных, отражающий доменно-специфические задачи и вопросы.

| Принципы             | Определение Microsoft                                | Соображения для разработчиков чатов                                   | Почему это важно                                                                     |
| ---------------------- | ----------------------------------------------------- | ---------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |

| Надежность и безопасность | Системы ИИ должны работать надежно и безопасно.        | Реализуйте тестирование и защитные механизмы для минимизации ошибок и рисков.         | Обеспечивает удовлетворенность пользователей и предотвращает возможный вред.                                 |
