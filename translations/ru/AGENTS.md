# AGENTS.md

## Обзор проекта

В этом репозитории содержится всесторонний курс из 21 урока, обучающий основам генеративного ИИ и разработке приложений. Курс предназначен для начинающих и охватывает всё — от базовых понятий до создания готовых к производству приложений.

**Основные технологии:**
- Python 3.9+ с библиотеками: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript с Node.js и библиотеками: `openai` (Azure OpenAI через v1 endpoint + Responses API), `@azure-rest/ai-inference` (Microsoft Foundry Models)
- Azure OpenAI Service, OpenAI API и Microsoft Foundry Models (GitHub Models будет закрыт в конце июля 2026)
- Jupyter Notebooks для интерактивного обучения
- Dev Containers для согласованной среды разработки

**Структура репозитория:**
- 21 каталог с уроками, пронумерованными от 00 до 21, содержащих README, примеры кода и задания
- Множественные реализации: Python, TypeScript и иногда примеры на .NET
- Каталог переводов с более чем 40 языковыми версиями
- Централизованная конфигурация через файл `.env` (используйте `.env.copy` как шаблон)

## Команды настройки

### Первоначальная настройка репозитория

```bash
# Клонируйте репозиторий
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# Скопируйте шаблон окружения
cp .env.copy .env
# Отредактируйте файл .env, указав ваши API-ключи и конечные точки
```

### Настройка среды Python

```bash
# Создать виртуальное окружение
python3 -m venv venv

# Активировать виртуальное окружение
# На macOS/Linux:
source venv/bin/activate
# На Windows:
venv\Scripts\activate

# Установить зависимости
pip install -r requirements.txt
```

### Настройка Node.js/TypeScript

```bash
# Установите зависимости на уровне root (для инструментов документации)
npm install

# Для отдельных примеров TypeScript уроков перейдите к конкретному уроку:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Настройка Dev Container (рекомендуется)

В репозитории включена конфигурация `.devcontainer` для GitHub Codespaces или VS Code Dev Containers:

1. Откройте репозиторий в GitHub Codespaces или в VS Code с расширением Dev Containers
2. Dev Container автоматически:
   - Устанавливает зависимости Python из `requirements.txt`
   - Запускает post-create скрипт (`.devcontainer/post-create.sh`)
   - Настраивает ядро Jupyter

## Рабочий процесс разработки

### Переменные окружения

Все уроки, требующие доступа к API, используют переменные окружения, определённые в `.env`:

- `OPENAI_API_KEY` - Для OpenAI API
- `AZURE_OPENAI_API_KEY` - Для Azure OpenAI в Microsoft Foundry (Azure OpenAI Service теперь часть Microsoft Foundry: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - URL конечной точки Azure OpenAI (endpoint ресурса Foundry)
- `AZURE_OPENAI_DEPLOYMENT` - Имя развертывания модели для чат-комплешенов
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - Имя развертывания модели для эмбеддингов
- `AZURE_OPENAI_API_VERSION` - Версия API (по умолчанию: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Для моделей Hugging Face
- `AZURE_INFERENCE_ENDPOINT` - Endpoint моделей Microsoft Foundry (мульти-провайдерный каталог моделей)
- `AZURE_INFERENCE_CREDENTIAL` - API ключ Microsoft Foundry Models (заменяет устаревающий `GITHUB_TOKEN`)

### Запуск примеров на Python

```bash
# Перейти в каталог урока
cd 06-text-generation-apps/python

# Запустить скрипт на Python
python aoai-app.py
```

### Запуск примеров на TypeScript

```bash
cd 06-text-generation-apps/typescript/recipe-app

# Собрать TypeScript код
```

### Запуск Jupyter Notebooks


## Руководство по стилю кода

### Python

- Используйте `python-dotenv` для управления переменными окружения
- Импортируйте библиотеку `openai` для работы с API
- Используйте `pylint` для статического анализа (некоторые примеры включают `# pylint: disable=all` для простоты)
- Следуйте соглашениям по наименованию из PEP 8
- Храните API креденшалы в файле `.env`, никогда не в коде

### TypeScript

- Используйте пакет `dotenv` для переменных окружения
- Конфигурация TypeScript в `tsconfig.json` для каждого приложения
- Используйте пакет `openai` для Azure OpenAI (указывайте клиенту `/openai/v1/` endpoint и вызывайте `client.responses.create`); используйте `@azure-rest/ai-inference` для Microsoft Foundry Models
- Используйте `nodemon` для разработки с автоперезагрузкой
- Сборка перед запуском: `npm run build`, затем `npm start`

### Общие соглашения

- Держите примеры кода простыми и образовательными
- Включайте комментарии с объяснением ключевых понятий
- Код каждого урока должен быть автономным и запускаться самостоятельно
- Используйте единообразные именования: префикс `aoai-` для Azure OpenAI, `oai-` для OpenAI API, `githubmodels-` для Microsoft Foundry Models (наследственный префикс с эпохи GitHub Models)

## Руководство по документации

### Стиль Markdown

- Все URL должны быть оформлены в формате `[текст](../../url)` без лишних пробелов
- Относительные ссылки должны начинаться с `./` или `../`
- Все ссылки на домены Microsoft должны включать ID отслеживания: `?WT.mc_id=academic-105485-koreyst`
- Избегайте локалей, специфичных для стран в URL (не используйте `/en-us/`)
- Изображения хранятся в папке `./images` с описательными именами
- В именах файлов используйте английские буквы, цифры и дефисы

### Поддержка переводов

- Репозиторий поддерживает более 40 языков через автоматические GitHub Actions
- Переводы хранятся в директории `translations/`
- Не отправляйте частичные переводы
- Машинные переводы не принимаются
- Переведённые изображения хранятся в `translated_images/`

### Проверки перед отправкой

Этот репозиторий использует GitHub Actions для валидации. Перед отправкой PR:

1. **Проверка Markdown ссылок**:
   ```bash

   # - Неисправные относительные пути
   # - Отсутствующие идентификаторы отслеживания на путях

   # - URL с локализацией по стране
   # - Неисправные внешние URL
   ```

2. **Ручное тестирование**:
   - Тестируйте Python примеры: активируйте venv и запускайте скрипты
   - Тестируйте TypeScript примеры: `npm install`, `npm run build`, `npm start`
   - Проверьте правильность настройки переменных окружения
   - Убедитесь, что API ключи работают с примерами кода

3. **Примеры кода**:
   - Убедитесь, что весь код запускается без ошибок
   - Тестируйте с Azure OpenAI и OpenAI API, когда применимо
   - Проверьте работу примеров с Microsoft Foundry Models там, где поддерживается

### Автоматических тестов нет

Это образовательный репозиторий, ориентированный на уроки и примеры. Юнит-тесты или интеграционные тесты отсутствуют. Валидация в основном:
- Ручное тестирование примеров кода
- GitHub Actions для проверки Markdown
- Обзор образовательного контента сообществом

## Устранение неполадок

### Частые проблемы

**Ошибки импорта Python**:
- Убедитесь, что виртуальная среда активирована
- Запустите `pip install -r requirements.txt`
- Проверьте, что версия Python 3.9 или выше

**Ошибки сборки TypeScript**:
- Запустите `npm install` в каталоге конкретного приложения
- Проверьте совместимость версии Node.js
- Очистите `node_modules` и переустановите при необходимости

**Ошибки аутентификации API**:
- Проверьте, что файл `.env` существует и содержит правильные значения
- Убедитесь, что ключи API действительны и не просрочены
- Проверьте правильность URL конечных точек для вашего региона

**Отсутствующие переменные окружения**:
- Скопируйте `.env.copy` в `.env`
- Заполните все необходимые значения для текущего урока
- Перезапустите приложение после обновления `.env`
