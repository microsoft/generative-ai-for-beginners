### Memanfaatkan fungsi terbina dengan SDK dan API

Apabila membina aplikasi sembang, langkah pertama yang bagus adalah menilai apa yang sudah tersedia. Menggunakan SDK dan API untuk membina aplikasi sembang adalah strategi yang menguntungkan atas pelbagai sebab. Dengan mengintegrasikan SDK dan API yang didokumentasi dengan baik, anda meletakkan aplikasi anda secara strategik untuk kejayaan jangka panjang, menangani kebimbangan kebolehaskalaan dan penyelenggaraan.

- **Mempercepat proses pembangunan dan mengurangkan beban kerja**: Bergantung pada fungsi terbina daripada proses membina sendiri yang mahal membolehkan anda menumpukan pada aspek lain aplikasi yang lebih penting, seperti logik perniagaan.
- **Prestasi lebih baik**: Apabila membina fungsi dari kosong, anda akan bertanya "Bagaimana ia skalakan? Adakah aplikasi ini mampu menangani lonjakan pengguna secara tiba-tiba?" SDK dan API yang dijaga dengan baik biasanya mempunyai penyelesaian terbina untuk kebimbangan ini.
- **Penyelenggaraan lebih mudah**: Kemaskini dan penambahbaikan lebih mudah diurus kerana kebanyakan API dan SDK hanya memerlukan kemaskini perpustakaan apabila versi baru dikeluarkan.
- **Akses kepada teknologi terkini**: Memanfaatkan model yang telah ditala halus dan dilatih pada set data besar menyediakan aplikasi anda dengan keupayaan bahasa semula jadi.

Mengakses fungsi SDK atau API biasanya melibatkan mendapatkan kebenaran untuk menggunakan perkhidmatan yang disediakan, yang sering melalui penggunaan kunci unik atau token pengesahan. Kita akan menggunakan Perpustakaan Python OpenAI untuk melihat bagaimana ia berfungsi. Anda juga boleh mencuba sendiri dalam [notebook untuk OpenAI](./python/oai-assignment.ipynb?WT.mc_id=academic-105485-koreyst) atau [notebook untuk Perkhidmatan Azure OpenAI](./python/aoai-assignment.ipynb?WT.mc_id=academic-105485-koreys) untuk pelajaran ini.

```python
import os
from openai import OpenAI

API_KEY = os.getenv("OPENAI_API_KEY","")

client = OpenAI(
    api_key=API_KEY
    )

response = client.responses.create(model="gpt-5-mini", input="Suggest two titles for an instructional lesson on chat applications for generative AI.", store=False)
print(response.output_text)
```

Contoh di atas menggunakan model GPT-5 mini dengan Responses API untuk melengkapkan prompt, tetapi perhatikan bahawa kunci API diset terlebih dahulu. Anda akan mendapat ralat jika tidak menetapkan kunci tersebut.

## Pengalaman Pengguna (UX)

Prinsip UX umum terpakai pada aplikasi sembang, tetapi berikut adalah beberapa pertimbangan tambahan yang menjadi penting kerana komponen pembelajaran mesin yang terlibat.

- **Mekanisme untuk menangani kekaburan**: Model AI generatif kadangkala menghasilkan jawapan kabur. Ciri yang membolehkan pengguna meminta penjelasan boleh membantu jika mereka menghadapi masalah ini.
- **Penahanan konteks**: Model AI generatif canggih mempunyai keupayaan untuk mengingati konteks dalam perbualan, yang boleh menjadi aset penting untuk pengalaman pengguna. Memberi pengguna keupayaan untuk mengawal dan mengurus konteks memperbaiki pengalaman pengguna, tetapi memperkenalkan risiko mengekalkan maklumat sensitif pengguna. Pertimbangan berapa lama maklumat ini disimpan, seperti memperkenalkan dasar penahanan, boleh mengimbangi keperluan konteks dengan privasi.
- **Personalisasi**: Dengan keupayaan untuk belajar dan menyesuaikan, model AI menawarkan pengalaman yang diperibadikan untuk pengguna. Mengubah suai pengalaman pengguna melalui ciri seperti profil pengguna bukan sahaja membuat pengguna merasa difahami, tetapi juga membantu pencarian jawapan khusus mereka, mewujudkan interaksi yang lebih cekap dan memuaskan.

Salah satu contoh personalisasi ialah tetapan "Arahan Tersuai" dalam ChatGPT OpenAI. Ia membolehkan anda memberikan maklumat tentang diri anda yang mungkin penting sebagai konteks untuk arahan anda. Berikut adalah contoh arahan tersuai.

## Penyesuaian dan Penalaan Halus untuk Model Bahasa Khusus Domain

Bayangkan aplikasi sembang yang memahami jargon syarikat anda dan menjangka pertanyaan khusus yang sering diajukan oleh pengguna. Terdapat beberapa pendekatan yang patut disebut:

- **Memanfaatkan model DSL**. DSL bermaksud bahasa khusus domain. Anda boleh memanfaatkan model DSL yang dilatih pada domain tertentu untuk memahami konsep dan senarionya.
- **Gunakan penalaan halus**. Penalaan halus adalah proses melatih model anda lebih lanjut dengan data khusus.

## Penyesuaian: Menggunakan DSL

Memanfaatkan model bahasa khusus domain (Model DSL) dapat meningkatkan keterlibatan pengguna dengan menyediakan interaksi khusus yang relevan konteks. Ia adalah model yang dilatih atau ditala untuk memahami dan menghasilkan teks berkaitan bidang, industri, atau subjek tertentu. Pilihan menggunakan model DSL boleh berbeza dari melatih dari awal, hingga menggunakan yang sudah ada melalui SDK dan API. Pilihan lain ialah penalaan halus, yang melibatkan mengambil model sedia ada terlatih dan menyesuaikannya untuk domain tertentu.

## Penyesuaian: Gunakan penalaan halus

Penalaan halus sering dipertimbangkan apabila model terlatih sebelumnya kurang dalam domain khusus atau tugasan khusus.

Contohnya, pertanyaan perubatan adalah rumit dan memerlukan banyak konteks. Apabila profesional perubatan membuat diagnosis, ia berdasarkan pelbagai faktor seperti gaya hidup atau kondisi sedia ada, dan mungkin bergantung pada jurnal perubatan terkini untuk mengesahkan diagnosis mereka. Dalam senario bernuansa seperti ini, aplikasi sembang AI umum tidak boleh menjadi sumber yang boleh dipercayai.

### Senario: aplikasi perubatan

Pertimbangkan aplikasi sembang yang direka untuk membantu pengamal perubatan dengan menyediakan rujukan cepat kepada garis panduan rawatan, interaksi ubat, atau penemuan kajian terkini.

Model tujuan umum mungkin memadai untuk menjawab soalan perubatan asas atau memberikan nasihat umum, tetapi mungkin menghadapi kesukaran dengan perkara berikut:

- **Kes sangat khusus atau kompleks**. Contohnya, seorang pakar neurologi mungkin bertanya, "Apakah amalan terbaik terkini untuk mengurus epilepsi tahan ubat dalam pesakit pediatrik?"
- **Kekurangan kemajuan terkini**. Model tujuan umum mungkin sukar memberikan jawapan terkini yang menggabungkan kemajuan terbaru dalam neurologi dan farmakologi.

Dalam keadaan seperti ini, penalaan halus model dengan dataset perubatan khusus boleh meningkatkan kebolehannya menangani pertanyaan perubatan rumit ini dengan lebih tepat dan boleh dipercayai. Ini memerlukan akses kepada dataset besar dan relevan yang mewakili cabaran dan soalan spesifik domain yang perlu ditangani.
