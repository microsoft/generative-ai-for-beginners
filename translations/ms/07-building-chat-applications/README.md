### Memanfaatkan fungsi terbina dengan SDK dan API

Apabila membina aplikasi sembang, satu langkah awal yang baik adalah menilai apa yang sudah tersedia. Menggunakan SDK dan API untuk membina aplikasi sembang adalah strategi yang menguntungkan atas pelbagai sebab. Dengan mengintegrasikan SDK dan API yang didokumentasi dengan baik, anda meletakkan aplikasi anda secara strategik untuk kejayaan jangka panjang, menangani kebimbangan penskalaan dan penyelenggaraan.

- **Mempercepatkan proses pembangunan dan mengurangkan beban**: Bergantung pada fungsi terbina daripada proses mahal membina sendiri membolehkan anda menumpukan perhatian pada aspek lain aplikasi yang anda anggap lebih penting, seperti logik perniagaan.
- **Prestasi lebih baik**: Apabila membina fungsi dari awal, anda akan bertanya pada diri sendiri "Bagaimana ia berskala? Adakah aplikasi ini mampu menangani lonjakan pengguna secara tiba-tiba?" SDK dan API yang dijaga dengan baik selalunya mempunyai penyelesaian terbina untuk kebimbangan ini.
- **Penyelenggaraan lebih mudah**: Kemas kini dan penambahbaikan lebih mudah diurus kerana kebanyakan API dan SDK hanya memerlukan kemas kini perpustakaan apabila versi baru dikeluarkan.
- **Akses kepada teknologi terkini**: Memanfaatkan model yang telah ditala halus dan dilatih dengan dataset yang luas memberikan aplikasi anda keupayaan bahasa semula jadi.

Mengakses fungsi SDK atau API biasanya melibatkan mendapatkan kebenaran untuk menggunakan perkhidmatan yang disediakan, yang selalunya melalui penggunaan kunci unik atau token pengesahan. Kita akan menggunakan Perpustakaan Python OpenAI untuk meneroka bagaimana ini dilakukan. Anda juga boleh mencubanya sendiri dalam [notebook untuk OpenAI](./python/oai-assignment.ipynb?WT.mc_id=academic-105485-koreyst) atau [notebook untuk Azure OpenAI Services](./python/aoai-assignment.ipynb?WT.mc_id=academic-105485-koreys) untuk pelajaran ini.

```python
import os
from openai import OpenAI

API_KEY = os.getenv("OPENAI_API_KEY","")

client = OpenAI(
    api_key=API_KEY
    )

response = client.responses.create(model="gpt-4o-mini", input="Suggest two titles for an instructional lesson on chat applications for generative AI.", store=False)
print(response.output_text)
```

Contoh di atas menggunakan model GPT-4o mini dengan API Respons untuk melengkapkan arahan, tetapi perhatikan bahawa kunci API ditetapkan sebelum itu. Anda akan menerima ralat jika tidak menetapkan kunci tersebut.

## Pengalaman Pengguna (UX)

Prinsip UX umum terpakai pada aplikasi sembang, tetapi berikut adalah beberapa pertimbangan tambahan yang menjadi penting disebabkan komponen pembelajaran mesin yang terlibat.

- **Mekanisme untuk menangani kekaburan**: Model AI generatif kadang-kadang menghasilkan jawapan yang kabur. Ciri yang membenarkan pengguna meminta penjelasan boleh membantu jika mereka menemui masalah ini.
- **Penyimpanan konteks**: Model AI generatif lanjutan mempunyai keupayaan untuk mengingati konteks dalam perbualan, yang boleh menjadi aset penting dalam pengalaman pengguna. Memberi kuasa kepada pengguna untuk mengawal dan mengurus konteks menambah baik pengalaman pengguna, tetapi memperkenalkan risiko penyimpanan maklumat pengguna yang sensitif. Pertimbangan berapa lama maklumat ini disimpan, seperti memperkenalkan polisi penyimpanan, boleh mengimbangi keperluan konteks dengan privasi.
- **Personalisasi**: Dengan keupayaan belajar dan menyesuaikan diri, model AI menawarkan pengalaman yang diperibadikan kepada pengguna. Menyesuaikan pengalaman pengguna melalui ciri seperti profil pengguna bukan sahaja membuat pengguna rasa difahami, tetapi juga membantu mereka mencari jawapan tertentu dengan lebih cekap dan memuaskan.

Salah satu contoh personalisasi adalah tetapan "Arahan tersuai" di ChatGPT OpenAI. Ia membenarkan anda menyediakan maklumat tentang diri anda yang mungkin menjadi konteks penting untuk arahan anda. Berikut adalah contoh arahan tersuai.

## Penyesuaian dan Penalaan Halus untuk Model Bahasa Khusus Domain

Bayangkan aplikasi sembang yang memahami jargon syarikat anda dan menjangka soalan khusus yang sering diajukan oleh pengguna. Terdapat beberapa pendekatan yang patut disebut:

- **Memanfaatkan model DSL**. DSL bermaksud bahasa khusus domain. Anda boleh menggunakan model DSL yang dilatih pada domain tertentu untuk memahami konsep dan senarionya.
- **Menerapkan penalaan halus**. Penalaan halus adalah proses melatih model anda lebih lanjut dengan data khusus.

## Penyesuaian: Menggunakan DSL

Memanfaatkan model bahasa khusus domain (Model DSL) boleh meningkatkan penglibatan pengguna dengan menyediakan interaksi khusus dan relevan secara konteks. Ia adalah model yang dilatih atau ditala halus untuk memahami dan menjana teks berkaitan bidang, industri, atau subjek tertentu. Pilihan menggunakan model DSL boleh berbeza dari melatih satu dari awal, hingga menggunakan yang telah ada melalui SDK dan API. Pilihan lain adalah penalaan halus, yang melibatkan mengambil model terlatih sedia ada dan menyesuaikannya untuk domain tertentu.

## Penyesuaian: Menerapkan penalaan halus

Penalaan halus biasanya dipertimbangkan apabila model pra-latih tidak mencukupi dalam domain khusus atau tugasan tertentu.

Sebagai contoh, soalan perubatan adalah kompleks dan memerlukan banyak konteks. Apabila seorang profesional perubatan mendiagnosis pesakit, ia berdasarkan pelbagai faktor seperti gaya hidup atau keadaan sedia ada, dan mungkin bergantung pada jurnal perubatan terkini untuk mengesahkan diagnosis mereka. Dalam scenario bernuansa sedemikian, aplikasi sembang AI umum tidak boleh menjadi sumber yang boleh dipercayai.

### Senario: aplikasi perubatan

Pertimbangkan aplikasi sembang yang direka untuk membantu pengamal perubatan dengan menyediakan rujukan pantas kepada garis panduan rawatan, interaksi ubat, atau penemuan kajian terkini.

Model umum mungkin memadai untuk menjawab soalan perubatan asas atau memberikan nasihat umum, tetapi ia mungkin menghadapi kesukaran dengan:

- **Kes yang sangat khusus atau kompleks**. Contohnya, seorang ahli neurologi mungkin bertanya aplikasi itu, "Apakah amalan terbaik terkini untuk menguruskan epilepsi tahan ubat pada pesakit pediatrik?"
- **Kekurangan kemajuan terkini**. Model am boleh menghadapi kesukaran memberikan jawapan terkini yang menggabungkan kemajuan terbaru dalam neurologi dan farmakologi.

Dalam kes seperti ini, menala halus model dengan dataset perubatan khusus boleh meningkatkan dengan ketara kemampuannya untuk menangani pertanyaan perubatan rumit ini dengan lebih tepat dan boleh dipercayai. Ini memerlukan akses kepada dataset yang besar dan berkaitan yang mewakili cabaran dan soalan domain khusus yang perlu ditangani.
