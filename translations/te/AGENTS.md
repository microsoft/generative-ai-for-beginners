ఈ రిపోజిటరీ జనరేటివ్ AI ప్రాథమికాలు మరియు అప్లికేషన్ అభివృద్ధిని నేర్పించేందుకు 21 పాఠాల సమగ్ర పాఠ్యক্রমాన్ని కలిగి ఉంది. కోర్సు ప్రారంభికుల కోసం రూపొందించబడింది మరియు ప్రాథమిక సూత్రాల నుంచి ఉత్పత్తి-తయారైన అప్లికేషన్ల నిర్మాణం వరకు అన్ని అంశాలను కవర్ చేస్తుంది.

**ప్రధాన సాంకేతికతలు:**

**రిపోజిటరీ నిర్మాణం:**

### Dev Container సెటప్ (సిఫార్సు చేయబడింది)

ఈ రిపోజిటరీ GitHub Codespaces లేదా VS Code Dev Containers కోసం `.devcontainer` కాన్ఫిగరేషన్‌ను కలిగి ఉంది:

## అభివృద్ధి పని ప్రవాహం

### వాతావరణ変数లు

API యాక్సెస్ అవసరమయ్యే అన్ని పాఠాలు `.env`లో నిర్వచించిన వాతావరణ変数లను ఉపయోగిస్తాయి:

- `OPENAI_API_KEY` - OpenAI API కోసం
- `AZURE_OPENAI_API_KEY` - Microsoft Foundryలో Azure OpenAI కోసం (Azure OpenAI సేవ ఇప్పుడు Microsoft Foundry యొక్క భాగం: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI ఎండ్‌పాయింట్ URL (Foundry రిసోర్స్ ఎండ్‌పాయింట్)
- `AZURE_OPENAI_DEPLOYMENT` - చాట్ కంప్లీషన్ మోడల్ డిప్లాయ్‌మెంట్ పేరు (కోర్సు డీఫాల్ట్: `gpt-5-mini`)
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - ఎంబెడింగ్స్ మోడల్ డిప్లాయ్‌మెంట్ పేరు (కోర్సు డీఫాల్ట్: `text-embedding-3-small`)
- `AZURE_OPENAI_API_VERSION` - API సంచిక (డీఫాల్ట్: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - Hugging Face మోడల్స్ కోసం
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry మోడల్స్ ఎండ్‌పాయింట్ (బహు-ప్రొవైడర్ మోడల్ క్యాటలాగ్)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry మోడల్స్ API కీ (`GITHUB_TOKEN` రిటైర్ అవుతోన్నది కోసం ప్రతిస్థానం)
- `AZURE_INFERENCE_CHAT_MODEL` - ఒక రీజనింగ్-లేని మోడల్ (ఉదా: `Llama-3.3-70B-Instruct`) `temperature` ఉదాహరణల్లో ఉపయోగించబడుతుంది, ఎందుకంటే రీజనింగ్ మోడల్స్ శాంప్లింగ్ నియంత్రణలను మద్ధతు ఇవ్వవు

### మోడల్ సంప్రదాయాలు (ముఖ్యమైనవి)

- **డీఫాల్ట్ చాట్ మోడల్ `gpt-5-mini`** - ప్రస్తుతంలో ఉన్న, డిప్రికేటెడ్ కాని **రీజనింగ్** మోడల్. 2026 నాటికి పాత `gpt-4o-mini` మరియు `gpt-4.1-mini` మోడల్స్ *డిప్రికేట్ అవుతున్నాయి*, అందుచేత పాఠ్యक्रमం GPT-5 కుటుంబాన్ని ప్రామాణికంగా తీసుకుంటుంది.
- **రీజనింగ్ మోడల్స్ `temperature` మరియు `top_p` ని తిరస్కరిస్తాయి**, మరియు `max_output_tokens` (Responses API) / `max_completion_tokens` (చాట్ కంప్లీషన్లలో) ను `max_tokens` స్థానంలో ఉపయోగిస్తాయి. `gpt-5-mini` ని పిలిచే నమూనాల్లో `temperature`/`top_p`/`max_tokens` జోడించవద్దు.
- **`temperature` చూపించడానికి**, Microsoft Foundry మోడల్స్ ఎండ్‌పాయింట్ (`AZURE_INFERENCE_CHAT_MODEL`) ద్వారా **Llama** మోడల్ (`Llama-3.3-70B-Instruct`) ఉపయోగపడుతుంది. రీజనింగ్ మోడల్స్‌ను శాంప్లింగ్ నియంత్రణల బదులు ప్రాంప్ట్ ఇంజినీరింగ్ + రీజనింగ్ నియంత్రణలతో సవరించండి.
- **ఫైన్-ట్యూనింగ్ (పాఠం 18)** `gpt-4.1-mini` ని ఉపయోగిస్తుంది: GPT-5 కేవలం రీన్‌ఫోర్స్‌మెంట్ ఫైన్-ట్యూనింగ్ (RFT) ని మద్దతు ఇస్తుంది, ఇక్కడ చూపించిన సూపర్వైజ్డ్ ఫైన్-ట్యూనింగ్ (SFT) కాదు.
- పాఠాలు 20 (Mistral) మరియు 21 (Meta) `temperature`/`max_tokens` మెయింటైన్ చేస్తాయి ఎందుకంటే అవి Mistral/Llama మోడల్స్‌ను లక్ష్యంగా పెట్టుకున్నాయి, అవి మద్దతు ఇస్తాయి.

## కోడ్ శైలి మార్గదర్శకాలు

### Python

- వాతావరణ変数 నిర్వహణకు `python-dotenv` ఉపయోగించండి
- API చర్యల కోసం `openai` లైబ్రరీని దిగుమతి చేసుకోండి
- లింటింగ్ కోసం `pylint` ఉపయోగించండి (కొన్ని ఉదాహరణలలో సులభతరం కోసం `# pylint: disable=all` వుంటుంది)
- PEP 8 నామకరణ నియమాలకు అనుగుణంగా ఉండండి
- API సాంకేతిక వివరాలు `.env` ఫైలులో భద్రపరచండి, కోడ్‌లో ఎప్పుడూ భద్రపరచవద్దు

### TypeScript

- వాతావరణ変数లకు `dotenv` ప్యాకేజ్ ఉపయోగించండి
- ప్రతి యాప్‌కు `tsconfig.json` లో TypeScript కాన్ఫిగరేషన్
- Azure OpenAI కోసం `openai` ప్యాకేజ్ (క్లయింట్‌ను `/openai/v1/` ఎండ్‌పాయింట్ వద్ద పాయింట్ చేసి `client.responses.create` ను పిలవండి); Microsoft Foundry మోడల్స్ కోసం `@azure-rest/ai-inference` ఉపయోగించండి
- స్వయంచాలక రీలొడ్‌తో అభివృద్ధి కోసం `nodemon` ఉపయోగించండి
- నడపడానికి ముందు నిర్మించండి: `npm run build` ఆ తర్వాత `npm start`

## పరీక్ష మరియు ధృవీకరణ

### సమర్పణ ముందు తనిఖీలు

ఈ రిపోజిటరీ ధృవీకరణ కోసం GitHub Actions ఉపయోగిస్తుంది. PR లను సమర్పించకముందు:

1. **మార్క్డౌన్ లింకులు తనిఖీ చేయండి**:
   ```bash
   # validate-markdown.yml వర్క్‌ఫ్లో పరిశీలిస్తుంది:
   # - లోపించిన పరస్పర మార్గాలు
   # - మార్గాలపై మరచిపోయిన ట్రాకింగ్ IDలు
   # - URLలపై ట్రాకింగ్ IDలు లేని స్తితి
   # - దేశ స్థానికీకరణతో కూడిన URLలు
   # - లోపించిన బాహ్య URLలు
   ```

2. **మానువల్ పరీక్ష**:
   - Python ఉదాహరణలు: వేన్ ను యాక్టివేట్ చేసి స్క్రిప్టులను నడపండి
   - TypeScript ఉదాహరణలు: `npm install`, `npm run build`, `npm start`
   - వాతావరణ変数లు సరిగ్గా కాన్ఫిగర్ అయ్యాయో చూసుకోండి
   - API కీలు కోడ్ ఉదాహరణలతో పనిచేస్తున్నాయో చెక్ చేయండి

3. **కోడ్ ఉదాహరణలు**:
   - అన్ని కోడ్ తప్పుల లేకుండా నడవాలి
   - అవసరమైతే Azure OpenAI మరియు OpenAI APIతో పరీక్షించండి
   - మద్దతు ఉన్న చోట Microsoft Foundry Models తో ఉదాహరణలు పనిచేసేలా చూసుకోండి

- విద్యా కంటెంట్ కోసం కమ్యూనిటీ సమీక్ష

## సాధారణ పని ప్రవాహాలు

### కొత్త కోడ్ ఉదాహరణ చేర్చడం

1. సంబంధిత పాఠపు డైరెక్టరీకి వెళ్లండి
2. `python/` లేదా `typescript/` ఉపడైరెక్టరీలో ఉదాహరణ సృష్టించండి
3. నామకరణ సంప్రదాయాన్ని పాటించండి: `{provider}-{example-name}.{py|ts|js}`
4. అసలయ API సాంకేతిక వివరాలతో పరీక్షించండి
5. కొత్త వాతావరణ変数లను పాఠ READMEలో డాక్యుమెంట్ చేయండి

### డాక్యుమెంటేషన్ నవీకరణ

1. పాఠ డైరెక్టరీలో README.md ని సవరించండి
2. మార్క్డౌన్ మార్గదర్శకాలను పాటించండి (ట్రాకింగ్ IDలు, సంబంధిత లింకులు)
3. అనువాద నవీకరణలను GitHub Actions నిర్వహిస్తాయి (కైంగా మార్పులు చేయకండి)
4. అన్ని లింకులు చెల్లుబాటు అయ్యాయో పరీక్షించండి

### Dev Containers తో పని చేయడం

1. రిపోజిటరీలో `.devcontainer/devcontainer.json` ఉంది
2. పోస్ట్-క్రియేట్ స్క్రిప్ట్ ఆటోమేటిగ్గా Python ఆధారాలను ఇన్స్టాల్ చేస్తుంది
3. Python మరియు జుపైటర్ విస్తరణలు ముందుగానే కాన్ఫిగర్ అయ్యుండటం
4. వాతావరణం `mcr.microsoft.com/devcontainers/universal:2.11.2` ఆధారంగా ఉంటుంది

## విడుదల మరియు ప్రచురణ

ఇది నేర్చుకునే రిపోజిటరీ - వాడకం ప్రక్రియ లేదు. పాఠ్యక్రమం ఇలా వినియోగిస్తారు:

1. **GitHub రిపోజిటరీ**: కోడ్ మరియు డాక్యుమెంటేషన్‌కు ప్రత్యక్ష యాక్సెస్
2. **GitHub Codespaces**: ముందుగానే సెటప్ చేసిన తక్షణ అభివృద్ధి వాతావరణం
3. **Microsoft Learn**: కంటెంట్ అధికారిక అభ్యసన వేదికకు పంపబడవచ్చు
4. **docsify**: Markdown నుండి రూపొందించిన డాక్యుమెంటేషన్ సైట్ (`docsifytopdf.js` మరియు `package.json` చూడండి)

### డాక్యుమెంటేషన్ సైట్ నిర్మాణం

```bash
# డాక్యుమెంటేషన్ నుండి PDF రూపొందించండి (కావాల్సినట్లు ఉన్నట్లయితే)
npm run convert
```

## సమస్యల పరిష్కారం

### సాధారణ సమస్యలు

**Python దిగుమతి లోపాలు**:
- వర్చువల్ వాతావరణం యాక్టివేట్ అయిందని నిర్ధారించుకోండి
- `pip install -r requirements.txt` నడపండి
- Python సంచిక 3.9+ కంటైనా తనిఖీ చేయండి

**TypeScript నిర్మాణ లోపాలు**:
- నిర్దిష్ట యాప్ డైరెక్టరీలో `npm install` నడపండి
- Node.js సంచిక సరిపోతుందో చూడండి
- అవసరమైతే `node_modules`లను శుభ్రం చేసి మళ్లీ ఇన్స్టాల్ చేయండి

**API ధృవీకరణ లోపాలు**:
- `.env` ఫైల్ ఉన్నదని మరియు సరిగా విలువలు ఉన్నాయో నిర్ధారించండి
- API కీలు చెల్లుబాటు అయ్యాయో మరియు కాలం ముగిసినవని కాదో తనిఖీ చేయండి
- మీ ప్రాంతానికి సరిపోయే ఎండ్‌పాయింట్ URLలు ఉన్నాయో చూసుకోండి

**హేరుకున్న వాతావరణ変数లు**:
- `.env.copy` ను `.env` గా కాపీ చేయండి
- మీరు పనిచేస్తున్న పాఠం కోసం అవసరమైన అన్ని విలువలను పూరించండి
- `.env` నవీకరించిన తరువాత అప్లికేషన్‌ను మళ్ళీ ప్రారంభించండి
