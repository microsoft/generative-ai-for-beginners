# AGENTS.md

## ప్రాజెక్ట్ అవలోకనం

ఈ రిపోజిటరీ జనరేటివ్ AI ప్రాథమికాలు మరియు అప్లికేషన్ అభివృద్ధిని బోధించే 21 పాఠాల సమగ్ర పాఠ్యక్రమాన్ని కలిగి ఉంది. ఈ కోర్సు ప్రారంభికులకు రూపొందించబడింది మరియు ప్రాథమిక సూత్రాల నుండి ఉత్పత్తి-సిద్ధమైన అప్లికేషన్ల నిర్మాణం వరకు అన్ని అంశాలను కవర్ చేస్తుంది.

**ప్రధాన సాంకేతికతలు:**
- Python 3.9+ లైబ్రరీలతో: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript Node.js మరియు లైబ్రరీలతో: `@azure/openai`, `@azure-rest/ai-inference`, `openai`
- Azure OpenAI సర్వీస్, OpenAI API, మరియు GitHub మోడల్స్
- ఇంటరాక్టివ్ అభ్యాసానికి Jupyter నోట్‌బుక్స్
- స్థిరమైన అభివృద్ధి వాతావరణానికి Dev Containers

**రిపోజిటరీ నిర్మాణం:**
- 21 సంఖ్యాబద్ధమైన పాఠాల డైరెక్టరీలు (00-21) వాటిలో READMEలు, కోడ్ ఉదాహరణలు, మరియు అసైన్‌మెంట్లు
- బహుళ అమలు: Python, TypeScript, మరియు కొన్నిసార్లు .NET ఉదాహరణలు
- 40+ భాషా వెర్షన్లతో అనువాదాల డైరెక్టరీ
- `.env` ఫైల్ ద్వారా కేంద్రీకృత కాన్ఫిగరేషన్ (టెంప్లేట్‌గా `.env.copy` ఉపయోగించండి)

## సెటప్ కమాండ్లు

### ప్రారంభ రిపోజిటరీ సెటప్

```bash
# రిపోజిటరీని క్లోన్ చేయండి
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# పర్యావరణ టెంప్లేట్‌ను కాపీ చేయండి
cp .env.copy .env
# మీ API కీలు మరియు ఎండ్పాయింట్లతో .env ను సవరించండి
```

### Python వాతావరణ సెటప్

```bash
# వర్చువల్ ఎన్విరాన్‌మెంట్ సృష్టించండి
python3 -m venv venv

# వర్చువల్ ఎన్విరాన్‌మెంట్‌ను యాక్టివేట్ చేయండి
# macOS/Linux పై:
source venv/bin/activate
# Windows పై:
venv\Scripts\activate

# డిపెండెన్సీలను ఇన్‌స్టాల్ చేయండి
pip install -r requirements.txt
```

### Node.js/TypeScript సెటప్

```bash
# రూట్-స్థాయి డిపెండెన్సీలను ఇన్‌స్టాల్ చేయండి (డాక్యుమెంటేషన్ టూలింగ్ కోసం)
npm install

# వ్యక్తిగత పాఠం TypeScript ఉదాహరణల కోసం, నిర్దిష్ట పాఠానికి నావిగేట్ చేయండి:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Dev Container సెటప్ (సిఫార్సు చేయబడింది)

GitHub Codespaces లేదా VS Code Dev Containers కోసం `.devcontainer` కాన్ఫిగరేషన్ ఈ రిపోజిటరీలో ఉంది:

1. GitHub Codespaces లేదా VS Code Dev Containers ఎక్స్‌టెన్షన్‌తో రిపోజిటరీని తెరవండి
2. Dev Container ఆటోమేటిక్‌గా:
   - `requirements.txt` నుండి Python డిపెండెన్సీలను ఇన్‌స్టాల్ చేస్తుంది
   - పోస్ట్-క్రియేట్ స్క్రిప్ట్ (`.devcontainer/post-create.sh`) నడుపుతుంది
   - Jupyter కర్నెల్ సెటప్ చేస్తుంది

## అభివృద్ధి వర్క్‌ఫ్లో

### వాతావరణ వేరియబుల్స్

API యాక్సెస్ అవసరమైన అన్ని పాఠాలు `.env` లో నిర్వచించిన వాతావరణ వేరియబుల్స్ ఉపయోగిస్తాయి:

- `OPENAI_API_KEY` - OpenAI API కోసం
- `AZURE_OPENAI_API_KEY` - Azure OpenAI సర్వీస్ కోసం
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI ఎండ్‌పాయింట్ URL
- `AZURE_OPENAI_DEPLOYMENT` - చాట్ కంప్లీషన్ మోడల్ డిప్లాయ్‌మెంట్ పేరు
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - ఎంబెడ్డింగ్స్ మోడల్ డిప్లాయ్‌మెంట్ పేరు
- `AZURE_OPENAI_API_VERSION` - API వెర్షన్ (డిఫాల్ట్: `2024-02-01`)
- `HUGGING_FACE_API_KEY` - Hugging Face మోడల్స్ కోసం
- `GITHUB_TOKEN` - GitHub మోడల్స్ కోసం

## కోడ్ శైలి మార్గదర్శకాలు

### Python

- వాతావరణ వేరియబుల్స్ నిర్వహణకు `python-dotenv` ఉపయోగించండి
- API ఇంటరాక్షన్ కోసం `openai` లైబ్రరీని దిగుమతి చేసుకోండి
- లింటింగ్ కోసం `pylint` ఉపయోగించండి (కొన్ని ఉదాహరణల్లో సౌలభ్యానికి `# pylint: disable=all` ఉంటుంది)
- PEP 8 నామకరణ నియమాలు పాటించండి
- API క్రెడెన్షియల్స్ `.env` ఫైల్‌లో నిల్వ చేయండి, కోడ్‌లో కాదు

### TypeScript

- వాతావరణ వేరియబుల్స్ కోసం `dotenv` ప్యాకేజీ ఉపయోగించండి
- ప్రతి యాప్ కోసం `tsconfig.json` లో TypeScript కాన్ఫిగరేషన్
- Azure సర్వీసుల కోసం `@azure/openai` లేదా `@azure-rest/ai-inference` ఉపయోగించండి
- ఆటో-రిలోడ్‌తో అభివృద్ధికి `nodemon` ఉపయోగించండి
- నడపడానికి ముందు నిర్మించండి: `npm run build` తరువాత `npm start`

### సాధారణ నియమాలు

- కోడ్ ఉదాహరణలు సులభంగా మరియు విద్యా ప్రయోజనకరంగా ఉంచండి
- ముఖ్యమైన సూత్రాలను వివరిస్తూ వ్యాఖ్యలు చేర్చండి
- ప్రతి పాఠం కోడ్ స్వతంత్రంగా ఉండాలి మరియు నడిపించదగినది కావాలి
- Azure OpenAI కోసం `aoai-`, OpenAI API కోసం `oai-`, GitHub మోడల్స్ కోసం `githubmodels-` అనే పేర్లను అనుసరించండి

   ```bash

   # - దేశ స్థానికీకరణతో URLలు
   # - పాడైన బాహ్య URLలు
   ```

2. **మాన్యువల్ పరీక్ష**:
   - Python ఉదాహరణలను పరీక్షించండి: వర్చువల్ ఎన్విరాన్‌మెంట్ యాక్టివేట్ చేసి స్క్రిప్ట్‌లు నడపండి

   - వాతావరణ వేరియబుల్స్ సరిగా కాన్ఫిగర్ అయ్యాయో చూడండి
   - API కీలు కోడ్ ఉదాహరణలతో పనిచేస్తున్నాయో తనిఖీ చేయండి

3. **కోడ్ ఉదాహరణలు**:
   - అన్ని కోడ్ తప్పుల్లేకుండా నడవాలి
   - అవసరమైతే Azure OpenAI మరియు OpenAI API రెండింటితో పరీక్షించండి
   - GitHub మోడల్స్ మద్దతు ఉన్న చోట్ల ఉదాహరణలు పనిచేస్తున్నాయో ధృవీకరించండి

- వివరణాత్మక శీర్షికలు ఉపయోగించండి: `[Lesson 06] Fix Python example typo` లేదా `Update README for lesson 08`
- అవసరమైతే ఇష్యూ నంబర్లను సూచించండి: `Fixes #123`

### PR వివరణ

- ఏం మార్చారో మరియు ఎందుకు మార్చారో వివరించండి
- సంబంధిత ఇష్యూలకు లింక్ ఇవ్వండి
- కోడ్ మార్పుల కోసం, పరీక్షించిన ఉదాహరణలను పేర్కొనండి
- అనువాద PRల కోసం, పూర్తి అనువాదం కోసం అన్ని ఫైళ్లను చేర్చండి

### సహకార అవసరాలు

- Microsoft CLA సంతకం చేయండి (మొదటి PRపై ఆటోమేటిక్)
- మార్పులు చేయడానికి ముందు రిపోజిటరీని మీ ఖాతాకు ఫోర్క్ చేయండి
- ఒక్కో లాజికల్ మార్పుకు ఒక్క PR (అసంబంధిత సవరణలు కలపవద్దు)
- PRలను సాధ్యమైనంత చిన్నగా మరియు లక్ష్యంగా ఉంచండి

## సాధారణ వర్క్‌ఫ్లోలు

### కొత్త కోడ్ ఉదాహరణ చేర్చడం

1. సంబంధిత పాఠం డైరెక్టరీకి వెళ్లండి
2. `python/` లేదా `typescript/` సబ్‌డైరెక్టరీలో ఉదాహరణ సృష్టించండి
3. నామకరణ నియమం పాటించండి: `{provider}-{example-name}.{py|ts|js}`
4. వాస్తవ API క్రెడెన్షియల్స్‌తో పరీక్షించండి
5. కొత్త వాతావరణ వేరియబుల్స్ ఉంటే పాఠం READMEలో డాక్యుమెంట్ చేయండి

### డాక్యుమెంటేషన్ నవీకరణ

1. పాఠం డైరెక్టరీలో README.md ఎడిట్ చేయండి
2. Markdown మార్గదర్శకాలు పాటించండి (ట్రాకింగ్ IDలు, సంబంధిత లింకులు)
3. అనువాదాలు GitHub Actions ద్వారా నిర్వహించబడతాయి (మాన్యువల్ ఎడిట్ చేయవద్దు)
4. అన్ని లింకులు సరైనవో పరీక్షించండి

### Dev Containersతో పని చేయడం

1. రిపోజిటరీలో `.devcontainer/devcontainer.json` ఉంది
2. పోస్ట్-క్రియేట్ స్క్రిప్ట్ ఆటోమేటిక్‌గా Python డిపెండెన్సీలను ఇన్‌స్టాల్ చేస్తుంది
3. Python మరియు Jupyter కోసం ఎక్స్‌టెన్షన్లు ముందుగా కాన్ఫిగర్ చేయబడ్డాయి
4. వాతావరణం `mcr.microsoft.com/devcontainers/universal:2.11.2` ఆధారంగా ఉంటుంది

## డిప్లాయ్‌మెంట్ మరియు ప్రచురణ

ఇది ఒక అభ్యాస రిపోజిటరీ - డిప్లాయ్‌మెంట్ ప్రక్రియ లేదు. పాఠ్యక్రమం ఈ విధంగా వినియోగించబడుతుంది:

1. **GitHub రిపోజిటరీ**: కోడ్ మరియు డాక్యుమెంటేషన్‌కు ప్రత్యక్ష యాక్సెస్
2. **GitHub Codespaces**: ముందుగా కాన్ఫిగర్ చేసిన డెవ్ వాతావరణం
3. **Microsoft Learn**: అధికారిక అభ్యాస వేదికకు కంటెంట్ సిండికేషన్
4. **docsify**: Markdown నుండి నిర్మించిన డాక్యుమెంటేషన్ సైట్ (చూడండి `docsifytopdf.js` మరియు `package.json`)

### డాక్యుమెంటేషన్ సైట్ నిర్మాణం

```bash
# డాక్యుమెంటేషన్ నుండి PDF సృష్టించండి (అవసరమైతే)
npm run convert
```

## సమస్య పరిష్కారం

### సాధారణ సమస్యలు

**Python దిగుమతి లోపాలు**:
- వర్చువల్ ఎన్విరాన్‌మెంట్ యాక్టివేట్ చేయబడిందో చూసుకోండి
- `pip install -r requirements.txt` నడపండి
- Python వెర్షన్ 3.9+ ఉందో తనిఖీ చేయండి

**TypeScript నిర్మాణ లోపాలు**:
- ప్రత్యేక యాప్ డైరెక్టరీలో `npm install` నడపండి
- Node.js వెర్షన్ అనుకూలమో చూసుకోండి
- అవసరమైతే `node_modules` క్లియర్ చేసి మళ్లీ ఇన్‌స్టాల్ చేయండి

**API ధృవీకరణ లోపాలు**:
- `.env` ఫైల్ ఉందో మరియు సరైన విలువలు ఉన్నాయో తనిఖీ చేయండి
- API కీలు చెల్లుబాటు అయ్యాయో మరియు కాలం ముగియలేదో చూసుకోండి
- మీ ప్రాంతానికి సరైన ఎండ్‌పాయింట్ URLలు ఉన్నాయో నిర్ధారించండి

**వాతావరణ వేరియబుల్స్ లేమి**:
- `.env.copy` ను `.env` గా కాపీ చేయండి
- మీరు పని చేస్తున్న పాఠానికి అవసరమైన అన్ని విలువలు పూరించండి
- `.env` నవీకరణ తర్వాత మీ అప్లికేషన్ రీస్టార్ట్ చేయండి
