# ఏజెంట్లు.md

## ప్రాజెక్ట్ అవలోకనం

ఈ రిపోజిటరీ జనరేటివ్ AI ఆధారాలు మరియు అప్లికేషన్ అభివృద్ధిని బోధించే 21 పాఠాల సమగ్రమైన పాఠ్యक्रमాన్ని కలిగి ఉంది. ఈ కోర్సు ప్రారంభకుల కోసం రూపొందించబడింది మరియు ప్రాథమిక భావనల నుండి తయారైన ఉత్పత్తి-సిద్ధ అప్లికేషన్ల నిర్మాణం వరకు అన్ని విషయాలను కవర్ చేస్తుంది.

**ప్రధాన సాంకేతికతలు:**
- పython 3.9+ లైబ్రరీలతో: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript Node.js మరియు లైబ్రరీలతో: `openai` (Azure OpenAI v1 ఎండ్‌పాయింట్ + Responses API ద్వారా), `@azure-rest/ai-inference` (Microsoft Foundry Models)
- Azure OpenAI సేవ, OpenAI API మరియు Microsoft Foundry Models (GitHub Models జూలై 2026 చివరికి రిటైర్ అవుతుందని)
- ఇంటరాక్టివ్ బోధనకి జూపిటర్ నోట్‌బుక్స్
- స్థిరమైన అభివృద్ధి వాతావరణానికి డెవ్ కంటెయినర్లు

**రిపోజిటరీ నిర్మాణం:**
- 21 సంఖ్యాగత పాఠాల డైరక్టరీలు (00-21) వాటిలో READMEలు, కోడ్ ఉదాహరణలు, మరియు కార్యక్రమాలు
- బహుళ అమలు: పython, TypeScript, మరియు కొన్నిసార్లు .NET ఉదాహరణలు
- 40+ భాషా వెర్షన్లతో అనువాదాల డైరెక్టరీ
- .env ఫైల్ ద్వారా కేంద్రీకృత కాన్ఫిగరేషన్ (`.env.copy` టెంప్లేట్ గా ఉపయోగించండి)


## అభివృద్ధి వర్క్‌ఫ్లో

### పరిసర వేరియబుల్స్

API యాక్సెస్ అవసరం ఉన్న అన్ని పాఠాలు `.env` ఫైల్లో నిర్వచించబడిన పరిసర వేరియబుల్స్ ఉపయోగిస్తాయి:

- `OPENAI_API_KEY` - OpenAI API కోసం
- `AZURE_OPENAI_API_KEY` - Microsoft Foundryలో Azure OpenAI కోసం (Azure OpenAI సేవ ఇప్పుడు Microsoft Foundry భాగం: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - Azure OpenAI ఎండ్‌పాయింట్ URL (Foundry వనరు ఎండ్‌పాయింట్)
- `AZURE_OPENAI_DEPLOYMENT` - చాట్ కంప్లీషన్ మోడల్ డిప్లాయ్ పేరు
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - ఎంబెడ్డింగ్స్ మోడల్ డిప్లాయ్ పేరు
- `AZURE_OPENAI_API_VERSION` - API వెర్షన్ (డిఫాల్ట్: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - హెగ్గింగ్ ఫేస్ మోడల్స్ కోసం
- `AZURE_INFERENCE_ENDPOINT` - Microsoft Foundry Models ఎండ్‌పాయింట్ (బహుళ-ప్రొవైడర్ మోడల్ క్యాటలాగ్)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry Models API కీ (`GITHUB_TOKEN` రిటైర్ అయినది స్థానంలో)

## కోడ్ స్టైల్ గైడ్‌లు

### పython

- పరిసర వేరియబుల్స్ నిర్వహణ కోసం `python-dotenv` ఉపయోగించండి
- API వ్యవహారాల కోసం `openai` లైబ్రరీని దిగుమతి చేసుకోండి
- లింటింగ్ కోసం `pylint` వాడండి (కొన్ని ఉదాహరణల్లో సౌకర్యార్థం `# pylint: disable=all` వాడబడింది)
- PEP 8 నామకరణ నియమాలను అనుసరించండి
- API క్రెడెన్షియల్స్ `.env` ఫైల్‌లో నిలుపుకోండి, కోడ్‌లో కాదు

### TypeScript

- పరిసర వేరియబుల్స్ కోసం `dotenv` ప్యాకేజీ వాడండి
- ప్రతి యాప్ కోసం `tsconfig.json` లో TypeScript కాన్ఫిగరేషన్
- Azure OpenAI కోసం `openai` ప్యాకేజీ వాడండి (క్లయింట్‌ను `/openai/v1/` ఎండ్‌పాయింట్ వద్ద పెట్టి `client.responses.create` పిలవండి); Microsoft Foundry Models కోసం `@azure-rest/ai-inference` వాడండి
- ఆటో-రిలోడ్‌తో అభివృద్ధికి `nodemon` వాడండి
- నడపడానికి ముందు బిల్డ్ చేయండి: `npm run build` తర్వాత `npm start`

- కోడ్ ఉదాహరణలను సులభమైన మరియు విద్యాసంబంధిగా ఉంచండి
- ముఖ్య భావనలను వివరిస్తూ వ్యాఖ్యానాలు చేర్చండి
- ప్రతి పాఠం కోడ్ స్వయంకుటుంబం మరియు నడిచేలా ఉండాలి
- సిధ్ధాంతపూర్వక నామకరణం వాడండి: Azure OpenAI కోసం `aoai-`, OpenAI API కోసం `oai-`, Microsoft Foundry Models కోసం `githubmodels-` (GitHub Models యుగం నుండి ఉన్న పురాతన ప్రిఫిక్స్)

1. పython మరియు TypeScript రెండింటిలో కోడ్ మార్పులను పరీక్షించండి అవసరమైతే
2. మార్క్డౌన్ ధృవీకరణ నడపండి (PRపై ఆటోమేటిక్‌గా ప్రారంభమవుతుంది)
3. అన్ని Microsoft URLsలో ట్రాకింగ్ IDs ఉండాలని నిర్ధారించండి
4. సంబంధిత లింకులు చెల్లుబాటు అయ్యాయా చూడండి
5. చిత్రాలను సరిగా సూచించిందని నిర్ధారించండి

### PR టైటిల్ ఫార్మాట్

- వివరణాత్మక శీర్షికలు వాడండి: `[Lesson 06] Fix Python example typo` లేదా `Update README for lesson 08`
- అవసరమైతే ఇష్యూ నెంబర్లను సూచించండి: `Fixes #123`

### PR వివరణ

- ఏమి మార్చామో ఎందుకు మార్చామో వివరించండి
- సంబంధిత ఇష్యూలకు లింకులు ఇవ్వండి
- కోడ్ మార్పులకు, ఏ ఉదాహరణలు పరీక్షించబడ్డాయో తెలియజేయండి
- అనువాద PRలకు, పూర్తి అనువాదం కోసం అన్ని ఫైళ్లు సమర్పించండి

### సహకార అవసరాలు

- Microsoft CLA సంతకం చేయండి (మొదటి PRపై ఆటోమాటిక్‌గా)
- మార్పులు చేయడానికి ముందు రిపోజిటరీని ఫోర్క్ చేయండి
- ఒక లాజికల్ మార్పుకు ఒక PR మాత్రమే (అసంబంధమైన ఫిక్స్‌లను కలపకండి)
- PRలను సాధ్యమైనంత వరకు ఫోకస్ చేసినవి మరియు చిన్నవిగా ఉంచండి

## సాధారణ వర్క్‌ఫ్లోలు

### కొత్త కోడ్ ఉదాహరణ జోడించడం

1. సరైన పాఠం డైరెక్టరీకి వెళ్ళండి
2. `python/` లేదా `typescript/` సబ్‌డైరెక్టరీలో ఉదాహరణ సృష్టించండి
3. నామకరణ నియమాన్ని అనుసరించండి: `{provider}-{example-name}.{py|ts|js}`
4. వాస్తవ API క్రెడెన్షియల్స్‌తో పరీక్షించండి
5. కొత్త పరిసర వేరియబుల్స్ ఉంటే పాఠ READMEలో డాక్యుమెంట్ చేయండి

### డాక్యుమెంటేషన్ అప్డేట్లు

1. పాఠం డైరెక్టరీలో README.md సవరించండి
2. మార్క్డౌన్ గైడ్‌లను అనుసరించండి (ట్రాకింగ్ IDs, సంబంధిత లింకులు)
3. అనువాదాలు GitHub Actions ద్వారా నిర్వహించబడతాయి (మానవీయంగా సవరించవద్దు)
4. అన్ని లింకులు చెల్లుబాటు అయ్యాయా పరీక్షించండి

### డెవ్ కంటెయినర్లు తో పని

1. రిపోజిటరీ `.devcontainer/devcontainer.json` కలిగి ఉంటుంది
2. పోస్ట్-క్రియేట్ స్క్రిప్ట్ ఆటోమేటిక్‌గా Python డిపెండెన్సీలు ఇన్‌స్టాల్ చేస్తుంది
3. Python మరియు Jupyter ఎక్స్‌టెన్షన్లు ముందుగా కాన్ఫిగర్ చేయబడ్డాయి
4. వాతావరణం `mcr.microsoft.com/devcontainers/universal:2.11.2` ఆధారంగా ఉంటుంది

## డిప్లాయ్‌మెంట్ మరియు ప్రచురణ

ఇది ఒక బోధనా రిపోజిటరీ - డిప్లాయ్‌మెంట్ ప్రక్రియ లేదు. పాఠ్యक्रमం ఇలా వినియోగించబడుతుంది:

1. **GitHub రిపోజిటరీ**: కోడ్ మరియు డాక్యుమెంటేషన్‌కు ప్రత్యక్ష యాక్సెస్
2. **GitHub Codespaces**: ముందుగా కాన్ఫిగర్ చేయబడిన సెటప్‌తో తక్షణ అభివృద్ధి వాతావరణం
3. **Microsoft Learn**: కంటెంట్ అధికారిక అభ్యాస వేదికకు ప్రసారం కావచ్చు
4. **docsify**: మార్క్డౌన్ నుండి రూపొందించిన డాక్యుమెంటేషన్ సైట్ (`docsifytopdf.js` మరియు `package.json` చూడండి)

### డాక్యుమెంటేషన్ సైట్ నిర్మాణం

```bash
# డాక్యుమెంటేషన్ నుండి PDF సృష్టించండి (అవసరమైతే)
npm run convert
```

## సమస్య పరిష్కారం

### సాధారణ సమస్యలు

**పython దిగుమతి లోపాలు**:
- వర్చువల్ ఎన్విరాన్మెంట్ సక్రియం చేసుకున్నారా అని నిర్ధారించండి
- `pip install -r requirements.txt` నడపండి
- పython వెర్షన్ 3.9+ ఉంది కాదా చూసుకోండి

**TypeScript బిల్డ్ తప్పులు**:
- నిర్దిష్ట యాప్ డైరెక్టరీలో `npm install` నడపండి
- Node.js వెర్షన్ అనుకూలత ఉంది కాదా చూడు
- `node_modules` క్లియర్ చేసి అవసరమైతే మళ్లీ ఇన్‌స్టాల్ చేయండి

**API ఆథెంటికేషన్ తప్పులు**:
- `.env` ఫైల్ ఉన్నదా మరియు విలువలు సరైనవా అని చెక్ చేయండి
- API కీలు చెల్లుబాటు అవుతాయా లేదో చూసుకోండి
- మీ ప్రాంతానికి సరిపడే ఎండ్‌పాయింట్ URLలు సరిగా ఉన్నాయా చూసుకోండి

**పరిసర వేరియబుల్స్ లేకపోవడం**:
- `.env.copy` ను `.env` గా కాపీ చేయండి
- మీరు పనిచేస్తున్న పాఠానికి కావలసిన అన్ని విలువలు పూరించడం
- `.env` నవీకరణ తర్వాత మీ అప్లికేషన్ రీస్టార్ట్ చేయండి
