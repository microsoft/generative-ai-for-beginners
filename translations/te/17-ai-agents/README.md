## పరిచయం

AI ఏజెంట్లు జనరేటివ్ AIలో ఒక ఉత్సాహభరితమైన అభివృద్ధిని ప్రతినిధి చేస్తున్నాయి, వీటి ద్వారా పెద్ద భాషా నమూనాలు (LLMs) సహాయకుల నుండి చర్యలు తీసుకునే ఏజెంట్లుగా అభివృద్ధి చెందవచ్చు. AI ఏజెంట్ ఫ్రేమ్‌వర్క్‌లు అభివృద్ధికారులకు LLMsకు టూల్స్ మరియు స్టేట్ నిర్వహణను అందించే అనువర్తనాలను సృష్టించడానికి ఉపకరిస్తాయి. ఈ ఫ్రేమ్‌వర్క్‌లు వాడుకరి మరియు అభివృద్ధిిఫారులకు LLMలు రూపొందించిన చర్యలను పర్యవేక్షించడానికి వీలు కల్పించి అనుభవాన్ని మెరుగుపరుస్తాయి.

ఈ పాఠం క్రింది అంశాలను చేర్చును:

- AI ఏజెంట్ అంటే ఏమిటి అనే సంగతిని అర్థం చేసుకోవడం - AI ఏజెంట్ ఏమిటి?
- ఐదు విభిన్న AI ఏజెంట్ ఫ్రేమ్‌వర్క్‌లను అన్వేషించడం - వాటి ప్రత్యేకత ఏమిటి?
- వివిధ ఉపయోగ సందర్భాలలో ఈ AI ఏజెంట్లను ఎలా ఉపయోగించవచ్చు - ఎప్పుడు AI ఏజెంట్లను ఉపయోగించాలి?

## నేర్చుకునే లక్ష్యాలు

ఈ పాఠం పూర్తైన తర్వాత, మీరు చేయగలుగుతారు:

- AI ఏజెంట్లు ఏమిటి మరియు వాటిని ఎలా ఉపయోగించవచ్చో వివరించగలరని.
- కొన్ని ప్రసిద్ధ AI ఏజెంట్ ఫ్రేమ్‌వర్క్‌ల మధ్య తేడాలను అర్థం చేసుకొని అవి ఎలా వేరుగా ఉన్నాయో తెలిసుకోవాలని.
- AI ఏజెంట్లు ఎలా పనిచేస్తాయో అర్థం చేసుకొని వాటితో అనువర్తనాలు నిర్మించగలగాలని.

## AI ఏజెంట్లు అంటే ఏమిటి?

AI ఏజెంట్లు జనరేటివ్ AI ప్రపంచంలో ఒక అత్యంత ఉత్సాహభరిత రంగం. ఈ ఉత్సాహంతో పలు పదాల గందరగోళం మరియు వాటి వినియోగం కలగొంటుంది. AI ఏజెంట్లను సూచించే పలు యంత్రాంగాలను సౌకర్యవంతంగా ప్రచారంలోకి తీసుకునేందుకు, ఈ నిర్వచనాన్ని ఉపయోగించుకుందాం:

AI ఏజెంట్లు పెద్ద భాషా నమూనాలు (LLMs) పనులను చేయడానికి **స్టేట్** మరియు **టూల్స్** కు యాక్సెస్ ఇస్తాయి.

![Agent Model](../../../translated_images/te/what-agent.21f2893bdfd01e6a.webp)

ఈ పదాలను నిర్వచిద్దాం:

**పెద్ద భాషా నమూనాలు** - ఈ కోర్సులో GPT-5, GPT-4o, Llama 3.3 మొదలైనవి సూచిస్తున్న నమూనాలు.

**స్టేట్** - ఇది LLM పని చేస్తున్న సందర్భాన్ని సూచిస్తుంది. LLM దాని గత చర్యల మరియు ప్రస్తుత సందర్భాన్ని ఉపయోగించి తదుపరి చర్యలకు మార్గనిర్దేశనం చేస్తుంది. AI ఏజెంట్ ఫ్రేమ్‌వర్క్‌లు అభివృద్ధికారులు ఈ సందర్భాన్ని సులభంగా నిర్వహించడానికి సహాయపడతాయి.

**టూల్స్** - వాడుకరి అభ్యర్థనను మరియు LLM ప్రణాళిక చేసిన పనిని పూర్తిచేయడానికి LLMకు టూల్స్ అవసరం. కొన్ని ఉదాహరణలు డేటాబేస్, API, బాహ్య అనువర్తనం లేదా ఇంకొక LLM కూడా కావచ్చు!

ఈ నిర్వచనలు ఎలా అమలు చేయబడుతాయో తెలుసుకునేముందు సరైన ఆధారం ఇస్తాయని ఆశిస్తున్నాము. కొన్ని AI ఏజెంట్ ఫ్రేమ్‌వర్క్‌లను పరిశీలిద్దాం:

## LangChain ఏజెంట్లు

[LangChain Agents](https://python.langchain.com/docs/how_to/#agents?WT.mc_id=academic-105485-koreyst) పైన ఇచ్చిన నిర్వచనాలను అనుసరించే అమలు.

**స్టేట్** ను నిర్వహించడానికి ఇది `AgentExecutor` అనే బిల్ట్-ఇన్ ఫంక్షన్‌ను ఉపయోగిస్తుంది. ఇది నిర్వచించిన `agent` మరియు అందుబాటులో ఉన్న `tools` ను స్వీకరిస్తుంది.

`Agent Executor` చాట్ చరిత్రను సేకరించి చాట్ సందర్భాన్ని అందిస్తుంది.

![Langchain Agents](../../../translated_images/te/langchain-agents.edcc55b5d5c43716.webp)

LangChain ఒక [టూల్స్ క్యాటలాగ్](https://integrations.langchain.com/tools?WT.mc_id=academic-105485-koreyst) అందిస్తుంది, దీన్ని మీ అనువర్తనంలో దిగుమతి చేసుకుని LLMకు యాక్సెస్ ఇవ్వవచ్చు. ఇవి కమ్యూనిటీ మరియు LangChain బృందం రూపొందించాయి.

మీరు వీటిని నిర్వచించి `Agent Executor` కు పంపవచ్చు.

AI ఏజెంట్ల గురించి చర్చించే ముందు చూడవలసిన మరో ముఖ్య అంశం వీక్షణం — LLM ఏ టూల్ ఉపయోగించుకుంటోంది మరియు ఎందుకు అనేది అభివృద్ధికారులకు తెలుస్తుండాలి. అందుకు LangChain బృందం LangSmithను అభివృద్ధి చేసింది.

## AutoGen

మనం దృష్టి పెట్టవలసిన తదుపరి AI ఏజెంట్ ఫ్రేమ్‌వర్క్ [AutoGen](https://microsoft.github.io/autogen/?WT.mc_id=academic-105485-koreyst). AutoGen ప్రధానంగా సంభాషణలపై దృష్టి సారిస్తుంది. ఏజెంట్లు **సంభాషణాత్మకమయినవి** మరియు **సనుకూలపరచదగినవి**.

**సంభాషణాత్మక -** LLMలు మరొక LLMతో సంభాషణ ప్రారంభించి, కొనసాగిస్తూ పనిని పూర్తి చేసుకోవచ్చు. ఇది `AssistantAgents` సృష్టించి ఒక నిర్దిష్ట సిస్టమ్ సందేశం ఇవ్వడం ద్వారా జరుగుతుంది.

```python

autogen.AssistantAgent( name="Coder", llm_config=llm_config, ) pm = autogen.AssistantAgent( name="Product_manager", system_message="Creative in software product ideas.", llm_config=llm_config, )

```

**సనుకూలపరచదగిన** - ఏజెంట్లు LLM మాత్రమే కాకుండా వాడుకరి లేదా టూల్ గా కూడా నిర్వచించబడవచ్చు. అభివృద్ధికారుగా మీరు `UserProxyAgent` ను నిర్వచించవచ్చు, ఇది వాడుకరితో ఇన్టరాక్ట్ చేసి పనిని పూర్తి చేయడానికి అభిప్రాయం సేకరిస్తుంది. ఆ అభిప్రాయం పని కొనసాగించడానికి లేదా ఆపడానికి ఉపయోగపడుతుంది.

```python
user_proxy = UserProxyAgent(name="user_proxy")
```

### స్టేట్ మరియు టూల్స్

స్టేట్ మార్చడానికి మరియు నిర్వహించడానికి, సహాయక ఏజెంట్ పనిని పూర్తి చేయడానికి Python కోడ్‌ను సృష్టిస్తుంది.

ఇది ఒక పనితీరు ఉదాహరణ:

![AutoGen](../../../translated_images/te/autogen.dee9a25a45fde584.webp)

#### సిస్టమ్ సందేశంతో LLM నిర్వచనము

```python
system_message="For weather related tasks, only use the functions you have been provided with. Reply TERMINATE when the task is done."
```

ఈ సిస్టమ్ సందేశం ఈ LLMకు పనికి సంబంధించిన ఫంక్షన్లను సూచిస్తుంది. AutoGenతో మీరు విభిన్న సిస్టమ్ సందేశాలు కలిగిన AssistantAgents ను సృష్టించవచ్చు.

#### వాడుకరి ద్వారా సంభాషణ ప్రారంభించడం

```python
user_proxy.initiate_chat( chatbot, message="I am planning a trip to NYC next week, can you help me pick out what to wear? ", )

```

వాడుకరి_ప్రాక్సీ (మానవ) నుండి ఈ సందేశం ఏజెంట్ పనిని మొదలు పెట్టడానికి అవసరమైన ఫంక్షన్లను పరిశీలించడానికి ప్రారంభం.

#### ఫంక్షన్ అమలు చేయబడటం

```bash
chatbot (to user_proxy):

***** Suggested tool Call: get_weather ***** Arguments: {"location":"New York City, NY","time_periond:"7","temperature_unit":"Celsius"} ******************************************************** --------------------------------------------------------------------------------

>>>>>>>> EXECUTING FUNCTION get_weather... user_proxy (to chatbot): ***** Response from calling function "get_weather" ***** 112.22727272727272 EUR ****************************************************************

```

మొదటి చాట్ ప్రక్రియ కావడం తర్వాత, ఏజెంట్ సూచించిన టూల్ ను పిలవాలి. ఈ సందర్భంలో, `get_weather` అనే ఫంక్షన్. మీ ఆకృతీకరణ ఆధారంగా, ఇది ఆటోమేటిక్‌గా అమలు కావచ్చు లేదా వాడుకరి ఇన్‌పుట్ ఆధారంగా అమలు అవ్వచ్చు.

మీరు [AutoGen కోడ్ నమూనాలు](https://microsoft.github.io/autogen/docs/Examples/?WT.mc_id=academic-105485-koreyst) ని తెలుసుకుని నిర్మాణం ప్రారంభించవచ్చు.

## Microsoft Agent ఫ్రేమ్‌వర్క్

[Microsoft Agent Framework](https://learn.microsoft.com/agent-framework/?WT.mc_id=academic-105485-koreyst) Microsoft యొక్క ఓపెన్ సోర్స్ SDK, ఇది AI ఏజెంట్లు మరియు బహుళ ఏజెంట్ వ్యవస్థలను **Python** మరియు **.NET**లో నిర్మించడానికి ఉపయోగించవచ్చు. ఇది రెండు పూర్వ ప్రాజెక్టుల శక్తులను సమీకరిస్తుంది — సంస్థ లక్షణాలు **Semantic Kernel** నుంచి, బహుళ ఏజెంట్ సమన్వయంకు **AutoGen** నుంచి, అంతటా ఒక మద్దతు పొందిన ఫ్రేమ్‌వర్క్‌గా. మీరు కొత్త ఏజెంట్ ప్రాజెక్ట్ ప్రారంభిస్తున్నట్లైతే, ఇది AutoGenకు సిఫార్సు చేయబడిన వారసత్వం.

ఈ ఫ్రేమ్‌వర్క్ ఒకే ఒక **చాట్ ఏజెంట్** నుండి సుదీర్ఘమైన **బహుళ ఏజెంట్ వర్క్‌ఫ్లోలు** వరకు విస్తరిస్తుంది, మరియు Microsoft Foundry, Azure OpenAI, OpenAIతో నేరుగా ఇంటిగ్రేట్ అవుతుంది. ఇది OpenTelemetry ద్వారా బిల్ట్-ఇన్ ఆబ్జర్వబిలిటీని అందిస్తుంది, తద్వారా ఏజెంట్లు ఏమి చేస్తున్నాయో గమనించవచ్చు.

### స్టేట్ మరియు టూల్స్

**స్టేట్** - ఫ్రేమ్‌వర్క్ సంభాషణ సందర్భాన్ని **థ్రెడ్స్** ద్వారా నిర్వహిస్తుంది. ఏజెంట్ చాట్ చరిత్రను (వాడుకరి అభ్యర్థనలు, టూల్ కాల్స్, మరియు ఫలితాలు) ట్రాక్ చేస్తుంది, ప్రతి టర్న్ మొన్నటి టర్న్లపై ఆధారపడి ఉంటుంది. థ్రెడ్స్ నిల్వ చేయబడవచ్చు, సంభాషణని ఆపి తరువాత తిరిగి మొదలుపెట్టవచ్చు.

**టూల్స్** - మీరు ఏజెంట్‌కు సాధారణ Python ఫంక్షన్లను ఇచ్చవచ్చు. టైపు సూచనలతో ఉన్న పరామితులు ఆటోమేటిక్‌గా స్కీమాగా మారిపోతాయి, అందువల్ల మోడల్ అవిని ఎలా మరియు ఎప్పుడు పిలవాలో తెలుసుకుంటుంది (ఫంక్షన్ కాలింగ్). ఫ్రేమ్‌వర్క్ Model Context Protocol (MCP) సర్వర్లు మరియు కోడ్ ఇంటర్ప్రిటర్ వంటి హోస్టెడ్ టూల్స్ ను కూడా మద్దతు ఇస్తుంది.

ఇక్కడ ఒక కస్టమ్ టూల్ కలిగిన ఏకైక ఏజెంట్ ఉదాహరణ:

```python
import asyncio
from typing import Annotated

from pydantic import Field
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient


def get_weather(
    location: Annotated[str, Field(description="The location to get the weather for.")],
) -> str:
    """Get the weather for a given location."""
    return f"The weather in {location} is sunny with a high of 22°C."


async def main():
    agent = Agent(
        client=OpenAIChatClient(),
        instructions="You are a helpful assistant that can answer weather questions.",
        tools=[get_weather],
    )

    response = await agent.run("What's the weather in Amsterdam?")
    print(response)


asyncio.run(main())
```

Microsoft Foundryలో Azure OpenAIకి కనెక్ట్ కావాలంటే, మీరు ఎపాయింట్ మరియు సర్టిఫికేట్లను క్లయింట్‌కు ఇవ్వండి:

```python
from azure.identity.aio import AzureCliCredential
from agent_framework.openai import OpenAIChatClient

client = OpenAIChatClient(
    model="my-gpt-5-mini-deployment",
    azure_endpoint="https://my-resource.openai.azure.com",
    credential=AzureCliCredential(),
)
```

### బహుళ ఏజెంట్ వర్క్‌ఫ్లోలు

ఫ్రేమ్‌వర్క్ ప్రత్యేకంగా ఒకదానిపై మరొక ఏజెంట్‌లను క్రమంగా నడిపించడం లేదా సమాంతరంగా అనేక ఏజెంట్‌లకు విస్తరించి ఫలితాలను సమీకరించడం వంటి బహుళ ఏజెంట్ల సమన్వయాన్ని చేస్తుంది:

```python
from agent_framework.orchestrations import SequentialBuilder, ConcurrentBuilder

# ఏజెంట్లను వరసగా నడిపించండి, చర్చా సందర్భాన్ని చైన్లోపలుగా పంపుతూ
sequential = SequentialBuilder(participants=[researcher, writer, editor]).build()

# ఏజెంట్లను సమాంతరంగా విస్తరించి, తరువాత వారి ప్రతిస్పందనలను సమీకరించండి
concurrent = ConcurrentBuilder(participants=[analyst_a, analyst_b, analyst_c]).build()
```

ఫ్రేమ్‌వర్క్‌ను ఇన్‌స్టాల్ చేసి ప్రారంభించడానికి:

```bash
pip install agent-framework-core
# ఐచ్ఛిక ఇంటిగ్రేషన్లు
pip install agent-framework-openai       # OpenAI మరియు Azure OpenAI
pip install agent-framework-foundry      # మైక్రోసాఫ్ట్ ఫౌండ్రీ
```

## Taskweaver

తదుపరి ఏజెంట్ ఫ్రేమ్‌వర్క్ [Taskweaver](https://microsoft.github.io/TaskWeaver/?WT.mc_id=academic-105485-koreyst). ఇది "కోడ్-ఫస్ట్" ఏజెంట్‌గా పిలవబడుతుంది ఎందుకంటే ఇది కేవలం స్ట్రింగ్‌లతో కాకుండా Pythonలో DataFramesతో కూడా పనిచేయవచ్చు. ఇది డేటా విశ్లేషణ మరియు ఉత్పత్తి పనుల కోసం ఎంతో ఉపయోగకరం అవుతుంది. ఉదాహరణకు గ్రాఫ్‌లు మరియు చార్ట్‌లు తయారుచేయడం లేదా రాండమ్ సంఖ్యలు ఉత్పత్తి చేయడం.

### స్టేట్ మరియు టూల్స్

సంభాషణ స్టేట్‌ని నిర్వహించడానికి, TaskWeaver `Planner` అనే సంకల్పనను ఉపయోగిస్తుంది. `Planner` LLM వాడుకరి అభ్యర్థనను తీసుకుని పూర్తి చేయవలసిన పనుల పథకం రూపొందిస్తుంది.

పనులు పూర్తి చేయడానికి `Planner` కు `Plugins` అనే టూల్స్ సేకరణ అందుబాటులో ఉంచబడుతుంది. ఇవి Python క్లాసులు లేదా సాధారణ కోడ్ ఇంటర్ప్రిటర్ కావచ్చు. ఈ ప్లగిన్లు ఎంబెడ్డింగ్స్ గా నిల్వ చేయబడి LLMకు సరైన ప్లగిన్‌ను మెరుగ్గా వెతకడానికి సహాయపడతాయి.

![Taskweaver](../../../translated_images/te/taskweaver.da8559999267715a.webp)

అనామలీ గుర్తింపు నిర్వహించేందుకు ఒక ప్లగిన్ ఉదాహరణ:

```python
class AnomalyDetectionPlugin(Plugin): def __call__(self, df: pd.DataFrame, time_col_name: str, value_col_name: str):
```

కోడ్ అమలు చేసే ముందు ధృవీకరించబడుతుంది. Taskweaverలో పరిస్థితిని నిర్వహించడానికి మరో లక్షణం `experience`. Experience లో YAML ఫైల్ ద్వారా సంభాషణ సందర్భం నిల్వ చేయబడుతుంది. ఇది ముందటి సంభాషణలపై ఆధారపడుతూ LLM కొంత కాలం పాటు మెరుగుపడుతుంది.

## JARVIS

చివరి ఏజెంట్ ఫ్రేమ్‌వర్క్ [JARVIS](https://github.com/microsoft/JARVIS?tab=readme-ov-file&WT.mc_id=academic-105485-koreyst). JARVIS ప్రత్యేకత ఏమిటంటే ఇది LLMను సంభాషణ **స్టేట్** నిర్వహణకు ఉపయోగించి, `tools` అనేవి ఇతర AI మోడళ్లయి ఉంటాయి. ప్రతీ AI మోడల్ ప్రత్యేక పనులను నిర్వహిస్తుంది, ఉదాహరణకు ఆబ్జెక్ట్ గుర్తింపు, ట్రాన్స్క్రిప్షన్ లేదా చిత్రం వివరణ.

![JARVIS](../../../translated_images/te/jarvis.762ddbadbd1a3a33.webp)

సాధారణ ఉద్దేశ్య మోడల్ అయిన LLM వాడుకరి అభ్యర్థనలను అందుకొని, నిర్దిష్ట పని మరియు పనిని పూర్తి చేయడానికి అవసరమైన వాదనలు/డేటాను గుర్తిస్తుంది.

```python
[{"task": "object-detection", "id": 0, "dep": [-1], "args": {"image": "e1.jpg" }}]
```

LLM ఆ అభ్యర్థనను ప్రత్యేక AI మోడల్ అర్థం చేసుకునే JSON వంటి రూపంలో మార్చుతుంది. AI మోడల్ ఆ పనిపై తన అంచనాలు ఇచ్చిన తరువాత LLM ప్రతిస్పందనను అందుతుంది.

పనిని పూర్తి చేయడానికి ఒకటి కంటే ఎక్కువ మోడల్స్ అవసరమైతే, LLM ప్రతిస్పందనలను కూడా అర్థం చేసుకుని, వాటిని కలిపి వాడుకరికి జవాబు ఇస్తుంది.

దిగువ ఉదాహరణలో వాడుకరి చిత్రం లోని వస్తువుల వివరణ మరియు సంఖ్య అభ్యర్థిస్తుంటే ఈ విధంగా ఎలా పని చేస్తుందో చూపిస్తుంది:

## అసైన్‌మెంట్

AI ఏజెంట్లపై మీ నేర్చుకుపోవడం కొనసాగించేందుకు మీరు Microsoft Agent Frameworkతో నిర్మించవచ్చు:

- విద్య అవసరంలో విభిన్న విభాగాలతో ఒక వ్యాపార సమావేశం సిమ్యులేషన్ చేసే అనువర్తనం.
- LLMలకు విభిన్న వ్యక్తిత్వాలు మరియు ప్రాధాన్యాలను అర్థం చేసుకునేలా సిస్టమ్ సందేశాలు సృష్టించి, వాడుకరికి ఒక కొత్త ఉత్పత్తి ఆలోచనను పరిచయం చేయనివ్వండి.
- LLM ఆ అంశాన్ని మెరుగుపరచడానికి ప్రతి విభాగం నుండి అనుసరణ ప్రశ్నలు రూపొందించాలి.
