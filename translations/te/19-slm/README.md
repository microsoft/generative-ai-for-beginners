# జనరేటివ్ AI కోసం ప్రారంభికుల కోసం చిన్న భాషా నమూనాల పరిచయం
జనరేటివ్ AI అనేది కొత్త కంటెంట్‌ను ఉత్పత్తి చేయగల వ్యవస్థలను సృష్టించడంపై కేంద్రీకృతమైన ఆర్టిఫిషియల్ ఇంటెలిజెన్స్ రంగం. ఈ కంటెంట్ టెక్స్ట్, చిత్రాలు, సంగీతం మరియు పూర్తి వర్చువల్ వాతావరణాల వరకు ఉండవచ్చు. జనరేటివ్ AI యొక్క అత్యంత ఉత్తేజకరమైన అనువర్తనాల్లో ఒకటి భాషా నమూనాల రంగంలో ఉంటుంది.

## చిన్న భాషా నమూనాలు అంటే ఏమిటి?

చిన్న భాషా నమూనా (SLM) అనేది పెద్ద భాషా నమూనా (LLM) యొక్క చిన్న పరిమాణం, LLM లు ఉపయోగించే పలు నిర్మాణ సూత్రాలు మరియు సాంకేతికతలను ఉపయోగించడం ద్వారా, అయితే గణనా ఖరచు తగ్గిపోవడం కనిపిస్తుంది.

SLM లు మానవలాగే టెక్స్ట్ సృష్టించడానికి రూపొందించిన భాషా నమూనాల ఉపసమూహం. వాటి పెద్ద ప్రత్యామ్నాయాలాగా GPT-4 లాంటి వాటి కంటే SLM లు చిన్నవి మరియు సమర్థవంతమైనవి, గణన సాధనాలు పరిమితమైన అనువర్తనాలకు ఇవి చక్కగా సరిపోతాయి. వాటి చిన్న పరిమాణం అయినా, అవి విభిన్న పనులను చేయగలవు. సాధారణంగా, SLM లను LLM లను సంస్కరించడం లేదా సారాంశం చేయడం ద్వారా నిర్మిస్తారు, ఆ మొట్టమొదటి నమూనా యొక్క పనితీరు మరియు భాషా సామర్థ్యాలను ఎక్కువగా నిలుపుకోవడానికి ప్రయత్నిస్తారు. నమూనా పరిమాణం తగ్గించటం వల్ల మొత్తం సంక్లిష్టత తగ్గి, SLM లు స్మృతి వినియోగం మరియు గణన అవసరాల పరంగా మరింత సమర్థవంతమవుతాయి. ఈ ఆప్టిమైజేషన్లు ఉన్నప్పటికీ, SLM లు విస్తృత శ్రేణి ప్రకృత భాషా ప్రాసెసింగ్ (NLP) పనులను చేయగలవు:

- టెక్స్ట్ సృష్టి: సవ్యమైన మరియు సందర్భాత్మకంగా సంబంధిత వాక్యాలు లేదా పేరాలలను సృష్టించడం.
- టెక్స్ట్ పూర్తి: ఇచ్చిన ప్రాంప్ట్ ఆధారంగా వాక్యాలను అంచనా వేసి పూర్తి చేయడం.
- అనువాదం: ఒక భాష నుండి మరొక భాషలో టెక్స్ట్ మార్చడం.
- సారాంశం: పొడవైన టెక్స్ట్ ను చిన్న, తేలికైన సారాంశాలుగా సంక్షిప్తం చేయడం.

పెద్ద ప్రత్యామ్నాయాలతో పోలిస్తే కొంత పనితీరు లేదా అర్థంపడగల లోతు తో కొంత ట్రేడ్-ఆఫ్లు ఉంటాయి.

## చిన్న భాషా నమూనాలు ఎలా పని చేస్తాయి?
SLM లు విస్తృతమైన టెక్స్ట్ డేటా మీద శిక్షణ పొందుతాయి. శిక్షణ సమయంలో, అవి భాషా థీమ్ మరియు నిర్మాణాలను నేర్చుకుంటూ, వ్యాకరణపరంగా సరైన మరియు సందర్భానుకూల టెక్స్ట్ సృష్టించగలవు. శిక్షణ ప్రక్రియలో:

- డేటా సేకరణ: వివిధ వనరుల నుండి పెద్ద టెక్స్ట్ డేటా సెట్‌లను సేకరించడం.
- ముందస్తు ప్రాసెసింగ్: డేటాను శుభ్రం చేసి, శిక్షణకు అనుకూలంగా వ్యవస్థీకరించడం.
- శిక్షణ: మిషన్ లెర్నింగ్ అల్గోరిథమ్లను ఉపయోగించి నమూనాను టెక్స్ట్ అర్థం చేసుకోవడం మరియు సృష్టించడం నేర్పించడం.
- ఫైన్-ట్యూనింగ్: నిర్దిష్ట పనులపై పనితీరు మెరుగుపరచడం కోసం నమూనాను సర్దుబాటు చేయడం.

SLM ల అభివృద్ధి కంప్యూటింగ్ వనరుల పరిమితి ఉన్న పరిసరాలలో, ఉదాహరణకు మొబైల్ పరికరాలు లేదా ఎడ్జి కంప్యూటింగ్ వేదికలు, అందుబాటు కోసం అవసరమైన నమూనాలకు అనుగుణంగా ఉంటుంది, అక్కడ పూర్తి స్థాయి LLM లు పేర్చడం సాధ్యం కాకపోవచ్చు. సమర్థతపై దృష్టి పెట్టడం ద్వారా, SLM లు పనితీరు మరియు జత చేరుకోవడాన్ని సమతుల్యం చేస్తాయి, తద్వారా వివిధ రంగాలలో విస్తృత అనువర్తనాన్ని సాధించగలవు.

## నేర్చుకోవాల్సిన లక్ష్యాలు

ఈ పాఠంలో, మనం SLM గురించి జ్ఞానం పొందగలము మరియు Microsoft Phi-3 తో కలిపి టెక్స్ట్ కంటెంట్, విజన్ మరియు MoE లో వేర్వేరు సన్నివేశాలను నేర్చుకోబోతున్నాము.

ఈ పాఠం చివరి వరకు, మీరు ఈ ప్రశ్నలకు సమాధానం ఇవ్వగలిగే స్థాయిలో ఉండాలి:

- SLM అంటే ఏమిటి?
- SLM మరియు LLM మధ్య వ్యత్యాసం ఏమిటి?
- Microsoft Phi-3/3.5 కుటుంబం అంటే ఏమిటి?
- Microsoft Phi-3/3.5 కుటుంబంతో ఇన్ఫెరెన్స్ ఎలా నడపాలి?

రెడీనా? మొదలుపెట్టుదాం.

## పెద్ద భాషా నమూనాలు (LLMs) మరియు చిన్న భాషా నమూనాలు (SLMs) మధ్య తేడాలు

LLM లు మరియు SLM లు రెండూ ప్రమాణాత్మక యంత్ర అభ్యాస స్థూల సూత్రాలపై నిర్మింపబడి ఉంటాయి, నిర్మాణాత్మక రూపకల్పన, శిక్షణ పద్ధతులు, డేటా సృష్టి ప్రక్రియలు మరియు నమూనా మూల్యాంకనం లా సమాన విధానాలు పాటిస్తాయి. అయితే, ఈ రెండు నమూనాల మధ్య కొన్ని ప్రధాన తేడాలు ఉంటాయి.

## చిన్న భాషా నమూనాల అనువర్తనాలు

SLM లకు విస్తృత అనువర్తనాలు ఉన్నాయి, ఈ క్రింది విధంగా:

- చాట్‌బాట్‌లు: వినియోగదారుల సహాయం అందించడం మరియు సంభాషణ రూపంలో ఉపయోగదారులతో సంభాషించడం.
- కంటెంట్ సృష్టి: రచయితలకు సాయపడటం, ఆలోచనలు సృష్టించడం లేదా సంపూర్ణ వ్యాసాలను డ్రాఫ్ట్ చేయడం.
- విద్య: విద్యార్థులకు రాయడంలో సహాయం లేదా కొత్త భాషలు నేర్చుకోవడంలో సాయం.
- యాక్సెస్ చేయగలిగే సೌಲభ్యం: అంగవైకల్యం ఉన్న వ్యక్తులకు టెక్స్ట్-టు-స్పీచ్ వంటి సాధనాలు సృష్టించడం.

**పరిమాణం**
  
LLM లు మరియు SLM ల మధ్య ప్రధాన తేడా నమూనాల పరిమాణంలో ఉంటుంది. ChatGPT (GPT-4) వంటి LLM లు సుమారు 1.76 ట్రిలియన్ ప్యారామీటర్లను కలిగివుంటే, మిస్ట్రాల్ 7B వంటి ఓపెన్-సోర్స్ SLM లు సుమారు 7 బిలియన్ ప్యారామీటర్లతో రూపొందించబడ్డాయి. ఈ తేడా ప్రధానంగా నమూనా నిర్మాణం మరియు శిక్షణ ప్రక్రియల్లోని వ్యత్యాసాల వల్ల. ఉదాహరణకు, ChatGPT ఒక ఎన్‌కోడర్-డికోడర్ ఫ్రేమ్‌వర్క్‌లో స్వీయ-ఆటెన్షన్ మెకానిజంను ఉపయోగిస్తుంటే, మిస్ట్రాల్ 7B స్లైడింగ్ విండో ఆటెన్షన్ ఉపయోగిస్తుంది, ఇది డికోడర్-ఓన్లీ నమూనాలలో మరింత సమర్థమైన శిక్షణను సాధిస్తుంది. ఈ నిర్మాణ భేదం నమూనాల సంక్లిష్టత మరియు పనితీరు పరంగా ఎక్కువ ప్రభావం చూపుతుంది.

**అర్థం చేసుకోవడంలో**

SLM లు సాధారణంగా నిర్దిష్ట విభాగాల్లో పనితీరు కోసం ఆప్టిమైజ్ చేయబడి ఉంటాయి, అందువల్ల అవి ఎక్కువగా ప్రత్యేకమైనవి కానీ విస్తృత సందర్భజ్ఞానాన్ని అందించడంలో పరిమితమైనవి కావచ్చు. విరుద్ధంగా, LLM లు విస్తృతంగా మానవ-సమాన బుద్ధిమత్తను అనుకరించడం లక్ష్యం గా పెట్టుకొంటాయి. విస్తృత, వివిధ డేటాసెట్ లపై శిక్షణ పొందిన LLM లు పలు విభాగాలలో బాగా పని చేస్తాయి, ఎక్కువ పరుపడత మరియు అనుకూలత కలిగివుండి. అందువల్ల LLM లు ప్రకృత భాషా ప్రాసెసింగ్ మరియు ప్రోగ్రామింగ్ వంటి విస్తృత దిశల పనులకు అనుకూలంగా ఉంటాయి.

**గణన**

LLM ల శిక్షణ మరియు అమలు భారీ వనరులు అవసరం చేస్తుంది, సాధారణంగా పెద్ద స్థాయి GPU క్లస్టర్లు అవసరం. ఉదాహరణకు ChatGPT లాంటి నమూనాను మొదటికుండానే శిక్షణ ఇవ్వడానికి వేలలాది GPU లు అనేక కాలవ్యవధులపాటు అవసరం. విరుద్ధంగా, SLM లు తక్కువ ప్యారామీటర్లతో గణన వనరుల పరంగా మరింత అందుబాటులో ఉంటాయి. మిస్ట్రాల్ 7B వంటి నమూనాలను సగటు GPU సామర్థ్యాలున్న స్థానిక యంత్రాలలో శిక్షించవచ్చు మరియు అమలు చేయవచ్చు, కానీ శిక్షణకు ఇంకా అనేక గంటలు ఒకదానికొరకు అవసరం.

**పక్షపాతం**

పక్షపాతం అనేది LLM లలో గుర్తొచ్చిన సమస్య, ఇది ముఖ్యంగా శిక్షణ డేటా స్వభావంతో గలదు. ఈ నమూనాలు ఇంటర్నెట్ నుండి అందుబాటులో ఉన్న కచ్చితమైన, అజ్ఞాత డేటాను ఆధారంగా చేసుకుంటాయి, దీనివల్ల కొన్ని గుంపులు తక్కువ ప్రాతినిధ్యం పొందవచ్చు లేదా తప్పుగా ప్రదర్శించబడవచ్చు, తప్పుదారుల లేబులింగ్, భాషా పక్షపాతాలు (ఉదాహరణకు ప్రాంతీయ భేదాలు, వ్యాకరణ నియమాలు) ఉండవచ్చు. అదనంగా, LLM నిర్మాణ సంక్లిష్టత ఈ పక్షపాతాన్ని అనుకోకుండా పెంచవచ్చు, ఫైన్-ట్యూనింగ్ లేకుండా ఇది గమనించకుండా ఉండవచ్చు. మరొకవైపు, SLM లు మరింత పరిమిత, విభాగ-ప్రత్యేక డేటాసెట్‌లపై శిక్షణ పొందుతున్నాయి, అందువల్ల ఇవి ఈ రకం పక్షపాతాలకు తక్కువ అతిసంవేదనశీలంగా ఉంటాయి, అయినప్పటికీ అవి పూర్తిగా నిమగ్నమయ్యేవి కాదని చెప్పవచ్చు.

**ఇన్ఫెరెన్స్**

SLM ల పరిమాణం తగ్గడం వల్ల వీటి ఇన్ఫెరెన్స్ వేగతపై పెద్ద లాభం ఉంది, ఇవి విస్తృత సమాంతర ఒకేలేని ప్రాసెసింగ్ అవసరం లేకుండా స్థానిక హార్డ్వేర్‌పై సమర్థంగా అవుట్పుట్లను సృష్టించగలవు. LLM ల పరిమాణం మరియు సంక్లిష్టత కారణంగా, ఇన్ఫెరెన్స్ సమయాలను అంగీకరించగల స్థాయికి తీసుకోవడానికి భారీ సమాంతర గణన వనరులు అవసరం. బహుళ సమాంతర ఉపయోగదారుల ఉనికి LLM ల ప్రతిస్పందన సమయాలను కాలంగా తగ్గిస్తుంది, ముఖ్యంగా పెద్ద స్థాయిలో అమలు చేస్తే.

సారాంశంగా, LLM లు మరియు SLM లు రెండూ యంత్ర అభ్యాసంలో ప్రాథమిక మౌలికతను పంచుకుంటూ, నమూనా పరిమాణం, వనరుల అవసరాలు, సందర్భజ్ఞానం, పక్షపాతం ప్రభావం మరియు ఇన్ఫెరెన్స్ వేగతలో గణనీయంగా భిన్నంగా ఉంటాయి. ఈ విభిన్నతలు వాటి అనుకూలతలో ప్రతిబింబిస్తాయి, LLM లు విస్తృతంగా ఉపయోగించదగినవి కానీ వనరుల ప్రమాదకరమైనవి, SLM లు నిర్దిష్ట డొమైన్ సమర్థత కలిగినవి, తక్కువ గణన అవసరాలతో.

***గమనిక: ఈ పాఠంలో, మనం Microsoft Phi-3 / 3.5 ను ఉదాహరణగా తీసుకొని SLM ను పరిచయం చేస్తాము.***

## Phi-3 / Phi-3.5 కుటుంబం పరిచయం

Phi-3 / 3.5 కుటుంబం ప్రధానంగా టెక్స్ట్, విజన్, మరియు ఏజెంట్ (MoE) అనువర్తన సన్నివేశాలకు లక్ష్యంగా రూపొందించబడింది:

### Phi-3 / 3.5 ఇనస్ట్రక్చ్

ప్రధానంగా టెక్స్ట్ సృష్టి, చాట్ పూర్తి, మరియు కంటెంట్ సమాచారాన్ని తీయడం మొదలైన పనులకు సదుపాయపడుతుంది.

**Phi-3-mini**

3.8B ప్యారామీటర్ల భాషా నమూనా Microsoft Foundry, Hugging Face, Ollama లో అందుబాటులో ఉంది. Phi-3 నమూనాలు సమాన లేదా పెద్ద పరిమాణాల భాషా నమూనాలతో కీలక బెంచ్‌మార్క్‌లపై గణనీయంగా మెరుగ్గ ఉంటాయి (క్రింద బెంచ్‌మార్క్ సంఖ్యలను చూడండి, అధిక సంఖ్యలు మంచి సూచికలు). Phi-3-mini సుమారు ద్విగుణ పరిమాణమున్న నమూనాలకంటే మెరుగ్గా ప్రదర్శిస్తుంటుంది, అలాగే Phi-3-small మరియు Phi-3-medium పెద్ద నమూనాలతోపాటు GPT-3.5 కంటే మెరుగ్గా ఉంటాయి.

**Phi-3-small & medium**

కేవలం 7B ప్యారామీటర్లతో కూడిన Phi-3-small అనేక భాష, తర్కసారాంశాలు, కోడింగ్ మరియు గణితం బెంచ్‌మార్క్‌లలో GPT-3.5T ను అంతకంటే అధికంగా జయిస్తుంది.

14B ప్యారామీటర్ల Phi-3-medium ఈ ధోరణిని కొనసాగిస్తూ Gemini 1.0 Pro కంటే మెరుగ్గా ఉంటుంది.

**Phi-3.5-mini**

దీన్ని Phi-3-mini యొక్క అప్‌గ్రేడ్‌గా భావించవచ్చు. ప్యారామీటర్లు మార్చకపోతే కూడా, ఇది బహుభాషల మద్దతును మెరుగుపరుస్తుంది (20+ భాషల మద్దతు: అరబిక్, చైనా, చెక్, డానిష్, డచ్చ్, ఇంగ్లీష్, ఫిన్నిష్, ఫ్రెంచ్, జర్మన్, హీబ్రూ, హంగేరియన్, ఇటాలియన్, జపనీస్, కొరియన్, నార్వేజియన్, పోలిష్, పోర్చుగీస్, రష్యన్, స్పానిష్, స్వీడిష్, థాయ్, టర్కిష్, ఉక్రెయిన్) మరియు దీర్ఘ సూటిమూలక సందర్భానికి బలమైన మద్దతును కలుపుతుంది.

3.8B ప్యారామీటర్ల Phi-3.5-mini అదే పరిమాణం ఉన్న భాషా నమూనాల కన్నా మెరుగ్గా ఉంది మరియు ద.hg. ద్విగుణ పరిమాణం ఉన్న నమూనాలతో సమానంగా ఉంటుంది.

### Phi-3 / 3.5 విజన్

Phi-3/3.5 యొక్క ఇనస్ట్రక్చ్ నమూనాను Phi యొక్క అర్థం చేసుకునే సామర్థ్యంగా భావించవచ్చు, మరియు విజన్ Phi కి ప్రపంచాన్ని అర్థం చేసుకునేందుకు కళ్లను ఇస్తుంది.


**Phi-3-విజన్**

4.2B ప్యారామీటర్ల Phi-3-విజన్ సాధారణ విజువల్ తర్కము, OCR, పట్టికలు మరియు గీయబడిన నమూనాలను అర్థం చేసుకునే పనులలో Claude-3 Haiku, Gemini 1.0 Pro V లాంటి పెద్ద నమూనాలను దాటిచూపుతుంది.


**Phi-3.5-విజన్**

Phi-3.5-విజన్ Phi-3-విజన్ యొక్క అప్‌గ్రేడ్, ఇది బహుళ చిత్ర మద్దతును జోడిస్తుంది. దీన్ని విజన్ లో మెరుగుదలగా పరిగణించవచ్చు, మీరు కేవలం చిత్రాలు కాదు, వీడియోలను కూడా చూడగలుగుతారు.

Phi-3.5-విజన్ Claude-3.5 Sonnet, Gemini 1.5 Flash లాంటి పెద్ద నమూనాలను OCR, పట్టికలు మరియు చార్ట్ అర్థం చేసుకునే పనులలో అధిగమిస్తుంది మరియు సాధారణ విజువల్ జ్ఞాన తర్కము పనుల్లో సమానంగా ఉంటుంది. బహు-ఫ్రేమ్ ఇన్‌పుట్ మద్దతు, అంటే బహు-ఇన్‌పుట్ చిత్రాలపై తర్కం చేయగలదు.


### Phi-3.5-MoE

***మిక్చర్ ఆఫ్ ఎక్స్‌పర్ట్స్ (MoE)*** నమూనాలను తక్కువ గణనతో ముందుగా శిక్షించడం సాధ్యమే చేస్తుంది, అంటే మీరు ఒకే గణనా బడ్జెట్‌తో గణనాత్మక నమూనా లేదా డేటాసెట్ పరిమాణాన్ని గణనీయంగా పెంచవచ్చు. ముఖ్యంగా, ఎమో నమూనా తన సాంద్రీకృత ప్రతిరూపానికి సమాన నాణ్యతను చాలా వేగంగా సాధించగలదు ముందస్తు శిక్షణలో.

Phi-3.5-MoE 16x3.8B ఎక్స్‌పర్ట్ మాడ్యూల్స్ కలిగి ఉంది. 6.6B యాక్టివ్ ప్యారామీటర్లతో Phi-3.5-MoE చాల పెద్ద నమూనాల మాదిరిగానే తర్కం, భాషా అర్థం చేసుకోవడం, మరియు గణితం సామర్ధ్యాన్ని పొందగలదు.

మేము Phi-3/3.5 కుటుంబ నమూనాలను విభిన్న సన్నివేశాలపై ఉపయోగించవచ్చు. LLM లా కాకుండా, మీరు Phi-3/3.5-mini లేదా Phi-3/3.5-విజన్ ను ఎడ్జ్ పరికరాలపై అమలు చేయవచ్చు.


## Phi-3/3.5 కుటుంబ నమూనాలను ఎలా ఉపయోగించాలి

మేము విభిన్న సన్నివేశాల్లో Phi-3/3.5 ఉపయోగించాలని ఆశిస్తున్నాము. తదుపరి, విభిన్న సన్నివేశాలపై ఆధారపడి Phi-3/3.5 ఉపయోగిస్తాము.

**NVIDIA NIM**

క్లౌడ్-ఆధారిత Microsoft Foundry Models క్యాటలాగ్ కాకుండా, మీరు సంబంధిత పిలుపులను పూర్తి చేయడానికి [NVIDIA NIM](https://developer.nvidia.com/nim?WT.mc_id=academic-105485-koreyst) ను కూడా ఉపయోగించవచ్చు. NVIDIA NIM ఆర్టిఫిషియల్ ఇంటెలిజెన్స్ నమూనాలను క్లౌడ్లు, డేటా సెంటర్లు, వర్క్‌స్టేషన్‌లు వంటి వివిధ వాతావరణాల్లో సమర్థవంతంగా అమలు చేయడంలో సహాయపడే కలిపిన వేగవంతమైన ఇన్ఫెరెన్స్ మైక్రోసర్వీసుల సమాహారం.

NVIDIA NIM యొక్క కొన్ని ప్రధాన లక్షణాలు ఈ క్రింది విధంగా ఉన్నాయి:

- **అమలు సౌలభ్యం:** NIM ఒకే ఆజ్ఞతో AI నమూనాలను అమలు చేయడానికి అనుమతిస్తుంది, దీని వల్ల మునుపటి పనితీరు ప్రవాహాల్లో దానిని సులభంగా కలుపుకోవచ్చు.

- **ఆప్టిమైజ్ செய்யப்பட்ட పనితీరు:** తక్కువ లేటెన్సీ మరియు అధిక మీదుకువలనును నిర్ధారించడానికి NVIDIA యొక్క ప్రీ-ఆప్టిమైజ్ చేసిన ఇంటెఫరెన్స్ ఇంజిన్‌లు, ఉదాహరణకు TensorRT మరియు TensorRT-LLM ను ఉపయోగిస్తుంది.
- **పరిమాణం పెరిగే సామర్థ్యం:** NIM కుబెర్నేట్స్ పై ఆటోస్కేలింగ్‌ను మద్దతు ఇస్తుంది, దీనివల్ల వివిధ వర్క్ లోడ్‌లను సమర్థవంతంగా నిర్వహించగలదు.
- **భద్రత మరియు నియంత్రణ:** సంస్థలు తమ డేటా మరియు అప్లికేషన్లు పై నియంత్రణను కలిగి ఉండేందుకు తమ సొంత నిర్వహించిన మౌలిక సదుపాయాల్లో NIM మైక్రోసర్వీసెస్‌ను స్వయం-హోస్టింగ్ చేయవచ్చు.
- **ప్రామాణిక APIలు:** NIM పరిశ్రమ ప్రామాణిక APIలను అందిస్తుంది, దీనివల్ల చాట్ బాట్లు, AI సహాయకులు, మరియు మరిన్ని AI అప్లికేషన్లు సులభంగా నిర్మించవచ్చు మరియు అనుసంధానం చేయవచ్చు.

NIM NVIDIA AI ఎంటర్‌ప్రైజ్ భాగంగా ఉంది, ఇది AI మోడళ్లు సులభంగా డిప్లాయ్ చేయడం మరియు ఆపరేషన్లు సుసాధ్యం చేయడాన్ని లక్ష్యంగా పెట్టుకుని, అవి NVIDIA GPUలపై సమర్థవంతంగా పరుగెత్తేలా చేస్తుంది.

### Phi-3/3.5 లోకల్ గా పరిరక్షించడం
Phi-3 లేదా GPT-3 వంటి ఏదైనా భాషా మోడల్‌కు సంబంధించి ఇన్ఫరెన్స్ అనగా అది అందే ఇన్‌పుట్ ఆధారంగా ప్రతిస్పందనలు లేదా అంచనాలు ఉత్పత్తి చేసే ప్రక్రియ. మీరు Phi-3 కు ఒక ప్రాంప్ట్ లేదా ప్రశ్న అందించినప్పుడు, అది తాను శిక్షణ పొందిన డేటాలోని నమూనాలు మరియు సంబంధాలను విశ్లేషించి అత్యంత సంభావ్యమైన మరియు సంబంధిత ప్రతిస్పందనను తరచుతుంది.

**Hugging Face ట్రాన్స్‌ఫార్మర్**
Hugging Face ట్రాన్స్‌ఫార్మర్స్ ఒక శక్తివంతమైన లైబ్రరీ, ఇది సహజ భాషా ప్రాసెసింగ్ (NLP) మరియు ఇతర మెషీన్ లెర్నింగ్ పనుల కోసం రూపొందించబడింది. దీని గురించి కొన్ని ముఖ్య అంశాలు:

1. **ప్రీట్రెయిన్ చేసిన మోడళ్లు:** ఇది వేర్వేరు పనుల కోసం వాడుకునేకి వేర్వేరు అనేక ప్రీట్రెయిన్ చేసిన మోడళ్లను అందిస్తుంది, వీటిలో టెక్స్ట్ వర్గీకరణ, పేరున్న ఎంటిటీ గుర్తింపు, ప్రశ్నల సమాధానం, సారాంశం, అనువాదం, టెక్స్ట్ ఉత్పత్తి మొదలైనవి ఉన్నాయి.

2. **ఫ్రేమ్‌వర్క్ ఇంటర్‌ఓపరబిలిటీ:** లైబ్రరీ PyTorch, TensorFlow, మరియు JAX వంటి అనేక డీప్ లెర్నింగ్ ఫ్రేమ్‌వర్క్లను మద్దతు ఇస్తుంది. దీని వల్ల మీరు ఒక ఫ్రేమ్‌వర్క్‌లో మోడల్‌ను శిక్షణ ఇచ్చి మరొకదానిలో వాడుకోవచ్చు.

3. **మల్టీమోడల్ సామర్థ్యాలు:** NLP తో పాటు, Hugging Face ట్రాన్స్‌ఫార్మర్స్ కంప్యూటర్ విజన్ (ఉదా: చిత్రం వర్గీకరణ, వస্তু గుర్తింపు) మరియు ఆడియో ప్రాసెసింగ్ (ఉదా: ప్రసంగ గుర్తింపు, ఆడియో వర్గీకరణ) పనులను మద్దతు ఇస్తుంది.

4. **సులభంగా వాడుకోవడం:** లైబ్రరీ APIs మరియు టూల్స్‌ను అందిస్తుంది, ఇవి మోడళ్లను సులువుగా డౌన్‌లోడ్ చేసుకుని ఫైన్-ట్యూన్ చేయడానికి సహాయపడతాయి, అందువల్ల ప్రారంభించేవారికి మరియు నిపుణులకు రెండింటికీ సులభం.

5. **సమాజం మరియు వనరులు:** హెగ్గింగ్ ఫేస్‌కి ఒక ఉత్సాహభరిత సమాజం మరియు విస్తృతమైన డాక్యుమెంటేషన్, ట్యుటోరియల్స్, మరియు గైడ్‌లు ఉన్నాయి, ఇవి వినియోగదారులను ప్రారంభించడంలో మరియు లైబ్రరీను మరింతగా ఉపయోగించడంలో సహాయపడతాయి.

**Ollama**
[Ollama](https://ollama.com/?WT.mc_id=academic-105485-koreyst) అనేది పెద్ద భాషా మోడళ్లను (LLMs) మీ యంత్రం పై స్థానికంగా నడపడం సులభతరం చేసే ప్లాట్‌ఫామ్. ఇది Llama 3.1, Phi 3, Mistral, మరియు Gemma 2 వంటి వివిధ మోడళ్లకు మద్దతు ఇస్తుంది. ఈ ప్లాట్‌ఫామ్ మోడల్ వెయిట్స్, కాన్ఫిగరేషన్ మరియు డేటాను ఒకే ప్యాకేజీగా గానీ, సులభం చేస్తూ యూజర్లకు తమ సొంత మోడళ్లను కస్టమైజ్ చేయడానికి మరియు సృష్టించడానికి అవకాశాన్ని ఇస్తుంది. Ollama మాక్‌ఓఎస్, లినక్స్, మరియు విండోస్‌కు అందుబాటులో ఉంది. క్లౌడ్ సర్వీసులు మీద ఆధారపడకుండా LLMలను అన్వేషించడానికి లేదా డిప్లాయ్ చేయడానికి ఇది అద్భుతమైన సాధనం. Ollama ఉపయోగించడానికి మీరు కేవలం క్రింది కమాండ్‌ను అమలు చేయాలి.

**Foundry Local**

[Foundry Local](https://foundrylocal.ai?WT.mc_id=academic-105485-koreyst) అనేది Microsoft యొక్క ఆఫ్‌లైన్, పరికరం-అంతర్గత రన్ టైమ్, ఇది Phi వంటి మోడళ్లను మీ స్వంత హార్డ్‌వేర్ పై పూర్తిగా నడపడానికి రూపొందించబడింది - ఏ Azure సబ్స్క్రిప్షన్, API కీ, లేదా నెట్‌వర్క్ కనెక్షన్ అవసరం లేదు. ఇది అందుబాటులో ఉన్న ఉత్తమ ఎగ్జిక్యూషన్ ప్రొవైడర్ (NPU, GPU, లేక CPU) ను ఆటోమేటిగ్గా ఎంచుకుంటుంది మరియు OpenAI-అనుకూల ఎండ్పాయింట్‌ను అందిస్తుంది, అందువల్ల ఉన్న `openai`/Azure AI Inference SDK కోడ్‌ను కేవలం తక్కువ మార్పులతో దానిపై సంకేతం చేయవచ్చు. ప్రారంభించడాన్ని చూడాలనుకుంటే [Foundry Local డాక్యుమెంటేషన్](https://learn.microsoft.com/azure/ai-foundry/foundry-local/get-started?WT.mc_id=academic-105485-koreyst) చూడండి.

```bash

winget install Microsoft.FoundryLocal
foundry model run phi-3.5-mini

```

లేదా Python లో నేరుగా SDK ఉపయోగించండి:

```bash

pip install foundry-local-sdk

```

```python

from foundry_local import FoundryLocalManager

manager = FoundryLocalManager("phi-3.5-mini")
print(manager.endpoint, manager.api_key)

```

Generative AI కోసం ONNX Runtime (GENAI) అనేది వివిధ ప్లాట్‌ఫారమ్‌లపై జనరేటివ్ AI మోడళ్లను సమర్థవంతంగా నడపడానికి శక్తివంతమైన సాధనం.

## ONNX రన్‌టైమ్ అంటే ఏమిటి?
ONNX రన్‌టైమ్ అనేది ఓపెన్-సోర్స్ ప్రాజెక్ట్, ఇది మెషీన్ లెర్నింగ్ మోడళ్లకు ఉన్నత పనితీరు ఇన్ఫరెన్స్‌ను సక్రమంగా అందిస్తుంది. ఇది Open Neural Network Exchange (ONNX) ఫార్మాట్‌లోని మోడళ్లకు మద్దతు ఇస్తుంది, ఇది మెషీన్ లెర్నింగ్ మోడళ్లను ప్రాతినిధ్యం వహించే ప్రామాణిక ప్రమాణం. ONNX రన్‌టైమ్ ఇన్ఫరెన్స్ కస్టమర్ అనుభవాలను వేగవంతం చేసి ఖర్చులను తగ్గిస్తుంది, PyTorch మరియు TensorFlow/Keras వంటి డీప్ లెర్నింగ్ ఫ్రేమ్‌వర్క్‌లతో పాటు scikit-learn, LightGBM, XGBoost వంటి క్లాసికల్ మెషీన్ లెర్నింగ్ లైబ్రరీలకు మద్దతు ఇస్తుంది. ONNX రన్‌టైమ్ వివిధ హార్డ్‌వేర్, డ్రైవర్, ఆపరేటింగ్ సిస్టమ్‌లకు అనుకూలంగా ఉంటుంది మరియు గ్రాఫ్ ఆప్టిమైజేషన్లు మరియు ట్రాన్స్‌ఫార్మ్‌లతో పాటు ఒకప్పుడు హార్డ్‌వేర్ యాక్సిలరేటర్లను వినియోగించి ఉత్తమ పనితీరును అందిస్తుంది.

## జనరేటివ్ AI అంటే ఏమిటి?
జనరేటివ్ AI అనగా శిక్షణ పొందిన డేటా ఆధారంగా కొత్త కంటెంట్ (టెక్స్ట్, చిత్రాలు, సంగీతం మొదలైనవి)ను ఉత్పత్తి చేయగల AI సిస్టమ్స్. ఉదాహరణకు GPT-3 వంటి భాషా మోడళ్లు మరియు Stable Diffusion వంటి చిత్రం ఉత్పత్తి మోడళ్లు. ONNX Runtime కోసం GenAI లైబ్రరీ ONNX మోడళ్ల కోసం జనరేటివ్ AI లూప్‌ను అందిస్తుంది, ఇందులో ONNX Runtime తో ఇన్ఫరెన్స్, లాజిట్స్ ప్రాసెసింగ్, వెతికే మరియు శాంప్లింగ్, మరియు KV క్యాచ్ నిర్వహణ ఉంటాయి.

## ONNX Runtime కోసం GENAI
ONNX Runtime కోసం GENAI, ONNX Runtime సామర్థ్యాలను విస్తరించి జనరేటివ్ AI మోడళ్లకు మద్దతు ఇస్తుంది. కొన్ని ముఖ్య లక్షణాలు:

- **విశాలమైన ప్లాట్‌ఫార్మ్ మద్దతు:** ఇది Windows, Linux, macOS, Android, మరియు iOS వంటి వివిధ ప్లాట్‌ఫార్మ్‌లపై పనిచేస్తుంది.
- **మోడల్ మద్దతు:** LLaMA, GPT-Neo, BLOOM వంటి అనేక პოპულర్ జనరేటివ్ AI మోడళ్లకు మద్దతు ఇస్తుంది.
- **పనితీరు ఆప్టిమైజేషన్:** NVIDIA GPUలు, AMD GPUలు, మరియు మరిన్ని వంటి వివిధ హార్డ్‌వేర్ యాక్సిలరేటర్లకు అనుకూలమైన ఆప్టిమైజేషన్లు ఉన్నాయి.
- **సులభంగా వాడుకోవడం:** అప్లికేషన్లలో సులభంగా అనుసంధానం చేసుకునేందుకు APIలను అందిస్తుంది, కాస్తే కోడ్‌తో మీరు టెక్స్ట్, చిత్రాలు మరియు ఇతర కంటెంట్ ఉత్పత్తి చేయవచ్చు.
- వినియోగదారులు ఒక హై లెవల్ generate() విధానాన్ని పిలవవచ్చు లేదా మోడల్ ప్రతీ ఐటరేషన్‌ను లూప్‌లో నడిపించి ఒకరు ఒక్క టోకన్‌ను ఉత్పత్తి చేస్తూ, అవసరమైతే లూప్ లోనైనా జనరేషన్ పరామితులను అప్డేట్ చేసుకోవచ్చు.
- ONNX రన్‌టైమ్ గ్రీడి/బీమ్ వెతికి, TopP, TopK శాంప్లింగ్‌ను టోకెన్ సీక్వెన్స్ ఉత్పత్తికి, మరియు రిపిటిషన్ పెనాల్టీల వంటి బిల్ట్-ఇన్ లాజిట్స్ ప్రాసెసింగ్‌ను మద్దతు ఇస్తుంది. మీరు స్వంత స్కోరింగ్‌ను సులువుగా జోడించవచ్చు.

## ప్రారంభించడం ఎలా
ONNX Runtime కోసం GENAI తో ప్రారంభం కావడానికి, మీరు ఈ దశలను అనుసరించవచ్చు:

### ONNX Runtime ఇన్‌స్టాల్ చేయండి:
```Python
pip install onnxruntime
```
### Generative AI విస్తరణలను ఇన్‌స్టాల్ చేయండి:
```Python
pip install onnxruntime-genai
```

### ఒక మోడల్ నడపండి: Python లో సరళమైన ఉదాహరణ ఇక్కడ:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### డెమో: ONNX Runtime GenAI ఉపయోగించి Phi-3.5-Vision కాల్ చేయడం


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
