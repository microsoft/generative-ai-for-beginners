# ఆరంభంకు చిన్న భాషా నమూనాలు జనరేటివ్ AI కోసం ప్రారంభకులకు
జనరేటివ్ AI అనేది ఒక ఆసక్తికరమైన కృత్రిమ మేధస్సు రంగం, ఇది కొత్త కంటెంట్‌ను సృష్టించగల సిస్టమ్స్ ను రూపొందించడంపై దృష్టి పెట్టి ఉంటుంది. ఈ కంటెంట్ టెక్స్ట్, చిత్రాలు, సంగీతం మరియు పూర్తి వర్చువల్ పర్యావరణాల వరకు విస్తరించవచ్చు. జనరేటివ్ AI యొక్క అత్యంత ఉత్సాహభరితమైన అనువర్తనాల్లో ఒకటి భాషా నమూనాల రంగంలో ఉంటుంది.

## చిన్న భాషా నమూనాలు అంటే ఏమిటి?

చిన్న భాషా నమూనా (SLM) అనేది పెద్ద భాషా నమూనా (LLM) యొక్క స్తూపిస్తున్న వెర్షన్, ఇది LLMల యొక్క ఎన్నో నిర్మాణాత్మక సూత్రాలు మరియు సాంకేతికతలను ఉపయోగించి ఉంటుంది, కానీ గణనీయంగా తక్కువ కంప్యూటేషనల్ ఫుట్ ప్రింట్ ప్రదర్శిస్తుంది.

SLMలు మానవ విన్యాసానికి సాంగత్యం కలిగిన టెక్స్ట్‌ను సృష్టించడానికి రూపొందించిన భాషా నమూనాల ఉపసమూహం. వాటి పెద్ద సహచరులు, ఉదాహరణకు GPT-4 లాగా కాకుండా, SLMలు మరింత సన్నగా, సమర్థంగా ఉంటాయి, ఇవి గణనీయ వనరులు పరిమితమై ఉన్న అనువర్తనాలకు అందుబాటులో ఉంటాయి. సైజు చిన్న అయినప్పటికీ, వీరు విభిన్న కార్యాలను నిర్వహించగలరు. సాధారణంగా, SLMలు LLMలను సంకోచించడం లేదా డిస్టిల్ చేయడం ద్వారా రూపొందించబడతాయి, అసలు నమూనా యొక్క ముఖ్యా ఫంక్షనాలిటీ మరియు భాషా సామర్థ్యాలను పెద్ద శాతం నిలుపుకోవాలని లక్ష్యంగా ఉంచుకుంటాయి. ఈ నమూనా పరిమాణం తగ్గించడం మొత్తంలో సంక్లిష్టతను తగ్గిస్తుంది, ఇది SLMలను మెమరీ వినియోగం మరియు గణనీయ అవసరాల పరంగా మరింత సమర్థవంతంగా మలుస్తుంది. ఈ సవరణల నేపధ్యంలో కూడా, SLMలు త్వరితగతిన విస్తృతమైన సహజ భాషా ప్రాసెసింగ్ (NLP) కార్యాలను నిర్వహించగలవు:

- టెక్స్ట్ సృష్టి: సమకాలీనంగా సారాంశమైన, సందర్భోచితమైన వాక్యాలు లేదా పేరాలు సృష్టించడం.
- టెక్స్ట్ పూర్తి: ఇచ్చిన ప్రాంప్ట్ ఆధారంగా వాక్యాలను ఊహించి పూర్తి చేయడం.
- అనువాదం: ఒక భాష నుండి మరొక భాషకు టెక్స్ట్ మార్పిడి.
- సంక్షిప్తీకరణ: పొడవైన టెక్స్ట్‌లను చిన్న, సులభంగా గ్రహించదగిన సారాంశాలకు సంక్లిష్టీకరించడం.

పెద్ద సహచరులకన్నా పనితీరులో లేదా సమగ్రత లో కొంత తేడా ఉండవచ్చు.

## చిన్న భాషా నమూనాలు ఎలా పని చేస్తాయి?
SLMలు విస్తృతమైన టెక్స్ట్ డేటాపై శిక్షణ పొందుతాయి. శిక్షణ సమయంలో, అవి భాష యొక్క నమూనాలు మరియు నిర్మాణాలను నేర్చుకుని, వ్యాకరణపరమైన మరియు సందర్భానికి తగిన పదాలు సృష్టించగలవు. శిక్షణ ప్రక్రియలో ఉన్నాయి:

- డేటా సేకరణ: వివిధ మూలాల నుండి విస్తృతమైన టెక్స్ట్ డేటాసెట్‌ల సేకరణ.
- ప్రీప్రాసెసింగ్: శిక్షణకు అనువుగా డేటాను శుభ్రం చేసి నిర్వహించడం.
- శిక్షణ: మెషీన్ లెర్నింగ్ అల్గోరిథమ్స్ ఉపయోగించి నమూనాకు టెక్స్ట్ ను అర్థం చేసుకోవటం మరియు సృష్టించటం నేర్పించడం.
- ఫైన్-ట్యూనింగ్: నిర్దిష్ట కార్యాలపై పనితీరును మెరుగుపరచేందుకు నమూనాను సవరించడం.

SLMల అభివృద్ధి వనరుల పరిమిత ప్రాంతాలలో అమర్చదగిన నమూనాలకు పెరుగుతున్న అవసరాన్ని నిశితంగా పూరించి ఉంటుంది, ఉదాహరణకు మొబైల్ డివైజ్లు లేదా ఎడ్జ్ కంప్యూటింగ్ ప్లాట్‌ఫారమ్లు, ఇక్కడ పెద్ద పూర్తి స్థాయి LLMలు భారీ వనరుల అవసరం కారణంగా అనుకోలేని అవగాహన పొందవచ్చని. సమర్థత పై దృష్టి పెట్టి, SLMలు పనితీరు మరియు అందుబాటును సమతుల్యం చేస్తూ విస్తృత అనువర్తనాలను సాధ్యమైనవులుగా చేస్తాయి.

## నేర్చుకోవాల్సిన లక్ష్యాలు

ఈ పాఠంలో, మేము SLM పరిజ్ఞానాన్ని పరిచయం చేస్తూ, Microsoft Phi-3తో కలిపి టెక్స్ట్ కంటెంట్, విజన్ మరియు MoEలో వివిధ సందర్భాలను నేర్చుకుంటాము.

పాఠం ముగింపులో, మీరు ఈ ప్రశ్నలకు సమాధానాలు ఇవ్వగలిగేటట్లు ఉంటారు:

- SLM అంటే ఏమిటి?
- SLM మరియు LLM మధ్య తేడా ఏమిటి?
- Microsoft Phi-3/3.5 కుటుంబం అంటే ఏమిటి?
- Microsoft Phi-3/3.5 కుటుంబంతో ఎలా ఇన్ఫరెన్స్ చేయాలి?

సిద్ధమా? మొదలవుదాం.

## పెద్ద భాషా నమూనాలు (LLMs) మరియు చిన్న భాషా నమూనాలు (SLMs) మధ్య తేడాలు

LLMలు మరియు SLMలు probability machine learning ప్రాథమిక సూత్రాలపై నిర్మించబడి ఉండి, వాటి నిర్మాణ డిజైన్, శిక్షణ విధానాలు, డేటా సృష్టి ప్రక్రియలు మరియు నమూనా మూల్యాంకన సాంకేతికతలలో సారూప్యమైన పద్ధతులను అనుసరిస్తాయి. అయితే, ఈ రెండు రకాల నమూనాల మధ్య కొన్ని కీలక కారణాలు తేడాలు కలిగిస్తాయి.

## చిన్న భాషా నమూనాల అనువర్తనాలు

SLMలకు విస్తృతమైన అనువర్తనాలు ఉన్నాయి, వీటిలో ఉన్నాయి:

- చాట్బాట్స్: కస్టమర్ మద్దతు అందించటం మరియు సంభాషణాత్మకంగా వినియోగదారులతో సంబంధం ఉంచడం.
- కంటెంట్ సృష్టి: రచయితలకు సహాయం చేయటం, ఆలోచనలు సృష్టించడం లేదా మొత్తం వ్యాసాలను రూపొందించడం.
- విద్య: విద్యార్థులకు రచనా పనుల్లో సహాయం చేయటం లేదా కొత్త భాషలు నేర్చుకోవడంలో సహాయం.
- ప్రవేశయోగ్యత: వঞ্চనాసాధ్యుల కోసం ఉపకరణాలు సృష్టించడం, ఉదాహరణకు టెక్స్ట్-టు-స్పీచ్ సిస్టమ్స్.

**పరిమాణం**
  
LLMలు మరియు SLMలు మధ్య ప్రధాన తేడా నమూనాల పరిమాణంలో ఉంది. ChatGPT (GPT-4) వంటి LLMలు సుమారు 1.76 ట్రిలియన్ పరిమాణాలను కలిగి ఉండవచ్చు, అయితే Mistral 7B వంటి ఓపెన్-సోర్స్ SLMలు సుమారు 7 బిల్లియన్ పరిమాణాలతో రూపొందించబడ్డాయి. ఈ తేడా ప్రధానంగా నమూనా నిర్మాణం మరియు శిక్షణ ప్రక్రియలలో ఉన్న తేడిలకు తోడ్పడింది. ఉదాహరణకు, ChatGPT ఒక ఎంకోడర్-డికోడర్ నిర్మాణంలో స్వీయ-ఆటెంషన్ మెకానిజాన్ని ఉపయోగిస్తుంది, Mistral 7B డికోడర్-ఒకటే నమూనాలో మరింత సమర్థవంతమైన శిక్షణకు వీలు కలిపే స్లైడింగ్ విండో ఆప్టెన్షన్ ఉపయోగిస్తుంది. ఈ నిర్మాణ తేడా నమూనాల సంక్లిష్టత మరియు పనితీరుపై తీవ్ర ప్రభావం చూపుతుంది.

**అర్థం చేసుకోవటం**

SLMలు సాధారణంగా నిర్దిష్ట రంగాలలో పనితీరు కోసం ఆప్టిమైజ్ చేయబడతాయి, ఇవి వృత్తిపరంగా ప్రావీణ్యం కలిగి ఉంటాయి కానీ విస్తృత కాంటెక్స్ట్ల అర్థగ్రహణలో పరిమితంగా ఉండవచ్చు. వ్యతిరేకంగా, LLMలు మరింత వృత్తిపరమైన మానవ-సమాన మేధస్సును అనుకరించడానికి రూపొందించబడ్డాయి. విస్తృత, వైవిధ్యభరిత డేటాసెట్లపై శిక్షణ పొందిన LLMలు విభిన్న రంగాలలో బాగా పనిచేయడానికి రూపొందించబడ్డాయి, మరింత అనుకూలత మరియు అన్వయించుకోవడం కాను కలిగి ఉంటాయి. ఫలితంగా, LLMలు సహజ భాషా ప్రాసెసింగ్ మరియు ప్రోగ్రామింగ్ వంటి విస్తృత డౌన్స్ట్రీం పనుల కోసం మరింత అనువైనవి.

**కంప్యూటింగ్**

LLMల శిక్షణ మరియు అమలుకు భారీ వనరుల అవసరం, సాధారణంగా పెద్ద స్థాయి GPU క్లస్టర్లను అవసరం. ఉదాహరణకు, ChatGPT మాదిరి నమూనాను ప్రారంభదశ నుండి శిక్షణ చేయడం కోసం సహజాగా వేల వరకు GPUలు తరచుగా చాలా పెద్ద వ్యవధిలో అవసరం అవుతాయి. ప్రత్యామ్నాయంగా, SLMలు తక్కువ పరిమాణంతో గణనీయ వనరుల పరంగా మరింత అందుబాటులో ఉంటాయి. Mistral 7B వంటి నమూనాలను అనుకూల GPUలతో స్థానిక యంత్రాల్లో శిక్షణ మరియు అమలు చేయవచ్చు, అయితే శిక్షణకు ఇంకా అనేక గంటలు, బహుళ GPUలు అవసరం.

**పక్షపాతం**

LLMలలో పక్షపాతం ప్రధానంగా శిక్షణ డేటా స్వభావం వల్ల కలుగుతుంది. ఈ నమూనాలు సాధారణంగా ఇంటర్నెట్ నుండి అందుబాటులో ఉన్న అసంపూర్ణ సమాచారం లేదా ఒప్పందం లేని డేటాను ఆధారం చేస్తాయి, ఇది కొన్ని గ్రూపులను తక్కువగా చూపించవచ్చు లేదా తప్పుగా సూచించవచ్చు, తప్పులుచేసిన లేబులింగ్ మరియు భాషా పక్షపాతాలను లేదా దైలెక్ట్, భూభాగాల వేరియేషన్లు మరియు వ్యాకరణ నియమాల ప్రభావాలను ప్రతిబింబిస్తాయి. అదనంగా, LLM నిర్మాణాల సంక్లిష్టత అనుకోకుండా పక్షపాతాన్ని పెంచే అవకాశం కలదు, ఇది జాగ్రత్తగా ఫైన్-ట్యూనింగ్ లేకపోతే గమనించబడకుండానే ఉంటుంది. మరోవైపు, SLMలు ఎక్కువగా పరిమిత, రంగం-నిర్దిష్ట డేటాసెట్‌లపై శిక్షణ పొందుతాయి కనుక అవి తక్కువగా పక్షపాతాలకు గురవుతాయి, అయినప్పటికీ అవి పూర్తిగా రక్షితం కావు.

**ఇన్ఫరెన్స్**

SLMల చిన్న పరిమాణం ఇన్ఫరెన్స్ వేగం గణనీయంగా వేగవంతం చేస్తుంది, ఇవి పెద్ద రకమైన అంతకుముందు ప్రాసెసింగ్ అవసరం లేకుండా స్థానిక హార్డ్వేర్ పై సమర్థవంతంగా అవుట్‌పుట్‌లను సృష్టించగలవు. వ్యతిరేకంగా, LLMలు వారి పరిమాణం మరియు సంక్లిష్టత కారణంగా సమర్థవంతమైన ఇన్ఫరెన్స్ సమయాలను పొందటానికి భారీ సమాంతర గణన వనరులు అవసరం పడుతుంది. బహుళ సమకాలీన వినియోగదారుల ఉనికితో కూడినపుడు, పెద్దస్థాయిలో అమలు చేసినపుడు LLMల ప్రతిస్పందన సమయాలు మరింత మందగిస్తాయి.

సంక్షేపంగా, LLMలు మరియు SLMలు ఇద్దరూ మెషీన్ లెర్నింగ్ ప్రాథమికతలలో పునాదిలా ఉన్నప్పటికీ, నమూనా పరిమాణం, వనరు అవసరాలు, సందర్భ అవగాహన, పక్షపాతానికి ఆవేదన, మరియు ఇన్ఫరెన్స్ వేగం పరంగా గణనీయ తేడాలు కలిగి ఉన్నాయి. ఈ తేడాలు ఒప్పుకునే ఒకదాన్ని మరొకదానికి తదు ఉపయోగకారితా కొరకు తేడా సూచిస్తాయి, LLMలు విస్తృతమే కానీ వనరు-భారంగా ఉన్నవి, SLMలు మరింత రంగ నిర్దిష్ట సమర్థతను తక్కువ గణన వనరులతో అందిస్తాయి.

***గమనిక: ఈ పాఠంలో, మేము Microsoft Phi-3 / 3.5 ను ఉదాహరణగా తీసుకొని SLM పరిచయం చేస్తాము.***

## Phi-3 / Phi-3.5 కుటుంబం పరిచయం

Phi-3 / 3.5 కుటుంబం ప్రధానంగా టెక్స్ట్, విజన్, మరియు ఏజెంట్ (MoE) అనువర్తన సన్నివేశాలకు లక్ష్యంగా ఉంటుంది:

### Phi-3 / 3.5 ఇనస్ట్రక్ట్

ప్రధానంగా టెక్స్ట్ జనరేషన్, చాట్ పూర్తి, మరియు కాంటెంట్ సమాచారం సేకరణ కోసం.

**Phi-3 మినీ**

3.8B పరిమాణం గల భాషా నమూనా Microsoft Foundry, Hugging Face, మరియు Ollamaలో అందుబాటులో ఉంది. Phi-3 నమూనాలు సమాన పెద్ద పరిమాణ భాషా నమూనాల కంటే కీలక బెంచ్‌మార్క్లలో (కింది అంకెలు చూడండి, ఎక్కువ సంఖ్యలు మంచివి) గణనీయంగా మెరుగ్గా ఉంటాయి. Phi-3 మినీ తన పరిమాణం కంటే రెండు రెట్లు పెద్ద నమూనాలు మించిన పనితీరు కలిగి ఉంటుంది, Phi-3-స్మాల్ మరియు Phi-3-మీడియం GPT-3.5 సహా పెద్ద నమూనాలను మించిన పనితీరును అందిస్తున్నాయి.

**Phi-3-స్మాల్ & మీడియం**

కేవలం 7B పరిమాణాలతో, Phi-3-స్మాల్ వివిధ భాషా, తర్క, కోడింగ్, మరియు గణిత బెంచ్‌మార్క్లలో GPT-3.5Tను మించి ఉంటుంది.

14B పరిమాణం గల Phi-3-మీడియం ఈ ధోరణిని కొనసాగిస్తూ Gemini 1.0 Proను మించి పనిచేస్తుంది.

**Phi-3.5-మినీ**

దీన్ని Phi-3-మినీకి అప్‌గ్రేడ్‌గా భావించవచ్చు. పరిమాణాలు మారకుండా ఉన్నప్పటికీ, ఇది అనేక భాషలను మద్దతు ఇవ్వగల సామర్థ్యాన్ని మెరుగుపరుస్తుంది (20+ భాషలు మద్దతు: అరబిక్, చైనీస్, చెక్, డానిష్, డచ్, ఇంగ్లీష్, ఫిన్నిష్, ఫ్రెంచ్, జర్మన్, హీబ్రూ, హంగేరియన్, ఇటాలియన్, జపనీస్, కొరీయన్, నార్వీషియన్, పొలిష్, పోర్చుగీస్, రష్యన్, స్పానిష్, స్వీడిష్, థాయ్, టర్కిష్, ఉక్రెయిన్) మరియు ఎక్కువగా కాంటెక్స్ట్ మద్దతు చేస్తుంది.

3.8B పరిమాణంతో Phi-3.5-మినీ ఒకే పరిమాణ భాషా నమూనాలను మించి పనిచేస్తుంది మరియు రెండె రెట్లు పెద్ద నమూనాల సమానం.

### Phi-3 / 3.5 విజన్

Phi-3/3.5 ఇన్స్ట్ర క్ట్ నమూనా Phi యొక్క అర్థం చేసుకునే సామర్థ్యంగా భావించవచ్చు, మరియు విజన్ Phiకు ప్రపంచాన్ని అర్ధం చేసుకునే కళ్ళను ఇస్తుంది.


**Phi-3-విజన్**

కేవలం 4.2B పరిమాణాలతో Phi-3-విజన్ ఈ ధోరణిని కొనసాగిస్తూ సాధారణ దృశ్య తర్క, OCR, పట్టిక మరియు చిత్ర వివరణ పనులను Claude-3 Haiku మరియు Gemini 1.0 Pro V కన్నా మెరుగ్గా చేస్తుంది.


**Phi-3.5-విజన్**

Phi-3.5-విజన్ కూడా Phi-3-విజన్ యొక్క అప్‌గ్రేడ్, ఇది బహుళ చిత్రాలను మద్దతు ఇస్తుంది. ఇది విజన్లో మెరుగుదల గా భావించవచ్చు, మీరు చిత్రాలు మాత్రమే కాకుండా వీడియోలను కూడా చూడగలుగుతారు.

Phi-3.5-విజన్ OCR, పట్టిక, మరియు చార్ట్ అర్థం చేసుకోవడంలో Claude-3.5 Sonnet మరియు Gemini 1.5 Flash వంటి పెద్ద నమూనాలను మించి పనిచేస్తుంది మరియు సాధారణ విజ్యువల్ జ్ఞాన తర్కలలో సమానం. బహుళ ఫ్రేమ్ ఇన్‌పుట్ మద్దతు, అంటే బహుళ ఇన్‌పుట్ చిత్రాలపై తర్క చేయడం.


### Phi-3.5-MoE

***Mixture of Experts (MoE)*** నమూనాలను చాలా తక్కువ గణనతో ప్రీట్రైన్ చేయడానికి అనుమతిస్తుంది, అదే గణన బడ్జెట్‌తో మీరు నమూనా లేదా డేటాసెట్ పరిమాణాన్ని డ్రమాటిక్‌గా పెంచవచ్చు. ముఖ్యంగా, MoE నమూనా తన డెన్స్ సహచరి ద్వారా ఉత్పన్నం గా ఉన్న అదే నాణ్యతను త్వరగా సాదించగలదు.

Phi-3.5-MoE 16x3.8B నిపుణ మాడ్యూల్‌లతో రూపొందుంది. 6.6B సక్రియ పరామితులతో Phi-3.5-MoE భారీ పెద్ద నమూనాలతో సమానమైన తర్క, భాషా అవగాహన, మరియు గణిత సామర్థ్యాన్ని అందిస్తుంది.

మనం భిన్న సందర్భాలకు ఆధారంగా Phi-3/3.5 కుటుంబ నమూనాలను ఉపయోగించవచ్చు. LLMల నుండి భిన్నంగా, Phi-3/3.5-మినీ లేదా Phi-3/3.5-విజన్‌ను ఎడ్జ్ పరికరాలలో అమర్చవచ్చు.

**NVIDIA NIM**

క్లౌడ్-ఆధారిత Microsoft Foundry Models కాటలాగ్‌కు అదనంగా, మీరు [NVIDIA NIM](https://developer.nvidia.com/nim?WT.mc_id=academic-105485-koreyst) ఉపయోగించి సంబంధిత కాల్‌లను పూర్తి చేయవచ్చు. మీరు NVIDIA NIMని సందర్శించి Phi-3/3.5 కుటుంబ API కాల్‌లను పూర్తి చేయవచ్చు. NVIDIA NIM (NVIDIA ఇన్ఫరెన్స్ మైక్రోసర్వీసులు) అనేది వివిధ వాతావరణాలలో, క్లౌడ్లు, డేటా సెంటర్లు మరియు వర్క్‌స్టేషన్లు లో AI నమూనాలను సమర్థవంతంగా అమర్చేందుకు రూపొందించిన వేగవంతమైన ఇన్ఫరెన్స్ మైక్రోసర్వీసులు సెట్.

ఇక్కడ NVIDIA NIM కొన్ని ప్రధాన లక్షణాలు:

- **సులభతర అమరిక:** NIM ఒక కమాండ్ ద్వారా AI నమూనాలను అమర్చవచ్చు, దీని వల్ల ఇప్పటికే ఉన్న వర్క్‌ఫ్లోలలో సులభంగా ఏకీకృతం చేయడం సాధ్యం.

- **అప్టిమైజ్డ్ పనితీరు:** ఇది NVIDIA యొక్క ముందుగా ఆప్టిమైజ్డ్ ఇన్ఫరెన్స్ ఇంజన్లను ఉపయోగిస్తుంది, ఉదాహరణకు TensorRT మరియు TensorRT-LLM, తక్కువ ఆలస్యం మరియు అధిక ప్రొడക്ഷన్‌ను నిర్ధారించడానికి.
- **స్కేలబిలిటీ:** NIM Kubernetes లో ఆటోస్కేలింగ్‌ను మద్దతు ఇస్తుంది, దాని ద్వారా ఇది మార్మోగుతున్న పని లోడ్లను ప్రభావవంతంగా నిర్వహించగలదు.
- **సెక్యూరిటీ మరియు కంట్రోల్:** సంస్థలు తమ డేటా మరియు అనువర్తనాలపై నియంత్రణను తమ స్వంత నిర్వహించబడిన ఇन्फ్రాస్ట్రక్చర్‌పై NIM మైక్రోసర్వీసులను స్వయం-హోస్టింగ్ ద్వారా అనుసరించవచ్చు.
- **స్టాండర్డ్ APIలు:** NIM పరిశ్రమలో ప్రామాణిక APIలను అందిస్తుంది, చాట్బాట్స్, AI అసిస్టెంట్లు మరియు మరిన్ని వంటి AI అనువర్తనాలను సులభంగా నిర్మించడానికి మరియు అనంతరం అనుసంధానం చేయడానికి.

NIM NVIDIA AI ఎంటర్ప్రైజ్ భాగం, ఇది AI నమూనాలను పంపిణీ చేయడం మరియు ఆపరేషనల్ చేయడాన్ని సరళీకృతం చేసే లక్ష్యంతో ఉంది, అవి NVIDIA GPUs పై సమర్థవంతంగా రన్ అవుతున్నాయని నిర్ధారిస్తుంది.

### Phi-3/3.5 లోకల్‌గా రన్ చేయడం
Phi-3 లేదా GPT-3 వంటి ఏదైనా భాషా నమూనా సంబంధమైన ఇన్ఫరెన్స్ అంటే అందుకున్న ఇన్‌పుట్ ఆధారంగా ప్రతిస్పందనలు లేదా ఊహాగానాలు రూపొందించే ప్రక్రియ. మీరు Phi-3 కు ప్రాంప్ట్ లేదా ప్రశ్న ఇస్తే, అది తన శిక్షణ పొందిన న్యూరల్ నెట్‌వర్క్ ఉపయోగించి శిక్షణ పొందిన డేటాలోని నమూనాలు మరియు సంబంధాలను విశ్లేషించి అత్యంత సాధ్యమైన మరియు సంబంధిత ప్రతిస్పందనను భావిస్తుంది.

**Hugging Face ట్రాన్స్‌ఫార్మర్**
Hugging Face Transformers ఒక శక్తివంతమైన లైబ్రరీ, ఇది సహజ భాషా ప్రాసెసింగ్ (NLP) మరియు ఇతర మెషీన్ లెర్నింగ్ పనుల కోసం రూపొందించబడింది. దీని గురించి కొన్ని ముఖ్యమైన విషయాలు ఈ క్రింది విధంగా ఉన్నాయి:

1. **మునుపటి శిక్షణ పొందిన నమూనాలు:** పాఠ్య వర్గీకరణ, పేరైన ఎంటిటీ గుర్తింపు, ప్రశ్న సమాధానం, సారాంశం, అనువాదం మరియు పాఠ్య ఉత్పత్తి వంటి వివిధ పనులకు ఉపయోగించే వేలాది శిక్షణ పొందిన నమూనాలను ఇది అందిస్తుంది.

2. **ఫ్రేమ్‌వర్క్ పరస్పర సంబంధం:** ఈ లైబ్రరీ PyTorch, TensorFlow, మరియు JAX వంటి బహుముఖీగల డీప్ లెర్నింగ్ ఫ్రేమ్‌వర్క్‌లను మద్దతు ఇస్తుంది. దీని వలన మీరు ఒక ఫ్రేమ్‌వర్క్‌లో నమూనాను శిక్షణ ఇచ్చి, మరొకలో ఉపయోగించవచ్చు.

3. **బహుమాధ్యమ సామర్థ్యాలు:** NLPతో పాటు Hugging Face Transformers కంప్యూటర్ విజన్ (ఉదా: చిత్రం వర్గీకరణ, వస్తువు గుర్తింపు) మరియు ఆడియో ప్రాసెసింగ్ (ఉదా: భాష గుర్తింపు, ఆడియో వర్గీకరణ) పనులను కూడా మద్దతు ఇస్తుంది.

4. **సులభమైన వాడకం:** ఈ లైబ్రరీ మోడళ్లు డౌన్లోడ్ చేసుకోవడానికి మరియు ఫైన్-ట్యూన్ చేయడానికి APIs మరియు సాధనాలను అందిస్తుంది, అందువల్ల ప్రారంభకులు మరియు నిపుణులు క్యూట్ గా ఉపయోగించుకోవచ్చు.

5. **సమాజం మరియు వనరులు:** Hugging Faceకు ఉత్సాహభరితమైన సమాజం, విస్తృతమైన డాక్యుమెంటేషన్, పాఠాలు మరియు మార్గదర్శకాలు ఉన్నాయి, వీటి ద్వారా వినియోగదారులు సులభంగా ప్రారంభించి పూర్తి లైబ్రరీ ఉపయోగించవచ్చు.
[అధికారిక డాక్యుమెంటేషన్](https://huggingface.co/docs/transformers/index?WT.mc_id=academic-105485-koreyst) లేదా వారి [GitHub రిపాజిటరీ](https://github.com/huggingface/transformers?WT.mc_id=academic-105485-koreyst).

**Ollama**
[Ollama](https://ollama.com/?WT.mc_id=academic-105485-koreyst) ఒక వేదిక, ఇది పెద్ద భాషా నమూనాలను (LLMs) మీ కంప్యూటర్లో సులభంగా నడుపుటకు డిజైన్ చేయబడింది. ఇది Llama 3.1, Phi 3, Mistral, మరియు Gemma 2 వంటివి మద్దతు ఇస్తుంది. ఈ వేదిక మోడల్ బరువులు, కాన్ఫిగరేషన్ మరియు డేటాను ఒకే ప్యాకేజీలో ప్యాక్ చేసి వినియోగదారులకు మోడళ్లను తాము అనుసరించడానికి, రూపొందించడానికి మరింత చేరువనైనదిగా చేస్తుంది. Ollama MacOS, Linux, మరియు Windows అందుబాటులో ఉంది. మీరు క్లౌడ్ సేవలపై ఆధారపడకుండా LLMs అన్వేషించడానికి లేదా విస్తరించడానికి ఇది ఒక అద్భుతమైన సాధనం. Ollama చాలా నేరుగా పనీ, మీరు క్రింది కమాండ్‌ను నడపాలి.


```bash

ollama run phi3.5
```

**Foundry Local**

[Foundry Local](https://foundrylocal.ai?WT.mc_id=academic-105485-koreyst) Microsoft యొక్క ఆఫ్‌లైన్, ఆన్-డివైస్ రన్‌టైమ్, ఇది Phi వంటి మోడళ్లను పూర్తిగా మీ స్వంత హార్డ్‌వేర్‌పై నడిపించడానికి ఉద్దేశించబడింది - ఏ Azure సబ్స్క్రిప్షన్, API కీ లేదా నెట్‌వర్క్ కనెక్షన్ అవసరం లేదు. ఇది స్వయంచాలకంగా ఉత్తమ ఎగ్జిక్యుషన్ ప్రొవైడర్‌ను ఎంచుకుంటుంది (NPU, GPU, లేదా CPU) మరియు OpenAI-అనుకూలమైన ఎండ్‌పాయింట్‌ను అందిస్తుంది, కనుక ప్రస్తుత `openai`/Azure AI ఇన్ఫరెన్స్ SDK కోడ్ చిత్తగించకుండా దీన్ని పాయింట్ చేయవచ్చు. ప్రారంభానికి [Foundry Local డాక్యుమెంటేషన్](https://learn.microsoft.com/en-us/azure/ai-foundry/foundry-local/get-started?WT.mc_id=academic-105485-koreyst) చూడండి.

```bash

winget install Microsoft.FoundryLocal
foundry model run phi-3.5-mini

```

లేదా Pythonలో నేరుగా SDK ఉపయోగించండి:

```bash

pip install foundry-local-sdk

```

```python

from foundry_local import FoundryLocalManager

manager = FoundryLocalManager("phi-3.5-mini")
print(manager.endpoint, manager.api_key)

```

**GenAI కోసం ONNX రన్‌టైం**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) ఒక క్రాస్-ప్లాట్‌ఫారమ్ ఇన్ఫరెన్స్ మరియు శిక్షణ మెషీన్ లెర్నింగ్ యాక్సిలరేటర్. ONNX Runtime for Generative AI (GENAI) అనేది మీకు వివిధ ప్లాట్‌ఫారమ్‌లపై జనరేటివ్ AI మోడళ్లను సమర్థవంతంగా నడపడానికి సహాయపడే శక్తివంతమైన సాధనం.

## ONNX Runtime అంటే ఏమిటి?
ONNX Runtime ఒక ఓపెన్-సోర్స్ ప్రాజెక్ట్, ఇది మెషీన్ లెర్నింగ్ మోడళ్ల యొక్క హై-పర్ఫార్మెన్స్ ఇన్ఫరెన్స్‌ను సాద్యం చేస్తుంది. ఇది Open Neural Network Exchange (ONNX) ఫార్మాట్‌లోని మోడళ్లను మద్దతు ఇస్తుంది, ఇది మెషీన్ లెర్నింగ్ మోడళ్ల ప్రాతినిధ్యానికి ఒక ప్రమాణం. ONNX Runtime ఇన్ఫరెన్స్ వేగవంతమైన కస్టమర్ అనుభవం మరియు తక్కువ ఖర్చులు సాధ్యం చేస్తుంది, ఇది PyTorch మరియు TensorFlow/Keras వంటి డీప్ లెర్నింగ్ ఫ్రేమ్‌వర్క్‌ల మోడళ్లను మరియు scikit-learn, LightGBM, XGBoost వంటి క్లాసికల్ మెషీన్ లెర్నింగ్ లైబ్రరీలను మద్దతు ఇస్తుంది. ONNX Runtime వివిధ హార్డ్‌వేర్, డ్రైవర్స్, మరియు ఆపరేటింగ్ సిస్టమ్‌లతో అనుకూలంగా ఉంటుంది, అందులో గమనించాలని హార్డ్వేర్ యాక్సిలరేటర్లను ఉపయోగించి ఆప్టిమైజ్డ్ పనితీరు అందిస్తుంది, అలాగే గ్రాఫ్ ఆప్టిమైజేషన్లు మరియు ట్రాన్స్‌ఫార్మ్స్ కూడా ఉంటాయి.

## జనరేటివ్ AI అంటే ఏమిటి?
జనరేటివ్ AI అనేది శిక్షణ పొందిన డేటా ఆధారంగా కొత్త కంటెంట్, ఉదాహరణకు పాఠ్యం, చిత్రాలు లేదా సంగీతం సృష్టించగల AI వ్యవస్థలను సూచిస్తుంది. ఉదా: GPT-3 వంటి భాషా నమూనాలు మరియు Stable Diffusion వంటి చిత్రం సృష్టింపు మోడళ్లు. ONNX Runtime for GenAI లైబ్రరీ ONNX మోడళ్ల కోసం జనరేటివ్ AI లూప్‌ని అందిస్తుంది, అందులో ONNX Runtime తో ఇన్ఫరెన్స్, లాజిట్స్ ప్రాసెసింగ్, సెర్చ్ మరియు శాంప్లింగ్, మరియు KV క్యాష్ నిర్వహణ ఉంటాయి.

## ONNX Runtime for GENAI
ONNX Runtime for GENAI ONNX Runtime యొక్క సామర్థ్యాలను విస్తరించి జనరేటివ్ AI మోడళ్లను మద్దతు ఇస్తుంది. కొన్ని ముఖ్యమైన ఫీచర్లు:

- **విస్తృతమైన వేదిక మద్దతు:** ఇది Windows, Linux, macOS, Android, మరియు iOS వంటి వేర్వేరు వేదికల్లో పనిచేస్తుంది.
- **మోడల్ మద్దతు:** ఇది LLaMA, GPT-Neo, BLOOM మరియు మరిన్ని ప్రాచుర్యమైన జనరేటివ్ AI మోడళ్లను మద్దతు ఇస్తుంది.
- **పనితీరు ఆప్టిమైజేషన్:** ఇది NVIDIA GPUs, AMD GPUs వంటి వివిధ హార్డ్‌వేర్ యాక్సిలరేటర్లకు ఆప్టిమైజేషన్లు కలిగి ఉంది.
- **సులభమైన వాడకం:** ఇది అనువర్తనాలలో సులభంగా అనుసంధానం కోసం APIs అందిస్తుంది, తక్కువ కోడ్‌తో పాఠ్యం, చిత్రాలు మరియు ఇతర కంటెంట్ రూపొందించడం సులభం.
- వినియోగదారులు ఒక హై లెవల్ generate() పద్ధతిని పిలవవచ్చు, లేదా మోడల్ ప్రతీ ఇటరేషన్ లూప్‌లో ఒక టోకెన్ సృష్టించి, ఆ లూప్ లో జనరేషన్ పారామితులని మార్చవచ్చు.
- ONNX రన్‌టైమ్ శ్రేణి/బీమ్ సెర్చ్, TopP, TopK శాంప్లింగ్ మద్దతు కలిగి ఇక్కడ టోకెన్ సీక్వెన్స్‌లు మరియు పునరావృత దండనలు వంటి ఇంటిగ్రేటెడ్ లాజిట్స్ ప్రాసెసింగ్ ఉన్నాయి. మీరు సులభంగా కస్టమ్ స్కోరింగ్ కూడా జోడించవచ్చు.

## ప్రారంభించాలని
ONNX Runtime for GENAI ప్రారంభించడానికి మీరు ఈ దశలను అనుసరించవచ్చు:

### ONNX Runtime ఇన్‌స్టాల్ చేయండి:
```Python
pip install onnxruntime
```
### జనరేటివ్ AI ఎక్స్‌టెన్షన్లను ఇన్‌స్టాల్ చేయండి:
```Python
pip install onnxruntime-genai
```

### మోడల్ ను రన్ చేయండి: ఇక్కడ Pythonలో ఒక సాదా ఉదాహరణ ఉంది:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### డెమో: ONNX Runtime GenAI ఉపయోగించి Phi-3.5-Vision కాల్ చేయడం


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
