# మెరుగైన ఫీచర్లు మరియు మెరుగుదలల రోడ్మ్యాప్

ఈ దస్తావేజు జనరేటివ్ AI ఫర్ బిగినర్స్ పాఠ్యాంశానికి సిఫార్సు చేయబడిన మెరుగుదలలు మరియు అభివృద్ధులను ఒక సమగ్ర కోడ్ రివ్యూ మరియు పరిశ్రమ ఉత్తమ అభ్యాసాల విశ్లేషణ ఆధారంగా వివరించును.

## కార్యనిర్వాహక సారాంశం

కోడ్‌బేస్‌ను భద్రత, కోడ్ నాణ్యత మరియు విద్యాత్మక సమర్థత కోసం విశ్లేషించాము. ఈ దస్తావేజు తక్షణ పరిష్కారాలు, దగ్గర కాల మెరుగుదలలు మరియు భవిష్యత్తు అభివృద్ధుల కోసం సిఫార్సులను అందిస్తుంది.

## 4. API ఆధునీకరణ

### 4.1 పాత API నమూనాలు (మైగ్రేషన్ పూర్తయింది)

అన్ని పైథాన్ మరియు టైప్‌స్క్రిప్ట్ **చాట్** ఉదాహరణలు చాట్ కంప్లీషన్స్ API నుండి **రెస్పాన్స్ API** (`client.responses.create(...)` → `response.output_text`) కి మార్చబడ్డాయి.

| పాత నమూనా | కొత్త నమూనా | స్థితి |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (చాట్) | `OpenAI(base_url="<endpoint>/openai/v1/")` (రెస్పాన్స్ API) | పూర్తయింది |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | పూర్తయింది |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (టైప్‌స్క్రిప్ట్) | `openai` ప్యాకేజీ `client.responses.create()` → `response.output_text` | పూర్తయింది |
| `df.append()` (పాండాస్) | `pd.concat()` | పూర్తయింది |

> **గమనిక:** Microsoft Foundry Models ఉదాహరణలు `azure-ai-inference` / `@azure-rest/ai-inference` SDK (`client.complete()`) ఉపయోగించుటతో మోడల్ ఇన్ఫరెన్స్ API పై కొనసాగుతాయి, ఇది రెస్పాన్స్ API ను మద్దతు ఇవ్వదు. `AzureOpenAI()` ఇంకా చెల్లుబాటు అయ్యే చోట (ఎంబెడ్డింగ్స్ మరియు ఇమేజ్ జనరేషన్) ఉంచబడింది.

### 4.2 ప్రదర్శించవలసిన కొత్త API ఫీచర్లు

1. **సంరచించబడిన అవుట్‌పుట్లు** (OpenAI)
   - JSON మోడ్

## 5. సంకల్పణ మెరుగుదలలు

### 5.1 CI/CD మెరుగుదలలు

[`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml) లో అమలు చేయబడింది: పైథాన్ లింటింగ్/ఫార్మాటింగ్ (రఫ్ + బ్లాక్) **ఉపాధానపూర్వకంగా** నిర్వహించబడుతున్న `shared/` యుటిలిటీస్ మాడ్యూల్ పై కఠినంగా అమలు చేస్తుంది మరియు మిగతా పాఠ్యాంశం పై **సలహా మాత్రమే** నడుస్తుంది, అదనంగా జావాస్క్రిప్ట్/టైప్‌స్క్రిప్ట్ కోసం ESLint సలహా పాస్ ఉన్నది. సంకేతాత్మక బేస్‌లైన్:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### 5.2 భద్రతా స్క్యానింగ్

[`.github/workflows/security.yml`](../../../.github/workflows/security.yml) లో అమలు చేయబడింది: కోడ్QL విశ్లేషణ పైథాన్ మరియు జావాస్క్రిప్ట్/టైప్‌స్క్రిప్ట్ కోసం (పుష్, పుల్ అభ్యర్థన, మరియు వారపు షెడ్యూల్ పై) మరియు పుల్ అభ్యర్థనలపై డిపెండెన్సీ సమీక్ష కూడా. సంకేతాత్మక బేస్‌లైన్:

```yaml
# .github/workflows/security.yml
name: Security Scan

on: [push, pull_request]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: javascript, python
      - uses: github/codeql-action/analyze@v3

  dependency-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
```

### 6.1 డెవ్‌కంటైనర్ మెరుగుదలలు

[`.devcontainer/devcontainer.json`](../../../.devcontainer/devcontainer.json) మరియు [`.devcontainer/post-create.sh`](../../../.devcontainer/post-create.sh) లో అమలు చేయబడింది: ఇప్పుడు కంటైనర్ Pylance, Black ఫార్మాటర్, Ruff, ESLint, Prettier, మరియు Copilot ఎక్స్‌టెన్షన్లను సహా సరబరుస్తుంది, రిపో యొక్క Black/Prettier కాన్ఫిగరేషన్‌కు ఫార్మాట్-ఆన్-సేవ్ ను ఎనేబుల్ చేస్తుంది, మరియు డెవలపర్ టూలింగ్ (`ruff`, `black`, `mypy`, `pytest`)ను ఇన్‌స్టాల్ చేయడంతో [code-quality workflow](../../../.github/workflows/code-quality.yml) స్థానికంగా పునరుత్పత్తి అయ్యేలా ఉంటుంది. `mcr.microsoft.com/devcontainers/universal` బేస్ ఇమేజ్ ఇప్పటికే పైథాన్ మరియు నోడ్ బండిల్స్ ఉ౦టువంటి కారణంగా అదనపు ఫీచర్లు అవసరం లేవు. సంకేతాత్మక బేస్‌లైన్:

## 9. 접근성 및 국제화

### 9.1 ప్రస్తుత అనువాద స్థితి

అన్ని అనువాదాలు **పూర్తయినవి** మరియు [Azure Co-op Translator](https://github.com/Azure/co-op-translator?WT.mc_id=academic-105485-koreyst) ద్వారా ఆటోమేటిక్‌గా ఉత్పత్తి అయ్యాయి, ఇది ఆంగ్ల మూలంతో సమకాలీకృతంగా 50+ భాషా సంస్కరణలను ఉత్పత్తి చేస్తుంది మరియు నిర్వహిస్తుంది. అనువదించిన కంటెంట్ `translations/` లో మరియు స్థానికీకరించిన చిత్రాలు `translated_images/` లో ఉంటాయి; అందుబాటులో ఉన్న భాషల పూర్తి జాబితా రిపోజిటరీ README టాప్ వద్ద ప్రచురించబడింది.

| అంశం | స్థితి |
|--------|--------|
| అనువాద కవరేజ్ | పూర్తి — 50+ భాషలు, అన్ని పాఠ్యాంశాలు |
| అనువాద విధానం | [Azure Co-op Translator](https://github.com/Azure/co-op-translator?WT.mc_id=academic-105485-koreyst) ద్వారా ఆటోమేటిక్ |
| ఆంగ్ల మూలంతో సమకాలీకృతం | అవును — ఆటోమేటిక్ పునరుత్పత్తి |
