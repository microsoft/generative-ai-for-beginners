## Увод

АИ агенти представљају узбудљив развој у Генеративној АИ, омогућавајући Великим Језичким Моделима (LLM) да еволуирају од асистената до агената способних за предузимање акција. Оквири за АИ агенте омогућавају програмерима да креирају апликације које дају LLM приступ алатима и управљању стањем. Ови оквири такође побољшавају видљивост, омогућавајући корисницима и програмерима да прате акције које LLM планира, чиме се побољшава управљање искуством.

У лекцији ће бити обрађене следеће области:

- Разумевање шта је АИ агент - Шта тачно значи АИ агент?
- Истраживање пет различитих оквира за АИ агенте - По чему су јединствени?
- Примена ових АИ агената у различитим случајевима коришћења - Када треба користити АИ агенте?

## Циљеви учења

Након ове лекције, моћи ћете да:

- Објасните шта су АИ агенти и како се могу користити.
- Разумете разлике између неких од популарних оквира за АИ агенте.
- Разумете како АИ агенти функционишу у циљу израде апликација користећи их.

## Шта су АИ агенти?

АИ агенти су веома узбудљиво подручје у свету Генеративне АИ. Уз ово узбуђење понекад долази и конфузија око термина и њихове примене. Да бисмо све поједноставили и укључили већину алата који се односе на АИ агенте, користићемо следећу дефиницију:

АИ агенти омогућавају Великим Језичким Моделима (LLM) да обављају задатке тако што им дају приступ **стању** и **алатима**.

![Agent Model](../../../translated_images/sr/what-agent.21f2893bdfd01e6a.webp)

Хајде да дефинишемо ове термине:

**Велики Језички Модели** - То су модели о којима се говори у овом курсу као што су GPT-3.5, GPT-4, Llama-2 итд.

**Стање** - Односи се на контекст у којем LLM ради. LLM користи контекст својих претходних акција и тренутни контекст, што води до доношења одлука за следеће акције. Оквири за АИ агенте омогућавају програмерима да лакше одржавају овај контекст.

**Алате** - Да би завршио корисничке захтеве које је LLM планирао, потребан му је приступ алатима. Примери алата могу бити база података, API, спољна апликација или чак други LLM!

Ове дефиниције ће вам помоћи да боље разумете како се они имплементирају. Хајде да истражимо неке различите оквире за АИ агенте:

## LangChain агенти

[LangChain агенти](https://python.langchain.com/docs/how_to/#agents?WT.mc_id=academic-105485-koreyst) су имплементација горе наведених дефиниција.

За управљање **стaњем** користи уграђену функцију која се зове `AgentExecutor`. Ова функција прихвата дефинисаног `агента` и `алате` које он може користити.

`Agent Executor` такође чува историју разговора како би обезбедио контекст сарадње.

![Langchain Agents](../../../translated_images/sr/langchain-agents.edcc55b5d5c43716.webp)

LangChain нуди [каталог алата](https://integrations.langchain.com/tools?WT.mc_id=academic-105485-koreyst) који се могу увозити у вашу апликацију и којима LLM може приступити. Ове алате прави заједница и LangChain тим.

Можете дефинисати ове алате и проследити их `Agent Executor-у`.

Видљивост је такође важан аспект када говоримо о АИ агентима. Важно је да програмери апликација разумеју који алат користи LLM и зашто. Управо због тога тим из LangChain-а је развио LangSmith.

## AutoGen

Следећи оквир за АИ агенте који ћемо размотрити је [AutoGen](https://microsoft.github.io/autogen/?WT.mc_id=academic-105485-koreyst). Главни фокус AutoGen-а је разговор. Агенти су и **разговорни** и **подесиви**.

**Разговорни -** LLM могу започети и наставити разговор са другим LLM да би завршили задатак. Ово се постиже креирањем `AssistantAgents` и давањем одговарајуће системске поруке.

```python

autogen.AssistantAgent( name="Coder", llm_config=llm_config, ) pm = autogen.AssistantAgent( name="Product_manager", system_message="Creative in software product ideas.", llm_config=llm_config, )

```

**Подесиви** - Агенти могу бити дефинисани не само као LLM, већ и као корисник или алат. Као програмер можете дефинисати `UserProxyAgent` који је задужен за интеракцију са корисником у циљу добијања повратних информација за обављање задатка. Ове повратне информације могу наставити извршење задатка или га зауставити.

```python
user_proxy = UserProxyAgent(name="user_proxy")
```

### Стање и алати

Да би променио и управљао стањем, асистент агент генерише Python код за завршетак задатка.

Ево примера процеса:

![AutoGen](../../../translated_images/sr/autogen.dee9a25a45fde584.webp)

#### LLM дефинисан системском поруком

```python
system_message="For weather related tasks, only use the functions you have been provided with. Reply TERMINATE when the task is done."
```

Ова системска порука усмерава овај конкретни LLM ка функцијама које су релевантне за његов задатак. Запамтите, са AutoGen-ом можете имати више дефинисаних AssistantAgents са различитим системским порукама.

#### Корисник покреће разговор

```python
user_proxy.initiate_chat( chatbot, message="I am planning a trip to NYC next week, can you help me pick out what to wear? ", )

```

Ова порука од user_proxy (човек) покреће процес у коме агент истражује могуће функције које треба извршити.

#### Функција се извршава

```bash
chatbot (to user_proxy):

***** Suggested tool Call: get_weather ***** Arguments: {"location":"New York City, NY","time_periond:"7","temperature_unit":"Celsius"} ******************************************************** --------------------------------------------------------------------------------

>>>>>>>> EXECUTING FUNCTION get_weather... user_proxy (to chatbot): ***** Response from calling function "get_weather" ***** 112.22727272727272 EUR ****************************************************************

```

Након што се иницијални разговор обради, агент шаље предлог алата за позив. У овом случају то је функција која се зове `get_weather`. У зависности од конфигурације, ова функција може бити аутоматски извршена и процењена од стране агента или може бити извршена на основу корисничког уноса.

Можете пронаћи листу [AutoGen примера кода](https://microsoft.github.io/autogen/docs/Examples/?WT.mc_id=academic-105485-koreyst) даље за испитивање како започети изградњу.

## Microsoft Agent Framework

[Microsoft Agent Framework](https://learn.microsoft.com/agent-framework/?WT.mc_id=academic-105485-koreyst) је Microsoft-ов open-source SDK за изградњу АИ агената и мулти-агент система у **Python** и **.NET**. Он спаја снаге два претходна Microsoft пројекта — предузетничке функције **Semantic Kernel** и мулти-агентску оркестрацију **AutoGen** — у један подржани оквир. Ако данас крећете са новим агентским пројектом, ово је препоручени наследник AutoGen-а.

Оквир се скалира од једног **chat агента** до сложених **мулти-агентских токова рада**, и интегрише се директно са Microsoft Foundry, Azure OpenAI и OpenAI. Такође пружа уграђену опсервацију кроз OpenTelemetry тако да можете пратити шта ваши агенти тачно раде.

### Стање и алати

**Стање** - Оквир управља контекстом разговора за вас кроз **теме**. Агент прати историју порука (кориснички захтеви, позиви алата и њихови резултати), тако да сваки нови корак надовезује на претходне. Теме се могу такође сачувати, што омогућава паузирање и наставак разговора касније.

**Алате** - Дајете агенту алате тако што прослеђујете обичне Python функције. Типски означени параметри се аутоматски претварају у шему, па модел зна како и када их позвати (function calling). Оквир такође подржава сервере Model Context Protocol (MCP) и хостоване алате као што је кодски интерпретер.

Ево примера једног агента са прилагођеним алатом:

```python
import asyncio
from typing import Annotated

from pydantic import Field
from agent_framework import Agent
from agent_framework.openai import OpenAIChatClient


def get_weather(
    location: Annotated[str, Field(description="The location to get the weather for.")],
) -> str:
    """Get the weather for a given location."""
    return f"The weather in {location} is sunny with a high of 22°C."


async def main():
    agent = Agent(
        client=OpenAIChatClient(),
        instructions="You are a helpful assistant that can answer weather questions.",
        tools=[get_weather],
    )

    response = await agent.run("What's the weather in Amsterdam?")
    print(response)


asyncio.run(main())
```

Да бисте се уместо тога повезали на Azure OpenAI у Microsoft Foundry, проследите своју крајњу тачку и акредитиве клијенту:

```python
from azure.identity.aio import AzureCliCredential
from agent_framework.openai import OpenAIChatClient

client = OpenAIChatClient(
    model="my-gpt-4o-deployment",
    azure_endpoint="https://my-resource.openai.azure.com",
    credential=AzureCliCredential(),
)
```

### Мулти-агентски токови рада

Оквир заиста сјаји у оркестрацији више агената заједно. На пример, можете покретати агенте један за другим (сваки прослеђује свој контекст следећем) или разгранати их у више паралелних агената и агрегирати њихове резултате:

```python
from agent_framework.orchestrations import SequentialBuilder, ConcurrentBuilder

# Покрените агенте низано, прослеђујући контекст разговора дуж ланца
sequential = SequentialBuilder(participants=[researcher, writer, editor]).build()

# Распоредите агенте паралелно, а затим агрегирајте њихове одговоре
concurrent = ConcurrentBuilder(participants=[analyst_a, analyst_b, analyst_c]).build()
```

## Taskweaver

Следећи оквир агената који ћемо истражити је [Taskweaver](https://microsoft.github.io/TaskWeaver/?WT.mc_id=academic-105485-koreyst). Познат је као агент „прво код“ јер уместо да ради строго са `стринговима`, може радити са DataFrames у Python-у. Ово постаје изузетно корисно за задатке анализе и генерисања података, као што су креирање графикона и дијаграма или генерација случајних бројева.

### Стање и алати

За управљање стањем разговора, TaskWeaver користи појам `Планирача`. `Планирач` је LLM који узима захтев од корисника и мапира задатке које треба завршити да би испунио тај захтев.

За извршење задатака `Планирач` има приступ колекцији алата који се зову `Plugins`. То могу бити Python класе или општи кодни интерпретер. Ови plugin-ови се чувају као ембеддинзи како би LLM боље тражио одговарајући plugin.

![Taskweaver](../../../translated_images/sr/taskweaver.da8559999267715a.webp)

Ево примера plugin-а за детекцију аномалија:

```python
class AnomalyDetectionPlugin(Plugin): def __call__(self, df: pd.DataFrame, time_col_name: str, value_col_name: str):
```

Код се верификује пре извршења. Још једна функција за управљање контекстом у Taskweaver-у је `искуство`. Искусто омогућава да се контекст разговора чува током дужег периода у YAML фајлу. Ово се може конфигурисати тако да се LLM временом побољшава у одређеним задацима, под условом да је изложен претходним разговорима.

## JARVIS

Последњи оквир агената који ћемо истражити је [JARVIS](https://github.com/microsoft/JARVIS?tab=readme-ov-file&WT.mc_id=academic-105485-koreyst). ЈАРВИС је јединствен јер користи LLM за управљање `стањем` разговора, а `алати` су други АИ модели. Сваки од АИ модела су специјализовани модели који обављају одређене задатке као што су детекција објеката, транскрипција или описивање слика.

![JARVIS](../../../translated_images/sr/jarvis.762ddbadbd1a3a33.webp)

LLM, као модел опште намене, прима захтев корисника и идентификује конкретан задатак и све аргументе/пodatke потребне за завршетак задатка.

```python
[{"task": "object-detection", "id": 0, "dep": [-1], "args": {"image": "e1.jpg" }}]
```

Затим LLM форматира захтев на начин који специјализовани АИ модел може интерпретирати, као што је JSON. Када АИ модел врати прогнозу на основу задатка, LLM прими одговор.

Ако је потребно више модела за завршетак задатка, LLM ће такође интерпретирати одговоре тих модела пре него што их споји у коначан одговор кориснику.
