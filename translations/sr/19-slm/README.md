# Увод у мале језичке моделе за генеративну вештачку интелигенцију за почетнике
Генеративна вештачка интелигенција је фасцинантно поље вештачке интелигенције које се фокусира на креирање система способних да генеришу нови садржај. Тај садржај може бити у распону од текста и слика до музике и чак целих виртуелних окружења. Једна од најузбудљивијих примена генеративне вештачке интелигенције јесте у домену језичких модела.

## Шта су мали језички модели?

Мали језички модел (SLM) представља умањену варијанту великог језичког модела (LLM), користећи многе архитектонске принципе и технике LLM-ова, уз значајно смањен рачунарски отпечатак.

SLM су подскуп језичких модела дизајнираних да генеришу текст сличан људском. За разлику од својих већих колега, као што је GPT-4, SLM су компактнији и ефикаснији, што их чини идеалним за примене где су рачунарски ресурси ограничени. Упркос мањој величини, могу извршавати разне задатке. Обично се SLM граде компресијом или дистилацијом LLM модела, са циљем да задрже значајан део функционалности и језичких способности оригиналног модела. Ово смањење величине модела смањује укупну сложеност, чинећи SLM-ове ефикаснијим како у коришћењу меморије, тако и у рачунарским захтевима. Упркос овим оптимизацијама, SLM-и и даље могу извршавати широк спектар задатака обраде природног језика (NLP):

- Генерисање текста: Креирање кохерентних и контекстуално релевантних реченица или пасуса.
- Довршавање текста: Предвиђање и довршавање реченица на основу дате упуте.
- Превођење: Претварање текста са једног језика на други.
- Сумирање: Сажимање дугачких текстова у краће, лакше сварљиве прегледе.

Иако уз неке компромисе у перформансама или дубини разумевања у поређењу са већим моделима.

## Како раде мали језички модели?
SLM-ови се обучавају на огромним количинама текстуалних података. Током обуке, они уче обрасце и структуре језика, омогућавајући им да генеришу текст који је како граматички исправан, тако и контекстуално прикладан. Процес обуке укључује:

- Прикупљање података: Сакупљање великих скупова текстуалних података из различитих извора.
- Предобрада: Чишћење и организовање података како би били погодни за обуку.
- Обука: Коришћење алгоритама машинског учења да се модели науче како да разумеју и генеришу текст.
- Фино подешавање: Прилагођавање модела за побољшање перформанси у одређеним задацима.

Развој SLM-ова одговара растућој потреби за моделима који се могу распоредити у окружењима са ограниченим ресурсима, као што су мобилни уређаји или edge computing платформе, где пуноразмерни LLM може бити непрактичан због великих захтева за ресурсима. Фокусирајући се на ефикасност, SLM постижу баланс између перформанси и приступачности, омогућавајући широку примену у различитим областима.

## Разлике између великих језичких модела (LLM) и малих језичких модела (SLM)

И LLM и SLM се граде на основним принципима вероватноћног машинског учења, пратећи сличне приступе у архитектонском дизајну, методологијама обуке, процесима генерисања података и техникама процене модела. Међутим, неколико кључних фактора разликује ова два типа модела.

## Примене малих језичких модела

SLM имају широк спектар примена, укључујући:

- Четботови: Пружање корисничке подршке и интеракција са корисницима на конверзацијалан начин.
- Креирање садржаја: Помоћ писцима кроз генерисање идеја или чак састављање читавих чланака.
- Образовање: Помоћ ученицима у писању задатака или учењу нових језика.
- Приступачност: Креирање алата за особе са инвалидитетом, као што су системи претварања текста у говор.

**Величина**
  
Главна разлика између LLM и SLM лежи у обиму модела. LLM, као што је ChatGPT (GPT-4), могу имати процењено 1,76 билиона параметара, док open-source SLM као што је Mistral 7B имају значајно мањи број параметара — око 7 милијарди. Ова разлика углавном произилази из разлика у архитектури модела и процесима обуке. На пример, ChatGPT користи механизам самопажње у оквиру енкодер-декодер архитектуре, док Mistral 7B користи sliding window пажњу, што омогућава ефикаснију обуку у декодер-оружаном моделу. Ова архитектонска разлика има дубок утицај на сложеност и перформансе ових модела.

**Разумевање**

SLM обично оптимизују перформансе у специфичним доменима, што их чини веома специјализованим, али потенцијално ограниченим у способности да пруже широку контекстуалну разумевање преко више области знања. Насупрот томе, LLM настоје да симулирају интелигенцију сличну људској на обухватнијем нивоу. Обучени на обимним и разноликим скуповима података, LLM су дизајнирани да добро обављају задатке у разним областима, нудећи већу флексибилност и прилагодљивост. Последично, LLM су погоднији за шири спектар задатака, као што су обрада природног језика и програмирање.

**Рачунарска снага**

Обука и имплементација LLM су процеси који захтевају значајне ресурсе, често укључујући велике GPU кластерe. На пример, обука модела као што је ChatGPT од нуле може захтевати хиљаде GPU-ова током дугих периода. Насупрот томе, SLM, са мањим бројем параметара, су приступачнији у смислу рачунарских ресурса. Модели као што је Mistral 7B могу се обучавати и покретати на локалним машинама опремљеним умереним GPU капацитетом, мада је тренинг још увек потребно обавити током неколико сати на више GPU-ова.

**Пристраност**

Пристрастност је познат проблем у LLM, углавном због природе података за обуку. Ови модели често користе сирове, отворено доступне податке са интернета, који могу мање представљати или погрешно представљати одређене групе, уносити погрешно означавање или одражавати језичке пристрасности узроковане дијалектом, географским варијацијама и граматичким правилима. Поред тога, сложеност LLM архитектуре може ненамерно погоршати пристрасности, које могу остати непримећене без пажљивог фино подешавања. Са друге стране, SLM обучени на уским, доменски специфичним скупова података мање су подложни овим пристрасностима, иако нису потпуно имуни.

**Инференција**

Мања величина SLM-а пружа им значајну предност у брзини инференције, омогућавајући им да ефикасно генеришу излаз на локалном хардверу без потребе за обимном паралелном обрадом. Напротив, LLM, због своје величине и сложености, често захтевају значајне паралелне рачунарске ресурсе да би достигли прихватљиво време инференције. Присуство више истовремених корисника додатно успорава време одговора LLM-а, посебно при великом оптерећењу.

Укратко, иако и LLM и SLM деле основну базу у машинском учењу, они се знатно разликују по величини модела, захтевима за ресурсима, контекстуалном разумевању, подложности пристрасностима и брзини инференције. Ове разлике одражавају њихову прикладност за различите употребне случајеве, при чему су LLM свестранији али захтевају више ресурса, док SLM нуде већу ефикасност у специфичним доменима уз смањене рачунарске захтеве.

## Представљање Phi-3 / Phi-3.5 породице

Phi-3 / 3.5 породица је усмерена углавном на текст, визуелне и Agent (MoE) апликационе сценарије:

### Phi-3 / 3.5 Instruct

Углавном за генерисање текста, довршавање разговора и екстракцију информација из садржаја и слично.

**Phi-3-mini**

Језички модел од 3.8 милијарди параметара доступан је на Microsoft Foundry, Hugging Face и Ollama. Phi-3 модели значајно су супериорни од језичких модела исте или веће величине на кључним мерилима (погледајте бројке мерила у наставку, веће вредности су боље). Phi-3-mini надмашује моделе двоструко веће величине, док Phi-3-small и Phi-3-medium надмашују веће моделе, укључујући GPT-3.5.

**Phi-3-small & medium**

Са само 7 милијарди параметара, Phi-3-small побеђује GPT-3.5T на разним језичким, резоновањским, кодерским и математичким тестовима.

Phi-3-medium са 14 милијарди параметара наставља овај тренд и надмашује Gemini 1.0 Pro.

**Phi-3.5-mini**

Можемо га сматрати надоградњом Phi-3-mini. Иако параметри остају непромењени, побољшава способност подршке више језика (подршка за преко 20 језика: арапски, кинески, чешки, дански, холандски, енглески, фински, француски, немачки, хебрејски, мађарски, италијански, јапански, корејски, норвешки, пољски, португалски, руски, шпански, шведски, тајландски, турски, украјински) и додаје јачу подршку за дуги контекст.

Phi-3.5-mini са 3.8 милијарди параметара надмашује језичке моделе исте величине и упоредив је са моделима двоструко веће величине.

### Phi-3 / 3.5 Vision

Можемо Phi-3/3.5 Instruct назвати Phi-јевом способношћу за разумевање, док је Vision оно што Phi-ју даје очи да разуме свет.


**Phi-3-Vision**

Phi-3-vision, са само 4.2 милијарде параметара, наставља овај тренд и надмашује веће моделе као што су Claude-3 Haiku и Gemini 1.0 Pro V на општим задацима визуелног резоновања, OCR-у и разумевању табела и дијаграма.


**Phi-3.5-Vision**

Phi-3.5-Vision такође је надоградња Phi-3-Vision, додајући подршку за више слика. Можемо га сматрати побољшањем у виду, јер не можете само да видите слике, већ и видео записе.

Phi-3.5-vision надмашује веће моделе као што су Claude-3.5 Sonnet и Gemini 1.5 Flash у OCR-у, разумевању табела и графикона, и упоредив је у општем визуелном резоновању. Подржава унос више кадрова, тј. извођење резоновања на више улазних слика.


### Phi-3.5-MoE

***Mixture of Experts (MoE)*** омогућава моделима да буду преттренирани са далеко мањом рачунарском снагом, што значи да можете драматично повећати величину модела или скупа података у оквиру истог буџета рачунарских ресурса као код густог (dense) модела. Посебно, MoE модел би требао постићи исту квалитету као густ модел много брже током преттренирања.

Phi-3.5-MoE обухвата 16 модула стручњака од по 3.8 милијарди параметара. Phi-3.5-MoE са само 6.6 милијарди активних параметара постиже сличан ниво резоновања, разумевања језика и математике као још већи модели.

Можемо користити модел из Phi-3/3.5 породице заснован на различитим сценаријима. За разлику од LLM, Phi-3/3.5-mini или Phi-3/3.5-Vision можете распоредити на edge уређајима.

### Покретање Phi-3/3.5 Локално
Инференција у вези са Phi-3, или било којим језичким моделом као што је GPT-3, односи се на процес генерисања одговора или предвиђања на основу уноса који прими. Када дате упит или питање Phi-3, он користи своју тренирану неуронску мрежу да изведе највероватнији и најрелевантнији одговор анализом образаца и односа у подацима на којима је трениран.

**Hugging Face трансформер**
Hugging Face Transformers је моћна библиотека дизајнирана за обраду природног језика (NLP) и друге задатке машинског учења. Ево неколико кључних тачака о њој:

1. **Предтренирани модели**: Пружа хиљаде предтренираних модела који се могу користити за разне задатке као што су класификација текста, препознавање именованих ентитета, одговарање на питања, сумирање, превођење и генерисање текста.

2. **Интероперабилност оквира:** Библиотека подржава више оквира за дубоко учење, укључујући PyTorch, TensorFlow и JAX. Ово омогућава тренирање модела у једном оквиру и коришћење у другом.

3. **Мултимодалне могућности:** Поред NLP-а, Hugging Face Transformers подржава и задатке у рачунарском виду (нпр. класификација слика, детекција објеката) и обради аудија (нпр. препознавање говора, класификација аудија).

4. **Лакоћа коришћења:** Библиотека нуди API-је и алате за лако преузимање и фину-тјунинг модела, чинећи је приступачном како за почетнике тако и за експерте.

5. **Заједница и ресурси:** Hugging Face има живу заједницу и обимну документацију, туторијале и водиче који помажу корисницима да започну и максимално искористе библиотеку.
[службена документација](https://huggingface.co/docs/transformers/index?WT.mc_id=academic-105485-koreyst) или њихов [GitHub репозиторијум](https://github.com/huggingface/transformers?WT.mc_id=academic-105485-koreyst).

Ово је најчешће коришћена метода, али такође захтева GPU акцелерацију. На крају крајева, сценарији као што су Vision и MoE захтевају много прорачуна, што ће на CPU-у бити веома споро ако нису квантовани.

**Ollama**
[Ollama](https://ollama.com/?WT.mc_id=academic-105485-koreyst) је платформа дизајнирана да олакша локално покретање великих језичких модела (LLM) на вашем уређају. Подржава различите моделе као што су Llama 3.1, Phi 3, Mistral, и Gemma 2, између осталих. Платформа поједностављује процес паковањем тежина модела, конфигурације и података у један пакет, чинећи га приступачнијим за кориснике да прилагоде и креирају своје моделе. Ollama је доступна за macOS, Linux и Windows. Одличан је алат ако желите да експериментишете или примените LLM без ослањања на облачне сервисе. Ollama је најдиректнији начин, потребно је само да извршите следећу команду.


```bash

ollama run phi3.5

```

**Foundry Local**

[Foundry Local](https://foundrylocal.ai?WT.mc_id=academic-105485-koreyst) је Microsoft-ов оффлајн, на уређају runtime за покретање модела као што је Phi у потпуности на вашем хардверу - без претплате на Azure, API кључа или мрежне везе. Аутоматски бира најбољег извршног провајдера доступног (NPU, GPU или CPU) и излаже endpoint компатибилан са OpenAI-јем, тако да постојећи код из `openai`/Azure AI Inference SDK може да га користи са минималним изменама. Погледајте [Foundry Local документацију](https://learn.microsoft.com/azure/ai-foundry/foundry-local/get-started?WT.mc_id=academic-105485-koreyst) за почетак.

```bash

winget install Microsoft.FoundryLocal
foundry model run phi-3.5-mini

```

Или користите SDK директно у Python-у:

```bash

pip install foundry-local-sdk
```

**ONNX Runtime за GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) је крос-платформски акцелератор за inference и тренирање машинског учења. ONNX Runtime за Generative AI (GENAI) је моћан алат који вам помаже да ефикасно покрећете generative AI моделе на разним платформама.

## Шта је ONNX Runtime?
ONNX Runtime је пројекат отвореног кода који омогућава извођење inference-а машинског учења високе перформансе. Подржава моделе у Open Neural Network Exchange (ONNX) формату, који је стандард за представљање модела машинског учења. ONNX Runtime inference може омогућити бржа корисничка искуства и ниже трошкове, подржавајући моделе из оквира за дубоко учење као што су PyTorch и TensorFlow/Keras, као и класичне библиотеке машинског учења као што су scikit-learn, LightGBM, XGBoost итд. ONNX Runtime је компатибилан са различитим хардвером, драјверима и оперативним системима, и пружа оптималне перформансе коришћењем хардверских акцелератора где је могуће заједно са оптимизацијама графова и трансформацијама.

## Шта је Generative AI?
Generative AI се односи на AI системе који могу генерисати нови садржај, као што су текст, слике или музика, на основу података на којима су тренирани. Примери укључују језичке моделе као што је GPT-3 и моделе за генерисање слика као што је Stable Diffusion. ONNX Runtime за GenAI библиотека пружа generative AI петљу за ONNX моделе, укључујући inference са ONNX Runtime-ом, процесирање логитс-а, претрагу и узорковање, као и управљање KV кешом.

## ONNX Runtime за GENAI
ONNX Runtime за GENAI проширује могућности ONNX Runtime-а за подршку generative AI модела. Ево неких кључних карактеристика:

- **Широка подршка платформи:** Ради на разним платформама, укључујући Windows, Linux, macOS, Android и iOS.
- **Подршка модела:** Подржава многе популарне generative AI моделе, као што су LLaMA, GPT-Neo, BLOOM и други.
- **Оптимизација перформанси:** Укључује оптимизације за различите хардверске акцелераторе као што су NVIDIA GPU-ови, AMD GPU-ови и други2.
- **Лакоћа коришћења:** Пружа API-је за лаку интеграцију у апликације, омогућавајући генерисање текста, слика и другог садржаја уз минималан код.
- Корисници могу позвати high level generate() метод, или покретати по једну итерацију модела у петљи, генеришући по један токен у времену, и по потреби ажурирати параметре генерације унутар петље.
- ONNX runtime такође подржава greedy/beam претрагу и TopP, TopK узорковање за генерисање секвенци токена и уграђено процесирање логитс-а као што су казне за понављање. Лако је додати и прилагођено оцењивање.

## Почетак рада
Да бисте почели са ONNX Runtime за GENAI, пратите следеће кораке:

### Инсталирајте ONNX Runtime:
```Python
pip install onnxruntime
```
### Инсталирајте Generative AI екстензије:
```Python
pip install onnxruntime-genai
```

### Покрени модел: Ево једноставног примера у Python-у:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### Демо: Коришћење ONNX Runtime GenAI за позив Phi-3.5-Vision


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
