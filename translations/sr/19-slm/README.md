``` **ONNX Runtime за GenAI** [ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) је интерплатформски акцелератор за инференцију и тренинг машинског учења. ONNX Runtime за Generative AI (GENAI) је моћан алат који вам помаже да ефикасно покрећете генеративне AI моделе на различитим платформама. ## Шта је ONNX Runtime? ONNX Runtime је отворени пројекат који омогућава високоперформансну инференцију машинских модела. Подржава моделе у Open Neural Network Exchange (ONNX) формату, који је стандард за представљање машинских модела. ONNX Runtime инференција може омогућити брже корисничко искуство и ниже трошкове, подржавајући моделе из оквира дубоког учења као што су PyTorch и TensorFlow/Keras, као и класичне библиотеке машинског учења као што су scikit-learn, LightGBM, XGBoost, итд. ONNX Runtime је компатибилан са различитим хардвером, драјверима и оперативним системима, и пружа оптималне перформансе користећи хардверске акцелераторе где је применљиво уз графичке оптимизације и трансформације ## Шта је Generative AI? Generative AI се односи на AI системе који могу генерисати нови садржај, као што су текст, слике или музика, на основу података на којима су обучени. Примери укључују језичке моделе као GPT-3 и моделе за генерисање слика као Stable Diffusion. ONNX Runtime за GenAI библиотека пружа генеративни AI циклус за ONNX моделе, укључујући инференцију са ONNX Runtime-ом, обраду логита, претрагу и узорковање, и управљање KV кешом. ## ONNX Runtime за GENAI ONNX Runtime за GENAI проширује могућности ONNX Runtime-а да подржи генеративне AI моделе. Ево неких кључних карактеристика: - **Широка подршка платформама:** Ради на различитим платформама, укључујући Windows, Linux, macOS, Android и iOS. - **Подршка модела:** Подржава многе популарне генеративне AI моделе, као што су LLaMA, GPT-Neo, BLOOM, и више. - **Оптимизација перформанси:** Укључује оптимизације за различите хардверске акцелераторе као што су NVIDIA GPU-ови, AMD GPU-ови, и више2. - **Лакоћа коришћења:** Пружа API-је за лаку интеграцију у апликације, омогућавајући вам да генеришете текст, слике и други садржај са минималним кодом - Корисници могу позвати висок ниво generate() метод, или покренути сваку итерацију модела у петљи, генеришући један токен у исто време, и опционално ажурирајући параметре генерисања унутар петље. - ONNX runtime такође има подршку за алчну/beam претрагу и TopP, TopK узорковање за генерисање секвенци токена и уграђену обраду логита као казне за понављање. Можете такође лако додати прилагођено оцењивање. ## Почетак Да започнете са ONNX Runtime за GENAI, можете следити ове кораке: ### Инсталирајте ONNX Runtime: ```Python
pip install onnxruntime
``` ### Инсталирајте Extensions за Generative AI: ```Python

``` ### Покрените модел: Ево једноставног примера у Python-у: ```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
``` ### Демонстрација: Коришћење ONNX Runtime GenAI за позивање Phi-3.5-Vision ```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    code += tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)
