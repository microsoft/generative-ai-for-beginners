## 4. Модернизација API-ја

### 4.1 Застарели API обрасци (Миграција завршена)

Сви Python и TypeScript **chat** примери су мигрирани са Chat Completions API на **Responses API** (`client.responses.create(...)` → `response.output_text`).

| Стари образац | Нови образац | Статус |
|-------------|-------------|--------|
| `openai.api_type = "azure"` / `AzureOpenAI()` (chat) | `OpenAI(base_url="<endpoint>/openai/v1/")` (Responses API) | Завршено |
| `openai.ChatCompletion.create()` / `client.chat.completions.create()` | `client.responses.create(input=...)` → `response.output_text` | Завршено |
| `@azure/openai` `OpenAIClient.getChatCompletions()` (TypeScript) | `openai` пакет `client.responses.create()` → `response.output_text` | Завршено |
| `df.append()` (pandas) | `pd.concat()` | Завршено |

> **Напомена:** Microsoft Foundry Models примери који користе `azure-ai-inference` / `@azure-rest/ai-inference` SDK (`client.complete()`) остају на Model Inference API-ју, који не подржава Responses API. `AzureOpenAI()` се намерно задржава тамо где је и даље валидан (ембеддинзи и генерисање слика).

### 4.2 Нове функције API-а за демонстрацију

1. **Структурирани излазни подаци** (OpenAI)

## 5. Побољшања инфраструктуре

### 5.1 Унапређења CI/CD

Имплементирано у [`.github/workflows/code-quality.yml`](../../../.github/workflows/code-quality.yml): Питхон проверу/форматирање кода (Ruff + Black) је **обавезно** за одржавани модул `shared/` и покреће се као **саветодавно** за остатак наставног плана, плус саветодавна провера ESLint за JavaScript/TypeScript. Илустративна основа била је:

```yaml
# .github/workflows/code-quality.yml
name: Code Quality

on: [push, pull_request]

jobs:
  python-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install ruff black mypy
      - run: ruff check .
      - run: black --check .

  js-lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
      - run: npm ci
      - run: npx eslint .
```

### 5.2 Безбедносно скенирање

Имплементирано у [`.github/workflows/security.yml`](../../../.github/workflows/security.yml): CodeQL анализа за Python и JavaScript/TypeScript (при пушу, pull request-у и недељном распореду), као и преглед зависности приликом pull request-а. Илустративна основа била је:

```yaml
# .github/workflows/security.yml
name: Security Scan

on: [push, pull_request]

jobs:
  codeql:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: javascript, python
      - uses: github/codeql-action/analyze@v3

  dependency-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/dependency-review-action@v4
```
