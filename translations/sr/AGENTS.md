# AGENTS.md

## Преглед пројекта

Ово спремиште садржи свеобухватан курикулум од 21 лекције који учи основе генеративне вештачке интелигенције и развој апликација. Курс је дизајниран за почетнике и обухвата све од основних појмова до израде апликација спремних за производњу.

**Кључне технологије:**
- Python 3.9+ са библиотекама: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript са Node.js и библиотекама: `openai` (Azure OpenAI преко v1 крајње тачке + Responses API), `@azure-rest/ai-inference` (Microsoft Foundry модели)
- Azure OpenAI сервис, OpenAI API и Microsoft Foundry модели (GitHub Models се повлаче крајем јула 2026)
- Jupyter нотебоок-ови за интерактивно учење
- Dev контейнер-и за доследно развојно окружење

**Структура спремишта:**
- 21 директоријум са нумерисаним лекцијама (00-21) који садрже README-ове, примере кода и задатке
- Вишеструке имплементације: примери у Python-у, TypeScript-у, и понекад .NET-у
- Директоријум за преводе са више од 40 језичких верзија
- Централизована конфигурација преко `.env` фајла (користити `.env.copy` као шаблон)

## Команде за подешавање

### Почетно подешавање спремишта

```bash
# Клонирај репозиторијум
git clone https://github.com/microsoft/generative-ai-for-beginners.git
cd generative-ai-for-beginners

# Копирај шаблон окружења
cp .env.copy .env
# Измени .env са својим API кључевима и крајњим тачкама
```

### Подешавање Python окружења

```bash
# Креирај виртуелно окружење
python3 -m venv venv

# Активирај виртуелно окружење
# На macOS/Linux:
source venv/bin/activate
# На Windows:
venv\Scripts\activate

# Инсталирај зависности
pip install -r requirements.txt
```

### Подешавање Node.js/TypeScript окружења

```bash
# Инсталирајте зависности на коренском нивоу (за алате за документацију)
npm install

# За примере TypeScript-а појединачних лекција, идите до конкретне лекције:
cd 06-text-generation-apps/typescript/recipe-app
npm install
```

### Подешавање Dev контејнера (препоручено)

Спремиште укључује `.devcontainer` конфигурацију за GitHub Codespaces или VS Code Dev Containers:

1. Отворите спремиште у GitHub Codespaces или VS Code са Dev Containers екстензијом
2. Dev контејнер ће аутоматски:
   - Инсталирати Python зависности из `requirements.txt`
   - Покренути post-create скрипту (`.devcontainer/post-create.sh`)
   - Подесити Jupyter кернел

## Радни ток развоја

### Променљиве окружења

Све лекције које захтевају приступ API-ју користе променљиве окружења дефинисане у `.env` фајлу:

- `OPENAI_API_KEY` - За OpenAI API
- `AZURE_OPENAI_API_KEY` - За Azure OpenAI у Microsoft Foundry (Azure OpenAI сервис је сада део Microsoft Foundry: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - URL крајње тачке Azure OpenAI (Foundry ресурс крајња тачка)
- `AZURE_OPENAI_DEPLOYMENT` - Име распореда Chat completion модела
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - Име распореда Embeddings модела
- `AZURE_OPENAI_API_VERSION` - Верзија API-ја (подразумевано: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - За Hugging Face моделе
- `AZURE_INFERENCE_ENDPOINT` - Крајња тачка Microsoft Foundry модела (каталог модела са више провајдера)
- `AZURE_INFERENCE_CREDENTIAL` - Microsoft Foundry Models API кључ (замењује повлачећи се `GITHUB_TOKEN`)

## Упутства за стил кода

### Python

- Користити `python-dotenv` за управљање променљивима окружења
- Увозити `openai` библиотеку за интеракцију са API-јем
- Користити `pylint` за линеарисање (неке примере имају `# pylint: disable=all` због једноставности)
- Следити PEP 8 конвенције именовања
- Чувати API акредитиве у `.env` фајлу, никада у коду

### TypeScript

- Користити `dotenv` пакет за променљиве окружења
- TypeScript конфигурација у `tsconfig.json` за сваку апликацију
- Користити `openai` пакет за Azure OpenAI (направити клијента ка `/openai/v1/` крајњој тачки и позивати `client.responses.create`); користити `@azure-rest/ai-inference` за Microsoft Foundry моделе
- Користити `nodemon` за развој са аутоматским поновним учитавањем
- Прво компајлирати: `npm run build` па `npm start`

### Опште конвенције

- Држати примере кода једноставним и едукативним
- Укључити коментаре који објашњавају кључне појмове
- Код сваке лекције треба бити самосталан и извршив
- Користити доследно именовање: `aoai-` префикс за Azure OpenAI, `oai-` за OpenAI API, `githubmodels-` за Microsoft Foundry моделе (задржан легаси префикс из доба GitHub Models)

## Тестирање и валидација

### Провере пре слања

Ово спремиште користи GitHub Actions за валидацију. Пре слања PR-ова:

1. **Проверити Markdown линкове**:
   ```bash
   # validate-markdown.yml радни ток проверава:
   # - Покварене релативне путање
   # - Недостајући идентификатори праћења на путевима
   # - Недостајући идентификатори праћења на URL-овима
   # - URL-ови са локализацијом земље
   # - Покварени спољашњи URL-ови
   ```

2. **Ручно тестирање**:
   - Тестирати Python примере: активирати вируелно окружење и покренути скрипте
   - Тестирати TypeScript примере: `npm install`, `npm run build`, `npm start`
   - Проверити да ли су променљиве окружења исправно подешене
   - Проверити да ли API кључеви раде са примерима кода

3. **Примери кода**:
   - Осигурати да сав код ради без грешака
   - Тестирати са оба Azure OpenAI и OpenAI API када је то применљиво
   - Проверити да ли примери раде са Microsoft Foundry моделима тамо где се подржавају

### Нема аутоматизованих тестова

Ово је едукативно спремиште фокусирано на туторијале и примере. Нема јединичних или интеграционих тестова за покретање. Валидација је углавном:
- Ручно тестирање примера кода
- GitHub Actions за Markdown проверу
- Заједнички преглед едукативног садржаја

## Решавање проблема

### Уобичајени проблеми

**Грешке приликом увоза у Python-у**:
- Осигурати да је виртуелно окружење активирано
- Покренути `pip install -r requirements.txt`
- Проверити да је Python верзија 3.9+

**Грешке при градњи у TypeScript-у**:
- Покренути `npm install` у конкретном апликационом директоријуму
- Проверити да Node.js верзија одговара
- Обрисати `node_modules` и поново инсталирати ако је потребно

**Грешке аутентификације API-ја**:
- Проверити да `.env` фајл постоји и садржи исправне вредности
- Проверити да API кључеви важе и нису истекли
- Осигурати да су URL-ови крајњих тачака исправни за ваш регион

**Недостају променљиве окружења**:
- Копирати `.env.copy` у `.env`
- Попунити све потребне вредности за лекцију на којој радите
- Поново покренути апликацију након ажурирања `.env` фајла
