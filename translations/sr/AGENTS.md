# AGENTS.md

## Преглед пројекта

Овај репозиторијум садржи свеобухватан курс од 21 лекције која учи основе Генеративне вештачке интелигенције и развој апликација. Курс је дизајниран за почетнике и обухвата све од основних концепата до изградње апликација спремних за продукцију.

**Кључне технологије:**
- Python 3.9+ са библиотекама: `openai`, `python-dotenv`, `tiktoken`, `azure-ai-inference`, `pandas`, `numpy`, `matplotlib`
- TypeScript/JavaScript са Node.js и библиотекама: `openai` (Azure OpenAI преко v1 ентпоинта + Responses API), `@azure-rest/ai-inference` (Microsoft Foundry модели)
- Azure OpenAI сервис, OpenAI API и Microsoft Foundry модели (GitHub модели ће бити укинути крајем јула 2026)
- Jupyter бележнице за интерактивно учење
- Dev контейјнери за конзистентно развојно окружење

**Структура репозиторијума:**
- 21 фасцикле са нумерисаним лекцијама (00-21) које садрже README, примере кода и задатке
- Више имплементација: Python, TypeScript, а понекад и .NET примери
- Директоријум са преводима на преко 40 језика
- Централизована конфигурација преко `.env` фајла (користити `.env.copy` као шаблон)

## Радни ток развоја

### Променљиве окружења

Све лекције које захтевају приступ API-ју користе променљиве окружења дефинисане у `.env`:

- `OPENAI_API_KEY` - За OpenAI API
- `AZURE_OPENAI_API_KEY` - За Azure OpenAI у Microsoft Foundry (Azure OpenAI Service је сада део Microsoft Foundry: https://ai.azure.com)
- `AZURE_OPENAI_ENDPOINT` - URL ентпоинта Azure OpenAI (Foundry ресурсни ентпоинт)
- `AZURE_OPENAI_DEPLOYMENT` - Име распоређеног модела за chat completion (подразумевано у курсу: `gpt-5-mini`)
- `AZURE_OPENAI_EMBEDDINGS_DEPLOYMENT` - Име распоређеног модела за embeddings (подразумевано у курсу: `text-embedding-3-small`)
- `AZURE_OPENAI_API_VERSION` - Верзија API-ја (подразумевано: `2024-10-21`)
- `HUGGING_FACE_API_KEY` - За Hugging Face моделе
- `AZURE_INFERENCE_ENDPOINT` - Ентпоинт Microsoft Foundry модела (мулти-провајдер каталог модела)
- `AZURE_INFERENCE_CREDENTIAL` - API кључ за Microsoft Foundry моделе (замењује укидајући `GITHUB_TOKEN`)
- `AZURE_INFERENCE_CHAT_MODEL` - Модел без резоновања (нпр. `Llama-3.3-70B-Instruct`) који се користи у примерима са `temperature`, јер модели са резоновањем не подржавају контролу узорковања

### Конвенције модела (важно)

- **Подразумевани chat модел је `gpt-5-mini`** - тренутни, неутакнути **модел са резоновањем**. Од 2026 старији „мини“ модели са способношћу temperature (`gpt-4o-mini`, `gpt-4.1-mini`) се *укидају*, па курс стандардује на GPT-5 породицу.
- **Модели са резоновањем не прихватају `temperature` и `top_p`**, и уместо `max_tokens` користе `max_output_tokens` (Responses API) / `max_completion_tokens` (chat completions). Не додајте `temperature`/`top_p`/`max_tokens` у узорцима који позивају `gpt-5-mini`.
- **За демонстрацију `temperature`**, узорци користе **Llama** модел (`Llama-3.3-70B-Instruct`) преко Microsoft Foundry Models ентпоинта (`AZURE_INFERENCE_CHAT_MODEL`). Уравнотежење модела са резоновањем се врши инжењерингом упита + контролама резоновања, а не параметрима узорковања.
- **Фајн-тунинг (лекција 18)** задржава `gpt-4.1-mini`: GPT-5 подржава само појачани фајн-тунинг (RFT), не и надгледани фајн-тунинг (SFT) приказан у тој лекцији.
- Лекције 20 (Mistral) и 21 (Meta) задржавају `temperature`/`max_tokens` јер циљају на Mistral/Llama моделе који их подржавају.

## Упутства за стил кода

### Python

- Користити `python-dotenv` за управљање променљивим окружења
- Увозити `openai` библиотеку за интеракцију са API-јем
- Користити `pylint` за анализу кода (неке примере имају `# pylint: disable=all` због поједностављења)
- Пратити PEP 8 конвенције именовања
- Складиштити API креденцијале у `.env` фајлу, никада у коду

### TypeScript

- Користити `dotenv` пакет за променљиве окружења
- Конфигурација TypeScript у `tsconfig.json` за сваку апликацију
- Користити `openai` пакет за Azure OpenAI (клијент треба да користи `/openai/v1/` ентпоинт и позива `client.responses.create`); користити `@azure-rest/ai-inference` за Microsoft Foundry моделе
- Користити `nodemon` за развој са аутоматским поновним покретањем
- Прво покренути `npm run build`, па онда `npm start`

### Опште конвенције

- Држати примере кода једноставним и образовним
- Укључити коментаре који објашњавају битне концепте
- Код сваке лекције треба да буде самосталан и извршив
- Користити доследно именовање: префикс `aoai-` за Azure OpenAI, `oai-` за OpenAI API, `githubmodels-` за Microsoft Foundry моделе (остављен из доба GitHub мoдела)

## Решавање проблема

### Чести проблеми

**Грешке приликом увоза у Python-у**:
- Осигурати да је виртуелно окружење активирано
- Покренути `pip install -r requirements.txt`
- Проверити да верзија Python-а буде 3.9+

**Грешке у TypeScript билд-у**:
- Покренути `npm install` у одговарајућем директоријуму апликације
- Проверити компатибилност Node.js верзије
- Очистити `node_modules` и поново инсталирати ако је потребно

**Грешке аутентификације API-ја**:
- Проверити да постоји `.env` фајл са исправним вредностима
- Проверити да API кључеви важе и нису истекли
- Осигурати да су URL-ови ентпоинта исправни за вашу регију

**Недостају променљиве окружења**:
- Копирати `.env.copy` као `.env`
- Попунити све потребне вредности за лекцију на којој радите
- Поново покренути апликацију након ажурирања `.env`
