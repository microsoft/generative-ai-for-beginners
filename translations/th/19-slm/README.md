# บทนำเกี่ยวกับโมเดลภาษาขนาดเล็กสำหรับ AI สร้างเนื้อหาสำหรับผู้เริ่มต้น
Generative AI เป็นสาขาที่น่าหลงใหลของปัญญาประดิษฐ์ที่เน้นการสร้างระบบที่สามารถสร้างเนื้อหาใหม่ได้ เนื้อหานี้อาจมีตั้งแต่ข้อความและภาพไปจนถึงดนตรีและแม้แต่วิสัยทัศน์เสมือนจริงทั้งหมด หนึ่งในแอปพลิเคชันที่น่าตื่นเต้นที่สุดของ generative AI คือในวงการโมเดลภาษา

## โมเดลภาษาขนาดเล็กคืออะไร?

โมเดลภาษาขนาดเล็ก (SLM) เป็นตัวแทนของเวอร์ชันที่ย่อส่วนลงของโมเดลภาษาขนาดใหญ่ (LLM) โดยใช้หลักการสถาปัตยกรรมและเทคนิคมากมายของ LLMs แต่มีการใช้คอมพิวเตอร์ทางคณิตศาสตร์ที่ลดลงอย่างมาก

SLMs เป็นกลุ่มย่อยของโมเดลภาษาออกแบบมาเพื่อสร้างข้อความที่เหมือนมนุษย์ แตกต่างจากโมเดลขนาดใหญ่ เช่น GPT-4, SLMs มีขนาดเล็กกว่าและมีประสิทธิภาพกว่า ทำให้เหมาะสมสำหรับแอปพลิเคชันที่มีข้อจำกัดด้านทรัพยากรคอมพิวเตอร์ แม้จะมีขนาดเล็ก แต่ก็ยังสามารถทำงานหลายประเภทได้ โดยปกติ SLMs ถูกสร้างขึ้นโดยการบีบอัดหรือกลั่นกรอง LLMs โดยมีเป้าหมายเพื่อรักษาฟังก์ชันการทำงานและความสามารถทางภาษาของโมเดลต้นฉบับในระดับมาก การลดขนาดโมเดลนี้ทำให้ความซับซ้อนโดยรวมลดลง ทำให้ SLM มีประสิทธิภาพมากขึ้นทั้งในการใช้หน่วยความจำและความต้องการทางคอมพิวเตอร์ แม้จะมีการปรับแต่งเหล่านี้ SLMs ก็ยังสามารถดำเนินงานในงานประมวลผลภาษาธรรมชาติ (NLP) ได้หลากหลาย:

- การสร้างข้อความ: สร้างประโยคหรือย่อหน้าที่สอดคล้องและเหมาะสมกับบริบท
- การเติมข้อความ: ทำนายและเติมประโยคตามคำถามที่ได้รับ
- การแปลภาษา: แปลงข้อความจากภาษาหนึ่งไปยังอีกภาษา
- การสรุป: ย่อข้อความยาวให้สั้นลงและเข้าใจง่าย

แม้จะมีข้อแลกเปลี่ยนบางอย่างในประสิทธิภาพหรือความลึกของความเข้าใจเมื่อเทียบกับโมเดลขนาดใหญ่กว่า

## โมเดลภาษาขนาดเล็กทำงานอย่างไร?
SLMs ถูกฝึกบนข้อมูลข้อความจำนวนมหาศาล ในระหว่างการฝึก พวกมันเรียนรู้รูปแบบและโครงสร้างของภาษา ทำให้สามารถสร้างข้อความที่ถูกต้องตามไวยากรณ์และเหมาะสมกับบริบท กระบวนการฝึกประกอบด้วย:

- การรวบรวมข้อมูล: รวบรวมชุดข้อมูลขนาดใหญ่ของข้อความจากแหล่งต่างๆ
- การเตรียมข้อมูล: ทำความสะอาดและจัดระเบียบข้อมูลให้เหมาะสำหรับการฝึก
- การฝึก: ใช้อัลกอริทึมการเรียนรู้ของเครื่องเพื่อสอนโมเดลให้เข้าใจและสร้างข้อความ
- การปรับแต่ง: ปรับโมเดลเพื่อเพิ่มประสิทธิภาพในงานเฉพาะ

การพัฒนา SLMs สอดคล้องกับความต้องการที่เพิ่มขึ้นสำหรับโมเดลที่สามารถนำไปใช้ในสภาพแวดล้อมที่มีทรัพยากรจำกัด เช่น อุปกรณ์เคลื่อนที่หรือแพลตฟอร์ม edge computing ซึ่ง LLM ขนาดเต็มอาจไม่เหมาะสมเนื่องจากความต้องการทรัพยากรที่สูง โดยมุ่งเน้นที่ประสิทธิภาพ SLMs จึงสร้างสมดุลระหว่างประสิทธิภาพกับการเข้าถึง ใช้งานได้กว้างขวางในหลายโดเมน

## ความแตกต่างระหว่างโมเดลภาษาขนาดใหญ่ (LLMs) กับโมเดลภาษาขนาดเล็ก (SLMs)

ทั้ง LLMs และ SLMs ถูกสร้างขึ้นบนพื้นฐานของหลักการเรียนรู้ด้วยความน่าจะเป็นโดยใช้แนวทางคล้ายกันในด้านการออกแบบสถาปัตยกรรม วิธีการฝึก กระบวนการสร้างข้อมูล และเทคนิคการประเมินโมเดล แต่มีหลายปัจจัยสำคัญที่ทำให้สองประเภทนี้แตกต่างกัน

## การใช้งานของโมเดลภาษาขนาดเล็ก

SLMs มีแอปพลิเคชันหลากหลาย รวมถึง:

- แชทบอท: ให้บริการลูกค้าและโต้ตอบกับผู้ใช้ในรูปแบบการสนทนา
- การสร้างเนื้อหา: ช่วยนักเขียนโดยสร้างไอเดียหรือร่างบทความทั้งหมด
- การศึกษา: ช่วยนักเรียนในการทำงานเขียนหรือเรียนรู้ภาษาใหม่
- การเข้าถึง: สร้างเครื่องมือสำหรับผู้ที่มีความบกพร่อง เช่น ระบบข้อความเป็นเสียง

**ขนาด**
  
ความแตกต่างหลักระหว่าง LLMs และ SLMs อยู่ที่ขนาดของโมเดล LLMs เช่น ChatGPT (GPT-4) อาจประกอบด้วยพารามิเตอร์ประมาณ 1.76 ล้านล้าน ในขณะที่ SLMs แบบเปิดเช่น Mistral 7B ถูกออกแบบให้มีพารามิเตอร์จำนวนน้อยมาก ประมาณ 7 พันล้าน ความแตกต่างนี้หลัก ๆ มาจากความแตกต่างในสถาปัตยกรรมโมเดลและกระบวนการฝึก เช่น ChatGPT ใช้กลไก self-attention ในกรอบ encoder-decoder ในขณะที่ Mistral 7B ใช้ sliding window attention ซึ่งทำให้การฝึกในโมเดลแบบ decoder-only มีประสิทธิภาพมากขึ้น ความแตกต่างในสถาปัตยกรรมนี้มีผลกระทบอย่างลึกซึ้งต่อความซับซ้อนและประสิทธิภาพของโมเดลเหล่านี้

**ความเข้าใจ**

SLMs มักได้รับการปรับแต่งเพื่อประสิทธิภาพในโดเมนเฉพาะ ทำให้มีความชำนาญสูงแต่มีข้อจำกัดในความสามารถที่จะให้ความเข้าใจบริบทกว้าง ๆ ในหลายด้าน ขณะที่ LLMs มีเป้าหมายเพื่อจำลองสติปัญญาเหมือนมนุษย์ในระดับที่ครอบคลุมมากกว่า โดยผ่านการฝึกบนชุดข้อมูลที่หลากหลายและกว้าง LLMs ถูกออกแบบมาเพื่อทำงานได้ดีในหลายโดเมน มีความยืดหยุ่นและปรับตัวสูง ผลคือ LLMs เหมาะกับงาน downstream หลากหลายประเภท เช่น การประมวลผลภาษาธรรมชาติและการเขียนโปรแกรม

**การคำนวณ**

การฝึกและใช้งาน LLMs เป็นกระบวนการที่ใช้ทรัพยากรอย่างมาก มักต้องการโครงสร้างพื้นฐานการประมวลผลขนาดใหญ่ รวมถึงคลัสเตอร์ GPU ขนาดใหญ่ ตัวอย่างเช่น การฝึกโมเดลเช่น ChatGPT ตั้งแต่ต้นอาจต้องใช้ GPU หลายพันตัวในระยะเวลายาวนาน ในขณะที่ SLMs ที่มีพารามิเตอร์น้อยกว่า มีความเข้าถึงได้มากกว่าในแง่ของทรัพยากรการประมวลผล โมเดลเช่น Mistral 7B สามารถฝึกและรันบนเครื่องท้องถิ่นที่มี GPU ที่มีความสามารถระดับปานกลางได้ ถึงแม้ว่าการฝึกยังคงต้องใช้เวลาหลายชั่วโมงกับหลาย GPU ก็ตาม

**อคติ**

อคติเป็นปัญหาที่รู้จักกันใน LLMs โดยส่วนใหญ่เกิดจากลักษณะของข้อมูลฝึก โมเดลเหล่านี้มักพึ่งพาข้อมูลดิบที่เปิดเผยจากอินเทอร์เน็ต ซึ่งอาจขาดการแทนกลุ่มบางกลุ่มหรือให้ข้อมูลที่ไม่ถูกต้อง รวมถึงการติดป้ายกำกับผิดพลาด หรือสะท้อนอคติทางภาษาอันเนื่องมาจากสำเนียง ต่างถิ่น ภูมิศาสตร์ และกฎไวยากรณ์ นอกจากนี้ความซับซ้อนของสถาปัตยกรรม LLM อาจทำให้อคติทวีความรุนแรงโดยไม่ถูกสังเกตหากไม่มีการปรับแต่งอย่างระมัดระวัง ในทางตรงกันข้าม SLMs ซึ่งถูกฝึกบนชุดข้อมูลจำกัดและเฉพาะโดเมน มีแนวโน้มอ่อนแอต่ออคติน้อยกว่า แม้ว่าจะไม่ปลอดภัยจากอคติอย่างสมบูรณ์

**การอนุมาน**

ขนาดที่ลดลงของ SLMs ทำให้มีข้อได้เปรียบที่สำคัญในเรื่องความเร็วในการอนุมาน ช่วยให้พวกมันสร้างผลลัพธ์ได้อย่างมีประสิทธิภาพบนฮาร์ดแวร์ท้องถิ่นโดยไม่ต้องใช้การประมวลผลแบบขนานจำนวนมาก ในทางตรงกันข้าม LLMs ด้วยขนาดและความซับซ้อน มีความต้องการทรัพยากรการประมวลผลแบบขนานจำนวนมากเพื่อให้ได้เวลาตอบสนองที่ยอมรับได้ การมีผู้ใช้หลายคนพร้อมกันยิ่งทำให้เวลาตอบสนองของ LLMs ช้าลงโดยเฉพาะเมื่อใช้งานในระดับขนาดใหญ่

สรุปว่า แม้ว่า LLMs และ SLMs จะมีพื้นฐานร่วมกันในการเรียนรู้ของเครื่อง แต่พวกมันแตกต่างกันอย่างมากในแง่ของขนาดโมเดล ความต้องการทรัพยากร ความเข้าใจบริบท ความเปราะบางต่ออคติ และความเร็วในการอนุมาน ความแตกต่างเหล่านี้สะท้อนถึงความเหมาะสมในการใช้งานที่แตกต่างกัน โดย LLMs มีความยืดหยุ่นมากกว่าแต่ใช้ทรัพยากรมาก ในขณะที่ SLMs เสนอประสิทธิภาพในโดเมนเฉพาะด้วยความต้องการทางคอมพิวเตอร์ที่ลดลง

***หมายเหตุ: ในบทเรียนนี้ เราจะแนะนำ SLM โดยใช้ Microsoft Phi-3 / 3.5 เป็นตัวอย่าง***

## แนะนำตระกูล Phi-3 / Phi-3.5

ตระกูล Phi-3 / 3.5 มุ่งเน้นไปที่สถานการณ์แอปพลิเคชันด้านข้อความ ภาพ และ Agent (MoE):

### Phi-3 / 3.5 Instruct

เน้นการสร้างข้อความ การเติมข้อความ และการสกัดข้อมูลเนื้อหา เป็นต้น

**Phi-3-mini**

โมเดลภาษาขนาด 3.8B ใช้งานได้บน Microsoft Foundry, Hugging Face และ Ollama โมเดล Phi-3 ให้ผลลัพธ์ที่เหนือกว่าโมเดลภาษาในขนาดที่เท่ากันและใหญ่กว่าบนมาตรฐานสำคัญ ๆ (ดูตัวเลขมาตรฐานด้านล่าง ยิ่งตัวเลขสูงยิ่งดี) Phi-3-mini ทำคะแนนได้ดีกว่าโมเดลที่มีขนาดสองเท่า ในขณะที่ Phi-3-small และ Phi-3-medium ทำคะแนนเหนือกว่าโมเดลที่ใหญ่กว่า รวมถึง GPT-3.5

**Phi-3-small & medium**

ด้วยพารามิเตอร์เพียง 7B, Phi-3-small ชนะ GPT-3.5T ในมาตรฐานต่าง ๆ ด้านภาษา การให้เหตุผล การเขียนโค้ด และคณิตศาสตร์

Phi-3-medium ที่มีพารามิเตอร์ 14B ต่อเนื่องแนวโน้มนี้และทำคะแนนเหนือกว่า Gemini 1.0 Pro

**Phi-3.5-mini**

เราสามารถถือมันเป็นการอัพเกรดของ Phi-3-mini แม้ว่าพารามิเตอร์จะไม่เปลี่ยนแปลง แต่มันปรับปรุงความสามารถในการรองรับหลายภาษา (รองรับ 20+ ภาษา: อาหรับ จีน เช็ก เดนมาร์ก ดัตช์ อังกฤษ ฟินแลนด์ ฝรั่งเศส เยอรมัน ฮีบรู ฮังกาเรียน อิตาลี ญี่ปุ่น เกาหลี นอร์เวย์ โปแลนด์ โปรตุเกส รัสเซีย สเปน สวีเดน ไทย ตุรกี ยูเครน) และเพิ่มการรองรับบริบทยาวได้อย่างแข็งแกร่งขึ้น

Phi-3.5-mini ที่มีพารามิเตอร์ 3.8B ทำคะแนนเหนือกว่าโมเดลภาษาขนาดเดียวกันและเทียบเท่ากับโมเดลที่มีขนาดสองเท่า

### Phi-3 / 3.5 Vision

เราสามารถมองว่าโมเดล Instruct ของ Phi-3/3.5 เป็นความสามารถด้านความเข้าใจของ Phi และ Vision คือสิ่งที่ทำให้ Phi มี "ตา" เพื่อเข้าใจโลก


**Phi-3-Vision**

Phi-3-vision ที่มีพารามิเตอร์เพียง 4.2B ยังคงแนวโน้มนี้และทำคะแนนเหนือกว่าโมเดลที่ใหญ่กว่า เช่น Claude-3 Haiku และ Gemini 1.0 Pro V ในงานการให้เหตุผลเชิงภาพทั่วไป OCR และงานความเข้าใจตารางและแผนภูมิ


**Phi-3.5-Vision**

Phi-3.5-Vision เป็นการอัพเกรดของ Phi-3-Vision เพิ่มการรองรับภาพหลายภาพ คุณสามารถมองว่ามันเป็นการปรับปรุงทางด้านวิสัยทัศน์ ไม่เพียงแต่สามารถเห็นภาพนิ่งได้ แต่ยังเห็นวิดีโอด้วย

Phi-3.5-vision ทำคะแนนเหนือกว่าโมเดลที่ใหญ่กว่า เช่น Claude-3.5 Sonnet และ Gemini 1.5 Flash ในงาน OCR งานเข้าใจตารางและแผนภูมิ และเทียบเท่าในการให้เหตุผลความรู้เชิงภาพทั่วไป รองรับการป้อนข้อมูลหลายเฟรม คือสามารถให้เหตุผลจากภาพหลายภาพพร้อมกัน


### Phi-3.5-MoE

***Mixture of Experts(MoE)*** ช่วยให้โมเดลถูกฝึกล่วงหน้าด้วยการใช้คำนวณน้อยมาก ซึ่งหมายความว่าคุณสามารถขยายขนาดโมเดลหรือชุดข้อมูลได้อย่างมากโดยใช้งบประมาณการคำนวณเท่ากับโมเดลแน่นทั่วไป โดยเฉพาะอย่างยิ่ง โมเดล MoE ควรให้คุณภาพเทียบเท่าโมเดลแน่นในเวลาเร็วกว่าในการฝึกล่วงหน้า

Phi-3.5-MoE ประกอบด้วยโมดูลผู้เชี่ยวชาญ 16x3.8B Phi-3.5-MoE ที่มีพารามิเตอร์แอคทีฟเพียง 6.6B ให้ระดับของการให้เหตุผล ความเข้าใจภาษา และคณิตศาสตร์ที่ใกล้เคียงกับโมเดลที่ใหญ่กว่ามาก

เราสามารถใช้โมเดลตระกูล Phi-3/3.5 ตามสถานการณ์ต่างๆ แตกต่างจาก LLM สามารถติดตั้ง Phi-3/3.5-mini หรือ Phi-3/3.5-Vision บนอุปกรณ์ edge ได้

- **การปรับขนาด:** NIM รองรับการปรับขนาดอัตโนมัติบน Kubernetes ช่วยให้จัดการภาระงานที่หลากหลายได้อย่างมีประสิทธิภาพ

**Hugging Face Transformer**
Hugging Face Transformers คือไลบรารีอันทรงพลังที่ออกแบบมาสำหรับการประมวลผลภาษาธรรมชาติ (NLP) และงานการเรียนรู้เครื่องอื่น ๆ นี่คือจุดสำคัญบางประการเกี่ยวกับมัน:

1. **โมเดลที่ผ่านการฝึกล่วงหน้า**: มีโมเดลที่ผ่านการฝึกล่วงหน้าหลายพันแบบที่สามารถใช้กับงานหลากหลาย เช่น การจำแนกข้อความ การรู้จำชื่อเฉพาะ คำถาม-คำตอบ การสรุป การแปล และการสร้างข้อความ

2. **ความเข้ากันได้กับเฟรมเวิร์กหลายแบบ**: ไลบรารีรองรับเฟรมเวิร์กการเรียนรู้ลึกหลากหลาย เช่น PyTorch, TensorFlow, และ JAX ทำให้สามารถฝึกโมเดลในเฟรมเวิร์กหนึ่งและใช้ในอีกเฟรมเวิร์กหนึ่งได้

3. **ความสามารถด้านมัลติโหมด**: นอกจาก NLP แล้ว Hugging Face Transformers ยังรองรับงานในด้านคอมพิวเตอร์วิชัน (เช่น การจำแนกรูปภาพ การตรวจจับวัตถุ) และการประมวลผลเสียง (เช่น การรู้จำเสียง การจำแนกเสียง)

4. **ใช้งานง่าย**: ไลบรารีเสนอตัว API และเครื่องมือที่ง่ายต่อการดาวน์โหลดและปรับแต่งโมเดล ทำให้เข้าถึงได้ทั้งผู้เริ่มต้นและผู้เชี่ยวชาญ

5. **ชุมชนและทรัพยากร**: Hugging Face มีชุมชนที่มีชีวิตชีวาและเอกสารประกอบอย่างครอบคลุม พร้อมบทช่วยสอนและคู่มือเพื่อช่วยผู้ใช้เริ่มต้นและใช้ไลบรารีได้เต็มประสิทธิภาพ
[เอกสารทางการ](https://huggingface.co/docs/transformers/index?WT.mc_id=academic-105485-koreyst) หรือที่ [GitHub repository](https://github.com/huggingface/transformers?WT.mc_id=academic-105485-koreyst)

**ONNX Runtime for GenAI**

[ONNX Runtime](https://github.com/microsoft/onnxruntime-genai?WT.mc_id=academic-105485-koreyst) คือเครื่องมือเร่งการอนุมานและการฝึกสอนการเรียนรู้เครื่องข้ามแพลตฟอร์ม ONNX Runtime สำหรับ Generative AI (GENAI) เป็นเครื่องมือทรงพลังที่ช่วยให้คุณรันโมเดล AI สร้างสรรค์ได้อย่างมีประสิทธิภาพบนหลายแพลตฟอร์ม

## ONNX Runtime คืออะไร?
ONNX Runtime คือโครงการโอเพนซอร์สที่ช่วยให้อนุมานโมเดลการเรียนรู้เครื่องด้วยประสิทธิภาพสูง รองรับโมเดลในรูปแบบ Open Neural Network Exchange (ONNX) ซึ่งเป็นมาตรฐานสำหรับแทนโมเดลการเรียนรู้เครื่อง ONNX Runtime สามารถช่วยให้ประสบการณ์ผู้ใช้เร็วขึ้นและลดต้นทุน รองรับโมเดลจากเฟรมเวิร์กการเรียนรู้ลึก เช่น PyTorch และ TensorFlow/Keras รวมถึงไลบรารีแมชชีนเลิร์นนิงแบบคลาสสิก เช่น scikit-learn, LightGBM, XGBoost เป็นต้น ONNX Runtime เข้ากันได้กับฮาร์ดแวร์ ไดรเวอร์ และระบบปฏิบัติการต่าง ๆ และให้ประสิทธิภาพสูงสุดโดยใช้ฮาร์ดแวร์เร่งการประมวลผลร่วมกับการปรับแต่งกราฟและการแปลง

## Generative AI คืออะไร?
Generative AI หมายถึงระบบ AI ที่สามารถสร้างเนื้อหาใหม่ เช่น ข้อความ รูปภาพ หรือดนตรี โดยอิงจากข้อมูลที่ได้รับการฝึก ตัวอย่างเช่น โมเดลภาษาอย่าง GPT-3 และโมเดลสร้างรูปภาพเช่น Stable Diffusion ไลบรารี ONNX Runtime สำหรับ GenAI ให้วงจรการสร้างเนื้อหา AI สำหรับโมเดล ONNX รวมถึงการอนุมานด้วย ONNX Runtime, การประมวลผล logits, การค้นหาและการสุ่มตัวอย่าง, และการจัดการแคช KV

## ONNX Runtime สำหรับ GENAI
ONNX Runtime สำหรับ GENAI ขยายขีดความสามารถของ ONNX Runtime เพื่อรองรับโมเดล generative AI นี่คือคุณสมบัติสำคัญบางประการ:

- **รองรับแพลตฟอร์มกว้าง:** ทำงานบนแพลตฟอร์มต่าง ๆ รวมถึง Windows, Linux, macOS, Android และ iOS
- **รองรับโมเดล:** รองรับโมเดล generative AI ที่เป็นที่นิยม เช่น LLaMA, GPT-Neo, BLOOM และอื่น ๆ
- **การปรับประสิทธิภาพ:** รวมการปรับแต่งสำหรับฮาร์ดแวร์เร่งความเร็วต่าง ๆ เช่น NVIDIA GPUs, AMD GPUs และอื่น ๆ
- **ใช้งานง่าย:** มี API สำหรับการผนวกเข้ากับแอปพลิเคชันได้ง่าย ช่วยให้คุณสร้างข้อความ รูปภาพ และเนื้อหาอื่น ๆ ด้วยโค้ดน้อยที่สุด
- ผู้ใช้สามารถเรียกใช้เมธอด generate() ในระดับสูง หรือรันแต่ละรอบของโมเดลในลูป เพื่อสร้างโทเค็นทีละตัว และปรับพารามิเตอร์การสร้างภายในลูปได้ตามต้องการ
- ONNX runtime ยังรองรับการค้นหาแบบ greedy/beam และการสุ่มแบบ TopP, TopK เพื่อสร้างลำดับโทเค็น รวมถึงการประมวลผล logits ในตัวเช่นการลงโทษการทำซ้ำ คุณสามารถเพิ่มการให้คะแนนแบบกำหนดเองได้ง่าย ๆ ด้วย

## การเริ่มต้น
เพื่อเริ่มใช้งาน ONNX Runtime สำหรับ GENAI คุณสามารถทำตามขั้นตอนเหล่านี้:

### ติดตั้ง ONNX Runtime:
```Python
pip install onnxruntime
```
### ติดตั้งการขยาย Generative AI:
```Python
pip install onnxruntime-genai
```

### รันโมเดล: ตัวอย่างง่าย ๆ ใน Python:
```Python
import onnxruntime_genai as og

model = og.Model('path_to_your_model.onnx')

tokenizer = og.Tokenizer(model)

input_text = "Hello, how are you?"

input_tokens = tokenizer.encode(input_text)

output_tokens = model.generate(input_tokens)

output_text = tokenizer.decode(output_tokens)

print(output_text) 
```
### สาธิต: การใช้ ONNX Runtime GenAI เรียก Phi-3.5-Vision


```python

import onnxruntime_genai as og

model_path = './Your Phi-3.5-vision-instruct ONNX Path'

img_path = './Your Image Path'

model = og.Model(model_path)

processor = model.create_multimodal_processor()

tokenizer_stream = processor.create_stream()

text = "Your Prompt"

prompt = "<|user|>\n"

prompt += "<|image_1|>\n"

prompt += f"{text}<|end|>\n"

prompt += "<|assistant|>\n"

image = og.Images.open(img_path)

inputs = processor(prompt, images=image)

params = og.GeneratorParams(model)

params.set_inputs(inputs)

params.set_search_options(max_length=3072)

generator = og.Generator(model, params)

while not generator.is_done():

    generator.compute_logits()
    
    generator.generate_next_token()

    new_token = generator.get_next_tokens()[0]
    
    output = tokenizer_stream.decode(new_token)
    
    print(tokenizer_stream.decode(new_token), end='', flush=True)

```
