# การปรับแต่งโมเดลภาษาใหญ่ของคุณ

การใช้โมเดลภาษาใหญ่เพื่อสร้างแอปพลิเคชัน AI สร้างสรรค์นั้นมาพร้อมกับความท้าทายใหม่ ๆ ประเด็นสำคัญคือการรับประกันคุณภาพคำตอบ (ความถูกต้องและความเกี่ยวข้อง) ในเนื้อหาที่สร้างขึ้นโดยโมเดลสำหรับคำขอของผู้ใช้ ในบทเรียนก่อนหน้า เราได้พูดถึงเทคนิคอย่างการออกแบบ prompt และการสร้างคำตอบด้วยการดึงข้อมูลมาเสริมที่พยายามแก้ปัญหาด้วยการ _ปรับเปลี่ยนข้อความ prompt_ ของโมเดลที่มีอยู่

ในบทเรียนวันนี้ เราจะพูดถึงเทคนิคที่สาม คือ **การปรับแต่งอย่างละเอียด (fine-tuning)** ซึ่งพยายามแก้ไขความท้าทายนี้โดยการ _ฝึกอบรมโมเดลใหม่เอง_ ด้วยข้อมูลเพิ่มเติม มาดูรายละเอียดกันเลย

## เป้าหมายการเรียนรู้

บทเรียนนี้แนะนำแนวคิดการปรับแต่งสำหรับโมเดลภาษาที่ได้รับการฝึกอบรมล่วงหน้า สำรวจประโยชน์และความท้าทายของแนวทางนี้ และให้คำแนะนำว่าควรใช้ fine-tuning เมื่อไรและอย่างไรเพื่อปรับปรุงประสิทธิภาพของโมเดล AI สร้างสรรค์ของคุณ

เมื่อจบบทเรียนนี้ คุณควรจะสามารถตอบคำถามต่อไปนี้ได้:

- การปรับแต่งสำหรับโมเดลภาษาใหญ่คืออะไร?
- ควรและทำไมควรทำการปรับแต่ง?
- ฉันจะปรับแต่งโมเดลที่ฝึกมาแล้วได้อย่างไร?
- ข้อจำกัดของการปรับแต่งมีอะไรบ้าง?

พร้อมหรือยัง? มาเริ่มกันเลย

## คู่มือภาพรวม

อยากเห็นภาพรวมว่าบทเรียนนี้จะครอบคลุมอะไรบ้างก่อนเริ่มไหม? ลองดูคู่มือภาพรวมนี้ซึ่งอธิบายเส้นทางการเรียนรู้ของบทเรียนนี้ — ตั้งแต่การเรียนรู้แนวคิดหลักและแรงจูงใจของการปรับแต่ง ไปจนถึงการเข้าใจกระบวนการและแนวทางปฏิบัติที่ดีที่สุดสำหรับการทำ fine-tuning งานนี้เป็นหัวข้อที่น่าสนใจในการสำรวจก็อย่าลืมดูที่หน้าทรัพยากร [Resources](./RESOURCES.md?WT.mc_id=academic-105485-koreyst) สำหรับลิงก์เพิ่มเติมที่ช่วยสนับสนุนการเรียนรู้ด้วยตัวเองของคุณ!

## การปรับแต่งโมเดลภาษาใหญ่คืออะไร?

โดยคำนิยาม โมเดลภาษาใหญ่ถูก _ฝึกล่วงหน้า_ ด้วยข้อความจำนวนมากมาจากแหล่งข้อมูลหลากหลายรวมถึงอินเทอร์เน็ต ตามที่เราได้เรียนรู้ในบทเรียนก่อนหน้า เราต้องใช้เทคนิคเช่น _การออกแบบ prompt_ และ _การเสริมการสร้างจากการดึงข้อมูล_ เพื่อเพิ่มคุณภาพคำตอบของโมเดลต่อคำถามของผู้ใช้ ("prompt")

เทคนิคการออกแบบ prompt ที่เป็นที่นิยมคือการให้คำแนะนำกับโมเดลมากขึ้นเกี่ยวกับสิ่งที่คาดหวังในคำตอบ ทั้งโดยการให้ _คำแนะนำที่ชัดเจน_ (guidance แบบชัดเจน) หรือ _ให้ตัวอย่างบางส่วน_ (guidance แบบแอบแฝง) เรียกกันว่า _few-shot learning_ แต่มีข้อจำกัดสองประการคือ:

- ขีดจำกัดของจำนวน token ในโมเดลจำกัดจำนวนตัวอย่างที่คุณสามารถให้และจำกัดประสิทธิภาพ
- ค่าใช้จ่ายของ token ของโมเดลทำให้การเพิ่มตัวอย่างในทุก prompt มีค่าใช้จ่ายสูงและจำกัดความยืดหยุ่น

การปรับแต่งคือการปฏิบัติที่แพร่หลายในระบบการเรียนรู้ของเครื่อง ที่เรานำโมเดลที่ฝึกแล้วมาและฝึกใหม่ด้วยข้อมูลชุดใหม่เพื่อปรับปรุงประสิทธิภาพสำหรับงานเฉพาะ ในบริบทของโมเดลภาษา เราสามารถปรับแต่งโมเดลที่ฝึกแล้วได้ _ด้วยชุดตัวอย่างที่คัดเลือกมาแล้วสำหรับงานหรือโดเมนแอปพลิเคชันเฉพาะ_ เพื่อสร้าง **โมเดลที่กำหนดเอง** ซึ่งอาจแม่นยำและเกี่ยวข้องมากขึ้นกับงานหรือโดเมนนั้น ๆ ข้อดีอีกอย่างของการปรับแต่งคือมันสามารถลดจำนวนตัวอย่างที่จำเป็นสำหรับ few-shot learning — ลดการใช้ token และค่าใช้จ่ายที่เกี่ยวข้อง

## ควรปรับแต่งโมเดลเมื่อไหร่และทำไม?

ใน _บริบทนี้_ เมื่อเราพูดถึงการปรับแต่ง เรากำลังหมายถึงการปรับแต่งแบบ **ควบคุมด้วยข้อมูล** (supervised fine-tuning) ที่การฝึกอบรมใหม่ทำโดย **เพิ่มข้อมูลใหม่** ที่ไม่ได้เป็นส่วนหนึ่งของชุดข้อมูลฝึกอบรมดั้งเดิม ซึ่งแตกต่างจากการปรับแต่งแบบไม่ควบคุม (unsupervised fine-tuning) ที่โมเดลถูกฝึกใหม่บนข้อมูลเดิมแต่ใช้พารามิเตอร์การปรับแต่งที่แตกต่างกัน

สิ่งสำคัญที่ต้องจำคือการปรับแต่งเป็นเทคนิคขั้นสูงที่ต้องการความเชี่ยวชาญระดับหนึ่งเพื่อให้ได้ผลลัพธ์ที่ต้องการ หากทำไม่ถูกวิธี อาจไม่ได้ผลลัพธ์ที่คาดหวัง และอาจทำให้ประสิทธิภาพของโมเดลสำหรับโดเมนเป้าหมายของคุณลดลง

ดังนั้น ก่อนที่คุณจะเรียนรู้ “วิธี” การปรับแต่งโมเดลภาษา คุณต้องรู้ “ทำไม” ถึงควรใช้วิธีนี้ และ “เมื่อไหร่” ที่ควรเริ่มกระบวนการปรับแต่ง เริ่มด้วยการถามตัวเองคำถามเหล่านี้:

- **กรณีการใช้งาน**: กรณีการใช้งานของคุณคืออะไรสำหรับการปรับแต่ง? คุณต้องการปรับปรุงอะไรในโมเดลที่ฝึกล่วงหน้า?
- **ทางเลือกอื่น**: คุณได้ลอง _เทคนิคอื่น_ เพื่อให้ได้ผลลัพธ์ที่ต้องการหรือยัง? ใช้วิธีเหล่านั้นเพื่อสร้างฐานในการเปรียบเทียบ
  - การออกแบบ prompt: ลองใช้เทคนิคแบบ few-shot prompting ด้วยตัวอย่างของคำตอบ prompt ที่เกี่ยวข้อง ประเมินคุณภาพคำตอบ
  - การเสริมการสร้างจากการดึงข้อมูล: ลองเสริม prompt ด้วยผลการค้นหาจากข้อมูลของคุณ ประเมินคุณภาพคำตอบ
- **ค่าใช้จ่าย**: คุณได้ประเมินค่าใช้จ่ายสำหรับการปรับแต่งหรือยัง?
  - ความสามารถในการปรับแต่ง - โมเดลที่ฝึกล่วงหน้ามีให้ปรับแต่งหรือไม่?
  - ความพยายาม - สำหรับการเตรียมข้อมูลฝึกอบรม การประเมินและปรับปรุงโมเดล
  - การคำนวณ - สำหรับการรันงานปรับแต่งและการนำโมเดลที่ปรับแล้วไปใช้งาน
  - ข้อมูล - การเข้าถึงตัวอย่างคุณภาพเพียงพอสำหรับผลกระทบจากการปรับแต่ง
- **ประโยชน์**: คุณได้ยืนยันประโยชน์ของการปรับแต่งหรือยัง?
  - คุณภาพ - โมเดลที่ปรับแต่งแล้วทำงานดีกว่าฐานหรือไม่?
  - ค่าใช้จ่าย - ช่วยลดการใช้ token โดยทำให้ prompt ง่ายขึ้นหรือไม่?
  - ความสามารถขยาย - คุณสามารถใช้โมเดลฐานซ้ำกับโดเมนใหม่ได้หรือไม่?

ด้วยการตอบคำถามเหล่านี้ คุณจะสามารถตัดสินใจได้ว่าการปรับแต่งเป็นแนวทางที่เหมาะสมสำหรับกรณีการใช้งานของคุณหรือไม่ โดยปกติแล้วแนวทางนี้จะเหมาะสมก็ต่อเมื่อประโยชน์มีมากกว่าค่าใช้จ่าย เมื่อคุณตัดสินใจดำเนินการแล้ว ก็ถึงเวลาคิดว่า _คุณจะทำการปรับแต่งโมเดลที่ฝึกล่วงหน้าอย่างไร_

อยากได้ข้อมูลเชิงลึกเพิ่มเติมเกี่ยวกับกระบวนการตัดสินใจไหม? ดูวิดีโอ [จะปรับแต่งหรือไม่ปรับแต่งดี](https://www.youtube.com/watch?v=0Jo-z-MFxJs)

## วิธีการปรับแต่งโมเดลที่ฝึกล่วงหน้า

ในการปรับแต่งโมเดลที่ฝึกล่วงหน้า คุณต้องมี:

- โมเดลที่ฝึกล่วงหน้าเพื่อนำมาปรับแต่ง
- ชุดข้อมูลสำหรับใช้ในการปรับแต่ง
- สภาพแวดล้อมการฝึกอบรมเพื่อรันงาน fine-tuning
- สภาพแวดล้อมการโฮสต์เพื่อปรับใช้โมเดลที่ปรับแต่งแล้ว
