# เฟรมเวิร์กเครือข่ายประสาท

อย่างที่เราได้เรียนรู้มาแล้ว เพื่อให้สามารถฝึกเครือข่ายประสาทได้อย่างมีประสิทธิภาพ เราต้องทำสองสิ่ง:

* ดำเนินการกับเทนเซอร์ เช่น การคูณ การบวก และคำนวณฟังก์ชันบางอย่าง เช่น sigmoid หรือ softmax
* คำนวณกราดิเอนต์ของทุกนิพจน์ เพื่อทำการปรับแต่งกราดิเอนต์เซนต์

ในขณะที่ไลบรารี `numpy` สามารถทำส่วนแรกได้ เราต้องการกลไกบางอย่างในการคำนวณกราดิเอนต์ ในเฟรมเวิร์กของเราที่เราได้พัฒนาในส่วนก่อนหน้านี้ เราต้องเขียนโปรแกรมฟังก์ชันอนุพันธ์ทั้งหมดด้วยตนเองภายในเมธอด `backward` ซึ่งทำการถอยหลังอัตโนมัติ อุดมคติคือเฟรมเวิร์กควรให้โอกาสเราในการคำนวณกราดิเอนต์ของ *ทุกนิพจน์* ที่เราสามารถกำหนดได้

อีกสิ่งสำคัญคือการสามารถดำเนินการคำนวณบน GPU หรือหน่วยคำนวณเฉพาะทางอื่นๆ เช่น TPU การฝึกเครือข่ายประสาทลึกต้องการการคำนวณ *จำนวนมาก* และการสามารถทำให้การคำนวณเหล่านั้นขนานบน GPU นั้นสำคัญมาก

> ✅ คำว่า 'parallelize' หมายถึงการกระจายการคำนวณไปยังอุปกรณ์หลายตัว

ปัจจุบันเฟรมเวิร์กเครือข่ายประสาทที่ได้รับความนิยมมากที่สุดสองตัวคือ: TensorFlow และ PyTorch ทั้งสองให้ API ระดับต่ำเพื่อดำเนินการกับเทนเซอร์ทั้งบน CPU และ GPU บน API ระดับต่ำยังมี API ระดับสูง ซึ่งเรียกว่า Keras และ PyTorch Lightning ตามลำดับ

API ระดับต่ำ | TensorFlow | PyTorch
--------------|-------------------------------------|--------------------------------
API ระดับสูง | Keras | PyTorch Lightning

**API ระดับต่ำ** ในทั้งสองเฟรมเวิร์กช่วยให้คุณสร้างสิ่งที่เรียกว่า **กราฟการคำนวณ** กราฟนี้กำหนดวิธีการคำนวณผลลัพธ์ (โดยปกติคือฟังก์ชันการสูญเสีย) ด้วยพารามิเตอร์นำเข้า และสามารถผลักดันให้คำนวณบน GPU หากมี ฟังก์ชันสำหรับทำอนุพันธ์ของกราฟการคำนวณนี้และคำนวณกราดิเอนต์ซึ่งสามารถใช้ในการปรับแต่งพารามิเตอร์ของโมเดลได้

**API ระดับสูง** มักจะพิจารณาเครือข่ายประสาทเป็น **ลำดับของชั้น** และทำให้การสร้างเครือข่ายประสาทส่วนใหญ่เป็นเรื่องง่ายขึ้นมาก การฝึกโมเดลมักจะต้องเตรียมข้อมูลแล้วเรียกใช้ฟังก์ชัน `fit` เพื่อทำงาน

API ระดับสูงช่วยให้คุณสร้างเครือข่ายประสาททั่วไปได้อย่างรวดเร็วโดยไม่ต้องกังวลเกี่ยวกับรายละเอียดมากมาย ในขณะเดียวกัน API ระดับต่ำให้การควบคุมมากขึ้นในกระบวนการฝึกฝน และดังนั้นจึงถูกใช้มากในการวิจัยเมื่อคุณกำลังจัดการกับสถาปัตยกรรมเครือข่ายประสาทใหม่

ยังเป็นสิ่งสำคัญที่จะเข้าใจว่าคุณสามารถใช้ทั้งสอง API ร่วมกันได้ เช่น คุณสามารถพัฒนาสถาปัตยกรรมชั้นเครือข่ายของคุณเองโดยใช้ API ระดับต่ำ แล้วใช้มันภายในเครือข่ายขนาดใหญ่ที่สร้างและฝึกฝนด้วย API ระดับสูง หรือคุณสามารถกำหนดเครือข่ายโดยใช้ API ระดับสูงเป็นลำดับของชั้น แล้วใช้ลูปการฝึกฝนระดับต่ำของคุณเองเพื่อทำการปรับแต่ง ทั้งสอง API ใช้แนวคิดพื้นฐานเดียวกัน และออกแบบมาให้ทำงานร่วมกันได้ดี

## การเรียนรู้

ในคอร์สนี้ เราเสนอเนื้อหาส่วนใหญ่ทั้งสำหรับ PyTorch และ TensorFlow คุณสามารถเลือกเฟรมเวิร์กที่คุณชอบและไปที่โน้ตบุ๊กที่สอดคล้องกันเท่านั้น หากคุณไม่แน่ใจว่าจะเลือกเฟรมเวิร์กใด อ่านการสนทนาบางอย่างบนอินเทอร์เน็ตเกี่ยวกับ **PyTorch vs. TensorFlow** คุณสามารถดูทั้งสองเฟรมเวิร์กเพื่อทำความเข้าใจให้ดีขึ้น

เมื่อเป็นไปได้ เราจะใช้ API ระดับสูงเพื่อความง่าย อย่างไรก็ตาม เราเชื่อว่าเป็นสิ่งสำคัญที่จะเข้าใจว่าเครือข่ายประสาททำงานจากพื้นฐานอย่างไร ดังนั้นในตอนเริ่มต้นเราจะเริ่มต้นด้วยการทำงานกับ API ระดับต่ำและเทนเซอร์ แต่หากคุณต้องการเริ่มต้นเร็วและไม่ต้องการใช้เวลามากในการเรียนรู้รายละเอียดเหล่านี้ คุณสามารถข้ามและไปยังโน้ตบุ๊ก API ระดับสูงได้เลย

## ✍️ แบบฝึกหัด: เฟรมเวิร์ก

ดำเนินการเรียนรู้ของคุณในโน้ตบุ๊กต่อไปนี้:

API ระดับต่ำ | โน้ตบุ๊ก TensorFlow+Keras | PyTorch
--------------|-------------------------------------|--------------------------------
API ระดับสูง | Keras | *PyTorch Lightning*

หลังจากเชี่ยวชาญเฟรมเวิร์กแล้ว มาทบทวนแนวคิดของ overfitting กัน

# Overfitting

Overfitting เป็นแนวคิดที่สำคัญอย่างยิ่งในการเรียนรู้ของเครื่อง และมันสำคัญมากที่จะเข้าใจให้ถูกต้อง!

พิจารณาปัญหาต่อไปนี้ของการประมาณจุด 5 จุด (แทนด้วย `x` ในกราฟด้านล่าง):

!linear | overfit
-------------------------|--------------------------
**โมเดลเชิงเส้น, 2 พารามิเตอร์** | **โมเดลไม่เชิงเส้น, 7 พารามิเตอร์**
ข้อผิดพลาดการฝึก = 5.3 | ข้อผิดพลาดการฝึก = 0
ข้อผิดพลาดการตรวจสอบ = 5.1 | ข้อผิดพลาดการตรวจสอบ = 20

* ทางซ้าย เราเห็นการประมาณเส้นตรงที่ดี เนื่องจากจำนวนพารามิเตอร์เหมาะสม โมเดลสามารถจับแนวคิดเบื้องหลังการกระจายจุดได้ถูกต้อง
* ทางขวา โมเดลทรงพลังเกินไป เนื่องจากเรามีเพียง 5 จุดและโมเดลมี 7 พารามิเตอร์ มันสามารถปรับให้ผ่านทุกจุดได้ ทำให้ข้อผิดพลาดการฝึกเป็น 0 อย่างไรก็ตาม สิ่งนี้ทำให้โมเดลไม่สามารถเข้าใจรูปแบบที่ถูกต้องเบื้องหลังข้อมูลได้ ดังนั้นข้อผิดพลาดการตรวจสอบจึงสูงมาก

มันสำคัญมากที่จะหาสมดุลที่ถูกต้องระหว่างความร่ำรวยของโมเดล (จำนวนพารามิเตอร์) และจำนวนตัวอย่างการฝึก

## ทำไม overfitting เกิดขึ้น

  * ข้อมูลการฝึกไม่เพียงพอ
  * โมเดลทรงพลังเกินไป
  * เสียงรบกวนมากเกินไปในข้อมูลนำเข้า

## วิธีการตรวจจับ overfitting

จากกราฟด้านบน คุณสามารถเห็นว่า overfitting สามารถตรวจจับได้จากข้อผิดพลาดการฝึกที่ต่ำมาก และข้อผิดพลาดการตรวจสอบที่สูง โดยปกติระหว่างการฝึกเราจะเห็นทั้งข้อผิดพลาดการฝึกและการตรวจสอบเริ่มลดลง และแล้วในบางจุดข้อผิดพลาดการตรวจสอบอาจหยุดลดลงและเริ่มเพิ่มขึ้น นี่จะเป็นสัญญาณของ overfitting และตัวบ่งชี้ว่าเราควรหยุดการฝึกในจุดนี้ (หรืออย่างน้อยทำการบันทึกภาพโมเดล)

overfitting

## วิธีการป้องกัน overfitting

หากคุณเห็นว่า overfitting เกิดขึ้น คุณสามารถทำสิ่งต่อไปนี้:

 * เพิ่มจำนวนข้อมูลการฝึก
 * ลดความซับซ้อนของโมเดล
 * ใช้เทคนิคการปรับแต่งบางอย่าง เช่น Dropout ซึ่งเราจะพิจารณาในภายหลัง

## Overfitting และการแลกเปลี่ยน Bias-Variance

Overfitting จริงๆ แล้วเป็นกรณีของปัญหาทั่วไปในสถิติที่เรียกว่าการแลกเปลี่ยน Bias-Variance หากเราพิจารณาแหล่งที่มาของข้อผิดพลาดในโมเดลของเรา เราสามารถเห็นข้อผิดพลาดสองประเภท:

* **ข้อผิดพลาด Bias** เกิดจากอัลกอริธึมของเราไม่สามารถจับความสัมพันธ์ระหว่างข้อมูลการฝึกได้อย่างถูกต้อง มันอาจเกิดจากข้อเท็จจริงที่ว่าโมเดลของเราไม่ทรงพลังพอ (**underfitting**)
* **ข้อผิดพลาด Variance** ซึ่งเกิดจากโมเดลประมาณเสียงรบกวนในข้อมูลนำเข้าแทนที่จะเป็นความสัมพันธ์ที่มีความหมาย (**overfitting**)

ระหว่างการฝึก ข้อผิดพลาด Bias ลดลง (เมื่อโมเดลของเราเรียนรู้ที่จะประมาณข้อมูล) และข้อผิดพลาด Variance เพิ่มขึ้น มันสำคัญที่จะหยุดการฝึก - ไม่ว่าจะด้วยตนเอง (เมื่อเราตรวจพบ overfitting) หรืออัตโนมัติ (โดยการนำการปรับแต่งเข้ามา) - เพื่อป้องกัน overfitting
