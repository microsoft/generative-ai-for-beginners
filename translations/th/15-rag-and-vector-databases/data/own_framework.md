# บทนำสู่โครงข่ายประสาทเทียม: Multi-Layered Perceptron

ในส่วนก่อนหน้านี้ คุณได้เรียนรู้เกี่ยวกับโมเดลโครงข่ายประสาทเทียมที่ง่ายที่สุด - หนึ่งชั้น perceptron ซึ่งเป็นโมเดลการจำแนกประเภทสองคลาสเชิงเส้น

ในส่วนนี้เราจะขยายโมเดลนี้ไปสู่กรอบที่ยืดหยุ่นมากขึ้น ซึ่งจะช่วยให้เรา:

* ทำ **การจำแนกประเภทหลายคลาส** นอกเหนือจากสองคลาส
* แก้ปัญหา **การถดถอย** นอกเหนือจากการจำแนกประเภท
* แยกคลาสที่ไม่สามารถแยกออกจากกันได้เชิงเส้น

เราจะพัฒนากรอบงานแบบแยกส่วนของเราเองใน Python ซึ่งจะช่วยให้เราสร้างสถาปัตยกรรมโครงข่ายประสาทเทียมที่แตกต่างกันได้

## การทำให้ปัญหา Machine Learning เป็นรูปแบบทางการ

เริ่มต้นด้วยการทำให้ปัญหา Machine Learning เป็นรูปแบบทางการ สมมติว่าเรามีชุดข้อมูลการฝึกอบรม **X** พร้อมป้ายกำกับ **Y** และเราจำเป็นต้องสร้างโมเดล *f* ที่จะทำให้การทำนายมีความแม่นยำมากที่สุด คุณภาพของการทำนายวัดโดย **Loss function** ℒ ฟังก์ชันความสูญเสียต่อไปนี้มักถูกใช้:

* สำหรับปัญหาการถดถอย เมื่อเราต้องการทำนายตัวเลข เราสามารถใช้ **absolute error** ∑<sub>i</sub>|f(x<sup>(i)</sup>)-y<sup>(i)</sup>| หรือ **squared error** ∑<sub>i</sub>(f(x<sup>(i)</sup>)-y<sup>(i)</sup>)<sup>2</sup>
* สำหรับการจำแนกประเภท เราใช้ **0-1 loss** (ซึ่งโดยพื้นฐานแล้วเหมือนกับ **accuracy** ของโมเดล) หรือ **logistic loss**

สำหรับ perceptron หนึ่งระดับ ฟังก์ชัน *f* ถูกกำหนดให้เป็นฟังก์ชันเชิงเส้น *f(x)=wx+b* (ที่นี่ *w* คือเมทริกซ์น้ำหนัก *x* คือเวกเตอร์ของคุณสมบัติอินพุต และ *b* คือเวกเตอร์อคติ) สำหรับสถาปัตยกรรมโครงข่ายประสาทเทียมที่แตกต่างกัน ฟังก์ชันนี้สามารถมีรูปแบบที่ซับซ้อนมากขึ้น

> ในกรณีของการจำแนกประเภท มักจะต้องการให้ได้ความน่าจะเป็นของคลาสที่สอดคล้องกันเป็นผลลัพธ์ของโครงข่าย เพื่อแปลงตัวเลขตามอำเภอใจให้เป็นความน่าจะเป็น (เช่น เพื่อทำให้ผลลัพธ์เป็นปกติ) เรามักใช้ฟังก์ชัน **softmax** σ และฟังก์ชัน *f* กลายเป็น *f(x)=σ(wx+b)*

ในคำจำกัดความของ *f* ข้างต้น *w* และ *b* ถูกเรียกว่า **parameters** θ=⟨*w,b*⟩ เมื่อได้รับชุดข้อมูล ⟨**X**,**Y**⟩ เราสามารถคำนวณข้อผิดพลาดโดยรวมในชุดข้อมูลทั้งหมดเป็นฟังก์ชันของพารามิเตอร์ θ

> ✅ **เป้าหมายของการฝึกอบรมโครงข่ายประสาทเทียมคือการลดข้อผิดพลาดโดยการเปลี่ยนแปลงพารามิเตอร์ θ**

## การปรับให้เหมาะสมด้วย Gradient Descent

มีวิธีการปรับให้เหมาะสมของฟังก์ชันที่เป็นที่รู้จักกันดีเรียกว่า **gradient descent** แนวคิดคือเราสามารถคำนวณอนุพันธ์ (ในกรณีหลายมิติเรียกว่า **gradient**) ของฟังก์ชันความสูญเสียในแง่ของพารามิเตอร์ และเปลี่ยนแปลงพารามิเตอร์ในลักษณะที่ข้อผิดพลาดจะลดลง สิ่งนี้สามารถทำให้เป็นรูปแบบทางการได้ดังนี้:

* เริ่มต้นพารามิเตอร์ด้วยค่าสุ่มบางค่า w<sup>(0)</sup>, b<sup>(0)</sup>
* ทำซ้ำขั้นตอนต่อไปนี้หลายครั้ง:
    - w<sup>(i+1)</sup> = w<sup>(i)</sup>-η∂ℒ/∂w
    - b<sup>(i+1)</sup> = b<sup>(i)</sup>-η∂ℒ/∂b

ในระหว่างการฝึกอบรม ขั้นตอนการปรับให้เหมาะสมควรคำนวณโดยพิจารณาชุดข้อมูลทั้งหมด (โปรดจำไว้ว่า loss คำนวณเป็นผลรวมผ่านตัวอย่างการฝึกอบรมทั้งหมด) อย่างไรก็ตาม ในชีวิตจริงเราจะใช้ส่วนเล็กๆ ของชุดข้อมูลที่เรียกว่า **minibatches** และคำนวณ gradients ตามชุดข้อมูลย่อย เนื่องจากชุดข้อมูลย่อยถูกสุ่มเลือกในแต่ละครั้ง วิธีนี้เรียกว่า **stochastic gradient descent** (SGD)

## Multi-Layered Perceptrons และ Backpropagation

โครงข่ายหนึ่งชั้น ตามที่เราเห็นข้างต้น สามารถจำแนกคลาสที่แยกออกจากกันได้เชิงเส้น เพื่อสร้างโมเดลที่มีความหลากหลายมากขึ้น เราสามารถรวมหลายชั้นของโครงข่ายเข้าด้วยกัน ทางคณิตศาสตร์หมายความว่าฟังก์ชัน *f* จะมีรูปแบบที่ซับซ้อนมากขึ้น และจะคำนวณในหลายขั้นตอน:
* z<sub>1</sub>=w<sub>1</sub>x+b<sub>1</sub>
* z<sub>2</sub>=w<sub>2</sub>α(z<sub>1</sub>)+b<sub>2</sub>
* f = σ(z<sub>2</sub>)

ที่นี่ α เป็น **non-linear activation function**, σ เป็นฟังก์ชัน softmax, และพารามิเตอร์ θ=<*w<sub>1</sub>,b<sub>1</sub>,w<sub>2</sub>,b<sub>2</sub>*>

อัลกอริทึม gradient descent จะยังคงเหมือนเดิม แต่จะคำนวณ gradients ได้ยากขึ้น โดยใช้กฎการหาอนุพันธ์แบบลูกโซ่ เราสามารถคำนวณอนุพันธ์ได้ดังนี้:

* ∂ℒ/∂w<sub>2</sub> = (∂ℒ/∂σ)(∂σ/∂z<sub>2</sub>)(∂z<sub>2</sub>/∂w<sub>2</sub>)
* ∂ℒ/∂w<sub>1</sub> = (∂ℒ/∂σ)(∂σ/∂z<sub>2</sub>)(∂z<sub>2</sub>/∂α)(∂α/∂z<sub>1</sub>)(∂z<sub>1</sub>/∂w<sub>1</sub>)

> ✅ กฎการหาอนุพันธ์แบบลูกโซ่ถูกใช้ในการคำนวณอนุพันธ์ของฟังก์ชันความสูญเสียในแง่ของพารามิเตอร์

โปรดทราบว่าด้านซ้ายสุดของนิพจน์ทั้งหมดเหล่านี้เหมือนกัน และดังนั้นเราสามารถคำนวณอนุพันธ์ได้อย่างมีประสิทธิภาพโดยเริ่มจากฟังก์ชันความสูญเสียและไป "ย้อนกลับ" ผ่านกราฟการคำนวณ ดังนั้นวิธีการฝึกอบรม multi-layered perceptron จึงเรียกว่า **backpropagation** หรือ 'backprop'
