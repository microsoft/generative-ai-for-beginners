## บทนำ

ในบทเรียนนี้เราจะครอบคลุมหัวข้อต่อไปนี้:

- การแนะนำ RAG คืออะไรและทำไมจึงนำไปใช้ใน AI (ปัญญาประดิษฐ์)

- ความเข้าใจว่าฐานข้อมูลเวกเตอร์คืออะไรและการสร้างฐานข้อมูลหนึ่งสำหรับแอปพลิเคชันของเรา

- ตัวอย่างแบบปฏิบัติในการผสาน RAG เข้ากับแอปพลิเคชัน

## เป้าหมายการเรียนรู้

หลังจากจบบทเรียนนี้ คุณจะสามารถ:

- อธิบายความสำคัญของ RAG ในการดึงและประมวลผลข้อมูลได้

- ตั้งค่าแอป RAG และเสริมฐานข้อมูลของคุณกับ LLM

- การผสาน RAG และฐานข้อมูลเวกเตอร์อย่างมีประสิทธิภาพในแอปพลิเคชัน LLM

## กรณีของเรา: การเพิ่มประสิทธิภาพ LLM ของเราด้วยข้อมูลของเราเอง

สำหรับบทเรียนนี้ เราต้องการเพิ่มบันทึกของเราเองเข้าไปในสตาร์ทอัปการศึกษา ซึ่งช่วยให้แชทบอทได้รับข้อมูลเพิ่มเติมในหัวข้อต่าง ๆ การใช้บันทึกที่เรามี ผู้เรียนจะสามารถศึกษาได้ดีขึ้นและเข้าใจหัวข้อต่าง ๆ ง่ายขึ้น ทำให้การทบทวนสำหรับการสอบเป็นไปได้สะดวกขึ้น ในการสร้างกรณีของเรา เราจะใช้:

- `Azure OpenAI:` LLM ที่เราจะใช้สร้างแชทบอทของเรา

- `บทเรียน AI สำหรับผู้เริ่มต้นเกี่ยวกับ Neural Networks`: ข้อมูลที่เราจะใช้เสริมฐานกับ LLM ของเรา

- `Azure AI Search` และ `Azure Cosmos DB:` ฐานข้อมูลเวกเตอร์สำหรับจัดเก็บข้อมูลและสร้างดัชนีการค้นหา

ผู้ใช้จะสามารถสร้างแบบทดสอบฝึกฝนจากบันทึกของตนเอง การ์ดทบทวน และสรุปเป็นภาพรวมที่กระชับ เพื่อเริ่มต้น เรามาดูว่า RAG คืออะไรและทำงานอย่างไร:

## การสร้างข้อความที่เสริมด้วยการดึงข้อมูล (Retrieval Augmented Generation, RAG)

แชทบอทที่ขับเคลื่อนด้วย LLM จะประมวลผลคำถามจากผู้ใช้เพื่อสร้างคำตอบ ออกแบบมาให้โต้ตอบได้และมีส่วนร่วมกับผู้ใช้ในหัวข้อต่าง ๆ อย่างกว้างขวาง อย่างไรก็ตามคำตอบที่ได้จะจำกัดอยู่ที่บริบทที่จัดเตรียมและข้อมูลพื้นฐานที่ได้รับการฝึกฝน ตัวอย่างเช่น GPT-4 มีความรู้จำกัดถึงกันยายน 2021 หมายความว่าขาดความรู้เกี่ยวกับเหตุการณ์ที่เกิดขึ้นหลังจากนั้น นอกจากนี้ข้อมูลที่ใช้ฝึก LLM จะไม่รวมข้อมูลลับเช่นบันทึกส่วนตัวหรือคู่มือผลิตภัณฑ์ของบริษัท

### RAG (Retrieval Augmented Generation) ทำงานอย่างไร

![ภาพวาดที่แสดงวิธีการทำงานของ RAG](../../../translated_images/th/how-rag-works.f5d0ff63942bd3a6.webp)

สมมติว่าคุณต้องการเปิดตัวแชทบอทที่สร้างแบบทดสอบจากบันทึกของคุณ คุณจะต้องเชื่อมต่อกับฐานความรู้ นี่คือที่ที่ RAG ช่วยได้ RAG ทำงานดังนี้:

- **ฐานความรู้:** ก่อนการดึงข้อมูล เอกสารเหล่านี้จะต้องถูกนำเข้าและประมวลผลล่วงหน้า ปกติจะมีการแบ่งเอกสารขนาดใหญ่ออกเป็นชิ้นเล็ก ๆ แปลงเป็น embedding ข้อความและจัดเก็บในฐานข้อมูล

- **คำถามผู้ใช้:** ผู้ใช้ถามคำถาม

- **การดึงข้อมูล:** เมื่อผู้ใช้ถามคำถาม โมเดล embedding จะดึงข้อมูลที่เกี่ยวข้องจากฐานความรู้ของเราเพื่อจัดเตรียมบริบทเพิ่มเติมที่จะรวมเข้ากับ prompt

- **การสร้างข้อความเสริม:** LLM จะปรับปรุงคำตอบโดยอิงจากข้อมูลที่ดึงมา ซึ่งช่วยให้คำตอบที่สร้างไม่ใช่แค่ขึ้นอยู่กับข้อมูลที่ได้รับการฝึกฝน แต่ยังรวมข้อมูลบริบทที่เกี่ยวข้องจากข้อมูลที่เพิ่มเข้ามา ข้อมูลที่ดึงมาใช้เสริมคำตอบของ LLM จากนั้น LLM จะส่งคำตอบกลับคำถามของผู้ใช้

![ภาพวาดแสดงสถาปัตยกรรมของ RAG](../../../translated_images/th/encoder-decode.f2658c25d0eadee2.webp)

สถาปัตยกรรมของ RAG ถูกนำไปใช้โดยใช้ตัวแปลง(transformers) ซึ่งประกอบด้วยสองส่วน: ตัวเข้ารหัส (encoder) และตัวถอดรหัส (decoder) ตัวอย่างเช่น เมื่อผู้ใช้ถามคำถาม ข้อความนำเข้า 'เข้ารหัส' เป็นเวกเตอร์ที่จับความหมายของคำ และเวกเตอร์จะถูก 'ถอดรหัส' ลงในดัชนีเอกสารของเราและสร้างข้อความใหม่โดยอิงจากคำถามของผู้ใช้ LLM ใช้ทั้งโมเดล encoder-decoder เพื่อสร้างผลลัพธ์

สองแนวทางในการนำ RAG ไปใช้ตามเอกสารที่เสนอ: [Retrieval-Augmented Generation for Knowledge intensive NLP (natural language processing software) Tasks](https://arxiv.org/pdf/2005.11401.pdf?WT.mc_id=academic-105485-koreyst) ได้แก่:

- **_RAG-Sequence_** ใช้เอกสารที่ดึงมาเพื่อทำนายคำตอบที่ดีที่สุดสำหรับคำถามของผู้ใช้

- **RAG-Token** ใช้เอกสารเพื่อสร้างโทเค็นถัดไป จากนั้นดึงเอกสารเหล่านั้นมาเพื่อตอบคำถามของผู้ใช้

### ทำไมคุณจึงควรใช้ RAG?

- **ข้อมูลที่มีความหลากหลาย:** ทำให้คำตอบเป็นปัจจุบันและล่าสุด ดังนั้นจะช่วยเพิ่มประสิทธิภาพงานเฉพาะด้านโดยเข้าถึงฐานความรู้ภายใน

- ลดการแต่งเรื่องโดยใช้ **ข้อมูลที่ตรวจสอบได้** ในฐานความรู้เพื่อจัดเตรียมบริบทสำหรับคำถามของผู้ใช้

- **ประหยัดค่าใช้จ่าย** เพราะมีค่าใช้จ่ายถูกกว่าการปรับแต่ง LLM อย่างละเอียด

## การสร้างฐานความรู้

แอปพลิเคชันของเราขึ้นอยู่กับข้อมูลส่วนบุคคลของเรา เช่นบทเรียน Neural Network ในหลักสูตร AI สำหรับผู้เริ่มต้น

### ฐานข้อมูลเวกเตอร์

ฐานข้อมูลเวกเตอร์ ต่างจากฐานข้อมูลแบบเดิม ๆ เป็นฐานข้อมูลเฉพาะเพื่อจัดเก็บ จัดการ และค้นหาเวกเตอร์ฝังตัว มันจัดเก็บการแทนค่าตัวเลขของเอกสาร การแตกข้อมูลเป็น embedding ตัวเลขช่วยให้ระบบ AI ของเราเข้าใจและประมวลผลข้อมูลได้ง่ายขึ้น

เราจัดเก็บ embedding ของเราในฐานข้อมูลเวกเตอร์เนื่องจาก LLM มีข้อจำกัดจำนวนโทเค็นที่รับเป็นอินพุต คุณไม่สามารถส่ง embedding ทั้งหมดให้ LLM ได้ เราจึงต้องแบ่งเป็นชิ้นเล็ก ๆ และเมื่อผู้ใช้ถามคำถาม embedding ที่ใกล้เคียงกับคำถามที่สุดจะถูกส่งกลับพร้อม prompt การแบ่งเป็นชิ้นยังช่วยลดต้นทุนในจำนวนโทเค็นที่ส่งผ่าน LLM

บางฐานข้อมูลเวกเตอร์ยอดนิยมเช่น Azure Cosmos DB, Clarifyai, Pinecone, Chromadb, ScaNN, Qdrant และ DeepLake คุณสามารถสร้างโมเดล Azure Cosmos DB โดยใช้ Azure CLI ด้วยคำสั่งต่อไปนี้:

```bash

az cosmosdb create -n <cosmos-db-name> -r <resource-group-name>
az cosmosdb list-keys -n <cosmos-db-name> -g <resource-group-name>
```

### จากข้อความเป็น embeddings

ก่อนจัดเก็บข้อมูล เราจะต้องแปลงข้อมูลเป็น vector embeddings ก่อนจัดเก็บในฐานข้อมูล หากคุณทำงานกับเอกสารขนาดใหญ่หรือข้อความยาว คุณสามารถแบ่งชิ้นตามคำถามที่คาดหวังได้ การแบ่งชิ้นสามารถทำได้ที่ระดับประโยค หรือย่อหน้า เนื่องจากการแบ่งชิ้นอิงความหมายจากคำรอบ ๆ คุณสามารถเพิ่มบริบทอื่น ๆ ให้ชิ้น เช่น เพิ่มชื่อเอกสารหรือข้อความก่อนหรือหลังชิ้นได้ คุณสามารถแบ่งข้อมูลได้ดังนี้:

```python
def split_text(text, max_length, min_length):
    words = text.split()
    chunks = []
    current_chunk = []

    for word in words:
        current_chunk.append(word)
        if len(' '.join(current_chunk)) < max_length and len(' '.join(current_chunk)) > min_length:
            chunks.append(' '.join(current_chunk))
            current_chunk = []

    # หากชิ้นสุดท้ายไม่ถึงความยาวขั้นต่ำ ก็ให้เพิ่มเข้าไปอย่างนั้นแหละ
    if current_chunk:
        chunks.append(' '.join(current_chunk))

    return chunks
```

เมื่อแบ่งชิ้นแล้ว เราสามารถฝังข้อความโดยใช้โมเดล embedding ต่าง ๆ ได้ โมเดลที่ใช้ได้เช่น word2vec, ada-002 โดย OpenAI, Azure Computer Vision และอื่น ๆ การเลือกโมเดลใช้ขึ้นกับภาษาที่คุณใช้ ประเภทเนื้อหาที่เข้ารหัส (ข้อความ/ภาพ/เสียง) ขนาดอินพุตที่รองรับ และความยาวผลลัพธ์ embedding

## การดึงข้อมูลและการค้นหาเวกเตอร์

เมื่อผู้ใช้ถามคำถาม ตัวดึงข้อมูลจะเปลี่ยนคำถามเป็นเวกเตอร์โดยใช้ query encoder จากนั้นค้นหาเวกเตอร์ที่เกี่ยวข้องในดัชนีเอกสาร เมื่อเสร็จแล้วจะแปลงอินพุตเวกเตอร์และเวกเตอร์เอกสารเป็นข้อความแล้วส่งผ่าน LLM

### การดึงข้อมูล

การดึงข้อมูลเกิดขึ้นเมื่อระบบพยายามค้นหาเอกสารจากดัชนีที่ตรงตามเกณฑ์การค้นหา เป้าหมายของตัวดึงข้อมูลคือการได้เอกสารที่ใช้ให้บริบทและเสริมฐาน LLM ของคุณ

มีหลายวิธีค้นหาในฐานข้อมูล เช่น:

- **การค้นหาด้วยคำสำคัญ** - ใช้สำหรับการค้นหาข้อความ

- **การค้นหาเวกเตอร์** - แปลงเอกสารจากข้อความเป็นเวกเตอร์โดยใช้โมเดล embedding อนุญาตให้ค้นหาเชิงความหมาย **semantic search** โดยใช้ความหมายของคำ การดึงข้อมูลจะใช้ออนไลน์เอกสารที่เวกเตอร์ใกล้เคียงกับคำถามผู้ใช้มากที่สุด

- **ผสมผสาน** - การรวมกันของการค้นหาด้วยคำสำคัญและเวกเตอร์

ความท้าทายของการดึงข้อมูลคือเมื่อไม่มีคำตอบใกล้เคียงในฐานข้อมูล ระบบจะคืนข้อมูลที่ดีที่สุดที่หาได้ อย่างไรก็ตามคุณสามารถใช้วิธีตั้งค่าระยะทางสูงสุดสำหรับความเกี่ยวข้อง หรือใช้การค้นหาผสมที่รวมทั้งคำสำคัญและเวกเตอร์ ในบทเรียนนี้เราจะใช้การค้นหาผสม โดยจัดเก็บข้อมูลใน dataframe ที่มีคอลัมน์บรรจุชิ้นและ embedding

### ความคล้ายคลึงของเวกเตอร์

ตัวดึงข้อมูลจะค้นหาในฐานความรู้สำหรับ embedding ที่อยู่ใกล้กันที่สุด ซึ่งเป็นเพื่อนบ้านใกล้สุด เพราะเป็นข้อความที่คล้ายกัน ในกรณีเมื่อผู้ใช้ถาม จะทำ embedding แล้วจับคู่กับ embedding ที่คล้ายกัน การวัดทั่วไปที่ใช้หาความคล้ายคลึงระหว่างเวกเตอร์ต่าง ๆ คือ cosine similarity ที่อิงจากมุมระหว่างเวกเตอร์สองอัน

เรายังสามารถวัดความคล้ายคลึงด้วยทางเลือกอื่น เช่น ระยะทาง Euclidean ซึ่งคือเส้นตรงระหว่างจุดปลายเวกเตอร์ และ dot product ที่วัดผลรวมของผลคูณของสมาชิกที่ตรงกันในเวกเตอร์สองอัน

### ดัชนีการค้นหา

เมื่อต้องทำการดึงข้อมูล เราจะต้องสร้างดัชนีการค้นหาสำหรับฐานความรู้ก่อน ดัชนีจะเก็บ embedding และช่วยดึงชิ้นที่เหมือนกันที่สุดได้อย่างรวดเร็วแม้ในฐานข้อมูลขนาดใหญ่ เราสามารถสร้างดัชนีของเราท้องถิ่นได้โดยใช้:

```python
from sklearn.neighbors import NearestNeighbors

embeddings = flattened_df['embeddings'].to_list()

# สร้างดัชนีการค้นหา
nbrs = NearestNeighbors(n_neighbors=5, algorithm='ball_tree').fit(embeddings)

# ในการค้นหาดัชนี คุณสามารถใช้เมธอด kneighbors ได้
distances, indices = nbrs.kneighbors(embeddings)
```

### การเรียงลำดับใหม่

เมื่อคุณสอบถามฐานข้อมูลแล้ว อาจจำเป็นต้องเรียงลำดับผลลัพธ์จากที่เกี่ยวข้องมากที่สุด LLM สำหรับการเรียงลำดับใหม่จะใช้ Machine Learning เพื่อปรับปรุงความเกี่ยวข้องโดยจัดอันดับผลลัพธ์จากที่สำคัญที่สุด ใช้ Azure AI Search การเรียงลำดับใหม่นี้จะทำโดยอัตโนมัติด้วย semantic reranker ตัวอย่างการทำงานของการเรียงลำดับใหม่โดยใช้เพื่อนบ้านใกล้สุด:

```python
# ค้นหาเอกสารที่เหมือนกันมากที่สุด
distances, indices = nbrs.kneighbors([query_vector])

index = []
# พิมพ์เอกสารที่เหมือนกันมากที่สุด
for i in range(3):
    index = indices[0][i]
    for index in indices[0]:
        print(flattened_df['chunks'].iloc[index])
        print(flattened_df['path'].iloc[index])
        print(flattened_df['distances'].iloc[index])
    else:
        print(f"Index {index} not found in DataFrame")
```

## รวมทุกอย่างเข้าด้วยกัน

ขั้นตอนสุดท้ายคือการเพิ่ม LLM ของเราเข้าไปเพื่อให้ได้คำตอบที่เสริมด้วยข้อมูลของเรา เราสามารถทำได้ดังนี้:

```python
user_input = "what is a perceptron?"

def chatbot(user_input):
    # แปลงคำถามเป็นเวกเตอร์คำค้น
    query_vector = create_embeddings(user_input)

    # ค้นหาเอกสารที่คล้ายกันมากที่สุด
    distances, indices = nbrs.kneighbors([query_vector])

    # เพิ่มเอกสารลงในคำค้นเพื่อให้บริบท
    history = []
    for index in indices[0]:
        history.append(flattened_df['chunks'].iloc[index])

    # รวมประวัติและข้อมูลผู้ใช้
    history.append(user_input)

    # สร้างวัตถุข้อความ
    messages=[
        {"role": "system", "content": "You are an AI assistant that helps with AI questions."},
        {"role": "user", "content": "\n\n".join(history) }
    ]

    # ใช้ Responses API เพื่อสร้างการตอบกลับ
    response = client.responses.create(
        model="gpt-5-mini",
        max_output_tokens=800,
        input=messages,
        store=False,
    )

    return response.output_text

chatbot(user_input)
```

## การประเมินแอปพลิเคชันของเรา

### ตัวชี้วัดการประเมิน

- คุณภาพของคำตอบที่ให้ ต้องแน่ใจว่าฟังดูเป็นธรรมชาติ คล่องแคล่ว และเหมือนมนุษย์

- ความสอดคล้องของข้อมูล: ประเมินว่าคำตอบมาจากเอกสารที่ให้ไว้หรือไม่

- ความเกี่ยวข้อง: ประเมินว่าคำตอบตรงประเด็นและเกี่ยวข้องกับคำถามหรือไม่

- ความคล่องแคล่ว - ว่าคำตอบเป็นไวยากรณ์ที่สมเหตุสมผลหรือไม่

## กรณีการใช้งานสำหรับ RAG และฐานข้อมูลเวกเตอร์

มีกรณีการใช้งานมากมายที่คำสั่งฟังก์ชันสามารถปรับปรุงแอปของคุณได้ เช่น:

- คำถามและการตอบ: เสริมฐานข้อมูลของบริษัทกับแชทที่พนักงานใช้ถามคำถาม

- ระบบแนะนำ: คุณสามารถสร้างระบบจับคู่อันดับที่เหมือนกันที่สุด เช่น ภาพยนตร์ ร้านอาหาร และอื่น ๆ

- บริการแชทบอท: คุณสามารถเก็บประวัติการแชทและปรับแต่งการสนทนาตามข้อมูลผู้ใช้

- การค้นหาภาพโดยใช้ embedding เวกเตอร์ มีประโยชน์สำหรับการรู้จำภาพและการตรวจจับความผิดปกติ

## สรุป

เราได้ครอบคลุมพื้นที่พื้นฐานของ RAG ตั้งแต่การเพิ่มข้อมูลของเราเข้าแอป การถามและผลลัพธ์ของผู้ใช้ เพื่อให้ง่ายต่อการสร้าง RAG คุณสามารถใช้เฟรมเวิร์กเช่น Semanti Kernel, Langchain หรือ Autogen

- ใช้เฟรมเวิร์ก เช่น LangChain หรือ Semantic Kernel และสร้างแอปของคุณอีกครั้ง
