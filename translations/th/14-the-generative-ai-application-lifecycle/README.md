# วัฏจักรชีวิตของแอปพลิเคชัน Generative AI

คำถามสำคัญสำหรับแอปพลิเคชัน AI ทุกตัวคือความเกี่ยวข้องของฟีเจอร์ AI เนื่องจาก AI เป็นสาขาที่พัฒนาอย่างรวดเร็ว เพื่อให้มั่นใจว่าแอปพลิเคชันของคุณยังคงเกี่ยวข้อง เชื่อถือได้ และมีความมั่นคง คุณจึงต้องติดตาม ประเมิน และปรับปรุงอย่างต่อเนื่อง นี่คือที่มาของวัฏจักรชีวิตของ generative AI

วัฏจักรชีวิตของ generative AI เป็นกรอบแนวทางที่ช่วยนำทางคุณผ่านขั้นตอนการพัฒนา การปรับใช้ และการดูแลรักษาแอปพลิเคชัน generative AI ช่วยให้คุณกำหนดเป้าหมาย วัดผลการทำงาน ระบุความท้าทาย และนำเสนอวิธีแก้ไข นอกจากนี้ยังช่วยให้แอปพลิเคชันของคุณสอดคล้องกับมาตรฐานจริยธรรมและกฎหมายของโดเมนและผู้มีส่วนได้ ส่วนเสียของคุณ การทำตามวัฏจักรชีวิตของ generative AI คุณจะมั่นใจว่าแอปพลิเคชันของคุณสร้างคุณค่าและตอบสนองผู้ใช้เสมอ

## บทนำ

ในบทนี้ คุณจะได้:

- วัฏจักรชีวิตของ LLM
- เครื่องมือในวัฏจักรชีวิต
- การวัดและประเมินผลวัฏจักรชีวิต

## เข้าใจการเปลี่ยนแปลงแนวคิดจาก MLOps ไปสู่ LLMOps

LLM เป็นเครื่องมือใหม่ในอาวุธปัญญาประดิษฐ์ มีประสิทธิภาพอย่างมากในการวิเคราะห์และงานสร้างสรรค์สำหรับแอปพลิเคชัน อย่างไรก็ตาม ความสามารถนี้ก็มาพร้อมผลกระทบบางอย่างต่อวิธีการจัดการงาน AI และงานเรียนรู้ของเครื่องแบบดั้งเดิม

ด้วยเหตุนี้ เราจึงต้องมีแนวคิดใหม่เพื่อปรับใช้เครื่องมือนี้อย่างมีพลวัตและมีแรงจูงใจที่ถูกต้อง เราสามารถจำแนกแอป AI เก่าเป็น "ML Apps" และแอป AI ใหม่เป็น "GenAI Apps" หรือเพียงแค่ "AI Apps" สะท้อนเทคโนโลยีและเทคนิคหลักที่ใช้ในเวลานั้น ซึ่งเปลี่ยนแปลงแนวคิดในหลาย ๆ ด้าน ดูการเปรียบเทียบต่อไปนี้

![การเปรียบเทียบ LLMOps vs. MLOps](../../../translated_images/th/01-llmops-shift.29bc933cb3bb0080.webp)

สังเกตว่าใน LLMOps เรามุ่งเน้นที่นักพัฒนาแอปมากขึ้น ใช้การผสานรวมเป็นจุดสำคัญ ใช้ "โมเดลในรูปแบบบริการ" และคำนึงถึงจุดต่อไปนี้สำหรับเมตริก

- คุณภาพ: คุณภาพของการตอบกลับ
- อันตราย: AI ที่รับผิดชอบ
- ความซื่อสัตย์: ความเป็นเหตุเป็นผลของการตอบกลับ (สมเหตุสมผล? ถูกต้องหรือไม่?)
- ต้นทุน: งบประมาณโซลูชัน
- ความหน่วง: เวลาตอบกลับเฉลี่ยต่อโทเค็น

## วัฏจักรชีวิตของ LLM

ก่อนอื่น เพื่อเข้าใจวัฏจักรชีวิตและการเปลี่ยนแปลง ให้สังเกตแผนภาพข้อมูลต่อไปนี้

![ข้อมูลแผนภาพ LLMOps](../../../translated_images/th/02-llmops.70a942ead05a7645.webp)

อย่างที่คุณอาจสังเกต นี่แตกต่างจากวัฏจักรชีวิตทั่วไปของ MLOps LLM มีความต้องการใหม่มากมาย เช่น การใช้คำสั่ง (Prompting) เทคนิคต่าง ๆ เพื่อปรับปรุงคุณภาพ (Fine-Tuning, RAG, Meta-Prompts) การประเมินและความรับผิดชอบกับ AI ที่รับผิดชอบ และล่าสุด เมตริกการประเมินใหม่ (คุณภาพ, อันตราย, ความซื่อสัตย์, ต้นทุน และความหน่วง)

ตัวอย่างเช่น ชมวิธีที่เราสร้างไอเดีย โดยใช้วิศวกรรม prompt เพื่อทดลองกับ LLM ต่าง ๆ เพื่อสำรวจความเป็นไปได้และทดสอบสมมติฐานว่าถูกต้องหรือไม่

สังเกตว่านี่ไม่ใช่กระบวนการเชิงเส้น แต่เป็นวงจรที่รวมกัน วนซ้ำ และมีวงจรหลักครอบคลุม

เราจะสำรวจขั้นตอนเหล่านี้อย่างไร? มาเจาะลึกว่าเราจะสร้างวัฏจักรชีวิตได้อย่างไร

![เวิร์กโฟลว์ LLMOps](../../../translated_images/th/03-llm-stage-flows.3a1e1c401235a6cf.webp)

อาจดูซับซ้อนเล็กน้อย ให้เรามุ่งเน้นที่ 3 ขั้นตอนใหญ่ก่อน

1. การคิดและสำรวจ: การสำรวจ ที่นี่เราสามารถสำรวจตามความต้องการทางธุรกิจ การสร้างต้นแบบ สร้าง [PromptFlow](https://microsoft.github.io/promptflow/index.html?WT.mc_id=academic-105485-koreyst) และทดสอบว่ามีประสิทธิภาพพอสำหรับสมมติฐานของเราหรือไม่
1. การสร้างและเพิ่มคุณค่า: การนำไปใช้ ตอนนี้ เราเริ่มประเมินสำหรับชุดข้อมูลขนาดใหญ่ ใช้เทคนิค เช่น การปรับจูน (Fine-tuning) และ RAG เพื่อตรวจสอบความมั่นคงของโซลูชัน หากไม่ได้ผล ให้ทำซ้ำ เพิ่มขั้นตอนใหม่ในเวิร์กโฟลว์ หรือจัดโครงสร้างข้อมูลใหม่อาจช่วย หลังจากทดสอบเวิร์กโฟลว์และขนาด หากทำงานได้และตรวจสอบเมตริกของเรา พร้อมสำหรับขั้นตอนถัดไป
1. การดำเนินงาน: การผสานรวม ตอนนี้เพิ่มระบบการตรวจสอบและแจ้งเตือนไปยังระบบของเรา การปรับใช้และการผสานรวมแอปพลิเคชันกับแอปของเรา

จากนั้น เรามีวงจรหลักของการจัดการที่เน้นความปลอดภัย การปฏิบัติตาม และธรรมาภิบาล

ขอแสดงความยินดี ตอนนี้คุณมีแอป AI พร้อมใช้งานและทำงานได้ สำหรับประสบการณ์ใช้งานจริง ให้ดูที่ [Contoso Chat Demo.](https://nitya.github.io/contoso-chat/?WT.mc_id=academic-105485-koreyst)

## เครื่องมือในวัฏจักรชีวิต

สำหรับเครื่องมือ Microsoft มี [Azure AI Platform](https://azure.microsoft.com/solutions/ai/?WT.mc_id=academic-105485-koreyst) และ [PromptFlow](https://microsoft.github.io/promptflow/index.html?WT.mc_id=academic-105485-koreyst) ที่ช่วยอำนวยความสะดวกและทำให้วงจรง่ายต่อการนำไปใช้และพร้อมใช้งาน

[Azure AI Platform](https://azure.microsoft.com/solutions/ai/?WT.mc_id=academic-105485-koreyst) ช่วยให้คุณใช้ [Microsoft Foundry](https://ai.azure.com/?WT.mc_id=academic-105485-koreyst) Microsoft Foundry (เดิมชื่อ Azure AI Studio) เป็นพอร์ทัลเว็บที่ช่วยให้คุณสำรวจโมเดล ตัวอย่าง และเครื่องมือ จัดการทรัพยากร และใช้เวิร์กโฟลว์การพัฒนาระบบ UI รวมถึงตัวเลือก SDK/CLI สำหรับการพัฒนาด้วยโค้ดเป็นหลัก

![ความเป็นไปได้ของ Azure AI](../../../translated_images/th/04-azure-ai-platform.80203baf03a12fa8.webp)

Azure AI ช่วยให้ใช้ทรัพยากรหลายอย่างเพื่อจัดการการดำเนินงาน บริการ โครงการ การค้นหาด้วยเวกเตอร์ และความต้องการฐานข้อมูล

![LLMOps กับ Azure AI](../../../translated_images/th/05-llm-azure-ai-prompt.a5ce85cdbb494bdf.webp)

สร้าง ตั้งแต่ Proof-of-Concept(POC) จนถึงแอปพลิเคชันขนาดใหญ่ด้วย PromptFlow:

- ออกแบบและสร้างแอปจาก VS Code ด้วยเครื่องมือเชิงภาพและฟังก์ชัน
- ทดสอบและปรับแต่งแอปของคุณเพื่อคุณภาพ AI ได้อย่างง่ายดาย
- ใช้ Microsoft Foundry เพื่อผสานรวมและทำซ้ำกับคลาวด์ กดและปรับใช้เพื่อผสานรวมรวดเร็ว

![LLMOps กับ PromptFlow](../../../translated_images/th/06-llm-promptflow.a183eba07a3a7fdf.webp)
