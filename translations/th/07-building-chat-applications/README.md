> _(คลิกที่ภาพด้านบนเพื่อดูวิดีโอบทเรียนนี้)_

ตอนนี้เราได้เห็นแล้วว่าเราสามารถสร้างแอปสร้างข้อความได้อย่างไร ลองมาดูแอปแชทกันบ้าง

แอปแชทได้กลายเป็นส่วนหนึ่งในชีวิตประจำวันของเรา มอบมากกว่าการสนทนาแบบทั่วไป พวกมันเป็นส่วนสำคัญของบริการลูกค้า การสนับสนุนทางเทคนิค และแม้แต่ระบบให้คำปรึกษาที่ซับซ้อน เป็นไปได้ว่าคุณเพิ่งได้รับความช่วยเหลือจากแอปแชทเมื่อไม่นานมานี้ เมื่อเรานำเทคโนโลยีที่ล้ำหน้ามากขึ้นอย่าง generative AI มาใช้ในแพลตฟอร์มเหล่านี้ ความซับซ้อนและความท้าทายก็เพิ่มขึ้นตามไปด้วย

คำถามบางอย่างที่เราต้องการคำตอบคือ:

- **การสร้างแอป** เราจะสร้างและผสานรวมแอป AI เหล่านี้อย่างมีประสิทธิภาพเพื่อกรณีการใช้งานเฉพาะได้อย่างไร?
- **การตรวจสอบ** หลังจากใช้งานแล้ว เราจะตรวจสอบและทำให้แน่ใจว่าแอปทำงานในระดับคุณภาพสูงสุด ทั้งในแง่การทำงานและการปฏิบัติตาม [หลักการหกข้อของ AI ที่รับผิดชอบ](https://www.microsoft.com/ai/responsible-ai?WT.mc_id=academic-105485-koreyst) ได้อย่างไร?

เมื่อเราเข้าสู่ยุคที่ถูกกำหนดด้วยระบบอัตโนมัติและการโต้ตอบระหว่างมนุษย์และเครื่องจักรอย่างไม่มีสะดุด การเข้าใจว่าก generative AI เปลี่ยนแปลงขอบเขต ความลึก และความสามารถในการปรับตัวของแอปแชทอย่างไรนั้นถือเป็นสิ่งสำคัญ บทเรียนนี้จะสำรวจด้านสถาปัตยกรรมที่สนับสนุนระบบที่ซับซ้อนเหล่านี้ เจาะลึกวิธีการปรับแต่งเฉพาะ และประเมินตัวชี้วัดและประเด็นสำคัญในการใช้งาน AI อย่างรับผิดชอบ

## บทนำ

บทเรียนนี้ครอบคลุม:

- วิธีการปรับแต่งและทำ fine-tuning กับแอป
- กลยุทธ์และข้อควรพิจารณาในการตรวจสอบแอปแชทอย่างมีประสิทธิผล

## เป้าหมายการเรียนรู้

เมื่อจบบทเรียนนี้ คุณจะสามารถ:

- ปรับแต่งแอปแชทสำหรับกรณีการใช้งานเฉพาะ
- ระบุตัวชี้วัดและข้อพิจารณาหลักในการตรวจสอบและรักษาคุณภาพของแอปแชทที่ใช้ AI
- ทำให้มั่นใจว่าแอปแชทใช้ AI อย่างรับผิดชอบ

## การผสาน Generative AI เข้ากับแอปแชท

การยกระดับแอปแชทด้วย generative AI ไม่ได้เน้นแค่การทำให้มันฉลาดขึ้นเท่านั้น แต่ยังเกี่ยวกับการเพิ่มประสิทธิภาพสถาปัตยกรรม ประสิทธิภาพ และส่วนต่อประสานผู้ใช้เพื่อมอบประสบการณ์ผู้ใช้ที่มีคุณภาพ ซึ่งรวมถึงการสำรวจพื้นฐานสถาปัตยกรรม การผสานรวม API และข้อควรพิจารณาด้านส่วนต่อประสานผู้ใช้ ส่วนนี้มีเป้าหมายที่จะมอบแผนที่ทางที่ครอบคลุมสำหรับนำทางในภูมิทัศน์ที่ซับซ้อนเหล่านี้ ไม่ว่าคุณจะเชื่อมต่อกับระบบที่มีอยู่หรือสร้างเป็นแพลตฟอร์มแยกต่างหาก

เมื่อสิ้นสุดส่วนนี้ คุณจะได้รับความรู้ความชำนาญที่จำเป็นในการสร้างและผสานรวมแอปแชทได้อย่างมีประสิทธิภาพ

### แชทบอทหรือแอปแชท?

ก่อนที่เราจะลงมือสร้างแอปแชท มาลองเปรียบเทียบ ‘แชทบอท’ กับ ‘แอปแชทที่ขับเคลื่อนด้วย AI’ ซึ่งมีบทบาทและฟังก์ชันต่างกัน แชทบอทมีจุดประสงค์หลักเพื่ออัตโนมัติงานสนทนาเฉพาะ เช่น การตอบคำถามที่ถูกถามบ่อยหรือการติดตามพัสดุ โดยมักอยู่ภายใต้ตรรกะที่ใช้กฎหรืออัลกอริธึม AI ที่ซับซ้อน ในทางกลับกัน แอปแชทที่ขับเคลื่อนด้วย AI เป็นสภาพแวดล้อมที่กว้างขวางยิ่งขึ้น ออกแบบมาเพื่ออำนวยความสะดวกในการสื่อสารดิจิทัลหลากหลายรูปแบบ เช่น การแชทด้วยข้อความ เสียง และวิดีโอระหว่างผู้ใช้มนุษย์ ฟีเจอร์เด่นของมันคือการรวมโมเดล generative AI ที่จำลองบทสนทนาแบบมีความละเอียดอ่อนและเหมือนมนุษย์ โดยสร้างคำตอบจากอินพุตและบริบทที่หลากหลาย แอปแชทที่ขับเคลื่อนด้วย generative AI สามารถสนทนาแบบเปิดกว้าง ปรับตัวตามบริบทที่เปลี่ยนไป และแม้กระทั่งสร้างบทสนทนาที่มีความคิดสร้างสรรค์หรือซับซ้อนได้

ตารางด้านล่างสรุปความแตกต่างและความเหมือนที่สำคัญเพื่อช่วยให้เราเข้าใจบทบาทเฉพาะในสื่อสารดิจิทัล

| แชทบอท                               | แอปแชทที่ขับเคลื่อนด้วย Generative AI        |
| ------------------------------------- | ---------------------------------------------- |
| มุ่งเน้นงานเฉพาะและใช้กฎ            | ตระหนักถึงบริบท                               |
| มักถูกผสานรวมในระบบขนาดใหญ่          | อาจโฮสต์แชทบอทหนึ่งตัวหรือหลายตัว            |
| จำกัดเฉพาะฟังก์ชันที่กำหนดไว้ล่วงหน้า | รวมโมเดล generative AI                        |
| การโต้ตอบแบบเฉพาะและมีโครงสร้าง      | สามารถสนทนาแบบเปิดกว้างได้                    |

### ใช้ฟังก์ชันที่สร้างไว้ล่วงหน้าด้วย SDK และ API

เมื่อสร้างแอปแชท ก้าวแรกที่ดีคือการประเมินว่าอะไรมีอยู่แล้ว การใช้ SDK และ API สร้างแอปแชทเป็นกลยุทธ์ที่ได้เปรียบด้วยเหตุผลหลายประการ โดยการผสาน SDK และ API ที่มีเอกสารครบถ้วน คุณกำลังวางตำแหน่งแอปของคุณสำหรับความสำเร็จระยะยาว โดยแก้ไขปัญหาด้านการขยายตัวและการดูแลรักษา

- **เร่งกระบวนการพัฒนาและลดภาระ**: การพึ่งพาฟังก์ชันที่สร้างไว้ล่วงหน้าซึ่งไม่ต้องเสียเวลาสร้างเอง ทำให้คุณสามารถโฟกัสกับส่วนอื่น ๆ ของแอปที่มีความสำคัญมากกว่า เช่น ตรรกะทางธุรกิจ
- **ประสิทธิภาพที่ดีกว่า**: เมื่อสร้างฟังก์ชันจากศูนย์ คุณจะต้องถามตัวเองในท้ายที่สุดว่า “มันปรับขนาดได้อย่างไร? แอปตัวนี้รับมือกับจำนวนผู้ใช้ที่เพิ่มขึ้นกระทันหันได้หรือไม่?” SDK และ API ที่ได้รับการดูแลอย่างดีมักจะมีโซลูชันในตัวสำหรับปัญหาเหล่านี้
- **การดูแลรักษาที่ง่ายกว่า**: การอัปเดตและพัฒนาทำได้ง่ายขึ้นเพราะ API และ SDK ส่วนใหญ่แค่ต้องอัปเดตไลบรารีเมื่อมีเวอร์ชันใหม่
- **เข้าถึงเทคโนโลยีล้ำสมัย**: การใช้โมเดลที่ได้รับการปรับแต่งและฝึกฝนกับชุดข้อมูลขนาดใหญ่ ให้ความสามารถทางภาษาธรรมชาติแก่แอปของคุณ

การเข้าถึงฟังก์ชันของ SDK หรือ API มักเกี่ยวข้องกับการได้รับอนุญาตใช้บริการ ผ่านการใช้คีย์เฉพาะหรือโทเคนยืนยันตัวตน เราจะใช้ OpenAI Python Library เพื่อสำรวจว่ารูปแบบนี้เป็นอย่างไร คุณยังสามารถลองใช้ด้วยตัวเองใน [notebook สำหรับ OpenAI](./python/oai-assignment.ipynb?WT.mc_id=academic-105485-koreyst) หรือ [notebook สำหรับ Azure OpenAI Services](./python/aoai-assignment.ipynb?WT.mc_id=academic-105485-koreys) สำหรับบทเรียนนี้

```python
import os
from openai import OpenAI

API_KEY = os.getenv("OPENAI_API_KEY","")

client = OpenAI(
    api_key=API_KEY
    )

chat_completion = client.chat.completions.create(model="gpt-3.5-turbo", messages=[{"role": "user", "content": "Suggest two titles for an instructional lesson on chat applications for generative AI."}])
```
  
ตัวอย่างข้างต้นใช้โมเดล GPT-3.5 Turbo เพื่อเติมเต็มคำสั่ง แต่สังเกตว่าคีย์ API ถูกตั้งค่าก่อนการทำงาน หากคุณไม่ตั้งค่าคีย์ คุณจะได้รับข้อผิดพลาด

## ประสบการณ์ผู้ใช้ (UX)

หลักการ UX ทั่วไปใช้กับแอปแชทได้ แต่มีข้อควรพิจารณาเพิ่มเติมที่สำคัญโดยเฉพาะอย่างยิ่งเนื่องจากส่วนประกอบการเรียนรู้ของเครื่องที่เกี่ยวข้อง

- **กลไกในการจัดการความคลุมเครือ**: โมเดล generative AI อาจสร้างคำตอบคลุมเครือได้ เป็นฟีเจอร์ที่ช่วยให้ผู้ใช้สามารถขอคำชี้แจงเมื่อต้องพบปัญหานี้
- **การเก็บบริบท**: โมเดล generative AI ขั้นสูงมีความสามารถในการจำบริบทในบทสนทนา ซึ่งเป็นทรัพย์สินจำเป็นต่อประสบการณ์ผู้ใช้ การให้ผู้ใช้ควบคุมและจัดการบริบทช่วยปรับปรุงประสบการณ์ผู้ใช้ แต่ก็เพิ่มความเสี่ยงในเรื่องการเก็บข้อมูลที่อ่อนไหว ต้องพิจารณาว่าจะเก็บข้อมูลนานแค่ไหน เช่น การกำหนดนโยบายทางการเก็บรักษา เพื่อให้สมดุลระหว่างความต้องการบริบทและความเป็นส่วนตัว
- **การปรับแต่งส่วนตัว**: ด้วยความสามารถในการเรียนรู้และปรับตัว โมเดล AI เสนอประสบการณ์ที่เป็นรายบุคคล การปรับแต่งประสบการณ์ผู้ใช้ผ่านฟีเจอร์ต่าง ๆ เช่น โปรไฟล์ผู้ใช้ ไม่เพียงแต่ทำให้ผู้ใช้รู้สึกว่าได้รับการเข้าใจ แต่ยังช่วยให้ผู้ใช้ค้นหาคำตอบเฉพาะเจาะจงได้มีประสิทธิภาพและน่าพึงพอใจยิ่งขึ้น

ตัวอย่างหนึ่งของการปรับแต่งคือการตั้งค่า "Custom instructions" ใน ChatGPT ของ OpenAI ซึ่งช่วยให้คุณใส่ข้อมูลเกี่ยวกับตัวคุณเองที่อาจเป็นบริบทสำคัญสำหรับคำสั่งของคุณ นี่คือตัวอย่างคำสั่งที่ปรับแต่ง

### การเข้าถึง

ไม่ว่าผู้ใช้จะมีความบกพร่องทางการมองเห็น การได้ยิน การเคลื่อนไหว หรือการรับรู้ แอปแชทที่ออกแบบมาอย่างดีควรใช้งานได้สำหรับทุกคน รายการต่อไปนี้แยกฟีเจอร์เฉพาะเพื่อเพิ่มความเข้าถึงสำหรับความบกพร่องต่าง ๆ

- **ฟีเจอร์สำหรับความบกพร่องทางการมองเห็น**: ธีมความคอนทราสต์สูงและข้อความปรับขนาดได้ รองรับการอ่านหน้าจอ
- **ฟีเจอร์สำหรับความบกพร่องทางการได้ยิน**: ฟังก์ชันข้อความเป็นเสียงและเสียงเป็นข้อความ สัญญาณภาพสำหรับแจ้งเตือนเสียง
- **ฟีเจอร์สำหรับความบกพร่องทางการเคลื่อนไหว**: รองรับการนำทางด้วยคีย์บอร์ด คำสั่งเสียง
- **ฟีเจอร์สำหรับความบกพร่องทางการรับรู้**: ตัวเลือกภาษาที่ง่ายขึ้น

## การปรับแต่งและการทำ Fine-tuning สำหรับโมเดลภาษาที่เฉพาะโดเมน

ลองนึกภาพแอปแชทที่เข้าใจศัพท์เฉพาะของบริษัทคุณและคาดเดาคำถามที่ผู้ใช้งานของตนมักถาม มีแนวทางที่ควรกล่าวถึงสองแนวทาง:

- **ใช้โมเดล DSL** DSL หมายถึง domain specific language คุณสามารถใช้โมเดล DSL ที่ฝึกฝนในโดเมนเฉพาะเพื่อเข้าใจแนวคิดและสถานการณ์นั้น ๆ
- **ทำ fine-tuning** fine-tuning คือกระบวนการฝึกโมเดลเพิ่มเติมด้วยข้อมูลเฉพาะ

## การปรับแต่ง: ใช้ DSL

การใช้โมเดลภาษาที่เฉพาะโดเมน (DSL Models) สามารถเพิ่มการมีส่วนร่วมของผู้ใช้และมอบการโต้ตอบที่มีความเชี่ยวชาญและเข้ากับบริบทแท้จริง โมเดลนี้ได้รับการฝึกหรือทำ fine-tune เพื่อเข้าใจและสร้างข้อความในสาขา อุตสาหกรรม หรือหัวข้อเฉพาะ ตัวเลือกในการใช้โมเดล DSL มีตั้งแต่การฝึกขึ้นมาใหม่จากศูนย์ ไปจนถึงการใช้โมเดลที่มีอยู่แล้วผ่าน SDK และ API อีกทางเลือกคือการทำ fine-tuning ซึ่งคือการนำโมเดลที่ฝึกแล้วมาปรับแต่งให้เหมาะกับโดเมนเฉพาะ

## การปรับแต่ง: ทำ Fine-tuning

การทำ fine-tuning มักพิจารณาเมื่อโมเดลที่ฝึกมาแล้วล้มเหลวในโดเมนเฉพาะหรือภารกิจเฉพาะ

เช่น คำถามทางการแพทย์ซับซ้อนและต้องการบริบทมาก เมื่อแพทย์วินิจฉัยผู้ป่วย จะพิจารณาจากหลายปัจจัย เช่น วิถีชีวิตหรือโรคประจำตัว และอาจต้องอิงบทความการแพทย์ล่าสุดเพื่อยืนยันผล ในสถานการณ์ที่ละเอียดอ่อนเช่นนี้ แอปแชท AI ทั่วไปไม่สามารถเป็นแหล่งข้อมูลที่น่าเชื่อถือได้

### กรณีศึกษา: แอปแชทด้านการแพทย์

พิจารณาแอปแชทที่ออกแบบมาเพื่อช่วยแพทย์ด้วยการอ้างอิงอย่างรวดเร็วถึงแนวทางการรักษา ปฏิกิริยาระหว่างยา หรือผลงานวิจัยล่าสุด

โมเดลทั่วไปอาจเพียงพอสำหรับตอบคำถามทางการแพทย์พื้นฐานหรือให้คำแนะนำทั่วไป แต่จะแสดงข้อจำกัดในเรื่องเหล่านี้:

- **กรณีที่เฉพาะเจาะจงหรือซับซ้อนมาก** เช่น แพทย์ระบบประสาทอาจถามแอปว่า “แนวทางปฏิบัติที่ดีที่สุดปัจจุบันสำหรับควบคุมโรคลมชักที่ดื้อยาในผู้ป่วยเด็กคืออะไร?”
- **ขาดการอัปเดตล่าสุด** โมเดลทั่วไปอาจไม่สามารถจัดหาคำตอบที่รวมความก้าวหน้าล่าสุดในสาขาประสาทวิทยาและเภสัชวิทยาได้

ในเหตุการณ์เช่นนี้ การทำ fine-tuning โมเดลด้วยชุดข้อมูลทางการแพทย์เฉพาะสามารถปรับปรุงความสามารถให้จัดการคำถามทางการแพทย์ที่ซับซ้อนเหล่านี้ได้แม่นยำและน่าเชื่อถือยิ่งขึ้น ซึ่งต้องมีการเข้าถึงชุดข้อมูลขนาดใหญ่และเกี่ยวข้องที่แทนความท้าทายในโดเมนและคำถามที่ต้องตอบ

## ข้อควรพิจารณาสำหรับประสบการณ์แชท AI คุณภาพสูง

ส่วนนี้สรุปเกณฑ์สำหรับแอปแชท "คุณภาพสูง" ซึ่งรวมถึงการเก็บตัวชี้วัดที่นำไปปฏิบัติได้และการปฏิบัติตามกรอบการทำงานที่ใช้ AI อย่างรับผิดชอบ

### ตัวชี้วัดสำคัญ

เพื่อรักษาประสิทธิภาพคุณภาพสูงของแอป จำเป็นต้องติดตามตัวชี้วัดและข้อพิจารณาที่สำคัญ การวัดเหล่านี้ไม่เพียงแต่รับรองการทำงานของแอปเท่านั้น แต่ยังประเมินคุณภาพของโมเดล AI และประสบการณ์ผู้ใช้อีกด้วย ด้านล่างคือรายการตัวชี้วัดพื้นฐาน AI และ UX ที่ควรพิจารณา

| ตัวชี้วัด                     | คำจำกัดความ                                                                                                               | ข้อพิจารณาสำหรับนักพัฒนาแชท                                      |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| **Uptime**                   | วัดระยะเวลาที่แอปสามารถทำงานและเข้าถึงได้โดยผู้ใช้                                                                    | คุณจะลดเวลาหยุดทำงานได้อย่างไร?                                     |
| **Response Time**            | เวลาที่แอปใช้ในการตอบสนองต่อคำถามของผู้ใช้                                                                             | คุณจะปรับปรุงการประมวลผลคำถามเพื่อเพิ่มความเร็วในการตอบสนองอย่างไร? |
| **Precision**                | สัดส่วนของการทำนายผลบวกจริงเทียบกับจำนวนการทำนายผลบวกทั้งหมด                                                           | คุณจะตรวจสอบความแม่นยำของโมเดลอย่างไร?                            |
| **Recall (Sensitivity)**     | สัดส่วนของการทำนายผลบวกจริงเทียบกับจำนวนผลบวกจริงทั้งหมด                                                               | คุณจะวัดและปรับปรุงค่า recall อย่างไร?                              |
| **F1 Score**                 | ค่าเฉลี่ยเชิงฮาร์มอนิกของ precision และ recall ซึ่งสร้างสมดุลระหว่างทั้งสอง                                              | เป้าหมาย F1 Score ของคุณคืออะไร? คุณจะสร้างสมดุล precision กับ recall อย่างไร? |
| **Perplexity**               | วัดว่าโมเดลทำนายการแจกแจงความน่าจะเป็นสอดคล้องกับข้อมูลจริงมากน้อยเพียงใด                                              | คุณจะลด perplexity อย่างไร?                                         |
| **User Satisfaction Metrics**| วัดการรับรู้ของผู้ใช้ต่อแอป โดยมักใช้แบบสำรวจ                                                                             | คุณจะเก็บข้อมูลฟีดแบ็กผู้ใช้บ่อยแค่ไหน? คุณจะปรับเปลี่ยนแอปตามฟีดแบ็กอย่างไร? |
| **Error Rate**               | อัตราความผิดพลาดในการเข้าใจหรือผลลัพธ์ของโมเดล                                                                          | คุณมีวิธีลดอัตราความผิดพลาดอย่างไร?                               |
| **Retraining Cycles**        | ความถี่ในการอัปเดตโมเดลเพื่อรวมข้อมูลและความรู้ใหม่                                                                       | คุณจะฝึกโมเดลใหม่บ่อยแค่ไหน? อะไรเป็นตัวกระตุ้นให้ทำการฝึกซ้ำ?     |
| **การตรวจจับความผิดปกติ**         | เครื่องมือและเทคนิคสำหรับระบุรูปแบบที่ผิดปกติซึ่งไม่ตรงตามพฤติกรรมที่คาดไว้                        | คุณจะตอบสนองต่อความผิดปกติอย่างไร?                                        |

### การนำแนวปฏิบัติ AI ที่รับผิดชอบมาใช้ในแอปพลิเคชันแชท

แนวทางของ Microsoft สำหรับ AI ที่รับผิดชอบได้ระบุหลักการหกประการที่ควรเป็นแนวทางในการพัฒนาและใช้งาน AI ด้านล่างนี้คือหลักการ คำนิยาม และสิ่งที่นักพัฒนาแชทควรพิจารณา รวมถึงเหตุผลที่ควรเอาจริงเอาจังกับสิ่งเหล่านี้

| หลักการ                | คำนิยามของ Microsoft                             | สิ่งที่นักพัฒนาแชทควรพิจารณา                                            | เหตุใดสิ่งนี้จึงสำคัญ                                                               |
| ---------------------- | ------------------------------------------------- | ------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| ความเป็นธรรม           | ระบบ AI ควรปฏิบัติต่อทุกคนอย่างเท่าเทียม            | ตรวจสอบให้แน่ใจว่าแอปพลิเคชันแชทไม่เลือกปฏิบัติตามข้อมูลผู้ใช้            | เพื่อสร้างความไว้วางใจและความครอบคลุมในหมู่ผู้ใช้ หลีกเลี่ยงปัญหาทางกฎหมาย          |
| ความน่าเชื่อถือและความปลอดภัย | ระบบ AI ควรทำงานได้อย่างน่าเชื่อถือและปลอดภัย        | ดำเนินการทดสอบและมาตรการป้องกันความผิดพลาดเพื่อลดข้อผิดพลาดและความเสี่ยง  | เพื่อสร้างความพึงพอใจแก่ผู้ใช้และป้องกันอันตรายที่อาจเกิดขึ้น                         |
| ความเป็นส่วนตัวและความปลอดภัย     | ระบบ AI ควรปลอดภัยและเคารพความเป็นส่วนตัว               | ใช้มาตรการเข้ารหัสที่แข็งแกร่งและปกป้องข้อมูล                          | เพื่อปกป้องข้อมูลผู้ใช้ที่ละเอียดอ่อนและปฏิบัติตามกฎหมายความเป็นส่วนตัว             |
| ความครอบคลุม           | ระบบ AI ควรให้พลังและมีส่วนร่วมกับทุกคน                  | ออกแบบ UI/UX ให้เข้าถึงได้ง่ายและใช้งานได้สะดวกสำหรับกลุ่มผู้ใช้หลากหลาย | เพื่อให้ผู้คนในวงกว้างสามารถใช้งานแอปพลิเคชันได้อย่างมีประสิทธิภาพ               |
| ความโปร่งใส           | ระบบ AI ควรเข้าใจได้                                      | ให้เอกสารที่ชัดเจนและเหตุผลสำหรับคำตอบของ AI                           | ผู้ใช้มีแนวโน้มไว้วางใจระบบมากขึ้นหากสามารถเข้าใจได้ว่ามีการตัดสินใจอย่างไร         |
| ความรับผิดชอบ          | คนควรมีความรับผิดชอบต่อระบบ AI                            | สร้างกระบวนการที่ชัดเจนสำหรับการตรวจสอบและปรับปรุงการตัดสินใจของ AI     | สนับสนุนการปรับปรุงอย่างต่อเนื่องและมาตรการแก้ไขในกรณีเกิดข้อผิดพลาด               |
