# การสร้างแอปแชทที่ขับเคลื่อนด้วย AI สร้างสรรค์

[![การสร้างแอปแชทที่ขับเคลื่อนด้วย AI สร้างสรรค์](../../../translated_images/th/07-lesson-banner.a279b937f2843833.webp)](https://youtu.be/R9V0ZY1BEQo?si=IHuU-fS9YWT8s4sA)

> _(คลิกที่ภาพด้านบนเพื่อดูวิดีโอของบทเรียนนี้)_

ตอนนี้ที่เราได้เห็นแล้วว่าเราสามารถสร้างแอปการสร้างข้อความได้อย่างไร มาเจาะลึกแอปแชทกันเถอะ

แอปแชทได้กลายเป็นส่วนหนึ่งของชีวิตประจำวันของเราแล้ว โดยไม่ได้เป็นเพียงแค่ช่องทางการสนทนาทั่วไปเท่านั้น แต่ยังเป็นส่วนสำคัญของการบริการลูกค้า การสนับสนุนทางเทคนิค และแม้กระทั่งระบบให้คำปรึกษาที่ซับซ้อน คุณน่าจะได้รับความช่วยเหลือจากแอปแชทเมื่อไม่นานมานี้ ในขณะที่เรานำเทคโนโลยีขั้นสูงเช่น AI สร้างสรรค์เข้ามาใช้งานในแพลตฟอร์มเหล่านี้ ความซับซ้อนและความท้าทายก็เพิ่มขึ้นด้วย

คำถามบางข้อที่เราต้องหาคำตอบคือ:

- **การสร้างแอป**: เราจะสร้างและผนวกแอปที่ขับเคลื่อนด้วย AI เหล่านี้อย่างมีประสิทธิภาพสำหรับกรณีการใช้งานเฉพาะอย่างไร?
- **การติดตามดูแล**: เมื่อเปิดใช้งานแล้ว เราจะติดตามและมั่นใจได้อย่างไรว่าการทำงานของแอปเป็นไปอย่างมีคุณภาพสูงสุด ทั้งในแง่ของฟังก์ชันและการปฏิบัติตาม [หกหลักการของ AI ที่มีความรับผิดชอบ](https://www.microsoft.com/ai/responsible-ai?WT.mc_id=academic-105485-koreyst)?

เมื่อเราเดินหน้าสู่ยุคที่ถูกกำหนดโดยการทำงานอัตโนมัติและการโต้ตอบระหว่างมนุษย์กับเครื่องจักรอย่างไร้รอยต่อ ความเข้าใจว่า AI สร้างสรรค์เปลี่ยนแปลงขอบเขต ความลึก และความสามารถในการปรับตัวของแอปแชทอย่างไรจึงเป็นสิ่งสำคัญ บทเรียนนี้จะสำรวจแง่มุมของสถาปัตยกรรมที่รองรับระบบซับซ้อนเหล่านี้ เจาะลึกวิธีการปรับแต่งเพื่อภารกิจเฉพาะ และประเมินตัวชี้วัดและข้อควรพิจารณาที่เกี่ยวข้องในการใช้งาน AI อย่างรับผิดชอบ

## บทนำ

บทเรียนนี้ครอบคลุมถึง:

- เทคนิคในการสร้างและผนวกแอปแชทอย่างมีประสิทธิภาพ
- วิธีการปรับแต่งและปรับจูนแอป
- กลยุทธ์และข้อพิจารณาในการติดตามดูแลแอปแชทอย่างมีประสิทธิภาพ

## เป้าหมายการเรียนรู้

เมื่อจบบทเรียนนี้ คุณจะสามารถ:

- อธิบายข้อพิจารณาในการสร้างและผนวกแอปแชทเข้ากับระบบที่มีอยู่
- ปรับแต่งแอปแชทให้เหมาะกับกรณีการใช้งานเฉพาะ
- ระบุตัวชี้วัดและข้อพิจารณาที่สำคัญสำหรับติดตามและรักษาคุณภาพของแอปแชทที่ขับเคลื่อนด้วย AI ได้อย่างมีประสิทธิภาพ
- ทำให้มั่นใจว่าแอปแชทใช้ AI อย่างรับผิดชอบ

## การผนวก AI สร้างสรรค์เข้ากับแอปแชท

การยกระดับแอปแชทด้วย AI สร้างสรรค์ไม่ได้หมายความแค่ทำให้มันฉลาดขึ้น แต่หมายถึงการเพิ่มประสิทธิภาพสถาปัตยกรรม ประสิทธิภาพ และส่วนติดต่อผู้ใช้เพื่อมอบประสบการณ์ที่มีคุณภาพ ซึ่งรวมถึงการสำรวจพื้นฐานสถาปัตยกรรม การผนวก API และข้อพิจารณาเรื่องส่วนติดต่อผู้ใช้ ส่วนนี้มุ่งเสนอเส้นทางครอบคลุมสำหรับนำทางในภูมิทัศน์ซับซ้อนนี้ ไม่ว่าคุณจะเชื่อมต่อกับระบบที่มีอยู่หรือสร้างเป็นแพลตฟอร์มแยกต่างหาก

เมื่อจบบทนี้ คุณจะมีความเชี่ยวชาญที่จำเป็นในการสร้างและผนวกแอปแชทได้อย่างมีประสิทธิภาพ

### แชทบอทหรือแอปแชท?

ก่อนที่เราจะเริ่มสร้างแอปแชท ให้เปรียบเทียบระหว่าง 'แชทบอท' กับ 'แอปแชทที่ขับเคลื่อนด้วย AI' ซึ่งทำหน้าที่และมีฟังก์ชันที่แตกต่างกัน แชทบอทมีวัตถุประสงค์เพื่อทำให้งานการสนทนาเฉพาะเป็นอัตโนมัติ เช่น ตอบคำถามที่พบบ่อยหรือติดตามพัสดุ โดยปกติจะถูกควบคุมด้วยตรรกะตามกฎหรืออัลกอริทึม AI ซับซ้อน ในทางตรงกันข้าม แอปแชทที่ขับเคลื่อนด้วย AI เป็นสภาพแวดล้อมที่กว้างกว่ามาก ออกแบบมาเพื่ออำนวยความสะดวกในการสื่อสารดิจิทัลหลายรูปแบบ เช่น แชทข้อความ เสียง และวิดีโอระหว่างผู้ใช้มนุษย์ ฟีเจอร์สำคัญคือการผนวกโมเดล AI สร้างสรรค์ที่จำลองบทสนทนาอย่างละเอียดอ่อนในแบบคนจริง สร้างคำตอบตามป้อนข้อมูลและบริบทที่หลากหลาย แอปแชทที่ใช้ AI สร้างสรรค์สามารถสนทนาแบบเปิดกว้าง ปรับตัวกับบริบทที่เปลี่ยนแปลง และแม้แต่สร้างบทสนทนาที่สร้างสรรค์หรือซับซ้อนได้

ตารางด้านล่างแสดงความแตกต่างและความคล้ายคลึงหลักเพื่อช่วยให้เราเข้าใจบทบาทเฉพาะตัวของพวกเขาในสื่อสารดิจิทัล

| แชทบอท                             | แอปแชทที่ขับเคลื่อนด้วย AI สร้างสรรค์ |
| ------------------------------------- | -------------------------------------- |
| เน้นงานเฉพาะและอิงกฎ                     | ตระหนักถึงบริบท                       |
| มักถูกรวมเข้ากับระบบใหญ่                | อาจรองรับแชทบอทหนึ่งตัวหรือหลายตัว   |
| จำกัดในฟังก์ชันที่โปรแกรมไว้           | ผนวกโมเดล AI สร้างสรรค์              |
| การโต้ตอบเฉพาะด้านและมีโครงสร้าง      | สามารถสนทนาแบบเปิดกว้าง               |

### การใช้ฟังก์ชันสำเร็จรูปด้วย SDK และ API

เมื่อสร้างแอปแชท ขั้นตอนแรกที่ดีคือประเมินสิ่งที่มีอยู่แล้ว การใช้ SDK และ API ในการสร้างแอปแชทเป็นกลยุทธ์ที่ได้เปรียบด้วยเหตุผลหลายประการ โดยการผนวก SDK และ API ที่มีเอกสารดี คุณจึงวางตำแหน่งแอปของคุณสำหรับความสำเร็จในระยะยาว พร้อมทั้งแก้ปัญหาเรื่องการขยายและการบำรุงรักษา

- **เร่งกระบวนการพัฒนาและลดภาระ**: การใช้ฟังก์ชันสำเร็จรูปแทนการสร้างเองซึ่งมีค่าใช้จ่ายสูงช่วยให้คุณโฟกัสกับส่วนอื่น ๆ ของแอปที่สำคัญกว่า เช่น ตรรกะทางธุรกิจ
- **ประสิทธิภาพที่ดีขึ้น**: เมื่อสร้างฟังก์ชันขึ้นเอง คุณจะต้องถามตัวเองว่า "แอปนี้สามารถรองรับผู้ใช้จำนวนมากได้อย่างไร?" SDK และ API ที่ได้รับการดูแลดีมักมีการแก้ไขปัญหานี้ในตัว
- **การบำรุงรักษาง่ายกว่า**: การอัปเดตและปรับปรุงง่ายขึ้นเพราะส่วนใหญ่ API และ SDK เพียงแค่ต้องอัปเดตไลบรารีเมื่อมีเวอร์ชันใหม่
- **เข้าถึงเทคโนโลยีล้ำสมัย**: การใช้โมเดลที่ผ่านการปรับแต่งและฝึกบนชุดข้อมูลขนาดใหญ่ช่วยให้อินพุตภาษาเป็นธรรมชาติ

การเข้าถึงฟังก์ชันของ SDK หรือ API มักต้องได้รับอนุญาตให้ใช้บริการนั้น ซึ่งมักผ่านการใช้คีย์เฉพาะหรือโทเค็นการตรวจสอบ เราจะใช้ไลบรารี Python ของ OpenAI เพื่อสำรวจสิ่งนี้ คุณยังสามารถลองใช้เองใน [โน้ตบุ๊กสำหรับ OpenAI](./python/oai-assignment.ipynb?WT.mc_id=academic-105485-koreyst) หรือ [โน้ตบุ๊กสำหรับ Azure OpenAI Services](./python/aoai-assignment.ipynb?WT.mc_id=academic-105485-koreys) สำหรับบทเรียนนี้

```python
import os
from openai import OpenAI

API_KEY = os.getenv("OPENAI_API_KEY","")

client = OpenAI(
    api_key=API_KEY
    )

response = client.responses.create(model="gpt-4o-mini", input="Suggest two titles for an instructional lesson on chat applications for generative AI.", store=False)
print(response.output_text)
```

ตัวอย่างด้านบนใช้โมเดล GPT-4o mini กับ API Responses เพื่อเติมข้อความคำสั่ง แต่สังเกตว่าได้ตั้งค่า API key ไว้ก่อน หากไม่ตั้งค่าคีย์ คุณจะได้รับข้อผิดพลาด

## ประสบการณ์ผู้ใช้ (UX)

หลักการ UX ทั่วไปใช้กับแอปแชทได้ แต่มีข้อควรพิจารณาเพิ่มเติมซึ่งสำคัญมากเพราะส่วนประกอบการเรียนรู้ของเครื่อง

- **กลไกแก้ความกำกวม**: โมเดล AI สร้างสรรค์บางครั้งให้คำตอบที่คลุมเครือ ฟีเจอร์ที่ให้ผู้ใช้ขอคำชี้แจงจะเป็นประโยชน์เมื่อเจอปัญหานี้
- **การเก็บบริบท**: โมเดล AI สร้างสรรค์ขั้นสูงสามารถจดจำบริบทภายในบทสนทนาได้ ซึ่งเป็นสิ่งจำเป็นสำหรับประสบการณ์ผู้ใช้ การให้ผู้ใช้ควบคุมและจัดการบริบทช่วยปรับปรุง UX แต่เสี่ยงต่อการเก็บข้อมูลส่วนตัว พิจารณาการเก็บข้อมูล เช่น การตั้งนโยบายการเก็บรักษาข้อมูล เพื่อสมดุลความต้องการบริบทกับความเป็นส่วนตัว
- **การปรับแต่งส่วนบุคคล**: ด้วยความสามารถในการเรียนรู้และปรับตัว โมเดล AI มอบประสบการณ์เฉพาะตัวให้ผู้ใช้ การปรับ UX ผ่านฟีเจอร์อย่างโปรไฟล์ผู้ใช้ไม่เพียงทำให้ผู้ใช้รู้สึกถูกเข้าใจ แต่ยังช่วยให้ค้นหาคำตอบเฉพาะได้รวดเร็วและพึงพอใจมากขึ้น

ตัวอย่างหนึ่งของการปรับแต่งส่วนบุคคลคือการตั้งค่า "คำแนะนำเอกเทศ" ใน ChatGPT ของ OpenAI ซึ่งช่วยให้คุณป้อนข้อมูลส่วนตัวที่อาจเป็นบริบทสำคัญต่อคำสั่งของคุณ ต่อไปนี้คือตัวอย่างของคำแนะนำเอกเทศ

### การเข้าถึง

ไม่ว่าผู้ใช้จะมีความบกพร่องทางการมองเห็น การได้ยิน การเคลื่อนไหว หรือสติปัญญา แอปแชทที่ออกแบบดีควรใช้งานได้สำหรับทุกคน รายการต่อไปนี้แยกฟีเจอร์เฉพาะที่มุ่งเน้นปรับปรุงการเข้าถึงสำหรับผู้บกพร่องแต่ละประเภท

- **ฟีเจอร์สำหรับผู้บกพร่องทางการมองเห็น**: ธีมความคมชัดสูง ข้อความปรับขนาดได้ ความเข้ากันได้กับเครื่องอ่านหน้าจอ
- **ฟีเจอร์สำหรับผู้บกพร่องทางการได้ยิน**: ฟังก์ชันแปลงข้อความเป็นเสียงและเสียงเป็นข้อความ สัญญาณภาพแจ้งเตือนเสียง
- **ฟีเจอร์สำหรับผู้บกพร่องทางการเคลื่อนไหว**: รองรับการนำทางผ่านคีย์บอร์ด คำสั่งเสียง
- **ฟีเจอร์สำหรับผู้บกพร่องทางสติปัญญา**: ตัวเลือกภาษาเรียบง่าย

## การปรับแต่งและการปรับจูนสำหรับโมเดลภาษาที่เฉพาะเจาะจงโดเมน

ลองนึกถึงแอปแชทที่เข้าใจศัพท์เฉพาะของบริษัทคุณและคาดเดาคำถามเฉพาะที่ผู้ใช้ของมันมักถาม มีสองแนวทางที่ควรกล่าวถึง:

- **ใช้โมเดล DSL**: DSL ย่อมาจาก domain specific language คุณสามารถใช้โมเดล DSL ที่ฝึกบนโดเมนเฉพาะเพื่อเข้าใจแนวคิดและสถานการณ์ที่เกี่ยวข้อง
- **ปรับจูนเพิ่มเติม**: การปรับจูนคือกระบวนการฝึกโมเดลของคุณเพิ่มเติมด้วยข้อมูลเฉพาะ

## การปรับแต่ง: ใช้ DSL

การใช้โมเดลภาษาเฉพาะโดเมน (DSL) ช่วยเพิ่มการมีส่วนร่วมของผู้ใช้โดยการให้การโต้ตอบที่เชี่ยวชาญและตรงตามบริบท โมเดลนี้ถูกฝึกหรือปรับจูนเพื่อเข้าใจและสร้างข้อความที่เกี่ยวข้องกับสาขา อุตสาหกรรม หรือหัวข้อเฉพาะ ตัวเลือกในการใช้โมเดล DSL อาจเป็นการฝึกจากศูนย์หรือใช้โมเดลที่มีอยู่ผ่าน SDK และ API อีกตัวเลือกคือการปรับจูนเพิ่มเติมโดยใช้โมเดลที่ผ่านการฝึกมาแล้วและปรับให้เหมาะกับโดเมนเฉพาะ

## การปรับแต่ง: ปรับจูนเพิ่มเติม

การปรับจูนมักถูกพิจารณาเมื่อโมเดลที่ฝึกมาแล้วนั้นไม่เพียงพอในโดเมนหรือภารกิจเฉพาะ

ยกตัวอย่างคำถามทางการแพทย์ที่ซับซ้อนและต้องใช้บริบทมาก เมื่อแพทย์วินิจฉัยผู้ป่วย จะพิจารณาหลายปัจจัย เช่น วิธีการใช้ชีวิตหรือโรคประจำตัว และอาจอ้างอิงวารสารทางการแพทย์ล่าสุดเพื่อยืนยันการวินิจฉัย ในสถานการณ์ละเอียดอ่อนเช่นนี้ แอปแชท AI ทั่วไปไม่สามารถเป็นแหล่งข้อมูลที่เชื่อถือได้

### กรณีศึกษา: แอปทางการแพทย์

ลองพิจารณาแอปแชทที่ออกแบบเพื่อช่วยแพทย์โดยให้ข้อมูลอ้างอิงอย่างรวดเร็วเกี่ยวกับแนวทางการรักษา ปฏิกิริยาระหว่างยา หรือผลวิจัยล่าสุด

โมเดลทั่วไปอาจเพียงพอสำหรับตอบคำถามพื้นฐานหรือให้คำแนะนำทั่วไป แต่จะพบปัญหาในเรื่องต่อไปนี้:

- **กรณีเฉพาะหรือซับซ้อนสูง**: เช่น นักประสาทวิทยาอาจถามแอปว่า "แนวทางปัจจุบันที่ดีที่สุดสำหรับจัดการโรคลมชักดื้อยาในเด็กคืออะไร?"
- **ขาดความก้าวหน้าล่าสุด**: โมเดลทั่วไปอาจไม่สามารถให้คำตอบปัจจุบันที่รวมความก้าวหน้าล่าสุดในสาขาประสาทวิทยาและเภสัชวิทยาได้

ในกรณีเช่นนี้ การปรับจูนโมเดลด้วยชุดข้อมูลทางการแพทย์เฉพาะจะช่วยเพิ่มความสามารถในการจัดการคำถามที่ซับซ้อนเหล่านี้อย่างแม่นยำและเชื่อถือได้ จำเป็นต้องเข้าถึงชุดข้อมูลขนาดใหญ่และเกี่ยวข้องที่ครอบคลุมความท้าทายและคำถามเฉพาะโดเมน

## ข้อพิจารณาสำหรับประสบการณ์แชทที่ขับเคลื่อนด้วย AI ที่มีคุณภาพสูง

ส่วนนี้สรุปเกณฑ์สำหรับแอปแชท "คุณภาพสูง" ซึ่งรวมถึงการเก็บตัวชี้วัดเชิงปฏิบัติและการปฏิบัติตามกรอบงานที่ใช้ AI อย่างรับผิดชอบ

### ตัวชี้วัดสำคัญ

เพื่อรักษาประสิทธิภาพของแอปที่มีคุณภาพสูง จำเป็นต้องติดตามตัวชี้วัดและข้อพิจารณาที่สำคัญ การวัดเหล่านี้ไม่เพียงรับประกันการทำงานของแอปแต่ยังประเมินคุณภาพของโมเดล AI และประสบการณ์ผู้ใช้ รายการด้านล่างครอบคลุมตัวชี้วัดพื้นฐาน AI และประสบการณ์ผู้ใช้ที่ควรพิจารณา

| ตัวชี้วัด                        | ความหมาย                                                                                                              | ข้อพิจารณาสำหรับนักพัฒนาแชท                                       |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------- |
| **เวลาทำงาน (Uptime)**        | วัดเวลาที่แอปเปิดใช้งานและผู้ใช้เข้าถึงได้                                                                         | คุณจะลดเวลาหยุดทำงานอย่างไร?                                           |
| **เวลาตอบสนอง**               | เวลาที่แอปตอบสนองต่อคำถามของผู้ใช้                                                                                  | คุณจะเพิ่มประสิทธิภาพการประมวลผลคำถามอย่างไรเพื่อปรับปรุงเวลาตอบสนอง?   |
| **ความแม่นยำ (Precision)**    | อัตราส่วนของการทำนายเป็นบวกที่ถูกต้องเทียบกับจำนวนทำนายเป็นบวกทั้งหมด                                             | คุณจะตรวจสอบความแม่นยำของโมเดลอย่างไร?                              |
| **การดึงข้อมูลกลับ (Recall)** | อัตราส่วนของการทำนายเป็นบวกที่ถูกต้องเทียบกับจำนวนบวกจริง                                                        | คุณจะวัดและปรับปรุงการดึงข้อมูลกลับอย่างไร?                          |
| **คะแนน F1**                  | ค่าเฉลี่ยฮาร์มอนิกของความแม่นยำและการดึงข้อมูลกลับ ซึ่งสร้างสมดุลระหว่างทั้งสอง                                  | ค่าคะแนน F1 เป้าหมายของคุณคืออะไร? คุณจะสมดุลความแม่นยำและการดึงข้อมูลกลับอย่างไร?  |
| **Perplexity**                | วัดว่าการแจกแจงความน่าจะเป็นที่โมเดลคาดการณ์สอดคล้องกับการแจกแจงจริงของข้อมูลมากน้อยเพียงใด                      | คุณจะลดค่า perplexity อย่างไร?                                         |
| **ตัวชี้วัดความพึงพอใจผู้ใช้** | วัดความรับรู้ของผู้ใช้ต่อแอป มักเก็บผ่านการสำรวจ                                                                         | คุณจะเก็บข้อมูลความคิดเห็นผู้ใช้บ่อยแค่ไหน? และจะปรับปรุงอย่างไรจากข้อมูลนั้น?  |
| **อัตราความผิดพลาด**         | อัตราที่โมเดลทำผิดพลาดในการทำความเข้าใจหรือส่งผลลัพธ์                                                                | คุณมีนโยบายลดอัตราความผิดพลาดอย่างไร?                               |
| **รอบการฝึกซ้ำ**              | ความถี่ที่โมเดลได้รับการอัปเดตเพื่อรวมข้อมูลและข้อมูลเชิงลึกใหม่                                                        | คุณจะฝึกซ้ำโมเดลบ่อยแค่ไหน? ปัจจัยไหนเป็นตัวกระตุ้นรอบการฝึกซ้ำ?  |

| **การตรวจจับความผิดปกติ**        | เครื่องมือและเทคนิคสำหรับการระบุรูปแบบที่ผิดปกติซึ่งไม่สอดคล้องกับพฤติกรรมที่คาดหวัง                       | คุณจะตอบสนองต่อความผิดปกติอย่างไร?                                      |

### การนำแนวปฏิบัติ AI ที่รับผิดชอบไปใช้ในแอปพลิเคชันแชท

แนวทางของ Microsoft สำหรับ AI ที่รับผิดชอบได้ระบุหลักการหกข้อที่ควรเป็นแนวทางในการพัฒนาและใช้งาน AI ด้านล่างคือหลักการ คำนิยาม และสิ่งที่นักพัฒนาแชทควรพิจารณาและเหตุผลที่ควรให้ความสำคัญอย่างจริงจัง

| หลักการ                | คำนิยามของ Microsoft                                | สิ่งที่นักพัฒนาแชทควรพิจารณา                                      | เหตุผลที่สำคัญ                                                                      |
| ---------------------- | ----------------------------------------------------- | ---------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| ความเป็นธรรม          | ระบบ AI ควรปฏิบัติต่อทุกคนอย่างเป็นธรรม           | ตรวจสอบให้แน่ใจว่าแอปแชทไม่เลือกปฏิบัติตามข้อมูลผู้ใช้              | เพื่อสร้างความไว้วางใจและความครอบคลุมในกลุ่มผู้ใช้; หลีกเลี่ยงผลทางกฎหมาย       |
| ความน่าเชื่อถือและความปลอดภัย | ระบบ AI ควรทำงานได้อย่างน่าเชื่อถือและปลอดภัย         | ดำเนินการทดสอบและระบบสำรองเพื่อป้องกันข้อผิดพลาดและความเสี่ยง     | เพื่อความพึงพอใจของผู้ใช้และป้องกันอันตรายที่อาจเกิดขึ้น                            |
| ความเป็นส่วนตัวและความปลอดภัย | ระบบ AI ควรมีความปลอดภัยและเคารพความเป็นส่วนตัว       | ใช้มาตรการเข้ารหัสที่เข้มงวดและการปกป้องข้อมูล                       | เพื่อปกป้องข้อมูลผู้ใช้ที่ละเอียดอ่อนและปฏิบัติตามกฎหมายความเป็นส่วนตัว          |
| ความครอบคลุม          | ระบบ AI ควรเสริมพลังทุกคนและดึงดูดผู้คน           | ออกแบบ UI/UX ที่เข้าถึงง่ายและใช้งานได้สะดวกสำหรับกลุ่มเป้าหมายที่หลากหลาย | เพื่อให้คนจำนวนมากขึ้นสามารถใช้งานแอปพลิเคชันได้อย่างมีประสิทธิภาพ              |
| ความโปร่งใส           | ระบบ AI ควรเข้าใจได้ง่าย                            | ให้เอกสารและเหตุผลที่ชัดเจนสำหรับการตอบสนองของ AI                    | ผู้ใช้มีแนวโน้มที่จะไว้วางใจระบบหากพวกเขาเข้าใจวิธีการตัดสินใจ                   |
| ความรับผิดชอบ         | ผู้คนควรมีความรับผิดชอบต่ระบบ AI                   | กำหนดกระบวนการที่ชัดเจนสำหรับการตรวจสอบและปรับปรุงการตัดสินใจของ AI | เพื่อให้สามารถปรับปรุงอย่างต่อเนื่องและดำเนินมาตรการแก้ไขเมื่อเกิดข้อผิดพลาด        |
